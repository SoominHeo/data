<p><b>Kruskal's algorithm</b> is a <a href="/wiki/Minimum_spanning_tree#Algorithms" title="Minimum spanning tree">minimum-spanning-tree algorithm</a> which finds an edge of the least possible weight that connects any two trees in the forest.<sup class="reference" id="cite_ref-:0_1-0"><a href="#cite_note-:0-1">[1]</a></sup> It is a <a href="/wiki/Greedy_algorithm" title="Greedy algorithm">greedy algorithm</a> in <a href="/wiki/Graph_theory" title="Graph theory">graph theory</a> as it finds a <a href="/wiki/Minimum_spanning_tree" title="Minimum spanning tree">minimum spanning tree</a> for a <a href="/wiki/Connectivity_(graph_theory)" title="Connectivity (graph theory)">connected</a> <a class="mw-redirect" href="/wiki/Glossary_of_graph_theory#Weighted_graphs_and_networks" title="Glossary of graph theory">weighted graph</a> adding increasing cost arcs at each step.<sup class="reference" id="cite_ref-:0_1-1"><a href="#cite_note-:0-1">[1]</a></sup> This means it finds a subset of the <a class="mw-redirect" href="/wiki/Edge_(graph_theory)" title="Edge (graph theory)">edges</a> that forms a tree that includes every <a href="/wiki/Vertex_(graph_theory)" title="Vertex (graph theory)">vertex</a>, where the total weight of all the edges in the tree is minimized. If the graph is not connected, then it finds a <i>minimum spanning forest</i> (a minimum spanning tree for each <a href="/wiki/Connected_component_(graph_theory)" title="Connected component (graph theory)">connected component</a>).</p>
<p>This algorithm first appeared in <i><a href="/wiki/Proceedings_of_the_American_Mathematical_Society" title="Proceedings of the American Mathematical Society">Proceedings of the American Mathematical Society</a></i>, pp. 48–50 in 1956, and was written by <a href="/wiki/Joseph_Kruskal" title="Joseph Kruskal">Joseph Kruskal</a>.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>Other algorithms for this problem include <a href="/wiki/Prim%27s_algorithm" title="Prim's algorithm">Prim's algorithm</a>, <a href="/wiki/Reverse-delete_algorithm" title="Reverse-delete algorithm">Reverse-delete algorithm</a>, and <a href="/wiki/Bor%C5%AFvka%27s_algorithm" title="Borůvka's algorithm">Borůvka's algorithm</a>.</p>

<h2><span class="mw-headline" id="Algorithm">Algorithm</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Kruskal%27s_algorithm&amp;action=edit&amp;section=1" title="Edit section: Algorithm">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>create a forest <i>F</i> (a set of trees), where each vertex in the graph is a separate <a href="/wiki/Tree_(graph_theory)" title="Tree (graph theory)">tree</a></li>
<li>create a set <i>S</i> containing all the edges in the graph</li>
<li>while <i>S</i> is <a class="mw-redirect" href="/wiki/Nonempty" title="Nonempty">nonempty</a> and <i>F</i> is not yet <a href="/wiki/Spanning_tree" title="Spanning tree">spanning</a>
<ul>
<li>remove an edge with minimum weight from <i>S</i></li>
<li>if the removed edge connects two different trees then add it to the forest <i>F</i>, combining two trees into a single tree</li>
</ul>
</li>
</ul>
<p>At the termination of the algorithm, the forest forms a minimum spanning forest of the graph. If the graph is connected, the forest has a single component and forms a minimum spanning tree.</p>
<h2><span class="mw-headline" id="Pseudocode">Pseudocode</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Kruskal%27s_algorithm&amp;action=edit&amp;section=2" title="Edit section: Pseudocode">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:KruskalDemo.gif"><img alt="" class="thumbimage" data-file-height="323" data-file-width="314" height="226" src="//upload.wikimedia.org/wikipedia/commons/thumb/b/bb/KruskalDemo.gif/220px-KruskalDemo.gif" srcset="//upload.wikimedia.org/wikipedia/commons/b/bb/KruskalDemo.gif 1.5x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:KruskalDemo.gif" title="Enlarge"></a></div>
A demo for Kruskal's algorithm based on Euclidean distance.</div>
</div>
</div>
<p>The following code is implemented with <a href="/wiki/Disjoint-set_data_structure" title="Disjoint-set data structure">disjoint-set data structure</a>:</p>
<pre>
KRUSKAL(G):
1 A = ∅
2 <b>foreach</b> v ∈ G.V:
3    MAKE-SET(v)
4 <b>foreach</b> (u, v) in G.E ordered by weight(u, v), increasing:
5    <b>if</b> FIND-SET(u) ≠ FIND-SET(v):
6       A = A ∪ {(u, v)}
7       UNION(u, v)
8 <b>return</b> 
</pre>
<p>complexity:</p>
<p>O(Elog(v))</p>
<h2><span class="mw-headline" id="Complexity">Complexity</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Kruskal%27s_algorithm&amp;action=edit&amp;section=3" title="Edit section: Complexity">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

These running times are equivalent because:</p>

<li>Each isolated vertex is a separate component of the minimum spanning forest. If we ignore isolated vertices we obtain <i>V</i> ≤ 2<i>E</i>, so log <i>V</i> is <i>O</i>(log <i>E</i>).</li>
</ul>
<p>We can achieve this bound as follows: first sort the edges by weight using a <a href="/wiki/Comparison_sort" title="Comparison sort">comparison sort</a> in <i>O</i>(<i>E</i> log <i>E</i>) time; this allows the step "remove an edge with minimum weight from <i>S</i>" to operate in constant time. Next, we use a <a href="/wiki/Disjoint-set_data_structure" title="Disjoint-set data structure">disjoint-set data structure</a> (Union&amp;Find) to keep track of which vertices are in which components. We need to perform O(<i>V</i>) operations, as in each iteration we connect a vertex to the spanning tree, two 'find' operations and possibly one union for each edge. Even a simple disjoint-set data structure such as disjoint-set forests with union by rank can perform O(<i>V</i>) operations in <i>O</i>(<i>V</i> log <i>V</i>) time. Thus the total time is <i>O</i>(<i>E</i> log <i>E</i>) = <i>O</i>(<i>E</i> log <i>V</i>).</p>
<p>Provided that the edges are either already sorted or can be sorted in linear time (for example with <a href="/wiki/Counting_sort" title="Counting sort">counting sort</a> or <a href="/wiki/Radix_sort" title="Radix sort">radix sort</a>), the algorithm can use a more sophisticated <a href="/wiki/Disjoint-set_data_structure" title="Disjoint-set data structure">disjoint-set data structure</a> to run in <i>O</i>(<i>E</i> α(<i>V</i>)) time, where α is the extremely slowly growing inverse of the single-valued <a href="/wiki/Ackermann_function" title="Ackermann function">Ackermann function</a>.</p>

<h2><span class="mw-headline" id="Proof_of_correctness">Proof of correctness</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Kruskal%27s_algorithm&amp;action=edit&amp;section=5" title="Edit section: Proof of correctness">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The proof consists of two parts. First, it is proved that the algorithm produces a <a href="/wiki/Spanning_tree" title="Spanning tree">spanning tree</a>. Second, it is proved that the constructed spanning tree is of minimal weight.</p>

<h3><span class="mw-headline" id="Minimality">Minimality</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Kruskal%27s_algorithm&amp;action=edit&amp;section=7" title="Edit section: Minimality">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>We show that the following proposition <i><b>P</b></i> is true by <a href="/wiki/Mathematical_induction" title="Mathematical induction">induction</a>: If <i>F</i> is the set of edges chosen at any stage of the algorithm, then there is some minimum spanning tree that contains <i>F</i>.</p>
<ul>
<li>Clearly <i><b>P</b></i> is true at the beginning, when <i>F</i> is empty: any minimum spanning tree will do, and there exists one because a weighted connected graph always has a minimum spanning tree.</li>
<li>Now assume <i><b>P</b></i> is true for some non-final edge set <i>F</i> and let <i>T</i> be a minimum spanning tree that contains <i>F</i>. If the next chosen edge <i>e</i> is also in <i>T</i>, then <i><b>P</b></i> is true for <i>F</i> + <i>e</i>. Otherwise, <i>T</i> + <i>e</i> has a cycle <i>C</i> and there is another edge <i>f</i> that is in <i>C</i> but not <i>F</i>. (If there were no such edge <i>f</i>, then <i>e</i> could not have been added to <i>F</i>, since doing so would have created the cycle <i>C</i>.) Then <i>T</i> − <i>f</i> + <i>e</i> is a tree, and it has the same weight as <i>T</i>, since <i>T</i> has minimum weight and the weight of <i>f</i> cannot be less than the weight of <i>e</i>, otherwise the algorithm would have chosen <i>f</i> instead of <i>e</i>. So <i>T</i> − <i>f</i> + <i>e</i> is a minimum spanning tree containing <i>F</i> + <i>e</i> and again <i><b>P</b></i> holds.</li>
<li>Therefore, by the principle of induction, <i><b>P</b></i> holds when <i>F</i> has become a spanning tree, which is only possible if <i>F</i> is a minimum spanning tree itself.</li>
</ul>
