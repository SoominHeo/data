<p>The <b>Open Document Format for Office Applications</b> (<b>ODF</b>), also known as <b>OpenDocument</b>, is an <a href="/wiki/XML" title="XML">XML</a>-based <a href="/wiki/File_format" title="File format">file format</a> for <a href="/wiki/Spreadsheet" title="Spreadsheet">spreadsheets</a>, <a href="/wiki/Chart" title="Chart">charts</a>, <a href="/wiki/Presentation_program" title="Presentation program">presentations</a> and <a href="/wiki/Word_processor" title="Word processor">word processing</a> documents. It was developed with the aim of providing an open, XML-based file format specification for office applications.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>The standard was developed by a technical committee in the <a href="/wiki/OASIS_(organization)" title="OASIS (organization)">Organization for the Advancement of Structured Information Standards</a> (OASIS) consortium.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> It was based on the <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a> specification for <a href="/wiki/OpenOffice.org_XML" title="OpenOffice.org XML">OpenOffice.org XML</a>, the default format for <a href="/wiki/OpenOffice.org" title="OpenOffice.org">OpenOffice.org</a>, which had been specifically intended "to provide an open standard for office documents."<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<p>In addition to being an OASIS standard, it was published as an <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a>/<a href="/wiki/International_Electrotechnical_Commission" title="International Electrotechnical Commission">IEC</a> international standard ISO/IEC 26300 –  Open Document Format for Office Applications (OpenDocument).<sup class="reference" id="cite_ref-odf12-part1_2-4"><a href="#cite_note-odf12-part1-2">[2]</a></sup><sup class="reference" id="cite_ref-odf12-part2_3-4"><a href="#cite_note-odf12-part2-3">[3]</a></sup><sup class="reference" id="cite_ref-odf12-part3_4-4"><a href="#cite_note-odf12-part3-4">[4]</a></sup><sup class="reference" id="cite_ref-odf10_5-4"><a href="#cite_note-odf10-5">[5]</a></sup><sup class="reference" id="cite_ref-iso.org_9-0"><a href="#cite_note-iso.org-9">[9]</a></sup><sup class="reference" id="cite_ref-http_10-0"><a href="#cite_note-http-10">[10]</a></sup></p>

<h2><span class="mw-headline" id="Specifications">Specifications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenDocument&amp;action=edit&amp;section=1" title="Edit section: Specifications">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/OpenDocument_technical_specification" title="OpenDocument technical specification">OpenDocument technical specification</a></div>
<p>The most common <a href="/wiki/Filename_extension" title="Filename extension">filename extensions</a> used for OpenDocument documents are:<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup><sup class="reference" id="cite_ref-hg_flat_opendocument_12-0"><a href="#cite_note-hg_flat_opendocument-12">[12]</a></sup></p>
<ul>
<li><code>.odt</code> and <code>.fodt</code> for <a class="mw-redirect" href="/wiki/Word_processing" title="Word processing">word processing</a> (text) documents</li>
<li><code>.ods</code> and <code>.fods</code> for <a href="/wiki/Spreadsheet" title="Spreadsheet">spreadsheets</a></li>
<li><code>.odp</code> and <code>.fodp</code> for <a href="/wiki/Presentation" title="Presentation">presentations</a></li>
<li><code>.odg</code> and <code>.fodg</code> for <a class="mw-redirect" href="/wiki/Graphic" title="Graphic">graphics</a></li>
<li><code>.odf</code> for <a href="/wiki/Formula" title="Formula">formulae</a>, mathematical equations</li>
</ul>
<p>The original OpenDocument format consists of an <a href="/wiki/XML" title="XML">XML</a> document that has <code>&lt;document&gt;</code> as its <a href="/wiki/Root_element" title="Root element">root element</a>. OpenDocument files can also take the format of a <a class="mw-redirect" href="/wiki/ZIP_(file_format)" title="ZIP (file format)">ZIP</a> compressed archive containing a number of files and directories; these can contain binary content and benefit from ZIP's <a href="/wiki/Lossless_compression" title="Lossless compression">lossless compression</a> to reduce file size. OpenDocument benefits from <a href="/wiki/Separation_of_concerns" title="Separation of concerns">separation of concerns</a> by separating the content, styles, metadata, and application settings into four separate XML files.</p>
<p>There is a comprehensive set of example documents in OpenDocument format available.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> The whole test suite is available under the <a class="mw-redirect" href="/wiki/Creative_Commons_licenses" title="Creative Commons licenses">Creative Commons Attribution 2.5 license</a>.</p>
<h2><span class="mw-headline" id="Standardization">Standardization</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenDocument&amp;action=edit&amp;section=2" title="Edit section: Standardization">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/OpenDocument_standardization" title="OpenDocument standardization">OpenDocument standardization</a></div>
<p>The OpenDocument standard was developed by a Technical Committee (TC) under the OASIS (<a class="mw-redirect" href="/wiki/Organization_for_the_Advancement_of_Structured_Information_Standards" title="Organization for the Advancement of Structured Information Standards">Organization for the Advancement of Structured Information Standards</a>) industry consortium. The ODF-TC has members from a diverse set of companies and individuals. Active TC members have voting rights. Members associated with Sun and IBM have sometimes had a large voting influence.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> The standardization process involved the developers of many office suites or related document systems. The first official ODF-TC meeting to discuss the standard was 16 December 2002; OASIS approved OpenDocument as an OASIS standard on 1 May 2005. OASIS submitted the ODF specification to <a class="mw-redirect" href="/wiki/ISO/IEC_JTC1" title="ISO/IEC JTC1">ISO/IEC Joint Technical Committee 1</a> (JTC 1) on 16 November 2005, under Publicly Available Specification (PAS) rules. ISO/IEC standardization for an open document standard including text, spreadsheet and presentation was proposed for the first time in <a class="mw-redirect" href="/wiki/DKUUG" title="DKUUG">DKUUG</a> 28 August 2001.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup></p>
<p>After a six-month review period, on 3 May 2006, OpenDocument unanimously passed its six-month DIS (Draft International Standard) ballot in <a class="mw-redirect" href="/wiki/JTC_1" title="JTC 1">JTC 1</a> (<a class="mw-redirect" href="/wiki/ISO/IEC_JTC1/SC34" title="ISO/IEC JTC1/SC34">ISO/IEC JTC1/SC34</a>), with broad participation,<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> after which the OpenDocument specification was "approved for release as an ISO and IEC International Standard" under the name ISO/IEC 26300:2006.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<p>After responding to all written ballot comments, and a 30-day default ballot, the OpenDocument <a href="/wiki/International_standard" title="International standard">international standard</a> went to publication in ISO, officially published 30 November 2006.</p>
<p>Further standardization work with OpenDocument includes:</p>
<ul>
<li>The <a class="mw-redirect" href="/wiki/Organization_for_the_Advancement_of_Structured_Information_Standards" title="Organization for the Advancement of Structured Information Standards">OASIS</a> Committee Specification <a class="external text" href="http://www.oasis-open.org/committees/download.php/19274/OpenDocument-v1.0ed2-cs1.pdf" rel="nofollow"><b>OpenDocument 1.0</b> (second edition)</a> corresponds to the published ISO/IEC 26300:2006 standard. The content of ISO/IEC 26300 and OASIS OpenDocument v1.0 2nd ed. is identical.<sup class="reference" id="cite_ref-odf10_5-5"><a href="#cite_note-odf10-5">[5]</a></sup> It includes the editorial changes made to address JTC1 ballot comments. It is available in ODF, HTML and PDF formats.</li>
<li><b><a class="external text" href="http://docs.oasis-open.org/office/v1.1/OS/OpenDocument-v1.1.pdf" rel="nofollow">OpenDocument 1.1</a></b> includes additional features to address accessibility concerns.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> It was approved as an OASIS Standard on 2007-02-01 following a call for vote issued on 2007-01-16.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup> The public announcement was made on 2007-02-13.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> This version was not initially submitted to ISO/IEC, because it is considered to be a minor update to ODF 1.0 only, and OASIS were working already on ODF 1.2 at the time ODF 1.1 was approved.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> However it was later submitted to ISO/IEC (as of March 2011, it was in "enquiry stage" as Draft Amendment 1 –  ISO/IEC 26300:2006/DAM 1) and published in March 2012 as "ISO/IEC 26300:2006/Amd 1:2012 – Open Document Format for Office Applications (OpenDocument) v1.1".<sup class="reference" id="cite_ref-iso.org_9-1"><a href="#cite_note-iso.org-9">[9]</a></sup><sup class="reference" id="cite_ref-http_10-1"><a href="#cite_note-http-10">[10]</a></sup></li>
<li><b><a class="external text" href="http://docs.oasis-open.org/office/v1.2/OpenDocument-v1.2.pdf" rel="nofollow">OpenDocument 1.2</a></b> includes additional accessibility features, <a href="/wiki/Resource_Description_Framework" title="Resource Description Framework">RDF</a>-based metadata,<sup class="reference" id="cite_ref-register_22-0"><a href="#cite_note-register-22">[22]</a></sup> a spreadsheet formula specification based on <a href="/wiki/OpenFormula" title="OpenFormula">OpenFormula</a>,<sup class="reference" id="cite_ref-register_22-1"><a href="#cite_note-register-22">[22]</a></sup> support for <a href="/wiki/Digital_signature" title="Digital signature">digital signatures</a> and some features suggested by the public. It consists of three parts: Part 1: OpenDocument Schema, Part 2: Recalculated Formula (OpenFormula) Format and Part 3: Packages. Version 1.2 of the specification was approved as an OASIS Standard on 29 September 2011.<sup class="reference" id="cite_ref-odf12_23-0"><a href="#cite_note-odf12-23">[23]</a></sup> It was submitted to the relevant ISO committee under the Publicly Available Specification (PAS) procedure in March 2014.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup> As of October 2014, it has been unanimously approved as a Draft International Standard, some comments have been raised in process that need to be addressed before OpenDocument 1.2 can proceed to become an International Standard.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> OpenDocument 1.2 was published as ISO/IEC standard on 17 June 2015.<sup class="reference" id="cite_ref-odf12-part1_2-5"><a href="#cite_note-odf12-part1-2">[2]</a></sup><sup class="reference" id="cite_ref-odf12-part2_3-5"><a href="#cite_note-odf12-part2-3">[3]</a></sup><sup class="reference" id="cite_ref-odf12-part3_4-5"><a href="#cite_note-odf12-part3-4">[4]</a></sup></li>
</ul>
<h3><span class="mw-headline" id="Future">Future</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenDocument&amp;action=edit&amp;section=3" title="Edit section: Future">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li><i><b>OpenDocument 1.3</b></i> (a.k.a. "ODF-Next") As of January, 2014, the current state of a possible future version of OpenDocument specification is a working draft (a preliminary unapproved sketch, outline, or version of the specification). The OASIS Advanced Document Collaboration subcommittee (created in December, 2010) is working on an update of OpenDocument change-tracking that will not only enhance the existing change-tracking feature set, but also lay the foundation for the standardization of <a class="mw-disambig" href="/wiki/Real-time_collaboration" title="Real-time collaboration">real-time collaboration</a> by making change tracking compatible with real-time collaboration.<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup><sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup><sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup></li>
</ul>

<li><a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a> <a href="/wiki/StarOffice" title="StarOffice">StarOffice</a></li>
<li><a href="/wiki/TextEdit" title="TextEdit">TextEdit</a></li>
<li><a href="/wiki/WordPad" title="WordPad">WordPad</a> 6.1 (Windows 7) partial support.</li>
<li><a class="mw-redirect" href="/wiki/Zoho_Office_Suite" title="Zoho Office Suite">Zoho Office Suite</a><sup class="reference" id="cite_ref-register_22-4"><a href="#cite_note-register-22">[22]</a></sup></li>
</ul>
<p>Various organizations have announced development of conversion software (including <i>plugins</i> and <i>filters</i>) to support OpenDocument on <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>'s products.<sup class="reference" id="cite_ref-43"><a href="#cite_note-43">[43]</a></sup><sup class="reference" id="cite_ref-44"><a href="#cite_note-44">[44]</a></sup> As of July 2007<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=OpenDocument&amp;action=edit">[update]</a></sup>, there are nine packages of conversion software. Microsoft first released support for the OpenDocument Format in Office 2007 SP2.<sup class="reference" id="cite_ref-45"><a href="#cite_note-45">[45]</a></sup> However, the implementation faced <a href="/wiki/OpenDocument_software#Microsoft_Office_2007_SP2_support_controversy" title="OpenDocument software">substantial criticism</a> and the <a class="mw-redirect" href="/wiki/ODF_Alliance" title="ODF Alliance">ODF Alliance</a> and others claimed that the third party plugins provided better support.<sup class="reference" id="cite_ref-sp2-fact-sheet_46-0"><a href="#cite_note-sp2-fact-sheet-46">[46]</a></sup> Microsoft Office 2010 can open and save OpenDocument Format documents natively, although not all features are supported.<sup class="reference" id="cite_ref-47"><a href="#cite_note-47">[47]</a></sup></p>
<p>Starting with <a class="mw-redirect" href="/wiki/Mac_OS_X_10.5" title="Mac OS X 10.5">Mac OS X 10.5</a>, the <a href="/wiki/TextEdit" title="TextEdit">TextEdit</a> application and <a href="/wiki/Quick_Look" title="Quick Look">Quick Look</a> preview feature support the OpenDocument Text format.</p>

<h2><span class="mw-headline" id="Licensing">Licensing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenDocument&amp;action=edit&amp;section=7" title="Edit section: Licensing">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Public_access_to_the_standard">Public access to the standard</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenDocument&amp;action=edit&amp;section=8" title="Edit section: Public access to the standard">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Versions of the OpenDocument Format approved by <a href="/wiki/OASIS_(organization)" title="OASIS (organization)">OASIS</a> are available for free download and use.<sup class="reference" id="cite_ref-48"><a href="#cite_note-48">[48]</a></sup> The <a href="/wiki/Information_Technology_Task_Force" title="Information Technology Task Force">ITTF</a> has added ISO/IEC 26300 to its "list of freely available standards"; anyone may download and use this standard free-of-charge under the terms of a click-through license.<sup class="reference" id="cite_ref-49"><a href="#cite_note-49">[49]</a></sup></p>
<h3><span class="mw-headline" id="Additional_royalty-free_licensing">Additional royalty-free licensing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenDocument&amp;action=edit&amp;section=9" title="Edit section: Additional royalty-free licensing">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Obligated <a class="external text" href="http://www.oasis-open.org/committees/daycount/tc/office.html" rel="nofollow">members</a> of the OASIS ODF TC have agreed to make deliverables available to implementors under the OASIS Royalty Free with Limited Terms <a class="external text" href="http://www.oasis-open.org/who/intellectualproperty.php" rel="nofollow">policy</a>.</p>
<p>Key contributor <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a> made an irrevocable intellectual property covenant, providing all implementers with the guarantee that Sun will not seek to enforce any of its enforceable U.S. or foreign patents against any implementation of the OpenDocument specification in which development Sun participates to the point of incurring an obligation.<sup class="reference" id="cite_ref-SunOpenDocumentPatentStatement_50-0"><a href="#cite_note-SunOpenDocumentPatentStatement-50">[50]</a></sup></p>
<p>A second contributor to ODF development, <a href="/wiki/IBM" title="IBM">IBM</a> – which, for instance, has contributed Lotus spreadsheet documentation<sup class="reference" id="cite_ref-51"><a href="#cite_note-51">[51]</a></sup> – has made their patent rights available through their <i>Interoperability Specifications Pledge</i> in which "IBM irrevocably covenants to you that it will not assert any Necessary Claims against you for your making, using, importing, selling, or offering for sale Covered Implementations."<sup class="reference" id="cite_ref-IBM-ISP-list_52-0"><a href="#cite_note-IBM-ISP-list-52">[52]</a></sup></p>
<p>The <a href="/wiki/Software_Freedom_Law_Center" title="Software Freedom Law Center">Software Freedom Law Center</a> has <a class="external text" href="http://www.softwarefreedom.org/resources/2006/OpenDocument.html" rel="nofollow">examined</a> whether there are any legal barriers to the use of the OpenDocument Format (ODF) in free and open source software arising from the standardization process. In their opinion ODF is free of legal encumbrances that would prevent its use in free and open source software, as distributed under licenses authored by Apache and the FSF.</p>
<h2><span class="mw-headline" id="Response">Response</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenDocument&amp;action=edit&amp;section=10" title="Edit section: Response">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Support_for_OpenDocument">Support for OpenDocument</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenDocument&amp;action=edit&amp;section=11" title="Edit section: Support for OpenDocument">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Several governments, companies, organizations and software products support the OpenDocument format. For example:</p>
<ul>
<li>The OpenDoc Society run frequent <a class="mw-redirect" href="/wiki/Plugfest" title="Plugfest">Plugfests</a> in association with industry groups and Public Sector organisations. The 10th Plugfest<sup class="reference" id="cite_ref-53"><a href="#cite_note-53">[53]</a></sup> was hosted by the UK <a href="/wiki/Government_Digital_Service" title="Government Digital Service">Government Digital Service</a> in conjunction with industry associations including the <a href="/wiki/OpenForum_Europe" title="OpenForum Europe">OpenForum Europe</a> and the <a class="new" href="/w/index.php?title=Open_Source_Consortium&amp;action=edit&amp;redlink=1" title="Open Source Consortium (page does not exist)">Open Source Consortium</a>.
<ul>
<li>An output of the 10th Plugfest was an ODF toolkit<sup class="reference" id="cite_ref-54"><a href="#cite_note-54">[54]</a></sup> which includes "Open Document Format principles for Government Technology" that has the purpose of simply explaining the case for ODF directed at the "average civil servant" and includes an extract from the UK Government policy relating to Open Document Format.</li>
<li>The toolkit also includes a single page graphical image<sup class="reference" id="cite_ref-55"><a href="#cite_note-55">[55]</a></sup> designed to articulate the consequences of not choosing Open Document Format. The illustration has now been translated in to more than 10 languages.</li>
</ul>
</li>
<li>Information technology companies like <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple Inc.</a>, <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe Systems</a>, <a href="/wiki/Google" title="Google">Google</a>, <a href="/wiki/IBM" title="IBM">IBM</a>, <a href="/wiki/Intel" title="Intel">Intel</a>, <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>, <a href="/wiki/Nokia" title="Nokia">Nokia</a>, <a href="/wiki/Novell" title="Novell">Novell</a>, <a href="/wiki/Red_Hat" title="Red Hat">Red Hat</a>, <a href="/wiki/Oracle_Corporation" title="Oracle Corporation">Oracle</a> as well as other companies who may or may not be working inside the OASIS OpenDocument Adoption Technical Committee.</li>
<li>Over 600 companies and organizations promote OpenDocument format through The <a href="/wiki/OpenDocument_Format_Alliance" title="OpenDocument Format Alliance">OpenDocument Format Alliance</a>.<sup class="reference" id="cite_ref-56"><a href="#cite_note-56">[56]</a></sup></li>
<li><a href="/wiki/NATO" title="NATO">NATO</a> with its 26 members uses ODF as a mandatory standard for all members.<sup class="reference" id="cite_ref-57"><a href="#cite_note-57">[57]</a></sup></li>
<li>The TAC (Telematics between Administrations Committee), composed of e-government policy-makers from the 25 <a href="/wiki/European_Union" title="European Union">European Union</a> Member States, endorsed a set of recommendations for promoting the use of open document formats in the public sector.<sup class="reference" id="cite_ref-ec.europa.eu_58-0"><a href="#cite_note-ec.europa.eu-58">[58]</a></sup></li>
<li>The free office suites <a href="/wiki/Apache_OpenOffice" title="Apache OpenOffice">Apache OpenOffice</a>, <a class="mw-redirect" href="/wiki/Calligra" title="Calligra">Calligra</a>, <a href="/wiki/KOffice" title="KOffice">KOffice</a>, <a href="/wiki/NeoOffice" title="NeoOffice">NeoOffice</a> and <a href="/wiki/LibreOffice" title="LibreOffice">LibreOffice</a> all use OpenDocument as their default file format.</li>
<li>Several organisations, such as the <a class="external text" href="http://opendocumentfellowship.com/" rel="nofollow">OpenDocument Fellowship</a> and <a class="external text" href="http://opendocsociety.org/" rel="nofollow">OpenDoc Society</a> were founded to support and promote OpenDocument.</li>
<li>The UK government has adopted ODF as the standard for all documents in the UK civil service<sup class="reference" id="cite_ref-59"><a href="#cite_note-59">[59]</a></sup></li>
<li>The <a href="/wiki/Wikimedia_Foundation" title="Wikimedia Foundation">Wikimedia Foundation</a> supports ODF export from <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a>, which powers <a href="/wiki/Wikipedia" title="Wikipedia">Wikipedia</a> and a number of other Internet <a href="/wiki/Wiki" title="Wiki">wiki</a>-based sites.<sup class="reference" id="cite_ref-60"><a href="#cite_note-60">[60]</a></sup></li>
<li>The default text processing applications in Windows 10 (<a href="/wiki/WordPad" title="WordPad">WordPad</a>) and Mac OS 10.9 (<a href="/wiki/TextEdit" title="TextEdit">TextEdit</a>) support OpenDocument Text.</li>
</ul>
<p>On 4 November 2005, IBM and Sun Microsystems convened the "OpenDocument (ODF) Summit" in <a href="/wiki/Armonk,_New_York" title="Armonk, New York">Armonk, New York</a>, to discuss how to boost OpenDocument adoption. The ODF Summit brought together representatives from several industry groups and technology companies, including Oracle, Google, Adobe, Novell, Red Hat, Computer Associates, Corel, Nokia, Intel, and Linux e-mail company Scalix (LaMonica, 10 November 2005). The providers committed resources to technically improve OpenDocument through existing standards bodies and to promote its usage in the marketplace, possibly through a stand-alone foundation.<sup class="reference" id="cite_ref-61"><a href="#cite_note-61">[61]</a></sup> Scholars have suggested that the "OpenDocument standard is the wedge that can hold open the door for competition, particularly with regard to the specific concerns of the public sector."<sup class="reference" id="cite_ref-papers.ssrn.com_62-0"><a href="#cite_note-papers.ssrn.com-62">[62]</a></sup> Indeed, adoption by the public sector has risen considerably since the promulgation of the OpenDocument format initiated the 2005/2006 time period.<sup class="reference" id="cite_ref-papers.ssrn.com_62-1"><a href="#cite_note-papers.ssrn.com-62">[62]</a></sup></p>
<ul>
<li>Different applications using ODF as a standard document format have different methods of providing macro/scripting capabilities. There is no <a class="mw-redirect" href="/wiki/Macro_language" title="Macro language">macro language</a> specified in ODF. Users and developers differ on whether inclusion of a standard scripting language would be desirable.<sup class="reference" id="cite_ref-63"><a href="#cite_note-63">[63]</a></sup></li>
<li>The ODF specification for tracked changes is limited and does not fully specify all cases, resulting in implementation-specific behaviors.<sup class="reference" id="cite_ref-64"><a href="#cite_note-64">[64]</a></sup> In addition, OpenDocument does not support change tracking in elements like tables or <a href="/wiki/MathML" title="MathML">MathML</a>.<sup class="reference" id="cite_ref-doyourmath_65-0"><a href="#cite_note-doyourmath-65">[65]</a></sup></li>
<li>It is not permitted to use generic ODF formatting style elements (like font information) for the MathML elements.<sup class="reference" id="cite_ref-doyourmath_65-1"><a href="#cite_note-doyourmath-65">[65]</a></sup></li>
</ul>
<h2><span class="mw-headline" id="Adoption">Adoption</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenDocument&amp;action=edit&amp;section=12" title="Edit section: Adoption">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/OpenDocument_adoption" title="OpenDocument adoption">OpenDocument adoption</a></div>
<p>One objective of open formats like OpenDocument is to guarantee long-term access to data without legal or technical barriers, and some governments have come to view open formats as a public policy issue. Several governments around the world have introduced policies of partial or complete adoption.<sup class="reference" id="cite_ref-papers.ssrn.com_62-2"><a href="#cite_note-papers.ssrn.com-62">[62]</a></sup> What this means varies from case to case; in some cases, it means that the ODF standard has a national standard identifier; in some cases, it means that the ODF standard is permitted to be used where national regulation says that non-proprietary formats must be used, and in still other cases, it means that some government body has actually decided that ODF will be used in some specific context.
