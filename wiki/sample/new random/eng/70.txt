<p><b>Windows 3.1x</b> (codenamed Janus)<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> is a series of <a href="/wiki/16-bit#16-bit_application" title="16-bit">16-bit</a> <a href="/wiki/Operating_environment" title="Operating environment">operating environments</a> produced by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> for use on personal computers. The series began with Windows 3.1, which was first sold during April 1992 as a successor to <a href="/wiki/Windows_3.0" title="Windows 3.0">Windows 3.0</a>. Subsequent versions were released between 1992 and 1994 until the series was superseded by <a href="/wiki/Windows_95" title="Windows 95">Windows 95</a>. During its lifespan, Windows 3.1 introduced several enhancements to the still <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>-based platform, including improved system stability, expanded support for multimedia, <a href="/wiki/TrueType" title="TrueType">TrueType</a> <a href="/wiki/Typeface" title="Typeface">fonts</a>, and <a href="/wiki/Workgroup_(computer_networking)" title="Workgroup (computer networking)">workgroup networking</a>.</p>
<p>Windows 3.1 was originally released on April 6, 1992; official support for Windows 3.1 ended on December 31, 2001, and OEM licensing for Windows for Workgroups 3.11 on <a href="/wiki/Embedded_system" title="Embedded system">embedded systems</a> continued to be available until November 1, 2008.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>

<h2><span class="mw-headline" id="Editions">Editions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=1" title="Edit section: Editions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Windows_3.1">Windows 3.1</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=2" title="Edit section: Windows 3.1">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><i>Windows 3.1</i> (originally codenamed <i><a href="/wiki/Janus" title="Janus">Janus</a></i>), released on April 6, 1992, introduced a <a href="/wiki/TrueType" title="TrueType">TrueType</a> <a href="/wiki/Typeface" title="Typeface">font</a> system (and a set of highly legible fonts), which effectively made Windows a viable <a href="/wiki/Desktop_publishing" title="Desktop publishing">desktop publishing</a> platform for the first time. Similar functionality was available for <a href="/wiki/Windows_3.0" title="Windows 3.0">Windows 3.0</a> through <a href="/wiki/Adobe_Type_Manager" title="Adobe Type Manager">Adobe Type Manager</a> (ATM) font system from <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe</a>.</p>
<p>Windows 3.1 was designed to have <a href="/wiki/Backward_compatibility" title="Backward compatibility">backward compatibility</a> with older Windows platforms. As with <a href="/wiki/Windows_3.0" title="Windows 3.0">Windows 3.0</a>, version 3.1 had <a href="/wiki/File_Manager_(Windows)" title="File Manager (Windows)">File Manager</a> and <a href="/wiki/Program_Manager" title="Program Manager">Program Manager</a>, but unlike all previous versions, Windows 3.1 cannot run in <a href="/wiki/Real_mode" title="Real mode">real mode</a>. It included <a href="/wiki/Minesweeper_(video_game)" title="Minesweeper (video game)">Minesweeper</a> as a replacement for <a href="/wiki/Reversi" title="Reversi">Reversi</a> (though Reversi was still included in some copies).</p>
<p>Windows 3.1 Multimedia PC Version (Beta only, released Nov 1992 – codenamed Bombay) included a media viewer, and the ability to play video files. It was targeted to the new <a href="/wiki/Multimedia_PC" title="Multimedia PC">multimedia PC</a> and included sound and video integration with CD-ROM support.</p>
<h4><span class="mw-headline" id="Improvements_over_Windows_3.0">Improvements over Windows 3.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=3" title="Edit section: Improvements over Windows 3.0">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Windows_3.1.png"><img alt="" class="thumbimage" data-file-height="281" data-file-width="374" height="165" src="//upload.wikimedia.org/wikipedia/en/thumb/6/64/Windows_3.1.png/220px-Windows_3.1.png" srcset="//upload.wikimedia.org/wikipedia/en/thumb/6/64/Windows_3.1.png/330px-Windows_3.1.png 1.5x, //upload.wikimedia.org/wikipedia/en/6/64/Windows_3.1.png 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Windows_3.1.png" title="Enlarge"></a></div>
Windows 3.1 showing some of the personalization options available.</div>
</div>
</div>
<p>Windows 3.1 dropped <a href="/wiki/Real_mode" title="Real mode">real mode</a> support and required a minimum of a <a href="/wiki/Intel_80286" title="Intel 80286">286</a> PC with 1 MB of <a href="/wiki/Random-access_memory" title="Random-access memory">RAM</a> to run. The effect of this was to increase system stability over the crash-prone Windows 3.0. Some older features were removed, like CGA graphics support (although Windows 3.0's CGA driver still worked on 3.1) and compatibility with real mode Windows 2.x applications.</p>
<p>Truetype font support was added, providing scalable fonts to Windows applications, without having to resort to using a third-party font technology such as <a href="/wiki/Adobe_Type_Manager" title="Adobe Type Manager">Adobe Type Manager</a>. Windows 3.1 included the following fonts: Arial, Courier New, Times New Roman, and Symbol (a collection of scalable symbols) in regular, bold, italic, and bold-italic versions. Truetype fonts could be scaled to any size and rotated, depending on the calling application.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>In 386 Enhanced Mode, windowed DOS applications gained the ability for users to manipulate menus and other objects in the program using the Windows mouse pointer, provided that a DOS application supported mice. A few DOS applications, such as late releases of Microsoft Word, could access Windows Clipboard. Windows' own drivers couldn't work directly with DOS applications; hardware such as mice required a DOS driver to be loaded before starting Windows.</p>
<p>Icons could be dragged and dropped for the first time, in addition to having a more detailed appearance. A file could be dragged onto Print Manager icon and the file would be printed by the current printer, assuming it was associated with an application capable of printing, such as a word processor. Alternatively, the file could be dragged out of File Manager and dropped onto an application icon or window for processing.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></p>
<p>While Windows 3.0 was limited to 16 MB maximum memory, Windows 3.1 can access a theoretical 4 GB in 386 Enhanced Mode. (The actual practical ceiling is 256 MB.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup>) However, no single process can use more than 16 MB. File Manager was significantly improved over Windows 3.0. <a href="/wiki/Multimedia" title="Multimedia">Multimedia</a> support was enhanced over what was available in <a href="/wiki/Windows_3.0" title="Windows 3.0">Windows 3.0 with Multimedia Extensions</a> and available to all Windows 3.1 users.</p>
<p>Windows 3.1 was available via 720 KB, 1.2 MB, and 1.44 MB floppy distributions. It was also the first version of Windows to be distributed on CD-ROM — although this was more common for Windows for Workgroups 3.11, which typically came with MS-DOS 6.22 on one CD. Installed size on the hard disk was between 10 MB and 15 MB.</p>
<p><a href="/wiki/32-bit_disk_access" title="32-bit disk access">32-bit disk access</a> (386 Enhanced Mode only) brought improved performance by using a 32-bit protected mode driver instead of the 16-bit BIOS functions (which necessitate Windows temporarily dropping out of protected mode).</p>
<p>Windows 3.1's calendar saves its files ending with <i>.cal</i>.<sup class="reference" id="cite_ref-CALENDAR_9-0"><a href="#cite_note-CALENDAR-9">[9]</a></sup></p>
<p>Windows 3.1 also introduced <a href="/wiki/Windows_Registry" title="Windows Registry">Windows Registry</a>, a centralized <a href="/wiki/Database" title="Database">database</a> that can store configuration information and settings for various operating systems components and applications.</p>
<p>Windows 3.1 was the first version of Windows that could also launch Windows programs via <a class="mw-redirect" href="/wiki/Command.com" title="Command.com">Command.com</a> while running Windows.</p>
<h3><span class="mw-headline" id="Windows_3.1_for_Central_and_Eastern_Europe">Windows 3.1 for Central and Eastern Europe</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=4" title="Edit section: Windows 3.1 for Central and Eastern Europe">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A special version named <i>Windows 3.1 for Central and Eastern Europe</i> was released that allowed use of Cyrillic and had fonts with diacritical marks characteristic of Central and Eastern European languages. Microsoft introduced its own <a href="/wiki/Code_page" title="Code page">code page</a> (<a href="/wiki/Windows-1250" title="Windows-1250">Windows-1250</a>) and supported its use in violation of many countries' ISO standards (<i>e.g.</i>, the official <a href="/wiki/Polish_language" title="Polish language">Polish</a> codepage is <a class="mw-redirect" href="/wiki/ISO-8859-2" title="ISO-8859-2">ISO-8859-2</a>, which was ignored by Microsoft but is supported by contemporary <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> versions). Similarly, Microsoft also released <i>Windows 3.1J</i> with support for Japanese, which shipped 1.46 million copies in its first year on the market (1993) in Japan.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></p>
<h3><span class="mw-headline" id="Modular_Windows">Modular Windows</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=5" title="Edit section: Modular Windows">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><i><a href="/wiki/Tandy_Video_Information_System#Modular_Windows" title="Tandy Video Information System">Modular Windows</a></i> is a special version of <i>Windows 3.1</i>, designed to run on <a href="/wiki/Tandy_Video_Information_System" title="Tandy Video Information System">Tandy Video Information System</a>.</p>
<h3><span class="mw-headline" id="Windows_3.11">Windows 3.11</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=6" title="Edit section: Windows 3.11">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Windows 3.11 was released on November 8, 1993. It did not add any feature improvements over Windows 3.1; it only corrected problems. Microsoft replaced all retail versions of Windows 3.1 with Windows 3.11 and provided a free upgrade to anyone who currently owned Windows 3.1.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></p>

<h3><span class="mw-headline" id="Windows_3.2">Windows 3.2</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=7" title="Edit section: Windows 3.2">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>On November 22, 1993, Microsoft released a <a class="mw-redirect" href="/wiki/Simplified_Chinese_character" title="Simplified Chinese character">Simplified Chinese</a> version of Windows for the Chinese market. A year later, an update was released, which identified itself as <i>Windows 3.2</i>. Thus, Windows 3.2 is an updated version of the Chinese version of Windows 3.1.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> The update was limited to this language version, as it only fixed issues related to the complex input system for the Chinese language.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></p>
<p>Windows 3.2 was generally sold by computer manufacturers with a ten-disk version of <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a> that also had Simplified Chinese characters in basic output and some translated utilities.</p>
<h2><span class="mw-headline" id="Windows_for_Workgroups">Windows for Workgroups</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=8" title="Edit section: Windows for Workgroups">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Windows_for_Workgroups.png"><img alt="" class="thumbimage" data-file-height="480" data-file-width="640" height="165" src="//upload.wikimedia.org/wikipedia/en/thumb/c/c2/Windows_for_Workgroups.png/220px-Windows_for_Workgroups.png" srcset="//upload.wikimedia.org/wikipedia/en/thumb/c/c2/Windows_for_Workgroups.png/330px-Windows_for_Workgroups.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/c/c2/Windows_for_Workgroups.png/440px-Windows_for_Workgroups.png 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Windows_for_Workgroups.png" title="Enlarge"></a></div>
Network capabilities of Windows for Workgroups 3.11</div>
</div>
</div>
<p>Windows for Workgroups is an extension that allowed users to share their resources and to request those of others without a centralized authentication server. It used <a href="/wiki/Server_Message_Block" title="Server Message Block">SMB</a> protocol over <a href="/wiki/NetBIOS" title="NetBIOS">NetBIOS</a>.</p>
<h3><span class="mw-headline" id="Windows_for_Workgroups_3.1">Windows for Workgroups 3.1</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=9" title="Edit section: Windows for Workgroups 3.1">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><i>Windows for Workgroups 3.1</i> (originally codenamed <i>Winball</i> and later <i>Sparta</i>), released in October 1992,<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> is an extended version of Windows 3.1 that features native networking support. It comes with <a href="/wiki/Server_Message_Block" title="Server Message Block">SMB</a> file sharing support via <a href="/wiki/NetBIOS" title="NetBIOS">NetBIOS</a>-based <a href="/wiki/NetBIOS_Frames_protocol" title="NetBIOS Frames protocol">NBF</a> and/or <a class="mw-redirect" href="/wiki/IPX" title="IPX">IPX</a> network transport protocols and introduces the <a class="mw-redirect" href="/wiki/Hearts_(Windows)" title="Hearts (Windows)">Hearts</a> card game and VSHARE.386, a <a href="/wiki/VxD" title="VxD">VxD</a> version of <a class="mw-redirect" href="/wiki/SHARE.EXE" title="SHARE.EXE">SHARE.EXE</a> (a <a href="/wiki/Terminate_and_stay_resident_program" title="Terminate and stay resident program">terminate-and-stay-resident program</a>).</p>
<h3><span class="mw-headline" id="Windows_for_Workgroups_3.11">Windows for Workgroups 3.11</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=10" title="Edit section: Windows for Workgroups 3.11">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><i>Windows for Workgroups 3.11</i> (originally codenamed <i>Snowball</i>) was released on August 11, 1993,<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> and shipped in November 1993.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> It supported <a href="/wiki/32-bit_file_access" title="32-bit file access">32-bit file access</a>, full 32-bit network redirectors, and VCACHE.386 file cache, shared between them. WFW 3.11 dropped standard mode support and requires a 386 machine to run.</p>
<p>A <a href="/wiki/Winsock" title="Winsock">Winsock</a> package was required to support <a class="mw-redirect" href="/wiki/TCP/IP" title="TCP/IP">TCP/IP</a> <a href="/wiki/Computer_network" title="Computer network">networking</a> in Windows 3.x. Usually third-party packages were used, but in August 1994, Microsoft released an add-on package (codenamed <i>Wolverine</i>) that provided TCP/IP support in Windows for Workgroups 3.11. Wolverine was a 32-bit stack (accessible from 16-bit Windows applications via WinSock <a class="mw-redirect" href="/wiki/Thunk_(compatibility_mapping)" title="Thunk (compatibility mapping)">Thunk</a>), which gave it superior performance to most of the third-party TCP/IP Windows stacks available. However, it was only compatible with Windows for Workgroups 3.11, and lacked support for dial-up. Wolverine stack was an early version of the TCP/IP stack that would later ship with Windows 95, and provided an early testbed for the 16-to-32-bit compatibility layer that was crucial to Windows 95's success.</p>
<p>Following the release of MS-DOS 6.22 in 1994, WFW 3.11 largely replaced Windows 3.1 for OEM installations on new PCs due to its improved capabilities and greater stability.</p>
<h2><span class="mw-headline" id="Add-ons">Add-ons</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=11" title="Edit section: Add-ons">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Video_for_Windows">Video for Windows</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=12" title="Edit section: Video for Windows">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Video_for_Windows" title="Video for Windows">Video for Windows</a></div>
<p>Video for Windows was first introduced in November 1992 as a reaction to <a class="mw-redirect" href="/wiki/Apple_Computer" title="Apple Computer">Apple Computer</a>'s <a href="/wiki/QuickTime" title="QuickTime">QuickTime</a> technology which added digital video to <a class="mw-redirect" href="/wiki/Apple_Macintosh" title="Apple Macintosh">Macintosh</a>. Costing around $200,<sup class="reference" id="cite_ref-Issue_80_17-0"><a href="#cite_note-Issue_80-17">[17]</a></sup> the software included editing and encoding programs for use with video input boards. A runtime version for viewing videos only was also made available. Originally released as a free add-on to Windows 3.1 and Windows 3.11, it then became an integral component of <a href="/wiki/Windows_95" title="Windows 95">Windows 95</a> and later. Like QuickTime there were three components in Video for Windows. The technology introduced a file format designed to store digital video, <a href="/wiki/Audio_Video_Interleave" title="Audio Video Interleave">Audio Video Interleave</a> (AVI). The technology provided an <a href="/wiki/Application_programming_interface" title="Application programming interface">application programming interface</a> that allowed Windows <a href="/wiki/Software_developer" title="Software developer">software developers</a> to add the ability to play or manipulate <a href="/wiki/Digital_video" title="Digital video">digital video</a> to their own applications. Lastly, it included a suite of software for playing and manipulating digital video.</p>
<h3><span class="mw-headline" id="Windows_for_Pen_Computing">Windows for Pen Computing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=13" title="Edit section: Windows for Pen Computing">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Windows_for_Pen_Computing" title="Windows for Pen Computing">Windows for Pen Computing</a></div>
<p>Windows for Pen Computing was a series of <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>-produced add-ons for <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> versions in the mid-1990s with additional tools for <a href="/wiki/Tablet_computer" title="Tablet computer">tablet PCs</a>. Windows for Pen Computing (also known as <i>Pen Windows</i> and <i>W4PC</i>) was developed as Microsoft's <a href="/wiki/Pen_computing" title="Pen computing">Pen computing</a> response to <a href="/wiki/PenPoint_OS" title="PenPoint OS">PenPoint OS</a> by <a class="mw-redirect" href="/wiki/GO_Corporation" title="GO Corporation">GO Corporation</a>.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> Windows for Pen Computing was rendered obsolete by <a href="/wiki/Tablet_computer" title="Tablet computer">Tablet PC</a> support for <a class="mw-redirect" href="/wiki/Windows_XP_Tablet_PC_Edition" title="Windows XP Tablet PC Edition">Windows XP Tablet PC Edition</a> in 2002.</p>
<h3><span class="mw-headline" id="Win32s">Win32s</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=14" title="Edit section: Win32s">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Win32s" title="Win32s">Win32s</a></div>
<p>Windows 3.1x was given limited compatibility with the then-new 32-bit <a href="/wiki/Windows_API" title="Windows API">Windows API</a> used by <a href="/wiki/Windows_NT" title="Windows NT">Windows NT</a> by another add-on package, <a href="/wiki/Win32s" title="Win32s">Win32s</a>. There was a rumor that Microsoft did not want to increase any mainstream Windows 3.1x version to something like "Windows 3.2" because it could be confused with the <a class="mw-redirect" href="/wiki/Win32" title="Win32">Win32</a> API or otherwise distract consumers from upgrading to a "real 32-bit OS" like the then-upcoming Windows 95 was, though <a href="/wiki/Windows_NT_3.1" title="Windows NT 3.1">Windows NT 3.1</a> and <a href="/wiki/Windows_NT_3.5" title="Windows NT 3.5">3.5</a> were both 32-bit operating systems that looked similar in appearance. A game called <a href="/wiki/Microsoft_FreeCell" title="Microsoft FreeCell">FreeCell</a> was included for testing the new Win32s functions.</p>
<h3><span class="mw-headline" id="WinG">WinG</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=15" title="Edit section: WinG">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/WinG" title="WinG">WinG</a></div>
<p>To entice game manufacturers to move from DOS to Windows, Microsoft provided a first attempt at high-speed graphics and animation capabilities for Windows 3.1x, introduced in September 1994. Windows' <a href="/wiki/Graphics_Device_Interface" title="Graphics Device Interface">GDI</a> capabilities were originally designed with static images in-mind, allowing only for write-only graphics calls. WinG provided a device-independent interface to graphics and printer hardware, and allowed programs to have both read and write capabilities to the WinGDC (WinG device context).<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup></p>
<h2><span class="mw-headline" id="Applications">Applications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=16" title="Edit section: Applications">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Windows 3.1x introduced new possibilities for applications, especially multimedia applications. During this era, Microsoft developed a new range of software that was implemented on this operating environment, called <a href="/wiki/Microsoft_Home" title="Microsoft Home">Microsoft Home</a>, <a href="/wiki/Microsoft_Bob" title="Microsoft Bob">Microsoft Bob</a> being one of the programs.</p>
<p>As the first versions of Windows to enjoy major commercial success and software support, Windows 3.1 and WFW 3.11 quickly replaced DOS as the platform for application software on PC compatibles. Multimedia software (especially games) proliferated, although many games continued to run on DOS until Windows 95.</p>
<h3><span class="mw-headline" id="Program_Manager">Program Manager</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=17" title="Edit section: Program Manager">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Program_Manager" title="Program Manager">Program Manager</a></div>
<p>Program Manager was included in all versions of Windows from <a href="/wiki/Windows_3.0" title="Windows 3.0">version 3.0</a> until Windows XP Service Pack 1. A non-operable icon library named progman.exe is included in <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a> Service Pack 2, and the file was removed entirely from <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a>.</p>
<p>If Program Manager is started under Windows XP Service Pack 2 and later, it does not appear to run, but when a .grp file created for Windows 3.1 is processed, it converts .grp file contents to a <a class="mw-redirect" href="/wiki/Start_Menu" title="Start Menu">Start Menu</a> folder.</p>
<h3><span class="mw-headline" id="Internet_Explorer">Internet Explorer</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=18" title="Edit section: Internet Explorer">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Internet_Explorer_2" title="Internet Explorer 2">Internet Explorer 2</a> through <a href="/wiki/Internet_Explorer_5" title="Internet Explorer 5">Internet Explorer 5</a> were released for Windows 3.1.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup></p>
<h2><span class="mw-headline" id="Promotion_and_reception">Promotion and reception</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=19" title="Edit section: Promotion and reception">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Microsoft began a television advertising campaign for the first time on March 1, 1992. The advertisements, developed by <a href="/wiki/Ogilvy_%26_Mather" title="Ogilvy &amp; Mather">Ogilvy &amp; Mather</a>, were designed to introduce a broader audience to Windows.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> Windows 3.1 was shipped worldwide on April 6, 1992, and reached three million sales two months later.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup> The year of Windows 3.1's release was successful for Microsoft, which was named the "Most Innovative Company Operating in the U.S." by <i><a href="/wiki/Forbes" title="Forbes">Forbes</a></i> magazine, while Windows became the most widely used <a href="/wiki/Graphical_user_interface" title="Graphical user interface">GUI</a>-based operating environment.<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup></p>
<h2><span class="mw-headline" id="Controversy">Controversy</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=20" title="Edit section: Controversy">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="DR-DOS_compatibility">DR-DOS compatibility</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=21" title="Edit section: DR-DOS compatibility">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/AARD_code" title="AARD code">AARD code</a></div>
<p>The installer to the beta release used code that checked whether it was running on Microsoft-licensed DOS or another DOS operating system (such as <a href="/wiki/DR-DOS" title="DR-DOS">DR-DOS</a>). The code ran several functional tests that succeeded on MS-DOS and <a href="/wiki/IBM_PC_DOS" title="IBM PC DOS">IBM PC DOS</a>, but resulted in a technical support message on competing operating systems. If the system was not MS-DOS, the installer would fail. Digital Research, who owned DR-DOS, released a patch within weeks to allow the installer to continue. Microsoft disabled, but did not remove, this warning message for the final release of Windows 3.1. When <a href="/wiki/Caldera_(company)" title="Caldera (company)">Caldera</a> bought DR-DOS from <a href="/wiki/Novell" title="Novell">Novell</a>, they brought a lawsuit against Microsoft over the AARD code, which was later settled.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup><sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup></p>
<h2><span class="mw-headline" id="Legacy">Legacy</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Windows_3.1x&amp;action=edit&amp;section=22" title="Edit section: Legacy">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Windows 3.x was superseded by the release of <a href="/wiki/Windows_95" title="Windows 95">Windows 95</a> in August 1995. Microsoft officially dropped support for all 16-bit versions of Windows on December 31, 2001.</p>
<p>Windows 3.1 found a niche market as an <a href="/wiki/Embedded_operating_system" title="Embedded operating system">embedded operating system</a> after becoming obsolete in the PC world. As of November 2008, both <a href="/wiki/Virgin_Atlantic" title="Virgin Atlantic">Virgin Atlantic</a> and <a href="/wiki/Qantas" title="Qantas">Qantas</a> employed it for some of the onboard entertainment systems on long-distance jets. It also sees continued use as an embedded OS in retail cash tills.<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup> It is also used as a secondary application in <a href="/wiki/DOSBox" title="DOSBox">DOSBox</a> to enable emulation of <a class="mw-redirect" href="/wiki/Win16" title="Win16">Win16</a> games on 64-bit Windows.</p>
<p>On July 9, 2008, it was announced that Windows for Workgroups 3.11 for the embedded devices channel would no longer be made available for <a class="mw-redirect" href="/wiki/Original_Equipment_Manufacturer" title="Original Equipment Manufacturer">OEM</a> distribution as of November 1, 2008.<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup></p>
<p>On July 14, 2013, <a href="/wiki/Linux_kernel" title="Linux kernel">Linux kernel</a> 3.11 was officially named "Linux For Workgroups" as a <a href="/wiki/Tongue-in-cheek" title="Tongue-in-cheek">tongue-in-cheek</a> reference to "Windows for Workgroups 3.11".<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup></p>
<p>On November 7, 2015, <a href="/wiki/Orly_Airport" title="Orly Airport">Orly Airport</a> near <a href="/wiki/Paris" title="Paris">Paris</a>, <a href="/wiki/France" title="France">France</a>, had a major computer glitch that interrupted its operations for some time. The newspaper <i><a class="mw-redirect" href="/wiki/Le_Canard_Encha%C3%AEn%C3%A9" title="Le Canard Enchaîné">Le Canard Enchaîné</a></i> later revealed that the glitch happened in an essential <a href="/wiki/Meteorology#Aviation_meteorology" title="Meteorology">meteorological</a> system called DECOR, which at the time of the incident still ran on Windows 3.1 – 23 years after the operating system's release and 14 years after Microsoft ceased to support it. The French Transportation Minister promised to have the system replaced by 2017, but the secretary general of the French <a href="/wiki/Air_traffic_controller" title="Air traffic controller">air traffic controller</a> union expressed his skepticism.<sup class="reference" id="cite_ref-vice-orly_29-0"><a href="#cite_note-vice-orly-29">[29]</a></sup></p>
