<p><b>자료구조</b>(資料構造, <span style="font-size: smaller;"><a href="/wiki/%EC%98%81%EC%96%B4" title="영어">영어</a>:</span> <span lang="en" xml:lang="en">data structure</span>)는 <a class="mw-redirect" href="/wiki/%EC%A0%84%EC%82%B0%ED%95%99" title="전산학">전산학</a>에서 <a href="/wiki/%EC%9E%90%EB%A3%8C" title="자료">자료</a>를 효율적으로 이용할 수 있도록 컴퓨터에 저장하는 방법이다. 신중히 선택한 자료구조는 보다 효율적인 <a href="/wiki/%EC%95%8C%EA%B3%A0%EB%A6%AC%EC%A6%98" title="알고리즘">알고리즘</a>을 사용할 수 있게 한다. 이러한 자료구조의 선택문제는 대개 <a class="mw-redirect" href="/wiki/%EC%B6%94%EC%83%81%EC%A0%81_%EC%9E%90%EB%A3%8C%EA%B5%AC%EC%A1%B0" title="추상적 자료구조">추상적 자료구조</a>의 선택으로부터 시작하는 경우가 많다. 효과적으로 설계된 자료구조는 실행시간 혹은 메모리 용량과 같은 자원을 최소한으로 사용하면서 연산을 수행하도록 해준다.</p>
<p>자료구조에는 여러 종류가 있으며, 이러한 각각의 자료구조는 각자의 연산 및 목적에 맞추어져 있다. 예를 들어 <a class="mw-redirect" href="/wiki/B-%ED%8A%B8%EB%A6%AC" title="B-트리">B-트리</a>는 데이터베이스에 효율적이며, <a href="/wiki/%EB%9D%BC%EC%9A%B0%ED%8C%85_%ED%85%8C%EC%9D%B4%EB%B8%94" title="라우팅 테이블">라우팅 테이블</a>은 네트워크(인터넷, 인트라넷)에 일반적이다.</p>
<p>다양한 프로그램을 설계할 때, 어떠한 자료구조를 선택할지는 가장 우선적으로 고려되어야 한다. 이는 큰 시스템을 제작할 때 구현의 난이도나 최종 결과물의 성능이 자료구조에 크게 의존한다는 것을 많은 경험이 뒷받침하기 때문이다. 일단 자료구조가 선택되면 적용할 <a href="/wiki/%EC%95%8C%EA%B3%A0%EB%A6%AC%EC%A6%98" title="알고리즘">알고리즘</a>은 상대적으로 명확해지기 마련이다. 때로는 반대 순서로 정해지기도 하는데, 이는 목표로 하는 연산이 특정한 알고리즘을 반드시 필요로 하며, 해당 알고리즘은 특정 자료구조에서 가장 나은 성능을 발휘할 때와 같은 경우이다. 어떠한 경우든, 적절한 자료구조의 선택은 필수적이다.</p>
<p>이러한 관점은 알고리즘보다 자료구조가 보다 중요한 요소로 적용되는 많은 정형화된 개발론 그리고 <a href="/wiki/%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4" title="프로그래밍 언어">프로그래밍 언어</a>의 개발을 촉발시켰다. 대부분의 언어는 일정 수준의 모듈개념을 가지고 있으며, 이는 자료구조가 검증된 구현은 감춘 채 인터페이스만을 이용하여 다양한 프로그램에서 사용되는 것을 가능하게 해준다. <a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="C (프로그래밍 언어)">C</a>나 <a href="/wiki/%EC%9E%90%EB%B0%94_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="자바 (프로그래밍 언어)">자바</a>와 같은 <a class="mw-redirect" href="/wiki/%EA%B0%9D%EC%B2%B4%EC%A7%80%ED%96%A5" title="객체지향">객체지향</a> 프로그래밍 언어는 특별히 이러한 목적으로 객체를 사용한다.</p>
<p>이러한 자료구조의 중요성으로 말미암아, 최근의 프로그래밍 언어 및 개발 환경은 다양한 <a href="/wiki/%ED%91%9C%EC%A4%80_%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="표준 라이브러리">표준 라이브러리</a>를 제공하고 있다. 예로, C++의 <a href="/wiki/%ED%91%9C%EC%A4%80_%ED%85%9C%ED%94%8C%EB%A6%BF_%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="표준 템플릿 라이브러리">표준 템플릿 라이브러리</a>나 자바의 <a href="/wiki/%EC%9E%90%EB%B0%94_API" title="자바 API">자바 API</a>, <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_.NET" title="마이크로소프트 .NET">마이크로소프트 .NET</a>과 같은 것을 들 수 있다.</p>
<p>자료구조에서 가장 기초적인 단위는 <a href="/wiki/%ED%96%89%EB%A0%AC" title="행렬">행렬</a>, <a class="mw-disambig" href="/wiki/%EB%A0%88%EC%BD%94%EB%93%9C" title="레코드">레코드</a>, <a class="mw-redirect mw-disambig" href="/wiki/%EC%9C%A0%EB%8B%88%EC%98%A8" title="유니온">유니온</a>, <a class="new" href="/w/index.php?title=%EC%B0%B8%EC%A1%B0&amp;action=edit&amp;redlink=1" title="참조 (없는 문서)">참조</a>와 같은 것이다. 예를 들어, Nullable 참조는 참조와 유니온의 조합으로 나타낼 수 있으며, 가장 단순한 자료구조 가운데 하나인 <a href="/wiki/%EC%97%B0%EA%B2%B0_%EB%A6%AC%EC%8A%A4%ED%8A%B8" title="연결 리스트">연결 리스트</a>는 레코드와 Nullable 참조로 나타낼 수 있다.</p>

<h2><span class="mw-headline" id=".EB.B6.84.EB.A5.98">분류</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%9E%90%EB%A3%8C_%EA%B5%AC%EC%A1%B0&amp;action=edit&amp;section=1" title="부분 편집: 분류">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a class="new" href="/w/index.php?title=%EC%9E%90%EB%A3%8C%EA%B5%AC%EC%A1%B0_%EB%AA%A9%EB%A1%9D&amp;action=edit&amp;redlink=1" title="자료구조 목록 (없는 문서)">자료구조 목록</a>입니다.</div>
<p>자료의 특성과 크기, 주요 사용법과 수행하는 연산의 종류, 구현에 필요한 기억 공간 크기에 따라 여러 가지 종류의 자료구조 중 하나를 선택할 수 있다. 자료구조의 종류로는 자료형의 따라 분류하는 단순 구조와 자료 간 관계가 1 대 1인 선형 구조, 1 대 다 혹은 다 대 다 구조인 비선형 구조, 마지막으로 파일 구조가 있다..<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<h3><span class="mw-headline" id=".EA.B5.AC.ED.98.84.EC.97.90_.EB.94.B0.EB.9D.BC">구현에 따라</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%9E%90%EB%A3%8C_%EA%B5%AC%EC%A1%B0&amp;action=edit&amp;section=2" title="부분 편집: 구현에 따라">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li><a href="/wiki/%EB%B0%B0%EC%97%B4" title="배열">배열</a> - 가장 일반적인 구조이다. 메모리 상에 같은 타입의 자료가 연속적으로 저장된다. 자료값을 나타내는 가장 작은 단위가 자료를 다루는 단위이다.</li>
<li><a href="/wiki/%ED%8A%9C%ED%94%8C" title="튜플">튜플</a> - 둘 이상의 자료형을 묶음으로 다루는 구조이다.</li>
<li><a href="/wiki/%EC%97%B0%EA%B2%B0_%EB%A6%AC%EC%8A%A4%ED%8A%B8" title="연결 리스트">연결 리스트</a> - 노드를 단위로 한다. 노드는 자료와 다음 노드를 가리키는 참조값으로 구성되어 있다. 노드가 다음 노드로 아무것도 가리키지 않으면 리스트의 끝이다.</li>
<li><a class="new" href="/w/index.php?title=%ED%99%98%ED%98%95_%EC%97%B0%EA%B2%B0_%EB%A6%AC%EC%8A%A4%ED%8A%B8&amp;action=edit&amp;redlink=1" title="환형 연결 리스트 (없는 문서)">환형 연결 리스트</a> - 각 노드는 다음 노드를 가리키고, 마지막 노드가 처음 노드를 가리키는 연결 리스트이다.</li>
<li><a class="new" href="/w/index.php?title=%EC%9D%B4%EC%A4%91_%EC%97%B0%EA%B2%B0_%EB%A6%AC%EC%8A%A4%ED%8A%B8&amp;action=edit&amp;redlink=1" title="이중 연결 리스트 (없는 문서)">이중 연결 리스트</a> - 각 노드는 이전 노드와 다음 노드를 가리키는 참조값으로 구성된다. 처음 노드의 이전 노드와 마지막 노드의 다음 노드는 없다.</li>
<li><a class="new" href="/w/index.php?title=%ED%99%98%ED%98%95_%EC%9D%B4%EC%A4%91_%EC%97%B0%EA%B2%B0_%EB%A6%AC%EC%8A%A4%ED%8A%B8&amp;action=edit&amp;redlink=1" title="환형 이중 연결 리스트 (없는 문서)">환형 이중 연결 리스트</a> - 처음 노드가 이전 노드로 마지막 노드를 가리키고, 마지막 노드가 다음 노드로 처음 노드를 가리키는 이중 연결 리스트이다.</li>
<li><a href="/wiki/%ED%95%B4%EC%8B%9C_%ED%85%8C%EC%9D%B4%EB%B8%94" title="해시 테이블">해시 테이블</a> - 개체가 <a class="mw-disambig" href="/wiki/%ED%95%B4%EC%8B%9C" title="해시">해시</a>값에 따라 인덱싱된다.</li>
</ul>
<h3><span class="mw-headline" id=".ED.98.95.ED.83.9C.EC.97.90_.EB.94.B0.EB.9D.BC">형태에 따라</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%9E%90%EB%A3%8C_%EA%B5%AC%EC%A1%B0&amp;action=edit&amp;section=3" title="부분 편집: 형태에 따라">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>선형 구조</li>
</ul>
<dl>
<dd>
<ul>
<li><a href="/wiki/%EC%8A%A4%ED%83%9D" title="스택">스택</a> - 스택 자료구조에 먼저 저장된 것이 꺼내어 쓸 때는 제일 나중에 나온다. 반대로, 가장 최근에 저장된 것이 꺼내어 쓸 때는 제일 먼저 나온다. 만약, 자료들의 나열 순서를 바꾸고 싶다면 스택에 집어 넣었다가 꺼내면 역순으로 바뀐다.</li>
<li><a class="new" href="/w/index.php?title=%ED%81%90_(%EC%9E%90%EB%A3%8C%EA%B5%AC%EC%A1%B0)&amp;action=edit&amp;redlink=1" title="큐 (자료구조) (없는 문서)">큐</a> - 스택과 반대로 큐 자료구조에 먼저 저장된 것이 제일 먼저 나온다. 반대로, 가장 나중에 저장된 것이 꺼내어 쓸 때는 가장 나중에 나온다.</li>
</ul>
<dl>
<dd>
<ul>
<li><a class="new" href="/w/index.php?title=%ED%99%98%ED%98%95_%ED%81%90&amp;action=edit&amp;redlink=1" title="환형 큐 (없는 문서)">환형 큐</a> - 한정된 길이 안에서 부수적인 작업 없이 읽고 쓰기를 할 수 있는 큐이다.</li>
</ul>
</dd>
</dl>
<ul>
<li><a class="new" href="/w/index.php?title=%EB%8D%B1_(%EC%9E%90%EB%A3%8C%EA%B5%AC%EC%A1%B0)&amp;action=edit&amp;redlink=1" title="덱 (자료구조) (없는 문서)">덱</a> - 양쪽에서 넣기와 빼기를 할 수 있는 일반화된 선형 구조이다.</li>
</ul>
</dd>
</dl>
<ul>
<li>비선형 구조</li>
</ul>
<dl>
<dd>
<ul>
<li><a class="new" href="/w/index.php?title=%EA%B7%B8%EB%9E%98%ED%94%84_(%EC%9E%90%EB%A3%8C%EA%B5%AC%EC%A1%B0)&amp;action=edit&amp;redlink=1" title="그래프 (자료구조) (없는 문서)">그래프</a> - 꼭짓점과 꼭짓점을 잇는 변으로 구성된다.</li>
</ul>
<dl>
<dd>
<ul>
<li><a href="/wiki/%EC%9C%A0%ED%96%A5_%EA%B7%B8%EB%9E%98%ED%94%84" title="유향 그래프">유향 그래프</a>, <a class="mw-redirect" href="/wiki/%EB%AC%B4%ED%96%A5_%EA%B7%B8%EB%9E%98%ED%94%84" title="무향 그래프">무향 그래프</a> - 변이 방향성을 갖는지 갖지 않는지에 따른 그래프의 분류이다.</li>
</ul>
</dd>
</dl>
<ul>
<li><a href="/wiki/%ED%8A%B8%EB%A6%AC_%EA%B5%AC%EC%A1%B0" title="트리 구조">트리</a> - 뿌리와, 뿌리 또는 다른 꼭짓점을 단 하나의 부모로 갖는 꼭짓점들로 이루어진 구조. 부모 자식 관계는 변으로 표현된다. 무향 그래프의 경우, <a class="new" href="/w/index.php?title=%EC%88%9C%ED%99%98_(%EA%B7%B8%EB%9E%98%ED%94%84)&amp;action=edit&amp;redlink=1" title="순환 (그래프) (없는 문서)">순환</a>이 없는 연결 그래프를 뜻한다. 유향 그래프의 경우 변의 방향은 보통 부모를 가리키도록 구현된다.</li>
</ul>
<dl>
<dd>* <a href="/wiki/%EC%9D%B4%EC%A7%84_%ED%8A%B8%EB%A6%AC" title="이진 트리">이진 트리</a> - 자식이 최대 두 개인 트리.
<ul>
<li><a href="/wiki/%ED%9E%99_(%EC%9E%90%EB%A3%8C_%EA%B5%AC%EC%A1%B0)" title="힙 (자료 구조)">힙</a> - 이진트리의 일종으로 이진트리에 어떤 특성을 부여한 것이라 할 수 있다.</li>
</ul>
</dd>
</dl>
</dd>
