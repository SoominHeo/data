<p><b>DivX</b>(<b>디빅</b> 또는 <b>디빅스</b>)는 <a class="new" href="/w/index.php?title=DivX_%EC%82%AC&amp;action=edit&amp;redlink=1" title="DivX 사 (없는 문서)">DivX 사</a>가 <a href="/wiki/MPEG-4" title="MPEG-4">MPEG-4</a> 기술을 기반으로 독자적으로 개발한 영상 <a href="/wiki/%EC%BD%94%EB%8D%B1" title="코덱">코덱</a>이다. 크고 긴 영상도 원본 품질에 가까우면서도 작은 크기로 압축시켜주는 강력한 기능을 갖고 있다. 원래 크기의 약 15%까지 압축하는 것이 가능하다.<sup class="reference" id="cite_ref-info_1-0"><a href="#cite_note-info-1">[1]</a></sup></p>
<p>DivX 코덱에는 두가지 종류가 있다. <a href="/wiki/MPEG-4_%ED%8C%8C%ED%8A%B8_2" title="MPEG-4 파트 2">MPEG-4 파트 2</a> DivX 코덱과 <a href="/wiki/H.264/MPEG-4_AVC" title="H.264/MPEG-4 AVC">H.264/MPEG-4 AVC</a> DivX 플러스 HD 코덱이다. 디빅스 코덱은 주로 영화 같은 멀티미디어 데이터를 <a href="/wiki/%EB%A6%AC%ED%95%91" title="리핑">리핑</a>하는 데 주로 이용된다.</p>
<p>처음 개발 당시에는 오디오 코덱만 존재하였으나, 이제는 <a href="/wiki/%EB%B9%84%EB%94%94%EC%98%A4" title="비디오">영상</a> 코덱만이 제공되고 있다.<sup class="정리_필요 noprint">[<a href="/wiki/%EC%9C%84%ED%82%A4%EB%B0%B1%EA%B3%BC:%EC%B6%9C%EC%B2%98_%EB%B0%9D%ED%9E%88%EA%B8%B0" title="위키백과:출처 밝히기">출처 필요</a>]</sup> 따라서 지금은 DivX을 영상 코덱으로 가리키는 경우가 대부분이다.</p>

<h2><span class="mw-headline" id="DivX_.ED.8F.AC.EB.A7.B7">DivX 포맷</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DivX&amp;action=edit&amp;section=1" title="부분 편집: DivX 포맷">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="DivX_.EB.AF.B8.EB.94.94.EC.96.B4_.ED.8F.AC.EB.A7.B7.28DMF.29">DivX 미디어 포맷(DMF)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DivX&amp;action=edit&amp;section=2" title="부분 편집: DivX 미디어 포맷(DMF)">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<table class="infobox">

<tr>
<th style="text-align: left;">파일 포맷 종류</th>
<td><a class="mw-redirect" href="/wiki/%EB%AF%B8%EB%94%94%EC%96%B4_%EC%BB%A8%ED%85%8C%EC%9D%B4%EB%84%88" title="미디어 컨테이너">미디어 컨테이너</a></td>

<p>DivX 6은 DivX의 범위를 코덱과 플레이어에서, 'DivX 미디어 포맷(DMF)'이라 불리는 <a href="/wiki/%EB%94%94%EC%A7%80%ED%84%B8_%EC%BB%A8%ED%85%8C%EC%9D%B4%EB%84%88_%ED%8F%AC%EB%A7%B7" title="디지털 컨테이너 포맷">미디어 컨테이너 포맷</a>을 추가함으로써 넓혀주었다.<sup class="reference" id="cite_ref-info_1-1"><a href="#cite_note-info-1">[1]</a></sup> DMF는 다음과 같은 기능을 지원한다.</p>
<ul>
<li>인터렉티브 비디오 메뉴</li>
<li>다중 자막(XSUB)</li>
<li>다중 오디오 트랙</li>
<li>다중 비디오 스트림</li>
<li>책갈피 기능(Chapter points)</li>
<li>다른 메타데이터(XTAG)</li>
<li>다중 포맷</li>
<li>AVI와의 부분적 <a href="/wiki/%ED%95%98%EC%9C%84_%ED%98%B8%ED%99%98%EC%84%B1" title="하위 호환성">하위 호환성</a></li>
</ul>
<p>디빅스 미디어 포맷은 'DivX Ultra Certified' 프로파일과 함께 출시되었다. '<b>Ultra'</b> 인증 플레이어는 모든 DivX 미디어 포맷 기능을 지원한다. 디빅스 미디어 포맷으로 인코딩된 영상은 MPEG-4 비디오 스트림이지만, 디빅스 미디어 포맷은 애플의 퀵타임과 같은 미디어 컨테이너 포맷들과 유사하다. DVD와 같은 미디어 포맷이 MPEG-2 영상을 자신의 사양으로 명시하는 것과 같은 방식으로 DivX 미디어 포맷은 MPEG-4 호환 가능 영상을 자신의 사양으로 명시한다. 그러나 .divx 확장자를 사용함에도 불구하고 이 포맷은 AVI 파일 포맷의 확장판이다. 다중 오디오와 자막 트랙을 포함하는 방식은 <a href="/wiki/RIFF" title="RIFF">RIFF</a> 헤더와 기타 AVI 해킹판을 저장하는 것을 포함한다. 이는 플레이어가 .divx 컨테이너 포맷에 사용가능한 새로운 기능을 지원하지 않는 플레이어가 적어도 기초 비디오 스트림을 재생할 수 있게 하기 위해 DivX 사가 일부러 AVI와의 최소한의 하위 호환성을 유지하려고 한 것이었다. 물론 디빅스 코덱은 여전히 전통적인 표준 AVI 파일 생성 방식을 지원한다.</p>
<h3><span class="mw-headline" id="DivX_.ED.94.8C.EB.9F.AC.EC.8A.A4_HD">DivX 플러스 HD</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DivX&amp;action=edit&amp;section=3" title="부분 편집: DivX 플러스 HD">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>DivX 플러스 HD는 사유 DivX 미디어 포맷이 아닌 <a href="/wiki/Matroska" title="Matroska">마트료시카</a> 미디어 포맷을 사용하는 파일 포맷의 브랜드 이름이다. 디빅스 플러스 HD 파일은 <a class="mw-redirect" href="/wiki/H.264" title="H.264">H.264</a> 비디오 비트스트림, <a href="/wiki/%EA%B3%A0%EA%B8%89_%EC%98%A4%EB%94%94%EC%98%A4_%EB%B6%80%ED%98%B8%ED%99%94" title="고급 오디오 부호화">AAC</a> 서라운드 사운드 오디오, 그리고 챕터, 자막, 메타데이터를 정의하는 다양한 XML 기반 첨부문서를 담는다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> 그리고 이 미디어 컨테이너 포맷은 H.264/MPEG-4 AVC 코덱을 위해 사용된다.</p>
