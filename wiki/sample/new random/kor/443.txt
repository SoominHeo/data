<p><b>C</b>는 <a href="/wiki/1972%EB%85%84" title="1972년">1972년</a> <a class="mw-redirect" href="/wiki/%EC%BC%84_%ED%86%B0%EC%8A%A8" title="켄 톰슨">켄 톰슨</a>과 <a href="/wiki/%EB%8D%B0%EB%8B%88%EC%8A%A4_%EB%A6%AC%EC%B9%98" title="데니스 리치">데니스 리치</a>가 벨 연구소에서 일할 당시 새로 개발된 <a href="/wiki/%EC%9C%A0%EB%8B%89%EC%8A%A4" title="유닉스">유닉스</a> <a href="/wiki/%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C" title="운영 체제">운영 체제</a>에서 사용하기 위해 개발한 <a href="/wiki/%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4" title="프로그래밍 언어">프로그래밍 언어</a>이다. 켄 톰슨은 BCPL언어를 필요에 맞추어 개조해서 "B"언어(언어를 개발한 벨 연구소의 B를 따서)라 명명했고, <a href="/wiki/%EB%8D%B0%EB%8B%88%EC%8A%A4_%EB%A6%AC%EC%B9%98" title="데니스 리치">데니스 리치</a>가 이것을 개선하여 C 언어가 탄생했다. 유닉스 시스템의 바탕 프로그램은 모두 C로 작성되었고, 수많은 운영 체제의 <a href="/wiki/%EC%BB%A4%EB%84%90_(%EC%BB%B4%ED%93%A8%ED%8C%85)" title="커널 (컴퓨팅)">커널</a> 또한 C로 만들어졌다. 오늘날 많이 쓰이는 <a href="/wiki/C%2B%2B" title="C++">C++</a>는 C에서 객체 지향형 언어로 발전된 것이다. 또 다른 다양한 최신 언어들도 그 뿌리를 C에 두고 있다.</p>

<h2><span class="mw-headline" id=".EA.B0.9C.EC.9A.94">개요</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=1" title="부분 편집: 개요">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>C는 실질적으로 모든 컴퓨터 시스템에서 사용할 수 있는 프로그래밍 언어이다. 예를 들어 BASIC 등과는 달리 다양한 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%8C%85_%ED%94%8C%EB%9E%AB%ED%8F%BC" title="컴퓨팅 플랫폼">플랫폼</a>에서 <a class="new" href="/w/index.php?title=ANSI_C&amp;action=edit&amp;redlink=1" title="ANSI C (없는 문서)">ANSI C</a>의 정의에 따르는 비교적 동일한 구현이 가능하다. 모든 C 시스템에는 정규화된 <a class="mw-redirect" href="/wiki/%ED%91%9C%EC%A4%80_C_%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="표준 C 라이브러리">표준 C 라이브러리</a>가 존재한다. 이런 이유와 생성된 프로그램의 높은 성능이 아직까지도 C 언어가 사랑받는 이유 중 하나이다.</p>
<p>그러나 C 언어가 기술적으로 보아 현재 기술 수준에 부합하지 않는다는 의견이 있으며, C를 "이식가능한 고급 <a href="/wiki/%EC%96%B4%EC%85%88%EB%B8%94%EB%A6%AC%EC%96%B4" title="어셈블리어">어셈블리어</a>" 정도로 낮추어 부르기도 한다. 이는 반면 오늘날의 널리 쓰이는 거의 모든 운영 체제 <a href="/wiki/%EC%BB%A4%EB%84%90_(%EC%BB%B4%ED%93%A8%ED%8C%85)" title="커널 (컴퓨팅)">커널</a>이 C를 이용해 구현된 이유이기도 하다. 이처럼 C는 <a class="mw-redirect" href="/wiki/%EC%8B%9C%EC%8A%A4%ED%85%9C_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%A8" title="시스템 프로그램">시스템 프로그램</a> 개발에 매우 적합하지만, 응용 프로그램 개발에도 많이 쓰이기도 한다.</p>
<p><span class="fix" title="이 부분은 출처가 필요합니다.">최근 CT를 이용한 소프트웨어 교육에 대한 중요성이 강조되면서 점차 C 언어 교육에 대한 관심이 뜨거워지고있다.</span><sup class="정리_필요 noprint">[<a href="/wiki/%EC%9C%84%ED%82%A4%EB%B0%B1%EA%B3%BC:%EC%B6%9C%EC%B2%98_%EB%B0%9D%ED%9E%88%EA%B8%B0" title="위키백과:출처 밝히기">출처 필요</a>]</sup></p>

<h3><span class="mw-headline" id=".EC.B4.88.EA.B8.B0_.EA.B0.9C.EB.B0.9C">초기 개발</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=3" title="부분 편집: 초기 개발">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/%ED%8C%8C%EC%9D%BC:Ken_n_dennis.jpg"><img alt="" class="thumbimage" data-file-height="201" data-file-width="310" height="143" src="//upload.wikimedia.org/wikipedia/commons/thumb/3/36/Ken_n_dennis.jpg/220px-Ken_n_dennis.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/3/36/Ken_n_dennis.jpg 1.5x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/%ED%8C%8C%EC%9D%BC:Ken_n_dennis.jpg" title="실제 크기로"></a></div>
C 언어의 개발자인 켄 톰슨과 데니스 리치.</div>
</div>
</div>
<p>C 언어의 초기 개발은 1969년부터 1973년까지 4년에 걸쳐 <a class="mw-redirect" href="/wiki/AT%26T_%EB%B2%A8_%EC%97%B0%EA%B5%AC%EC%86%8C" title="AT&amp;T 벨 연구소">AT&amp;T 벨 연구소</a>에서 이루어 졌으며,<sup class="reference" id="cite_ref-dottcl_1-0"><a href="#cite_note-dottcl-1">[1]</a></sup> <a href="/wiki/%EB%8D%B0%EB%8B%88%EC%8A%A4_%EB%A6%AC%EC%B9%98" title="데니스 리치">데니스 리치</a>의 말에 따르면, 가장 창의적인 작업이 이루어진 기간은 1972년이었다. 언어의 이름이 "C"인 이유는 그 특징이 "B" 언어에서 유래되었기 때문이며, <a class="mw-redirect" href="/wiki/%EC%BC%84_%ED%86%B0%EC%8A%A8" title="켄 톰슨">켄 톰슨</a>에 의하면, B 언어는 BCPL 언어의 기본만 남긴 버전이다.</p>
<p>리치와 톰슨에 의해 초기에 <a class="new" href="/w/index.php?title=PDP-7&amp;action=edit&amp;redlink=1" title="PDP-7 (없는 문서)">PDP-7</a>의 어셈블리 언어로 구현되었던 <a href="/wiki/%EC%9C%A0%EB%8B%89%EC%8A%A4" title="유닉스">유닉스</a> 운영체제의 개발과 C 언어의 기원은 밀접하게 연관되어 있다. 결국 그들은 유닉스 운영체제를 <a href="/wiki/PDP-11" title="PDP-11">PDP-11</a>로 포팅하기로 결정하였다. PDP-11의 기능의 일부, 특히 <a href="/wiki/%EB%B0%94%EC%9D%B4%ED%8A%B8" title="바이트">바이트</a> 접근기능을 활용하지 못하는 B 언어의 부족함이 C 언어의 초기 버전의 개발을 이끌었다.</p>
<p>유닉스의 초기 PDP-11 버전은 어셈블리로 개발되었다. 1973년에 <code>struct</code> 자료형의 추가로, C 언어는 유닉스의 대부분을 C로 쓸 수 있을 정도로 강력해 졌다. 유닉스는 어셈블리가 아닌 언어로 구현된 최초의 운영체제 커널 중의 하나였다.(더 빠른 사례는 <a href="/wiki/PL/I" title="PL/I">PL/I</a>로 쓰인 <a class="mw-redirect" href="/wiki/Multics" title="Multics">Multics</a> 시스템, <a class="mw-redirect" href="/wiki/ALGOL" title="ALGOL">ALGOL</a>로 쓰인 Burroughs B5000을 위한 MCP(Master Control Program)가 있다.) 1977년 경, 리치와 스티븐 C. 존슨이 유닉스 운영체제의 이식성을 향상시키기 위해, C 언어를 추가적으로 변경하였다. 존슨의 <a class="new" href="/w/index.php?title=Portable_C_Compiler&amp;action=edit&amp;redlink=1" title="Portable C Compiler (없는 문서)">Portable C Compiler</a>는 새로운 플랫폼에서의 C의 구현의 기초가 되었다.<sup class="reference" id="cite_ref-unixport_2-0"><a href="#cite_note-unixport-2">[2]</a></sup></p>
<h3><span class="mw-headline" id="K.26R_C">K&amp;R C</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=4" title="부분 편집: K&amp;R C">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="thumb tright">
<div class="thumbinner" style="width:173px;"><a class="image" href="/wiki/%ED%8C%8C%EC%9D%BC:The_C_Programming_Language,_First_Edition_Cover_(2).svg"><img alt="" class="thumbimage" data-file-height="1405" data-file-width="1000" height="240" src="//upload.wikimedia.org/wikipedia/commons/thumb/9/95/The_C_Programming_Language%2C_First_Edition_Cover_%282%29.svg/171px-The_C_Programming_Language%2C_First_Edition_Cover_%282%29.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/9/95/The_C_Programming_Language%2C_First_Edition_Cover_%282%29.svg/256px-The_C_Programming_Language%2C_First_Edition_Cover_%282%29.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/9/95/The_C_Programming_Language%2C_First_Edition_Cover_%282%29.svg/341px-The_C_Programming_Language%2C_First_Edition_Cover_%282%29.svg.png 2x" width="171"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/%ED%8C%8C%EC%9D%BC:The_C_Programming_Language,_First_Edition_Cover_(2).svg" title="실제 크기로"></a></div>
<i>The C Programming Language</i>의 표지</div>
</div>
</div>
<p><a href="/wiki/1978%EB%85%84" title="1978년">1978년</a>에 <a href="/wiki/%EB%B8%8C%EB%9D%BC%EC%9D%B4%EC%96%B8_%EC%BB%A4%EB%8B%88%ED%95%B8" title="브라이언 커니핸">브라이언 커니핸</a>과 <a href="/wiki/%EB%8D%B0%EB%8B%88%EC%8A%A4_%EB%A6%AC%EC%B9%98" title="데니스 리치">데니스 리치</a>가 <i>The C Programming Language</i>라는 책의 초판을 출간했다. 커니핸과 리치의 앞 글자만 따서 C 프로그래머들에게는 "K&amp;R"로 불리는 이 책은, 비공식적이지만 오랫동안 C의 규격과 같은 역할을 했다. 그리하여, 이 책에서 서술하는 C의 판이 "K&amp;R C"란 명칭으로 불리게 되었고, 이 책의 2판에서는 후의 <a class="new" href="/w/index.php?title=ANSI_C&amp;action=edit&amp;redlink=1" title="ANSI C (없는 문서)">ANSI C</a> 표준을 다루게 되었다.</p>
<p>K&amp;R에서 다음과 같은 기능이 등장한다.</p>
<ul>
<li>표준 입출력 라이브러리</li>
<li><code>long int</code> 자료형</li>
<li><code>unsigned int</code> 자료형</li>
<li><code>=-</code>와 같은 형태의 복합 대입 연산자를 <code>-=</code> 형태로 변경
<ul>
<li><code>i=-10</code>라는 코드가 원래 의도한 <code>i =- 10</code> (i의 값을 10 차감)이 아닌, <code>i = -10</code> (i의 값을 -10으로 설정)으로 해석될 수 있기 때문에 중의성을 없애기 위해서 형태를 변경했다.</li>
</ul>
</li>
</ul>
<h3><span class="mw-headline" id="C99">C99</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=5" title="부분 편집: C99">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a href="/wiki/C99" title="C99">C99</a>입니다.</div>
<p>C 언어 표준이 상대적으로 정적으로 남아 있었던 동안, <a href="/wiki/C%2B%2B" title="C++">C++</a>는 표준화를 위하여 계속 진화하고 있었다. 1995년에 1990년의 C 표준에 대한 규약 수정안 1이 출판되었는데, 이는 약간의 세부 사항을 교정하고 국제적 문자 세트에 대한 보다 확장된 지원을 위한 것이었다. C 표준은 1990년대 후반에 더 개정되어, 1999년 ISO/IEC 9899:1999가 출간되었고, 여기서 명시한 규범을 흔히 <a href="/wiki/C99" title="C99">C99</a>라 부른다. 이는 기술적 교정에 의하여 현재까지 3번의 수정이 있었다. 국제 C 표준은 실무 그룹 ISO/IEC JTC1/SC22/WG14에 의해 관리되고 있다.</p>

<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a href="/wiki/C11" title="C11">C11</a>입니다.</div>
<p>2011년 ISO/IEC 9899:2011이 출간되었고, 간단하게 <a href="/wiki/C11" title="C11">C11</a>라고 부른다. <a href="/wiki/C11" title="C11">C11</a> 나오기 전까지 개발 버전을<a class="new" href="/w/index.php?title=C1X&amp;action=edit&amp;redlink=1" title="C1X (없는 문서)">C1X</a>라고 부른다. 최종 개발 버전은 2011년도 4월 N1570이었다.</p>

<h2><span class="mw-headline" id=".EB.A9.94.EB.AA.A8.EB.A6.AC_.EA.B4.80.EB.A6.AC">메모리 관리</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=12" title="부분 편집: 메모리 관리">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>OS에서 응용 프로그램을 실행하거나, CPU의 프로그램을 실행하기 위해 여러가지 영역으로 나누어 메모리를 할당하고 이를 메모리에 올려 실행 한다.</p>
<ul>
<li>OS: 주로 <a class="mw-redirect" href="/wiki/%EC%A0%80%EC%9E%A5%EC%9E%A5%EC%B9%98" title="저장장치">저장장치</a>에서 실행 파일을 메모리에 올려 실행 한다. 따라서 프로그램에 필요한 메모리는 거의 RAM에 할당하고 실행 한다.</li>
<li>CPU을 사용한 장치: OS가 없이 개발자가 설정한 메모리 배치에 따라 코드를 ROM/FLASH에 생산할 때 쓰고, 전원 공급시 실행 한다.</li>
</ul>
<p>C 언어로 개발된 프로그램은 메모리 입장에서 다음과 같은 할당 영역으로 나누어 생각할 수 있다.</p>
<ul>
<li>정적 변수: static을 이용하여 정적 변수임을 명시하고, 단 한 번만 초기화되며 프로그램이 시작할 때 생성되어 종료될 때까지 유지된다.</li>
<li>동적 변수: <a class="mw-disambig" href="/wiki/%ED%9E%99" title="힙">힙</a>(HEAP)영역을 이용하여 할당 함수를 호출하여 변수 영역을 할당 받아 사용하고, 해제 함수에 의해 반납한다.</li>
<li>자동변수: 함수나 블록({ } 이용) 안에서 선언하는 지역변수를 사용하면 <a href="/wiki/%EC%8A%A4%ED%83%9D" title="스택">스택</a>(STACK) 영역에서 자동 할당을 받는다.</li>
</ul>
<p>CPU을 사용하여 개발하여 장치에 넣어 코드를 실행할 때, 힙 영역을 많이 사용하지는 않는다. 따라서 필요 없다면 메모리 공간을 할당할 필요도 없고 힙관리 프로그램 코드(함수를 개발툴에서 라이브러리 형태로 제공)도 필요하지 않는다. 만약 <a class="new" href="/w/index.php?title=Malloc&amp;action=edit&amp;redlink=1" title="Malloc (없는 문서)">malloc</a> 등의 함수를 사용하면, 힙 영역을 사용하겠다는 의미 이기 때문에 힙 영역을 개발자가 선언하여 관리 해야 한다. 이 때 힙관리 프로그램 코드는 자동으로 링크 된다. 물론 저 사양의 CPU 경우, 이 함수를 제공하지 않을 수 있는 컴파일러도 있을 수 있다.</p>

<p>Notes:</p>
<div class="reflist" style="list-style-type: decimal;">
<ol class="references">
<li id="cite_note-MemMapConst-3"><span class="mw-cite-backlink"><a href="#cite_ref-MemMapConst_3-0">↑</a></span> <span class="reference-text">상수값 "홍길동"은 main() 함수가 시작 되기 전, 초기과정에서 <code>char name[]</code>에 복사 된다. 여러개의 초기치 변수가 있을 경우 블록으로 복사 된다. 따라서 개별적으로 복사하는 기계어 코드가 존재하지 않고, 초기설정과정에서 한번에 복사 된다. 이를 구현하기 위해 컴파일러와 링커는 초기치 값만 모아서 처리 한다. 블록 내 지역변수에서 사용하는 초기치 과정은 다르다. 지역변수에서 초기치를 설정하면 그 위치에 초기치 설정하는 기계어 코드가 삽입된다.</span></li>
<li id="cite_note-MemMapR-4"><span class="mw-cite-backlink"><a href="#cite_ref-MemMapR_4-0">↑</a></span> <span class="reference-text">레지스터는 메모리가 아니다. 따라서 레지스터는 메모리 맵에 없는 내용이다. 지역변수를 스택영역에 잡을 것인가 CPU의 레지스터에 잡을 것인가는 프로그램의 복잡도와 최적화에 따라 달라진다. 함수의 기능이 간단하고 변수의 숫자가 적을 경우 레지스터로 자동 배치 될 가능성이 높아진다.</span></li>
</ol>
</div>
<p>각 특성별로 나누어 그룹을 지어 메모리에 배치 하는데, 이것을 링커가 한다. 이렇게 그룹은 나누는 것을 세그먼트(SEGMENT) 또는 섹션(SECTION)이라는 단어를 사용 한다.</p>
<p>위의 그룹은 가장 기본적인 내용만을 표시 한것이다. CPU와 컴파일 마다 다르다. 어떤 컴파일러는 더욱 세밀히 하기도 한다. 그리고 각 세그먼트 이름도 다르다.</p>

<p>컴파일 마다 각 세그먼트 이름과 구조가 다르지만, 예를 들어 중요한 세그먼트 만 표시 하였다, TEXT와 CONST는 ROM/FLASH에 배치 해도 되는 변하지 않는 세그먼트 이므로 같은 부류이고, CPU를 설계하고 코드를 직접 쓰는 경우 ROM/FLASH을 이용한다.</p>
<p>거의 모든 툴에서 이 맵을 파일로 만들어 준다. 물론 옵션으로 설정을 해야하는 경우도 있지만 구조를 얻을 수 있다. 함수와 변수의 위치와 이름 등을 확인할 수 있고, 각각의 세그먼트 크기 등의 데이터를 알 수 있다. 실제 CPU을 다루는 C 언어에서 이런 정보는 중요하다. 내가 사용하는 MCU의 메모리는 얼마나 사용하는지 등을 확인할 필요가 있기 때문이다.</p>
<h2><span class="mw-headline" id=".EB.9D.BC.EC.9D.B4.EB.B8.8C.EB.9F.AC.EB.A6.AC"><a class="mw-disambig" href="/wiki/%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="라이브러리">라이브러리</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=14" title="부분 편집: 라이브러리">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>C 언어 함수는 표준함수가 있고, 개발 툴에서 제공하는 함수가 있다. 여러가지 부류가 있고 특성 별로 나누어 lib 파일로 코드를 제공하고 헤더파일로 선언을 알 수 있다.</p>
<h3><span class="mw-headline" id="C_.ED.91.9C.EC.A4.80_.EB.9D.BC.EC.9D.B4.EB.B8.8C.EB.9F.AC.EB.A6.AC">C 표준 라이브러리</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=15" title="부분 편집: C 표준 라이브러리">편집</a><span class="mw-editsection-bracket">]</span></span></h3>

<h2><span class="mw-headline" id=".EA.B0.9C.EB.B0.9C_.EB.8F.84.EA.B5.AC">개발 도구</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=17" title="부분 편집: 개발 도구">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="GCC"><a class="mw-disambig" href="/wiki/GCC" title="GCC">GCC</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=18" title="부분 편집: GCC">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink boilerplate seealso"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> <a href="/wiki/GNU_%EC%BB%B4%ED%8C%8C%EC%9D%BC%EB%9F%AC_%EB%AA%A8%EC%9D%8C" title="GNU 컴파일러 모음">GNU 컴파일러 모음</a> 문서를 참고하십시오.</div>
<p><a href="/wiki/%EC%9C%A0%EB%8B%89%EC%8A%A4_%EA%B3%84%EC%97%B4" title="유닉스 계열">유닉스 계열</a>(리눅스)의 시스템에서 주로 사용하는 C/C++ 언어 개발 도구이다. 리눅스의 OS을 제 컴파일하거나, 각종 응용 프로그램 개발에 사용한다. 또한 X-Windows의 개발 도구로도 사용 할 수 있다.</p>
<p>전자 장치의 개발 시 임베디드 OS 포팅에서, 리눅스 커널이나 리눅스 커널 기반으로 하는 OS 커널 자체를 개발하는 도구로 사용 한다. 리눅스 커널 기반 임베디드에서 실행되는 응용 프로그램 역시 <a class="mw-redirect mw-disambig" href="/wiki/Gcc" title="Gcc">gcc</a>을 많이 사용 한다.</p>
<h4><span class="mw-headline" id="make"><a href="/wiki/Make_(%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4)" title="Make (소프트웨어)">make</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=19" title="부분 편집: make">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a href="/wiki/Make_(%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4)" title="Make (소프트웨어)">make (소프트웨어)</a>입니다.</div>
<p>여러 파일들끼리의 의존성과 각 파일에 필요한 명령을 정의함으로써 프로그램을 컴파일할 수 있으며 최종 프로그램을 만들 수 있는 과정을 서술할 수 있는 표준적인 문법을 가지고 있고, 구조로 기술된 파일(주로 Makefile이라는 파일명)을 [make]가 해석하여 프로그램 빌드를 수행하게 된다.</p>
<h4><span class="mw-headline" id="Cygwin">Cygwin</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=20" title="부분 편집: Cygwin">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a href="/wiki/%EC%8B%9C%EA%B7%B8%EC%9C%88" title="시그윈">시그윈</a>입니다.</div>
<p><a class="mw-redirect mw-disambig" href="/wiki/Gcc" title="Gcc">gcc</a>을 <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">윈도</a>에서 실행 할 수 있도록 재 포팅한 것이다.</p>
<h4><span class="mw-headline" id="MinGW">MinGW</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=21" title="부분 편집: MinGW">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Cygwin에서 분화 된 gcc 기반 개발 라이브러리 이다.</p>
<h3><span class="mw-headline" id=".EC.9D.B4.ED.81.B4.EB.A6.BD.EC.8A.A4"><a class="mw-disambig" href="/wiki/%EC%9D%B4%ED%81%B4%EB%A6%BD%EC%8A%A4" title="이클립스">이클립스</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=22" title="부분 편집: 이클립스">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink boilerplate seealso"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> <a class="mw-disambig" href="/wiki/%EC%9D%B4%ED%81%B4%EB%A6%BD%EC%8A%A4" title="이클립스">이클립스</a> 문서를 참고하십시오.</div>
<p>이클립스는 다양한 언어와 다양한 <a class="mw-disambig" href="/wiki/OS" title="OS">OS</a>에서 실행 되는 <a class="mw-disambig" href="/wiki/IDE" title="IDE">IDE</a> 이다. 따라서 여러가지 상황에서 다양하게 적용할 수 있다.</p>
<h4><span class="mw-headline" id="Eclipse_IDE_for_C.2FC.2B.2B_Developers">Eclipse IDE for C/C++ Developers</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=23" title="부분 편집: Eclipse IDE for C/C++ Developers">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>C/C++언어를 제공하는 <a class="mw-disambig" href="/wiki/IDE" title="IDE">IDE</a>으로 리눅스의 경우 기존의 <a class="mw-redirect mw-disambig" href="/wiki/Gcc" title="Gcc">gcc</a>을 사용할 수 있도록 연결 설정만 하면 된다.</p>
<dl>
<dt><a class="external text" href="http://www.mingw.org/" rel="nofollow">MinGW</a></dt>
</dl>
<p><a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">윈도</a>에서 <a class="mw-redirect mw-disambig" href="/wiki/Gcc" title="Gcc">gcc</a>와 연결하여 C/C++ 언어를 사용하여 프로그램을 개발 할 수 있다. MinGW는 다양한 언어를 지원하므로 다른 언어로도 이클립스와 연결하여 개발 도구로 사용할 수 있다.</p>
<h3><span class="mw-headline" id=".EB.B9.84.EC.A3.BC.EC.96.BC_C.2B.2B.28.EB.B9.84.EC.A3.BC.EC.96.BC_.EC.8A.A4.ED.8A.9C.EB.94.94.EC.98.A4_C.2B.2B.29"><a href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_C%2B%2B" title="비주얼 C++">비주얼 C++</a>(비주얼 스튜디오 C++)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=24" title="부분 편집: 비주얼 C++(비주얼 스튜디오 C++)">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>비주얼 프로그래밍은 크게 윈도 프로그램과 윈도가 아닌 프로그램으로 나누어 생각할 수 있다. 초기의 마이크로소프트의 개발 도구는 C 언어로 부터 출발 하였다. 오브젝트 프로그램의 필요성이 대두되면서 C++가 추가 되었다.</p>
<p>윈도 프로그램은 초기에 Win32 SDK을 기반으로 하였다. 오브젝트의 개념없이 함수를 사용하여 윈도 프로그램을 작성 한다. 후에 C++로 가면서 <a class="mw-disambig" href="/wiki/MFC" title="MFC">MFC</a>로 진화 하였다.</p>
<h4><span class="mw-headline" id="Win32_Console_Application">Win32 Console Application</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=25" title="부분 편집: Win32 Console Application">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>C/C++의 라이브러리를 이용하여 윈도 없이 실행되는 프로그램을 말 한다. 비주얼 프로그램에서는 C와 C++가 통합 되어 있으므로 <a href="/wiki/C_%ED%91%9C%EC%A4%80_%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="C 표준 라이브러리">C 표준 라이브러리</a> 뿐만 아니라 <a class="mw-disambig" href="/wiki/STL" title="STL">STL</a> 등도 사용할 수 있다.</p>
<h4><span class="mw-headline" id="Win32_SDK">Win32 SDK</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=26" title="부분 편집: Win32 SDK">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>SDK(Software Development Kit)라는 말은 일반적인 소프트웨어 개발 도구를 나타내는 말이지만 마이크로소프트의 윈도 개발에서는 클래스화 되지 않는 C 라이브러리를 윈도 SDK라고 부르기도 했다. 윈도 개발 도구를 만들 초기에 붙인 이름이다.</p>
<p>이것은 Visual Studio에 통합되어 C 언어의 라이브러리 형태로 개발할 수 있다.</p>
<h4><span class="mw-headline" id="MFC"><a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%ED%8C%8C%EC%9A%B4%EB%8D%B0%EC%9D%B4%EC%85%98_%ED%81%B4%EB%9E%98%EC%8A%A4_%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="마이크로소프트 파운데이션 클래스 라이브러리">MFC</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=27" title="부분 편집: MFC">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%ED%8C%8C%EC%9A%B4%EB%8D%B0%EC%9D%B4%EC%85%98_%ED%81%B4%EB%9E%98%EC%8A%A4_%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="마이크로소프트 파운데이션 클래스 라이브러리">마이크로소프트 파운데이션 클래스 라이브러리</a>입니다.</div>
<p>윈도 개발에 필요한 각종 윈도 요소를 클래스화 하였다. Win32 SDK도 같이 사용할 수 있다.</p>
<h4><span class="mw-headline" id="DirectX"><a href="/wiki/DirectX" title="DirectX">DirectX</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=28" title="부분 편집: DirectX">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>윈도에서 주로 게임 등을 개발할 때 사용하는 툴킷 이다. 고속의 화면 제어, 음성지원, 3D 등을 지원한다.</p>
<h2><span class="mw-headline" id=".EB.94.94.EB.B2.84.EA.B9.85">디버깅</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=29" title="부분 편집: 디버깅">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>보통 소프트웨어 개발에서 디버깅의 가장 일반적인 방법은 두가지 이다.</p>
<ol>
<li>IDE를 통해 소스 수준에서 디버깅 한다. 일명 'BREAK POINT'를 설정하면 그 위치에서 멈추고 해당 상황을 파악할 수 있다. 가장 중요한 파악 대상이 변수의 값으로 표시 화면에 해당 변수의 형과 맞게 변수의 값을 표시 한다. 또한 줄단위로 실행하기, 함수로 들어가기와 나오기 등 IDE에 따라 구성이 다양하다.</li>
<li>내부 변수나 기타 상황을 printf 형태의 함수를 UART, 네트워크, USB 등과 연결하여 출력함으로써 프로그램 상황을 표시 할 수 있다. 보통 IDE 처럼 다양한 구성이 불가능 할 때 많이 사용 한다.
<ol>
<li>리눅스 커널의 경우 printfk 함수로 출력하면 정해진 출력 메시지 파일태로 표시할 수 있다.</li>
<li><a class="mw-disambig" href="/wiki/MCU" title="MCU">MCU</a>와 같은 전자장치의 개발에서 개인용 컴퓨터 처럼 성능이 좋은 IDE을 만들기 쉽지 않다. 따라서 저 성능의 IDE을 사용하거나 UART등으로 상황을 출력해서 디버깅 하는 방법을 사용 할 수 있다. 구성이 간단해서 개발 소스에 printf 형의 메시지 출력을 추가하면 된다. 개발자가 직접 해야하는 것이 번거롭다.</li>
</ol>
</li>
</ol>
<h3><span class="mw-headline" id="GDB"><a class="mw-redirect" href="/wiki/GDB" title="GDB">GDB</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=30" title="부분 편집: GDB">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a href="/wiki/GNU_%EB%94%94%EB%B2%84%EA%B1%B0" title="GNU 디버거">GNU 디버거</a>입니다.</div>
<p><a class="mw-disambig" href="/wiki/GCC" title="GCC">GCC</a>을 기반으로 하는 <a class="mw-redirect" href="/wiki/%EB%94%94%EB%B2%84%EA%B9%85" title="디버깅">디버깅</a> 도구이다. 따라서 <a href="/wiki/%EC%9C%A0%EB%8B%89%EC%8A%A4" title="유닉스">유닉스</a> 계열에서 가장 일반적으로 실행된다.</p>
<h4><span class="mw-headline" id=".EC.9D.91.EC.9A.A9_.ED.94.84.EB.A1.9C.EA.B7.B8.EB.9E.A8_.EB.94.94.EB.B2.84.EA.B9.85">응용 프로그램 디버깅</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=31" title="부분 편집: 응용 프로그램 디버깅">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>GCC 옵션을 디버깅이 되도록 설정하면 디버깅 테이블을 만든다. gdb 실행 중에 이것을 사용한다. GDB을 실행하여 응용 프로그램을 실행하면서 break, 변수, 함수 등의 디버깅을 할 수 있다.</p>
<h4><span class="mw-headline" id=".EC.9B.90.EA.B2.A9_.EB.94.94.EB.B2.84.EA.B9.85">원격 디버깅</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=32" title="부분 편집: 원격 디버깅">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>GCC에서 <a class="mw-redirect" href="/wiki/Gdb" title="Gdb">gdb</a>는 서버 구조를 사용할 수 있다. <a class="new" href="/w/index.php?title=Gdb-server&amp;action=edit&amp;redlink=1" title="Gdb-server (없는 문서)">gdb-server</a>을 설치하면 <a class="mw-disambig" href="/wiki/%EB%84%A4%ED%8A%B8%EC%9B%8C%ED%81%AC" title="네트워크">네트워크</a>를 통해 디버깅 환경을 구성할 수 있다. 예를 들어 임베디드 개발 시 리눅스 커널을 포팅하고, 해당 리눅스 시스템에 gdb-server를 설치하면 다른 환경에서 이를 통해 응용 프로그램을 디버깅 할 수 있다. 임베디드의 많은 경우 자신의 시스템에서는 디버깅이 만만치 않다. 따라서 원격으로 gdb의 실행 결과를 전송 할 수 있고 이 정보를 바탕으로 이클립스와 같은 IDE와 연동할 수 있다. 보통 리눅스 기반의 임베디드 개발 환경은 이클립스 C++를 사용할 수 있는데, 이것과 결합할 수 있다.</p>
<h4><span class="mw-headline" id=".EC.BB.A4.EB.84.90_.EB.94.94.EB.B2.84.EA.B9.85">커널 디버깅</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=33" title="부분 편집: 커널 디버깅">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>원격 디버깅 모드는 리눅스 커널에 사용되는 소스 수준의 디버거인 KGDB에서도 사용된다. KGDB를 사용하면 커널 개발자는 일반 응용 프로그램과 마찬가지로 커널을 디버깅할 수 있다.</p>
<h3><span class="mw-headline" id="IDE_.EB.94.94.EB.B2.84.EA.B9.85">IDE 디버깅</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=34" title="부분 편집: IDE 디버깅">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a class="mw-redirect" href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EC%8A%A4%ED%8A%9C%EB%94%94%EC%98%A4" title="비주얼 스튜디오">비주얼 스튜디오</a>나 <a class="mw-disambig" href="/wiki/%EC%9D%B4%ED%81%B4%EB%A6%BD%EC%8A%A4" title="이클립스">이클립스</a> 등의 도구 들은 기본적으로 디버깅 방법을 제시 한다. 이클립스 디버깅은 <a class="mw-redirect" href="/wiki/GDB" title="GDB">GDB</a>와 연동해서 구성할 수 있다.</p>
<h2><span class="mw-headline" id="Hello_World_.ED.94.84.EB.A1.9C.EA.B7.B8.EB.9E.A8"><a class="mw-redirect" href="/wiki/Hello_World_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%A8" title="Hello World 프로그램">Hello World 프로그램</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)&amp;action=edit&amp;section=35" title="부분 편집: Hello World 프로그램">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="cp">#include</span> <span class="cpf">&lt;stdio.h&gt; // C 표준 라이브러리 stdio.h 를 include 한다. 이것으로 printf 함수등을 사용할 수 있다.</span>

<span class="kt">int</span> <span class="nf">main</span><span class="p">()</span> <span class="c1">// 프로그램 호출시 가장 먼저실행되는 함수</span>
<span class="p">{</span>
    <span class="n">printf</span><span class="p">(</span><span class="s">"hello, world</span><span class="se">\n</span><span class="s">"</span><span class="p">);</span> <span class="c1">// 표준출력에 hello, world라는 단어를 출력하고 \n 에 의해 개행하게 된다.</span>
    <span class="k">return</span> <span class="mi">0</span><span class="p">;</span>                 <span class="c1">// 0을 반환하고 main() 종료.</span>
<span class="p">}</span>
</pre></div>
<p>이 프로그램은 표준출력(stdout)으로 <tt>hello, world</tt>를 출력한다.</p>
