<p><b>gzip</b>은 <a class="mw-redirect" href="/wiki/%ED%8C%8C%EC%9D%BC_%EC%95%95%EC%B6%95" title="파일 압축">파일 압축</a>에 쓰이는 <a href="/wiki/%EC%9D%91%EC%9A%A9_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="응용 소프트웨어">응용 소프트웨어</a>이다. gzip은 GNU zip의 준말이며, 초기 <a href="/wiki/%EC%9C%A0%EB%8B%89%EC%8A%A4" title="유닉스">유닉스</a> 시스템에 쓰이던 <a href="/wiki/%EB%8D%B0%EC%9D%B4%ED%84%B0_%EC%95%95%EC%B6%95" title="데이터 압축">압축</a> 프로그램을 대체하기 위한 <a href="/wiki/%EC%9E%90%EC%9C%A0_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="자유 소프트웨어">자유 소프트웨어</a>이다. gzip은 <a class="new" href="/w/index.php?title=Jean-loup_Gailly&amp;action=edit&amp;redlink=1" title="Jean-loup Gailly (없는 문서)">Jean-loup Gailly</a>와 <a class="new" href="/w/index.php?title=%EB%A7%88%ED%81%AC_%EC%95%A0%EB%93%A4%EB%9F%AC&amp;action=edit&amp;redlink=1" title="마크 애들러 (없는 문서)">마크 애들러</a>가 만들었다. 버전 0.1은 <a href="/wiki/1992%EB%85%84" title="1992년">1992년</a> <a href="/wiki/10%EC%9B%94_31%EC%9D%BC" title="10월 31일">10월 31일</a>에 처음 공개되었으며, 버전 1.0이 1993년 2월에 뒤따라 나왔다. <a class="mw-redirect" href="/wiki/%EC%98%A4%ED%94%88BSD" title="오픈BSD">오픈BSD</a>의 gzip 버전은 더 오래된 <a href="/wiki/%EB%8D%B0%EC%9D%B4%ED%84%B0_%EC%95%95%EC%B6%95" title="데이터 압축">압축</a> 프로그램을 기반으로 하고 있으며, 오픈BSD 3.4에 추가되었다.</p>

<h2><span class="mw-headline" id=".ED.8C.8C.EC.9D.BC_.ED.8F.AC.EB.A7.B7">파일 포맷</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gzip&amp;action=edit&amp;section=1" title="부분 편집: 파일 포맷">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>gzip은 <a href="/wiki/ZIP_(%ED%8C%8C%EC%9D%BC_%ED%8F%AC%EB%A7%B7)" title="ZIP (파일 포맷)">ZIP (파일 포맷)</a>과 같이 <a href="/wiki/DEFLATE" title="DEFLATE">DEFLATE</a> 알고리즘을 따르지만, 여러 파일을 하나의 파일로 압축하는 옵션이 없다는 점에서 차이가 난다. 여러 파일 또는 디렉터리를 하나의 파일로 압축하기 위해서 gzip은 보통 <a href="/wiki/Tar_(%ED%8C%8C%EC%9D%BC_%ED%8F%AC%EB%A7%B7)" title="Tar (파일 포맷)">Tar (파일 포맷)</a>와 같이 사용되는 것이 일반적이다. .tar.gz 로 압축된 파일의 경우 zip과 압축 알고리즘은 같지만 더 용량이 작다. 이는 .tar.gz의 경우 서로 다른 파일끼리의 중복되는 부분을 압축시킬 수 있기 때문이다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<h2><span class="mw-headline" id=".EC.9D.91.EC.9A.A9.EA.B3.BC_.ED.8C.8C.EC.83.9D">응용과 파생</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Gzip&amp;action=edit&amp;section=2" title="부분 편집: 응용과 파생">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>대부분의 리눅스 배포판에 포함되어 있는 tar 유틸리티는 .tar.gz 파일을 z 옵션으로 압축을 풀 수 있다. <i>e.g., tar -zxf file.tar.gz</i></p>
<p>1990년대 이후로 블록 소팅 알고리즘을 이용한 <a href="/wiki/Bzip2" title="Bzip2">Bzip2</a>와 같은 더 발전된 포맷이 gzip을 대체하는 경우가 있다.</p>
