<h1 class="firstHeading" id="firstHeading" lang="ko">COM 파일</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">위키백과, 우리 모두의 백과사전.</div>
<div id="contentSub"><span class="mw-redirectedfrom">(<a class="mw-redirect" href="/w/index.php?title=.COM&amp;redirect=no" title=".COM">.COM</a>에서 넘어옴)</span></div>
<div class="mw-jump" id="jump-to-nav">
					이동:					<a href="#mw-head">둘러보기</a>, 					<a href="#p-search">검색</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%99%95%EC%9E%A5%EC%9E%90" title="파일 확장자">파일 확장자</a> <b>.com</b>은 다양한 컴퓨터 시스템에서 다른 목적으로 사용되어 왔다. 초기에, 이 용어는 "명령 파일"을 대표하는 말이었으며 운영 체제에 내보낼 명령어를 포함하는 텍스트 파일이었다. 1970년대로 거슬러 올라가서, <a href="/wiki/%EB%94%94%EC%A7%80%ED%84%B8_%EC%9D%B4%ED%81%85%EB%A8%BC%ED%8A%B8_%EC%BD%94%ED%8D%BC%EB%A0%88%EC%9D%B4%EC%85%98" title="디지털 이큅먼트 코퍼레이션">디지털 이큅먼트 코퍼레이션</a> <a href="/wiki/%EB%AF%B8%EB%8B%88%EC%BB%B4%ED%93%A8%ED%84%B0" title="미니컴퓨터">미니컴퓨터</a> 및 <a href="/wiki/%EB%A9%94%EC%9D%B8%ED%94%84%EB%A0%88%EC%9E%84" title="메인프레임">메인프레임</a> 컴퓨터 시스템에 실용적으로 쓰이게 되었다.</p>
<p><a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%BB%B4%ED%93%A8%ED%84%B0" title="마이크로컴퓨터">마이크로컴퓨터</a>의 도입과 더불어, 확장자 .com으로 끝나는 파일의 사용은 점차 바뀌어 갔다. <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a> 호환 도스와 8 비트 <a href="/wiki/CP/M" title="CP/M">CP/M</a>에서, <b>COM 파일</b>은 단순히 <a href="/wiki/%EC%8B%A4%ED%96%89_%ED%8C%8C%EC%9D%BC" title="실행 파일">실행 파일</a>의 종류이다.</p>
<p>이 파일 형식의 이름은 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%99%95%EC%9E%A5%EC%9E%90" title="파일 확장자">파일 확장자</a> .com에서 나온 것이며(최상위 도메인 <a href="/wiki/.com" title=".com">.com</a>과 헷갈리지 말 것) 각 파일에 쓰이는 초기의 확장자였다. 그러나 CP/M과 매우 초기의 MS-DOS 버전을 제외하고는 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%98%95%EC%8B%9D" title="파일 형식">파일 형식</a>과 파일 확장자 사이의 실제적인 통합은 없었다.</p>

<h2><span class="mw-headline" id=".ED.94.8C.EB.9E.AB.ED.8F.BC_.EC.A7.80.EC.9B.90">플랫폼 지원</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=COM_%ED%8C%8C%EC%9D%BC&amp;action=edit&amp;section=1" title="부분 편집: 플랫폼 지원">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>이 포맷은 여전히 현대의 <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">윈도</a> 기반의 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%8C%85_%ED%94%8C%EB%9E%AB%ED%8F%BC" title="컴퓨팅 플랫폼">플랫폼</a>의 <a href="/wiki/%EC%8B%A4%ED%96%89_%ED%8C%8C%EC%9D%BC" title="실행 파일">실행 파일</a>이지만 MS-DOS 가상 구현 체계에서 돌아간다. x64 계열의 운영 체제에서는 이러한 가상 구현 체계가 없으므로 실행되지 않는다. (역자 주: 가상 구현에 대해서는 <a class="mw-redirect" href="/wiki/NTVDM" title="NTVDM">NTVDM</a> 참조)</p>
<p>COM 파일은 또한 <a href="/wiki/%EB%8F%84%EC%8A%A4%EB%B0%95%EC%8A%A4" title="도스박스">도스박스</a>와 같은 도스 에뮬레이터를 사용하여 실행할 수 있다. 이 에뮬레이터는 모든 플랫폼에서 지원한다. "COM"은 "core image"(코어 이미지)의 준말이며, 이러한 .com 파일이 컴퓨터에서 실행할 수 있는 기본 명령어에 대한 코드를 포함했기 때문에 "command"(명령어)로 해석되기도 한다.</p>
<h2><span class="mw-headline" id=".EC.9D.B4.EC.A7.84_.ED.8F.AC.EB.A7.B7">이진 포맷</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=COM_%ED%8C%8C%EC%9D%BC&amp;action=edit&amp;section=2" title="부분 편집: 이진 포맷">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>COM 포맷은 가장 단순한 실행 포맷이며, <a href="/wiki/%EB%A9%94%ED%83%80%EB%8D%B0%EC%9D%B4%ED%84%B0" title="메타데이터">메타데이터</a>는 없고, 코드와 데이터만 포함하고 있으며 일부 세그먼트의 오프셋 <a href="/wiki/%EC%8B%AD%EC%9C%A1%EC%A7%84%EB%B2%95" title="십육진법">0x</a>0100에서 <a href="/wiki/%EB%A1%9C%EB%8D%94_(%EC%BB%B4%ED%93%A8%ED%8C%85)" title="로더 (컴퓨팅)">로드</a>되어 실행된다. <a class="new" href="/w/index.php?title=X86_%EB%A9%94%EB%AA%A8%EB%A6%AC_%EC%84%B8%EA%B7%B8%EB%A8%BC%ED%8A%B8&amp;action=edit&amp;redlink=1" title="X86 메모리 세그먼트 (없는 문서)">세그멘테이션</a> 모델이 잘 동작하기 때문에 <a class="new" href="/w/index.php?title=%EB%A6%B4%EB%A1%9C%EC%BC%80%EC%9D%B4%EC%85%98&amp;action=edit&amp;redlink=1" title="릴로케이션 (없는 문서)">구조 배치</a>가 필요하지 않다. 파일 크기가 파일 시스템의 최대 용량을 지원하는 <a href="/wiki/EXE" title="EXE">EXE</a> 파일과는 달리 COM의 최대 지원 용량은 65536바이트(64<a href="/wiki/%ED%82%AC%EB%A1%9C%EB%B0%94%EC%9D%B4%ED%8A%B8" title="킬로바이트">킬로바이트</a>)이다.</p>
<h2><span class="mw-headline" id=".EC.8B.A4.ED.96.89_.EC.9A.B0.EC.84.A0.EB.8F.84">실행 우선도</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=COM_%ED%8C%8C%EC%9D%BC&amp;action=edit&amp;section=3" title="부분 편집: 실행 우선도">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>실행 파일로는 EXE, COM, BAT 등이 있다. 이를테면, 한 디렉터리가 COM 파일과 EXE 파일이 확장자만 다르고 같은 이름을 가지고 있다고 하면 이 경우 COM 파일이 우선하여 실행된다.</p>
