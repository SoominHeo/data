<p><b>Flash Video</b> is a <a class="mw-redirect" href="/wiki/Container_format_(digital)" title="Container format (digital)">container file format</a> used to deliver <a href="/wiki/Digital_video" title="Digital video">digital video</a> content (e.g., <a class="mw-redirect" href="/wiki/TV_show" title="TV show">TV shows</a>, <a class="mw-redirect" href="/wiki/Movie" title="Movie">movies</a>, etc.) over the <a href="/wiki/Internet" title="Internet">Internet</a> using <a href="/wiki/Adobe_Flash_Player" title="Adobe Flash Player">Adobe Flash Player</a> version 6 and newer. Flash Video content may also be embedded within <a href="/wiki/SWF" title="SWF">SWF</a> files. There are two different video file formats known as Flash Video: <b>FLV</b> and <b>F4V</b>. The audio and video data within FLV files are encoded in the same manner as they are within SWF files. The F4V file format is based on the <a href="/wiki/ISO_base_media_file_format" title="ISO base media file format">ISO base media file format</a> and is starting with Flash Player 9 update 3.<sup class="reference" id="cite_ref-FLV-F4V_1-0"><a href="#cite_note-FLV-F4V-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> Both formats are supported in Adobe Flash Player and developed by Adobe Systems. FLV was originally developed by <a href="/wiki/Macromedia" title="Macromedia">Macromedia</a>. In the early 2000s, Flash Video used to be the <i>de facto</i> standard for web-based streaming video (over <a class="mw-redirect" href="/wiki/Real_Time_Messaging_Protocol" title="Real Time Messaging Protocol">RTMP</a>). Notable users of it include <a href="/wiki/Hulu" title="Hulu">Hulu</a>, <a class="mw-redirect" href="/wiki/VEVO" title="VEVO">VEVO</a>, <a class="mw-redirect" href="/wiki/Yahoo!_Video" title="Yahoo! Video">Yahoo! Video</a>, <a href="/wiki/Metacafe" title="Metacafe">metacafe</a>, <a href="/wiki/Reuters" title="Reuters">Reuters.com</a>, and many other news providers.</p>
<p>Flash Video FLV files usually contain material encoded with <a href="/wiki/Codec" title="Codec">codecs</a> following the <a class="mw-redirect" href="/wiki/Sorenson_Spark" title="Sorenson Spark">Sorenson Spark</a> or <a href="/wiki/VP6" title="VP6">VP6</a> <a class="mw-redirect" href="/wiki/Video_compression_format" title="Video compression format">video compression formats</a>. The most recent public releases of Flash Player (collaboration between <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe Systems</a> and <a href="/wiki/MainConcept" title="MainConcept">MainConcept</a>) also support <a class="mw-redirect" href="/wiki/H.264" title="H.264">H.264</a> video and <a class="mw-redirect" href="/wiki/HE-AAC" title="HE-AAC">HE-AAC</a> audio.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> All of these compression formats are restricted by patents. Flash Video is viewable on most <a class="mw-redirect" href="/wiki/Operating_systems" title="Operating systems">operating systems</a> via the Adobe Flash Player and <a href="/wiki/Web_browser" title="Web browser">web browser</a> <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plugin</a> or one of several third-party programs. Apple's <a class="mw-redirect" href="/wiki/IOS_(Apple)" title="IOS (Apple)">iOS</a> devices, along with almost all other mobile devices, do not support the Flash Player plugin and so require other delivery methods such as provided by the <a href="/wiki/Adobe_Flash_Media_Server" title="Adobe Flash Media Server">Adobe Flash Media Server</a>.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flash_Video&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Support for video in SWF file format was added in Flash Player 6, released in 2002. In 2003, Flash Player 7 added direct support for FLV file format. Because of restrictions in the FLV file format, Adobe Systems has created in 2007 new file formats listed below, based on the <a href="/wiki/ISO_base_media_file_format" title="ISO base media file format">ISO base media file format</a> (MPEG-4 Part 12). Flash Player does not check the extension of the file, but rather looks inside the file to detect which format it is.<sup class="reference" id="cite_ref-MPEG4_5-0"><a href="#cite_note-MPEG4-5">[5]</a></sup><sup class="reference" id="cite_ref-kaourantin_6-0"><a href="#cite_note-kaourantin-6">[6]</a></sup> The new file formats are completely different from the older FLV file format. For example, F4V does not support Screen video, Sorenson Spark, VP6 video compression formats and ADPCM, Nellymoser audio compression formats.<sup class="reference" id="cite_ref-FLV-F4V_1-1"><a href="#cite_note-FLV-F4V-1">[1]</a></sup><sup class="reference" id="cite_ref-kaourantin_6-1"><a href="#cite_note-kaourantin-6">[6]</a></sup> Authors of Flash Player strongly encourage everyone to embrace the new standard file format F4V (ISO base media file format). There are functional limits with the FLV structure when streaming H.264 or AAC which could not be overcome without a redesign of the file format. This is one reason why Adobe Systems is moving away from the traditional FLV file structure.<sup class="reference" id="cite_ref-kaourantin_6-2"><a href="#cite_note-kaourantin-6">[6]</a></sup> Initial format since 2002 is <i>Flash Video</i>, file suffix is <i>.flv</i> with a MIME derived <a class="mw-redirect" href="/wiki/Internet_media_type" title="Internet media type">Internet media type</a> of <i>video/x-flv</i>.</p>
<p>Extended from 2007 to support the <i><a href="/wiki/ISO_base_media_file_format" title="ISO base media file format">ISO base media file format</a></i>, Adobe branded file suffix is <i>.f4v</i> with the same MIME derived <a class="mw-redirect" href="/wiki/Internet_media_type" title="Internet media type">Internet media type</a> of <i>video/mp4</i> as the Apple file suffix of <i>.m4v</i> and the general file suffix of <i>.mp4</i>. Adobe branded file suffixes exist for <i>.f4p</i> which relates to media encrypted with their <i>Adobe Access</i> DRM scheme with <i>.f4a</i> and <i>.f4b</i> which relate respectively to <i>.m4a</i> and <i>.m4b</i> with the same MIME derived <a class="mw-redirect" href="/wiki/Internet_media_type" title="Internet media type">Internet media type</a> of <i>audio/mp4</i>. SWF files published for Flash Player 6 and later versions are able to exchange audio, video, and data over <a class="mw-redirect" href="/wiki/Real_Time_Messaging_Protocol" title="Real Time Messaging Protocol">RTMP</a> connections with the Adobe Flash Media Server. One way to feed data to Flash Media Server is from files in the FLV file format. Starting with SWF files created for Flash Player 7, Flash Player can play FLV file format directly (MIME type video/x-flv). Starting with SWF files created for Flash Player 9 Update 3, Flash Player can also play the new F4V file format.<sup class="reference" id="cite_ref-FLV-F4V_1-2"><a href="#cite_note-FLV-F4V-1">[1]</a></sup></p>

<li><span class="citation wikicite" id="endnote_h264.2A"><a href="#ref_h264.2A"></a></span> Use of the H.264 and AAC compression formats in the FLV file format has some limitations and authors of Flash Player strongly encourage everyone to embrace the new standard F4V file format.<sup class="reference" id="cite_ref-kaourantin_6-3"><a href="#cite_note-kaourantin-6">[6]</a></sup></li>
</ul>
<h2><span class="mw-headline" id="Encoding">Encoding</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flash_Video&amp;action=edit&amp;section=2" title="Edit section: Encoding">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Commonly, Flash Video FLV files contain video <a class="mw-redirect" href="/wiki/Bit_stream" title="Bit stream">bit streams</a> which are a proprietary variant of the <a href="/wiki/H.263" title="H.263">H.263</a> video standard,<sup class="reference" id="cite_ref-almost_10-0"><a href="#cite_note-almost-10">[10]</a></sup> under the name of <a class="mw-redirect" href="/wiki/Sorenson_codec" title="Sorenson codec">Sorenson Spark</a> (<a href="/wiki/FourCC" title="FourCC">FourCC</a> FLV1).<sup class="reference" id="cite_ref-incomplete-h263_11-0"><a href="#cite_note-incomplete-h263-11">[11]</a></sup><sup class="reference" id="cite_ref-multimediawiki-spark_12-0"><a href="#cite_note-multimediawiki-spark-12">[12]</a></sup> Sorenson Spark is an older codec for FLV files but it is also a widely available and compatible one, because it was the first video codec supported in Flash Player.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> It is the required video compression format for Flash Player 6 and 7.<sup class="reference" id="cite_ref-flash8-versions_14-0"><a href="#cite_note-flash8-versions-14">[14]</a></sup><sup class="reference" id="cite_ref-flash10-livedocs_15-0"><a href="#cite_note-flash10-livedocs-15">[15]</a></sup> Flash Player 8 and newer revisions also support the playback of <a class="mw-redirect" href="/wiki/On2" title="On2">On2</a> <a href="/wiki/VP6" title="VP6">TrueMotion VP6</a> video bit streams (FourCC VP6F or FLV4). On2 VP6 is the preferred video compression format for use with Flash Player 8 and higher.<sup class="reference" id="cite_ref-flash10_7-1"><a href="#cite_note-flash10-7">[7]</a></sup><sup class="reference" id="cite_ref-incomplete-h263_11-1"><a href="#cite_note-incomplete-h263-11">[11]</a></sup> On2 VP6 can provide a higher visual quality than Sorenson Spark, especially when using lower <a href="/wiki/Bit_rate" title="Bit rate">bit rates</a>. On the other hand, it is computationally more complex and therefore will not run as well on certain older system configurations.<sup class="reference" id="cite_ref-flash8-versions_14-1"><a href="#cite_note-flash8-versions-14">[14]</a></sup><sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></p>
<p>The Flash Video FLV file format supports two versions of a so-called 'screenshare' (Screen video) codec which is an encoding format designed for <a href="/wiki/Screencast" title="Screencast">screencasts</a>. Both these formats are bitmap tile based, can be <a class="mw-redirect" href="/wiki/Lossy" title="Lossy">lossy</a> by reducing <a href="/wiki/Color_depth" title="Color depth">color depths</a> and are compressed using <a href="/wiki/Zlib" title="Zlib">zlib</a>. The second version is only playable in Flash Player 8 and newer. Audio in Flash Video files is usually encoded as MP3. However, audio in Flash Video FLV files recorded from the user's microphone use the proprietary <a class="mw-redirect" href="/wiki/Nellymoser_Asao_Codec" title="Nellymoser Asao Codec">Nellymoser Asao Codec</a>.<sup class="reference" id="cite_ref-nelly_8-1"><a href="#cite_note-nelly-8">[8]</a></sup> (Flash Player 10 released in 2008 also supports the <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> <a href="/wiki/Speex" title="Speex">Speex</a> codec.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup>) FLV files also support uncompressed audio or <a class="mw-redirect" href="/wiki/ADPCM" title="ADPCM">ADPCM</a> format audio. Recent versions of Flash Player 9 support <a href="/wiki/Advanced_Audio_Coding" title="Advanced Audio Coding">AAC</a> (<a class="mw-redirect" href="/wiki/HE-AAC" title="HE-AAC">HE-AAC</a>/<a href="/wiki/Advanced_Audio_Coding" title="Advanced Audio Coding">AAC</a> <a href="/wiki/Spectral_band_replication" title="Spectral band replication">SBR</a>, AAC Main Profile, and AAC-LC). Support for encoding Flash Video files is provided by an encoding tool included with Adobe's Flash Professional and Creative Suite products, <a class="mw-redirect" href="/wiki/On2" title="On2">On2</a>'s Flix encoding tools, <a href="/wiki/Sorenson_Squeeze" title="Sorenson Squeeze">Sorenson Squeeze</a>, <a href="/wiki/FFmpeg" title="FFmpeg">FFmpeg</a> and other third party tools.</p>

<h3><span class="mw-headline" id="Packets">Packets</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flash_Video&amp;action=edit&amp;section=7" title="Edit section: Packets">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

<p>The <i>Packet Type</i> byte of a packet/tag header is based on the RTMP message ID byte with the AMF metadata value of 18 (0x12), video payload value of 9 (0x09) and audio payload value of 8 (0x08) being the only valid types used. The third bit indicates the payload is encrypted using the same mechanism as RTMP uses, however this is rarely used due to encrypted transports such as RTMP being used instead. The FLV packet encryption is generally inherited from a MP4 file that is stored on a <a href="/wiki/Adobe_Flash_Media_Server" title="Adobe Flash Media Server">Adobe Flash Media Server</a>.</p>
<ul>
<li>Packet types enumerated as 1 is a RTMP set packet size.</li>
<li>Packet types enumerated from 3 are RTMP bytes read report, RTMP ping, RTMP server bandwidth, RTMP client bandwidth.</li>
<li>Packet types enumerated from 8 are Audio payload, Video payload.</li>
<li>Packet types enumerated from 15 are RTMP flex stream send, RTMP flex shared object, RTMP flex message, AMF metadata, shared object, RTMP invoke.</li>
<li>Packet type enumerated as 24 is an encapsulated flash video.</li>
</ul>

FLV Tag Structure</div>

<p>Following that, there are three bytes for the <i>Payload Size</i> denoting length of the <i>Payload Data</i>, then four bytes for the <i>Timestamp</i> in milliseconds (with the last byte used to extend the first three bytes), the next 3 bytes for the <i>Stream ID</i> (incremented for multiple streams of the same type), and finally followed by the actual payload data. There is a direct relation between the fields encountered in an FLV Tag and those found in a <a class="mw-redirect" href="/wiki/Real_Time_Messaging_Protocol#Packet_structure" title="Real Time Messaging Protocol">RTMP packet</a>, as for example the FLV Packet Type field uses the same numeric values as the RTMP Chunk Type field (ex. 0x08 for audio and 0x09 for video). FLV tags are thus converted into RTMP packets when the file is streamed through a Flash Media Server or equivalent RTMP Server.</p>
<p>The first packet encountered is usually a metadata packet which contains information such as:</p>
<ul>
<li>"duration" - 64-bit IEEE floating point value in seconds</li>
<li>"width" and "height" – 64-bit IEEE floating point value in pixels</li>
<li>"framerate" – 64-bit IEEE floating point value in frames per second</li>
<li>"keyframes" – an array with the positions of <a class="mw-redirect" href="/wiki/Inter_Frame" title="Inter Frame">p-frames</a>, needed when random access is sought.</li>
<li>"|AdditionalHeader" - an array of required stream decoding informational pairs
<ul>
<li>"Encryption" - an array of required encryption informational pairs</li>
<li>"Metadata" - <a href="/wiki/Base64" title="Base64">Base64</a> encoded string of a signed <a href="/wiki/X.509" title="X.509">X.509</a> certificate containing the <i>Adobe Access</i> AES decryption key required</li>
</ul>
</li>
</ul>
<p>When streamed using an <a class="mw-redirect" href="/wiki/Actionscript" title="Actionscript">Actionscript</a> built player, the <a href="/wiki/Metadata" title="Metadata">metadata</a> values above are passed as arguments on the onMetaData callback function. Audio packets have the first byte of the payload defining the decoding details with the first four bits for the encoding used and the last four bits for the parameters required to process the encoding. Video packets have this order reversed.</p>

<h2><span class="mw-headline" id="FLV_players">FLV players</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flash_Video&amp;action=edit&amp;section=8" title="Edit section: FLV players">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>An FLV player is a type of media player that is used for playing Flash video from PC as well as from Internet websites. An FLV player can be used standalone, without the need of the Adobe Flash authoring or developmental tools. It can also be embedded in the website using Flash component or embeddable version of FLV player.</p>
<h3><span class="mw-headline" id="Flash_Player">Flash Player</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flash_Video&amp;action=edit&amp;section=9" title="Edit section: Flash Player">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Adobe_Flash_Player" title="Adobe Flash Player">Adobe Flash Player</a></div>
<p><b>Adobe Flash Player</b> is a multimedia and application player originally developed by <a href="/wiki/Macromedia" title="Macromedia">Macromedia</a> and acquired by <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe Systems</a>. It plays <a href="/wiki/SWF" title="SWF">SWF</a> files, which can be created by <a class="mw-redirect" href="/wiki/Adobe_Flash_Professional" title="Adobe Flash Professional">Adobe Flash Professional</a>, <a href="/wiki/Apache_Flex" title="Apache Flex">Apache Flex</a>, or a number of other <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe Systems</a> and 3rd party tools. It has support for a scripting language called <a href="/wiki/ActionScript" title="ActionScript">ActionScript</a>, which can be used to display Flash Video from an SWF file. Because the Flash Player runs as a <a href="/wiki/Web_browser" title="Web browser">browser</a> <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plug-in</a>, it is possible to embed Flash Video in web pages and view the video within a web browser. Flash Player supported display of Flash Video files since version 6, with the <a class="mw-redirect" href="/wiki/Sorenson_Spark" title="Sorenson Spark">Sorenson Spark</a> and On <a href="/wiki/VP6" title="VP6">VP6</a> video codecs. Support was recently added for <a class="mw-redirect" href="/wiki/H.264" title="H.264">H.264</a> video content as well.</p>
<h4><span class="mw-headline" id="H.264">H.264</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flash_Video&amp;action=edit&amp;section=10" title="Edit section: H.264">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Flash Player 9 Update 3, released on 3 December 2007,<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> also includes support for the new Flash Video file format F4V and <a class="mw-redirect" href="/wiki/H.264" title="H.264">H.264</a> video standard (also known as MPEG-4 part 10, or AVC) which is even more computationally demanding, but offers significantly better quality/bitrate ratio.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> Specifically, Flash Player now supports video compressed in H.264 (MPEG-4 Part 10), audio compressed using <a href="/wiki/Advanced_Audio_Coding" title="Advanced Audio Coding">AAC</a> (MPEG-4 Part 3), the F4V, MP4 (MPEG-4 Part 14), M4V, M4A, <a class="mw-redirect" href="/wiki/3GP" title="3GP">3GP</a> and <a href="/wiki/QuickTime_File_Format" title="QuickTime File Format">MOV</a> multimedia <a class="mw-redirect" href="/wiki/Container_format_(digital)" title="Container format (digital)">container formats</a>, <a class="mw-redirect" href="/wiki/3GPP_Timed_Text" title="3GPP Timed Text">3GPP Timed Text</a> specification (MPEG-4 Part 17) which is a standardized subtitle format and partial parsing support for the 'ilst' atom which is the <a href="/wiki/ID3" title="ID3">ID3</a> equivalent <a href="/wiki/ITunes" title="ITunes">iTunes</a> uses to store <a href="/wiki/Metadata" title="Metadata">metadata</a>. <a href="/wiki/MPEG-4_Part_2" title="MPEG-4 Part 2">MPEG-4 Part 2</a> video (e.g. created with DivX or Xvid) is not supported.<sup class="reference" id="cite_ref-MPEG4_5-1"><a href="#cite_note-MPEG4-5">[5]</a></sup> In an interview with <a href="/wiki/BBC_News" title="BBC News">BBC News</a>, the main programmer of Flash <a href="/wiki/Jonathan_Gay" title="Jonathan Gay">Jonathan Gay</a> said that the company had wanted to use H.264 when video support was originally added to Flash, but had been deterred by the patent licensing fees of around $5 million (£3.5 million) per year.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup></p>
<h4><span class="mw-headline" id="Playback">Playback</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flash_Video&amp;action=edit&amp;section=11" title="Edit section: Playback">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Flash Player supports two distinct modes of video playback:</p>
<ul>
<li><b>Software Rendered Video</b> : Flash Player supports software rendered video since version 6. Such video supports vector animations displayed above the video content. Such content is typically rendered using software decoding.<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup></li>
</ul>
<ul>
<li><b>Hardware Accelerated Video</b> : Flash Player supports hardware accelerated video playback since version 10.2, for <a class="mw-redirect" href="/wiki/H.264" title="H.264">H.264</a>, F4V, and <a class="mw-redirect" href="/wiki/FLV" title="FLV">FLV</a> video formats. Such video is displayed above all Flash content, and takes advantage of <a href="/wiki/Video_codec" title="Video codec">video codec</a> chipsets installed on the user's device. Developers must specifically use the "StageVideo" technology within Flash Player in order for hardware decoding to be enabled. Flash Player internally uses technologies such as <a href="/wiki/DirectX_Video_Acceleration" title="DirectX Video Acceleration">DirectX Video Acceleration</a> and <a href="/wiki/OpenGL" title="OpenGL">OpenGL</a> to do so.</li>
</ul>
<h3><span class="mw-headline" id="Desktop-based">Desktop-based</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flash_Video&amp;action=edit&amp;section=12" title="Edit section: Desktop-based">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

<ul>
<li><a href="/wiki/Adobe_Media_Player" title="Adobe Media Player">Adobe Media Player</a> (freeware) (no longer published or supported by Adobe)</li>
<li><a href="/wiki/Media_Player_Classic" title="Media Player Classic">Media Player Classic</a> (free software)</li>
<li><a href="/wiki/MPlayer" title="MPlayer">MPlayer</a> (free software)</li>
<li><a href="/wiki/RealPlayer" title="RealPlayer">RealPlayer</a> (freeware)</li>

<li><a href="/wiki/Winamp" title="Winamp">Winamp</a> (freeware)</li>
</ul>
<p>Mac OS devices can play flash videos in <a href="/wiki/QuickTime" title="QuickTime">QuickTime</a> with the help of additional software (such as the open source <a href="/wiki/Perian" title="Perian">Perian</a> component.)</p>
<h3><span class="mw-headline" id="PDA-based_and_smartphone-based">PDA-based and smartphone-based</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flash_Video&amp;action=edit&amp;section=13" title="Edit section: PDA-based and smartphone-based">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

<ul>

</ul>
<p>The <a href="/wiki/IPhone" title="IPhone">iPhone</a> and <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android</a> devices can play flash videos with the help of additional software (such as the <a class="mw-redirect" href="/wiki/Skyfire_(web_browser)" title="Skyfire (web browser)">Skyfire</a> web browser application.) Apple iOS has never supported Flash. Android versions above 2.1 through 4.0 supported Flash; Adobe has discontinued Flash Player for Android.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup> On 29 June 2012, Adobe announced that, as they were discontinuing development of the mobile version of Flash, they would prevent the Flash player from installing from the Google Play Store on any Android devices that do not already have the Flash player installed, and that they would not be releasing a version of Flash certified for use with Android version 4.1, codenamed <a href="/wiki/Android_version_history" title="Android version history">Jelly Bean</a>.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup></p>
<h2><span class="mw-headline" id="Delivery_options">Delivery options</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flash_Video&amp;action=edit&amp;section=14" title="Edit section: Delivery options">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Flash Video files can be delivered in several different ways:</p>
<ul>
<li>As a standalone .FLV file. Although Flash Video files are normally delivered using a Flash player for control, the .FLV file itself is only playable with an FLV player. There are several third party players available.</li>
<li>Embedded in an SWF file using the Flash authoring tool (supported in Flash Player 6 and later). The entire file must be transferred before playback can begin. Changing the video requires rebuilding the SWF file.</li>
<li><a href="/wiki/Progressive_download" title="Progressive download">Progressive download</a> via <a class="mw-redirect" href="/wiki/HTTP" title="HTTP">HTTP</a> (supported in Flash Player 7 and later). This method uses <a href="/wiki/ActionScript" title="ActionScript">ActionScript</a> to include an externally hosted Flash Video file client-side for playback. <a href="/wiki/Progressive_download" title="Progressive download">Progressive download</a> has several advantages, including buffering, use of generic HTTP servers, and the ability to reuse a single SWF player for multiple Flash Video sources. Flash Player 8 includes support for random access within video files using the partial download functionality of HTTP, sometimes this is referred to as streaming. However, unlike streaming using <a class="mw-redirect" href="/wiki/Real_Time_Messaging_Protocol" title="Real Time Messaging Protocol">RTMP</a>, HTTP "streaming" does not support real-time broadcasting. Streaming via HTTP requires a custom player and the injection of specific Flash Video metadata containing the exact starting position in bytes and timecode of each keyframe. Using this specific information, a custom Flash Video player can request any part of the Flash Video file starting at a specified keyframe. For example, <a class="mw-redirect" href="/wiki/Google_Video" title="Google Video">Google Video</a>, YouTube, and <a class="new" href="/w/index.php?title=Bitgravity&amp;action=edit&amp;redlink=1" title="Bitgravity (page does not exist)">Bitgravity</a> support progressive downloading and can seek to any part of the video before buffering is complete.</li>
<li>Streamed via <a class="mw-redirect" href="/wiki/Real_Time_Messaging_Protocol" title="Real Time Messaging Protocol">RTMP</a> to the Flash Player using the <a class="mw-redirect" href="/wiki/Flash_Media_Server" title="Flash Media Server">Flash Media Server</a> (formerly called Flash Communication Server), free <a class="external text" href="http://www.rtmpd.com/" rel="nofollow">C++ RTMP Server</a>, VCS, ElectroServer, <a href="/wiki/Helix_Universal_Server" title="Helix Universal Server">Helix Universal Server</a>, <a class="mw-redirect" href="/wiki/Wowza_Media_Server" title="Wowza Media Server">Wowza Media Server</a>, <a class="mw-redirect" href="/wiki/Unified_Streaming_Platform" title="Unified Streaming Platform">Unified Streaming Platform</a> WebORB for .NET, WebORB for Java, erlyvideo (in erlang), or the open source <a href="/wiki/Red5_(media_server)" title="Red5 (media server)">Red5</a> server. As of April 2008, there are stream recorders available for this protocol, re-encoding screencast software excluded.</li>
<li><a href="/wiki/Adobe_Flash_Media_Server" title="Adobe Flash Media Server">Flash Media Server</a> 4.5 allows Flash video streaming to <a href="/wiki/IPhone" title="IPhone">iPhones</a> and <a href="/wiki/IPad" title="IPad">iPads</a>. It achieves this by delivering the content in an <a href="/wiki/MPEG-2" title="MPEG-2">MPEG-2</a> stream using the <a href="/wiki/HTTP_Live_Streaming" title="HTTP Live Streaming">HTTP Live Streaming</a> format.<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup></li>
</ul>
<h2><span class="mw-headline" id="Flash_video_recording">Flash video recording</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flash_Video&amp;action=edit&amp;section=15" title="Edit section: Flash video recording">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>It is possible to record online flash videos using <a href="/wiki/Stream_recorder" title="Stream recorder">stream recorders</a> and <a href="/wiki/Screencast" title="Screencast">screen video capture software</a>. The former gives lossless results, while the latter allows recording any video even when anti-<a class="mw-redirect" href="/wiki/Bandwidth_theft" title="Bandwidth theft">leeching</a> measures are used.</p>
