<p><b>GNU bison</b>, commonly known as Bison, is a <a class="mw-redirect" href="/wiki/Parser_generator" title="Parser generator">parser generator</a> that is part of the <a href="/wiki/GNU_Project" title="GNU Project">GNU Project</a>. Bison reads a specification of a <a href="/wiki/Context-free_language" title="Context-free language">context-free language</a>, warns about any <a href="/wiki/Parsing" title="Parsing">parsing</a> ambiguities, and generates a parser (either in <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a>, or <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>) which reads sequences of <a href="/wiki/Lexical_analysis#Token" title="Lexical analysis">tokens</a> and decides whether the sequence conforms to the syntax specified by the grammar. Bison by default generates <a class="mw-redirect" href="/wiki/LALR_parsers" title="LALR parsers">LALR parsers</a> but can also create <a href="/wiki/GLR_parser" title="GLR parser">GLR parsers</a>.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>In <a href="/wiki/POSIX" title="POSIX">POSIX</a> mode, Bison is compatible with <a href="/wiki/Yacc" title="Yacc">yacc</a>, but also has several extensions over this earlier program. <a class="mw-redirect" href="/wiki/Flex_lexical_analyser" title="Flex lexical analyser">flex</a>, an automatic <a href="/wiki/Lexical_analysis" title="Lexical analysis">lexical analyser</a>, is often used with Bison, to tokenise input data and provide Bison with tokens.</p>
<p>Bison was originally written by Robert Corbett in 1988.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2013)">citation needed</span></a></i>]</sup> Later, in 1990, Robert Corbett wrote another <a class="mw-redirect" href="/wiki/Parser_generator" title="Parser generator">parser generator</a> named <a href="/wiki/Berkeley_Yacc" title="Berkeley Yacc">Berkeley Yacc</a>. Bison was made <a href="/wiki/Yacc" title="Yacc">Yacc</a>-compatible by <a href="/wiki/Richard_Stallman" title="Richard Stallman">Richard Stallman</a>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2013)">citation needed</span></a></i>]</sup></p>
<p>Bison is <a href="/wiki/Free_software" title="Free software">free software</a> and is available under the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a>, with an exception (discussed below) allowing its generated code to be used without triggering the <a href="/wiki/Copyleft" title="Copyleft">copyleft</a> requirements of the licence.</p>

<p>The following example shows how to use Bison and flex to write a simple calculator program (only addition and multiplication) and a program for creating an <a href="/wiki/Abstract_syntax_tree" title="Abstract syntax tree">abstract syntax tree</a>. The next two files provide definition and implementation of the syntax tree functions.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="cm">/*</span>
<span class="cm"> * Expression.h</span>
<span class="cm"> * Definition of the structure used to build the syntax tree.</span>
<span class="cm"> */</span>
<span class="cp">#ifndef __EXPRESSION_H__</span>
<span class="cp">#define __EXPRESSION_H__</span>

<span class="cm">/**</span>
<span class="cm"> * @brief The operation type</span>
<span class="cm"> */</span>
<span class="k">typedef</span> <span class="k">enum</span> <span class="n">tagEOperationType</span>
<span class="p">{</span>
    <span class="n">eVALUE</span><span class="p">,</span>
    <span class="n">eMULTIPLY</span><span class="p">,</span>
    <span class="n">ePLUS</span>
<span class="p">}</span> <span class="n">EOperationType</span><span class="p">;</span>

<span class="cm">/**</span>
<span class="cm"> * @brief The expression structure</span>
<span class="cm"> */</span>
<span class="k">typedef</span> <span class="k">struct</span> <span class="n">tagSExpression</span>
<span class="p">{</span>
    <span class="n">EOperationType</span> <span class="n">type</span><span class="p">;</span><span class="c1">///&lt; type of operation</span>

    <span class="kt">int</span> <span class="n">value</span><span class="p">;</span><span class="c1">///&lt; valid only when type is eVALUE</span>
    <span class="k">struct</span> <span class="n">tagSExpression</span> <span class="o">*</span><span class="n">left</span><span class="p">;</span> <span class="c1">///&lt; left side of the tree</span>
    <span class="k">struct</span> <span class="n">tagSExpression</span> <span class="o">*</span><span class="n">right</span><span class="p">;</span><span class="c1">///&lt; right side of the tree</span>
<span class="p">}</span> <span class="n">SExpression</span><span class="p">;</span>

<span class="cm">/**</span>
<span class="cm"> * @brief It creates an identifier</span>
<span class="cm"> * @param value The number value</span>
<span class="cm"> * @return The expression or NULL in case of no memory</span>
<span class="cm"> */</span>
<span class="n">SExpression</span> <span class="o">*</span><span class="nf">createNumber</span><span class="p">(</span><span class="kt">int</span> <span class="n">value</span><span class="p">);</span>

<span class="cm">/**</span>
<span class="cm"> * @brief It creates an operation</span>
<span class="cm"> * @param type The operation type</span>
<span class="cm"> * @param left The left operand</span>
<span class="cm"> * @param right The right operand</span>
<span class="cm"> * @return The expression or NULL in case of no memory</span>
<span class="cm"> */</span>
<span class="n">SExpression</span> <span class="o">*</span><span class="nf">createOperation</span><span class="p">(</span><span class="n">EOperationType</span> <span class="n">type</span><span class="p">,</span> <span class="n">SExpression</span> <span class="o">*</span><span class="n">left</span><span class="p">,</span> <span class="n">SExpression</span> <span class="o">*</span><span class="n">right</span><span class="p">);</span>

<span class="cm">/**</span>
<span class="cm"> * @brief Deletes a expression</span>
<span class="cm"> * @param b The expression</span>
<span class="cm"> */</span>
<span class="kt">void</span> <span class="nf">deleteExpression</span><span class="p">(</span><span class="n">SExpression</span> <span class="o">*</span><span class="n">b</span><span class="p">);</span>

<span class="cp">#endif </span><span class="c1">// __EXPRESSION_H__</span>
</pre></div>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="cm">/*</span>
<span class="cm"> * Expression.c</span>
<span class="cm"> * Implementation of functions used to build the syntax tree.</span>
<span class="cm"> */</span>

<span class="cp">#include</span> <span class="cpf">"Expression.h"</span>

<span class="cp">#include</span> <span class="cpf">&lt;stdlib.h&gt;</span>

<span class="cm">/**</span>
<span class="cm"> * @brief Allocates space for expression</span>
<span class="cm"> * @return The expression or NULL if not enough memory</span>
<span class="cm"> */</span>
<span class="k">static</span> <span class="n">SExpression</span> <span class="o">*</span><span class="nf">allocateExpression</span><span class="p">()</span>
<span class="p">{</span>
    <span class="n">SExpression</span> <span class="o">*</span><span class="n">b</span> <span class="o">=</span> <span class="p">(</span><span class="n">SExpression</span> <span class="o">*</span><span class="p">)</span><span class="n">malloc</span><span class="p">(</span><span class="k">sizeof</span><span class="p">(</span><span class="n">SExpression</span><span class="p">));</span>

    <span class="k">if</span> <span class="p">(</span><span class="n">b</span> <span class="o">==</span> <span class="nb">NULL</span><span class="p">)</span>
        <span class="k">return</span> <span class="nb">NULL</span><span class="p">;</span>

    <span class="n">b</span><span class="o">-&gt;</span><span class="n">type</span> <span class="o">=</span> <span class="n">eVALUE</span><span class="p">;</span>
    <span class="n">b</span><span class="o">-&gt;</span><span class="n">value</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span>

    <span class="n">b</span><span class="o">-&gt;</span><span class="n">left</span> <span class="o">=</span> <span class="nb">NULL</span><span class="p">;</span>
    <span class="n">b</span><span class="o">-&gt;</span><span class="n">right</span> <span class="o">=</span> <span class="nb">NULL</span><span class="p">;</span>

    <span class="k">return</span> <span class="n">b</span><span class="p">;</span>
<span class="p">}</span>

<span class="n">SExpression</span> <span class="o">*</span><span class="nf">createNumber</span><span class="p">(</span><span class="kt">int</span> <span class="n">value</span><span class="p">)</span>
<span class="p">{</span>
    <span class="n">SExpression</span> <span class="o">*</span><span class="n">b</span> <span class="o">=</span> <span class="n">allocateExpression</span><span class="p">();</span>

    <span class="k">if</span> <span class="p">(</span><span class="n">b</span> <span class="o">==</span> <span class="nb">NULL</span><span class="p">)</span>
        <span class="k">return</span> <span class="nb">NULL</span><span class="p">;</span>

    <span class="n">b</span><span class="o">-&gt;</span><span class="n">type</span> <span class="o">=</span> <span class="n">eVALUE</span><span class="p">;</span>
    <span class="n">b</span><span class="o">-&gt;</span><span class="n">value</span> <span class="o">=</span> <span class="n">value</span><span class="p">;</span>

    <span class="k">return</span> <span class="n">b</span><span class="p">;</span>
<span class="p">}</span>

<span class="n">SExpression</span> <span class="o">*</span><span class="nf">createOperation</span><span class="p">(</span><span class="n">EOperationType</span> <span class="n">type</span><span class="p">,</span> <span class="n">SExpression</span> <span class="o">*</span><span class="n">left</span><span class="p">,</span> <span class="n">SExpression</span> <span class="o">*</span><span class="n">right</span><span class="p">)</span>
<span class="p">{</span>
    <span class="n">SExpression</span> <span class="o">*</span><span class="n">b</span> <span class="o">=</span> <span class="n">allocateExpression</span><span class="p">();</span>

    <span class="k">if</span> <span class="p">(</span><span class="n">b</span> <span class="o">==</span> <span class="nb">NULL</span><span class="p">)</span>
        <span class="k">return</span> <span class="nb">NULL</span><span class="p">;</span>

    <span class="n">b</span><span class="o">-&gt;</span><span class="n">type</span> <span class="o">=</span> <span class="n">type</span><span class="p">;</span>

<h2><span class="mw-headline" id="Reentrancy">Reentrancy</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GNU_bison&amp;action=edit&amp;section=2" title="Edit section: Reentrancy">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Reentrancy is a feature which has been added to Bison and does not exist in Yacc.</p>
<p>Normally, Bison generates a parser which is not <a class="mw-redirect" href="/wiki/Reentrant_(subroutine)" title="Reentrant (subroutine)">reentrant</a>. In order to achieve reentrancy the declaration <code>%define api.pure</code> must be used. More details on Bison reentrancy can be found in the Bison manual.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<h2><span class="mw-headline" id="Using_Bison_from_other_languages">Using Bison from other languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GNU_bison&amp;action=edit&amp;section=3" title="Edit section: Using Bison from other languages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Bison can only generate code for <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a> and <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> For using the Bison generated parser from other languages a <a href="/wiki/Language_binding" title="Language binding">language binding</a> tool such as <a href="/wiki/SWIG" title="SWIG">SWIG</a> can be used.</p>
<h2><span class="mw-headline" id="Licence_and_distribution_of_generated_code">Licence and distribution of generated code</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GNU_bison&amp;action=edit&amp;section=4" title="Edit section: Licence and distribution of generated code">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Because Bison generates source code that in turn gets added to the source code of other software projects, it raises some simple but interesting copyright questions.</p>
<h3><span class="mw-headline" id="A_GPL-compatible_licence_is_not_required">A GPL-compatible licence is not required</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GNU_bison&amp;action=edit&amp;section=5" title="Edit section: A GPL-compatible licence is not required">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The code generated by Bison includes significant amounts of code from the Bison project itself. The Bison package is distributed under the terms of the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a> (GPL) but an exception has been added so that the GPL does not apply to output.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup><sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<p>Earlier releases of Bison stipulated that parts of its output were also licensed under the GPL, due to the inclusion of the yyparse() function from the original source code in the output.</p>
<h3><span class="mw-headline" id="Distribution_of_packages_using_Bison">Distribution of packages using Bison</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GNU_bison&amp;action=edit&amp;section=6" title="Edit section: Distribution of packages using Bison">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Free software projects that use Bison may have a choice of whether to distribute the source code which their project feeds into Bison, or the resulting C code outputted by Bison. Both are sufficient for a recipient to be able to compile the project's source code. However, distributing only the input carries the minor inconvenience that the recipients must have a compatible copy of Bison installed so that they can generate the necessary C code when compiling the project. And distributing only the outputted C code creates the problem of making it very difficult for the recipients to modify the parser since this code was written neither <i>by</i> a human nor <i>for</i> humans - its purpose is to be fed directly into a C compiler.</p>
<p>These problems can be avoided by distributing both the input files and the generated code. Most people will compile using the generated code, no different from any other software package, but anyone who wants to modify the parser component can modify the input files first and re-generate the generated files before compiling. Projects distributing both usually don't have the generated files in their <a class="mw-redirect" href="/wiki/Revision_control" title="Revision control">revision control</a> systems. The files are only generated when making a release.</p>
<p>Some licences, such as the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GPL</a>, require that the source code be in "<i>the preferred form of the work for making modifications to it</i>". GPL'd projects using Bison must thus distribute the files which are the input for Bison. Of course, they can also include the generated files.</p>
<h2><span class="mw-headline" id="Use">Use</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GNU_bison&amp;action=edit&amp;section=7" title="Edit section: Use">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Because Bison was written as a replacement for Yacc, and is largely compatible, the code from a lot of projects using Bison could equally be fed into Yacc, so it's difficult to say that a project's source code "uses" Bison. In many cases, the "use" of Bison could be trivially replaced by the equivalent use of Yacc.</p>
<p>Bison does have features not found in Yacc, so some projects can be truly said to "use" Bison, since Yacc wouldn't suffice.</p>
<p>The following list is of projects which are known to "use" Bison in the looser sense, that they use free software development tools and distribute code which is intended to be fed into Bison or a Bison-compatible package.</p>
<ul>
<li>The <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a> programming language (YARV)</li>
<li>The <a href="/wiki/PHP" title="PHP">PHP</a> programming language (Zend Parser)</li>
<li><a href="/wiki/GNU_Compiler_Collection" title="GNU Compiler Collection">GCC</a> started out using Bison, but switched to a hand-written <a href="/wiki/Recursive_descent_parser" title="Recursive descent parser">recursive-descent parser</a> for C++ in 2004 (version 3.4),<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> and for C and Objective-C in 2006 (version 4.1)<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></li>
<li>The <a href="/wiki/Go_(programming_language)" title="Go (programming language)">Go</a> programming language (GC)</li>
<li><a href="/wiki/Bash_(Unix_shell)" title="Bash (Unix shell)">Bash</a> shell uses a yacc grammar for parsing the command input.</li>
<li><a href="/wiki/LilyPond" title="LilyPond">LilyPond</a></li>
<li><a href="/wiki/PostgreSQL" title="PostgreSQL">PostgreSQL</a><sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></li>
<li><a href="/wiki/MySQL" title="MySQL">MySQL</a><sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></li>
<li><a href="/wiki/GNU_Octave" title="GNU Octave">GNU Octave</a> uses a Bison-generated parser.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></li>
</ul>
