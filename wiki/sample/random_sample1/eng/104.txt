<p>In <a href="/wiki/Computer_architecture" title="Computer architecture">computer architecture</a>, <b>16-bit</b> <a href="/wiki/Integer_(computer_science)" title="Integer (computer science)">integers</a>, <a href="/wiki/Memory_address" title="Memory address">memory addresses</a>, or other <a href="/wiki/Data#Uses_of_data_in_computing" title="Data">data</a> units are those that are at most 16 <a href="/wiki/Bit" title="Bit">bits</a> (2 octets) wide. Also, 16-bit <a href="/wiki/Central_processing_unit" title="Central processing unit">CPU</a> and <a href="/wiki/Arithmetic_logic_unit" title="Arithmetic logic unit">ALU</a> architectures are those that are based on <a href="/wiki/Processor_register" title="Processor register">registers</a>, <a href="/wiki/Address_bus" title="Address bus">address buses</a>, or <a href="/wiki/Bus_(computing)" title="Bus (computing)">data buses</a> of that size. <b>16-bit</b> is also a term given to a generation of <a href="/wiki/Microcomputer" title="Microcomputer">microcomputers</a> in which 16-bit <a href="/wiki/Microprocessor" title="Microprocessor">microprocessors</a> were the norm.</p>
<p>A 16-bit register can store 2<sup>16</sup> different values. The <a href="/wiki/Signedness" title="Signedness">signed</a> range of <a href="/wiki/Integer" title="Integer">integer</a> values that can be stored in 16 bits is -32,768 (-1 × 2<sup>15</sup>) through 32,767 (2<sup>15</sup> - 1); the <a href="/wiki/Signedness" title="Signedness">unsigned</a> range is 0 through 65,535 (2<sup>16</sup> - 1). Since 2<sup>16</sup> is 65,536, a processor with 16-bit <a href="/wiki/Memory_address" title="Memory address">memory addresses</a> can directly access 64 <a href="/wiki/Kibibyte" title="Kibibyte">KiB</a> of <a href="/wiki/Byte_addressing" title="Byte addressing">byte-addressable</a> memory.</p>

<h2><span class="mw-headline" id="16-bit_architecture">16-bit architecture</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=16-bit&amp;action=edit&amp;section=1" title="Edit section: 16-bit architecture">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The <a href="/wiki/Whirlwind_I" title="Whirlwind I">MIT Whirlwind</a> (<abbr title="circa">c.</abbr> 1951)<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> was quite possibly the first-ever 16-bit computer. Other early (<abbr title="circa">c.</abbr> 1965–70) 16-bit computers include the <a href="/wiki/IBM_1130" title="IBM 1130">IBM 1130</a>,<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> the <a href="/wiki/HP_2100" title="HP 2100">HP 2100</a>,<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> the <a href="/wiki/Data_General_Nova" title="Data General Nova">Data General Nova</a>,<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> and the <a href="/wiki/PDP-11" title="PDP-11">DEC PDP-11</a>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> Early (<abbr title="circa">c.</abbr> 1973–75) multi-chip <a class="mw-redirect" href="/wiki/Microprocessors#16-bit_designs" title="Microprocessors">16-bit microprocessors</a> include the <a href="/wiki/IMP-16" title="IMP-16">National Semiconductor IMP-16</a> and the <a href="/wiki/MCP-1600" title="MCP-1600">Western Digital MCP-1600</a>. Early (<abbr title="circa">c.</abbr> 1975–76) single-chip 16-bit microprocessors include the <a href="/wiki/National_Semiconductor_PACE" title="National Semiconductor PACE">National Semiconductor PACE</a>, the <a href="/wiki/HP_2100#Descendants_and_variants" title="HP 2100">HP BPC</a>, and the <a href="/wiki/Texas_Instruments_TMS9900" title="Texas Instruments TMS9900">TI TMS9900</a>. Other notable 16-bit processors include the <a href="/wiki/Intel_8086" title="Intel 8086">Intel 8086</a>, the <a href="/wiki/Intel_80286" title="Intel 80286">Intel 80286</a>, the <a href="/wiki/WDC_65816/65802" title="WDC 65816/65802">WDC 65C816</a>, and the <a href="/wiki/Zilog_Z8000" title="Zilog Z8000">Zilog Z8000</a>. The <a href="/wiki/Intel_8088" title="Intel 8088">Intel 8088</a> was <a class="mw-redirect" href="/wiki/Binary_compatible" title="Binary compatible">binary compatible</a> with the Intel 8086, and was 16-bit in that its registers were 16 bits wide, and arithmetic instructions could operate on 16-bit quantities, even though its external bus was 8 bits wide.</p>
<p>A 16-bit integer can store 2<sup>16</sup> (or 65,536) distinct values. In an <a href="/wiki/Signedness" title="Signedness">unsigned</a> representation, these values are the integers between <a class="mw-redirect" href="/wiki/0_(number)" title="0 (number)">0</a> and 65,535; using <a href="/wiki/Two%27s_complement" title="Two's complement">two's complement</a>, possible values range from −32,768 to 32,767. Hence, a processor with 16-bit <a href="/wiki/Memory_address" title="Memory address">memory addresses</a> can directly access 64 <a href="/wiki/Kibibyte" title="Kibibyte">KB</a> of <a href="/wiki/Byte_addressing" title="Byte addressing">byte-addressable</a> memory.</p>
<p>16-bit processors have been almost entirely supplanted in the <a href="/wiki/Personal_computer" title="Personal computer">personal computer</a> industry, and are used less than 32-bit (or 8-bit) CPUs in embedded applications.</p>
<h3><span class="mw-headline" id="The_16.2F32-bit_Motorola_68000_and_Intel_386SX">The 16/32-bit Motorola 68000 and Intel 386SX</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=16-bit&amp;action=edit&amp;section=2" title="Edit section: The 16/32-bit Motorola 68000 and Intel 386SX">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/Motorola_68000" title="Motorola 68000">Motorola 68000</a> is sometimes called <i>16-bit</i> because its internal and external data buses were 16 bits wide, however it could be considered a <a href="/wiki/32-bit" title="32-bit">32-bit</a> processor in that the general purpose registers were 32 bits wide and most arithmetic instructions supported 32-bit arithmetic. The 68000 was a <a href="/wiki/Microcode" title="Microcode">microcoded</a> processor with three internal 16-bit ALU units. Only <a href="/wiki/24-bit" title="24-bit">24-bits</a> of the <a href="/wiki/Program_counter" title="Program counter">program counter</a> (PC) were available on original <a href="/wiki/Dual_in-line_package" title="Dual in-line package">DIP</a> packages, with up to 16 megabytes of addressable <a href="/wiki/Random-access_memory" title="Random-access memory">RAM</a>. 68000 software is 32-bit in nature and forward-compatible with other 32-bit processors in the same family.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> The 68008 was a version of the 68000 with 8-bit external data path and 1 megabyte addressing for the 48-pin DIP version and 4 megabyte for the 52-pin PLCC version. Several <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple Inc.</a> <a href="/wiki/Macintosh" title="Macintosh">Macintosh</a> models; e.g., LC series, used 32-bit 68020 and 68030 processors on a 16-bit data bus to save cost.</p>
<p>Similar analysis applies to Intel's <a class="mw-redirect" href="/wiki/80286" title="80286">80286</a> CPU replacement called the <a href="/wiki/Intel_80386" title="Intel 80386">386SX</a> which is a 32-bit processor with 32-bit <a href="/wiki/Arithmetic_logic_unit" title="Arithmetic logic unit">ALU</a> and internal 32-bit data paths with a 16-bit external bus and 24-bit addressing of the processor it replaced.</p>
<p>The 68000 processor of the <a class="mw-redirect" href="/wiki/Sega_Mega_Drive" title="Sega Mega Drive">Sega Mega Drive</a> was a highly advertised feature of the video game system. Due to the saturation of this advertising, the 1988–95 era (<a class="mw-redirect" href="/wiki/History_of_video_game_consoles_(fourth_generation)" title="History of video game consoles (fourth generation)">fourth generation</a>) of video game consoles is often called <i>the 16-bit era</i>.</p>
<h2><span class="mw-headline" id="Intel_16-bit_memory_models">Intel 16-bit memory models</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=16-bit&amp;action=edit&amp;section=3" title="Edit section: Intel 16-bit memory models">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/Intel_Memory_Model" title="Intel Memory Model">Intel Memory Model</a></div>
<p>Just as there are <a class="mw-redirect" href="/wiki/64-bit#64-bit_data_models" title="64-bit">multiple data models for 64-bit architectures</a>, the 16-bit Intel architecture allows for different memory models—ways to access a particular memory location. The reason for using a specific memory model is the size of the assembler instructions or required storage for pointers. Compilers of the 16-bit era generally had the following type-width characteristic:</p>
<table class="wikitable">
<caption>16-bit data model</caption>
<tr>
<th>Data model</th>

<th><code>int</code></th>
<th><code>long</code></th>
<th>Pointers</th>
</tr>
<tr>
<th>IP16L32 (near)</th>
<td>16</td>
<td>16</td>
<td>32</td>
<td>16</td>
</tr>
<tr>
<th>I16LP32 (far)</th>
<td>16</td>

<td>32</td>

<dl>
<dt>Tiny</dt>
<dd>Code and data will be in the same segment (especially, the registers CS, DS, ES, SS will point to the same segment); <i>near</i> (16-bit) pointers are always used. Code, <a href="/wiki/Heap_(data_structure)" title="Heap (data structure)">data</a> and stack together cannot exceed 64 KB.</dd>
<dt>Small</dt>
<dd>Code and data will be in different segments, and near pointers are always used. There will be 64 KB of space for code and 64 KB for data/stack.</dd>
<dt>Medium</dt>
<dd>Code pointers will use <i>far</i> pointers (16:16 bit), enabling access to 1 MB. Data pointers remain to be of the near type.</dd>
<dt>Compact</dt>
<dd>Data pointers will use far and code will use near pointers.</dd>
<dt>Large/huge</dt>
<dd>Code and data pointers will be far.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></dd>

<h2><span class="mw-headline" id="16-bit_application">16-bit application</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=16-bit&amp;action=edit&amp;section=4" title="Edit section: 16-bit application">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In the context of <a href="/wiki/IBM_PC_compatible" title="IBM PC compatible">IBM PC compatible</a> and <a href="/wiki/Wintel" title="Wintel">Wintel</a> platforms, a 16-bit application is any software written for <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>, <a href="/wiki/OS/2" title="OS/2">OS/2</a> 1.x or early versions of <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> which originally ran on the 16-bit <a href="/wiki/Intel_8088" title="Intel 8088">Intel 8088</a> and <a href="/wiki/Intel_80286" title="Intel 80286">Intel 80286</a> <a href="/wiki/Microprocessor" title="Microprocessor">microprocessors</a>. Such applications used a 20-<a href="/wiki/Bit" title="Bit">bit</a> or 24-bit <a href="/wiki/X86_memory_segmentation" title="X86 memory segmentation">segment or selector-offset</a> address representation to extend the range of addressable memory locations beyond what was possible using only 16-bit addresses. Programs containing more than 2<sup>16</sup> <a href="/wiki/Byte" title="Byte">bytes</a> (64 <a href="/wiki/Kilobyte" title="Kilobyte">kilobytes</a>) of instructions and data therefore required special instructions to switch between their 64-kilobyte <a class="mw-redirect" href="/wiki/Memory_segment" title="Memory segment">segments</a>, increasing the complexity of programming 16-bit applications.</p>
