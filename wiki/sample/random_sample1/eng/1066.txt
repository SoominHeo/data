<p><b>HFS Plus</b> or <b>HFS+</b> is a <a href="/wiki/File_system" title="File system">file system</a> developed by <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple Inc.</a> It serves as the primary file system of <a href="/wiki/MacOS" title="MacOS">macOS</a>. HFS+ was developed to replace the <a href="/wiki/Hierarchical_File_System" title="Hierarchical File System">Hierarchical File System</a> (HFS) as the primary file system used in <a href="/wiki/Macintosh" title="Macintosh">Macintosh</a> computers (or other systems running the <a href="/wiki/Classic_Mac_OS" title="Classic Mac OS">classic Mac OS</a>). It is also one of the formats used by the <a href="/wiki/IPod" title="IPod">iPod</a> digital music player. HFS Plus is also referred to as <b>Mac OS Extended</b> or <b>HFS Extended</b>, where its predecessor, HFS, is also referred to as <i>Mac OS Standard</i> or <i>HFS Standard</i>. During development, Apple referred to this file system with the codename <i>Sequoia</i>.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<p>HFS Plus is an improved version of HFS, supporting much larger files (block addresses are 32-bit length instead of 16-bit) and using <a href="/wiki/Unicode" title="Unicode">Unicode</a> (instead of <a href="/wiki/Mac_OS_Roman" title="Mac OS Roman">Mac OS Roman</a> or any of several other character sets) for naming items. Like HFS, HFS Plus uses <a href="/wiki/B-tree" title="B-tree">B-trees</a> to store most volume <a href="/wiki/Metadata" title="Metadata">metadata</a>, but unlike most other file systems, HFS Plus supports <a href="/wiki/Hard_link" title="Hard link">hard links</a> to directories. HFS Plus permits filenames up to 255 characters in length, and <a href="/wiki/Fork_(file_system)" title="Fork (file system)">n-forked files</a> similar to <a href="/wiki/NTFS" title="NTFS">NTFS</a>, though until 2005 almost no system software took advantage of forks other than the <a href="/wiki/Resource_fork" title="Resource fork">data fork</a> and <a href="/wiki/Resource_fork" title="Resource fork">resource fork</a>. HFS Plus also uses a full 32-bit allocation mapping table rather than HFS's 16 bits, significantly improving space utilization with large disks.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HFS_Plus&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>HFS+ was introduced with the January 19, 1998 release of <a href="/wiki/Mac_OS_8" title="Mac OS 8">Mac OS 8.1</a>.<sup class="reference" id="cite_ref-tn1150_2-1"><a href="#cite_note-tn1150-2">[2]</a></sup></p>
<p>With the release of the <a class="mw-redirect" href="/wiki/Mac_OS_X_v10.2" title="Mac OS X v10.2">Mac OS X 10.2.2</a> update on November 11, 2002, Apple added optional <a href="/wiki/Journaling_file_system" title="Journaling file system">journaling</a> features to HFS Plus for improved data reliability. These features were easily accessible in Mac OS X Server, but only accessible through the command line in the standard desktop client.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>With Mac OS X v10.3, all HFS Plus volumes on all Macs were set to be journaled by default. Within the system, an HFS Plus volume with a journal is identified as <b>HFSJ</b>.</p>
<p>Mac OS X 10.3 also introduced another version of HFS Plus called <b>HFSX</b>. HFSX volumes are almost identical to HFS Plus volumes, except that they are never surrounded by the <a href="/wiki/HFS_Plus#Design" title="HFS Plus">HFS Wrapper</a> that is typical of HFS Plus volumes and they optionally support <a href="/wiki/Case_sensitivity" title="Case sensitivity">case sensitivity</a> for file and folder names. HFSX volumes can be recognized by two entries in the Volume Header, a value of HX in the signature field and 5 in the version field.<sup class="reference" id="cite_ref-tn1150_2-2"><a href="#cite_note-tn1150-2">[2]</a></sup></p>
<p>Mac OS X 10.3 also marked Apple's adoption of Unicode 3.2 decomposition, superseding the Unicode 2.1 decomposition used previously. This change caused problems for developers writing software for Mac OS X.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></p>
<p>With Mac OS X 10.4, Apple added support for Inline Attribute Data records, something that had been a part of the Mac OS X implementation of HFS Plus since at least <a class="mw-redirect" href="/wiki/Mac_OS_X_v10.0" title="Mac OS X v10.0">10.0</a>, but always marked as "reserved for future use".<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> Until the release of <a href="/wiki/MacOS_Server" title="MacOS Server">Mac OS X Server 10.4</a>, HFS Plus supported only the standard UNIX <a href="/wiki/File_system_permissions" title="File system permissions">file system permissions</a>; however, 10.4 introduced support for <a href="/wiki/Access_control_list" title="Access control list">access control list</a>âbased file security, which provides a richer mechanism to define file permissions and is also designed to be fully compatible with the file permission models on other platforms such as <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a> and <a href="/wiki/Windows_Server_2003" title="Windows Server 2003">Windows Server 2003</a>.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
<p>In Mac OS X Leopard 10.5, directory hard-linking was added as a fundamental part of Time Machine.</p>
<p>In Mac OS X Snow Leopard 10.6, HFS+ compression was added. In open source and some other areas this is referred to as AppleFSCompression. Compressed data may be stored in either an extended attribute or the resource fork.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> When using non-Apple APIs, AppleFSCompression is not always completely transparent.</p>
<p>In Mac OS X Lion 10.7, logical volume encryption (known as <a href="/wiki/FileVault" title="FileVault">FileVault 2</a>) was added to the operating system. This addition to the operating system in no way changed the logical structure of the file system. Apple's logical volume manager is known as Core Storage and its encryption at the volume level can apply to file systems other than HFS Plus. With appropriate hardware, both encryption and decryption should be transparent.</p>
<h2><span class="mw-headline" id="Design">Design</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HFS_Plus&amp;action=edit&amp;section=2" title="Edit section: Design">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>HFS Plus volumes are divided into sectors (called logical blocks in HFS), that are usually 512 bytes in size. These sectors are then grouped together into allocation blocks which can contain one or more sectors; the number of allocation blocks depends on the total size of the volume. HFS Plus uses a larger value to address allocation blocks than HFS, 32 bits rather than 16 bits; this means it can access 4,294,967,296 (= 2<sup>32</sup>) allocation blocks rather than the 65,536 (= 2<sup>16</sup>) allocation blocks available to HFS.<sup class="reference" id="cite_ref-tn1150_2-3"><a href="#cite_note-tn1150-2">[2]</a></sup> When disks were small, this was of little consequence, but as larger-capacity drives became available, it meant that the smallest amount of space that any file could occupy (a single allocation block) became excessively large, wasting significant amounts of space. For example, on a 1 GB disk, the allocation block size under HFS is 16 KB, so even a 1 byte file would take up 16 KB of disk space. HFS Plus's system greatly improves space utilization on larger disks as a result.</p>
<p>File and folder names in HFS Plus are also character encoded in <a href="/wiki/UTF-16" title="UTF-16">UTF-16</a><sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability"><span title="The material near this tag needs to be fact-checked with the cited source(s). (February 2011)">verification needed</span></a></i>]</sup> and normalized to a form very nearly the same as <a href="/wiki/Unicode_equivalence" title="Unicode equivalence">Unicode Normalization Form D (NFD)</a><sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> (which means that precomposed characters like "Ã¥" are decomposed in the HFS+ filename and therefore count as two code units<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> and UTF-16 implies that characters from outside the <a href="/wiki/Plane_(Unicode)#Basic_Multilingual_Plane" title="Plane (Unicode)">Basic Multilingual Plane</a> also count as two code units in an HFS+ filename). HFS Plus permits filenames up to 255 UTF-16 code units in length.</p>
<p>Formerly, HFS Plus volumes were embedded inside an HFS standard filesystem. This was phased out by the Tiger transition to Intel Macs, where the HFS Plus filesystem was not embedded inside a wrapper. The wrapper had been designed for two purposes; it allowed Macintosh computers without HFS Plus support in their ROM to boot HFS Plus volumes and it also was designed to help users transition to HFS Plus by including a minimal HFS volume with a read-only file called <i>Where_have_all_my_files_gone?</i>, explaining to users with versions of Mac OS 8.0 and earlier without HFS Plus, that the volume requires a system with HFS Plus support. The original HFS volume contains a signature and an offset to the embedded HFS Plus volume within its volume header. All allocation blocks in the HFS volume which contain the embedded volume are mapped out of the HFS allocation file as <i>bad blocks</i>.<sup class="reference" id="cite_ref-tn1150_2-4"><a href="#cite_note-tn1150-2">[2]</a></sup></p>
<p>Notable among filesystems used for Unix systems, HFS Plus does not support <a href="/wiki/Sparse_file" title="Sparse file">sparse files</a>.</p>
<p>There are nine structures that make up a typical HFS Plus volume:<sup class="reference" id="cite_ref-tn1150_2-5"><a href="#cite_note-tn1150-2">[2]</a></sup></p>
<ol>
<li>Sectors 0 and 1 of the volume are HFS <b><a href="/wiki/Boot_sector" title="Boot sector">boot blocks</a></b>. These are identical to the boot blocks in an HFS volume. They are part of the HFS wrapper.</li>
<li>Sector 2 contains the <b>Volume Header</b> equivalent to the Master Directory Block in an HFS volume. The Volume Header stores a wide variety of data about the volume itself, for example the size of allocation blocks, a timestamp that indicates when the volume was created or the location of other volume structures such as the Catalog File or Extent Overflow File. The Volume Header is always located in the same place.</li>
<li>The <b>Allocation File</b> which keeps track of which allocation blocks are free and which are in use. It is similar to the Volume Bitmap in HFS, in which each allocation block is represented by one bit. A zero means the block is free and a one means the block is in use. The main difference with the HFS Volume Bitmap, is that the Allocation File is stored as a regular file, it does not occupy a special reserved space near the beginning of the volume. The Allocation File can also change size and does not have to be stored contiguously within a volume.</li>
<li>The <b>Catalog File</b> is a <a href="/wiki/B-tree" title="B-tree">B-tree</a> that contains records for all the files and directories stored in the volume. The HFS Plus Catalog File is very similar to the <a href="/wiki/Hierarchical_File_System#Design" title="Hierarchical File System">HFS Catalog File</a>, the main differences being records are larger to allow more fields and to allow for those fields to be larger (for example to allow the longer 255-character unicode file names in HFS Plus). A record in the HFS Catalog File is 512 bytes in size, a record in the HFS Plus Catalog File is 4Â KB in the classic Mac OS and 8Â KB in macOS. Fields in HFS are of fixed size, in HFS Plus the size can vary depending on the actual size of the data they store.</li>
<li>The <b>Extents Overflow File</b> is another <a href="/wiki/B-tree" title="B-tree">B-tree</a> that records the allocation blocks that are allocated to each file as extents. Each file record in the Catalog File is capable of recording eight extents for each fork of a file; once those are used additional extents are recorded in the Extents Overflow File. Bad blocks are also recorded as extents in the Extents Overflow File. The default size of an extent record in the classic Mac OS is 1 KB and 4 KB in macOS.</li>
<li>The <b>Attributes File</b> is a new <a href="/wiki/B-tree" title="B-tree">B-tree</a> in HFS Plus that does not have a corresponding structure in HFS. The Attributes File can store three different types of 4 KB records: <i>Inline Data Attribute</i> records, <i>Fork Data Attribute</i> records and <i>Extension Attribute</i> records. Inline Data Attribute records store small attributes that can fit within the record itself. Fork Data Attribute records contain references to a maximum of eight extents that can hold larger attributes. Extension Attributes are used to extend a Fork Data Attribute record when its eight extent records are already used.</li>
<li>The <b>Startup File</b> is designed for non-Mac OS systems that don't have HFS or HFS Plus support. It is similar to the Boot Blocks of an HFS volume.</li>
<li>The second to last sector contains the <b>Alternate Volume Header</b> equivalent to the Alternate Master Directory Block of HFS.</li>
<li>The last sector in the volume is reserved for use by Apple. It is used during the computer manufacturing process.<sup class="reference" id="cite_ref-tn1150_2-6"><a href="#cite_note-tn1150-2">[2]</a></sup></li>
</ol>
<h2><span class="mw-headline" id="Criticisms">Criticisms</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HFS_Plus&amp;action=edit&amp;section=3" title="Edit section: Criticisms">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Due to the age of HFS Plus, several features that are currently considered staples of modern file-systems like <a href="/wiki/ZFS" title="ZFS">ZFS</a>, or <a href="/wiki/NTFS" title="NTFS">NTFS</a> are not present in HFS Plus.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> One such feature, data checksums, is the most routinely cited missing feature.<sup class="reference" id="cite_ref-siracusa_14-0"><a href="#cite_note-siracusa-14">[14]</a></sup> Additionally, the core of the filesystem uses case-insensitive, <a href="/wiki/Unicode_equivalence#Normal_forms" title="Unicode equivalence">NFD</a> Unicode strings, which led <a href="/wiki/Linus_Torvalds" title="Linus Torvalds">Linus Torvalds</a> to state "HFS+ is probably the worst file-system ever."<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup></p>
<p>Examples of modern file-system features that HFS+ lacks include:</p>
<ul>
<li>nanosecond timestamps <sup class="reference" id="cite_ref-siracusa_14-1"><a href="#cite_note-siracusa-14">[14]</a></sup></li>
<li><a href="/wiki/Concurrency_(computer_science)" title="Concurrency (computer science)">concurrent</a> access (let more than one process at the time access the filesystem) <sup class="reference" id="cite_ref-siracusa_14-2"><a href="#cite_note-siracusa-14">[14]</a></sup></li>
<li>checksumming <sup class="reference" id="cite_ref-siracusa_14-3"><a href="#cite_note-siracusa-14">[14]</a></sup></li>
<li><a href="/wiki/Snapshot_(computer_storage)" title="Snapshot (computer storage)">snapshotting</a> <sup class="reference" id="cite_ref-siracusa_14-4"><a href="#cite_note-siracusa-14">[14]</a></sup></li>
<li>longer time frame (February 6, 2040 for HFS+)<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup></li>
<li>sparse file support <sup class="reference" id="cite_ref-siracusa_14-5"><a href="#cite_note-siracusa-14">[14]</a></sup></li>
<li>real hard links <sup class="reference" id="cite_ref-siracusa_14-6"><a href="#cite_note-siracusa-14">[14]</a></sup></li>
</ul>
<p>Initially HFS Plus was not designed for <a href="/wiki/MacOS" title="MacOS">macOS</a> (or any UNIX-like system) nor Intel architectures. As a result, all metadata read must be byte swapped, because Macs today use <a href="/wiki/Endianness#Little-endian" title="Endianness">little-endian</a>, while <a href="/wiki/PowerPC" title="PowerPC">PowerPC</a> based Macs, at the time when HFS Plus was designed, used <a href="/wiki/Endianness#Big-endian" title="Endianness">big-endian</a>.<sup class="reference" id="cite_ref-siracusa_14-7"><a href="#cite_note-siracusa-14">[14]</a></sup></p>
<h2><span class="mw-headline" id="Other_operating_systems">Other operating systems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HFS_Plus&amp;action=edit&amp;section=4" title="Edit section: Other operating systems">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Linux">Linux</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HFS_Plus&amp;action=edit&amp;section=5" title="Edit section: Linux">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/Linux_kernel" title="Linux kernel">Linux kernel</a> includes the hfsplus module<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> for mounting HFS+ filesystems read-write. HFS+ <a href="/wiki/Fsck" title="Fsck">fsck</a> and <a href="/wiki/Mkfs" title="Mkfs">mkfs</a> have been ported to <a href="/wiki/Linux" title="Linux">Linux</a> and are part of the hfsprogs package.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<p>In 2009, these drivers were diagnosed to be corrupting HFS+ drives with a capacity greater than 2Â TB.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> Consequently, Linux distributions such as Debian and Ubuntu also stopped allowing mounting of HFS+ drives or partitions greater than 2Â TB.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup> As of February 2011<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=HFS_Plus&amp;action=edit">[update]</a></sup>, work is in progress to lift this restriction.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> <sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Manual_of_Style/Dates_and_numbers#Precise_language" title="Wikipedia:Manual of Style/Dates and numbers"><span title="The text near this tag is dated.">needs update</span></a></i>]</sup></p>
<p>Under Linux's current HFS+ driver, journaling must be disabled in order to write data safely onto an HFS+ partition. Provided the partition isn't being used by Apple's <a href="/wiki/Time_Machine_(macOS)" title="Time Machine (macOS)">Time Machine</a> software, journaling can be disabled under macOS:<sup class="reference" id="cite_ref-superuser-force_21-0"><a href="#cite_note-superuser-force-21">[21]</a></sup> Using Disk Utility in OS X Yosemite, the user may hold Alt/Option and click "Disable Journaling" on the File menu, having first selected a mounted partition.</p>
<p>An HFS+ partition with journaling enabled may be forcibly mounted with write-access under Linux, but this is unsupported and unwise.<sup class="reference" id="cite_ref-superuser-force_21-1"><a href="#cite_note-superuser-force-21">[21]</a></sup><sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup></p>
<p>A <a href="/wiki/Google_Summer_of_Code" title="Google Summer of Code">Google Summer of Code</a> project to implement write-support to journaled HFS+ <sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup> was accepted by the Linux Foundation in 2011 but was not completed at that time and is still a work in progress. Progress and improvements to the HFS+ driver, including some updates to journaling support, are posted on the linux-fsdevel mailing list <sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup> from time to time.</p>
<p>As of July 2011, <a href="/wiki/Paragon_Software_Group" title="Paragon Software Group">Paragon Software Group</a> provided kernel drivers that allow full read-write on HFS+ journaled volumes.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> The product is a proprietary implementation of HFS+ based on Paragon's proprietary UFSD library. There are both free and paid editions of the driver, and they include a utility for checking and repairing HFS+ volumes. According to the online documentation (free version <sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup> or the paid edition <sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup>), both the free edition and the paid edition currently support Linux kernels from 2.6.36 up to 4.4.x. Only Ubuntu, Debian, Fedora, OpenSuse and CentOS are the Linux distributions officially supported. <sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup></p>
<h3><span class="mw-headline" id="Windows">Windows</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HFS_Plus&amp;action=edit&amp;section=6" title="Edit section: Windows">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>As of May 2012, Apple has only released read-only HFS+ drivers for Windows XP, Windows Vista, and Windows 7 as part of the <a href="/wiki/Boot_Camp_(software)" title="Boot Camp (software)">Boot Camp</a> software in <a href="/wiki/Mac_OS_X_Snow_Leopard" title="Mac OS X Snow Leopard">Mac OS X 10.6</a>.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> This means users on these systems can read data on the HFS+ drive, but not write to them. Microsoft has created an HFS+ driver for the Xbox 360 mainly for the purpose of reading HFS+ formatted iPods.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (October 2016)">citation needed</span></a></i>]</sup></p>
<p>A <a href="/wiki/Free_software" title="Free software">free</a> (<a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GPL</a>) alternative is HFSExplorer written by Erik Larsson.<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup> HFSExplorer is an application for viewing and extracting files from an HFS+ volume (Mac OS Extended) or an HFSX volume (Mac OS Extended, Case-sensitive) located either on a physical disk, on a <a href="/wiki/Apple_Disk_Image" title="Apple Disk Image">.dmg disk image</a>, or in a raw file system dump. However, HFSExplorer can only read from, but not write to, HFS formatted volumes.</p>
<p>A free and opensource software â jHFSplus, based on HFSExplorer and jpfm â can be used to mount hfs/hfs+ partitions as read-only virtual folders.<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup></p>
<p>A freeware plugin for <a href="/wiki/Total_Commander" title="Total Commander">Total Commander</a> exists, that can read, among others, HFS and HFS+ filesystems.<sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup></p>
<p>DiskInternals Linux Reader can be used to extract/save folders/files out of HFS and HFS+ Hard Drives/Partitions.<sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup></p>
<p>A commercial product, MacDrive, is also available for mounting HFS and HFS+ drives, optical discs, and other media in Windows Explorer, and allows both reading and writing to the volume, as well as repairing and formatting Mac disks.<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup></p>
<p>A commercial product, Paragon's HFS+ for Windows allows full read and write and disk management from all versions of Windows from Windows XP to Windows Server 2008.<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup></p>
