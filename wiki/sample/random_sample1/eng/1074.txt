<p>A <a href="/wiki/Web_server" title="Web server">web server</a> may return an HTTP <b>403 Forbidden</b> status in response to a request from a client for a web page, or it may indicate that the server can be reached and process the request but refuses to take any further action. Status code 403 responses are the result of the web server being configured to deny access to the requested resource by the client.</p>
<p>A common request that may result in a 403 Forbidden response is a <a class="mw-redirect" href="/wiki/GET_(HTTP)" title="GET (HTTP)">GET</a> for a web page, performed by a web browser to retrieve the page for display to a user in a browser window. The web server may return a 403 Forbidden status for other types of requests as well.</p>
<p>The <a href="/wiki/Apache_HTTP_Server" title="Apache HTTP Server">Apache</a> web server returns 403 Forbidden in response to requests for url paths that correspond to filesystem directories when directory listings have been disabled in the server and there is no Directory Index directive to specify an existing file to be returned to the browser. Some administrators configure the <a href="/wiki/Mod_proxy" title="Mod proxy">Mod proxy</a> extension to Apache to block such requests, and this will also return 403 Forbidden. Microsoft <a href="/wiki/Internet_Information_Services" title="Internet Information Services">IIS</a> responds in the same way when directory listings are denied in that server. In <a href="/wiki/WebDAV" title="WebDAV">WebDAV</a>, the 403 Forbidden response will be returned by the server if the client issued a PROPFIND request but did not also issue the required Depth header, or issued a Depth header of infinity.<sup class="reference" id="cite_ref-RFC_1-0"><a href="#cite_note-RFC-1">[1]</a></sup></p>

<h2><span class="mw-headline" id="Difference_from_status_.22401_Unauthorized.22">Difference from status "401 Unauthorized"</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTTP_403&amp;action=edit&amp;section=1" title='Edit section: Difference from status "401 Unauthorized"'>edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Status codes 401 (Unauthorized) and 403 (Forbidden) have distinct meanings.</p>
<p>A 401 response indicates that access to the resource is restricted, and the request did not provide any <a href="/wiki/Basic_access_authentication" title="Basic access authentication">HTTP authentication</a>. It is possible that a new request for the same resource will succeed if authentication is provided. The response must include an HTTP WWW-Authenticate header to prompt the user-agent to provide credentials. If valid credentials are not provided via HTTP Authorization, then 401 should not be used.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>A 403 response generally indicates one of two conditions:</p>
<ul>
<li>Authentication was provided, but the authenticated user is not permitted to perform the requested operation.</li>
<li>The operation is forbidden to all users. For example, requests for a directory listing return code 403 when directory listing has been disabled.</li>
</ul>

<p>The following nonstandard code are returned by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>'s <a href="/wiki/Internet_Information_Services" title="Internet Information Services">Internet Information Services</a> and are not officially recognized by <a href="/wiki/Internet_Assigned_Numbers_Authority" title="Internet Assigned Numbers Authority">IANA</a>.</p>
<ul>
<li>403.1 - Execute access forbidden.<sup class="reference" id="cite_ref-globalscape_3-0"><a href="#cite_note-globalscape-3">[3]</a></sup></li>
<li>403.2 - Read access forbidden.<sup class="reference" id="cite_ref-globalscape_3-1"><a href="#cite_note-globalscape-3">[3]</a></sup></li>
<li>403.3 - Write access forbidden.<sup class="reference" id="cite_ref-globalscape_3-2"><a href="#cite_note-globalscape-3">[3]</a></sup></li>
<li>403.4 - SSL required.<sup class="reference" id="cite_ref-globalscape_3-3"><a href="#cite_note-globalscape-3">[3]</a></sup></li>
<li>403.5 - SSL 128 required.<sup class="reference" id="cite_ref-globalscape_3-4"><a href="#cite_note-globalscape-3">[3]</a></sup></li>
<li>403.6 - IP address rejected.<sup class="reference" id="cite_ref-globalscape_3-5"><a href="#cite_note-globalscape-3">[3]</a></sup></li>
<li>403.7 - <a href="/wiki/Client_certificate" title="Client certificate">Client certificate</a> required.<sup class="reference" id="cite_ref-globalscape_3-6"><a href="#cite_note-globalscape-3">[3]</a></sup></li>
<li>403.8 - Site access denied.<sup class="reference" id="cite_ref-globalscape_3-7"><a href="#cite_note-globalscape-3">[3]</a></sup></li>
<li>403.9 - Too many users.<sup class="reference" id="cite_ref-globalscape_3-8"><a href="#cite_note-globalscape-3">[3]</a></sup></li>
<li>403.10 - Invalid configuration.<sup class="reference" id="cite_ref-globalscape_3-9"><a href="#cite_note-globalscape-3">[3]</a></sup></li>
<li>403.11 - Password change.</li>
<li>403.12 - Mapper denied access.</li>
<li>403.13 - Client certificate revoked.</li>
<li>403.14 - Directory listing denied.</li>
<li>403.15 - Client Access Licenses exceeded.</li>
<li>403.16 - Client certificate is untrusted or invalid.</li>
<li>403.17 - Client certificate has expired or is not yet valid.</li>
<li>403.18 - Cannot execute request from that application pool.</li>
<li>403.19 - Cannot execute CGIs for the client in this application pool.</li>
<li>403.20 - Passport logon failed.</li>
<li>403.21 - Source access denied.</li>
<li>403.22 - Infinite depth is denied.</li>
<li>403.502 - Too many requests from the same client IP; Dynamic IP Restriction limit reached.</li>
</ul>
