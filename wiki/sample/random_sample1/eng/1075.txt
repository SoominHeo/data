<p>The <b>404</b> or <b>Not Found</b> <a href="/wiki/Error_message" title="Error message">error message</a> is a <a href="/wiki/Hypertext_Transfer_Protocol" title="Hypertext Transfer Protocol">Hypertext Transfer Protocol</a> (HTTP) <a href="/wiki/List_of_HTTP_status_codes" title="List of HTTP status codes">standard response code</a>, in computer network communications, to indicate that the <a href="/wiki/Web_browser" title="Web browser">client</a> was able to communicate with a given <a href="/wiki/Server_(computing)" title="Server (computing)">server</a>, but the server could not find what was requested.</p>
<p>The web site hosting server will typically generate a "404 Not Found" web page when a user attempts to follow a <a href="/wiki/Link_rot" title="Link rot">broken or dead link</a>; hence the 404 error is one of the most recognizable errors encountered on the <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>.</p>

<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTTP_404&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>When communicating via HTTP, a server is required to respond to a request, such as a <a href="/wiki/Web_browser" title="Web browser">web browser</a> request for a <a href="/wiki/Web_page" title="Web page">web page</a>, with a numeric response code and an optional, mandatory, or disallowed (based upon the status code) message. In the code 404, the first digit indicates a client error, such as a mistyped <a href="/wiki/Uniform_Resource_Locator" title="Uniform Resource Locator">Uniform Resource Locator</a> (URL). The following two digits indicate the specific error encountered. HTTP's use of three-digit codes is similar to the use of such codes in earlier protocols such as <a href="/wiki/File_Transfer_Protocol" title="File Transfer Protocol">FTP</a> and <a href="/wiki/Network_News_Transfer_Protocol" title="Network News Transfer Protocol">NNTP</a>.</p>
<p>At the HTTP level, a 404 response code is followed by a human-readable "reason phrase". The HTTP specification suggests the phrase "Not Found"<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> and many web servers by default issue an HTML page that includes both the 404 code and the "Not Found" phrase.</p>
<p>A 404 error is often returned when pages have been moved or deleted. In the first case, it is better to employ <a class="mw-redirect" href="/wiki/URL_mapping" title="URL mapping">URL mapping</a> or <a href="/wiki/URL_redirection" title="URL redirection">URL redirection</a> by returning a 301 Moved Permanently response, which can be configured in most server configuration files, or through <a class="mw-redirect" href="/wiki/URL_rewriting" title="URL rewriting">URL rewriting</a>; in the second case, a 410 Gone should be returned. Because these two options require special server configuration, most websites do not make use of them.</p>
<p>404 errors should not be confused with <a href="/wiki/Domain_Name_System" title="Domain Name System">DNS</a> errors, which appear when the given URL refers to a server name that does not exist. A 404 error indicates that the server itself was found, but that the server was not able to retrieve the requested page.</p>
<h2><span class="mw-headline" id="Custom_error_pages">Custom error pages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTTP_404&amp;action=edit&amp;section=2" title="Edit section: Custom error pages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Wikimedia_error_404.png"><img alt="" class="thumbimage" data-file-height="1200" data-file-width="1600" height="165" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/e0/Wikimedia_error_404.png/220px-Wikimedia_error_404.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/e0/Wikimedia_error_404.png/330px-Wikimedia_error_404.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/e0/Wikimedia_error_404.png/440px-Wikimedia_error_404.png 2x" width="220"/></a>

<p><a href="/wiki/Web_server" title="Web server">Web servers</a> can typically be configured to display a customised 404 error page, including a more natural description, the parent site's branding, and sometimes a site map, a search form or 404 page widget. The protocol level phrase, which is hidden from the user, is rarely customized.</p>
<p><a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a>, however, will not display custom pages unless they are larger than 512 bytes, opting instead to display a "friendly" error page.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> <a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a> includes similar functionality, where the 404 is replaced with alternative suggestions generated by Google algorithms, if the page is under 512 bytes in size.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2013)">citation needed</span></a></i>]</sup></p>
<p>Another problem is that if the page does not provide a <a href="/wiki/Favicon" title="Favicon">favicon</a>, and a separate custom 404 page exists, extra traffic and longer loading times will be generated on every page view.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p>Many organizations use 404 error pages as an opportunity to inject humor into what may otherwise be a serious website. For example, Metro UK shows a polar bear on a skateboard, and the web development agency Left Logic has a simple drawing program.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<p>While many websites send additional information in a 404 error message—such as a link to the <a class="mw-redirect" href="/wiki/Homepage" title="Homepage">homepage</a> of a website or a <a href="/wiki/Search_box" title="Search box">search box</a>—some also endeavor to find the correct web page the user wanted. Extensions are available for some popular <a href="/wiki/Content_management_system" title="Content management system">content management systems (CMSs)</a> to do this.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>In Europe, the <a class="mw-redirect" href="/wiki/NotFound_project" title="NotFound project">NotFound project</a>, created by multiple European organizations including <a href="/wiki/Missing_Children_Europe" title="Missing Children Europe">Missing Children Europe</a> and <a href="/wiki/Child_Focus" title="Child Focus">Child Focus</a>, encourages site operators to add a <a href="/wiki/Snippet_(programming)" title="Snippet (programming)">snippet of code</a> to serve customised 404 error pages<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> which provide data about <a class="mw-redirect" href="/wiki/Missing_children" title="Missing children">missing children</a>.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<h3><span class="mw-headline" id="Tracking.2FChecking_404_errors">Tracking/Checking 404 errors</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTTP_404&amp;action=edit&amp;section=3" title="Edit section: Tracking/Checking 404 errors">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A number of tools exist that crawl through a website to find pages that return 404 status codes. These tools can be helpful in finding links that exist within a particular website. The limitation of these tools is that they only find links within one particular website, and ignore 404s resulting from links on other websites. As a result, these tools miss out on 83% of the 404s on websites.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> One way around this is to find 404 errors by analyzing external links.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></p>
<p>Another common method is tracking traffic to 404 pages using log file analysis.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> This can be useful to understand more about what 404s users reached on the site. Another method of tracking traffic to 404 pages is using JavaScript-based traffic tracking tools.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<h2><span class="mw-headline" id="Soft_404_Errors">Soft 404 Errors</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTTP_404&amp;action=edit&amp;section=4" title="Edit section: Soft 404 Errors">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><span id="Soft_404"></span>Some websites report a "not found" error by returning a standard web page with a "200 OK" response code, falsely reporting that the page loaded properly; this is known as a <b>soft 404</b>. Soft 404s are problematic for automated methods of discovering whether a link is broken. Some search engines, like Yahoo, use automated processes to detect soft 404s.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> Soft 404s can occur as a result of configuration errors when using certain HTTP server software, for example with the <a href="/wiki/Apache_HTTP_Server" title="Apache HTTP Server">Apache</a> software, when an Error Document 404 (specified in a <a href="/wiki/.htaccess" title=".htaccess">.htaccess</a> file) is specified as an absolute path (e.g. http://example.com/error.html) rather than a relative path (/error.html).<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> This can also be done on purpose to force some browsers (like <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a>) to display a customized 404 error message rather than replacing what is served with a browser-specific "friendly" error message (in Internet Explorer, this behavior is triggered when a 404 is served and the received HTML is shorter than a certain length, and can be manually disabled by the user).</p>
<p>Some proxy servers generate a 404 error when the remote host is not present, rather than returning the correct 500-range code when errors such as hostname resolution failures or refused TCP connections prevent the proxy server from satisfying the request. This can confuse programs that expect and act on specific responses, as they can no longer easily distinguish between an absent web server and a missing web page on a web server that is present.</p>
<p>In July 2004, the UK telecom provider <a href="/wiki/BT_Group" title="BT Group">BT Group</a> deployed the <a href="/wiki/Cleanfeed_(content_blocking_system)" title="Cleanfeed (content blocking system)">Cleanfeed</a> content blocking system, which returns a 404 error to any request for content identified as potentially illegal by the <a href="/wiki/Internet_Watch_Foundation" title="Internet Watch Foundation">Internet Watch Foundation</a>.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> Other ISPs return a <a href="/wiki/HTTP_403" title="HTTP 403">HTTP 403</a> "forbidden" error in the same circumstances.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> The practice of employing fake 404 errors as a means to conceal <a href="/wiki/Censorship" title="Censorship">censorship</a> has also been reported in <a href="/wiki/Thailand" title="Thailand">Thailand</a><sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> and <a href="/wiki/Tunisia" title="Tunisia">Tunisia</a>.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> In Tunisia, where censorship was severe before the <a href="/wiki/Tunisian_Revolution" title="Tunisian Revolution">2011 revolution</a>, people became aware of the nature of the fake 404 errors and created an imaginary character named "<a class="mw-redirect" href="/wiki/Ammar_404#Censorship_during_the_Ben_Ali_regime" title="Ammar 404">Ammar 404</a>" who represents "the invisible censor".<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup></p>
<h2><span class="mw-headline" id="Microsoft_Internet_Server_404_substatus_error_codes">Microsoft Internet Server 404 substatus error codes</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTTP_404&amp;action=edit&amp;section=5" title="Edit section: Microsoft Internet Server 404 substatus error codes">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The webserver software developed by Microsoft, <a href="/wiki/Internet_Information_Services" title="Internet Information Services">Microsoft's Internet Information Services</a> (IIS), returns a set of substatus codes with its 404 responses. The substatus codes take the form of decimal numbers appended to the 404 status code. The substatus codes are not officially recognized by <a href="/wiki/Internet_Assigned_Numbers_Authority" title="Internet Assigned Numbers Authority">IANA</a> and are not returned by non-Microsoft servers.</p>
<h3><span class="mw-headline" id="Substatus_codes">Substatus codes</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTTP_404&amp;action=edit&amp;section=6" title="Edit section: Substatus codes">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Microsoft's IIS 7.0, IIS 7.5, and IIS 8.0 servers define the following HTTP substatus codes to indicate a more specific cause of a 404 error:</p>
<ul>
<li>404.0 – Not found.</li>
<li>404.1 – Site Not Found.</li>
<li>404.2 – <a class="mw-redirect" href="/wiki/ISAPI" title="ISAPI">ISAPI</a> or <a href="/wiki/Common_Gateway_Interface" title="Common Gateway Interface">CGI</a> restriction.</li>
<li>404.3 – <a href="/wiki/MIME" title="MIME">MIME</a> type restriction.</li>
<li>404.4 – No handler configured.</li>
<li>404.5 – Denied by request filtering configuration.</li>
<li>404.6 – Verb denied.</li>
<li>404.7 – File extension denied.</li>
<li>404.8 – Hidden namespace.</li>
<li>404.9 – File attribute hidden.</li>
<li>404.10 – Request header too long.</li>
<li>404.11 – Request contains double escape sequence.</li>
<li>404.12 – Request contains high-bit characters.</li>
<li>404.13 – Content length too large.</li>
<li>404.14 – Request URL too long.</li>
<li>404.15 – Query string too long.</li>
<li>404.16 – DAV request sent to the static file handler.</li>
<li>404.17 – Dynamic content mapped to the static file handler via a wildcard MIME mapping.</li>
<li>404.18 – Query string sequence denied.</li>
<li>404.19 – Denied by filtering rule.</li>
<li>404.20 – Too Many URL Segments.</li>
</ul>
<h2><span class="mw-headline" id="Slang_use_of_404">Slang use of 404</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTTP_404&amp;action=edit&amp;section=7" title="Edit section: Slang use of 404">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In 2008, a study carried out by the telecommunications arm of the <a href="/wiki/Royal_Mail" title="Royal Mail">Royal Mail</a><sup class="reference" id="cite_ref-autogenerated1_20-0"><a href="#cite_note-autogenerated1-20">[20]</a></sup> found that "404" became a slang synonym for "clueless" in the <a href="/wiki/United_Kingdom" title="United Kingdom">United Kingdom</a>. Slang <a class="mw-redirect" href="/wiki/Lexicographer" title="Lexicographer">lexicographer</a> Jonathon Green said that "404" as a slang term had been driven by the "influence of technology" and young people, but at the time, such usage was relatively confined to London and <a class="mw-redirect" href="/wiki/List_of_conurbations_in_the_United_Kingdom" title="List of conurbations in the United Kingdom">other urban areas</a>.<sup class="reference" id="cite_ref-autogenerated1_20-1"><a href="#cite_note-autogenerated1-20">[20]</a></sup></p>
