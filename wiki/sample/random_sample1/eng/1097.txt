<p><a href="/wiki/IBM" title="IBM">IBM</a> has a series of high performance <a href="/wiki/Microprocessor" title="Microprocessor">microprocessors</a> called <b>POWER</b> followed by a number designating generation, i.e. <i>POWER1</i>, <i>POWER2</i>, <i>POWER3</i> and so forth up to the latest <i>POWER9</i>. These processor have been used by IBM in their <a href="/wiki/RS/6000" title="RS/6000">RS/6000</a>, <a class="mw-redirect" href="/wiki/AS/400" title="AS/400">AS/400</a>, <a class="mw-redirect" href="/wiki/PSeries" title="PSeries">pSeries</a>, <a class="mw-redirect" href="/wiki/ISeries" title="ISeries">iSeries</a>, <a class="mw-redirect" href="/wiki/System_p" title="System p">System p</a>, <a href="/wiki/IBM_System_i" title="IBM System i">System i</a> and <a href="/wiki/IBM_Power_Systems" title="IBM Power Systems">Power Systems</a> line of <a href="/wiki/Server_(computing)" title="Server (computing)">servers</a> and <a href="/wiki/Supercomputer" title="Supercomputer">supercomputers</a>. They have also been used in <a href="/wiki/Data_storage_device" title="Data storage device">data storage devices</a> by IBM and by other server manufacturers like <a href="/wiki/Groupe_Bull" title="Groupe Bull">Bull</a> and <a href="/wiki/Hitachi" title="Hitachi">Hitachi</a>.</p>
<p>The POWER<i>n</i> family of processors were developed in the late 1980s and are still in active development nearly 30 years later. In the beginning, they utilized the <a href="/wiki/IBM_POWER_Instruction_Set_Architecture" title="IBM POWER Instruction Set Architecture">POWER instruction set architecture</a> (ISA), but that evolved into <a href="/wiki/PowerPC" title="PowerPC">PowerPC</a> in later generations and then to <a href="/wiki/Power_Architecture" title="Power Architecture">Power Architecture</a>. Today, only the naming scheme remains the same; modern POWER processors do not use the POWER ISA.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Early_developments">Early developments</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=2" title="Edit section: Early developments">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<h4><span class="mw-headline" id="The_801_research_project">The 801 research project</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=3" title="Edit section: The 801 research project">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<div class="hatnote" role="note">Main article: <a href="/wiki/IBM_801" title="IBM 801">IBM 801</a></div>
<p>In 1974 IBM started a project to build a telephone switching computer with, for the time, immense computational power. Since the application was comparably simple, this machine would need only to perform <a href="/wiki/Input/output" title="Input/output">I/O</a>, <a href="/wiki/Branch_(computer_science)" title="Branch (computer science)">branches</a>, add <a href="/wiki/Processor_register" title="Processor register">register-register</a>, move data between <a href="/wiki/Processor_register" title="Processor register">registers</a> and <a href="/wiki/Random-access_memory" title="Random-access memory">memory</a>, and would have no need for special instructions to perform heavy arithmetic. This simple design philosophy, whereby each step of a complex operation is specified explicitly by one machine instruction, and all instructions are required to complete in the same constant time, would later come to be known as <a class="mw-redirect" href="/wiki/RISC" title="RISC">RISC</a>. When the telephone switch project was cancelled IBM kept the design for the general purpose processor and named it <b>801</b> after building #801 at <a href="/wiki/Thomas_J._Watson_Research_Center" title="Thomas J. Watson Research Center">Thomas J. Watson Research Center</a>.</p>
<h4><span class="mw-headline" id="The_Cheetah_project">The Cheetah project</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=4" title="Edit section: The Cheetah project">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>By 1982 IBM continued to explore the <a class="mw-redirect" href="/wiki/Superscalar" title="Superscalar">superscalar</a> limits of the 801 design by using multiple <a class="mw-redirect" href="/wiki/Execution_units" title="Execution units">execution units</a> to improve performance to determine if a RISC machine could maintain multiple instructions per cycle. Many changes were made to the 801 design to allow for multiple execution units and the Cheetah processor had separate <a href="/wiki/Branch_predictor" title="Branch predictor">branch prediction</a>, <a href="/wiki/Arithmetic_logic_unit" title="Arithmetic logic unit">fixed-point</a>, and <a href="/wiki/Floating-point_unit" title="Floating-point unit">floating-point</a> execution units. By 1984 <a href="/wiki/CMOS" title="CMOS">CMOS</a> was chosen since it allowed an increase in the level of circuit integration while improving transistor-logic performance.</p>
<h4><span class="mw-headline" id="The_America_project">The America project</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=5" title="Edit section: The America project">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>In 1985, research on a second-generation RISC architecture started at the IBM Thomas J. Watson Research Center, producing the "AMERICA architecture"; in 1986, <a href="/wiki/IBM_Austin_Research_Laboratory" title="IBM Austin Research Laboratory">IBM Austin</a> started developing the <a href="/wiki/RS/6000" title="RS/6000">RS/6000</a> series computers based on that architecture. This was to become the first POWER processors using the first POWER ISA.</p>
<h3><span class="mw-headline" id="POWER">POWER</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=6" title="Edit section: POWER">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/POWER1" title="POWER1">POWER1</a></div>
<div class="thumb tright">
<div class="thumbinner" style="width:502px;"><a class="image" href="/wiki/File:PowerISA-evolution.svg"><img alt="" class="thumbimage" data-file-height="484" data-file-width="2486" height="97" src="//upload.wikimedia.org/wikipedia/commons/thumb/3/3b/PowerISA-evolution.svg/500px-PowerISA-evolution.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/3/3b/PowerISA-evolution.svg/750px-PowerISA-evolution.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/3/3b/PowerISA-evolution.svg/1000px-PowerISA-evolution.svg.png 2x" width="500"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:PowerISA-evolution.svg" title="Enlarge"></a></div>
A schematic showing the evolution of the different <a href="/wiki/IBM_POWER_Instruction_Set_Architecture" title="IBM POWER Instruction Set Architecture">POWER</a>, <a href="/wiki/PowerPC" title="PowerPC">PowerPC</a> and Power <a class="mw-redirect" href="/wiki/Instruction_set_architecture" title="Instruction set architecture">ISAs</a></div>
</div>
</div>
<p>In February 1990, the first computers from IBM to incorporate the POWER ISA were called the "RISC System/6000" or <a href="/wiki/RS/6000" title="RS/6000">RS/6000</a>. These RS/6000 computers were divided into two classes, <a href="/wiki/Workstation" title="Workstation">workstations</a> and <a href="/wiki/Server_(computing)" title="Server (computing)">servers</a>, and hence introduced as the POWERstation and POWERserver. The RS/6000 CPU had 2 configurations, called the "RIOS-1" and "RIOS.9" (or more commonly the <i>POWER1</i> CPU). A RIOS-1 configuration had a total of 10 discrete chips — an instruction cache chip, fixed-point chip, floating-point chip, 4 <a href="/wiki/CPU_cache" title="CPU cache">data L1 cache</a> chips, storage control chip, input/output chips, and a clock chip. The lower cost RIOS.9 configuration had 8 discrete chips—an instruction cache chip, fixed-point chip, floating-point chip, 2 data cache chips, storage control chip, input/output chip, and a clock chip.</p>
<p>The POWER1 is the first microprocessor that used <a href="/wiki/Register_renaming" title="Register renaming">register renaming</a> and <a href="/wiki/Out-of-order_execution" title="Out-of-order execution">out-of-order execution</a>. A simplified and less powerful version of the 10 chip RIOS-1 was made in 1992 was developed for lower-end RS/6000s. It used only one chip and was called "<a href="/wiki/RISC_Single_Chip" title="RISC Single Chip">RISC Single Chip</a>" or <i>RSC</i>.</p>
<h4><span class="mw-headline" id="POWER1_processors">POWER1 processors</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=7" title="Edit section: POWER1 processors">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li><i>RIOS-1</i> –  the original 10-chip version</li>
<li><i>RIOS.9</i> –  a less powerful version of RIOS-1</li>
<li><i>POWER1+</i> –  a faster version of RIOS-1 made on a reduced fabrication process</li>
<li><i>POWER1++</i> –  an even faster version of RIOS-1</li>
<li><i>RSC</i> –  a single-chip implementation of RIOS-1</li>
<li><i>RAD6000</i> –  a radiation-hardened version of the RSC was made available for primarily use in space; it was a very popular design and was used extensively on many high-profile missions</li>
</ul>
<h3><span class="mw-headline" id="POWER2">POWER2</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=8" title="Edit section: POWER2">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/POWER2" title="POWER2">POWER2</a></div>
<p>IBM started the <a href="/wiki/POWER2" title="POWER2">POWER2</a> processor effort as a successor to the POWER1. By adding a second fixed-point unit, a second powerful floating point unit, and other performance enhancements and new instructions to the design, the POWER2 ISA had leadership performance when it was announced in November 1993. The POWER2 was a multi-chip design, but IBM also made a single chip design of it, called the <i>POWER2 Super Chip</i> or <i>P2SC</i> that went into high performance servers and supercomputers. At the time of its introduction in 1996, the P2SC was the largest processor with the highest transistor count in the industry and was a leader in floating point operations.</p>
<h4><span class="mw-headline" id="POWER2_processors">POWER2 processors</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=9" title="Edit section: POWER2 processors">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li><i>POWER2</i> –  6 to 8 chips were mounted on a <a class="mw-redirect" href="/wiki/High_temperature_co-fired_ceramic" title="High temperature co-fired ceramic">ceramic</a> <a href="/wiki/Multi-chip_module" title="Multi-chip module">multi chip module</a></li>
<li><i>POWER2+</i> –  a cheaper 6-chip version of POWER2 with support for external L2 caches</li>
<li><i>P2SC</i> –  a faster and single chip version of POWER2</li>
<li><i>P2SC+</i> –  an even faster version or P2SC due to reduced fabrication process</li>
</ul>
<h3><span class="mw-headline" id="PowerPC">PowerPC</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=10" title="Edit section: PowerPC">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/PowerPC" title="PowerPC">PowerPC</a></div>
<p>In 1991, <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple</a> looked for a future alternative to <a href="/wiki/Motorola" title="Motorola">Motorola</a>'s <a class="mw-redirect" href="/wiki/68000" title="68000">68000</a>-based <a href="/wiki/Complex_instruction_set_computing" title="Complex instruction set computing">CISC</a> platform, and Motorola experimented with a RISC platform of its own, the <a class="mw-redirect" href="/wiki/88000" title="88000">88000</a>. IBM joined the discussion and the three founded the <a href="/wiki/AIM_alliance" title="AIM alliance">AIM alliance</a> to build the <a href="/wiki/PowerPC" title="PowerPC">PowerPC</a> ISA, heavily based on the POWER ISA, but with additions from both Apple and Motorola. It was to be a complete <a href="/wiki/Word_(computer_architecture)" title="Word (computer architecture)">32/64 bit</a> RISC architecture, with a promise to range from very low end <a href="/wiki/Embedded_system" title="Embedded system">embedded</a> <a href="/wiki/Microcontroller" title="Microcontroller">microcontrollers</a> to the very high end <a href="/wiki/Supercomputer" title="Supercomputer">supercomputer</a> and server applications.</p>
<p>After two years of development, the resulting <a href="/wiki/PowerPC" title="PowerPC">PowerPC</a> ISA was introduced in 1993. A modified version of the RSC architecture, PowerPC added <a href="/wiki/Single-precision_floating-point_format" title="Single-precision floating-point format">single-precision floating point instructions</a> and general register-to-register multiply and divide instructions, and removed some POWER features. It also added a 64-bit version of the ISA and support for <a href="/wiki/Symmetric_multiprocessing" title="Symmetric multiprocessing">SMP</a>.</p>
<h3><span class="mw-headline" id="The_Amazon_project">The Amazon project</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=11" title="Edit section: The Amazon project">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/IBM_RS64" title="IBM RS64">IBM RS64</a></div>
<p>In 1990, IBM wanted to merge the low end server and mid range server architectures, the RS/6000 RISC ISA and AS/400 CISC ISA into one common RISC ISA that could host both IBM's <a class="mw-redirect" href="/wiki/AIX" title="AIX">AIX</a> and <a class="mw-redirect" href="/wiki/OS/400" title="OS/400">OS/400</a> operating systems. The existing POWER and the upcoming PowerPC ISAs were deemed unsuitable by the AS/400 team so an extension to the 64-bit PowerPC instruction set was developed called PowerPC AS for <i>Advances Series</i> or <i>Amazon Series</i>. Later, additions from the RS/6000 team and AIM Alliance PowerPC were included, and by 2001, with the introduction of POWER4, they were all joined into one instruction set architecture: the PowerPC v.2.0.</p>
<h3><span class="mw-headline" id="POWER3">POWER3</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=12" title="Edit section: POWER3">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/POWER3" title="POWER3">POWER3</a></div>
<p>The POWER3 began its life as "PowerPC 630", a successor of the commercially unsuccessful <a class="mw-redirect" href="/wiki/PowerPC_620" title="PowerPC 620">PowerPC 620</a>. It used a combination of the POWER2 ISA and the 32/64-bit PowerPC ISA set with support for SMP and single-chip implementation. It was used to great extent in IBM's RS/6000 computers, while the second generation version, the POWER3-II, was the first commercially available processor from IBM using <a href="/wiki/Copper_interconnect" title="Copper interconnect">copper interconnects</a>. The POWER3 was the last processor to use a POWER instruction set; all subsequent models used some version of the PowerPC instruction set.</p>
<h4><span class="mw-headline" id="POWER3_processors">POWER3 processors</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=13" title="Edit section: POWER3 processors">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li><b>POWER3</b> – Introduced in 1998, it combined the POWER and PowerPC instruction sets.</li>
<li><b>POWER3-II</b> – A faster POWER3 fabricated on a reduced size, copper based process.</li>
</ul>
<h3><span class="mw-headline" id="POWER4">POWER4</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=14" title="Edit section: POWER4">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/POWER4" title="POWER4">POWER4</a></div>
<p>The POWER4 merged the 32/64 bit PowerPC instruction set and the 64-bit PowerPC AS instruction set from the Amazon project to the new PowerPC v.2.0 specification, unifying IBM's RS/6000 and AS/400 families of computers. Besides the unification of the different platforms, POWER4 was also designed to reach very high <a href="/wiki/Clock_rate" title="Clock rate">frequencies</a> and have large on-die L2 caches. It was the first commercially available <a href="/wiki/Multi-core_processor" title="Multi-core processor">multi-core processor</a> and came in single-die versions as well as in four-chip multi-chip modules. In 2002, IBM also made a cost- and feature-reduced version of the POWER4 called <a href="/wiki/PowerPC_970" title="PowerPC 970">PowerPC 970</a> by Apple's request.</p>
<h4><span class="mw-headline" id="POWER4_processors">POWER4 processors</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=15" title="Edit section: POWER4 processors">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li><b>POWER4</b> – The first dual core microprocessor and the first PowerPC processor to reach beyond 1 GHz.</li>
<li><b>POWER4+</b> – A faster POWER4 fabricated on a reduced process.</li>
</ul>
<h3><span class="mw-headline" id="POWER5">POWER5</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=16" title="Edit section: POWER5">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/POWER5" title="POWER5">POWER5</a></div>
<p>The POWER5 processors built on the popular POWER4 and incorporated <a href="/wiki/Simultaneous_multithreading" title="Simultaneous multithreading">simultaneous multithreading</a> into the design, a technology pioneered in the PowerPC AS based <a class="mw-redirect" href="/wiki/RS64#RS64-III" title="RS64">RS64-III</a> processor, and on-die <a href="/wiki/Memory_controller" title="Memory controller">memory controllers</a>. It was designed for multiprocessing on a massive scale and came in multi-chip modules with onboard large L3 cache chips.</p>
<h4><span class="mw-headline" id="POWER5_processors">POWER5 processors</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=17" title="Edit section: POWER5 processors">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li><b>POWER5</b> – The iconic setup with four POWER5 chips and four L3 cache chips on a large multi-chip module.</li>
<li><b>POWER5+</b> – A faster POWER5 fabricated on a reduced process mainly to reduce power consumption.</li>
</ul>
<h3><span class="mw-headline" id="Power_Architecture">Power Architecture</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=18" title="Edit section: Power Architecture">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Power_Architecture" title="Power Architecture">Power Architecture</a></div>
<p>A joint organization was founded in 2004 called <a href="/wiki/Power.org" title="Power.org">Power.org</a> with the mission to unify and coordinate future development of the PowerPC specifications. By then, the PowerPC specification was fragmented since <a class="mw-redirect" href="/wiki/Freescale" title="Freescale">Freescale</a> (née Motorola) and IBM had taken different paths in their respective development of it. Freescale had prioritized 32-bit embedded applications and IBM high-end servers and supercomputers. There was also a collection of licensees of the specification like <a href="/wiki/Applied_Micro_Circuits_Corporation" title="Applied Micro Circuits Corporation">AMCC</a>, <a href="/wiki/Synopsys" title="Synopsys">Synopsys</a>, <a href="/wiki/Sony" title="Sony">Sony</a>, <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>, <a href="/wiki/P.A._Semi" title="P.A. Semi">P.A. Semi</a>, <a class="mw-redirect" href="/wiki/CRAY" title="CRAY">CRAY</a> and <a href="/wiki/Xilinx" title="Xilinx">Xilinx</a> that needed coordination. The joint effort was not only to streamline development of the technology but also to streamline marketing.</p>
<p>The new instruction set architecture was called <a href="/wiki/Power_Architecture" title="Power Architecture">Power Architecture</a> and merged the PowerPC v.2.02 from the POWER5 with the PowerPC Book E specification from Freescale as well as some related technologies like the Vector-Media Extensions known under the brand name <a href="/wiki/AltiVec" title="AltiVec">AltiVec</a> (also called <b>VMX</b> by IBM) and <a href="/wiki/Hardware_virtualization" title="Hardware virtualization">hardware virtualization</a>. This new ISA was called <b>Power ISA v.2.03</b> and POWER5 was the first high end processor from IBM to use it. Older POWER and PowerPC specifications did not make the cut and those instruction sets were henceforth <a href="/wiki/Deprecation" title="Deprecation">deprecated</a> for good. There is no active development on any processor type today that uses these older instruction sets.</p>
<h3><span class="mw-headline" id="POWER6">POWER6</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=19" title="Edit section: POWER6">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/POWER6" title="POWER6">POWER6</a></div>
<p>POWER6 was the fruit of the ambitions <i>eCLipz Project</i>, joining the <i>I</i> (AS/400), <i>P</i> (RS/6000) and <a href="/wiki/IBM_mainframe" title="IBM mainframe"><i>Z</i> (Mainframe)</a> instruction sets under one common platform. I and P was already joined with the POWER4, but the eCLipz effort failed to include the CISC based <a href="/wiki/Z/Architecture" title="Z/Architecture">z/Architecture</a> and where the <a class="mw-redirect" href="/wiki/IBM_z10_(microprocessor)" title="IBM z10 (microprocessor)">z10 processor</a> became POWER6's eCLipz sibling. z/Architecture remains a separate design track to this day not related to Power Architecture instruction set in any way.</p>
<p>Because of eCLipz, the POWER6 is an unusual design as it aimed for very high frequencies and sacrificed <a href="/wiki/Out-of-order_execution" title="Out-of-order execution">out-of-order execution</a>, something that has been a feature for POWER and PowerPC processors since their inception. POWER6 also introduced the <a href="/wiki/Decimal_floating_point" title="Decimal floating point">decimal floating point</a> unit to the Power ISA, something it shares with z/Architecture.</p>
<p>With the POWER6, in 2008 IBM merged the former <a class="mw-redirect" href="/wiki/System_p" title="System p">System p</a> and <a href="/wiki/IBM_System_i" title="IBM System i">System i</a> server and workstation families into one family called <a href="/wiki/IBM_Power_Systems" title="IBM Power Systems">Power Systems</a>. Power Systems machines can run different operating systems like <a class="mw-redirect" href="/wiki/AIX" title="AIX">AIX</a>, <a href="/wiki/Linux" title="Linux">Linux</a> and <a href="/wiki/IBM_i" title="IBM i">IBM i</a>.</p>
<h4><span class="mw-headline" id="POWER6_processors">POWER6 processors</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=20" title="Edit section: POWER6 processors">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li><b>POWER6</b> – Reached 5 GHz; comes in modules with a single chip on it, and in MCM with two L3 cache chips.</li>
<li><b>POWER6+</b> – A minor update, fabricated on the same process as POWER6.</li>
</ul>
<h3><span class="mw-headline" id="POWER7">POWER7</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=21" title="Edit section: POWER7">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/POWER7" title="POWER7">POWER7</a></div>
<p>The POWER7 <a href="/wiki/Symmetric_multiprocessing" title="Symmetric multiprocessing">symmetric multiprocessor</a> design was a substantial evolution from the POWER6 design, focusing more on power efficiency through multiple cores, simultaneous multithreading (SMT), out-of-order execution and large on-die eDRAM L3 caches. The eight-core chip could execute 32 threads in parallel, and has a mode in which it could disable cores to reach higher frequencies for the ones that are left. It uses a new high-performance floating point unit called VSX that merges the functionality of the traditional FPU with AltiVec. Even while the POWER7 run at lower frequencies than POWER6, each POWER7 core performed faster than its POWER6 counterpart.</p>
<h4><span class="mw-headline" id="POWER7_processors">POWER7 processors</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=22" title="Edit section: POWER7 processors">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li><b>POWER7</b> – Comes in single-chip modules or in quad-chip MCM-configurations for supercomputer applications.</li>
<li><b>POWER7+</b> – Scaled down fabrication process, and increased L3 cache and frequency.</li>
</ul>
<h3><span class="mw-headline" id="POWER8">POWER8</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=23" title="Edit section: POWER8">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/POWER8" title="POWER8">POWER8</a></div>
<p>POWER8 is a 4 GHz, 12 core processor with 8 hardware threads per core for a total of 96 threads of parallel execution. It uses 96 <a class="mw-redirect" href="/wiki/MiB" title="MiB">MB</a> of eDRAM L3 cache on chip and 128 MB off-chip L4 cache and a new extension bus called CAPI that runs on top of PCIe, replacing the older <a href="/wiki/PowerPC_600#6XX_and_GX_buses" title="PowerPC 600">GX bus</a>. The CAPI bus can be used to attach dedicated off-chip accelerator chips such as <a class="mw-redirect" href="/wiki/GPUs" title="GPUs">GPUs</a>, <a href="/wiki/Application-specific_integrated_circuit" title="Application-specific integrated circuit">ASICs</a> and <a class="mw-redirect" href="/wiki/FPGA" title="FPGA">FPGAs</a>. IBM states that it is two to three times as fast as its predecessor, the POWER7.</p>
<p>It will be built on a <a href="/wiki/22_nanometer" title="22 nanometer">22 nanometer</a> process at the end of 2013 or early 2014.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> In December 2012, IBM began submitting patches to the 3.8 version of the <a href="/wiki/Linux_kernel" title="Linux kernel">Linux kernel</a>, to support new POWER8 features including the VSX-2 instructions.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<h3><span class="mw-headline" id="POWER9">POWER9</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IBM_POWER_microprocessors&amp;action=edit&amp;section=24" title="Edit section: POWER9">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/POWER9" title="POWER9">POWER9</a></div>
<p>IBM has been designing the future POWER9 processor for quite a while according to William Starke, a systems architect for the POWER8 processor.<sup class="reference" id="cite_ref-TheRegHotChips2013_4-0"><a href="#cite_note-TheRegHotChips2013-4">[4]</a></sup> The POWER9 will be the first to incorporate elements of the Power ISA version 3.0 that was released in December 2015, including the VSX-3 instructions, and will also incorporate support for <a href="/wiki/Nvidia" title="Nvidia">Nvidia</a>'s <a href="/wiki/NVLink" title="NVLink">NVLink</a> bus technology.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup><sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>The <a href="/wiki/United_States_Department_of_Energy" title="United States Department of Energy">United States Department of Energy</a> together with <a href="/wiki/Oak_Ridge_National_Laboratory" title="Oak Ridge National Laboratory">Oak Ridge National Laboratory</a> and <a href="/wiki/Lawrence_Livermore_National_Laboratory" title="Lawrence Livermore National Laboratory">Lawrence Livermore National Laboratory</a> have contracted IBM and <a href="/wiki/Nvidia" title="Nvidia">Nvidia</a> to build two supercomputers, the <i>Summit</i> and the <i>Sierra</i>, that will be based on POWER9 processors coupled with Nvidia's <a href="/wiki/GeForce#VOLTA" title="GeForce">Volta</a> GPUs. These systems are slated to go online in 2017.<sup class="reference" id="cite_ref-sc14-power9_7-0"><a href="#cite_note-sc14-power9-7">[7]</a></sup><sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup><sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
<p>POWER9, coming in 2017, will be manufactured using a <a class="mw-redirect" href="/wiki/14_nm" title="14 nm">14 nm</a> <a class="mw-redirect" href="/wiki/FinFET" title="FinFET">FinFET</a> process, and will come in at least two versions, two 24 core versions for <a href="/wiki/Scalability#Horizontal_and_vertical_scaling" title="Scalability">scale up</a> and <a href="/wiki/Scalability#Horizontal_and_vertical_scaling" title="Scalability">scale out</a> applications, and possible more since the POWER9 architecture is open for licensing and modification by the <a href="/wiki/OpenPOWER_Foundation" title="OpenPOWER Foundation">OpenPOWER Foundation</a> members.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></p>
