<p><b>IC Manage</b> is a company that provides design data management software to <a href="/wiki/Semiconductor" title="Semiconductor">semiconductor</a> companies for managing their <a class="mw-redirect" href="/wiki/IC_design" title="IC design">IC design</a> databases between design teams. IC Manage also provides enterprise software for large scale <a class="mw-redirect" href="/wiki/Perforce" title="Perforce">Perforce</a> customers. IC Manage provides a structured IC design workflow for <a class="mw-redirect" href="/wiki/Analog_electronics" title="Analog electronics">analog electronics</a> and <a href="/wiki/Digital_electronics" title="Digital electronics">digital electronics</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IC_Manage&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In 2004, IC Manage introduced the beta version of its commercial design data management software system that provided <a href="/wiki/Version_control" title="Version control">version control</a>, configuration management and <a href="/wiki/Software_bug" title="Software bug">software bug</a> tracking. The design management system included a graphical <a class="mw-redirect" href="/wiki/Cadence_Design" title="Cadence Design">Cadence Design</a> Framework user interface and worked with both the Cadence and the <a href="/wiki/OpenAccess" title="OpenAccess">OpenAccess</a> databases. It tracked only modified files and stored <a href="/wiki/Metadata" title="Metadata">metadata</a> about revisions and configurations in a <a href="/wiki/Relational_database" title="Relational database">relational database</a> separate from the design data.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>In 2007, IC Manage announced its Global Design Platform (GDP) design data management system. GDP had bi-directional component tracking for IC revisions and derivatives; mixing, match, and reuse of components and <a href="/wiki/Semiconductor_intellectual_property_core" title="Semiconductor intellectual property core">semiconductor intellectual property cores</a> across local and remote design sites; <a href="/wiki/Defect_tracking" title="Defect tracking">defect tracking</a> and workspace synchronization to the same state when a designer reported, fixed, or verified a software bug.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<p>In 2007, Nvidia manufactured over 100 chips that used IC Manage software during design.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p>In 2009, IC Manage content management software was mentioned as a technology driver enabling semiconductor companies to do design work across local and remote geographic locations, as part of a significant trend toward outsourcing design work to other countries. IC Manage's software creates a workspace that abstracts the design data from the underlying directory organization.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<p>In 2011, Deepchip.com ranked IC Manage IP Central (IP Pro) as the #1 item to see at the Design Automation conference.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>From 2010 through 2014, IC Manage has been included in the annual "What to See at <a href="/wiki/Design_Automation_Conference" title="Design Automation Conference">DAC</a>" list by the GarySmith EDA analyst firm. In 2010 for Enterprise Tools,<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> in 2011 for Design Management,<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> in 2012 and 2013 for Design Debug,<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> and in 2014 for Enterprise tools.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></p>
<p>In 2014, IC Manage was selected as one of <a href="/wiki/Deloitte_Fast_500" title="Deloitte Fast 500">Deloitte Fast 500</a>, a ranking of the 500 fastest growing technology, media, telecommunications, life sciences and clean technology companies in North America. IC Manage grew 178 percent during the 4-year period from 2008 to 2012.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<h2><span class="mw-headline" id="Products">Products</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IC_Manage&amp;action=edit&amp;section=2" title="Edit section: Products">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>Global Design Platform (GDP) - In April 2007, IC Manage announced its Global Design Platform (GDP) design data management system.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></li>
<li>IP Central/ IP Pro - In June 2011, the company introduced its IP Central platform targeted to design and verification teams for <a href="/wiki/Semiconductor_intellectual_property_core" title="Semiconductor intellectual property core">IP reuse</a>, including publishing, sharing and integration and revisions of internal and third party IP, and IP bug dependency tracing.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup></li>
<li>IC Manage Views – In May 2012, the company announced its IC Manage Views workspace acceleration software, a version aware file system which presents complete virtual workspace views, and transfers data on demand to a local file cache.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup></li>
</ul>
<h2><span class="mw-headline" id="Customer_Applications">Customer Applications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IC_Manage&amp;action=edit&amp;section=3" title="Edit section: Customer Applications">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li><a href="/wiki/Broadcom" title="Broadcom">Broadcom</a>, <a class="mw-redirect" href="/wiki/AMD" title="AMD">AMD</a> and <a href="/wiki/Nvidia" title="Nvidia">Nvidia</a> on IP Reuse. At the 2012 <a href="/wiki/Design_Automation_Conference" title="Design Automation Conference">Design Automation Conference</a>, three IC Manage customers presented on best practices and use of IC Manage for Design IP reuse, including: top-down mandates for frequent check-ins, following a branching methodology, putting IP into project libraries, self-contained IP blocks within a library, and specification-driven IP design.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></li>
<li><a href="/wiki/Altera" title="Altera">Altera</a>, <a class="mw-redirect" href="/wiki/Cambridge_Silicon_Radio" title="Cambridge Silicon Radio">Cambridge Silicon Radio</a> and <a href="/wiki/Xilinx" title="Xilinx">Xilinx</a> on - IP-based Design and Verification Reuse. At <a href="/wiki/Design_Automation_Conference" title="Design Automation Conference">Design Automation Conference</a> 2013, three IC Manage customers discussed design and verification approaches utilizing IC Manage: Using one data repository for all designs, reusing simulation testbenches, linking the <a href="/wiki/Bug_tracking_system" title="Bug tracking system">bug tracking system</a> and data management system, making verification information part of the IP, designing by high-level block function rather than chips, minimizing IP modifications, and restricting IP access.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></li>
</ul>
