<p>The <b>Journaling Flash File System</b> (or <b>JFFS</b>) is a <a href="/wiki/Log-structured_file_system" title="Log-structured file system">log-structured file system</a> for use on NOR <a href="/wiki/Flash_memory" title="Flash memory">flash memory</a> devices on the <a href="/wiki/Linux" title="Linux">Linux</a> operating system. It has been superseded by <a href="/wiki/JFFS2" title="JFFS2">JFFS2</a>.</p>
<p></p>

<h2><span class="mw-headline" id="Design">Design</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JFFS&amp;action=edit&amp;section=1" title="Edit section: Design">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Flash memory (specifically <a class="mw-redirect" href="/wiki/NOR_flash" title="NOR flash">NOR flash</a>) must be erased prior to writing. The erase process has several limitations:</p>
<ul>
<li>Erasing is very slow (typically 1-100 ms per erase block, which is 10<sup>3</sup>-10<sup>5</sup> times slower than reading data from the same region)</li>
<li>It is only possible to erase flash in large segments (usually 64 <a class="mw-redirect" href="/wiki/KiB" title="KiB">KiB</a> or more), whereas it can be read or written in smaller blocks (often 512 bytes)</li>
<li>Flash memory <a href="/wiki/Flash_memory#Write_endurance" title="Flash memory">can only be erased a limited number of times</a> (typically 10<sup>3</sup>-10<sup>6</sup>) before it becomes worn out</li>
</ul>
<p>These constraints combine to produce a profound asymmetry between patterns of read and write access to flash memory. In contrast, magnetic hard disk drives offer nearly symmetrical read and write access: read speed and write speed are nearly identical (as both are constrained by the rate at which the disk spins), it is possible to both read and write small blocks or sectors (typically 512 or <a href="/wiki/Advanced_Format" title="Advanced Format">4096</a> bytes), and there is no practical limit to the number of times magnetic media can be written and rewritten.</p>
<p>Traditional file systems, such as <a href="/wiki/Ext2" title="Ext2">ext2</a> or <a class="mw-redirect" href="/wiki/File_allocation_table" title="File allocation table">FAT</a> which were designed for use on magnetic media typically update their data structures in-place, with data structures like inodes and directories updated on-disk after every modification. This concentrated lack of wear-levelling makes conventional file systems unsuitable for read-write use on flash devices.</p>
<p>JFFS enforces <a class="mw-redirect" href="/wiki/Wear_levelling" title="Wear levelling">wear levelling</a> by treating the flash device as a circular log. All changes to files and directories are written to the tail of the log in <i>nodes</i>. In each node, a header containing metadata is written first, followed by file data, if any. Nodes are chained together with offset pointers in the header. Nodes start out as <i>valid</i> and then become <i>obsolete</i> when a newer version of them is created.</p>
<p>The free space remaining in the file system is the gap between the log's tail and its head. When this runs low, a <a href="/wiki/Garbage_collection_(computer_science)" title="Garbage collection (computer science)">garbage collector</a> copies valid nodes from the head to the tail and skips obsolete ones, thus reclaiming space.</p>
<h2><span class="mw-headline" id="Disadvantages">Disadvantages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JFFS&amp;action=edit&amp;section=2" title="Edit section: Disadvantages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>At mount time, the file system driver must read the entire inode chain and then keep it in memory. This can be very slow. Memory consumption of JFFS is also proportional to the number of files in the file system.</li>
<li>The circular log design means <i>all</i> data in the filesystem is re-written, regardless of whether it is static or not. This generates many unnecessary erase cycles and reduces the life of the flash medium.</li>
</ul>
