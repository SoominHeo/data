<p>The <b>Java Naming and Directory Interface</b> (<b>JNDI</b>) is a Java <a href="/wiki/Application_programming_interface" title="Application programming interface">API</a> for a <a href="/wiki/Directory_service" title="Directory service">directory service</a> that allows Java software clients to discover and look up data and objects via a name. Like all <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> APIs that interface with host systems, JNDI is independent of the underlying implementation. Additionally, it specifies a <a href="/wiki/Service_provider_interface" title="Service provider interface">service provider interface</a> (SPI) that allows <a href="/wiki/Directory_service" title="Directory service">directory service</a> implementations to be plugged into the framework.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> It may make use of a server, a flat file, or a database; the choice is up to the vendor.</p>
<p>Typical uses of JNDI include:</p>
<ul>
<li>connecting a Java application to an external directory service (such as an address database or an <a class="mw-redirect" href="/wiki/LDAP" title="LDAP">LDAP</a> server)</li>
<li>allowing a <a class="mw-redirect" href="/wiki/Java_Servlet" title="Java Servlet">Java Servlet</a> to look up configuration information provided by the hosting <a href="/wiki/Web_container" title="Web container">web container</a><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></li>
</ul>

<h2><span class="mw-headline" id="Background">Background</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Naming_and_Directory_Interface&amp;action=edit&amp;section=1" title="Edit section: Background">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The Java <a href="/wiki/Java_remote_method_invocation" title="Java remote method invocation">RMI</a> and <a class="mw-redirect" href="/wiki/Java_EE" title="Java EE">Java EE</a> APIs use the JNDI API to look up objects in a network.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<p>The API provides:</p>
<ul>
<li>a mechanism to bind an object to a name</li>
<li>a directory-lookup interface that allows general queries</li>
<li>an event interface that allows clients to determine when directory entries have been modified</li>
<li>LDAP extensions to support the additional capabilities of an LDAP service</li>
</ul>
<p>The <a href="/wiki/Service_provider_interface" title="Service provider interface">SPI</a> portion allows support for practically any kind of naming or directory service, including:</p>
<ul>
<li><a href="/wiki/Lightweight_Directory_Access_Protocol" title="Lightweight Directory Access Protocol">LDAP</a></li>
<li><a href="/wiki/Domain_Name_System" title="Domain Name System">DNS</a></li>
<li><a href="/wiki/Network_Information_Service" title="Network Information Service">NIS</a></li>
<li><a class="mw-redirect" href="/wiki/CORBA" title="CORBA">CORBA</a> name service</li>
<li><a href="/wiki/File_system" title="File system">file system</a></li>
</ul>
<p><a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a> first released the JNDI specification on March 10, 1997.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> As of 2006<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Java_Naming_and_Directory_Interface&amp;action=edit">[update]</a></sup>, the current version is JNDI 1.2.</p>
<h2><span class="mw-headline" id="Basic_lookup">Basic lookup</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Naming_and_Directory_Interface&amp;action=edit&amp;section=2" title="Edit section: Basic lookup">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>JNDI (Java Naming and Directory Interface) organizes its names into a hierarchy. A name can be any string such as "com.mydomain.ejb.MyBean". A name can also be an object that implements the <code>Name</code> interface; however a string is the most common way to name an object. A name is bound to an object in the directory by storing either the object or a <a class="new" href="/w/index.php?title=JNDI_reference&amp;action=edit&amp;redlink=1" title="JNDI reference (page does not exist)">reference</a> to the object in the directory service identified by the name.</p>
<p>The JNDI API defines a context that specifies where to look for an object. The initial context is typically used as a starting point.</p>
<p>In the simplest case, an initial context must be created using the specific implementation and extra parameters required by the implementation. The initial context will be used to look up a name. The initial context is analogous to the root or top of a directory tree for a file system. Below is an example of creating an initial context:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">Hashtable</span> <span class="n">contextArgs</span> <span class="o">=</span> <span class="k">new</span> <span class="n">Hashtable</span><span class="o">();</span>

<span class="c1">// First you must specify the context factory.</span>
<span class="c1">// This is how you choose between jboss implementation</span>
<span class="c1">// vs. an implementation from Sun or other vendors.</span>
<span class="n">contextArgs</span><span class="o">.</span><span class="na">put</span><span class="o">(</span> <span class="n">Context</span><span class="o">.</span><span class="na">INITIAL_CONTEXT_FACTORY</span><span class="o">,</span> <span class="s">"com.jndiprovider.TheirContextFactory"</span> <span class="o">);</span>

<span class="c1">// The next argument is the URL specifying where the data store is:</span>
<span class="n">contextArgs</span><span class="o">.</span><span class="na">put</span><span class="o">(</span> <span class="n">Context</span><span class="o">.</span><span class="na">PROVIDER_URL</span><span class="o">,</span> <span class="s">"jndiprovider-database"</span> <span class="o">);</span>

<span class="c1">// (You may also have to provide security credentials)</span>

<span class="c1">// Next you create the initial context</span>
<span class="n">Context</span> <span class="n">myCurrentContext</span> <span class="o">=</span> <span class="k">new</span> <span class="n">InitialContext</span><span class="o">(</span><span class="n">contextArgs</span><span class="o">);</span>
</pre></div>
<p>A context is then used to look up previously bound names in that context. For example:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">MyBean</span> <span class="n">myBean</span> <span class="o">=</span> <span class="o">(</span><span class="n">MyBean</span><span class="o">)</span>  <span class="n">myCurrentContext</span><span class="o">.</span><span class="na">lookup</span><span class="o">(</span><span class="s">"com.mydomain.MyBean"</span><span class="o">);</span>
</pre></div>
<p>Alternative to above code is as below:</p>
<p>The Context object can also be configured by adding jndi.properties file in classpath containing initial context factory class name and provider URL. The above code will be reduced as shown below:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">//just need to create initial context object, it will try to read jndi.properties file from the classpath.</span>
<span class="n">Context</span> <span class="n">myCurrentContext</span> <span class="o">=</span> <span class="k">new</span> <span class="n">InitialContext</span><span class="o">();</span>
</pre></div>
<p>A context is then used to look up previously bound names in that context. For example:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">MyBean</span> <span class="n">myBean</span> <span class="o">=</span> <span class="o">(</span><span class="n">MyBean</span><span class="o">)</span>  <span class="n">myCurrentContext</span><span class="o">.</span><span class="na">lookup</span><span class="o">(</span><span class="s">"com.mydomain.MyBean"</span><span class="o">);</span>
</pre></div>
<h2><span class="mw-headline" id="Searching">Searching</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Naming_and_Directory_Interface&amp;action=edit&amp;section=3" title="Edit section: Searching">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Attributes may be attached to special entries called directories. Directories enable searching for objects by their associated attributes. Directories are a type of context; they restrict the name space much like a directory structure on a file system does.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
