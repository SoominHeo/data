<p><b>KHTML</b> is a <a href="/wiki/HTML" title="HTML">HTML</a> <a href="/wiki/Layout_engine" title="Layout engine">layout engine</a> developed by the <a href="/wiki/KDE" title="KDE">KDE</a> project. It is the engine used by the <a href="/wiki/Konqueror" title="Konqueror">Konqueror</a> web browser. Descendants of KHTML are used by some of the world's most popular browsers, among them <a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a>, <a href="/wiki/Safari_(web_browser)" title="Safari (web browser)">Safari</a> and <a href="/wiki/Opera_(web_browser)" title="Opera (web browser)">Opera</a>. Distributed under the terms of the <a href="/wiki/GNU_Lesser_General_Public_License" title="GNU Lesser General Public License">GNU Lesser General Public License</a>, KHTML is <a href="/wiki/Free_software" title="Free software">free software</a>.</p>
<p>Built on the <a class="mw-redirect" href="/wiki/KPart" title="KPart">KPart</a> framework and written in <a href="/wiki/C%2B%2B" title="C++">C++</a>, KHTML has relatively good <a href="/wiki/KHTML#Standards_compliance" title="KHTML">support for Web standards</a>. To render as many pages as possible, some extra abilities and quirks from <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> are also supported, even though those are non-standard.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=KHTML&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Origins">Origins</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=KHTML&amp;action=edit&amp;section=2" title="Edit section: Origins">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>KHTML was preceded by an earlier engine called <i>khtmlw</i> or <i>the KDE HTML Widget</i>, developed by Torben Weis and Martin Jones,<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> which implemented support for <a href="/wiki/HTML" title="HTML">HTML</a> 3.2, <a class="mw-redirect" href="/wiki/HTTP" title="HTTP">HTTP</a> 1.0, and <a href="/wiki/Framing_(World_Wide_Web)" title="Framing (World Wide Web)">HTML frames</a>, but not the <a class="mw-redirect" href="/wiki/W3C" title="W3C">W3C</a> <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a>, <a href="/wiki/Cascading_Style_Sheets" title="Cascading Style Sheets">CSS</a>, or <a class="mw-redirect" href="/wiki/DOM_scripting" title="DOM scripting">scripting</a>.</p>
<p>KHTML itself came into existence on November 4, 1998,<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> as a copy of the khtmlw library, with some slight refactoring and the addition of Unicode support and changes to support the move to <a class="mw-redirect" href="/wiki/Qt_(toolkit)" title="Qt (toolkit)">Qt</a> 2. Waldo Bastian<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> was among those who did the work of creating that early version of KHTML.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<h3><span class="mw-headline" id="Re-write_and_improvement">Re-write and improvement</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=KHTML&amp;action=edit&amp;section=3" title="Edit section: Re-write and improvement">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The real work on KHTML actually started between May and October 1999, with the realization that the choice facing the project was "either do a significant effort to move KHTML forward or to use <a href="/wiki/Mozilla" title="Mozilla">Mozilla</a>"<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> and with adding support for <a class="mw-redirect" href="/wiki/DOM_scripting" title="DOM scripting">scripting</a> as the highest priority. So in May 1999, Lars Knoll<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> began doing research with an eye toward implementing the <a class="mw-redirect" href="/wiki/W3C" title="W3C">W3C</a> <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a> specification, finally announcing<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> on August 16, 1999 that he had checked in<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> what amounted to a complete rewrite of the KHTML library — changing KHTML to use the standard <a class="mw-redirect" href="/wiki/W3C" title="W3C">W3C</a> <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a> as its internal document representation. That in turn allowed the beginnings of <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> support to be added in October 1999,<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> with the integration of <a href="/wiki/Harri_Porten" title="Harri Porten">Harri Porten</a>'s <a class="mw-redirect" href="/wiki/KJS_(KDE)" title="KJS (KDE)">KJS</a> following shortly afterwards.</p>
<p>In the closing months of 1999 and first few months of 2000, Knoll did further work with Antti Koivisto and Dirk Mueller<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup><sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup><sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> to add CSS support and to refine and stabilize the KHTML architecture,<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> with most of that work being completed by March 2000. Among other things, those changes enabled KHTML to become the second browser after <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> to correctly support <a class="mw-redirect" href="/wiki/Hebrew" title="Hebrew">Hebrew</a> and <a href="/wiki/Arabic" title="Arabic">Arabic</a> and languages written <a href="/wiki/Bi-directional_text" title="Bi-directional text">right-to-left</a><sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> — before <a href="/wiki/Mozilla" title="Mozilla">Mozilla</a> had such support.</p>
<p><a class="mw-redirect" href="/wiki/KDE_2.0" title="KDE 2.0">KDE 2.0</a> was the first KDE release (on October 23, 2000) to include KHTML<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> (as the rendering engine of the new <a href="/wiki/Konqueror" title="Konqueror">Konqueror</a> <a href="/wiki/File_manager" title="File manager">file</a> and <a href="/wiki/Web_browser" title="Web browser">web browser</a>, which replaced the monolithic KDE File Manager).</p>
<h3><span class="mw-headline" id="Other_modules">Other modules</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=KHTML&amp;action=edit&amp;section=4" title="Edit section: Other modules">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>KSVG was first developed in 2001 by <a class="new" href="/w/index.php?title=Nikolas_Zimmermann&amp;action=edit&amp;redlink=1" title="Nikolas Zimmermann (page does not exist)">Nikolas Zimmermann</a> and <a class="new" href="/w/index.php?title=Rob_Buis&amp;action=edit&amp;redlink=1" title="Rob Buis (page does not exist)">Rob Buis</a>; however, by 2003, it was decided to fork the then-current KSVG implementation into two new projects: <a class="mw-redirect" href="/wiki/KDOM_(module)" title="KDOM (module)">KDOM</a>/KSVG2 (to improve the state of DOM rendering in KHTML underneath a more formidable SVG 1.0 render state) and <a class="new" href="/w/index.php?title=Kcanvas&amp;action=edit&amp;redlink=1" title="Kcanvas (page does not exist)">Kcanvas</a> (to abstract any rendering done within khtml/ksvg2 in a single shared library, with multiple backends for it, e.g., Cairo/Qt, etc.).<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup></p>
<p>KSVG2 is also a part of <a href="/wiki/WebKit" title="WebKit">WebKit</a>.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup></p>
<h2><span class="mw-headline" id="Standards_compliance">Standards compliance</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=KHTML&amp;action=edit&amp;section=5" title="Edit section: Standards compliance">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The following standards are supported by the KHTML engine:</p>
<ul>
<li><a href="/wiki/HTML" title="HTML">HTML</a> 4.01</li>
<li><a href="/wiki/HTML5" title="HTML5">HTML 5</a> support</li>
<li><a href="/wiki/Cascading_Style_Sheets" title="Cascading Style Sheets">CSS</a> 1</li>
<li><a href="/wiki/Cascading_Style_Sheets" title="Cascading Style Sheets">CSS</a> 2.1 (screen and paged media)</li>
<li><a href="/wiki/Cascading_Style_Sheets" title="Cascading Style Sheets">CSS</a> 3 Selectors (fully as of KDE 3.5.6<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup>)</li>
<li><a href="/wiki/Cascading_Style_Sheets" title="Cascading Style Sheets">CSS</a> 3 Other (multiple backgrounds, box-sizing and text-shadow)</li>
<li><a href="/wiki/Portable_Network_Graphics" title="Portable Network Graphics">PNG</a>, <a href="/wiki/Multiple-image_Network_Graphics" title="Multiple-image Network Graphics">MNG</a>, <a href="/wiki/JPEG" title="JPEG">JPEG</a>, <a class="mw-redirect" href="/wiki/Graphics_Interchange_Format" title="Graphics Interchange Format">GIF</a> graphic formats</li>
<li><a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a> 1, 2 and partially 3</li>
<li><a href="/wiki/JavaScript" title="JavaScript">ECMA-262/JavaScript 1.5</a></li>
<li>Partial <a href="/wiki/Scalable_Vector_Graphics" title="Scalable Vector Graphics">Scalable Vector Graphics</a> support</li>
</ul>
<h2><span class="mw-headline" id="Descendants">Descendants</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=KHTML&amp;action=edit&amp;section=6" title="Edit section: Descendants">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/WebKit#Origins" title="WebKit">WebKit § Origins</a></div>
<p>KHTML and <a class="mw-redirect" href="/wiki/KDE%27s_JavaScript_engine" title="KDE's JavaScript engine">KJS</a> were adopted by <a class="mw-redirect" href="/wiki/Apple_Computer" title="Apple Computer">Apple</a> in 2002 for use in the <a href="/wiki/Safari_(web_browser)" title="Safari (web browser)">Safari</a> web browser. Apple publishes the source code for their <a href="/wiki/Fork_(software_development)" title="Fork (software development)">fork</a> of the KHTML engine, called <a href="/wiki/WebKit" title="WebKit">WebKit</a>. In 2013, Google began development on a fork of WebKit, called <a href="/wiki/Blink_(web_engine)" title="Blink (web engine)">Blink</a>.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup></p>
