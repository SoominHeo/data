<p><b>Cabinet</b> (or <b>CAB</b>) is an <a class="mw-redirect" href="/wiki/Archive_file_format" title="Archive file format">archive file format</a> for <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> that supports <a class="mw-redirect" href="/wiki/Lossless_data_compression" title="Lossless data compression">lossless data compression</a> and embedded <a class="mw-redirect" href="/wiki/Digital_certificate" title="Digital certificate">digital certificates</a> used for maintaining archive integrity. Cabinet files have <code>.cab</code> <a href="/wiki/Filename_extension" title="Filename extension">filename extensions</a> and are recognized by their first 4 bytes <i>MSCF</i>. Cabinet files were known originally as Diamond files.</p>
<p>The CAB file format may employ the following <a class="mw-redirect" href="/wiki/Compression_algorithm" title="Compression algorithm">compression algorithms</a>:</p>
<ul>
<li><a href="/wiki/DEFLATE" title="DEFLATE">DEFLATE</a> – invented by <a href="/wiki/Phil_Katz" title="Phil Katz">Phil Katz</a>, the author of the <a class="mw-redirect" href="/wiki/ZIP_file_format" title="ZIP file format">ZIP file format</a></li>
<li><a href="/wiki/Quantum_compression" title="Quantum compression">Quantum compression</a> – licensed from David Stafford, the author of the Quantum archiver</li>
<li><a href="/wiki/LZX_(algorithm)" title="LZX (algorithm)">LZX</a> – invented by <a class="new" href="/w/index.php?title=Jonathan_Forbes_(programmer)&amp;action=edit&amp;redlink=1" title="Jonathan Forbes (programmer) (page does not exist)">Jonathan Forbes</a> and Tomi Poutanen, given to Microsoft when Forbes joined the company</li>
</ul>
<p>A CAB archive can reserve empty spaces in the archive as well as for each file in the archive, for some application-specific uses like digital signatures or arbitrary data. CAB format is used by a variety of <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> installation technologies including <a href="/wiki/Windows_Installer" title="Windows Installer">Windows Installer</a>, <a class="mw-redirect" href="/wiki/Setup_API" title="Setup API">Setup API</a>, <a class="new" href="/w/index.php?title=Device_Installer&amp;action=edit&amp;redlink=1" title="Device Installer (page does not exist)">Device Installer</a> and <a class="new" href="/w/index.php?title=AdvPack&amp;action=edit&amp;redlink=1" title="AdvPack (page does not exist)">AdvPack</a> (used by <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> to install <a href="/wiki/ActiveX" title="ActiveX">ActiveX</a> components). CAB files are also often associated with self-extracting programs like <a href="/wiki/IExpress" title="IExpress">IExpress</a> where the executable program extracts the associated CAB file. CAB files are also sometimes embedded into other files. For example, MSI and MSU files (the latter are CAB files with just another filename extension) usually included one or more embedded CAB files.</p>

<h2><span class="mw-headline" id="File_structure">File structure</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Cabinet_(file_format)&amp;action=edit&amp;section=1" title="Edit section: File structure">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A CAB archive can contain up to 65535 CAB-folders, (not to be confused with <a class="mw-redirect" href="/wiki/Folder_(computing)" title="Folder (computing)">file system folders</a>) each can contain up to 65535 files. Internally, each CAB-folder is treated as a <a href="/wiki/Solid_compression" title="Solid compression">single compressed block</a>, which provides more efficient compression than individually compressing each file.</p>
<p>Every entry in a CAB-folder has to be a file.<sup class="reference" id="cite_ref-msdn.microsoft.com_1-0"><a href="#cite_note-msdn.microsoft.com-1">[1]</a></sup> Due to this structure, it is not possible to store empty folders in CAB archives.</p>

<p>How paths should be handled is not specified in the CAB file format, leaving it to the software implementation.</p>
<ul>
<li>Some affix file paths to filenames only, as if all files in a CAB archive are in a single folder. <a href="/wiki/IExpress" title="IExpress">IExpress</a> works this way, as does Microsoft Windows Explorer, which can open CAB archives as a folder.</li>
<li>Some can store the paths, and upon extraction, create folders as necessary. <tt>CABARC.EXE</tt> and <tt>EXTRACT.EXE</tt> (tools from Microsoft Cabinet SDK<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> ) as well as <code>lcab</code><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> and <code>cabextract</code><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> (third-party <a class="mw-redirect" href="/wiki/Open-source" title="Open-source">open-source</a> tools) work this way.</li>
<li><tt>EXPAND.EXE</tt>, only since version 6 (which is included from Windows Vista to above) can extract files to their paths. The previous versions don't do it.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></li>
</ul>
<h2><span class="mw-headline" id="Software">Software</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Cabinet_(file_format)&amp;action=edit&amp;section=2" title="Edit section: Software">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/Comparison_of_file_archivers" title="Comparison of file archivers">Comparison of file archivers</a></div>
<p>Software that can extract the contents of a CAB archive file are numerous, including <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> itself (through <a href="/wiki/File_Explorer" title="File Explorer">File Explorer</a>, <a class="mw-redirect" href="/wiki/Setup_API" title="Setup API">Setup API</a>, <code>expand.exe</code> or <code>extract.exe</code>) as well as well-known software such as <a href="/wiki/WinZip" title="WinZip">WinZip</a>, <a href="/wiki/WinRAR" title="WinRAR">WinRAR</a> or <a href="/wiki/7-Zip" title="7-Zip">7-Zip</a>. However, fewer programs can create CAB archives. For a full list, see <span style="padding: 0; font-size: xx-small; font-weight: bold; color: #000000; background-color: #f9f9f9;"><a href="/wiki/Wikipedia:Templates_for_discussion/Log/2017_January_9#Template:Section_link" title="Wikipedia:Templates for discussion/Log/2017 January 9">‹See Tfd›</a></span><a href="/wiki/Comparison_of_file_archivers#archive_formats" title="Comparison of file archivers">Comparison of file archivers § archive formats</a>.</p>
<p>The Windows program <code>makecab.exe</code> is used to create CAB archives:</p>
<ul>
<li>Compress a single file into a CAB archive</li>
</ul>
<dl>
<dd>
<dl>
<dd><code>makecab.exe &lt;input_file&gt;[ &lt;output_file&gt;]</code></dd>
</dl>
</dd>
</dl>
<ul>
<li>Read the <a class="new" href="/w/index.php?title=Diamond_directive_file&amp;action=edit&amp;redlink=1" title="Diamond directive file (page does not exist)">diamond directive file</a> (with <code>.ddf</code> filename extension) and create a CAB archive containing multiple files in a flat or hierarchical structure like a <a href="/wiki/File_system" title="File system">file system</a>.</li>
</ul>
<dl>
<dd>
<dl>
<dd><code>makecab.exe /F &lt;diamond_directive_file&gt;[ &lt;output_file&gt;]</code></dd>
</dl>
</dd>
</dl>
<h2><span class="mw-headline" id="Related_formats">Related formats</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Cabinet_(file_format)&amp;action=edit&amp;section=3" title="Edit section: Related formats">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The <code>.cab</code> filename extension is also used by other installer programs (e.g. <a href="/wiki/InstallShield" title="InstallShield">InstallShield</a>) for their own proprietary archiving formats. InstallShield uses <a href="/wiki/Zlib" title="Zlib">zlib</a> for compression (see <a class="mw-redirect" href="/wiki/Deflate" title="Deflate">Deflate</a>), but their headers are not the same as for Microsoft CAB files so they are incompatible and cannot be manipulated or edited with the programs that are made for standard Cabinet format. Specialized third-party utilities, such as Unshield,<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> can extract this specific proprietary format.</p>
<p>Microsoft Publisher has a "Pack and Go" feature that bundles a publisher document, together with all external links, into a CAB file with a <code>.PUZ</code> extension. These files are meant to be activated with a companion .EXE file which is distributed along with the <code>.PUZ</code> file. These files may be opened with any CAB file extraction program.</p>
