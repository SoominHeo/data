<h1 class="firstHeading" id="firstHeading" lang="en">MSCDEX</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>

<p><b>MSCDEX</b> or <b>Microsoft CD-ROM Extensions</b> is a <a href="/wiki/Computer_program" title="Computer program">software program</a> produced by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> and included with <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a> 6.x and certain versions of <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> Earlier versions of MSCDEX since 1986 were installable add-ons for MS-DOS 3.1 and higher.<sup class="reference" id="cite_ref-Barney_1986_2-0"><a href="#cite_note-Barney_1986-2">[2]</a></sup><sup class="reference" id="cite_ref-Mace_1986_3-0"><a href="#cite_note-Mace_1986-3">[3]</a></sup></p>
<p>It is a driver executable which allows DOS programs to recognize, read, and control <a href="/wiki/CD-ROM" title="CD-ROM">CD-ROMs</a> using the <a class="mw-redirect" href="/wiki/High_Sierra_file_system" title="High Sierra file system">High Sierra</a> and - since version 2.0 as of 1988 - also the <a href="/wiki/ISO_9660" title="ISO 9660">ISO 9660</a> <a href="/wiki/File_system" title="File system">file systems</a>.<sup class="reference" id="cite_ref-Johnston_1988_4-0"><a href="#cite_note-Johnston_1988-4">[4]</a></sup><sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> This requires the previous loading of an appropriate CD-ROM <a href="/wiki/Device_driver" title="Device driver">device driver</a> (example: OAKCDROM.SYS), usually from <a href="/wiki/CONFIG.SYS" title="CONFIG.SYS">CONFIG.SYS</a>.</p>
<p>The final version of the MSCDEX program was 2.96, included with Windows 95 and used when creating bootable floppy disks with CD-ROM support. Starting with Windows 95, CD-ROM access became possible through a 32-bit CDFS driver.</p>
<p>A cloaked variant of MSCDEX was provided as part of <a class="mw-redirect" href="/wiki/Helix_Software" title="Helix Software">Helix Software</a>'s <a class="mw-redirect" href="/wiki/Multimedia_Cloaking" title="Multimedia Cloaking">Multimedia Cloaking</a> product. It uses <a class="mw-redirect" href="/wiki/Helix_Cloaking" title="Helix Cloaking">Cloaking</a> to relocate and run in <a href="/wiki/Protected_mode" title="Protected mode">protected mode</a> on 386 processors.</p>
<p>The driver uses the Microsoft Networks interface in MS-DOS. This is the reason that at least version 3.1 of MS-DOS is required. The driver essentially looks like a network drive from the system perspective. It's implemented as a <a class="mw-redirect" href="/wiki/Terminate_and_Stay_Resident" title="Terminate and Stay Resident">TSR</a> program<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> and an extension to the <a href="/wiki/Network_redirector" title="Network redirector">redirector</a> interface (<b>CDEX</b>).</p>
<h2><span class="mw-headline" id="Alternatives">Alternatives</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MSCDEX&amp;action=edit&amp;section=1" title="Edit section: Alternatives">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><span id="NWCDEX"></span><a class="mw-redirect" href="/wiki/Novell_DOS_7" title="Novell DOS 7">Novell DOS 7</a>, <a class="mw-redirect" href="/wiki/Caldera_OpenDOS_7.01" title="Caldera OpenDOS 7.01">Caldera OpenDOS 7.01</a> and <a class="mw-redirect" href="/wiki/DR-DOS_7.02" title="DR-DOS 7.02">DR-DOS 7.02</a> and higher provide a functional equivalent to MSCDEX named <b>NWCDEX</b>, which also runs under MS-DOS and PC DOS. It has more flexible load-high capabilities, also allowing to relocate and run in <a href="/wiki/Protected_mode" title="Protected mode">protected mode</a> through <a href="/wiki/DOS_Protected_Mode_Services" title="DOS Protected Mode Services">DPMS</a> on <a href="/wiki/Intel_80286" title="Intel 80286">286</a> and higher processors, thereby leaving only a 7 KB stub in conventional or upper memory (in comparison to MSCDEX, which occupies some 16 KB).<sup class="reference" id="cite_ref-Paul_1997_NWDOSTIP_7-0"><a href="#cite_note-Paul_1997_NWDOSTIP-7">[7]</a></sup> Using <a class="mw-redirect" href="/wiki/Expanded_Memory" title="Expanded Memory">EMS</a> with a page frame, NWCDEX can reduce its footprint even down to a few bytes in conventional memory.<sup class="reference" id="cite_ref-Paul_1997_NWDOSTIP_7-1"><a href="#cite_note-Paul_1997_NWDOSTIP-7">[7]</a></sup> In contrast to MSCDEX, the driver does not depend on undocumented DOS APIs<sup class="reference" id="cite_ref-Paul_1997_NWDOSTIP_7-2"><a href="#cite_note-Paul_1997_NWDOSTIP-7">[7]</a></sup><sup class="reference" id="cite_ref-Schulman_1994_Undocumented-DOS_8-0"><a href="#cite_note-Schulman_1994_Undocumented-DOS-8">[8]</a></sup> and therefore, with a third-party helper tool named <b>INSTCDEX</b>,<sup class="reference" id="cite_ref-Paul_1997_NWDOSTIP_7-3"><a href="#cite_note-Paul_1997_NWDOSTIP-7">[7]</a></sup> can be loaded via <tt><a class="mw-redirect" href="/wiki/INSTALL_(CONFIG.SYS_directive)" title="INSTALL (CONFIG.SYS directive)">INSTALL</a></tt> statements and be fully functional in <a href="/wiki/CONFIG.SYS" title="CONFIG.SYS">CONFIG.SYS</a> to increase chances to load the driver high and, under these operating systems, allow to load other drivers not only from hard disk but also from CD-ROM while the operating system is still processing CONFIG.SYS.<sup class="reference" id="cite_ref-Paul_1997_NWDOSTIP_7-4"><a href="#cite_note-Paul_1997_NWDOSTIP-7">[7]</a></sup></p>
<p><span id="IMSCDEX"></span>Based on NWCDEX, <a class="mw-redirect" href="/wiki/Intelligent_Micro_Software" title="Intelligent Micro Software">IMS</a> <a class="mw-redirect" href="/wiki/REAL/32" title="REAL/32">REAL/32</a>, a successor to <a href="/wiki/Novell" title="Novell">Novell</a>'s <a href="/wiki/Multiuser_DOS" title="Multiuser DOS">Multiuser DOS</a> and <a href="/wiki/Digital_Research" title="Digital Research">Digital Research</a>'s <a class="mw-redirect" href="/wiki/Concurrent_DOS" title="Concurrent DOS">Concurrent DOS</a>, provides a similar driver named <b>IMSCDEX</b>.<sup class="reference" id="cite_ref-Paul_1997_NWDOSTIP_7-5"><a href="#cite_note-Paul_1997_NWDOSTIP-7">[7]</a></sup></p>
<p><span id="SHSUCDEX"></span>There's a free alternative called <b>SHSUCDX</b><sup class="reference" id="cite_ref-BOOTCD_9-0"><a href="#cite_note-BOOTCD-9">[9]</a></sup> that is used with the IDE/ATA driver <a class="external text" href="http://www.bootablecd.de/FreeDOS/help/en/hhstndrd/base/uide.htm" rel="nofollow">UIDE.SYS</a> first released in 2005<sup class="reference" id="cite_ref-SHSUCDEX_10-0"><a href="#cite_note-SHSUCDEX-10">[10]</a></sup> that is often used with <a href="/wiki/FreeDOS" title="FreeDOS">FreeDOS</a> and works with other DOSes as well.</p>
<p>In 1998 <a class="mw-redirect" href="/wiki/Caldera_UK" title="Caldera UK">Caldera</a> provided a <a class="mw-redirect" href="/wiki/DRFAT32" title="DRFAT32">DRFAT32</a> driver for DR-DOS to dynamically mount and unmount <a class="mw-redirect" href="/wiki/FAT32" title="FAT32">FAT32</a> volumes on DOS versions otherwise not natively supporting FAT32. DRFAT32 uses a variation and extension of the CDEX API in order to achieve this and work with older DOS versions.</p>
<p><a href="/wiki/Corel" title="Corel">Corel</a> also offered <b>CORELCDX.COM</b> as their alternative.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></p>
