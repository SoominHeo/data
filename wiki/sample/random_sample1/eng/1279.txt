<p><b>MSX BASIC</b> is a dialect of the <a class="mw-redirect" href="/wiki/BASIC_programming_language" title="BASIC programming language">BASIC programming language</a>. It is an extended version of <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> <a href="/wiki/MBASIC" title="MBASIC">Standard BASIC</a> Version 4.5, and includes support for graphic, music, and various peripherals attached to <a href="/wiki/MSX" title="MSX">MSX</a> Personal Computers. Generally, MSX-BASIC is designed to follow <a href="/wiki/GW-BASIC" title="GW-BASIC">GW-BASIC</a>, which is one of the standard BASICs running on 16-bit computers.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> During the creation of MSX-BASIC, a major effort was made to make the system as flexible and expandable as possible.</p>

<h2><span class="mw-headline" id="Distribution">Distribution</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MSX_BASIC&amp;action=edit&amp;section=1" title="Edit section: Distribution">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:258px;"><a class="image" href="/wiki/File:Msxbasic.png"><img alt="" class="thumbimage" data-file-height="192" data-file-width="256" height="192" src="//upload.wikimedia.org/wikipedia/commons/7/71/Msxbasic.png" width="256"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Msxbasic.png" title="Enlarge"></a></div>
MSX BASIC version 3.0</div>
</div>
</div>
<p>MSX BASIC came bundled in the <a href="/wiki/Read-only_memory" title="Read-only memory">ROM</a> of all MSX computers. At system start-up MSX BASIC is invoked, causing its command prompt to be displayed, unless other software placed in ROM takes control (which is the typical case of game cartridges and disk interfaces, the latter causing the <a href="/wiki/MSX-DOS" title="MSX-DOS">MSX-DOS</a> prompt to be shown if there is a disk present which contains the DOS system files).</p>
<p>When MSX BASIC is invoked, the ROM code for <a href="/wiki/BIOS" title="BIOS">BIOS</a> and the BASIC interpreter itself are visible on the lower 32K of the Z80 addressing space. The upper 32K are set to RAM, of which about 23K to 28K are available for BASIC code and data (the exact amount depends on the presence of disk controller and on the MSX-DOS kernel version).</p>
<h2><span class="mw-headline" id="Development_Environment">Development Environment</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MSX_BASIC&amp;action=edit&amp;section=2" title="Edit section: Development Environment">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>MSX BASIC development environment is very similar to that of the Dartmouth Time Sharing System associated with <a href="/wiki/Dartmouth_BASIC" title="Dartmouth BASIC">Dartmouth BASIC</a>. It has a command line-based Integrated Development Environment (IDE) system; all program lines must be numbered, all non-numbered lines are considered to be commands in direct mode (i.e., to be executed immediately). The user interface is almost completely command line, except for the function key shortcuts at the bottom and an initial copyright notice at the top of the screen.</p>
<h2><span class="mw-headline" id="Versions_of_MSX_BASIC">Versions of MSX BASIC</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MSX_BASIC&amp;action=edit&amp;section=3" title="Edit section: Versions of MSX BASIC">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Every new version of the MSX computer was bundled with an updated version of MSX BASIC. All versions are <a class="mw-redirect" href="/wiki/Backward_compatible" title="Backward compatible">backward compatible</a> and provide new capabilities to fully explore the new and extended hardware found on the newer MSX computers.</p>
<h3><span class="mw-headline" id="MSX_BASIC_1.0">MSX BASIC 1.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MSX_BASIC&amp;action=edit&amp;section=4" title="Edit section: MSX BASIC 1.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>Bundled with <a href="/wiki/MSX" title="MSX">MSX</a>1 computers</li>
<li>16 KB in size</li>
<li>No native support for floppy disk requiring the Disk BASIC cartridge extension (4 KB overhead)</li>
<li>Support for all available screen modes:
<ul>
<li>Screen 0 (text mode 40 x 24 characters)</li>
<li>Screen 1 (mixed text mode 32 x 24 characters, sprites and colored custom characters)</li>
<li>Screen 2 (high resolution graphic mode 256 x 192 pixels, 16 colors)</li>
<li>Screen 3 (low resolution graphic mode 64×48 - 4×4 pixel blocks over the screen 2 resolution)</li>
</ul>
</li>
<li>Full support for hardware sprites and <a href="/wiki/Interrupt" title="Interrupt">interrupt</a>-driven automatic <a href="/wiki/Collision_detection" title="Collision detection">collision detection</a></li>
<li>Full support for the <a class="mw-redirect" href="/wiki/General_Instruments_AY-3-8912" title="General Instruments AY-3-8912">General Instruments AY-3-8910</a> <a class="mw-redirect" href="/wiki/Programmable_Sound_Generator" title="Programmable Sound Generator">Programmable Sound Generator</a> (PSG)</li>
</ul>
<p>Note that the Brazilian MSX "clones" by Sharp and Gradiente show other versions of MSX-BASIC (on the Sharps even called HOT-BASIC), but they're basically just unlicensed MSX BASIC 1.0.</p>
<h3><span class="mw-headline" id="MSX_BASIC_2.0_.2F_2.1">MSX BASIC 2.0 / 2.1</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MSX_BASIC&amp;action=edit&amp;section=5" title="Edit section: MSX BASIC 2.0 / 2.1">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>Bundled with <a href="/wiki/MSX" title="MSX">MSX</a>2 computers</li>
<li>32 KB in size (First 16 KB directly available, second 16 KB in other slot and has to be paged in/out for usage)</li>
<li>Added support for new available screen modes, including graphic modes with 212 progressive or 424 interlaced lines:
<ul>
<li>Updated Screen 0 (text mode 80 x 24)</li>
<li>Screen 5 (graphic mode 256 x 212/424 pixels, 16 colors out of 512)</li>
<li>Screen 6 (graphic mode 512 x 212/424 pixels, 4 colors out of 512)</li>
<li>Screen 7 (graphic mode 512 x 212/424 pixels, 16 colors out of 512)</li>
<li>Screen 8 (graphic mode 256 x 212/424 pixels, 256 colors, no palette)</li>
</ul>
</li>
<li>Added support for multicolored sprites (16 colors)</li>
<li>Added support for hardware accelerated graphics functions (copy, fill, blittering, etc.)</li>
<li>Added support for using the lower 32K RAM of the computer (not directly visible because the BIOS and BASIC interpreter ROMs take over the addressing space) as a limited RAM disk (only certain types of files could be saved). MSX BASIC 2.1 supports using the memory mapper (if available on the machine) to expand this RAM disk to almost 90kB.</li>
</ul>
<p>MSX BASIC 2.1 exists on computers like the Philips MSX2 machines (except for the VG 8230), the Yamaha YIS-805<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> and Sanyo MPC-2300.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<h3><span class="mw-headline" id="MSX_BASIC_3.0">MSX BASIC 3.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MSX_BASIC&amp;action=edit&amp;section=6" title="Edit section: MSX BASIC 3.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>Bundled with <a class="mw-redirect" href="/wiki/MSX2%2B" title="MSX2+">MSX2+</a> computers</li>
<li>32 KB in size (First 16 KB directly available, second 16 KB in other slot and has to be paged in/out for usage)</li>
<li>Added command <b>SET SCROLL</b> for smooth, hardware based scrolling in BASIC</li>
<li>Added support for new available screen modes:
<ul>
<li>Screen 10 (graphic mode 256 x 212/424 pixels, 12499 YJK at once + 16 colors out of 512 RGB in ML)</li>
<li>Screen 11 (graphic mode 256 x 212/424 pixels, 12499 YJK at once + 16 colors out of 512 RGB)</li>
<li>Screen 12 (graphic mode 256 x 212/424 pixels, 19268 YJK at once)</li>
</ul>
</li>
</ul>
<h3><span class="mw-headline" id="MSX_BASIC_4.0">MSX BASIC 4.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MSX_BASIC&amp;action=edit&amp;section=7" title="Edit section: MSX BASIC 4.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>Bundled with the Panasonic FS-A1ST <a class="mw-redirect" href="/wiki/MSX_turbo_R" title="MSX turbo R">MSX turbo R</a> model</li>
<li>Added <b>_PAUSE</b> command to make delays in BASIC independent of the current CPU and clock</li>
<li>Added extra commands for the <a href="/wiki/Pulse-code_modulation" title="Pulse-code modulation">PCM</a> device (<b>_PCMPLAY</b>, <b>_PCMREC</b>)</li>
</ul>
<h3><span class="mw-headline" id="MSX_BASIC_4.1">MSX BASIC 4.1</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MSX_BASIC&amp;action=edit&amp;section=8" title="Edit section: MSX BASIC 4.1">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>Bundled with the Panasonic FS-A1GT <a class="mw-redirect" href="/wiki/MSX_turbo_R" title="MSX turbo R">MSX turbo R</a> model</li>
<li>Added <a href="/wiki/MIDI" title="MIDI">MIDI</a> extensions</li>
</ul>
<h2><span class="mw-headline" id="Extensions_of_MSX_BASIC">Extensions of MSX BASIC</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MSX_BASIC&amp;action=edit&amp;section=9" title="Edit section: Extensions of MSX BASIC">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Since MSX BASIC was meant to be expandable from inception, it was possible to write add-on modules quite easily. Support for specific hardware was commonly added by means of expansion cartridges, which also served as the interface to the hardware in question. MSX Disk-BASIC is an example, bundled in the cartridge that provides the hardware interface to the disk drives, it adds commands to access the floppy disk drives.</p>
<p>Exploring the MSX BASIC extensibility it was possible to extend MSX BASIC solely via software. A very interesting extension to the MSX BASIC 1.0/1.1 was <a class="new" href="/w/index.php?title=Screen_IV&amp;action=edit&amp;redlink=1" title="Screen IV (page does not exist)">Screen IV</a>, a tweak of MSX BASIC adding the mixed graphic/text mode Screen 4 (hence its name Screen IV), with 64 x 24 characters and Screen 2 resolution for graphics (256 x 192 pixels, 16 colors).</p>
