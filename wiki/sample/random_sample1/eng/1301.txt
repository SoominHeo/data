<p>The <b>NEC V20</b> (μPD70108) was a <a href="/wiki/Central_processing_unit" title="Central processing unit">processor</a> made by <a href="/wiki/NEC" title="NEC">NEC</a> that was a <a href="/wiki/Reverse_engineering" title="Reverse engineering">reverse-engineered</a>, pin-compatible version of the <a href="/wiki/Intel_8088" title="Intel 8088">Intel 8088</a> with an <a href="/wiki/Instruction_set" title="Instruction set">instruction set</a> compatible with the <a href="/wiki/Intel_80186" title="Intel 80186">Intel 80186</a>. The V20 was introduced in 1982,<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> and the V30 debuted in 1983.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<p>The chip featured much more than the 29,000 <a href="/wiki/Transistor" title="Transistor">transistors</a> of the simpler 8088 CPU, ran at 5 to 10 <a class="mw-redirect" href="/wiki/MHz" title="MHz">MHz</a><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> and was around 30% faster (application dependent) than the 8088 at the same clock speed, primarily due to faster <a href="/wiki/Addressing_mode" title="Addressing mode">effective address calculation</a>, along with faster loop counters, shift registers and multiplier.</p>

<h2><span class="mw-headline" id="Uses">Uses</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NEC_V20&amp;action=edit&amp;section=1" title="Edit section: Uses">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>NEC V20 was used in "turbo" versions of some PC clones such as <a href="/wiki/Commodore_PC_compatible_systems" title="Commodore PC compatible systems">Commodore PC compatible systems</a>, <a class="new" href="/w/index.php?title=Copam&amp;action=edit&amp;redlink=1" title="Copam (page does not exist)">Copam</a>, and <a href="/wiki/Tandy_Corporation" title="Tandy Corporation">Tandy</a> 1000 laptop series. It was also used in the Casio PV-S450 PDA and Hewlett-Packard's <a href="/wiki/HP_95LX" title="HP 95LX">HP 95LX</a>. It was also used in the Bandai <a href="/wiki/WonderSwan" title="WonderSwan">WonderSwan</a>, a handheld gaming system released in Japan in 1999. <a href="/wiki/Sony" title="Sony">Sony</a> also produced this microprocessor under license from NEC as the V20H (Sony CXQ70108).</p>
<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NEC_V20&amp;action=edit&amp;section=2" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>An unusual feature of the NEC V20 was that it added an Intel 8080 emulation mode, in which it could execute programs written for the <a href="/wiki/Intel_8080" title="Intel 8080">Intel 8080</a> processors. The instructions <code>BRKEM</code> executed in 8086 mode (NEC used a different notation for the instructions than Intel and BRK in NEC notation = INT in Intel notation) and <code>RETEM</code> and <code>CALLN</code> executed in 8080 mode was used to switch or return to or from the emulation mode. There were some programs which allowed to run 8080-based <a href="/wiki/CP/M" title="CP/M">CP/M-80</a> programs on a MS-DOS machine: V2080 <a href="/wiki/CPMulator" title="CPMulator">CPMulator</a> (later ZRUN) by Michael Day and 22nice from SYDEX.</p>
<p>Another unusual feature was the existence of several families of unique instructions. Instructions ADD4S, SUB4S, CMP4S were able to add, subtract and compare huge packed binary-coded decimal numbers stored in memory. Instructions ROL4 and ROR4 rotate <a href="/wiki/Nibble" title="Nibble">nibbles</a>. Another family consisted of instructions TEST1, SET1, CLR1, NOT1. These instructions test, set, clear and invert single bits but are far less efficient than the later <a class="mw-redirect" href="/wiki/I80386" title="I80386">I80386</a> equivalents <a class="new" href="/w/index.php?title=BT_(x86_instruction)&amp;action=edit&amp;redlink=1" title="BT (x86 instruction) (page does not exist)">BT</a>, <a class="new" href="/w/index.php?title=BTS_(x86_instruction)&amp;action=edit&amp;redlink=1" title="BTS (x86 instruction) (page does not exist)">BTS</a>, <a class="new" href="/w/index.php?title=BTR_(x86_instruction)&amp;action=edit&amp;redlink=1" title="BTR (x86 instruction) (page does not exist)">BTR</a> and <a class="new" href="/w/index.php?title=BTC_(x86_instruction)&amp;action=edit&amp;redlink=1" title="BTC (x86 instruction) (page does not exist)">BTC</a>, neither are their encodings compatible. There were two instructions to extract and insert bit fields of arbitrary lengths (EXT, INS). And finally, there were two additional repeat prefixes, namely REPC and REPNC, which amended the original <a class="new" href="/w/index.php?title=REP_(x86_instruction)&amp;action=edit&amp;redlink=1" title="REP (x86 instruction) (page does not exist)">REPE</a> and <a class="new" href="/w/index.php?title=REP_(x86_instruction)&amp;action=edit&amp;redlink=1" title="REP (x86 instruction) (page does not exist)">REPNE</a> and allowed a string of bytes of words to be scanned (with instructions <a class="new" href="/w/index.php?title=SCAS_(x86_instruction)&amp;action=edit&amp;redlink=1" title="SCAS (x86 instruction) (page does not exist)">SCAS</a> and <a class="new" href="/w/index.php?title=CMPS_(x86_instruction)&amp;action=edit&amp;redlink=1" title="CMPS (x86 instruction) (page does not exist)">CMPS</a>) while less or not less.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>

<ul>
<li>The <b>NEC V30</b> (μPD70116) was a version of the NEC V20 that was pin compatible with the 16-bit data bus <a href="/wiki/Intel_8086" title="Intel 8086">Intel 8086</a> processor. It also supports the 8080 emulation mode. The V30 was used in the <a href="/wiki/GTD-5_EAX" title="GTD-5 EAX">GTD-5 EAX</a> Class 5 central office as a drop-in performance upgrade for the processor complex in the late 1980s. It was also used in the <a href="/wiki/Psion_Series_3" title="Psion Series 3">Psion Series 3</a>, in the <a class="mw-redirect" href="/wiki/NEC_PC-9801" title="NEC PC-9801">NEC PC-9801</a> (originally released in 1982), the <a href="/wiki/Olivetti" title="Olivetti">Olivetti</a> PCS86, the <a href="/wiki/Applied_Engineering" title="Applied Engineering">Applied Engineering</a> "PC Transporter" emulator for the <a href="/wiki/Apple_II" title="Apple II">Apple II</a> series of computers, and in various arcade machines (particularly ones made by <a href="/wiki/Irem" title="Irem">Irem</a>) in the late 1980s.</li>
<li>The <b>NEC V20HL</b> (μPD70108H) and <b>NEC V30HL</b> (μPD70116H) were high-speed (up to 16 MHz) and low-power versions.</li>
<li>The <b><a href="/wiki/NEC_V25" title="NEC V25">NEC V25</a></b> (µPD70320) is the microcontroller version of the NEC V20 processor.</li>
<li><b>NEC V25HS</b> (μPD79011) V25 version with built-in <a class="mw-redirect" href="/wiki/RX116" title="RX116">RX116</a> RTOS</li>
<li><b>NEC V25+</b> (μPD70325) High-speed version of V25</li>
<li>The <b>NEC V33</b> is a super version of the V30 that separates address bus and data bus, and executes all instructions with wired logic instead of micro-codes, making it twice as fast as a V30 for the same clock frequency. V33 has the performance equivalent to <a href="/wiki/Intel_80286" title="Intel 80286">Intel 80286</a>. NEC V33 offers a method to expanding the memory address space to 16M bytes. It has two additional instructions <code>BRKXA</code> and <code>RETXA</code> to support extended addressing mode. The 8080 emulation mode was not supported.</li>
<li>The <b>NEC V33A</b> (μPD70136A) differs from the <b>NEC V33</b> in that it has interrupt vector numbers compatible with intel 80X86 processors.</li>
<li>The <b>NEC V35</b> (μPD70330) is the microcontroller version of the NEC V30 processor. Has 16-bit external data bus.</li>
<li><b>NEC V35HS</b> (μPD79021) V35 version with built-in <a class="mw-redirect" href="/wiki/RX116" title="RX116">RX116</a> RTOS</li>
<li><b>NEC V35+</b> (μPD70335) High-speed version of V35</li>
<li><b>NEC V40</b> (μPD70208) embedded version of V20, integrated Intel-compatible <a href="/wiki/Intel_8251" title="Intel 8251">8251</a> <a class="mw-redirect" href="/wiki/USART" title="USART">USART</a>, <a href="/wiki/Intel_8253" title="Intel 8253">8253</a> programmable interval timer, and <a href="/wiki/Intel_8255" title="Intel 8255">8255</a> parallel port interface. Used in the Olivetti PC1.</li>
<li><b>NEC V40HL</b> (μPD70208H) high-speed low-voltage version of V40</li>
<li><b>NEC V50</b> (μPD70216) embedded version of V30 with 16-bit data bus. It is the main CPU of the <a href="/wiki/Korg_M1" title="Korg M1">Korg M1</a>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></li>
<li><b>NEC V50HL</b> (μPD70216H) high-speed low-voltage version of V50</li>
<li>The <b>NEC V41</b> (μPD70270) <b>NEC V51</b> (μPD70280) integrated V30HL core and <a href="/wiki/IBM_Personal_Computer_XT" title="IBM Personal Computer XT">PC-XT</a> peripherals: <a href="/wiki/Intel_8255" title="Intel 8255">8255</a> parallel port interface, <a href="/wiki/Intel_8253" title="Intel 8253">8254</a> programmable interval timer, <a href="/wiki/Intel_8259" title="Intel 8259">8259</a> PIC, <a href="/wiki/Intel_8237" title="Intel 8237">8237</a> DMA controller and <a class="mw-redirect" href="/wiki/Intel_8042" title="Intel 8042">8042</a> keyboard controller. Also integrates full DRAM controller. Was used in <a href="/wiki/Olivetti" title="Olivetti">Olivetti</a> Quaderno XT-20.</li>
<li>The <b>NEC V53</b> (μPD70236) integrates a V33 core with 4-channel DMA (μPD71087/i8237), UART (μPD71051/i8251), three timer/counters (μPD71054/<a href="/wiki/Intel_8253" title="Intel 8253">i8254</a>) and interrupt controller (μPD71059/<a href="/wiki/Intel_8259" title="Intel 8259">i8259</a>).</li>
<li>The <b>NEC V53A</b> (μPD70236A) integrates some peripherals with a V33A core.</li>
<li><b>NEC V55PI</b> (μPD70433)</li>
<li>The <b><a class="new" href="/w/index.php?title=Vadem&amp;action=edit&amp;redlink=1" title="Vadem (page does not exist)">Vadem</a> VG230</b> was a single-chip PC platform. The VG230 contained a 16 MHz NEC V30HL processor and IBM PC/XT-compatible core logic, LCD controller (CGA/AT&amp;T640x400) with touch-plane support, keyboard matrix scanner, dual <a class="mw-redirect" href="/wiki/PCMCIA" title="PCMCIA">PCMCIA</a> 2.1 card controller, <a href="/wiki/Expanded_memory" title="Expanded memory">EMS</a> 4.0 hardware support for up to 64 MB, and built-in timer, PIC, DMA, UART and RTC controllers. It was used in the <a class="mw-redirect" href="/wiki/HP_OmniGo_100" title="HP OmniGo 100">HP OmniGo 100</a> and <a class="mw-redirect" href="/wiki/HP_OmniGo_120" title="HP OmniGo 120">120</a>.</li>
<li>The enhanced <b><a class="new" href="/w/index.php?title=Vadem&amp;action=edit&amp;redlink=1" title="Vadem (page does not exist)">Vadem</a> VG330</b> contained a 32 MHz NEC V30MX processor and IBM PC/AT-compatible core logic with dual PICs, LCD controller (640x480), keyboard matrix scanner, <a href="/wiki/PC_Card" title="PC Card">PC Card</a> ExCA 2.1 controller and SIR port.</li>
<li>Starting with the <a href="/wiki/NEC_V60" title="NEC V60">NEC V60</a>, NEC departed from the x86 design.</li>
</ul>
