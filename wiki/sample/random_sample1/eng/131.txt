<h1 class="firstHeading" id="firstHeading" lang="en">Batch file</h1>
<div class="mw-body-content" id="bodyContent">

<p>A <b>batch file</b> is a kind of <a href="/wiki/Scripting_language" title="Scripting language">script file</a> in <a href="/wiki/DOS" title="DOS">DOS</a>, <a href="/wiki/OS/2" title="OS/2">OS/2</a> and <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>. It consists of a series of <a href="/wiki/Command_(computing)" title="Command (computing)">commands</a> to be executed by the <a href="/wiki/Command-line_interface#Command-line_interpreter" title="Command-line interface">command-line interpreter</a>, stored in a <a href="/wiki/Plain_text" title="Plain text">plain text</a> file. A batch file may contain any command the interpreter accepts interactively and use constructs that enable conditional branching and looping within the batch file, such as "<a href="/wiki/Conditional_(computer_programming)#If.E2.80.93then.28.E2.80.93else.29" title="Conditional (computer programming)">if</a>", "<a href="/wiki/Foreach_loop" title="Foreach loop">for</a>", "<a href="/wiki/Goto" title="Goto">goto</a>" and <a href="/wiki/Label_(computer_science)" title="Label (computer science)">labels</a>. The term "batch" is from <a href="/wiki/Batch_processing" title="Batch processing">batch processing</a>, meaning "non-interactive execution", though a batch file may not process a <i>batch</i> of multiple data.</p>
<p>Similar to <a href="/wiki/Job_Control_Language" title="Job Control Language">Job Control Language</a> (JCL) and other systems on mainframe and minicomputer systems, batch files were added to ease the work required for certain regular tasks by allowing the user to set up a script to automate them. When a batch file is run, the <a href="/wiki/Shell_(computing)" title="Shell (computing)">shell</a> program (usually <a href="/wiki/COMMAND.COM" title="COMMAND.COM">COMMAND.COM</a> or <a href="/wiki/Cmd.exe" title="Cmd.exe">cmd.exe</a>) reads the file and executes its commands, normally line-by-line.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> <a href="/wiki/Operating_system" title="Operating system">operating systems</a>, such as <a href="/wiki/Linux" title="Linux">Linux</a>, have a similar, but more flexible, type of file called a <a href="/wiki/Shell_script" title="Shell script">shell script</a>.</p>
<p>The <a href="/wiki/Filename_extension" title="Filename extension">filename extension</a> <b>.bat</b> is used in DOS and Windows. <a href="/wiki/Windows_NT" title="Windows NT">Windows NT</a> and OS/2 also added <b>.cmd</b>. Batch files for other environments may have different extensions, e.g., <b>.btm</b> in <a href="/wiki/4DOS" title="4DOS">4DOS</a>, <a href="/wiki/4OS2" title="4OS2">4OS2</a> and <a href="/wiki/Take_Command_Console" title="Take Command Console">4NT</a> related shells.</p>
<p>The detailed handling of batch files has changed. Some of the detail in this article applies to all batch files, while other details apply only to certain versions.</p>

<h2><span class="mw-headline" id="Variants">Variants</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=1" title="Edit section: Variants">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="DOS">DOS</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=2" title="Edit section: DOS">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In DOS, a batch file can be started from the <a href="/wiki/Command-line_interface" title="Command-line interface">command-line interface</a> by typing its name, followed by any required parameters and pressing the "Enter" key. When DOS loads, the file <a href="/wiki/AUTOEXEC.BAT" title="AUTOEXEC.BAT">AUTOEXEC.BAT</a>, when present, is automatically executed, so any commands that need to be run to set up the DOS environment may be placed in this file. Computer users would have the AUTOEXEC.BAT file set up the system date and time, initialize the DOS environment, load any resident programs or device drivers, or initialize network connections and assignments.</p>
<p>A ".bat" file name extension identifies a file containing commands that are executed by the command interpreter <a href="/wiki/COMMAND.COM" title="COMMAND.COM">COMMAND.COM</a> line by line, as if it were a list of commands entered manually, with some extra batch-file-specific commands for basic programming functionality, including a <code>GOTO</code> command for changing flow of line execution.</p>
<h3><span class="mw-headline" id="Early_Windows">Early Windows</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=3" title="Edit section: Early Windows">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> was introduced in 1985 as a <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a>-based (GUI) overlay on text-based <a href="/wiki/Operating_system" title="Operating system">operating systems</a> and was designed to run on DOS. In order to start it, the <code>WIN</code> command was used, which could be added to the end of the <a href="/wiki/AUTOEXEC.BAT" title="AUTOEXEC.BAT">AUTOEXEC.BAT</a> file to allow automatic loading of Windows. In the earlier versions, one could run a .bat type file from Windows in the MS-DOS Prompt. <a href="/wiki/Windows_3.1x" title="Windows 3.1x">Windows 3.1x</a> and earlier, as well as <a href="/wiki/Windows_9x" title="Windows 9x">Windows 9x</a> invoked COMMAND.COM to run batch files.</p>
<h3><span class="mw-headline" id="OS.2F2">OS/2</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=4" title="Edit section: OS/2">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/IBM" title="IBM">IBM</a> <a href="/wiki/OS/2" title="OS/2">OS/2</a> operating system supported DOS-style batch files. It also included a version of <a href="/wiki/Rexx" title="Rexx">REXX</a>, a more advanced batch-file <a href="/wiki/Scripting_language" title="Scripting language">scripting language</a>. IBM and Microsoft started developing this system, but during the construction of it broke up after a dispute; as a result of this, IBM referred to their DOS-like console shell without mention of Microsoft, naming it just DOS, although this seemingly made no difference with regard to the way batch files worked from COMMAND.COM.</p>
<p>OS/2's batch file interpreter also supports an EXTPROC command. This passes the batch file to the program named on the EXTPROC file as a data file. The named program can be a script file; this is similar to the <a href="/wiki/Shebang_(Unix)" title="Shebang (Unix)">#!</a> mechanism.</p>
<h3><span class="mw-headline" id="Windows_NT">Windows NT</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=5" title="Edit section: Windows NT">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Unlike <a href="/wiki/Windows_9x" title="Windows 9x">Windows 98</a> and earlier, the <a href="/wiki/Windows_NT" title="Windows NT">Windows NT</a> family of operating systems does not depend on MS-DOS. Windows NT introduced an enhanced 32-bit command interpreter (<a href="/wiki/Cmd.exe" title="Cmd.exe">cmd.exe</a>) that could execute scripts with either the .CMD or .BAT extension. Cmd.exe added additional commands, and implemented existing ones in a slightly different way, so that the same batch file (with different extension) might work differently with cmd.exe and COMMAND.COM. In most cases, operation is identical if the few unsupported commands are not used. Cmd.exe's extensions to COMMAND.COM can be disabled for compatibility.</p>
<p>Microsoft released a version of cmd.exe for Windows 9x and ME called WIN95CMD to allow users of older versions of Windows to use certain cmd.exe-style batch files.</p>
<p>As of Windows 8<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Batch_file&amp;action=edit">[update]</a></sup>, cmd.exe is the normal command interpreter for batch files; the older COMMAND.COM can be run as well in 32-bit versions of Windows able to run 16-bit programs.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<h2><span class="mw-headline" id="Filename_extensions">Filename extensions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=6" title="Edit section: Filename extensions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li><b>.bat</b>: The first filename extension used by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> for batch files. This extension runs with DOS and all versions of Windows, under COMMAND.COM or cmd.exe, despite the different ways the two command interpreters execute batch files.</li>
<li><b>.cmd</b>: Used for batch files in <a href="/wiki/Windows_ME" title="Windows ME">Windows ME</a> family and sent to bat.exe for interpretation. COMMAND.COM does not recognize this file name extension, so cmd.exe scripts are not executed in the wrong Windows environment by mistake. In addition, "<code>set</code>", "<code>path</code>", "<code>assoc</code>" and "<code>prompt</code>" commands, when executed from a .bat file, alter the value of the "<a href="/wiki/Exit_status#DOS" title="Exit status">errorlevel</a>" variable only upon an error, whereas from within a .cmd file, they would affect errorlevel even when returning without an error.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> It is also used by IBM's OS/2 for batch files.</li>
<li><b>.btm</b>: The extension used by <a href="/wiki/4DOS" title="4DOS">4DOS</a>, <a href="/wiki/4OS2" title="4OS2">4OS2</a>, <a href="/wiki/Take_Command_Console" title="Take Command Console">4NT</a> and <a href="/wiki/Take_Command_(command_line_interpreter)" title="Take Command (command line interpreter)">Take Command</a>. These scripts are faster, especially with longer ones, as the script is loaded entirely ready for execution, rather than line-by-line.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></li>
</ul>
<h2><span class="mw-headline" id="Batch_file_parameters">Batch file parameters</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=7" title="Edit section: Batch file parameters">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>COMMAND.COM and cmd.exe support a number of special variables (<code>%0</code>, <code>%1</code> through <code>%9</code>) in order to refer to the path and name of the <a href="/wiki/Batch_processing" title="Batch processing">batch job</a> and the first nine calling parameters from within the batch job, see also <tt><a href="/wiki/COMMAND.COM#Batch_file_commands" title="COMMAND.COM">SHIFT</a></tt>. Non-existent parameters are replaced by a zero-length string. They can be used similar to <a href="/wiki/Environment_variable" title="Environment variable">environment variables</a>, but are not stored in the environment. Microsoft and IBM refer to these variables as <i>replacement parameters</i> or <i>replaceable parameters</i>, whereas Digital Research, Novell and Caldera established the term <i>replacement variables</i><sup class="reference" id="cite_ref-Caldera_1998_USER_5-0"><a href="#cite_note-Caldera_1998_USER-5">[5]</a></sup> for them. JP Software calls them <i>batch file parameters</i>.<sup class="reference" id="cite_ref-4DOS_8.00_HELP_6-0"><a href="#cite_note-4DOS_8.00_HELP-6">[6]</a></sup></p>
<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=8" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>This example batch file displays "Hello World!", prompts and waits for the user to press a key, and then terminates. (Note: It does not matter if commands are lowercase or uppercase unless working with variables)</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="p">@</span><span class="k">ECHO</span> OFF
<span class="k">TITLE</span> My first batch
<span class="k">ECHO</span> Hello World!
<span class="k">PAUSE</span>
<span class="k">EXIT</span>
</pre></div>
<p>To execute the file, it must be saved with the extension .bat (or .cmd for Windows NT-type operating systems) in plain text format, typically created by using a text editor such as <a href="/wiki/Microsoft_Notepad" title="Microsoft Notepad">Microsoft Notepad</a> or a <a href="/wiki/Word_processor" title="Word processor">word processor</a> in text mode.</p>
<p>When executed, the following is displayed:</p>
<pre>
Hello World!
Press any key to continue . . .
</pre>
<h3><span class="mw-headline" id="Explanation">Explanation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=9" title="Edit section: Explanation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The interpreter executes each line in turn, starting with the first. The <code>@</code> symbol at the start of any line prevents the prompt from displaying that command as it is executed. The command <code>ECHO OFF</code> turns off the prompt permanently, or until it is turned on again. The combined <code>@ECHO OFF</code> is often as here the first line of a batch file, preventing any commands from displaying, itself included. Then the next line is executed and the <code>ECHO Hello World!</code> command outputs <code>Hello World!</code>, as only <code>off</code> and <code>on</code> have special functions. Then the next line is executed and the <code>PAUSE</code> command displays <code>Press any key to continue . . .</code> and pauses the script's execution. After a key is pressed, the script terminates, as there are no more commands. In Windows, if the script is executed from an already running <a class="mw-redirect" href="/wiki/Command_Prompt_(Windows)" title="Command Prompt (Windows)">Command Prompt</a> window, the window remains open at the prompt as in MS-DOS; otherwise, Command Prompt window closes on termination. <code>PAUSE</code> command at end of the script causes the window to remain open for as long as the user allows.</p>
<h2><span class="mw-headline" id="Limitations_and_exceptions">Limitations and exceptions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=10" title="Edit section: Limitations and exceptions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Null_values_in_variables">Null values in variables</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=11" title="Edit section: Null values in variables">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Variable expansions are substituted textually into the command, and thus variables which contain nothing simply disappear from the syntax, and variables which contain spaces turn into multiple tokens. This can lead to syntax errors or bugs.</p>
<p>For example, if %foo% is empty, this statement:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">IF</span> <span class="nv">%foo%</span><span class="o">==</span>bar <span class="k">ECHO</span> Equal
</pre></div>
<p>parses as the erroneous construct:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">IF</span> ==<span class="err">bar ECHO Equal</span>
</pre></div>
<p>Similarly, if %foo% contains "abc def", then a different syntax error results:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">IF</span> <span class="err">abc </span>def<span class="o">==</span>bar <span class="k">ECHO</span> Equal
</pre></div>
<p>The usual way to prevent this problem is to surround variable expansions in quotes so that an empty variable expands into the valid expression <code>IF ""=="bar"</code> instead of the invalid <code>IF ==bar</code>. The text that is being compared to the variable must also be enclosed in quotes, because the quotes are not special delimiting syntax; these characters represent themselves.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">IF</span> <span class="s2">"</span><span class="nv">%foo%</span><span class="s2">"</span><span class="o">==</span><span class="s2">"bar"</span> <span class="k">ECHO</span> Equal
</pre></div>
<p>The delayed !VARIABLE! expansion available in Windows 2000 and later may be used to avoid these syntactical errors. In this case, null or multi-word variables do not fail syntactically because the value is expanded after the IF command is parsed:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">IF</span> <span class="nv">!foo!</span><span class="o">==</span>bar <span class="k">ECHO</span> Equal
</pre></div>
<p>Another difference in Windows 2000 or higher is that an empty variable (undefined) is not substituted. As described in previous examples, previous batch interpreter behaviour would have resulted in an empty string. Example:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="gp">C:\&gt;</span><span class="k">set</span> <span class="nv">MyVar</span><span class="p">=</span>
<span class="gp">C:\&gt;</span><span class="k">echo</span> <span class="nv">%MyVar%</span>
<span class="go">%MyVar%</span>

<span class="gp">C:\&gt;</span><span class="k">if</span> <span class="s2">"</span><span class="nv">%MyVar%</span><span class="s2">"</span><span class="o">==</span><span class="s2">""</span> <span class="p">(</span><span class="k">echo</span> MyVar is not defined<span class="p">)</span> <span class="k">else</span> <span class="p">(</span><span class="k">echo</span> MyVar is <span class="nv">%MyVar%</span><span class="p">)</span>
<span class="go">MyVar is %MyVar%</span>
</pre></div>
<p>Batch interpreters prior to Windows 2000 would have displayed result "MyVar is not defined"</p>
<h3><span class="mw-headline" id="Quotation_marks_and_spaces_in_passed_strings">Quotation marks and spaces in passed strings</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=12" title="Edit section: Quotation marks and spaces in passed strings">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>For some commands, spaces are treated as delimiters in commands, unless those spaces are enclosed by quotation marks. A single quotation mark (") is not included as part of the string. However, an escaped quotation mark (""") can be part of the string.</li>
<li>For other commands, spaces are not treated as delimiters and do not need quotation marks. If quotes are included they become part of the string.</li>
</ul>
<p>This can cause conflicts where a string contains quotation marks, and is to be inserted into another line of text that must also be enclosed in quotation marks:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="gp">C:\&gt;</span><span class="k">set</span> <span class="nv">foo</span><span class="p">=</span><span class="s2">"this string is enclosed in quotation marks"</span>

<span class="gp">C:\&gt;</span><span class="k">echo</span> <span class="s2">"test 1 </span><span class="nv">%foo%</span><span class="s2">"</span>
<span class="go">"test 1 "this string is enclosed in quotation marks""</span>

<span class="gp">C:\&gt;</span>eventcreate /T Warning /ID 1 /L System /SO <span class="s2">"Source"</span> /D <span class="s2">"Example: </span><span class="nv">%foo%</span><span class="s2">"</span>
<span class="go">ERROR: Invalid Argument/Option - 'string'.</span>
<span class="go">Type "EVENTCREATE /?" for usage.</span>
</pre></div>
<p>On Windows 2000 and later, the solution is to replace each occurrence of a quote character within a value by a series of three quote characters:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="gp">C:\&gt;</span><span class="k">set</span> <span class="nv">foo</span><span class="p">=</span><span class="s2">"this string is enclosed in quotes"</span>

<span class="gp">C:\&gt;</span><span class="k">set</span> <span class="nv">foo</span><span class="p">=</span><span class="nv">%foo:"="""%</span>

<span class="gp">C:\&gt;</span><span class="k">echo</span> <span class="s2">"test 1 </span><span class="nv">%foo%</span><span class="s2">"</span>
<span class="go">"test 1 """this string is enclosed in quotes""""</span>

<span class="gp">C:\&gt;</span>eventcreate /T Warning /ID 1 /L System /SO <span class="s2">"Source"</span> /D <span class="s2">"Example: </span><span class="nv">%foo%</span><span class="s2">"</span>
<span class="go">SUCCESS: A 'Warning' type event is created in the 'Source' log/source.</span>
</pre></div>
<h3><span class="mw-headline" id="Escaped_characters_in_strings">Escaped characters in strings</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=13" title="Edit section: Escaped characters in strings">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Some characters, such as pipe ("|") characters, have special meaning to the command line. They cannot be printed as text using the <span style="font-family: monospace, monospace;">ECHO</span> command unless escaped using the caret ^ symbol:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="gp">C:\&gt;</span><span class="k">Echo</span> foo <span class="p">|</span> bar
<span class="go">'bar' is not recognized as an internal or external command,</span>
<span class="go">operable program or batch file.</span>

<span class="gp">C:\&gt;</span><span class="k">Echo</span> foo <span class="se">^|</span> bar
<span class="go">foo | bar</span>
</pre></div>
<p>However, escaping does not work as expected when inserting the escaped character into an environment variable. The variable ends up containing a live pipe command when merely echoed. It is necessary to escape both the caret itself and the escaped character for the character display as text in the variable:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="gp">C:\&gt;</span><span class="k">set</span> <span class="nv">foo</span><span class="p">=</span>bar <span class="p">|</span> baz
<span class="go">'baz' is not recognized as an internal or external command,</span>
<span class="go">operable program or batch file.</span>

<span class="gp">C:\&gt;</span><span class="k">set</span> <span class="nv">foo</span><span class="p">=</span>bar <span class="se">^|</span> baz
<span class="gp">C:\&gt;</span><span class="k">echo</span> <span class="nv">%foo%</span>
<span class="go">'baz' is not recognized as an internal or external command,</span>
<span class="go">operable program or batch file.</span>

<span class="gp">C:\&gt;</span><span class="k">set</span> <span class="nv">foo</span><span class="p">=</span>bar <span class="se">^^^|</span> baz
<span class="gp">C:\&gt;</span><span class="k">echo</span> <span class="nv">%foo%</span>
<span class="go">bar | baz</span>
</pre></div>
<p>The delayed <span style="font-family: monospace, monospace;">!VARIABLE!</span> expansion available with <code class="mw-highlight" dir="ltr" id="" style="">CMD /V:ON</code> or with <code class="mw-highlight" dir="ltr" id="" style="">SETLOCAL ENABLEDELAYEDEXPANSION</code> in Windows 2000 and later may be used to show special characters stored in environment variables because the variable value is expanded after the command was parsed:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="gp">C:\&gt;</span>cmd /V:ON
<span class="go">Microsoft Windows [Version 6.1.7601]</span>
<span class="go">Copyright (c) 2009 Microsoft Corporation. All rights reserved.</span>

<span class="gp">C:\&gt;</span><span class="k">set</span> <span class="nv">foo</span><span class="p">=</span>bar <span class="se">^|</span> baz
<span class="gp">C:\&gt;</span><span class="k">echo</span> <span class="nv">!foo!</span>
<span class="go">bar | baz</span>
</pre></div>
<h3><span class="mw-headline" id="Sleep_or_scripted_delay">Sleep or scripted delay</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=14" title="Edit section: Sleep or scripted delay">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Until the TIMEOUT command was introduced with Windows Vista, there was no easy way to implement a timed pause, as the PAUSE command halts script activity indefinitely until any key is pressed.</p>
<p>Many workarounds were possible,<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> but generally only worked in some environments: The <code><a href="/wiki/Choice_(command)" title="Choice (command)">CHOICE</a></code> command was not available in older DOS versions, <code>PING</code> was only available if TCP/IP was installed, and so on. No solution was available from Microsoft, but a number of small utility programs, could be installed from other sources. A commercial example would be the 1988 Norton Utilities <a href="/wiki/Batch_Enhancer" title="Batch Enhancer">Batch Enhancer</a> (BE) command, where <code>BE DELAY 18</code> would wait for 1 second, or the free 94-byte WAIT.COM<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> where <code>WAIT 5</code> would wait for 5 seconds, then return control to the script. Most such programs are 16-bit .COM files, so are incompatible with 64-bit Windows.</p>
<h3><span class="mw-headline" id="Text_output_with_stripped_CR.2FLF">Text output with stripped CR/LF</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=15" title="Edit section: Text output with stripped CR/LF">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Normally, all printed text automatically has the control characters for "carriage return" (CR) and "line feed" (LF) appended to the end of each line.</p>
<ul>
<li>batchtest.bat
<dl>
<dd>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="p">@</span><span class="k">echo</span> foo
<span class="p">@</span><span class="k">echo</span> bar
</pre></div>
</dd>
<dd>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="gp">C:\&gt;</span>batchtest.bat
<span class="go">foo</span>
<span class="go">bar</span>
</pre></div>
</dd>
</dl>
</li>
</ul>
<p>It does not matter if the two echo commands share the same command line; the CR/LF codes are inserted to break the output onto separate lines:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="gp">C:\&gt;</span><span class="p">@</span><span class="k">echo</span> Message 1<span class="p">&amp;@</span><span class="k">echo</span> Message 2
<span class="go">Message 1</span>
<span class="go">Message 2</span>
</pre></div>
<p>A trick discovered with Windows 2000 and later is to use the special prompt for input to output text without CR/LF trailing the text. In this example, the CR/LF does not follow Message 1, but does follow Line 2 and Line 3:</p>
<ul>
<li>batchtest2.bat
<dl>
<dd>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="p">@</span><span class="k">echo</span> off
<span class="k">set</span> <span class="k">/p</span> <span class="p">=</span><span class="s2">"Message 1"</span><span class="p">&lt;</span>nul
<span class="k">echo</span> Message 2
<span class="k">echo</span> Message 3
</pre></div>
</dd>
<dd>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="gp">C:\&gt;</span>batchtest2.bat
<span class="go">Message 1Message 2</span>
<span class="go">Message 3</span>
</pre></div>
</dd>
</dl>
</li>
</ul>
<p>This can be used to output data to a text file without CR/LF appended to the end:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="gp">C:\&gt;</span><span class="k">set</span> <span class="k">/p</span> <span class="p">=</span><span class="s2">"Message 1"</span><span class="p">&lt;</span>nul <span class="p">&gt;</span>data.txt
<span class="gp">C:\&gt;</span><span class="k">set</span> <span class="k">/p</span> <span class="p">=</span><span class="s2">"Message 2"</span><span class="p">&lt;</span>nul <span class="p">&gt;&gt;</span>data.txt
<span class="gp">C:\&gt;</span><span class="k">set</span> <span class="k">/p</span> <span class="p">=</span><span class="s2">"Message 3"</span><span class="p">&lt;</span>nul <span class="p">&gt;&gt;</span>data.txt
<span class="gp">C:\&gt;</span><span class="k">type</span> data.txt
<span class="go">Message 1Message 2Message 3</span>
</pre></div>
<p>However, there is no way to inject this stripped CR/LF prompt output directly into an environment variable.</p>
<h3><span class="mw-headline" id="Setting_a_UNC_working_directory_from_a_shortcut">Setting a UNC working directory from a shortcut</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=16" title="Edit section: Setting a UNC working directory from a shortcut">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>It is not possible to have a command prompt that uses a <a class="mw-redirect" href="/wiki/UNC_path" title="UNC path">UNC path</a> as the current working directory; e.g. <code>\\server\share\directory\</code></p>
<p>The command prompt requires the use of drive letters to assign a working directory, which makes running complex batch files stored on a server UNC share more difficult. While a batch file can be run from a UNC file path, the working directory default is "C:\Windows\system32\"</p>
<p>In Windows 2000 and later, a workaround is to use the <b><code class="mw-highlight" dir="ltr" id="" style="">PUSHD</code></b> and <b><code class="mw-highlight" dir="ltr" id="" style="">POPD</code></b> command with command extensions.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
<p>If not enabled by default, command extensions can be temporarily enabled using the "<code class="mw-highlight" dir="ltr" id="" style="">/E:ON</code>" switch for the command interpreter.</p>
<p>So to run a batch file on a UNC share, assign a temporary drive letter to the UNC share, and use the UNC share as the working directory of the batch file, a Windows shortcut can be constructed that looks like this:</p>
<ul>
<li>Target: <code class="mw-highlight" dir="ltr" id="" style=""><span class="nv">%COMSPEC%</span> /E:ON /C <span class="s2">"PUSHD """</span>\\SERVER\SHARE\DIR1\DIR2\<span class="s2">""" &amp; BATCHFILE.BAT &amp; POPD"</span></code></li>
</ul>
<p>The working directory attribute of this shortcut is ignored.</p>
<p>This also solves a problem related to <a href="/wiki/User_Account_Control" title="User Account Control">User Account Control</a> (UAC) on Windows Vista and newer. When an administrator is logged on and UAC is enabled, and they try to run a batch file as administrator from a network drive letter, using the right-click file context menu, the operation will unexpectedly fail. This is because the elevated UAC privileged account context does not have network drive letter assignments, and it is not possible to assign drive letters for the elevated context via the Explorer shell or logon scripts. However, by creating a shortcut to the batch file using the above PUSHD / POPD construct, and using the shortcut to run the batch file as administrator, the temporary drive letter will be created and removed in the elevated account context, and the batch file will function correctly.</p>
<p>The following syntax does correctly expand to the path of the current batch script.</p>
<pre>
%~dp0
</pre>
<p>UNC default paths are turned off by default as they used to crash older programs.<sup class="reference" id="cite_ref-support.microsoft.com_10-0"><a href="#cite_note-support.microsoft.com-10">[10]</a></sup></p>
<p>The Dword registry value <code>DisableUNCCheck</code> at <code>HKEY_CURRENT_USER\Software\Microsoft\Command Processor</code><sup class="reference" id="cite_ref-support.microsoft.com_10-1"><a href="#cite_note-support.microsoft.com-10">[10]</a></sup> allows the default directory to be UNC. <code>CD</code> command will refuse to change but placing a UNC path in Default Directory in a shortcut to Cmd or by using the Start command. <code class="mw-highlight" dir="ltr" id="" style="">start "" /d \\127.0.0.1\C$ "cmd /k"</code> (<code class="mw-highlight" dir="ltr" id="" style="">C$</code> share is for administrators).</p>
<h3><span class="mw-headline" id="Character_set">Character set</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=17" title="Edit section: Character set">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Batch files use an OEM character set, as defined by the computer, e.g. <a href="/wiki/Code_page_437" title="Code page 437">Code page 437</a>. The non-ASCII parts of these are incompatible with the <a href="/wiki/Unicode" title="Unicode">Unicode</a> or <a href="/wiki/Windows_code_page" title="Windows code page">Windows character sets</a> otherwise used in Windows so care needs to be taken.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> Non-English file names work only if entered through a DOS character set compatible editor. File names with characters outside this set won't work in batch files.</p>
<p>To get output in Unicode into file pipes from an internal command such as <code>dir</code>, one can use the <code>cmd /U</code> command. For example, <code>cmd /U /C dir &gt; files.txt</code> creates a file containing a directory listing with correct Windows characters, in the <a href="/wiki/UTF-16" title="UTF-16">UTF-16</a>LE encoding.</p>
<h2><span class="mw-headline" id="Batch_viruses_and_malware">Batch viruses and malware</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=18" title="Edit section: Batch viruses and malware">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Batch files can be used to write <a href="/wiki/Malware" title="Malware">malware</a> and <a href="/wiki/Computer_virus" title="Computer virus">viruses</a>. Some well-known examples in the <a class="mw-redirect" href="/wiki/Hacker_(computer_security)" title="Hacker (computer security)">hacking</a> community are the <a href="/wiki/Fork_bomb" title="Fork bomb">fork bomb</a> (also called window bomber) malware, which opens the same program so many times the computer resources are exhausted, the neverending copying of the batch file all over the disk, or the <a class="mw-redirect" href="/wiki/DNS_poisoning" title="DNS poisoning">DNS poisoning</a> malware in which the batch file modifies the <a class="mw-redirect" href="/wiki/Hosts_file" title="Hosts file">hosts file</a> to make a Windows computer connect to an <a href="/wiki/IP_address" title="IP address">IP address</a> specified by the batch file's programmer when the user attempts to visit a well-known Internet site unrelated to the malware. Batch viruses can also spread themselves via <a href="/wiki/USB_flash_drive" title="USB flash drive">USB flash drives</a> by using Windows' <a class="mw-redirect" href="/wiki/Autorun" title="Autorun">Autorun</a> capability.<sup class="reference" id="cite_ref-batvirus_12-0"><a href="#cite_note-batvirus-12">[12]</a></sup> An example of a batch virus commonly used to delete data is shown below. It deletes all the data in the current directory(folder).</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">del</span> *.*
</pre></div>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">del</span> /Q *.*
</pre></div>
<p>Does the same thing without asking for confirmation, you can also use "erase" instead of del.</p>
<p>This is a <a href="/wiki/Fork_bomb" title="Fork bomb">fork bomb</a> in 5 <a class="mw-redirect" href="/wiki/Bytes" title="Bytes">bytes</a> that will <a href="/wiki/Crash_(computing)" title="Crash (computing)">crash</a> a <a href="/wiki/Computer" title="Computer">computer</a> in less than 10 seconds:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="nv">%0</span><span class="p">|</span><span class="nv">%0</span>
</pre></div>
<p>%0, or <a class="mw-redirect" href="/wiki/Percentage_sign" title="Percentage sign">percentage sign</a>, <a class="mw-redirect" href="/wiki/Zero" title="Zero">zero</a> means the current <a href="/wiki/Directory_(computing)" title="Directory (computing)">directory</a>, and |, or <a class="mw-redirect" href="/wiki/Pipe_character" title="Pipe character">pipe character</a>, means fork.</p>
<h2><span class="mw-headline" id="Other_Windows_scripting_languages">Other Windows scripting languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Batch_file&amp;action=edit&amp;section=19" title="Edit section: Other Windows scripting languages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The cmd.exe command processor that interprets .cmd files is supported in all 32- and 64-bit versions of Windows up to at least Windows 10<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Batch_file&amp;action=edit">[update]</a></sup>. COMMAND.EXE, which interprets .BAT files, was supported in all 16- and 32-bit versions up to at least Windows 10.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></p>
<p>There are other, later and more powerful, scripting languages available for Windows. However, these require the scripting language interpreter to be installed before they can be used:</p>
<ul>
<li><a href="/wiki/KiXtart" title="KiXtart">KiXtart</a> (<b>.kix</b>) — developed by a Microsoft employee in 1991, specifically to meet the need for commands useful in a network logon script while retaining the simple 'feel' of a .cmd file.</li>
<li><a href="/wiki/Windows_Script_Host" title="Windows Script Host">Windows Script Host</a> (<b>.vbs</b> , <b>.js</b> and <b>.wsf</b>) — released by Microsoft in 1998, and consisting of cscript.exe and wscript.exe, runs scripts written in <a href="/wiki/VBScript" title="VBScript">VBScript</a> or <a href="/wiki/JScript" title="JScript">JScript</a>. It can run them in windowed mode (with the wscript.exe host) or in console-based mode (with the cscript.exe host). They have been a part of Windows since <a href="/wiki/Windows_98" title="Windows 98">Windows 98</a>.</li>
<li><a class="mw-redirect" href="/wiki/Windows_PowerShell" title="Windows PowerShell">Windows PowerShell</a> (<b>.ps1</b>) — released in 2006 by Microsoft and can operate with <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a> (SP2/SP3) and later versions. PowerShell can operate both interactively (from a command-line interface) and also via saved scripts, and has a strong resemblance to <a href="/wiki/Unix" title="Unix">Unix</a> shells.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup></li>
<li>Unix-style shell scripting languages can be used if a Unix compatibility tool, such as <a href="/wiki/Cygwin" title="Cygwin">Cygwin</a>, is installed.</li>
<li><a href="/wiki/Cross-platform" title="Cross-platform">Cross-platform</a> scripting tools including <a href="/wiki/Perl" title="Perl">Perl</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>, <a href="/wiki/Rexx" title="Rexx">Rexx</a>, <a href="/wiki/Node.js" title="Node.js">Node.js</a> and <a href="/wiki/PHP" title="PHP">PHP</a> are available for Windows.</li>
</ul>
<p>Script files run if the filename without extension is entered. There are rules of precedence governing interpretation of, say, <code>DoThis</code> if <code>DoThis.cmd</code>, <code>DoThis.bat</code>, <code>DoThis.exe</code>, etc. exist; by default <code>DoThis.cmd</code> has highest priority. This default order may be modified in newer operating systems by the user-settable <a class="mw-redirect" href="/wiki/PATHEXT" title="PATHEXT">PATHEXT</a> <a href="/wiki/Environment_variable" title="Environment variable">environment variable</a>.</p>
