<p>In <a href="/wiki/Computer_science" title="Computer science">computer science</a>, a <b>NOP</b> or <b>NOOP</b> (short for <b>No Operation</b>) is an <a href="/wiki/Assembly_language" title="Assembly language">assembly language</a> instruction, <a href="/wiki/Programming_language" title="Programming language">programming language</a> statement, or <a class="mw-redirect" href="/wiki/Protocol_(computing)" title="Protocol (computing)">computer protocol</a> command that does nothing.</p>

<h2><span class="mw-headline" id="Machine_instruction">Machine instruction</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=1" title="Edit section: Machine instruction">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Some computer <a href="/wiki/Instruction_set" title="Instruction set">instruction sets</a> include an instruction whose explicit purpose is to not change the state of any of the programmer-accessible <a href="/wiki/Processor_register" title="Processor register">registers</a>, <a class="mw-redirect" href="/wiki/Process_states" title="Process states">status flags</a>, or <a class="mw-redirect" href="/wiki/Main_memory" title="Main memory">memory</a> and which may require a specific number of <a class="mw-redirect" href="/wiki/Clock_cycles" title="Clock cycles">clock cycles</a> to execute. In other instruction sets, a NOP has to be simulated by executing an instruction having operands that cause the same effect (e.g., on the <a href="/wiki/SPARC" title="SPARC">SPARC</a> processor, the instruction <code>sethi 0, %g0</code> is the recommended solution).</p>
<p>A NOP is most commonly used for timing purposes, to force <a href="/wiki/Data_structure_alignment" title="Data structure alignment">memory alignment</a>, to prevent <a href="/wiki/Hazard_(computer_architecture)" title="Hazard (computer architecture)">hazards</a>, to occupy a <a class="mw-redirect" href="/wiki/Branch_delay_slot" title="Branch delay slot">branch delay slot</a>, to render void an existing instruction such as a jump, or as a place-holder to be replaced by active instructions later on in program development (or to replace removed instructions when refactoring would be problematic or time-consuming). In some cases, a NOP can have minor side effects; for example, on the <a href="/wiki/Motorola_68000" title="Motorola 68000">Motorola 68000</a> series of processors, the NOP opcode causes a synchronization of the <a href="/wiki/Pipeline_(computing)" title="Pipeline (computing)">pipeline</a>.<sup class="reference" id="cite_ref-m68ref_1-0"><a href="#cite_note-m68ref-1">[1]</a></sup></p>

<p>From a hardware design point of view, unmapped areas of a bus are often designed to return zeroes; since the <a href="/wiki/NOP_slide" title="NOP slide">NOP slide</a> behavior is often desirable, it gives a bias to coding it with the all-zeroes opcode.</p>
<h2><span class="mw-headline" id="Code">Code</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=2" title="Edit section: Code">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>NOP is sometimes used as a description for the action performed by a function or a sequence of programming language statements if the function or code has no effect (it might also be called <a href="/wiki/Redundant_code" title="Redundant code">redundant code</a>). A common <a class="mw-redirect" href="/wiki/Compiler_optimization" title="Compiler optimization">compiler optimization</a> is the <a href="/wiki/Dead_code_elimination" title="Dead code elimination">detection and removal of this kind of code</a>. Such code may be required by the grammar of the programming language, which does not allow a blank.</p>
<h3><span class="mw-headline" id="Ada">Ada</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=3" title="Edit section: Ada">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In a <a href="/wiki/Ada_(programming_language)" title="Ada (programming language)">Ada</a>, the <code>null</code> statement serves as a NOP.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> As the syntax forbids that control statements or functions be empty, the <code>null</code> statement must be used to specify that no action is required. (Thus, if the programmer forgets to write a sequence of statements, the program will fail to compile.)</p>
<h3><span class="mw-headline" id="C_and_derivatives">C and derivatives</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=4" title="Edit section: C and derivatives">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The following is an example of a single <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> statement that <i>behaves</i> like a NOP. In practice, most <a class="mw-redirect" href="/wiki/Compilers" title="Compilers">compilers</a> will not generate code for this statement:</p>
<pre>
  i+1;
</pre>
<p>This statement performs an addition and discards the result. Indeed, any statement without side effects (and that does not affect control flow, e.g., break, return) can be removed, as the result of the computation is discarded.</p>
<p>The simplest possible statement in C that behaves like a NOP is the so-called <i>null statement</i>, which is just a semi-colon in a context requiring a statement. (A compiler is not required to generate a NOP instruction in this case; typically, no instructions whatsoever would be generated.)</p>
<pre>
  ;
</pre>
<p>Alternatively, an empty block (compound statement) may be used, and may be more legible:</p>
<pre>
  {}
</pre>
<p>In some cases, such as the body of a function, a block must be used, but this can be empty. In C, statements cannot be empty – simple statements must end with a ; (semicolon) while compound statements are enclosed in {} (braces), which does not itself need a following semicolon. Thus in contexts where a statement is grammatically required, some such null statement can be used.</p>
<p>The null statement is useless by itself, but it can have a syntactic use in a wider context, e.g., within the context of a loop:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
  <span class="k">while</span> <span class="p">(</span><span class="n">getchar</span><span class="p">()</span> <span class="o">!=</span> <span class="sc">'\n'</span><span class="p">)</span> <span class="p">{}</span>
</pre></div>
<p>alternatively,</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
  <span class="k">while</span> <span class="p">(</span><span class="n">getchar</span><span class="p">()</span> <span class="o">!=</span> <span class="sc">'\n'</span><span class="p">)</span>
      <span class="p">;</span>
</pre></div>
<p>or more tersely:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
  <span class="k">while</span> <span class="p">(</span><span class="n">getchar</span><span class="p">()</span> <span class="o">!=</span> <span class="sc">'\n'</span><span class="p">);</span>
</pre></div>
<p>(note that the last form may be confusing, and as such generates a warning with some compilers or compiler options, as semicolon usually indicates an end of function call instruction when placed after a round parenthesis on the end of line).</p>
<p>The above code continues calling the function <i>getchar()</i> until it returns a \n (newline) character, essentially fast-forwarding the current reading location of standard input to the beginning of next line.</p>
<h3><span class="mw-headline" id="Fortran"><a href="/wiki/Fortran" title="Fortran">Fortran</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=5" title="Edit section: Fortran">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In Fortran, the <code>CONTINUE</code> statement is used in some contexts such as the last statement in a DO loop, although it can be used anywhere, and does not have any functionality.</p>
<h3><span class="mw-headline" id="Pascal"><a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=6" title="Edit section: Pascal">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>As with C, the ; used by itself can be used as a null statement. In fact, due to the specification of the language, in a BEGIN / END block, the semicolon is optional before the END statement, thus a semicolon used there is superfluous.</p>
<p>Also, a block consisting of <code>BEGIN END;</code> may be used as a placeholder to indicate no action, even if placed inside another BEGIN / END block.</p>
<h3><span class="mw-headline" id="Python">Python</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=7" title="Edit section: Python">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a class="mw-redirect" href="/wiki/Python_programming_language" title="Python programming language">Python programming language</a> has a <a class="mw-redirect" href="/wiki/Python_programming_language#Statements_and_control_flow" title="Python programming language"><code>pass</code> statement</a> which has no effect when executed and thus serves as a NOP. It is primarily used to ensure correct syntax due to Python's <a href="/wiki/Off-side_rule" title="Off-side rule">indentation-sensitive syntax</a>; for example the syntax for definition of a <a href="/wiki/Class_(computer_programming)" title="Class (computer programming)">class</a> requires an indented block with the class logic, which has to be expressed as <code>pass</code> when it should be empty.</p>
<h3><span class="mw-headline" id="jQuery">jQuery</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=8" title="Edit section: jQuery">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/JQuery" title="JQuery">jQuery</a> library provides a function <code>jQuery.noop()</code>, which does nothing.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></p>
<h3><span class="mw-headline" id="Angular">Angular</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=9" title="Edit section: Angular">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/AngularJS" title="AngularJS">Angular</a> framework provides <a class="external text" href="https://docs.angularjs.org/api/ng/function/angular.noop" rel="nofollow">angular.noop</a> function that performs no operations.</p>
<h3><span class="mw-headline" id="Shell_Scripting_.28bash.2C_zsh.2C_etc.29">Shell Scripting (bash, zsh, etc)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=10" title="Edit section: Shell Scripting (bash, zsh, etc)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The ':' [colon] character is a shell builtin that has similar effect to a "NOP" (a do-nothing operation). It's not technically an NOP, as it changes the special parameter $? (exit status of last command) to 0. It may be considered a synonym for the shell builtin 'true', and its exit status is true (0).<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup><sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></p>
<h3><span class="mw-headline" id="Visual_Basic">Visual Basic</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=11" title="Edit section: Visual Basic">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic</a> language has a <code>;</code> statement, which does nothing.</p>
<h2><span class="mw-headline" id="NOP_protocol_commands">NOP protocol commands</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=12" title="Edit section: NOP protocol commands">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Many <a class="mw-redirect" href="/wiki/Protocol_(computing)" title="Protocol (computing)">computer protocols</a>, such as <a href="/wiki/Telnet" title="Telnet">telnet</a>, include a NOP command that a client can issue to request a response from the server without requesting any other actions. Such a command can be used to <a href="/wiki/Keepalive" title="Keepalive">ensure the connection is still alive</a> or that the server is responsive. A NOOP command is part of the following protocols (<i>this is a partial list</i>):</p>
<ul>
<li><a href="/wiki/Telnet" title="Telnet">telnet</a></li>
<li><a href="/wiki/File_Transfer_Protocol" title="File Transfer Protocol">FTP</a></li>
<li><a href="/wiki/Simple_Mail_Transfer_Protocol" title="Simple Mail Transfer Protocol">SMTP</a></li>
<li><a class="mw-redirect" href="/wiki/X_Window_core_protocol" title="X Window core protocol">X11</a></li>
<li><a href="/wiki/Post_Office_Protocol" title="Post Office Protocol">POP3</a></li>
<li><a href="/wiki/Network_News_Transfer_Protocol" title="Network News Transfer Protocol">NNTP</a></li>
<li><a href="/wiki/Finger_protocol" title="Finger protocol">finger</a></li>
<li><a href="/wiki/Internet_Message_Access_Protocol" title="Internet Message Access Protocol">IMAP4</a></li>
<li><a class="mw-redirect" href="/wiki/BitTorrent_(protocol)" title="BitTorrent (protocol)">BitTorrent</a></li>
<li><a href="/wiki/Character_encoding" title="Character encoding">Character encoding</a>: the <a class="mw-redirect" href="/wiki/Null_(Character)" title="Null (Character)">null</a> <a href="/wiki/Control_character" title="Control character">control character</a></li>
</ul>
<p>Note that unlike the other protocols listed, the IMAP4 NOOP command has a specific purpose - it allows the server to send any pending notifications to the client.</p>
<p>While most <a href="/wiki/Telnet" title="Telnet">telnet</a> or <a class="mw-redirect" href="/wiki/FTP" title="FTP">FTP</a> servers respond to a NOOP command with "OK" or "+OK", some programmers have added quirky responses to the client. For example, the <code>ftpd</code> daemon of <a href="/wiki/MINIX" title="MINIX">MINIX</a> responds to NOOP with the message:<sup class="reference" id="cite_ref-minix_ftpd_11-0"><a href="#cite_note-minix_ftpd-11">[11]</a></sup></p>
<pre>
<a href="/wiki/List_of_FTP_server_return_codes" title="List of FTP server return codes">200</a> NOOP to you too!
</pre>
<h2><span class="mw-headline" id="Cracking">Cracking</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=13" title="Edit section: Cracking">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>NOPs are often involved when <a href="/wiki/Software_cracking" title="Software cracking">cracking</a> software that checks for serial numbers, specific hardware or software requirements, presence or absence of hardware <a href="/wiki/Dongle" title="Dongle">dongles</a>, etc. This is accomplished by altering <a href="/wiki/Subroutine" title="Subroutine">functions and subroutines</a> to bypass security checks and instead simply return the expected value being checked for. Because most of the instructions in the security check routine will be unused, these would be replaced with NOPs, thus removing the software's security functionality without attracting any attention.</p>
<h2><span class="mw-headline" id="Security_exploits">Security exploits</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NOP&amp;action=edit&amp;section=14" title="Edit section: Security exploits">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The NOP opcode can be used to form a <a href="/wiki/NOP_slide" title="NOP slide">NOP slide</a>, which allows code to execute when the exact value of the instruction pointer is indeterminate (e.g., when a buffer overflow causes a function's return address on the stack to be overwritten).</p>
