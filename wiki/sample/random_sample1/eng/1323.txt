<h1 class="firstHeading" id="firstHeading" lang="en">NP-hardness</h1>
<div class="mw-body-content" id="bodyContent">

<p><b>NP-hardness</b> (<a href="/wiki/NP_(complexity)" title="NP (complexity)"><i>n</i>on-deterministic <i>p</i>olynomial-time</a> hard), in <a href="/wiki/Computational_complexity_theory" title="Computational complexity theory">computational complexity theory</a>, is a class of problems that are, informally, "at least as hard as the hardest problems in <a href="/wiki/NP_(complexity)" title="NP (complexity)">NP</a>". More precisely, a problem <i>H</i> is NP-hard when every problem <i>L</i> in <a href="/wiki/NP_(complexity)" title="NP (complexity)">NP</a> can be <a href="/wiki/Reduction_(complexity)" title="Reduction (complexity)">reduced</a> in <a class="mw-redirect" href="/wiki/Polynomial_time" title="Polynomial time">polynomial time</a> to <i>H</i>, that is: assuming a solution for <i>H</i> takes 1 unit time, we can use <i>H</i>‎'s solution to solve <i>L</i> in polynomial time.<sup class="reference" id="cite_ref-Leeuwen_1-0"><a href="#cite_note-Leeuwen-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> As a consequence, finding a polynomial algorithm to solve any NP-hard problem would give polynomial algorithms for all the problems in NP, which is unlikely as many of them are considered hard.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<p>A common misconception is that the <i>NP</i> in "NP-hard" stands for "non-polynomial" when in fact it stands for "<b>N</b>on-deterministic <b>P</b>olynomial acceptable problems".<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> Although it is suspected that there are no polynomial-time algorithms for NP-hard problems, this has never been proven.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> Moreover, the class <a href="/wiki/P_(complexity)" title="P (complexity)">P</a> in which all problems can be solved in polynomial time, is contained in the <a href="/wiki/NP_(complexity)" title="NP (complexity)">NP</a> class.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>

<h2><span class="mw-headline" id="Definition">Definition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NP-hardness&amp;action=edit&amp;section=1" title="Edit section: Definition">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A <a href="/wiki/Decision_problem" title="Decision problem">decision problem</a> <i>H</i> is NP-hard when for every problem <i>L</i> in NP, there is a <a href="/wiki/Polynomial-time_reduction" title="Polynomial-time reduction">polynomial-time reduction</a> from <i>L</i> to <i>H</i><sup class="reference" id="cite_ref-Leeuwen_1-1"><a href="#cite_note-Leeuwen-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:80</sup> An equivalent definition is to require that every problem <i>L</i> in NP can be solved in <a class="mw-redirect" href="/wiki/Polynomial_time" title="Polynomial time">polynomial time</a> by an <a href="/wiki/Oracle_machine" title="Oracle machine">oracle machine</a> with an oracle for <i>H</i>.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> Informally, we can think of an algorithm that can call such an oracle machine as a subroutine for solving <i>H</i>, and solves <i>L</i> in polynomial time, if the subroutine call takes only one step to compute.</p>
<p>Another definition is to require that there is a polynomial-time reduction from an <a class="mw-redirect" href="/wiki/NP-complete" title="NP-complete">NP-complete</a> problem <i>G</i> to <i>H</i>.<sup class="reference" id="cite_ref-Leeuwen_1-2"><a href="#cite_note-Leeuwen-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:91</sup> As any problem <i>L</i> in NP reduces in polynomial time to <i>G</i>, <i>L</i> reduces in turn to <i>H</i> in polynomial time so this new definition implies the previous one. Awkwardly, it does not restrict the class NP-hard to decision problems, for instance it also includes <a href="/wiki/Search_problem" title="Search problem">search problems</a>, or <a href="/wiki/Optimization_problem" title="Optimization problem">optimization problems</a>.</p>

</ul>
<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NP-hardness&amp;action=edit&amp;section=3" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>An example of an NP-hard problem is the decision <a href="/wiki/Subset_sum_problem" title="Subset sum problem">subset sum problem</a>, which is this: given a set of integers, does any non-empty subset of them add up to zero? That is a <a href="/wiki/Decision_problem" title="Decision problem">decision problem</a>, and happens to be NP-complete. Another example of an NP-hard problem is the optimization problem of finding the least-cost cyclic route through all nodes of a weighted graph. This is commonly known as the <a class="mw-redirect" href="/wiki/Traveling_salesman_problem" title="Traveling salesman problem">traveling salesman problem</a>.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<p>There are decision problems that are NP-hard but not NP-complete, for example the <a href="/wiki/Halting_problem" title="Halting problem">halting problem</a>. This is the problem which asks "given a program and its input, will it run forever?" That is a <i>yes</i>/<i>no</i> question, so this is a decision problem. It is easy to prove that the halting problem is <i>NP-hard</i> but not <i>NP-complete</i>. For example, the <a href="/wiki/Boolean_satisfiability_problem" title="Boolean satisfiability problem">Boolean satisfiability problem</a> can be reduced to the halting problem by transforming it to the description of a <a href="/wiki/Turing_machine" title="Turing machine">Turing machine</a> that tries all <a href="/wiki/Truth_value" title="Truth value">truth value</a> assignments and when it finds one that satisfies the formula it halts and otherwise it goes into an infinite loop. It is also easy to see that the halting problem is not in <i>NP</i> since all problems in NP are decidable in a finite number of operations, while the halting problem, in general, is <a href="/wiki/Undecidable_problem" title="Undecidable problem">undecidable</a>. There are also NP-hard problems that are neither NP-complete nor undecidable. For instance, the language of <a href="/wiki/True_quantified_Boolean_formula" title="True quantified Boolean formula">True quantified Boolean formulas</a> is decidable in <a href="/wiki/PSPACE" title="PSPACE">polynomial space</a>, but not non-deterministic polynomial time (unless NP = <a href="/wiki/PSPACE" title="PSPACE">PSPACE</a>).<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
<h2><span class="mw-headline" id="NP-naming_convention">NP-naming convention</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NP-hardness&amp;action=edit&amp;section=4" title="Edit section: NP-naming convention">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>NP-hard problems do not have to be elements of the complexity class NP. As NP plays a central role in <a href="/wiki/Computational_complexity_theory" title="Computational complexity theory">computational complexity</a>, it is used as the basis of several classes:</p>
<dl>
<dt><a href="/wiki/NP_(complexity)" title="NP (complexity)">NP</a></dt>
<dd>Class of computational decision problems for which a given solution can be verified as a solution in polynomial time by a deterministic Turing machine (or <i>solvable</i> by a <i>non-deterministic</i> Turing machine in polynomial time).</dd>
<dt>NP-hard</dt>
<dd>Class of decision problems which are at least as hard as the hardest problems in NP. Problems that are NP-hard do not have to be elements of NP; indeed, they may not even be decidable.</dd>
<dt><a class="mw-redirect" href="/wiki/NP-complete" title="NP-complete">NP-complete</a></dt>
<dd>Class of decision problems which contains the hardest problems in NP. Each NP-complete problem has to be in NP.</dd>
<dt><a href="/wiki/NP-easy" title="NP-easy">NP-easy</a></dt>
<dd>At most as hard as NP, but not necessarily in NP.</dd>
<dt><a href="/wiki/NP-equivalent" title="NP-equivalent">NP-equivalent</a></dt>
<dd>Decision problems that are both NP-hard and NP-easy, but not necessarily in NP.</dd>
<dt><a href="/wiki/NP-intermediate" title="NP-intermediate">NP-intermediate</a></dt>
<dd>If P and NP are different, then there exist decision problems in the region of NP that fall between P and the NP-complete problems. (If P and NP are the same class, then NP-intermediate problems do not exist.)</dd>
</dl>
