<p><b>OpenCV</b> (<i>Open Source Computer Vision</i>) is a <a href="/wiki/Library_(computing)" title="Library (computing)">library of programming functions</a> mainly aimed at real-time <a href="/wiki/Computer_vision" title="Computer vision">computer vision</a>.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> Originally developed by <a class="mw-redirect" href="/wiki/Intel_Corporation" title="Intel Corporation">Intel</a>'s research center in <a href="/wiki/Nizhny_Novgorod" title="Nizhny Novgorod">Nizhny Novgorod</a> (Russia), it was later supported by <a href="/wiki/Willow_Garage" title="Willow Garage">Willow Garage</a> and is now maintained by Itseez.<sup class="reference" id="cite_ref-Itseez_1-1"><a href="#cite_note-Itseez-1">[1]</a></sup> The library is <a href="/wiki/Cross-platform" title="Cross-platform">cross-platform</a> and free for use under the <a class="mw-redirect" href="/wiki/Open-source" title="Open-source">open-source</a> <a class="mw-redirect" href="/wiki/BSD_license" title="BSD license">BSD license</a>.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenCV&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Officially launched in 1999, the OpenCV project was initially an <a href="/wiki/Intel_Research_Lablets" title="Intel Research Lablets">Intel Research</a> initiative to advance <a href="/wiki/Central_processing_unit" title="Central processing unit">CPU</a>-intensive applications, part of a series of projects including <a href="/wiki/Real-time_computing" title="Real-time computing">real-time</a> <a href="/wiki/Ray_tracing_(graphics)" title="Ray tracing (graphics)">ray tracing</a> and <a class="mw-redirect" href="/wiki/3D_Display" title="3D Display">3D display</a> walls. The main contributors to the project included a number of optimization experts in Intel Russia, as well as Intel’s Performance Library Team. In the early days of OpenCV, the goals of the project were described<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> as:</p>
<blockquote>
<ul>
<li>Advance vision research by providing not only open but also <a class="mw-redirect" href="/wiki/Code_optimization" title="Code optimization">optimized code</a> for basic vision infrastructure. No more <a href="/wiki/Reinventing_the_wheel" title="Reinventing the wheel">reinventing the wheel</a>.</li>
<li>Disseminate vision knowledge by providing a common infrastructure that developers could build on, so that code would be more readily readable and transferable.</li>
<li>Advance vision-based commercial applications by making <a class="mw-redirect" href="/wiki/Portability_(computer_science)" title="Portability (computer science)">portable</a>, performance-optimized code available for free—with a license that did not require to be open or free themselves.</li>
</ul>
</blockquote>
<p>The first alpha version of OpenCV was released to the public at the <a href="/wiki/Conference_on_Computer_Vision_and_Pattern_Recognition" title="Conference on Computer Vision and Pattern Recognition">IEEE Conference on Computer Vision and Pattern Recognition</a> in 2000, and five betas were released between 2001 and 2005. The first 1.0 version was released in 2006. In mid-2008, OpenCV obtained corporate support from <a href="/wiki/Willow_Garage" title="Willow Garage">Willow Garage</a>, and is now again under active development. A version 1.1 "pre-release" was released in October 2008.</p>
<p>The second major release of the OpenCV was on October 2009. OpenCV 2 includes major changes to the <a href="/wiki/C%2B%2B" title="C++">C++</a> interface, aiming at easier, more type-safe patterns, new functions, and better implementations for existing ones in terms of performance (especially on multi-core systems). Official releases now occur every six months<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> and development is now done by an independent Russian team supported by commercial corporations.</p>
<p>In August 2012, support for OpenCV was taken over by a non-profit foundation OpenCV.org, which maintains a developer<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> and user site.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>

<p>OpenCV's application areas include:</p>
<ul>
<li>2D and 3D feature toolkits</li>
<li><a class="mw-redirect" href="/wiki/Egomotion" title="Egomotion">Egomotion</a> estimation</li>
<li><a href="/wiki/Facial_recognition_system" title="Facial recognition system">Facial recognition system</a></li>
<li><a href="/wiki/Gesture_recognition" title="Gesture recognition">Gesture recognition</a></li>
<li><a href="/wiki/Human%E2%80%93computer_interaction" title="Human–computer interaction">Human–computer interaction</a> (HCI)</li>
<li><a class="mw-redirect" href="/wiki/Mobile_robotics" title="Mobile robotics">Mobile robotics</a></li>
<li><a class="new" href="/w/index.php?title=Motion_understanding&amp;action=edit&amp;redlink=1" title="Motion understanding (page does not exist)">Motion understanding</a></li>
<li><a class="new" href="/w/index.php?title=Object_identification&amp;action=edit&amp;redlink=1" title="Object identification (page does not exist)">Object identification</a></li>
<li><a class="mw-redirect" href="/wiki/Segmentation_(image_processing)" title="Segmentation (image processing)">Segmentation</a> and recognition</li>
<li><a href="/wiki/Stereopsis" title="Stereopsis">Stereopsis</a> stereo vision: depth perception from 2 cameras</li>
<li><a href="/wiki/Structure_from_motion" title="Structure from motion">Structure from motion</a> (SFM)</li>
<li><a href="/wiki/Video_tracking" title="Video tracking">Motion tracking</a></li>
<li><a href="/wiki/Augmented_reality" title="Augmented reality">Augmented reality</a></li>
</ul>
<p>To support some of the above areas, OpenCV includes a statistical <a href="/wiki/Machine_learning" title="Machine learning">machine learning</a> library that contains:</p>
<ul>
<li><a class="mw-redirect" href="/wiki/Boosting_(meta-algorithm)" title="Boosting (meta-algorithm)">Boosting</a></li>
<li><a href="/wiki/Decision_tree_learning" title="Decision tree learning">Decision tree learning</a></li>
<li><a href="/wiki/Gradient_boosting" title="Gradient boosting">Gradient boosting</a> trees</li>
<li><a class="mw-redirect" href="/wiki/Expectation-maximization_algorithm" title="Expectation-maximization algorithm">Expectation-maximization algorithm</a></li>
<li><a class="mw-redirect" href="/wiki/K-nearest_neighbor_algorithm" title="K-nearest neighbor algorithm">k-nearest neighbor algorithm</a></li>
<li><a href="/wiki/Naive_Bayes_classifier" title="Naive Bayes classifier">Naive Bayes classifier</a></li>
<li><a href="/wiki/Artificial_neural_network" title="Artificial neural network">Artificial neural networks</a></li>
<li><a href="/wiki/Random_forest" title="Random forest">Random forest</a></li>
<li><a href="/wiki/Support_vector_machine" title="Support vector machine">Support vector machine</a> (SVM)</li>
</ul>
<h2><span class="mw-headline" id="Programming_language">Programming language</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenCV&amp;action=edit&amp;section=3" title="Edit section: Programming language">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>OpenCV is written in <a href="/wiki/C%2B%2B" title="C++">C++</a> and its primary interface is in C++, but it still retains a less comprehensive though extensive older <a href="/wiki/C_(programming_language)" title="C (programming language)">C interface</a>. There are bindings in <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> and <a href="/wiki/MATLAB" title="MATLAB">MATLAB</a>/<a href="/wiki/GNU_Octave" title="GNU Octave">OCTAVE</a>. The API for these interfaces can be found in the online documentation.<sup class="reference" id="cite_ref-Cdocs_7-0"><a href="#cite_note-Cdocs-7">[7]</a></sup> Wrappers in other languages such as <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a>, <a href="/wiki/Perl" title="Perl">Perl</a>,<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> <a href="/wiki/Ch_(computer_programming)" title="Ch (computer programming)">Ch</a>,<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> <a href="/wiki/Haskell_(programming_language)" title="Haskell (programming language)">Haskell</a><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> and <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a> have been developed to encourage adoption by a wider audience.</p>
<p>All of the new developments and algorithms in OpenCV are now developed in the C++ interface.</p>
<h2><span class="mw-headline" id="Hardware_acceleration">Hardware acceleration</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenCV&amp;action=edit&amp;section=4" title="Edit section: Hardware acceleration">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>If the library finds Intel's <a href="/wiki/Integrated_Performance_Primitives" title="Integrated Performance Primitives">Integrated Performance Primitives</a> on the system, it will use these proprietary optimized routines to accelerate itself.</p>
<p>A <a href="/wiki/CUDA" title="CUDA">CUDA</a>-based <a href="/wiki/Graphics_processing_unit" title="Graphics processing unit">GPU</a> interface has been in progress since September 2010.<sup class="reference" id="cite_ref-OpenCVGPU_11-0"><a href="#cite_note-OpenCVGPU-11">[11]</a></sup></p>
<p>An <a href="/wiki/OpenCL" title="OpenCL">OpenCL</a>-based <a href="/wiki/Graphics_processing_unit" title="Graphics processing unit">GPU</a> interface has been in progress since October 2012,<sup class="reference" id="cite_ref-OpenCVOCL_12-0"><a href="#cite_note-OpenCVOCL-12">[12]</a></sup> documentation for version 2.4.9.0 can be found at docs.opencv.org.<sup class="reference" id="cite_ref-OpenCVOCL2.4.5_13-0"><a href="#cite_note-OpenCVOCL2.4.5-13">[13]</a></sup></p>
<h2><span class="mw-headline" id="OS_support">OS support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenCV&amp;action=edit&amp;section=5" title="Edit section: OS support">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>OpenCV runs on a variety of platforms. Desktop: <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a>, <a href="/wiki/Linux" title="Linux">Linux</a>, <a href="/wiki/MacOS" title="MacOS">macOS</a>, <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a>, <a href="/wiki/NetBSD" title="NetBSD">NetBSD</a>, <a href="/wiki/OpenBSD" title="OpenBSD">OpenBSD</a>; Mobile: <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android</a>, <a href="/wiki/IOS" title="IOS">iOS</a>, <a href="/wiki/Maemo" title="Maemo">Maemo</a>,<sup class="reference" id="cite_ref-Maemo_Port_14-0"><a href="#cite_note-Maemo_Port-14">[14]</a></sup> <a href="/wiki/BlackBerry_10" title="BlackBerry 10">BlackBerry 10</a>.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> The user can get official releases from <a href="/wiki/SourceForge" title="SourceForge">SourceForge</a> or take the latest sources from <a href="/wiki/GitHub" title="GitHub">GitHub</a>.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> OpenCV uses <a href="/wiki/CMake" title="CMake">CMake</a>.</p>

<ul>
<li><a href="/wiki/AForge.NET" title="AForge.NET">AForge.NET</a>, a computer vision library for the <a href="/wiki/Common_Language_Runtime" title="Common Language Runtime">Common Language Runtime</a> (<a href="/wiki/.NET_Framework" title=".NET Framework">.NET Framework</a> and <a href="/wiki/Mono_(software)" title="Mono (software)">Mono</a>).</li>
