<p><b>Apple Wallet</b> (referred to as simply <b>Wallet</b>) is an <a href="/wiki/Application_software" title="Application software">application</a> in <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple</a>'s <a href="/wiki/IOS" title="IOS">iOS</a> (previously known as <b>Passbook</b> in <a href="/wiki/IOS_6" title="IOS 6">iOS 6</a> to <a href="/wiki/IOS_8" title="IOS 8">iOS 8</a>) that allows users to store <a href="/wiki/Coupon" title="Coupon">coupons</a>, <a href="/wiki/Boarding_pass" title="Boarding pass">boarding passes</a>, event <a href="/wiki/Ticket_(admission)" title="Ticket (admission)">tickets</a>, store cards and, starting with iOS 8.1, <a href="/wiki/Credit_card" title="Credit card">credit cards</a>, loyalty cards, and <a href="/wiki/Debit_card" title="Debit card">debit cards</a> via <a href="/wiki/Apple_Pay" title="Apple Pay">Apple Pay</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> The technology is designed by <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple Inc.</a> and was presented at the 2012 <a href="/wiki/Apple_Worldwide_Developers_Conference" title="Apple Worldwide Developers Conference">Apple Worldwide Developers Conference</a> (WWDC) on June 11, 2012 under the name Passbook (now <b>Wallet</b>, as of iOS 9).<sup class="reference" id="cite_ref-enwwdc_2-0"><a href="#cite_note-enwwdc-2">[2]</a></sup> The application was released as a built-in app for iPhones and iPod Touch devices alongside <a href="/wiki/IOS_6" title="IOS 6">iOS 6</a> on September 19, 2012.</p>

<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Wallet_(application)&amp;action=edit&amp;section=1" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The Wallet application displays <a href="/wiki/Aztec_Code" title="Aztec Code">Aztec</a>, <a href="/wiki/PDF417" title="PDF417">PDF417</a> and <a href="/wiki/QR_code" title="QR code">QR</a> 2D barcodes and <a href="/wiki/Code_128" title="Code 128">Code 128</a> 1D barcodes beginning with iOS 9.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> Each digital coupon or ticket is known as a "pass".<sup class="reference" id="cite_ref-tnoozpassbookimpact_4-0"><a href="#cite_note-tnoozpassbookimpact-4">[4]</a></sup> When the user launches Wallet for the first time, a brief introduction screen will appear with a button inviting users to browse applications on the <a href="/wiki/App_Store_(iOS)" title="App Store (iOS)">App Store</a> with Wallet integration. Passes can also be distributed online via <a href="/wiki/Safari_(web_browser)" title="Safari (web browser)">Safari</a>, sent to the user via email, or scanned using the built-in scanner in the Wallet app.</p>
<p>Passes are synced between iOS devices using iCloud, and <a href="/wiki/OS_X_Mountain_Lion#Release_history" title="OS X Mountain Lion">OS X 10.8.2</a> and higher also support opening passes to be sent to users' iOS devices. Although the application is available in <a href="/wiki/IOS_6" title="IOS 6">iOS 6</a> or later, it is currently only available on <a href="/wiki/IPhone" title="IPhone">iPhone</a> and <a href="/wiki/IPod_Touch" title="IPod Touch">iPod Touch</a>, but not on <a href="/wiki/IPad" title="IPad">iPad</a>.<sup class="reference" id="cite_ref-iOS_What.27s_New:_Passbook_5-0"><a href="#cite_note-iOS_What.27s_New:_Passbook-5">[5]</a></sup></p>
<p>Wallet has the following features:</p>
<ul>
<li>Displays 2D barcodes of following types: <a href="/wiki/Aztec_Code" title="Aztec Code">Aztec</a>, <a href="/wiki/PDF417" title="PDF417">PDF417</a> and <a href="/wiki/QR_code" title="QR code">QR</a>.</li>
<li>Displays 1D barcodes of following types: <a href="/wiki/Code_128" title="Code 128">Code 128</a> beginning with iOS 9.</li>
<li>Triggered by location. Up to 10 locations can be added to each Pass. A location is programmed as GPS coordinates (longitude, latitude, and altitude) and/or <a href="/wiki/IBeacon" title="IBeacon">iBeacon</a> UUID (The UUID is a Universally Unique Identifier which is a 32 ASCII character code or a code automatically generated from a name using the PassKit API).</li>
<li>Triggered by time of pass.</li>
<li>Localisation of the pass. Up to 35 languages can be stored per each pass in Wallet.<sup class="reference" id="cite_ref-PassLocales_6-0"><a href="#cite_note-PassLocales-6">[6]</a></sup></li>
<li>Pass changes can be pushed via the <a href="/wiki/Apple_Push_Notification_Service" title="Apple Push Notification Service">Apple Push Notification Service</a> by the pass provider, or manually updated by the user themselves.</li>
</ul>
<p>As Wallet is only officially provided by Apple for iOS, several third party developers have created unsupported applications for other operating systems, such as Pass2U Wallet or PassWallet for Android &amp; BlackBerry, that support importing and viewing Wallet passes. Windows 8.1 Phone also<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> supports Apple's pass format as well, although dynamic updates are not supported. Some issuers<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> of passes also support viewing the passes through any web browser.</p>
<h2><span class="mw-headline" id="Wallet_Ecosystem">Wallet Ecosystem</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Wallet_(application)&amp;action=edit&amp;section=2" title="Edit section: Wallet Ecosystem">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Passes exist in a larger ecosystem, because passes are created as a package. The package is a pass template, that is created with a pass signer, along with relevant data and a private key. Passes can be updated at any time using the PassKit API and an iOS app can interact directly with passes stored in Wallet.</p>
<p>Passes are presented and managed by the Wallet app. Systems and apps interact with passes via the <a class="external text" href="https://developer.apple.com/library/prerelease/ios/documentation/UserExperience/Reference/PassKit_Framework/index.html" rel="nofollow">PassKit</a> <a class="mw-redirect" href="/wiki/API" title="API">API</a>.</p>
<p>In its simplest form, an interaction (or transaction) between a Pass and a system is facilitated by a 2D Barcode or the modern QR code although it requires the customer to initiate the activity.</p>
<p>In late 2014 the first known implementations utilizing the iBeacon wireless geofencing started to appear in retail locations in the US. The iBeacon solutions allowed the retailer to broadcast an unsolicited lock-screen notice onto smartphones within <a href="/wiki/Wi-Fi" title="Wi-Fi">Wi-Fi</a> range, which is about one thousand feet.</p>
<h2><span class="mw-headline" id="Wallet_Pass_Distribution">Wallet Pass Distribution</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Wallet_(application)&amp;action=edit&amp;section=3" title="Edit section: Wallet Pass Distribution">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Passes can be distributed via email, SMS, MMS, the web, social media, an app, and as QR codes <sup class="reference" id="cite_ref-Passbook_Pass_Distribution_9-0"><a href="#cite_note-Passbook_Pass_Distribution-9">[9]</a></sup></p>
