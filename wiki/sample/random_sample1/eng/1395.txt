<h1 class="firstHeading" id="firstHeading" lang="en">QR code</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>

<p><b>QR code</b> (abbreviated from <b>Quick Response Code</b>) is the trademark for a type of <a class="mw-redirect" href="/wiki/Matrix_barcode" title="Matrix barcode">matrix barcode</a> (or two-dimensional <a href="/wiki/Barcode" title="Barcode">barcode</a>) first designed for the <a href="/wiki/Automotive_industry_in_Japan" title="Automotive industry in Japan">automotive industry in Japan</a>. A barcode is a machine-readable optical label that contains information about the item to which it is attached. A QR code uses four standardized encoding modes (numeric, alphanumeric, byte/binary, and <a href="/wiki/Kanji" title="Kanji">kanji</a>) to efficiently store data; extensions may also be used.<sup class="reference" id="cite_ref-QRCodefeatures_1-0"><a href="#cite_note-QRCodefeatures-1">[1]</a></sup></p>
<p>The QR code system became popular outside the automotive industry due to its fast readability and greater storage capacity compared to standard <a href="/wiki/Universal_Product_Code" title="Universal Product Code">UPC barcodes</a>. Applications include product tracking, item identification, time tracking, document management, and general marketing.<sup class="reference" id="cite_ref-autogenerated1_2-0"><a href="#cite_note-autogenerated1-2">[2]</a></sup></p>
<p>A QR code consists of black squares arranged in a square grid on a white background, which can be read by an imaging device such as a camera, and processed using <a href="/wiki/Reed%E2%80%93Solomon_error_correction" title="Reed–Solomon error correction">Reed–Solomon error correction</a> until the image can be appropriately interpreted. The required data are then extracted from patterns that are present in both horizontal and vertical components of the image.<sup class="reference" id="cite_ref-autogenerated1_2-1"><a href="#cite_note-autogenerated1-2">[2]</a></sup></p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The QR code system was invented in 1994 by <a href="/wiki/Denso#Denso_Wave" title="Denso">Denso Wave</a>. Its purpose was to track vehicles during manufacturing; it was designed to allow high-speed component scanning.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> QR codes now are used in a much broader context, including both commercial tracking applications and convenience-oriented applications aimed at mobile-phone users (termed mobile tagging). QR codes may be used to display text to the user, to add a <a href="/wiki/VCard" title="VCard">vCard</a> contact to the user's device, to open a <a href="/wiki/Uniform_Resource_Identifier" title="Uniform Resource Identifier">Uniform Resource Identifier</a> (URI), or to compose an e-mail or text message. Users can generate and print their own QR codes for others to scan and use by visiting one of several paid and free QR code generating sites or apps. The technology has since become one of the most-used types of two-dimensional barcode.<sup class="reference" id="cite_ref-About2DCode_4-0"><a href="#cite_note-About2DCode-4">[4]</a></sup></p>
<h2><span class="mw-headline" id="Standards">Standards</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=2" title="Edit section: Standards">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:402px;"><a class="image" href="/wiki/File:QR_Code_Structure_Example_3.svg"><img alt="" class="thumbimage" data-file-height="736" data-file-width="1312" height="224" src="//upload.wikimedia.org/wikipedia/commons/thumb/1/1d/QR_Code_Structure_Example_3.svg/400px-QR_Code_Structure_Example_3.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/1/1d/QR_Code_Structure_Example_3.svg/600px-QR_Code_Structure_Example_3.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/1/1d/QR_Code_Structure_Example_3.svg/800px-QR_Code_Structure_Example_3.svg.png 2x" width="400"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:QR_Code_Structure_Example_3.svg" title="Enlarge"></a></div>
Structure of a QR code, highlighting functional elements</div>
</div>

<p>There are several standards that cover the encoding of data as QR codes:<sup class="reference" id="cite_ref-qrstandard1_5-0"><a href="#cite_note-qrstandard1-5">[5]</a></sup></p>
<ul>
<li>October 1997 – AIM (Association for Automatic Identification and Mobility) International<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></li>
<li>January 1999 – <a href="/wiki/Japanese_Industrial_Standards" title="Japanese Industrial Standards">JIS</a> X 0510</li>
<li>June 2000 – <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a>/IEC 18004:2000 <i><a class="external text" href="http://www.iso.org/iso/iso_catalogue/catalogue_ics/catalogue_detail_ics.htm?csnumber=30789" rel="nofollow">Information technology – Automatic identification and data capture techniques – Bar code symbology – QR code</a></i> (now withdrawn)<br/>
Defines QR code models 1 and 2 symbols.</li>
<li>1 September 2006 – ISO/IEC 18004:2006 <i><a class="external text" href="http://www.iso.org/iso/iso_catalogue/catalogue_tc/catalogue_detail.htm?csnumber=43655" rel="nofollow">Information technology – Automatic identification and data capture techniques – QR code 2005 bar code symbology specification</a></i><br/>
Defines QR code 2005 symbols, an extension of QR code model 2. Does not specify how to read QR code model 1 symbols, or require this for compliance.</li>
<li>1 February 2015 – ISO/IEC 18004:2015 <i><a class="external text" href="http://www.iso.org/iso/catalogue_detail.htm?csnumber=62021" rel="nofollow">Information – Automatic identification and data capture techniques – QR Code barcode symbology specification</a></i><br/>
Renames the QR Code 2005 symbol to QR Code and adds clarification to some procedures and minor corrections.</li>
</ul>
<p>At the application layer, there is some variation between most of the implementations. Japan's <a href="/wiki/NTT_DoCoMo" title="NTT DoCoMo">NTT DoCoMo</a> has established <a href="/wiki/De_facto" title="De facto">de facto</a> standards for the encoding of URLs, contact information, and several other data types.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> The open-source "ZXing" project maintains a list of QR code data types.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<h2><span class="mw-headline" id="Uses">Uses</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=3" title="Edit section: Uses">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Japan-qr-code-billboard.jpg"><img alt="" class="thumbimage" data-file-height="3072" data-file-width="2304" height="293" src="//upload.wikimedia.org/wikipedia/commons/thumb/3/32/Japan-qr-code-billboard.jpg/220px-Japan-qr-code-billboard.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/3/32/Japan-qr-code-billboard.jpg/330px-Japan-qr-code-billboard.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/3/32/Japan-qr-code-billboard.jpg/440px-Japan-qr-code-billboard.jpg 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Japan-qr-code-billboard.jpg" title="Enlarge"></a></div>
A QR code used on a large billboard in Japan, linking to the sagasou.mobi website</div>
</div>

<p>QR codes have become common in consumer advertising. Typically, a <a href="/wiki/Smartphone" title="Smartphone">smartphone</a> is used as a QR code scanner, displaying the code and converting it to some useful form (such as a standard <a class="mw-redirect" href="/wiki/URL" title="URL">URL</a> for a website, thereby obviating the need for a user to type it into a <a href="/wiki/Web_browser" title="Web browser">web browser</a>). QR code has become a focus of <a href="/wiki/Advertising" title="Advertising">advertising</a> strategy, since it provides a way to access a brand's website more quickly than by manually entering a URL.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> Beyond mere convenience to the consumer, the importance of this capability is that it increases the <a class="mw-redirect" href="/wiki/Conversion_rate" title="Conversion rate">conversion rate</a> (the chance that contact with the advertisement will convert to a sale), by coaxing interested prospects further down the <a href="/wiki/Conversion_funnel" title="Conversion funnel">conversion funnel</a> with little delay or effort, bringing the viewer to the advertiser's website immediately, where a longer and more targeted sales pitch may lose the viewer's interest.</p>
<p>Although initially used to track parts in vehicle manufacturing, QR codes are used over a much wider range of applications, including commercial tracking, entertainment and transport ticketing, product and loyalty marketing (examples: mobile couponing where a company's discounted and percent discount can be captured using a QR code decoder which is a mobile app, or storing a company's information such as address and related information alongside its alpha-numeric text data as can be seen in Yellow Pages directory), and in-store product labeling. It can also be used in storing personal information for use by organizations. An example of this is Philippines National Bureau of Investigation (NBI) where NBI clearances now come with a QR code. Many of these applications target <a href="/wiki/Mobile_phone" title="Mobile phone">mobile-phone</a> users (via <a href="/wiki/Mobile_tagging" title="Mobile tagging">mobile tagging</a>). Users may receive text, add a <a href="/wiki/VCard" title="VCard">vCard</a> contact to their device, open a URI, or compose an <a href="/wiki/Email#Message_format" title="Email">e-mail</a> or text message after scanning QR codes. They can generate and print their own QR codes for others to scan and use by visiting one of several pay or free QR code-generating sites or apps. <a href="/wiki/Google" title="Google">Google</a> had an <a class="mw-redirect" href="/wiki/API" title="API">API</a>, now deprecated, to generate QR codes,<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> and apps for scanning QR codes can be found on nearly all smartphone devices.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<p>QR codes storing addresses and URLs may appear in magazines, on signs, on buses, on business cards, or on almost any object about which users might want information. Users with a <a href="/wiki/Camera_phone" title="Camera phone">camera phone</a> equipped with the correct reader application can scan the image of the QR code to display text, contact information, connect to a <a href="/wiki/Wireless_LAN" title="Wireless LAN">wireless network</a>, or open a web page in the telephone's browser. This act of linking from physical world objects is termed <a class="mw-redirect" href="/wiki/Hardlink_(hyperlink)" title="Hardlink (hyperlink)">hardlinking</a> or <a href="/wiki/Object_hyperlinking" title="Object hyperlinking">object hyperlinking</a>. QR codes also may be linked to a location to track where a code has been scanned. Either the application that scans the QR code retrieves the geo information by using GPS and cell tower triangulation (aGPS) or the URL encoded in the QR code itself is associated with a location.<sup class="reference" id="cite_ref-GeotaggedQRCodes_13-0"><a href="#cite_note-GeotaggedQRCodes-13">[13]</a></sup></p>

<p>Recruiters have started placing QR codes in job advertisements,<sup class="reference" id="cite_ref-articles.timesofindia.indiatimes.com_15-0"><a href="#cite_note-articles.timesofindia.indiatimes.com-15">[15]</a></sup> while applicants have started sporting it in their CVs and visiting cards.<sup class="reference" id="cite_ref-articles.timesofindia.indiatimes.com_15-1"><a href="#cite_note-articles.timesofindia.indiatimes.com-15">[15]</a></sup></p>
<p>In June 2011 The <a href="/wiki/Royal_Dutch_Mint" title="Royal Dutch Mint">Royal Dutch Mint</a> (<i>Koninklijke Nederlandse Munt</i>) issued the world's first official coin with a QR code to celebrate the centenary of its current building and premises. The coin can be scanned by a <a href="/wiki/Smartphone" title="Smartphone">smartphone</a> and link to a special website with contents about the historical event and design of the coin.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> In 2014 the Central Bank of Nigeria issued a 100-naira banknote to commemorate its centennial, the first banknote to incorporate a QR code in its design. When scanned with an internet-enabled mobile device, the code goes to a website which tells the centenary story of Nigeria.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> In 2015, the <a href="/wiki/Central_Bank_of_Russia" title="Central Bank of Russia">Central Bank of the Russian Federation</a> issued a 100-ruble note to commemorate the annexation of Crimea by the Russian Federation. It contains a QR code into its design, and when scanned with an internet-enabled mobile device, the code goes to a website that details the historical and technical background of the commemorative note. In 2008, a Japanese stonemason announced plans to engrave QR codes on gravestones, allowing visitors to view information about the deceased, and family members to keep track of visits.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup></p>
<p>Psychologist <a href="/wiki/Richard_Wiseman" title="Richard Wiseman">Richard Wiseman</a> was one of the first authors to include QR codes in a book, in <i>Paranormality: Why We See What Isn't There</i> (2011), allowing his readers to follow up on <a href="/wiki/Paranormal" title="Paranormal">paranormal</a> claims by accessing his research through the codes.<sup class="reference" id="cite_ref-Skepicality160_19-0"><a href="#cite_note-Skepicality160-19">[19]</a></sup></p>
<h3><span class="mw-headline" id="Mobile_operating_systems">Mobile operating systems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=4" title="Edit section: Mobile operating systems">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>QR codes can be used on various mobile device operating systems. These devices support <a href="/wiki/URL_redirection" title="URL redirection">URL redirection</a>, which allows QR codes to send <a href="/wiki/Metadata" title="Metadata">metadata</a> to existing applications on the device. Many paid or free apps are available with the ability to scan the codes and hard-link to an external URL.</p>
<h3><span class="mw-headline" id="URLs">URLs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=5" title="Edit section: URLs">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>URLs aided <a class="mw-redirect" href="/wiki/Conversion_rate" title="Conversion rate">marketing conversion rates</a> even in the pre-smartphone era, but during those years faced several limitations: ad viewers usually had to type the URL and often did not have a web browser in front of them when they first viewed the ad. The chances were high that they would forget to visit the site later, not bother to type a URL, or forget what URL to type. <a href="/wiki/Semantic_URL" title="Semantic URL">Semantic URLs</a> decreased these risks but did not eliminate them. Some of these disadvantages to URL conversion rates are fading away now that smartphones are putting web access and voice recognition in constant reach, with QR code providing the URL for instant access.</p>
<h3><span class="mw-headline" id="Virtual_stores">Virtual stores</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=6" title="Edit section: Virtual stores">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>During the month of June 2011, according to one study, 14 million mobile users scanned a QR code or a barcode. Some 58% of those users scanned a QR or barcode from their homes, while 39% scanned from retail stores; 53% of the 14 million users were men between the ages of 18 and 34.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> The use of QR codes for "virtual store" formats started in South Korea,<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> and Argentina,<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup> but is currently expanding globally.<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup> Walmart, Procter &amp; Gamble and Woolworths have already adopted the Virtual Store concept.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup></p>
<h3><span class="mw-headline" id="Code_payments">Code payments</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=7" title="Edit section: Code payments">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>QR codes can be used to store bank account information or credit card information, or they can be specifically designed to work with particular payment provider applications. There are several trial applications of QR code payments across the world.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup><sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup></p>
<p>In November 2012, QR code payments were deployed on a larger scale in the <a href="/wiki/Czech_Republic" title="Czech Republic">Czech Republic</a> when an open format for payment information exchange - a <a href="/wiki/Short_Payment_Descriptor" title="Short Payment Descriptor">Short Payment Descriptor</a> - was introduced and endorsed by the Czech Banking Association as the official local solution for QR payments.<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup></p>
<p>QR codes are commonly used in the field of cryptographic currencies, particularly those based on and including <a href="/wiki/Bitcoin" title="Bitcoin">Bitcoin</a>.<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup> Payment addresses, cryptographic keys and transaction information are often shared between digital wallets in this way.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup></p>
<h3><span class="mw-headline" id="Website_login">Website login</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=8" title="Edit section: Website login">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>QR codes can be used to log in into websites: a QR code is shown on the login page on a computer screen, and when a registered user scans it with a verified smartphone, they will automatically be logged in. Authentication is performed by the smartphone which contacts the server. Google tested such a login method in January 2012.<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup></p>
<h3><span class="mw-headline" id="Funerary_use">Funerary use</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=9" title="Edit section: Funerary use">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In 2008, Ishinokoe in Yamanashi Prefecture, Japan began to sell tombstones with QR codes produced by IT DeSign, where the code leads to a virtual grave site of the deceased.<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup><sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup><sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup> Other companies, such as Wisconsin based Interactive Headstones, have begun implementing QR codes into tombstones.<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup> In 2014 the <a class="mw-redirect" href="/wiki/Jewish_Cemetery_of_La_Paz" title="Jewish Cemetery of La Paz">Jewish Cemetery of La Paz</a> in Uruguay, began implementing QR codes for tombstones.<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup></p>

<p>Encrypted QR codes, which are not very common, have a few implementations. <h2><span class="mw-headline" id="Design">Design</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=11" title="Edit section: Design">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Unlike the older, one-dimensional barcodes that were designed to be mechanically scanned by a narrow beam of light, a QR code is detected by a 2-dimensional digital <a href="/wiki/Image_sensor" title="Image sensor">image sensor</a> and then digitally analyzed by a programmed processor. The processor locates the three distinctive squares at the corners of the QR code image, using a smaller square (or multiple squares) near the fourth corner to normalize the image for size, orientation, and angle of viewing. The small dots throughout the QR code are then converted to binary numbers and validated with an error-correcting algorithm.</p>

<h3><span class="mw-headline" id="Error_correction">Error correction</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=13" title="Edit section: Error correction">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="thumb tright">
<div class="thumbinner" style="width:143px;"><a class="image" href="/wiki/File:QR_Code_Damaged.jpg"><img alt="" class="thumbimage" data-file-height="1680" data-file-width="1680" height="141" src="//upload.wikimedia.org/wikipedia/commons/thumb/8/8d/QR_Code_Damaged.jpg/141px-QR_Code_Damaged.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/8/8d/QR_Code_Damaged.jpg/212px-QR_Code_Damaged.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/8/8d/QR_Code_Damaged.jpg/282px-QR_Code_Damaged.jpg 2x" width="141"/></a>

<p>Codewords are <a href="/wiki/Octet_(computing)" title="Octet (computing)">8 bits</a> long and use the <a href="/wiki/Reed%E2%80%93Solomon_error_correction" title="Reed–Solomon error correction">Reed–Solomon error correction</a> algorithm with four error correction levels. The higher the error correction level, the less storage capacity. The following table lists the approximate error correction capability at each of the four levels:</p>
<table>
<tr>
<td>Level L (Low)</td>
<td>7% of codewords can be restored.</td>
</tr>
<tr>
<td>Level M (Medium)</td>
<td>15% of codewords can be restored.</td>
</tr>
<tr>
<td>Level Q (Quartile)<sup class="reference" id="cite_ref-41"><a href="#cite_note-41">[41]</a></sup></td>
<td>25% of codewords can be restored.</td>
</tr>
<tr>
<td>Level H (High)</td>
<td>30% of codewords can be restored.</td>
</tr>
</table>
<p>In larger QR symbols, the message is broken up into several Reed–Solomon code blocks. The block size is chosen so that at most 15 errors can be corrected in each block; this limits the complexity of the decoding algorithm. The code blocks are then interleaved together, making it less likely that localized damage to a QR symbol will overwhelm the capacity of any single block.</p>
<p>Due to error correction, it is possible to create artistic QR codes that still scan correctly, but contain intentional errors to make them more readable or attractive to the human eye, as well as to incorporate colors, logos, and other features into the QR code block.<sup class="reference" id="cite_ref-42"><a href="#cite_note-42">[42]</a></sup><sup class="reference" id="cite_ref-43"><a href="#cite_note-43">[43]</a></sup></p>
<p>It is also possible to design artistic QR codes without reducing the error correction capacity by manipulating the underlying mathematical constructs.<sup class="reference" id="cite_ref-44"><a href="#cite_note-44">[44]</a></sup><sup class="reference" id="cite_ref-45"><a href="#cite_note-45">[45]</a></sup></p>
<h3><span class="mw-headline" id="Encoding">Encoding</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=14" title="Edit section: Encoding">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The format information records two things: the error correction level and the mask pattern used for the symbol. Masking is used to break up patterns in the data area that might confuse a scanner, such as large blank areas or misleading features that look like the locator marks. The mask patterns are defined on a grid that is repeated as necessary to cover the whole symbol. Modules corresponding to the dark areas of the mask are inverted. The format information is protected from errors with a <a href="/wiki/BCH_code" title="BCH code">BCH code</a>, and two complete copies are included in each QR symbol.<sup class="reference" id="cite_ref-autogenerated1_2-2"><a href="#cite_note-autogenerated1-2">[2]</a></sup></p>

<h2><span class="mw-headline" id="Variants">Variants</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=16" title="Edit section: Variants">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><i>Micro QR code</i> is a smaller version of the QR code standard for applications where symbol size is limited. There are four different versions (sizes) of Micro QR codes: the smallest is 11×11 modules; the largest can hold 35 numeric characters.<sup class="reference" id="cite_ref-47"><a href="#cite_note-47">[47]</a></sup></p>
<p><i>IQR code</i> is an alternative to existing QR codes developed by Denso Wave. IQR codes can be created in square or rectangular formations; this is intended for situations where a rectangular barcode would otherwise be more appropriate, such as cylindrical objects. IQR codes can fit the same amount of information in 30% less space. There are 61 versions of square IQR codes, and 15 versions of rectangular codes. For squares, the minimum size is 9x9 modules; rectangles have a minimum of 19x5 modules. IQR codes add error correction level S, which allows for 50% error correction.<sup class="reference" id="cite_ref-48"><a href="#cite_note-48">[48]</a></sup> IQR Codes have not yet been given an ISO specification, and only proprietary Denso Wave products can create or read IQR codes.<sup class="reference" id="cite_ref-49"><a href="#cite_note-49">[49]</a></sup></p>
<p><i>Model 1 QR code</i> is an older version of the specification. It is visually similar to the widely seen model 2 codes, but lacks alignment patterns.</p>

<h2><span class="mw-headline" id="License">License</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=17" title="Edit section: License">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The use of QR Code technology is freely licensed as long as users follow the standards for QR Code documented with <a href="/wiki/Japanese_Industrial_Standards" title="Japanese Industrial Standards">JIS</a> or <a class="mw-redirect" href="/wiki/ISO" title="ISO">ISO</a>. Non-standardized codes may require special licensing.<sup class="reference" id="cite_ref-qrpatent01_50-0"><a href="#cite_note-qrpatent01-50">[50]</a></sup></p>
<p><a href="/wiki/Denso#Denso_Wave" title="Denso">Denso Wave</a> owns a number of <a href="/wiki/Patent" title="Patent">patents</a> on QR Code technology, but has chosen to exercise them in a limited fashion.<sup class="reference" id="cite_ref-qrpatent01_50-1"><a href="#cite_note-qrpatent01-50">[50]</a></sup> In order to promote widespread usage of the technology Denso Wave chose to waive its rights to a key patent in its possession for <i>standardized</i> codes only.<sup class="reference" id="cite_ref-qrstandard1_5-1"><a href="#cite_note-qrstandard1-5">[5]</a></sup> In the USA, the granted QR Code patent is <span class="citation patent"><a class="external text" href="https://worldwide.espacenet.com/textdoc?DB=EPODOC&amp;IDX=US5726435" rel="nofollow">US 5726435</a></span><span class="Z3988" title="ctx_ver=Z39.88-2004&amp;rft_val_fmt=info%3Aofi%2Ffmt%3Akev%3Amtx%3Apatent&amp;rft.number=5726435&amp;rft.cc=US&amp;rft.title="><span style="display: none;"> </span></span>, and in Japan <span class="citation patent"><a class="external text" href="https://worldwide.espacenet.com/textdoc?DB=EPODOC&amp;IDX=JP2938338" rel="nofollow">JP 2938338</a></span><span class="Z3988" title="ctx_ver=Z39.88-2004&amp;rft_val_fmt=info%3Aofi%2Ffmt%3Akev%3Amtx%3Apatent&amp;rft.number=2938338&amp;rft.cc=JP&amp;rft.title="><span style="display: none;"> </span></span>. <h2><span class="mw-headline" id="Risks">Risks</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=18" title="Edit section: Risks">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The only context in which common QR codes can carry executable data is the <a class="mw-redirect" href="/wiki/URL" title="URL">URL</a> data type. These URLs may host JavaScript code, which can be used to exploit vulnerabilities in applications on the host system, such as the reader, the web browser or the image viewer, since a reader will typically send the data to the application associated with the data type used by the QR code.</p>
<p>In the case of no software exploits, malicious QR codes combined with a permissive reader can still put a computer's contents and user's privacy at risk. This practice is known as "attagging", a <a href="/wiki/Portmanteau" title="Portmanteau">portmanteau</a> of "attack tagging".<sup class="reference" id="cite_ref-Wired_56-0"><a href="#cite_note-Wired-56">[56]</a></sup> They are easily created and can be affixed over legitimate QR codes.<sup class="reference" id="cite_ref-57"><a href="#cite_note-57">[57]</a></sup> On a <a href="/wiki/Smartphone" title="Smartphone">smartphone</a>, the reader's permissions may allow use of the camera, full Internet access, read/write contact data, <a class="mw-redirect" href="/wiki/GPS" title="GPS">GPS</a>, read <a href="/wiki/Web_browser" title="Web browser">browser</a> history, read/write local storage, and global system changes.<sup class="reference" id="cite_ref-58"><a href="#cite_note-58">[58]</a></sup><sup class="reference" id="cite_ref-59"><a href="#cite_note-59">[59]</a></sup><sup class="reference" id="cite_ref-60"><a href="#cite_note-60">[60]</a></sup></p>
<p>Risks include linking to dangerous web sites with browser exploits, enabling the microphone/camera/GPS, and then streaming those feeds to a remote server, analysis of sensitive data (passwords, files, contacts, transactions),<sup class="reference" id="cite_ref-61"><a href="#cite_note-61">[61]</a></sup> and sending email/<a href="/wiki/Short_Message_Service" title="Short Message Service">SMS</a>/IM messages or <a class="mw-redirect" href="/wiki/DDOS" title="DDOS">DDOS</a> packets as part of a <a href="/wiki/Botnet" title="Botnet">botnet</a>, corrupting privacy settings, stealing identity,<sup class="reference" id="cite_ref-62"><a href="#cite_note-62">[62]</a></sup> and even containing malicious logic themselves such as <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a><sup class="reference" id="cite_ref-63"><a href="#cite_note-63">[63]</a></sup> or a virus.<sup class="reference" id="cite_ref-64"><a href="#cite_note-64">[64]</a></sup><sup class="reference" id="cite_ref-65"><a href="#cite_note-65">[65]</a></sup> These actions could occur in the background while the user is only seeing the reader opening a seemingly harmless web page.<sup class="reference" id="cite_ref-66"><a href="#cite_note-66">[66]</a></sup> In Russia, a malicious QR code caused phones that scanned it to send premium texts at a fee of US$6 each.<sup class="reference" id="cite_ref-Wired_56-1"><a href="#cite_note-Wired-56">[56]</a></sup></p>
<h2><span class="mw-headline" id="Extension">Extension</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=QR_code&amp;action=edit&amp;section=19" title="Edit section: Extension">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Hcc2d.png"><img alt="" class="thumbimage" data-file-height="592" data-file-width="1367" height="95" src="//upload.wikimedia.org/wikipedia/commons/thumb/b/b0/Hcc2d.png/220px-Hcc2d.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/b/b0/Hcc2d.png/330px-Hcc2d.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/b/b0/Hcc2d.png/440px-Hcc2d.png 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Hcc2d.png" title="Enlarge"></a></div>
Samples of the High Capacity Colored 2-Dimensional (HCC2D) code: (a) 4-color HCC2D code and (b) 8-color HCC2D code.</div>
</div>
</div>
<p>Researchers have proposed a new High Capacity Colored 2-Dimensional (HCC2D) Code, which builds upon a QR code basis for preserving the QR robustness to distortions and use colors for increasing data density (at this stage it is still in prototyping phase). The HCC2D code specification is described in details in Querini <i>et al.</i> (2014),<sup class="reference" id="cite_ref-Querini_et_al._.282014.29_67-0"><a href="#cite_note-Querini_et_al._.282014.29-67">[67]</a></sup> while techniques for color classification of HCC2D code cells are described in detail in Querini and <a href="/wiki/Giuseppe_F._Italiano" title="Giuseppe F. Italiano">Italiano</a> (2014),<sup class="reference" id="cite_ref-Querini_and_Italiano_.282014.29_68-0"><a href="#cite_note-Querini_and_Italiano_.282014.29-68">[68]</a></sup> which is an extended version of Querini and Italiano (2013).<sup class="reference" id="cite_ref-Querini_and_Italiano_.282013.29_69-0"><a href="#cite_note-Querini_and_Italiano_.282013.29-69">[69]</a></sup></p>
<p>Introducing colors into QR codes requires addressing additional issues. In particular, during QR code reading only the brightness information is taken into account, while HCC2D codes have to cope with chromatic distortions during the decoding phase. In order to ensure adaptation to chromatic distortions which arise in each scanned code, HCC2D codes make use of an additional field: the Color Palette Pattern. This is because color cells of a Color Palette Pattern are supposed to be distorted in the same way as color cells of the Encoding Region. Replicated color palettes are used for training machine learning classifiers.</p>
