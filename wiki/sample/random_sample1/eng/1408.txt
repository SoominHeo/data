<h1 class="firstHeading" id="firstHeading" lang="en">RE (complexity)</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p>In <a href="/wiki/Computability_theory" title="Computability theory">computability theory</a> and <a href="/wiki/Computational_complexity_theory" title="Computational complexity theory">computational complexity theory</a>, <b>RE</b> (<a href="/wiki/Recursively_enumerable_set" title="Recursively enumerable set">recursively enumerable</a>) is the <a href="/wiki/Complexity_class" title="Complexity class">class</a> of <a href="/wiki/Decision_problem" title="Decision problem">decision problems</a> for which a 'yes' answer can be verified by a <a href="/wiki/Turing_machine" title="Turing machine">Turing machine</a> in a finite amount of time.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> Informally, it means that if the answer to a problem instance is 'yes', then there is some procedure which takes finite time to determine this, and this procedure never falsely reports 'yes' when the true answer is 'no'. However, when the true answer is 'no', the procedure is not required to halt; it may go into an "<a href="/wiki/Infinite_loop" title="Infinite loop">infinite loop</a>" for some 'no' cases. Such a procedure is sometimes called a <b>semi-algorithm</b>, to distinguish it from an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a>, defined as a complete solution to a decision problem.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>Equivalently, <b>RE</b> is the class of decision problems for which a Turing machine can list all the 'yes' instances, one by one (this is what 'enumerable' means). Each member of <b>RE</b> is a <a href="/wiki/Recursively_enumerable_set" title="Recursively enumerable set">recursively enumerable set</a> and therefore a <a href="/wiki/Diophantine_set" title="Diophantine set">Diophantine set</a>.</p>
<p>Similarly, <b>co-RE</b> is the set of all languages that are complements of a language in <b>RE</b>. In a sense, <b>co-RE</b> contains languages of which membership can be disproved in a finite amount of time, but proving membership might take forever.</p>

<h2><span class="mw-headline" id="Relations_to_other_classes">Relations to other classes</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=RE_(complexity)&amp;action=edit&amp;section=1" title="Edit section: Relations to other classes">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The set of <a href="/wiki/Recursive_language" title="Recursive language">recursive languages</a> (<b><a href="/wiki/R_(complexity)" title="R (complexity)">R</a></b>) is a subset of both <b>RE</b> and <b>co-RE</b>.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> In fact, it is the intersection of those two classes, because we can decide any problem for which there exists a recogniser and also a co-recogniser by simply interleaving them until one obtains a result. <h2><span class="mw-headline" id="RE-complete">RE-complete</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=RE_(complexity)&amp;action=edit&amp;section=2" title="Edit section: RE-complete">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><b>RE-complete</b> is the set of decision problems that are complete for <b>RE</b>. In a sense, these are the "hardest" recursively enumerable problems. All such problems are non<a href="/wiki/Recursive_language" title="Recursive language">recursive</a>. Generally, no constraint is placed on the reductions used except that they must be <a href="/wiki/Many-one_reduction" title="Many-one reduction">many-one reductions</a>.</p>
<p>Examples of RE-complete problems:</p>
<ol>
<li><a href="/wiki/Halting_problem" title="Halting problem">Halting problem</a>: Whether a program given a finite input finishes running or will run forever.</li>
<li>By <a class="mw-redirect" href="/wiki/Rice%27s_Theorem" title="Rice's Theorem">Rice's Theorem</a>, deciding membership in any nontrivial subset of the set of <a href="/wiki/Computable_function" title="Computable function">recursive functions</a> is <b>RE</b>-hard. It will be complete whenever the set is recursively enumerable.</li>
<li><a href="/wiki/John_Myhill" title="John Myhill">John Myhill</a> (<a href="#CITEREFMyhill1955">1955</a>)<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> has proven that all <a class="mw-redirect" href="/wiki/Creative_set" title="Creative set">creative sets</a> are <b>RE</b>-complete.</li>
<li>The uniform <a href="/wiki/Word_problem_(mathematics)" title="Word problem (mathematics)">word problem</a> for <a href="/wiki/Group_(mathematics)" title="Group (mathematics)">groups</a> or <a href="/wiki/Semigroup" title="Semigroup">semigroups</a>. [Indeed, the <a href="/wiki/Word_problem_for_groups" title="Word problem for groups">word problem for some individual groups</a> is <b>RE</b>-complete.]</li>
<li>Deciding membership in a general <a href="/wiki/Unrestricted_grammar" title="Unrestricted grammar">unrestricted</a> <a href="/wiki/Formal_grammar" title="Formal grammar">formal grammar</a>. [Again, certain individual grammars have <b>RE</b>-complete membership problem.]</li>
<li>The <a href="/wiki/Validity" title="Validity">validity</a> problem for <a href="/wiki/First-order_logic" title="First-order logic">first-order logic</a>.</li>
<li><a href="/wiki/Post_correspondence_problem" title="Post correspondence problem">Post correspondence problem</a>: Given a finite set of strings, determine if there is a string that can be factored into a composition of the strings (allowing repeats) in two different ways.</li>
<li>Determining if a <a href="/wiki/Diophantine_equation" title="Diophantine equation">Diophantine equation</a> has any integer solutions.</li>
</ol>
<h2><span class="mw-headline" id="co-RE-complete">co-RE-complete</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=RE_(complexity)&amp;action=edit&amp;section=3" title="Edit section: co-RE-complete">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><b>co-RE-complete</b> is the set of decision problems that are complete for <b>co-RE</b>. In a sense, these are the complements of the hardest recursively enumerable problems.</p>
<p>Examples of co-RE-complete problems:</p>
<ol>
<li>The <a class="mw-redirect" href="/wiki/Domino_Problem" title="Domino Problem">Domino Problem</a> for <a href="/wiki/Wang_tile" title="Wang tile">Wang tiles</a>.</li>
<li>The <a href="/wiki/Satisfiability" title="Satisfiability">satisfiability</a> problem for <a href="/wiki/First-order_logic" title="First-order logic">first-order logic</a></li>
</ol>
