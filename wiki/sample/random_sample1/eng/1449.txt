<p><b>SCons</b> is a <a class="mw-redirect" href="/wiki/Computer_software" title="Computer software">computer software</a> construction tool that automatically analyzes source code file dependencies and operating system adaptation requirements from a software project description and generates final binary executables for installation on the target operating system platform. Its function is analogous to the traditional <a class="mw-redirect" href="/wiki/GNU_build_system" title="GNU build system">GNU build system</a> based on the <a href="/wiki/Make_(software)" title="Make (software)">make</a> utility and the <a href="/wiki/Autoconf" title="Autoconf">autoconf</a> tools.</p>
<p>SCons generates project configurations and build process implementations in the form of <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> scripts.</p>

<h2><span class="mw-headline" id="Major_features">Major features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=SCons&amp;action=edit&amp;section=1" title="Edit section: Major features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Major SCons features include the following:</p>
<ul>
<li>Configuration files are Python scripts, which means that user-written builds have access to a complete general-purpose programming language.</li>
<li>Automatic <a href="/wiki/Coupling_(computer_programming)" title="Coupling (computer programming)">dependency</a> analysis built-in for <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a> and <a href="/wiki/Fortran" title="Fortran">Fortran</a>. Dependency analysis is extensible through user-defined dependency scanners for other languages or file types. Unlike the <a href="/wiki/GNU_Compiler_Collection" title="GNU Compiler Collection">GNU Compiler Collection</a>'s (GCC) built-in dependency analysis, it uses a <a href="/wiki/Regular_expression" title="Regular expression">regular expression</a> scan for included source files.</li>
<li>Built-in support for C, C++, <a href="/wiki/D_(programming_language)" title="D (programming language)">D</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, <a href="/wiki/Fortran" title="Fortran">Fortran</a>, <a href="/wiki/Objective-C" title="Objective-C">Objective-C</a>, <a href="/wiki/Yacc" title="Yacc">Yacc</a>, <a href="/wiki/Lex_(software)" title="Lex (software)">Lex</a>, <a class="mw-redirect" href="/wiki/Qt_(framework)" title="Qt (framework)">Qt</a> and <a href="/wiki/SWIG" title="SWIG">SWIG</a>, as well as <a href="/wiki/TeX" title="TeX">TeX</a> and <a href="/wiki/LaTeX" title="LaTeX">LaTeX</a> documents. SCons can also handle other languages or file types through user-defined builders.</li>
<li>Building from central repositories of source code and pre-built targets.</li>
<li>Built-in ability to fetch source files from <a class="mw-redirect" href="/wiki/Revision_control" title="Revision control">revision control</a> systems, such as <a href="/wiki/Source_Code_Control_System" title="Source Code Control System">SCCS</a>, <a href="/wiki/Revision_Control_System" title="Revision Control System">RCS</a>, <a href="/wiki/Concurrent_Versions_System" title="Concurrent Versions System">CVS</a>, <a href="/wiki/Apache_Subversion" title="Apache Subversion">Subversion</a>, <a href="/wiki/BitKeeper" title="BitKeeper">BitKeeper</a> and <a class="mw-redirect" href="/wiki/Perforce" title="Perforce">Perforce</a>.</li>
<li>Built-in ability to use <a href="/wiki/Microsoft_Visual_Studio" title="Microsoft Visual Studio">Microsoft Visual Studio</a>, including the generation of .dsp, .dsw, .sln and .vcproj files.</li>
<li>Detection of file content changes using <a href="/wiki/MD5" title="MD5">MD5</a> signatures; optional, configurable abiltity to use traditional timestamps.</li>
<li>Ability to do parallel builds, maintaining a specified number of jobs running simultaneously regardless of directory hierarchy.</li>
<li>Integrated Autoconf-like support for finding #include files, libraries, functions and typedefs.</li>
<li>Global view of all dependencies, so multiple build passes or reordering targets is not required.</li>
<li>Ability to share built files in a cache to speed up multiple builds - like <a href="/wiki/Ccache" title="Ccache">ccache</a> but for any type of target file, not just C/C++ compilation.</li>
<li>Designed from the ground up for cross-platform builds, and known to work on <a href="/wiki/POSIX" title="POSIX">POSIX</a> systems (including <a href="/wiki/GNU" title="GNU">GNU</a>/<a href="/wiki/Linux" title="Linux">Linux</a>, <a href="/wiki/IBM" title="IBM">IBM</a> <a href="/wiki/IBM_AIX" title="IBM AIX">AIX</a> and <a href="/wiki/OS/2" title="OS/2">OS/2</a>, <a href="/wiki/Berkeley_Software_Distribution" title="Berkeley Software Distribution">*BSD Unices</a>, <a href="/wiki/HP-UX" title="HP-UX">HP-UX</a>, <a href="/wiki/Silicon_Graphics" title="Silicon Graphics">SGI</a> <a href="/wiki/IRIX" title="IRIX">IRIX</a>, <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">Solaris</a>), <a href="/wiki/Microsoft" title="Microsoft">MS</a> <a href="/wiki/Windows_NT" title="Windows NT">Windows NT</a>, <a class="mw-redirect" href="/wiki/Apple,_Inc." title="Apple, Inc.">Apple</a> <a class="mw-redirect" href="/wiki/OS_X" title="OS X">OS X</a>.</li>
</ul>
<h2><span class="mw-headline" id="History_and_related_projects">History and related projects</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=SCons&amp;action=edit&amp;section=2" title="Edit section: History and related projects">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>SCons software history started with the <i>Cons</i> software construction utility created by Bob Sidebotham in 1999.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> Cons was written in the <a href="/wiki/Perl" title="Perl">Perl</a> language. It served as a base for the <i>ScCons</i> build tool, a design which won the <a class="new" href="/w/index.php?title=Software_Carpentry&amp;action=edit&amp;redlink=1" title="Software Carpentry (page does not exist)">Software Carpentry</a> project SC Build competition in August 2000.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> ScCons was the foundation for SCons.</p>
<p>SCons inspired the creation of <a href="/wiki/Waf" title="Waf">Waf</a>, formerly known as <i>SCons/BKsys</i>, which emerged in the <a href="/wiki/KDE" title="KDE">KDE</a> community. For some time, there were plans to use it as the build tool for KDE 4 and beyond, but that effort was abandoned in favor of <a href="/wiki/CMake" title="CMake">CMake</a>.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>

<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=SCons&amp;action=edit&amp;section=3" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The following example is a very simple SConstruct file that compiles the C program file hello-world.c using the default platform compiler:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">Program</span><span class="p">(</span><span class="s1">'hello-world.c'</span><span class="p">)</span>
</pre></div>
<p>The following is a more complex example that creates an environment used to build the program hello:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">env</span> <span class="o">=</span> <span class="n">Environment</span><span class="p">()</span>
<span class="n">env</span><span class="o">.</span><span class="n">Append</span><span class="p">(</span><span class="n">CPPFLAGS</span><span class="o">=</span><span class="p">[</span><span class="s1">'-Wall'</span><span class="p">,</span><span class="s1">'-g'</span><span class="p">])</span>
<span class="n">env</span><span class="o">.</span><span class="n">Program</span><span class="p">(</span><span class="s1">'hello'</span><span class="p">,</span>
            <span class="p">[</span><span class="s1">'hello.c'</span><span class="p">,</span> <span class="s1">'main.c'</span><span class="p">])</span>
</pre></div>
