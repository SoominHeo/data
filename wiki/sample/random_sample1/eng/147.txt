<h1 class="firstHeading" id="firstHeading" lang="en">Computer font</h1>

<p>A <b>computer font</b> (or <b><a href="/wiki/Font" title="Font">font</a></b>) is an electronic <a href="/wiki/Data_file" title="Data file">data file</a> containing a set of <a href="/wiki/Glyph" title="Glyph">glyphs</a>, characters, or symbols such as <a href="/wiki/Dingbat" title="Dingbat">dingbats</a>. Although the term <i><a href="/wiki/Font" title="Font">font</a></i> first referred to a set of metal type sorts in one style and size, since the 1990s it is generally used to refer to a scalable set of digital shapes that may be printed at many different sizes.</p>
<p>There are three basic kinds of computer font file data formats:</p>
<ul>
<li><b>Bitmap</b> fonts consist of a matrix of dots or <a href="/wiki/Pixel" title="Pixel">pixels</a> representing the image of each glyph in each face and size.</li>
<li><b>Outline</b> fonts (also called <b>vector</b> fonts) use <a href="/wiki/B%C3%A9zier_curve" title="Bézier curve">Bézier curves</a>, drawing instructions and mathematical formulae to describe each glyph, which make the character outlines scalable to any size.</li>
<li><b>Stroke</b> fonts use a series of specified lines and additional information to define the <i>profile</i>, or size and shape of the line in a specific face, which together describe the appearance of the glyph.</li>
</ul>
<p>Bitmap fonts are faster and easier to use in computer code, but non-scalable, requiring a separate font for each size.<sup class="reference" id="cite_ref-Anti-Anti-Aliasing_1-0"><a href="#cite_note-Anti-Anti-Aliasing-1">[1]</a></sup> Outline and stroke fonts can be resized using a single font and substituting different measurements for components of each glyph, but are somewhat more complicated to render on screen than bitmap fonts, as they require additional computer code to render the outline to a bitmap for display on screen or in print. Although all types are still in use, most fonts seen and used on computers are outline fonts.</p>
<p>Fonts are designed and created using <a href="/wiki/Font_editor" title="Font editor">font editors</a>. Fonts specifically designed for the computer screen and not printing are known as <i>screen fonts</i>.</p>
<p>Fonts can be <a href="/wiki/Monospaced_font" title="Monospaced font">monospaced</a> (i.e., every character is plotted a constant distance from the previous character that it is next to, while drawing) or proportional (each character has its own width). However, the particular font-handling application can affect the spacing, particularly when doing <a class="mw-redirect" href="/wiki/Justification_(typesetting)" title="Justification (typesetting)">justification</a>.</p>
<p></p>

<h2><span class="mw-headline" id="Font_types">Font types</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_font&amp;action=edit&amp;section=1" title="Edit section: Font types">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Bitmap_fonts"><span id="BITMAP"></span>Bitmap fonts</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_font&amp;action=edit&amp;section=2" title="Edit section: Bitmap fonts">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Original_Mac_fonts.png"><img alt="" class="thumbimage" data-file-height="237" data-file-width="381" height="137" src="//upload.wikimedia.org/wikipedia/commons/thumb/2/29/Original_Mac_fonts.png/220px-Original_Mac_fonts.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/2/29/Original_Mac_fonts.png/330px-Original_Mac_fonts.png 1.5x, //upload.wikimedia.org/wikipedia/commons/2/29/Original_Mac_fonts.png 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Original_Mac_fonts.png" title="Enlarge"></a></div>
An assortment of bitmap fonts from the first version of the <a href="/wiki/Classic_Mac_OS" title="Classic Mac OS">Macintosh operating system</a>.</div>
</div>
</div>
<p>A bitmap font is one that stores each <a href="/wiki/Glyph" title="Glyph">glyph</a> as an array of <a class="mw-redirect" href="/wiki/Pixels" title="Pixels">pixels</a> (that is, a <a href="/wiki/Bitmap" title="Bitmap">bitmap</a>). It is less commonly known as a <b>raster font</b>. Bitmap fonts are simply collections of <a href="/wiki/Raster_graphics" title="Raster graphics">raster images</a> of glyphs. For each variant of the font, there is a complete set of glyph images, with each set containing an image for each character. For example, if a font has three sizes, and any combination of bold and italic, then there must be 12 complete sets of images.</p>
<p>Advantages of bitmap fonts include:</p>
<ul>
<li>Extremely fast and simple to render</li>
<li>Easier to create than other kinds.</li>
<li>Unscaled bitmap fonts always give exactly the same output when displayed on the same specification display</li>
<li>Best for very low-quality or small-size displays where the font needs to be fine-tuned to display clearly</li>
</ul>
<p>The primary disadvantage of bitmap fonts is that the visual quality tends to be poor when scaled or otherwise transformed, compared to outline and stroke fonts, and providing many optimized and purpose-made sizes of the same font dramatically increases memory usage. The earliest bitmap fonts were only available in certain optimized sizes such as 8, 9, 10, 12, 14, 18, 24, 36, 48, 72, and 96 points (assuming a resolution of 96 <a href="/wiki/Dots_per_inch" title="Dots per inch">DPI</a>), with custom fonts often available in only one specific size, such as a headline font at only 72 points.</p>
<p>The limited processing power and memory of early computer systems forced exclusive use of bitmap fonts. Improvements in hardware have allowed them to be replaced with outline or stroke fonts in cases where arbitrary scaling is desirable, but bitmap fonts are still in common use in embedded systems and other places where speed and simplicity are considered important.</p>
<p>Bitmap fonts are used in the <a href="/wiki/Linux" title="Linux">Linux</a> console, the <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a> <a href="/wiki/Recovery_Console" title="Recovery Console">recovery console</a>, and <a class="mw-redirect" href="/wiki/Embedded_systems" title="Embedded systems">embedded systems</a>. Older <a class="mw-redirect" href="/wiki/Dot_matrix_printer" title="Dot matrix printer">dot matrix printers</a> used bitmap fonts; often stored in the memory of the printer and addressed by the computer's <a class="mw-redirect" href="/wiki/Print_driver" title="Print driver">print driver</a>. Bitmap fonts may be used in <a href="/wiki/Cross-stitch" title="Cross-stitch">cross-stitch</a>.</p>
<p>To draw a string using a bitmap font, means to successively output bitmaps of each character that the string comprises, performing per-character indentation.</p>
<h4><span class="mw-headline" id="Monochrome_fonts_vs._fonts_with_shades_of_gray">Monochrome fonts vs. fonts with shades of gray</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_font&amp;action=edit&amp;section=3" title="Edit section: Monochrome fonts vs. fonts with shades of gray">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Digital bitmap fonts (and the <a href="/wiki/Font_rasterization" title="Font rasterization">final rendering of vector fonts</a>) may use <a href="/wiki/Monochrome" title="Monochrome">monochrome</a> or <a href="/wiki/Grayscale" title="Grayscale">shades of gray</a>. The latter is <a href="/wiki/Spatial_anti-aliasing" title="Spatial anti-aliasing">anti-aliased</a>. When displaying a text, typically an operating system properly represents the "shades of gray" as intermediate colors between the color of the font and that of the background. However, if the text is represented as an <i>image</i> with <a class="mw-redirect" href="/wiki/Transparency_in_graphic_files" title="Transparency in graphic files">transparent</a> background, "shades of gray" require an image format allowing <a href="/wiki/Alpha_compositing" title="Alpha compositing">partial transparency</a>.</p>
<h4><span class="mw-headline" id="Scaling">Scaling</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_font&amp;action=edit&amp;section=4" title="Edit section: Scaling">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Bitmap fonts look best at their native <a href="/wiki/Pixel" title="Pixel">pixel</a> size. Some systems using bitmap fonts can create some font variants algorithmically. For example, the original <a class="mw-redirect" href="/wiki/Apple_Macintosh" title="Apple Macintosh">Apple Macintosh</a> computer could produce bold by widening vertical strokes and oblique by <a href="/wiki/Shear_mapping" title="Shear mapping">shearing</a> the image. At non-native sizes, many text rendering systems perform <a href="/wiki/Nearest-neighbor_interpolation" title="Nearest-neighbor interpolation">nearest-neighbor resampling</a>, introducing rough jagged edges. More advanced systems perform <a href="/wiki/Spatial_anti-aliasing" title="Spatial anti-aliasing">anti-aliasing</a> on bitmap fonts whose size does not match the size that the application requests. This technique works well for making the font smaller but not as well for increasing the size, as it tends to blur the edges. Some graphics systems that use bitmap fonts, especially those of <a href="/wiki/Emulator" title="Emulator">emulators</a>, apply curve-sensitive <a class="new" href="/w/index.php?title=Nonlinear_resampling&amp;action=edit&amp;redlink=1" title="Nonlinear resampling (page does not exist)">nonlinear resampling</a> algorithms such as <a class="mw-redirect" href="/wiki/2xSaI" title="2xSaI">2xSaI</a> or <a class="mw-redirect" href="/wiki/Hq3x" title="Hq3x">hq3x</a> on fonts and other bitmaps, which avoids blurring the font while introducing little objectionable distortion at moderate increases in size.</p>
<p>The difference between bitmap fonts and outline fonts is similar to the difference between bitmap and vector image file formats. Bitmap fonts are like image formats such as <i><a class="mw-redirect" href="/wiki/Windows_bitmap" title="Windows bitmap">Windows Bitmap</a></i> (.bmp), <i><a href="/wiki/Portable_Network_Graphics" title="Portable Network Graphics">Portable Network Graphics</a></i> (.png) and <i><a class="mw-redirect" href="/wiki/TIF" title="TIF">Tagged Image Format</a></i> (.tif or .tiff), which store the image data as a grid of pixels, in some cases with compression. <b>Outline</b> or <b>stroke</b> image formats such as <i><a href="/wiki/Windows_Metafile" title="Windows Metafile">Windows Metafile</a></i> format (.wmf) and <i><a href="/wiki/Scalable_Vector_Graphics" title="Scalable Vector Graphics">Scalable Vector Graphics</a></i> format (.svg), store instructions in the form of lines and curves of how to draw the image rather than storing the image itself.</p>
<p>A "trace" program can follow the outline of a high-resolution bitmap font and create an initial outline that a font designer uses to create an <a class="mw-redirect" href="/wiki/Outline_font" title="Outline font">outline font</a> useful in systems such as <a href="/wiki/PostScript" title="PostScript">PostScript</a> or <a href="/wiki/TrueType" title="TrueType">TrueType</a>. Outline fonts scale easily without jagged edges or blurriness.</p>

<h3><span class="mw-headline" id="Outline_fonts"><span id="OUTLINE"></span>Outline fonts</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_font&amp;action=edit&amp;section=6" title="Edit section: Outline fonts">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><i>Outline fonts</i> or <i>vector fonts</i> are collections of <a href="/wiki/Vector_graphics" title="Vector graphics">vector images</a>, consisting of lines and curves defining the boundary of <a class="mw-redirect" href="/wiki/Glyphs" title="Glyphs">glyphs</a>. Early vector fonts were used by <a href="/wiki/Vector_monitor" title="Vector monitor">vector monitors</a> and <a href="/wiki/Plotter" title="Plotter">vector plotters</a> using their own internal fonts, usually with thin single strokes instead of thick outlined glyphs. The advent of desktop publishing brought the need for a universal standard to integrate the <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a> of the first <a href="/wiki/Macintosh" title="Macintosh">Macintosh</a> and <a class="mw-redirect" href="/wiki/Laser_printer" title="Laser printer">laser printers</a>. The term to describe the integration technology was <a href="/wiki/WYSIWYG" title="WYSIWYG">WYSIWYG</a> (What You See Is What You Get). The universal standard was (and still is) Adobe <a href="/wiki/PostScript" title="PostScript">PostScript</a>. Examples are PostScript <a class="mw-redirect" href="/wiki/Type_1_and_Type_3_fonts" title="Type 1 and Type 3 fonts">Type 1 and Type 3 fonts</a>, <a href="/wiki/TrueType" title="TrueType">TrueType</a> and <a href="/wiki/OpenType" title="OpenType">OpenType</a>.</p>
<p>The primary advantage of outline fonts is that, unlike <a class="mw-redirect" href="/wiki/Bitmap_fonts" title="Bitmap fonts">bitmap fonts</a>, they are a set of lines and curves instead of pixels; they can be scaled without causing <a class="mw-redirect" href="/wiki/Pixellation" title="Pixellation">pixellation</a>. Therefore, outline font characters can be scaled to any size and otherwise transformed with more attractive results than bitmap fonts, but require considerably more processing and may yield undesirable rendering, depending on the font, rendering software, and output size. Even so, outline fonts can be transformed into bitmap fonts beforehand if necessary. The converse transformation is considerably harder, since bitmap fonts requires <a class="mw-redirect" href="/wiki/Heuristic_algorithm" title="Heuristic algorithm">heuristic algorithm</a> to guess and approximate the corresponding curves if the pixels do not make a straight line.</p>
<p>Outline fonts have a major problem, in that the <a href="/wiki/B%C3%A9zier_curve" title="Bézier curve">Bézier curves</a> used by them cannot be rendered accurately onto a raster display (such as most computer monitors and printers), and their rendering can change shape depending on the desired size and position.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> Measures such as <a href="/wiki/Font_hinting" title="Font hinting">font hinting</a> have to be used to reduce the visual impact of this problem, which require sophisticated software that is difficult to implement correctly. Many modern desktop computer systems include software to do this, but they use considerably more processing power than bitmap fonts, and there can be minor rendering defects, particularly at small font sizes. Despite this, they are frequently used because people often consider the processing time and defects to be acceptable when compared to the ability to scale fonts freely.</p>
<h4><span class="mw-headline" id="Outline_font_formats">Outline font formats</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_font&amp;action=edit&amp;section=7" title="Edit section: Outline font formats">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<h5><span class="mw-headline" id="Type_1_and_Type_3_fonts">Type 1 and Type 3 fonts</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_font&amp;action=edit&amp;section=8" title="Edit section: Type 1 and Type 3 fonts">edit</a><span class="mw-editsection-bracket">]</span></span></h5>
<p><a class="mw-redirect" href="/wiki/Type_1_and_Type_3_fonts" title="Type 1 and Type 3 fonts">Type 1 and Type 3 fonts</a> were developed by <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe</a> for professional digital typesetting. Using <a href="/wiki/PostScript" title="PostScript">PostScript</a>, the glyphs are outline fonts described with cubic <a class="mw-redirect" href="/wiki/Bezier_curves" title="Bezier curves">Bezier curves</a>. Type 1 fonts were restricted to a subset of the PostScript language, and used Adobe's hinting system, which used to be very expensive. Type 3 allowed unrestricted use of the PostScript language, but didn't include any hint information, which could lead to visible rendering artifacts on low-resolution devices (such as computer screens and dot-matrix printers).</p>
<h5><span class="mw-headline" id="TrueType_fonts">TrueType fonts</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_font&amp;action=edit&amp;section=9" title="Edit section: TrueType fonts">edit</a><span class="mw-editsection-bracket">]</span></span></h5>
<p><a href="/wiki/TrueType" title="TrueType">TrueType</a> is a font system originally developed by <a class="mw-redirect" href="/wiki/Apple_Inc" title="Apple Inc">Apple Inc</a>. It was intended to replace Type 1 fonts, which many felt were too expensive. Unlike Type 1 fonts, TrueType glyphs are described with quadratic Bezier curves. It is currently very popular and implementations exist for all major operating systems.</p>
<h5><span class="mw-headline" id="OpenType_fonts">OpenType fonts</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_font&amp;action=edit&amp;section=10" title="Edit section: OpenType fonts">edit</a><span class="mw-editsection-bracket">]</span></span></h5>
<p><a href="/wiki/OpenType" title="OpenType">OpenType</a> is a <a class="new" href="/w/index.php?title=Smartfont&amp;action=edit&amp;redlink=1" title="Smartfont (page does not exist)">smartfont</a> system designed by <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe</a> and <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>. OpenType fonts contain outlines in either the TrueType or Type 1 (actually <a class="mw-redirect" href="/wiki/Compact_Font_Format" title="Compact Font Format">CFF</a>) format together with a wide range of metadata.</p>
<h3><span class="mw-headline" id="Stroke-based_fonts">Stroke-based fonts</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_font&amp;action=edit&amp;section=11" title="Edit section: Stroke-based fonts">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Metafont_pens.png"><img alt="" class="thumbimage" data-file-height="260" data-file-width="271" height="211" src="//upload.wikimedia.org/wikipedia/commons/thumb/3/34/Metafont_pens.png/220px-Metafont_pens.png" srcset="//upload.wikimedia.org/wikipedia/commons/3/34/Metafont_pens.png 1.5x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Metafont_pens.png" title="Enlarge"></a></div>
With stroke-based fonts, the same stroke paths can be filled with different stroke profiles resulting in different visual shapes without the need to specify the vertex positions of each outline, as is the case with outline fonts.</div>
</div>
</div>
<p>A glyph's outline is defined by the vertices of individual stroke paths, and the corresponding stroke profiles. The stroke paths are a kind of <a href="/wiki/Topological_skeleton" title="Topological skeleton">topological skeleton</a> of the glyph. The advantages of stroke-based fonts over outline fonts include reducing number of vertices needed to define a glyph, allowing the same vertices to be used to generate a font with a different weight, glyph width, or serifs using different stroke rules, and the associated size savings. For a font developer, editing a glyph by stroke is easier and less prone to error than editing outlines. A stroke-based system also allows scaling glyphs in height or width without altering stroke thickness of the base glyphs. Stroke-based fonts are heavily marketed for East Asian markets for use on embedded devices, but the technology is not limited to <a href="/wiki/Ideogram" title="Ideogram">ideograms</a>.</p>
<p>Commercial developers included <a class="mw-redirect" href="/wiki/Agfa_Monotype" title="Agfa Monotype">Agfa Monotype</a> (iType), Type Solutions, Inc. (owned by <a href="/wiki/Bitstream_Inc." title="Bitstream Inc.">Bitstream Inc.</a>) (Font Fusion (FFS), btX2), <a class="new" href="/w/index.php?title=Fontworks&amp;action=edit&amp;redlink=1" title="Fontworks (page does not exist)">Fontworks</a> (Gaiji Master), which have independently developed stroke-based font types and font engines.</p>
<p>Although Monotype and Bitstream have claimed tremendous space saving using stroke-based fonts on East Asian character sets, most of the space saving comes from building composite glyphs, which is part of the TrueType specification and does not require a stroke-based approach.</p>
<h4><span class="mw-headline" id="Stroke-based_font_formats">Stroke-based font formats</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_font&amp;action=edit&amp;section=12" title="Edit section: Stroke-based font formats">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p><a class="mw-redirect" href="/wiki/METAFONT" title="METAFONT">METAFONT</a> uses a different sort of glyph description. Like TrueType, it is a vector font description system. It draws glyphs using strokes produced by moving a polygonal or elliptical pen approximated by a polygon along a path made from cubic <a href="/wiki/Composite_B%C3%A9zier_curve" title="Composite Bézier curve">composite Bézier curves</a> and straight line segments, or by filling such paths. Although when stroking a path the envelope of the stroke is never actually generated, the method causes no loss of accuracy or resolution. The method Metafont uses is more mathematically complex because the <a href="/wiki/Parallel_curve" title="Parallel curve">parallel curves</a> of a Bézier can be 10th order algebraic curves.<sup class="reference" id="cite_ref-Kilgard_4-0"><a href="#cite_note-Kilgard-4">[4]</a></sup></p>
<p>In 2004, <a href="/wiki/DynaComware" title="DynaComware">DynaComware</a> developed DigiType, a stroke-based font format. In 2006, the creators of the <a href="/wiki/Saffron_Type_System" title="Saffron Type System">Saffron Type System</a> announced a representation for stroke-based fonts called <a href="/wiki/Saffron_Type_System#Stylized_stroke_fonts" title="Saffron Type System">Stylized Stroke Fonts</a> (SSFs) with the aim of providing the expressiveness of traditional outline-based fonts and the small memory footprint of uniform-width stroke-based fonts (USFs).<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
