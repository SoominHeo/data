<p><b>Berkeley Software Distribution</b> (<b>BSD</b>) is a <a href="/wiki/Unix" title="Unix">Unix</a> <a href="/wiki/Operating_system" title="Operating system">operating system</a> derivative developed and distributed by the <a href="/wiki/Computer_Systems_Research_Group" title="Computer Systems Research Group">Computer Systems Research Group</a> (CSRG) of the <a href="/wiki/University_of_California,_Berkeley" title="University of California, Berkeley">University of California, Berkeley</a>, from 1977 to 1995. Today the term "BSD" is often used non-specifically to refer to any of the BSD descendants which together form a branch of the family of <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> operating systems. Operating systems derived from the original BSD <a href="/wiki/Source_code" title="Source code">code</a> remain actively developed and widely used.</p>
<p>Historically, BSD has been considered a branch of Unix, <b>Berkeley Unix</b>, because it shared the initial codebase and design with the original <a href="/wiki/AT%26T" title="AT&amp;T">AT&amp;T</a> Unix operating system. In the 1980s, BSD was widely adopted by vendors of <a class="mw-redirect" href="/wiki/Computer_workstation" title="Computer workstation">workstation</a>-class systems in the form of proprietary Unix variants such as <a href="/wiki/Digital_Equipment_Corporation" title="Digital Equipment Corporation">DEC</a> <a class="mw-redirect" href="/wiki/ULTRIX" title="ULTRIX">ULTRIX</a> and <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a> <a href="/wiki/SunOS" title="SunOS">SunOS</a>. This can be attributed to the ease with which it could be licensed, and the familiarity the founders of many technology companies of the time had with it.</p>
<p>Although these proprietary BSD derivatives were largely superseded by the UNIX <a class="mw-redirect" href="/wiki/System_V_Release_4" title="System V Release 4">System V Release 4</a> and <a class="mw-redirect" href="/wiki/OSF/1" title="OSF/1">OSF/1</a> systems in the 1990s (both of which incorporated BSD code and are the basis of other modern Unix systems), later BSD releases provided a basis for several <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> development projects that are ongoing, e.g., <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a>, <a href="/wiki/OpenBSD" title="OpenBSD">OpenBSD</a>, <a href="/wiki/NetBSD" title="NetBSD">NetBSD</a>, <a href="/wiki/Darwin_(operating_system)" title="Darwin (operating system)">Darwin,</a> and <a class="mw-redirect" href="/wiki/PC-BSD" title="PC-BSD">PC-BSD</a>. These, in turn, have been incorporated in whole or in part in modern proprietary operating systems, e.g., the <a href="/wiki/Internet_protocol_suite" title="Internet protocol suite">TCP/IP</a> networking code in <a href="/wiki/Windows_NT_3.1" title="Windows NT 3.1">Windows NT 3.1</a><sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> and most of the foundation of <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple</a>'s <a href="/wiki/MacOS" title="MacOS">macOS</a> and <a href="/wiki/IOS" title="IOS">iOS</a>.</p>

<h3><span class="mw-headline" id="1BSD_.28PDP-11.29">1BSD (PDP-11)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=2" title="Edit section: 1BSD (PDP-11)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/Research_Unix" title="Research Unix">earliest distributions</a> of Unix from <a href="/wiki/Bell_Labs" title="Bell Labs">Bell Labs</a> in the 1970s included the <a href="/wiki/Source_code" title="Source code">source code</a> to the operating system, allowing researchers at universities to modify and extend Unix. The operating system arrived at Berkeley in 1974, at the request of computer science professor <a href="/wiki/Bob_Fabry" title="Bob Fabry">Bob Fabry</a> who had been on the program committee for the <a href="/wiki/Symposium_on_Operating_Systems_Principles" title="Symposium on Operating Systems Principles">Symposium on Operating Systems Principles</a> where Unix was first presented. A <a class="mw-redirect" href="/wiki/PDP-11/45" title="PDP-11/45">PDP-11/45</a> was bought to run the system, but for budgetary reasons, this machine was shared with the mathematics and statistics groups at Berkeley, who used <a href="/wiki/RSTS/E" title="RSTS/E">RSTS</a>, so that Unix only ran on the machine eight hours per day (sometimes during the day, sometimes during the night). A larger <a class="mw-redirect" href="/wiki/PDP-11/70" title="PDP-11/70">PDP-11/70</a> was installed at Berkeley the following year, using money from the <a href="/wiki/Ingres_(database)" title="Ingres (database)">Ingres</a> database project.<sup class="reference" id="cite_ref-penguin7_2-0"><a href="#cite_note-penguin7-2">[2]</a></sup></p>
<p>Also in 1975, <a href="/wiki/Ken_Thompson" title="Ken Thompson">Ken Thompson</a> took a <a href="/wiki/Sabbatical" title="Sabbatical">sabbatical</a> from Bell Labs and came to Berkeley as a visiting professor. He helped to install <a href="/wiki/Version_6_Unix" title="Version 6 Unix">Version 6 Unix</a> and started working on a <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a> implementation for the system. Graduate students Chuck Haley and <a href="/wiki/Bill_Joy" title="Bill Joy">Bill Joy</a> improved Thompson's Pascal and implemented an improved text editor, <a href="/wiki/Ex_(text_editor)" title="Ex (text editor)">ex</a>.<sup class="reference" id="cite_ref-penguin7_2-1"><a href="#cite_note-penguin7-2">[2]</a></sup> Other universities became interested in the software at Berkeley, and so in 1977 Joy started compiling the first Berkeley Software Distribution (1BSD), which was released on March 9, 1978.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> 1BSD was an add-on to Version 6 Unix rather than a complete operating system in its own right. Some thirty copies were sent out.<sup class="reference" id="cite_ref-penguin7_2-2"><a href="#cite_note-penguin7-2">[2]</a></sup></p>
<h3><span class="mw-headline" id="2BSD_.28PDP-11.29">2BSD (PDP-11)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=3" title="Edit section: 2BSD (PDP-11)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The Second Berkeley Software Distribution (2BSD), released in May 1979,<sup class="reference" id="cite_ref-tuhs-2bsd_4-0"><a href="#cite_note-tuhs-2bsd-4">[4]</a></sup> included updated versions of the 1BSD software as well as two new programs by Joy that persist on Unix systems to this day: the <a href="/wiki/Vi" title="Vi">vi</a> text editor (a <a href="/wiki/Visual_editor" title="Visual editor">visual</a> version of <a href="/wiki/Ex_(text_editor)" title="Ex (text editor)">ex</a>) and the <a href="/wiki/C_shell" title="C shell">C shell</a>. Some 75 copies of 2BSD were sent out by Bill Joy.<sup class="reference" id="cite_ref-penguin7_2-3"><a href="#cite_note-penguin7-2">[2]</a></sup> A further feature was a networking package called <a href="/wiki/Berknet" title="Berknet">Berknet</a>, developed by <a href="/wiki/Eric_Schmidt" title="Eric Schmidt">Eric Schmidt</a> as part of his <a class="mw-redirect" href="/wiki/Master%27s_thesis" title="Master's thesis">master's thesis</a> work, that could connect up to twenty-six computers and provided email and file transfer.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<p>After 3BSD (see below) had came out for the <a href="/wiki/VAX" title="VAX">VAX</a> line of computers, new releases of 2BSD for the PDP-11 were still issued and distributed through <a href="/wiki/USENIX" title="USENIX">USENIX</a>; for example, 1982's 2.8.1BSD included a collection of fixes for performance problems in <a href="/wiki/Version_7_Unix" title="Version 7 Unix">Version 7 Unix</a>,<sup class="reference" id="cite_ref-penguin_6-0"><a href="#cite_note-penguin-6">[6]</a></sup> and later releases contained ports of changes from the VAX-based releases of BSD back to the PDP-11 architecture. 2.9BSD from 1983 included code from 4.1cBSD, and was the first release that was a full OS (a modified V7 Unix) rather than a set of applications and patches. The most recent release, 2.11BSD, was first issued in 1992. As of 2008, maintenance updates from volunteers are still continuing, with patch 448 being released on June 17, 2012.</p>

<p>A VAX computer was installed at Berkeley in 1978, but the <a href="/wiki/Porting" title="Porting">port</a> of Unix to the VAX architecture, <a href="/wiki/UNIX/32V" title="UNIX/32V">UNIX/32V</a>, did not take advantage of the VAX's <a href="/wiki/Virtual_memory" title="Virtual memory">virtual memory</a> capabilities. The <a class="mw-redirect" href="/wiki/Kernel_(computer_science)" title="Kernel (computer science)">kernel</a> of 32V was largely rewritten by Berkeley students to include a virtual memory implementation, and a complete operating system including the new kernel, ports of the 2BSD utilities to the VAX, and the utilities from 32V was released as 3BSD at the end of 1979. 3BSD was also alternatively called Virtual VAX/UNIX or VMUNIX (for Virtual Memory Unix), and BSD kernel images were normally called <code>/vmunix</code> until 4.4BSD.</p>
<p>The success of 3BSD was a major factor in the <a class="mw-redirect" href="/wiki/Defense_Advanced_Research_Projects_Agency" title="Defense Advanced Research Projects Agency">Defense Advanced Research Projects Agency</a>'s (DARPA) decision to fund Berkeley's <a href="/wiki/Computer_Systems_Research_Group" title="Computer Systems Research Group">Computer Systems Research Group</a> (CSRG), which would develop a standard Unix platform for future DARPA research in the <a href="/wiki/VLSI_Project" title="VLSI Project">VLSI Project</a>.</p>
<h3><span class="mw-headline" id="4BSD">4BSD</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=5" title="Edit section: 4BSD">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>4BSD (November 1980) offered a number of enhancements over 3BSD, notably <a href="/wiki/Job_control_(Unix)" title="Job control (Unix)">job control</a> in the previously released <a class="mw-redirect" href="/wiki/C_Shell" title="C Shell">csh</a>, <a href="/wiki/Delivermail" title="Delivermail">delivermail</a> (the antecedent of <a href="/wiki/Sendmail" title="Sendmail">sendmail</a>), "reliable" <a class="mw-redirect" href="/wiki/Signal_(computing)" title="Signal (computing)">signals</a>, and the <a href="/wiki/Curses_(programming_library)" title="Curses (programming library)">Curses</a> programming library. In a 1985 review of BSD releases, <a href="/wiki/John_Quarterman" title="John Quarterman">John Quarterman</a> <i>et al.</i>, wrote:<sup class="reference" id="cite_ref-quarterman42bsd_7-0"><a href="#cite_note-quarterman42bsd-7">[7]</a></sup></p>
<blockquote class="templatequote">
<p>4BSD was the operating system of choice for VAXs from the beginning until the release of <a href="/wiki/UNIX_System_III" title="UNIX System III">System III</a> (1979–1982) [...] Most organizations would buy a 32V license and order 4BSD from Berkeley without ever bothering to get a 32V tape. Many installations inside the <a href="/wiki/Bell_System" title="Bell System">Bell System</a> ran 4.1BSD (many still do, and many others run 4.2BSD).</p>
</blockquote>
<h3><span class="mw-headline" id="4.1BSD">4.1BSD</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=6" title="Edit section: 4.1BSD">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>4.1BSD (June 1981) was a response to criticisms of BSD's performance relative to the dominant VAX operating system, <a href="/wiki/OpenVMS" title="OpenVMS">VMS</a>. The 4.1BSD kernel was systematically tuned up by <a href="/wiki/Bill_Joy" title="Bill Joy">Bill Joy</a> until it could perform as well as VMS on several benchmarks. The release would have been called <i>5BSD</i>, but after objections from <a href="/wiki/AT%26T" title="AT&amp;T">AT&amp;T</a> the name was changed; AT&amp;T feared confusion with <a href="/wiki/AT%26T" title="AT&amp;T">AT&amp;T</a>'s <a href="/wiki/UNIX_System_V" title="UNIX System V">UNIX System V</a>.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<h3><span class="mw-headline" id="4.2BSD">4.2BSD</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=7" title="Edit section: 4.2BSD">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>4.2BSD (August 1983) would take over two years to implement and contained several major overhauls. Before its official release came three intermediate versions: <i>4.1a</i> incorporated a modified version of <a class="mw-redirect" href="/wiki/Bolt,_Beranek_and_Newman" title="Bolt, Beranek and Newman">BBN's</a> preliminary <a class="mw-redirect" href="/wiki/TCP/IP" title="TCP/IP">TCP/IP</a> implementation; <i>4.1b</i> included the new <a class="mw-redirect" href="/wiki/Berkeley_Fast_File_System" title="Berkeley Fast File System">Berkeley Fast File System</a>, implemented by <a href="/wiki/Marshall_Kirk_McKusick" title="Marshall Kirk McKusick">Marshall Kirk McKusick</a>; and <i>4.1c</i> was an interim release during the last few months of 4.2BSD's development. Back at Bell Labs, 4.1cBSD became the basis of the <a class="mw-redirect" href="/wiki/Version_8_Unix" title="Version 8 Unix">8th Edition</a> of <a href="/wiki/Research_Unix" title="Research Unix">Research Unix</a>, and a commercially supported version was available from <a href="/wiki/MtXinu" title="MtXinu">mtXinu</a>.</p>
<p>To guide the design of 4.2BSD, <a class="new" href="/w/index.php?title=Duane_Adams&amp;action=edit&amp;redlink=1" title="Duane Adams (page does not exist)">Duane Adams</a> of <a href="/wiki/DARPA" title="DARPA">DARPA</a> formed a "steering committee" consisting of <a href="/wiki/Bob_Fabry" title="Bob Fabry">Bob Fabry</a>, <a href="/wiki/Bill_Joy" title="Bill Joy">Bill Joy</a> and <a class="mw-redirect" href="/wiki/Sam_Leffler" title="Sam Leffler">Sam Leffler</a> from <a href="/wiki/University_of_California,_Berkeley" title="University of California, Berkeley">UCB</a>, <a class="new" href="/w/index.php?title=Alan_Nemeth&amp;action=edit&amp;redlink=1" title="Alan Nemeth (page does not exist)">Alan Nemeth</a> and <a class="new" href="/w/index.php?title=Rob_Gurwitz&amp;action=edit&amp;redlink=1" title="Rob Gurwitz (page does not exist)">Rob Gurwitz</a> from <a href="/wiki/BBN_Technologies" title="BBN Technologies">BBN</a>, <a href="/wiki/Dennis_Ritchie" title="Dennis Ritchie">Dennis Ritchie</a> from <a href="/wiki/Bell_Labs" title="Bell Labs">Bell Labs</a>, <a class="new" href="/w/index.php?title=Keith_Lantz&amp;action=edit&amp;redlink=1" title="Keith Lantz (page does not exist)">Keith Lantz</a> from <a class="mw-redirect" href="/wiki/Stanford" title="Stanford">Stanford</a>, <a class="mw-redirect" href="/wiki/Rick_Rashid" title="Rick Rashid">Rick Rashid</a> from <a class="mw-redirect" href="/wiki/Carnegie-Mellon" title="Carnegie-Mellon">Carnegie-Mellon</a>, <a class="new" href="/w/index.php?title=Bert_Halstead&amp;action=edit&amp;redlink=1" title="Bert Halstead (page does not exist)">Bert Halstead</a> from <a href="/wiki/Massachusetts_Institute_of_Technology" title="Massachusetts Institute of Technology">MIT</a>, <a class="new" href="/w/index.php?title=Dan_Lynch_(ISI)&amp;action=edit&amp;redlink=1" title="Dan Lynch (ISI) (page does not exist)">Dan Lynch</a> from <a href="/wiki/Information_Sciences_Institute" title="Information Sciences Institute">ISI</a>, and <a href="/wiki/Gerald_J._Popek" title="Gerald J. Popek">Gerald J. Popek</a> of <a href="/wiki/University_of_California,_Los_Angeles" title="University of California, Los Angeles">UCLA</a>. The committee met from April 1981 to June 1983.</p>
<p>Apart from the Fast File System, several features from outside contributors were accepted, including disk quotas and job control. <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a> provided testing on its <a href="/wiki/Motorola_68000" title="Motorola 68000">Motorola 68000</a> machines prior to release, ensuring portability of the system.<sup class="reference" id="cite_ref-quarterman42bsd_7-1"><a href="#cite_note-quarterman42bsd-7">[7]</a></sup></p>
<p>The official 4.2BSD release came in August 1983. It was notable as the first version released after the 1982 departure of Bill Joy to co-found Sun Microsystems; <a class="mw-redirect" href="/wiki/Mike_Karels" title="Mike Karels">Mike Karels</a> and <a href="/wiki/Marshall_Kirk_McKusick" title="Marshall Kirk McKusick">Marshall Kirk McKusick</a> took on leadership roles within the project from that point forward. On a lighter note, it also marked the debut of <a href="/wiki/BSD_Daemon" title="BSD Daemon">BSD's daemon mascot</a> in a drawing by <a href="/wiki/John_Lasseter" title="John Lasseter">John Lasseter</a> that appeared on the cover of the printed manuals distributed by <a href="/wiki/USENIX" title="USENIX">USENIX</a>.</p>
<h3><span class="mw-headline" id="4.3BSD">4.3BSD</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=8" title="Edit section: 4.3BSD">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

<p>4.3BSD was released in June 1986. Its main changes were to improve the performance of many of the new contributions of 4.2BSD that had not been as heavily tuned as the 4.1BSD code. Prior to the release, BSD's implementation of TCP/IP had diverged considerably from BBN's official implementation. After several months of testing, DARPA determined that the 4.2BSD version was superior and would remain in 4.3BSD. (See also <a href="/wiki/History_of_the_Internet" title="History of the Internet">History of the Internet</a>.)</p>
<p>After 4.3BSD, it was determined that BSD would move away from the aging VAX platform. The <a href="/wiki/Computer_Consoles_Inc.#Power_5_and_Power_6_computers" title="Computer Consoles Inc.">Power 6/32</a> platform (codenamed "Tahoe") developed by <a href="/wiki/Computer_Consoles_Inc." title="Computer Consoles Inc.">Computer Consoles Inc.</a> seemed promising at the time, but was abandoned by its developers shortly thereafter. Nonetheless, the <b>4.3BSD-Tahoe</b> port (June 1988) proved valuable, as it led to a separation of machine-dependent and machine-independent code in BSD which would improve the system's future portability.</p>
<p>Apart from portability, the CSRG worked on an implementation of the <a class="mw-redirect" href="/wiki/Open_Systems_Interconnect" title="Open Systems Interconnect">OSI</a> network protocol stack, improvements to the kernel virtual memory system and (with <a href="/wiki/Van_Jacobson" title="Van Jacobson">Van Jacobson</a> of <a class="mw-redirect" href="/wiki/Lawrence_Berkeley_Laboratory" title="Lawrence Berkeley Laboratory">LBL</a>) new TCP/IP algorithms to accommodate the growth of the Internet.<sup class="reference" id="cite_ref-beyond43_9-0"><a href="#cite_note-beyond43-9">[9]</a></sup></p>
<p>Until then, all versions of BSD incorporated proprietary AT&amp;T Unix code and were, therefore, subject to an AT&amp;T software license. Source code licenses had become very expensive and several outside parties had expressed interest in a separate release of the networking code, which had been developed entirely outside AT&amp;T and would not be subject to the licensing requirement. This led to <b>Networking Release 1</b> (<b>Net/1</b>), which was made available to non-licensees of AT&amp;T code and was <a href="/wiki/Free_software" title="Free software">freely redistributable</a> under the terms of the <a class="mw-redirect" href="/wiki/BSD_license" title="BSD license">BSD license</a>. It was released in June 1989.</p>
<p><b>4.3BSD-Reno</b> came in early 1990. It was an interim release during the early development of 4.4BSD, and its use was considered a "gamble", hence the naming after the gambling center of <a href="/wiki/Reno,_Nevada" title="Reno, Nevada">Reno, Nevada</a>. This release was explicitly moving towards <a href="/wiki/POSIX" title="POSIX">POSIX</a> compliance,<sup class="reference" id="cite_ref-beyond43_9-1"><a href="#cite_note-beyond43-9">[9]</a></sup> and, according to some, away from the BSD philosophy (as POSIX is very much based on System V, and Reno was quite bloated compared to previous releases<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (February 2016)">citation needed</span></a></i>]</sup>). Among the new features were an <a href="/wiki/Network_File_System" title="Network File System">NFS</a> implementation from the <a href="/wiki/University_of_Guelph" title="University of Guelph">University of Guelph</a> and support for the <a href="/wiki/HP_9000" title="HP 9000">HP 9000</a> range of computers, originating in the <a href="/wiki/University_of_Utah" title="University of Utah">University of Utah</a>'s "HPBSD" port.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></p>
<p>In August 2006, <i><a href="/wiki/InformationWeek" title="InformationWeek">InformationWeek</a></i> magazine rated 4.3BSD as the "Greatest Software Ever Written".<sup class="reference" id="cite_ref-iw_11-0"><a href="#cite_note-iw-11">[11]</a></sup> They commented: "BSD 4.3 represents the single biggest theoretical undergirder of the Internet."</p>
<h3><span class="mw-headline" id="Net.2F2_and_legal_troubles">Net/2 and legal troubles</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=9" title="Edit section: Net/2 and legal troubles">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:386BSD_installer_screenshot.png"><img alt="Black and white 386BSD installer screenshot" class="thumbimage" data-file-height="400" data-file-width="720" height="122" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/e6/386BSD_installer_screenshot.png/220px-386BSD_installer_screenshot.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/e6/386BSD_installer_screenshot.png/330px-386BSD_installer_screenshot.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/e6/386BSD_installer_screenshot.png/440px-386BSD_installer_screenshot.png 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:386BSD_installer_screenshot.png" title="Enlarge"></a></div>
Installation of <a href="/wiki/386BSD" title="386BSD">386BSD</a>. 386BSD was an early port of BSD to the <a href="/wiki/Intel_80386" title="Intel 80386">Intel 80386</a> architecture.</div>
</div>
</div>
<p>After Net/1, BSD developer <a href="/wiki/Keith_Bostic" title="Keith Bostic">Keith Bostic</a> proposed that more non-AT&amp;T sections of the BSD system be released under the same license as Net/1. To this end, he started a project to reimplement most of the standard Unix utilities without using the AT&amp;T code. For example, <a href="/wiki/Vi" title="Vi">vi</a>, which had been based on the original Unix version of <a class="mw-redirect" href="/wiki/Ed_(Unix)" title="Ed (Unix)">ed</a>, was rewritten as <a href="/wiki/Nvi" title="Nvi">nvi</a> (new vi). Within eighteen months, all of the AT&amp;T utilities had been replaced, and it was determined that only a few AT&amp;T files remained in the kernel. These files were removed, and the result was the June 1991 release of Networking Release 2 (Net/2), a nearly complete operating system that was freely distributable.</p>
<p>Net/2 was the basis for two separate ports of BSD to the <a href="/wiki/Intel_80386" title="Intel 80386">Intel 80386</a> architecture: the free <a href="/wiki/386BSD" title="386BSD">386BSD</a> by <a href="/wiki/William_Jolitz" title="William Jolitz">William Jolitz</a> and the <a href="/wiki/Proprietary_software" title="Proprietary software">proprietary</a> <a href="/wiki/BSD/OS" title="BSD/OS">BSD/386</a> (later renamed BSD/OS) by <a href="/wiki/Berkeley_Software_Design" title="Berkeley Software Design">Berkeley Software Design</a> (BSDi). 386BSD itself was short-lived, but became the initial code base of the <a href="/wiki/NetBSD" title="NetBSD">NetBSD</a> and <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a> projects that were started shortly thereafter.</p>
<p>BSDi soon found itself in legal trouble with AT&amp;T's <a href="/wiki/Unix_System_Laboratories" title="Unix System Laboratories">Unix System Laboratories</a> (USL) subsidiary, then the owners of the System V <a href="/wiki/Copyright" title="Copyright">copyright</a> and the Unix trademark. The <i><a class="mw-redirect" href="/wiki/USL_v._BSDi" title="USL v. BSDi">USL v. BSDi</a></i> lawsuit was filed in 1992 and led to an <a href="/wiki/Injunction" title="Injunction">injunction</a> on the distribution of Net/2 until the validity of USL's copyright claims on the source could be determined.</p>
<p>The lawsuit slowed development of the free-software descendants of BSD for nearly two years while their legal status was in question, and as a result systems based on the <a href="/wiki/Linux_kernel" title="Linux kernel">Linux kernel</a>, which did not have such legal ambiguity, gained greater support. Although not released until 1992, development of <a href="/wiki/386BSD" title="386BSD">386BSD</a> predated that of Linux. <a href="/wiki/Linus_Torvalds" title="Linus Torvalds">Linus Torvalds</a> has said that if 386BSD or the <a class="mw-redirect" href="/wiki/GNU_kernel" title="GNU kernel">GNU kernel</a> had been available at the time, he probably would not have created Linux.<sup class="reference" id="cite_ref-meta_12-0"><a href="#cite_note-meta-12">[12]</a></sup><sup class="reference" id="cite_ref-TorvaldsPost3_13-0"><a href="#cite_note-TorvaldsPost3-13">[13]</a></sup></p>
<h3><span class="mw-headline" id="4.4BSD_and_descendants">4.4BSD and descendants</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=10" title="Edit section: 4.4BSD and descendants">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">See also: <a href="/wiki/Comparison_of_BSD_operating_systems" title="Comparison of BSD operating systems">Comparison of BSD operating systems</a></div>
<p>The lawsuit was settled in January 1994, largely in Berkeley's favor. Of the 18,000 files in the Berkeley distribution, only three had to be removed and 70 modified to show USL copyright notices. A further condition of the settlement was that USL would not file further lawsuits against users and distributors of the Berkeley-owned code in the upcoming 4.4BSD release. Marshall Kirk McKusick summarizes the lawsuit and its outcome:<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup></p>
<blockquote class="templatequote">
<p>Code copying and theft of trade secrets was alleged. The actual infringing code was not identified for nearly two years. The lawsuit could have dragged on for much longer but for the fact that <a href="/wiki/Novell" title="Novell">Novell</a> bought USL from AT&amp;T and sought a settlement. In the end, three files were removed from the 18,000 that made up the distribution, and a number of minor changes were made to other files. In addition, the University agreed to add USL copyrights to about 70 files, with the stipulation that those files continued to be freely redistributed.</p>
</blockquote>
<p>In June 1994, 4.4BSD was released in two forms: the freely distributable <b>4.4BSD-Lite</b> contained no AT&amp;T source, whereas <b>4.4BSD-Encumbered</b> was available, as earlier releases had been, only to AT&amp;T licensees.</p>
<p>The final release from Berkeley was 1995's <b>4.4BSD-Lite Release 2</b>, after which the CSRG was dissolved and development of BSD at Berkeley ceased. Since then, several variants based directly or indirectly on 4.4BSD-Lite (such as <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a>, <a href="/wiki/NetBSD" title="NetBSD">NetBSD</a>, <a href="/wiki/OpenBSD" title="OpenBSD">OpenBSD</a> and <a href="/wiki/DragonFly_BSD" title="DragonFly BSD">DragonFly BSD</a>) have been maintained.</p>
<p>In addition, the permissive nature of the BSD license has allowed many other operating systems, both free and proprietary, to incorporate BSD code. For example, <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> has used BSD-derived code in its implementation of TCP/IP<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> and bundles recompiled versions of BSD's <a href="/wiki/Command-line_interface" title="Command-line interface">command-line</a> networking tools since <a href="/wiki/Windows_2000" title="Windows 2000">Windows 2000</a>.<sup class="reference" id="cite_ref-windows_16-0"><a href="#cite_note-windows-16">[16]</a></sup> Also <a href="/wiki/Darwin_(operating_system)" title="Darwin (operating system)">Darwin</a>, the system on which Apple's <a href="/wiki/MacOS" title="MacOS">macOS</a> is built, is a derivative of 4.4BSD-Lite2 and FreeBSD. Various commercial Unix operating systems, such as <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">Solaris</a>, also contain varying amounts of BSD code.</p>
<h2><span class="mw-headline" id="Relationship_to_Research_Unix">Relationship to Research Unix</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=11" title="Edit section: Relationship to Research Unix">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Starting with the 8th Edition, versions of Research Unix at Bell Labs had a close relationship to BSD. This began when 4.1cBSD for the VAX was used as the basis for Research Unix 8th Edition. This continued in subsequent versions, such as the 9th Edition, which incorporated source code and improvements from 4.3BSD. The result was that these later versions of Research Unix were closer to BSD than they were to System V. In a <a href="/wiki/Usenet" title="Usenet">Usenet</a> posting from 2000, Dennis Ritchie described this relationship between BSD and Research Unix:<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<blockquote class="templatequote">
<p>Research Unix 8th Edition started from (I think) BSD 4.1c, but with enormous amounts scooped out and replaced by our own stuff. This continued with 9th and 10th. The ordinary user command-set was, I guess, a bit more BSD-flavored than SysVish, but it was pretty eclectic.</p>
</blockquote>
<h2><span class="mw-headline" id="Relationship_to_System_V">Relationship to System V</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=12" title="Edit section: Relationship to System V">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/Eric_S._Raymond" title="Eric S. Raymond">Eric S. Raymond</a> summarizes the longstanding relationship between <a class="mw-redirect" href="/wiki/System_V" title="System V">System V</a> and BSD, stating, "The divide was roughly between longhairs and shorthairs; <a href="/wiki/Programmer" title="Programmer">programmers</a> and technical people tended to line up with Berkeley and BSD, more business-oriented types with AT&amp;T and System V."<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup></p>
<p>In 1989, David A. Curry wrote about the differences between BSD and System V. He characterized System V as being often regarded as the "standard Unix." However, he described BSD as more popular among university and government computer centers, due to its advanced features and performance:<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup></p>
<blockquote class="templatequote">
<p>Most university and government computer centers that use UNIX use Berkeley UNIX, rather than System V. There are several reasons for this, but perhaps the two most significant are that Berkeley UNIX provides networking capabilities that until recently (Release 3.0) were completely unavailable in System V, and that Berkeley UNIX is much more suited to a research environment, which requires a faster file system, better virtual memory handling, and a larger variety of <a href="/wiki/Programming_language" title="Programming language">programming languages</a>.</p>
</blockquote>

<p>Berkeley's Unix was the first Unix to include libraries supporting the <a href="/wiki/Internet_Protocol" title="Internet Protocol">Internet Protocol</a> stacks: <i><a href="/wiki/Berkeley_sockets" title="Berkeley sockets">Berkeley sockets</a></i>. A Unix implementation of IP's predecessor, the ARPAnet's <a href="/wiki/Network_Control_Program" title="Network Control Program">NCP</a>, with <a href="/wiki/File_Transfer_Protocol" title="File Transfer Protocol">FTP</a> and <a href="/wiki/Telnet" title="Telnet">Telnet</a> clients, had been produced at <a class="mw-redirect" href="/wiki/University_of_Illinois" title="University of Illinois">U. Illinois</a> in 1975, and was available at Berkeley.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup><sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> However, the memory scarcity on the PDP-11 forced a complicated design and performance problems.<sup class="reference" id="cite_ref-quarterman42bsd_7-2"><a href="#cite_note-quarterman42bsd-7">[7]</a></sup></p>
<p>By integrating sockets with the Unix operating system's <a href="/wiki/File_descriptor" title="File descriptor">file descriptors</a>, it became almost as easy to read and write data across a <a href="/wiki/Computer_network" title="Computer network">network</a> as it was to access a disk. The AT&amp;T laboratory eventually released their own <a href="/wiki/STREAMS" title="STREAMS">STREAMS</a> library, which incorporated much of the same functionality in a software stack with a different architecture, but the wide distribution of the existing sockets library reduced the impact of the new <a href="/wiki/Application_programming_interface" title="Application programming interface">API</a>. Early versions of BSD were used to form <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a>' <a href="/wiki/SunOS" title="SunOS">SunOS</a>, founding the first wave of popular Unix workstations.</p>
<h3><span class="mw-headline" id="Binary_compatibility">Binary compatibility</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=15" title="Edit section: Binary compatibility">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>BSD operating systems can run much native software of several other operating systems on the same <a href="/wiki/Computer_architecture" title="Computer architecture">architecture</a>, using a binary <a href="/wiki/Compatibility_layer" title="Compatibility layer">compatibility layer</a>. Much simpler and faster than <a href="/wiki/Emulator" title="Emulator">emulation</a>, this allows, for instance, applications intended for <a href="/wiki/Linux" title="Linux">Linux</a> to be run at effectively full speed. This makes BSDs not only suitable for server environments, but also for workstation ones, given the increasing availability of commercial or closed-source software for Linux only. This also allows administrators to migrate legacy commercial applications, which may have only supported commercial Unix variants, to a more modern operating system, retaining the functionality of such applications until they can be replaced by a better alternative.</p>
<h3><span class="mw-headline" id="Standards_adherence">Standards adherence</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=16" title="Edit section: Standards adherence">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Current BSD operating system variants support many of the common <a href="/wiki/Institute_of_Electrical_and_Electronics_Engineers" title="Institute of Electrical and Electronics Engineers">IEEE</a>, <a href="/wiki/American_National_Standards_Institute" title="American National Standards Institute">ANSI</a>, <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a>, and <a href="/wiki/POSIX" title="POSIX">POSIX</a> standards, while retaining most of the traditional BSD behavior. Like <a class="mw-redirect" href="/wiki/AT%26T_Unix" title="AT&amp;T Unix">AT&amp;T Unix</a>, the BSD kernel is <a href="/wiki/Monolithic_kernel" title="Monolithic kernel">monolithic</a>, meaning that device drivers in the kernel run in <a class="mw-redirect" href="/wiki/Privileged_mode" title="Privileged mode">privileged mode</a>, as part of the core of the operating system.</p>
<h2><span class="mw-headline" id="Significant_BSD_descendants">Significant BSD descendants</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Berkeley_Software_Distribution&amp;action=edit&amp;section=17" title="Edit section: Significant BSD descendants">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

<p>BSD has been the base of a large number of operating systems. Most notable among these today are perhaps the major <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> BSDs: FreeBSD, NetBSD and OpenBSD, which are all derived from <a href="/wiki/386BSD" title="386BSD">386BSD</a> and <a class="mw-redirect" href="/wiki/4.4BSD" title="4.4BSD">4.4BSD</a>-Lite by various routes. Both NetBSD and FreeBSD started life in 1993, initially derived from 386BSD, but in 1994 migrating to a 4.4BSD-Lite code base. OpenBSD was <a href="/wiki/Fork_(software_development)" title="Fork (software development)">forked</a> in 1995 from NetBSD. The three most notable descendants in current use—sometimes known as <i>the BSDs</i>—have themselves spawned a number of children, including <a href="/wiki/DragonFly_BSD" title="DragonFly BSD">DragonFly BSD</a>, <a href="/wiki/FreeSBIE" title="FreeSBIE">FreeSBIE</a>, <a href="/wiki/MirOS_BSD" title="MirOS BSD">MirOS BSD</a>, <a href="/wiki/DesktopBSD" title="DesktopBSD">DesktopBSD</a>, and <a class="mw-redirect" href="/wiki/PC-BSD" title="PC-BSD">PC-BSD</a>. They are targeted at an array of systems for different purposes and are common in government facilities, universities and in commercial use. A number of commercial operating systems are also partly or wholly based on BSD or its descendants, including <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun's</a> <a href="/wiki/SunOS" title="SunOS">SunOS</a> and <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple Inc.</a>'s <a href="/wiki/MacOS" title="MacOS">macOS</a>.</p>
<p>Most of the current BSD operating systems are <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> and available for download, free of charge, under the <a class="mw-redirect" href="/wiki/BSD_License" title="BSD License">BSD License</a>, the most notable exception being <a href="/wiki/MacOS" title="MacOS">macOS</a>. They also generally use a <a href="/wiki/Monolithic_kernel" title="Monolithic kernel">monolithic kernel</a> architecture, apart from macOS and DragonFly BSD which feature <a href="/wiki/Hybrid_kernel" title="Hybrid kernel">hybrid kernels</a>. The various open source BSD projects generally develop the kernel and <a href="/wiki/User_space" title="User space">userland</a> programs and libraries together, the source code being managed using a single central source repository.</p>
<p>In the past, BSD was also used as a basis for several proprietary versions of Unix, such as <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun</a>'s <a href="/wiki/SunOS" title="SunOS">SunOS</a>, <a href="/wiki/Sequent_Computer_Systems" title="Sequent Computer Systems">Sequent</a>'s <a class="mw-redirect" href="/wiki/Dynix" title="Dynix">Dynix</a>, <a href="/wiki/NeXT" title="NeXT">NeXT</a>'s <a href="/wiki/NeXTSTEP" title="NeXTSTEP">NeXTSTEP</a>, <a href="/wiki/Digital_Equipment_Corporation" title="Digital Equipment Corporation">DEC</a>'s <a href="/wiki/Ultrix" title="Ultrix">Ultrix</a> and OSF/1 AXP (now <a href="/wiki/Tru64_UNIX" title="Tru64 UNIX">Tru64 UNIX</a>). Parts of NeXT's software became the foundation for <a href="/wiki/MacOS" title="MacOS">macOS</a>, among the most commercially successful BSD variants in the general market.</p>
