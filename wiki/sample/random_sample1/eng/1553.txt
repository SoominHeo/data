<p><b>Things</b> is a task management app for <a href="/wiki/MacOS" title="MacOS">macOS</a>, <a href="/wiki/IOS" title="IOS">iOS</a>, and <a href="/wiki/WatchOS" title="WatchOS">watchOS</a> made by Cultured Code, a software startup based in <a href="/wiki/Stuttgart" title="Stuttgart">Stuttgart</a>, <a href="/wiki/Germany" title="Germany">Germany</a>. It first released for <a href="/wiki/Macintosh" title="Macintosh">Mac</a> as an alpha that went out in late 2007 to 12,000 people<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> and quickly gained popularity. The following July, when the <a class="mw-redirect" href="/wiki/App_Store" title="App Store">App Store</a> launched, it was among the first 552 apps available for <a href="/wiki/IPhone" title="IPhone">iPhone</a>.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> It was then released alongside the <a href="/wiki/IPad" title="IPad">iPad</a> in April 2010,<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> and became one of the first apps available for <a href="/wiki/Apple_Watch" title="Apple Watch">Apple Watch</a> in March 2015.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p>In December 2013, Cultured Code announced that they had sold one million copies of the software to date,<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> and in December 2014 the company announced that downloads had increased by an additional three million.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>

<h2><span class="mw-headline" id="Awards">Awards</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Things_(application)&amp;action=edit&amp;section=1" title="Edit section: Awards">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Things has won multiple awards over the years. It first won the <a href="/wiki/MacLife" title="MacLife">MacLife</a> Editors' Choice Award in 2008, and then in 2009 it went on to win the <a class="mw-redirect" href="/wiki/Apple_Design_Award" title="Apple Design Award">Apple Design Award</a>,<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> the <a href="/wiki/Macworld" title="Macworld">Macworld</a> Editors’ Choice Award,<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> and the Macworld Best of Show Award.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> In 2012, after the release of Things 2, <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple</a> selected it as Editors’ Choice, named it among the App Store Best of 2012, and inducted it into the App Store Hall of Fame.</p>
<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Things_(application)&amp;action=edit&amp;section=2" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Main_features">Main features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Things_(application)&amp;action=edit&amp;section=3" title="Edit section: Main features">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Things allows to-dos to be subdivided into several sections, which correspond to parts of the <a href="/wiki/Getting_Things_Done" title="Getting Things Done">Getting Things Done</a> methodology:</p>
<h4><span class="mw-headline" id="Collect">Collect</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Things_(application)&amp;action=edit&amp;section=4" title="Edit section: Collect">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li><b>Inbox</b> is used to temporarily collect to-dos which have not been filed in a specific place yet.</li>
</ul>
<h4><span class="mw-headline" id="Focus">Focus</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Things_(application)&amp;action=edit&amp;section=5" title="Edit section: Focus">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li><b>Today</b> highlights to-dos which are due, or scheduled to begin, on the current day; they are the user’s priorities.</li>
<li><b>Next</b> is an overview of upcoming to-dos across all of the user’s projects and areas.</li>
<li><b>Scheduled</b> contains to-dos and projects that have been postponed to a specific date, as well as to-dos that are automatically generated based on repeating patterns chosen by the user.</li>
<li><b>Someday</b> is used to store to-dos which need to be done, but are not time critical.</li>
</ul>
<h4><span class="mw-headline" id="Organize">Organize</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Things_(application)&amp;action=edit&amp;section=6" title="Edit section: Organize">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li><b>Projects</b> are collections of to-dos that contribute to the completion of a larger goal (e.g., "Plan Holiday"). Once the project is finished, the user marks it complete and it moves to the Logbook with all the to-dos it contains.</li>
<li><b>Areas</b> can be used to organize projects and to-dos which correspond to the same, ongoing theme (e.g., "Work"). Unlike projects, areas are perpetual, do not have a checkbox, and are never completed.</li>
<li><b>Logbook</b> is where projects and to-dos are stored for future reference after they’ve been completed.</li>
</ul>
<h3><span class="mw-headline" id="Additional_features">Additional features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Things_(application)&amp;action=edit&amp;section=7" title="Edit section: Additional features">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li><b>Quick Entry</b> is an extension on the Mac that allows the user to create a new to-do while working in other programs. Activated by a global <a href="/wiki/Keyboard_shortcut" title="Keyboard shortcut">keyboard shortcut</a>, it invokes a small pop-up window which automatically includes links to files or websites, and any text that was selected.</li>
<li><b>Add to Things</b> is an extension on iPhone and iPad that, like Quick Entry, allows the user to send to-dos to Things from other apps. When saved, the new to-do is sent to Things’ inbox in the background.</li>
<li><b>Siri integration</b> allows the user to speak to-dos to their iPhone, Apple Watch, or iPad, and have them automatically appear in Things’ inbox for later review.</li>
<li><b>Repeating Tasks</b> are automatically generated by the app based on flexible recurrence rules, such as the last day of every month, every other Thursday, or two weeks after the last one was completed, etc. – whatever the user chooses.</li>
<li><b>Today Widget</b> allows the user to quickly access their Today list from the Notification Center, either by pulling it down from the top of the screen on iPhone or iPad, or in from the right side of the screen on Mac.</li>
<li><b>Tags</b> allow the user to further describe to-dos using the popular <a href="/wiki/Tag_(metadata)" title="Tag (metadata)">tagging</a> organisation paradigm; lists can then be filtered by these tags to search for and focus on specific types of tasks.</li>
<li><b>Global Search</b> allows the user to search through all their to-dos across projects, areas, and the logbook.</li>
</ul>
<h3><span class="mw-headline" id="Things_Cloud">Things Cloud</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Things_(application)&amp;action=edit&amp;section=8" title="Edit section: Things Cloud">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a class="external text" href="http://culturedcode.com/things/cloud/" rel="nofollow">Things Cloud</a> is a sync service that stores the user’s to-dos and automatically keeps them updated across all their <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple</a> devices. It was developed to replace the app’s original sync technology, which only worked over a local network. The company began alpha testing the new service in 2011 and, after an extended beta period, launched it publicly on August 9, 2012 with the release of Things 2.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> A user can create a free Things Cloud account from within the apps’ settings.</p>
<p>In 2015, the company announced "Nimbus"<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> – an update to Things Cloud’s architecture which introduced push sync through the cloud (in addition to the push sync it already had across the local network<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup>). The main benefit of the new push technology is that it delivers sync to iOS devices when the app isn’t actively running, by utilizing <a href="/wiki/Apple_Push_Notification_Service" title="Apple Push Notification Service">APNs</a>.</p>
