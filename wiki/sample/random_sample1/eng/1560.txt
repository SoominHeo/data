<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><b>TwinVQ</b> (transform-domain weighted interleave <a href="/wiki/Vector_quantization" title="Vector quantization">vector quantization</a>) is an <a class="mw-redirect" href="/wiki/Audio_data_compression" title="Audio data compression">audio compression</a> technique developed by <a class="mw-redirect" href="/wiki/Nippon_Telegraph_and_Telephone_Corporation" title="Nippon Telegraph and Telephone Corporation">Nippon Telegraph and Telephone Corporation</a> (NTT) Human Interface Laboratories (now Cyber Space Laboratories) in 1994.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> The compression technique has been used in both standardized and proprietary designs.</p>

<h2><span class="mw-headline" id="TwinVQ_in_MPEG-4">TwinVQ in MPEG-4</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=TwinVQ&amp;action=edit&amp;section=1" title="Edit section: TwinVQ in MPEG-4">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In the context of the <a href="/wiki/MPEG-4" title="MPEG-4">MPEG-4</a> Audio (<a href="/wiki/MPEG-4_Part_3" title="MPEG-4 Part 3">MPEG-4 Part 3</a>), TwinVQ is an audio <a href="/wiki/Codec" title="Codec">codec</a> optimized for audio coding at ultra low bitrates around 8 kbit/s.</p>
<p>TwinVQ is one of the object types defined in MPEG-4 Audio, published as subpart 4 of <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a>/<a href="/wiki/International_Electrotechnical_Commission" title="International Electrotechnical Commission">IEC</a> 14496-3 (for the first time in 1999 - a.k.a. MPEG-4 Audio version 1).<sup class="reference" id="cite_ref-mpeg4audio_5-0"><a href="#cite_note-mpeg4audio-5">[5]</a></sup><sup class="reference" id="cite_ref-mpeg-audio-faq-introduction_6-0"><a href="#cite_note-mpeg-audio-faq-introduction-6">[6]</a></sup><sup class="reference" id="cite_ref-mpeg4audio-version2_7-0"><a href="#cite_note-mpeg4audio-version2-7">[7]</a></sup><sup class="reference" id="cite_ref-mpeg4audio1_8-0"><a href="#cite_note-mpeg4audio1-8">[8]</a></sup><sup class="reference" id="cite_ref-mpeg4audio1-draft_9-0"><a href="#cite_note-mpeg4audio1-draft-9">[9]</a></sup> This object type is based on a general audio transform coding scheme which is integrated with the AAC coding frame work, a spectral flattening module, and a weighted interleave <a href="/wiki/Vector_quantization" title="Vector quantization">vector quantization</a> module. This scheme reportedly has high coding gain for low bit rate and potential robustness against channel errors and packet loss, since it does not use any variable length coding and adaptive bit allocation. It supports bitrate scalability, both by means of layered TwinVQ coding and in combination with the scalable AAC.</p>
<p>Note that some commercialized products such as Metasound (Voxware),<sup class="reference" id="cite_ref-metasound-voxware_10-0"><a href="#cite_note-metasound-voxware-10">[10]</a></sup><sup class="reference" id="cite_ref-metasound-voxware-2_11-0"><a href="#cite_note-metasound-voxware-2-11">[11]</a></sup> SoundVQ (Yamaha),<sup class="reference" id="cite_ref-yamaha-soundvq_12-0"><a href="#cite_note-yamaha-soundvq-12">[12]</a></sup><sup class="reference" id="cite_ref-yamaha_13-0"><a href="#cite_note-yamaha-13">[13]</a></sup><sup class="reference" id="cite_ref-ntt-twinvq_14-0"><a href="#cite_note-ntt-twinvq-14">[14]</a></sup> and SolidAudio (Hagiwara) are also based on the TwinVQ technology, but the configurations are different from the MPEG-4 TwinVQ.<sup class="reference" id="cite_ref-mpeg-audio-faq-introduction_6-1"><a href="#cite_note-mpeg-audio-faq-introduction-6">[6]</a></sup></p>
<h2><span class="mw-headline" id="TwinVQ_as_a_proprietary_audio_format">TwinVQ as a proprietary audio format</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=TwinVQ&amp;action=edit&amp;section=2" title="Edit section: TwinVQ as a proprietary audio format">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A <a href="/wiki/Proprietary_software" title="Proprietary software">proprietary</a> audio <a class="mw-redirect" href="/wiki/Audio_data_compression" title="Audio data compression">compression</a> format called TwinVQ was developed by <a class="mw-redirect" href="/wiki/Nippon_Telegraph_and_Telephone_Corporation" title="Nippon Telegraph and Telephone Corporation">Nippon Telegraph and Telephone Corporation</a> (NTT) (in NTT's Human Interface Laboratories)<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup><sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> and marketed by <a href="/wiki/Yamaha_Corporation" title="Yamaha Corporation">Yamaha</a> under the name SoundVQ.<sup class="reference" id="cite_ref-yamaha_13-1"><a href="#cite_note-yamaha-13">[13]</a></sup> The NTT also offered a TwinVQ demonstration software for non-commercial purposes - NTT TwinVQ Encoder and TwinVQ Player, encoder <a class="mw-redirect" href="/wiki/API" title="API">API</a>, decoder <a class="mw-redirect" href="/wiki/API" title="API">API</a> and header file format.<sup class="reference" id="cite_ref-twinvq-software-download_17-0"><a href="#cite_note-twinvq-software-download-17">[17]</a></sup><sup class="reference" id="cite_ref-twinvq-libraries_18-0"><a href="#cite_note-twinvq-libraries-18">[18]</a></sup> The <a href="/wiki/Filename_extension" title="Filename extension">filename extension</a> is .vqf.</p>
<p>TwinVQ uses <a href="/wiki/Twin_vector_quantization" title="Twin vector quantization">Twin vector quantization</a>. The proprietary TwinVQ codec supports <a class="mw-redirect" href="/wiki/Constant_bit_rate" title="Constant bit rate">constant bit rate</a> <a href="/wiki/Code" title="Code">encoding</a> at 80, 96, 112, 128, 160 and 192 kbit/s. It was claimed that TwinVQ files are about 30 to 35% smaller than <a href="/wiki/MP3" title="MP3">MP3</a> files of adequate quality. For example, a 96 kbit/s TwinVQ file allegedly has roughly the same quality as a 128 kbit/s MP3 file. The higher quality is achieved at the cost of higher processor usage.</p>
<p>Yamaha marketed TwinVQ as an alternative to MP3, but the format never became very popular. This could be attributed to the proprietary nature of the format—third party software was scarce and there was no hardware support. Also the encoding was extremely slow and there was not much music available in TwinVQ format. As other MP3 alternatives emerged, TwinVQ quickly became obsolete.</p>
<p>The proprietary version of TwinVQ can be also used for <a class="mw-redirect" href="/wiki/Speech_encoding" title="Speech encoding">speech encoding</a>. Compression technology specifically designed to handle voice compression was published by NTT. The NTT TwinVQ implementation supported <a class="mw-redirect" href="/wiki/Sampling_frequency" title="Sampling frequency">sampling frequencies</a> from 8 kHz or 11.025 kHz and <a href="/wiki/Bit_rate" title="Bit rate">bit rate</a> from 8 kbit/s.<sup class="reference" id="cite_ref-ntt-twinvq_14-1"><a href="#cite_note-ntt-twinvq-14">[14]</a></sup><sup class="reference" id="cite_ref-ntt-twinvq-faq_19-0"><a href="#cite_note-ntt-twinvq-faq-19">[19]</a></sup><sup class="reference" id="cite_ref-twinvq-music_20-0"><a href="#cite_note-twinvq-music-20">[20]</a></sup><sup class="reference" id="cite_ref-vqf-multimediawiki_21-0"><a href="#cite_note-vqf-multimediawiki-21">[21]</a></sup><sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup></p>
<h2><span class="mw-headline" id="Software_support">Software support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=TwinVQ&amp;action=edit&amp;section=3" title="Edit section: Software support">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Official">Official</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=TwinVQ&amp;action=edit&amp;section=4" title="Edit section: Official">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>NTT in Japan is said to have once offered (on its website) a player/encoder for download (citation needed). This was not as successful as the Yamaha version (see below) and, nowadays, is almost impossible to find (citation needed).</p>
<p>Yamaha released an English player application called <b>SoundVQ</b>. Several third party players also supported the format including WinAmp (with the appropriate input plugin) and K-Jöfol (which supported the format natively).</p>
<h3><span class="mw-headline" id="Third-Party_Software">Third-Party Software</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=TwinVQ&amp;action=edit&amp;section=5" title="Edit section: Third-Party Software">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The format was reverse-engineered in 2009 by the <a href="/wiki/FFmpeg" title="FFmpeg">FFmpeg</a> project and decoding of vqf files is supported by the open-source <a href="/wiki/Libavcodec" title="Libavcodec">libavcodec</a> library,<sup class="reference" id="cite_ref-ffdev20080608_23-0"><a href="#cite_note-ffdev20080608-23">[23]</a></sup> which makes it supported in players that utilize the library, such as <a href="/wiki/VLC_media_player" title="VLC media player">VLC media player</a>.</p>
<p>Some older versions of <a href="/wiki/Nero_Burning_ROM" title="Nero Burning ROM">Nero Burning ROM</a> are able to encode to TwinVQ/VQF.</p>
<p><br/>
Some CD-Ripping/Converter software also support encoding to .vqf format.</p>
