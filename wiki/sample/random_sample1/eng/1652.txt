<p><b>WebGL</b> (<b>Web Graphics Library</b>) is a <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> <a href="/wiki/Application_programming_interface" title="Application programming interface">API</a> for rendering 3D graphics within any compatible <a href="/wiki/Web_browser" title="Web browser">web browser</a> without the use of <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plug-ins</a>.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> WebGL is integrated completely into all the web standards of the browser allowing GPU accelerated usage of physics and image processing and effects as part of the web page canvas. WebGL elements can be mixed with other HTML elements and composited with other parts of the page or page background.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> WebGL programs consist of control code written in JavaScript and <a href="/wiki/Shader" title="Shader">shader</a> code that is written in <a href="/wiki/OpenGL_Shading_Language" title="OpenGL Shading Language">GLSL</a>, a language similar to <a href="/wiki/C_(programming_language)" title="C (programming language)">C or C++</a>, and is executed on a computer's <a class="mw-redirect" href="/wiki/Graphics_Processing_Unit" title="Graphics Processing Unit">Graphics Processing Unit</a> (GPU). WebGL is designed and maintained by the <a class="mw-redirect" href="/wiki/Non-profit_organization" title="Non-profit organization">non-profit</a> <a href="/wiki/Khronos_Group" title="Khronos Group">Khronos Group</a>.<sup class="reference" id="cite_ref-WebGLWebsite_4-0"><a href="#cite_note-WebGLWebsite-4">[4]</a></sup></p>

<h2><span class="mw-headline" id="Design">Design</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=WebGL&amp;action=edit&amp;section=1" title="Edit section: Design">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>WebGL 1.0 is based on <a href="/wiki/OpenGL_ES" title="OpenGL ES">OpenGL ES 2.0</a> and provides an <a class="mw-redirect" href="/wiki/API" title="API">API</a> for 3D graphics.<sup class="reference" id="cite_ref-WebGLSpecification_5-0"><a href="#cite_note-WebGLSpecification-5">[5]</a></sup> It uses the <a href="/wiki/HTML5" title="HTML5">HTML5</a> <a href="/wiki/Canvas_element" title="Canvas element">canvas element</a> and is accessed using <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a> interfaces. <a href="/wiki/Garbage_collection_(computer_science)" title="Garbage collection (computer science)">Automatic memory management</a> is provided as part of the <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> language.<sup class="reference" id="cite_ref-WebGLWebsite_4-1"><a href="#cite_note-WebGLWebsite-4">[4]</a></sup></p>
<p>Like OpenGL ES 2.0, WebGL does not have the <a href="/wiki/Fixed-function" title="Fixed-function">fixed-function</a> APIs introduced in OpenGL 1.0 and <a href="/wiki/Deprecation" title="Deprecation">deprecated</a> in OpenGL 3.0. This functionality can instead be provided by the user in the JavaScript code space.</p>
<p>Shaders in WebGL are expressed directly in <a class="mw-redirect" href="/wiki/GLSL" title="GLSL">GLSL</a>.</p>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=WebGL&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>WebGL evolved out of the Canvas 3D experiments started by <a href="/wiki/Vladimir_Vuki%C4%87evi%C4%87" title="Vladimir Vukićević">Vladimir Vukićević</a> at <a href="/wiki/Mozilla_Foundation" title="Mozilla Foundation">Mozilla</a>. Vukićević first demonstrated a Canvas 3D prototype in 2006. By the end of 2007, both Mozilla<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> and Opera<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> had made their own separate implementations.</p>
<p>In early 2009, the <a class="mw-redirect" href="/wiki/Non-profit_organization" title="Non-profit organization">non-profit</a> technology <a href="/wiki/Consortium" title="Consortium">consortium</a> <a href="/wiki/Khronos_Group" title="Khronos Group">Khronos Group</a> started the WebGL Working Group, with initial participation from <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple</a>, <a href="/wiki/Google" title="Google">Google</a>, <a href="/wiki/Mozilla_Foundation" title="Mozilla Foundation">Mozilla</a>, <a href="/wiki/Opera_Software" title="Opera Software">Opera</a>, and others.<sup class="reference" id="cite_ref-WebGLWebsite_4-2"><a href="#cite_note-WebGLWebsite-4">[4]</a></sup><sup class="reference" id="cite_ref-WebGLAnnouncement_8-0"><a href="#cite_note-WebGLAnnouncement-8">[8]</a></sup> Version 1.0 of the WebGL specification was released March 2011.<sup class="reference" id="cite_ref-WebGLFinal_1-1"><a href="#cite_note-WebGLFinal-1">[1]</a></sup> As of March 2012, the chair of the working group is Ken Russell.</p>
<p>Early applications of WebGL include <a class="mw-redirect" href="/wiki/Zygote_Body" title="Zygote Body">Zygote Body</a>.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> In November 2012 <a href="/wiki/Autodesk" title="Autodesk">Autodesk</a> announced that they ported most of their applications to the cloud running on local WebGL clients. These applications included Fusion 360 and AutoCAD 360.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></p>
<p>Development of the WebGL 2 specification started in 2013.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> This specification is based on OpenGL ES 3.0.</p>
<h2><span class="mw-headline" id="Support">Support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=WebGL&amp;action=edit&amp;section=3" title="Edit section: Support">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>WebGL is widely supported in modern browsers. However its availability is dependent on other factors like the GPU supporting it. The official WebGL website offers a simple test page.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> More detailed information (like what renderer the browser uses, and what extensions are available) is provided at third-party websites.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup><sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup></p>
<h3><span class="mw-headline" id="Desktop_browsers">Desktop browsers</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=WebGL&amp;action=edit&amp;section=4" title="Edit section: Desktop browsers">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li><a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a> – WebGL has been enabled on all platforms that have a capable graphics card with updated drivers since version 9, released in February 2011.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup><sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> By default on Windows, Chrome uses the <a href="/wiki/ANGLE_(software)" title="ANGLE (software)">ANGLE</a> (Almost Native Graphics Layer Engine) renderer to translate OpenGL ES to <a class="mw-redirect" href="/wiki/Direct_X" title="Direct X">Direct X</a> 9.0c or 11.0, which have better driver support.<sup class="reference" id="cite_ref-geek3d_18-0"><a href="#cite_note-geek3d-18">[18]</a></sup> On Linux and Mac OS X the default renderer is OpenGL however.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup> It is also possible to force OpenGL as the renderer on Windows.<sup class="reference" id="cite_ref-geek3d_18-1"><a href="#cite_note-geek3d-18">[18]</a></sup> Since September 2013, Chrome also has a newer <a href="/wiki/Direct3D" title="Direct3D">Direct3D</a> 11 renderer, which however requires a newer graphics card.<sup class="reference" id="cite_ref-L5956_20-0"><a href="#cite_note-L5956-20">[20]</a></sup><sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup></li>
<li><a class="mw-redirect" href="/wiki/Mozilla_Firefox" title="Mozilla Firefox">Mozilla Firefox</a> – WebGL has been enabled on all platforms that have a capable graphics card with updated drivers since version 4.0.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup> Since 2013 Firefox also uses <a href="/wiki/DirectX" title="DirectX">DirectX</a> on the Windows platform via <a href="/wiki/ANGLE_(software)" title="ANGLE (software)">ANGLE</a>.<sup class="reference" id="cite_ref-geek3d_18-2"><a href="#cite_note-geek3d-18">[18]</a></sup></li>
<li><a href="/wiki/Safari_(web_browser)" title="Safari (web browser)">Safari</a> – Safari 6.0 and newer versions installed on <a href="/wiki/OS_X_Mountain_Lion" title="OS X Mountain Lion">OS X Mountain Lion</a>, <a href="/wiki/Mac_OS_X_Lion" title="Mac OS X Lion">Mac OS X Lion</a> and Safari 5.1 on <a href="/wiki/Mac_OS_X_Snow_Leopard" title="Mac OS X Snow Leopard">Mac OS X Snow Leopard</a> implemented support for WebGL, which was disabled by default before Safari 8.0.<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup><sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup><sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup><sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup><sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup></li>
<li><a href="/wiki/Opera_(web_browser)" title="Opera (web browser)">Opera</a> – WebGL has been implemented in Opera 11 and 12, although was disabled by default in 2014.<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup><sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup></li>
<li><a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> – WebGL is partially supported in <a href="/wiki/Internet_Explorer_11" title="Internet Explorer 11">Internet Explorer 11</a>.<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup><sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup><sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup><sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup> It initially failed the majority of official WebGL conformance tests, but Microsoft later released several updates. The latest 0.94 WebGL engine currently passes ~97% of Khronos tests.<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup> WebGL support can also be manually added to earlier versions of Internet Explorer using third-party plugins such as IEWebGL.<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup></li>
<li><a href="/wiki/Microsoft_Edge" title="Microsoft Edge">Microsoft Edge</a> – The initial stable release supports WebGL version 0.95 (context name: "experimental-webgl") with an open source GLSL to <a class="mw-redirect" href="/wiki/HLSL" title="HLSL">HLSL</a> transpiler.<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup></li>
<li><a href="/wiki/Vivaldi_(web_browser)" title="Vivaldi (web browser)">Vivaldi</a></li>
</ul>
<h3><span class="mw-headline" id="Mobile_browsers">Mobile browsers</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=WebGL&amp;action=edit&amp;section=5" title="Edit section: Mobile browsers">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li><a href="/wiki/BlackBerry_10" title="BlackBerry 10">BlackBerry 10</a> – WebGL is available for BlackBerry devices since OS version 10.00<sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup></li>
<li><a href="/wiki/BlackBerry_PlayBook" title="BlackBerry PlayBook">BlackBerry PlayBook</a> – WebGL is available via WebWorks and browser in PlayBook OS 2.00<sup class="reference" id="cite_ref-38"><a href="#cite_note-38">[38]</a></sup></li>
<li>Android Browser – Basically unsupported, but the <a class="mw-redirect" href="/wiki/Sony_Ericsson_Xperia" title="Sony Ericsson Xperia">Sony Ericsson Xperia</a> range of Android smartphones have had WebGL capabilities following a firmware upgrade.<sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[39]</a></sup> Samsung smartphones also have WebGL enabled (verified on Galaxy SII (4.1.2) and Galaxy Note 8.0 (4.2)). Supported in Google Chrome that replaced Android browser in many phones (but is not a new standard Android Browser).</li>
<li><a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> – WebGL is available on Windows Phone 8.1</li>
<li><a class="mw-redirect" href="/wiki/Firefox_for_mobile" title="Firefox for mobile">Firefox for mobile</a> – WebGL is available for Android and MeeGo devices since Firefox 4.<sup class="reference" id="cite_ref-40"><a href="#cite_note-40">[40]</a></sup><sup class="reference" id="cite_ref-mobilehtml5.org_41-0"><a href="#cite_note-mobilehtml5.org-41">[41]</a></sup></li>
<li><a href="/wiki/Firefox_OS" title="Firefox OS">Firefox OS</a><sup class="reference" id="cite_ref-mobilehtml5.org_41-1"><a href="#cite_note-mobilehtml5.org-41">[41]</a></sup></li>
<li><a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a> – WebGL is available for Android devices since Google Chrome 25 and enabled by default since version 30.<sup class="reference" id="cite_ref-42"><a href="#cite_note-42">[42]</a></sup></li>
<li><a href="/wiki/Maemo" title="Maemo">Maemo</a> – In <a href="/wiki/Nokia_N900" title="Nokia N900">Nokia N900</a>, WebGL is available in the stock microB browser from the PR1.2 firmware update onwards.<sup class="reference" id="cite_ref-43"><a href="#cite_note-43">[43]</a></sup></li>
<li><a href="/wiki/MeeGo" title="MeeGo">MeeGo</a> – WebGL is unsupported in the stock browser "Web." However, it is available through Firefox.<sup class="reference" id="cite_ref-mobilehtml5.org_41-2"><a href="#cite_note-mobilehtml5.org-41">[41]</a></sup></li>
<li><a href="/wiki/Microsoft_Edge" title="Microsoft Edge">Microsoft Edge</a> – WebGL is available on Windows 10 Mobile.<sup class="reference" id="cite_ref-44"><a href="#cite_note-44">[44]</a></sup></li>
<li><a href="/wiki/Opera_Mobile" title="Opera Mobile">Opera Mobile</a> – Opera Mobile 12 supports WebGL (on Android only).<sup class="reference" id="cite_ref-45"><a href="#cite_note-45">[45]</a></sup></li>
<li><a href="/wiki/Sailfish_OS" title="Sailfish OS">Sailfish OS</a> – WebGL is supported in the default Sailfish browser.<sup class="reference" id="cite_ref-46"><a href="#cite_note-46">[46]</a></sup></li>
<li><a href="/wiki/Tizen" title="Tizen">Tizen</a> – WebGL is supported<sup class="reference" id="cite_ref-47"><a href="#cite_note-47">[47]</a></sup></li>
<li><a href="/wiki/Ubuntu_Touch" title="Ubuntu Touch">Ubuntu Touch</a></li>
<li><a href="/wiki/WebOS" title="WebOS">WebOS</a></li>
<li><a href="/wiki/IOS" title="IOS">iOS</a> – WebGL is available for mobile Safari, in <a href="/wiki/IOS_8" title="IOS 8">iOS 8</a>.<sup class="reference" id="cite_ref-48"><a href="#cite_note-48">[48]</a></sup></li>
</ul>
<h2><span class="mw-headline" id="Content_creation_and_ecosystem">Content creation and ecosystem</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=WebGL&amp;action=edit&amp;section=6" title="Edit section: Content creation and ecosystem">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The WebGL API may be too tedious to use directly without some utility libraries, which for example set up typical view transformation shaders (e.g. for <a class="mw-redirect" href="/wiki/View_frustum" title="View frustum">view frustum</a>). Loading <a href="/wiki/Scene_graph" title="Scene graph">scene graphs</a> and 3D objects in the popular industry formats is also not directly provided for. JavaScript libraries have been built (or sometimes <a href="/wiki/Porting" title="Porting">ported</a> to WebGL) to provide the additional functionality. A non-exhaustive list of libraries that provide many high-level features includes <a href="/wiki/A-Frame_(VR)" title="A-Frame (VR)">A-Frame (VR)</a>, <a class="external text" href="http://www.babylonjs.com/" rel="nofollow">BabylonJS</a>, <a href="/wiki/PlayCanvas" title="PlayCanvas">PlayCanvas</a>, <a href="/wiki/Three.js" title="Three.js">three.js</a>, <a href="/wiki/OSG.JS" title="OSG.JS">OSG.JS</a> and <a href="/wiki/CopperLicht" title="CopperLicht">CopperLicht</a>. There also has been a rapid emergence of <a href="/wiki/Game_engine" title="Game engine">game engines</a> for WebGL,<sup class="reference" id="cite_ref-Parisi2014_49-0"><a href="#cite_note-Parisi2014-49">[49]</a></sup> including <a href="/wiki/Unreal_Engine" title="Unreal Engine">Unreal Engine</a> 4 and <a href="/wiki/Unity_(game_engine)" title="Unity (game engine)">Unity</a> 5.<sup class="reference" id="cite_ref-50"><a href="#cite_note-50">[50]</a></sup> The <a href="/wiki/Stage3D" title="Stage3D">Stage3D</a>/Flash-based <a href="/wiki/Away3D" title="Away3D">Away3D</a> high-level library also has a port to WebGL via <a href="/wiki/TypeScript" title="TypeScript">TypeScript</a>.<sup class="reference" id="cite_ref-L5956_20-1"><a href="#cite_note-L5956-20">[20]</a></sup><sup class="reference" id="cite_ref-51"><a href="#cite_note-51">[51]</a></sup> A more light-weight utility library that provides just the vector and matrix math utilities for shaders is sylvester.js.<sup class="reference" id="cite_ref-BoreskovShikin2013_52-0"><a href="#cite_note-BoreskovShikin2013-52">[52]</a></sup><sup class="reference" id="cite_ref-Anyuru2012_53-0"><a href="#cite_note-Anyuru2012-53">[53]</a></sup> It is sometimes used in conjunction with a WebGL specific extension called glUtils.js.<sup class="reference" id="cite_ref-BoreskovShikin2013_52-1"><a href="#cite_note-BoreskovShikin2013-52">[52]</a></sup><sup class="reference" id="cite_ref-FultonFulton2013_54-0"><a href="#cite_note-FultonFulton2013-54">[54]</a></sup> Although not very common, some 3D libraries, like the one used by <a class="external text" href="http://www.lai4d.org" rel="nofollow">LAI4D</a>, can work with <a class="mw-redirect" href="/wiki/HTML5_canvas" title="HTML5 canvas">HTML5 canvas</a> when WebGL is not supported.<sup class="reference" id="cite_ref-lai4dlib_55-0"><a href="#cite_note-lai4dlib-55">[55]</a></sup></p>
<p>There are also some 2D libraries built on top of WebGL like <a href="/wiki/Cocos2d" title="Cocos2d">Cocos2d</a>-x or <a href="/wiki/Pixi.js" title="Pixi.js">Pixi.js</a>, which were implemented this way for performance reasons, in a move that parallels what happened with the <a href="/wiki/Starling_Framework" title="Starling Framework">Starling Framework</a> over Stage3D in the Flash world. The WebGL-based 2D libraries fall back to HTML5 canvas when WebGL is not available.<sup class="reference" id="cite_ref-pot_56-0"><a href="#cite_note-pot-56">[56]</a></sup></p>
<p>Removing the rendering bottleneck by giving almost direct access to the GPU also exposed performance limitations in the JavaScript implementations. Some were addressed by <a href="/wiki/Asm.js" title="Asm.js">asm.js</a>. (Similarly, the introduction of Stage3D exposed performance problems within <a href="/wiki/ActionScript" title="ActionScript">ActionScript</a>, which were addressed by projects like <a href="/wiki/CrossBridge" title="CrossBridge">CrossBridge</a>.)<sup class="reference" id="cite_ref-pot_56-1"><a href="#cite_note-pot-56">[56]</a></sup></p>
<p>Creating content for WebGL scenes often means using a regular <a href="/wiki/3D_computer_graphics_software" title="3D computer graphics software">3D content creation tool</a> and exporting the scene to a format that is readable by the viewer or helper library. Desktop 3D authoring software such as <a href="/wiki/Blender_(software)" title="Blender (software)">Blender</a>, <a href="/wiki/Autodesk_Maya" title="Autodesk Maya">Autodesk Maya</a> or <a class="new" href="/w/index.php?title=SimLab_Composer&amp;action=edit&amp;redlink=1" title="SimLab Composer (page does not exist)">SimLab Composer</a> can be used for this purpose. Particularly, <a href="/wiki/Blend4Web" title="Blend4Web">Blend4Web</a> allows a WebGL scene to be authored entirely in Blender and exported to a browser with a single click, even as a standalone web page.<sup class="reference" id="cite_ref-blend4web_about_57-0"><a href="#cite_note-blend4web_about-57">[57]</a></sup> There are also some WebGL-specific software such as <a href="/wiki/CopperCube" title="CopperCube">CopperCube</a> and the online WebGL-based editor <a href="/wiki/Clara.io" title="Clara.io">Clara.io</a>. Online platforms such as <a href="/wiki/Sketchfab" title="Sketchfab">Sketchfab</a> and <a href="/wiki/Clara.io" title="Clara.io">Clara.io</a> allow users to directly upload their 3D models and display them using a hosted WebGL viewer.</p>
<p>Additionally, Mozilla Firefox implemented built-in WebGL tools starting with version 27 that allow editing vertex and fragment shaders.<sup class="reference" id="cite_ref-58"><a href="#cite_note-58">[58]</a></sup> A number of other debugging and profiling tools have also emerged.<sup class="reference" id="cite_ref-59"><a href="#cite_note-59">[59]</a></sup></p>
<p><a href="/wiki/X3D" title="X3D">X3D</a> also made a project called X3DOM to make X3D and <a href="/wiki/VRML" title="VRML">VRML</a> content running on WebGL. The 3D model will in XML tag <code>&lt;X3D&gt;</code> in HTML5 and interactive script will use JavaScript and DOM. BS Content Studio and InstantReality X3D exporter can exported X3D in HTML and running by WebGL.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (August 2014)">citation needed</span></a></i>]</sup></p>
