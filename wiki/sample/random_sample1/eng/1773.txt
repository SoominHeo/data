<h1 class="firstHeading" id="firstHeading" lang="en">Virtual tape library</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p>A <b>virtual tape library</b> (<b>VTL</b>) is a <a class="mw-redirect" href="/wiki/Computer_storage" title="Computer storage">data storage</a> <a href="/wiki/Storage_virtualization" title="Storage virtualization">virtualization</a> technology used typically for backup and recovery purposes. A VTL presents a storage component (usually hard disk storage) as <a href="/wiki/Tape_library" title="Tape library">tape libraries</a> or <a href="/wiki/Tape_drive" title="Tape drive">tape drives</a> for use with existing backup software.</p>
<p>Virtualizing the disk storage as tape allows integration of VTLs with existing <a href="/wiki/Backup_software" title="Backup software">backup software</a> and existing backup and recovery processes and policies. The benefits of such virtualization include storage consolidation and faster data restore processes.For most mainframe data centers, the storage capacity varies, however protecting its business and mission critical data is always vital.</p>
<p>Most current VTL solutions use <a class="mw-redirect" href="/wiki/Serial_attached_SCSI" title="Serial attached SCSI">SAS</a> or <a class="mw-redirect" href="/wiki/SATA" title="SATA">SATA</a> <a href="/wiki/Disk_array" title="Disk array">disk arrays</a> as the primary storage component due to their relatively low cost. The use of array enclosures increases the scalability of the solution by allowing the addition of more disk drives and enclosures to increase the storage capacity.</p>
<p>The shift to VTL also eliminates streaming problems that often impair efficiency in tape drives as disk technology does not rely on streaming and can write effectively regardless of data transfer speeds.</p>
<p>By backing up data to disks instead of tapes, VTL often increases performance of both backup and recovery operations. Restore processes are found to be faster than backup regardless of implementations. In some cases, the data stored on the VTL's disk array is exported to other media, such as physical tapes, for <a href="/wiki/Disaster_recovery" title="Disaster recovery">disaster recovery</a> purposes (scheme called <i>disk-to-disk-to-tape</i>, or <i>D2D2T</i>).<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p>Alternatively, most contemporary backup software products introduced also direct usage of the <a href="/wiki/File_system" title="File system">file system</a> storage (especially <a href="/wiki/Network-attached_storage" title="Network-attached storage">network-attached storage</a>, accessed through <a class="mw-redirect" href="/wiki/Network_File_System_(protocol)" title="Network File System (protocol)">NFS</a> and <a class="mw-redirect" href="/wiki/CIFS" title="CIFS">CIFS</a> protocols over <a href="/wiki/Internet_Protocol" title="Internet Protocol">IP</a> networks) not requiring a tape library emulation at all. They also often offer a <a href="/wiki/Disk_staging" title="Disk staging">disk staging</a> feature: moving the data from disk to a physical tape for a long-term storage.</p>
<p>While a virtual tape library is very fast, the disk storage within is not designed to be removable, and does not usually involve physically removable external disk drives to be used for data archiving in place of tape. Since the disk storage is always connected to power and data sources and is never physically electrically isolated, it is vulnerable to potential damage and corruption due to nearby building or power grid lightning strikes.</p>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Virtual_tape_library&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The first VTL solution was an IBM Virtual Tape Server (VTS) introduced in 1997. It was targeted for a <a href="/wiki/Mainframe_computer" title="Mainframe computer">mainframe</a> market, where many legacy applications tend to use a lot of very short tape volumes. It used <a href="/wiki/ESCON" title="ESCON">ESCON</a> interface, and acted as a disk cache for <a class="new" href="/w/index.php?title=IBM_3494&amp;action=edit&amp;redlink=1" title="IBM 3494 (page does not exist)">3494</a> tape library. A competitive offering from StorageTek (acquired in 2005 by SUN Microsystems, then subsequently by Oracle Corporation) was known as Virtual Storage Manager (VSM) which leveraged the market dominant STK Powderhorn library as a back store. Each product line has been enhanced to support larger disk buffer capacities, FICON, and more recently (c. 2010) "tapeless" disk-only environments.</p>
<p>Other offerings in the mainframe space are also "tapeless". DLm has been developed by EMC Corporation, while <a href="/wiki/Luminex_Software,_Inc." title="Luminex Software, Inc.">Luminex</a> has gained popularity and wide acceptance by teaming with Data Domain to provide the benefits of <a href="/wiki/Data_deduplication" title="Data deduplication">data deduplication</a> behind its Channel Gateway platform. With the consequent reduction in off-site replication bandwidth afforded by deduplication, it is possible and practical for this form of virtual tape to reduce <a class="mw-redirect" href="/wiki/Recovery_Point_Objective" title="Recovery Point Objective">Recovery Point Objective</a> /RPO and <a class="mw-redirect" href="/wiki/Recovery_Time_Objective" title="Recovery Time Objective">Recovery Time Objective</a> /RTO to approaching zero (or instantaneous).</p>
<p>Outside of <a href="/wiki/Mainframe_computer" title="Mainframe computer">mainframe</a> environment, tape drives and libraries mostly featured <a href="/wiki/SCSI" title="SCSI">SCSI</a>. Likewise, VTLs were developed supporting popular SCSI transport protocols such as <a class="mw-redirect" href="/wiki/SCSI_Parallel_Interface" title="SCSI Parallel Interface">SPI</a> (legacy systems), <a href="/wiki/Fibre_Channel" title="Fibre Channel">Fibre Channel</a>, and <a href="/wiki/ISCSI" title="ISCSI">iSCSI</a>.</p>
<p>The <a class="mw-redirect" href="/wiki/FalconStor" title="FalconStor">FalconStor</a> VTL is the foundation of nearly half of the products sold in the VTL market, according to an Enterprise Strategy Group analyst.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>In mid-2010s VTLs got a rebirth thanks to hi-capacity "archive" drives from <a href="/wiki/Seagate_Technology" title="Seagate Technology">Seagate</a> and <a href="/wiki/HGST" title="HGST">HGST</a> and more popular "tape in cloud" and Disk-to-Disk-to-Tape (often in cloud) scenarios. <sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
