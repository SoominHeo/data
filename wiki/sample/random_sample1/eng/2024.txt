<p>A <b>decision tree</b> is a <a href="/wiki/Decision_support_system" title="Decision support system">decision support</a> tool that uses a tree-like <a href="/wiki/Diagram" title="Diagram">graph</a> or <a href="/wiki/Causal_model" title="Causal model">model</a> of decisions and their possible consequences, including <a href="/wiki/Probability" title="Probability">chance</a> event outcomes, resource costs, and <a href="/wiki/Utility" title="Utility">utility</a>. It is one way to display an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a>.</p>
<p>Decision trees are commonly used in <a href="/wiki/Operations_research" title="Operations research">operations research</a>, specifically in <a href="/wiki/Decision_analysis" title="Decision analysis">decision analysis</a>, to help identify a strategy most likely to reach a <a href="/wiki/Goal" title="Goal">goal</a>, but are also a popular tool in <a href="/wiki/Decision_tree_learning" title="Decision tree learning">machine learning</a>.</p>
<p></p>

<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decision_tree&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A decision tree is a <a href="/wiki/Flowchart" title="Flowchart">flowchart</a>-like structure in which each internal node represents a "test" on an attribute (e.g. whether a coin flip comes up heads or tails), each branch represents the outcome of the test and each leaf node represents a class label (decision taken after computing all attributes). The paths from root to leaf represents classification rules.</p>
<p>In <a href="/wiki/Decision_analysis" title="Decision analysis">decision analysis</a> a decision tree and the closely related <a href="/wiki/Influence_diagram" title="Influence diagram">influence diagram</a> are used as a visual and analytical decision support tool, where the <a href="/wiki/Expected_value" title="Expected value">expected values</a> (or <a class="mw-redirect" href="/wiki/Expected_utility" title="Expected utility">expected utility</a>) of competing alternatives are calculated.</p>
<p>A decision tree consists of 3 types of nodes:</p>
<ol>
<li>Decision nodes - commonly represented by squares</li>
<li>Chance nodes - represented by circles</li>
<li>End nodes - represented by triangles</li>
</ol>
<p>Decision trees are commonly used in <a href="/wiki/Operations_research" title="Operations research">operations research</a> and <a href="/wiki/Operations_management" title="Operations management">operations management</a>. If in practice decisions have to be taken online with no recall under incomplete knowledge, a decision tree should be paralleled by a <a href="/wiki/Probability" title="Probability">probability</a> model as a best choice model or online selection model <a href="/wiki/Algorithm" title="Algorithm">algorithm</a>. Another use of decision trees is as a descriptive means for calculating <a href="/wiki/Conditional_probability" title="Conditional probability">conditional probabilities</a>.</p>
<p>Decision trees, <a class="mw-redirect" href="/wiki/Influence_diagrams" title="Influence diagrams">influence diagrams</a>, <a class="mw-redirect" href="/wiki/Utility_function" title="Utility function">utility functions</a>, and other <a href="/wiki/Decision_analysis" title="Decision analysis">decision analysis</a> tools and methods are taught to undergraduate students in schools of business, health economics, and public health, and are examples of operations research or <a href="/wiki/Management_science" title="Management science">management science</a> methods.</p>
<h2><span class="mw-headline" id="Decision_tree_building_blocks">Decision tree building blocks</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decision_tree&amp;action=edit&amp;section=2" title="Edit section: Decision tree building blocks">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Decision_tree_elements">Decision tree elements</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decision_tree&amp;action=edit&amp;section=3" title="Edit section: Decision tree elements">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="center">
<div class="floatnone"><a class="image" href="/wiki/File:Decision-Tree-Elements.png"><img alt="Decision-Tree-Elements.png" data-file-height="141" data-file-width="491" height="141" src="//upload.wikimedia.org/wikipedia/commons/a/ad/Decision-Tree-Elements.png" width="491"/></a></div>
</div>
<p>Drawn from left to right, a decision tree has only burst nodes (splitting paths) but no sink nodes (converging paths). Therefore, used manually, they can grow very big and are then often hard to draw fully by hand. Traditionally, decision trees have been created manually — as the aside example shows — although increasingly, specialized software is employed.</p>
<h3><span class="mw-headline" id="Decision_rules">Decision rules</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decision_tree&amp;action=edit&amp;section=4" title="Edit section: Decision rules">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The decision tree can be <a href="/wiki/Linearization" title="Linearization">linearized</a> into <b>decision rules</b>,<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> where the outcome is the contents of the leaf node, and the conditions along the path form a conjunction in the if clause. In general, the rules have the form:</p>
<dl>
<dd><i>if</i> condition1 <i>and</i> condition2 <i>and</i> condition3 <i>then</i> outcome.</dd>
</dl>
<p>Decision rules can be generated by constructing <a href="/wiki/Association_rule_learning" title="Association rule learning">association rules</a> with the target variable on the right. They can also denote <a class="mw-disambig" href="/wiki/Temporal" title="Temporal">temporal</a> or <a class="mw-redirect" href="/wiki/Causal" title="Causal">causal</a> relations.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<h3><span class="mw-headline" id="Decision_tree_using_flowchart_symbols">Decision tree using flowchart symbols</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decision_tree&amp;action=edit&amp;section=5" title="Edit section: Decision tree using flowchart symbols">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Commonly a decision tree is drawn using <a href="/wiki/Flowchart" title="Flowchart">flowchart</a> symbols as it is easier for many to read and understand.</p>

<div class="floatnone"><a class="image" href="/wiki/File:DecisionCalcs.jpg"><img alt="DecisionCalcs.jpg" data-file-height="525" data-file-width="800" height="525" src="//upload.wikimedia.org/wikipedia/commons/4/48/DecisionCalcs.jpg" width="800"/></a></div>

<h3><span class="mw-headline" id="Analysis_example">Analysis example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decision_tree&amp;action=edit&amp;section=6" title="Edit section: Analysis example">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Analysis can take into account the decision maker's (e.g., the company's) <a href="/wiki/Preference" title="Preference">preference</a> or <a class="mw-redirect" href="/wiki/Utility_function" title="Utility function">utility function</a>, for example:</p>
<div class="center">
<div class="floatnone"><a class="image" href="/wiki/File:RiskPrefSensitivity2Threshold.png"><img alt="RiskPrefSensitivity2Threshold.png" data-file-height="289" data-file-width="411" height="289" src="//upload.wikimedia.org/wikipedia/commons/5/5b/RiskPrefSensitivity2Threshold.png" width="411"/></a></div>
</div>
<p>The basic interpretation in this situation is that the company prefers B's risk and payoffs under realistic risk preference coefficients (greater than $400K—in that range of risk aversion, the company would need to model a third strategy, "Neither A nor B").</p>
<h3><span class="mw-headline" id="Influence_diagram">Influence diagram</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decision_tree&amp;action=edit&amp;section=7" title="Edit section: Influence diagram">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Much of the information in a decision tree can be represented more compactly as an <a href="/wiki/Influence_diagram" title="Influence diagram">influence diagram</a>, focusing attention on the issues and relationships between events.</p>

<h2><span class="mw-headline" id="Association_rule_induction">Association rule induction</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decision_tree&amp;action=edit&amp;section=8" title="Edit section: Association rule induction">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Decision trees can also be seen as generative models of induction rules from empirical data. An optimal decision tree is then defined as a tree that accounts for most of the data, while minimizing the number of levels (or "questions").<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> Several algorithms to generate such optimal trees have been devised, such as ID3/4/5,<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> CLS, ASSISTANT, and CART.</p>
<h2><span class="mw-headline" id="Advantages_and_disadvantages">Advantages and disadvantages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decision_tree&amp;action=edit&amp;section=9" title="Edit section: Advantages and disadvantages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Among decision support tools, decision trees (and <a class="mw-redirect" href="/wiki/Influence_diagrams" title="Influence diagrams">influence diagrams</a>) have several advantages. Decision trees:</p>
<ul>
<li>Are simple to understand and interpret. People are able to understand decision tree models after a brief explanation.</li>
<li>Have value even with little hard data. Important insights can be generated based on experts describing a situation (its alternatives, probabilities, and costs) and their preferences for outcomes.</li>
<li>Allow the addition of new possible scenarios</li>
<li>Help determine worst, best and expected values for different scenarios</li>
<li>Use a <a href="/wiki/White_box_(software_engineering)" title="White box (software engineering)">white box</a> model. If a given result is provided by a model.</li>
<li>Can be combined with other decision techniques.</li>
</ul>
<p>Disadvantages of decision trees:</p>
<ul>
<li>For data including categorical variables with different number of levels, <a href="/wiki/Information_gain_in_decision_trees" title="Information gain in decision trees">information gain in decision trees</a> are biased in favor of those attributes with more levels.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></li>
<li>Calculations can get very complex particularly if many values are uncertain and/or if many outcomes are linked.</li>
</ul>
