<h1 class="firstHeading" id="firstHeading" lang="en">Advanced Host Controller Interface</h1>
<div class="mw-body-content" id="bodyContent">

<p>The <b>Advanced Host Controller Interface</b> (<b>AHCI</b>) is a technical standard defined by <a href="/wiki/Intel" title="Intel">Intel</a> that specifies the operation of <a href="/wiki/Serial_ATA" title="Serial ATA">Serial ATA</a> (SATA) host bus adapters in a non-implementation-specific manner.</p>
<p>The specification describes a system memory structure for computer hardware vendors to exchange data between host system <a href="/wiki/Computer_memory" title="Computer memory">memory</a> and attached <a class="mw-redirect" href="/wiki/Computer_storage_device" title="Computer storage device">storage devices</a>. AHCI gives software developers and hardware designers a standard method for detecting, configuring, and programming SATA/AHCI adapters. AHCI is separate from the SATA 3 Gbit/s standard, although it exposes SATA's advanced capabilities (such as <a href="/wiki/Hot_swapping" title="Hot swapping">hot swapping</a> and <a href="/wiki/Native_Command_Queuing" title="Native Command Queuing">native command queuing</a>) such that host systems can utilize them.</p>
<p>As of March 2014<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Advanced_Host_Controller_Interface&amp;action=edit">[update]</a></sup>, the current version of the specification is 1.3.1.</p>

<h2><span class="mw-headline" id="Operating_modes">Operating modes</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Advanced_Host_Controller_Interface&amp;action=edit&amp;section=1" title="Edit section: Operating modes">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Many SATA controllers offer selectable modes of operation: legacy <a class="mw-redirect" href="/wiki/AT_Attachment" title="AT Attachment">Parallel ATA</a> emulation, standard AHCI mode (also known as native mode), or vendor-specific <a href="/wiki/RAID" title="RAID">RAID</a> (which generally enables AHCI in order to take advantage of its capabilities). Intel recommends choosing RAID mode on their <a href="/wiki/Motherboard" title="Motherboard">motherboards</a> (which also enables AHCI) rather than AHCI/SATA mode for maximum flexibility.<sup class="reference" id="cite_ref-intel_1-0"><a href="#cite_note-intel-1">[1]</a></sup> Legacy mode is a software <a href="/wiki/Backward_compatibility" title="Backward compatibility">backward-compatibility</a> mechanism intended to allow the SATA controller to run in legacy operating systems which are not SATA-aware or where a driver does not exist to make the operating system SATA-aware.</p>
<p>When a SATA controller is configured to operate in legacy mode, the number of storage devices per controller is usually limited to four (two IDE channels, primary and secondary, with up to two devices per channel), compared to the maximum of 32 devices/ports when configured in AHCI mode.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<h2><span class="mw-headline" id="Operating_system_support">Operating system support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Advanced_Host_Controller_Interface&amp;action=edit&amp;section=2" title="Edit section: Operating system support">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>AHCI is supported out of the box on <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a> and later, <a href="/wiki/Linux" title="Linux">Linux</a>-based operating systems (since version 2.6.19 of the <a href="/wiki/Linux_kernel" title="Linux kernel">kernel</a>), <a href="/wiki/OpenBSD" title="OpenBSD">OpenBSD</a> (since version 4.1), <a href="/wiki/NetBSD" title="NetBSD">NetBSD</a> (since version 4.0), <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a> (since version 8.0), <a class="mw-redirect" href="/wiki/OS_X" title="OS X">OS X</a>, <a href="/wiki/EComStation" title="EComStation">eComStation</a> (since version 2.1), and <a class="mw-redirect" href="/wiki/Solaris_10" title="Solaris 10">Solaris 10</a> (since version 8/07).<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> <a class="mw-redirect" href="/wiki/DragonFlyBSD" title="DragonFlyBSD">DragonFlyBSD</a> based its AHCI implementation on OpenBSD's and added extended features such as port multiplier support. Older versions of operating systems require hardware-specific drivers in order to support AHCI. <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a> and older do not provide support out of the box.</p>
<h3><span class="mw-headline" id="Boot_issues">Boot issues</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Advanced_Host_Controller_Interface&amp;action=edit&amp;section=3" title="Edit section: Boot issues">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Some operating systems, notably <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a>, <a href="/wiki/Windows_7" title="Windows 7">Windows 7</a>, <a href="/wiki/Windows_8" title="Windows 8">Windows 8</a> and <a href="/wiki/Windows_10" title="Windows 10">Windows 10</a>, do not configure themselves to load the AHCI driver upon boot if the SATA controller was not in AHCI mode at the time of installation. This can cause failure to boot, with an error message, if the SATA controller is later switched to AHCI mode. For this reason, Intel recommends changing the drive controller to AHCI or RAID before installing an operating system.<sup class="reference" id="cite_ref-intel_1-1"><a href="#cite_note-intel-1">[1]</a></sup> (It may also be necessary to load chipset-specific AHCI or RAID drivers at installation time, for example from a USB flash drive.)</p>
<p>On Windows Vista and Windows 7, this can be fixed by configuring the <code>msahci</code> device driver to start at boot time (rather than on-demand). Setting non-AHCI mode in the <a href="/wiki/BIOS" title="BIOS">BIOS</a> will allow the user to boot Windows, where the required <a class="mw-redirect" href="/wiki/Windows_registry" title="Windows registry">registry</a> change can be performed. The BIOS can then be changed to AHCI.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<p>On Windows 10, this can be fixed by forcing the correct drivers to reload during safe mode.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>On Windows 8 and <a href="/wiki/Windows_Server_2012" title="Windows Server 2012">Windows Server 2012</a>, the name of the controller has changed from <code>msahci</code> to <code>storahci</code>,<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> and the procedure to upgrade to the new controller to it is similar to that of Windows 7.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> On Windows 8 and Windows Server 2012, changing the SATA mode to AHCI without updating the registry will make the boot drive inaccessible.</p>
<p>A similar problem can occur on Linux systems if the AHCI driver is compiled as a <a class="mw-redirect" href="/wiki/Kernel_module" title="Kernel module">kernel module</a> rather than built into the <a class="mw-redirect" href="/wiki/Kernel_image" title="Kernel image">kernel image</a>, as it may not be included in the <a href="/wiki/Initrd" title="Initrd">initrd</a> (initial RAM disk) created when the controller is configured to run in legacy mode. The solution is either to build a new initrd containing the AHCI module, or to build the AHCI driver into the kernel image.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
