<p><b>iCalendar</b> is a computer <a href="/wiki/File_format" title="File format">file format</a> which allows Internet users to send meeting requests and tasks to other Internet users by sharing or sending files in this format through various methods. The files usually have an <a href="/wiki/Filename_extension" title="Filename extension">extension</a> of <code>.ics</code>. With <a href="/wiki/List_of_applications_with_iCalendar_support" title="List of applications with iCalendar support">supporting software</a>, such as an <a href="/wiki/Email_client" title="Email client">email reader</a> or <a href="/wiki/Calendaring_software" title="Calendaring software">calendar application</a>, recipients of an iCalendar data file can respond to the sender easily or counter-propose another meeting date/time. The file format is specified in a <a class="mw-redirect" href="/wiki/Internet_standard" title="Internet standard">proposed internet standard</a> (<a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc5545" rel="nofollow">RFC 5545</a>) for calendar data exchange.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p>ICalendar is used and supported by a large number of products, including <a href="/wiki/Google_Calendar" title="Google Calendar">Google Calendar</a>, <a href="/wiki/Calendar_(Apple)" title="Calendar (Apple)">Apple Calendar</a> (formerly iCal), <a class="mw-redirect" href="/wiki/IBM_Lotus_Notes" title="IBM Lotus Notes">IBM Lotus Notes</a>,<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> <a href="/wiki/Yahoo!_Calendar" title="Yahoo! Calendar">Yahoo! Calendar</a>, <a class="mw-redirect" href="/wiki/Novell_Evolution" title="Novell Evolution">Evolution (software)</a>, <a href="/wiki/EM_Client" title="EM Client">eM Client</a>, <a href="/wiki/Lightning_(software)" title="Lightning (software)">Lightning</a> extension for <a href="/wiki/Mozilla_Thunderbird" title="Mozilla Thunderbird">Mozilla Thunderbird</a> and <a href="/wiki/SeaMonkey" title="SeaMonkey">SeaMonkey</a>, and partially by <a href="/wiki/Microsoft_Outlook" title="Microsoft Outlook">Microsoft Outlook</a> and <a class="mw-redirect" href="/wiki/Novell_GroupWise" title="Novell GroupWise">Novell GroupWise</a>.</p>
<p>iCalendar is designed to be independent of the transport protocol. For example, certain events can be sent by traditional email or whole calendar files can be shared and edited by using a <a class="mw-redirect" href="/wiki/WebDav" title="WebDav">WebDav</a> server, or <a href="/wiki/SyncML" title="SyncML">SyncML</a>. Simple web servers (using just the HTTP protocol) are often used to distribute iCalendar data about an event and to publish busy times of an individual. Publishers can embed iCalendar data in web pages using <a href="/wiki/HCalendar" title="HCalendar">hCalendar</a>, a 1:1 <a class="mw-redirect" href="/wiki/Microformats" title="Microformats">microformat</a> representation of iCalendar in semantic <a href="/wiki/XHTML" title="XHTML">(X)</a><a href="/wiki/HTML" title="HTML">HTML</a>.</p>

<h2><span class="mw-headline" id="History_and_design">History and design</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ICalendar&amp;action=edit&amp;section=1" title="Edit section: History and design">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>iCalendar was created by the Calendaring and Scheduling Working Group of the <a href="/wiki/Internet_Engineering_Task_Force" title="Internet Engineering Task Force">Internet Engineering Task Force</a>, chaired by <a class="new" href="/w/index.php?title=Anik_Ganguly&amp;action=edit&amp;redlink=1" title="Anik Ganguly (page does not exist)">Anik Ganguly</a> of <a class="mw-redirect" href="/wiki/Open_Text_Corporation" title="Open Text Corporation">Open Text Corporation</a>), and was authored by Frank Dawson of <a class="mw-redirect" href="/wiki/Lotus_Development_Corporation" title="Lotus Development Corporation">Lotus Development Corporation</a> and Derik Stenerson of <a class="mw-redirect" href="/wiki/Microsoft_Corporation" title="Microsoft Corporation">Microsoft Corporation</a>. iCalendar is heavily based on the earlier <a class="mw-redirect" href="/wiki/VCalendar" title="VCalendar">vCalendar</a> by the <a href="/wiki/Internet_Mail_Consortium" title="Internet Mail Consortium">Internet Mail Consortium</a> (IMC). iCalendar data files are <a href="/wiki/Plain_text" title="Plain text">plain text</a> files with the <a href="/wiki/Filename_extension" title="Filename extension">extension</a> <code>.ics</code> or <code>.ifb</code> (for files containing availability information only). <a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc5545" rel="nofollow">RFC 5545</a> replaced <a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc2445" rel="nofollow">RFC 2445</a> in September 2009 and now defines the standard.</p>
<p>iCalendar data has the <a href="/wiki/MIME" title="MIME">MIME</a> content type <i>text/calendar</i>.</p>
<p>The filename extension of <code>ics</code> is to be used for files containing calendaring and scheduling information, <code>ifb</code> for files with free or busy time information consistent with this MIME content type. The equivalent file <a href="/wiki/Type_code" title="Type code">type codes</a> in Apple Macintosh operating system environments are <code>iCal</code> and <code>iFBf</code>.</p>
<p>By default, iCalendar uses the <a href="/wiki/UTF-8" title="UTF-8">UTF-8</a> <a class="mw-redirect" href="/wiki/Character_set" title="Character set">character set</a>; a different character set can be specified using the "charset" MIME parameter (if the transport method used supports MIME, such as <a href="/wiki/Email" title="Email">Email</a> or <a class="mw-redirect" href="/wiki/HTTP" title="HTTP">HTTP</a>).</p>
<p>Each line is terminated by <a href="/wiki/Carriage_return" title="Carriage return">CR</a>+<a class="mw-redirect" href="/wiki/Line_feed" title="Line feed">LF</a> (in <a href="/wiki/Hexadecimal" title="Hexadecimal">hexadecimal</a>: 0D0A). Lines should be limited to 75 <a href="/wiki/Octet_(computing)" title="Octet (computing)">octets</a> (not characters) long. Where a data item is too long to fit on a single line it can be continued on following lines by starting the continuation lines with a space character (in hex: 20) or a tab character (in hex: 09).</p>
<p>Actual line feeds in data items are encoded as a backslash followed by the letter n or N (the bytes 5C 6E or 5C 4E in UTF-8).</p>
<h2><span class="mw-headline" id="Limitations_and_future">Limitations and future</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ICalendar&amp;action=edit&amp;section=2" title="Edit section: Limitations and future">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The iCalendar format is designed to transmit calendar-based data, such as events, and intentionally does not describe what to do with that data. Thus, other programming may be needed to negotiate what to do with this data.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<p>iCalendar is meant to "provide the definition of a common format for openly exchanging calendaring and scheduling information across the Internet". While the features most often used by users are widely supported by iCalendar, some more advanced capabilities have problems. For example, most vendors do not support Journals (VJOURNAL). VTODOs have had conversion problems as well.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p>iCalendar's calendar is also not compatible with some non-<a href="/wiki/Gregorian_calendar" title="Gregorian calendar">Gregorian</a> calendars such as the <a href="/wiki/Lunar_calendar" title="Lunar calendar">lunar calendars</a> used in <a href="/wiki/Israel" title="Israel">Israel</a> and <a href="/wiki/Saudi_Arabia" title="Saudi Arabia">Saudi Arabia</a>.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<p>The memo "Calendar Access Protocol" (<a href="/wiki/Request_for_Comments" title="Request for Comments">RFC</a> 4324) was an initial attempt at a universal system to create real-time calendars. This protocol was eventually abandoned, possibly because of excessive complexity. Regardless, iCalendar-based code such as <a href="/wiki/GroupDAV" title="GroupDAV">GroupDAV</a> and <a href="/wiki/CalDAV" title="CalDAV">CalDAV</a> are now being used more frequently in both client and server <a href="/wiki/Software" title="Software">software</a> packages.</p>
<p>The <a class="mw-redirect" href="/wiki/IETF" title="IETF">IETF</a> "Calendaring and Scheduling Working Group" (ietf-calsify WG) has published additional proposed revisions to the iCalendar standards<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> (as RFC-drafts/"Proposed"),<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> but as of January 2011, the group has 'ended'.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> The work being picked up by the "Calendaring Extensions Working Group" (ietf-calext WG).<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
<h2><span class="mw-headline" id="Technical_specifications">Technical specifications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ICalendar&amp;action=edit&amp;section=3" title="Edit section: Technical specifications">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Core_object">Core object</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ICalendar&amp;action=edit&amp;section=4" title="Edit section: Core object">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The top-level element in iCalendar is the Calendaring and Scheduling Core Object, a collection of calendar and scheduling information. Typically, this information will consist of a single iCalendar object. However, multiple iCalendar objects can be grouped together.</p>
<p>The first line must be <code>BEGIN:VCALENDAR</code>, and the last line must be <code>END:VCALENDAR</code>; the contents between these lines is called the "icalbody".<br/>
The second line <code>VERSION:2.0</code> indicates that the data is in iCalendar format. <code>VERSION:1.0</code> was used to specify that data is in the old vCalendar format.</p>
<p>The body of the iCalendar object (the icalbody) is made up of a list of calendar properties and one or more calendar components. The calendar properties apply to the entire calendar. The calendar components are several calendar properties which create a calendar schematic (design). For example, the calendar component can specify an event, a to-do list, a journal entry, time zone information, or free/busy time information, or an alarm. Empty lines are not allowed in some versions of usage (Google calendar).</p>
<p>Here is a simple example<sup class="reference" id="cite_ref-From_RFC_2445_10-0"><a href="#cite_note-From_RFC_2445-10">[10]</a></sup> of an iCalendar object, "<a href="/wiki/Bastille_Day" title="Bastille Day">Bastille Day</a> Party" event which occurs July 14, 1997 17:00 (UTC) through July 15, 1997 03:59:59 (UTC):</p>
<pre>
BEGIN:VCALENDAR
VERSION:2.0
PRODID:-//hacksw/handcal//NONSGML v1.0//EN
BEGIN:VEVENT
UID:uid1@example.com
DTSTAMP:19970714T170000Z
ORGANIZER;CN=John Doe:MAILTO:john.doe@example.com
DTSTART:19970714T170000Z
DTEND:19970715T035959Z
SUMMARY:Bastille Day Party
END:VEVENT
END:VCALENDAR
</pre>
<p>There are many different types of components which can be used in iCalendar, as described below.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></p>
<h4><span class="mw-headline" id="Events_.28VEVENT.29">Events (VEVENT)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ICalendar&amp;action=edit&amp;section=5" title="Edit section: Events (VEVENT)">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p><code>VEVENT</code> describes an event, which has a scheduled amount of time on a calendar. Normally, when a user accepts the calendar event, this will cause that time to be considered busy.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> A <code>VEVENT</code> may include a <code>VALARM</code> which allows an alarm. Such events have a <code>DTSTART</code> which sets a starting time, and a <code>DTEND</code> which sets an ending time. If the calendar event is recurring, <code>DTSTART</code> sets up the start of the first event.</p>
<p>A VALARM code (reminder 1 day before):</p>
<pre>
BEGIN:VALARM
TRIGGER:-PT1440M
ACTION:DISPLAY
DESCRIPTION:Reminder
END:VALARM
</pre>
<p><code>VEVENT</code> also is used for calendar events without a specific time, such as anniversaries and daily reminders.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> If the user needs to send in a cancellation for an event, the <a href="/wiki/Unique_identifier" title="Unique identifier">UID</a> should be same as the original event, and the component properties should be set to cancel Ex.</p>
<pre>
METHOD:CANCEL
STATUS:CANCELLED
</pre>
<p>For sending an <code>UPDATE</code> for an event the <code>UID</code> should match the original <code>UID</code>. The other component property to be set is:</p>
<pre>
SEQUENCE:&lt;Num of Update&gt;
</pre>
<p>I.e., for the first update:</p>
<pre>
SEQUENCE:1
</pre>
<p>In Microsoft Outlook, the <code>SUMMARY</code> corresponds to the "Subject" entry in the "Appointment" form, and <code>DESCRIPTION</code> to the descriptive text below it. In addition, Outlook 2002 and Outlook 2003 demand a <code>UID</code> and a <code>DTSTAMP</code>.</p>
<h4><span class="mw-headline" id="To-do_.28VTODO.29">To-do (VTODO)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ICalendar&amp;action=edit&amp;section=6" title="Edit section: To-do (VTODO)">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>VTODO explains a to-do item, i.e., an action-item or assignment.</p>
<p>Not all calendar applications recognize VTODO items. In particular, Outlook does not export Tasks as VTODO items, and ignores VTODO items in imported calendars.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup></p>
<p>The following is an example of a to-do due on April 15, 1998.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> An audio alarm has been specified to remind the calendar user at noon, the day before the to-do is expected to be completed and repeat hourly, four additional times. The SEQUENCE element shows this to-do has been modified twice since it was initially created.</p>
<pre>
BEGIN:VCALENDAR
VERSION:2.0
PRODID:-//ABC Corporation//NONSGML My Product//EN
BEGIN:VTODO
DTSTAMP:19980130T134500Z
SEQUENCE:2
UID:uid4@example.com
DUE:19980415T235959
STATUS:NEEDS-ACTION
SUMMARY:Submit Income Taxes
BEGIN:VALARM
ACTION:AUDIO
TRIGGER:19980414T120000
ATTACH;FMTTYPE=audio/basic:http://example.com/pub/audio-
 files/ssbanner.aud
REPEAT:4
DURATION:PT1H
END:VALARM
END:VTODO
END:VCALENDAR
</pre>
<h4><span class="mw-headline" id="Journal_entry_.28VJOURNAL.29">Journal entry (VJOURNAL)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ICalendar&amp;action=edit&amp;section=7" title="Edit section: Journal entry (VJOURNAL)">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>VJOURNAL is a journal entry. They attach descriptive text to a particular calendar date, may be used to record a daily record of activities or accomplishments, or describe progress with a related to-do entry. A "VJOURNAL" calendar component does not take up time on a calendar, so it has no effect on free or busy time (just like TRANSPARENT entries). In practice, few programs support VJOURNAL entries, although examples exist: Plum Canary's Chirp software uses VTODO and VJOURNAL together. Also <a class="mw-redirect" href="/wiki/KOrganizer" title="KOrganizer">KOrganizer</a> from the KDE desktop and <a href="/wiki/Evolution_(software)" title="Evolution (software)">Evolution</a> from the <a href="/wiki/GNOME" title="GNOME">GNOME</a> desktop support VJOURNAL.</p>
<p>The following is an example of a journal entry:<sup class="reference" id="cite_ref-From_RFC_2445_10-1"><a href="#cite_note-From_RFC_2445-10">[10]</a></sup></p>
<pre>
BEGIN:VCALENDAR
VERSION:2.0
PRODID:-//ABC Corporation//NONSGML My Product//EN
BEGIN:VJOURNAL
DTSTAMP:19970324T120000Z
UID:uid5@example.com
ORGANIZER:MAILTO:jsmith@example.com
STATUS:DRAFT
CLASS:PUBLIC
CATEGORIES:Project Report, XYZ, Weekly Meeting
DESCRIPTION:Project xyz Review Meeting Minutes\n
 Agenda\n1. Review of project version 1.0 requirements.\n2.
 Definition
 of project processes.\n3. Review of project schedule.\n
 Participants: John Smith, Jane Doe, Jim Dandy\n-It was
  decided that the requirements need to be signed off by
  product marketing.\n-Project processes were accepted.\n
 -Project schedule needs to account for scheduled holidays
  and employee vacation time. Check with HR for specific
  dates.\n-New schedule will be distributed by Friday.\n-
 Next weeks meeting is cancelled. No meeting until 3/23.
END:VJOURNAL
END:VCALENDAR
</pre>
<p>(Note: This example is taken from <a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc2445" rel="nofollow">RFC 2445</a> with the correction of changing the word 'CATEGORY' to 'CATEGORIES', which is a mistake in the original RFC)</p>
<h4><span class="mw-headline" id="Free.2Fbusy_time_.28VFREEBUSY.29">Free/busy time (VFREEBUSY)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ICalendar&amp;action=edit&amp;section=8" title="Edit section: Free/busy time (VFREEBUSY)">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>VFREEBUSY is a request for free/busy time, is a response to a request, or is a published set of busy time.<sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (December 2008)">clarification needed</span></a></i>]</sup> <sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></p>
<p>The following is an example of published busy time information:<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<pre>
BEGIN:VCALENDAR
VERSION:2.0
PRODID:-//RDU Software//NONSGML HandCal//EN
BEGIN:VFREEBUSY
DTSTAMP:20151013T080000Z
UID:uid6@example.com
ORGANIZER:MAILTO:jsmith@example.com
DTSTART:19980313T141711Z
DTEND:19980410T141711Z
FREEBUSY:19980314T233000Z/19980315T003000Z
FREEBUSY:19980316T153000Z/19980316T163000Z
FREEBUSY:19980318T030000Z/19980318T040000Z
URL:http://www.example.com/calendar/busytime/jsmith.ifb
END:VFREEBUSY
END:VCALENDAR
</pre>
<h4><span class="mw-headline" id="Other_component_types">Other component types</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ICalendar&amp;action=edit&amp;section=9" title="Edit section: Other component types">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Other component types include <b>VTIMEZONE</b> (time zones) and <b>VALARM</b> (alarms). Some components can include other components (VALARM is often included in other components).<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup></p>
<h4><span class="mw-headline" id="Distributing_updates">Distributing updates</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ICalendar&amp;action=edit&amp;section=10" title="Edit section: Distributing updates">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>The UID field distributes updates when a scheduled event changes. When the event is first generated a <a class="mw-redirect" href="/wiki/UUID" title="UUID">globally unique</a> identifier is created. If a later event is distributed with the same UID, it replaces the original one.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup></p>
<h3><span class="mw-headline" id="Calendar_extensions">Calendar extensions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ICalendar&amp;action=edit&amp;section=11" title="Edit section: Calendar extensions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>vCalendar and iCalendar support private <a class="mw-redirect" href="/wiki/Software_extensions" title="Software extensions">software extensions</a>, with a "X-" prefix, a number of which are in common usage.</p>
<p>Some of these include:</p>
<ul>
<li>X-RECURRENCE-ID - vCalendar 1.0 extension which mimics the iCalendar 2.0 RECURRENCE-ID (Nokia S60 3rd Edition)</li>
<li>X-EPOCAGENDAENTRYTYPE - defines the client calendar type</li>
<li>X-FUNAMBOL-AALARMOPTIONS</li>
<li>X-FUNAMBOL-ALLDAY - All Day event flag</li>
<li>X-MICROSOFT-CDO-ALLDAYEVENT - Microsoft Outlook all day event flag</li>
<li>X-MICROSOFT-CDO-BUSYSTATUS - Microsoft Outlook status information</li>
<li>X-MICROSOFT-CDO-INTENDEDSTATUS</li>
<li>X-WR-CALNAME - The display name of the calendar</li>
<li>X-WR-CALDESC - A description of the calendar</li>
<li>X-WR-RELCALID - A globally unique identifier for the calendar<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup></li>
<li>X-WR-TIMEZONE</li>
<li>X-PUBLISHED-TTL - Recommended update interval for subscription to the calendar</li>
<li>X-ALT-DESC - Used to include HTML markup in an event's description. Standard DESCRIPTION tag should contain non-HTML version.</li>
</ul>
<h3><span class="mw-headline" id="vCalendar_1.0">vCalendar 1.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ICalendar&amp;action=edit&amp;section=12" title="Edit section: vCalendar 1.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>iCalendar's design was based on the previous file format <b>vCalendar</b> created by the Internet Mail Consortium (IMC).<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup></p>
<p>Here is an example of information in vCalendar format:</p>
<pre>
BEGIN:VCALENDAR
VERSION:1.0
BEGIN:VEVENT
CATEGORIES:MEETING
STATUS:TENTATIVE
DTSTART:19960401T033000Z
DTEND:19960401T043000Z
SUMMARY:Your Proposal Review
DESCRIPTION:Steve and John to review newest proposal material
CLASS:PRIVATE
END:VEVENT
END:VCALENDAR
</pre>
<p>It has the .vcs file extension. After iCalendar was released, the Internet Mail Consortium stated that it "hopes that all vCalendar developers take advantage of these new open standards and make their software compatible with both vCalendar 1.0 and iCalendar."<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup></p>

<li id="cite_note-14"><span class="mw-cite-backlink"><b><a href="#cite_ref-14">^</a></b></span> <span class="reference-text"><cite class="citation web"><a class="external text" href="https://msdn.microsoft.com/en-us/library/ee624464%28v=exchg.80%29.aspx" rel="nofollow">"[RFC5546] Section 3.4 Methods for VTODO Components"</a>. <i>Microsoft Developer Network</i><span class="reference-accessdate">. Retrieved <span class="nowrap">7 August</span> 2015</span>.</cite><span class="Z3988" title="ctx_ver=Z39.88-2004&amp;rfr_id=info%3Asid%2Fen.wikipedia.org%3AICalendar&amp;rft.atitle=%5BRFC5546%5D+Section+3.4+Methods+for+VTODO+Components&amp;rft.genre=unknown&amp;rft_id=https%3A%2F%2Fmsdn.microsoft.com%2Fen-us%2Flibrary%2Fee624464%2528v%3Dexchg.80%2529.aspx&amp;rft.jtitle=Microsoft+Developer+Network&amp;rft_val_fmt=info%3Aofi%2Ffmt%3Akev%3Amtx%3Ajournal"><span style="display:none;">Â </span></span></span></li>
<li id="cite_note-15"><span class="mw-cite-backlink"><b><a href="#cite_ref-15">^</a></b></span> <span class="reference-text">From <a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc2445" rel="nofollow">RFC 2445</a>.</span></li>
<li id="cite_note-16"><span class="mw-cite-backlink"><b><a href="#cite_ref-16">^</a></b></span> <span class="reference-text">As described in <a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc2445" rel="nofollow">RFC 2445</a>:</span>
<blockquote>
<p><span class="reference-text">When used to request free/busy time information, the "ATTENDEE" property specifies the calendar users whose free/busy time is being requested; the "ORGANIZER" property specifies the calendar user who is requesting the free/busy time; the "DTSTART" and "DTEND" properties specify the window of time for which the free/busy time is being requested; the "UID" and "DTSTAMP" properties are specified to assist in proper sequencing of multiple free/busy time requests.</span></p>
</blockquote>
<blockquote>
<p><span class="reference-text">When used to reply to a request for free/busy time, the "ATTENDEE" property specifies the calendar user responding to the free/busy time request; the "ORGANIZER" property specifies the calendar user that originally requested the free/busy time; the "FREEBUSY" property specifies the free/busy time information (if it exists); and the "UID" and "DTSTAMP" properties are specified to assist in proper sequencing of multiple free/busy time replies.</span></p>
</blockquote>
<blockquote>
<p><span class="reference-text">When used to publish busy time, the "ORGANIZER" property specifies the calendar user associated with the published busy time; the "DTSTART" and "DTEND" properties specify an inclusive time window that surrounds the busy time information; the "FREEBUSY" property specifies the published busy time information; and the "DTSTAMP" property specifies the date/time that iCalendar object was created.</span></p>
</blockquote>
</li>
