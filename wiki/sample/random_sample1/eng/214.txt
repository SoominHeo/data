<p><b>Android application package</b> (<b>APK</b>) is the <a href="/wiki/Package_format" title="Package format">package</a> <a href="/wiki/File_format" title="File format">file format</a> used by the <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android</a> operating system for distribution and installation of <a href="/wiki/Mobile_app" title="Mobile app">mobile apps</a> and <a href="/wiki/Middleware" title="Middleware">middleware</a>.</p>
<p>APK files are analogous to other <a class="mw-redirect mw-disambig" href="/wiki/Software_package_(disambiguation)" title="Software package (disambiguation)">software packages</a> such as <a class="mw-redirect" href="/wiki/APPX" title="APPX">APPX</a> in <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> or <a href="/wiki/Deb_(file_format)" title="Deb (file format)">Deb packages</a> in <a href="/wiki/Debian" title="Debian">Debian</a>-based operating systems like Ubuntu. To make an APK file, a program for Android is first compiled, and then all of its parts are packaged into one file. An APK file contains all of that program's code (such as <a class="mw-redirect" href="/wiki/.dex" title=".dex">.dex</a> files), resources, assets, certificates, and <a href="/wiki/Manifest_file" title="Manifest file">manifest file</a>. As is the case with many file formats, APK files can have any name needed, provided that the file name ends in ".apk".<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p>APK files are a type of <a href="/wiki/Archive_file" title="Archive file">archive file</a>, specifically in <a href="/wiki/Zip_(file_format)" title="Zip (file format)">zip format</a> packages based on the <a href="/wiki/JAR_(file_format)" title="JAR (file format)">JAR file format</a>, with <code>.apk</code> as the <a href="/wiki/Filename_extension" title="Filename extension">filename extension</a>. The <a class="mw-redirect" href="/wiki/Internet_media_type" title="Internet media type">MIME type</a> associated with APK files is <code>application/vnd.android.package-archive</code>.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<p>APK files can be installed on <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android</a> powered devices just like installing software on <a href="/wiki/Personal_computer" title="Personal computer">PC</a>. When a user downloads and installs an Android application from either an official source (such as <a href="/wiki/Google_Play" title="Google Play">Google Play</a>), or from some other (unofficial) site, they are installing an APK file on their device. A user or developer can also install an APK file directly to a device (that is, not via download from the network) from a desktop computer, using a communication program such as <a href="/wiki/Android_software_development#Android_Debug_Bridge" title="Android software development">adb</a>, or from within a file manager app in a process known as <a href="/wiki/Sideloading" title="Sideloading">sideloading</a>. By default, the ability to install from unofficial sites or directly from a desktop or file manager is disabled for security reasons on most Android devices. Users can enable it by changing the setting "Unknown sources" in the Settings menu.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<h2><span class="mw-headline" id="Package_contents">Package contents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Android_application_package&amp;action=edit&amp;section=1" title="Edit section: Package contents">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>An APK file is an <a href="/wiki/Archive_file" title="Archive file">archive</a> that usually contains the following files and directories:</p>
<ul>
<li><code>META-INF</code> directory:
<ul>
<li><code>MANIFEST.MF</code>: the <a href="/wiki/Manifest_file" title="Manifest file">Manifest file</a></li>
<li><code>CERT.RSA</code>: The certificate of the application.</li>
<li><code>CERT.SF</code>: The list of resources and <a href="/wiki/SHA-1" title="SHA-1">SHA-1</a> digest of the corresponding lines in the MANIFEST.MF file; for example:</li>
</ul>
</li>
</ul>
<pre>
 Signature-Version: 1.0
 Created-By: 1.0 (Android)
 SHA1-Digest-Manifest: wxqnEAI0UA5nO5QJ8CGMwjkGGWE=
 ...
 Name: res/layout/exchange_component_back_bottom.xml
 SHA1-Digest: eACjMjESj7Zkf0cBFTZ0nqWrt7w=
 ...
 Name: res/drawable-hdpi/icon.png
 SHA1-Digest: DGEqylP8W0n0iV/ZzBx3MW0WGCA=
</pre>
<ul>
<li><code>lib</code>: the directory containing the compiled code that is specific to a software layer of a processor, the directory is split into more directories within it:
<ul>
<li><code>armeabi</code>: compiled code for all <a href="/wiki/ARM_architecture" title="ARM architecture">ARM</a> based processors only</li>
<li><code>armeabi-v7a</code>: compiled code for all ARMv7 and above based processors only</li>
<li><code>arm64-v8a</code>: compiled code for all ARMv8 arm64 and above based processors only<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup><sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></li>
<li><code>x86</code>: compiled code for <a href="/wiki/X86" title="X86">x86</a> processors only</li>
<li><code>x86_64</code>: compiled code for <a class="mw-redirect" href="/wiki/X86_64" title="X86 64">x86 64</a> processors only</li>
<li><code>mips</code>: compiled code for <a class="mw-redirect" href="/wiki/MIPS_architecture" title="MIPS architecture">MIPS</a> processors only</li>
</ul>
</li>
<li><code>res</code>: the directory containing resources not compiled into resources.arsc (see below).</li>
<li><code>assets</code>: a directory containing applications assets, which can be retrieved by <code>AssetManager</code>.</li>
<li><code>AndroidManifest.xml</code>: An additional Android manifest file, describing the name, version, access rights, referenced library files for the application. This file may be in Android <a href="/wiki/Binary_XML" title="Binary XML">binary XML</a> that can be converted into human-readable plaintext XML with tools such as <a class="external text" href="https://code.google.com/p/android4me/downloads/list" rel="nofollow">AXMLPrinter2</a>, <a class="external text" href="https://code.google.com/p/android-apktool/" rel="nofollow">android-apktool</a>, or <a class="external text" href="https://code.google.com/p/androguard/wiki/Usage#Androaxml" rel="nofollow">Androguard</a>.</li>
<li><code>classes.dex</code>: The classes compiled in the <a class="mw-redirect" href="/wiki/DEX_(file_format)" title="DEX (file format)">dex file format</a> understandable by the <a href="/wiki/Dalvik_(software)" title="Dalvik (software)">Dalvik virtual machine</a></li>
<li><code>resources.arsc</code>: a file containing precompiled resources, such as binary XML for example.</li>
</ul>
