<p>An <b>ISO image</b> is an <a href="/wiki/Archive_file" title="Archive file">archive file</a> of an <a href="/wiki/Optical_disc" title="Optical disc">optical disc</a>, a type of <a href="/wiki/Disk_image" title="Disk image">disk image</a> composed of the data contents from every written <a href="/wiki/Disk_sector" title="Disk sector">sector</a> on an <a href="/wiki/Optical_disc" title="Optical disc">optical disc</a>, including the optical disc <a href="/wiki/File_system" title="File system">file system</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> ISO image files usually have a <a class="mw-redirect" href="/wiki/File_extension" title="File extension">file extension</a> of <i>.iso</i>. The name <i>ISO</i> is taken from the <a href="/wiki/ISO_9660" title="ISO 9660">ISO 9660</a> file system used with <a href="/wiki/CD-ROM" title="CD-ROM">CD-ROM</a> media, but what is known as an ISO image might also contain a <a href="/wiki/Universal_Disk_Format" title="Universal Disk Format">UDF</a> (ISO/IEC 13346) file system (commonly used by <a href="/wiki/DVD" title="DVD">DVDs</a> and <a class="mw-redirect" href="/wiki/Blu-ray_Disc" title="Blu-ray Disc">Blu-ray Discs</a>).</p>
<p>ISO images can be created from optical discs by <a href="/wiki/Comparison_of_disc_image_software" title="Comparison of disc image software">disk imaging software</a>, or from a collection of <a href="/wiki/Computer_file" title="Computer file">files</a> by <a href="/wiki/List_of_optical_disc_authoring_software" title="List of optical disc authoring software">optical disc authoring software</a>, or from a different <a href="/wiki/Disk_image#File_formats" title="Disk image">disk image file</a> by means of <a href="/wiki/Data_conversion" title="Data conversion">conversion</a>. Software distributed on bootable discs is often available for download in ISO image format. And like any other ISO image, it may be written to an optical disc such as CD or DVD.</p>

<h2><span class="mw-headline" id="Description">Description</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ISO_image&amp;action=edit&amp;section=1" title="Edit section: Description">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>There is no standard definition for ISO image files. ISO <a class="mw-redirect" href="/wiki/Disc_image" title="Disc image">disc images</a> are uncompressed and do not use a particular container format; they are a <a href="/wiki/Disk_sector" title="Disk sector">sector</a>-by-sector copy of the data on an optical disc, stored inside a binary file. ISO images are expected to contain the binary image of an optical media <a href="/wiki/File_system" title="File system">file system</a> (usually <a href="/wiki/ISO_9660" title="ISO 9660">ISO 9660</a> and its extensions or <a href="/wiki/Universal_Disk_Format" title="Universal Disk Format">UDF</a>), including the data in its files in binary format, copied exactly as they were stored on the disc. The data inside the ISO image will be structured according to the file system that was used on the optical disc from which it was created.</p>
<p>ISO files store only the user data from each sector on an optical disc, ignoring the <a href="/wiki/CD-ROM#CD-ROM_format" title="CD-ROM">control headers</a> and error correction data, and are therefore slightly smaller than a raw disc image of optical media. Since the size of the user data portion of a sector (logical sector) in data optical discs is 2,048 bytes, the size of an ISO image will be a multiple of 2,048.</p>
<p>The <i>.iso</i> <a class="mw-redirect" href="/wiki/File_extension" title="File extension">file extension</a> is the one most commonly used for this type of disc images. The <i>.img</i> extension can also be found on some ISO image files, such as in some images from Microsoft <a class="mw-redirect" href="/wiki/DreamSpark" title="DreamSpark">DreamSpark</a>; however, <a href="/wiki/IMG_(file_format)" title="IMG (file format)">IMG files</a>, which also use the <i>.img</i> extension, tend to have slightly different contents. The <i>.udf</i> file extension is sometimes used to indicate that the file system inside the ISO image is actually UDF and not ISO 9660.</p>
<p>Any single-<a href="/wiki/Track_(CD)" title="Track (CD)">track</a> <a href="/wiki/CD-ROM" title="CD-ROM">CD-ROM</a>, <a href="/wiki/DVD" title="DVD">DVD</a> or <a href="/wiki/Blu-ray" title="Blu-ray">Blu-ray</a> disc can be archived in ISO format as a true digital copy of the original. Unlike a physical optical disc, an image can be transferred over any data link or removable storage medium. An ISO image can be opened with almost every multi-format <a href="/wiki/File_archiver" title="File archiver">file archiver</a>. Native support for handling ISO images varies from operating system to operating system.</p>
<p><a href="/wiki/Hybrid_disc" title="Hybrid disc">Hybrid disc</a> formats include the ability to be read by different devices, operating systems, or hardware. In the past, one example of this use was for a disc that supported both <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> and <a href="/wiki/Macintosh" title="Macintosh">Macintosh</a> installations from a single disk image (by containing several file systems).</p>
<p>An ISO can be "mounted" with suitable <a href="/wiki/Device_driver" title="Device driver">driver</a> software, i.e. treated by the operating system as if it were a physical optical disc. Most <a href="/wiki/Unix" title="Unix">Unix</a>-based operating systems, including <a href="/wiki/Linux" title="Linux">Linux</a> and <a class="mw-redirect" href="/wiki/Mac_OS_X" title="Mac OS X">Mac OS X</a>, have built-in capability to mount an ISO. <a href="/wiki/Windows_8" title="Windows 8">Windows 8</a> also has such capability.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> For other operating systems software drivers can be installed to achieve the same objective.</p>
<p>Since there is no standard defining the ISO disc image file format, the term "ISO image" is sometimes used in a broader sense to refer to any disc image file of an optical disc, independent of the format it uses.</p>
<h2><span class="mw-headline" id="Limitations">Limitations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ISO_image&amp;action=edit&amp;section=2" title="Edit section: Limitations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A CD can have multiple <a href="/wiki/Track_(CD)" title="Track (CD)">tracks</a>, which can contain computer data, audio, or video. <a href="/wiki/File_system" title="File system">File systems</a> such as <a href="/wiki/ISO_9660" title="ISO 9660">ISO 9660</a> are stored inside one of these tracks. Since ISO images are expected to contain a binary copy of the file system and its contents, there is no concept of a "track" inside an ISO image, since a track is a container for the contents of an ISO image. This means that CDs with multiple tracks can't be stored inside a single ISO image; at most, an ISO image will contain the data inside one of those multiple tracks, and only if it is stored inside a standard file system.</p>
<p>This also means that <a href="/wiki/Compact_Disc_Digital_Audio" title="Compact Disc Digital Audio">audio CDs</a>, which are usually composed of multiple tracks, can't be stored inside an ISO image. Furthermore, not even a single track of an audio CD can be stored as an ISO image, since audio tracks do not contain a file system inside them, but only a continuous stream of encoded audio data. This audio is stored on <a href="/wiki/Track_(CD)#Subchannels" title="Track (CD)">sectors of 2352 bytes</a> different than those that store a file system and it is not stored inside files; it is addressed with <i>track numbers</i>, <i>index points</i> and a <i>CD time code</i> that are encoded into the <a class="mw-redirect" href="/wiki/Lead-in_(CD)" title="Lead-in (CD)">lead-in</a> of each session of the CD-Audio disc.</p>
<p><a href="/wiki/Video_CD" title="Video CD">Video CDs</a> and <a href="/wiki/Super_Video_CD" title="Super Video CD">Super Video CDs</a> require at least two tracks on a CD, so it is also not possible to store an image of one of these discs inside an ISO image file.</p>
<p>Formats such as <a href="/wiki/Cue_sheet_(computing)" title="Cue sheet (computing)">CUE/BIN</a>, <a href="/wiki/CloneCD_Control_File" title="CloneCD Control File">CCD/IMG</a> and <a class="mw-redirect" href="/wiki/MDF_and_MDS_file_pair" title="MDF and MDS file pair">MDS/MDF</a> formats can be used to store multi-track disc images, including audio CDs. These formats store a raw disc image of the complete disc, including information from all tracks, along with a companion file describing the multiple tracks and the characteristics of each of those tracks. This would allow an optical media burning tool to have all the information required to correctly burn the image on a new disc. For audio CDs, one can also transfer the audio data into uncompressed audio files like <a href="/wiki/WAV" title="WAV">WAV</a> or <a class="mw-redirect" href="/wiki/AIFF" title="AIFF">AIFF</a>, optionally reserving the metadata (see <a class="mw-redirect" href="/wiki/CD_ripping" title="CD ripping">CD ripping</a>).</p>
<p>Most software that is capable of writing from ISO images to hard disks or recordable media (CD / DVD / BD) is generally not able to write from ISO disk images to <a href="/wiki/USB_flash_drive" title="USB flash drive">flash drives</a>. This limitation is more related to the availability of software tools able to perform this task, than to problems in the format itself. However, since 2011, various software has existed to write raw image files to USB flash drives.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
