<p><b>Common Development and Distribution License</b> (<b>CDDL</b>) is a <a class="mw-redirect" href="/wiki/Free_and_open-source" title="Free and open-source">Free and open-source</a> <a href="/wiki/Software_license" title="Software license">software license</a>,<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> produced by <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a>, based on the <a href="/wiki/Mozilla_Public_License" title="Mozilla Public License">Mozilla Public License</a> (MPL). Files licensed under the CDDL can be combined with files licensed under other licenses, whether open source or proprietary.<sup class="reference" id="cite_ref-osi_2-1"><a href="#cite_note-osi-2">[2]</a></sup> In 2005 the <a href="/wiki/Open_Source_Initiative" title="Open Source Initiative">Open Source Initiative</a> approved the license. The <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a> (FSF) considers it a <a href="/wiki/Free_software" title="Free software">free software</a> license, but one which is <a href="/wiki/License_compatibility" title="License compatibility">incompatible</a> with the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a> (GPL).<sup class="reference" id="cite_ref-fsf_1-4"><a href="#cite_note-fsf-1">[1]</a></sup> In 2016 according to Black Duck Software<sup class="reference" id="cite_ref-blackduck2016_4-0"><a href="#cite_note-blackduck2016-4">[4]</a></sup> the CDDL is the 15th most popular FOSS license, before the <a href="/wiki/GNU_Affero_General_Public_License" title="GNU Affero General Public License">AGPL</a> v3.</p>

<h2><span class="mw-headline" id="Terms">Terms</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Development_and_Distribution_License&amp;action=edit&amp;section=1" title="Edit section: Terms">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Derived from the <a href="/wiki/Mozilla_Public_License" title="Mozilla Public License">Mozilla Public License</a> 1.1,<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> the CDDL tries to address some of the problems of the MPL.<sup class="reference" id="cite_ref-whyCDDL_6-0"><a href="#cite_note-whyCDDL-6">[6]</a></sup> Like the MPL, the CDDL is a weak <a href="/wiki/Copyleft" title="Copyleft">copyleft</a> license in-between <a class="mw-redirect" href="/wiki/GPL" title="GPL">GPL</a> license and <a class="mw-redirect" href="/wiki/BSD_license" title="BSD license">BSD</a>/<a class="mw-redirect" href="/wiki/MIT_license" title="MIT license">MIT</a> <a class="mw-redirect" href="/wiki/Permissive_license" title="Permissive license">permissive licenses</a>, requiring only source code files under CDDL to remain under CDDL. Unlike strong <a href="/wiki/Copyleft" title="Copyleft">copyleft</a> licenses like the GPL, mixing of CDDL licensed source code files with source code files under other licenses is permitted without relicensing. The resulting compiled software product ("binary") can be licensed and sold under a different license, as long as the source code is still available under CDDL, which should enable more commercial business cases, according to Sun.<sup class="reference" id="cite_ref-whyCDDL_6-1"><a href="#cite_note-whyCDDL-6">[6]</a></sup><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup><sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> Like the MPL the CDDL includes a patent grant to the licensee from all contributors ("patent peace").</p>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Development_and_Distribution_License&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The previous <a href="/wiki/Software_license" title="Software license">software license</a> used by <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun</a> for its <a class="mw-redirect" href="/wiki/Open_source_software" title="Open source software">open source</a> projects was the <a href="/wiki/Sun_Public_License" title="Sun Public License">Sun Public License</a> (SPL), also derived from the <a href="/wiki/Mozilla_Public_License" title="Mozilla Public License">Mozilla Public License</a>. The CDDL license is considered by <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun</a> (now <a href="/wiki/Oracle_Corporation" title="Oracle Corporation">Oracle</a>) to be <a href="/wiki/Sun_Public_License" title="Sun Public License">SPL</a> version 2.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
<p>The CDDL was developed by a <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a> team (among them <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">Solaris</a> kernel engineer Andrew Tucker<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup><sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> and Claire Giordano<sup class="reference" id="cite_ref-approval_12-0"><a href="#cite_note-approval-12">[12]</a></sup>), based on the <a href="/wiki/Mozilla_Public_License" title="Mozilla Public License">MPL</a> version 1.1. On December 1, 2004 the CDDL was submitted for approval to the <a href="/wiki/Open_Source_Initiative" title="Open Source Initiative">Open Source Initiative</a><sup class="reference" id="cite_ref-approval_12-1"><a href="#cite_note-approval-12">[12]</a></sup> and was approved as an <a class="mw-redirect" href="/wiki/Open_source_license" title="Open source license">open source license</a> in mid January 2005. The second CDDL proposal, submitted in early January 2005, includes some corrections that prevent the CDDL from being in conflict with European Copyright law and to allow single developers to use the CDDL for their work.</p>
<p>In 2006 in the first draft of the OSI's <a href="/wiki/License_proliferation" title="License proliferation">license proliferation</a> committee report, the CDDL is one of nine preferred licenses listed as popular, widely used, or with strong communities.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></p>
<p>While the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a> (FSF) also considered the CDDL a <a href="/wiki/Free_software" title="Free software">free software</a> license, they saw some <a href="/wiki/License_compatibility" title="License compatibility">incompatibilities</a> with their <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a> (GPL).<sup class="reference" id="cite_ref-fsf_1-5"><a href="#cite_note-fsf-1">[1]</a></sup></p>
<h3><span class="mw-headline" id="GPL_compatibility"><span id="GPL-INCOMPATIBILITY"></span>GPL compatibility</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Development_and_Distribution_License&amp;action=edit&amp;section=3" title="Edit section: GPL compatibility">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The question of whether and when both licenses are <a href="/wiki/License_compatibility" title="License compatibility">incompatible</a> sparked debates in the free software domain in 2004 to 2006.<sup class="reference" id="cite_ref-svn_14-0"><a href="#cite_note-svn-14">[14]</a></sup><sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup></p>
<p>For instance the FSF considered the CDDL incompatible to their GPL license (without going to details).<sup class="reference" id="cite_ref-fsf_1-6"><a href="#cite_note-fsf-1">[1]</a></sup></p>
<p>Some describe the incompatibility as inherited from the MPL 1.1 (fixed with the MPL 2.0 according to the FSF<sup class="reference" id="cite_ref-fsf_1-7"><a href="#cite_note-fsf-1">[1]</a></sup>) and as a complex interaction of several clauses.<sup class="reference" id="cite_ref-svn_14-1"><a href="#cite_note-svn-14">[14]</a></sup><sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></p>
<p>Some of the CDDL proponents describe the GPL/CDDL compatibility situation from another point of view, they see the problem more on the GPL side than the CDDL side.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<p>Some people argue that Sun (or the Sun engineer) as creator of the license made the CDDL intentionally GPL incompatible.<sup class="reference" id="cite_ref-svn_14-2"><a href="#cite_note-svn-14">[14]</a></sup> According to <a href="/wiki/Danese_Cooper" title="Danese Cooper">Danese Cooper</a> one of the reasons for basing the CDDL on the Mozilla license was that the Mozilla license is <a href="/wiki/License_compatibility" title="License compatibility">GPL-incompatible</a>. Cooper stated, at the 6th annual <a class="mw-redirect" href="/wiki/Debconf" title="Debconf">Debian conference</a>, that the engineers who had written the Solaris kernel requested that the license of OpenSolaris be GPL-incompatible.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup></p>
<blockquote class="templatequote">
<p>Mozilla was selected partially because it is GPL incompatible. That was part of the design when they released OpenSolaris. [...] the engineers who wrote Solaris [...] had some biases about how it should be released, and you have to respect that.</p>
</blockquote>
<p><a href="/wiki/Simon_Phipps_(programmer)" title="Simon Phipps (programmer)">Simon Phipps</a> (Sun's Chief Open Source Officer at the time), who had introduced Ms. Cooper as "the one who actually wrote the CDDL",<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup> did not immediately comment, but later in the same video, he says, referring back to the license issue, "I actually disagree with Danese to some degree",<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> while describing the strong preference among the engineers who wrote the code for a BSD-like license, which was in conflict with Sun's preference for something <a href="/wiki/Copyleft" title="Copyleft">copyleft</a>, and that waiting for legal clearance to release some parts of the code under the then unreleased GNU GPL v3 would have taken several years, and would probably also have involved massed resignations from engineers (unhappy with either the delay, the GPL, or both—this is not clear from the video). Later, in September 2006, Phipps rejected Cooper's assertion in even stronger terms.<sup class="reference" id="cite_ref-forumpost_21-0"><a href="#cite_note-forumpost-21">[21]</a></sup> Similarly, <a href="/wiki/Bryan_Cantrill" title="Bryan Cantrill">Bryan Cantrill</a>, who was on Sun at that time and involved in the release of CDDL licensed software stated in 2015 that he and his colleagues expected in 2006 the fast emerge of CDDL licensed software into the Linux ecosystem and the CDDL being not an obstacle.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup></p>
<h3><span class="mw-headline" id="cdrtools_controversy">cdrtools controversy</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Development_and_Distribution_License&amp;action=edit&amp;section=4" title="Edit section: cdrtools controversy">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The GPL compatibility question was also the source of a controversy behind a partial relicensing of <a href="/wiki/Cdrtools" title="Cdrtools">cdrtools</a> to the CDDL which had been previously all GPL. In 2006 the <a href="/wiki/Debian" title="Debian">Debian</a> project declared the cdrtools legally undistributable because the <a class="mw-redirect" href="/wiki/Build_system" title="Build system">build system</a> was licensed under the CDDL.<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup> The author Jörg Schilling claims <a class="new" href="/w/index.php?title=Smake&amp;action=edit&amp;redlink=1" title="Smake (page does not exist)">smake</a> to be an independent project and therefore not violating the <a class="mw-redirect" href="/wiki/GPLv3" title="GPLv3">GPLv3</a>.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup> And that even though the GPL requires all scripts required to build the work to be licensed freely but not necessarily under the GPL,<sup class="reference" id="cite_ref-GNU_GPL_25-0"><a href="#cite_note-GNU_GPL-25">[25]</a></sup><sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup><sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citing_sources" title="Wikipedia:Citing sources"><span title="This citation requires a reference to the specific page or range of pages in which the material appears. (February 2016)">page needed</span></a></i>]</sup> thus not causing an incompatibility that <a href="/wiki/Copyright_infringement" title="Copyright infringement">violates the license</a>. He argues, in "combined works" (in contrast to "<a href="/wiki/Derivative_work" title="Derivative work">derived works</a>") GPL and CDDL licensed code is compatible.<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup><sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup></p>
<h3><span class="mw-headline" id="CDDL.27d_ZFS_into_GPL.27d_Linux_kernel">CDDL'd ZFS into GPL'd Linux kernel</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Development_and_Distribution_License&amp;action=edit&amp;section=5" title="Edit section: CDDL'd ZFS into GPL'd Linux kernel">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In 2015 the CDDL to GPL compatibility question reemerged when the <a href="/wiki/Linux_distribution" title="Linux distribution">Linux distribution</a> <a href="/wiki/Ubuntu_(operating_system)" title="Ubuntu (operating system)">Ubuntu</a> announced to include <a href="/wiki/OpenZFS" title="OpenZFS">OpenZFS</a> by default.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> In 2016 Ubuntu announced that a legal review resulted in the conclusion that it is legally acceptable to use ZFS as binary <a class="mw-redirect" href="/wiki/Kernel_module" title="Kernel module">kernel module</a> in Linux.<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup> Others followed Ubuntu's conclusion, for instance lawyer James E.J. Bottomley argued there can't be <i>"a convincing theory of harm"</i> developed making it impossible to bring the case to court.<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup> <a href="/wiki/Eben_Moglen" title="Eben Moglen">Eben Moglen</a>, co-author of the <a class="mw-redirect" href="/wiki/GPLv3" title="GPLv3">GPLv3</a> and founder of the <a href="/wiki/Software_Freedom_Law_Center" title="Software Freedom Law Center">SFLC</a>, argued that while the letters of the GPL might be violated, the spirit of both licenses is unharmed, which would be the relevant aspect in court.<sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup> The SFLC mentioned also that precedent exist, the <a href="/wiki/Andrew_File_System" title="Andrew File System">Andrew File System</a>'s kernel module, which isn't considered a derivative work of the GPL'd kernel by the kernel developers.<sup class="reference" id="cite_ref-moglen2_33-0"><a href="#cite_note-moglen2-33">[33]</a></sup><sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup> On the other hand, <a href="/wiki/Bradley_M._Kuhn" title="Bradley M. Kuhn">Bradley M. Kuhn</a> and lawyer<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup> <a class="mw-redirect" href="/wiki/Karen_M._Sandler" title="Karen M. Sandler">Karen M. Sandler</a> from the <a href="/wiki/Software_Freedom_Conservancy" title="Software Freedom Conservancy">Software Freedom Conservancy</a> argued that Ubuntu would violate both licenses as a binary ZFS module would be a derivative work of the Linux kernel and announced their intent to achieve clarity in this question, even by court.<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup><sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup> In April 2016, the <a href="/wiki/Ubuntu_(operating_system)" title="Ubuntu (operating system)">Ubuntu</a> 16.04 <a href="/wiki/Long-term_support" title="Long-term support">LTS</a> release included the CDDL-licensed <a class="mw-redirect" href="/wiki/ZFS_on_Linux" title="ZFS on Linux">ZFS on Linux</a>, as announced before.<sup class="reference" id="cite_ref-38"><a href="#cite_note-38">[38]</a></sup></p>
