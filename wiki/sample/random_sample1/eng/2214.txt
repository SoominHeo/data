<h1 class="firstHeading" id="firstHeading" lang="en">Common Intermediate Language</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><div class="hatnote" role="note">For the counterpart to compiled assembly in the <a href="/wiki/Common_Language_Infrastructure" title="Common Language Infrastructure">Common Language Infrastructure</a>, see <a href="/wiki/Assembly_(CLI)" title="Assembly (CLI)">assembly (CLI)</a>.</div>
<p><b>Common Intermediate Language</b> (<b>CIL</b>, pronounced either "sil" or "kil"), formerly called <b>Microsoft Intermediate Language</b> or <b>MSIL</b>, is the lowest-level <a class="mw-redirect" href="/wiki/Human-readable" title="Human-readable">human-readable</a> <a href="/wiki/Programming_language" title="Programming language">programming language</a> defined by the <a href="/wiki/Common_Language_Infrastructure" title="Common Language Infrastructure">Common Language Infrastructure</a> (CLI) specification and is used by the <a href="/wiki/.NET_Framework" title=".NET Framework">.NET Framework</a> and <a href="/wiki/Mono_(software)" title="Mono (software)">Mono</a>. Languages which target a <a href="/wiki/Common_Language_Infrastructure" title="Common Language Infrastructure">CLI</a>-compatible runtime environment compile to CIL, which is assembled into an <a href="/wiki/Object_code" title="Object code">object code</a> that has a <a href="/wiki/Bytecode" title="Bytecode">bytecode</a>-style format. CIL is an <a class="mw-redirect" href="/wiki/Object-oriented" title="Object-oriented">object-oriented</a> <a href="/wiki/Assembly_language" title="Assembly language">assembly language</a>, and is entirely <a href="/wiki/Stack_machine" title="Stack machine">stack-based</a>. Its bytecode is translated into <a class="mw-redirect" href="/wiki/Native_code" title="Native code">native code</a> or — most commonly — executed by a <a href="/wiki/Virtual_machine" title="Virtual machine">virtual machine</a>.</p>
<p>CIL was originally known as Microsoft Intermediate Language (MSIL) during the beta releases of the .NET languages. Due to standardization of <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a> and the <a href="/wiki/Common_Language_Infrastructure" title="Common Language Infrastructure">Common Language Infrastructure</a>, the bytecode is now officially known as CIL.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p>In an unrelated usage, CIL also refers to the <a class="mw-redirect" href="/wiki/C_Intermediate_Language" title="C Intermediate Language">C Intermediate Language</a>, a simplified transformation of C used for further analysis.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p></p>

<h2><span class="mw-headline" id="General_information">General information</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Intermediate_Language&amp;action=edit&amp;section=1" title="Edit section: General information">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>During compilation of <a href="/wiki/List_of_CLI_languages" title="List of CLI languages">CLI programming languages</a>, the <a href="/wiki/Source_code" title="Source code">source code</a> is translated into CIL code rather than into platform- or processor-specific <a href="/wiki/Object_file" title="Object file">object code</a>. CIL is a <a class="mw-redirect" href="/wiki/CPU" title="CPU">CPU</a>- and platform-independent instruction set that can be executed in any environment supporting the Common Language Infrastructure,<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> such as the <a href="/wiki/Common_Language_Runtime" title="Common Language Runtime">.NET runtime</a> on <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a>, or the <a href="/wiki/Cross-platform" title="Cross-platform">cross-platform</a> <a href="/wiki/Mono_(software)" title="Mono (software)">Mono</a> runtime. In theory, this eliminates the need to distribute different executable files for different platforms and CPU types. CIL code is verified for safety during runtime, providing better security and reliability than natively compiled executable files.</p>
<p>The execution process looks like this:</p>
<ol>
<li>Source code is converted to CIL i.e. Common Intermediate Language, which is the CLI's equivalent to assembly language for a CPU.</li>
<li>CIL is then assembled into a form of so-called <a href="/wiki/Bytecode" title="Bytecode">bytecode</a> and a <a href="/wiki/Assembly_(CLI)" title="Assembly (CLI)">CLI assembly</a> is created.</li>
<li>Upon execution of a CLI assembly, its code is passed through the runtime's <a href="/wiki/Just-in-time_compilation" title="Just-in-time compilation">JIT compiler</a> to generate native code. Ahead-of-time compilation may also be used, which eliminates this step, but at the cost of executable-file portability.</li>
<li>The computer's processor executes the native code.</li>
</ol>
<h2><span class="mw-headline" id="Instructions">Instructions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Intermediate_Language&amp;action=edit&amp;section=2" title="Edit section: Instructions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">See also: <a href="/wiki/List_of_CIL_instructions" title="List of CIL instructions">List of CIL instructions</a></div>
<p>CIL bytecode has <a class="mw-redirect" href="/wiki/Instruction_(computer_science)" title="Instruction (computer science)">instructions</a> for the following groups of tasks:</p>
<ul>
<li>Load and store</li>
<li><a href="/wiki/Arithmetic" title="Arithmetic">Arithmetic</a></li>
<li><a href="/wiki/Type_conversion" title="Type conversion">Type conversion</a></li>
<li><a class="mw-redirect" href="/wiki/Dynamic_memory_allocation" title="Dynamic memory allocation">Object creation and manipulation</a></li>
<li><a class="mw-redirect" href="/wiki/Stack_(data_structure)" title="Stack (data structure)">Operand stack management (push / pop)</a></li>
<li><a href="/wiki/Branch_(computer_science)" title="Branch (computer science)">Control transfer (branching)</a></li>
<li><a href="/wiki/Subroutine" title="Subroutine">Method invocation and return</a></li>
<li><a href="/wiki/Exception_handling" title="Exception handling">Throwing exceptions</a></li>
<li><a href="/wiki/Monitor_(synchronization)" title="Monitor (synchronization)">Monitor-based concurrency</a></li>
<li>Data and function pointers manipulation needed for C++/CLI and unsafe C# code</li>
</ul>
<h2><span class="mw-headline" id="Computational_model">Computational model</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Intermediate_Language&amp;action=edit&amp;section=3" title="Edit section: Computational model">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The Common Intermediate Language is object-oriented and <a class="mw-redirect" href="/wiki/Stack-based" title="Stack-based">stack-based</a>. That means that data is pushed on a stack instead of pulled from registers as in most <a class="mw-redirect" href="/wiki/CPU_architecture" title="CPU architecture">CPU architectures</a>.</p>
<p>In <a href="/wiki/X86" title="X86">x86</a> it might look like this:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="nf">add</span> <span class="no">eax</span><span class="p">,</span> <span class="no">edx</span>
</pre></div>
<p>The corresponding code in <a class="mw-redirect" href="/wiki/Intermediate_language" title="Intermediate language">IL</a> can be rendered as this:</p>
<pre>
ldloc.0
ldloc.1
add
stloc.0    // a = a + b or a += b;
</pre>
<p>Here are two locals that are pushed on the stack. When the add-instruction is called the operands get popped and the result is pushed. The remaining value is then popped and stored in the first local.</p>
<h3><span class="mw-headline" id="Object-oriented_concepts">Object-oriented concepts</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Intermediate_Language&amp;action=edit&amp;section=4" title="Edit section: Object-oriented concepts">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>This extends to object-oriented concepts as well. You may create objects, call methods and use other types of members such as fields.</p>
<p>CIL is designed to be object-oriented and every <a class="mw-redirect" href="/wiki/Method_(programming)" title="Method (programming)">method</a> needs (with some exceptions) to reside in a class. So does this static method:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="p">.</span><span class="k">class</span> <span class="nc">public</span> <span class="n">Foo</span>
<span class="p">{</span>
    <span class="p">.</span><span class="n">method</span> <span class="k">public</span> <span class="k">static</span> <span class="n">int32</span> <span class="n">Add</span><span class="p">(</span><span class="n">int32</span><span class="p">,</span> <span class="n">int32</span><span class="p">)</span> <span class="n">cil</span> <span class="n">managed</span>
    <span class="p">{</span>
        <span class="p">.</span><span class="n">maxstack</span> <span class="m">2</span>
        <span class="n">ldarg</span><span class="p">.</span><span class="m">0</span> <span class="c1">// load the first argument;</span>
        <span class="n">ldarg</span><span class="p">.</span><span class="m">1</span> <span class="c1">// load the second argument;</span>
        <span class="k">add</span>     <span class="c1">// add them;</span>
        <span class="n">ret</span>     <span class="c1">// return the result;</span>
    <span class="p">}</span>
<span class="p">}</span>
</pre></div>
<p>This method does not require any instance of Foo to be declared because it is static. That means it belongs to the class and it may then be used like this in C#:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kt">int</span> <span class="n">r</span> <span class="p">=</span> <span class="n">Foo</span><span class="p">.</span><span class="n">Add</span><span class="p">(</span><span class="m">2</span><span class="p">,</span> <span class="m">3</span><span class="p">);</span>    <span class="c1">// 5</span>
</pre></div>
<p>In CIL:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">ldc</span><span class="p">.</span><span class="n">i4</span><span class="p">.</span><span class="m">2</span>
<span class="n">ldc</span><span class="p">.</span><span class="n">i4</span><span class="p">.</span><span class="m">3</span>
<span class="n">call</span> <span class="n">int32</span> <span class="n">Foo</span><span class="p">::</span><span class="n">Add</span><span class="p">(</span><span class="n">int32</span><span class="p">,</span> <span class="n">int32</span><span class="p">)</span>
<span class="n">stloc</span><span class="p">.</span><span class="m">0</span>
</pre></div>
<h4><span class="mw-headline" id="Instance_classes">Instance classes</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Intermediate_Language&amp;action=edit&amp;section=5" title="Edit section: Instance classes">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>An instance class contains at least one constructor and some instance members. This class has a set of methods representing actions of a Car-object.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="p">.</span><span class="k">class</span> <span class="nc">public</span> <span class="n">Car</span>
<span class="p">{</span>
    <span class="p">.</span><span class="n">method</span> <span class="k">public</span> <span class="n">specialname</span> <span class="n">rtspecialname</span> <span class="n">instance</span> <span class="k">void</span> <span class="p">.</span><span class="n">ctor</span><span class="p">(</span><span class="n">int32</span><span class="p">,</span> <span class="n">int32</span><span class="p">)</span> <span class="n">cil</span> <span class="n">managed</span>
    <span class="p">{</span>
        <span class="cm">/* Constructor */</span>
    <span class="p">}</span>

    <span class="p">.</span><span class="n">method</span> <span class="k">public</span> <span class="k">void</span> <span class="n">Move</span><span class="p">(</span><span class="n">int32</span><span class="p">)</span> <span class="n">cil</span> <span class="n">managed</span>
    <span class="p">{</span>
        <span class="cm">/* Omitting implementation */</span>
    <span class="p">}</span>

    <span class="p">.</span><span class="n">method</span> <span class="k">public</span> <span class="k">void</span> <span class="n">TurnRight</span><span class="p">()</span> <span class="n">cil</span> <span class="n">managed</span>
    <span class="p">{</span>
        <span class="cm">/* Omitting implementation */</span>
    <span class="p">}</span>

    <span class="p">.</span><span class="n">method</span> <span class="k">public</span> <span class="k">void</span> <span class="n">TurnLeft</span><span class="p">()</span> <span class="n">cil</span> <span class="n">managed</span>
    <span class="p">{</span>
        <span class="cm">/* Omitting implementation */</span>
    <span class="p">}</span>

    <span class="p">.</span><span class="n">method</span> <span class="k">public</span> <span class="k">void</span> <span class="n">Brake</span><span class="p">()</span> <span class="n">cil</span> <span class="n">managed</span>
    <span class="p">{</span>
        <span class="cm">/* Omitting implementation */</span>
    <span class="p">}</span>
<span class="p">}</span>
</pre></div>
<h4><span class="mw-headline" id="Creating_objects">Creating objects</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Intermediate_Language&amp;action=edit&amp;section=6" title="Edit section: Creating objects">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>In C# class instances are created like this:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">Car</span> <span class="n">myCar</span> <span class="p">=</span> <span class="k">new</span> <span class="n">Car</span><span class="p">(</span><span class="m">1</span><span class="p">,</span> <span class="m">4</span><span class="p">);</span> 
<span class="n">Car</span> <span class="n">yourCar</span> <span class="p">=</span> <span class="k">new</span> <span class="n">Car</span><span class="p">(</span><span class="m">1</span><span class="p">,</span> <span class="m">3</span><span class="p">);</span>
</pre></div>
<p>And these statements are roughly the same as these instructions:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">ldc</span><span class="p">.</span><span class="n">i4</span><span class="p">.</span><span class="m">1</span>
<span class="n">ldc</span><span class="p">.</span><span class="n">i4</span><span class="p">.</span><span class="m">4</span>

<h3><span class="mw-headline" id="Metadata">Metadata</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Intermediate_Language&amp;action=edit&amp;section=8" title="Edit section: Metadata">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Metadata_(CLI)" title="Metadata (CLI)">Metadata (CLI)</a></div>
<p>CLI records information about compiled classes as <a href="/wiki/Metadata" title="Metadata">Metadata</a>. Like the type library in the <a href="/wiki/Component_Object_Model" title="Component Object Model">Component Object Model</a>, this enables applications to support and discover the interfaces, classes, types, methods, and fields in the assembly. The process of reading such metadata is called <i><a class="mw-redirect" href="/wiki/Reflection_(computer_science)" title="Reflection (computer science)">reflection</a></i>.</p>
<p>Metadata can be data in the form of <i>attributes</i>. Attributes can be custom made by extending from the <code>Attribute</code> class. This is a very powerful feature. It allows the creator of the class the ability to adorn it with extra information that consumers of the class can use in various meaningful ways depending on the application domain.</p>

<p>This is just a representation of how CIL looks like near VM-level. When compiled the methods are stored in tables and the instructions are stored as bytes inside the assembly, which is a <a href="/wiki/Portable_Executable" title="Portable Executable">Portable Executable</a> (PE).</p>
<h2><span class="mw-headline" id="Generation">Generation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Intermediate_Language&amp;action=edit&amp;section=10" title="Edit section: Generation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A CIL assembly and instructions are generated by either a compiler or a utility called the <i>IL Assembler</i> (<a href="/wiki/ILAsm" title="ILAsm">ILAsm</a>) that is shipped with the execution environment.</p>
<p>Assembled CIL can also be disassembled into code again using the <i>IL Disassembler</i> (ILDASM). There are other tools such as <a href="/wiki/.NET_Reflector" title=".NET Reflector">.NET Reflector</a> that can decompile CIL into a high-level language (e. g. C# or <a href="/wiki/Visual_Basic_.NET" title="Visual Basic .NET">Visual Basic</a>). This makes CIL a very easy target for reverse engineering. This trait is shared with <a href="/wiki/Java_bytecode" title="Java bytecode">Java bytecode</a>. However, there are tools that can <a class="mw-redirect" href="/wiki/Obfuscated_code" title="Obfuscated code">obfuscate</a> the code, and do it so that the code cannot be easily readable but still be runnable.</p>
<h2><span class="mw-headline" id="Execution">Execution</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Intermediate_Language&amp;action=edit&amp;section=11" title="Edit section: Execution">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Just-in-time_compilation">Just-in-time compilation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Intermediate_Language&amp;action=edit&amp;section=12" title="Edit section: Just-in-time compilation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Just-in-time_compilation" title="Just-in-time compilation">Just-in-time compilation</a> (JIT) involves turning the byte-code into code immediately executable by the CPU. The conversion is performed gradually during the program's execution. JIT compilation provides environment-specific optimization, runtime type safety, and assembly verification. To accomplish this, the JIT compiler examines the assembly metadata for any illegal accesses and handles violations appropriately.</p>
<h3><span class="mw-headline" id="Ahead-of-time_compilation">Ahead-of-time compilation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Intermediate_Language&amp;action=edit&amp;section=13" title="Edit section: Ahead-of-time compilation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Common_Language_Infrastructure" title="Common Language Infrastructure">CLI</a>-compatible execution environments also come with the option to do an <a href="/wiki/Ahead-of-time_compilation" title="Ahead-of-time compilation">Ahead-of-time compilation</a> (AOT) of an assembly to make it execute faster by removing the JIT process at runtime.</p>
<p>In the <a href="/wiki/.NET_Framework" title=".NET Framework">.NET Framework</a> there is a special tool called the <a href="/wiki/Native_Image_Generator" title="Native Image Generator">Native Image Generator</a> (NGEN) that performs the AOT. In <a class="mw-redirect" href="/wiki/Mono_(programming)" title="Mono (programming)">Mono</a> there is also an option to do an AOT.</p>
<h2><span class="mw-headline" id="Pointer_instructions_-_C.2B.2B.2FCLI">Pointer instructions - C++/CLI</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Common_Intermediate_Language&amp;action=edit&amp;section=14" title="Edit section: Pointer instructions - C++/CLI">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A huge difference from Java's bytecode is that CIL comes with ldind, stind, ldloca, and many call instructions which are enough for data/function pointers manipulation needed to compile C/C++ code into CIL.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>

<span class="p">};</span>
<span class="kt">void</span> <span class="nf">test_pointer_operations</span><span class="p">(</span><span class="kt">int</span> <span class="n">param</span><span class="p">)</span> <span class="p">{</span>
	<span class="kt">int</span> <span class="n">k</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span>
	<span class="kt">int</span> <span class="o">*</span> <span class="n">ptr</span> <span class="o">=</span> <span class="o">&amp;</span><span class="n">k</span><span class="p">;</span>
	<span class="o">*</span><span class="n">ptr</span> <span class="o">=</span> <span class="mi">1</span><span class="p">;</span>
	<span class="n">ptr</span> <span class="o">=</span> <span class="o">&amp;</span><span class="n">param</span><span class="p">;</span>
	<span class="o">*</span><span class="n">ptr</span> <span class="o">=</span> <span class="mi">2</span><span class="p">;</span>
