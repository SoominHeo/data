<p><b>Portable Game Notation</b> (PGN) is a <a href="/wiki/Plain_text" title="Plain text">plain text</a> <a href="/wiki/Computer" title="Computer">computer</a>-processible format for recording <a href="/wiki/Chess" title="Chess">chess</a> games (both the moves and related <a href="/wiki/Data" title="Data">data</a>), supported by many chess programs.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Portable_Game_Notation&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>PGN was devised around 1993, by Steven J. Edwards, and was first popularized via the <a href="/wiki/Usenet" title="Usenet">Usenet</a> newsgroup rec.games.chess.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<h2><span class="mw-headline" id="Usage">Usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Portable_Game_Notation&amp;action=edit&amp;section=2" title="Edit section: Usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>PGN is structured "for easy <a class="mw-redirect" href="/wiki/Reading_(activity)" title="Reading (activity)">reading</a> and <a href="/wiki/Writing" title="Writing">writing</a> by <a href="/wiki/Human" title="Human">human</a> users and for easy <a href="/wiki/Parsing" title="Parsing">parsing</a> and <a href="/wiki/Generation" title="Generation">generation</a> by <a href="/wiki/Computer_program" title="Computer program">computer programs</a>." The chess moves themselves are given in <a class="mw-redirect" href="/wiki/Algebraic_chess_notation" title="Algebraic chess notation">algebraic chess notation</a>. The usual <a href="/wiki/Filename_extension" title="Filename extension">filename extension</a> is <code>.pgn</code>.</p>
<p>There are two formats in the PGN specification, the "import" format and the "export" format. The import format describes data that may have been prepared by hand, and is intentionally lax; a program that can read PGN data should be able to handle the somewhat lax import format. The export format is rather strict and describes data prepared under program control, similar to a <a href="/wiki/Prettyprint" title="Prettyprint">pretty printed</a> source program reformatted by a <a href="/wiki/Compiler" title="Compiler">compiler</a>. The export format representations generated by different programs on the same computer should be exactly equivalent, byte for byte.</p>
<p>PGN <a class="mw-redirect" href="/wiki/Computer_code" title="Computer code">code</a> begins with a set of "tag pairs" (a tag name and its value), followed by the "movetext" (chess moves with optional commentary).</p>
<h3><span class="mw-headline" id="Tag_pairs">Tag pairs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Portable_Game_Notation&amp;action=edit&amp;section=3" title="Edit section: Tag pairs">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Tag pairs begin with an initial left bracket <code>[</code>, followed by the name of the tag in plain text (<a href="/wiki/ASCII" title="ASCII">ASCII</a>). The tag value is enclosed in double-quotes, and the tag is then terminated with a closing right bracket <code>]</code>. A quote inside a tag value is represented by the backslash immediately followed by a quote. A backslash inside a tag value is represented by two adjacent backslashes. There are no special control codes involving escape characters, or carriage returns and linefeeds to separate the fields, and superfluous embedded spaces (or <i>SPC</i> characters) are usually skipped when parsing.</p>
<p>PGN data for archival storage is required to provide seven <i>bracketed</i> fields, referred to as "tags" and together known as the STR (Seven Tag Roster). In export format, the STR tag pairs must appear before any other tag pairs that may appear, and in this order:</p>
<ol>
<li>Event: the name of the tournament or match event.</li>
<li>Site: the location of the event. This is in <code>City, Region COUNTRY</code> format, where COUNTRY is the three-letter <a href="/wiki/International_Olympic_Committee" title="International Olympic Committee">International Olympic Committee</a> <a href="/wiki/List_of_IOC_country_codes" title="List of IOC country codes">code</a> for the country. An example is <code><a href="/wiki/New_York_City" title="New York City">New York City</a>, <a class="mw-redirect" href="/wiki/New_York_(state)" title="New York (state)">NY</a> <a href="/wiki/United_States" title="United States">USA</a></code>.</li>
<li>Date: the starting date of the game, in <code>YYYY.MM.DD</code> form. <code>??</code> is used for unknown values.</li>
<li>Round: the playing round ordinal of the game within the event.</li>
<li>White: the player of the white pieces, in <code>Lastname, Firstname</code> format.</li>
<li>Black: the player of the black pieces, same format as White.</li>
<li>Result: the result of the game. This can only have four possible values: <code>1-0</code> (White won), <code>0-1</code> (Black won), <code>1/2-1/2</code> (Draw), or <code>*</code> (other, e.g., the game is ongoing).</li>
</ol>
<p>The standard allows for supplementation in the form of other, optional, tag pairs. The more common tag pairs include:</p>
<ul>
<li>Annotator: The person providing notes to the game.</li>
<li>PlyCount: String value denoting total number of half-moves played.</li>
<li>TimeControl: e.g. <code>40/7200:3600</code> (moves per seconds: sudden death seconds)</li>
<li>Time: Time the game started, in <code>HH:MM:SS</code> format, in local clock time.</li>
<li>Termination: Gives more details about the termination of the game. It may be <code>abandoned</code>, <code>adjudication</code> (result determined by third-party adjudication), <code>death</code>, <code>emergency</code>, <code>normal</code>, <code>rules infraction</code>, <code>time forfeit</code>, or <code>unterminated</code>.</li>
<li>Mode: <code>OTB</code> (<a class="mw-redirect" href="/wiki/Over-the-board" title="Over-the-board">over-the-board</a>) <code>ICS</code> (Internet Chess Server)</li>
<li>FEN: The initial position of the chess board, in <a class="mw-redirect" href="/wiki/Forsyth-Edwards_Notation" title="Forsyth-Edwards Notation">Forsyth-Edwards Notation</a>. This is used to record partial games (starting at some initial position). It is also necessary for chess variants such as <a href="/wiki/Chess960" title="Chess960">Chess960</a>, where the initial position is not always the same as traditional chess. If a FEN tag is used, a separate tag pair <code>SetUp</code> must also appear and have its value set to <code>1</code>.</li>
</ul>
<h3><span class="mw-headline" id="Movetext">Movetext</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Portable_Game_Notation&amp;action=edit&amp;section=4" title="Edit section: Movetext">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="thumb tright">
<div class="thumbinner" style="width:244px;"><a class="image" href="/wiki/File:SCD_algebraic_notation.svg"><img alt="" class="thumbimage" data-file-height="246" data-file-width="242" height="246" src="//upload.wikimedia.org/wikipedia/commons/thumb/b/b6/SCD_algebraic_notation.svg/242px-SCD_algebraic_notation.svg.png" width="242"/></a>
<div class="thumbcaption">Chessboard notation</div>
</div>
</div>
<p>The movetext describes the actual moves of the game. This includes move number indicators (numbers followed by either one or three periods; one if the next move is White's move, three if the next move is Black's move) and movetext Standard Algebraic Notation (SAN).</p>
<p>For most moves the SAN consists of the letter abbreviation for the piece, an <code>x</code> if there is a capture, and the two-character algebraic name of the final square the piece moved to. The letter abbreviations are <code>K</code> (<a href="/wiki/King_(chess)" title="King (chess)">king</a>), <code>Q</code> (<a href="/wiki/Queen_(chess)" title="Queen (chess)">queen</a>), <code>R</code> (<a href="/wiki/Rook_(chess)" title="Rook (chess)">rook</a>), <code>B</code> (<a href="/wiki/Bishop_(chess)" title="Bishop (chess)">bishop</a>), and <code>N</code> (<a href="/wiki/Knight_(chess)" title="Knight (chess)">knight</a>). The <a href="/wiki/Pawn_(chess)" title="Pawn (chess)">pawn</a> is given an empty abbreviation in SAN movetext, but in other contexts the abbreviation <code>P</code> is used. The algebraic name of any square is as per usual <a class="mw-redirect" href="/wiki/Algebraic_chess_notation" title="Algebraic chess notation">algebraic chess notation</a>; from white's perspective, the leftmost square closest to white is <code>a1</code>, the rightmost square closest to the white is <code>h1</code>, and the rightmost (from white's perspective) square closest to black side is <code>h8</code>.</p>
<p>In a few cases a more detailed representation is needed to resolve ambiguity; if so, the piece's <a class="mw-redirect" href="/wiki/File_(chess)" title="File (chess)">file</a> letter, numerical <a class="mw-redirect" href="/wiki/Rank_(chess)" title="Rank (chess)">rank</a>, or the exact square is inserted after the moving piece's name (in that order of preference). Thus, <code>Nge2</code> specifies that the knight originally on the g-file moves to e2.</p>
<p>SAN <a class="mw-redirect" href="/wiki/Kingside" title="Kingside">kingside</a> <a href="/wiki/Castling" title="Castling">castling</a> is indicated by the sequence <code>O-O</code>; <a class="mw-redirect" href="/wiki/Queenside" title="Queenside">queenside</a> castling is indicated by the sequence <code>O-O-O</code> (note that these are capital Os, not zeroes, contrary to the <a href="/wiki/FIDE" title="FIDE">FIDE</a> standard for notation). Pawn promotions are notated by <a href="/wiki/Append" title="Append">appending</a> <code>=</code> to the destination square, followed by the piece the pawn is promoted to. For example: <code>e8=Q</code>. If the move is a checking move, <code>+</code> is also appended; if the move is a checkmating move, <code>#</code> is appended instead. For example: <code>e8=Q#</code>.</p>
<p>An <a href="/wiki/Glossary_of_chess#Annotation" title="Glossary of chess">annotator</a> who wishes to suggest alternative moves to those actually played in the game may insert variations enclosed in parentheses. He may also comment on the game by inserting <a href="/wiki/Numeric_Annotation_Glyphs" title="Numeric Annotation Glyphs">Numeric Annotation Glyphs</a> (NAGs) into the movetext. Each NAG reflects a subjective impression of the move preceding the NAG or of the resultant position.</p>
<p>If the game result is anything other than <code>*</code>, the result is repeated at the end of the movetext.</p>
<h3><span class="mw-headline" id="Comments">Comments</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Portable_Game_Notation&amp;action=edit&amp;section=5" title="Edit section: Comments">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Comments are inserted by either a <code>;</code> (a comment that continues to the end of the line) or a <code>{</code> (which continues until a matching <code>}</code>). Comments do not nest.</p>
<h3><span class="mw-headline" id="Example">Example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Portable_Game_Notation&amp;action=edit&amp;section=6" title="Edit section: Example">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Here is the PGN format of the 29th game of the <a href="/wiki/Fischer%E2%80%93Spassky_(1992_match)" title="Fischer–Spassky (1992 match)">1992 match</a> played in <a class="mw-redirect" href="/wiki/Federal_Republic_of_Yugoslavia" title="Federal Republic of Yugoslavia">Yugoslavia</a> between <a href="/wiki/Bobby_Fischer" title="Bobby Fischer">Bobby Fischer</a> and <a href="/wiki/Boris_Spassky" title="Boris Spassky">Boris Spassky</a>:</p>

[Result "1/2-1/2"]

4. Ba4 Nf6 5. O-O Be7 6. Re1 b5 7. Bb3 d6 8. c3 O-O 9. h3 Nb8 10. d4 Nbd7
11. c4 c6 12. cxb5 axb5 13. Nc3 Bb7 14. Bg5 b4 15. Nb1 h6 16. Bh4 c5 17. dxe5
Nxe4 18. Bxe7 Qxe7 19. exd6 Qf6 20. Nbd2 Nxd6 21. Nc4 Nxc4 22. Bxc4 Nb6
23. Ne5 Rae8 24. Bxf7+ Rxf7 25. Nxf7 Rxe1+ 26. Qxe1 Kxf7 27. Qe3 Qg5 28. Qxg5

<h3><span class="mw-headline" id="Handling_chess_variants">Handling chess variants</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Portable_Game_Notation&amp;action=edit&amp;section=7" title="Edit section: Handling chess variants">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Many <a class="mw-redirect" href="/wiki/Chess_variant" title="Chess variant">chess variants</a> can be recorded using PGN, provided the names of the pieces can be limited to one character, usually a letter and not a number. They are typically noted with a tag named "Variant" giving the name of the rules. The term "Variation" must be avoided, as that refers to the name of an opening variation. Note that traditional chess programs can only handle, at most, a few variants. <a class="mw-redirect" href="/wiki/Forsyth-Edwards_Notation" title="Forsyth-Edwards Notation">Forsyth-Edwards Notation</a> (FEN) is used to record the starting position for variants (such as <a href="/wiki/Chess960" title="Chess960">Chess960</a>) which have initial positions other than the orthodox chess initial position.</p>
<p>Plycount is a chess term for the total number of moves in a game, counting each player's move as one. It is an optional part of the standard PGN description of a chess game.</p>
