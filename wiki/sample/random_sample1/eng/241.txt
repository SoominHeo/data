<p><b>Visual Basic</b> is a <a href="/wiki/Third-generation_programming_language" title="Third-generation programming language">third-generation</a> <a href="/wiki/Event-driven_programming" title="Event-driven programming">event-driven programming language</a> and <a href="/wiki/Integrated_development_environment" title="Integrated development environment">integrated development environment</a> (IDE) from <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> for its <a href="/wiki/Component_Object_Model" title="Component Object Model">Component Object Model</a> (COM) programming model first released in 1991 and declared legacy in 2008. Microsoft intended Visual Basic to be relatively easy to learn and use.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> Visual Basic was derived from <a href="/wiki/BASIC" title="BASIC">BASIC</a>, a user-friendly programming language designed for beginners, and it enables the <a href="/wiki/Rapid_application_development" title="Rapid application development">rapid application development (RAD)</a> of <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface (GUI)</a> applications, access to <a href="/wiki/Database" title="Database">databases</a> using <a class="mw-redirect" href="/wiki/Data_Access_Object" title="Data Access Object">Data Access Objects</a>, <a href="/wiki/Remote_Data_Objects" title="Remote Data Objects">Remote Data Objects</a>, or <a class="mw-redirect" href="/wiki/ActiveX_Data_Object" title="ActiveX Data Object">ActiveX Data Objects</a>, and creation of <a href="/wiki/ActiveX" title="ActiveX">ActiveX</a> controls and objects.</p>
<p>A programmer can create an application using the <a href="/wiki/Component-based_software_engineering" title="Component-based software engineering">components</a> provided by the Visual Basic program itself. Over time the community of programmers developed third party components.<sup class="reference" id="cite_ref-ISpliter_3-0"><a href="#cite_note-ISpliter-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup><sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup><sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> Programs written in Visual Basic can also use the <a href="/wiki/Windows_API" title="Windows API">Windows API</a>, which requires external function declarations.</p>
<p>The final release was version 6 in 1998 (now known simply as Visual Basic). On April 8, 2008 Microsoft stopped supporting Visual Basic 6.0 <a href="/wiki/Integrated_development_environment" title="Integrated development environment">IDE</a>. The Microsoft Visual Basic team still maintains compatibility for Visual Basic 6.0 applications on <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a>, <a href="/wiki/Windows_Server_2008" title="Windows Server 2008">Windows Server 2008</a> including R2, <a href="/wiki/Windows_7" title="Windows 7">Windows 7</a>, <a href="/wiki/Windows_8" title="Windows 8">Windows 8</a>, <a href="/wiki/Windows_8.1" title="Windows 8.1">Windows 8.1</a>, <a href="/wiki/Windows_Server_2012" title="Windows Server 2012">Windows Server 2012</a> and <a href="/wiki/Windows_10" title="Windows 10">Windows 10</a> through its "It Just Works" program.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> In 2014, some <a href="/wiki/Software_developer" title="Software developer">software developers</a> still preferred Visual Basic 6.0 over its successor, <a href="/wiki/Visual_Basic_.NET" title="Visual Basic .NET">Visual Basic .NET</a>.<sup class="reference" id="cite_ref-ISpliter_3-1"><a href="#cite_note-ISpliter-3">[3]</a></sup><sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> In 2014 some developers lobbied for a new version of Visual Basic 6.0.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup><sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup><sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup><sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> In 2016, Visual Basic 6.0 won the technical impact award at The 19th Annual D.I.C.E. Awards.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup><sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup><sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> A dialect of Visual Basic, <a href="/wiki/Visual_Basic_for_Applications" title="Visual Basic for Applications">Visual Basic for Applications</a> (VBA), is used as a macro or scripting language within several Microsoft applications, including <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a>.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>

<h2><span class="mw-headline" id="Language_features">Language features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Visual_Basic&amp;action=edit&amp;section=1" title="Edit section: Language features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Like the <a href="/wiki/BASIC" title="BASIC">BASIC</a> programming language, Visual Basic was designed to accommodate a steep <a href="/wiki/Learning_curve" title="Learning curve">learning curve</a>. Programmers can create both simple and complex <a href="/wiki/Graphical_user_interface" title="Graphical user interface">GUI</a> applications. Programming in VB is a combination of visually arranging <a class="mw-redirect" href="/wiki/GUI_widget" title="GUI widget">components</a> or <a class="mw-redirect" href="/wiki/GUI_widget" title="GUI widget">controls</a> on a <a href="/wiki/Form_(programming)" title="Form (programming)">form</a>, specifying attributes and actions for those components, and writing additional lines of <a href="/wiki/Source_code" title="Source code">code</a> for more functionality. Since VB defines default attributes and actions for the components, a programmer can develop a simple program without writing much code. Programs built with earlier versions suffered performance problems, but faster computers and native code compilation has made this less of an issue.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2015)">citation needed</span></a></i>]</sup> Though VB programs can be compiled into native code executables <a href="#Timeline_of_Visual_Basic_.28VB1_to_VB6.29">from version 5 on</a>, they still require the presence of around 1 MB of runtime libraries. Core runtime libraries are included by default in <a href="/wiki/Windows_2000" title="Windows 2000">Windows 2000</a> and later, but extended runtime components still have to be installed. Earlier versions of <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a> (95/98/NT), require that the runtime libraries be distributed with the executable.</p>

<p>Forms are created using <a class="mw-redirect" href="/wiki/Drag-and-drop" title="Drag-and-drop">drag-and-drop</a> techniques. A tool is used to place controls (e.g., text boxes, buttons, etc.) on the form (window). Controls have <a href="/wiki/Attribute_(computing)" title="Attribute (computing)">attributes</a> and <a class="mw-redirect" href="/wiki/Event_handler" title="Event handler">event handlers</a> associated with them. Default values are provided when the control is created, but may be changed by the programmer. Many attribute values can be modified during run time based on user actions or changes in the environment, providing a dynamic application. For example, code can be inserted into the form resize event handler to reposition a control so that it remains centered on the form, expands to fill up the form, etc. By inserting code into the event handler for a keypress in a text box, the program can automatically translate the case of the text being entered, or even prevent certain characters from being inserted.</p>
<p>Visual Basic can create executables (EXE files), <a href="/wiki/ActiveX" title="ActiveX">ActiveX controls</a>, or DLL files, but is primarily used to develop Windows applications and to interface database systems. Dialog boxes with less functionality can be used to provide pop-up capabilities. Controls provide the basic functionality of the application, while programmers can insert additional logic within the appropriate event handlers. For example, a drop-down combination box automatically displays a list. When the user selects an element, an event handler is called that executes code that the programmer created to perform the action for that list item. Alternatively, a Visual Basic component can have no user interface, and instead provide ActiveX objects to other programs via <a href="/wiki/Component_Object_Model" title="Component Object Model">Component Object Model</a> (COM). This allows for <a href="/wiki/Server-side" title="Server-side">server-side</a> processing or an add-in module.</p>
<p>The runtime recovers unused memory using <a href="/wiki/Reference_counting" title="Reference counting">reference counting</a>, which depends on variables passing out of scope or being set to <i>Nothing</i>, avoiding the problem of <a href="/wiki/Memory_leak" title="Memory leak">memory leaks</a> common to other languages. There is a large library of utility objects, and the language provides basic <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object oriented support</a>. Unlike many other programming languages, Visual Basic is generally not case sensitive—though it transforms <a class="mw-redirect" href="/wiki/Keyword_(computer_programming)" title="Keyword (computer programming)">keywords</a> into a standard case configuration and forces the case of variable names to conform to the case of the entry in the symbol table. String comparisons are case sensitive by default. The Visual Basic compiler is shared with other Visual Studio languages (C, C++). Nevertheless, by default the restrictions in the IDE do not allow creation of some targets (Windows model DLLs) and threading models, but over the years, developers have bypassed these restrictions.</p>
<h2><span class="mw-headline" id="Characteristics">Characteristics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Visual_Basic&amp;action=edit&amp;section=2" title="Edit section: Characteristics">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Visual_Basic_function.PNG"><img alt="" class="thumbimage" data-file-height="359" data-file-width="690" height="114" src="//upload.wikimedia.org/wikipedia/commons/thumb/f/fb/Visual_Basic_function.PNG/220px-Visual_Basic_function.PNG" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/f/fb/Visual_Basic_function.PNG/330px-Visual_Basic_function.PNG 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/f/fb/Visual_Basic_function.PNG/440px-Visual_Basic_function.PNG 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Visual_Basic_function.PNG" title="Enlarge"></a></div>
The code windows in Visual Basic, showing a Function using the If, Then, Else and Dim statements.</div>
</div>
</div>
<p>The following Visual Basic traits differ from C-derived languages:</p>
<ul>
<li>Statements tend to terminate with keywords, such as "End If", instead of using "{}"s to group statements.</li>
<li>Multiple variable assignment is not possible. A = B = C does not imply that the values of A, B and C are equal. The boolean result of "Is B = C?" is stored in A. The result stored in A would therefore be either false or true.</li>
<li><a class="mw-redirect" href="/wiki/Boolean_datatype" title="Boolean datatype">Boolean</a> constant <code>True</code> has numeric value −1.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> This is because the Boolean data type is stored as a 16-bit signed integer. In this construct −1 evaluates to 16 binary 1s (the Boolean value <code>True</code>), and 0 as 16 0s (the Boolean value <code>False</code>). This is apparent when performing a <code>Not</code> operation on a 16 bit signed integer value 0, which returns the integer value −1, in other words <code>True = Not False</code>. This inherent functionality becomes especially useful when performing logical operations on the individual bits of an integer such as <code>And</code>, <code>Or</code>, <code>Xor</code> and <code>Not</code>.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup> This definition of <code>True</code> is also consistent with BASIC since the early 1970s Microsoft BASIC implementation and is also related to the characteristics of CPU instructions at the time.</li>
<li>Logical and bitwise operators are unified. This is unlike some C-derived languages (such as <a href="/wiki/Perl" title="Perl">Perl</a>), which have separate logical and bitwise operators. This again is a traditional feature of BASIC.</li>
<li>Variable <a href="/wiki/Array_data_type" title="Array data type">array</a> base. Arrays are declared by specifying the upper and lower bounds in a way similar to <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a> and <a href="/wiki/Fortran" title="Fortran">Fortran</a>. It is also possible to use the Option Base statement to set the default lower bound. Use of the Option Base statement can lead to confusion when reading Visual Basic code and is best avoided by always explicitly specifying the lower bound of the array. This lower bound is not limited to 0 or 1, because it can also be set by declaration. In this way, both the lower and upper bounds are programmable. In more subscript-limited languages, the lower bound of the array is not variable. This uncommon trait does exist in <a href="/wiki/Visual_Basic_.NET" title="Visual Basic .NET">Visual Basic .NET</a> but not in <a href="/wiki/VBScript" title="VBScript">VBScript</a>.</li>
</ul>
<dl>
<dd><code>OPTION BASE</code> was introduced by ANSI, with the standard for ANSI Minimal BASIC in the late 1970s.</dd>
</dl>
<ul>
<li>Relatively strong integration with the <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows operating system</a> and the <a href="/wiki/Component_Object_Model" title="Component Object Model">Component Object Model</a>. The native types for strings and arrays are the dedicated COM types, BSTR and SAFEARRAY.</li>
<li><a class="mw-redirect" href="/wiki/Banker%27s_rounding" title="Banker's rounding">Banker's rounding</a> as the default behavior when converting real numbers to integers with the <code>Round</code> function.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> <code>? Round(2.5, 0)</code> gives 2, <code>? Round(3.5, 0)</code> gives 4.</li>
<li>Integers are automatically promoted to reals in expressions that involve the normal division operator (<code>/</code>) so that division of one integer by another produces the intuitively correct result. VB provides a specific integer divide operator (<code>\</code>) that does truncate.</li>
<li>By default, if a variable has not been declared or if no type declaration character is specified, the variable is of type <code><a href="/wiki/Variant_type" title="Variant type">Variant</a></code>. However this can be changed with Deftype statements such as <code>DefInt</code>, <code>DefBool</code>, <code>DefVar</code>, <code>DefObj</code>, <code>DefStr</code>. There are 12 <code>Deftype</code> statements in total offered by Visual Basic 6.0. The default type may be overridden for a specific declaration by using a special suffix character on the variable name (<code>#</code> for Double, <code>!</code> for Single, <code>&amp;</code> for Long, <code>%</code> for Integer, <code>$</code> for String, and <code>@</code> for Currency) or using the key phrase <code>As (type)</code>. VB can also be set in a mode that only explicitly declared variables can be used with the command <code>Option Explicit</code>.</li>
</ul>

<p>VB 1.0 was introduced in 1991. The drag and drop design for creating the user interface is derived from a prototype form generator developed by <a href="/wiki/Alan_Cooper" title="Alan Cooper">Alan Cooper</a> and his company called <i>Tripod</i>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (June 2014)">citation needed</span></a></i>]</sup><sup class="noprint selfreference">[<i><a href="/wiki/Talk:Visual_Basic#promotional_edits_for_Alan_Cooper" title="Talk:Visual Basic">discuss</a></i>]</sup> Microsoft contracted with Cooper and his associates to develop Tripod into a programmable form system for <a href="/wiki/Windows_3.0" title="Windows 3.0">Windows 3.0</a>, under the code name <i>Ruby</i> (no relation to the later <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby programming language</a>). Tripod did not include a programming language at all. Microsoft decided to combine Ruby with the Basic language to create Visual Basic. The Ruby interface generator provided the "visual" part of Visual Basic and this was combined with the "EB" Embedded BASIC engine designed for Microsoft's abandoned "Omega" database system. Ruby also provided the ability to load <a href="/wiki/Dynamic-link_library" title="Dynamic-link library">dynamic link libraries</a> containing additional controls (then called "gizmos"), which later became the <a href="/wiki/Visual_Basic_Extension" title="Visual Basic Extension">VBX</a> interface.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup></p>

<h4><span class="mw-headline" id="1990s">1990s</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Visual_Basic&amp;action=edit&amp;section=5" title="Edit section: 1990s">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li>Project 'basic Thunder' was initiated in 1990.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup> Thunder persisted through to the last release of Visual Basic in the name of the primary internal function, "ThunderRTMain".</li>
<li>Visual Basic 1.0 (May 1991) was released for Windows at the Comdex/Windows World trade show in Atlanta, Georgia.</li>
<li>Visual Basic 1.0 for <a href="/wiki/DOS" title="DOS">DOS</a> was released in September 1992. The language itself was not quite compatible with Visual Basic for Windows, as it was actually the next version of Microsoft's DOS-based BASIC compilers, <a href="/wiki/QuickBASIC" title="QuickBASIC">QuickBASIC</a> and BASIC Professional Development System. The interface used a <a class="mw-redirect" href="/wiki/Text_user_interface" title="Text user interface">text user interface</a>, using <a href="/wiki/Extended_ASCII" title="Extended ASCII">extended ASCII</a> characters to simulate the appearance of a <a href="/wiki/Graphical_user_interface" title="Graphical user interface">GUI</a>.</li>
<li>Visual Basic 2.0 was released in November 1992. The programming environment was easier to use, and its speed was improved. Notably, forms became instantiable objects, thus laying the foundational concepts of class modules as were later offered in VB4.</li>
<li>Visual Basic 3.0 was released in the summer of 1993 and came in Standard and Professional versions. VB3 included version 1.1 of the <a href="/wiki/Microsoft_Jet_Database_Engine" title="Microsoft Jet Database Engine">Microsoft Jet Database Engine</a> that could read and write Jet (or Access) 1.x databases.</li>
<li>Visual Basic 4.0 (August 1995) was the first version that could create <a href="/wiki/32-bit" title="32-bit">32-bit</a> as well as <a href="/wiki/16-bit" title="16-bit">16-bit</a> Windows programs. It has three editions; Standard, Professional, and Enterprise. It also introduced the ability to write non-GUI classes in Visual Basic. Incompatibilities between different releases of VB4 caused installation and operation problems. While previous versions of Visual Basic had used VBX controls, Visual Basic now used OLE controls (with files names ending in .OCX) instead. These were later to be named ActiveX controls.</li>
<li>With version 5.0 (February 1997), Microsoft released Visual Basic exclusively for <a href="/wiki/32-bit" title="32-bit">32-bit</a> versions of Windows. Programmers who preferred to write 16-bit programs were able to import programs written in Visual Basic 4.0 to Visual Basic 5.0, and Visual Basic 5.0 programs can easily be converted with Visual Basic 4.0. Visual Basic 5.0 also introduced the ability to create custom user controls, as well as the ability to compile to native Windows executable code, speeding up calculation-intensive code execution. A free, downloadable Control Creation Edition was also released for creation of <a href="/wiki/ActiveX" title="ActiveX">ActiveX controls</a>. It was also used as an introductory form of Visual Basic: a regular .exe project could be created and run in the IDE, but not compiled.</li>
<li>Visual Basic 6.0 (Mid-1998) improved in a number of areas<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup> including the ability to create web-based applications.</li>
</ul>
<h4><span class="mw-headline" id="2000s">2000s</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Visual_Basic&amp;action=edit&amp;section=6" title="Edit section: 2000s">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li>Visual Basic 6.0 extended support ended in March 2008; however, primary components of the Visual Basic 6 <i>development</i> environment run in all 32-bit versions of Windows up to and including 8.1.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2015)">citation needed</span></a></i>]</sup></li>
<li>Mainstream Support for Microsoft Visual Basic 6.0 ended on March 31, 2005. Extended support ended in March 2008.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup> In response, the Visual Basic user community expressed its concern and lobbied users to sign a petition to keep the product alive, to no avail.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup></li>
</ul>
<h2><span class="mw-headline" id="Derivative_languages">Derivative languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Visual_Basic&amp;action=edit&amp;section=7" title="Edit section: Derivative languages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> has developed derivatives of Visual Basic for use in <a href="/wiki/Scripting_language" title="Scripting language">scripting</a>. Visual Basic itself is derived heavily from <a href="/wiki/BASIC" title="BASIC">BASIC</a>, and subsequently has been replaced with a <a href="/wiki/.NET_Framework" title=".NET Framework">.NET</a> platform version.</p>
<p>Some of the derived languages are:</p>
<ul>
<li><a href="/wiki/Visual_Basic_for_Applications" title="Visual Basic for Applications">Visual Basic for Applications</a> (VBA) is included in many Microsoft applications (<a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a>), and also in many third-party products like <a href="/wiki/SolidWorks" title="SolidWorks">SolidWorks</a>, <a href="/wiki/AutoCAD" title="AutoCAD">AutoCAD</a>, <a class="mw-redirect" href="/wiki/WordPerfect_Office" title="WordPerfect Office">WordPerfect Office 2002</a>, <a href="/wiki/ArcGIS" title="ArcGIS">ArcGIS</a>, <a class="mw-redirect" href="/wiki/Sage_300_ERP" title="Sage 300 ERP">Sage 300 ERP</a>, and Business Objects Desktop Intelligence. There are small inconsistencies in the way VBA is implemented in different applications, but it is largely the same language as Visual Basic 6.0 and uses the same runtime library. Visual Basic development ended with 6.0, but in 2010 Microsoft introduced VBA 7 to provide extended features and add 64-bit support.<sup class="reference" id="cite_ref-VBA_7_26-0"><a href="#cite_note-VBA_7-26">[26]</a></sup></li>
<li><a href="/wiki/VBScript" title="VBScript">VBScript</a> is the default language for <a href="/wiki/Active_Server_Pages" title="Active Server Pages">Active Server Pages</a>. It can be used in <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a> scripting and client-side <a href="/wiki/Web_page" title="Web page">web page</a> scripting. It resembles VB in <a href="/wiki/Syntax_(programming_languages)" title="Syntax (programming languages)">syntax</a>, but is a separate language—executed by vbscript.dll instead of the VB runtime. ASP and VBScript should not be confused with <a href="/wiki/ASP.NET" title="ASP.NET">ASP.NET</a>, which uses the <a href="/wiki/.NET_Framework" title=".NET Framework">.NET Framework</a> for compiled web pages.</li>
<li><a href="/wiki/Visual_Basic_.NET" title="Visual Basic .NET">Visual Basic .NET</a> is Microsoft's designated successor to Visual Basic 6.0, and is part of Microsoft's <a href="/wiki/.NET_Framework" title=".NET Framework">.NET</a> platform. Visual Basic .NET compiles and runs using the .NET Framework. It is not <a href="/wiki/Backward_compatibility" title="Backward compatibility">backwards compatible</a> with Visual Basic 6.0. An automated conversion tool exists, but fully automated conversion for most projects is impossible.<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup></li>
<li><a href="/wiki/OpenOffice_Basic" title="OpenOffice Basic">OpenOffice Basic</a> is a Visual Basic compatible interpreter that originated in <a href="/wiki/StarOffice" title="StarOffice">StarOffice</a> office suite.</li>
<li><a href="/wiki/Gambas" title="Gambas">Gambas</a> is a Visual Basic inspired <a href="/wiki/Free_software" title="Free software">free software</a> programming language for the Linux operating system. It is not a clone of Visual Basic, but it does have the ability to convert Visual Basic programs to Gambas.</li>
<li><a href="/wiki/WinWrap_Basic" title="WinWrap Basic">WinWrap Basic</a> is a third-party VBA variant used with various software, and available for programmers to use to build a macro facility into their programmes.</li>
<li><a href="/wiki/LotusScript" title="LotusScript">LotusScript</a> is a VBA variant available in <a class="mw-redirect" href="/wiki/Lotus_SmartSuite" title="Lotus SmartSuite">Lotus SmartSuite</a> and <a class="mw-redirect" href="/wiki/Lotus_Notes" title="Lotus Notes">Lotus Notes</a>.</li>
<li>Later versions of <a class="mw-redirect" href="/wiki/Corel_WordPerfect_Office" title="Corel WordPerfect Office">Corel WordPerfect Office</a> implement access to VBA as one of the macro/scripting languages, the other major ones being <a class="new" href="/w/index.php?title=CorelScript&amp;action=edit&amp;redlink=1" title="CorelScript (page does not exist)">CorelScript</a> and <a class="new" href="/w/index.php?title=PerfectScript&amp;action=edit&amp;redlink=1" title="PerfectScript (page does not exist)">PerfectScript</a></li>
<li>Earlier versions of <a href="/wiki/Microsoft_Word" title="Microsoft Word">Microsoft Word</a> use a variant of Visual Basic called <a class="mw-redirect" href="/wiki/WordBasic" title="WordBasic">WordBasic</a></li>
</ul>
<h2><span class="mw-headline" id="Performance_and_other_issues">Performance and other issues</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Visual_Basic&amp;action=edit&amp;section=8" title="Edit section: Performance and other issues">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Earlier versions of Visual Basic (prior to version 5) compiled the code to <a href="/wiki/Microsoft_P-Code" title="Microsoft P-Code">P-Code</a> only. The P-Code is interpreted by the language runtime. The benefits of P-Code include portability and smaller binary file sizes, but it usually slows down the execution, since having a runtime adds an additional layer of interpretation. Visual Basic applications require Microsoft Visual Basic runtime MSVBVMxx.DLL, where xx is the relevant version number, either 50 or 60. MSVBVM60.dll comes as standard with Windows in all editions after Windows 98. A Windows 95 machine would however require inclusion with the installer of whichever dll was needed by the program. Visual Basic 5 and 6 can compile code to either native or P-Code but in either case the runtime is still required for built in functions and forms management.</p>
<p>Criticisms levelled at Visual Basic editions prior to VB.NET include:<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup></p>
<ul>
<li>Versioning problems associated with various runtime <a href="/wiki/Dynamic-link_library" title="Dynamic-link library">DLLs</a>, known as "<a class="mw-redirect" href="/wiki/DLL_hell" title="DLL hell">DLL hell</a>"</li>
<li>Poor support for <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a><sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup></li>
<li>Can only create multi-<a class="mw-redirect" href="/wiki/Thread_(computer_science)" title="Thread (computer science)">threaded</a> applications, using <a class="new" href="/w/index.php?title=ActiveX.exe&amp;action=edit&amp;redlink=1" title="ActiveX.exe (page does not exist)">ActiveX.exe</a></li>
<li><a href="/wiki/Variant_type" title="Variant type">Variant types</a> have a greater performance and storage "overhead" than <a class="mw-redirect" href="/wiki/Strongly_typed_programming_language" title="Strongly typed programming language">strongly typed programming languages</a></li>
<li>Dependency on complex and fragile <a href="/wiki/Component_Object_Model" title="Component Object Model">Component Object Model</a> (COM) Registry entries<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup></li>
</ul>
<h2><span class="mw-headline" id="Legacy_development_and_support">Legacy development and support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Visual_Basic&amp;action=edit&amp;section=9" title="Edit section: Legacy development and support">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>All versions of the Visual Basic development environment from 1.0 to 6.0 were retired by Microsoft by 2008, and are therefore no longer supported. The associated runtime environments are also unsupported, except for the Visual Basic 6 core runtime environment, which Microsoft officially supports for the lifetime of <a href="/wiki/Windows_8" title="Windows 8">Windows 8</a><sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup> and <a href="/wiki/Windows_10" title="Windows 10">Windows 10</a>.<sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup> Third party components that shipped with Visual Studio 6.0 are not included in this support statement. Some legacy Visual Basic components may still work on newer platforms, despite being unsupported by Microsoft and other vendors. Documentation for Visual Basic 6.0, its application programming interface and tools is best covered in the last <a class="mw-redirect" href="/wiki/MSDN" title="MSDN">MSDN</a> release before Visual Studio.NET 2002. Later releases of MSDN focused on .NET development and had significant parts of the Visual Basic 6.0 programming documentation removed as the language evolved, and support for older code ended. Although vendor support for Visual Basic 6 has ended, and the product has never been supported on the latest versions of Windows, key parts of the environment still work on newer platforms. It is possible to get a subset of the development environment working on 32-bit versions of Windows Vista, Windows 7 and Windows 8; the primary limiting factor on 64-bit editions of Windows is the 16-bit setup program.</p>
<h2><span class="mw-headline" id="Example_code">Example code</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Visual_Basic&amp;action=edit&amp;section=10" title="Edit section: Example code">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The following code snippet displays a message box saying "Hello, World!" as the window loads:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">Private</span> <span class="k">Sub</span> <span class="nf">Form_Load</span><span class="p">()</span>
    <span class="c">' Execute a simple message box that says "Hello, World!"</span>
<span class="hll">    <span class="n">MsgBox</span> <span class="s">"Hello, World!"</span>
</span><span class="k">End</span> <span class="k">Sub</span>
</pre></div>
<p>This snippet makes a counter that moves up 1 every second (a label and a timer control need to be added to the form for this to work) until the form is closed or an integer overflow occurs:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
