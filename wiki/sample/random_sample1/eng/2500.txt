<p><b>GNOME Do</b> (often referred to as <b>Do</b>) is a <a class="mw-redirect" href="/wiki/Free_and_open_source_software" title="Free and open source software">free</a> <a class="mw-redirect" href="/wiki/Application_launcher" title="Application launcher">application launcher</a> for <a href="/wiki/Linux" title="Linux">Linux</a> originally created by David Siegel,<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> and currently maintained by Alex Launi. Like other application launchers, it allows searching for applications and files, but it also allows specifying actions to perform on search results. GNOME Do allows for quick finding of miscellaneous artifacts of GNOME environment (applications, Evolution and Pidgin contacts, Firefox bookmarks, Rhythmbox artists and albums, and so on) and execute the basic actions on them (launch, open, email, chat, play, etc.).<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>While it is designed primarily for the <a href="/wiki/GNOME" title="GNOME">GNOME</a> desktop, it works in other <a href="/wiki/Desktop_environment" title="Desktop environment">desktop environments</a>, such as <a href="/wiki/KDE" title="KDE">KDE</a>.</p>
<p>GNOME Do was inspired by <a href="/wiki/Quicksilver_(software)" title="Quicksilver (software)">Quicksilver</a> for <a class="mw-redirect" href="/wiki/Mac_OS_X" title="Mac OS X">Mac OS X</a>, and GNOME Launch Box.<sup class="reference" id="cite_ref-GNOME_Do_Main_Page_3-0"><a href="#cite_note-GNOME_Do_Main_Page-3">[3]</a></sup></p>

<h2><span class="mw-headline" id="Docky">Docky</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GNOME_Do&amp;action=edit&amp;section=1" title="Edit section: Docky">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:302px;"><a class="image" href="/wiki/File:GNOME_Do_Docky.png"><img alt="" class="thumbimage" data-file-height="319" data-file-width="547" height="175" src="//upload.wikimedia.org/wikipedia/commons/thumb/6/6c/GNOME_Do_Docky.png/300px-GNOME_Do_Docky.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/6/6c/GNOME_Do_Docky.png/450px-GNOME_Do_Docky.png 1.5x, //upload.wikimedia.org/wikipedia/commons/6/6c/GNOME_Do_Docky.png 2x" width="300"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:GNOME_Do_Docky.png" title="Enlarge"></a></div>
GNOME Do with the Docky interface</div>
</div>
</div>
<p>Docky is a theme for GNOME Do that behaves much like the Mac OS X <a class="mw-redirect" href="/wiki/Dock_(Mac_OS_X)" title="Dock (Mac OS X)">dock</a>.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> Unlike GNOME Do's traditional interface, Docky can be set to one of three modes for hiding:</p>
<ul>
<li><i>None</i> - Docky is always visible.</li>
<li><i>Autohide</i> - Docky is normally hidden, and appears when the mouse pointer reaches Docky's (unseen) edge at the lower/upper edge of the screen.</li>
<li><i>Intellihide</i> - Docky hides if it overlaps any window in the <a href="/wiki/Active_window" title="Active window">active window</a> group, but it can still be brought up with the mouse pointer as described above.</li>
</ul>
<p>Standard Do functionality is still present within Docky, and the Do hot-key will still produce the expected behavior.</p>
<p>Docky 2<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup><sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup><sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> is a separate application from GNOME Do. Integration with GNOME Do is planned for Docky 2.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
