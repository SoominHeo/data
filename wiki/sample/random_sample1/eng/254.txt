<p><b>Microsoft Project</b> is a <a href="/wiki/Project_management_software" title="Project management software">project management software</a> product, developed and sold by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>. It is designed to assist a <a href="/wiki/Project_manager" title="Project manager">project manager</a> in developing a <a href="/wiki/Plan" title="Plan">plan</a>, assigning <a href="/wiki/Resource_(project_management)" title="Resource (project management)">resources</a> to tasks, tracking progress, managing the <a href="/wiki/Budget" title="Budget">budget</a>, and analyzing workloads.</p>
<p>Microsoft Project was the company's third Microsoft Windows-based application, and within a couple of years of its introduction it became the dominant PC-based project management software.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p>It is part of the <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a> family but has never been included in any of the Office suites. It is available currently in two editions, Standard and Professional. Microsoft Project's <a href="/wiki/Proprietary_format" title="Proprietary format">proprietary file format</a> is <i>.mpp.</i></p>
<p>Microsoft Project and <a href="/wiki/Microsoft_Project_Server" title="Microsoft Project Server">Microsoft Project Server</a> are the cornerstones of the <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a> <a href="/wiki/Enterprise_project_management" title="Enterprise project management">enterprise project management</a> (EPM) product.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Project&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Screengrab_-_Microsoft_Project_9.0.2000.0224_-_(simple_Gantt_chart)_.png"><img alt="" class="thumbimage" data-file-height="430" data-file-width="720" height="131" src="//upload.wikimedia.org/wikipedia/en/thumb/6/65/Screengrab_-_Microsoft_Project_9.0.2000.0224_-_%28simple_Gantt_chart%29_.png/220px-Screengrab_-_Microsoft_Project_9.0.2000.0224_-_%28simple_Gantt_chart%29_.png" srcset="//upload.wikimedia.org/wikipedia/en/thumb/6/65/Screengrab_-_Microsoft_Project_9.0.2000.0224_-_%28simple_Gantt_chart%29_.png/330px-Screengrab_-_Microsoft_Project_9.0.2000.0224_-_%28simple_Gantt_chart%29_.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/6/65/Screengrab_-_Microsoft_Project_9.0.2000.0224_-_%28simple_Gantt_chart%29_.png/440px-Screengrab_-_Microsoft_Project_9.0.2000.0224_-_%28simple_Gantt_chart%29_.png 2x" width="220"/></a>

<p>'Project' was an MS-DOS software application originally written in Microsoft 'C' (and some assembly) language for the IBM PC. The idea originated from the vision of Ron Bredehoeft, a former IBM S/E and PC-enthusiast in the early 1980s. This original vision was simple: express the recipe and all preparation for a breakfast of <a href="/wiki/Eggs_Benedict" title="Eggs Benedict">eggs Benedict</a> in project management terms. Mr. Bredehoeft formed Microsoft Application Services (MAS) during the birth of the application and the company later entered an OEM agreement with Microsoft Corporation. Alan M. Boyd, Microsoft's Manager of Product Development, introduced the application as an internal tool to help manage the huge number of software projects that were in development at any time inside the company. Boyd wrote the specification and engaged a local Seattle company to develop the prototype.</p>
<p>The first commercial version of Project was released for <a href="/wiki/DOS" title="DOS">DOS</a> in 1984. Microsoft bought all rights to the software in 1985 and released version 2. Version 3 for DOS was released in 1986. Version 4 for DOS was the final DOS version, released in 1986. The first <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a> version was released in 1990, and was labelled version 1 for Windows.</p>
<p>In 1991 a <a href="/wiki/Macintosh" title="Macintosh">Macintosh</a> version was released. Development continued until Microsoft Project 4.0 for Mac in 1993. Microsoft Project 4 for the Mac included both 68k and PowerMac versions, Visual Basic for Applications and integration with Microsoft office 4.2 for the Mac. In 1994, Microsoft stopped development of most of its Mac applications and did not offer a new version of Office until 1998, after the creation of the new Microsoft Macintosh Business Unit the year prior. The Mac Business Unit never released an updated version of Project, and the last version does not run natively on <a class="mw-redirect" href="/wiki/Mac_OS_X" title="Mac OS X">Mac OS X</a>.</p>
<p>Microsoft Project 4.0 was the first to use common Office menus, and the last to support <a href="/wiki/Windows_3.1x" title="Windows 3.1x">Windows 3.1x</a>, <a href="/wiki/Windows_NT_3.1" title="Windows NT 3.1">Windows NT 3.1</a> and <a href="/wiki/Windows_NT_3.5" title="Windows NT 3.5">3.5</a>. It was the last 16-bit version.</p>
<p>Microsoft Project 95 was the first 32-bit version and it was designed for <a href="/wiki/Windows_95" title="Windows 95">Windows 95</a>, hence the name.</p>
<p>Microsoft Project 98 was the first to use <a href="/wiki/Tahoma_(typeface)" title="Tahoma (typeface)">Tahoma</a> font in the menu bars and to contain <a href="/wiki/Office_Assistant" title="Office Assistant">Office Assistant</a>, like all Office 97 applications. It was the last version to run on <a href="/wiki/Windows_NT_3.51" title="Windows NT 3.51">Windows NT 3.51</a>. Project 98 SR-1 was a major service release addressing several issues in Project 98.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>Microsoft Project 2000 was the first to use personalized menus and to use <a href="/wiki/Windows_Installer" title="Windows Installer">Windows Installer</a>-based setup interface, like all <a href="/wiki/Microsoft_Office_2000" title="Microsoft Office 2000">Office 2000</a> applications. Project 2000 was also the last version to run on <a href="/wiki/Windows_95" title="Windows 95">Windows 95</a>.</p>
<p>Microsoft Project 2002 was the first to contain mandatory <a href="/wiki/Product_activation" title="Product activation">product activation</a>, like <a href="/wiki/Microsoft_Office_XP" title="Microsoft Office XP">Office XP</a> and <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a>. It was also the last version to run on <a href="/wiki/Windows_NT_4.0" title="Windows NT 4.0">Windows NT 4.0</a>, <a href="/wiki/Windows_98" title="Windows 98">98 (SE)</a> and <a href="/wiki/Windows_ME" title="Windows ME">ME</a>.</p>
<p>Microsoft Project 2003 was the first to contain <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a>-style icons, like all <a href="/wiki/Microsoft_Office_2003" title="Microsoft Office 2003">Office 2003</a> applications, and the last to run on <a href="/wiki/Windows_2000" title="Windows 2000">Windows 2000</a>.</p>
<p>Versions were released in 1992 (v3.0), 1993 (v4.0), 1995 (v4.1a), 1998 (v9.0), 2000 (v10.0), 2003 (v11.0), 2007 (v12.0), 2010 (v14.0) and 2013 (v15.0).<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> There was no Version 2 on the Windows platform; the original design spec was augmented with the addition of macro capabilities and the extra work required to support a macro language pushed the development schedule out to early 1992 (Version 3).</p>
<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Project&amp;action=edit&amp;section=2" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">

<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Proj2007.jpg" title="Enlarge"></a></div>

<p>Project creates budgets based on assignment work and resource rates. As resources are assigned to tasks and assignment work estimated, the program calculates the cost, equal to the work times the rate, which rolls up to the task level and then to any summary tasks and finally to the project level. Resource definitions (people, equipment and materials) can be shared between projects using a shared resource pool. Each resource can have its own calendar, which defines what days and shifts a resource is available. Resource rates are used to calculate resource assignment costs which are rolled up and summarized at the resource level. Each resource can be assigned to multiple tasks in multiple plans and each task can be assigned multiple resources, and the application schedules task work based on the resource availability as defined in the resource calendars. All resources can be defined in label without limit. Therefore, it cannot determine how many finished products can be produced with a given amount of raw materials. This makes Microsoft Project unsuitable for solving problems of available materials constrained production. Additional software is necessary to manage a complex facility that produces physical goods.</p>
<p>The application creates <a href="/wiki/Critical_path_method" title="Critical path method">critical path</a> schedules, and <a class="mw-redirect" href="/wiki/Critical_chain" title="Critical chain">critical chain</a> and <a href="/wiki/Event_chain_methodology" title="Event chain methodology">event chain methodology</a> third-party add-ons also are available. Schedules can be <a href="/wiki/Resource_management" title="Resource management">resource leveled</a>, and chains are visualized in a <a href="/wiki/Gantt_chart" title="Gantt chart">Gantt chart</a>. Additionally, Microsoft Project can recognize different classes of users. These different classes of users can have differing access levels to projects, views, and other data. Custom objects such as calendars, views, tables, filters, and fields are stored in an enterprise global which is shared by all users.</p>
<h3><span class="mw-headline" id="Editions">Editions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Project&amp;action=edit&amp;section=3" title="Edit section: Editions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Project is available in two editions, Standard and Professional; both editions are available either as 32 or 64bit options. The Professional edition includes all the features of the Standard version, plus more features like team collaboration tools and ability to connect to <a href="/wiki/Microsoft_Project_Server" title="Microsoft Project Server">Microsoft Project Server</a>.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup><sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<h3><span class="mw-headline" id="Project_2010">Project 2010</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Project&amp;action=edit&amp;section=4" title="Edit section: Project 2010">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Microsoft Project 2010 includes the <a class="mw-redirect" href="/wiki/Fluent_(user_interface)" title="Fluent (user interface)">Fluent user interface</a> known as the <a href="/wiki/Ribbon_(computing)" title="Ribbon (computing)">Ribbon</a>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<dl>
<dt>Interoperability</dt>
<dd>Microsoft Project's capabilities were extended with the introduction of <a class="mw-redirect" href="/wiki/Microsoft_Office_Project_Server" title="Microsoft Office Project Server">Microsoft Office Project Server</a> and Microsoft Project Web Access. Project Server stores Project data in a central <a href="/wiki/SQL" title="SQL">SQL</a>-based database, allowing multiple, independent projects to access a shared resource pool. Web Access allows authorized users to access a Project Server database across the <a href="/wiki/Internet" title="Internet">Internet</a>, and includes timesheets, graphical analysis of resource workloads, and administrative tools.</dd>
<dt>User controlled scheduling</dt>
<dd>User-controlled scheduling offers flexible choices for developing and managing projects.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></dd>
<dt>Timeline</dt>
<dd>The timeline view allows the user to build a basic Visio-style graphical overview of the project schedule. The view can be copied and pasted into PowerPoint, Word, or any other application.</dd>
<dt>SharePoint 2010 list synchronization</dt>
<dd><a class="mw-redirect" href="/wiki/SharePoint_Foundation" title="SharePoint Foundation">SharePoint Foundation</a> and Project Professional project task status updates may be synchronized for team members.</dd>
<dt>Inactive tasks</dt>
<dd>helps experiment with project plans and perform what-if analysis</dd>
<dt>The Team Planner view</dt>
<dd>The new Team Planner shows resources and work over time, and helps spot problems and resolve issues.</dd>
</dl>
<h3><span class="mw-headline" id="Project_2013">Project 2013</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Project&amp;action=edit&amp;section=5" title="Edit section: Project 2013">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>What's new in Project 2013 includes new Reports section, better integration with other Microsoft products, and appearance of user interface items:<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<dl>
<dt>Reports</dt>
<dd>A Reports section is added to the ribbon for pre-installed reports. Project 2013 includes graphical reports so that you can create graphical reports and add clipart without having to export data to another program. For example, the Burndown reports show planned work, completed work, and remaining work as lines on a graph. Project 2013 adds pre-installed ability to compare projects, do dashboards, and to export to Visual Reports.</dd>
</dl>
<dl>
<dt>Trace task paths</dt>
<dd>This feature allows you to highlight the link chain (or 'task path') for any task. When you click on a specific task, all of its predecessor tasks show up in one color and all of its successor tasks show up in another color.</dd>
</dl>
<dl>
<dt>Sharing</dt>
<dd>Project 2013 improves the sharing and communication features of its predecessors in multiple ways without leaving Project. With Lync installed, hovering over a name allows you to start an IM session, a video chat, an email, or a phone call. You can copy and paste content to any of the Microsoft Office suite. You can sync content to Sharepoint or a SkyDrive to share without going through Project and Project Online provides an online project management web app that has all of the functionality of Project 2013 and can be accessed from any web enabled device.</dd>
</dl>
<h3><span class="mw-headline" id="Project_2016">Project 2016</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Project&amp;action=edit&amp;section=6" title="Edit section: Project 2016">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Project 2016 adds a new Reports section, backwards-compatibility with Project Server 2013, better integration with other Microsoft products, and improved appearance of user interface items:</p>
<dl>
<dt>Timeline</dt>
<dd>Allows user to customize views to have multiple timeline bars and custom date ranges in a single view.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></dd>
</dl>
<dl>
<dt>Resource Agreements</dt>
<dd>Gives features for resource planning coordination between Project Manager and Resource Manager.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></dd>
</dl>
<dl>
<dt>Office 2016 style theme and help</dt>
<dd>Uses the new Office query 'tell me what you want to do'.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></dd>
</dl>
<dl>
<dt>Backwards compatibility with <a href="/wiki/Microsoft_Project_Server" title="Microsoft Project Server">Microsoft Project Server</a> 2013</dt>
<dd>The transition of enterprises from one version to the next may be eased by this product being able to interact with the earlier version of server.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></dd>
</dl>
