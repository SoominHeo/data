<h1 class="firstHeading" id="firstHeading" lang="en">Unix shell</h1>
<div class="mw-body-content" id="bodyContent">

<p>A <b>Unix shell</b> is a <a class="mw-redirect" href="/wiki/Command-line_interpreter" title="Command-line interpreter">command-line interpreter</a> or <a href="/wiki/Shell_(computing)" title="Shell (computing)">shell</a> that provides a traditional <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> command line <a href="/wiki/User_interface" title="User interface">user interface</a>. Users direct the operation of the <a href="/wiki/Computer" title="Computer">computer</a> by entering commands as text for a <a class="mw-redirect" href="/wiki/Command_line_interpreter" title="Command line interpreter">command line interpreter</a> to execute, or by creating text scripts of one or more such commands. Users typically interact with a Unix shell using a <a href="/wiki/Terminal_emulator" title="Terminal emulator">terminal emulator</a>, however, direct operation via serial hardware connections, or networking session, are common for server systems. All Unix shells provide filename <a href="/wiki/Wildcard_character" title="Wildcard character">wildcarding</a>, <a href="/wiki/Pipeline_(Unix)" title="Pipeline (Unix)">piping</a>, <a href="/wiki/Here_document" title="Here document">here documents</a>, <a href="/wiki/Command_substitution" title="Command substitution">command substitution</a>, <a class="mw-redirect" href="/wiki/Variable_(programming)" title="Variable (programming)">variables</a> and <a href="/wiki/Control_flow" title="Control flow">control structures</a> for <a class="mw-redirect" href="/wiki/Conditional_(programming)" title="Conditional (programming)">condition-testing</a> and <a href="/wiki/Iteration" title="Iteration">iteration</a>.</p>

<h2><span class="mw-headline" id="Concept">Concept</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Unix_shell&amp;action=edit&amp;section=1" title="Edit section: Concept">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The most generic sense of the term <i>shell</i> means any program that users employ to type commands. A shell hides the details of the underlying operating system and manages the technical details of the operating system <a class="mw-redirect" href="/wiki/Kernel_(computer_science)" title="Kernel (computer science)">kernel</a> interface, which is the lowest-level, or "inner-most" component of most operating systems.</p>
<p>In Unix-like operating systems, users typically have many choices of command-line interpreters for interactive sessions. When a user <a class="mw-redirect" href="/wiki/Log_in" title="Log in">logs in</a> to the system interactively, a shell program is automatically executed for the duration of the session. The type of shell, which may be customized for each user, is typically stored in the user's profile, for example in the local <tt><a class="mw-redirect" href="/wiki/Passwd_(file)" title="Passwd (file)">passwd</a></tt> file or in a distributed configuration system such as <a href="/wiki/Network_Information_Service" title="Network Information Service">NIS</a> or <a href="/wiki/Lightweight_Directory_Access_Protocol" title="Lightweight Directory Access Protocol">LDAP</a>; however, the user may execute any other available shell interactively.</p>
<p>The Unix shell is both an interactive <a href="/wiki/Shell_(computing)" title="Shell (computing)">command language</a> as well as a <a class="mw-redirect" href="/wiki/Scripting_programming_language" title="Scripting programming language">scripting programming language</a>, and is used by the operating system as the facility to control (<a href="/wiki/Shell_script" title="Shell script">shell script</a>) the execution of the system.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> Shells created for other <a href="/wiki/Operating_system" title="Operating system">operating systems</a> often provide similar functionality.</p>
<p>On hosts with a <a href="/wiki/Windowing_system" title="Windowing system">windowing system</a>, like <a href="/wiki/MacOS" title="MacOS">macOS</a>, some users may never use the shell directly. On Unix systems, the shell has historically been the implementation language of system startup scripts, including the program that starts a windowing system, configures networking, and many other essential functions. However, some system vendors have replaced the traditional shell-based startup system (<a href="/wiki/Init" title="Init">init</a>) with different approaches, such as <a href="/wiki/Systemd" title="Systemd">systemd</a>.</p>
<h2><span class="mw-headline" id="Early_shells">Early shells</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Unix_shell&amp;action=edit&amp;section=2" title="Edit section: Early shells">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The first Unix shell was the <a href="/wiki/Thompson_shell" title="Thompson shell">Thompson shell</a>, <i>sh</i>, written by <a class="mw-redirect" href="/wiki/Ken_Thompson_(computer_programmer)" title="Ken Thompson (computer programmer)">Ken Thompson</a> at <a href="/wiki/Bell_Labs" title="Bell Labs">Bell Labs</a> and distributed with Versions 1 through 6 of Unix, from 1971 to 1975.<sup class="reference" id="cite_ref-v6hist_2-0"><a href="#cite_note-v6hist-2">[2]</a></sup> Though rudimentary by modern standards, it introduced many of the basic features common to all later Unix shells, including <a href="/wiki/Pipeline_(Unix)" title="Pipeline (Unix)">piping</a>, simple <a href="/wiki/Control_flow" title="Control flow">control structures</a> using if and goto, and filename <a href="/wiki/Wildcard_character" title="Wildcard character">wildcarding</a>. Though not in current use, it is still available as part of some <a class="mw-redirect" href="/wiki/Ancient_UNIX_Systems" title="Ancient UNIX Systems">Ancient UNIX Systems</a>.</p>
<p>It was modeled after the <a href="/wiki/Multics" title="Multics">Multics</a> shell, itself modeled after the <a class="mw-redirect" href="/wiki/RUNCOM" title="RUNCOM">RUNCOM</a> program <a href="/wiki/Louis_Pouzin" title="Louis Pouzin">Louis Pouzin</a> showed to the Multics Team. The "rc" suffix on some Unix configuration files (for example, ".vimrc"), is a remnant of the RUNCOM ancestry of Unix shells.<sup class="reference" id="cite_ref-Vleck_3-0"><a href="#cite_note-Vleck-3">[3]</a></sup><sup class="reference" id="cite_ref-Pouzin_4-0"><a href="#cite_note-Pouzin-4">[4]</a></sup></p>
<p>The <a href="/wiki/PWB_shell" title="PWB shell">PWB shell</a> or Mashey shell, <i>sh</i>, was an upward-compatible version of the Thompson shell, augmented by <a href="/wiki/John_Mashey" title="John Mashey">John Mashey</a> and others and distributed with the <a href="/wiki/PWB/UNIX" title="PWB/UNIX">Programmer's Workbench UNIX</a>, circa 1975-1977. It focused on making shell programming practical, especially in large shared computing centers. It added shell variables (precursors of environment variables, including the search path mechanism that evolved into $PATH), user-executable shell scripts, and interrupt-handling. Control structures were extended from if/goto to if/then/else/endif, switch/breaksw/endsw, and while/end/break/continue. As shell programming became widespread, these external commands were incorporated into the shell itself for performance.</p>
<p>But the most widely distributed and influential of the early Unix shells were the <a href="/wiki/Bourne_shell" title="Bourne shell">Bourne shell</a> and the <a href="/wiki/C_shell" title="C shell">C shell</a>. Both shells have been used as the coding base and model for many derivative and work-alike shells with extended feature sets.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<h3><span class="mw-headline" id="Bourne_shell">Bourne shell</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Unix_shell&amp;action=edit&amp;section=3" title="Edit section: Bourne shell">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Bourne_shell" title="Bourne shell">Bourne shell</a></div>
<p>The Bourne shell, <i>sh</i>, was a complete rewrite by <a href="/wiki/Stephen_R._Bourne" title="Stephen R. Bourne">Stephen Bourne</a> at Bell Labs.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> Distributed as the shell for UNIX Version 7 in 1979, it introduced the rest of the basic features considered common to all the Unix shells, including <a href="/wiki/Here_document" title="Here document">here documents</a>, <a href="/wiki/Command_substitution" title="Command substitution">command substitution</a>, more generic <a class="mw-redirect" href="/wiki/Variable_(programming)" title="Variable (programming)">variables</a> and more extensive builtin <a href="/wiki/Control_flow" title="Control flow">control structures</a>. The language, including the use of a reversed keyword to mark the end of a block, was influenced by <a href="/wiki/ALGOL_68" title="ALGOL 68">ALGOL 68</a>.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> Traditionally, the Bourne shell program name is <tt>sh</tt> and its path in the Unix file system hierarchy is <tt>/bin/sh</tt>. But a number of compatible work-alikes are also available with various improvements and additional features. On many systems, sh may be a <a href="/wiki/Symbolic_link" title="Symbolic link">symbolic link</a> or <a href="/wiki/Hard_link" title="Hard link">hard link</a> to one of these alternatives:</p>
<ul>
<li><a href="/wiki/Almquist_shell" title="Almquist shell">Almquist shell</a> (ash): written as a BSD-licensed replacement for the Bourne Shell; often used in resource-constrained environments. The sh of <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a>, <a href="/wiki/NetBSD" title="NetBSD">NetBSD</a> (and their derivatives) are based on ash that has been enhanced to be <a href="/wiki/POSIX" title="POSIX">POSIX</a> conformant for the occasion.</li>
<li><a href="/wiki/Bash_(Unix_shell)" title="Bash (Unix shell)">Bourne-Again shell</a> (bash): written as part of the <a href="/wiki/GNU_Project" title="GNU Project">GNU Project</a> to provide a superset of Bourne Shell functionality. This shell can be found installed and is the default interactive shell for users on most <a href="/wiki/Linux" title="Linux">Linux</a> and <a href="/wiki/MacOS" title="MacOS">macOS</a> systems.</li>
<li><a class="mw-redirect" href="/wiki/Debian_Almquist_shell" title="Debian Almquist shell">Debian Almquist shell</a> (dash): a modern replacement for ash in <a href="/wiki/Debian" title="Debian">Debian</a> and <a href="/wiki/Ubuntu_(operating_system)" title="Ubuntu (operating system)">Ubuntu</a></li>
<li><a class="mw-redirect" href="/wiki/Korn_shell" title="Korn shell">Korn shell</a> (ksh): written by <a href="/wiki/David_Korn_(computer_scientist)" title="David Korn (computer scientist)">David Korn</a> based on the Bourne shell sources<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> while working at <a href="/wiki/Bell_Labs" title="Bell Labs">Bell Labs</a></li>
<li><a class="mw-redirect" href="/wiki/Pdksh" title="Pdksh">Public domain Korn shell</a> (pdksh)</li>
<li><a class="mw-redirect" href="/wiki/Mksh" title="Mksh">MirBSD Korn shell</a> (mksh): a descendant of the <a href="/wiki/OpenBSD" title="OpenBSD">OpenBSD</a> /bin/ksh and pdksh, developed as part of <a href="/wiki/MirOS_BSD" title="MirOS BSD">MirOS BSD</a></li>
<li><a href="/wiki/Z_shell" title="Z shell">Z shell</a> (zsh): a relatively modern shell that is <a class="mw-redirect" href="/wiki/Backward_compatible" title="Backward compatible">backward compatible</a> with <a href="/wiki/Bash_(Unix_shell)" title="Bash (Unix shell)">bash</a></li>
<li><a class="mw-redirect" href="/wiki/Busybox" title="Busybox">Busybox</a>: a set of Unix utilities for small and embedded systems, which includes 2 shells: ash, a derivative of the Almquist shell; and hush, an independent implentation of a Bourne shell.</li>
</ul>
<p>The <a href="/wiki/POSIX" title="POSIX">POSIX</a> standard specifies its standard shell as a strict subset of the <a class="mw-redirect" href="/wiki/Korn_shell" title="Korn shell">Korn shell</a>, an enhanced version of the Bourne shell. From a user's perspective the Bourne shell was immediately recognized when active by its characteristic default command line prompt character, the dollar sign (<tt>$</tt>).</p>
<h3><span class="mw-headline" id="C_shell">C shell</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Unix_shell&amp;action=edit&amp;section=4" title="Edit section: C shell">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/C_shell" title="C shell">C shell</a></div>
<p>The <a href="/wiki/C_shell" title="C shell">C shell</a>, <i>csh</i>, was written by <a href="/wiki/Bill_Joy" title="Bill Joy">Bill Joy</a> while a graduate student at <a href="/wiki/University_of_California,_Berkeley" title="University of California, Berkeley">University of California, Berkeley</a> and widely distributed with <a href="/wiki/Berkeley_Software_Distribution" title="Berkeley Software Distribution">BSD Unix</a>.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> The language, including the control structures and the expression grammar, was modeled on C. The C shell also introduced a large number of features for interactive work, including the <a href="/wiki/C_shell#History" title="C shell">history</a> and <a href="/wiki/C_shell#Editing_operators" title="C shell">editing</a> mechanisms, <a href="/wiki/C_shell#Aliases" title="C shell">aliases</a>, <a href="/wiki/C_shell#Directory_stack" title="C shell">directory stacks</a>, <a href="/wiki/C_shell#Tilde_notation" title="C shell">tilde notation</a>, <a href="/wiki/C_shell#Cdpath" title="C shell">cdpath</a>, <a href="/wiki/C_shell#Job_control" title="C shell">job control</a> and <a href="/wiki/C_shell#Path_hashing" title="C shell">path hashing</a>. On many systems, csh may be a <a href="/wiki/Symbolic_link" title="Symbolic link">symbolic link</a> or <a href="/wiki/Hard_link" title="Hard link">hard link</a> to <a class="mw-redirect" href="/wiki/TENEX_C_shell" title="TENEX C shell">TENEX C shell</a> (tcsh), an improved version of Joy's original csh. Though the C shell's interactive features have been copied in most other current shells, the language itself has not been widely copied. The only work-alike is <a href="/wiki/Hamilton_C_shell" title="Hamilton C shell">Hamilton C shell</a>, written by Nicole Hamilton, first distributed on <a href="/wiki/OS/2" title="OS/2">OS/2</a> in 1988 and on <a class="mw-redirect" href="/wiki/Windows" title="Windows">Windows</a> since 1992.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></p>

<p>Explanation:</p>
<ul>
<li>blank means a file is not read by a shell at all.</li>
<li>"yes" means a file is always read by a shell upon startup.</li>
<li>"login" means a file is read if the shell is a login shell.</li>
<li>"n/login" means a file is read if the shell is not a login shell.</li>
<li>"int." means a file is read if the shell is interactive.</li>
</ul>

<h2><span class="mw-headline" id="Exotic_shells">Exotic shells</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Unix_shell&amp;action=edit&amp;section=6" title="Edit section: Exotic shells">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Other, more exotic variations on the Unix shell concept include the following:<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[15]</a></sup></p>
<ul>
<li><a class="mw-redirect" href="/wiki/Es_(Unix_shell)" title="Es (Unix shell)">es</a>: A <a href="/wiki/Functional_programming" title="Functional programming">functional programming</a> rc-compatible shell written in the mid-1990s.</li>
<li><a href="/wiki/Friendly_interactive_shell" title="Friendly interactive shell">Friendly interactive shell</a> (fish): First released in 2005.</li>
<li><a class="mw-redirect" href="/wiki/Rc_shell" title="Rc shell">rc</a>: The default shell on <a href="/wiki/Plan_9_from_Bell_Labs" title="Plan 9 from Bell Labs">Plan 9 from Bell Labs</a> and <a class="mw-redirect" href="/wiki/Version_10_Unix" title="Version 10 Unix">Version 10 Unix</a> written by <a href="/wiki/Tom_Duff" title="Tom Duff">Tom Duff</a>. Ports have been made to various <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> operating systems.</li>
<li><a href="/wiki/Scsh" title="Scsh">scsh</a>, a <a href="/wiki/Scheme_(programming_language)" title="Scheme (programming language)">Scheme</a> Shell.</li>
<li><a href="/wiki/Wish_(Unix_shell)" title="Wish (Unix shell)">wish</a>: A windowing shell for <a class="mw-redirect" href="/wiki/Tcl/Tk" title="Tcl/Tk">Tcl/Tk</a>.</li>
</ul>
