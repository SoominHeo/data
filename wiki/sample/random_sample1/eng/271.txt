<div class="hatnote" role="note">"Lzh" redirects here. For other uses, see <a class="mw-redirect mw-disambig" href="/wiki/LZH_(disambiguation)" title="LZH (disambiguation)">LZH</a>.</div>
<p><b>LHA</b> is a <a href="/wiki/Freeware" title="Freeware">freeware</a> <a href="/wiki/Data_compression" title="Data compression">compression</a> utility and associated file format. It was created in 1988 by <a class="new" href="/w/index.php?title=Haruyasu_Yoshizaki&amp;action=edit&amp;redlink=1" title="Haruyasu Yoshizaki (page does not exist)">Haruyasu Yoshizaki</a> <span style="font-weight: normal">(<span class="t_nihongo_kanji" lang="ja" xml:lang="ja">吉崎栄泰</span><span class="t_nihongo_comma" style="display:none">,</span> <i>Yoshizaki Haruyasu</i><sup class="t_nihongo_help noprint"><a href="/wiki/Help:Installing_Japanese_character_sets" title="Help:Installing Japanese character sets"><span class="t_nihongo_icon" style="color: #00e; font: bold 80% sans-serif; text-decoration: none; padding: 0 .1em;">?</span></a></sup>)</span>, and originally named <b>LHarc</b>. A complete rewrite of LHarc, tentatively named <i>LHx</i>, was eventually released as <i>LH</i>. It was then renamed to <i>LHA</i> to avoid conflicting with the then-new <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a> 5.0 <tt>LH</tt> ("load high") command. According to early documentation, LHA is pronounced like <i>La</i>.</p>
<p>Although no longer much used in the West, LHA remains popular in <a href="/wiki/Japan" title="Japan">Japan</a>. It was used by <a href="/wiki/Id_Software" title="Id Software">id Software</a> to compress installation files for their earlier games, including <i><a href="/wiki/Doom_(1993_video_game)" title="Doom (1993 video game)">Doom</a></i> and <i><a href="/wiki/Quake_(video_game)" title="Quake (video game)">Quake</a></i>. LHA has been ported to many operating systems, and is still the main archiving format used on the <a href="/wiki/Amiga" title="Amiga">Amiga</a> computer, although it competed with <a class="mw-redirect" href="/wiki/LZX" title="LZX">LZX</a> in the mid 1990s. This was due to <a href="/wiki/Aminet" title="Aminet">Aminet</a>, the world's largest archive of Amiga related software and files, standardising on <a class="new" href="/w/index.php?title=Stefan_Boberg&amp;action=edit&amp;redlink=1" title="Stefan Boberg (page does not exist)">Stefan Boberg</a>'s implementation of LHA for the Amiga. Microsoft has released a Windows XP add-on, Microsoft Compressed (LZH) Folder Add-on, designed for the Japanese version of the operating system.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> The Japanese version of <a href="/wiki/Windows_7" title="Windows 7">Windows 7</a> ships with the LZH folder add-on built-in.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> Users of non-Japanese versions of Windows 7 Enterprise and Ultimate can also install the LZH folder add-on by installing the optional Japanese language pack from <a href="/wiki/Windows_Update" title="Windows Update">Windows Update</a>.</p>

<h2><span class="mw-headline" id="Compression_methods">Compression methods</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LHA_(file_format)&amp;action=edit&amp;section=1" title="Edit section: Compression methods">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In an LZH archive, the compression method is stored as a five-byte text string. These are the third through seventh bytes of the file.</p>
<h3><span class="mw-headline" id="Canonical_LZH">Canonical LZH</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LHA_(file_format)&amp;action=edit&amp;section=2" title="Edit section: Canonical LZH">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>LHarc compresses files using an algorithm from Yoshizaki's earlier LZHUF product, which was modified from LZARI developed by <a class="new" href="/w/index.php?title=Haruhiko_Okumura&amp;action=edit&amp;redlink=1" title="Haruhiko Okumura (page does not exist)">Haruhiko Okumura</a> <span style="font-weight: normal">(<span class="t_nihongo_kanji" lang="ja" xml:lang="ja">奥村晴彦</span><span class="t_nihongo_comma" style="display:none">,</span> <i>Okumura Haruhiko</i><sup class="t_nihongo_help noprint"><a href="/wiki/Help:Installing_Japanese_character_sets" title="Help:Installing Japanese character sets"><span class="t_nihongo_icon" style="color: #00e; font: bold 80% sans-serif; text-decoration: none; padding: 0 .1em;">?</span></a></sup>)</span>, but uses <a href="/wiki/Huffman_coding" title="Huffman coding">Huffman coding</a> instead of <a href="/wiki/Arithmetic_coding" title="Arithmetic coding">arithmetic coding</a>. LZARI uses <a href="/wiki/Lempel%E2%80%93Ziv%E2%80%93Storer%E2%80%93Szymanski" title="Lempel–Ziv–Storer–Szymanski">Lempel–Ziv–Storer–Szymanski</a> with arithmetic coding.</p>
<dl>
<dt>lh0</dt>
</dl>
<p>No compression method is applied to the source data.</p>
<dl>
<dt>lh1</dt>
</dl>
<p>This method is introduced in LHarc version 1.</p>
<p>It supports 4KiByte <a class="mw-redirect" href="/wiki/Sliding_window" title="Sliding window">sliding window</a>, with support of maximum 60 bytes of matching length. Dynamic Huffman encoding is used.</p>
<dl>
<dt>lh2</dt>
</dl>
<p>This method supports 8KiByte <a class="mw-redirect" href="/wiki/Sliding_window" title="Sliding window">sliding window</a>, with support of maximum 256 bytes of matching length. Dynamic Huffman encoding is used.</p>
<dl>
<dt>lh4, lh5, lh6, lh7</dt>
</dl>
<p>Methods 4, 5, 6, 7 support 4, 8, 32, 64 KiByte <a class="mw-redirect" href="/wiki/Sliding_window" title="Sliding window">sliding window</a> respectively, with support of maximum 256 bytes of matching length. Static Huffman encoding is used. lh5 is first introduced in LHarc 2, followed by lh6 in LHA 2.66 (MSDOS), lh7 in LHA 2.67 beta (MSDOS). LHA itself never compresses into lh4.</p>
<dl>
<dt>lhd</dt>
</dl>
<p>Technically it is not a compression method, but it is used in .LZH archive to indicate the compressed object is an empty directory.</p>
<h3><span class="mw-headline" id="Joe_Jared_extensions">Joe Jared extensions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LHA_(file_format)&amp;action=edit&amp;section=3" title="Edit section: Joe Jared extensions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a class="new" href="/w/index.php?title=Joe_Jared&amp;action=edit&amp;redlink=1" title="Joe Jared (page does not exist)">Joe Jared</a> extended LZSS to use larger dictionaries.</p>
<dl>
<dt>lh8, lh9, lha, lhb, lhc, lhe</dt>
</dl>
<p>Dictionary sizes are 64, 128, 256, 512, 1024, 2048 Ki bytes respectively.</p>
<h3><span class="mw-headline" id="UNLHA32_extensions">UNLHA32 extensions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LHA_(file_format)&amp;action=edit&amp;section=4" title="Edit section: UNLHA32 extensions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>UNLHA32.DLL uses its own method for testing purposes.</p>
<dl>
<dt>lhx</dt>
</dl>
<p>It uses 128-256 Ki bytes dictionary.</p>
<h3><span class="mw-headline" id="PMarc_extensions">PMarc extensions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LHA_(file_format)&amp;action=edit&amp;section=5" title="Edit section: PMarc extensions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>These compression methods are created by PMarc, a <a href="/wiki/CP/M" title="CP/M">CP/M</a> archiver created by Miyo. The archive usually has a .PMA extension.</p>
<dl>
<dt>pc1</dt>
</dl>
<p>PopCom compressed executable archive.</p>
<dl>
<dt>pm0</dt>
</dl>
<p>No compression method is applied to the source data.</p>
<dl>
<dt>pm1</dt>
</dl>
<dl>
<dt>pm2</dt>
</dl>
<dl>
<dt>pms</dt>
</dl>
<p>Used to indicate PMarc self-extracting archive.</p>
<h3><span class="mw-headline" id="LArc_extensions">LArc extensions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LHA_(file_format)&amp;action=edit&amp;section=6" title="Edit section: LArc extensions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>LArc uses the same file format as .LZH, but was written by <a class="new" href="/w/index.php?title=Kazuhiko_Miki&amp;action=edit&amp;redlink=1" title="Kazuhiko Miki (page does not exist)">Kazuhiko Miki</a>, Haruhiko Okumura, <a class="new" href="/w/index.php?title=Ken_Masuyama&amp;action=edit&amp;redlink=1" title="Ken Masuyama (page does not exist)">Ken Masuyama</a>, with extension name '.LZS'.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<dl>
<dt>lzs</dt>
</dl>
<p>It supports 2KiByte <a class="mw-redirect" href="/wiki/Sliding_window" title="Sliding window">sliding window</a>, with support of maximum 17 bytes of matching length.</p>
<dl>
<dt>lz2</dt>
</dl>
<p>It is similar to lzs, except dictionary size and match length can be changed.</p>
<dl>
<dt>lz3</dt>
</dl>
<dl>
<dt>lz4</dt>
</dl>
<p>No compression method is applied to the source data.</p>
<dl>
<dt>lz5</dt>
</dl>
<p>It supports 4KiByte <a class="mw-redirect" href="/wiki/Sliding_window" title="Sliding window">sliding window</a>, with support of maximum 17 bytes of matching length.</p>
<dl>
<dt>lz7</dt>
</dl>
<dl>
<dt>lz8</dt>
</dl>
<h2><span class="mw-headline" id="Issues">Issues</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LHA_(file_format)&amp;action=edit&amp;section=7" title="Edit section: Issues">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="LHICE.2FICE">LHICE/ICE</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LHA_(file_format)&amp;action=edit&amp;section=8" title="Edit section: LHICE/ICE">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>There are copies of LHICE marked as version 1.14. According to Okumura, LHICE is not written by Yoshi.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
