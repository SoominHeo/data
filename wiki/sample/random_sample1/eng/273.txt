<p><b>Microsoft Access</b> is a <a class="mw-redirect" href="/wiki/Database_management_system" title="Database management system">database management system</a> (DBMS) from <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> that combines the <a href="/wiki/Relational_database" title="Relational database">relational</a> <a href="/wiki/Microsoft_Jet_Database_Engine" title="Microsoft Jet Database Engine">Microsoft Jet Database Engine</a> with a <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a> and software-development tools. It is a member of the <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a> suite of applications, included in the Professional and higher editions or sold separately.</p>
<p>Microsoft Access stores data in its own format based on the Access Jet Database Engine. It can also import or link directly to <a href="/wiki/Data" title="Data">data</a> stored in other applications and databases.<sup class="reference" id="cite_ref-ms-import_1-0"><a href="#cite_note-ms-import-1">[1]</a></sup></p>
<p><a href="/wiki/Software_developer" title="Software developer">Software developers</a>, <a href="/wiki/Data_architect" title="Data architect">data architects</a> and <a href="/wiki/Power_user" title="Power user">power users</a> can use Microsoft Access to develop <a href="/wiki/Application_software" title="Application software">application software</a>. Like other Microsoft Office applications, Access is supported by <a href="/wiki/Visual_Basic_for_Applications" title="Visual Basic for Applications">Visual Basic for Applications</a> (VBA), an <a class="mw-redirect" href="/wiki/Object-based" title="Object-based">object-based</a> programming language that can reference a variety of objects including DAO (Data Access Objects), <a href="/wiki/ActiveX" title="ActiveX">ActiveX</a> Data Objects, and many other ActiveX components. Visual objects used in forms and reports expose their methods and properties in the VBA programming environment, and VBA code modules may declare and call Windows <a href="/wiki/Operating_system" title="Operating system">operating system</a> operations.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Access&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Project_Omega">Project Omega</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Access&amp;action=edit&amp;section=2" title="Edit section: Project Omega">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Microsoft's first attempt to sell a <a href="/wiki/Relational_database" title="Relational database">relational database</a> product was during the mid 1980s, when Microsoft obtained the license to sell <a href="/wiki/R:Base" title="R:Base">R:Base</a>.<sup class="reference" id="cite_ref-Goodhew1996-11_2-0"><a href="#cite_note-Goodhew1996-11-2">[2]</a></sup> In the late 1980s Microsoft developed its own solution codenamed Omega.<sup class="reference" id="cite_ref-FergusonBirth_3-0"><a href="#cite_note-FergusonBirth-3">[3]</a></sup> It was confirmed in 1988 that a database product for Windows and OS/2 was in development.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup><sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> It was going to include the "EB" Embedded Basic language,<sup class="reference" id="cite_ref-FergusonBirth_3-1"><a href="#cite_note-FergusonBirth-3">[3]</a></sup> which was going to be the language for writing macros in all Microsoft applications,<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> but the unification of macro languages did not happen until the introduction of <a href="/wiki/Visual_Basic_for_Applications" title="Visual Basic for Applications">Visual Basic for Applications</a> (VBA). Omega was also expected to provide a front end to the <a href="/wiki/Microsoft_SQL_Server" title="Microsoft SQL Server">Microsoft SQL Server</a>.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> The application was very resource-hungry, and there were reports that it was working slowly on the <a href="/wiki/Intel_80386" title="Intel 80386">386</a> processors that were available at the time.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> It was scheduled to be released in the 1st quarter of 1990,<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> but in 1989 the development of the product was reset<sup class="reference" id="cite_ref-Goodhew1996-11_2-1"><a href="#cite_note-Goodhew1996-11-2">[2]</a></sup><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> and it was rescheduled to be delivered no sooner than in January 1991.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> Parts of the project were later used for other Microsoft projects: Cirrus (codename for Access) and Thunder (codename for <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic</a>, where the Embedded Basic engine was used).<sup class="reference" id="cite_ref-Goodhew1996-11_2-2"><a href="#cite_note-Goodhew1996-11-2">[2]</a></sup><sup class="reference" id="cite_ref-FergusonBirth_3-2"><a href="#cite_note-FergusonBirth-3">[3]</a></sup> After Access's premiere, the Omega project was demonstrated in 1992 to several journalists and included features that were not available in Access.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<h3><span class="mw-headline" id="Project_Cirrus">Project Cirrus</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Access&amp;action=edit&amp;section=3" title="Edit section: Project Cirrus">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>After the Omega project was scrapped, some of its developers were assigned to the Cirrus project (most were assigned to the team which created Visual Basic).<sup class="reference" id="cite_ref-Goodhew1996-11_2-3"><a href="#cite_note-Goodhew1996-11-2">[2]</a></sup> Its goal was to create a competitor for applications like <a href="/wiki/Paradox_(database)" title="Paradox (database)">Paradox</a> or <a href="/wiki/DBase" title="DBase">dBase</a> that would work on Windows.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> After Microsoft acquired <a class="mw-redirect" href="/wiki/FoxPro_2" title="FoxPro 2">FoxPro</a>, there were rumors that the Microsoft project might get replaced with it,<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> but the company decided to develop them in parallel. It was assumed that the project would make use of <a href="/wiki/Extensible_Storage_Engine" title="Extensible Storage Engine">Extensible Storage Engine</a> (Jet Blue)<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> but, in the end, only support for <a href="/wiki/Microsoft_Jet_Database_Engine" title="Microsoft Jet Database Engine">Microsoft Jet Database Engine</a> (Jet Red) was provided. The project used some of the code from both the Omega project and a pre-release version of Visual Basic.<sup class="reference" id="cite_ref-FergusonBirth_3-3"><a href="#cite_note-FergusonBirth-3">[3]</a></sup> In July 1992, betas of Cirrus shipped to developers<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> and the name Access became the official name of the product.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<h3><span class="mw-headline" id="Timeline">Timeline</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Access&amp;action=edit&amp;section=4" title="Edit section: Timeline">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>1992: Microsoft released Access version 1.0 on 13 November 1992, and an Access 1.1 release in May 1993 to improve compatibility with other Microsoft products and to include the Access Basic programming language.</p>
<p>1994: Microsoft specified the minimum hardware requirements for Access v2.0 as: Microsoft Windows v3.1 with 4 MB of <a class="mw-redirect" href="/wiki/Random_Access_Memory" title="Random Access Memory">RAM</a> required, 6 MB RAM recommended; 8 MB of available <a class="mw-redirect" href="/wiki/Hard_disk" title="Hard disk">hard disk</a> space required, 14 MB hard disk space recommended. The product shipped on seven 1.44 MB diskettes. The manual shows a 1994 copyright date.</p>
<p>Originally, the software worked well with relatively small databases but testing showed that some circumstances caused <a href="/wiki/Data_corruption" title="Data corruption">data corruption</a>. For example, file sizes over 10 MB proved problematic (note that most hard disks held less than 500 MB at the time this was in wide use), and the <i>Getting Started</i> manual warns about a number of circumstances where obsolete <a href="/wiki/Device_driver" title="Device driver">device drivers</a> or incorrect configurations can cause <a href="/wiki/Data_loss" title="Data loss">data loss</a>. With the phasing out of Windows 95, 98 and ME, improved network reliability, and Microsoft having released 8 <a href="/wiki/Service_pack" title="Service pack">service packs</a> for the Jet <a href="/wiki/Database_engine" title="Database engine">Database Engine</a>, the reliability of Access databases has improved<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Manual_of_Style/Dates_and_numbers#Chronological_items" title="Wikipedia:Manual of Style/Dates and numbers"><span title="The time period mentioned near this tag is ambiguous. (April 2011)">when?</span></a></i>]</sup> and it supports both more data and a larger number of users.</p>
<p>With Office 95, Microsoft Access 7.0 (a.k.a. "Access 95") became part of the Microsoft Office Professional Suite, joining Microsoft Excel, Word, and PowerPoint and transitioning from Access Basic to VBA. Since then, Microsoft has released new versions of Microsoft Access with each release of Microsoft Office. This includes Access 97 (version 8.0), Access 2000 (version 9.0), Access 2002 (version 10.0), Access 2003 (version 11.5), Access 2007 (version 12.0), Access 2010 (version 14.0), and Access 2013 (version 15.0).</p>
<p>Versions 3.0 and 3.5 of <a class="mw-redirect" href="/wiki/Microsoft_Jet" title="Microsoft Jet">Microsoft Jet</a> database engine (used by Access 7.0 and the later-released Access 97 respectively) had a critical issue which made these versions of Access unusable on a computer with more than 1 GB of memory.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> While Microsoft fixed this problem for Jet 3.5/Access 97 post-release, it never fixed the issue with Jet 3.0/Access 95.</p>
<p>The native Access database format (the Jet MDB Database) has also evolved over the years. Formats include Access 1.0, 1.1, 2.0, 7.0, 97, 2000, 2002, 2007, and 2010. The most significant transition was from the Access 97 to the Access 2000 format; which is not <a href="/wiki/Backward_compatibility" title="Backward compatibility">backward compatible</a> with earlier versions of Access. As of 2011<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Microsoft_Access&amp;action=edit">[update]</a></sup> all newer versions of Access support the Access 2000 format. New features were added to the Access 2002 format which can be used by Access 2002, 2003, 2007, and 2010.</p>
<p>Microsoft Access 2000 increased the maximum database size to 2GB from 1GB in Access 97.</p>
<p>Microsoft Access 2007 introduced a new database format: ACCDB. It supports links to SharePoint lists and complex <a href="/wiki/Data_type" title="Data type">data types</a> such as multivalue and attachment fields. These new field types are essentially recordsets in fields and allow the storage of multiple values or files in one field. Microsoft Access 2007 also introduced File Attachment field, which stored data more efficiently than the <a href="/wiki/Object_Linking_and_Embedding" title="Object Linking and Embedding">OLE (Object Linking and Embedding)</a> field.</p>
<p>Microsoft Access 2010 introduced a new version of the ACCDB format supported hosting Access Web solutions on a SharePoint 2010 server. For the first time, this allowed Access solutions to be run without having to install Access on their PC and was the first support of Mac users. Any user on the SharePoint site with sufficient rights could use the Access Web solution. A copy of Access was still required for the developer to create the Access Web solution, and the desktop version of Access remained part of Access 2010. The Access Web solutions were not the same as the desktop solutions. Automation was only through the macro language (not VBA) which Access automatically converted to JavaScript. The data was no longer in an Access database but SharePoint lists. An Access desktop database could link to the SharePoint data, so hybrid applications were possible so that SharePoint users needing basic views and edits could be supported while the more sophisticated, traditional solutions could remain in the desktop Access database.</p>
<p>Microsoft Access 2013 offers traditional Access desktop solutions plus a significantly updated SharePoint 2013 web solution.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup> The Access Web model in Access 2010 was replaced by a new architecture that stores its data in actual SQL Server databases. Unlike SharePoint lists, this offers true relational database design with referential integrity, scalability, extensibility and performance one would expect from SQL Server.<sup class="reference" id="cite_ref-New_in_Access_for_developers_20-0"><a href="#cite_note-New_in_Access_for_developers-20">[20]</a></sup> The database solutions that can be created on SharePoint 2013 offer a modern user interface designed to display multiple levels of relationships that can be viewed and edited, along with resizing for different devices and support for touch. The Access 2013 desktop is similar to Access 2010 but several features were discontinued including support for Access Data Projects (ADPs), pivot tables, pivot charts, Access data collections, source code control, replication, and other legacy features.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> Access desktop database maximum size remained 2GB (as it has been since the 2000 version).</p>
<p>Prior to the introduction of Access, <a href="/wiki/Borland" title="Borland">Borland</a> (with <a href="/wiki/Paradox_(database)" title="Paradox (database)">Paradox</a> and <a href="/wiki/DBase" title="DBase">dBase</a>) and Fox (with <a href="/wiki/Visual_FoxPro" title="Visual FoxPro">FoxPro</a>) dominated the desktop database market. Microsoft Access was the first mass-market database program for Windows. With Microsoft's purchase of FoxPro in 1992 and the incorporation of Fox's Rushmore <a href="/wiki/Query_optimization" title="Query optimization">query optimization</a> routines into Access, Microsoft Access quickly became the dominant database for Windows - effectively eliminating the competition which failed to transition from the <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a> world.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup></p>
<p>Access's initial codename was Cirrus; the forms engine was called Ruby. This was before <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic</a>. <a href="/wiki/Bill_Gates" title="Bill Gates">Bill Gates</a> saw the prototypes and decided that the <a href="/wiki/BASIC" title="BASIC">BASIC</a> language component should be co-developed as a separate expandable application, a project called Thunder. The two projects were developed separately.</p>
<p>Access was also the name of a communications program from Microsoft, meant to compete with <a class="mw-redirect" href="/wiki/ProComm" title="ProComm">ProComm</a> and other programs. This proved a failure and was dropped.<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup> Years later, Microsoft reused the name for its <a class="mw-redirect" href="/wiki/Database_management_system" title="Database management system">database software</a>.</p>
<h2><span class="mw-headline" id="Uses">Uses</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Access&amp;action=edit&amp;section=5" title="Edit section: Uses">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In addition to using its own database storage file, Microsoft Access also may be used as the 'front-end' of a program while other products act as the 'back-end' tables, such as <a href="/wiki/Microsoft_SQL_Server" title="Microsoft SQL Server">Microsoft SQL Server</a> and non-Microsoft products such as <a class="mw-redirect" href="/wiki/Oracle_database" title="Oracle database">Oracle</a> and <a href="/wiki/Sybase" title="Sybase">Sybase</a>. Multiple backend sources can be used by a Microsoft Access Jet Database (ACCDB and MDB formats). Similarly, some applications such as <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic</a>, <a href="/wiki/ASP.NET" title="ASP.NET">ASP.NET</a>, or Visual Studio .NET will use the Microsoft Access database format for its tables and queries. Microsoft Access may also be part of a more complex solution, where it may be integrated with other technologies such as <a href="/wiki/Microsoft_Excel" title="Microsoft Excel">Microsoft Excel</a>, <a href="/wiki/Microsoft_Outlook" title="Microsoft Outlook">Microsoft Outlook</a>, <a href="/wiki/Microsoft_Word" title="Microsoft Word">Microsoft Word</a>, <a href="/wiki/Microsoft_PowerPoint" title="Microsoft PowerPoint">Microsoft PowerPoint</a> and <a class="mw-redirect" href="/wiki/ActiveX_control" title="ActiveX control">ActiveX controls</a>.</p>
<p>Access tables support a variety of standard field types, <a class="mw-redirect" href="/wiki/Index_(database)" title="Index (database)">indices</a>, and <a href="/wiki/Referential_integrity" title="Referential integrity">referential integrity</a> including <a href="/wiki/Event_cascade" title="Event cascade">cascading</a> <a href="/wiki/Update_(SQL)" title="Update (SQL)">updates</a> and <a href="/wiki/Delete_(SQL)" title="Delete (SQL)">deletes</a>. Access also includes a <a href="/wiki/Information_retrieval" title="Information retrieval">query</a> <a href="/wiki/Graphical_user_interface" title="Graphical user interface">interface</a>, forms to display and enter data, and reports for printing. The underlying <a href="/wiki/Microsoft_Jet_Database_Engine" title="Microsoft Jet Database Engine">Jet database</a>, which contains these objects, is <a href="/wiki/Multi-user" title="Multi-user">multi-user</a> and handles <a href="/wiki/Lock_(database)" title="Lock (database)">record-locking</a>.</p>
<p>Repetitive tasks can be automated through macros with <a class="mw-redirect" href="/wiki/Point-and-click" title="Point-and-click">point-and-click</a> options. It is also easy to place a database on a network and have multiple users share and update data without overwriting each other's work. Data is <a href="/wiki/Lock_(database)" title="Lock (database)">locked</a> at the <a href="/wiki/Row_(database)" title="Row (database)">record</a> level which is significantly different from <a href="/wiki/Microsoft_Excel" title="Microsoft Excel">Excel</a> which <a href="/wiki/Lock_(computer_science)" title="Lock (computer science)">locks</a> the entire spreadsheet.</p>
<p>There are <a href="/wiki/Template_(file_format)" title="Template (file format)">template</a> databases within the program and for download from Microsoft's website. These options are available upon starting Access and allow users to enhance a database with predefined tables, <a href="/wiki/Query_language" title="Query language">queries</a>, forms, reports, and macros. Database templates support <a href="/wiki/Visual_Basic_for_Applications" title="Visual Basic for Applications">VBA</a> code but Microsoft's templates do not include VBA code.</p>
<p>Programmers can create solutions using VBA, which is similar to <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic 6.0 (VB6)</a> and used throughout the <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a> programs such as <a href="/wiki/Microsoft_Excel" title="Microsoft Excel">Excel</a>, <a href="/wiki/Microsoft_Word" title="Microsoft Word">Word</a>, <a href="/wiki/Microsoft_Outlook" title="Microsoft Outlook">Outlook</a> and <a href="/wiki/Microsoft_PowerPoint" title="Microsoft PowerPoint">PowerPoint</a>. Most VB6 code, including the use of Windows <a class="mw-redirect" href="/wiki/API" title="API">API</a> calls, can be used in VBA. <a class="mw-redirect" href="/wiki/Power_users" title="Power users">Power users</a> and developers can extend basic <a class="mw-redirect" href="/wiki/End-user" title="End-user">end-user</a> solutions to a professional solution with advanced automation, <a href="/wiki/Data_validation" title="Data validation">data validation</a>, <a href="/wiki/Exception_handling" title="Exception handling">error trapping</a>, and multi-user support.</p>
<p>The number of simultaneous users that can be supported depends on the amount of data, the <a href="/wiki/Task_(project_management)" title="Task (project management)">tasks</a> being performed, level of use, and <a href="/wiki/Software_design" title="Software design">application design</a>. Generally accepted limits are solutions with 1 GB or less of data (Access supports up to 2 GB) and performs quite well with 100 or fewer simultaneous connections (255 concurrent users are supported). This capability is often a good fit for department solutions. If using an Access database solution in a multi-user scenario, the application should be "split". This means that the tables are in one file called the <a class="mw-redirect" href="/wiki/Open_Back-End" title="Open Back-End">back end</a> (typically stored on a shared network folder) and the application components (forms, reports, queries, code, macros, linked tables) are in another file called the <a class="mw-redirect" href="/wiki/Open_Back-End" title="Open Back-End">front end</a>. The linked tables in the front end point to the back end file. Each user of the Access application would then receive his or her own copy of the front end file.</p>
<p>Applications that run complex queries or analysis across large datasets would naturally require greater <a href="/wiki/Bandwidth_(computing)" title="Bandwidth (computing)">bandwidth</a> and <a class="mw-redirect" href="/wiki/Memory_(computers)" title="Memory (computers)">memory</a>. Microsoft Access is designed to <a class="extiw" href="https://en.wiktionary.org/wiki/scale" title="wikt:scale">scale</a> to support more data and users by linking to multiple Access databases or using a <a href="/wiki/Back-end_database" title="Back-end database">back-end</a> database like Microsoft SQL Server. With the latter design, the amount of data and users can scale to enterprise-level solutions.</p>
<p>Microsoft Access's role in <a href="/wiki/Web_development" title="Web development">web development</a> prior to version 2010 is limited. <a href="/wiki/User_interface" title="User interface">User interface</a> features of Access, such as forms and reports, only work in <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a>. In versions 2000 through 2003 an Access object type called Data Access Pages created publishable web pages. Data Access Pages are no longer supported. The <a href="/wiki/Microsoft_Jet_Database_Engine" title="Microsoft Jet Database Engine">Microsoft Jet Database Engine</a>, core to Access, can be accessed through technologies such as <a class="mw-redirect" href="/wiki/ODBC" title="ODBC">ODBC</a> or <a href="/wiki/OLE_DB" title="OLE DB">OLE DB</a>. The data (i.e., tables and queries) can be accessed by <a href="/wiki/Web_application" title="Web application">web-based applications</a> developed in <a href="/wiki/ASP.NET" title="ASP.NET">ASP.NET</a>, <a href="/wiki/PHP" title="PHP">PHP</a>, or <a href="/wiki/Java_(software_platform)" title="Java (software platform)">Java</a>. With the use of Microsoft's Terminal Services and Remote Desktop Application in Windows Server 2008 R2, organizations can host Access applications so they can be run over the web.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup> This technique does not scale the way a web application would but is appropriate for a limited number of users depending on the configuration of the host.</p>
<p>Access 2010 allows databases to be published to <a class="mw-redirect" href="/wiki/Microsoft_SharePoint_Server" title="Microsoft SharePoint Server">SharePoint</a> 2010 web sites running Access Services. These web-based forms and reports run in any modern web browser. The resulting web forms and reports, when accessed via a web browser, don't require any add-ins or extensions (e.g. ActiveX, <a class="mw-redirect" href="/wiki/Silverlight" title="Silverlight">Silverlight</a>).</p>
<p>Access 2013 can create web applications directly in <a class="mw-redirect" href="/wiki/Microsoft_SharePoint_Server" title="Microsoft SharePoint Server">SharePoint</a> 2013 sites running Access Services. Access 2013 web solutions store its data in an underlying SQL Server database which is much more scalable and robust than the Access 2010 version which used SharePoint lists to store its data.</p>
<p>A compiled version of an Access database (File extensions: .MDE /ACCDE or .ADE; ACCDE only works with Access 2007 or later) can be created to prevent user from accessing the design surfaces to modify module code, forms, and reports. An MDE or ADE file is a Microsoft Access database file with all modules compiled and all editable source code removed. Both the .MDE and .ADE versions of an Access database are used when end-user modifications are not allowed or when the application’s source code should be kept confidential.</p>
<p>Microsoft also offers <a class="external text" href="http://www.microsoft.com/en-us/download/details.aspx?id=24569" rel="nofollow">developer extensions</a> for download to help distribute Access 2007 applications, create database templates, and integrate <a href="/wiki/Source_code" title="Source code">source code</a> control with <a href="/wiki/Microsoft_Visual_SourceSafe" title="Microsoft Visual SourceSafe">Microsoft Visual SourceSafe</a>.</p>
<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Access&amp;action=edit&amp;section=6" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Users can create tables, queries, forms and reports, and connect them together with <a href="/wiki/Macro_(computer_science)" title="Macro (computer science)">macros</a>. Advanced users can use <a href="/wiki/Visual_Basic_for_Applications" title="Visual Basic for Applications">VBA</a> to write rich solutions with advanced <a class="mw-redirect" href="/wiki/Data_manipulation" title="Data manipulation">data manipulation</a> and user control. Access also has report creation features that can work with any data source that Access can access.</p>
<p>The original concept of Access was for end users to be able to access data from any source. Other features include: the import and export of data to many formats including <a href="/wiki/Microsoft_Excel" title="Microsoft Excel">Excel</a>, <a href="/wiki/Microsoft_Outlook" title="Microsoft Outlook">Outlook</a>, <a href="/wiki/ASCII" title="ASCII">ASCII</a>, <a href="/wiki/DBase" title="DBase">dBase</a>, <a href="/wiki/Paradox_(database)" title="Paradox (database)">Paradox</a>, <a class="mw-redirect" href="/wiki/FoxPro_2" title="FoxPro 2">FoxPro</a>, <a href="/wiki/Microsoft_SQL_Server" title="Microsoft SQL Server">SQL Server</a> and <a href="/wiki/Oracle_Database" title="Oracle Database">Oracle</a>. It also has the ability to link to data in its existing location and use it for viewing, querying, editing, and reporting. This allows the existing data to change while ensuring that Access uses the latest data. It can perform <a class="mw-redirect" href="/wiki/Heterogeneity" title="Heterogeneity">heterogeneous</a> <a href="/wiki/Join_(SQL)" title="Join (SQL)">joins</a> between data sets stored across different platforms. Access is often used by people downloading data from <a href="/wiki/Enterprise_software" title="Enterprise software">enterprise level databases</a> for manipulation, analysis, and reporting locally.</p>
<p>There is also the <a href="/wiki/Microsoft_Jet_Database_Engine" title="Microsoft Jet Database Engine">Jet</a> Database format (MDB or ACCDB in Access 2007) which can contain the application and data in one file. This makes it very convenient to distribute the entire application to another user, who can run it in disconnected environments.</p>
<p>One of the benefits of Access from a programmer's perspective is its relative compatibility with <a href="/wiki/SQL" title="SQL">SQL</a> (structured <a href="/wiki/Query_language" title="Query language">query language</a>) — queries can be viewed <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphically</a> or edited as SQL statements, and SQL statements can be used directly in Macros and VBA <a href="/wiki/Modular_programming" title="Modular programming">Modules</a> to manipulate Access tables. Users can mix and use both VBA and "Macros" for programming forms and logic and offers <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented</a> possibilities. VBA can also be included in queries.</p>
<p>Microsoft Access offers <a class="mw-redirect" href="/wiki/Parameter_(computer_science)" title="Parameter (computer science)">parameterized queries</a>. These queries and Access tables can be referenced from other programs like <a href="/wiki/Visual_Basic" title="Visual Basic">VB6</a> and <a href="/wiki/.NET_Framework" title=".NET Framework">.NET</a> through <a class="mw-redirect" href="/wiki/Data_Access_Objects" title="Data Access Objects">DAO</a> or <a href="/wiki/ActiveX_Data_Objects" title="ActiveX Data Objects">ADO</a>. From Microsoft Access, VBA can reference parameterized stored procedures via ADO.</p>
<p>The <a href="/wiki/Desktop_environment" title="Desktop environment">desktop</a> editions of <a href="/wiki/Microsoft_SQL_Server" title="Microsoft SQL Server">Microsoft SQL Server</a> can be used with Access as an alternative to the <a href="/wiki/Microsoft_Jet_Database_Engine" title="Microsoft Jet Database Engine">Jet Database Engine</a>. This support started with <a href="/wiki/MSDE" title="MSDE">MSDE</a> (Microsoft SQL Server Desktop Engine), a scaled down version of Microsoft SQL Server 2000, and continues with the <a href="/wiki/SQL_Server_Express" title="SQL Server Express">SQL Server Express</a> versions of SQL Server 2005 and 2008.</p>
<p>Microsoft Access is a <a href="/wiki/File_server" title="File server">file server</a>-based database. Unlike <a class="mw-redirect" href="/wiki/Client%E2%80%93server" title="Client–server">client–server</a> <a href="/wiki/Relational_database_management_system" title="Relational database management system">relational database management systems</a> (RDBMS), Microsoft Access does not implement <a href="/wiki/Database_trigger" title="Database trigger">database triggers</a>, <a href="/wiki/Stored_procedure" title="Stored procedure">stored procedures</a>, or <a href="/wiki/Transaction_log" title="Transaction log">transaction logging</a>. Access 2010 includes table-level triggers and stored procedures built into the ACE data engine. Thus a Client-server database system is not a requirement for using stored procedures or table triggers with Access 2010. Tables, queries, forms, reports and macros can now be developed specifically for web base application in Access 2010. Integration with Microsoft SharePoint 2010 is also highly improved.</p>
<h3><span class="mw-headline" id="Access_Services_and_Web_database">Access Services and Web database</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Access&amp;action=edit&amp;section=7" title="Edit section: Access Services and Web database">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

This offers a true relational database with referential integrity, scalability, maintainability, and extensibility compared to the SharePoint views Access 2010 used. The macro language is enhanced to support more sophisticated programming logic and database level automation.<sup class="reference" id="cite_ref-New_in_Access_for_developers_20-1"><a href="#cite_note-New_in_Access_for_developers-20">[20]</a></sup></p>

<h2><span class="mw-headline" id="Microsoft_Access_Runtime">Microsoft Access Runtime</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Access&amp;action=edit&amp;section=9" title="Edit section: Microsoft Access Runtime">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Microsoft offers free <a class="mw-redirect" href="/wiki/Run-time_system" title="Run-time system">runtime</a> versions of Microsoft Access which allow users to run an Access desktop application without needing to purchase or install a retail version of Microsoft Access. This allows Access developers to create databases that can be freely distributed to an unlimited number of end-users. These runtime versions of Access 2007 and later can be downloaded for free from Microsoft.<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup> The runtime versions for Access 2003 and earlier were part of the Office Developer Extensions/Toolkit and required a separate purchase.</p>
<p>The runtime version allows users to view, edit and delete data, along with running queries, forms, reports, macros and VBA module code. The runtime version does not allow users to change the design of Microsoft Access tables, queries, forms, reports, macros or module code. The runtime versions are similar to their corresponding full version of Access and usually compatible with earlier versions; for example Access Runtime 2010 allows a user to run an Access application made with the 2010 version as well as 2007 through 2000. Due to deprecated features in Access 2013, its runtime version is also unable to support those older features. During development one can simulate the runtime environment from the fully functional version by using the /runtime <a class="mw-redirect" href="/wiki/Command_line_option" title="Command line option">command line option</a>.<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup></p>
<h2><span class="mw-headline" id="Development">Development</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Access&amp;action=edit&amp;section=10" title="Edit section: Development">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Access stores all database tables, queries, forms, reports, macros, and modules in the Access Jet database as a single file.</p>
<p>For query development, Access offers a "Query Designer", a graphical user interface that allows users to build queries without knowledge of structured query language. In the Query Designer, users can "show" the datasources of the query (which can be tables or queries) and select the fields they want returned by clicking and dragging them into the grid. One can set up joins by clicking and dragging fields in tables to fields in other tables. Access allows users to view and manipulate the SQL code if desired. Any Access table, including linked tables from different data sources, can be used in a query.</p>
<p>Access also supports the creation of "pass-through queries". These snippets of SQL code can address external data sources through the use of ODBC connections on the local machine. This enables users to interact with data stored outside the Access program without using linked tables or Jet.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> Users construct the pass-through queries using the SQL syntax supported by the external data source.</p>
<p>When developing reports (in "Design View") additions or changes to controls cause any linked queries to execute in the background and the designer is forced to wait for records to be returned before being able to make another change. This feature cannot be turned off.</p>
<p>Non-programmers can use the macro feature to automate simple tasks through a series of drop-down selections. Macros allow users to easily chain commands together such as running queries, importing or exporting data, opening and closing forms, previewing and printing reports, etc. Macros support basic logic (IF-conditions) and the ability to call other macros. Macros can also contain sub-macros which are similar to subroutines. In Access 2007, enhanced macros included error-handling and support for <a href="/wiki/Temporary_variable" title="Temporary variable">temporary variables</a>. Access 2007 also introduced embedded macros that are essentially properties of an object's event. This eliminated the need to store macros as individual objects. However, macros were limited in their functionality by a lack of <a class="mw-redirect" href="/wiki/Program_loops" title="Program loops">programming loops</a> and advanced coding logic until Access 2013. With significant further enhancements introduced in Access 2013, the capabilities of macros became fully comparable to VBA. They made feature rich web-based application deployments practical, via a greatly enhanced Microsoft SharePoint interface and tools, as well as on traditional Windows desktops.</p>
<p>In common with other products in the <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a> suite, the other programming language used in Access is Microsoft VBA. It is similar to Visual Basic 6.0 (VB6) and code can be stored in modules, classes, and code behind forms and reports. To create a richer, more efficient and maintainable finished product with good error handling, most professional Access applications are developed using the VBA programming language rather than macros, except where web deployment is a business requirement.</p>
<p>To manipulate data in tables and queries in VBA or macros, Microsoft provides two database access libraries of <a href="/wiki/Component_Object_Model" title="Component Object Model">COM</a> components:</p>
<ol>
<li><a class="mw-redirect" href="/wiki/Data_Access_Objects" title="Data Access Objects">Data Access Objects</a> (DAO) (32-bit only), which is included in Access and Windows and evolved to ACE in Microsoft Access 2007 for the ACCDE database format</li>
<li>ActiveX Data Objects <a href="/wiki/ActiveX_Data_Objects" title="ActiveX Data Objects">ActiveX Data Objects</a> (ADO) (both 32-bit and 64-bit versions)</li>
</ol>
<p>As well as DAO and ADO, developers can also use OLE DB and ODBC for developing native C/C++ programs for Access.<sup class="reference" id="cite_ref-Development_30-0"><a href="#cite_note-Development-30">[30]</a></sup> For ADPs and the direct manipulation of SQL Server data, ADO is required. DAO is most appropriate for managing data in Access/Jet databases, and the only way to manipulate the <a href="/wiki/Complex_number" title="Complex number">complex field</a> types in ACCDB tables.</p>
<p>In the database container or navigation pane in Access 2007 and later versions, the system automatically categorizes each object by type (e.g., table, query, macro). Many Access developers use the <a href="/wiki/Leszynski_naming_convention" title="Leszynski naming convention">Leszynski naming convention</a>, though this is not universal; it is a programming convention, not a DBMS-enforced rule.<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup><sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup> It is particularly helpful in VBA where references to object names may not indicate its data type (e.g. tbl for tables, qry for queries).</p>
<p>Developers deploy Microsoft Access most often for individual and workgroup projects (the Access 97 speed characterization was done for 32 users).<sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup> Since Access 97, and with Access 2003 and 2007, Microsoft Access and hardware have evolved significantly. Databases under 1 GB in size (which can now fit entirely in RAM) and 200 simultaneous users are well within the capabilities of Microsoft Access.<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup> Of course, performance depends on the <a href="/wiki/Database_design" title="Database design">database design</a> and tasks. Disk-intensive work such as complex searching and querying take the most time.</p>
<p>As data from a Microsoft Access database can be cached in RAM, processing speed may substantially improve when there is only a single user or if the data is not changing. In the past, the effect of packet latency on the record-locking system caused Access databases to run slowly on a <a class="mw-redirect" href="/wiki/Virtual_Private_Network" title="Virtual Private Network">Virtual Private Network</a> (VPN) or a <a class="mw-redirect" href="/wiki/Wide_Area_Network" title="Wide Area Network">Wide Area Network</a> (WAN) against a Jet database. As of 2010<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Microsoft_Access&amp;action=edit">[update]</a></sup> broadband connections have mitigated this issue. Performance can also be enhanced if a <a class="external text" href="http://www.fmsinc.com/MicrosoftAccess/Performance/LinkedDatabase.html" rel="nofollow">continuous connection</a> is maintained to the back-end database throughout the session rather than opening and closing it for each table access. If Access database performance over VPN or WAN suffers, then a client using <a href="/wiki/Remote_Desktop_Protocol" title="Remote Desktop Protocol">Remote Desktop Protocol</a> (such as Microsoft <a class="mw-redirect" href="/wiki/Terminal_Services" title="Terminal Services">Terminal Services</a>) can provide an effective solution. Access databases linked to SQL Server or to Access Data Projects work well<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (July 2010)">citation needed</span></a></i>]</sup> over VPNs and WANs.</p>
<p>In July 2011, Microsoft acknowledged an intermittent query performance problem with all versions of Access and <a href="/wiki/Windows_7" title="Windows 7">Windows 7</a> and <a href="/wiki/Windows_Server_2008_R2" title="Windows Server 2008 R2">Windows Server 2008 R2</a> due to the nature of resource management being vastly different in newer operating systems.<sup class="reference" id="cite_ref-AccessSlow_35-0"><a href="#cite_note-AccessSlow-35">[35]</a></sup> This issue severely affects query performance on both Access 2003 and earlier with the <a href="/wiki/Microsoft_Jet_Database_Engine" title="Microsoft Jet Database Engine">Jet Database Engine</a> code, as well as Access 2007 and later with the Access Database Engine (ACE).<sup class="reference" id="cite_ref-AccessSlow_35-1"><a href="#cite_note-AccessSlow-35">[35]</a></sup> Microsoft has issued hotfixes <a class="external text" href="http://support.microsoft.com/kb/2553029" rel="nofollow">KB2553029</a> for Access 2007 and <a class="external text" href="http://support.microsoft.com/kb/2553116" rel="nofollow">KB2553116</a> for Access 2010, but will not fix the issue with Jet 4.0 as it is out of mainstream support.<sup class="reference" id="cite_ref-AccessSlow_35-2"><a href="#cite_note-AccessSlow-35">[35]</a></sup></p>
<p>In earlier versions of Microsoft Access, the ability to distribute applications required the purchase of the Developer Toolkit; in Access 2007, 2010 and Access 2013 the "Runtime Only" version is offered as a free download,<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup> making the distribution of royalty-free applications possible on Windows XP, Vista, 7 and Windows 8.x.<sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup></p>
<h3><span class="mw-headline" id="Split_database_architecture">Split database architecture</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Access&amp;action=edit&amp;section=11" title="Edit section: Split database architecture">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Microsoft Access applications can adopt a split-database architecture. The single database can be divided into a separate "back-end" file that contains the data tables (shared on a file server) and a "front-end" (containing the application's objects such as queries, forms, reports, macros, and modules). The "front-end" Access application is distributed to each user's desktop and linked to the shared database. Using this approach, each user has a copy of Microsoft Access (or the runtime version) installed on their machine along with their application database. This reduces network traffic since the application is not retrieved for each use. The "front-end" database can still contain local tables for storing a user's settings or temporary data. This split-database design also allows development of the application independent of the data. One disadvantage is that users may make various changes to their own local copy of the application and this makes it hard to manage version control. When a new version is ready, the front-end database is replaced without impacting the data database. Microsoft Access has two built-in utilities, Database Splitter<sup class="reference" id="cite_ref-38"><a href="#cite_note-38">[38]</a></sup> and Linked Table Manager, to facilitate this architecture.</p>
<p>Linked tables in Access use absolute paths rather than relative paths, so the development environment either has to have the same path as the production environment or a "dynamic-linker" routine can be written in <a href="/wiki/Visual_Basic_for_Applications" title="Visual Basic for Applications">VBA</a>.</p>
<p>For very large Access databases, this may have performance issues and a SQL backend should be considered in these circumstances. This is less of an issue if the entire database can fit in the PC's RAM since Access caches data and indexes.</p>

<div class="hatnote" role="note">Main article: <a href="/wiki/Upsizing_(database)" title="Upsizing (database)">Upsizing (database)</a></div>
<p>To scale Access applications to enterprise or web solutions, one possible technique involves migrating to Microsoft SQL Server or equivalent server database. A client–server design significantly reduces maintenance and increases security, availability, stability, and transaction logging.</p>
<p>Access 2010 included a feature called the Upsizing Wizard that allowed users to upgrade their databases to <a href="/wiki/Microsoft_SQL_Server" title="Microsoft SQL Server">Microsoft SQL Server</a>, an ODBC client–server database. This feature was removed from Access 2013. An additional solution, the SQL Server Migration Assistant for Access (SSMA), continues to be available for free download from Microsoft.<sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[39]</a></sup></p>
<p>A variety of upgrading options are available.<sup class="reference" id="cite_ref-40"><a href="#cite_note-40">[40]</a></sup> After migrating the data and queries to SQL Server, the Access database can be linked to the SQL database. However, certain data types are problematic, most notably "Yes/No". In Microsoft Access there are three states for the Yes/No (True/False) data type: empty, no/false (zero) and yes/true (-1). The corresponding SQL Server data type is binary, with only two states, permissible values, zero and 1. Regardless, SQL Server is still the easiest migration, and most appropriate especially if the user does not have rights to create objects such as stored procedures on SQL Server. Retrieving data from linked tables is optimized to just the records needed, but this scenario may operate less efficiently than what would otherwise be optimal for SQL Server. For example, in instances where multi-table joins still require copying the whole table across the network.</p>
<p>In previous versions of Access, including Access 2010, databases can also be converted to Access Data Projects (ADP) which are tied directly to one SQL Server database. This feature was removed from Access 2013. ADP's support the ability to directly create and modify SQL Server objects such as tables, views, stored procedures, and SQL Server constraints. The views and stored procedures can significantly reduce the network traffic for multi-table joins. Fortunately, SQL Server supports temporary tables and links to other data sources beyond the single SQL Server database.</p>
<p>Finally, some Access databases are completely replaced by another technology such as ASP.NET or Java once the data is converted. However any migration may dictate major effort since the Access SQL language is a more powerful superset of standard SQL. Further, Access application procedures, whether VBA and macros, are written at a relatively higher level versus the currently available alternatives that are both robust and comprehensive. Note that the Access macro language, allowing an even higher level of abstraction than VBA, was significantly enhanced in Access 2010 and again in Access 2013.</p>
<p>In many cases, developers build direct web-to-data interfaces using ASP.NET, while keeping major business automation processes, administrative and reporting functions that don't need to be distributed to everyone in Access for information workers to maintain.</p>
<p>While all Access data can migrate to SQL Server directly, some queries cannot migrate successfully. In some situations, you may need to translate VBA functions and user defined functions into T–SQL or .NET functions / procedures. Crosstab queries can be migrated to SQL Server using the PIVOT command.</p>
<h2><span class="mw-headline" id="Protection">Protection</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft_Access&amp;action=edit&amp;section=13" title="Edit section: Protection">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Microsoft Access applications can be made secure by various methods, the most basic being password access control; this is a relatively weak form of protection.</p>
<p>A higher level of protection is the use of workgroup security requiring a <a href="/wiki/User_(computing)" title="User (computing)">user name</a> and <a href="/wiki/Password" title="Password">password</a>. Users and groups can be specified along with their rights at the <a class="mw-redirect" href="/wiki/Object_type" title="Object type">object type</a> or individual object level. This can be used to specify people with read-only or data entry rights but may be challenging to specify. A separate workgroup security file contains the settings which can be used to manage multiple databases. Workgroup security is not supported in the Access 2007 and Access 2010 ACCDB database format, although Access 2007 and Access 2010 still support it for MDB databases.</p>
<p>Databases can also be encrypted. The ACCDB format offers significantly advanced encryption from previous versions.<sup class="reference" id="cite_ref-41"><a href="#cite_note-41">[41]</a></sup></p>
<p>Additionally, if the database design needs to be secured to prevent changes, Access databases can be locked/protected (and the source code <a class="mw-redirect" href="/wiki/Compiled" title="Compiled">compiled</a>) by converting the database to a .MDE file. All changes to the VBA project (modules, forms, or reports) need to be made to the original MDB and then reconverted to MDE. In Access 2007 and Access 2010, the ACCDB database is converted to an ACCDE file. Some tools are available for unlocking and "<a class="mw-redirect" href="/wiki/Decompiling" title="Decompiling">decompiling</a>", although certain elements including original VBA comments and formatting are normally irretrievable.</p>
