<h1 class="firstHeading" id="firstHeading" lang="en">.htaccess</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p>A <b>.htaccess</b> (<i><a href="/wiki/Hypertext" title="Hypertext">hypertext</a> access</i>) file is a <a class="mw-redirect" href="/wiki/Directory_(file_systems)" title="Directory (file systems)">directory</a>-level <a href="/wiki/Configuration_file" title="Configuration file">configuration file</a> supported by several <a href="/wiki/Web_server" title="Web server">web servers</a>, used for configuration of site-access issues, such as <a href="/wiki/URL_redirection" title="URL redirection">URL redirection</a>, <a href="/wiki/URL_shortening" title="URL shortening">URL shortening</a>, Access-security control (for different webpages and files), and more.</p>
<p>A site could have more than one .htaccess file, and the files are placed inside the web tree (i.e. inside directories and their sub-directories), and hence their other name <i>distributed configuration files</i>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p>.htaccess files acts as a subset of the server's global configuration file (like <b>httpd.conf</b><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>) for the directory that they are in, or all sub-directories.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<p>The original purpose of .htaccess—reflected in its name—was to allow per-directory access control by, for example, requiring a password to access the content. More commonly, however, the .htaccess files define or override many other configuration settings such as content type, <a class="mw-redirect" href="/wiki/Character_set" title="Character set">character set</a>, <a href="/wiki/Common_Gateway_Interface" title="Common Gateway Interface">CGI</a> handlers, etc.</p>
<p></p>

<h2><span class="mw-headline" id="The_dot_prefix">The dot prefix</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.htaccess&amp;action=edit&amp;section=1" title="Edit section: The dot prefix">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The 'dot' (<a href="/wiki/Full_stop" title="Full stop">period</a> or <a href="/wiki/Full_stop" title="Full stop">full stop</a>) before the file name makes it a <a href="/wiki/Hidden_file_and_hidden_directory" title="Hidden file and hidden directory">hidden file</a> in <a href="/wiki/Unix" title="Unix">Unix</a> based environments.</p>
<h2><span class="mw-headline" id="Format_and_language">Format and language</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.htaccess&amp;action=edit&amp;section=2" title="Edit section: Format and language">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>.htaccess are written in the <b>Apache Directives</b> variant of the <a href="/wiki/Perl_Compatible_Regular_Expressions" title="Perl Compatible Regular Expressions">Perl Compatible Regular Expressions</a> (PCRE) language. Learning basic PCRE itself can help in mastering work with these files.</p>
<p>For historical reasons, the format of .htaccess is a limited subset of the <a class="mw-redirect" href="/wiki/Apache_HTTP_server" title="Apache HTTP server">Apache HTTP server</a>'s global configuration file <span style="font-family: monospace, monospace;"><a href="/wiki/Httpd.conf" title="Httpd.conf">httpd.conf</a></span> <sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> even when used with web servers such as <a href="/wiki/Oracle_iPlanet_Web_Server" title="Oracle iPlanet Web Server">Oracle iPlanet Web Server</a><sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> and <a href="/wiki/Zeus_Web_Server" title="Zeus Web Server">Zeus Web Server</a> which have very different native global configuration files.</p>
<h2><span class="mw-headline" id="Common_usage">Common usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.htaccess&amp;action=edit&amp;section=3" title="Edit section: Common usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<dl>
<dt><a href="/wiki/Authorization" title="Authorization">Authorization</a>, <a href="/wiki/Authentication" title="Authentication">authentication</a></dt>
<dd>A .htaccess file is often used to specify security restrictions for a directory, hence the filename "access". The .htaccess file is often accompanied by a <a href="/wiki/.htpasswd" title=".htpasswd">.htpasswd</a> file which stores valid <a class="mw-redirect" href="/wiki/Username" title="Username">usernames</a> and their <a href="/wiki/Password" title="Password">passwords</a>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></dd>
<dt><a class="mw-redirect" href="/wiki/URL_rewriting" title="URL rewriting">URL rewriting</a></dt>
<dd>Servers often use .htaccess for <a href="/wiki/Rewriting" title="Rewriting">rewriting</a> long, overly comprehensive URLs to shorter and more memorable ones.</dd>
<dt>Blocking</dt>
<dd>Use <i>allow/deny</i> to block users by IP address or domain. Also, use to block bad bots, rippers and referrers. Often used to restrict access by <a class="mw-redirect" href="/wiki/Web_crawlers" title="Web crawlers">web crawlers</a></dd>
<dt>SSI</dt>
<dd>Enable <a class="mw-redirect" href="/wiki/Server-side_include" title="Server-side include">server-side includes</a>.</dd>
<dt>Directory listing</dt>
<dd>Control how the server will react when no specific web page is specified.</dd>
<dt><a class="mw-redirect" href="/wiki/Custom_error_page" title="Custom error page">Customized error responses</a></dt>
<dd>Changing the page that is shown when a server-side error occurs, for example <a href="/wiki/HTTP_404" title="HTTP 404">HTTP 404 Not Found</a> or, to indicate to a search engine that a page has moved, <a href="/wiki/HTTP_301" title="HTTP 301">HTTP 301 Moved Permanently</a>.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></dd>
<dt><a class="mw-redirect" href="/wiki/MIME_types" title="MIME types">MIME types</a></dt>
<dd>Instruct the server how to treat different varying file types.</dd>
<dt>Cache Control</dt>
<dd>.htaccess files allow a server to control <a href="/wiki/Web_cache" title="Web cache">caching</a> by <a href="/wiki/Web_browser" title="Web browser">web browsers</a> and <a class="mw-redirect" href="/wiki/Caching_proxy" title="Caching proxy">proxies</a> to reduce <a href="/wiki/Bandwidth_(computing)" title="Bandwidth (computing)">bandwidth</a> usage, <a href="/wiki/Web_server" title="Web server">server</a> load, and perceived <a href="/wiki/Lag" title="Lag">lag</a>.</dd>
</dl>
<h2><span class="mw-headline" id="Advantages">Advantages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.htaccess&amp;action=edit&amp;section=4" title="Edit section: Advantages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<dl>
<dt>Immediate changes</dt>
<dd>Because .htaccess files are read on every request, changes made in these files take immediate effect – as opposed to the main configuration file which requires the server to be restarted for the new settings to take effect.</dd>
<dt>Non-privileged users</dt>
<dd>For servers with multiple users, it is often desirable to allow individual users the ability to alter their site configuration. The use of .htaccess files allows such individualization, and by unprivileged users – because the main server configuration files do not need to be changed.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></dd>
</dl>
<h2><span class="mw-headline" id="Disadvantages">Disadvantages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.htaccess&amp;action=edit&amp;section=5" title="Edit section: Disadvantages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Controlling Apache using the main server configuration file <code><a href="/wiki/Httpd.conf" title="Httpd.conf">httpd.conf</a></code><sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> is often preferred for security and performance reasons:<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></p>
<dl>
<dt>Performance loss</dt>
<dd>For each <a class="mw-redirect" href="/wiki/HTTP_request" title="HTTP request">HTTP request</a>, there are additional file-system accesses for parent directories when using .htaccess, to check for possibly existing .htaccess files in those parent directories which are allowed to hold .htaccess files. It is possible to programatically migrate directives from .htaccess to <code><a href="/wiki/Httpd.conf" title="Httpd.conf">httpd.conf</a></code> if this performance loss is a concern.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></dd>
<dt>Security</dt>
<dd>Allowing individual users to modify the configuration of a server can cause security concerns if not set up properly.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></dd>
</dl>
