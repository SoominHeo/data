<h1 class="firstHeading" id="firstHeading" lang="en">INI file</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p>The <b>INI file</b> <a href="/wiki/File_format" title="File format">format</a> is an informal standard for <a href="/wiki/Configuration_file" title="Configuration file">configuration files</a> for some <a class="mw-redirect" href="/wiki/Platform_(computing)" title="Platform (computing)">platforms</a> or software. INI files are simple <a href="/wiki/Text_file" title="Text file">text files</a> with a <a href="#Format">basic structure</a> composed of sections, <a class="mw-redirect" href="/wiki/Key-value_pair" title="Key-value pair">properties, and values</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p>In <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a> and 16-bit <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a> platforms up through <a href="/wiki/Windows_ME" title="Windows ME">Windows ME</a>, the INI file served as the primary mechanism to configure operating system and installed applications features, such as device drivers, fonts, startup launchers, and things that needed to be initialized in booting Windows. INI files were also generally used by applications to store their individual settings.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>In <a href="/wiki/Windows_NT" title="Windows NT">Windows NT</a> Microsoft introduced the <a href="/wiki/Windows_Registry" title="Windows Registry">registry</a>, and began to steer developers away from using INI files for configuration. All subsequent versions of Windows have used the Windows Registry for system configuration, and applications built on the .NET Framework use special <a href="/wiki/XML" title="XML">XML</a> .config files. The APIs still exist in Windows, however, and developers may still use them.</p>
<p>The name "INI file" comes from the commonly used <a href="/wiki/Filename_extension" title="Filename extension">filename extension</a> <code>.INI</code>, which stands for "initialization". Other common initialization file extensions are <code>.CFG</code>, <code>.conf</code>,<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> and <code>.TXT</code>,<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> especially 'config.txt' occurrences.</p>
<p><a href="/wiki/Linux" title="Linux">Linux</a> and <a href="/wiki/Unix" title="Unix">Unix</a> systems also use a similar file format for system configuration. In addition, platform-agnostic software may use this file format for configuration. It is human-readable and simple to parse, so it is a usable format for configuration files that do not require much greater complexity.</p>
<p></p>

<h2><span class="mw-headline" id="Format">Format</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=1" title="Edit section: Format">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Keys_.28properties.29">Keys (properties)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=2" title="Edit section: Keys (properties)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The basic element contained in an INI file is the <i>key</i> or <i>property</i>. Every key has a <i>name</i> and a <i>value</i>, <a href="/wiki/Delimiter" title="Delimiter">delimited</a> by an <a href="/wiki/Equals_sign" title="Equals sign">equals sign</a> (<code>=</code>). The name appears to the left of the equals sign.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="na">name</span><span class="o">=</span><span class="s">value</span>
</pre></div>
<h3><span class="mw-headline" id="Sections">Sections</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=3" title="Edit section: Sections">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Keys may (but need not) be grouped into arbitrarily named <i>sections</i>. The section name appears on a line by itself, in <a href="/wiki/Bracket" title="Bracket">square brackets</a> (<code>[</code> and <code>]</code>). All keys after the section declaration are associated with that section. There is no explicit "end of section" delimiter; sections end at the next section declaration, or the end of the file. Sections may not be nested.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">[section]</span>
<span class="na">a</span><span class="o">=</span><span class="s">a</span>
<span class="na">b</span><span class="o">=</span><span class="s">b</span>
</pre></div>
<h3><span class="mw-headline" id="Case_insensitivity">Case insensitivity</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=4" title="Edit section: Case insensitivity">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Section and property names are not <a class="mw-redirect" href="/wiki/Case_sensitive" title="Case sensitive">case sensitive</a> in the Windows implementation.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<h3><span class="mw-headline" id="Comments">Comments</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=5" title="Edit section: Comments">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Semicolon" title="Semicolon">Semicolons</a> (<code>;</code>) at the beginning of the line indicate a <a href="/wiki/Comment_(computer_programming)" title="Comment (computer programming)">comment</a>. Comment lines are ignored.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">; comment text</span>
</pre></div>
<h3><span class="mw-headline" id="Varying_features">Varying features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=6" title="Edit section: Varying features">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

<p>The INI file format is not well defined. Many <a href="/wiki/Computer_program" title="Computer program">programs</a> support features beyond the basics described above. The following is a list of some common features, which may or may not be <a href="/wiki/Implementation" title="Implementation">implemented</a> in any given program.</p>
<h4><span class="mw-headline" id="Blank_lines">Blank lines</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=7" title="Edit section: Blank lines">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Some rudimentary programs do not allow blank lines. Every line must therefore be a section head, a property, or a comment.</p>
<h4><span class="mw-headline" id="Comments_2">Comments</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=8" title="Edit section: Comments">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Some software supports the use of the <a href="/wiki/Number_sign" title="Number sign">number sign</a> (<code>#</code>) as an alternative to the semicolon for indicating comments. Practically speaking, using it to begin a line may effectively change a variable name on that line. For instance, the following line creates a variable named "#var", but not one named "var"; this is sometimes used to create a pseudo-implementation of a comment.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">#var=a</span>
</pre></div>
<p>More generally, use of the number sign is unpredictable, as in these following lines (note the space after the number sign in the second line). For this reason, the number sign character should not be used to begin comments.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">#[section]</span>
<span class="c1"># var=a</span>
</pre></div>
<p>In some implementations, a comment may begin anywhere on a line, including on the same line after properties or section declarations. In others, including the <a href="/wiki/Windows_API" title="Windows API">WinAPI</a> function <code>GetPrivateProfileString</code>, comments must occur on lines by themselves.</p>
<h4><span class="mw-headline" id="Duplicate_names">Duplicate names</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=9" title="Edit section: Duplicate names">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Most implementations only support having one property with a given name in a section. The second occurrence of a property name may cause an <a href="/wiki/Abort_(computing)" title="Abort (computing)">abort</a>, it may be ignored (and the value discarded), or it may override the first occurrence (with the first value discarded). Some programs use duplicate property names to implement multi-valued properties.</p>
<p>Interpretation of multiple section declarations with the same name also varies. In some implementations, duplicate sections simply merge their properties together, as if they occurred contiguously. Others may abort, or ignore some aspect of the INI file.</p>
<h4><span class="mw-headline" id="Escape_characters">Escape characters</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=10" title="Edit section: Escape characters">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Some implementations also offer varying support for an <a href="/wiki/Escape_character" title="Escape character">escape character</a>, typically with the <a href="/wiki/Backslash" title="Backslash">backslash</a> (<code>\</code>). Some support "line continuation", where a backslash followed immediately by EOL (end-of-line) causes the line break to be ignored, and the "logical line" to be continued on the next actual line from the INI file. Implementation of various "special characters" with escape sequences is also seen.</p>

<h4><span class="mw-headline" id="Global_properties">Global properties</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=11" title="Edit section: Global properties">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Optional "global" properties may also be allowed, that are declared before any section is declared.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<h4><span class="mw-headline" id="Hierarchy">Hierarchy</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=12" title="Edit section: Hierarchy">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Most commonly, INI files have no hierarchy of sections within sections. Some files appear to have a hierarchical naming convention, however. For section A, subsection B, sub-subsection C, property P and value V, they may accept entries such as <code>[A.B.C]</code> and <code>P=V</code> (Windows' <code>xstart.ini</code>), <code>[A\B\C]</code> and <code>P=V</code> (the IBM Windows driver file <code>devlist.ini</code>), or <code>[A]</code> and <code>B,C,P = V</code> (Microsoft Visual Studio file <code>AEMANAGR.INI</code>).</p>
<p>It is unclear whether these are simply naming conventions that an application happens to use in order to give the <i>appearance</i> of a hierarchy, or whether the file is being read by a module that actually presents this hierarchy to the application programmer.</p>
<h4><span class="mw-headline" id="Name.2Fvalue_delimiter">Name/value delimiter</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=13" title="Edit section: Name/value delimiter">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Some implementations allow a colon (<code>:</code>) as the name/value delimiter (instead of the equals sign).</p>
<h4><span class="mw-headline" id="Quoted_values">Quoted values</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=14" title="Edit section: Quoted values">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Some implementations allow values to be quoted, typically using <a href="/wiki/Quotation_mark" title="Quotation mark">double quotes</a> and/or <a href="/wiki/Apostrophe" title="Apostrophe">apostrophes</a>. This allows for explicit declaration of whitespace, and/or for quoting of special characters (equals, semicolon, etc.). The standard Windows function <a class="external text" href="http://msdn2.microsoft.com/en-us/library/ms724353.aspx" rel="nofollow">GetPrivateProfileString</a> supports this, and will remove quotation marks that surround the values.</p>
<h4><span class="mw-headline" id="Whitespace">Whitespace</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=15" title="Edit section: Whitespace">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Interpretation of <a class="mw-redirect" href="/wiki/Whitespace_(computer_science)" title="Whitespace (computer science)">whitespace</a> varies. Most implementations ignore leading and trailing whitespace around the outside of the property name. Some even ignore whitespace within values (for example, making "host name" and "hostname" equivalent). Some implementations also ignore leading and trailing whitespace around the property value; others consider all <a href="/wiki/Character_(computing)" title="Character (computing)">characters</a> following the equals sign (including whitespace) to be part of the value.</p>
<h3><span class="mw-headline" id="Order_of_sections_and_properties">Order of sections and properties</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=16" title="Edit section: Order of sections and properties">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In most cases the order of properties in a section and the order of sections in a file is irrelevant, but implementations may vary.</p>
<h2><span class="mw-headline" id="Example">Example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=17" title="Edit section: Example">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Following is an example INI file for an imaginary program. It has two sections: one for the owner of the software, and one for a payroll database connection. Comments note who modified the file last and why an <a href="/wiki/IP_address" title="IP address">IP address</a> is used instead of a <a class="mw-redirect" href="/wiki/Domain_name_system" title="Domain name system">DNS name</a>.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">; last modified 1 April 2001 by John Doe</span>
<span class="k">[owner]</span>
<span class="na">name</span><span class="o">=</span><span class="s">John Doe</span>
<span class="na">organization</span><span class="o">=</span><span class="s">Acme Widgets Inc.</span>

<span class="k">[database]</span>
<span class="c1">; use IP address in case network name resolution is not working</span>
<span class="na">server</span><span class="o">=</span><span class="s">192.0.2.62     </span>
<span class="na">port</span><span class="o">=</span><span class="s">143</span>
<span class="na">file</span><span class="o">=</span><span class="s">"payroll.dat"</span>
</pre></div>
<h2><span class="mw-headline" id="Accessing_INI_files">Accessing INI files</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=18" title="Edit section: Accessing INI files">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Under Windows, the <i>Profile API</i> is the programming interface used to read and write settings from classic Windows .ini files. For example, the <a class="external text" href="http://msdn.microsoft.com/en-us/library/ms724353(VS.85).aspx" rel="nofollow">GetPrivateProfileString</a> function retrieves a string from the specified section in an initialization file.</p>
<p>The following sample <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> program demonstrates reading property values from the above sample INI file (Let the name of configuration file be <code>dbsettings.ini</code>)</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="cp">#include</span> <span class="cpf">&lt;windows.h&gt;</span>

<span class="kt">int</span> <span class="nf">main</span><span class="p">(</span><span class="kt">int</span> <span class="n">argc</span><span class="p">,</span> <span class="n">_TCHAR</span> <span class="o">*</span><span class="n">argv</span><span class="p">[])</span>
<span class="p">{</span>
  <span class="n">_TCHAR</span> <span class="n">dbserver</span><span class="p">[</span><span class="mi">1000</span><span class="p">];</span>
  <span class="kt">int</span> <span class="n">dbport</span><span class="p">;</span>
  <span class="n">GetPrivateProfileString</span><span class="p">(</span><span class="s">"database"</span><span class="p">,</span> <span class="s">"server"</span><span class="p">,</span> <span class="s">"127.0.0.1"</span><span class="p">,</span> <span class="n">dbserver</span><span class="p">,</span> <span class="k">sizeof</span><span class="p">(</span><span class="n">dbserver</span><span class="p">)</span> <span class="o">/</span> <span class="k">sizeof</span><span class="p">(</span><span class="n">dbserver</span><span class="p">[</span><span class="mi">0</span><span class="p">]),</span> <span class="s">".</span><span class="se">\\</span><span class="s">dbsettings.ini"</span><span class="p">);</span>
  <span class="n">dbport</span> <span class="o">=</span> <span class="n">GetPrivateProfileInt</span><span class="p">(</span><span class="s">"database"</span><span class="p">,</span> <span class="s">"port"</span><span class="p">,</span> <span class="mi">143</span><span class="p">,</span> <span class="s">".</span><span class="se">\\</span><span class="s">dbsettings.ini"</span><span class="p">);</span>
  <span class="c1">// N.B. WritePrivateProfileInt() does not exist</span>
  <span class="k">return</span> <span class="mi">0</span><span class="p">;</span>
<span class="p">}</span>
</pre></div>
<p>The third parameter of the GetPrivateProfileString function is the default string, which are "127.0.0.1" and 143 respectively in the two function calls above. If the argument supplied for this parameter is NULL, the default is an empty string, "".</p>
<h2><span class="mw-headline" id="File_mapping">File mapping</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=19" title="Edit section: File mapping">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Initialization File Mapping<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup><sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> creates a mapping between an INI file and the <a class="mw-redirect" href="/wiki/Windows_registry" title="Windows registry">Registry</a>. It was introduced with Windows NT and Windows 95 as a way to migrate from storing settings in classic .ini files to the new <a href="/wiki/Windows_Registry" title="Windows Registry">Windows Registry</a>. File mapping traps the Profile API calls and, using settings from the <code>IniFileMapping</code> Registry section, directs reads and writes to appropriate places in the Registry.</p>
<p>Using the Example above, a string call could be made to fetch the <i>name</i> key from the <i>owner</i> section from a settings file called, say, <i>dbsettings.ini</i>. The returned value should be the string "John Doe":</p>
<pre style="font-size:90%">
GetPrivateProfileString("owner", "name", ... , "c:\\programs\\oldprogram\\dbsettings.ini");
</pre>
<p>INI mapping takes this Profile API call, ignores any path in the given filename and checks to see if there is a Registry key matching the filename under:</p>
<dl>
<dd><code>HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\<br/>
   CurrentVersion\IniFileMapping</code></dd>
</dl>
<p>If this exists, it looks for an entry name matching the requested section. If an entry is found, INI mapping uses its value as a pointer to another part of the Registry. It then looks up the requested INI setting in that part of the Registry.</p>
<p>If no matching entry name is found and there is an entry under the <code>(Default)</code> entry name, INI mapping uses that instead. Thus each section name does not need its own entry.</p>
<table class="wikitable">
<tr>
<th colspan="2">HKEY_LOCAL_MACHINE\Software\...\IniFileMapping\dbsettings.ini</th>
</tr>
<tr>
<td><code>(Default)</code></td>
<td>@USR:Software\oldprogs\inisettings\all</td>
</tr>
<tr>
<td><code>database</code></td>
<td>USR:Software\oldprogs\inisettings\db</td>
</tr>
</table>
<p>So, in this case the profile call for the [owner] section is mapped through to:</p>
<table class="wikitable">
<tr>
<th colspan="2">HKEY_CURRENT_USER\Software\oldprogs\inisettings\all</th>
</tr>
<tr>
<td><code>name</code></td>
<td>John Doe</td>
</tr>
<tr>
<td><code>organization</code></td>
<td>Acme Products</td>
</tr>
</table>
<p>where the "<code>name</code>" Registry entry name is found to match the requested INI key. The value of "John Doe" is then returned to the Profile call. In this case, the @ prefix on the default prevents any reads from going to the dbsettings.ini file on disk. The result is that any settings not found in the Registry are not looked for in the INI file.</p>
<p>The "<code>database</code>" Registry entry does not have the @ prefix on the value; thus, for the [database] section <i>only</i>, settings in the Registry are taken first followed by settings in the dbsettings.ini file on disk.</p>
<h2><span class="mw-headline" id="Alternatives">Alternatives</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_file&amp;action=edit&amp;section=20" title="Edit section: Alternatives">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Starting with <a href="/wiki/Windows_95" title="Windows 95">Windows 95</a>, Microsoft began strongly promoting the use of Windows registry over the INI file.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> INI files are typically limited to two levels (sections and properties) and do not handle binary data well</p>
<p>Later <a href="/wiki/XML" title="XML">XML</a>-based configuration files became a popular choice for encoding configuration in text files.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2014)">citation needed</span></a></i>]</sup> XML allows arbitrarily complex levels and nesting, and has standard mechanisms for encoding <a href="/wiki/Binary_file" title="Binary file">binary data</a>.</p>
<p>More recently, data serialization formats, such as <a href="/wiki/JSON" title="JSON">JSON</a>, <a href="/wiki/TOML" title="TOML">TOML</a>, and <a href="/wiki/YAML" title="YAML">YAML</a> can serve as configuration formats. These three alternative formats can nest arbitrarily, but have a more heavyweight syntax than the INI file.</p>
