<p><b>GIMP</b> <span class="nowrap"><span class="IPA nopopups"><a href="/wiki/Help:IPA_for_English" title="Help:IPA for English">/<span style="border-bottom:1px dotted"><span title="'g' in 'guy'">ɡ</span><span title="/ɪ/ short 'i' in 'bid'">ɪ</span><span title="'m' in 'my'">m</span><span title="'p' in 'pie'">p</span></span>/</a></span></span> (<b>GNU Image Manipulation Program</b>) is a <a href="/wiki/Free_and_open-source_software" title="Free and open-source software">free and open-source</a> <a href="/wiki/Raster_graphics_editor" title="Raster graphics editor">raster graphics editor</a><sup class="reference" id="cite_ref-what-is-gimp_7-0"><a href="#cite_note-what-is-gimp-7">[7]</a></sup> used for image <a href="/wiki/Photo_manipulation" title="Photo manipulation">retouching</a> and <a href="/wiki/Image_editing" title="Image editing">editing</a>, free-form <a href="/wiki/Drawing" title="Drawing">drawing</a>, resizing, <a href="/wiki/Cropping_(image)" title="Cropping (image)">cropping</a>, <a href="/wiki/Photomontage" title="Photomontage">photo-montages</a>, converting between different <a href="/wiki/Image_file_formats" title="Image file formats">image formats</a>, and more specialized tasks.</p>
<p>GIMP is released under <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GPLv3+</a> licenses and is available for <a href="/wiki/Linux" title="Linux">Linux</a>, <a href="/wiki/MacOS" title="MacOS">macOS</a>, and <a class="mw-redirect" href="/wiki/Windows" title="Windows">Windows</a>.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GIMP&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/GIMP_version_history" title="GIMP version history">GIMP version history</a></div>
<p>GIMP was originally released as the <i>General Image Manipulation Program.</i><sup class="reference" id="cite_ref-GIMP0.54-README_8-0"><a href="#cite_note-GIMP0.54-README-8">[8]</a></sup> In 1995 <a href="/wiki/Spencer_Kimball_(computer_programmer)" title="Spencer Kimball (computer programmer)">Spencer Kimball</a> and <a href="/wiki/Peter_Mattis" title="Peter Mattis">Peter Mattis</a> began developing GIMP as a semester-long project at the <a href="/wiki/University_of_California,_Berkeley" title="University of California, Berkeley">University of California, Berkeley</a>. In 1996 GIMP (0.54) was released as the first publicly available release.<sup class="reference" id="cite_ref-gimp-prehistory_9-0"><a href="#cite_note-gimp-prehistory-9">[9]</a></sup><sup class="reference" id="cite_ref-ancient-history_10-0"><a href="#cite_note-ancient-history-10">[10]</a></sup> In the following year <a href="/wiki/Richard_Stallman" title="Richard Stallman">Richard Stallman</a> visited UC Berkeley where Spencer Kimball and Peter Mattis asked if they could change <i>General</i> to <i>GNU</i> (the name given to the operating system created by Stallman).<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> Richard Stallman approved and the definition of the <a href="/wiki/Acronym" title="Acronym">acronym</a> GIMP was changed to be the <i>GNU</i> Image Manipulation Program. This reflected its new existence as being developed as Free Software as a part of the <a href="/wiki/GNU_Project" title="GNU Project">GNU Project</a>.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<p>The number of computer architectures and operating systems supported has expanded significantly since its first release. The first release supported <a href="/wiki/Unix" title="Unix">UNIX</a> systems, such as <a href="/wiki/Linux" title="Linux">Linux</a>, <a href="/wiki/Silicon_Graphics" title="Silicon Graphics">SGI</a> <a href="/wiki/IRIX" title="IRIX">IRIX</a> and <a href="/wiki/HP-UX" title="HP-UX">HP-UX</a>.<sup class="reference" id="cite_ref-GIMP0.54-README_8-1"><a href="#cite_note-GIMP0.54-README-8">[8]</a></sup><sup class="reference" id="cite_ref-why-win32_13-0"><a href="#cite_note-why-win32-13">[13]</a></sup> Since the initial release, GIMP has been ported to many operating systems, including <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> and <a href="/wiki/MacOS" title="MacOS">macOS</a>; the original port to the Windows 32-bit platform was started by Finnish programmer Tor M. Lillqvist (tml) in 1997 and was supported in the GIMP 1.1 release.<sup class="reference" id="cite_ref-why-win32_13-1"><a href="#cite_note-why-win32-13">[13]</a></sup></p>
<p>Following the first release GIMP was quickly adopted and a community of contributors formed. The community began developing tutorials, artwork and shared better work-flows and techniques.<sup class="reference" id="cite_ref-gimp-community_14-0"><a href="#cite_note-gimp-community-14">[14]</a></sup></p>
<p>A <a href="/wiki/Widget_toolkit" title="Widget toolkit">GUI toolkit</a> called <a href="/wiki/GTK%2B" title="GTK+">GTK</a> (GIMP tool kit) was developed to facilitate the development of GIMP. GTK was replaced by its successor GTK+ after being redesigned using <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a> techniques. The development of GTK+ has been attributed to Peter Mattis becoming disenchanted with the <a href="/wiki/Motif_(software)" title="Motif (software)">Motif</a> toolkit GIMP originally used; Motif was used up until GIMP 0.60.<sup class="reference" id="cite_ref-ancient-history_10-1"><a href="#cite_note-ancient-history-10">[10]</a></sup><sup class="reference" id="cite_ref-linuxworldinterview_15-0"><a href="#cite_note-linuxworldinterview-15">[15]</a></sup></p>
<div style="clear:both;"></div>
<h2><span class="mw-headline" id="Development">Development</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GIMP&amp;action=edit&amp;section=2" title="Edit section: Development">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>GIMP is primarily developed by volunteers as a free software project associated to both the GNU and GNOME Projects. Development takes place in a public <a class="mw-redirect" href="/wiki/Git_(software)" title="Git (software)">git</a> <a href="/wiki/Source_code" title="Source code">source code</a> repository,<sup class="reference" id="cite_ref-gimp-git_16-0"><a href="#cite_note-gimp-git-16">[16]</a></sup> on public mailing lists and in public chat channels on the GIMPNET <a href="/wiki/Internet_Relay_Chat" title="Internet Relay Chat">IRC</a> network.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<p>New features are held in public separate source code branches and merged into the main (or development) branch when the GIMP team is sure they won't damage existing functions.<sup class="reference" id="cite_ref-gimp-git_16-1"><a href="#cite_note-gimp-git-16">[16]</a></sup> Sometimes this means that features that appear complete do not get merged or take months or years before they become available in GIMP.</p>
<p>GIMP itself is released as source code. After a source code release installers and packages are made for different operating systems by parties who might not be in contact with the maintainers of GIMP.</p>
<p>The <a href="/wiki/Software_versioning" title="Software versioning">version number</a> used in GIMP is expressed in a <i>major-minor-micro</i> format, with each number carrying a specific meaning: the first (major) number is incremented only for major developments (and is currently 2). The second (minor) number is incremented with each release of new features, with odd numbers reserved for in-progress development versions and even numbers assigned to stable releases; the third (micro) number is incremented before and after each release (resulting in even numbers for releases, and odd numbers for development snapshots) with any bug fixes subsequently applied and released for a stable version.</p>
<p>Each year GIMP applies for several positions in the <a href="/wiki/Google_Summer_of_Code" title="Google Summer of Code">Google Summer of Code</a> (GSoC);<sup class="reference" id="cite_ref-gimp-gsoc_18-0"><a href="#cite_note-gimp-gsoc-18">[18]</a></sup><sup class="reference" id="cite_ref-gsoc-gimp-2009_19-0"><a href="#cite_note-gsoc-gimp-2009-19">[19]</a></sup> to date GIMP has participated in all years except 2007.<sup class="reference" id="cite_ref-list-gimpdev-2007gsoc_20-0"><a href="#cite_note-list-gimpdev-2007gsoc-20">[20]</a></sup> From 2006 to 2009 there have been nine GSoC projects that have been listed as successful,<sup class="reference" id="cite_ref-gimp-gsoc_18-1"><a href="#cite_note-gimp-gsoc-18">[18]</a></sup> although not all successful projects have been merged into GIMP yet. The healing brush and perspective clone tools and <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a> bindings were created as part of the 2006 GSoC and can be used in version 2.8.0 of GIMP, although there were three other projects that were completed and are not yet available in a stable version of GIMP; those projects being Vector Layers, and a <a href="/wiki/JPEG_2000" title="JPEG 2000">JPEG 2000</a> plug-in. Several of the GSoC projects were completed in 2008, but have not been merged into a stable GIMP release.</p>
<p>Actual Development Version is 2.9.4 with many deep improvements.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> Next stable Version in Roadmap is 2.10.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup></p>
<h3><span class="mw-headline" id="User_interface">User interface</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GIMP&amp;action=edit&amp;section=3" title="Edit section: User interface">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The user interface of GIMP is designed by a dedicated design and usability team. This team was formed after the developers of GIMP signed up to join the <a href="/wiki/OpenUsability" title="OpenUsability">OpenUsability</a> project.<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup> A user interface brainstorming group has since been created for GIMP,<sup class="reference" id="cite_ref-usability-team_24-0"><a href="#cite_note-usability-team-24">[24]</a></sup><sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> where users of GIMP can send in their suggestions as to how they think the GIMP user interface could be improved.</p>
<p>GIMP is presented in two forms, <i>single</i> and <i>multiple</i> window mode;<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup> GIMP 2.8 defaults to the multiple window mode. In multiple <a href="/wiki/Window_(computing)" title="Window (computing)">window</a> mode a set of windows contain all GIMPs functionality. By default, tools and tool settings are on the left and other dialogues are on the right.<sup class="reference" id="cite_ref-GIMP-Windows_27-0"><a href="#cite_note-GIMP-Windows-27">[27]</a></sup> A layers tab is often to the right of the tools tab, and allows a user to work individually on separate image layers. Layers can be edited by right-clicking on a particular layer to bring up edit options for that layer. The tools tab and layers tab are the most common dockable tabs.</p>
<p><a href="/wiki/GTK%2B" title="GTK+">GTK+</a> (GIMP tool kit) is used to create the graphical user interface. GTK+'s creation and history regarding GIMP is described in the history section above.</p>
<h3><span class="mw-headline" id="Libre_Graphics_Meetings">Libre Graphics Meetings</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GIMP&amp;action=edit&amp;section=4" title="Edit section: Libre Graphics Meetings">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Libre_Graphics_Meeting" title="Libre Graphics Meeting">Libre Graphics Meeting</a></div>
<p>The Libre Graphics Meeting (LGM) is a yearly event where developers of GIMP and other projects meet up to discuss issues related to free and open source graphics software. The GIMP developers hold <a href="/wiki/Birds_of_a_feather_(computing)" title="Birds of a feather (computing)">birds of a feather</a> (BOF) sessions at this event.</p>
<h2><span class="mw-headline" id="Distribution">Distribution</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GIMP&amp;action=edit&amp;section=5" title="Edit section: Distribution">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The current version of GIMP works with numerous operating systems, including <a href="/wiki/Linux" title="Linux">Linux</a>, <a href="/wiki/MacOS" title="MacOS">macOS</a> and <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>. Many Linux distributions include GIMP as a part of their desktop operating systems, including <a href="/wiki/Fedora_(operating_system)" title="Fedora (operating system)">Fedora</a> and <a href="/wiki/Debian" title="Debian">Debian</a>.</p>
<p>The GIMP website links to binary installers compiled by Jernej Simončič for the platform.<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup> <a href="/wiki/MacPorts" title="MacPorts">MacPorts</a> was listed as the recommended provider of Mac builds of GIMP.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> This is no longer needed as version 2.8.2 and later run natively on macOS.<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup> GTK+ was originally designed to run on an X11 server. Because macOS can optionally use an X11 server, porting GIMP to macOS is simpler compared to creating a Windows port. GIMP is also available as part of the <i>Ubuntu noroot</i> package from the <a href="/wiki/Google_Play#Play_Store_.28Android_application.29" title="Google Play">Google Play Store</a> on <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android</a>.<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup> In November 2013, GIMP removed its download from SourceForge, citing misleading download buttons that potentially confuse customers, as well as SourceForge's own Windows installer, which bundles <a href="/wiki/Potentially_unwanted_program" title="Potentially unwanted program">potentially unwanted programs</a>. In a statement, GIMP called SourceForge a once "useful and trustworthy place to develop and host FLOSS applications" that now faces "a problem with the ads they allow on their sites ..."<sup class="reference" id="cite_ref-Register:_Gimp_departs_SourceForge_32-0"><a href="#cite_note-Register:_Gimp_departs_SourceForge-32">[32]</a></sup><sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup><sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup></p>
<h3><span class="mw-headline" id="Sourceforge_controversy">Sourceforge controversy</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GIMP&amp;action=edit&amp;section=6" title="Edit section: Sourceforge controversy">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>GIMP, who discontinued their use of <a href="/wiki/SourceForge" title="SourceForge">SourceForge</a> as a download mirror in November 2013,<sup class="reference" id="cite_ref-Register:_Gimp_departs_SourceForge_32-1"><a href="#cite_note-Register:_Gimp_departs_SourceForge-32">[32]</a></sup><sup class="reference" id="cite_ref-sf-bundle_35-0"><a href="#cite_note-sf-bundle-35">[35]</a></sup> reported in May 2015 that SourceForge was hosting infected versions of their Windows binaries on their Open Source Mirror directory.<sup class="reference" id="cite_ref-gimp-20150531_36-0"><a href="#cite_note-gimp-20150531-36">[36]</a></sup><sup class="reference" id="cite_ref-ars-gimp_37-0"><a href="#cite_note-ars-gimp-37">[37]</a></sup></p>
<h2><span class="mw-headline" id="Professional_reviews">Professional reviews</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GIMP&amp;action=edit&amp;section=7" title="Edit section: Professional reviews">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>GIMP's fitness for use in professional environments is regularly reviewed; it is often compared to and suggested as a possible replacement for <a href="/wiki/Adobe_Photoshop" title="Adobe Photoshop">Adobe Photoshop</a>.<sup class="reference" id="cite_ref-ars-2.6_38-0"><a href="#cite_note-ars-2.6-38">[38]</a></sup><sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[39]</a></sup> GIMP has similar functionality to Photoshop, but has a different user interface.<sup class="reference" id="cite_ref-40"><a href="#cite_note-40">[40]</a></sup></p>
<p>GIMP 2.6 was used to create nearly all of the art in <i>Lucas the Game</i>, an independent <a href="/wiki/Video_game" title="Video game">video game</a> by developer Timothy Courtney. Courtney started development of <i>Lucas the Game</i> in early 2014, and the video game was published in July 2015 for PC and Mac. Courtney explains GIMP is a powerful tool, fully capable of large professional projects, such as video games. This is the first case of GIMP having played a major role in the production of a published video game.<sup class="reference" id="cite_ref-41"><a href="#cite_note-41">[41]</a></sup></p>
<p>The single-window mode introduced in GIMP 2.8 was reviewed in 2012 by Ryan Paul of <i><a href="/wiki/Ars_Technica" title="Ars Technica">Ars Technica</a></i>, who noted that it made the user experience feel "more streamlined and less cluttered."<sup class="reference" id="cite_ref-42"><a href="#cite_note-42">[42]</a></sup> Michael Burns, writing for <i><a href="/wiki/Macworld" title="Macworld">Macworld</a></i> in 2014, described the single-window interface of GIMP 2.8.10 as a "big improvement".<sup class="reference" id="cite_ref-43"><a href="#cite_note-43">[43]</a></sup></p>
<p>In his review of GIMP for <i><a href="/wiki/ExtremeTech" title="ExtremeTech">ExtremeTech</a></i> in October 2013, David Cardinal noted that GIMP's reputation of being hard to use and lacking features has "changed dramatically over the last couple years", and that it was "no longer a crippled alternative to Photoshop". He described GIMP's scripting as one of its strengths, but also remarked that some of Photoshop's features - such as Text, 3D commands, Adjustment Layers and History - are either less powerful or missing in GIMP. Cardinal favorably described the UFRaw converter for raw images used with GIMP, noting that it still "requires some patience to figure out how to use those more advanced capabilities". Cardinal stated that GIMP is "easy enough to try" despite not having as well developed documentation and help system as those for Photoshop, concluding that it "has become a worthy alternative to Photoshop for anyone on a budget who doesn’t need all of Photoshop’s vast feature set".<sup class="reference" id="cite_ref-44"><a href="#cite_note-44">[44]</a></sup></p>
<p>A 2016 comparison recommended GIMP for use with Linux, for its low (no) cost, for occasional use, and Photoshop for professional users such as photographers and designers, and for some things that GIMP cannot do.<sup class="reference" id="cite_ref-45"><a href="#cite_note-45">[45]</a></sup> The author commented "GIMP has improved a lot in the last few years, going from unbearably ugly to bearably ugly — but what’s indisputable is that Photoshop is much easier to learn thanks to the countless awesome tutorials available online.<sup class="reference" id="cite_ref-46"><a href="#cite_note-46">[46]</a></sup></p>

<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GIMP&amp;action=edit&amp;section=9" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Further information: <a href="/wiki/Comparison_of_raster_graphics_editors" title="Comparison of raster graphics editors">Comparison of raster graphics editors</a></div>
<div class="thumb tright">
<div class="thumbinner" style="width:212px;"><a class="image" href="/wiki/File:Cycle_of_gradients_brushes_pratterns_in_gimp-2.8.gif"><img alt="" class="thumbimage" data-file-height="394" data-file-width="210" height="394" src="//upload.wikimedia.org/wikipedia/commons/2/28/Cycle_of_gradients_brushes_pratterns_in_gimp-2.8.gif" width="210"/></a>

<p>Tools used to perform <a href="/wiki/Image_editing" title="Image editing">image editing</a> can be accessed via the toolbox, through menus and dialogue windows. They include filters and brushes, as well as transformation, selection, <a href="/wiki/Layers_(digital_image_editing)" title="Layers (digital image editing)">layer</a> and masking tools.</p>
<dl>
<dt>Color</dt>
</dl>
<p>There are several ways of selecting colors, including palettes, color choosers and using an eyedropper tool to select a colour on the canvas. The built-in color choosers include <a href="/wiki/RGB_color_model" title="RGB color model">RGB</a>/<a href="/wiki/HSL_and_HSV" title="HSL and HSV">HSV</a> selector or scales, water-color selector, <a href="/wiki/CMYK_color_model" title="CMYK color model">CMYK</a> selector and a color-wheel selector. Colors can also be selected using hexadecimal color codes as used in HTML color selection. GIMP has native support for indexed colour and <a href="/wiki/RGB_color_model" title="RGB color model">RGB</a> color spaces; other color spaces are supported using decomposition where each channel of the new color space becomes a black-and-white image. CMYK, <a href="/wiki/Lab_color_space" title="Lab color space">LAB</a> and <a href="/wiki/HSL_and_HSV" title="HSL and HSV">HSV</a> (<a href="/wiki/Hue" title="Hue">hue</a>, <a href="/wiki/Colorfulness" title="Colorfulness">saturation</a>, <a href="/wiki/Lightness" title="Lightness">value</a>) are supported this way.<sup class="reference" id="cite_ref-51"><a href="#cite_note-51">[51]</a></sup><sup class="reference" id="cite_ref-52"><a href="#cite_note-52">[52]</a></sup> Color blending can be achieved using the Blend tool, by applying a <a href="/wiki/Image_gradient" title="Image gradient">gradient</a> to the surface of an image and using GIMP's color modes. Gradients are also integrated into tools such as the brush tool, when the user paints this way the output color slowly changes. There are a number of default gradients included with GIMP; a user can also create custom gradients with tools provided. Gradient plug-ins are also available.</p>
<dl>
<dt>Selections and paths</dt>
</dl>
<p>GIMP <a href="/wiki/Selection_(user_interface)" title="Selection (user interface)">selection</a> tools include a rectangular and circular selection tool, free select tool, and fuzzy select tool (also known as magic wand). More advanced selection tools include the select by color tool for selecting contiguous regions of color—and the scissors select tool, which creates selections semi-automatically between areas of highly contrasting colors. GIMP also supports a quick mask mode where a user can use a brush to paint the area of a selection. Visibly this looks like a red colored overlay being added or removed. The foreground select tool is an implementation of <a href="/wiki/Simple_interactive_object_extraction" title="Simple interactive object extraction">Simple Interactive Object Extraction</a> (SIOX) a method used to perform the extraction of foreground elements, such as a person or a tree in focus. The Paths Tool allows a user to create vectors (also known as <a href="/wiki/B%C3%A9zier_curve" title="Bézier curve">Bézier curves</a>). Users can use paths to create complex selections, including around natural curves. They can paint (or "stroke") the paths with brushes, patterns, or various line styles. Users can name and save paths for reuse.</p>
<dl>
<dt>Image editing</dt>
</dl>
<p>There are many tools that can be used for <a href="/wiki/Image_editing" title="Image editing">editing images</a> in GIMP. The more common tools include a paint brush, pencil, airbrush, eraser and ink tools used to create new or blended pixels. The Bucket Fill tool can be used to fill a selection with a color or pattern. The Blend tool can be used to fill a selection with a color gradient. These color transitions can be applied to large regions or smaller custom path selections.</p>
<p>GIMP also provides "smart" tools that use a more complex algorithm to do things that otherwise would be time consuming or impossible. These include:</p>
<ul>
<li>Clone tool, which copies pixels using a brush</li>
<li>Healing brush, which copies pixels from an area and corrects tone and color</li>
<li>Perspective clone tool, which works like the clone tool but corrects for distance changes</li>
<li>Blur and sharpen tool blurs and sharpens using a brush</li>
<li>The Smudge tool can be used to subtly smear a selection where it stands.</li>
<li>Dodge and burn tool is a brush that makes target pixels lighter (dodges) or darker (burns)</li>
</ul>

<dl>
<dt>Layers, layer masks and channels</dt>
</dl>
<p>An image being edited in GIMP can consist of many <a href="/wiki/Layers_(digital_image_editing)" title="Layers (digital image editing)">layers</a> in a stack. The user manual suggests that "A good way to visualize a GIMP image is as a stack of transparencies," where in GIMP terminology, each transparency is a layer.<sup class="reference" id="cite_ref-53"><a href="#cite_note-53">[53]</a></sup> Each layer in an image is made up of several channels. In an <a href="/wiki/RGB_color_model" title="RGB color model">RGB</a> image, there are normally 3 or 4 channels, each consisting of a red, green and blue channel. Color sublayers look like slightly different gray images, but when put together they make a complete image. The fourth channel that may be part of a layer is the <a href="/wiki/Alpha_compositing" title="Alpha compositing">alpha channel</a> (or layer mask). This channel measures opacity where a whole or part of an image can be completely visible, partially visible or invisible. Each layer has a layer mode that can be set to change the colors in the image.<sup class="reference" id="cite_ref-54"><a href="#cite_note-54">[54]</a></sup></p>
<p>Text layers can be created using the text tool, allowing a user to write on an image. Text layers can be transformed in several ways, such as converting them to a path or selection.<sup class="reference" id="cite_ref-55"><a href="#cite_note-55">[55]</a></sup><sup class="reference" id="cite_ref-56"><a href="#cite_note-56">[56]</a></sup></p>

<dl>
<dt>Automation, scripts and plug-ins</dt>
</dl>
<p>GIMP has approximately 150 standard effects and filters, including Drop Shadow, Blur, Motion Blur and Noise.</p>
<p>GIMP operations can be automated with <a href="/wiki/Scripting_language" title="Scripting language">scripting languages</a>. The Script-Fu is a <a href="/wiki/Scheme_(programming_language)" title="Scheme (programming language)">Scheme</a>-based language implemented using a <a href="/wiki/TinyScheme" title="TinyScheme">TinyScheme</a> interpreter built into GIMP.<sup class="reference" id="cite_ref-57"><a href="#cite_note-57">[57]</a></sup> GIMP can also be scripted in <a href="/wiki/Perl" title="Perl">Perl</a>,<sup class="reference" id="cite_ref-58"><a href="#cite_note-58">[58]</a></sup><sup class="reference" id="cite_ref-59"><a href="#cite_note-59">[59]</a></sup> <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> (Python-Fu),<sup class="reference" id="cite_ref-60"><a href="#cite_note-60">[60]</a></sup><sup class="reference" id="cite_ref-61"><a href="#cite_note-61">[61]</a></sup> or <a href="/wiki/Tcl" title="Tcl">Tcl</a>, using interpreters external to GIMP.<sup class="reference" id="cite_ref-62"><a href="#cite_note-62">[62]</a></sup> New features can be added to GIMP not only by changing program code (GIMP core), but also by creating plug-ins. These are external programs that are executed and controlled by the main GIMP program.<sup class="reference" id="cite_ref-63"><a href="#cite_note-63">[63]</a></sup><sup class="reference" id="cite_ref-64"><a href="#cite_note-64">[64]</a></sup> MathMap is an example of a plug-in written in C.</p>
<p>There is support for several methods of sharpening and blurring images, including the blur and sharpen tool. The <a href="/wiki/Unsharp_masking#Digital_unsharp_masking" title="Unsharp masking">unsharp mask</a> tool is used to sharpen an image selectively — it only sharpens areas of an image that are sufficiently detailed. The Unsharp Mask tool is considered to give more targeted results for photographs than a normal sharpening filter.<sup class="reference" id="cite_ref-65"><a href="#cite_note-65">[65]</a></sup><sup class="reference" id="cite_ref-66"><a href="#cite_note-66">[66]</a></sup> The Selective Gaussian Blur tool works in a similar way, except it blurs areas of an image with little detail.</p>
<dl>
<dt>GEGL</dt>
</dl>
<p>The <i><a href="/wiki/GEGL" title="GEGL">Generic Graphics Library</a></i> (<i>GEGL</i>) was first introduced as part of GIMP on the 2.6 release of GIMP. This initial introduction does not yet exploit all of the capabilities of GEGL; as of the 2.6 release, GIMP can use GEGL to perform high bit-depth color operations; because of this less information is lost when performing color operations.<sup class="reference" id="cite_ref-67"><a href="#cite_note-67">[67]</a></sup> When GEGL is fully integrated, GIMP will have a higher color bit depth and better non-destructive work-flow. Current distribution versions of GIMP only support 8-bit of color, which is much less than what e.g. digital cameras produce (12-bit or more). Full support for high bit depth is included with actual Gimp 2.9 Development version. For accelerations OpenCL is available for some operations. <sup class="reference" id="cite_ref-68"><a href="#cite_note-68">[68]</a></sup></p>
<dl>
<dt>File formats</dt>
</dl>
<p>GIMP supports importing and exporting with a large number of different <a href="/wiki/File_format" title="File format">file formats</a>,<sup class="reference" id="cite_ref-69"><a href="#cite_note-69">[69]</a></sup> GIMP's native format <a href="/wiki/XCF_(file_format)" title="XCF (file format)">XCF</a> is designed to store all information GIMP can contain about an image; XCF is named after the e<i>X</i>perimental <i>C</i>omputing <i>F</i>acility where GIMP was authored. Import and export capability can be extended to additional file formats by means of plug-ins.</p>
<table class="wikitable">
<tr>
<th></th>
<th>File formats</th>
</tr>
<tr>
<th>Import and export</th>
<td>GIMP has import and export support for image formats such as <a href="/wiki/BMP_file_format" title="BMP file format">BMP</a>, <a href="/wiki/JPEG" title="JPEG">JPEG</a>, <a href="/wiki/Portable_Network_Graphics" title="Portable Network Graphics">PNG</a>, <a href="/wiki/GIF" title="GIF">GIF</a> and <a class="mw-redirect" href="/wiki/Tagged_Image_File_Format" title="Tagged Image File Format">TIFF</a>, along with the file formats of several other applications such as <a href="/wiki/Autodesk" title="Autodesk">Autodesk</a> flic animations, Corel <a href="/wiki/PaintShop_Pro" title="PaintShop Pro">PaintShop Pro</a> images, and Adobe Photoshop documents. Other formats with read/write support include <a href="/wiki/PostScript" title="PostScript">PostScript</a> documents, <a href="/wiki/X_Window_System" title="X Window System">X</a> bitmap image, <a href="/wiki/Xwd" title="Xwd">xwd</a>, and Zsoft <a href="/wiki/PCX" title="PCX">PCX</a>. GIMP can also read and write path information from <a href="/wiki/Scalable_Vector_Graphics" title="Scalable Vector Graphics">SVG</a> files and read/write <a href="/wiki/ICO_(file_format)" title="ICO (file format)">ICO</a> Windows icon files.</td>
</tr>
<tr>
<th>Import only</th>
<td>GIMP can import Adobe <a href="/wiki/Portable_Document_Format" title="Portable Document Format">PDF</a> documents and the <a href="/wiki/Raw_image_format" title="Raw image format">raw image formats</a> used by many <a href="/wiki/Digital_camera" title="Digital camera">digital cameras</a>, but cannot save to these formats. An open source plug-in, <a href="/wiki/UFRaw" title="UFRaw">UFRaw</a>, adds full raw compatibility, and has been noted several times for being updated for new camera models quicker than Adobe's UFRaw support.</td>
</tr>
<tr>
<th>Export only</th>
<td>GIMP can export to <a href="/wiki/Multiple-image_Network_Graphics" title="Multiple-image Network Graphics">MNG</a> layered image files (Linux version only) and <a href="/wiki/HTML" title="HTML">HTML</a> (as a table with colored cells), <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> source code files (as an array) and <a href="/wiki/ASCII_art" title="ASCII art">ASCII Art</a> (using a plug-in to represent images with characters and punctuation making up images), though it cannot read these formats.</td>
</tr>
</table>
<h2><span class="mw-headline" id="Forks_and_derivatives">Forks and derivatives</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GIMP&amp;action=edit&amp;section=10" title="Edit section: Forks and derivatives">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Because of the <a href="/wiki/Free_and_open-source_software" title="Free and open-source software">free and open-source</a> nature of GIMP, several <a href="/wiki/Fork_(software_development)" title="Fork (software development)">forks</a>, variants and derivatives of the computer program have been created to fit the needs of their creators. While GIMP is available for popular operating systems, variants of GIMP may be OS-specific. These variants are neither hosted nor linked on the GIMP site. The GIMP site does not host GIMP builds for Windows or Unix-like operating systems either, although it does include a link to a Windows build.</p>
<p>Well-known variants include:</p>
<ul>
<li><a href="/wiki/CinePaint" title="CinePaint">CinePaint</a>: Formerly Film Gimp, it is a fork of GIMP version 1.0.4, used for frame-by-frame retouching of feature film. CinePaint supports up to 32-bit IEEE-floating point <a href="/wiki/Color_depth" title="Color depth">color depth</a> per channel, as well as <a href="/wiki/Color_management" title="Color management">color management</a> and <a href="/wiki/High-dynamic-range_imaging" title="High-dynamic-range imaging">HDR</a>. CinePaint is used primarily within the <a href="/wiki/Film_industry" title="Film industry">film industry</a> due mainly to its support of high-fidelity image formats. It is available for <a href="/wiki/Berkeley_Software_Distribution" title="Berkeley Software Distribution">BSD</a>, Linux, and macOS.</li>
<li>GIMP classic: A patch<sup class="reference" id="cite_ref-70"><a href="#cite_note-70">[70]</a></sup> against GIMP v2.6.8 source code created to undo changes made to the user interface in GIMP v2.4 through v2.6. A build of GIMP classic for Ubuntu is available.<sup class="reference" id="cite_ref-71"><a href="#cite_note-71">[71]</a></sup> As of March 2011, a new patch could be downloaded that patches against the experimental GIMP v2.7.</li>
<li>GIMP Portable: A <a href="/wiki/Portable_application" title="Portable application">portable</a> version of GIMP for Microsoft Windows XP or later that preserves brushes and presets between computers<sup class="reference" id="cite_ref-72"><a href="#cite_note-72">[72]</a></sup></li>
<li><a href="/wiki/GimPhoto" title="GimPhoto">GimPhoto</a> and <a href="/wiki/GIMPshop" title="GIMPshop">GIMPshop</a>: Derivatives that aim to replicate the <a href="/wiki/Adobe_Photoshop" title="Adobe Photoshop">Adobe Photoshop</a> in some form.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (July 2016)">citation needed</span></a></i>]</sup> <span class="citation-needed-content" style="padding-left:0.1em; padding-right:0.1em; color:#595959; border:1px solid #DDD;">Development of GIMPshop was halted in 2006 and the project disavowed by the developer, Scott Moschella, after an unrelated party registered "GIMPshop" as part of an Internet domain name and passed off the website as belonging to Moschella while accepting donations and making revenue from advertising but passing on none of the income to Moschella.</span><sup class="noprint Inline-Template Template-Fact" style="margin-left:0.1em; white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="curious recent addition (July 2016)">citation needed</span></a></i>]</sup></li>
<li>Instrumented GIMP (ingimp): Created at the <a href="/wiki/University_of_Waterloo" title="University of Waterloo">University of Waterloo</a> to track and report user interaction with the program to generate statistics about how GIMP is used, first released on 5 May 2007. Statistics collected by ingimp were publicly available freely of charge on the project's site after being anonymized.<sup class="reference" id="cite_ref-73"><a href="#cite_note-73">[73]</a></sup> The ingimp site is no longer functioning as of 2014.</li>
<li><a href="/wiki/Seashore_(software)" title="Seashore (software)">Seashore</a>, GIMP.app and GIMP on macOS: Releases and derivatives of GIMP for Mac OS<sup class="reference" id="cite_ref-74"><a href="#cite_note-74">[74]</a></sup><sup class="reference" id="cite_ref-75"><a href="#cite_note-75">[75]</a></sup></li>
</ul>

<p><b>GIMP Animation Package (GAP)</b> A GIMP <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plug-in</a> for creating animations. GAP can save animations in several formats, including <a href="/wiki/GIF" title="GIF">GIF</a> and <a href="/wiki/Audio_Video_Interleave" title="Audio Video Interleave">AVI</a>.<sup class="reference" id="cite_ref-76"><a href="#cite_note-76">[76]</a></sup> The animation function relies on GIMP's layering and image file name numbering capability. Animations are created either by placing each frame on its own layer (in other words, treating each layer as an animation cel), or by manipulating each numbered file as if it were a frame in the video: moving, rotating, flipping, changing colors, applying filters, etc. to the layers by taking advantage of interpolation within function calls(plug-in usage), within a specified frame range. The resulting project can be saved as an animated GIF or encoded video file. GAP also provides programmed layer transitions, frame rate change, and move paths, allowing the creation of sophisticated animations.</p>
<p><b>GIMP Paint Studio (GPS)</b> A collection of brushes and accompanying tool presets, aimed at artists and graphic designers. It speeds up repetitive tasks and can save tool settings between sessions.<sup class="reference" id="cite_ref-gps-google-code_77-0"><a href="#cite_note-gps-google-code-77">[77]</a></sup></p>
<p><b>Resynthesizer</b> A set of plugins originally developed as part of Paul Harrison's PhD thesis<sup class="reference" id="cite_ref-78"><a href="#cite_note-78">[78]</a></sup> providing "context-aware fill" features, including heal selection, heal transparency, uncrop and general resynthesize (the other plugins are user-friendly specialisations of this plugin). The plugin is now maintained by Lloyd Konneker.<sup class="reference" id="cite_ref-79"><a href="#cite_note-79">[79]</a></sup><sup class="reference" id="cite_ref-80"><a href="#cite_note-80">[80]</a></sup> Some uses for the plugin are creating more of a texture, including creation of tileable textures, removing objects from images for touching up photos, and creating themed images.</p>
