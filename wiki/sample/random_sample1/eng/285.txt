<p>A camera <b>raw image file</b> contains minimally processed data from the <a href="/wiki/Image_sensor" title="Image sensor">image sensor</a> of either a <a href="/wiki/Digital_camera" title="Digital camera">digital camera</a>, <a href="/wiki/Image_scanner" title="Image scanner">image scanner</a>, or <a href="/wiki/Motion_picture_film_scanner" title="Motion picture film scanner">motion picture film scanner</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-LoC_2-0"><a href="#cite_note-LoC-2">[2]</a></sup> Raw files are named so because they are not yet processed and therefore are not ready to be <a href="/wiki/Photographic_printing" title="Photographic printing">printed</a> or edited with a <a class="mw-redirect" href="/wiki/Bitmap_graphics_editor" title="Bitmap graphics editor">bitmap graphics editor</a>. Normally, the image is processed by a raw converter in a wide-<a href="/wiki/Gamut" title="Gamut">gamut</a> internal <a class="mw-redirect" href="/wiki/Colorspace" title="Colorspace">colorspace</a> where precise adjustments can be made before conversion to a "positive" file format such as <a href="/wiki/TIFF" title="TIFF">TIFF</a> or <a href="/wiki/JPEG" title="JPEG">JPEG</a> for storage, printing, or further manipulation. This often encodes the image in a device-dependent colorspace. There are dozens, if not hundreds, of raw formats in use by different models of digital equipment (like cameras or film scanners).<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>

<h2><span class="mw-headline" id="Rationale">Rationale</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=1" title="Edit section: Rationale">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Raw image files are sometimes called <b>digital negatives</b>, as they fulfill the same role as <a href="/wiki/Negative_(photography)" title="Negative (photography)">negatives</a> in film photography: that is, the negative is not directly usable as an image, but has all of the information needed to create an image. Likewise, the process of converting a raw image file into a viewable format is sometimes called developing a raw image, by analogy with the <a class="mw-redirect" href="/wiki/Film_development" title="Film development">film development</a> process used to convert <a href="/wiki/Photographic_film" title="Photographic film">photographic film</a> into viewable prints. The selection of the final choice of image rendering is part of the process of <a class="mw-redirect" href="/wiki/White_balance" title="White balance">white balancing</a> and <a href="/wiki/Color_grading" title="Color grading">color grading</a>.</p>
<p>Like a photographic negative, a raw digital image may have a wider <a href="/wiki/Dynamic_range#Photography" title="Dynamic range">dynamic range</a> or color <a href="/wiki/Gamut" title="Gamut">gamut</a> than the eventual final image format, and it preserves most of the information of the captured image. The purpose of raw image formats is to save, with minimum loss of information, data obtained from the sensor, and the conditions surrounding the capturing of the image (the <a href="/wiki/Metadata" title="Metadata">metadata</a>).</p>
<p>Raw image formats are intended to capture as closely as possible (i.e. at the best of the specific sensor's performance) the <a class="mw-redirect" href="/wiki/Radiometric" title="Radiometric">radiometric</a> characteristics of the scene, that is, physical information about the light intensity and color of the scene.</p>
<p>Most raw image file formats store information sensed according to the geometry of the sensor's individual photo-receptive elements (sometimes called pixels) rather than points in the expected final image: sensors with hexagonal element displacement, for example, record information for each of their hexagonally-displaced cells, which a decoding software will eventually transform into the rectangular geometry during "digital developing".</p>
<h2><span class="mw-headline" id="File_contents">File contents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=2" title="Edit section: File contents">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Raw files contain the information required to produce a viewable image from the camera's sensor data. The structure of raw files often follows a common pattern:</p>
<ul>
<li>A short file header which typically contains an indicator of the <a href="/wiki/Endianness" title="Endianness">byte-ordering</a> of the file, a file identifier and an offset into the main file data</li>
<li>Camera sensor <a href="/wiki/Metadata" title="Metadata">metadata</a> which is required to interpret the sensor image data, including the size of the sensor, the attributes of the <a href="/wiki/Color_filter_array" title="Color filter array">CFA</a> and its <a href="/wiki/ICC_profile" title="ICC profile">color profile</a></li>
<li>Image <a href="/wiki/Metadata" title="Metadata">metadata</a> which is required for inclusion in any <a href="/wiki/Content_management_system" title="Content management system">CMS</a> environment or <a href="/wiki/Database" title="Database">database</a>. These include the exposure settings, camera/scanner/lens model, date (and, optionally, place) of shoot/scan, authoring information and other. Some raw files contain a standardized metadata section with data in <a class="mw-redirect" href="/wiki/Exchangeable_image_file_format" title="Exchangeable image file format">Exif</a> format.</li>
<li>An image <a href="/wiki/Thumbnail" title="Thumbnail">thumbnail</a></li>
<li>Optionally a reduced-size image in JPEG format, which can be used for a quick preview</li>
<li>In the case of motion picture film scans, either the <a href="/wiki/Timecode" title="Timecode">timecode</a>, <a href="/wiki/Keykode" title="Keykode">keycode</a> or frame number in the <a href="/wiki/File_sequence" title="File sequence">file sequence</a> which represents the frame sequence in a scanned reel. This item allows the file to be ordered in a <a href="/wiki/File_sequence" title="File sequence">frame sequence</a> (without relying on its filename).</li>
<li>The sensor image data</li>
</ul>
<p>Many raw file formats, including IIQ (<a href="/wiki/Phase_One_(company)" title="Phase One (company)">Phase One</a>), 3FR (<a href="/wiki/Hasselblad" title="Hasselblad">Hasselblad</a>), DCR, K25, KDC (<a href="/wiki/Kodak" title="Kodak">Kodak</a>), CR2 (<a class="mw-redirect" href="/wiki/Canon_(company)" title="Canon (company)">Canon</a>), ERF (<a class="mw-redirect" href="/wiki/Epson" title="Epson">Epson</a>), MEF (<a href="/wiki/Mamiya" title="Mamiya">Mamiya</a>), MOS (<a class="mw-redirect" href="/wiki/Leaf_Medium_Format_Photography" title="Leaf Medium Format Photography">Leaf</a>), NEF (<a href="/wiki/Nikon" title="Nikon">Nikon</a>), <a class="mw-redirect" href="/wiki/Orf_format" title="Orf format">ORF</a> (<a href="/wiki/Olympus_Corporation" title="Olympus Corporation">Olympus</a>), PEF (<a href="/wiki/Pentax" title="Pentax">Pentax</a>), RW2 (<a href="/wiki/Panasonic" title="Panasonic">Panasonic</a>) and ARW, SRF, SR2 (<a href="/wiki/Sony" title="Sony">Sony</a>), are based on the TIFF file format.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> These files may deviate from the TIFF standard in a number of ways, including the use of a non-standard file header, the inclusion of additional image tags and the encryption of some of the tagged data.</p>
<p>Panasonic's raw converter corrects geometric distortion and <a href="/wiki/Chromatic_aberration" title="Chromatic aberration">chromatic aberration</a> on such cameras as the <a href="/wiki/Lumix" title="Lumix">LX3</a>,<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup><sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> with necessary correction information presumably included in the raw.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> <a href="/wiki/Phase_One_(company)" title="Phase One (company)">Phase One</a>'s raw converter <a href="/wiki/Capture_One" title="Capture One">Capture One</a> also offers corrections for geometrical distortion, <a href="/wiki/Chromatic_aberration" title="Chromatic aberration">chromatic aberration</a>, purple fringing and keystone correction emulating the shift capability of <a class="mw-redirect" href="/wiki/Tilt-shift" title="Tilt-shift">tilt-shift</a> in software and specially designed hardware, on most raw files from over 100 different cameras.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> The same holds for Canon's DPP application, at least for all more expensive cameras like all <a href="/wiki/Canon_EOS" title="Canon EOS">EOS DSLRs</a> and the <a href="/wiki/Canon_PowerShot_G" title="Canon PowerShot G">G&lt;n&gt; series</a> of compact cameras.</p>
<p><a class="mw-redirect" href="/wiki/Digital_Negative_(file_format)" title="Digital Negative (file format)">DNG</a>, the <a class="mw-redirect" href="/wiki/Digital_Negative_(file_format)" title="Digital Negative (file format)">Adobe digital negative</a> format, is an extension of the TIFF 6.0 format and is compatible with <a class="mw-redirect" href="/wiki/Tag_Image_File_Format_/_Electronic_Photography" title="Tag Image File Format / Electronic Photography">TIFF/EP</a>, and uses various <a href="/wiki/Open_format" title="Open format">open formats</a> and/or <a href="/wiki/Open_standard" title="Open standard">standards</a>, including <a class="mw-redirect" href="/wiki/Exchangeable_image_file_format" title="Exchangeable image file format">Exif metadata</a>, <a href="/wiki/Extensible_Metadata_Platform" title="Extensible Metadata Platform">XMP metadata</a>, <a href="/wiki/International_Press_Telecommunications_Council" title="International Press Telecommunications Council">IPTC metadata</a>, <a href="/wiki/CIE_1931_color_space" title="CIE 1931 color space">CIE XYZ coordinates</a>, <a href="/wiki/ICC_profile" title="ICC profile">ICC profiles</a>, and <a href="/wiki/JPEG" title="JPEG">JPEG</a>.<sup class="reference" id="cite_ref-DNGspecification_11-0"><a href="#cite_note-DNGspecification-11">[11]</a></sup></p>
<h3><span class="mw-headline" id="Sensor_image_data">Sensor image data</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=3" title="Edit section: Sensor image data">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In <a href="/wiki/Digital_photography" title="Digital photography">digital photography</a>, the raw file plays the role that <a href="/wiki/Photographic_film" title="Photographic film">photographic film</a> plays in <a href="/wiki/Digital_versus_film_photography" title="Digital versus film photography">film photography</a>. Raw files thus contain the full resolution (typically 12- or 14-bit) data as read out from each of the camera's <a href="/wiki/Image_sensor" title="Image sensor">image sensor</a> <a href="/wiki/Pixel" title="Pixel">pixels</a>.</p>
<p>The camera's sensor is almost invariably overlaid with a <a href="/wiki/Color_filter_array" title="Color filter array">color filter array</a> (CFA), usually a <a href="/wiki/Bayer_filter" title="Bayer filter">Bayer filter</a>, consisting of a mosaic of a 2x2 matrix of red, green, blue and (second) green filters.</p>
<p>One variation on the Bayer filter is the <a href="/wiki/RGBE_filter" title="RGBE filter">RGBE filter</a> of the <a href="/wiki/Sony_Cyber-shot_DSC-F828" title="Sony Cyber-shot DSC-F828">Sony Cyber-shot DSC-F828</a>, which exchanged the green in the RG rows with "<a class="mw-redirect" href="/wiki/Emerald_(colour)#Emerald" title="Emerald (colour)">emerald</a>"<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> (a <a href="/wiki/Blue-green" title="Blue-green">blue-green</a><sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> or <a href="/wiki/Cyan" title="Cyan">cyan</a><sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> color). Other sensors, such as the <a href="/wiki/Foveon_X3_sensor" title="Foveon X3 sensor">Foveon X3 sensor</a>, capture information directly in RGB form (using three pixel sensors in each location). These RGB raw data still need to be processed to make an image file, because the raw RGB values correspond to the responses of the sensors, not to a standard color space like <a href="/wiki/SRGB" title="SRGB">sRGB</a>. These data do not need to be <a href="/wiki/Demosaicing" title="Demosaicing">demosaiced</a>, however.</p>
<p><a href="/wiki/Image_scanner" title="Image scanner">Flatbed</a> and <a href="/wiki/Film_scanner" title="Film scanner">film scanner</a> sensors are typically straight narrow RGB or RGBI (where "I" stands for the additional infra-red channel for automatic dust removal) strips that are swept across an image. The <a href="/wiki/HDRi_(data_format)" title="HDRi (data format)">HDRi raw data format</a> is able to store the infrared raw data, which can be used for <a href="/wiki/Infrared_cleaning" title="Infrared cleaning">infrared cleaning</a>, as an additional 16-bit channel. The remainder of the discussion about raw files applies to them as well. (Some scanners do not allow the host system access to the raw data at all, as a speed compromise. The raw data are processed very rapidly inside the scanner to select out the best part of the available dynamic range so only the result is passed to the computer for permanent storage, reducing the amount of data transferred and therefore the bandwidth requirement for any given speed of image throughput.)</p>
<p>To obtain an image from a raw file, this mosaic of data must be converted into standard RGB form. This is often referred to as "raw development".</p>
<p>When converting from the four-sensor 2x2 Bayer-matrix raw form into RGB pixels, the green pair is used to control the luminance detail of the processed output pixel, while the red and blue, which each have half as many samples, are used mostly for the more slowly-varying <a href="/wiki/Chrominance" title="Chrominance">chroma</a> component of the image.</p>
<p>If raw format data is available, it can be used in <a href="/wiki/High-dynamic-range_imaging" title="High-dynamic-range imaging">high-dynamic-range imaging</a> conversion, as a simpler alternative to the multi-exposure HDI approach of capturing three separate images, one underexposed, one correct and one overexposed, and "overlaying" one on top of the other.</p>
<h3><span class="mw-headline" id="Standardization">Standardization</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=4" title="Edit section: Standardization">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Providing a detailed and concise description of the content of raw files is highly problematic. There is no single raw format; formats can be similar or radically different. Different manufacturers use their own proprietary and typically undocumented formats, which are collectively known as raw format. Often they also change the format from one camera model to the next. Several major camera manufacturers, including Nikon, Canon and Sony, <a href="/wiki/Encryption" title="Encryption">encrypt</a> portions of the file in an attempt to prevent third-party tools from accessing them.<sup class="reference" id="cite_ref-dcinterview_15-0"><a href="#cite_note-dcinterview-15">[15]</a></sup></p>
<p>This industry-wide situation of inconsistent formatting has concerned many photographers who worry that their valuable raw photos may someday become inaccessible, as computer operating systems and software programs become obsolete and abandoned raw formats are dropped from new software. The availability of high-quality <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> software which decodes raw image formats, particularly <a href="/wiki/Dcraw" title="Dcraw">dcraw</a>, has helped to alleviate these concerns. An essay by <a href="/wiki/Michael_Reichmann" title="Michael Reichmann">Michael Reichmann</a> and Juergen Specht stated "here are two solutions – the adoption by the camera industry of A: Public documentation of RAW formats; past, present and future, or, more likely B: Adoption of a universal RAW format".<sup class="reference" id="cite_ref-LLtherawflaw_16-0"><a href="#cite_note-LLtherawflaw-16">[16]</a></sup><sup class="reference" id="cite_ref-RawFlawLLWord_17-0"><a href="#cite_note-RawFlawLLWord-17">[17]</a></sup><sup class="reference" id="cite_ref-RawFlawLLPDF_18-0"><a href="#cite_note-RawFlawLLPDF-18">[18]</a></sup> "Planning for [US] Library of Congress Collections" identifies raw-file formats as "less desirable file formats", and identifies DNG as a suggested alternative.<sup class="reference" id="cite_ref-USLOCpreferences_19-0"><a href="#cite_note-USLOCpreferences-19">[19]</a></sup></p>
<p>DNG is the only raw image format for which industry-wide buy-in is being sought. It is based upon, and compatible with, the ISO standard raw image format ISO 12234-2, <a class="mw-redirect" href="/wiki/Tag_Image_File_Format_/_Electronic_Photography" title="Tag Image File Format / Electronic Photography">TIFF/EP</a>, and is being used by ISO in their revision of that standard.</p>

<p>The ISO standard raw image format is ISO 12234-2, better known as <a class="mw-redirect" href="/wiki/Tag_Image_File_Format_/_Electronic_Photography" title="Tag Image File Format / Electronic Photography">TIFF/EP</a>. (TIFF/EP also supports "non-raw", or "processed", images). TIFF/EP provided a basis for the raw image formats of a number of cameras. For example, <a href="/wiki/Nikon" title="Nikon">Nikon</a>'s NEF raw files are based on TIFF/EP, and include a tag which identifies the version of TIFF/EP they are based on.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> Adobe's DNG raw file format was based on TIFF/EP, and the DNG specification states "DNG ... is compatible with the TIFF-EP standard".<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> Several cameras use DNG as their raw image format, so in that limited sense they use TIFF/EP too.<sup class="reference" id="cite_ref-barrypearson.co.uk_22-0"><a href="#cite_note-barrypearson.co.uk-22">[22]</a></sup></p>
<p><a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe Systems</a> launched this DNG raw image format in September 2004. By September 2006, several camera manufacturers had started to announce support for DNG in newer camera models, including <a href="/wiki/Leica_Camera" title="Leica Camera">Leica</a>, <a href="/wiki/Samsung" title="Samsung">Samsung</a>, <a href="/wiki/Ricoh" title="Ricoh">Ricoh</a>, Pentax, <a href="/wiki/Hasselblad" title="Hasselblad">Hasselblad</a> (native camera support); and, <a href="/wiki/Better_Light" title="Better Light">Better Light</a> (export).<sup class="reference" id="cite_ref-PEARSONproductssept2006_23-0"><a href="#cite_note-PEARSONproductssept2006-23">[23]</a></sup> The Leica Digital-Modul-R (DMR) was first to use DNG as its native format.<sup class="reference" id="cite_ref-PEARSONhistory_24-0"><a href="#cite_note-PEARSONhistory-24">[24]</a></sup> In September 2009 Adobe stated that there were no known intellectual property encumbrances or license requirements for DNG.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> (There is a "Digital Negative (DNG) Specification Patent License",<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup> but it does not actually state that there <i>are</i> any patents held on DNG, and the September 2009 statement was made at least 4 years after this license was published).</p>
<p>TIFF/EP began its 5-year revision cycle in 2006.<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup> Adobe offered the DNG specification to <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a> to be part of ISO's revised TIFF/EP standard.<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup><sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> A progress report in October 2008 from ISO about the revision of TIFF/EP stated that the revision "... currently includes two "interoperability-profiles," "IP 1" for processed image data, using ".TIF" extension, and "IP 2" for "raw" image data, ".DNG" extension".<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup> It is "IP 2" that is relevant here. A progress report in September 2009 states that "This format will be similar to DNG 1.3, which serves as the starting point for development."<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup></p>
<p>DNG has been used by open-source developers.<sup class="reference" id="cite_ref-dcinterview_15-1"><a href="#cite_note-dcinterview-15">[15]</a></sup> Use by camera makers varies: the largest companies such as Canon, Nikon, Sony, and some others, do not use DNG. Smaller companies and makers of "niche" cameras who might otherwise have difficulty getting support from software companies frequently use DNG as their native raw image format. Pentax uses DNG as an optional alternative to their own raw image format. There are 15 or more such companies, even including a few that specialize in movie cameras.<sup class="reference" id="cite_ref-barrypearson.co.uk_22-1"><a href="#cite_note-barrypearson.co.uk-22">[22]</a></sup> In addition, most Canon point &amp; shoot cameras can support DNG by using <a class="mw-redirect" href="/wiki/CHDK" title="CHDK">CHDK</a>.</p>
<h2><span class="mw-headline" id="Processing">Processing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=5" title="Edit section: Processing">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">See also: <a href="/wiki/Color_image_pipeline" title="Color image pipeline">Color image pipeline</a></div>
<p>To be viewed or printed, the output from a camera's <a href="/wiki/Image_sensor" title="Image sensor">image sensor</a> has to be processed, that is, converted to a photographic rendering of the scene, and then stored in a standard <a href="/wiki/Raster_graphics" title="Raster graphics">raster graphics</a> format such as <a href="/wiki/JPEG" title="JPEG">JPEG</a>. This processing, whether done in-camera or later in a raw-file converter, involves a number of operations, typically including:<sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup><sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup></p>
<ul>
<li>decoding – image data of raw files are typically encoded for compression purpose, but also often for obfuscation purpose (e.g. raw files from Canon<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup> or Nikon cameras).<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup></li>
<li><a href="/wiki/Demosaicing" title="Demosaicing">demosaicing</a> – interpolating the partial raw data received from the <a href="/wiki/Color_filter_array" title="Color filter array">color-filtered</a> image sensor into a matrix of colored pixels.</li>
<li><a href="/wiki/Defective_pixel" title="Defective pixel">defective pixel</a> removal – replacing data in known bad locations with interpolations from nearby locations</li>
<li><a class="mw-redirect" href="/wiki/White_balancing" title="White balancing">white balancing</a> – accounting for <a href="/wiki/Color_temperature" title="Color temperature">color temperature</a> of the light that was used to take the photograph</li>
<li><a href="/wiki/Noise_reduction" title="Noise reduction">noise reduction</a> – trading off detail for smoothness by removing small fluctuations</li>
<li><a class="mw-redirect" href="/wiki/Color_translation#Color_translation" title="Color translation">color translation</a> – converting from the camera native <a href="/wiki/Color_space" title="Color space">color space</a> defined by the <a href="/wiki/Spectral_sensitivity" title="Spectral sensitivity">spectral sensitivities</a> of the image sensor to an output color space (typically <a href="/wiki/SRGB" title="SRGB">sRGB</a> for JPEG)</li>
<li><a href="/wiki/Tone_reproduction" title="Tone reproduction">tone reproduction</a><sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup><sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup> – the scene luminance captured by the camera sensors and stored in the raw file (with a <a href="/wiki/Dynamic_range" title="Dynamic range">dynamic range</a> of typically 10 or more bits) needs to be rendered for pleasing effect and correct viewing on low-dynamic-range monitors or prints; the tone-reproduction rendering often includes separate <a href="/wiki/Tone_mapping" title="Tone mapping">tone mapping</a> and <a class="mw-redirect" href="/wiki/Gamma_compression" title="Gamma compression">gamma compression</a> steps.</li>
<li><a href="/wiki/Image_compression" title="Image compression">compression</a> – for example <a href="/wiki/JPEG#JPEG_compression" title="JPEG">JPEG compression</a></li>
</ul>
<p>Note that <a href="/wiki/Demosaicing" title="Demosaicing">demosaicing</a> is only performed for <a href="/wiki/Color_filter_array" title="Color filter array">CFA</a> sensors; it is not required for <a class="mw-redirect" href="/wiki/Three-CCD" title="Three-CCD">3CCD</a> or <a href="/wiki/Foveon_X3_sensor" title="Foveon X3 sensor">Foveon X3</a> sensors.</p>
<p>Cameras and image processing software may also perform additional processing to improve image quality, for example:</p>
<ul>
<li>removal of systematic noise – <a href="/wiki/Bias_frame" title="Bias frame">bias frame</a> subtraction and <a href="/wiki/Flat-field_correction" title="Flat-field correction">flat-field correction</a></li>
<li><a class="mw-redirect" href="/wiki/Dark_frame" title="Dark frame">dark frame</a> subtraction</li>
<li>optical correction – <a href="/wiki/Distortion_(optics)" title="Distortion (optics)">lens distortion</a>, <a href="/wiki/Vignetting" title="Vignetting">vignetting</a>, <a href="/wiki/Chromatic_aberration" title="Chromatic aberration">chromatic aberration</a> and <a href="/wiki/Purple_fringing" title="Purple fringing">color fringing</a> correction</li>
<li><a href="/wiki/Display_contrast" title="Display contrast">contrast manipulation</a></li>
<li>increasing <a href="/wiki/Visual_acuity" title="Visual acuity">visual acuity</a> by <a href="/wiki/Unsharp_masking" title="Unsharp masking">unsharp masking</a></li>
<li><a href="/wiki/High-dynamic-range_imaging" title="High-dynamic-range imaging">dynamic range compression</a> – lighten shadow regions without blowing out highlight regions</li>
</ul>
<p>When a camera saves a raw file it defers most of this processing; typically the only processing performed is the removal of defective pixels (the DNG specification requires that defective pixels are removed before creating the file<sup class="reference" id="cite_ref-38"><a href="#cite_note-38">[38]</a></sup>). Some camera manufacturers do additional processing before saving raw files; for example, Nikon has been criticized by <a href="/wiki/Astrophotography" title="Astrophotography">astrophotographers</a> for applying noise reduction before saving the raw file.<sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[39]</a></sup></p>
<p>Some raw formats also allow <a href="/wiki/Quantization_(image_processing)" title="Quantization (image processing)">nonlinear quantization</a>.<sup class="reference" id="cite_ref-adobe.com_40-0"><a href="#cite_note-adobe.com-40">[40]</a></sup><sup class="reference" id="cite_ref-majid.info_41-0"><a href="#cite_note-majid.info-41">[41]</a></sup> This nonlinearity allows the compression of the raw data without visible degradation of the image by removing invisible and irrelevant information from the image. Although noise is discarded this has nothing to do with (visible) noise reduction.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2008)">citation needed</span></a></i>]</sup></p>
<h3><span class="mw-headline" id="Benefits">Benefits</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=6" title="Edit section: Benefits">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Nearly all digital cameras can process the image from the <a href="/wiki/Charge-coupled_device" title="Charge-coupled device">sensor</a> into a JPEG file using settings for <a class="mw-redirect" href="/wiki/White_balance" title="White balance">white balance</a>, <a class="mw-redirect" href="/wiki/Saturation_(colour_theory)" title="Saturation (colour theory)">colour saturation</a>, <a href="/wiki/Contrast_(vision)" title="Contrast (vision)">contrast</a>, and <a href="/wiki/Unsharp_masking" title="Unsharp masking">sharpness</a> that are either selected automatically or entered by the photographer before taking the picture. Cameras that produce raw files save these settings in the file, but defer the processing. This results in an extra step for the photographer, so raw is normally only used when additional computer processing is intended. However, raw has numerous advantages over JPEG such as:</p>
<ul>
<li>Many more shades of colors compared to JPEG files - raw files have 12 or 14 bits of intensity information per channel (4096-16384 shades), compared to JPEG's <a href="/wiki/Gamma_correction" title="Gamma correction">gamma-compressed</a> 8 bits (256 shades).</li>
<li>Higher image quality. Because all the calculations (such as applying <a href="/wiki/Gamma_correction" title="Gamma correction">gamma correction</a>, <a href="/wiki/Demosaicing" title="Demosaicing">demosaicing</a>, white balance, <a href="/wiki/Brightness" title="Brightness">brightness</a>, contrast, etc...) used to generate pixel values (in RGB format for most images) are performed in one step on the base data, the resultant pixel values will be more accurate and exhibit less <a href="/wiki/Posterization" title="Posterization">posterization</a>.</li>
<li>Bypassing of undesired steps in the camera's processing, including <a href="/wiki/Sharpening" title="Sharpening">sharpening</a> and <a href="/wiki/Noise_reduction" title="Noise reduction">noise reduction</a></li>
<li>JPEG images are typically saved using a <a href="/wiki/Lossy_compression" title="Lossy compression">lossy compression</a> format (though a <a href="/wiki/Lossless_JPEG" title="Lossless JPEG">lossless JPEG</a> compression is now available). Raw formats typically use lossless compression or high-quality lossy compression.</li>
<li>Finer control. Raw conversion software allows users to manipulate more parameters (such as <a class="new" href="/w/index.php?title=Lightness_(colour)&amp;action=edit&amp;redlink=1" title="Lightness (colour) (page does not exist)">lightness</a>, white balance, <a href="/wiki/Hue" title="Hue">hue</a>, saturation, etc...) and do so with greater variability. For example, the white point can be set to any value, not just discrete preset values like "daylight" or "incandescent". Furthermore, the user can typically see a preview while adjusting these parameters.</li>
<li>The <a class="mw-redirect" href="/wiki/Colour_space" title="Colour space">colour space</a> can be set to whatever is desired.</li>
<li>Different <a href="/wiki/Demosaicing" title="Demosaicing">demosaicing</a> algorithms can be used, not just the one coded into the camera.</li>
<li>The contents of raw files include more information, and potentially higher quality, than the converted results, in which the rendering parameters are fixed, the colour gamut is clipped, and there may be <a href="/wiki/Quantization_(image_processing)" title="Quantization (image processing)">quantization</a> and <a href="/wiki/Compression_artifact" title="Compression artifact">compression</a> artifacts.</li>
<li>Large transformations of the data, such as increasing the exposure of a dramatically under-exposed photo, result in fewer visible artifacts when done from raw data than when done from already rendered image files. Raw data leave more scope for both corrections and artistic manipulations, without resulting in images with visible flaws such as <a href="/wiki/Posterization" title="Posterization">posterization</a>.</li>
<li>All the changes made on a raw image file are non-destructive; that is, only the metadata that controls the rendering is changed to make different output versions, leaving the original data unchanged.</li>
<li>To some extent, raw-format photography eliminates the need to use the <a class="mw-redirect" href="/wiki/HDRI" title="HDRI">HDRI</a> technique, allowing a much better control over the mapping of the scene intensity range into the output tonal range, compared to the process of automatically mapping to <a href="/wiki/JPEG" title="JPEG">JPEG</a> or other 8-bit representation.</li>
</ul>
<h3><span class="mw-headline" id="Drawbacks">Drawbacks</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=7" title="Edit section: Drawbacks">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>Camera raw file size is typically 2–6 times larger than JPEG file size.<sup class="reference" id="cite_ref-42"><a href="#cite_note-42">[42]</a></sup> While use of raw formats avoids the compression artifacts inherent in JPEG, fewer images can fit on a given <a href="/wiki/Memory_card" title="Memory card">memory card</a>. However, the large sizes and low prices of modern memory cards mitigate this. <a href="/wiki/Burst_mode_(photography)" title="Burst mode (photography)">Burst mode shooting</a> tends to be slower and shorter due to the larger file size.</li>
<li>Most raw formats implement <a class="mw-redirect" href="/wiki/Lossless_data_compression" title="Lossless data compression">lossless data compression</a> to reduce the size of the files without affecting image quality. But some others use <a class="mw-redirect" href="/wiki/Lossy_data_compression" title="Lossy data compression">lossy data compression</a> where quantization and filtering is performed on the image data.<sup class="reference" id="cite_ref-adobe.com_40-1"><a href="#cite_note-adobe.com-40">[40]</a></sup><sup class="reference" id="cite_ref-majid.info_41-1"><a href="#cite_note-majid.info-41">[41]</a></sup> Sony’s lossy 11+7 bit delta compression of RAW data causes posterization under certain conditions.<sup class="reference" id="cite_ref-43"><a href="#cite_note-43">[43]</a></sup> Several Nikon cameras let photographers choose between no compression, lossless compression or lossy compression for their raw images. <a href="/wiki/Red_Digital_Cinema_Camera_Company" title="Red Digital Cinema Camera Company">Red Digital Cinema Camera Company</a> introduced <code>.r3d</code> <a class="mw-redirect" href="/wiki/Red_Digital_Cinema" title="Red Digital Cinema">REDCODE RAW</a> with compression ratio from 3:1 to 18:1 which depends on resolution and frame rates.<sup class="reference" id="cite_ref-44"><a href="#cite_note-44">[44]</a></sup></li>
<li>The standard raw image format (ISO 12234-2, <a class="mw-redirect" href="/wiki/Tag_Image_File_Format_/_Electronic_Photography" title="Tag Image File Format / Electronic Photography">TIFF/EP</a>) is not widely accepted. <a class="mw-redirect" href="/wiki/Digital_Negative_(file_format)" title="Digital Negative (file format)">DNG</a>, the potential candidate for a new standard format, has not been adopted by many major camera companies. (See "<a href="#Standardization">Standardization</a>" section). Numerous different raw formats are currently in use and new raw formats keep appearing, while others are abandoned.<sup class="reference" id="cite_ref-45"><a href="#cite_note-45">[45]</a></sup></li>
<li>Because of the lack of widespread adoption of a standard raw format, more specialized software may be required to open raw files than for standardized formats like JPEG or TIFF. Software developers have to frequently update their products to support the raw formats of the latest cameras but open source implementations like <a href="/wiki/Dcraw" title="Dcraw">dcraw</a> make it easier.</li>
<li>The time taken in the image workflow is an important factor when choosing between raw and ready-to-use image formats. With modern photo editing software the additional time needed to process raw images has been greatly reduced but it still requires an extra step in workflow in comparison with using out-of-camera JPEGs.</li>
</ul>
<h2><span class="mw-headline" id="Software_support">Software support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=8" title="Edit section: Software support">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Cameras that support raw files typically come with <a href="/wiki/Proprietary_software" title="Proprietary software">proprietary software</a> for conversion of their raw image data into standard RGB images. Other processing and conversion <a href="/wiki/Computer_program" title="Computer program">programs</a> and <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plugins</a> are available from vendors that have either licensed the technology from the camera manufacturer or <a href="/wiki/Reverse_engineering" title="Reverse engineering">reverse-engineered</a> the particular raw format and provided their own processing algorithms.</p>
<h3><span class="mw-headline" id="Operating_system_support">Operating system support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=9" title="Edit section: Operating system support">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<h4><span class="mw-headline" id="Apple_Mac_OS_X_and_iOS">Apple Mac OS X and iOS</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=10" title="Edit section: Apple Mac OS X and iOS">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>In January 2005, <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple</a> released <a href="/wiki/IPhoto" title="IPhoto">iPhoto</a> 5, which offered basic support for viewing and editing many raw file formats.</p>
<p>In April 2005, Apple's <a href="/wiki/Mac_OS_X_Tiger" title="Mac OS X Tiger">OS X 10.4</a> brought raw support to the operating system's ImageIO framework, enabling raw support automatically in the majority of OS X applications both from Apple (such as Preview, OS X's PDF and image viewing application, and <a href="/wiki/Aperture_(software)" title="Aperture (software)">Aperture</a>, a photo post-production software package for professionals) as well as all third party applications which make use of the ImageIO frameworks.</p>
<p>Semi-regular updates to OS X generally include updated support for new raw formats introduced in the intervening months by camera manufacturers.</p>
<p>In 2016, Apple announced that <a href="/wiki/IOS_10" title="IOS 10">iOS 10</a> would allow capturing raw images on selected hardware, and third party applications will be able to edit raw images through the operating system's <a href="/wiki/Core_Image" title="Core Image">Core Image</a> framework.<sup class="reference" id="cite_ref-46"><a href="#cite_note-46">[46]</a></sup></p>
<h4><span class="mw-headline" id="Microsoft_Windows">Microsoft Windows</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=11" title="Edit section: Microsoft Windows">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<h5><span class="mw-headline" id="Windows_Camera_Codec_Pack">Windows Camera Codec Pack</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=12" title="Edit section: Windows Camera Codec Pack">edit</a><span class="mw-editsection-bracket">]</span></span></h5>
<p><a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> supplies the free Windows Camera Codec Pack for <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a> and later versions of <a class="mw-redirect" href="/wiki/Windows" title="Windows">Windows</a>, to integrate raw file viewing and printing into some Windows tools.<sup class="reference" id="cite_ref-47"><a href="#cite_note-47">[47]</a></sup> The codecs allow native viewing of raw files from a variety of specific cameras in <a href="/wiki/File_Explorer" title="File Explorer">Windows Explorer / File Explorer</a> and <a href="/wiki/Windows_Photo_Gallery" title="Windows Photo Gallery">Windows Live Photo Gallery / Windows Photo Gallery</a>, in <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a> and <a href="/wiki/Windows_7" title="Windows 7">Windows 7</a>.<sup class="reference" id="cite_ref-mccp_48-0"><a href="#cite_note-mccp-48">[48]</a></sup> As of October 2016, Microsoft had not released an updated version since April 2014, which supported some specific cameras by the following manufacturers: Canon, Casio, Epson, Fujifilm, Kodak, Konica Minolta, Leica, Nikon, Olympus, Panasonic, Pentax, Samsung, and Sony.<sup class="reference" id="cite_ref-mccp_48-1"><a href="#cite_note-mccp-48">[48]</a></sup></p>
<h5><span class="mw-headline" id="Windows_Imaging_Component_.28WIC.29">Windows Imaging Component (WIC)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=13" title="Edit section: Windows Imaging Component (WIC)">edit</a><span class="mw-editsection-bracket">]</span></span></h5>
<div class="hatnote" role="note">Main article: <a href="/wiki/Windows_Imaging_Component" title="Windows Imaging Component">Windows Imaging Component</a></div>
<p>Windows supports the Windows Imaging Component (WIC) codec standard. WIC was available as a stand-alone downloadable program for Windows XP Service Pack 2, and built into <a href="/wiki/Windows_XP#Service_Pack_3" title="Windows XP">Windows XP Service Pack 3</a>, <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a>, and later versions. Windows Explorer / File Explorer, and Windows Live Photo Gallery / Windows Photo Gallery can view raw formats for which the necessary WIC codecs are installed. Canon, Nikon, Sony, Olympus and Pentax have released WIC codecs for their cameras, although some manufactures only provide codec support for the 32-bit versions of Windows.<sup class="reference" id="cite_ref-49"><a href="#cite_note-49">[49]</a></sup></p>
<p>Commercial <a class="mw-redirect" href="/wiki/Digital_Negative_(file_format)" title="Digital Negative (file format)">DNG</a> WIC codecs are also available from Ardfry Imaging,<sup class="reference" id="cite_ref-50"><a href="#cite_note-50">[50]</a></sup> and others; and <a class="mw-redirect" href="/wiki/Fast_Picture_Viewer" title="Fast Picture Viewer">FastPictureViewer Professional</a> installs a set of WIC-enabled image decoders.<sup class="reference" id="cite_ref-51"><a href="#cite_note-51">[51]</a></sup><sup class="reference" id="cite_ref-52"><a href="#cite_note-52">[52]</a></sup></p>
<h4><span class="mw-headline" id="Android">Android</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=14" title="Edit section: Android">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p><a href="/wiki/Android_Lollipop" title="Android Lollipop">Android Lollipop</a> 5.0, introduced in late 2014, can allow <a href="/wiki/Smartphone" title="Smartphone">smartphones</a> to take RAW images, useful in low-light situations.<sup class="reference" id="cite_ref-53"><a href="#cite_note-53">[53]</a></sup></p>
<h3><span class="mw-headline" id="Free_and_open_source_software">Free and open source software</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=15" title="Edit section: Free and open source software">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li><a href="/wiki/Darktable" title="Darktable">darktable</a> is a raw-workflow tool for Linux and other open Unix-like operating systems. Features native 32-bit floating point processing and a plugin architecture.</li>
<li><a href="/wiki/Dcraw" title="Dcraw">dcraw</a> is a program which reads most raw formats and can be made to run on operating systems not supported by most commercial software (such as <a href="/wiki/Unix" title="Unix">Unix</a>). Libraw<sup class="reference" id="cite_ref-54"><a href="#cite_note-54">[54]</a></sup> is an <a class="mw-redirect" href="/wiki/API" title="API">API</a> library based on dcraw, offering a more convenient interface for reading and converting raw files. <a href="/wiki/HDR_PhotoStudio" title="HDR PhotoStudio">HDR PhotoStudio</a> and AZImage<sup class="reference" id="cite_ref-55"><a href="#cite_note-55">[55]</a></sup> are some of the commercial applications that use Libraw. Jrawio is another <a class="mw-redirect" href="/wiki/API" title="API">API</a> library, written in pure <a href="/wiki/Java_(software_platform)" title="Java (software platform)">Java</a> code and compliant to the standard Java Image I/O API.</li>
<li><a href="/wiki/DigiKam" title="DigiKam">digiKam</a> is an advanced digital photo management application for Linux, Windows, and Mac OS X that supports raw processing.</li>
<li><a href="/wiki/ExifTool" title="ExifTool">ExifTool</a> supports the reading, writing and editing of metadata in raw image files. ExifTool supports many different types of metadata including <a href="/wiki/Exif" title="Exif">Exif</a>, GPS, <a href="/wiki/IPTC_Information_Interchange_Model" title="IPTC Information Interchange Model">IPTC</a>, <a href="/wiki/Extensible_Metadata_Platform" title="Extensible Metadata Platform">XMP</a>, <a href="/wiki/JPEG_File_Interchange_Format" title="JPEG File Interchange Format">JFIF</a>, <a href="/wiki/GeoTIFF" title="GeoTIFF">GeoTIFF</a>, <a class="mw-redirect" href="/wiki/ICC_Profile" title="ICC Profile">ICC Profile</a>, Photoshop IRB, <a href="/wiki/FlashPix" title="FlashPix">FlashPix</a>, AFCP and ID3, as well as the maker notes of many digital cameras.</li>
<li><a href="/wiki/ImageMagick" title="ImageMagick">ImageMagick</a>, a popular software suite for image manipulation and conversion, reads many different raw file formats.<sup class="reference" id="cite_ref-56"><a href="#cite_note-56">[56]</a></sup> <a href="/wiki/ImageMagick" title="ImageMagick">ImageMagick</a> is available for Linux/Unix, Mac OS, Windows, and other platforms.</li>
<li><a href="/wiki/LightZone" title="LightZone">LightZone</a> is a photo editing program providing the ability to edit many raw formats natively. Most tools are raw converters, but LightZone allows a user to edit a raw file as if it were TIFF or JPEG. The project was discontinued in September 2011<sup class="reference" id="cite_ref-57"><a href="#cite_note-57">[57]</a></sup> and reinstated as an open source project in December 2012.</li>
<li><a href="/wiki/Rawstudio" title="Rawstudio">Rawstudio</a> is a raw format developer.</li>
<li><a href="/wiki/RawTherapee" title="RawTherapee">RawTherapee</a> is a raw developer supporting Linux, OS X and Windows operating systems. It features a native 32-bit floating point pipeline.</li>
<li><a href="/wiki/Shotwell_(software)" title="Shotwell (software)">Shotwell</a> is an image organizer available for all major operating systems with the ability to view and edit raw images and has built-in social networking upload capability.</li>
<li><a href="/wiki/UFRaw" title="UFRaw">UFRaw</a> is a <a class="mw-redirect" href="/wiki/Front_end_processor_(program)" title="Front end processor (program)">frontend</a> which uses <a href="/wiki/Dcraw" title="Dcraw">dcraw</a> as a <a href="/wiki/Front_and_back_ends" title="Front and back ends">back end</a>. It can be used as a <a href="/wiki/GIMP" title="GIMP">GIMP</a> plugin and is available for most operating systems.</li>
</ul>
<h3><span class="mw-headline" id="Proprietary_software">Proprietary software</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=16" title="Edit section: Proprietary software">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In addition to those listed under operating system support, above, the commercial software described below support raw formats.</p>
<h4><span class="mw-headline" id="Dedicated_raw_converters">Dedicated raw converters</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=17" title="Edit section: Dedicated raw converters">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>The following products were launched as RAW processing software to process a wide range of raw files, and have this as their main purpose:</p>
<ul>
<li><a href="/wiki/Adobe_Photoshop_Lightroom" title="Adobe Photoshop Lightroom">Adobe Photoshop Lightroom</a></li>
<li><a href="/wiki/Bibble_(software)" title="Bibble (software)">Bibble Pro</a> (now <a href="/wiki/Corel_AfterShot_Pro" title="Corel AfterShot Pro">Corel AfterShot Pro</a>)</li>
<li><a href="/wiki/Capture_One" title="Capture One">Capture One</a><sup class="reference" id="cite_ref-58"><a href="#cite_note-58">[58]</a></sup></li>
<li><a class="mw-redirect" href="/wiki/DxO_Optics_Pro" title="DxO Optics Pro">DxO Optics Pro</a></li>
<li><a href="/wiki/Hasselblad" title="Hasselblad">Hasselblad</a>'s Phocus relies on operating system support to process non-Hasselblad files</li>
<li><a href="/wiki/Photo_Ninja" title="Photo Ninja">Photo Ninja</a></li>
<li><a href="/wiki/Silkypix_Developer_Studio" title="Silkypix Developer Studio">Silkypix Developer Studio</a></li>
<li><a href="/wiki/MagicRaw" title="MagicRaw">MagicRaw</a></li>
</ul>
<h4><span class="mw-headline" id="Others">Others</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=18" title="Edit section: Others">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<ul>
<li><a class="mw-redirect" href="/wiki/ACDSee_Pro" title="ACDSee Pro">ACDSee Pro</a> is photo management and editing software that supports the raw formats of 21 camera manufacturers.<sup class="reference" id="cite_ref-59"><a href="#cite_note-59">[59]</a></sup></li>
<li><a href="/wiki/Adobe_Photoshop" title="Adobe Photoshop">Adobe Photoshop</a> supports raw formats (as of version CS2).</li>
<li><i>DNG Viewer</i> is a free Windows (32bit) viewer based on <a href="/wiki/Dcraw" title="Dcraw">dcraw</a>. The very simple viewer is installed as <i>RAW Image Viewer</i>, supports some lossless operations, and can save raw images as <a href="/wiki/BMP_file_format" title="BMP file format">BMP</a>, <a href="/wiki/JPEG" title="JPEG">JPEG</a>, <a href="/wiki/Portable_Network_Graphics" title="Portable Network Graphics">PNG</a>, or <a class="mw-redirect" href="/wiki/Tagged_Image_File_Format" title="Tagged Image File Format">TIFF</a>.<sup class="reference" id="cite_ref-60"><a href="#cite_note-60">[60]</a></sup></li>
<li>FastRawViewer is a dedicated RAW viewer that runs on Mac and Windows, and currently claims to support all RAW formats except Foveon.<sup class="reference" id="cite_ref-61"><a href="#cite_note-61">[61]</a></sup></li>
<li><a href="/wiki/Helicon_Filter" title="Helicon Filter">Helicon Filter</a> supports raw formats.</li>
<li><a href="/wiki/IrfanView" title="IrfanView">IrfanView</a> is a freeware/shareware basic editor with support for raw files.</li>
<li><a class="new" href="/w/index.php?title=Konvertor&amp;action=edit&amp;redlink=1" title="Konvertor (page does not exist)">Konvertor</a> support for raw formats is based on dcraw.</li>
<li><a class="mw-redirect" href="/wiki/Paint_Shop_Pro" title="Paint Shop Pro">Paint Shop Pro</a> contains raw support, although as in the case of most editors updates to the program may be necessary to attain compatibility with newer raw formats as they are released.</li>
<li><a href="/wiki/PhotoLine" title="PhotoLine">PhotoLine</a> supports raw formats.</li>
<li><a href="/wiki/Picasa" title="Picasa">Picasa</a> (development discontinued) is a free editor and organizer from <a href="/wiki/Google" title="Google">Google</a>. It can read and display many raw formats, but like <a href="/wiki/IPhoto" title="IPhoto">iPhoto</a>, Picasa provides only limited tools for processing the data in a raw file.</li>
<li><a href="/wiki/SilverFast" title="SilverFast">SilverFast</a> supports raw formats.</li>
<li><a href="/wiki/XnView" title="XnView">XnView</a> support for raw formats is mostly based on dcraw.</li>
</ul>
<h4><span class="mw-headline" id="HTML5_browser-based_apps">HTML5 browser-based apps</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_image_format&amp;action=edit&amp;section=19" title="Edit section: HTML5 browser-based apps">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>A new class of raw file processing tools appeared with the development of <a href="/wiki/HTML5" title="HTML5">HTML5</a> - <a href="/wiki/Rich_Internet_application" title="Rich Internet application">rich Internet applications</a>.</p>
<ul>
<li><a href="/wiki/Pics.io" title="Pics.io">Raw.pics.io</a> is able to render and apply basic adjustments to raw and DNG files.</li>
</ul>
