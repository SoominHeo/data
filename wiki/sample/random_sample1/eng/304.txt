<h1 class="firstHeading" id="firstHeading" lang="en">.exe</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">

</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><div class="hatnote" role="note">"EXE" redirects here. For other uses, see <a class="mw-redirect mw-disambig" href="/wiki/EXE_(disambiguation)" title="EXE (disambiguation)">EXE (disambiguation)</a>.</div>
<p><b>.exe</b> is a common <a href="/wiki/Filename_extension" title="Filename extension">filename extension</a> denoting an <a class="mw-redirect" href="/wiki/Executable_file" title="Executable file">executable file</a> (the main execution point of a <a href="/wiki/Computer_program" title="Computer program">computer program</a>) for <a href="/wiki/DOS" title="DOS">DOS</a>, <a href="/wiki/OpenVMS" title="OpenVMS">OpenVMS</a>, <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>, <a href="/wiki/Symbian" title="Symbian">Symbian</a> or <a href="/wiki/OS/2" title="OS/2">OS/2</a>. Besides the executable program, many .exe files contain other components called <a href="/wiki/Resource_(Windows)" title="Resource (Windows)">resources</a>, such as <a class="mw-redirect" href="/wiki/Bitmap_graphics" title="Bitmap graphics">bitmap graphics</a> and <a class="mw-redirect" href="/wiki/Computer_icon" title="Computer icon">icons</a> which the executable program may use for its <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a>.</p>
<p></p>

<h2><span class="mw-headline" id="File_formats">File formats</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.exe&amp;action=edit&amp;section=1" title="Edit section: File formats">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>There are several <a class="mw-redirect" href="/wiki/File_formats" title="File formats">file formats</a> which may be used by a file with a <span style="font-family: monospace, monospace;">.exe</span> extension:</p>
<h3><span class="mw-headline" id="DOS"><span id="MZ"></span><span id="NE"></span>DOS</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.exe&amp;action=edit&amp;section=2" title="Edit section: DOS">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/DOS_MZ_executable" title="DOS MZ executable">DOS MZ executable</a></div>
<dl>
<dt>16-bit DOS MZ executable</dt>
<dd>The original DOS executable file format. These can be identified by the letters "MZ" at the beginning of the file in ASCII.</dd>
<dt>16-bit New Executable</dt>
<dd>Introduced with the multitasking <a href="/wiki/MS-DOS_4.0_(multitasking)" title="MS-DOS 4.0 (multitasking)">MS-DOS 4.0</a> and also used by 16-bit OS/2 and Windows, <a href="/wiki/New_Executable" title="New Executable">NE</a> can be identified by the "NE" in ASCII.</dd>
</dl>
<h3><span class="mw-headline" id="OS.2F2">OS/2</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.exe&amp;action=edit&amp;section=3" title="Edit section: OS/2">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><span id="LX"></span><span id="LE"></span></p>
<dl>
<dt>32-bit Linear Executable</dt>
<dd>Introduced with OS/2 2.0, these can be identified by the "LX" in ASCII. These can only be run by OS/2 2.0 and higher.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> They are also used by some <a href="/wiki/DOS_extender" title="DOS extender">DOS extenders</a>.</dd>
<dt>Mixed 16/32/64-bit Linear Executable</dt>
<dd>Introduced with OS/2 2.0, these can be identified by the "LE" in ASCII. This format is used for <a href="/wiki/VxD" title="VxD">VxD</a> drivers under <a href="/wiki/Windows_3.1x" title="Windows 3.1x">Windows 3.x</a>, <a href="/wiki/OS/2" title="OS/2">OS/2</a>, and Windows 9x; it is also used by some DOS extenders.</dd>
</dl>
<h3><span class="mw-headline" id="Windows">Windows</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.exe&amp;action=edit&amp;section=4" title="Edit section: Windows">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><span id="PE"></span><span id="PE32.2B"></span></p>
<div class="hatnote" role="note">Main article: <a href="/wiki/Portable_Executable" title="Portable Executable">Portable Executable</a></div>
<p>When a 16-bit or 32-bit Windows executable is run by VLC, execution starts at either the NE or the PE, and ignores the MZ code known as <a href="/wiki/New_Executable#DOS_stub" title="New Executable">DOS stub</a>.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> Started in <a href="/wiki/DOS" title="DOS">DOS</a> the stub typically displays a message "This program cannot be run in DOS mode" (or similar) before exiting cleanly. A few dual-mode programs (MZ-NE or MZ-PE) such as <a href="/wiki/Windows_Registry" title="Windows Registry">regedit</a><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> and older WinZIP self extractors include a more functional DOS section.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<dl>
<dt>32-bit <a href="/wiki/Portable_Executable" title="Portable Executable">Portable Executable</a></dt>
<dd>Introduced with Windows NT, these can be identified by the "PE" in ASCII (although not at the beginning; these files also begin with "MZ").<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></dd>
<dt>64-bit Portable Executable (PE32+)</dt>
<dd>Introduced by 64-bit versions of Windows, this is a PE file with wider fields. In most cases, code can be written to simply work as either a 32 or 64-bit PE file.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></dd>
</dl>

<p>Besides these, there are also many custom EXE formats, such as <i>W3</i> (a collection of LE files, only used in WIN386.EXE), <i>W4</i> (a compressed collection of LE files, only used in VMM32.VXD), <i>DL</i>, <i>MP</i>, <i>P2</i>, <i>P3</i> (last three used by <a href="/wiki/Phar_Lap_(company)" title="Phar Lap (company)">Phar Lap</a> extenders), and probably more.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (February 2007)">citation needed</span></a></i>]</sup></p>
<p>The PE format was also used in <a href="/wiki/BeOS" title="BeOS">BeOS</a> R3, although the format used by BeOS somewhat violated the PE specification as it didn't specify a correct subsystem.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (February 2007)">citation needed</span></a></i>]</sup></p>
