<h1 class="firstHeading" id="firstHeading" lang="en">.bss</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">

</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><div class="hatnote" role="note">For the wifi <b>Basic Service Set</b> (BSS), see <a href="/wiki/Wireless_LAN#Basic_service_set" title="Wireless LAN">Wireless_LAN § Basic_service_set</a>.</div>
<p>In <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a>, the name <b>.bss</b> or <b>bss</b> is used by many <a href="/wiki/Compiler" title="Compiler">compilers</a> and <a href="/wiki/Linker_(computing)" title="Linker (computing)">linkers</a> for a part of the <a href="/wiki/Data_segment" title="Data segment">data segment</a> containing <a href="/wiki/Static_variable" title="Static variable">statically-allocated variables</a> represented solely by zero-valued <a href="/wiki/Bit" title="Bit">bits</a> initially (i.e., when execution begins). It is often referred to as the "bss section" or "bss segment".</p>
<p>Typically only the length of the bss section, but no data, is stored in the <a href="/wiki/Object_file" title="Object file">object file</a>. The <a class="mw-redirect" href="/wiki/Program_loader" title="Program loader">program loader</a> allocates and initializes memory for the bss section when it loads the program. Operating systems may use a technique called <a class="mw-redirect" href="/wiki/Copy_on_write" title="Copy on write">copy on write</a> to efficiently implement the bss segment (<a href="#CITEREFMcKusickKarels1986">McKusick &amp; Karels 1986</a>). In embedded software, the bss segment is mapped into memory that is initialized to zero by the C <a class="mw-redirect" href="/wiki/Run-time_system" title="Run-time system">run-time system</a> before <code><a class="mw-redirect" href="/wiki/Main_function_(programming)#C_and_C.2B.2B" title="Main function (programming)">main()</a></code> is entered.</p>
<p>On some <a href="/wiki/Computer_architecture" title="Computer architecture">computer architectures</a> the <a href="/wiki/Application_binary_interface" title="Application binary interface">application binary interface</a> also supports an <b>sbss</b> segment for "small data". Typically, these data items can be accessed using shorter instructions that may only be able to access a certain range of addresses.</p>

<h2><span class="mw-headline" id="Origin">Origin</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.bss&amp;action=edit&amp;section=1" title="Edit section: Origin">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Historically, <b>BSS</b> (from <b>Block Started by Symbol</b>) was a <a href="/wiki/Assembly_language#Related_terminology" title="Assembly language">pseudo-operation</a> in <a class="new" href="/w/index.php?title=UA-SAP&amp;action=edit&amp;redlink=1" title="UA-SAP (page does not exist)">UA-SAP</a> (United Aircraft Symbolic Assembly Program), the <a href="/wiki/Assembly_language#Assembler" title="Assembly language">assembler</a> developed in the mid-1950s for the <a href="/wiki/IBM_704" title="IBM 704">IBM 704</a> by Roy Nutt, Walter Ramshaw, and others at <a href="/wiki/United_Aircraft" title="United Aircraft">United Aircraft Corporation</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> The BSS keyword was later incorporated into <a class="mw-redirect" href="/wiki/FORTRAN_Assembly_Program" title="FORTRAN Assembly Program">FAP</a> (FORTRAN Assembly Program), <a href="/wiki/IBM" title="IBM">IBM</a>'s standard assembler for its <a href="/wiki/IBM_700/7000_series#Scientific_Architecture" title="IBM 700/7000 series">709 and 7090/94</a> computers. It defined a label (i.e. symbol) and reserved a block of uninitialized space for a given number of <a class="mw-redirect" href="/wiki/Word_(data_type)" title="Word (data type)">words</a> (<a href="#CITEREFTimar1996">Timar 1996</a>). In this situation BSS served as a shorthand in place of individually reserving a number of separate smaller data locations. Some assemblers support a complementary or alternative directive <b>BES</b>, for <b>Block Ended by Symbol</b>, where the specified symbol corresponds to the end of the reserved block.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<h2><span class="mw-headline" id="BSS_in_C">BSS in C</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.bss&amp;action=edit&amp;section=2" title="Edit section: BSS in C">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, statically-allocated objects without an explicit initializer are initialized to zero (for arithmetic types) or a null pointer (for pointer types). Implementations of C typically represent zero values and null pointer values using a bit pattern consisting solely of zero-valued bits (though this is not required by the C standard). Hence, the BSS segment typically includes all uninitialized objects (both variables and <a href="/wiki/Constant_(computer_programming)" title="Constant (computer programming)">constants</a>) declared at file scope (i.e., outside any function) as well as uninitialized <a class="mw-redirect" href="/wiki/Static_local_variable" title="Static local variable">static local variables</a> (<a href="/wiki/Local_variable" title="Local variable">local variables</a> declared with the <code><a href="/wiki/Static_(keyword)" title="Static (keyword)">static</a></code> <a class="mw-redirect" href="/wiki/Keyword_(computer_programming)" title="Keyword (computer programming)">keyword</a>); static local constants must be initialized at declaration, however, as they do not have a separate declaration, and thus are typically not in the BSS section, though they may be implicitly or explicitly initialized to zero. An implementation may also assign statically-allocated variables and constants initialized with a value consisting solely of zero-valued bits to the BSS section.</p>
<p><a href="/wiki/Peter_van_der_Linden" title="Peter van der Linden">Peter van der Linden</a>, a C programmer and author, says, "Some people like to remember it as 'Better Save Space.' Since the BSS segment only holds variables that don't have any value yet, it doesn't actually need to store the image of these variables. The size that BSS will require at runtime is recorded in the object file, but BSS (unlike the data segment) doesn't take up any actual space in the object file."<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<h2><span class="mw-headline" id="BSS_in_Fortran">BSS in Fortran</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.bss&amp;action=edit&amp;section=3" title="Edit section: BSS in Fortran">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In <a href="/wiki/Fortran" title="Fortran">Fortran</a>, common block variables are allocated in this segment <sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup>. Note that even on x86_64, 32-bit pointers are often used to address this data, limiting the size of the common blocks to 2 GB. <sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
