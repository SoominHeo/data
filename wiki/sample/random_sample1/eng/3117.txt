<p><b>Nexenta OS</b>, officially known as the <b>Nexenta Core Platform</b>, was a computer <a href="/wiki/Operating_system" title="Operating system">operating system</a> based on <a href="/wiki/OpenSolaris" title="OpenSolaris">OpenSolaris</a> and <a href="/wiki/Ubuntu_(operating_system)" title="Ubuntu (operating system)">Ubuntu</a> that runs on <a href="/wiki/IA-32" title="IA-32">IA-32</a>- and <a href="/wiki/X86-64" title="X86-64">x86-64</a>-based systems. It emerged in fall 2005, after <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a> started the <a href="/wiki/OpenSolaris" title="OpenSolaris">OpenSolaris</a> project in June of that year.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> <a href="/wiki/Nexenta_Systems" title="Nexenta Systems">Nexenta Systems</a>, Inc. initiated the project and sponsored its development. Nexenta OS version 1.0 was released in February 2008.</p>
<p>In late 2011, the Nexenta OS brand was terminated and replaced with <b>Illumian</b>, which is derived from community development for <a href="/wiki/Illumos" title="Illumos">illumos</a> and <a href="/wiki/OpenIndiana" title="OpenIndiana">OpenIndiana</a>, but was distinguished by its use of Debian packaging. Illumian version 1.0 was released in February 2012. Following the initial release of Illumian in 2012, the Illumian project was discontinued.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>

<h2><span class="mw-headline" id="Repositories">Repositories</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Nexenta_OS&amp;action=edit&amp;section=1" title="Edit section: Repositories">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>There were initially two official <a href="/wiki/Advanced_Packaging_Tool" title="Advanced Packaging Tool">APT</a> repositories: "testing" and "unstable", totalling more than 9000 packages. A third "stable" was added for the first official release. Nexenta OS is available as InstallCD and <a href="/wiki/VMware" title="VMware">VMware</a> images. A <a href="/wiki/Live_CD" title="Live CD">Live CD</a> is also available, but only for releases up to alpha 5.</p>
<p>Nexenta repositories follow the general Debian structure (main contrib non-free). Stable releases use a subset of this structure. Packages originate from Debian GNU/Linux. The nexenta.org website was retired in early 2012.</p>
<h2><span class="mw-headline" id="Development">Development</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Nexenta_OS&amp;action=edit&amp;section=2" title="Edit section: Development">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Since Nexenta OS does not use the <a href="/wiki/Linux_kernel" title="Linux kernel">Linux kernel</a>, and <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun</a> only recently began releasing the code of their <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">Solaris</a> operating system as <a href="/wiki/Free_software" title="Free software">free</a> and <a class="mw-redirect" href="/wiki/Open_source_software" title="Open source software">open source software</a>, it supports less diverse hardware than other <a href="/wiki/Debian" title="Debian">Debian</a> variants.</p>
<p>The Nexenta OS team has decided to focus on a minimal OpenSolaris effort called the Nexenta Core Platform (NCP)<sup class="reference" id="cite_ref-ncp_4-0"><a href="#cite_note-ncp-4">[4]</a></sup> which forms the basis of the <a href="/wiki/NexentaStor" title="NexentaStor">NexentaStor</a> NAS storage solution. Version 1.0 of Nexenta Core Platform was released on February 10, 2008.</p>
<p>Nexenta Core Platform was the first operating system to combine the OpenSolaris <a class="mw-redirect" href="/wiki/Kernel_(computer_science)" title="Kernel (computer science)">kernel</a> with <a href="/wiki/GNU" title="GNU">GNU</a> <a class="mw-redirect" href="/wiki/Userland_(computing)" title="Userland (computing)">userland</a> tools. It aimed to bring technologies such as <a href="/wiki/ZFS" title="ZFS">ZFS</a> and <a href="/wiki/Solaris_Containers" title="Solaris Containers">Zones</a> to the <a href="/wiki/GNU" title="GNU">GNU</a>/<a href="/wiki/Debian" title="Debian">Debian</a> community. An additional purpose of the OS was to provide a kernel allowing proprietary closed source hardware drivers to be produced for inclusion in an <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> operating system.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<h2><span class="mw-headline" id="Releases">Releases</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Nexenta_OS&amp;action=edit&amp;section=3" title="Edit section: Releases">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>Nexenta Core Platform 1.0 was released on February 10, 2008. It was based on Ubuntu 6.06 "Dapper Drake" and OpenSolaris b82.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> The release included:
<ul>
<li><a href="/wiki/ZFS" title="ZFS">ZFS</a> root/boot</li>
<li><a href="/wiki/Solaris_Containers" title="Solaris Containers">zones</a> integrated with <a href="/wiki/Dpkg" title="Dpkg">dpkg</a>/<a class="mw-redirect" href="/wiki/Apt-get" title="Apt-get">apt-get</a></li>
<li><a class="mw-redirect" href="/wiki/Branded_zones" title="Branded zones">brandz</a> with preconfigured support for <a href="/wiki/Debian" title="Debian">Debian</a> and <a href="/wiki/Ubuntu_(operating_system)" title="Ubuntu (operating system)">Ubuntu</a></li>
<li>stable toolchain</li>
<li>server/storage software support</li>
<li><a href="/wiki/Xen" title="Xen">Xen</a> <a class="mw-redirect" href="/wiki/DomU" title="DomU">DomU</a>/<a class="mw-redirect" href="/wiki/Dom0" title="Dom0">Dom0</a></li>
</ul>
</li>
<li>Nexenta Core Platform 2.0 was released on May 25, 2009. It included packages based on Ubuntu 8.04 "Hardy Heron" LTS.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></li>
<li>Nexenta Core Platform 3.0.1 was released on September 17, 2010. The release included many updated packages, support for ZFS deduplication, and Crossbow support, among other fixes.<sup class="reference" id="cite_ref-DW_8-0"><a href="#cite_note-DW-8">[8]</a></sup></li>
</ul>
<h2><span class="mw-headline" id="NexentaStor">NexentaStor</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Nexenta_OS&amp;action=edit&amp;section=4" title="Edit section: NexentaStor">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><b>NexentaStor</b> is a proprietary operating system built by Nexenta Systems on top of the Nexenta Core Platform. It is optimized for use in virtualized server environments including <a href="/wiki/Network-attached_storage" title="Network-attached storage">NAS</a>, <a href="/wiki/ISCSI" title="ISCSI">iSCSI</a>, and <a href="/wiki/Fibre_Channel" title="Fibre Channel">Fibre Channel</a> applications, and is built around the <a href="/wiki/ZFS" title="ZFS">ZFS</a> file system. It features <a href="/wiki/ISCSI" title="ISCSI">iSCSI</a> support, unlimited incremental backups or 'snapshots', snapshot mirroring (replication), block level mirroring (<a href="/wiki/Continuous_data_protection" title="Continuous data protection">CDP</a>), integrated search within ZFS snapshots and a custom API. Through its focus on ZFS, it carries with it potential benefits for virtualized server farms in terms of performance and <a href="/wiki/Thin_provisioning" title="Thin provisioning">thin provisioning</a>. The operating system is currently distributed as <a href="/wiki/ISO_9660" title="ISO 9660">ISO</a> and <a href="/wiki/VMware" title="VMware">VMware</a> <a href="/wiki/Disk_image" title="Disk image">disk images</a> with pricing determined on a per-terabyte and per-server licensing basis.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
<p>A "community edition" is available free of charge for users with less than 18 terabytes of raw disk space (pre-redundancy and volume creation).<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> This free tier model is likely to change with the release of version 5.0 (community edition only) to a 10 terabyte limit of assigned and usable (post-redundancy and volume creation) disk space. It is the same as the enterprise version, except for the lack of paid plugins and commercial support. It can install many of the free and open source plugins that are developed and hosted on the community edition website. The "community edition" though is only intended for hobbyist use. Production use of the "community edition" is forbidden by the NexentaStor EULA. Therefore, the use of the term "community edition" remains very disputable.</p>
<h2><span class="mw-headline" id="ReadyDATAOS">ReadyDATAOS</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Nexenta_OS&amp;action=edit&amp;section=5" title="Edit section: ReadyDATAOS">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The <a class="new" href="/w/index.php?title=ReadyDATAOS&amp;action=edit&amp;redlink=1" title="ReadyDATAOS (page does not exist)">ReadyDATAOS</a> firmware is an embedded <a href="/wiki/Network-attached_storage" title="Network-attached storage">network-attached storage</a> (NAS) distribution based on Nexenta OS and has the same ZFS support as Nexenta OS. It runs on NetGear ReadyDATA hardware.</p>
