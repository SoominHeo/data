<p><b>7z</b> is a compressed <a class="mw-redirect" href="/wiki/Archive_file_format" title="Archive file format">archive file format</a> that supports several different <a href="/wiki/Data_compression" title="Data compression">data compression</a>, <a href="/wiki/Encryption" title="Encryption">encryption</a> and pre-processing algorithms. The 7z format initially appeared as implemented by the <a href="/wiki/7-Zip" title="7-Zip">7-Zip</a> archiver. The 7-Zip program is publicly available under the terms of the <a href="/wiki/GNU_Lesser_General_Public_License" title="GNU Lesser General Public License">GNU Lesser General Public License</a>. The LZMA SDK 4.62 was placed in the <a href="/wiki/Public_domain" title="Public domain">public domain</a> in December 2008. The latest stable version of 7-Zip and <a class="mw-redirect" href="/wiki/Lempel-Ziv-Markov_chain_algorithm" title="Lempel-Ziv-Markov chain algorithm">LZMA</a> SDK is version 16.<sup class="reference" id="cite_ref-History_1-1"><a href="#cite_note-History-1">[1]</a></sup></p>
<p>The 7z file format specification is distributed with 7-Zip's source code. The specification can be found in plain text format in the 'doc' sub-directory of the source code distribution.</p>

<h2><span class="mw-headline" id="Features_and_enhancements">Features and enhancements</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=7z&amp;action=edit&amp;section=1" title="Edit section: Features and enhancements">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The 7z format provides the following main features:</p>
<ul>
<li><a href="/wiki/Open_format" title="Open format">Open</a>, modular architecture that allows any compression, conversion, or encryption method to be stacked.</li>
<li>High <a href="/wiki/Data_compression_ratio" title="Data compression ratio">compression ratios</a> (depending on the compression method used)</li>
<li><a href="/wiki/Advanced_Encryption_Standard" title="Advanced Encryption Standard">AES</a>-256 <a href="/wiki/Encryption" title="Encryption">encryption</a>.</li>
<li>Large file support (up to approximately 16 <a href="/wiki/Exbibyte" title="Exbibyte">exbibytes</a>, or 2<sup>64</sup> bytes).</li>
<li><a href="/wiki/Unicode" title="Unicode">Unicode</a> file names</li>
<li>Support for <a href="/wiki/Solid_compression" title="Solid compression">solid compression</a>, where multiple files of like type are compressed within a single stream, in order to exploit the combined redundancy inherent in similar files.</li>
<li>Compression and encryption of archive <a href="/wiki/Header_(computing)" title="Header (computing)">headers</a>.</li>
<li>Support for multi-part archives : e.g. xxx.7z.001, xxx.7z.002, ... (see the context menu items <i>Split File...</i> to create them and <i>Combine Files...</i> to re-assemble an archive from a set of multi-part component files)</li>
<li>Support for custom codec plugin DLLs.</li>
</ul>
<p>The format's open architecture allows additional future compression methods to be added to the standard.</p>
<h3><span class="mw-headline" id="Compression_methods">Compression methods</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=7z&amp;action=edit&amp;section=2" title="Edit section: Compression methods">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The following compression methods are currently defined:</p>
<ul>
<li><a class="mw-redirect" href="/wiki/LZMA" title="LZMA">LZMA</a> – A variation of the <a href="/wiki/LZ77_and_LZ78" title="LZ77 and LZ78">LZ77</a> algorithm, using a sliding dictionary up to 4 GB in length for duplicate string elimination. The LZ stage is followed by <a class="mw-redirect" href="/wiki/Entropy_coding" title="Entropy coding">entropy coding</a> using a <a href="/wiki/Markov_chain" title="Markov chain">Markov chain</a>-based <a href="/wiki/Range_encoding" title="Range encoding">range coder</a> and <a href="/wiki/Binary_tree" title="Binary tree">binary trees</a>.</li>
<li><a href="/wiki/Lempel%E2%80%93Ziv%E2%80%93Markov_chain_algorithm#LZMA2_format" title="Lempel–Ziv–Markov chain algorithm">LZMA2</a> – modified version of LZMA providing better multithreading support and less expansion of incompressible data.<sup class="reference" id="cite_ref-lzma2_source_code_2-0"><a href="#cite_note-lzma2_source_code-2">[2]</a></sup></li>
<li><a href="/wiki/Bzip2" title="Bzip2">Bzip2</a> – The standard <a href="/wiki/Burrows%E2%80%93Wheeler_transform" title="Burrows–Wheeler transform">Burrows–Wheeler transform</a> algorithm. Bzip2 uses two reversible transformations; BWT, then <a class="mw-redirect" href="/wiki/Move_to_front" title="Move to front">Move to front</a> with <a href="/wiki/Huffman_coding" title="Huffman coding">Huffman coding</a> for symbol reduction (the actual compression element).</li>
<li><a class="mw-redirect" href="/wiki/Prediction_by_Partial_Matching" title="Prediction by Partial Matching">PPMd</a> – Dmitry Shkarin's 2002 PPMdH (PPMII/cPPMII) with small changes: PPMII is an improved version of the 1984 <a class="mw-redirect" href="/wiki/PPM_compression_algorithm" title="PPM compression algorithm">PPM compression algorithm</a> (prediction by partial matching).</li>
<li><a href="/wiki/DEFLATE" title="DEFLATE">DEFLATE</a> – Standard algorithm based on 32 kB <a href="/wiki/LZ77_and_LZ78" title="LZ77 and LZ78">LZ77</a> (<a class="mw-redirect" href="/wiki/Lempel-Ziv-Storer-Szymanski" title="Lempel-Ziv-Storer-Szymanski">LZSS</a> actually<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (June 2014)">citation needed</span></a></i>]</sup>) and <a href="/wiki/Huffman_coding" title="Huffman coding">Huffman coding</a>. Deflate is found in several file formats including <a class="mw-redirect" href="/wiki/ZIP_(file_format)" title="ZIP (file format)">ZIP</a>, <a href="/wiki/Gzip" title="Gzip">gzip</a>, <a href="/wiki/Portable_Network_Graphics" title="Portable Network Graphics">PNG</a> and <a class="mw-redirect" href="/wiki/PDF" title="PDF">PDF</a>. 7-Zip contains a from-scratch DEFLATE encoder that frequently beats the <i>de facto</i> standard <a href="/wiki/Zlib" title="Zlib">zlib</a> version in compression size, but at the expense of CPU usage.</li>
</ul>
<p>A suite of recompression tools called <a href="/wiki/AdvanceCOMP" title="AdvanceCOMP">AdvanceCOMP</a> contains a copy of the DEFLATE encoder from the 7-Zip implementation; these utilities can often be used to further compress the size of existing <a href="/wiki/Gzip" title="Gzip">gzip</a>, <a class="mw-redirect" href="/wiki/ZIP_(file_format)" title="ZIP (file format)">ZIP</a>, <a href="/wiki/Portable_Network_Graphics" title="Portable Network Graphics">PNG</a>, or <a href="/wiki/Multiple-image_Network_Graphics" title="Multiple-image Network Graphics">MNG</a> files.</p>
<h3><span class="mw-headline" id="Pre-processing_filters">Pre-processing filters</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=7z&amp;action=edit&amp;section=3" title="Edit section: Pre-processing filters">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The LZMA SDK comes with the BCJ / BCJ2 preprocessor included, so that later stages are able to achieve greater compression: For <a href="/wiki/X86" title="X86">x86</a>, <a href="/wiki/ARM_architecture" title="ARM architecture">ARM</a>, <a href="/wiki/PowerPC" title="PowerPC">PowerPC</a> (PPC), IA-64 <a href="/wiki/Itanium" title="Itanium">Itanium</a>, and <a class="mw-redirect" href="/wiki/ARM_Thumb" title="ARM Thumb">ARM Thumb</a> processors, jump targets are normalized before compression by changing relative position into absolute values. For x86, this means that near jumps, calls and conditional jumps (but not short jumps and conditional jumps) are converted from the machine language "jump 1655 bytes backwards" style notation to normalized "jump to address 5554" style notation; all jumps to 5554, perhaps a common subroutine, are thus encoded identically, making them more compressible.</p>
<ul>
<li>BCJ – Converter for 32-bit x86 executables. Normalise target addresses of near jumps and calls from relative distances to absolute destinations.</li>
<li><a class="new" href="/w/index.php?title=BCJ2&amp;action=edit&amp;redlink=1" title="BCJ2 (page does not exist)">BCJ2</a> – Pre-processor for 32-bit x86 executables. BCJ2 is an improvement on BCJ, adding additional x86 jump/call instruction processing. Near jump, near call, conditional near jump targets are split out and compressed separately in another stream.</li>
<li><a href="/wiki/Delta_encoding" title="Delta encoding">Delta encoding</a> – delta filter, basic preprocessor for multimedia data.</li>
</ul>
<p>Similar executable pre-processing technology is included in other software; the <a href="/wiki/RAR_(file_format)" title="RAR (file format)">RAR</a> compressor features displacement compression for 32-bit x86 executables and IA-64 executables, and the <a href="/wiki/UPX" title="UPX">UPX</a> runtime executable file compressor includes support for working with 16-bit values within <a href="/wiki/DOS" title="DOS">DOS</a> binary files.</p>
<h3><span class="mw-headline" id="Encryption">Encryption</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=7z&amp;action=edit&amp;section=4" title="Edit section: Encryption">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The 7z format supports <a href="/wiki/Encryption" title="Encryption">encryption</a> with the <a href="/wiki/Advanced_Encryption_Standard" title="Advanced Encryption Standard">AES</a> algorithm with a 256-bit key. The key is generated from a user-supplied <a href="/wiki/Passphrase" title="Passphrase">passphrase</a> using an algorithm based on the <a class="mw-redirect" href="/wiki/SHA-256" title="SHA-256">SHA-256</a> hash function. The SHA-256 is executed 2<sup>18</sup> (262144) times,<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> which causes a significant delay on slow PCs before compression or extraction starts. This technique is called <a href="/wiki/Key_stretching" title="Key stretching">key stretching</a> and is used to make a <a href="/wiki/Brute-force_search" title="Brute-force search">brute-force search</a> for the passphrase more difficult. Current GPU-based, and custom hardware attacks limit the effectiveness of this particular method of key stretching,<sup class="reference" id="cite_ref-percival2009_4-0"><a href="#cite_note-percival2009-4">[4]</a></sup> so it is still important to choose a strong password. The 7z format provides the option to encrypt the filenames of a 7z archive.</p>
<h3><span class="mw-headline" id="Limitations">Limitations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=7z&amp;action=edit&amp;section=5" title="Edit section: Limitations">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The 7z format does not store <a class="mw-redirect" href="/wiki/Filesystem_permissions" title="Filesystem permissions">filesystem permissions</a> (such as <a class="mw-redirect" href="/wiki/UNIX" title="UNIX">UNIX</a> owner/group permissions or <a href="/wiki/NTFS" title="NTFS">NTFS</a> <a href="/wiki/Access_control_list" title="Access control list">ACLs</a>), and hence can be inappropriate for backup/archival purposes. A workaround on UNIX-like systems for this is to convert data to a <a class="mw-redirect" href="/wiki/Tar_(file_format)" title="Tar (file format)">tar bitstream</a> before compressing with 7z. But it is worth noting that GNU tar (common in many UNIX environments) can also compress with the LZMA algorithm natively, without the use of 7z, and that in this case the suggested<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> file extension for the archive is ".tar.lzma" (or just ".tlz"), and not ".tar.7z". On the other hand, it is important to note, that tar does not save the filesystem encoding, which means that tar compressed filenames can become unreadable if decompressed on a different computer. It is also possible to use LZMA2 by running it through the <a href="/wiki/Xz" title="Xz">xz</a> tool. Recent versions of GNU tar support the <code>-J</code> switch, which runs TAR through XZ. The file extension is ".tar.xz" or ".txz". This method of compression has been adopted with many distributions for packaging, such as Arch, Debian (deb), Fedora (rpm) and Slackware.</p>
<p>The 7z format does not allow extraction of some "broken files"—that is (for example) if one has the first segment of a series of 7z files, 7z cannot give the start of the files within the archive—it must wait until all segments are downloaded. The 7z format also lacks recovery records, which might be a problem when limited file corruption has occurred. By way of comparison, <a href="/wiki/Zip_(file_format)" title="Zip (file format)">zip</a> files also lack a recovery feature. In contrast the proprietary <a href="/wiki/RAR_(file_format)" title="RAR (file format)">rar</a> format permits recoveries as well as the extraction of broken files and <a href="/wiki/File_spanning" title="File spanning">file spanning</a>.</p>
