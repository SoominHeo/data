<p><b>MXML</b> is an <a href="/wiki/XML" title="XML">XML</a>-based <a href="/wiki/User_interface_markup_language" title="User interface markup language">user interface markup language</a> first introduced by <a href="/wiki/Macromedia" title="Macromedia">Macromedia</a> in March 2004. Application developers use MXML in combination with <a href="/wiki/ActionScript" title="ActionScript">ActionScript</a> to develop <a href="/wiki/Rich_Internet_application" title="Rich Internet application">rich Internet applications</a>, with products such as <a href="/wiki/Apache_Flex" title="Apache Flex">Apache Flex</a>.</p>
<p><a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe Systems</a>, which acquired Macromedia in December 2005, gives no official meaning for the acronym MXML. Some developers suggest it should stand for "Magic eXtensible Markup Language" (which is a <a href="/wiki/Backronym" title="Backronym">backronym</a>). It is likely that the name comes from the MX suffix given to Macromedia Studio products released in 2002 and 2004, or simply "Macromedia eXtensible Markup Language".</p>
<p>MXML is used mainly to declaratively lay out the interface of applications and can also be used to implement business logic and internet application behaviors. It can contain chunks of ActionScript code, either when creating the body of an event handler function, or with data binding where the curly braces (<b>{</b>) syntax is used.</p>
<p>MXML is often used with Flex Server, which dynamically compiles it into standard binary <a href="/wiki/SWF" title="SWF">SWF</a> files. However, the Adobe <a class="mw-redirect" href="/wiki/Flash_Builder" title="Flash Builder">Flash Builder</a> IDE (formerly Adobe Flex Builder) and free Flex SDK can also compile MXML into SWF files without the use of a Flex Server.</p>
<p>There is also a <a href="/wiki/PHP" title="PHP">PHP</a> <a class="mw-redirect" href="/wiki/PHP_Extension_and_Application_Repository" title="PHP Extension and Application Repository">PEAR</a> package called <a class="external text" href="http://pear.php.net/package/XML_MXML" rel="nofollow">XML_MXML</a>, which is a framework to build Adobe Flex applications.</p>
<p>MXML is considered a proprietary standard due to its tight integration with Adobe technologies. It is like <a href="/wiki/Extensible_Application_Markup_Language" title="Extensible Application Markup Language">XAML</a> in this respect. No published translators exist for converting an MXML document to another user interface language such as <a href="/wiki/UIML" title="UIML">UIML</a>, <a href="/wiki/XUL" title="XUL">XUL</a>, <a href="/wiki/XForms" title="XForms">XForms</a>, <a href="/wiki/Extensible_Application_Markup_Language" title="Extensible Application Markup Language">XAML</a>, or <a href="/wiki/Scalable_Vector_Graphics" title="Scalable Vector Graphics">SVG</a>. However, there do exist third party vendor plugins for Flex Builder that are capable of generating a result other than a SWF file from Flex applications, for instance native mobile applications.<sup class="reference" id="cite_ref-elipsstudio_1-0"><a href="#cite_note-elipsstudio-1">[1]</a></sup></p>
