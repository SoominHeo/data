<h1 class="firstHeading" id="firstHeading" lang="en">Multi-monitor</h1>

<p><b>Multi-monitor</b>, also called <b>multi-display</b> and <b>multi-head</b>, is the use of multiple physical <a href="/wiki/Display_device" title="Display device">display devices</a>, such as <a href="/wiki/Computer_monitor" title="Computer monitor">monitors</a>, <a href="/wiki/Television" title="Television">televisions</a>, and <a href="/wiki/Video_projector" title="Video projector">projectors</a>, in order to increase the area available for <a href="/wiki/Computer_program" title="Computer program">computer programs</a> running on a single <a href="/wiki/Computer" title="Computer">computer</a> system. Research studies<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> show that, depending on the type of work, multi-head may increase the productivity by up to 50-70%.</p>
<p></p>

<h2><span class="mw-headline" id="Implementation">Implementation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=1" title="Edit section: Implementation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Analog_Monitor_Splitter_(unbranded)-2312.jpg"><img alt="" class="thumbimage" data-file-height="2609" data-file-width="3966" height="145" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/e2/Analog_Monitor_Splitter_%28unbranded%29-2312.jpg/220px-Analog_Monitor_Splitter_%28unbranded%29-2312.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/e2/Analog_Monitor_Splitter_%28unbranded%29-2312.jpg/330px-Analog_Monitor_Splitter_%28unbranded%29-2312.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/e2/Analog_Monitor_Splitter_%28unbranded%29-2312.jpg/440px-Analog_Monitor_Splitter_%28unbranded%29-2312.jpg 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Analog_Monitor_Splitter_(unbranded)-2312.jpg" title="Enlarge"></a></div>
Analog Monitor Splitter for VGA cable</div>

</div>
<p>Support for a multi-monitor setup is either achieved by installing multiple <a class="mw-redirect" href="/wiki/Graphics_card" title="Graphics card">graphics cards</a> into one computer or by special <a class="mw-redirect" href="/wiki/Display_controller" title="Display controller">display controllers</a>, that have the ability to feed multiple monitors independently with a signal. Monitors supporting <a href="/wiki/DisplayPort" title="DisplayPort">DisplayPort</a> allow to drive multiple monitors from only one external clock. DisplayPort version 1.2 supports <a class="mw-redirect" href="/wiki/Multi-Stream_Transport" title="Multi-Stream Transport">Multi-Stream Transport</a>; this makes it possible to drive multiple displays on one single DisplayPort connector using a multi-head cable or loop through.</p>
<p>Furthermore, multiple computers can be connected to provide a single display, e.g. over <a href="/wiki/Gigabit_Ethernet" title="Gigabit Ethernet">Gigabit Ethernet</a>/<a href="/wiki/Ethernet" title="Ethernet">Ethernet</a> to drive a large <a href="/wiki/Video_wall" title="Video wall">video wall</a>.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<h2><span class="mw-headline" id="Display_modes">Display modes</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=2" title="Edit section: Display modes">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Clone_mode">Clone mode</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=3" title="Edit section: Clone mode">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Since <a href="/wiki/History_of_personal_computers" title="History of personal computers">before personal computers existed</a>, video signals have been split with simple <a href="/wiki/Y-cable" title="Y-cable">Y-adapters</a> to provide duplicate signals to multiple monitors for various reasons. When personal computers came to have video outputs, this naturally carried over—sometimes for <a href="/wiki/Presentation" title="Presentation">presentations</a>, and sometimes to provide a different representation of the same output (for example color alongside the higher resolution monochrome interpretation of the output of an Apple II). Later systems – particularly portable machines with built-in displays — provided built-in redundant outputs for this. Even later systems, in addition to being capable of the discrete modes described below, are able to mimic this "<b>cloning</b>" or "<b>mirroring</b>" behavior (typically defaulting to it upon power-up/reset).</p>
<h3><span class="mw-headline" id="Extended_mode">Extended mode</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=4" title="Edit section: Extended mode">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Extended mode creates one virtual display with the resolution of all participating monitors. Depending on the employed hardware and software, the monitors must be driven all with the same resolution or each monitor can participate with a different resolution.</p>
<p>Both of these modes present the display space to the user as a contiguous area, allowing objects to be moved between, or even straddled across displays as if they are one.</p>
<h2><span class="mw-headline" id="Graphics_devices">Graphics devices</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=5" title="Edit section: Graphics devices">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The biggest manufacturers of graphics card or on-board graphics products support multiple monitor configurations. This includes Nvidia, AMD, Matrox and Intel. Limiting factors are now the number of physical ports than can be supplied rather than the abilities of the devices themselves.</p>
<h3><span class="mw-headline" id="USB">USB</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=6" title="Edit section: USB">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Another way to extend the number of displays on one computer is to add displays via USB. Starting in 2006, <a href="/wiki/DisplayLink" title="DisplayLink">DisplayLink</a> released several chips for USB support on VGA/DVI/LVDS and other interfaces.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>

<h3><span class="mw-headline" id="In_the_office">In the office</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=8" title="Edit section: In the office">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

<p>Specialized application environments such as <a href="/wiki/Computer-aided_design" title="Computer-aided design">CAD</a>, day trading of corporate stocks, and <a href="/wiki/Software_development" title="Software development">software development</a> are increasingly using six or more monitors on one production system.</p>
<p>In many professions, including graphic design, architecture, communications, accounting, engineering and video editing, the idea of two or more monitors being driven from one machine is not a new one. While in the past, it has meant multiple graphics adapters and specialized software, it was common for engineers to have at least two, if not more, displays to enhance productivity.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>Multi-display setups are also very common in <a href="/wiki/Investment_banking" title="Investment banking">investment banks</a>, particularly in <a href="/wiki/Market_maker" title="Market maker">market making</a>, where they allow the simultaneous display of several screens of prices as reference data, allowing the trader to keep an eye on the market. Setups of 6 displays (2×3: 2 rows of 3) are common on <a href="/wiki/Interest_rate" title="Interest rate">interest rate</a> trading desks,<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (February 2011)">citation needed</span></a></i>]</sup> which involve many numbers, while 8 displays (2×4: 2 rows of 4) are not uncommon. Financial multi-display setups may also incorporate <a href="/wiki/Bloomberg_Terminal" title="Bloomberg Terminal">Bloomberg Terminals</a>, or these may be adjunct to the main display.</p>
<p>Now that multi-monitor setups are more budget-friendly,<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (February 2011)">citation needed</span></a></i>]</sup> it is not uncommon to see a wide range of business professionals using multiple monitors to increase visual area.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (February 2011)">citation needed</span></a></i>]</sup> This advantage helps promote the concept of a <a href="/wiki/Paperless_office" title="Paperless office">paperless office</a> by increasing the quantity of simultaneous media that can be viewed digitally, although the advantage of viewing two documents simultaneously is also feasible on many larger <a class="new" href="/w/index.php?title=Widescreen_monitor&amp;action=edit&amp;redlink=1" title="Widescreen monitor (page does not exist)">widescreen monitors</a>.</p>
<h3><span class="mw-headline" id="Professional_audio_production">Professional audio production</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=9" title="Edit section: Professional audio production">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Digital_audio_workstation" title="Digital audio workstation">Digital audio workstation</a> are known to rely on multi-monitor setups to increase the available area for the user interface.</p>

<p>Multi-monitor gaming/simulation is also becoming more common; however, the hardware expense can be a limiting factor.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></p>
<h4><span class="mw-headline" id="Video_games_that_support_multi-monitor_set-ups">Video games that support multi-monitor set-ups</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=11" title="Edit section: Video games that support multi-monitor set-ups">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Early versions of <a href="/wiki/Doom_(series)" title="Doom (series)">DOOM</a> permitted a three-monitor display mode, using three networked machines to show left, right, and center views.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<p>More recently, games have used multiple monitor to show a more absorbing interface to the player or to display game information . Various flight simulators can use these monitor setups to create an artificial cockpit with more realistic interfaces.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> Others such as <a href="/wiki/Supreme_Commander_(video_game)" title="Supreme Commander (video game)">Supreme Commander</a> and <a href="/wiki/World_in_Conflict" title="World in Conflict">World in Conflict</a> can use an additional monitor for a large scale map of the battlefield.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></p>
<p>A large number of older games support multi-monitor set-ups by treating the total screen space as effectively a single monitor to the game, a technique known as spanning. Many games without inherent multi-monitor support such as <a href="/wiki/Guild_Wars" title="Guild Wars">Guild Wars</a> and <a href="/wiki/World_of_Warcraft" title="World of Warcraft">World of Warcraft</a> can also be made to run in multi-monitor set-ups, with this technique or in conjunction with additional of third party software<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> A larger list of games that support dual/multi screen modes is available at WSGF.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<h3><span class="mw-headline" id="Other_uses">Other uses</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=12" title="Edit section: Other uses">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<h4><span class="mw-headline" id="Older_uses">Older uses</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=13" title="Edit section: Older uses">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>From the mid-1980s through 1990s, a popular configuration for software developers was to employ a general-purpose <a href="/wiki/Video_Graphics_Array" title="Video Graphics Array">VGA</a>, <a href="/wiki/Enhanced_Graphics_Adapter" title="Enhanced Graphics Adapter">EGA</a>, or <a href="/wiki/Color_Graphics_Adapter" title="Color Graphics Adapter">CGA</a> display for managing the program under development and an independent monochrome <a href="/wiki/Hercules_Graphics_Card" title="Hercules Graphics Card">Hercules</a> or <a href="/wiki/IBM_Monochrome_Display_Adapter" title="IBM Monochrome Display Adapter">MDA</a> card driving a second monitor for displaying debugging information. Many DOS debugging applications supported this configuation. It was possible to operate two display cards in this fashion, even with operating systems such as <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a> which did not support multi-monitor natively, because the Hercules and MDA cards used a different hardware memory address than conventional graphics cards and could operate concurrently without creating hardware conflicts. Modern hardware is not affected by the limitations of earlier systems like these when running modern operating systems, because the hardware and software are both designed such that the operating system can abstract the various hardware devices from each other and then manage them appropriately. The first Macintosh computer to support multiple monitors was the <a href="/wiki/Macintosh_II" title="Macintosh II">Macintosh II</a>. The <a href="/wiki/Macintosh_SE/30" title="Macintosh SE/30">Macintosh SE/30</a>, which had one slot in it, also supported a second monitor which could be color even though the main monitor was monochrome.</p>
<p><a href="/wiki/Interactive_television#Interactivity_with_TV_program_content" title="Interactive television">Interactive television</a> sometimes coordinates the use of a television screen and a computer display.</p>
<h4><span class="mw-headline" id="Arcade_machines">Arcade machines</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=14" title="Edit section: Arcade machines">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Arcade games were released in the 1980s and 1990s which used a multi-monitor configuration. The earliest of these is the game <a href="/wiki/TX-1" title="TX-1">TX-1</a>, a 1983 driving game developed by Tatsumi and published by <a href="/wiki/Namco" title="Namco">Namco</a>/<a href="/wiki/Atari" title="Atari">Atari</a>, which used a cockpit cabinet with 3 19" CRT monitors side-by-side to give a wide viewing area. Its successor, <a class="mw-redirect" href="/wiki/TX-1_V8" title="TX-1 V8">TX-1 V8</a>, released in 1984, also used the same 3 monitor configuration. Tatsumi released <a href="/wiki/Buggy_Boy" title="Buggy Boy">Buggy Boy</a> in 1985, in both cockpit 3 monitor and upright single monitor cabinets. <a href="/wiki/Darius_(video_game)" title="Darius (video game)">Darius</a> from 1986 used an upright cabinet which was around half as wide again than a standard arcade cabinet of the time. It used 3 14" FST (flatter squarer tube) monitors but in this case, 2 monitors were mounted on the left and right with the screens pointing upwards and one was mounted in a box at the centre rear of the cabinet, and using a two way mirror a seamless wide image was obtained. A similar effect was seen in <a href="/wiki/Darius_II_(video_game)" title="Darius II (video game)">Darius II</a>, also known as Sagaia (which was also released in twin larger monitor format) and <a class="mw-redirect" href="/wiki/Ninja_Warriors" title="Ninja Warriors">The Ninja Warriors</a>. The original 1987 arcade version of <a href="/wiki/Tecmo_Bowl" title="Tecmo Bowl">Tecmo Bowl</a> also used a dual-monitor setup. In 1990, <i><a href="/wiki/Galaxian_3" title="Galaxian 3">Galaxian 3</a></i> used up to 16 projected screens.</p>
<p>As arcade technology entered the 1990s, larger cabinets were being built which in turn also housed larger monitors such as the 3 28" screen version of <a href="/wiki/Namco" title="Namco">Namco</a>'s <a href="/wiki/Ridge_Racer" title="Ridge Racer">Ridge Racer</a> from 1993. Although large screen technology such as CRT rear projection was beginning to be used more often, multi-monitor games were still occasionally released, such as <a href="/wiki/Sega" title="Sega">Sega</a>'s <a href="/wiki/F355_Challenge" title="F355 Challenge">F355 Challenge</a> from 1999 which again used 3 28" monitors for the sit-down cockpit version. The most recent use of a multi-monitor setup in arcades occurred with Taito's Dariusburst: Another Chronicle game, released in Japan in December 2010<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> and worldwide the following year.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> It uses 2 32" LCD screens and an angled mirror to create a seamless widescreen effect.</p>
<h2><span class="mw-headline" id="Developing_software_for_multiple_monitor_workstations">Developing software for multiple monitor workstations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Multi-monitor&amp;action=edit&amp;section=15" title="Edit section: Developing software for multiple monitor workstations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Ordinary software does not need special support for multiple screens even if it uses graphic accelerator. At the usual application level multihead is presented just as a single larger monitor spanning over all screens. However some special approaches may increase the multihead performance.</p>
<p>With multiple monitors present, each screen will have its own graphics buffer. One possible scenario for programming is to present to OpenGL or DirectX a continuous, virtual frame buffer in which the OS or graphics driver writes out to each individual buffer. With some graphics cards, its possible to enable a mode called "horizontal span" which accomplishes this. The OpenGL/DirectX programmer then renders to a very large frame buffer for output. In practice, and with recent cards, this mode is being phased out because it does not make very good use of GPU parallelism, and does not support arbitrary arrangements of monitors (they must all be horizontal). A more recent technique uses the wglShareLists feature of OpenGL to share data across multiple GPUs, and then render to each individual monitor's frame buffer.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup></p>
