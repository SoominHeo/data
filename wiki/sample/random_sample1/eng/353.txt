<h1 class="firstHeading" id="firstHeading" lang="en">Non-breaking space</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p>In <a class="mw-redirect" href="/wiki/Word_processing" title="Word processing">word processing</a> and <a class="mw-redirect" href="/wiki/Digital_typesetting" title="Digital typesetting">digital typesetting</a>, a <b>non-breaking space</b> (" ") (also called <b>no-break space</b>, <b>non-breakable space</b> (<b>NBSP</b>), <b>hard space</b>, or <b>fixed space</b>) is a <a class="mw-redirect" href="/wiki/Space_character" title="Space character">space character</a> that prevents an <a href="/wiki/Line_wrap_and_word_wrap" title="Line wrap and word wrap">automatic line break</a> at its position. In some formats, including <a href="/wiki/HTML" title="HTML">HTML</a>, it also prevents consecutive <a class="mw-redirect" href="/wiki/Whitespace_(computer_science)" title="Whitespace (computer science)">whitespace characters</a> from collapsing into a single space.</p>
<p>In HTML, the common non-breaking space, which is the same width as the ordinary space character, is encoded as <code>&amp;nbsp;</code> or <code>&amp;#160;</code>. In <a href="/wiki/Unicode" title="Unicode">Unicode</a>, it is encoded as <code>U+00A0</code>.</p>
<p>Non-breaking space characters <a href="#Width_variation">with other widths</a> also exist.</p>

<h2><span class="mw-headline" id="Uses_and_variations">Uses and variations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Non-breaking_space&amp;action=edit&amp;section=1" title="Edit section: Uses and variations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Despite having layout and uses similar to those of <a class="mw-redirect" href="/wiki/Whitespace_(computer_science)" title="Whitespace (computer science)">whitespace</a>, it differs in contextual behavior.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<h3><span class="mw-headline" id="Non-breaking_behavior">Non-breaking behavior</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Non-breaking_space&amp;action=edit&amp;section=2" title="Edit section: Non-breaking behavior">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Text-processing software typically assumes that an automatic line break may be inserted anywhere a space character occurs; a non-breaking space prevents this from happening (provided the software recognizes the character). For example, if the text "100 km" will not quite fit at the end of a line, the software may insert a line break between "100" and "km". An editor who finds this behaviour undesirable may choose to use a non-breaking space between "100" and "km". This guarantees that the text "100 km" will not be broken: if it does not fit at the end of a line, it is moved in its entirety to the next line.</p>
<h3><span class="mw-headline" id="Non-collapsing_behavior">Non-collapsing behavior</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Non-breaking_space&amp;action=edit&amp;section=3" title="Edit section: Non-collapsing behavior">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A second common application of non-breaking spaces is in <a href="/wiki/Plain_text" title="Plain text">plain text</a> file formats such as <a class="mw-redirect" href="/wiki/SGML" title="SGML">SGML</a>, <a href="/wiki/HTML" title="HTML">HTML</a>, <a href="/wiki/TeX" title="TeX">TeX</a> and <a href="/wiki/LaTeX" title="LaTeX">LaTeX</a>, whose rendering engines are programmed to treat sequences of <a class="mw-redirect" href="/wiki/Whitespace_(computer_science)" title="Whitespace (computer science)">whitespace characters</a> (space, newline, tab, <a class="mw-redirect" href="/wiki/Form_feed" title="Form feed">form feed</a>, etc.) as if they were a single character (but this behavior can be overridden). Such "collapsing" of whitespace allows the author to neatly arrange the source text using line breaks, indentation and other forms of spacing without affecting the final typeset result.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p>In contrast, non-breaking spaces are not merged with neighboring whitespace characters when displayed, and can therefore be used by an author to simply insert additional visible space in the resulting output without using spans styled with peculiar values of the <a class="mw-redirect" href="/wiki/CSS" title="CSS">CSS</a> “white-space” property. Conversely, indiscriminate use (see the recommended use in <a href="/wiki/Style_guide" title="Style guide">style guides</a>), in addition to a normal space, gives extraneous space in the output.</p>
<h3><span class="mw-headline" id="Width_variation">Width variation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Non-breaking_space&amp;action=edit&amp;section=4" title="Edit section: Width variation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Other non-breaking variants, <a href="/wiki/Space_(punctuation)#Spaces_in_Unicode" title="Space (punctuation)">defined in Unicode</a>:</p>
<ul>
<li><span class="nowrap"><span style="font-family: monospace, monospace;">U+202F</span></span> <span class="Unicode" style="background:lightblue"> </span> <span class="smallcaps" style="font-variant: small-caps; font-size: smaller;">NARROW NO-BREAK SPACE</span> (HTML <code>&amp;#8239;</code> <b>·</b> <code>NNBSP</code>). It was introduced in Unicode 3.0 for Mongolian,<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> to separate a suffix<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> from the word stem without indicating a word boundary. It is also required for big <a href="/wiki/Punctuation" title="Punctuation">punctuation</a> in <a href="/wiki/French_language" title="French language">French</a>, sometimes inaccurately referred to as “double punctuation” (before " ; ? ! » › " and after " « ‹ "; today often also before " : ") and <a href="/wiki/Russian_language" title="Russian language">Russian</a> (before " — "), and in <a href="/wiki/German_language" title="German language">German</a> between multi-part abbreviations (z. B., d. h., v. l. n. r.).<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> When used with Mongolian, its width is usually one third of the normal space; in other contexts, its width is about 70% of the normal space but may resemble that of the <a href="/wiki/Thin_space" title="Thin space">thin space</a> (U+2009), at least with some fonts.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></li>
<li><span class="nowrap"><span style="font-family: monospace, monospace;">U+2007</span></span> <span class="Unicode" style="background:lightblue"> </span> <a href="/wiki/Figure_space" title="Figure space"><span class="smallcaps" style="font-variant: small-caps; font-size: smaller;">FIGURE SPACE</span></a> (HTML <code>&amp;#8199;</code>). Produces a space equal to the figure (0–9) characters.</li>
<li><span class="nowrap"><span style="font-family: monospace, monospace;">U+2060</span></span> <a class="mw-redirect" href="/wiki/Word-joiner" title="Word-joiner"><span class="smallcaps" style="font-variant: small-caps; font-size: smaller;">WORD-JOINER</span></a> (HTML <code>&amp;#8288;</code> <b>·</b> <code>WJ</code>): encoded in Unicode since version 3.2. The word-joiner does not produce any space, and prohibits a line break at its position.</li>
</ul>

<p>Unicode defines several other non-break space characters. See <a href="#Width_variation">#Width variation</a>. Encoding remarks:</p>
<ul>
<li>Word joiner, encoded in Unicode 3.2 and above as U+2060, and in HTML as <code class="mw-highlight" dir="ltr" id="" style="">&amp;#x2060;</code> or <code class="mw-highlight" dir="ltr" id="" style="">&amp;#8288;</code>.</li>
<li>The <a class="mw-redirect" href="/wiki/Byte_Order_Mark" title="Byte Order Mark">Byte Order Mark</a>, U+FEFF, officially named "Zero Width No-Break Space", was also used with the same meaning as the word joiner, but in current documents this use is deprecated. See also <a class="mw-redirect" href="/wiki/Zero-width_non-breaking_space" title="Zero-width non-breaking space">Zero-width non-breaking space</a>.</li>
</ul>
<h2><span class="mw-headline" id="Keyboard_entry_methods">Keyboard entry methods</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Non-breaking_space&amp;action=edit&amp;section=6" title="Edit section: Keyboard entry methods">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>It is rare for national or international standards on <a href="/wiki/Keyboard_layout" title="Keyboard layout">keyboard layouts</a> to define an input method for the non-breaking space. An exception is the Finnish multilingual keyboard, accepted as the national standard SFS 5966 in 2008. According to the SFS setting, the non-breaking space can be entered with the key combination <a class="mw-redirect" href="/wiki/AltGr" title="AltGr">AltGr</a> + <a href="/wiki/Space_bar" title="Space bar">Space</a>.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>

<p>Apart from this, applications and environments often have methods of entering unicode entities directly via their code point, e.g. via the <a class="mw-redirect" href="/wiki/Alt_Numpad" title="Alt Numpad">Alt Numpad</a> input method. (Non-breaking space has codepoint <tt>255</tt> decimal (<tt>FF</tt> hex) in <a class="mw-redirect" href="/wiki/Codepage_437" title="Codepage 437">codepage 437</a> and <a class="mw-redirect" href="/wiki/Codepage_850" title="Codepage 850">codepage 850</a>, and codepoint <tt>160</tt> decimal (<tt>A0</tt> hex) in <a class="mw-redirect" href="/wiki/Codepage_1252" title="Codepage 1252">codepage 1252</a>.)</p>
