<h1 class="firstHeading" id="firstHeading" lang="en">Injective module</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p>In <a href="/wiki/Mathematics" title="Mathematics">mathematics</a>, especially in the area of <a href="/wiki/Abstract_algebra" title="Abstract algebra">abstract algebra</a> known as <a class="mw-redirect" href="/wiki/Module_theory" title="Module theory">module theory</a>, an <b>injective module</b> is a <a href="/wiki/Module_(mathematics)" title="Module (mathematics)">module</a> <i>Q</i> that shares certain desirable properties with the <b>Z</b>-module <b>Q</b> of all <a href="/wiki/Rational_number" title="Rational number">rational numbers</a>. Specifically, if <i>Q</i> is a <a class="mw-redirect" href="/wiki/Submodule" title="Submodule">submodule</a> of some other module, then it is already a <a class="mw-redirect" href="/wiki/Direct_summand" title="Direct summand">direct summand</a> of that module; also, given a submodule of a module <i>Y</i>, then any <a href="/wiki/Module_homomorphism" title="Module homomorphism">module homomorphism</a> from this submodule to <i>Q</i> can be extended to a homomorphism from all of <i>Y</i> to <i>Q</i>. This concept is dual to that of <a href="/wiki/Projective_module" title="Projective module">projective modules</a>. Injective modules were introduced in (<a href="#CITEREFBaer1940">Baer 1940</a>) and are discussed in some detail in the textbook (<a href="#CITEREFLam1999">Lam 1999</a>, §3).</p>
<p>Injective modules have been heavily studied, and a variety of additional notions are defined in terms of them: <a href="/wiki/Injective_cogenerator" title="Injective cogenerator">Injective cogenerators</a> are injective modules that faithfully represent the entire category of modules. Injective resolutions measure how far from injective a module is in terms of the <a href="/wiki/Injective_module#Injective_resolutions" title="Injective module">injective dimension</a> and represent modules in the <a href="/wiki/Derived_category" title="Derived category">derived category</a>. <a href="/wiki/Injective_hull" title="Injective hull">Injective hulls</a> are maximal <a href="/wiki/Essential_extension" title="Essential extension">essential extensions</a>, and turn out to be minimal injective extensions. Over a <a href="/wiki/Noetherian_ring" title="Noetherian ring">Noetherian ring</a>, every injective module is uniquely a direct sum of <a href="/wiki/Indecomposable_module" title="Indecomposable module">indecomposable</a> modules, and their structure is well understood. An injective module over one ring, may not be injective over another, but there are well-understood methods of changing rings which handle special cases. Rings which are themselves injective modules have a number of interesting properties and include rings such as <a href="/wiki/Group_ring" title="Group ring">group rings</a> of <a href="/wiki/Finite_group" title="Finite group">finite groups</a> over <a href="/wiki/Field_(mathematics)" title="Field (mathematics)">fields</a>. Injective modules include <a href="/wiki/Divisible_group" title="Divisible group">divisible groups</a> and are generalized by the notion of <a href="/wiki/Injective_object" title="Injective object">injective objects</a> in <a href="/wiki/Category_theory" title="Category theory">category theory</a>.</p>

<h2><span class="mw-headline" id="Definition">Definition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=1" title="Edit section: Definition">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A left module <i>Q</i> over the <a href="/wiki/Ring_(mathematics)" title="Ring (mathematics)">ring</a> <i>R</i> is injective if it satisfies one (and therefore all) of the following equivalent conditions:</p>
<ul>
<li>If <i>Q</i> is a submodule of some other left <i>R</i>-module <i>M</i>, then there exists another submodule <i>K</i> of <i>M</i> such that <i>M</i> is the <a href="/wiki/Direct_sum_of_modules" title="Direct sum of modules">internal direct sum</a> of <i>Q</i> and <i>K</i>, i.e. <i>Q</i> + <i>K</i> = <i>M</i> and <i>Q</i> ∩ <i>K</i> = {0}.</li>
<li>Any <a class="mw-redirect" href="/wiki/Short_exact_sequence" title="Short exact sequence">short exact sequence</a> 0 →<i>Q</i> → <i>M</i> → <i>K</i> → 0 of left <i>R</i>-modules <a class="mw-redirect" href="/wiki/Split_exact_sequence" title="Split exact sequence">splits</a>.</li>
<li>If <i>X</i> and <i>Y</i> are left <i>R</i>-modules and <i>f</i> : <i>X</i> → <i>Y</i> is an <a class="mw-redirect" href="/wiki/Injective" title="Injective">injective</a> module homomorphism and <i>g</i> : <i>X</i> → <i>Q</i> is an arbitrary module homomorphism, then there exists a module homomorphism <i>h</i> : <i>Y</i> → <i>Q</i> such that <i>hf</i> = <i>g</i>, i.e. such that the following diagram <a href="/wiki/Commutative_diagram" title="Commutative diagram">commutes</a>:</li>
</ul>
<dl>
<dd>
<dl>
<dd><a class="image" href="/wiki/File:Injective_module.png" title="commutative diagram defining injective module Q"><img alt="commutative diagram defining injective module Q" data-file-height="115" data-file-width="151" height="115" src="//upload.wikimedia.org/wikipedia/commons/4/47/Injective_module.png" width="151"/></a></dd>
</dl>
</dd>
</dl>
<ul>
<li>The <a class="mw-redirect" href="/wiki/Contravariant_functor" title="Contravariant functor">contravariant functor</a> Hom(-,<i>Q</i>) from the <a href="/wiki/Category_theory" title="Category theory">category</a> of left <i>R</i>-modules to the category of <a href="/wiki/Abelian_group" title="Abelian group">abelian groups</a> is <a href="/wiki/Exact_functor" title="Exact functor">exact</a>.</li>
</ul>
<p>Injective right <i>R</i>-modules are defined in complete analogy.</p>
<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=2" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="First_examples">First examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=3" title="Edit section: First examples">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Trivially, the zero module {0} is injective.</p>
<p>Given a <a href="/wiki/Field_(mathematics)" title="Field (mathematics)">field</a> <i>k</i>, every <i>k</i>-<a href="/wiki/Vector_space" title="Vector space">vector space</a> <i>Q</i> is an injective <i>k</i>-module. Reason: if <i>Q</i> is a subspace of <i>V</i>, we can find a <a class="mw-redirect" href="/wiki/Basis_of_a_vector_space" title="Basis of a vector space">basis</a> of <i>Q</i> and extend it to a basis of <i>V</i>. The new extending basis vectors <a href="/wiki/Linear_span" title="Linear span">span</a> a subspace <i>K</i> of <i>V</i> and <i>V</i> is the internal direct sum of <i>Q</i> and <i>K</i>. Note that the direct complement <i>K</i> of <i>Q</i> is not uniquely determined by <i>Q</i>, and likewise the extending map <i>h</i> in the above definition is typically not unique.</p>
<p>The rationals <b>Q</b> (with addition) form an injective abelian group (i.e. an injective <b>Z</b>-module). The <a class="mw-redirect" href="/wiki/Factor_group" title="Factor group">factor group</a> <b>Q</b>/<b>Z</b> and the <a href="/wiki/Circle_group" title="Circle group">circle group</a> are also injective <b>Z</b>-modules. The factor group <b>Z</b>/<i>n</i><b>Z</b> for <i>n</i> &gt; 1 is injective as a <b>Z</b>/<i>n</i><b>Z</b>-module, but <i>not</i> injective as an abelian group.</p>
<h3><span class="mw-headline" id="Commutative_examples">Commutative examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=4" title="Edit section: Commutative examples">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>More generally, for any <a href="/wiki/Integral_domain" title="Integral domain">integral domain</a> <i>R</i> with field of fractions <i>K</i>, the <i>R</i>-module <i>K</i> is an injective <i>R</i>-module, and indeed the smallest injective <i>R</i>-module containing <i>R</i>. <p>Two examples are the injective hull of the <b>Z</b>-module <b>Z</b>/<i>p</i><b>Z</b> (the <a href="/wiki/Pr%C3%BCfer_group" title="Prüfer group">Prüfer group</a>), and the injective hull of the <i>k</i>[<i>x</i>]-module <i>k</i> (the ring of inverse polynomials). The latter is easily described as <i>k</i>[<i>x</i>,<i>x</i><sup>−1</sup>]/<i>xk</i>[<i>x</i>]. This module has a basis consisting of "inverse monomials", that is <i>x</i><sup>−<i>n</i></sup> for <i>n</i> = 0, 1, 2, …. Multiplication by scalars is as expected, and multiplication by <i>x</i> behaves normally except that <i>x</i>·1 = 0. The endomorphism ring is simply the ring of <a href="/wiki/Formal_power_series" title="Formal power series">formal power series</a>.</p>
<h3><span class="mw-headline" id="Artinian_examples">Artinian examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=5" title="Edit section: Artinian examples">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>If <i>G</i> is a <a href="/wiki/Finite_group" title="Finite group">finite group</a> and <i>k</i> a field with <a href="/wiki/Characteristic_(algebra)" title="Characteristic (algebra)">characteristic</a> 0, then one shows in the theory of <a href="/wiki/Group_representation" title="Group representation">group representations</a> that any subrepresentation of a given one is already a direct summand of the given one. Translated into module language, this means that all modules over the <a href="/wiki/Group_algebra" title="Group algebra">group algebra</a> <i>kG</i> are injective. If the characteristic of <i>k</i> is not zero, the following example may help.</p>
<p>If <i>A</i> is a unital <a href="/wiki/Associative_algebra" title="Associative algebra">associative algebra</a> over the field <i>k</i> with finite <a class="mw-redirect" href="/wiki/Dimension_of_a_vector_space" title="Dimension of a vector space">dimension</a> over <i>k</i>, then Hom<sub><i>k</i></sub>(−, <i>k</i>) is a <a class="mw-redirect" href="/wiki/Duality_of_categories" title="Duality of categories">duality</a> between finitely generated left <i>A</i>-modules and finitely generated right <i>A</i>-modules. Therefore, the finitely generated injective left <i>A</i>-modules are precisely the modules of the form Hom<sub><i>k</i></sub>(<i>P</i>, <i>k</i>) where <i>P</i> is a finitely generated projective right <i>A</i>-module. For <a href="/wiki/Frobenius_algebra" title="Frobenius algebra">symmetric algebras</a>, the duality is particularly well-behaved and projective modules and injective modules coincide.</p>
<p>For any <a href="/wiki/Artinian_ring" title="Artinian ring">Artinian ring</a>, just as for <a href="/wiki/Commutative_ring" title="Commutative ring">commutative rings</a>, there is a 1-1 correspondence between prime ideals and indecomposable injective modules. The correspondence in this case is perhaps even simpler: a prime ideal is an annihilator of a unique simple module, and the corresponding indecomposable injective module is its <a href="/wiki/Injective_hull" title="Injective hull">injective hull</a>. For finite-dimensional algebras over fields, these injective hulls are <a class="mw-redirect" href="/wiki/Finitely-generated_module" title="Finitely-generated module">finitely-generated modules</a> (<a href="#CITEREFLam1999">Lam 1999</a>, §3G, §3J).</p>
<h2><span class="mw-headline" id="Theory">Theory</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=6" title="Edit section: Theory">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Submodules.2C_quotients.2C_products.2C_and_sums">Submodules, quotients, products, and sums</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=7" title="Edit section: Submodules, quotients, products, and sums">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Any <a href="/wiki/Product_(category_theory)" title="Product (category theory)">product</a> of (even infinitely many) injective modules is injective; conversely, if a direct product of modules is injective, then each module is injective (<a href="#CITEREFLam1999">Lam 1999</a>, p. 61). Every direct sum of finitely many injective modules is injective. In general, submodules, factor modules, or infinite <a href="/wiki/Direct_sum_of_modules" title="Direct sum of modules">direct sums</a> of injective modules need not be injective. Every submodule of every injective module is injective if and only if the ring is <a href="/wiki/Artinian_ring" title="Artinian ring">Artinian</a> <a class="mw-redirect" href="/wiki/Semisimple_ring" title="Semisimple ring">semisimple</a> (<a href="#CITEREFGolanHead1991">Golan &amp; Head 1991</a>, p. 152); every factor module of every injective module is injective if and only if the ring is <a href="/wiki/Hereditary_ring" title="Hereditary ring">hereditary</a>, (<a href="#CITEREFLam1999">Lam 1999</a>, Th. 3.22); every infinite direct sum of injective modules is injective if and only if the ring is <a href="/wiki/Noetherian_ring" title="Noetherian ring">Noetherian</a>, (<a href="#CITEREFLam1999">Lam 1999</a>, Th 3.46).<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<h3><span class="mw-headline" id="Baer.27s_criterion">Baer's criterion</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=8" title="Edit section: Baer's criterion">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In Baer's original paper, he proved a useful result, usually known as Baer's Criterion, for checking whether a module is injective: a left <i>R</i>-module <i>Q</i> is injective if and only if any homomorphism <i>g</i> : <i>I</i> → <i>Q</i> defined on a <a href="/wiki/Ideal_(ring_theory)" title="Ideal (ring theory)">left ideal</a> <i>I</i> of <i>R</i> can be extended to all of <i>R</i>.</p>
<p>Using this criterion, one can show that <b>Q</b> is an injective <a href="/wiki/Abelian_group" title="Abelian group">abelian group</a> (i.e. an injective module over <b>Z</b>). More generally, an abelian group is injective if and only if it is <a class="mw-redirect" href="/wiki/Divisible_module" title="Divisible module">divisible</a>. More generally still: a module over a <a href="/wiki/Principal_ideal_domain" title="Principal ideal domain">principal ideal domain</a> is injective if and only if it is divisible (the case of vector spaces is an example of this theorem, as every field is a principal ideal domain and every vector space is divisible). Over a general integral domain, we still have one implication: every injective module over an integral domain is divisible.</p>
<p>Baer's criterion has been refined in many ways (<a href="#CITEREFGolanHead1991">Golan &amp; Head 1991</a>, p. 119), including a result of (<a href="#CITEREFSmith1981">Smith 1981</a>) and (<a href="#CITEREFVamos1983">Vamos 1983</a>) that for a commutative Noetherian ring, it suffices to consider only <a href="/wiki/Prime_ideal" title="Prime ideal">prime ideals</a> <i>I</i>. The dual of Baer's Criterion would give a simple test for projectivity, but even for the ring <b>Z</b> of integers, this becomes the unsolvable <a href="/wiki/Whitehead_problem" title="Whitehead problem">Whitehead problem</a>.</p>
<h3><span class="mw-headline" id="Injective_cogenerators">Injective cogenerators</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=9" title="Edit section: Injective cogenerators">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Injective_cogenerator" title="Injective cogenerator">injective cogenerator</a></div>
<p>Maybe the most important injective module is the abelian group <b>Q</b>/<b>Z</b>. It is an <a href="/wiki/Injective_cogenerator" title="Injective cogenerator">injective cogenerator</a> in the <a href="/wiki/Category_of_abelian_groups" title="Category of abelian groups">category of abelian groups</a>, which means that it is injective and any other module is contained in a suitably large product of copies of <b>Q</b>/<b>Z</b>. So in particular, every abelian group is subgroup of an injective one. It is quite significant that this is also true over any ring: every module is a submodule of an injective one, or "the category of left <i>R</i>-modules has enough injectives." To prove this, one uses the peculiar properties of the abelian group <b>Q</b>/<b>Z</b> to construct an injective cogenerator in the category of left <i>R</i>-modules.</p>
<p>For a left <i>R</i>-module <i>M</i>, the so-called "character module" <i>M</i><sup>+</sup> = Hom<sub><b>Z</b></sub>(<i>M</i>,<b>Q</b>/<b>Z</b>) is a right <i>R</i>-module that exhibits an interesting duality, not between injective modules and <a href="/wiki/Projective_module" title="Projective module">projective modules</a>, but between injective modules and <a href="/wiki/Flat_module" title="Flat module">flat modules</a> (<a href="#CITEREFEnochsJenda2001">Enochs &amp; Jenda 2001</a>, pp. 78–80). For any ring <i>R</i>, a left <i>R</i>-module is flat if and only if its character module is injective. If <i>R</i> is left noetherian, then a left <i>R</i>-module is injective if and only if its character module is flat.</p>
<h3><span class="mw-headline" id="Injective_hulls">Injective hulls</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=10" title="Edit section: Injective hulls">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Injective_hull" title="Injective hull">injective hull</a></div>
<p>The <a href="/wiki/Injective_hull" title="Injective hull">injective hull</a> of a module is the smallest injective module containing the given one and was described in (<a href="#CITEREFEckmannShopf1953">Eckmann &amp; Shopf 1953</a>).</p>
<p>One can use injective hulls to define a minimal injective resolution (see below). If each term of the injective resolution is the injective hull of the cokernel of the previous map, then the injective resolution has minimal length.</p>
<h3><span class="mw-headline" id="Injective_resolutions">Injective resolutions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=11" title="Edit section: Injective resolutions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Every module <i>M</i> also has an <b>injective <a href="/wiki/Resolution_(algebra)" title="Resolution (algebra)">resolution</a></b>: an <a href="/wiki/Exact_sequence" title="Exact sequence">exact sequence</a> of the form</p>
<dl>
<dd>0 → <i>M</i> → <i>I</i><sup>0</sup> → <i>I</i><sup>1</sup> → <i>I</i><sup>2</sup> → ...</dd>
</dl>
<p>where the <i>I</i> <sup><i>j</i></sup> are injective modules. Injective resolutions can be used to define <a href="/wiki/Derived_functor" title="Derived functor">derived functors</a> such as the <a href="/wiki/Ext_functor" title="Ext functor">Ext functor</a>.</p>
<p>The <i>length</i> of a finite injective resolution is the first index <i>n</i> such that <i>I</i><sup><i>n</i></sup> is nonzero and <i>I</i><sup><i>i</i></sup> = 0 for <i>i</i> greater than <i>n</i>. If a module <i>M</i> admits a finite injective resolution, the minimal length among all finite injective resolutions of <i>M</i> is called its <b>injective dimension</b> and denoted id(<i>M</i>). If <i>M</i> does not admit a finite injective resolution, then by convention the injective dimension is said to be infinite. (<a href="#CITEREFLam1999">Lam 1999</a>, §5C) As an example, consider a module <i>M</i> such that id(<i>M</i>) = 0. In this situation, the exactness of the sequence 0 → <i>M</i> → <i>I</i><sub>0</sub> → 0 indicates that the arrow in the center is an isomorphism, and hence <i>M</i> itself is injective.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>Equivalently, the injective dimension of <i>M</i> is the minimal integer (if there is such, otherwise ∞) <i>n</i> such that Ext<span style="display:inline-block;margin-bottom:-0.3em;vertical-align:-0.4em;line-height:1.2em;font-size:80%;text-align:left"><i>N</i><br/>
<i>A</i></span>(–,<i>M</i>) = 0 for all <i>N</i> &gt; <i>n</i>.</p>
<h3><span class="mw-headline" id="Indecomposables">Indecomposables</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=12" title="Edit section: Indecomposables">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Every injective submodule of an injective module is a direct summand, so it is important to understand <a href="/wiki/Indecomposable_module" title="Indecomposable module">indecomposable</a> injective modules, (<a href="#CITEREFLam1999">Lam 1999</a>, §3F).</p>
<p>Every indecomposable injective module has a <a href="/wiki/Local_ring" title="Local ring">local</a> <a href="/wiki/Endomorphism_ring" title="Endomorphism ring">endomorphism ring</a>. A module is called a <i><a href="/wiki/Uniform_module" title="Uniform module">uniform module</a></i> if every two nonzero submodules have nonzero intersection. For an injective module <i>M</i> the following are equivalent:</p>
<ul>
<li><i>M</i> is indecomposable</li>
<li><i>M</i> is nonzero and is the injective hull of every nonzero submodule</li>
<li><i>M</i> is uniform</li>
<li><i>M</i> is the injective hull of a uniform module</li>
<li><i>M</i> is the injective hull of a uniform <a href="/wiki/Cyclic_module" title="Cyclic module">cyclic module</a></li>
<li><i>M</i> has a local endomorphism ring</li>
</ul>
<p>Over a Noetherian ring, every injective module is the direct sum of (uniquely determined) indecomposable injective modules. Over a commutative Noetherian ring, this gives a particularly nice understanding of all injective modules, described in (<a href="#CITEREFMatlis1958">Matlis 1958</a>). The indecomposable injective modules are the injective hulls of the modules <i>R</i>/<i>p</i> for <i>p</i> a prime ideal of the ring <i>R</i>. Moreover the injective hull <i>M</i> of <i>R</i>/<i>p</i> has an increasing filtration by modules <i>M</i><sub><i>n</i></sub> given by the annihilators of the ideals <i>p</i><sup><i>n</i></sup>, and <i>M</i><sub><i>n</i>+1</sub>/<i>M</i><sub><i>n</i></sub> is isomorphic as finite-dimensional vector space over the quotient field <i>k</i>(<i>p</i>) of <i>R</i>/<i>p</i> to Hom<sub><i>R</i>/<i>p</i></sub>(<i>p</i><sup><i>n</i></sup>/<i>p</i><sup><i>n</i>+1</sup>, <i>k</i>(<i>p</i>)).</p>
<h3><span class="mw-headline" id="Change_of_rings">Change of rings</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=13" title="Edit section: Change of rings">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>It is important to be able to consider modules over <a href="/wiki/Subring" title="Subring">subrings</a> or <a href="/wiki/Quotient_ring" title="Quotient ring">quotient rings</a>, especially for instance <a href="/wiki/Polynomial_ring" title="Polynomial ring">polynomial rings</a>. In general, this is difficult, but a number of results are known, (<a href="#CITEREFLam1999">Lam 1999</a>, p. 62).</p>
<p>Let <i>S</i> and <i>R</i> be rings, and <i>P</i> be a left-<i>R</i>, right-<i>S</i> <a href="/wiki/Bimodule" title="Bimodule">bimodule</a> that is <a href="/wiki/Flat_module" title="Flat module">flat</a> as a left-<i>R</i> module. For any injective right <i>S</i>-module <i>M</i>, the set of <a href="/wiki/Module_homomorphism" title="Module homomorphism">module homomorphisms</a> Hom<sub><i>S</i></sub>( <i>P</i>, <i>M</i> ) is an injective right <i>R</i>-module. For instance, if <i>R</i> is a subring of <i>S</i> such that <i>S</i> is a flat <i>R</i>-module, then every injective <i>S</i>-module is an injective <i>R</i>-module. In particular, if <i>R</i> is an integral domain and <i>S</i> its <a href="/wiki/Field_of_fractions" title="Field of fractions">field of fractions</a>, then every vector space over <i>S</i> is an injective <i>R</i>-module. Similarly, every injective <i>R</i>[<i>x</i>]-module is an injective <i>R</i>-module.</p>
<p>For quotient rings <i>R</i>/<i>I</i>, the change of rings is also very clear. An <i>R</i>-module is an <i>R</i>/<i>I</i>-module precisely when it is annihilated by <i>I</i>. The submodule ann<sub><i>I</i></sub>(<i>M</i>) = { <i>m</i> in <i>M</i> : <i>im</i> = 0 for all <i>i</i> in <i>I</i> } is a left submodule of the left <i>R</i>-module <i>M</i>, and is the largest submodule of <i>M</i> that is an <i>R</i>/<i>I</i>-module. If <i>M</i> is an injective left <i>R</i>-module, then ann<sub><i>I</i></sub>(<i>M</i>) is an injective left <i>R</i>/<i>I</i>-module. Applying this to <i>R</i>=<b>Z</b>, <i>I</i>=<i>n</i><b>Z</b> and <i>M</i>=<b>Q</b>/<b>Z</b>, one gets the familiar fact that <b>Z</b>/<i>n</i><b>Z</b> is injective as a module over itself. While it is easy to convert injective <i>R</i>-modules into injective <i>R</i>/<i>I</i>-modules, this process does not convert injective <i>R</i>-resolutions into injective <i>R</i>/<i>I</i>-resolutions, and the homology of the resulting complex is one of the early and fundamental areas of study of relative homological algebra.</p>
<p>The textbook (<a href="#CITEREFRotman1979">Rotman 1979</a>, p. 103) has an erroneous proof that <a href="/wiki/Localization_of_a_ring" title="Localization of a ring">localization</a> preserves injectives, but a counterexample was given in (<a href="#CITEREFDade1981">Dade 1981</a>).</p>
<h3><span class="mw-headline" id="Self-injective_rings">Self-injective rings</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=14" title="Edit section: Self-injective rings">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Every ring with unity is a <a href="/wiki/Free_module" title="Free module">free module</a> and hence is a <a href="/wiki/Projective_module" title="Projective module">projective</a> as a module over itself, but it is rarer for a ring to be injective as a module over itself, (<a href="#CITEREFLam1999">Lam 1999</a>, §3B). If a ring is injective over itself as a right module, then it is called a <b>right self-injective ring</b>. Every <a href="/wiki/Frobenius_algebra" title="Frobenius algebra">Frobenius algebra</a> is self-injective, but no <a href="/wiki/Integral_domain" title="Integral domain">integral domain</a> that is not a <a href="/wiki/Field_(mathematics)" title="Field (mathematics)">field</a> is self-injective. Every proper <a href="/wiki/Quotient_ring" title="Quotient ring">quotient</a> of a <a href="/wiki/Dedekind_domain" title="Dedekind domain">Dedekind domain</a> is self-injective.</p>
<p>A right <a href="/wiki/Noetherian_ring" title="Noetherian ring">Noetherian</a>, right self-injective ring is called a <a href="/wiki/Quasi-Frobenius_ring" title="Quasi-Frobenius ring">quasi-Frobenius ring</a>, and is two-sided <a href="/wiki/Artinian_ring" title="Artinian ring">Artinian</a> and two-sided injective, (<a href="#CITEREFLam1999">Lam 1999</a>, Th. 15.1). An important module theoretic property of quasi-Frobenius rings is that the projective modules are exactly the injective modules.</p>
<h2><span class="mw-headline" id="Generalizations_and_specializations">Generalizations and specializations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=15" title="Edit section: Generalizations and specializations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Injective_objects">Injective objects</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=16" title="Edit section: Injective objects">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Injective_object" title="Injective object">injective object</a></div>
<p>One also talks about <a href="/wiki/Injective_object" title="Injective object">injective objects</a> in <a href="/wiki/Category_(mathematics)" title="Category (mathematics)">categories</a> more general than module categories, for instance in <a href="/wiki/Functor_category" title="Functor category">functor categories</a> or in categories of <a href="/wiki/Sheaf_(mathematics)" title="Sheaf (mathematics)">sheaves</a> of O<sub><i>X</i></sub>-modules over some <a href="/wiki/Ringed_space" title="Ringed space">ringed space</a> (<i>X</i>,O<sub><i>X</i></sub>). The following general definition is used: an object <i>Q</i> of the category <i>C</i> is <b>injective</b> if for any <a href="/wiki/Monomorphism" title="Monomorphism">monomorphism</a> <i>f</i> : <i>X</i> → <i>Y</i> in <i>C</i> and any morphism <i>g</i> : <i>X</i> → <i>Q</i> there exists a morphism <i>h</i> : <i>Y</i> → <i>Q</i> with <i>hf</i> = <i>g</i>.</p>
<h3><span class="mw-headline" id="Divisible_groups">Divisible groups</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=17" title="Edit section: Divisible groups">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Divisible_group" title="Divisible group">divisible group</a></div>
<p>The notion of injective object in the category of abelian groups was studied somewhat independently of injective modules under the term <a href="/wiki/Divisible_group" title="Divisible group">divisible group</a>. Here a <b>Z</b>-module <i>M</i> is injective if and only if <i>n</i>⋅<i>M</i> = <i>M</i> for every nonzero integer <i>n</i>. Here the relationships between <a href="/wiki/Flat_module" title="Flat module">flat modules</a>, <a href="/wiki/Pure_submodule" title="Pure submodule">pure submodules</a>, and injective modules is more clear, as it simply refers to certain divisibility properties of module elements by integers.</p>
<h3><span class="mw-headline" id="Pure_injectives">Pure injectives</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Injective_module&amp;action=edit&amp;section=18" title="Edit section: Pure injectives">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a class="mw-redirect" href="/wiki/Pure_injective_module" title="Pure injective module">pure injective module</a></div>
<p>In relative homological algebra, the extension property of homomorphisms may be required only for certain submodules, rather than for all. For instance, a <a class="mw-redirect" href="/wiki/Pure_injective_module" title="Pure injective module">pure injective module</a> is a module in which a homomorphism from a <a href="/wiki/Pure_submodule" title="Pure submodule">pure submodule</a> can be extended to the whole module.</p>
