<h1 class="firstHeading" id="firstHeading" lang="en">Double-click</h1>

<p>A <b>double-click</b> is the act of pressing a <a href="/wiki/Computer_mouse" title="Computer mouse">computer mouse</a> button twice quickly without moving the mouse. Double-clicking allows two different actions to be associated with the same mouse button. It was developed by <a href="/wiki/Bill_Atkinson" title="Bill Atkinson">Bill Atkinson</a> of <a class="mw-redirect" href="/wiki/Apple_Computer" title="Apple Computer">Apple Computer</a> (now Apple Inc.) for their <a href="/wiki/Apple_Lisa" title="Apple Lisa">Lisa</a> project.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> Often, <a href="/wiki/Point_and_click#Single_click" title="Point and click">single-clicking</a> selects (or highlights) an object, while a double-click executes the function associated with that object. Following a link in a <a href="/wiki/Web_browser" title="Web browser">web browser</a> is accomplished with only a single click, requiring the use of a second mouse button, "click and hold" delay, or modifier key to gain access to actions other than following the link. On <a href="/wiki/Touchscreen" title="Touchscreen">touchscreens</a>, the double-click is called "double-tap"; it's not used as much as double-click, but typically it functions as a zoom feature. ("triple-tap" sometimes used to zoom the whole screen.)</p>

<h2><span class="mw-headline" id="On_icons">On icons</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Double-click&amp;action=edit&amp;section=1" title="Edit section: On icons">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>On most systems, double-clicking an icon in the <a href="/wiki/File_manager" title="File manager">file manager</a> will perform a default action on the object represented by the icon. Double-clicking an application program will launch the program, and double-clicking a file icon will open the file in a default application for that file's type or format.</p>
<h2><span class="mw-headline" id="On_text">On text</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Double-click&amp;action=edit&amp;section=2" title="Edit section: On text">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In many programs, double-clicking on text selects an entire word.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> (In <a href="/wiki/X_Window_System" title="X Window System">X Window</a>, it will also copy that piece of text into a <a href="/wiki/Data_buffer" title="Data buffer">buffer</a> separate from the system clipboard, as with all selected text. The selected text is not also put into clipboard until an overt cut or copy action takes place. A person can retrieve the information from this buffer, which is not the system clipboard, later by pressing the middle mouse button.)</p>
<h2><span class="mw-headline" id="Difficulties">Difficulties</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Double-click&amp;action=edit&amp;section=3" title="Edit section: Difficulties">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>New mouse users or the elderly often have difficulty with double-clicking due to a need for specific <a href="/wiki/Fine_motor_skill" title="Fine motor skill">fine motor skills</a>. They may have trouble clicking fast enough or keeping the mouse still while double-clicking.</p>
<p>Solutions to this may include:</p>
<ul>
<li>Click once to select and press Enter on keyboard (on Windows systems).</li>
<li>Using keyboard navigation instead of a mouse.</li>
<li>Configuring the system to use single clicks for actions usually associated with double-clicks.</li>
<li>Configuring the system to allow for more delay time between the two clicks for it to be registered as a double-click (See below for how to on several operating systems)</li>
<li>Remapping the double-click function to a single click on an additional button, for example the often unused middle button. This effectively creates a Unix style 3-button scheme of select/action/context.</li>
<li>To prevent the mouse from moving during a double-click, bracing the mouse by putting the thumb on the side of the mouse and the bottom of the hand on the bottom of the mouse.</li>
<li>In Windows, the threshold of movement can be increased by changing the associated registry keys in HKEY_CURRENT_USER/Control Panel/Mouse</li>
</ul>
<p>Additionally, applications and operating systems will often not require the mouse to be completely still. Instead, they implement <a href="/wiki/Hysteresis" title="Hysteresis">hysteresis</a>, allowing for a small amount of movement between the two clicks.</p>
<p>Another complication lies in the fact that some systems associate one action with a single click, another with a double-click, and yet another with a two consecutive single clicks. Even advanced users sometimes fail to differentiate between these properly. An example is the most common way of renaming a file in <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>. A single click highlights the file's icon and another single click (on the filename, not the icon) makes the name of the file editable. A user who tries to execute this action may inadvertently open the file (a double-click) by clicking too quickly, while a user who tries to open the file may find it being renamed by clicking too slowly. This may be avoided by Windows' users by using the menu (or <a href="/wiki/Function_key" title="Function key">F2</a>/<a href="/wiki/Enter_key" title="Enter key">Enter</a>) to initiate renaming and opening rather than multiple clicks. In <a href="/wiki/GNOME" title="GNOME">GNOME</a>, this problem is avoided entirely by simply not allowing file renaming by this method. In the <a href="/wiki/Classic_Mac_OS" title="Classic Mac OS">classic Mac OS</a>, which originated this technique, moving the mouse after the first click would immediately highlight the name. This was the result of a bug in the first versions of the system, one that was deliberately continued after users had come to rely upon it.</p>
<h2><span class="mw-headline" id="Speed_and_timing">Speed and timing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Double-click&amp;action=edit&amp;section=4" title="Edit section: Speed and timing">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The maximum delay required for two consecutive clicks to be interpreted as a double-click is not standardized. According to Microsoft's MSDN website, the default timing in Windows is 500 ms (half a second).<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> The double-click time is also used as a basis for other timed actions.</p>
<p>The double-click timing delay can usually be configured by the user. For example, adjusting double-click settings can be done by:</p>
<ul>
<li><a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a> and <a href="/wiki/Windows_7" title="Windows 7">Windows 7</a> - Start &gt; Control Panel &gt; Mouse &gt; Buttons (Start &gt; Control Panel &gt; Printers &amp; Other Hardware &gt; Mouse &gt; Buttons if Control Panel is in Category view). If you prefer, you may use Start &gt; Run &gt; main.cpl.</li>
<li><a href="/wiki/MacOS" title="MacOS">macOS</a> - Applications (or Apple menu) &gt; System Preferences &gt; Keyboard &amp; Mouse &gt; Mouse</li>
<li>In the <a href="/wiki/KDE" title="KDE">KDE</a> Desktop under <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> operating systems - K Menu &gt; Control Center (or Alt+F2 "kcontrol") &gt; Peripherals &gt; Mouse &gt; Adva<u>n</u>ced &gt; Double-click <u>i</u>nterval</li>
<li>In the <a href="/wiki/GNOME" title="GNOME">GNOME</a> Desktop under <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> operating systems - System &gt; Preferences &gt; Mouse</li>
</ul>
<h2><span class="mw-headline" id="Patent">Patent</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Double-click&amp;action=edit&amp;section=5" title="Edit section: Patent">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In 2004, <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> was granted a patent on using a double-click on "limited resource computing devices".<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> As a result of this, some observers fear that any U.S. company which uses double-clicking may have to change their product not to use the technology, pay licensing fees to Microsoft, or give Microsoft access to intellectual property.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
