<p>In <a href="/wiki/Computing" title="Computing">computing</a>, the <b>desktop metaphor</b> is an <a href="/wiki/Interface_metaphor" title="Interface metaphor">interface metaphor</a> which is a set of unifying concepts used by <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interfaces</a> to help users interact more easily with the computer.<sup class="reference" id="cite_ref-autogenerated1_1-0"><a href="#cite_note-autogenerated1-1">[1]</a></sup> The desktop metaphor treats the <a href="/wiki/Computer_monitor" title="Computer monitor">computer monitor</a> as if it is the user's <a href="/wiki/Desk" title="Desk">desktop</a>, upon which <i>objects</i> such as <a href="/wiki/Document" title="Document">documents</a> and <a href="/wiki/File_folder" title="File folder">folders</a> of documents can be placed. A document can be opened into a <a href="/wiki/Window_(computing)" title="Window (computing)">window</a>, which represents a paper copy of the document placed on the desktop. Small applications called <a class="mw-redirect" href="/wiki/Desk_Accessory" title="Desk Accessory">desk accessories</a> are also available, such as a desk calculator or notepad, etc.</p>
<p>The desktop metaphor itself has been extended and stretched with various implementations of <a href="/wiki/Desktop_environment" title="Desktop environment">desktop environments</a>, since access to features and <a href="/wiki/Usability" title="Usability">usability</a> of the computer are usually more important than maintaining the ‘purity’ of the <a href="/wiki/Metaphor" title="Metaphor">metaphor</a>. Hence we find trash cans on the desktop, as well as disks and network volumes (which can be thought of as <a href="/wiki/Filing_cabinet" title="Filing cabinet">filing cabinets</a>—not something normally found <i>on</i> a desktop). Other features such as <a href="/wiki/Menu_bar" title="Menu bar">menu bars</a>, <a class="mw-redirect" href="/wiki/Task_bar" title="Task bar">task bars</a>, or <a class="mw-redirect" href="/wiki/Dock_(computing)" title="Dock (computing)">docks</a> have no counterpart on a real-world desktop.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Desktop_metaphor&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tleft">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Apple_Macintosh_Desktop.png"><img alt="A screenshot of the original Mac OS. See caption." class="thumbimage" data-file-height="342" data-file-width="512" height="147" src="//upload.wikimedia.org/wikipedia/en/thumb/5/50/Apple_Macintosh_Desktop.png/220px-Apple_Macintosh_Desktop.png" srcset="//upload.wikimedia.org/wikipedia/en/thumb/5/50/Apple_Macintosh_Desktop.png/330px-Apple_Macintosh_Desktop.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/5/50/Apple_Macintosh_Desktop.png/440px-Apple_Macintosh_Desktop.png 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Apple_Macintosh_Desktop.png" title="Enlarge"></a></div>
The original 1984 <a href="/wiki/Classic_Mac_OS" title="Classic Mac OS">Mac OS</a> desktop popularized the new <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a>. Users communicated with the computer not through abstract textual <a href="/wiki/Command-line_interface" title="Command-line interface">commands</a> but rather using a metaphorical desktop that included <a class="mw-redirect" href="/wiki/Computer_icon" title="Computer icon">icons</a> of <a href="/wiki/Real_life" title="Real life">real life</a> items with which the user was already familiar.</div>
</div>
</div>
<p>The desktop metaphor was first introduced by Alan Kay at <a class="mw-redirect" href="/wiki/Palo_Alto_Research_Center" title="Palo Alto Research Center">Xerox PARC</a> in 1970 and elaborated in a series of innovative software applications developed by PARC scientists throughout the ensuing decade. The first computer to use an early version of the desktop metaphor was the experimental <a href="/wiki/Xerox_Alto" title="Xerox Alto">Xerox Alto</a>,<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> and the first commercial computer that adopted this kind of interface was the <a href="/wiki/Xerox_Star" title="Xerox Star">Xerox Star</a>. The use of <a class="mw-redirect" href="/wiki/Window_control" title="Window control">window controls</a> to contain related information predates the desktop metaphor, with a primitive version appearing in <a href="/wiki/Douglas_Engelbart" title="Douglas Engelbart">Douglas Engelbart</a>'s "<a class="mw-redirect" href="/wiki/Mother_of_All_Demos" title="Mother of All Demos">Mother of All Demos</a>",<sup class="reference" id="cite_ref-ArsTechnica2_4-0"><a href="#cite_note-ArsTechnica2-4">[4]</a></sup> though it was incorporated by PARC in the environment of the <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a> language.<sup class="reference" id="cite_ref-ArsTechnica3_5-0"><a href="#cite_note-ArsTechnica3-5">[5]</a></sup></p>
<p>One of the very first desktop-like interfaces on the market was a program called Magic Desk I built on a cartridge for the <a href="/wiki/Commodore_64" title="Commodore 64">Commodore 64</a> <a href="/wiki/Home_computer" title="Home computer">home computer</a> in 1983. A very primitive GUI presented a <a class="mw-redirect" href="/wiki/Low_resolution" title="Low resolution">low resolution</a> sketch of a desktop, complete with telephone, drawers, calculator, etc. The user made their choices by moving a <a href="/wiki/Sprite_(computer_graphics)" title="Sprite (computer graphics)">sprite</a> depicting a hand pointing by using the same <a href="/wiki/Joystick" title="Joystick">joystick</a> the user may have used for <a class="mw-redirect" href="/wiki/Video_gaming" title="Video gaming">video gaming</a>. Onscreen options were chosen by pushing the fire button on the joystick. The Magic Desk program featured a <a href="/wiki/Typewriter" title="Typewriter">typewriter</a> graphically emulated complete with audio effects. Other applications included a calculator, <a href="/wiki/Rolodex" title="Rolodex">rolodex</a> organiser, and a <a href="/wiki/Terminal_emulator" title="Terminal emulator">terminal emulator</a>. Files could be archived into the drawers of the desktop. A <a href="/wiki/Trash_(computing)" title="Trash (computing)">trashcan</a> was also present.</p>
<p>The first computer to popularise the desktop metaphor, using it as a standard feature over the earlier <a href="/wiki/Command-line_interface" title="Command-line interface">command-line interface</a> was the <a class="mw-redirect" href="/wiki/Apple_Macintosh" title="Apple Macintosh">Apple Macintosh</a> in 1984. The desktop metaphor is ubiquitous in modern-day personal computing; it is found in most <a href="/wiki/Desktop_environment" title="Desktop environment">desktop environments</a> of modern operating systems: <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a> as well as <a href="/wiki/MacOS" title="MacOS">macOS</a>, <a href="/wiki/Linux" title="Linux">Linux</a>, and other <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> systems.</p>
<p><a href="/wiki/BeOS" title="BeOS">BeOS</a> observed the desktop metaphor more strictly than many systems. For example, external hard drives appeared on the ‘desktop’, while internal ones were accessed clicking on an <a href="/wiki/Icon_(computing)" title="Icon (computing)">icon</a> representing the computer itself. By comparison, the Mac OS places all drives on the desktop itself by default, while in Windows the user can access the drives through an icon labelled "Computer".</p>
<p><a href="/wiki/Amiga" title="Amiga">Amiga</a> terminology for its desktop metaphor was taken directly from workshop jargon. The desktop was called <a href="/wiki/Workbench_(AmigaOS)" title="Workbench (AmigaOS)">Workbench</a>, programs were called <a class="mw-redirect" href="/wiki/Tools" title="Tools">tools</a>, small applications (<a class="mw-redirect" href="/wiki/Applets" title="Applets">applets</a>) were utilities, directories were drawers, etc. Icons of objects were animated and the directories are shown as drawers which were represented either open or closed. As in the <a href="/wiki/Classic_Mac_OS" title="Classic Mac OS">classic Mac OS</a> and <a href="/wiki/MacOS" title="MacOS">macOS</a> desktop, an icon for a <a href="/wiki/Floppy_disk" title="Floppy disk">floppy disk</a> or <a href="/wiki/CD-ROM" title="CD-ROM">CD-ROM</a> would appear on the desktop when the disk was inserted into the drive, as it was a virtual counterpart of a physical floppy disk or CD-ROM on the surface of a workbench.</p>
<h2><span class="mw-headline" id="Paper_paradigm">Paper paradigm</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Desktop_metaphor&amp;action=edit&amp;section=2" title="Edit section: Paper paradigm">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

<p>The <i>paper paradigm</i> refers to the <a href="/wiki/Paradigm" title="Paradigm">paradigm</a> used by most modern computers and operating systems. The paper paradigm consists of, usually, black text on a white background, files within folders, and a "desktop." The paper paradigm was created by many individuals and organisations, such as <a href="/wiki/Douglas_Engelbart" title="Douglas Engelbart">Douglas Engelbart</a>, <a class="mw-redirect" href="/wiki/Xerox_PARC" title="Xerox PARC">Xerox PARC</a>, and <a class="mw-redirect" href="/wiki/Apple_Computer" title="Apple Computer">Apple Computer</a>, and was an attempt to make computers more user-friendly by making them resemble the common workplace of the time (with papers, folders, and a desktop).<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> It was first presented to the public by Engelbart in 1968, in what is now referred to as "<a href="/wiki/The_Mother_of_All_Demos" title="The Mother of All Demos">The Mother of All Demos</a>".</p>
<p>From John Siracusa:<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></p>
<blockquote>
<p>Back in 1984, explanations of the original <a class="mw-redirect" href="/wiki/Apple_Macintosh" title="Apple Macintosh">Mac</a> interface to users who had never seen a <a class="mw-redirect" href="/wiki/GUI" title="GUI">GUI</a> before inevitably included an explanation of <a href="/wiki/Icon_(computing)" title="Icon (computing)">icons</a> that went something like this: "This icon represents your file on disk." But to the surprise of many, users very quickly discarded any semblance of indirection. This icon <i>is</i> my file. My file <i>is</i> this icon. One is not a "representation of" or an "interface to" the other. Such relationships were foreign to most people, and constituted unnecessary mental baggage when there was a much more simple and direct connection to what they knew of reality.</p>
</blockquote>
<p>Since then, many aspects of computers have wandered away from the paper paradigm by implementing features such as "shortcuts" to files, <a href="/wiki/Hypertext" title="Hypertext">hypertext</a>, and non-spatial file browsing. A shortcut (a link to a file that acts as a redirecting proxy, not the actual file) and hypertext have no real-world equivalent. Non-spatial file browsing, as well, may confuse novice users, as they can often have more than one window representing the same folder open at the same time, something that is impossible in reality. These and other departures from real-world equivalents are violations of the pure paper paradigm.</p>
