<p><b>Microsoft WinHelp</b> is a proprietary format for <a href="/wiki/Online_help" title="Online help">online help</a> files that can be displayed by the Microsoft Help browser <i>winhelp.exe</i> or <i>winhlp32.exe</i>. The file format is based on <a href="/wiki/Rich_Text_Format" title="Rich Text Format">Rich Text Format</a> (RTF). It remained a popular Help platform from <a href="/wiki/Windows_3.0" title="Windows 3.0">Windows 3.0</a> platform through <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a>. WinHelp was removed in <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a> to discourage software developers from using the obsolete format and encourage use of newer help formats.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=WinHelp&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>1990 - WinHelp 1.0 shipped with <a href="/wiki/Windows_3.0" title="Windows 3.0">Windows 3.0</a>.</li>
<li>1995 - WinHelp 4.0 shipped with <a href="/wiki/Windows_95" title="Windows 95">Windows 95</a> / <a href="/wiki/Windows_NT" title="Windows NT">Windows NT</a>.</li>
<li>2006 - Microsoft announced its intentions to phase out WinHelp as a supported platform. WinHelp is not part of <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a> out of the box. WinHelp files come in 16 bit and 32 bit types. Vista treats these files types differently. When starting an application that uses the 32 bit .hlp format, Windows warns that the format is no longer supported. A downloadable viewer for 32 bit .hlp files is available from the Microsoft Download Center.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-KB917607_2-0"><a href="#cite_note-KB917607-2">[2]</a></sup> The 16 bit WinHelp files continue to display in Windows Vista (32 bit only) without the viewer download.</li>
<li>October 14, 2009 - Microsoft announced the downloadability of Windows Help program (WinHlp32.exe) for Windows 7 at the Microsoft Download Center.<sup class="reference" id="cite_ref-DownloadWinHlp32Windows7_3-0"><a href="#cite_note-DownloadWinHlp32Windows7-3">[3]</a></sup></li>
</ul>
<h2><span class="mw-headline" id="File_format">File format</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=WinHelp&amp;action=edit&amp;section=2" title="Edit section: File format">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A WinHelp file has a ".hlp" suffix. It can be accompanied by an optional table of contents (.cnt) file if the help developer created one. When Windows opens a WinHelp file, it creates a .gid file in the same directory or in "<code>%LOCALAPPDATA%\Help</code>", containing information about the .hlp file such as the window size and location. If the user clicks the "Find" tab and enables keyword indexing, Windows creates an index file with a .fts (full text search) extension. Annotations and bookmarks for each Windows help file have the extension ".ann" and ".bmk".</p>
<p>A number of software tools can decompile a WinHelp file into its source documents—HPJ, CNT, <a href="/wiki/Rich_Text_Format" title="Rich Text Format">RTF</a>, <a href="/wiki/Bitmap" title="Bitmap">BMP</a>, <a href="/wiki/Segmented_Hyper_Graphics" title="Segmented Hyper Graphics">SHG</a>). An HPJ file is the project file that is created and edited in the Help Workshop (or a third party help authoring tool). The HPJ contains information about what RTF files to compile into the help, the MAP IDs and Aliases that provide links from a calling application to the help file, and help file appearance (window size, default buttons, color schemes, etc.). The CNT file provides the table of contents for the help file. An SHG file is a "SHED" graphics file that essentially creates an image map of help calls for a graphic file (e.g., a BMP).</p>
<p>A number of tools can read and explore these files. (See, for example, <a class="external text" href="http://www.herdsoft.com/catalog/ehlp2rtf.html" rel="nofollow">Help to RTF</a> and <a class="external text" href="http://www.herdsoft.com/linux/themen/online_publishing_winhelp.html" rel="nofollow">winhelpcgi</a>).</p>
<h2><span class="mw-headline" id="Source_files_and_compilation">Source files and compilation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=WinHelp&amp;action=edit&amp;section=3" title="Edit section: Source files and compilation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Source files required to compile a .hlp file consist of one or more documents with the extension .rtf and a help project file with the extension .hpj, along with any image files (.bmp, .wmf, or .shg) that are used within the Help file. An optional table of contents file with the extension .cnt can also be created for use with the .hlp file.</p>
<p>Within the .rtf files, topics are separated by page breaks. Each topic has a series of <a class="mw-redirect" href="/wiki/Footnotes" title="Footnotes">footnotes</a> that contain information for the help compiler:</p>
<dl>
<dd><b>#</b> footnotes contain the topic ID (used to create links to that topic).</dd>
<dd><b>$</b> footnotes contain the topic name as it displays in the table of contents, index, and other locations.</dd>
<dd><b>K</b> footnotes contain keywords for the index.</dd>
<dd><b>A</b> footnotes contain See Also keywords.</dd>
<dd><b>*</b> footnotes contain build tags.</dd>
<dd><b>+</b> footnotes contain browse sequence information.</dd>
<dd><b>!</b> footnotes contain topic entry macros.</dd>
</dl>
<p>Only the # footnote is required. All others are optional.</p>
<p>Text in each topic can contain limited formatting, including <a class="mw-redirect" href="/wiki/Bold" title="Bold">bold</a> text, <a class="mw-redirect" href="/wiki/Italics" title="Italics">italics</a>, and colors. Superscript and subscript are not allowed. Jumps between topics in the same Help file usually appear in the source document as double-underlined text (green by default, though this can be overridden) followed by a topic ID in hidden text. Popup links appear in the source document as text with a single underline (also green by default) followed by a topic ID in hidden text. (In the .hlp file, the jumps show up as green text with a single underline, and popups show up as green text with a dotted underline.)</p>
<p>Images can be added using codes such as {bmc image.bmp}. Supported image formats include <a class="mw-redirect" href="/wiki/.bmp" title=".bmp">.bmp</a>, <a class="mw-redirect" href="/wiki/.wmf" title=".wmf">.wmf</a>, and .shg (used for image maps, which can contain jumps or popups that are triggered by clicking on specific parts of the image).</p>
<p>After the source files have been created, the help file can be compiled using a WinHelp compiler such as HCW.exe or by using a commercial software program such as <a class="mw-redirect" href="/wiki/RoboHelp" title="RoboHelp">RoboHelp</a> or <a class="external text" href="http://www.solutionsoft.com/hlpbrz.htm" rel="nofollow">HelpBreeze</a>, most of which (included the two cited here) also use hcw.exe as the <a href="/wiki/Front_and_back_ends" title="Front and back ends">backend</a> compiler.</p>
<h2><span class="mw-headline" id="WinHelp_appearance_and_features">WinHelp appearance and features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=WinHelp&amp;action=edit&amp;section=4" title="Edit section: WinHelp appearance and features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Depending on how it has launched and what settings the Help author chose, a WinHelp file opens either to its default topic, its table of contents, or its index.</p>
<p>A topic in a WinHelp file opens in a separate window, in a size and initial position that the Help author may choose. Users can resize or reposition the window. The Help author can control whether the Help file stores the user's settings between sessions, or always opens in the default size and position.</p>
<p>When a topic is open, a title bar at the top of the Help window displays the topic title. Below that is a row of <a href="/wiki/Menu_(computing)" title="Menu (computing)">menus</a> (<b>File</b>, <b>Edit</b>, <b>Bookmark</b>, <b>Options</b>, and <b>Help</b>), which control various aspects of the file. A row of <a href="/wiki/Button_(computing)" title="Button (computing)">buttons</a> usually appears below the menus. The Help author controls which buttons, if any, appear. Typical buttons include <b>Contents</b>, <b>Index</b>, <b>Back</b>, and <b>Print</b>, along with <b>&lt;&lt;</b> and <b>&gt;&gt;</b> buttons to browse through the file. Help authors can also create custom buttons to jump to specific topics or perform other actions.</p>
<p>Below the buttons is the main text area of the window. Typically, the text begins with a heading, often bold or in a larger font than the rest of the text. This heading may sometimes be in a non-scrolling region—an area of the window that does not move up or down via the scrollbar at the side of the window. Non-scrolling regions can only be used at the beginning of a topic. The Help author can control size and background color of a non-scrolling region.</p>
<p>Help authors can also control the background color of the main text area, where the actual text of the topic appears. This text can be formatted and arranged in many ways. Within the text, jumps appear as green text with a single <a href="/wiki/Underline" title="Underline">underline</a>. Single-clicking on a jump opens a different topic. Some jumps may open secondary Help windows to display information. Popups appear in the text as green text with a dotted underline. Single-clicking on a popup opens a small window with no menus, buttons, or scrollbars, sized to fit the text. Often, popups provide short definitions of key terms or other supplemental information about the main text. The popup automatically disappears the next time the user clicks or presses a key.</p>
<p>Many, though not all Help topics have <b>See Also</b> jumps at the end of the text. Depending on the Help author's preference, this feature may be a simple list of jumps under the heading <i>See Also,</i> or it may be a small button that, when clicked, brings up a dialog box displaying all the relevant topics. Clicking on the name of a topic in that <a href="/wiki/Dialog_box" title="Dialog box">dialog box</a> then clicking <b>Display</b> opens that topic.</p>
<p>Most Help files also contain a table of contents and an index to help users locate information. These appear in a separate, tabbed window. Clicking on the <b>Contents</b> tab opens the table of contents, in which users can click on headings to see the topics. Often, headings are marked with icons that look like small books and the topics have icons that look like pages. Double-clicking on a topic (or clicking on a topic then clicking <b>Display</b>) opens that topic. Clicking on the <b>Index</b> tab opens the index, which has a typing field and an alphabetical keyword list. Typing in the typing field automatically scrolls the list of keywords to the closest match. Double-clicking on a keyword (or clicking on a keyword then clicking <b>Display</b>) displays the topic associated with that keyword (if only one) or brings up a list of all topics associated with it. The index is important in helping users locate information. Sometimes Help files also have a <b>Find</b> tab, which lets the user search for any word used in the text of the file, not just for keywords.</p>
<p>WinHelp also supports a feature known as <a href="/wiki/Context-sensitive_help" title="Context-sensitive help">context-sensitive help</a>. Context-sensitive help is assistance that is appropriate to where the user is in the software application, and what they are trying to do.</p>
<p>A rather security critical feature is that one can also include a <a href="/wiki/Dynamic-link_library" title="Dynamic-link library">DLL</a> file containing custom code and associating it with WinHelp topics. Effectively this makes .HLP files equivalent to executables.</p>
<h2><span class="mw-headline" id="End_of_support">End of support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=WinHelp&amp;action=edit&amp;section=5" title="Edit section: End of support">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>At the 2006 WritersUA conference, Microsoft announced its intentions to phase out WinHelp as a supported platform. Ted Dworkin (Partner Director of WinHelp Experience) stated, "WinHelp does not meet the code standards established for Vista. These standards include security, reliability, and performance." He went on to say that WinHelp is designed in such a way that, "...we would have to rewrite it from the ground up to meet the Vista code standards. And that approach doesn't make sense given that we have two other Help systems in Vista."</p>
<p>The updated licensing agreement prohibits application developers from packaging the WinHelp libraries with their installers. This means that WinHelp manuals for legacy applications are not readable on a new Windows Vista installation. To read them, the end-user must obtain the 32-bit WinHelp viewer from Microsoft's website and manually install it.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<h2><span class="mw-headline" id="Other_Documentation_File_Formats">Other Documentation File Formats</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=WinHelp&amp;action=edit&amp;section=6" title="Edit section: Other Documentation File Formats">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Although documentation <i>can</i> be maintained entirely in a vendor-specific presentation format such as WinHelp, it is more often the case that documentation must be published in multiple presentation formats at once: <a href="/wiki/Microsoft_Compiled_HTML_Help" title="Microsoft Compiled HTML Help">Microsoft Compiled HTML Help</a> (CHM), WinHelp, <a href="/wiki/HTML" title="HTML">HTML</a> pages, Java Help, <a class="mw-redirect" href="/wiki/PDF" title="PDF">PDF</a>, <i>etc.</i> It would be very expensive and error-prone to maintain each format separately.</p>
<p>For this reason, authors often maintain documentation in an industry-standard, vendor-neutral authoring format—such as <a href="/wiki/DocBook" title="DocBook">DocBook</a> or <a class="mw-redirect" href="/wiki/FrameMaker" title="FrameMaker">FrameMaker</a>—that can be used to generate several different presentation formats (including WinHelp).<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2011)">citation needed</span></a></i>]</sup> Various presentation files thus produced (with WinHelp or other tools) contain consistent content because they were generated from the same source.</p>
