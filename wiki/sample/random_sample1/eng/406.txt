<p>The <b>ampersand</b> is the <a href="/wiki/Logogram" title="Logogram">logogram</a> "<b>&amp;</b>", representing the <a class="mw-redirect" href="/wiki/Grammatical_conjunction" title="Grammatical conjunction">conjunction</a> word "and". It originated as a <a href="/wiki/Typographic_ligature" title="Typographic ligature">ligature</a> of the letters <i>et</i>, <a href="/wiki/Latin" title="Latin">Latin</a> for "and".<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>

<h2><span class="mw-headline" id="Etymology">Etymology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ampersand&amp;action=edit&amp;section=1" title="Edit section: Etymology">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tleft">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Alphabet_with_ampersand.jpg"><img alt="" class="thumbimage" data-file-height="410" data-file-width="309" height="292" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Alphabet_with_ampersand.jpg/220px-Alphabet_with_ampersand.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/e/ec/Alphabet_with_ampersand.jpg 1.5x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Alphabet_with_ampersand.jpg" title="Enlarge"></a></div>
A page from an 1863 textbook displaying the alphabet. Note the &amp; as the 27th character.</div>
</div>
</div>
<p>The word <i>ampersand</i> is a corruption of the phrase "and <i><a class="mw-redirect" href="/wiki/Per_se_(terminology)" title="Per se (terminology)">per se</a></i> &amp; (and)", meaning "and intrinsically the word <i>and</i> (represented by the symbol &amp;)".<sup class="reference" id="cite_ref-adobe_2-0"><a href="#cite_note-adobe-2">[2]</a></sup></p>
<p>Traditionally, when reciting the <a href="/wiki/English_alphabet" title="English alphabet">alphabet</a> in English-speaking schools, any letter that could also be used as a word in itself ("A", "I", and, at one point, "<a href="/wiki/Vocative_case#English" title="Vocative case">O</a>") was repeated with the Latin expression <i><a class="extiw" href="https://en.wiktionary.org/wiki/per_se" title="wikt:per se">per se</a></i> ("by itself").<sup class="reference" id="cite_ref-aglossary_3-0"><a href="#cite_note-aglossary-3">[3]</a></sup><sup class="reference" id="cite_ref-worddetective_4-0"><a href="#cite_note-worddetective-4">[4]</a></sup><sup class="reference" id="cite_ref-merriam-webster_5-0"><a href="#cite_note-merriam-webster-5">[5]</a></sup> This habit was useful in spelling where a word or syllable was repeated after spelling; e.g. "d, o, g—dog" would be clear but simply saying "a—a" would be confusing without the clarifying "per se" added. It was also common practice to add the "&amp;" sign at the end of the alphabet as if it were the 27th letter, pronounced as the Latin <i>et</i> or later in English as <i>and</i>. As a result, the recitation of the alphabet would end in "X, Y, Z, <i>and per se and</i>". This last phrase was routinely slurred to "ampersand" and the term had entered common English usage by 1837.<sup class="reference" id="cite_ref-worddetective_4-1"><a href="#cite_note-worddetective-4">[4]</a></sup><sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> However, in contrast to the 26 letters, the ampersand does not represent a speech sound—although other characters that were dropped from the English alphabet did, such as the Old English <a href="/wiki/Thorn_(letter)" title="Thorn (letter)">thorn</a>, <a href="/wiki/Wynn" title="Wynn">wynn</a>, and <a href="/wiki/Eth" title="Eth">eth</a>.</p>
<p>Through <a class="mw-redirect" href="/wiki/Popular_etymology" title="Popular etymology">popular etymology</a>, it has been falsely claimed that <a href="/wiki/Andr%C3%A9-Marie_Amp%C3%A8re" title="André-Marie Ampère">André-Marie Ampère</a> used the symbol in his widely read publications and that people began calling the new shape "Ampère's and".<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>

<div class="thumb tleft">
<div class="thumbinner" style="width:392px;"><a class="image" href="/wiki/File:Historical_ampersand_evolution.svg"><img alt="" class="thumbimage" data-file-height="200" data-file-width="780" height="100" src="//upload.wikimedia.org/wikipedia/commons/thumb/1/1c/Historical_ampersand_evolution.svg/390px-Historical_ampersand_evolution.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/1/1c/Historical_ampersand_evolution.svg/585px-Historical_ampersand_evolution.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/1/1c/Historical_ampersand_evolution.svg/780px-Historical_ampersand_evolution.svg.png 2x" width="390"/></a>

<p>The ampersand can be traced back to the 1st century A.D. and the Old <a href="/wiki/Roman_cursive" title="Roman cursive">Roman cursive</a>, in which the letters E and T occasionally were written together to form a <a href="/wiki/Typographic_ligature" title="Typographic ligature">ligature</a> (figure 1). In the later and more flowing New Roman Cursive, ligatures of all kinds were extremely common; figures 2 and 3 from the middle of 4th century are both examples of how the et-ligature could look in this script. During the following development of the Latin script that led up to the <a href="/wiki/Carolingian_minuscule" title="Carolingian minuscule">Carolingian minuscule</a> (9th century) the use of ligatures in general diminished. The et-ligature, however, continued to be used and gradually became more stylized and less revealing of its origin (figures 4–6).<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
<p>The modern <a href="/wiki/Italic_type" title="Italic type">italic type</a> ampersand is a kind of "<a class="extiw" href="https://en.wiktionary.org/wiki/et" title="wikt:et">et</a>" ligature that goes back to the cursive scripts developed during the <a href="/wiki/Renaissance" title="Renaissance">Renaissance</a>. After the advent of <a href="/wiki/Printing" title="Printing">printing</a> in Europe in 1455, printers made extensive use of both the italic and Roman ampersands. Since the ampersand's roots go back to Roman times, many languages that use a variation of the <a href="/wiki/Latin_alphabet" title="Latin alphabet">Latin alphabet</a> make use of it.</p>
<p>The ampersand often appeared as a letter at the end of the Latin alphabet, as for example in <a href="/wiki/Byrhtferth" title="Byrhtferth">Byrhtferð's</a> list of letters from 1011.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> Similarly, <i>&amp;</i> was regarded as the 27th letter of the <a href="/wiki/English_alphabet" title="English alphabet">English alphabet</a>, as used by children (in the US)<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (August 2016)">citation needed</span></a></i>]</sup>. An example may be seen in M. B. Moore's 1863 book <i>The Dixie Primer, for the Little Folks</i>.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> In her 1859 novel <i><a href="/wiki/Adam_Bede" title="Adam Bede">Adam Bede</a></i>, <a href="/wiki/George_Eliot" title="George Eliot">George Eliot</a> refers to this when she makes Jacob Storey say: "He thought it [Z] had only been put to finish off th' alphabet like; though ampusand would ha' done as well, for what he could see."<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> The popular <a href="/wiki/Apple_Pie_ABC" title="Apple Pie ABC">Apple Pie ABC</a> finishes with the lines "X, Y, Z, and ampersand, All wished for a piece in hand".</p>
<p>The ampersand should not be confused with the <a href="/wiki/Tironian_notes" title="Tironian notes">Tironian "et"</a> ("⁊"), which is a symbol similar to the numeral <i>7</i>. Both symbols have their roots in the classical antiquity, and both signs were used up through the Middle Ages as a representation for the Latin word "et" ("and"). However, while the ampersand was in origin a common ligature in the everyday script, the Tironian "et" was part of a highly specialised stenographic <a href="/wiki/Shorthand" title="Shorthand">shorthand</a>.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> The Tironian "et" ("⁊") is found in old Irish language script, a Latin-based script generally only used for decorative purposes today, where it signifies <i>agus</i> ("and") in Irish. This symbol may have entered the script language by way of monastic influence in the time of the early Christian church in Ireland.</p>
<h2><span class="mw-headline" id="Writing_the_ampersand">Writing the ampersand</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ampersand&amp;action=edit&amp;section=3" title="Edit section: Writing the ampersand">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In everyday <a href="/wiki/Handwriting" title="Handwriting">handwriting</a>, the ampersand is sometimes simplified in design as a large lowercase <a href="/wiki/Epsilon" title="Epsilon">epsilon</a> (<b>Ɛ</b>) or a backwards numeral <b>3</b> superimposed by a vertical line. The ampersand is also often shown as a backwards <b>3</b> with a vertical line above and below it or a dot above and below it.</p>
<p>The <a class="mw-redirect" href="/wiki/Plus_sign" title="Plus sign"><b>+</b> sign</a> is often informally used in place of an ampersand, sometimes with an added loop and resembling ɬ.</p>

<h2><span class="mw-headline" id="Usage">Usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ampersand&amp;action=edit&amp;section=4" title="Edit section: Usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Ampersands are commonly seen in business names formed from partnership of two or more people, such as <i><a href="/wiki/Johnson_%26_Johnson" title="Johnson &amp; Johnson">Johnson &amp; Johnson</a></i>, <i><a href="/wiki/Dolce_%26_Gabbana" title="Dolce &amp; Gabbana">Dolce &amp; Gabbana</a></i>, <i><a href="/wiki/Marks_%26_Spencer" title="Marks &amp; Spencer">Marks &amp; Spencer</a></i>, <i><a class="mw-redirect" href="/wiki/A%26P" title="A&amp;P">A&amp;P</a></i> (supermarkets), and <i><a href="/wiki/Tiffany_%26_Co." title="Tiffany &amp; Co.">Tiffany &amp; Co.</a></i>, as well as some abbreviations containing the word <i>and</i>, such as <i>AT&amp;T</i> (<a class="mw-redirect" href="/wiki/American_Telephone_and_Telegraph" title="American Telephone and Telegraph">American Telephone and Telegraph</a>), <i>R&amp;D</i> (<a href="/wiki/Research_and_development" title="Research and development">research and development</a>), <i>R&amp;B</i> (<a href="/wiki/Rhythm_and_blues" title="Rhythm and blues">rhythm and blues</a>), <i>B&amp;B</i> (<a href="/wiki/Bed_and_breakfast" title="Bed and breakfast">bed and breakfast</a>), and <i>P&amp;L</i> (<a class="mw-redirect" href="/wiki/Profit_and_loss" title="Profit and loss">profit and loss</a>).<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup><sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></p>
<p>In <a href="/wiki/Film" title="Film">film</a> credits for stories, <a href="/wiki/Screenplay" title="Screenplay">screenplays</a>, etc., <i>&amp;</i> indicates a closer collaboration than <i>and</i>. The ampersand is used by the <a href="/wiki/Writers_Guild_of_America" title="Writers Guild of America">Writers Guild of America</a> to denote two writers collaborating on a specific script, rather than one writer rewriting another's work. In screenplays, two authors joined with <i>&amp;</i> collaborated on the script, while two authors joined with <i>and</i> worked on the script at different times and may not have consulted each other at all.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup><sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> In the latter case, they both contributed enough significant material to the screenplay to receive credit but did not work together.</p>
<p>In <a href="/wiki/APA_style" title="APA style">APA style</a>, the ampersand is used when citing sources in text such as (Jones &amp; Jones, 2005). In the list of references, an ampersand precedes the last author's name when there is more than one author.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup> (This does not apply to <a class="mw-redirect" href="/wiki/The_MLA_Style_Manual" title="The MLA Style Manual">MLA style</a>, which calls for the "and" to be spelled.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup>)</p>
<p>The phrase <i><a href="/wiki/Et_cetera" title="Et cetera">et cetera</a></i> ("and so forth"), usually written as <i>etc.</i> can be abbreviated <i>&amp;c.</i> representing the combination <i>et</i> + <i>c(etera)</i>.</p>
<p>The ampersand can be used to indicate that the "and" in a listed item is a part of the item's name and not a separator (e.g. "Rock, pop, rhythm &amp; blues, and hip hop").</p>
<p>The ampersand may still be used as an abbreviation for "and" in informal writing regardless of how "and" is used.</p>

<h3><span class="mw-headline" id="Programming_languages">Programming languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ampersand&amp;action=edit&amp;section=7" title="Edit section: Programming languages">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In the 20th century, following the development of <a class="mw-redirect" href="/wiki/Formal_logic" title="Formal logic">formal logic</a>, the ampersand became a commonly used logical notation for the <a class="mw-redirect" href="/wiki/Binary_operator" title="Binary operator">binary operator</a> or <a class="mw-redirect" href="/wiki/Sentential_connective" title="Sentential connective">sentential connective</a> <i><a href="/wiki/Logical_conjunction" title="Logical conjunction">AND</a></i>. This usage was adopted in computing.</p>
<p>Many languages with syntax derived from <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, including <a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/Perl" title="Perl">Perl</a>,<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> and more differentiate between:</p>
<ul>
<li><tt>&amp;</tt> for <a href="/wiki/Bitwise_operation" title="Bitwise operation">bitwise</a> AND. <tt>(4 &amp; 2)</tt> is zero, <tt>(4 &amp; 5)</tt> is 4.</li>
<li><tt>&amp;&amp;</tt> for <a class="mw-redirect" href="/wiki/Minimal_evaluation" title="Minimal evaluation">short-circuit</a> <a href="/wiki/Logical_conjunction" title="Logical conjunction">logical AND</a>. <tt>(4 &amp;&amp; 2)</tt> is true.</li>
</ul>
<p>In <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a>, and <a href="/wiki/Go_(programming_language)" title="Go (programming language)">Go</a>, a prefix "&amp;" is a unary operator denoting the <a href="/wiki/Memory_address" title="Memory address">address in memory</a> of the argument, <i>e.g.</i> <code>&amp;x, &amp;func, &amp;a[3]</code>.</p>
<p>In <a href="/wiki/C%2B%2B" title="C++">C++</a> and <a href="/wiki/PHP" title="PHP">PHP</a>, unary prefix &amp; before a <a class="mw-redirect" href="/wiki/Parameter_(computer_science)" title="Parameter (computer science)">formal parameter</a> of a <a class="mw-redirect" href="/wiki/Function_(programming)" title="Function (programming)">function</a> denotes <a href="/wiki/Reference_(C%2B%2B)" title="Reference (C++)">pass-by-reference</a>.</p>
<p>In <a href="/wiki/Fortran" title="Fortran">Fortran</a>, the ampersand forces the compiler to treat two lines as one. This is accomplished by placing an ampersand at the end of the first line and at the beginning of the second line.</p>
<p>In <a href="/wiki/Common_Lisp" title="Common Lisp">Common Lisp</a>, the ampersand is the prefix for lambda list keywords.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup></p>
<p>Ampersand is the <a class="mw-redirect" href="/wiki/String_concatenation" title="String concatenation">string concatenation</a> operator in many <a class="mw-redirect" href="/wiki/BASIC_programming_language" title="BASIC programming language">BASIC dialects</a>, <a href="/wiki/AppleScript" title="AppleScript">AppleScript</a>, <a href="/wiki/Lingo_(programming_language)" title="Lingo (programming language)">Lingo</a>, <a href="/wiki/HyperTalk" title="HyperTalk">HyperTalk</a>, and <a href="/wiki/FileMaker" title="FileMaker">FileMaker</a>. In <a href="/wiki/Ada_(programming_language)" title="Ada (programming language)">Ada</a> it applies to all one-dimensional arrays, not just strings.</p>
<p><a href="/wiki/BASIC-PLUS" title="BASIC-PLUS">BASIC-PLUS</a> on the <a href="/wiki/Digital_Equipment_Corporation" title="Digital Equipment Corporation">DEC</a> <a href="/wiki/PDP-11" title="PDP-11">PDP-11</a> uses the ampersand as a short form of the verb <tt>PRINT</tt>.</p>
<p><a href="/wiki/Applesoft_BASIC" title="Applesoft BASIC">Applesoft BASIC</a> used the ampersand as an internal command, not intended to be used for general programming, that invoked a <a class="mw-redirect" href="/wiki/Machine_language" title="Machine language">machine language</a> program in the computer's <a href="/wiki/Read-only_memory" title="Read-only memory">ROM</a>.</p>
<p>In some versions of BASIC, unary suffix &amp; denotes a variable is of type <i>long</i>, or 32 <a href="/wiki/Bit" title="Bit">bits</a> in length.</p>
<p>The ampersand is occasionally used as a prefix to denote a <a href="/wiki/Hexadecimal" title="Hexadecimal">hexadecimal</a> number, such as <code>&amp;FF</code> for decimal 255, for instance in <a href="/wiki/BBC_BASIC" title="BBC BASIC">BBC BASIC</a>. Some other languages, such as the <a href="/wiki/Machine_code_monitor" title="Machine code monitor">Monitor</a> built into ROM on the <a href="/wiki/Commodore_128" title="Commodore 128">Commodore 128</a>, used it to indicate <a href="/wiki/Octal" title="Octal">octal</a> instead, a convention that spread throughout the Commodore community and is now used in the <a href="/wiki/VICE" title="VICE">VICE</a> emulator.</p>
<p>In <a href="/wiki/MySQL" title="MySQL">MySQL</a> the '&amp;' has dual roles. As well as a logical AND, it additionally serves as the bitwise operator of an intersection between elements.</p>
<p>In more recent years, the ampersand has made its way into the <a href="/wiki/Haskell_(programming_language)" title="Haskell (programming language)">Haskell</a> standard library, representing flipped <a href="/wiki/Function_application" title="Function application">function application</a>: <code>x &amp; f</code> means the same thing as <code>f x</code>.</p>
<p>The ampersand character is used as a special character in at least some versions of the database software originally created in Denmark under the name <a class="mw-redirect" href="/wiki/Navision" title="Navision">Navision</a> (the software has since been acquired by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>). Using this character in either "Text" or "Code" fields could create difficulties for performing certain tasks in Navision, such as filtering records (either by the user or by programming). It is also used as described below to indicate shortcuts in menu items and labels.</p>
<p><a href="/wiki/Perl" title="Perl">Perl</a> uses the ampersand as a <a href="/wiki/Sigil_(computer_programming)" title="Sigil (computer programming)">sigil</a> to refer to subroutines:</p>
<ul>
<li>In Perl 4 and earlier, it was effectively required to call user-defined subroutines<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup></li>
<li>In Perl 5, it can still be used to modify the way user-defined subroutines are called<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup></li>
<li>In Perl 6, the ampersand <a href="/wiki/Sigil_(computer_programming)" title="Sigil (computer programming)">sigil</a> is only used when referring to a subroutine as an object, never when calling it<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup></li>
</ul>
<p>Ampersand is the name of a programming language, which uses <a href="/wiki/Relation_algebra" title="Relation algebra">relation algebra</a> to specify <a href="/wiki/Information_system" title="Information system">information systems</a>.<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup></p>
<h3><span class="mw-headline" id="Text_markup">Text markup</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ampersand&amp;action=edit&amp;section=8" title="Edit section: Text markup">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In <a class="mw-redirect" href="/wiki/SGML" title="SGML">SGML</a>, <a href="/wiki/XML" title="XML">XML</a>, and <a href="/wiki/HTML" title="HTML">HTML</a>, the ampersand is used to introduce an <a href="/wiki/SGML_entity" title="SGML entity">SGML entity</a>. The HTML and XML encoding for the ampersand character is the entity "<code>&amp;amp;</code>".<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup> This can create a problem known as <a href="/wiki/Delimiter#Delimiter_collision" title="Delimiter">delimiter collision</a> when converting text into one of these markup languages. For instance, when putting URLs or other material containing ampersands into XML format files such as <a class="mw-redirect" href="/wiki/RSS_(file_format)" title="RSS (file format)">RSS</a> files the &amp; must be replaced with &amp;amp; or they are considered not well formed, and computers will be unable to read the files correctly. SGML derived the use from <a href="/wiki/IBM_Generalized_Markup_Language" title="IBM Generalized Markup Language">IBM Generalized Markup Language</a>, which was one of many IBM-mainframe languages to use the ampersand to signal a text substitution, eventually going back to <a href="/wiki/IBM_System/360" title="IBM System/360">System/360</a> macro assembly language.</p>
<p>In the plain <a href="/wiki/TeX" title="TeX">TeX</a> <a href="/wiki/Markup_language" title="Markup language">markup language</a>, the ampersand is used to mark <a class="mw-redirect" href="/wiki/Tabstop" title="Tabstop">tabstops</a>. The ampersand itself can be applied in TeX with <code>\&amp;</code>. The <a href="/wiki/Computer_Modern" title="Computer Modern">Computer Modern</a> fonts replace it with an "E.T." symbol in the <tt>cmti<i>#</i></tt>(text italic) fonts, so it can be entered as <code>{\it\&amp;}</code> in running text when using the default (Computer Modern) fonts.<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup></p>
<p>In <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> menus, labels, and other captions, the ampersand is used to denote the keyboard shortcut for that option (<a href="/wiki/Alt_key" title="Alt key">Alt</a> + that letter, which appears underlined). A double ampersand is needed in order to display a real ampersand. This convention originated in the first WIN32 api, and is used in <a href="/wiki/Windows_Forms" title="Windows Forms">Windows Forms</a>,<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> and is also copied into many other tookits on multiple operating systems.</p>
<h3><span class="mw-headline" id="Unix_shells">Unix shells</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ampersand&amp;action=edit&amp;section=9" title="Edit section: Unix shells">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Some Unix shells use the ampersand as a <a href="/wiki/Metacharacter" title="Metacharacter">metacharacter</a>:</p>
<p>Some Unix shells, like the <a href="/wiki/POSIX" title="POSIX">POSIX</a> standard <code>sh</code> shell, use the ampersand to <a href="/wiki/Job_control_(Unix)" title="Job control (Unix)">execute a process in the background</a> and to duplicate <a href="/wiki/File_descriptor" title="File descriptor">file descriptors</a>.</p>
<ul>
<li>In <a href="/wiki/Bash_(Unix_shell)" title="Bash (Unix shell)">Bash</a>, the ampersand can separate words, control the command history, duplicate file descriptors, perform logical operations, control jobs, and participate in <a href="/wiki/Regular_expression" title="Regular expression">regular expressions</a>.<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup></li>
</ul>
<h3><span class="mw-headline" id="Web_standards">Web standards</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ampersand&amp;action=edit&amp;section=10" title="Edit section: Web standards">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The generic <a href="/wiki/Uniform_Resource_Locator" title="Uniform Resource Locator">URL</a> (Uniform Resource Locator) syntax allows for a <a href="/wiki/Query_string" title="Query string">query string</a> to be appended to a file name in a web address so that additional information can be passed to a script; the <a href="/wiki/Question_mark" title="Question mark">question mark</a>, or query mark, ?, is used to indicate the start of a query string.<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup> A query string is usually made up of a number of different name–value pairs, each separated by the ampersand symbol, &amp;. For example, <code>http://www.example.com/login.php?username=test&amp;password=blank</code>.</p>
