<p><b>Advanced Systems Format</b> (formerly <b>Advanced Streaming Format</b>, <b>Active Streaming Format</b>) is <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>'s <a href="/wiki/Proprietary_software" title="Proprietary software">proprietary</a> <a href="/wiki/Digital_audio" title="Digital audio">digital audio</a>/<a href="/wiki/Digital_video" title="Digital video">digital video</a> <a class="mw-redirect" href="/wiki/Container_format_(digital)" title="Container format (digital)">container format</a>, especially meant for <a href="/wiki/Streaming_media" title="Streaming media">streaming media</a>. ASF is part of the <a href="/wiki/Media_Foundation" title="Media Foundation">Media Foundation</a> framework.</p>

<h2><span class="mw-headline" id="Overview_and_features">Overview and features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Advanced_Systems_Format&amp;action=edit&amp;section=1" title="Edit section: Overview and features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>ASF is based on <a href="/wiki/Serialization" title="Serialization">serialized</a> <i><a href="/wiki/Object-oriented_programming" title="Object-oriented programming">objects</a></i> which are essentially <a href="/wiki/Byte" title="Byte">byte</a> sequences identified by a <a class="mw-redirect" href="/wiki/Globally_Unique_Identifier" title="Globally Unique Identifier">GUID</a> marker.</p>
<p>The format does not specify how (i.e. with which <a href="/wiki/Codec" title="Codec">codec</a>) the video or audio should be encoded; it just specifies the structure of the video/audio stream. This is similar to the function performed by the <a href="/wiki/QuickTime" title="QuickTime">QuickTime</a>, <a href="/wiki/Audio_Video_Interleave" title="Audio Video Interleave">AVI</a>, or <a href="/wiki/Ogg" title="Ogg">Ogg</a> container formats. One of the objectives of ASF was to support playback from <a href="/wiki/Streaming_media" title="Streaming media">digital media servers</a>, <a class="mw-redirect" href="/wiki/HTTP_server" title="HTTP server">HTTP servers</a>, and local storage devices such as <a href="/wiki/Hard_disk_drive" title="Hard disk drive">hard disk drives</a>.</p>
<p>The most common media contained within an ASF file are <a href="/wiki/Windows_Media_Audio" title="Windows Media Audio">Windows Media Audio</a> (WMA) and <a href="/wiki/Windows_Media_Video" title="Windows Media Video">Windows Media Video</a> (WMV). The most common file extensions for ASF files are extension <tt>.WMA</tt> (audio-only files using Windows Media Audio, with MIME-type <code>audio/x-ms-wma</code>) and <tt>.WMV</tt> (files containing video, using the Windows Media Audio and Video codecs, with MIME-type <code>video/x-ms-asf</code>). These files are identical to the old <tt>.ASF</tt> files but for their extension and MIME-type. The different extensions are used to make it easier to identify the content of a media file.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<p>ASF files can also contain objects representing <a class="mw-redirect" href="/wiki/Metadata_(computing)" title="Metadata (computing)">metadata</a>, such as the artist, title, album and genre for an audio track, or the director of a video track, much like the <a href="/wiki/ID3" title="ID3">ID3</a> tags of <a href="/wiki/MP3" title="MP3">MP3</a> files. It supports scalable media types and stream prioritization; as such, it is a format optimized for streaming.</p>
<p>The ASF container provides the framework for <a href="/wiki/Digital_rights_management" title="Digital rights management">digital rights management</a> in Windows Media Audio and Windows Media Video. An analysis of an older scheme used in WMA reveals that it is using a combination of <a href="/wiki/Elliptic_curve_cryptography" title="Elliptic curve cryptography">elliptic curve cryptography</a> key exchange, <a href="/wiki/Data_Encryption_Standard" title="Data Encryption Standard">DES</a> block cipher, a custom block cipher, <a class="mw-redirect" href="/wiki/RC4_cipher" title="RC4 cipher">RC4</a> stream cipher and the <a href="/wiki/SHA-1" title="SHA-1">SHA-1</a> hashing function.</p>
<p>ASF container-based media are sometimes still streamed on the internet either through the <a class="mw-redirect" href="/wiki/Microsoft_Media_Services" title="Microsoft Media Services">MMS</a> protocol or the <a href="/wiki/Real_Time_Streaming_Protocol" title="Real Time Streaming Protocol">RTSP</a> protocol. Mostly, however, they contain material encoded for 'progressive download', which can be distributed by any webserver and then offers the same advantages as streaming: the file starts playing as soon as a minimum number of bytes is received and the rest of the download continues in the background while one is watching or listening.</p>
<p>The <a href="/wiki/Library_of_Congress" title="Library of Congress">Library of Congress</a> <i>Digital Preservation</i> project considers ASF as <i>de facto</i> successor of <a href="/wiki/Resource_Interchange_File_Format" title="Resource Interchange File Format">RIFF</a>.<sup class="reference" id="cite_ref-LoC_2-1"><a href="#cite_note-LoC-2">[2]</a></sup> In 2010 <a href="/wiki/Google" title="Google">Google</a> picked RIFF as container format for <a href="/wiki/WebP" title="WebP">WebP</a>.</p>
<h2><span class="mw-headline" id="License">License</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Advanced_Systems_Format&amp;action=edit&amp;section=2" title="Edit section: License">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The specification is downloadable from the Microsoft website,<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> and the format can be implemented under a <a href="/wiki/License" title="License">license</a> from Microsoft that however does not allow distribution of sources and is not compatible with <a href="/wiki/Open-source_software" title="Open-source software">open source</a> licenses. The author of the free software project <a href="/wiki/VirtualDub" title="VirtualDub">VirtualDub</a> reported that a Microsoft employee informed him that his software violated a Microsoft patent regarding ASF playback.<sup class="reference" id="cite_ref-adv_5-0"><a href="#cite_note-adv-5">[5]</a></sup></p>
<p>Certain <a href="/wiki/Error_detection_and_correction" title="Error detection and correction">error-correcting</a> techniques related to ASF are patented in the United States (<a class="external text" href="http://www.google.com/patents?id=kY57AAAAEBAJ&amp;printsec=abstract&amp;zoom=4&amp;source=gbs_overview_r&amp;cad=0#v=onepage&amp;q&amp;f=false" rel="nofollow">United States Patent 6,041,345</a> Levi, et al. March 21, 2000) by Microsoft.</p>
