<p><b>EPUB</b> is an <a href="/wiki/Comparison_of_e-book_formats" title="Comparison of e-book formats">e-book file format</a> with the <a href="/wiki/Filename_extension" title="Filename extension">extension</a> <code>.epub</code> that can be downloaded and read on devices like smartphones, tablets, computers, or <a href="/wiki/E-reader" title="E-reader">e-readers</a>. It is a <a href="/wiki/Technical_standard" title="Technical standard">technical standard</a> published by the <a href="/wiki/International_Digital_Publishing_Forum" title="International Digital Publishing Forum">International Digital Publishing Forum</a> (IDPF). The term is short for <i>electronic publication</i> and is sometimes styled <i>ePub</i>. EPUB became an official standard of the IDPF in September 2007, superseding the older Open eBook standard.<sup class="reference" id="cite_ref-ops_approved_2-0"><a href="#cite_note-ops_approved-2">[2]</a></sup> The <a href="/wiki/Book_Industry_Study_Group" title="Book Industry Study Group">Book Industry Study Group</a> endorses EPUB 3 as the format of choice for packaging content and has stated that the global book publishing industry should rally around a single standard.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> EPUB is the most widely supported vendor-independent <a href="/wiki/XML" title="XML">XML</a>-based (as opposed to <a class="mw-redirect" href="/wiki/PDF" title="PDF">PDF</a>) e-book format; that is, it is supported by the largest number of hardware readers.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>EPUB 2.0 was approved in October 2007, with a maintenance update (2.0.1) approved in September 2010.<sup class="reference" id="cite_ref-epub2.0_history_4-0"><a href="#cite_note-epub2.0_history-4">[4]</a></sup> The EPUB 3.0 specification became effective in October 2011, superseded by a minor maintenance update (3.0.1) in June 2014.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> New major features include support for precise layout or specialized formatting (Fixed Layout Documents), such as for comic books,<sup class="reference" id="cite_ref-teleread_epub_test_6-0"><a href="#cite_note-teleread_epub_test-6">[6]</a></sup> and <a href="/wiki/MathML" title="MathML">MathML</a> support. The current version of EPUB is 3.1, effective January 5, 2017.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> The (text of) format specification underwent reorganization<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> and clean-up; format supports remotely-hosted resources and new font formats (<a class="mw-redirect" href="/wiki/WOFF" title="WOFF">WOFF 2.0</a> and <a href="/wiki/SFNT" title="SFNT">SFNT</a>)<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> and uses more pure <a href="/wiki/HTML" title="HTML">HTML</a> and <a class="mw-redirect" href="/wiki/CSS" title="CSS">CSS</a>.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></p>
<p>In May 2016 <a href="/wiki/International_Digital_Publishing_Forum" title="International Digital Publishing Forum">IDPF</a> Members approved <a href="/wiki/World_Wide_Web_Consortium" title="World Wide Web Consortium">World Wide Web Consortium</a> (W3C) merger,<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> "to fully align the publishing industry and core Web technology".<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=2" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The format and many readers support the following:</p>
<ul>
<li><a href="/wiki/Reflowable_document" title="Reflowable document">Reflowable document</a>: optimize text for a particular display</li>
<li>Fixed-layout content:<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> pre-paginated content can be useful for certain kinds of highly designed content, such as illustrated books intended only for larger screens, such as tablets.<sup class="reference" id="cite_ref-understanding-epub3_14-0"><a href="#cite_note-understanding-epub3-14">[14]</a></sup></li>
<li>Like an <a href="/wiki/HTML" title="HTML">HTML</a> web site, the format supports inline <a href="/wiki/Raster_graphics" title="Raster graphics">raster</a> and <a href="/wiki/Vector_graphics" title="Vector graphics">vector</a> images, <a href="/wiki/Metadata" title="Metadata">metadata</a>, and <a class="mw-redirect" href="/wiki/CSS" title="CSS">CSS</a> styling.</li>
<li>Page bookmarking</li>
<li>Passage highlighting and notes</li>
<li>A library that stores books and can be searched</li>
<li><a class="mw-redirect" href="/wiki/Font_size" title="Font size">Re-sizable</a> fonts, and changeable text and background colors</li>
<li>Support for a subset of <a href="/wiki/MathML" title="MathML">MathML</a><sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup></li>
<li>Digital rights management—can contain <a href="/wiki/Digital_rights_management" title="Digital rights management">digital rights management</a> (DRM) as an optional layer<sup class="reference" id="cite_ref-idpf_faq_16-0"><a href="#cite_note-idpf_faq-16">[16]</a></sup></li>
</ul>
<p>The EPUB specification does not enforce or suggest a particular DRM scheme. This could affect the level of support for various DRM systems on <a class="mw-redirect" href="/wiki/E-book_reader" title="E-book reader">devices</a> and the portability of purchased e-books. Consequently, such DRM incompatibility may segment the EPUB format along the lines of DRM systems, undermining the advantages of a single standard format and confusing the consumer.<sup class="reference" id="cite_ref-teleread_epub_drm1_17-0"><a href="#cite_note-teleread_epub_drm1-17">[17]</a></sup></p>
<h2><span class="mw-headline" id="Adoption_and_market_share">Adoption and market share</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=3" title="Edit section: Adoption and market share">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>It is also widely used on many software readers such as <a href="/wiki/IBooks" title="IBooks">iBooks</a> on <a href="/wiki/IOS" title="IOS">iOS</a> and <a href="/wiki/Google_Books" title="Google Books">Google Books</a> on <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android</a>, but notably not by the <a href="/wiki/Amazon_Kindle" title="Amazon Kindle">Amazon Kindle</a> e-readers. iBooks also supports the proprietary iBook format, which is based on the EPUB format but depends upon code from the iBooks app to function.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup></p>
<dl>
<dt>As final product</dt>
<dd>As in the eBook market <a href="/wiki/Portable_Document_Format" title="Portable Document Format">PDF format</a> is not generally sold at sites that sell eBooks, the EPUB participation in the eBook market is very low. The main EPUB producers are suppliers of <a href="/wiki/Public_domain" title="Public domain">public domain</a> and <a class="mw-redirect" href="/wiki/Open_license" title="Open license">open licensed</a> content, as <a href="/wiki/Project_Gutenberg" title="Project Gutenberg">Project Gutenberg</a>, <a href="/wiki/PubMed_Central" title="PubMed Central">PubMed Central</a>, <a href="/wiki/SciELO" title="SciELO">SciELO</a> and many others. In that context is growing and replacing the PDF format.</dd>
</dl>
<dl>
<dt><a href="/wiki/Electronic_data_interchange" title="Electronic data interchange">Data interchange</a></dt>
<dd>EPUB is a popular way to feed ebook creation process, because is an open format and is based in HTML, while Amazon's format is proprietary. EPUB is the "first step" content format in many production process and <a href="/wiki/Supply_chain" title="Supply chain">supply chains</a>.</dd>
</dl>
<h2><span class="mw-headline" id="Implementation">Implementation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=4" title="Edit section: Implementation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>An EPUB file is a <a class="mw-redirect" href="/wiki/ZIP_archive" title="ZIP archive">ZIP archive</a> that contains, in effect, a website—including HTML files, images, CSS style sheets, and other assets. It also contains <a href="/wiki/Metadata" title="Metadata">metadata</a>. EPUB 3 is the latest version. By using <a href="/wiki/HTML5" title="HTML5">HTML5</a>, publications can contain video, audio, and interactivity, just like websites in web browsers.<sup class="reference" id="cite_ref-understanding-epub3_14-1"><a href="#cite_note-understanding-epub3-14">[14]</a></sup></p>
<h3><span class="mw-headline" id="Container">Container</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=5" title="Edit section: Container">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>An ePub publication is delivered as a single file. This file is an unencrypted zipped archive containing a set of interrelated resources.<sup class="reference" id="cite_ref-ocf3.0_spec_19-0"><a href="#cite_note-ocf3.0_spec-19">[19]</a></sup></p>
<p>An OCF Abstract Container defines a file system model for the contents of the container. The file system model uses a single common root directory for all contents in the container. All (non-remote) resources for publications are in the directory tree headed by the container's root directory, though EPUB mandates no specific file system structure for this. The file system model includes a mandatory directory named META-INF that is a direct child of the container's root directory. META-INF stores container.xml.</p>
<p>The first file in the archive must be the <a class="mw-redirect" href="/wiki/Mimetype" title="Mimetype">mimetype</a> file. It must be uncompressed so that non-ZIP utilities can read the mimetype. The mimetype file must be an <a href="/wiki/ASCII" title="ASCII">ASCII</a> file that contains the string application/epub+zip. It must be unencrypted, and the first file in the ZIP archive. This file provides a more reliable way for applications to identify the mimetype of the file than just the .epub extension.<sup class="reference" id="cite_ref-ocf3.0_spec_19-1"><a href="#cite_note-ocf3.0_spec-19">[19]</a></sup></p>
<p>An example file structure:</p>
<pre>
--ZIP Container--
mimetype
META-INF/
  container.xml
OEBPS/
  content.opf
  chapter1.xhtml
  ch1-pic.png
  css/
    style.css
    myfont.otf
  toc.ncx
</pre>
<p>There must be a META-INF directory containing container.xml. This file points to the file defining the contents of the book, the OPF file, though additional alternative rootfile elements are allowed.<sup class="reference" id="cite_ref-ocf3.0_spec_19-2"><a href="#cite_note-ocf3.0_spec-19">[19]</a></sup> Apart from mimetype and META-INF/container.xml, the other files (OPF, NCX, XHTML, CSS and images files) are traditionally put in a directory named OEBPS. An example container.xml:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="cp">&lt;?xml version="1.0" encoding="UTF-8" ?&gt;</span>
<span class="nt">&lt;container</span> <span class="na">version=</span><span class="s">"1.0"</span> <span class="na">xmlns=</span><span class="s">"urn:oasis:names:tc:opendocument:xmlns:container"</span><span class="nt">&gt;</span>
  <span class="nt">&lt;rootfiles&gt;</span>
    <span class="nt">&lt;rootfile</span> <span class="na">full-path=</span><span class="s">"OEBPS/content.opf"</span> <span class="na">media-type=</span><span class="s">"application/oebps-package+xml"</span><span class="nt">/&gt;</span>
  <span class="nt">&lt;/rootfiles&gt;</span>
<span class="nt">&lt;/container&gt;</span>
</pre></div>
<h3><span class="mw-headline" id="Publication">Publication</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=6" title="Edit section: Publication">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The ePUB container must contain:<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup></p>
<ul>
<li>At least one content document.</li>
<li>One navigation document.</li>
<li>One package document listing all publication resources. This file should use the file extension <i>.opf</i>. It contains metadata, a manifest, fallback chains, bindings, and a spine. This is an ordered sequence of ID references defining the default reading order.</li>
</ul>
<p>The ePUB container may contain:</p>
<ul>
<li>Zero or more style sheets.</li>
<li>Zero or more <a href="/wiki/Pronunciation_Lexicon_Specification" title="Pronunciation Lexicon Specification">PLS</a> Documents.</li>
<li>Zero or more media overlay documents.</li>
</ul>
<h3><span class="mw-headline" id="Contents">Contents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=7" title="Edit section: Contents">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Content documents include:<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> <a class="mw-redirect" href="/wiki/HTML_5" title="HTML 5">HTML 5</a> content, navigation documents, <a class="mw-redirect" href="/wiki/SVG" title="SVG">SVG</a> documents, scripted content documents, and fixed layout documents. Contents also include CSS and PLS documents. Navigation documents supersedes the NCX grammar used in EPUB 2.</p>
<h3><span class="mw-headline" id="Media_overlays">Media overlays</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=8" title="Edit section: Media overlays">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Books with synchronized audio narration are created in EPUB 3 by using media overlay documents to describe the timing for the pre-recorded audio narration and how it relates to the EPUB Content Document markup. The file format for Media Overlays is defined as a subset of <a href="/wiki/Synchronized_Multimedia_Integration_Language" title="Synchronized Multimedia Integration Language">SMIL</a>.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup></p>
<h2><span class="mw-headline" id="Software">Software</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=9" title="Edit section: Software">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Many editors exist including <a href="/wiki/Calibre_(software)" title="Calibre (software)">calibre</a>, <a href="/wiki/Sigil_(application)" title="Sigil (application)">Sigil</a>, <a href="/wiki/LaTeX" title="LaTeX">LaTeX</a>, and <a href="/wiki/Genebook" title="Genebook">Genebook</a>. An <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> tool, called epubcheck, can be used for validating and detecting errors in the structural markup (OCF, OPF, OPS), image, and <a href="/wiki/XHTML" title="XHTML">XHTML</a> files; the tool can either be run from the command line or used in applications or <a href="/wiki/Web_application" title="Web application">webapps</a> as a <a href="/wiki/Library_(computing)" title="Library (computing)">library</a>; it is also available on EPUB Validator. Readers exist for all major hardware platforms, including <a href="/wiki/Google_Play_Books" title="Google Play Books">Google Play Books</a> (Android and iOS) and <a href="/wiki/IBooks" title="IBooks">Apple iBooks</a> (MacOS and iOS).</p>
