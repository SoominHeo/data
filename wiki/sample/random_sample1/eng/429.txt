<p><b>DWG</b> (from <i>drawing</i>) is a <a href="/wiki/Proprietary_format" title="Proprietary format">proprietary</a><sup class="reference" id="cite_ref-guides_to_good_practice_cad_3-2_2-1"><a href="#cite_note-guides_to_good_practice_cad_3-2-2">[2]</a></sup> binary <a href="/wiki/File_format" title="File format">file format</a> used for storing two- and three- dimensional design data and metadata. It is the native format for several <a href="/wiki/Computer-aided_design" title="Computer-aided design">CAD</a> packages including <a class="new" href="/w/index.php?title=DraftSight&amp;action=edit&amp;redlink=1" title="DraftSight (page does not exist)">DraftSight</a>, <a href="/wiki/AutoCAD" title="AutoCAD">AutoCAD</a>, <a class="mw-redirect" href="/wiki/IntelliCAD_Technology_Consortium" title="IntelliCAD Technology Consortium">IntelliCAD</a> (and its variants), <a href="/wiki/Caddie_(CAD_system)" title="Caddie (CAD system)">Caddie</a> and <a href="/wiki/Open_Design_Alliance" title="Open Design Alliance">Open Design Alliance</a> compliant applications. In addition, DWG is supported non-natively by many other CAD applications. The .bak (drawing backup), .dws (drawing standards), .dwt (drawing template) and .sv$ (temporary automatic save) files are also DWG files.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.dwg&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>DWG (denoted by the .dwg <a href="/wiki/Filename_extension" title="Filename extension">filename extension</a>) was the native file format for the <a class="new" href="/w/index.php?title=Interact_CAD&amp;action=edit&amp;redlink=1" title="Interact CAD (page does not exist)">Interact CAD</a> package, developed by Mike Riddle in the late 1970s,<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> and subsequently <a href="/wiki/Software_license" title="Software license">licensed</a> by <a href="/wiki/Autodesk" title="Autodesk">Autodesk</a> in 1982 as the basis for <a href="/wiki/AutoCAD" title="AutoCAD">AutoCAD</a>.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup><sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup><sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> From 1982 to 2009, Autodesk created versions of AutoCAD which wrote no fewer than 18 major variants of the DWG file format,<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> none of which is publicly documented.</p>
<p>The DWG format is probably the most widely used format for CAD drawings. Autodesk estimates that in 1998 there were in excess of two billion DWG files in existence.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<p>There are several claims to control of the DWG format.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> As the biggest and most influential creator of DWG files it is Autodesk who designs, defines, and iterates the DWG format as the native format for their CAD applications. Autodesk sells a read/write library, called RealDWG,<sup class="reference" id="cite_ref-autogenerated1_10-0"><a href="#cite_note-autogenerated1-10">[10]</a></sup> under selective licensing terms for use in non-competitive applications. Several companies have attempted to <a class="mw-redirect" href="/wiki/Reverse_engineer" title="Reverse engineer">reverse engineer</a> Autodesk's DWG format, and offer software libraries to read and write Autodesk DWG files. The most successful is Open Design Alliance,<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> a non-profit consortium created in 1998 by a number of software developers (including competitors to Autodesk), released a read/write/view library called the <a class="mw-redirect" href="/wiki/OpenDWG" title="OpenDWG">OpenDWG</a> Toolkit, which was based on the MarComp AUTODIRECT libraries.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> (ODA has since rewritten and updated that code.)</p>
<p>In 1998, Autodesk added <a href="/wiki/File_verification" title="File verification">file verification</a> to AutoCAD R14.01, through a function called DWGCHECK. This function was supported by an encrypted checksum and product code (called a "watermark" by Autodesk), written into DWG files created by the program.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup><sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> In 2006 Autodesk modified AutoCAD 2007, to include "TrustedDWG technology", a function which would embed a text string within DWG files written by the program: "Autodesk DWG. This file is a Trusted DWG last saved by an Autodesk application or Autodesk licensed application."<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> This helped Autodesk software users ensure that the files they were opening were created by an Autodesk, or RealDWG application, reducing risk of incompatibilities.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> AutoCAD would pop up a message, warning of potential stability problems, if a user opened a 2007 version DWG file which did not include this text string.</p>
<p>In 2008 the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a> asserted the need for an open replacement for the DWG format, as neither RealDWG<sup class="reference" id="cite_ref-autogenerated1_10-1"><a href="#cite_note-autogenerated1-10">[10]</a></sup> nor <a href="/wiki/Open_Design_Alliance" title="Open Design Alliance">DWGdirect</a> are <a href="/wiki/Software_license" title="Software license">licensed</a> on terms that are <a href="/wiki/License_compatibility" title="License compatibility">compatible</a> with <a class="mw-redirect" href="/wiki/Free_software_licenses" title="Free software licenses">free software license</a> like the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU GPL</a>. Therefore, the FSF placed the goal 'Replacement for OpenDWG libraries' in 10th place on their High Priority Free Software Projects list.<sup class="reference" id="cite_ref-opendwgreplacement_17-0"><a href="#cite_note-opendwgreplacement-17">[17]</a></sup> Created in late 2009, <a href="/wiki/GNU_Project" title="GNU Project">GNU</a> <a href="/wiki/LibreDWG" title="LibreDWG">LibreDWG</a><sup class="reference" id="cite_ref-GNU_LibreDWG_18-0"><a href="#cite_note-GNU_LibreDWG-18">[18]</a></sup> is a <a href="/wiki/Free_software" title="Free software">free software</a> library released under the terms of the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU GPLv3</a> license. Despite being currently under development, it can read most parts of DWG files from version R13 up to 2004.</p>
<p>Also in 2008 Autodesk and <a href="/wiki/Bentley_Systems" title="Bentley Systems">Bentley Systems</a> agreed on exchange of software libraries, including Autodesk RealDWG, to improve the ability to read and write the companies' respective DWG and DGN formats in mixed environments with greater fidelity. In addition, the two companies will facilitate work process interoperability between their AEC applications through supporting the reciprocal use of available <a class="mw-redirect" href="/wiki/Application_Programming_Interface" title="Application Programming Interface">Application Programming Interfaces</a> (APIs).<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup></p>
<h3><span class="mw-headline" id="Autodesk_trademark">Autodesk trademark</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.dwg&amp;action=edit&amp;section=3" title="Edit section: Autodesk trademark">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>On 13 November 2006, Autodesk sued the <a href="/wiki/Open_Design_Alliance" title="Open Design Alliance">Open Design Alliance</a> alleging that its DWGdirect libraries infringed Autodesk's trademark for the word "Autodesk", by writing the TrustedDWG watermark (including the word "AutoCAD") into DWG files it created.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> Nine days later, Autodesk's attorneys won a broad and deep <a class="mw-redirect" href="/wiki/Temporary_restraining_order" title="Temporary restraining order">temporary restraining order</a> against the Open Design Alliance.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> In April 2007, the suit was settled, essentially on Autodesk's terms, with Autodesk modifying the warning message in AutoCAD 2008 (to make it somewhat less alarming), and the Open Design Alliance removing support for writing the TrustedDWG watermark from its DWGdirect libraries. The effect of the temporary restraining order and subsequent consent decree was to render the Open Design Alliance's DWGdirect libraries, from one point of view, incapable of creating DWG files that are 100% compatible with AutoCAD. Others point out that the failure of "100% compatibility" means only that loading such a drawing triggers an essentially irrelevant warning message when the file is opened in AutoCAD.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup></p>
<p>In 2006, Autodesk applied for registration of US <a href="/wiki/Trademark" title="Trademark">trademarks</a> on "DWG",<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup><sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup> "DWG EXTREME",<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> "DWG TRUECONVERT",<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup> "REALDWG",<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup> "DWGX",<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup> "DWG TRUEVIEW".<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup><sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup> As early as 1996, Autodesk has disclaimed exclusive use of the DWG mark in US trademark filings.<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup> Out of these applications, only TRUSTEDDWG has been registered as a trademark by the <a class="mw-redirect" href="/wiki/USPTO" title="USPTO">USPTO</a>. The REALDWG and DWGX registrations were opposed by SolidWorks. The DWG EXTREME, DWG TRUECONVERT, and DWG TRUEVIEW trademark registration applications all received substantial resistance, with the USPTO examining attorney requiring Autodesk to disclaim exclusive use of DWG as a condition for their registration.</p>
<p>In a non-final action in May, 2007, the USPTO examining attorney refused to register the two DWG marks, as they are "merely descriptive" of the use of DWG as a file format name. In September, 2007, Autodesk responded, claiming that DWG has gained a "secondary meaning," separate from its use as a generic file format name.<sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup></p>
<p>As of June 22, 2008, all of Autodesk's DWG-related trademark registration proceedings were suspended by the USPTO, pending disposition of trademark opposition and cancellation petitions Autodesk had filed against the Open Design Alliance and Dassault Systèmes <a href="/wiki/SolidWorks" title="SolidWorks">SolidWorks</a> Corporation. The USPTO office actions notifying Autodesk of this noted that Autodesk was not the exclusive source of files with the format name DWG, and Autodesk does not control the use of DWG by others, either as a trademark or as a file format name, among other points.</p>
<p>In 2006, Autodesk filed an opposition with the USPTO to the trademark registration of DWGGATEWAY by <a href="/wiki/SolidWorks" title="SolidWorks">SolidWorks</a>.<sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup> Autodesk subsequently filed a petition for cancellation of SolidWorks' trademark registration for DWGEDITOR.<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup> In both cases, Autodesk's basis was that they had "been using the DWG name with its CAD software products since at least as early as 1983." The opposition and cancellation actions were consolidated, and suspended pending disposition of Autodesk's <a class="mw-redirect" href="/wiki/US_District_Court" title="US District Court">US District Court</a> suit against SolidWorks.<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup></p>
<p>In early 2007, Autodesk petitioned the USPTO to cancel the Open Design Alliance's "OpenDWG" trademarks, claiming that they had been abandoned.<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup> This cancellation action was suspended pending disposition of Autodesk's US District Court suit against SolidWorks.<sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup></p>
<p>In 2008, Autodesk sued SolidWorks in US District Court, arguing that through its marketing efforts, the term "DWG" has lost its original generic meaning and taken on a secondary meaning referring specifically to Autodesk's proprietary drawing file format, and therefore any use of "DWG" in competitive products amounted to trademark infringement.<sup class="reference" id="cite_ref-38"><a href="#cite_note-38">[38]</a></sup> In January, 2010, on the morning that trial was scheduled to begin, Autodesk and SolidWorks settled the suit, with SolidWorks acknowledging Autodesk's trademark rights for DWG, surrendering its trademark registrations for its DWG related projects, and withdrawing its opposition to Autodesk's DWG-related trademark registrations.<sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[39]</a></sup></p>
<p>In April, 2010, Autodesk and the Open Design Alliance settled their suit, with the Open Design Alliance agreeing to cancel its DWG-based trademark registrations and cease use of DWG and DWG-based trademarks in its product marketing and branding.<sup class="reference" id="cite_ref-40"><a href="#cite_note-40">[40]</a></sup> Because there was no adjudication in either case, the agreements between the parties are not binding upon the USPTO. In March, 2010, the Office of the Deputy Commissioner for Trademark Examination Policy at the USPTO determined that evidence submitted by the Open Design Alliance two years earlier was relevant and supported a reasonable ground for refusal to register DWG as a trademark.<sup class="reference" id="cite_ref-41"><a href="#cite_note-41">[41]</a></sup></p>
<p>In June, 2011 the <a class="mw-redirect" href="/wiki/USPTO" title="USPTO">USPTO</a> issued a final refusal<sup class="reference" id="cite_ref-42"><a href="#cite_note-42">[42]</a></sup> to register DWG as a trademark owned by Autodesk. They were quoted as saying:<sup class="reference" id="cite_ref-WorldCAD_Access_Blog_43-0"><a href="#cite_note-WorldCAD_Access_Blog-43">[43]</a></sup></p>
<blockquote class="templatequote">
<p>DWG is merely descriptive of applicant's goods under Section 2(e)(1) of the Trademark Act for two reasons: (1) DWG is a recognized abbreviation for “drawing," and (2) .dwg is a file format used for computer-aided design (CAD) drawings made both with applicant's CAD software and others' CAD software.</p>
</blockquote>
<p>Autodesk appealed the decision. The <a class="mw-redirect" href="/wiki/USPTO" title="USPTO">USPTO</a> affirmed in 2013 their refusal to recognise DWG as a trademark.<sup class="reference" id="cite_ref-44"><a href="#cite_note-44">[44]</a></sup> Despite this, Autodesk websites still claimed DWG as a trademark after the decision.<sup class="reference" id="cite_ref-45"><a href="#cite_note-45">[45]</a></sup></p>
<p>In late 2014 Autodesk again lost, this time at the United States District Court for the Eastern District of Virginia. The judge dismissed all their arguments.<sup class="reference" id="cite_ref-46"><a href="#cite_note-46">[46]</a></sup></p>
<p>In 2015 Autodesk's website has a section title <i>About DWG</i><sup class="reference" id="cite_ref-47"><a href="#cite_note-47">[47]</a></sup> in which they try to make a distinction between .dwg as a file format and the <i>DWG technology environment</i>.</p>
<h2><span class="mw-headline" id="DWG_Support_in_Freemium_and_Free_Software">DWG Support in Freemium and Free Software</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.dwg&amp;action=edit&amp;section=4" title="Edit section: DWG Support in Freemium and Free Software">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>As neither RealDWG<sup class="reference" id="cite_ref-autogenerated1_10-2"><a href="#cite_note-autogenerated1-10">[10]</a></sup> nor DWGdirect are licensed on terms that are compatible with <a class="mw-redirect" href="/wiki/Free_software_licenses" title="Free software licenses">free software licenses</a> like the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU GPL</a>, in 2008 the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a> asserted the need for an open replacement for the DWG format. Therefore, the FSF placed the goal 'Replacement for OpenDWG libraries'<sup class="reference" id="cite_ref-opendwgreplacement_17-1"><a href="#cite_note-opendwgreplacement-17">[17]</a></sup> in 10th place on their High Priority Free Software Projects list.<sup class="reference" id="cite_ref-phoronix1_48-0"><a href="#cite_note-phoronix1-48">[48]</a></sup> Forked in late 2009 from libDWG as <a href="/wiki/GNU_Project" title="GNU Project">GNU</a> <a href="/wiki/LibreDWG" title="LibreDWG">LibreDWG</a> project<sup class="reference" id="cite_ref-GNU_LibreDWG_18-1"><a href="#cite_note-GNU_LibreDWG-18">[18]</a></sup> it can read most parts of DWG files from version R13 up to 2004. But as the libreDWG library is released under the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU GPLv3</a> it can't be used by most targeted <a class="mw-redirect" href="/wiki/Free_and_open_source_software" title="Free and open source software">FOSS</a> graphic software, like <a href="/wiki/FreeCAD" title="FreeCAD">FreeCAD</a>, <a href="/wiki/LibreCAD" title="LibreCAD">LibreCAD</a> and <a href="/wiki/Blender_(software)" title="Blender (software)">Blender</a>, due to a GPLv2/GPLv3 <a href="/wiki/License_compatibility" title="License compatibility">license incompatibility</a>.<sup class="reference" id="cite_ref-gpl2gpl3comp_49-0"><a href="#cite_note-gpl2gpl3comp-49">[49]</a></sup><sup class="reference" id="cite_ref-libredwg-gplv3-opinion_50-0"><a href="#cite_note-libredwg-gplv3-opinion-50">[50]</a></sup><sup class="reference" id="cite_ref-norelicensing_51-0"><a href="#cite_note-norelicensing-51">[51]</a></sup> Due to this struggles in September 2013, the original project LibDWG <a href="/wiki/Fork_(software_development)" title="Fork (software development)">re-forked</a> its code from LibreDWG.<sup class="reference" id="cite_ref-52"><a href="#cite_note-52">[52]</a></sup> A GPLv2 licensed alternative is the libdxfrw project, which can read simple DWGs.<sup class="reference" id="cite_ref-53"><a href="#cite_note-53">[53]</a></sup></p>
<p><a href="/wiki/FreeCAD" title="FreeCAD">FreeCAD</a> is a <a class="mw-redirect" href="/wiki/Free_and_open-source" title="Free and open-source">free and open-source</a> application that can work with the DWG files by utilizing the proprietary <a class="external text" href="https://www.opendesign.com/guestfiles/TeighaFileConverter" rel="nofollow">Teigha file converter</a> for .dwg and .dxf files from the <a href="/wiki/Open_Design_Alliance" title="Open Design Alliance">Open Design Alliance</a>.<sup class="reference" id="cite_ref-54"><a href="#cite_note-54">[54]</a></sup></p>
<p><a href="/wiki/LibreCAD" title="LibreCAD">LibreCAD</a> is a <a class="mw-redirect" href="/wiki/Free_and_open-source" title="Free and open-source">free and open-source</a> 2D CAD application that can open DWG &amp; DXF files using your own library.</p>
<p><a class="external text" href="https://www.opendesign.com/guestfiles/teigha_viewer" rel="nofollow">Teigha Viewer</a> is a freeware stand-alone viewer for .dwg and .dgn files built on the Teigha development platform from the <a href="/wiki/Open_Design_Alliance" title="Open Design Alliance">Open Design Alliance</a>. It runs on Windows, Linux, MacOS and Android operating systems.</p>
<p>Autodesk DWG TrueView is a freeware stand-alone DWG viewer with DWG TrueConvert software included, built on the same viewing engine as AutoCAD software. The freeware Autodesk Design Review software adds a possibility to open DWG files in Design Review to take advantage of measure and markup capabilities, sheet set organization, and status tracking.</p>
<p><a class="new" href="/w/index.php?title=DraftSight&amp;action=edit&amp;redlink=1" title="DraftSight (page does not exist)">DraftSight</a> is a <a href="/wiki/Freemium" title="Freemium">freemium</a> CAD software from <a href="/wiki/Dassault_Syst%C3%A8mes" title="Dassault Systèmes">Dassault Systèmes</a> that lets users create, edit and view DWG files.<sup class="reference" id="cite_ref-55"><a href="#cite_note-55">[55]</a></sup> It runs on <a href="/wiki/Linux" title="Linux">Linux</a>, <a href="/wiki/MacOS" title="MacOS">Mac</a> and <a class="mw-redirect" href="/wiki/Windows" title="Windows">Windows</a> operating systems.<sup class="reference" id="cite_ref-56"><a href="#cite_note-56">[56]</a></sup></p>
<p>DWG files can be displayed online by <a href="/wiki/ShareCAD" title="ShareCAD">ShareCAD</a>, a free online viewer. This service also offers a free IFrame plug-in for viewing DWG files at a site.</p>
