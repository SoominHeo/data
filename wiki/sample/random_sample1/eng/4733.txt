<p><b>Ruby characters</b> <span style="font-weight: normal">(<span class="t_nihongo_kanji" lang="ja">ルビ</span><span class="t_nihongo_comma" style="display:none">,</span> <i>rubi</i><sup class="t_nihongo_help noprint"><a href="/wiki/Help:Installing_Japanese_character_sets" title="Help:Installing Japanese character sets"><span class="t_nihongo_icon" style="color: #00e; font: bold 80% sans-serif; text-decoration: none; padding: 0 .1em;">?</span></a></sup>)</span> are small, annotative <a href="/wiki/Gloss_(annotation)" title="Gloss (annotation)">glosses</a> that can be placed above or to the right of a <a class="mw-redirect" href="/wiki/Chinese_character" title="Chinese character">Chinese character</a> when writing languages with <a href="/wiki/Logogram" title="Logogram">logographic</a> characters such as <a href="/wiki/Chinese_language" title="Chinese language">Chinese</a>, <a href="/wiki/Japanese_language" title="Japanese language">Japanese</a>, or <a href="/wiki/Korean_language" title="Korean language">Korean</a> to show the pronunciation. Typically called just <b>ruby</b> or <b>rubi</b>, such annotations are used as pronunciation guides for characters that are likely to be unfamiliar to the reader.</p>

<p>Most <i>furigana</i> (Japanese ruby characters) are written with the <i><a href="/wiki/Hiragana" title="Hiragana">hiragana</a></i> syllabary, but <i><a href="/wiki/Katakana" title="Katakana">katakana</a></i> and <i><a class="mw-redirect" href="/wiki/Romaji" title="Romaji">romaji</a></i> are also occasionally used. Alternatively, sometimes foreign words (usually English) are printed with furigana implying the meaning, and vice versa. Textbooks usually write <a href="/wiki/Kanji#On.27yomi_.28Sino-Japanese_reading.29" title="Kanji">on-readings</a> with katakana and <a class="mw-redirect" href="/wiki/Kunyomi" title="Kunyomi">kun-readings</a> with hiragana.</p>

<p>In Taiwan, the syllabary used for Chinese ruby characters is <i><a href="/wiki/Bopomofo" title="Bopomofo">Zhuyin fuhao</a></i> (also known as <i>Bopomofo</i>); in mainland China <i><a href="/wiki/Pinyin" title="Pinyin">pinyin</a></i> is used. Typically, unlike the example shown above, zhuyin is used with a vertical traditional writing and zhuyin is written on the right side of the characters. In mainland China, horizontal script is used and ruby characters (pinyin) are written above the Chinese characters.</p>
<p>Books with phonetic guides are popular with children and foreigners learning Chinese (especially pinyin).</p>

<p>Romaja is normally used in foreign textbooks until Hangul is introduced. Ruby characters can be quite common on signs in certain parts of South Korea.</p>
<h2><span class="mw-headline" id="Uses">Uses</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ruby_character&amp;action=edit&amp;section=2" title="Edit section: Uses">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Ruby may be used for different reasons:</p>
<ul>
<li>because the character is rare and the pronunciation unknown to many—personal name characters often fall into this category;</li>
<li>because the character has more than one pronunciation, and the context is insufficient to determine which to use;</li>
<li>because the intended readers of the text are still learning the language and are not expected to always know the pronunciation and/or meaning of a term;</li>
<li>because the author is using a nonstandard pronunciation for a character or a term—for example, comic books often employ ruby to emphasize <a href="/wiki/Dajare" title="Dajare">dajare</a> puns, as in <a class="mw-redirect" href="/wiki/Hana_Yori_Dango" title="Hana Yori Dango">Hana Yori Dango</a> (rather than standard "Dan<i>shi</i>" reading), and show both of the pronunciation and meaning, as in "One Piece" in <a href="/wiki/One_Piece" title="One Piece">One Piece</a> (displayed by ruby character "<i>Wan Piisu</i>" ("One Piece") as the pronunciation and main character "<i>Hitotsunagi no Daihihou</i>" ("The Great Treasure of One Piece") as meaning).</li>
</ul>
<p>Also, ruby may be used to show the meaning, rather than pronunciation, of a possibly-unfamiliar (usually foreign) or slang word. This is generally used with spoken dialogue and applies only to Japanese publications. The most common form of ruby is called <i>furigana</i> or <i>yomigana</i> and is found in Japanese instructional books, newspapers, comics and books for children.</p>
<p>In Japanese, certain characters, such as the <a href="/wiki/Sokuon" title="Sokuon">sokuon</a> <span style="font-weight: normal">(<span class="t_nihongo_kanji" lang="ja">促音</span><span class="t_nihongo_comma" style="display:none">,</span> <i>tsu</i><sup class="t_nihongo_help noprint"><a href="/wiki/Help:Installing_Japanese_character_sets" title="Help:Installing Japanese character sets"><span class="t_nihongo_icon" style="color: #00e; font: bold 80% sans-serif; text-decoration: none; padding: 0 .1em;">?</span></a></sup>, 小さいつ literally <i>"little</i> tsu<i>"</i>)</span> (っ) that indicates a pause before the consonant it precedes, are normally written at about half the size of normal characters. When written as ruby, such characters are usually the same size as other ruby characters. Advancements in technology now allow certain characters to render accurately.<sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (December 2016)">clarification needed</span></a></i>]</sup><sup class="reference" id="cite_ref-w3c-spec_1-0"><a href="#cite_note-w3c-spec-1">[1]</a></sup></p>
<p>In Chinese, the practice of providing phonetic cues via ruby is rare, but does occur systematically in grade-school level text books or dictionaries. The Chinese have no special name for this practice, as it is not as widespread as in Japan. In Taiwan, it is known as "<a href="/wiki/Bopomofo" title="Bopomofo">zhuyin</a>", from the name of the phonetic system employed for this purpose there. It is virtually always used vertically, because publications are normally in a vertical format, and zhuyin is not as easy to read when presented horizontally<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (February 2012)">citation needed</span></a></i>]</sup>. Where zhuyin is not used, other Chinese phonetic systems like <a href="/wiki/Pinyin" title="Pinyin">pinyin</a> are employed.</p>
<p>Sometimes <a class="mw-redirect" href="/wiki/Interlinear_glosses" title="Interlinear glosses">interlinear glosses</a> are visually similar to ruby, appearing above or below the main text in smaller type. However, this is a distinct practice used for helping students of a foreign language by giving glosses for the words in a text, as opposed to the pronunciation of lesser-known characters.</p>
<p>Ruby annotation can also be used in handwriting.</p>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ruby_character&amp;action=edit&amp;section=3" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">See also: <a href="/wiki/Agate_(typography)" title="Agate (typography)">Agate (typography)</a></div>

<p>In British typography, <i>ruby</i> was originally the <a href="/wiki/Point_(typography)#Traditional_point-size_names" title="Point (typography)">name for type</a> with a height of 5.5 points, which printers used for interlinear annotations in printed documents. In Japanese, rather than referring to a font size, the word became the name for typeset <i>furigana</i>. When transliterated back into English, some texts rendered the word as <i>rubi</i>, (a typical <a href="/wiki/Romanization" title="Romanization">romanization</a> of the Japanese word <span lang="ja">ルビ</span>, instead of ルビー (<i>rubī</i>), the expected transliteration of <i>ruby</i>). However, the spelling "ruby" has become more common since the <a class="mw-redirect" href="/wiki/W3C" title="W3C">W3C</a> published a recommendation for <i>ruby <a href="/wiki/Markup_language" title="Markup language">markup</a></i>. In the US, the font size had been called "<a href="/wiki/Agate_(typography)" title="Agate (typography)">agate</a>", a term in use since 1831 according to the <i><a href="/wiki/Oxford_English_Dictionary" title="Oxford English Dictionary">Oxford English Dictionary</a></i>.</p>
<h2><span class="mw-headline" id="HTML_markup">HTML markup</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ruby_character&amp;action=edit&amp;section=4" title="Edit section: HTML markup">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><span id="HTML"></span> This information is out of date and has been superceeded by the inclusion of Ruby support in HTML5. For more information visit <a class="external text" href="https://www.w3.org/International/articles/ruby/markup.en.html" rel="nofollow">w3 ruby markup reference</a>.</p>

<p>In 2001, the W3C published the Ruby Annotation specification<sup class="reference" id="cite_ref-w3c-spec_1-1"><a href="#cite_note-w3c-spec-1">[1]</a></sup> for supplementing <a href="/wiki/XHTML" title="XHTML">XHTML</a> with ruby markup. Ruby markup is not a standard part of HTML 4.01 or any of the XHTML 1.0 specifications (XHTML-1.0-Strict, XHTML-1.0-Transitional, and XHTML-1.0-Frameset), but was incorporated into the XHTML 1.1 specification, and is expected to be a core part of HTML5 once the specification becomes finalised by the W3C.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>Support for ruby markup in web browsers is limited, as XHTML 1.1 is not yet widely implemented. Ruby markup is partially supported by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> (5.0+) for <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a> and <a class="mw-redirect" href="/wiki/Apple_Macintosh" title="Apple Macintosh">Macintosh</a>, supported by Chrome, but is not supported by <a href="/wiki/Konqueror" title="Konqueror">Konqueror</a> or <a href="/wiki/Opera_(web_browser)" title="Opera (web browser)">Opera</a>.<sup class="reference" id="cite_ref-opera_specs_3-0"><a href="#cite_note-opera_specs-3">[3]</a></sup> The WebKit nightly builds added support for Ruby HTML markup in January 2010.<sup class="reference" id="cite_ref-webkit_ruby_4-0"><a href="#cite_note-webkit_ruby-4">[4]</a></sup> <a href="/wiki/Safari_(web_browser)" title="Safari (web browser)">Safari</a> has included support in version 5.0.6.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> It is also supported in Mozilla Firefox as of version 38.<sup class="reference" id="cite_ref-fx38_6-0"><a href="#cite_note-fx38-6">[6]</a></sup></p>
<p>For those browsers that don't support Ruby natively, Ruby support is most easily added by using <a class="mw-redirect" href="/wiki/CSS" title="CSS">CSS</a> rules that are easily found on the web.<sup class="reference" id="cite_ref-ruby_stylesheet_hack_7-0"><a href="#cite_note-ruby_stylesheet_hack-7">[7]</a></sup></p>
<p>Ruby markup support can also be added to some browsers that support custom extensions.</p>
<p>Ruby markup is structured such that a fallback rendering, consisting of the ruby characters in parentheses immediately after the main text, appears if the browser does not support ruby.</p>
<p>The W3C is also working on a specific ruby module for <a href="/wiki/Cascading_Style_Sheets" title="Cascading Style Sheets">CSS</a> level 2, which additionally allows the grouping of ruby and automatic omission of furigana matching their annotated part.<sup class="reference" id="cite_ref-css2_ruby_8-0"><a href="#cite_note-css2_ruby-8">[8]</a></sup> This is currently only supported by Firefox 38.<sup class="reference" id="cite_ref-fx38_6-1"><a href="#cite_note-fx38-6">[6]</a></sup></p>

<h3><span class="mw-headline" id="Complex_ruby_markup">Complex ruby markup</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ruby_character&amp;action=edit&amp;section=6" title="Edit section: Complex ruby markup">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Complex ruby markup makes it possible to associate more than one ruby text with a base text, or parts of ruby text with parts of base text.<sup class="reference" id="cite_ref-complex_ruby_9-0"><a href="#cite_note-complex_ruby-9">[9]</a></sup> It is currently only supported in Firefox 38.<sup class="reference" id="cite_ref-fx38_6-2"><a href="#cite_note-fx38-6">[6]</a></sup></p>
<h2><span class="mw-headline" id="Unicode">Unicode</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ruby_character&amp;action=edit&amp;section=7" title="Edit section: Unicode">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/Unicode" title="Unicode">Unicode</a> and its companion standard, the <a class="mw-redirect" href="/wiki/Universal_Character_Set" title="Universal Character Set">Universal Character Set</a>, support ruby via these <i>interlinear annotation</i> characters:</p>
<ul>
<li>Code point <code>FFF9</code> (<a href="/wiki/Hexadecimal" title="Hexadecimal">hex</a>)—Interlinear annotation anchor—marks start of annotated text</li>
<li>Code point <code>FFFA</code> (hex)—Interlinear annotation separator—marks start of annotating character(s)</li>
<li>Code point <code>FFFB</code> (hex)—Interlinear annotation terminator—marks end of annotated text</li>
</ul>
<p>Few applications implement these characters. Unicode Technical Report #20<sup class="reference" id="cite_ref-unicode20_10-0"><a href="#cite_note-unicode20-10">[10]</a></sup> clarifies that these characters are not intended to be exposed to users of markup languages and software applications. It suggests that ruby markup be used instead, where appropriate.</p>
<h2><span class="mw-headline" id="ANSI">ANSI</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ruby_character&amp;action=edit&amp;section=8" title="Edit section: ANSI">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a class="mw-redirect" href="/wiki/ISO/IEC_6429" title="ISO/IEC 6429">ISO/IEC 6429</a> (also known as <a href="/wiki/Ecma_International" title="Ecma International">ECMA</a>-48) which defines the <a href="/wiki/ANSI_escape_code" title="ANSI escape code">ANSI escape codes</a> also provided a mechanism for ruby text for use by text terminals, although few terminals and terminal emulators implement it. The PARALLEL TEXTS (PTX) escape code accepted six parameter values giving the following escape sequences for marking ruby text:</p>
<ul>
<li><code><a class="mw-redirect" href="/wiki/Control_Sequence_Introducer" title="Control Sequence Introducer">CSI</a> 0 \</code> (or simply <code>CSI \</code> since 0 is used as the default value for this control) — end of parallel texts</li>
<li><code>CSI 1 \</code> — beginning of a string of principal parallel text</li>
<li><code>CSI 2 \</code> — beginning of a string of supplementary parallel text</li>
<li><code>CSI 3 \</code> — beginning of a string of supplementary Japanese phonetic annotation</li>
<li><code>CSI 4 \</code> — beginning of a string of supplementary Chinese phonetic annotation</li>
<li><code>CSI 5 \</code> — end of a string of supplementary phonetic annotations</li>
</ul>
