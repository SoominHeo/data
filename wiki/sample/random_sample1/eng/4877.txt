<p><b>Linux Mint</b> is a community-driven <a href="/wiki/Linux_distribution" title="Linux distribution">Linux distribution</a> based on <a href="/wiki/Debian" title="Debian">Debian</a> and <a href="/wiki/Ubuntu_(operating_system)" title="Ubuntu (operating system)">Ubuntu</a> that strives to be a "modern, elegant and comfortable <a href="/wiki/Operating_system" title="Operating system">operating system</a> which is both powerful and <a href="/wiki/Usability" title="Usability">easy to use</a>."<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> Linux Mint provides full <a href="/wiki/Out_of_the_box_(feature)" title="Out of the box (feature)">out-of-the-box</a> multimedia support by including some <a href="/wiki/Proprietary_software" title="Proprietary software">proprietary software</a> and comes <a href="/wiki/Pre-installed_software" title="Pre-installed software">bundled</a> with a variety of <a href="/wiki/Free_and_open-source_software" title="Free and open-source software">free and open-source</a> applications;<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> however, with the release of v18 "Sarah" some previously included proprietary software such as multimedia codecs was no longer included by default. Its motto is "from freedom came elegance."</p>
<p>The project was conceived by Clement Lefebvre and is being actively developed by the Linux Mint Team and community.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_Mint&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Linux Mint started in 2006 with a beta release of Linux Mint 1.0, <a href="/wiki/Code_name" title="Code name">code-named</a> "Ada",<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> based on <a href="/wiki/Kubuntu" title="Kubuntu">Kubuntu</a>. Following its release, Linux Mint 2.0 "Barbara" was the first version to use Ubuntu as its <a href="/wiki/Codebase" title="Codebase">codebase</a>. Linux Mint had few users from these early versions until the release of Linux Mint 3.0, "Cassandra."<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup><sup class="reference" id="cite_ref-allreleases_11-0"><a href="#cite_note-allreleases-11">[11]</a></sup></p>
<p>Linux Mint 2.0 was based on <a class="mw-redirect" href="/wiki/Ubuntu_6.10" title="Ubuntu 6.10">Ubuntu 6.10</a>,<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> using its package repositories and using it as a <a href="/wiki/Codebase" title="Codebase">codebase</a>. From there, Linux Mint followed its own codebase, building each release from its previous one, but continued to use the package repositories from the latest Ubuntu release. This resulted in making the base between the two systems almost identical, guaranteeing full compatibility between the two <a href="/wiki/Linux_distribution" title="Linux distribution">distributions</a> rather than having Mint become a <a href="/wiki/Fork_(software_development)" title="Fork (software development)">fork</a>.</p>
<p>In 2008, Linux Mint adopted the same release cycle as Ubuntu and dropped its minor version number before releasing version 5 "Elyssa." The same year, in an effort to increase the compatibility between the two systems, Linux Mint decided to abandon its code-base and changed the way it built its releases. Starting with Linux Mint 6 "Felicia," each release was now completely based on the latest Ubuntu release, built directly from it, and timed for approximately one month after the corresponding Ubuntu release, usually in May or November.</p>
<p>In 2010, Linux Mint released Linux Mint Debian Edition. Unlike the other Ubuntu-based editions, LMDE was originally a <a href="/wiki/Rolling_release" title="Rolling release">rolling release</a> based directly on <a href="/wiki/Debian" title="Debian">Debian</a> and was not tied to Ubuntu packages or its release schedule.<sup class="reference" id="cite_ref-allreleases_11-1"><a href="#cite_note-allreleases-11">[11]</a></sup> It was announced on May 27, 2015 that the Linux Mint team would no longer support the original rolling release version of LMDE after January 1, 2016.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> LMDE 2 "Betsy," the current release of LMDE, is a long term support release based on Debian <a href="/wiki/Debian#History" title="Debian">Jessie</a>.<sup class="reference" id="cite_ref-linuxmint.com_14-0"><a href="#cite_note-linuxmint.com-14">[14]</a></sup> When LMDE 2 was released it was announced that all LMDE users would be automatically upgraded to new versions of MintTools software and new Desktop Environments before they were released into the main edition of Linux Mint.<sup class="reference" id="cite_ref-linuxmint.com1_15-0"><a href="#cite_note-linuxmint.com1-15">[15]</a></sup></p>
<p>On 20 February 2016, the Linux Mint website was breached by unknown hackers, who briefly replaced download links for a version of Linux Mint with a modified version that contained malware. The hackers also breached the database of the website's user forum.<sup class="reference" id="cite_ref-ars_16-0"><a href="#cite_note-ars-16">[16]</a></sup><sup class="reference" id="cite_ref-pcw-lmhack_17-0"><a href="#cite_note-pcw-lmhack-17">[17]</a></sup> After the attack Linux Mint was severely criticized as being a distribution that, unlike many others at the time, "just worked" and became popular with non-technical users, but at the expense of security, with some security updates to the underlying Ubuntu or Debian blacklisted from running due to compatibility issues.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup><sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup></p>
<h2><span class="mw-headline" id="Releases">Releases</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_Mint&amp;action=edit&amp;section=2" title="Edit section: Releases">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/Linux_Mint_version_history" title="Linux Mint version history">Linux Mint version history</a></div>
<p>Every version of Linux Mint is given a version number and, until the 18.x series, was code-named with a feminine first name ending in "a" and beginning with a letter of the alphabet that increases with every iteration.<sup class="reference" id="cite_ref-allreleases_11-2"><a href="#cite_note-allreleases-11">[11]</a></sup> The 18.x series broke from the pattern with version 18 having the name "Sarah".<sup class="reference" id="cite_ref-:0_20-0"><a href="#cite_note-:0-20">[20]</a></sup></p>
<p>Initially, there were two Linux Mint releases per year. Following the release of Linux Mint 5 in 2008, every fourth release was labeled a <a href="/wiki/Long-term_support" title="Long-term support">long-term support</a> (LTS) version,<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2015)">citation needed</span></a></i>]</sup> indicating that it was supported (with updates) for longer than traditional releases. Versions 5 and 9 had three years of support, and all LTS versions following received five years of support.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2015)">citation needed</span></a></i>]</sup></p>
<p>On May 31, 2014, with the release of Linux Mint 17,<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> the Linux Mint team adopted a new release strategy. Starting with the release of Mint 17, all future versions were planned to use a LTS version of Ubuntu as a base, until 2016.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup> Under this strategy, Mint 17.1 was released on November 29, 2014,<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup> Mint 17.2 was released on June 30, 2015,<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup> and Mint 17.3 was released on December 4, 2015.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> The 17.x releases are intended to be an easy, optional upgrade.<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup> All three versions included upgrades to the Cinnamon and MATE Desktop Environments and various Mint tools. In addition, Mint 17.2 and 17.3 included an upgrade to the <a href="/wiki/LibreOffice" title="LibreOffice">LibreOffice</a> suite.<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup><sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup> The 18.x series will follow the pattern set by the 17.x series, by using Ubuntu 16.04 LTS as a base.<sup class="reference" id="cite_ref-:0_20-1"><a href="#cite_note-:0-20">[20]</a></sup></p>
<p>Linux Mint does not communicate specific release dates as new versions are published "when ready," meaning that they can be released early when the distribution is ahead of schedule or late when critical bugs are found.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> New releases are announced, with much other material, on the Linux Mint blog.<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup></p>
<p>Reviews of Linux Mint 18, Sarah, have been somewhat mixed, with several that were quite favorable<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup><sup class="reference" id="cite_ref-SCOTT_GILBERTSON_32-0"><a href="#cite_note-SCOTT_GILBERTSON-32">[32]</a></sup><sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup><sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup> and others critical of several specific new problems, with multiple reviews complaining about the lack of multimedia/codec support by default.<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup><sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup> Multimedia codecs, that were previously in the standard Mint distribution, were no longer included in Sarah, but could be loaded with a graphical application that one <i><a href="/wiki/Ars_Technica" title="Ars Technica">Ars Technica</a></i> reviewer said should be obvious for new users.<sup class="reference" id="cite_ref-SCOTT_GILBERTSON_32-1"><a href="#cite_note-SCOTT_GILBERTSON-32">[32]</a></sup></p>
<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_Mint&amp;action=edit&amp;section=3" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Linux Mint primarily uses <a href="/wiki/Free_and_open-source_software" title="Free and open-source software">free and open-source software</a>, making exceptions for some <a href="/wiki/Proprietary_software" title="Proprietary software">proprietary</a> software, such as <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plug-ins</a> and <a class="mw-redirect" href="/wiki/Codecs" title="Codecs">codecs</a> that provide <a href="/wiki/Adobe_Flash" title="Adobe Flash">Adobe Flash</a>, <a href="/wiki/MP3" title="MP3">MP3</a>, and <a href="/wiki/DVD" title="DVD">DVD</a> playback.<sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup><sup class="reference" id="cite_ref-computerw_38-0"><a href="#cite_note-computerw-38">[38]</a></sup><sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[39]</a></sup> Linux Mint's inclusion of proprietary software is unusual; many <a class="mw-redirect" href="/wiki/Linux_distributions" title="Linux distributions">Linux distributions</a> do not include proprietary software by default, as a common goal for some Linux distributions is to adhere to the model of free and open-source software.</p>
<p>Linux Mint comes with a wide range of software installed that includes <a href="/wiki/LibreOffice" title="LibreOffice">LibreOffice</a>, <a class="mw-redirect" href="/wiki/Mozilla_Firefox" title="Mozilla Firefox">Firefox</a>, <a href="/wiki/Mozilla_Thunderbird" title="Mozilla Thunderbird">Thunderbird</a>, <a class="mw-redirect" href="/wiki/Xchat" title="Xchat">HexChat</a>, <a href="/wiki/Pidgin_(software)" title="Pidgin (software)">Pidgin</a>, <a href="/wiki/Transmission_(BitTorrent_client)" title="Transmission (BitTorrent client)">Transmission</a>, <a href="/wiki/VLC_media_player" title="VLC media player">VLC media player</a> and <a href="/wiki/GIMP" title="GIMP">GIMP</a>. Additional software that is not installed by default can be downloaded using the package manager. Linux Mint allows <a class="mw-redirect" href="/wiki/TCP_and_UDP_port" title="TCP and UDP port">networking ports</a> to be closed using its <a href="/wiki/Firewall_(computing)" title="Firewall (computing)">firewall</a>, with customized port selection available. The default Linux Mint <a href="/wiki/Desktop_environment" title="Desktop environment">desktop environments</a>, <a href="/wiki/MATE_(software)" title="MATE (software)">MATE</a> and <a href="/wiki/Cinnamon_(software)" title="Cinnamon (software)">Cinnamon</a>, support many languages.<sup class="reference" id="cite_ref-40"><a href="#cite_note-40">[40]</a></sup><sup class="reference" id="cite_ref-41"><a href="#cite_note-41">[41]</a></sup> Linux Mint can also run many programs designed for <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> (such as <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a>), using the <a href="/wiki/Wine_(software)" title="Wine (software)">Wine</a> Windows <a href="/wiki/Compatibility_layer" title="Compatibility layer">compatibility layer</a> software for Linux, or <a href="/wiki/Virtualization" title="Virtualization">virtualization</a> software, including <a href="/wiki/VMware_Workstation" title="VMware Workstation">VMware Workstation</a> and <a href="/wiki/VirtualBox" title="VirtualBox">VirtualBox</a>, or <a href="/wiki/Kernel-based_Virtual_Machine" title="Kernel-based Virtual Machine">KVM</a> (Kernel based Virtual Machine, built into the Linux kernel) hypervisor using Virtual Machine Manager.</p>
<p>Linux Mint is available with a number of <a href="/wiki/Desktop_environment" title="Desktop environment">desktop environments</a> to choose from, including the default <a href="/wiki/Cinnamon_(software)" title="Cinnamon (software)">Cinnamon</a> desktop, <a href="/wiki/MATE_(software)" title="MATE (software)">MATE</a>, <a href="/wiki/KDE" title="KDE">KDE</a>, and <a href="/wiki/Xfce" title="Xfce">Xfce</a>. Other desktop environments can be installed via <a href="/wiki/Advanced_Packaging_Tool" title="Advanced Packaging Tool">APT</a>, Synaptic, or via the custom Mint Software Manager.</p>
<p>Linux Mint actively develops software for its operating system. Most of the development is done in Python and the source code is available on GitHub.<sup class="reference" id="cite_ref-42"><a href="#cite_note-42">[42]</a></sup></p>

<li><b>Desktop Settings (mintDesktop)</b>: A tool for configuration of the desktop.</li>
<li><b>Welcome screen (mintWelcome)</b>: Introduced in Linux Mint 7, an application that starts on the first login of any new account. It provides links to the Linux Mint website, user guide and community website.</li>

<h3><span class="mw-headline" id="Installation">Installation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_Mint&amp;action=edit&amp;section=5" title="Edit section: Installation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Linux Mint can be booted and run from a <a href="/wiki/USB_flash_drive" title="USB flash drive">USB flash drive</a> on any PC capable of booting from a USB drive, with the option of saving settings to the flash drive. A <a class="mw-redirect" href="/wiki/Ubuntu_Live_USB_creator" title="Ubuntu Live USB creator">USB creator</a> program is available to install an Ubuntu (not LMDE) Live Linux Mint on a USB drive.</p>
<p>The Windows installer "Mint4Win" allows Linux Mint to be installed from within Microsoft Windows, much like the <a class="mw-redirect" href="/wiki/Wubi_(Ubuntu_installer)" title="Wubi (Ubuntu installer)">Wubi</a> installer for Ubuntu. The operating system could then be removed, as with other Windows software, using the <a href="/wiki/Control_Panel_(Windows)" title="Control Panel (Windows)">Windows Control Panel</a>. This method requires no <a class="mw-redirect" href="/wiki/Disk_partition" title="Disk partition">partitioning</a> of the <a class="mw-redirect" href="/wiki/Hard_drive" title="Hard drive">hard drive</a>. It is only useful for Windows users, and is not meant for permanent installations because it incurs a slight performance loss. This installer was included on the Live DVD until Linux Mint 16, but removed in the Linux Mint 16 "Petra" release because the size of the Live DVD images would have exceeded what the software could reliably handle.</p>
<p>Installation supports a <a href="/wiki/Logical_Volume_Manager_(Linux)" title="Logical Volume Manager (Linux)">Logical Volume Manager</a> (LVM) with automatic partitioning only, and <a href="/wiki/Disk_encryption" title="Disk encryption">disk encryption</a> since Linux Mint 15. <a href="/wiki/UTF-8" title="UTF-8">UTF-8</a>, the default <a href="/wiki/Character_encoding" title="Character encoding">character encoding</a>, supports a variety of non-<a href="/wiki/Roman_type" title="Roman type">Roman</a> <a href="/wiki/Script_(Unicode)" title="Script (Unicode)">scripts</a>.</p>
<h2><span class="mw-headline" id="Editions">Editions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_Mint&amp;action=edit&amp;section=6" title="Edit section: Editions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Linux Mint has multiple editions that are based upon <a href="/wiki/Ubuntu_(operating_system)" title="Ubuntu (operating system)">Ubuntu</a>, with various <a href="/wiki/Desktop_environment" title="Desktop environment">desktop environments</a> available. Linux Mint also has an edition based upon <a href="/wiki/Debian" title="Debian">Debian</a>.</p>
<h3><span class="mw-headline" id="Ubuntu-based_editions">Ubuntu-based editions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_Mint&amp;action=edit&amp;section=7" title="Edit section: Ubuntu-based editions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>As of Linux Mint 13, there are two main editions of Linux Mint developed by the core development team and using <a href="/wiki/Ubuntu_(operating_system)" title="Ubuntu (operating system)">Ubuntu</a> as a base. One includes Linux Mint's own <a href="/wiki/Cinnamon_(software)" title="Cinnamon (software)">Cinnamon</a> as the desktop environment while the other uses <a href="/wiki/MATE_(software)" title="MATE (software)">MATE</a>. Linux Mint also develops editions that feature the <a href="/wiki/KDE" title="KDE">KDE</a> and <a href="/wiki/Xfce" title="Xfce">Xfce</a> desktop environments by default, but these have secondary priority and are generally released somewhat later than the two main editions.<sup class="reference" id="cite_ref-download-mint_4-1"><a href="#cite_note-download-mint-4">[4]</a></sup></p>
<p>Older releases, now obsolete, included editions that featured <a href="/wiki/GNOME" title="GNOME">GNOME</a>, <a href="/wiki/LXDE" title="LXDE">LXDE</a>, and <a href="/wiki/Fluxbox" title="Fluxbox">Fluxbox</a> desktop environments by default.</p>

<p>The distribution provided an <a href="/wiki/Original_equipment_manufacturer" title="Original equipment manufacturer">OEM</a> version for manufacturers to use;<sup class="reference" id="cite_ref-45"><a href="#cite_note-45">[45]</a></sup><sup class="reference" id="cite_ref-46"><a href="#cite_note-46">[46]</a></sup> however, this version was discontinued with the release of v18 Sarah in order to reduce the amount of <a href="/wiki/ISO_image" title="ISO image">ISO images</a> that needed to be maintained.<sup class="reference" id="cite_ref-47"><a href="#cite_note-47">[47]</a></sup> Manufacturers wanting to perform an OEM install now have the option to choose so in the live CD boot menu.</p>
<h4><span class="mw-headline" id="No_Codecs_version">No Codecs version</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_Mint&amp;action=edit&amp;section=9" title="Edit section: No Codecs version">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>The distribution provided a "No Codecs" version for magazines, companies, and distributors in the United States, Japan, and countries where the legislation allows <a href="/wiki/Software_patent" title="Software patent">patents to apply to software</a> and distribution of restricted technologies may require the acquisition of 3rd party licenses;<sup class="reference" id="cite_ref-allreleases_11-3"><a href="#cite_note-allreleases-11">[11]</a></sup><sup class="reference" id="cite_ref-48"><a href="#cite_note-48">[48]</a></sup><sup class="reference" id="cite_ref-49"><a href="#cite_note-49">[49]</a></sup> however, this version was discontinued with release of v18 Sarah. Users now have the option whether or not to install multimedia codecs during the installation; additionally, multimedia codecs can also be installed via a link on the Mint Welcome Screen any time after the installation.</p>
<h3><span class="mw-headline" id="Debian-based_edition">Debian-based edition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_Mint&amp;action=edit&amp;section=10" title="Edit section: Debian-based edition">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/Debian" title="Debian">Debian</a>-based edition (<a class="external text" href="https://www.linuxmint.com/download_lmde.php" rel="nofollow"><i>LMDE</i></a>) uses Debian as the package base rather than <a href="/wiki/Ubuntu_(operating_system)" title="Ubuntu (operating system)">Ubuntu</a>.<sup class="reference" id="cite_ref-50"><a href="#cite_note-50">[50]</a></sup> LMDE was originally based directly on <a href="/wiki/Debian#Branches" title="Debian">Debian's Testing branch</a>, but is designed to provide the same functionality and <a href="/wiki/Look_and_feel" title="Look and feel">look and feel</a> as the <a href="/wiki/Linux_Mint#Ubuntu-based_editions" title="Linux Mint">Ubuntu-based editions</a>.<sup class="reference" id="cite_ref-lmde_51-0"><a href="#cite_note-lmde-51">[51]</a></sup> LMDE has its own <a href="/wiki/Debian#Package_management" title="Debian">package repositories</a>.</p>
<p>LMDE claims certain advantages and disadvantages compared to "Mint Main" (i.e., the Ubuntu-based editions):<sup class="reference" id="cite_ref-downloadlmde_52-0"><a href="#cite_note-downloadlmde-52">[52]</a></sup></p>
<ul>
<li>LMDE is faster and more responsive than Ubuntu-based editions.<sup class="reference" id="cite_ref-53"><a href="#cite_note-53">[53]</a></sup></li>
<li>LMDE requires a deeper knowledge and experience with Linux and <a href="/wiki/Debian#Package_management" title="Debian">Debian package management</a>.</li>
<li>Debian is less user-friendly and desktop-ready than Ubuntu, with some rough edges.</li>
</ul>
<h4><span class="mw-headline" id="LMDE_1">LMDE 1</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_Mint&amp;action=edit&amp;section=11" title="Edit section: LMDE 1">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>The original LMDE (now often referred to as <i>LMDE 1</i>) had a semi-<a href="/wiki/Rolling_release" title="Rolling release">rolling release</a> development model, which periodically introduced "Update Packs" (tested <a href="/wiki/Snapshot_(computer_storage)" title="Snapshot (computer storage)">snapshots</a> of Debian Testing).<sup class="reference" id="cite_ref-lmde_51-1"><a href="#cite_note-lmde-51">[51]</a></sup> Installing an Update Pack allowed one to keep LMDE 1 current, without having to reinstall the system every six months as with Mint Main. As of May 17, 2015, it has an upgrade path to LMDE 2.<sup class="reference" id="cite_ref-54"><a href="#cite_note-54">[54]</a></sup></p>

<p>LMDE 2 (a.k.a. <i>Betsy</i>) was released on 10 Apr 2015,<sup class="reference" id="cite_ref-LMDE2_launch_blogpost_55-0"><a href="#cite_note-LMDE2_launch_blogpost-55">[55]</a></sup> becoming the current version of LMDE. LMDE 2 is based on <a href="/wiki/Debian#Branches" title="Debian">Debian Stable</a>, but receives automatic updates to the latest versions of MintTools and the installed <a href="/wiki/Desktop_environment" title="Desktop environment">desktop environment</a> before they are released into the Mint Main edition.<sup class="reference" id="cite_ref-linuxmint.com_14-1"><a href="#cite_note-linuxmint.com-14">[14]</a></sup><sup class="reference" id="cite_ref-linuxmint.com1_15-1"><a href="#cite_note-linuxmint.com1-15">[15]</a></sup> LMDE 2 is available with both the <a href="/wiki/MATE_(software)" title="MATE (software)">MATE</a> and <a href="/wiki/Cinnamon_(software)" title="Cinnamon (software)">Cinnamon</a> desktop environments.<sup class="reference" id="cite_ref-lmde_51-2"><a href="#cite_note-lmde-51">[51]</a></sup></p>
<p>LMDE 2 remains based on <code><a class="mw-redirect" href="/wiki/Sysvinit" title="Sysvinit">sysvinit</a></code> but with a "functional <code>logind</code>" from <code><a href="/wiki/Systemd" title="Systemd">systemd</a></code>.<sup class="reference" id="cite_ref-Clem_comment_1_sysvinit_systemd_56-0"><a href="#cite_note-Clem_comment_1_sysvinit_systemd-56">[56]</a></sup><sup class="reference" id="cite_ref-Clem_comment_2_sysvinit_systemd_57-0"><a href="#cite_note-Clem_comment_2_sysvinit_systemd-57">[57]</a></sup></p>
<h4><span class="mw-headline" id="LMDE_3">LMDE 3</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_Mint&amp;action=edit&amp;section=13" title="Edit section: LMDE 3">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>LMDE 3 is "very likely" to complete the switch to <code>systemd</code> from <code>sysvinit</code>.<sup class="reference" id="cite_ref-Clem_comment_1_sysvinit_systemd_56-1"><a href="#cite_note-Clem_comment_1_sysvinit_systemd-56">[56]</a></sup></p>
<h2><span class="mw-headline" id="Development">Development</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_Mint&amp;action=edit&amp;section=14" title="Edit section: Development">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Individual users and companies using the operating system act as donors,<sup class="reference" id="cite_ref-58"><a href="#cite_note-58">[58]</a></sup> sponsors<sup class="reference" id="cite_ref-59"><a href="#cite_note-59">[59]</a></sup> and partners<sup class="reference" id="cite_ref-60"><a href="#cite_note-60">[60]</a></sup> of the distribution. Linux Mint relies on user feedback to make decisions and orient its development. The official blog often features discussions where users are asked to voice their opinion about the latest features or decisions implemented for upcoming releases. Ideas can be submitted, commented upon and rated by users via the Linux Mint Community Website.<sup class="reference" id="cite_ref-61"><a href="#cite_note-61">[61]</a></sup></p>
<p>The community of Linux Mint users use <a href="/wiki/Launchpad_(website)" title="Launchpad (website)">Launchpad</a> to participate in the translation of the operating system and in reporting bugs.<sup class="reference" id="cite_ref-62"><a href="#cite_note-62">[62]</a></sup></p>
<p>Most extraneous development is done in <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> and organized online using <a href="/wiki/GitHub" title="GitHub">GitHub</a>, making it easy for developers to provide patches, implement additional features, and also fork Linux Mint sub-projects (for example the Linux Mint menu was ported to <a href="/wiki/Fedora_(operating_system)" title="Fedora (operating system)">Fedora</a>). With each release, features are added that are developed by the community. In Linux Mint 9 for instance, the ability to edit menu items is a feature that was contributed by a Linux Mint user.<sup class="reference" id="cite_ref-63"><a href="#cite_note-63">[63]</a></sup></p>
<h2><span class="mw-headline" id="Reception">Reception</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_Mint&amp;action=edit&amp;section=15" title="Edit section: Reception">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>TechRadar has praised Linux Mint for focusing on desktop users.<sup class="reference" id="cite_ref-techradar_64-0"><a href="#cite_note-techradar-64">[64]</a></sup></p>
<p>In a 2012 online poll at <a href="/wiki/Lifehacker" title="Lifehacker">Lifehacker</a>, Linux Mint was voted the second best Linux distribution, after Ubuntu, with almost 16% of the votes.<sup class="reference" id="cite_ref-pcworld_65-0"><a href="#cite_note-pcworld-65">[65]</a></sup> In October 2012 (Issue 162), <a href="/wiki/Linux_Format" title="Linux Format">Linux Format</a> named Linux Mint the best distro for 2012.<sup class="reference" id="cite_ref-66"><a href="#cite_note-66">[66]</a></sup> In July 2013 (Issue 128), <a href="/wiki/Linux_User_and_Developer" title="Linux User and Developer">Linux User &amp; Developer</a> gave Linux Mint 15 "Olivia" a score of 5/5, stating "We haven't found a single problem with the distro… we're more than satisfied with the smooth, user-friendly experience that Linux Mint 15, and Cinnamon 1.8, provides for it to be our main distro for at least another 6 months."<sup class="reference" id="cite_ref-lud-128_67-0"><a href="#cite_note-lud-128-67">[67]</a></sup></p>
<p><a href="/wiki/DistroWatch" title="DistroWatch">DistroWatch</a> provides information and statistics of Linux distributions. Since 2012, DistroWatch's Page Hit Ranking listed Linux Mint as the most viewed profile page there.<sup class="reference" id="cite_ref-68"><a href="#cite_note-68">[68]</a></sup><sup class="reference" id="cite_ref-69"><a href="#cite_note-69">[69]</a></sup><sup class="reference" id="cite_ref-Second_70-0"><a href="#cite_note-Second-70">[70]</a></sup><sup class="reference" id="cite_ref-71"><a href="#cite_note-71">[71]</a></sup> In December 2011, Wikimedia Traffic Analysis Report showed 29 million hits on Wikimedia pages from computers running Ubuntu and 642,000 hits from those running Linux Mint. In June 2015, Wikimedia Traffic Analysis Report shows 6.4 million hits for Linux Mint while the highest-placed Linux distribution, Ubuntu, had 1.2 billion hits.<sup class="reference" id="cite_ref-72"><a href="#cite_note-72">[72]</a></sup> <a href="/wiki/ZDNet" title="ZDNet">ZDNet</a>, <a href="/wiki/Ars_Technica" title="Ars Technica">Ars Technica</a> and OStatic argued that these numbers, based on operating systems actually in use, represent the popularity of Ubuntu and Mint <a href="/wiki/DistroWatch#Inappropriateness_of_the_hit_counters_as_a_measure_of_OS_usage" title="DistroWatch">far more accurately</a> than the DistroWatch rankings.<sup class="reference" id="cite_ref-techradar_64-1"><a href="#cite_note-techradar-64">[64]</a></sup><sup class="reference" id="cite_ref-pcworld_65-1"><a href="#cite_note-pcworld-65">[65]</a></sup><sup class="reference" id="cite_ref-73"><a href="#cite_note-73">[73]</a></sup><sup class="reference" id="cite_ref-74"><a href="#cite_note-74">[74]</a></sup><sup class="reference" id="cite_ref-75"><a href="#cite_note-75">[75]</a></sup><sup class="reference" id="cite_ref-76"><a href="#cite_note-76">[76]</a></sup></p>
<p>The <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">FSF</a> has criticized the Linux Mint operating system for using proprietary software such as Adobe Flash Player and proprietary codecs such as MP3 and MP4:</p>
<blockquote class="templatequote">
<p>Mint does not have a policy against including nonfree software, it includes nonfree binary blobs in drivers packaged with the kernel, and it includes nonfree programs in its repositories. It even includes proprietary codecs.<sup class="reference" id="cite_ref-77"><a href="#cite_note-77">[77]</a></sup></p>
</blockquote>
