<p><b>Linux on z Systems</b> (or <b>Linux on z</b> for short, and previously <b>Linux on System z</b>) is the collective term for the <a href="/wiki/Linux" title="Linux">Linux</a> operating system compiled to run on <a href="/wiki/IBM_mainframe" title="IBM mainframe">IBM mainframes</a>, especially IBM <a class="mw-redirect" href="/wiki/System_z" title="System z">System z</a> and IBM LinuxONE servers. Similar terms which imply the same meaning are <b>Linux on <a class="mw-redirect" href="/wiki/ZEnterprise" title="ZEnterprise">zEnterprise</a></b>, <b>Linux on <a class="mw-redirect" href="/wiki/ZSeries" title="ZSeries">zSeries</a></b>, <b>Linux/390</b>, <b>Linux/390x,</b> etc. The terms <b>zLinux</b> or <b>z/Linux</b> are also sometimes used, but these terms are discouraged by IBM as they create the implication of an IBM-offered or IBM-distributed version of Linux, which is incorrect.</p>
<p>IBM and Canonical promote LinuxONE collaboration running Ubuntu 16.04.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_on_z_Systems&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Linux on z originated as two separate efforts to port Linux to IBM's largest servers. The first effort, the "Bigfoot" project, developed by <a class="new" href="/w/index.php?title=Linas_Vepstas&amp;action=edit&amp;redlink=1" title="Linas Vepstas (page does not exist)">Linas Vepstas</a> in late 1998 through early 1999, was an independent distribution and has since been abandoned.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> IBM published a collection of patches and additions to the Linux 2.2.13 <a class="mw-redirect" href="/wiki/Kernel_(computer_science)" title="Kernel (computer science)">kernel</a> on December 18, 1999, to start today's mainline Linux on z.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> Formal product announcements quickly followed in 2000, including the <a href="/wiki/Integrated_Facility_for_Linux" title="Integrated Facility for Linux">Integrated Facility for Linux</a> (IFL) engines. <a href="/wiki/Think_Blue_Linux" title="Think Blue Linux">Think Blue Linux</a> was an early mainframe distribution consisting mainly of <a href="/wiki/Red_Hat" title="Red Hat">Red Hat</a> packages added to the IBM kernel.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> Commercial Linux distributors introduced mainframe editions very quickly after the initial kernel work.</p>
<p>At the start of IBM's involvement, Linux patches for System z included some <i>object code only</i> (OCO) modules, without source code.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> Soon after IBM replaced the OCO modules with <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> modules. Linux on z is <a href="/wiki/Free_software" title="Free software">free software</a> under the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a>.</p>
<p>According to IBM, by May, 2006, over 1,700 customers were running Linux on their mainframes.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<h2><span class="mw-headline" id="Virtualization">Virtualization</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_on_z_Systems&amp;action=edit&amp;section=2" title="Edit section: Virtualization">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Virtualization is required by default on IBM z Systems; there is no option to run Linux on z without some degree of virtualization. (Only the very first 64-bit mainframe models, the z900 and z800, included a non-virtualized "basic mode.") First layer virtualization is provided by the Processor Resource and System Manager (PR/SM) to deploy one or more Logical Partitions (LPARs). Each LPAR supports a variety of operating systems including Linux on z. A hypervisor called z/VM can also be run as the second layer virtualization in LPARs to create as many virtual machines (VMs) as there are resources assigned to the LPARs to support them. <a href="/wiki/Kernel-based_Virtual_Machine" title="Kernel-based Virtual Machine">KVM</a> on z is another hypervisor option.</p>
<p>When Linux applications in an LPAR access data and applications in other LPARs such as <a href="/wiki/CICS" title="CICS">CICS</a>, <a href="/wiki/IBM_DB2" title="IBM DB2">IBM DB2</a>, <a class="mw-redirect" href="/wiki/Information_Management_System" title="Information Management System">IMS</a>, <a href="/wiki/Linux" title="Linux">Linux</a>, and other mainframe subsystems, running on the same physical mainframe, they can utilize <a href="/wiki/HiperSocket" title="HiperSocket">HiperSockets</a> – fast, memory-only <a class="mw-redirect" href="/wiki/TCP/IP" title="TCP/IP">TCP/IP</a> connections. As compared to TCP/IP over standard <a class="mw-redirect" href="/wiki/Network_interface_card" title="Network interface card">network interface cards</a> (NICs, also known as Open System Adapters or OSAs in mainframes), HiperSockets can improve end-user responsiveness (reduce network latency and processing overhead), security (since there's no network connection to intercept), and reliability (since there's no network connection to lose).<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>With the zEC12, zBC12, and later models, the HiperSocket concept is extended beyond the physical machine boundary via an <a href="/wiki/RDMA_over_Converged_Ethernet" title="RDMA over Converged Ethernet">RDMA over Converged Ethernet</a> (RoCE) adapter to facilitate a secure and high speed inter-system communication. Applications in LPAR A in system A can thus use HiperSockets to communicate with applications in LPAR B in system B to ensure the security and performance attributes.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (August 2014)">citation needed</span></a></i>]</sup></p>
<h2><span class="mw-headline" id="Hardware">Hardware</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_on_z_Systems&amp;action=edit&amp;section=3" title="Edit section: Hardware">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Beginning with Linux kernel version 4.1 released in early 2015, Linux on z is only available as a <a class="mw-redirect" href="/wiki/64-bit" title="64-bit">64-bit</a> operating system compatible with <a href="/wiki/Z/Architecture" title="Z/Architecture">z/Architecture</a> mainframes. Previously Linux on z was also available as a <a href="/wiki/31-bit" title="31-bit">31-bit</a> operating system compatible with older model mainframes introduced prior to 2000's z900 model. However, the newer 64-bit Linux kernel and 64-bit Linux on z distributions are still backward compatible with applications compiled for 31-bit Linux on z. Historically the Linux kernel architecture designations were "s390" and "s390x" to distinguish between the 31-bit and 64-bit Linux on z kernels respectively, but "s390" now also refers generally to the one Linux on z kernel architecture.</p>
<p>Linux runs on standard, general purpose mainframe CPs (Central Processors) as well as IFLs (<a href="/wiki/Integrated_Facility_for_Linux" title="Integrated Facility for Linux">Integrated Facility for Linux</a>). IFLs are mainframe processors dedicated to running Linux, either natively or under a hypervisor (z/VM or KVM on z). <a href="/wiki/Microcode" title="Microcode">Microcode</a> restricts IFLs from running "traditional" workloads, such as <a href="/wiki/Z/OS" title="Z/OS">z/OS</a>, but they are physically identical to other z System processors. IFLs are typically less expensive to acquire from IBM than CPs.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></p>
<h2><span class="mw-headline" id="Advantages">Advantages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_on_z_Systems&amp;action=edit&amp;section=4" title="Edit section: Advantages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Linux on z gives the flexibility of running Linux with the advantages of <a href="/wiki/Mainframe_computer" title="Mainframe computer">fault-tolerant mainframe hardware</a> capable of over 90,000 I/O operations per second<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> and with a <a href="/wiki/Mean_time_between_failures" title="Mean time between failures">mean time between failure (MTBF)</a><sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> measured in decades.<sup class="reference" id="cite_ref-:0_10-0"><a href="#cite_note-:0-10">[10]</a></sup> Using virtualization, numerous smaller servers can be combined onto one mainframe, gaining some benefits of centralization and cost reduction, while still allowing specialized servers. Instead of <a href="/wiki/Paravirtualization" title="Paravirtualization">paravirtualization</a>, IBM mainframes use <a href="/wiki/Full_virtualization" title="Full virtualization">full virtualization</a>, which permits workload density far greater than paravirtualization does. Combining full virtualization of the hardware plus lightweight <a href="/wiki/Z/VM" title="Z/VM">Virtual Machine</a> containers that run Linux in isolation (somewhat similar in concept to <a href="/wiki/Docker_(software)" title="Docker (software)">Docker</a>) result in a platform that supports more virtual servers than any other in a single footprint,<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> which also can lower operating costs. Additional savings can be seen from reduced need for floor space, power, cooling, networking hardware, and the other <a href="/wiki/Data_center_infrastructure_management" title="Data center infrastructure management">infrastructure</a> needed to support a <a href="/wiki/Data_center" title="Data center">data center</a>. IBM mainframes allow transparent use of <a href="/wiki/Mainframe_computer" title="Mainframe computer">redundant processor execution steps</a> and <a href="/wiki/Mainframe_computer" title="Mainframe computer">integrity checking</a>, which is important for critical applications in certain industries such as banking.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="dubious (June 2013)">citation needed</span></a></i>]</sup>. Mainframes typically allow <a href="/wiki/Hot_swapping" title="Hot swapping">hot-swapping of hardware</a>, such as processors and memory. z Systems provides fault tolerance for all key components, including processors, memory, I/O Interconnect, power supply, channel paths, network cards, and others. Through internal monitoring, possible problems are detected and problem components are designed to be switched over without even failing a single transaction.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> In the rare event of failure, firmware will automatically enable a spare component, disable the failing component, and notify IBM to dispatch a service representative. This is transparent to the operating system, allowing routine repairs to be performed without shutting down the system. Industries continue to rely on mainframes because they are genuinely the best option, in terms of reliability, security, and often cost.<sup class="reference" id="cite_ref-:0_10-1"><a href="#cite_note-:0-10">[10]</a></sup></p>
<h2><span class="mw-headline" id="Pricing_and_costs">Pricing and costs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_on_z_Systems&amp;action=edit&amp;section=5" title="Edit section: Pricing and costs">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Linux on z is not generally appropriate on premises for small businesses that would have fewer than about 10 distributed Linux servers, although some expensive per-processor licensed software can quickly reduce that <a href="/wiki/Rule_of_thumb" title="Rule of thumb">rule of thumb</a>. Most software vendors, including IBM, treat the highly virtualized IFLs just like non-virtualized processors on other platforms for licensing purposes. In other words, a single IFL running scores of Linux instances still typically counts as one "ordinary" CPU, at the same CPU price, for software licensing. Test, development, quality assurance, training, and redundant production server instances can all run on one IFL (or more IFLs, but only if needed for peak demand performance capacity). Thus, beyond some minimum threshold, Linux on z can quickly become cost-advantageous when factoring in labor and software costs.</p>
<p>The cost equation for Linux on z is not always well understood and is controversial, and many businesses and governments have difficulty measuring, much less basing decisions on, software, labor, and other costs (such as the costs of outage and security breaches). Acquisition costs are often more visible, and small, non-scalable servers are "cheap." Nonetheless, non-acquisition costs are no less real and are usually far greater than hardware acquisition prices<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="dubious (June 2013)">citation needed</span></a></i>]</sup>. Also, individual users and departments within larger businesses and governments sometimes have difficulty sharing computing infrastructure (or any other resources, for that matter), citing a loss of control. Server centralization, as Linux on z provides, might reward cooperation with better service and lower costs, but that's not to say that cooperation is always easily accomplished within a corporate bureaucracy.</p>
<p>Linux on z also supports less expensive disk storage devices than z/OS because Linux does not require FICON or ESCON attachment, although z/OS may use disk space more efficiently, on balance, due to hardware-assisted database compression common on z/OS and the smaller number of operating system instances z/OS typically requires. There are also some operational advantages using some FICON-attached storage with Linux on z, for example support for z/VM Live Guest Relocation.</p>
<h2><span class="mw-headline" id="Appropriate_workloads">Appropriate workloads</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_on_z_Systems&amp;action=edit&amp;section=6" title="Edit section: Appropriate workloads">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Mainframe characteristics are designed for such business workloads as transaction processing (especially in conjunction with concurrent, high volume batch processing) and large database management. Mainframe design traditionally emphasizes "balanced" performance for all computing elements including input/output, implemented via <a href="/wiki/Channel_I/O" title="Channel I/O">channel I/O</a>. Mainframes offload I/O, system accounting, and other non-core computing tasks from the main CPUs as much as possible, and z/Architecture additionally offloads cryptographic calculations. For example, in a single IBM z13 machine up to 141 processor cores are available to configure as IFLs. However, every such machine also has 27 additional main cores: 2 as spares, 1 for firmware support, and the remainder running system accounting and I/O support tasks. In addition, each I/O adapter typically has two <a href="/wiki/PowerPC" title="PowerPC">PowerPC</a> processors, and a z13 supports hundreds of I/O adapters. There are also separate processors handling memory and cache control tasks, environmental monitoring, and internal interconnections, as examples.</p>
<p>Historically, mainframes in general, and Linux on z in particular, did not execute "CPU-intensive" single task computations with notably high performance compared to certain other platforms with a few notable exceptions such as cryptographic calculations. Examples included most scientific simulations, <a href="/wiki/Weather_forecasting" title="Weather forecasting">weather forecasting</a>, and <a href="/wiki/Computational_chemistry" title="Computational chemistry">molecular modeling</a>. <a class="mw-redirect" href="/wiki/Supercomputers" title="Supercomputers">Supercomputers</a>, including Linux-based supercomputers, excel at these workloads. This bifurcation between mainframes and other platforms has significantly blurred in recent years, starting with the introduction of 2008's System z10, a machine based on quad-core 4.4 GHz processors with hardware decimal floating point. As mainframe processor technology has continued to evolve, and especially with the introduction of the IBM LinuxONE and IBM z13 models in 2015, IBM has started promoting its mainframes as ideal platforms to run real-time analytics and other computationally intensive tasks that mainframes did not historically run well. IBM sold its X86 server product line to <a href="/wiki/Lenovo" title="Lenovo">Lenovo</a> in 2014.</p>
<p>Mainframes do not provide graphics or sound adapters, and are as such ill-suited for <a href="/wiki/Digital_media" title="Digital media">digital media</a> editing or <a href="/wiki/Computer-aided_design" title="Computer-aided design">computer-aided design</a> (CAD) except perhaps in support roles (e.g. content storage, parts inventories, metadata management, security services, etc.)</p>
<h2><span class="mw-headline" id="Support">Support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_on_z_Systems&amp;action=edit&amp;section=7" title="Edit section: Support">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Like all other versions of Linux, Linux on z is governed by the GPL <a href="/wiki/Free_software" title="Free software">free software</a> license. Complete Linux on z <a href="/wiki/Source_code" title="Source code">source code</a> is available from numerous parties on a free and equal basis, and architectural support is part of the main Linux kernel effort. IBM assigns several of its programmers to the community effort, but IBM is by no means the only participant.</p>
<p>Though there are no obstacles to running any Linux on z distribution on an IBM z System, IBM routinely tests three particular Linux on z distributions: <a href="/wiki/Red_Hat_Enterprise_Linux" title="Red Hat Enterprise Linux">Red Hat</a>, <a href="/wiki/SUSE_Linux_Enterprise_Server" title="SUSE Linux Enterprise Server">SUSE</a>,<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> and, starting in 2015, <a href="/wiki/Canonical_(company)" title="Canonical (company)">Canonical</a>'s <a class="mw-redirect" href="/wiki/Ubuntu_Linux" title="Ubuntu Linux">Ubuntu Linux</a>. Other notable Linux on z distributions include <a href="/wiki/Debian" title="Debian">Debian</a>,<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> <a href="/wiki/Fedora_(operating_system)" title="Fedora (operating system)">Fedora</a>,<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> <a href="/wiki/Slackware" title="Slackware">Slackware</a>,<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> <a href="/wiki/CentOS" title="CentOS">CentOS</a>, and <a href="/wiki/Gentoo_Linux" title="Gentoo Linux">Gentoo</a>.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<p>Nearly every free or open-source software package available for Linux generally is available for Linux on z, including <a href="/wiki/Apache_HTTP_Server" title="Apache HTTP Server">Apache HTTP Server</a>, <a class="mw-redirect" href="/wiki/Samba_software" title="Samba software">Samba software</a>, <a class="mw-redirect" href="/wiki/JBoss" title="JBoss">JBoss</a>, <a href="/wiki/PostgreSQL" title="PostgreSQL">PostgreSQL</a>, <a href="/wiki/MySQL" title="MySQL">MySQL</a>, <a href="/wiki/PHP" title="PHP">PHP</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python programming language</a>, <a href="/wiki/Concurrent_Versions_System" title="Concurrent Versions System">Concurrent Versions System</a> (CVS), <a href="/wiki/GNU_Compiler_Collection" title="GNU Compiler Collection">GNU Compiler Collection</a> (GCC), <a href="/wiki/LLVM" title="LLVM">LLVM</a>, and <a href="/wiki/Perl" title="Perl">Perl</a>, <a class="mw-redirect" href="/wiki/Rust_programming_language" title="Rust programming language">Rust</a>,<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> among many others.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup></p>
<p>Red Hat and SUSE offer mainline support for their distributions running Linux on z.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup><sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> In 2015 Canonical announced plans to offer official support for its distribution starting in early 2016. <a href="/wiki/IBM_Global_Services" title="IBM Global Services">IBM Global Services</a> also offers support contracts, including 24x7 coverage.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup> Some standard Linux software applications are readily available pre-compiled, including popular closed-source enterprise software packages such as <a class="mw-redirect" href="/wiki/WebSphere" title="WebSphere">WebSphere</a>,<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup> <a href="/wiki/IBM_DB2" title="IBM DB2">DB2</a><sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup> and <a class="mw-redirect" href="/wiki/Oracle_database" title="Oracle database">Oracle</a><sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> databases and applications, <a href="/wiki/SAP_R/3" title="SAP R/3">SAP R/3</a>, <a href="/wiki/SAP_ERP" title="SAP ERP">SAP ERP</a>,<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup> and IBM's <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> Developer's Kit (JDK),<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup> to name only a few.</p>
<h2><span class="mw-headline" id="Developer_resources">Developer resources</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_on_z_Systems&amp;action=edit&amp;section=8" title="Edit section: Developer resources">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>IBM offers resources to developers wishing to target Linux for z:</p>
<ul>
<li>The Linux Test Drive, a free program granting a single Linux on System z virtual machine for 30 days.<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup></li>
<li>The IBM Systems Application Advantage for Linux (<a class="mw-redirect" href="/wiki/Chiphopper" title="Chiphopper">Chiphopper</a>), a developer program to help developers write and publish cross-platform Linux software.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup></li>
<li>The Community Development System for Linux on System z (CDSL) program, a platform for providing <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> developers a platform for porting to Linux on System z.<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup></li>
<li>The Linux Remote Development Program, a fee-based extended developer support program.<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup></li>
</ul>
<p>Linux on z supports <a href="/wiki/Unicode" title="Unicode">Unicode</a> and <a href="/wiki/ASCII" title="ASCII">ASCII</a> just like any other Linux distribution—it is not an <a href="/wiki/EBCDIC" title="EBCDIC">EBCDIC</a>-based operating system.<sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup> However, for convenience Linux is able to read kernel parameters in EBCDIC. z/VM takes advantage of this capability.</p>
<p>Porting Linux applications to Linux on z is fairly straightforward. Potential issues include <a href="/wiki/Endianness" title="Endianness">endianness</a> (Linux on z is big-endian) and reliance on non-portable libraries particularly if source code is not available.<sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup> Programs can be easily <a href="/wiki/Cross_compiler" title="Cross compiler">cross compiled</a> to z/Architecture binaries on non-mainframe Linux systems.<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup></p>
<h3><span class="mw-headline" id="Emulators">Emulators</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Linux_on_z_Systems&amp;action=edit&amp;section=9" title="Edit section: Emulators">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>There are at least three software-based System z mainframe <a href="/wiki/Emulator" title="Emulator">emulators</a>.</p>
<ul>
<li>FLEX-ES from Fundamental Software is a commercially offered option.<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup></li>
<li>The open source <a class="mw-redirect" href="/wiki/Hercules_emulator" title="Hercules emulator">Hercules emulator</a> supports Linux on System z (and can even run on Linux on System z itself).</li>
<li>In 2010, IBM introduced the Rational Developer for System z Unit Test Feature (now called Rational Development and Test Environment for z, or sometimes RDTz for short) which provides a restricted use execution environment that can run on X86 hardware. IBM's license terms limit use of RDTz to certain application development tasks, not including final pre-production compiling or pre-production testing (such as stress testing). RDTz includes z/OS (with common middleware) and is also compatible with Linux on z.<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup></li>
</ul>
