Four of the Riemann summation <a href="#Methods">methods</a> for approximating the area under curves. <span style="color:#0081cd"><b>Right</b></span> and <span style="color:#fec200"><b>left</b></span> methods make the approximation using the right and left endpoints of each subinterval, respectively. <span style="color:#009246"><b>Maximum</b></span> and <span style="color:#bc1e47"><b>minimum</b></span> methods make the approximation using the largest and smallest endpoint values of each subinterval, respectively. The values of the sums converge as the subintervals halve from top-left to bottom-right.</div>

<p>In <a href="/wiki/Mathematics" title="Mathematics">mathematics</a>, a <b>Riemann sum</b> is an <a href="/wiki/Approximation" title="Approximation">approximation</a> that takes the form <span><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML">
<semantics>
<mrow class="MJX-TeXAtom-ORD">
<mstyle displaystyle="true" scriptlevel="0">
<mo>∑<!-- ∑ --></mo>
<mi>f</mi>
<mo stretchy="false">(</mo>
<mi>x</mi>
<mo stretchy="false">)</mo>
<mi mathvariant="normal">Δ<!-- Δ --></mi>
<mi>x</mi>
</mstyle>
</mrow>
<annotation encoding="application/x-tex">{\displaystyle \sum f(x)\Delta x}</annotation>
</semantics>
</math></span><img alt="\sum f(x)\Delta x" aria-hidden="true" class="mwe-math-fallback-image-inline" src="https://wikimedia.org/api/rest_v1/media/math/render/svg/9c5d867ba20c3f1217f6dc412ff0a337f536c026" style="vertical-align: -1.338ex; width:11.498ex; height:3.843ex;"/></span>. It is named after German mathematician <a href="/wiki/Bernhard_Riemann" title="Bernhard Riemann">Bernhard Riemann</a>. One very common application is approximating the area of functions or lines on a graph, but also the length of curves and other approximations.</p>
<p>The sum is calculated by dividing the region up into shapes (<a href="/wiki/Rectangle" title="Rectangle">rectangles</a>, <a href="/wiki/Trapezoid" title="Trapezoid">trapezoids</a>, <a href="/wiki/Parabola" title="Parabola">parabolas</a>, or <a href="/wiki/Cubic_function" title="Cubic function">cubics</a>) that together form a region that is similar to the region being measured, then calculating the area for each of these shapes, and finally adding all of these small areas together. This approach can be used to find a numerical approximation for a <a class="mw-redirect" href="/wiki/Definite_integral" title="Definite integral">definite integral</a> even if the <a href="/wiki/Fundamental_theorem_of_calculus" title="Fundamental theorem of calculus">fundamental theorem of calculus</a> does not make it easy to find a <a href="/wiki/Closed-form_expression" title="Closed-form expression">closed-form solution</a>.</p>
<p>Because the region filled by the small shapes is usually not exactly the same shape as the region being measured, the Riemann sum will differ from the area being measured. This error can be reduced by dividing up the region more finely, using smaller and smaller shapes. As the shapes get smaller and smaller, the sum approaches the <a href="/wiki/Riemann_integral" title="Riemann integral">Riemann integral</a>.</p>
