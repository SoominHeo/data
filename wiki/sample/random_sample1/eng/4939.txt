<h1 class="firstHeading" id="firstHeading" lang="en">RealVideo</h1>

<p><b>RealVideo</b> is a suite of <a href="/wiki/Proprietary_format" title="Proprietary format">proprietary</a> <a class="mw-redirect" href="/wiki/Video_compression_format" title="Video compression format">video compression formats</a> developed by <a href="/wiki/RealNetworks" title="RealNetworks">RealNetworks</a> – the specific format changes with the version. It was first released in 1997 and as of 2008<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=RealVideo&amp;action=edit">[update]</a></sup> is at version 10.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> RealVideo is supported on many platforms, including Windows, Mac, Linux, Solaris, and several mobile phones.</p>
<p>RealVideo is usually paired with <a href="/wiki/RealAudio" title="RealAudio">RealAudio</a> and packaged in a <a href="/wiki/RealMedia" title="RealMedia">RealMedia</a> (.rm) container. RealMedia is suitable for use as a <a href="/wiki/Streaming_media" title="Streaming media">streaming media</a> format, that is one which is viewed while it is being sent over the network. Streaming video can be used to watch live television, since it does not require downloading the entire video in advance. Compression and decompression software for each version are called "codecs".</p>

<h2><span class="mw-headline" id="Technology">Technology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=RealVideo&amp;action=edit&amp;section=1" title="Edit section: Technology">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The first version of RealVideo was announced in 1997 and was based on the <a href="/wiki/H.263" title="H.263">H.263</a> format. At the time, RealNetworks issued a press release saying they had licensed Iterated Systems' ClearVideo technology and were including it as the RealVideo Fractal Codec.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> However, support for ClearVideo quietly disappeared in the next version of RealVideo.</p>
<p>RealVideo continued to use H.263 until RealVideo 8, when the company switched to a <a href="/wiki/Proprietary_software" title="Proprietary software">proprietary</a> video format. RealVideo codecs are identified by <a href="/wiki/FourCC" title="FourCC">four-character codes</a>. RV10 and RV20 are the H.263-based codecs. RV30 and RV40 are RealNetworks' proprietary codecs. These identifiers have been the source of some confusion, as people may assume that RV10 is RealVideo version 10, when it is actually the first version of RealVideo. RealVideo 10 uses RV40.</p>
<p>RealVideo can be played from a RealMedia file or streamed over the network using the <a href="/wiki/Real_Time_Streaming_Protocol" title="Real Time Streaming Protocol">Real Time Streaming Protocol</a> (RTSP), a standard protocol for streaming media developed by the <a class="mw-redirect" href="/wiki/IETF" title="IETF">IETF</a>. However, RealNetworks uses RTSP only to set up and manage the connection. The actual video data is sent with their own proprietary <a href="/wiki/Real_Data_Transport" title="Real Data Transport">Real Data Transport</a> (RDT) protocol. This tactic has drawn criticism because it made it difficult to use RealVideo with other player and server software. However, the <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> <a href="/wiki/MPlayer" title="MPlayer">MPlayer</a> project has now developed software capable of playing the RDT streams.</p>
<p>To facilitate real-time streaming, RealVideo (and RealAudio) normally uses <a class="mw-redirect" href="/wiki/Constant_bit_rate" title="Constant bit rate">constant bit rate</a> encoding, so that the same amount of data is sent over the network each second. Recently, RealNetworks has introduced a <a class="mw-redirect" href="/wiki/Variable_bit_rate" title="Variable bit rate">variable bit rate</a> form called RealMedia Variable Bitrate (<a href="/wiki/RMVB" title="RMVB">RMVB</a>). This allows for better video quality, however this format is less suited for streaming because it is difficult to predict how much network capacity a certain video stream will need. Video with fast motion or rapidly changing scenes will require a higher bit rate. If the bit rate of a video stream increases significantly, it may exceed the speed at which data can be transmitted over the network, leading to an interruption in the video.</p>
<p>RealNetworks says that the RealVideo and RealAudio codecs are not available in source code under the <a href="/wiki/RealNetworks_Public_Source_License" title="RealNetworks Public Source License">RPSL</a> license. Source code is available only under <a href="/wiki/RealNetworks_Community_Source_License" title="RealNetworks Community Source License">RCSL</a> license for commercial porting to non-supported processors and operating systems. While RealNetworks owns most of the intellectual property for RealVideo and RealAudio, RealNetworks has licensed third party technology for certain aspects of those codecs. RealNetworks claims that it does not have the rights to license that technology under an open source license.<sup class="reference" id="cite_ref-helix-producer_5-0"><a href="#cite_note-helix-producer-5">[5]</a></sup></p>
<h2><span class="mw-headline" id="RealVideo_Players">RealVideo Players</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=RealVideo&amp;action=edit&amp;section=2" title="Edit section: RealVideo Players">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The official player for RealVideo is RealNetworks <a class="mw-redirect" href="/wiki/RealPlayer_SP" title="RealPlayer SP">RealPlayer SP</a>, currently at version 15, and is available for various platforms including <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a>, <a class="mw-redirect" href="/wiki/Apple_Macintosh" title="Apple Macintosh">Macintosh</a>, and <a href="/wiki/Linux" title="Linux">Linux</a>. Several other players exist, including <a href="/wiki/MPlayer" title="MPlayer">MPlayer</a> and <a href="/wiki/Media_Player_Classic" title="Media Player Classic">Media Player Classic</a>. Many of these rely on the <a class="mw-redirect" href="/wiki/Dynamic_Link_Library" title="Dynamic Link Library">dynamically linked libraries</a> (DLLs) from the official RealPlayer to play the video, and thus require RealPlayer to be installed (or at least its DLLs, if not the actual player). However, the open source <a href="/wiki/FFmpeg" title="FFmpeg">ffmpeg</a> library (and its <a href="/wiki/DirectShow" title="DirectShow">DirectShow</a> counterpart <a href="/wiki/Ffdshow" title="Ffdshow">ffdshow</a>) can play RealVideo and does not require RealPlayer or any parts thereof.</p>
<p>The latest version of RealPlayer that can run on Windows 9x is RealPlayer 8; but this version can be easily modified to play RealPlayer 9 and 10 files, by the manual addition of just three .dll files (codecs and plugins), from Microsoft's free distribution of RealPlayer 10, that aren't included in RealPlayer 8 Basic.</p>
<p>RealNetworks has also developed the open source <a class="mw-redirect" href="/wiki/Helix_player" title="Helix player">Helix player</a>, however support for RealVideo in the Helix Project is limited because RealNetworks is still keeping the codecs proprietary.</p>
<p>RealPlayer does not record RealVideo streams, and RealNetworks has advertised this feature to content owners such as broadcasters, film studios, and music labels, as a means of discouraging users from copying video. However, other software exists which can save the streams to files for later viewing. Such copying, known as <a class="mw-redirect" href="/wiki/Time-shifting" title="Time-shifting">time-shifting</a>, is legal in most countries.</p>
<h3><span class="mw-headline" id="Video_compression_formats_and_codecs">Video compression formats and codecs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=RealVideo&amp;action=edit&amp;section=3" title="Edit section: Video compression formats and codecs">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>RealVideo files are compressed using several different <a class="mw-redirect" href="/wiki/Video_compression_format" title="Video compression format">video compression formats</a>. Each video compression format is identified by a <a href="/wiki/FourCC" title="FourCC">four character code</a>. Below is a list of the video compression formats and the version in which each was introduced:</p>
<ul>
<li><tt>rv10, rv13</tt>: RealVideo 1.0, based on <a href="/wiki/H.263" title="H.263">H.263</a> (included with RealPlayer 5)<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></li>
<li><tt>rv20</tt>: RealVideo G2 and RealVideo G2+SVT, also based on h.263 (included with RealPlayer 6)<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></li>
<li><tt><span id="RV30"></span>rv30</tt>: RealVideo 8, suspected to based largely on an early draft of <a class="mw-redirect" href="/wiki/H.264" title="H.264">H.264</a> (included with RealPlayer 8)<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></li>
<li><tt><span id="RV40"></span>rv40</tt>: RealVideo 9, suspected to be based on H.264 (included with RealPlayer 9)<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></li>
<li><tt>rv40</tt>: RealVideo 10, aka RV9 EHQ (included with RealPlayer 10). This refers to an improved encoder for the RV9 format that is fully backwards compatible with RV9 players – the format and decoder did not change, only the encoder did. As a result, it uses the same <a href="/wiki/FourCC" title="FourCC">FourCC</a>.</li>
</ul>
<p>The newest version of RealPlayer can play any RealVideo file, as can programs using <a href="/wiki/FFmpeg" title="FFmpeg">FFmpeg</a>. Other programs may not support all video compression formats.</p>
