<p><b>CorelDraw</b> (styled <b>CorelDRAW</b>) is a <a href="/wiki/Vector_graphics_editor" title="Vector graphics editor">vector graphics editor</a> developed and marketed by <a href="/wiki/Corel" title="Corel">Corel Corporation</a>. It is also the name of Corel's Graphics Suite, which bundles CorelDraw with bitmap-image editor <a href="/wiki/Corel_Photo-Paint" title="Corel Photo-Paint">Corel Photo-Paint</a> as well as other graphics-related programs (see below). The latest version is designated X8 (equivalent to version 18), and was released in March 2016.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> CorelDraw is designed to edit two-dimensional images such as logos and posters.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=CorelDRAW&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In 1987, Corel hired software engineers Michel Bouillon and Pat Beirne to develop a vector-based illustration program to bundle with their desktop publishing systems. That program, CorelDraw, was initially released in 1989. CorelDraw 1.x and 2.x ran under Windows 2.x and 3.0. CorelDraw 3.0 came into its own with Microsoft's release of Windows 3.1. The inclusion of <a href="/wiki/TrueType" title="TrueType">TrueType</a> in Windows 3.1 transformed CorelDraw into a serious illustration program capable of using system-installed outline fonts without requiring third-party software such as <a href="/wiki/Adobe_Type_Manager" title="Adobe Type Manager">Adobe Type Manager</a>; paired with a photo-editing program (Corel Photo-Paint), a font manager and several other pieces of software, it was also part of the first all-in-one graphics suite.</p>

<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=CorelDRAW&amp;action=edit&amp;section=3" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Supported_platforms">Supported platforms</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=CorelDRAW&amp;action=edit&amp;section=4" title="Edit section: Supported platforms">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>CorelDraw was originally developed for Microsoft <a href="/wiki/Windows_3.0" title="Windows 3.0">Windows 3</a> and currently runs on <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a>, <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a>, <a href="/wiki/Windows_7" title="Windows 7">Windows 7</a>, <a href="/wiki/Windows_8" title="Windows 8">Windows 8</a> and <a href="/wiki/Windows_10" title="Windows 10">Windows 10</a> .<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> The latest version, X8, was released on 15 March 2016.</p>
<p>Versions for <a class="mw-redirect" href="/wiki/Mac_OS" title="Mac OS">Mac OS</a> and <a class="mw-redirect" href="/wiki/Mac_OS_X" title="Mac OS X">Mac OS X</a> were at one time available, but due to poor sales these were discontinued. The last port for Linux was version 9 (released in 2000, it did not run natively; instead, it used a modified version of <a href="/wiki/Wine_(software)" title="Wine (software)">Wine</a> to run) and the last version for OS X was version 11 (released in 2001). Also, up until version 5, CorelDraw was developed for <a href="/wiki/Windows_3.1x" title="Windows 3.1x">Windows 3.1x</a>, <a href="/wiki/Convergent_Technologies_Operating_System" title="Convergent Technologies Operating System">CTOS</a> and <a href="/wiki/OS/2" title="OS/2">OS/2</a>.</p>
<p>With version 6, CorelDraw introduced the automation of tasks using a Corel proprietary scripting language, COREL Script. With version 10, support for VBA (<a href="/wiki/Visual_Basic_for_Applications" title="Visual Basic for Applications">Visual Basic for Applications</a>) was introduced for scripting by what Corel calls now <i>macros</i>. Corel recommends to no longer use the COREL Script language but only VBA.</p>

<h3><span class="mw-headline" id="Structure">Structure</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=CorelDRAW&amp;action=edit&amp;section=6" title="Edit section: Structure">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In its first versions, the CDR file format was a completely <a href="/wiki/Proprietary_format" title="Proprietary format">proprietary</a> <a href="/wiki/File_format" title="File format">file format</a> primarily used for vector graphic drawings, recognizable by the first two bytes of the file being "WL". Starting with CorelDraw 3, the file format changed to a <a href="/wiki/Resource_Interchange_File_Format" title="Resource Interchange File Format">Resource Interchange File Format</a> (RIFF) envelope, recognizable by the first four bytes of the file being "RIFF", and a "CDR*vrsn" in bytes 9 to 15, with the asterisk "*" being in early versions just a blank, and beginning with CorelDraw 4, the version number of the writing program in hexadecimal ("4" meaning version 4, "D" meaning version 14). The actual data chunk of the RIFF remains a Corel proprietary format.</p>
<p>From version X4 (14) on, the CDR file is a ZIP-compressed directory of several files, among them XML-files and the RIFF-structured <i>riffdata.cdr</i> with the familiar version signature in versions X4 (CDREvrsn) and X5 (CDRFvrsn), and a <i>root.dat</i> with CorelDraw X6, where the bytes 9 to 15 look slightly different -- "CDRGfver" in a file created with X6. "F" was the last valid hex digit, and the "fver" now indicates that the letter before does no longer stand for a hex digit.</p>
<p>There is no publicly available CDR file format specification.<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup><sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup></p>
<p>Other CorelDraw file formats include CorelDraw Compressed (CDX), CorelDraw Template (CDT)<sup class="reference" id="cite_ref-CorelDRAW_Template_format_33-0"><a href="#cite_note-CorelDRAW_Template_format-33">[33]</a></sup> and Corel Presentation Exchange (CMX).<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup></p>
<h3><span class="mw-headline" id="Use_of_CDR-files_in_other_programs">Use of CDR-files in other programs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=CorelDRAW&amp;action=edit&amp;section=7" title="Edit section: Use of CDR-files in other programs">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In December 2006 the <a href="/wiki/SK1_(program)" title="SK1 (program)">sK1</a> <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> project team started to <a class="mw-redirect" href="/wiki/Reverse-engineer" title="Reverse-engineer">reverse-engineer</a> the CDR format.<sup class="reference" id="cite_ref-sk1_35-0"><a href="#cite_note-sk1-35">[35]</a></sup> The results and the first working snapshot of the CDR importer were presented at the Libre Graphics Meeting 2007 conference taking place in May 2007 in Montreal (Canada).<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup> Later on the team parsed the structure of other Corel formats with the help of the open source CDR Explorer.<sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup> As of 2008, the sK1 project claims to have the best import support for CorelDraw file formats among open source software programs. The sK1 project developed also the UniConvertor, a command line open source tool which supports conversion from CorelDraw ver.7-X4 formats (CDR/CDT/CCX/CDRX/CMX) to other formats. UniConvertor is also used in <a href="/wiki/Inkscape" title="Inkscape">Inkscape</a> and <a href="/wiki/Scribus" title="Scribus">Scribus</a> open source projects as an external tool for CorelDraw files importing.<sup class="reference" id="cite_ref-inkscape_38-0"><a href="#cite_note-inkscape-38">[38]</a></sup><sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[39]</a></sup><sup class="reference" id="cite_ref-uniconvertor_40-0"><a href="#cite_note-uniconvertor-40">[40]</a></sup></p>
<p>In 2007, <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> blocked CDR file format in Microsoft Office 2003 with the release of Service Pack 3 for Office 2003.<sup class="reference" id="cite_ref-41"><a href="#cite_note-41">[41]</a></sup><sup class="reference" id="cite_ref-42"><a href="#cite_note-42">[42]</a></sup> Microsoft later apologized for inaccurately blaming the CDR file format and other formats for security problems in Microsoft Office and released some tools for solving this problem.<sup class="reference" id="cite_ref-43"><a href="#cite_note-43">[43]</a></sup></p>
<p>In 2012 the joint LibreOffice/re-lab team implemented libcdr, a library for reading CDR files from v1 to the currently latest X7 version and CMX files.<sup class="reference" id="cite_ref-44"><a href="#cite_note-44">[44]</a></sup> The library has extensive support for shapes and their properties, including support for color management and spot colors, and has a basic support for text.<sup class="reference" id="cite_ref-45"><a href="#cite_note-45">[45]</a></sup> The library provides a built-in converter to SVG, and a converter to OpenDocument is provided by writerperfect package. The libcdr library is used in LibreOffice starting from version 3.6,<sup class="reference" id="cite_ref-46"><a href="#cite_note-46">[46]</a></sup> and thanks to public API it can be freely used by other applications.</p>
