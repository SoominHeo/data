<p><b>Maemo</b> is a <a class="mw-redirect" href="/wiki/Software_platform" title="Software platform">software platform</a> developed by <a href="/wiki/Nokia" title="Nokia">Nokia</a> and then handed over to <a href="/wiki/Hildon" title="Hildon">Hildon Foundation</a> for <a href="/wiki/Smartphone" title="Smartphone">smartphones</a> and <a class="mw-redirect" href="/wiki/Internet_tablet" title="Internet tablet">Internet tablets</a>.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> The platform comprises both the Maemo operating system and <a href="/wiki/Software_development_kit" title="Software development kit">SDK</a>.</p>
<p>Maemo is mostly based on <a href="/wiki/Free_and_open-source_software" title="Free and open-source software">open-source</a> code and has been developed by Maemo Devices within Nokia in collaboration with many open-source projects such as the <a href="/wiki/Linux_kernel" title="Linux kernel">Linux kernel</a>, <a href="/wiki/Debian" title="Debian">Debian</a>, and <a href="/wiki/GNOME" title="GNOME">GNOME</a>. Maemo is based on <a href="/wiki/Debian" title="Debian">Debian GNU/Linux</a> and draws much of its <a class="mw-redirect" href="/wiki/GUI" title="GUI">GUI</a>, <a href="/wiki/Software_framework" title="Software framework">frameworks</a>, and <a href="/wiki/Library_(computing)" title="Library (computing)">libraries</a> from the <a href="/wiki/GNOME" title="GNOME">GNOME</a> project. It uses the <a href="/wiki/Matchbox_(window_manager)" title="Matchbox (window manager)">Matchbox</a> window manager and the <a class="mw-redirect" href="/wiki/GTK" title="GTK">GTK</a>-based <a href="/wiki/Hildon" title="Hildon">Hildon</a> framework as its GUI and <a href="/wiki/Application_framework" title="Application framework">application framework</a>.</p>
<p>The <a href="/wiki/User_interface" title="User interface">user interface</a> in Maemo 4 is similar to many hand-held interfaces and features a "home" screen, from which all applications and settings are accessed. The home screen is divided into areas for launching applications, a menu bar, and a large customizable area that can display information such as an <a href="/wiki/News_aggregator" title="News aggregator">RSS reader</a>, <a href="/wiki/Internet_radio" title="Internet radio">Internet radio</a> player, and Google search box. The Maemo 5 user interface is slightly different; the menu bar and info area are consolidated to the top of the display, and the four desktops can be customized with shortcuts and widgets.</p>
<p>At the <a href="/wiki/Mobile_World_Congress" title="Mobile World Congress">Mobile World Congress</a> in February 2010, it was announced that the Maemo project would be merging with <a href="/wiki/Moblin" title="Moblin">Moblin</a> to create the <a href="/wiki/MeeGo" title="MeeGo">MeeGo</a> mobile software platform. Despite that, the Maemo community continued to be active, and in late 2012 Nokia began transferring Maemo ownership to the newly established <a class="mw-redirect" href="/wiki/Hildon_Foundation" title="Hildon Foundation">Hildon Foundation</a>.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup><sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>

<h2><span class="mw-headline" id="User_Interface">User Interface</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=1" title="Edit section: User Interface">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="OS2005.E2.80.93OS2008">OS2005–OS2008</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=2" title="Edit section: OS2005–OS2008">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Up to Maemo 4 (AKA OS2008), the default screen is the "Home" screen — the central point from which all applications and settings are accessed. The Home Screen is divided into the following areas:</p>
<ul>
<li>Vertically down the left hand side of the screen is the taskbar, with applets for the web browser, communications, and application menu by default. These can be modified using third party plug-ins (e.g. to provide a favorites or command menu).</li>
<li>Horizontally across the top left half is the menu bar, which shows the application name and window title, and gives access to the application's menu (which contains the typical file, edit, view, tools, etc., menus and sub-menus).</li>
<li>Horizontally across the top right quadrant is the status bar, containing icons such as battery life, wireless connection, volume, <a href="/wiki/Bluetooth" title="Bluetooth">Bluetooth</a> status, and brightness by default. These can be expanded using third party plug-ins in the same manner as the task-bar.</li>
<li>The remaining large part of the display contains Home applets (roughly analogous to <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple Inc.</a>'s <a class="mw-redirect" href="/wiki/Dashboard_(Mac_OS)" title="Dashboard (Mac OS)">Dashboard widgets</a>), which can display data as well as serving as a shortcut to applications. These include an RSS reader, Internet radio player, Google search box and contact list by default, but can also be expanded with third party plug-ins.</li>
</ul>
<p>The interface uses either the touch screen, or a directional pad and select button, with separate back, menu, and home buttons. It is capable of receiving text input through handwriting recognition, two different sizes of on-screen keyboard and hardware keyboard input with the <a href="/wiki/Nokia_N810" title="Nokia N810">N810</a>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<h3><span class="mw-headline" id="Maemo_5_.28Fremantle.29">Maemo 5 (Fremantle)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=3" title="Edit section: Maemo 5 (Fremantle)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

</div>
<p>The user interface in Maemo 5 is different from its predecessors. It provides four fully customizable (with the ability to add/remove widgets, move widgets around, change the background and customize shortcuts to applications/contacts) "Home" screens,<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> called Panorama Desktop.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> Switching from one desktop to the others is done by sliding one's finger horizontally on the background. The dashboard is accessed via the upper left icon and shows all the running applications, in a manner similar to the Exposé feature in Apple's <a class="mw-redirect" href="/wiki/Mac_OS_X" title="Mac OS X">Mac OS X</a> operating system. From the dashboard, running applications can be brought back to full screen by tapping the preview window, and applications can be closed by tapping an X-symbol located in the top right corner of the preview window, similar to the concept of closing applications in other operating systems. The application launcher, where all the installed applications can be launched, can also be accessed from the dashboard.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> If no task or application is running in the background, tapping the top left icon skips the dashboard and directly displays the application launcher.</p>
<p>Maemo provides the <a href="/wiki/Mozilla" title="Mozilla">Mozilla</a>-based <a href="/wiki/MicroB" title="MicroB">MicroB</a> web-browser with complete <a href="/wiki/Adobe_Flash" title="Adobe Flash">Adobe Flash</a> support. It supports an 800×480 display resolution, so some web pages can be viewed without horizontal scrolling. It can automatically connect to known wireless networks, download RSS feeds and email and disconnect automatically without user intervention.<sup class="reference" id="cite_ref-arsn800_11-0"><a href="#cite_note-arsn800-11">[11]</a></sup></p>
<p>Advanced Packaging Tool with a command-line apt-get client can be used to install applications. Users can subscribe to different <a class="mw-redirect" href="/wiki/Software_repositories" title="Software repositories">software repositories</a>, which can then be used to automatically keep software up to date. The application manager also provides an overview of everything currently installed on the system. Data can be synchronized with a PC via a USB connection, and the user's files can be accessed using the standard Removable Storage Device protocol.<sup class="reference" id="cite_ref-arsn800_11-1"><a href="#cite_note-arsn800-11">[11]</a></sup></p>
<p>A new update (Version 21.2011.38-1.002) was released onto the Nokia N900 on 2 November 2011<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> as an <a href="/wiki/Over-the-air_programming" title="Over-the-air programming">OTA</a> update. The new version mostly consists of security updates. This is considered to be the final official update to Maemo 5/Fremantle shipped by Nokia.</p>
<p>Further development of Maemo 5 happens as a community effort in Maemo-CSSU.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></p>
<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=4" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Updating">Updating</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=5" title="Edit section: Updating">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Maemo devices can be updated using a simple flashing method with a computer over USB.</p>
<p>Since Diablo (Maemo 4.1), Maemo supports "Seamless Software Update" (SSU), which allows incremental operating system upgrades "over the air" using the <a href="/wiki/Advanced_Packaging_Tool" title="Advanced Packaging Tool">Advanced Packaging Tool</a>, without the need for a full flash with every update.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup></p>
<p>Flashing remains available as a way to start over from scratch with a clean installation (much like formatting a hard drive and reinstalling an operating system on a PC).<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup></p>
<h3><span class="mw-headline" id="Security">Security</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=6" title="Edit section: Security">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">See also: <a href="/wiki/Mobile_security" title="Mobile security">Mobile security</a></div>
<p>The quick start guide for developers warns that Maemo security concentrates on preventing remote attacks (e.g. by <a class="mw-redirect" href="/wiki/Wireless_networking" title="Wireless networking">wireless networking</a> and <a href="/wiki/Bluetooth" title="Bluetooth">Bluetooth</a>). It also warns that Maemo's <a href="/wiki/Superuser" title="Superuser">root account</a> has a trivial default password (user: gainroot, password: rootme) which needs to be changed before enabling remote access.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></p>
<p>Maemo employs a numeric security code as a way to lock the device's controls and display independently of the root password, to help prevent unauthorised access.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<h2><span class="mw-headline" id="Software_architecture">Software architecture</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=7" title="Edit section: Software architecture">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Maemo is a modified version of the <a href="/wiki/Debian" title="Debian">Debian</a> <a href="/wiki/GNU" title="GNU">GNU</a>/<a href="/wiki/Linux_distribution" title="Linux distribution">Linux distribution</a>, slimmed down for mobile devices.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> It uses an <a href="/wiki/X_Window_System" title="X Window System">X Window System</a>-based <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a> using Xomap and the <a href="/wiki/Matchbox_(window_manager)" title="Matchbox (window manager)">Matchbox</a> <a href="/wiki/Window_manager" title="Window manager">window manager</a>. The GUI uses the <a href="/wiki/GTK%2B" title="GTK+">GTK+</a> toolkit and <a href="/wiki/Hildon" title="Hildon">Hildon</a> user interface <a class="mw-redirect" href="/wiki/Widget_(computing)" title="Widget (computing)">widgets</a> and <a class="mw-redirect" href="/wiki/API" title="API">API</a>.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup></p>
<p><a href="/wiki/BusyBox" title="BusyBox">BusyBox</a>, a software package for embedded and mobile devices, replaces the <a href="/wiki/GNU_Core_Utilities" title="GNU Core Utilities">GNU Core Utilities</a> used in <a href="/wiki/Debian" title="Debian">Debian</a>-proper to reduce memory usage and storage requirements (at the expense of some functionality).<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup></p>
<p><a href="/wiki/Enlightened_Sound_Daemon" title="Enlightened Sound Daemon">ESD</a> is used as the primary sound server, and <a href="/wiki/GStreamer" title="GStreamer">GStreamer</a> is used by the shipped media player to play back sounds and movies. The formats supported by GStreamer can be extended by compiling GStreamer plugins in scratchbox (<a href="/wiki/Maemo#Maemo_SDK" title="Maemo">Maemo SDK</a>), which was done, for example, to bring <a href="/wiki/Ogg" title="Ogg">Ogg</a> support to the platform,<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> as well as experimental features such as <a href="/wiki/WebM" title="WebM">WebM</a> and <a href="/wiki/VP8" title="VP8">VP8</a> after they were announced by <a href="/wiki/Google" title="Google">Google</a>.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup> Third-party media players can access GStreamer directly or via "osso-media-server".<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2008)">citation needed</span></a></i>]</sup></p>
<p>Window management is handled by the Matchbox window manager, which limits the screen to showing a single window at a time (<a href="/wiki/Ubuntu_Netbook_Edition" title="Ubuntu Netbook Edition">Ubuntu Netbook Edition</a> implements a similar system). This is to improve usability on a mobile device with a small screen.</p>
<p>Although Maemo is based on GNU/Linux and other open source software, some parts of Maemo remain closed source. These include some user-space software, like certain status bar and taskbar applets (including the display brightness applet) and applications, and some system <a class="mw-redirect" href="/wiki/Daemon_(computer_software)" title="Daemon (computer software)">daemons</a> related to connectivity and power management.<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup></p>
<h2><span class="mw-headline" id="Software">Software</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=8" title="Edit section: Software">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Maemo comes with a number of built-in applications, but additional applications can be installed from a number of sources, including various official and community <a href="/wiki/Software_repository" title="Software repository">software repositories</a>, and <a href="/wiki/Deb_(file_format)" title="Deb (file format)">deb</a> files through either the built-in <a class="mw-redirect" href="/wiki/Package_management_system" title="Package management system">package manager</a> "Application manager", or the Advanced Packaging Tool and <a href="/wiki/Dpkg" title="Dpkg">dpkg</a>. Bundled applications include the <a href="/wiki/Mozilla" title="Mozilla">Mozilla</a>-based <a href="/wiki/MicroB" title="MicroB">MicroB</a> browser, <a class="mw-redirect" href="/wiki/Macromedia_Flash" title="Macromedia Flash">Macromedia Flash</a> player, <a href="/wiki/Gizmo5" title="Gizmo5">Gizmo5</a>, and <a href="/wiki/Skype" title="Skype">Skype</a>.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup></p>

<h2><span class="mw-headline" id="Software_development">Software development</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=11" title="Edit section: Software development">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Software can be developed in <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> using the Maemo <a href="/wiki/Software_development_kit" title="Software development kit">SDK</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> (which is supported by the Jalimo JVM),<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup> <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>, <a href="/wiki/Mono_(software)" title="Mono (software)">Mono</a>, Vala, Perl and Pascal.</p>
<p>The Maemo SDK is based around the <a href="/wiki/Debian" title="Debian">Debian</a>-oriented <a href="/wiki/Scratchbox" title="Scratchbox">Scratchbox Cross Compilation Toolkit</a>, which provides a <a href="/wiki/Sandbox_(software_development)" title="Sandbox (software development)">sandbox</a> environment in which development may take place. Scratchbox uses <a href="/wiki/QEMU" title="QEMU">QEMU</a> to emulate an ARM processor or <a class="new" href="/w/index.php?title=Sbrsh&amp;action=edit&amp;redlink=1" title="Sbrsh (page does not exist)">sbrsh</a> to remotely execute instructions. Scratchbox-compatible rootstraps are available for both <a href="/wiki/X86" title="X86">x86</a> and <a href="/wiki/ARM_architecture" title="ARM architecture">ARM</a>, so the majority of development and debugging takes place on x86, with final packaging being for ARM.<sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup></p>
<p>As a new feature of the Maemo operating system, Maemo 5 offers the <a class="mw-redirect" href="/wiki/Qt_(toolkit)" title="Qt (toolkit)">Qt</a> library as a community-supported component, alongside the officially supported <a href="/wiki/GTK%2B" title="GTK+">GTK+</a> backend. This will change with the Harmattan release, which will add the Qt library as the default, with GTK+ becoming community-supported.<sup class="reference" id="cite_ref-38"><a href="#cite_note-38">[38]</a></sup> The programming languages <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> and <a href="/wiki/C%2B%2B" title="C++">C++</a> will also be supported.</p>
<p>The Nokia Developer Wiki community has articles about Qt development, and includes tutorials and articles about development for the Maemo operating system.</p>
<h2><span class="mw-headline" id="Hardware">Hardware</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=12" title="Edit section: Hardware">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The Maemo operating system is designed for <a class="mw-redirect" href="/wiki/Nokia_Internet_Tablet" title="Nokia Internet Tablet">Nokia Internet Tablets</a>, which feature very similar specifications to Nokia's high-end <a href="/wiki/Nokia_Nseries" title="Nokia Nseries">N-series</a> and <a href="/wiki/Nokia_Eseries" title="Nokia Eseries">E-series</a> cellphones, with <a href="/wiki/Texas_Instruments" title="Texas Instruments">TI</a> <a href="/wiki/OMAP" title="OMAP">OMAP</a> <a href="/wiki/ARM_architecture" title="ARM architecture">ARM</a> <a class="mw-redirect" href="/wiki/System-on-a-chip" title="System-on-a-chip">SoCs</a>, large screens, and expandable internal storage.<sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[39]</a></sup> Although the highly optimized, hardware-specific nature of Maemo renders its operation on non-Internet tablet hardware very difficult, most of the important non-proprietary parts of the operating system, along with some of the available third-party applications, are actively being packaged for Debian and are available for use on other distributions, which will open up a large range of other hardware options.<sup class="reference" id="cite_ref-40"><a href="#cite_note-40">[40]</a></sup></p>
<p>Nokia runs the Maemo operating system on the <a href="/wiki/Nokia_N900" title="Nokia N900">Nokia N900</a> and <a href="/wiki/Nokia_N9" title="Nokia N9">Nokia N9</a>.</p>

<h4><span class="mw-headline" id="Diablo">Diablo</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=18" title="Edit section: Diablo">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Diablo is the codename for the OS2008 Feature Upgrade (corresponding to Maemo 4.1),<sup class="reference" id="cite_ref-codenames_48-0"><a href="#cite_note-codenames-48">[48]</a></sup> released in June 2008.<sup class="reference" id="cite_ref-49"><a href="#cite_note-49">[49]</a></sup> Major features include significant improvements to the built-in Application manager, incremental operating system upgrades without reflashing ("Seamless Software Update", SSU), <a class="mw-redirect" href="/wiki/Modest_(e-mail_client)" title="Modest (e-mail client)">Modest</a> as the default mail client, and a new version of the <a href="/wiki/MicroB" title="MicroB">MicroB</a> browser with better performance and some interface improvements (though still based on the same <a class="mw-redirect" href="/wiki/Gecko_(layout_engine)" title="Gecko (layout engine)">Gecko</a> release as Chinook's MicroB).<sup class="reference" id="cite_ref-50"><a href="#cite_note-50">[50]</a></sup><sup class="reference" id="cite_ref-51"><a href="#cite_note-51">[51]</a></sup> Diablo also included an implementation of the WiMAX stack for the short-lived N810 variation called <a href="/wiki/Nokia_N810#Nokia_N810_WiMAX_Edition" title="Nokia N810">N810 WiMAX edition</a>.<sup class="reference" id="cite_ref-52"><a href="#cite_note-52">[52]</a></sup></p>
<h4><span class="mw-headline" id="SSU">SSU</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=19" title="Edit section: SSU">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>SSU, in particular, is one of the most significant parts of the Diablo release, as it eliminates the flash-based upgrade system of previous Maemo releases which required the user to completely wipe their device with each upgrade. With SSU, the user can receive "over-the-air" updates. This also decouples bundled software updates from the major system updates, as was required with the flash-based method, so Nokia can release more frequent updates to individual packages than before.<sup class="reference" id="cite_ref-53"><a href="#cite_note-53">[53]</a></sup></p>
<p>The first SSU update was pushed on 11 August 2008, and brought Diablo up to version 4.2008.30-2. The update primarily features MicroB, Modest, and connectivity framework updates as well as a number of other minor bugfixes.</p>

<p>Maemo 5, also known as Fremantle,<sup class="reference" id="cite_ref-codenames_48-1"><a href="#cite_note-codenames-48">[48]</a></sup> is the default operating system on the <a href="/wiki/Nokia_N900" title="Nokia N900">Nokia N900</a>. The release features a much more finger-friendly and consistent user interface, and an <a href="/wiki/X_Window_System" title="X Window System">X-server</a> based on <a class="mw-redirect" href="/wiki/Xorg" title="Xorg">Xorg</a> rather than <a class="mw-redirect" href="/wiki/KDrive" title="KDrive">KDrive</a>.<sup class="reference" id="cite_ref-54"><a href="#cite_note-54">[54]</a></sup><sup class="reference" id="cite_ref-55"><a href="#cite_note-55">[55]</a></sup> It also features several new technologies, including the <a class="mw-redirect" href="/wiki/Tracker_(desktop_search_software)" title="Tracker (desktop search software)">Tracker</a> search system, <a href="/wiki/PulseAudio" title="PulseAudio">PulseAudio</a> (replacing <a href="/wiki/Enlightened_Sound_Daemon" title="Enlightened Sound Daemon">ESD</a>), the OHM hardware management daemon, the gUPnP <a class="mw-redirect" href="/wiki/UPnP" title="UPnP">UPnP</a> framework, enhanced location framework, and <a href="/wiki/Upstart" title="Upstart">Upstart</a>. Several existing technologies have also been updated, including <a class="mw-redirect" href="/wiki/Gecko_(layout_engine)" title="Gecko (layout engine)">Gecko</a>, <a class="mw-redirect" href="/wiki/BlueZ" title="BlueZ">BlueZ</a>, <a class="mw-redirect" href="/wiki/Gstreamer" title="Gstreamer">gstreamer</a> (will include <a href="/wiki/OpenMAX" title="OpenMAX">OpenMAX</a>), <a href="/wiki/GTK%2B" title="GTK+">GTK+</a>, and <a href="/wiki/Telepathy_(software)" title="Telepathy (software)">Telepathy</a>.<sup class="reference" id="cite_ref-56"><a href="#cite_note-56">[56]</a></sup><sup class="reference" id="cite_ref-57"><a href="#cite_note-57">[57]</a></sup></p>
<p>Maemo 5 comes with new hardware, the <a href="/wiki/Nokia_N900" title="Nokia N900">Nokia N900</a> featuring a <a href="/wiki/Texas_Instruments" title="Texas Instruments">Texas Instruments</a> <a href="/wiki/OMAP" title="OMAP">OMAP</a>3 <a class="mw-redirect" href="/wiki/System-on-a-chip" title="System-on-a-chip">SoC</a>, with an <a href="/wiki/High_Speed_Packet_Access" title="High Speed Packet Access">HSPA</a> modem and HD camera, which provides significant improvements in speed, <a href="/wiki/3D_computer_graphics" title="3D computer graphics">3D</a> acceleration, and media playback. Nokia decided that the commercial Maemo 5 platform releases will not be available on N800/N810 hardware, for compatibility reasons.<sup class="reference" id="cite_ref-58"><a href="#cite_note-58">[58]</a></sup> An open source Linux project called <a class="mw-redirect" href="/wiki/Mer_(operating_system)" title="Mer (operating system)">Mer</a>, a variant of Maemo, has been formed to provide an alternative, fully open operating system for mobile devices including the Nokia N800 and N810.<sup class="reference" id="cite_ref-59"><a href="#cite_note-59">[59]</a></sup></p>
<p>Unlike most smartphones, the end-user is able to gain <a href="/wiki/Superuser" title="Superuser">root access</a> by installing an application, such as the "rootsh",<sup class="reference" id="cite_ref-60"><a href="#cite_note-60">[60]</a></sup> and then issuing the appropriate command in the terminal in Maemo 5.<sup class="reference" id="cite_ref-61"><a href="#cite_note-61">[61]</a></sup> The device does not need to be unlocked or "<a class="mw-redirect" href="/wiki/Jailbreak_(iPhone_OS)" title="Jailbreak (iPhone OS)">jailbroken</a>" in order to install an unsupported application. The UK cellular operator Vodafone has its own branded, somewhat more restricted version of the Maemo 5 operating system. Users can flash (change) to the global variant of the operating system if they wish.</p>

<h3><span class="mw-headline" id="Hacker_Editions">Hacker Editions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=21" title="Edit section: Hacker Editions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Nokia's release of the <a href="/wiki/Nokia_N800" title="Nokia N800">N800</a> in January 2007 included Internet Tablet OS 2007. OS2007 ran only on the N800 and Nokia had no plans to release it for the <a class="mw-redirect" href="/wiki/Nokia_770" title="Nokia 770">770</a>; however feedback from disgruntled 770 owners<sup class="reference" id="cite_ref-68"><a href="#cite_note-68">[68]</a></sup> led Nokia to release an unsupported hybrid of OS2006 and OS2007, dubbed Internet Tablet OS 2007 Hacker Edition, in February 2007.<sup class="reference" id="cite_ref-69"><a href="#cite_note-69">[69]</a></sup> OS2007HE combined the binary parts of OS2006 with most of the updated libraries and user-space applications of OS2007.</p>
<p>Nokia followed the release of the N810 and OS2008 in November 2007 with the OS2008 Hacker Edition for the 770 on 14 February 2008, employing a similar method to the OS2007HE release to create a hybrid between OS2006 and OS2007. The Hacker Editions allow 770 users to access the latest Internet Tablet OS releases and third-party software, although due to the 770's hardware limitations and the increased CPU requirements of OS2008, performance is inferior to that of OS2007 HE in most areas.<sup class="reference" id="cite_ref-70"><a href="#cite_note-70">[70]</a></sup></p>
<h3><span class="mw-headline" id="Alternative_desktops">Alternative desktops</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=22" title="Edit section: Alternative desktops">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In addition to the officially supported Hildon environment that comes standard on the tablets, several alternate desktop systems are available. Penguinbait, a member of the Internet Tablet Talk community, has successfully ported <a class="mw-redirect" href="/wiki/KDE_3" title="KDE 3">KDE 3</a>,<sup class="reference" id="cite_ref-71"><a href="#cite_note-71">[71]</a></sup> <a href="/wiki/Openbox" title="Openbox">Openbox</a>,<sup class="reference" id="cite_ref-72"><a href="#cite_note-72">[72]</a></sup> and <a href="/wiki/Xfce" title="Xfce">Xfce</a><sup class="reference" id="cite_ref-73"><a href="#cite_note-73">[73]</a></sup> to the N800 and N810 (the N770 is able to run an early version of the KDE port). A port of <a class="mw-redirect" href="/wiki/KDE_Plasma_Workspaces" title="KDE Plasma Workspaces">KDE Plasma Workspaces</a> were developed.<sup class="reference" id="cite_ref-74"><a href="#cite_note-74">[74]</a></sup> <a href="/wiki/LXDE" title="LXDE">LXDE</a> is the default desktop for the <i>Easy Debian</i> distribution, which is installed as a single application under Maemo but provides access to the full range of software in the ARM <a href="/wiki/Debian" title="Debian">Debian</a> distribution (including GIMP, Iceweasel/Firefox and OpenOffice)<sup class="reference" id="cite_ref-easydebian_34-1"><a href="#cite_note-easydebian-34">[34]</a></sup> via a <a href="/wiki/Chroot" title="Chroot">chroot</a> environment.</p>

<h3><span class="mw-headline" id="MeeGo">MeeGo</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=24" title="Edit section: MeeGo">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/MeeGo" title="MeeGo">MeeGo</a></div>
<p>In the wake of the release of the MeeGo code base, there are two main community efforts to bring MeeGo to the current Maemo devices (N800, N810, and N900), as Nokia has launched the N9, which is powered by MeeGo. These efforts are the MeeGo adaptation for the N800 and N810 devices,<sup class="reference" id="cite_ref-75"><a href="#cite_note-75">[75]</a></sup><sup class="reference" id="cite_ref-76"><a href="#cite_note-76">[76]</a></sup> and the MeeGo to N900 project.<sup class="reference" id="cite_ref-77"><a href="#cite_note-77">[77]</a></sup></p>
<h3><span class="mw-headline" id="Harmattan">Harmattan</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=25" title="Edit section: Harmattan">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Harmattan was originally planned as a transitional operating system developed by Nokia, meant to be a bridge between Nokia's Maemo and the MeeGo project. It is still using the APT package manager for applications, but the graphical user interface and major parts of the system were improved, rewritten from scratch or rebased on top of Meego frameworks. It is heavily using the <a class="mw-redirect" href="/wiki/Qt_(toolkit)" title="Qt (toolkit)">Qt</a> libraries instead of <a href="/wiki/GTK%2B" title="GTK+">GTK+</a>/<a href="/wiki/Hildon" title="Hildon">Hildon</a>, and introduced a new UI paradigm based on Swipe gestures, done from the edge of the screen.<sup class="reference" id="cite_ref-78"><a href="#cite_note-78">[78]</a></sup><sup class="reference" id="cite_ref-79"><a href="#cite_note-79">[79]</a></sup> After problems with the original Meego project, Nokia decided to rename Harmattan to Meego/Harmattan and is now shipping their N9 with this OS.</p>
<h3><span class="mw-headline" id="Mer">Mer</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=26" title="Edit section: Mer">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Mer_(software_distribution)" title="Mer (software distribution)">Mer (software distribution)</a></div>
<p>Mer is a completely <a class="mw-redirect" href="/wiki/Free_and_open-source" title="Free and open-source">free and open-source</a> software distribution targeting mobile and embedded systems, first developed for reimplementing Maemo in the open, and then rebased on top of the MeeGo tools.</p>
<h3><span class="mw-headline" id="Qt">Qt</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=27" title="Edit section: Qt">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In January 2008, Nokia began the process of acquiring <a class="mw-redirect" href="/wiki/Trolltech" title="Trolltech">Trolltech</a>, the developer of the <a class="mw-redirect" href="/wiki/Qt_(framework)" title="Qt (framework)">Qt</a> application framework. Nokia has since announced plans for the Qt libraries to be bundled with Maemo by <a href="/wiki/Fiscal_year" title="Fiscal year">Q4</a> 2008, though without a platform-specific "<a href="/wiki/Hildon" title="Hildon">hildonized</a>" user interface as is offered with <a class="mw-redirect" href="/wiki/GTK" title="GTK">GTK</a> (meaning Qt applications will look different from hildonized GTK applications).<sup class="reference" id="cite_ref-80"><a href="#cite_note-80">[80]</a></sup><sup class="reference" id="cite_ref-81"><a href="#cite_note-81">[81]</a></sup> This will make porting of Qt applications to the platform easier for developers, and make new applications available to users. Qt support on <a class="mw-redirect" href="/wiki/S60_platform" title="S60 platform">S60</a> devices will likely lead to cross-platform application sharing between the two platforms.<sup class="reference" id="cite_ref-82"><a href="#cite_note-82">[82]</a></sup><sup class="reference" id="cite_ref-83"><a href="#cite_note-83">[83]</a></sup> The announcement has been met with mixed reactions in the developer community.<sup class="reference" id="cite_ref-84"><a href="#cite_note-84">[84]</a></sup><sup class="reference" id="cite_ref-85"><a href="#cite_note-85">[85]</a></sup></p>
<h2><span class="mw-headline" id="Release_history">Release history</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=28" title="Edit section: Release history">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Naming">Naming</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=29" title="Edit section: Naming">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Maemo codenames are named for winds. For example, the latest release, codenamed "Fremantle", is a reference to the <a href="/wiki/Fremantle_Doctor" title="Fremantle Doctor">Fremantle Doctor</a>, the Western Australian vernacular term for the cooling afternoon sea breeze which occurs during summer months in south west coastal areas of Western Australia.</p>

<h2><span class="mw-headline" id="Maemo_Summit">Maemo Summit</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Maemo&amp;action=edit&amp;section=30" title="Edit section: Maemo Summit">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In 2008 and 2009, Maemo developers and users gathered at the Maemo Summit, the registration for which was free. Each participant got a badge and a Maemo T-shirt.</p>
<p>The 2009 Maemo Summit was held in <a href="/wiki/Amsterdam" title="Amsterdam">Amsterdam</a> on 9 October.<sup class="reference" id="cite_ref-91"><a href="#cite_note-91">[91]</a></sup> The first day was the Nokia day, with the other two days dedicated to community contributions. Nearly 400 developers attended the summit. Nokia gave out 300 <a href="/wiki/Nokia_N900" title="Nokia N900">N900</a> devices to independent developers during the summit.</p>
<p>The 2009 Maemo Summit was also the last Maemo Summit since MeeGo was launched. The event was replaced by the MeeGo Conference.</p>
