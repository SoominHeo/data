<h2><span class="mw-headline" id="Structured_Exception_Handling"><span id="SEH"></span>Structured Exception Handling</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft-specific_exception_handling_mechanisms&amp;action=edit&amp;section=1" title="Edit section: Structured Exception Handling">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Microsoft Structured Exception Handling is the native exception handling mechanism for Windows and a forerunner technology to <a href="#Vectored_Exception_Handling">Vectored Exception Handling</a> (VEH).<sup class="reference" id="cite_ref-devx_1-0"><a href="#cite_note-devx-1">[1]</a></sup> It features the <code>finally</code> mechanism not present in standard С++ exceptions (but present in most <a href="/wiki/Imperative_programming" title="Imperative programming">imperative</a> languages introduced later). SEH is set up and handled separately for each <a href="/wiki/Thread_(computing)" title="Thread (computing)">thread of execution</a>.</p>
<p>The Microsoft implementation of SEH is based on a patent licensed from <a href="/wiki/Borland" title="Borland">Borland</a>, <span><a class="external text" href="//www.google.com/patents/US5628016" rel="nofollow">U.S. Patent 5,628,016</a></span>.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<h3><span class="mw-headline" id="Usage">Usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft-specific_exception_handling_mechanisms&amp;action=edit&amp;section=2" title="Edit section: Usage">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">See also: <a href="/wiki/Exception_handling_syntax#Microsoft-specific" title="Exception handling syntax">Exception handling syntax § Microsoft-specific</a></div>
<p>Microsoft supports SEH as a programming technique at the compiler level only. MS Visual C++ compiler features three non-standard keywords: <code>__try</code>, <code>__except</code> and <code>__finally</code> — for this purpose. Other exception handling aspects are backed by a number of <a class="mw-redirect" href="/wiki/Win32_API" title="Win32 API">Win32 API</a> functions,<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> for example, <code>RaiseException</code> to raise SEH exceptions manually.</p>
<h3><span class="mw-headline" id="Implementation">Implementation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft-specific_exception_handling_mechanisms&amp;action=edit&amp;section=3" title="Edit section: Implementation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Each <a href="/wiki/Thread_(computing)" title="Thread (computing)">thread of execution</a> in Windows has a link to an undocumented _EXCEPTION_REGISTRATION_RECORD <a class="mw-redirect" href="/wiki/List_(computing)" title="List (computing)">list</a> at the start of its <a class="mw-redirect" href="/wiki/Thread_Information_Block" title="Thread Information Block">Thread Information Block</a>. The <code>__try</code> statement essentially calls a compiler-defined <code>EH_prolog</code> function. That function allocates an _EXCEPTION_REGISTRATION_RECORD <a href="/wiki/Stack-based_memory_allocation" title="Stack-based memory allocation">on the stack</a> pointing to <code>__except_handler3</code><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[a]</a></sup> function in <code>msvcrt.dll</code>,<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[b]</a></sup> then adds the record to the list's head. At the end of the <code>__try</code> <a href="/wiki/Block_(programming)" title="Block (programming)">block</a> a compiler-defined <code>EH_epilog</code> function is called that does the reverse operation. Either of these compiler-defined routines can be <a href="/wiki/Inline_expansion" title="Inline expansion">inline</a>. All the programmer-defined <code>__except</code> and <code>__finally</code> blocks are called from within <code>__except_handler3</code>. If such blocks are present, _EXCEPTION_REGISTRATION_RECORD being created is extended with a few additional fields used by <code>__except_handler3</code>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[4]</a></sup></p>
<p>In a case of an exception in a <a class="mw-redirect" href="/wiki/User_mode" title="User mode">user mode</a> code, the operating system<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[c]</a></sup> parses the thread's _EXCEPTION_REGISTRATION_RECORD list and calls each exception handler in sequence until a handler signals it has handled the exception (by <a class="mw-redirect" href="/wiki/Return_value" title="Return value">return value</a>) or the list is exhausted. The last one in the list is always the <code>kernel32!UnhandledExceptionFilter</code> which displays the <a href="/wiki/General_protection_fault" title="General protection fault">General protection fault</a> error message.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[d]</a></sup> Then the list is traversed once more giving handlers a chance to clean up any resources used. Finally, the execution returns to <a class="mw-redirect" href="/wiki/Kernel_mode" title="Kernel mode">kernel mode</a><sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[e]</a></sup> where the process is either resumed or terminated.</p>
<h2><span class="mw-headline" id="Vectored_Exception_Handling"><span id="VEH"></span>Vectored Exception Handling</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microsoft-specific_exception_handling_mechanisms&amp;action=edit&amp;section=4" title="Edit section: Vectored Exception Handling">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Vectored Exception Handling was introduced in <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a>.<sup class="reference" id="cite_ref-VEH_10-0"><a href="#cite_note-VEH-10">[5]</a></sup> Vectored Exception Handling is made available to Windows programmers using languages such as <a href="/wiki/C%2B%2B" title="C++">C++</a> and <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic</a>. VEH does not replace Structured Exception Handling (SEH), rather VEH and SEH coexist, with VEH handlers having priority over SEH handlers.<sup class="reference" id="cite_ref-devx_1-1"><a href="#cite_note-devx-1">[1]</a></sup><sup class="reference" id="cite_ref-VEH_10-1"><a href="#cite_note-VEH-10">[5]</a></sup> Compared with SEH, VEH works more like a traditional <a class="new" href="/w/index.php?title=Notification_callback_scheme&amp;action=edit&amp;redlink=1" title="Notification callback scheme (page does not exist)">notification callback scheme</a>.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[6]</a></sup></p>
