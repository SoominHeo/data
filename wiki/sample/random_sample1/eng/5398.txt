<h1 class="firstHeading" id="firstHeading" lang="en">McEliece cryptosystem</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p>In <a href="/wiki/Cryptography" title="Cryptography">cryptography</a>, the <b>McEliece cryptosystem</b> is an <a class="mw-redirect" href="/wiki/Asymmetric_encryption" title="Asymmetric encryption">asymmetric encryption</a> algorithm developed in 1978 by <a href="/wiki/Robert_McEliece" title="Robert McEliece">Robert McEliece</a>.<sup class="reference" id="cite_ref-McEliece_1-0"><a href="#cite_note-McEliece-1">[1]</a></sup> It was the first such scheme to use <a href="/wiki/Randomized_algorithm" title="Randomized algorithm">randomization</a> in the encryption process. The algorithm has never gained much acceptance in the cryptographic community, but is a candidate for "<a href="/wiki/Post-quantum_cryptography" title="Post-quantum cryptography">post-quantum cryptography</a>", as it is immune to attacks using <a href="/wiki/Shor%27s_algorithm" title="Shor's algorithm">Shor's algorithm</a> and — more generally — measuring cost states using Fourier sampling.<sup class="reference" id="cite_ref-quantum-fourier_2-0"><a href="#cite_note-quantum-fourier-2">[2]</a></sup></p>
<p>The algorithm is based on the hardness of <a href="/wiki/Decoding_methods#Syndrome_decoding" title="Decoding methods">decoding</a> a general <a href="/wiki/Linear_code" title="Linear code">linear code</a> (which is known to be <a class="mw-redirect" href="/wiki/NP-hard" title="NP-hard">NP-hard</a><sup class="reference" id="cite_ref-intractability_3-0"><a href="#cite_note-intractability-3">[3]</a></sup>). For a description of the private key, an <a class="mw-redirect" href="/wiki/Error-correcting_code" title="Error-correcting code">error-correcting code</a> is selected for which an efficient decoding algorithm is known, and which is able to correct <span><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML">
<semantics>

The original algorithm uses <a href="/wiki/Binary_Goppa_code" title="Binary Goppa code">binary Goppa codes</a> (subfield codes of geometric <a href="/wiki/Goppa_code" title="Goppa code">Goppa codes</a> of a genus-0 curve over finite fields of characteristic 2); these codes are easy to decode, thanks to an efficient algorithm due to Patterson.<sup class="reference" id="cite_ref-Patterson_4-0"><a href="#cite_note-Patterson-4">[4]</a></sup> The public key is derived from the private key by disguising the selected code as a general linear code. For this, the code's <a href="/wiki/Generator_matrix" title="Generator matrix">generator matrix</a> <span><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML">

<p>Variants of this cryptosystem exist, using different types of codes. Most of them were proven less secure; they were broken by structural decoding. On the other hand, Wang <sup class="reference" id="cite_ref-Wang_5-0"><a href="#cite_note-Wang-5">[5]</a></sup> proposed a secure McEliece scheme based on any efficient linear code, the hardness of Wang's variants depend on the NP-hardness of decoding random linear code.</p>
<p>McEliece with Goppa codes has resisted cryptanalysis so far. The most effective attacks known use <a href="/wiki/Decoding_methods#Syndrome_decoding" title="Decoding methods">information-set decoding</a> algorithms. A 2008 paper describes both an attack and a fix.<sup class="reference" id="cite_ref-fix_6-0"><a href="#cite_note-fix-6">[6]</a></sup> Another paper shows that for quantum computing, key sizes must be increased by a factor of four due to improvements in information set decoding.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></p>
<p>The McEliece cryptosystem has some advantages over, for example, <a class="mw-redirect" href="/wiki/RSA_(algorithm)" title="RSA (algorithm)">RSA</a>. The encryption and decryption are faster (for comparative benchmarks see the eBATS benchmarking project at <a class="external text" href="http://bench.cr.yp.to" rel="nofollow">bench.cr.yp.to</a>). For a long time, it was thought that McEliece could not be used to produce <a href="/wiki/Digital_signature" title="Digital signature">signatures</a>. However, a signature scheme can be constructed based on the <a href="/wiki/Niederreiter_cryptosystem" title="Niederreiter cryptosystem">Niederreiter</a> scheme, the dual variant of the McEliece scheme. One of the main disadvantages of McEliece is that the private and public keys are large matrices. For a standard selection of parameters, the public key is 512 kilobits long. This is why the algorithm is rarely used in practice. One exceptional case that used McEliece for encryption is the <a href="/wiki/Freenet" title="Freenet">Freenet</a>-like application <a href="/wiki/Entropy_(anonymous_data_store)" title="Entropy (anonymous data store)">Entropy</a>.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>

<h3><span class="mw-headline" id="Information_set_decoding">Information set decoding</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=McEliece_cryptosystem&amp;action=edit&amp;section=9" title="Edit section: Information set decoding">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a class="new" href="/w/index.php?title=Information_set_decoding&amp;action=edit&amp;redlink=1" title="Information set decoding (page does not exist)">Information set decoding</a> algorithms have turned out to be the most effective attacks against the McEliece and Niederreiter cryptosystems. Various forms have been introduced. An effective method is based on finding minimum- or low-weight codewords (see, for example, Canteaut &amp; Sendrier 1998<sup class="reference" id="cite_ref-min-weight_10-0"><a href="#cite_note-min-weight-10">[10]</a></sup>). In 2008, Bernstein, Lange and Peters<sup class="reference" id="cite_ref-fix_6-2"><a href="#cite_note-fix-6">[6]</a></sup> described a practical attack on the original McEliece cryptosystem, based on finding low-weight code words using an algorithm published by <a href="/wiki/Jacques_Stern" title="Jacques Stern">Jacques Stern</a> in 1989.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> Using the parameters originally suggested by McEliece, the attack could be carried out in 2<sup>60.55</sup> bit operations. Since the attack is <a href="/wiki/Embarrassingly_parallel" title="Embarrassingly parallel">embarrassingly parallel</a> (no communication between nodes is necessary), it can be carried out in days on modest computer clusters.</p>

<p>The McEliece cryptosystem has been implemented in the secure Instant Messenger and E-Mail Client <a class="external free" href="http://goldbug.sf.net" rel="nofollow">http://goldbug.sf.net</a> since the version 3.1 release (McEliece Release).</p>
