<h1 class="firstHeading" id="firstHeading" lang="en">Macroblock</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><b>Macroblock</b> is a processing unit in <a href="/wiki/Image_compression" title="Image compression">image</a> and <a class="mw-redirect" href="/wiki/Video_compression" title="Video compression">video compression</a> formats based on linear block transforms, such as the <a href="/wiki/Discrete_cosine_transform" title="Discrete cosine transform">discrete cosine transform</a> (DCT). A macroblock typically consists of 16×16 samples, and is further subdivided into transform blocks, and may be further subdivided into prediction blocks. Formats which are based on macroblocks include <a href="/wiki/JPEG" title="JPEG">JPEG</a>, where they are called <a href="/wiki/JPEG#Block_splitting" title="JPEG">MCU blocks</a>, <a href="/wiki/H.261" title="H.261">H.261</a>, <a class="mw-redirect" href="/wiki/MPEG-1_Part_2" title="MPEG-1 Part 2">MPEG-1 Part 2</a>, <a href="/wiki/H.262/MPEG-2_Part_2" title="H.262/MPEG-2 Part 2">H.262/MPEG-2 Part 2</a>, <a href="/wiki/H.263" title="H.263">H.263</a>, <a href="/wiki/MPEG-4_Part_2" title="MPEG-4 Part 2">MPEG-4 Part 2</a>, and <a href="/wiki/H.264/MPEG-4_AVC" title="H.264/MPEG-4 AVC">H.264/MPEG-4 AVC</a>.<sup class="reference" id="cite_ref-H261_1-0"><a href="#cite_note-H261-1">[1]</a></sup><sup class="reference" id="cite_ref-H262_2-0"><a href="#cite_note-H262-2">[2]</a></sup><sup class="reference" id="cite_ref-H263_3-0"><a href="#cite_note-H263-3">[3]</a></sup><sup class="reference" id="cite_ref-H264_4-0"><a href="#cite_note-H264-4">[4]</a></sup> In <a class="mw-redirect" href="/wiki/H.265/HEVC" title="H.265/HEVC">H.265/HEVC</a>, the macroblock as a basic processing unit has been replaced by the <a href="/wiki/Coding_tree_unit" title="Coding tree unit">coding tree unit</a>.<sup class="reference" id="cite_ref-OverviewHEVCIEEE2012_5-0"><a href="#cite_note-OverviewHEVCIEEE2012-5">[5]</a></sup></p>
<p></p>

<h2><span class="mw-headline" id="Technical_details">Technical details</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Macroblock&amp;action=edit&amp;section=1" title="Edit section: Technical details">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Transform_blocks">Transform blocks</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Macroblock&amp;action=edit&amp;section=2" title="Edit section: Transform blocks">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A macroblock is divided into transform blocks, which serve as input to the linear block transform, e.g. the DCT. In H.261, the first video codec to use macroblocks, transform blocks have a fixed size of 8×8 samples.<sup class="reference" id="cite_ref-H261_1-1"><a href="#cite_note-H261-1">[1]</a></sup> In the <a href="/wiki/YCbCr" title="YCbCr">YCbCr</a> color space with <a class="mw-redirect" href="/wiki/4:2:0" title="4:2:0">4:2:0</a> chroma subsampling, a 16×16 macroblock consists of 16×16 luma (Y) samples and 8×8 chroma (Cb and Cr) samples. These samples are split into four Y blocks, one Cb block and one Cr block. This design is also used in <a href="/wiki/JPEG" title="JPEG">JPEG</a> and most other macroblock-based video codecs with a fixed transform block size, such as MPEG-1 Part 2 and H.262/MPEG-2 Part 2. In other <a href="/wiki/Chroma_subsampling" title="Chroma subsampling">chroma subsampling</a> formats, e.g. 4:0:0, 4:2:2, or 4:4:4, the number of chroma samples in a macroblock will be smaller or larger, and the grouping of chroma samples into blocks will differ accordingly.</p>
<p>In more modern macroblock-based video coding standards such as H.263 and H.264/AVC, transform blocks can be of sizes other than 8×8 samples. For instance, in H.264/AVC main profile, the transform block size is 4×4.<sup class="reference" id="cite_ref-H264_4-1"><a href="#cite_note-H264-4">[4]</a></sup> In H.264/AVC High profile, the transform block size can be either 4×4 or 8×8, adapted on a per-macroblock basis.<sup class="reference" id="cite_ref-H264_4-2"><a href="#cite_note-H264-4">[4]</a></sup></p>
<h3><span class="mw-headline" id="Prediction_blocks">Prediction blocks</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Macroblock&amp;action=edit&amp;section=3" title="Edit section: Prediction blocks">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Distinct from the division into transform blocks, a macroblock can be split into prediction blocks. In early standards such as H.261, MPEG-1 Part 2, and H.262/MPEG-2 Part 2, motion compensation is performed with one motion vector per macroblock.<sup class="reference" id="cite_ref-H261_1-2"><a href="#cite_note-H261-1">[1]</a></sup><sup class="reference" id="cite_ref-H262_2-1"><a href="#cite_note-H262-2">[2]</a></sup> In more modern standards such as H.264/AVC, a macroblock can be split into multiple variable-sized prediction blocks, called partitions.<sup class="reference" id="cite_ref-H264_4-3"><a href="#cite_note-H264-4">[4]</a></sup> In an inter-predicted macroblock in H.264/AVC, a separate motion vector is specified for each partition.<sup class="reference" id="cite_ref-H264_4-4"><a href="#cite_note-H264-4">[4]</a></sup> Correspondingly, in an intra-predicted macroblock, where samples are predicted by extrapolating from the edges of neighboring blocks, the prediction direction that is specified on a per-partition basis.<sup class="reference" id="cite_ref-H264_4-5"><a href="#cite_note-H264-4">[4]</a></sup> In H.264/AVC, prediction partition size ranges from 4×4 to 16×16 samples for both inter-prediction (motion compensation) and intra-prediction.<sup class="reference" id="cite_ref-H264_4-6"><a href="#cite_note-H264-4">[4]</a></sup></p>
<h2><span class="mw-headline" id="Bitstream_representation">Bitstream representation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Macroblock&amp;action=edit&amp;section=4" title="Edit section: Bitstream representation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A possible bitstream representation of a macroblock in a video codec which uses <a href="/wiki/Motion_compensation" title="Motion compensation">motion compensation</a> and <a href="/wiki/Transform_coding#Digital" title="Transform coding">transform coding</a> is given below.<sup class="reference" id="cite_ref-D-Marshall_6-0"><a href="#cite_note-D-Marshall-6">[6]</a></sup> It is similar to the format used in <a href="/wiki/H.261" title="H.261">H.261</a>.<sup class="reference" id="cite_ref-H261_1-3"><a href="#cite_note-H261-1">[1]</a></sup></p>
<pre>
+------+------+-------+--------+-----+----+----+--------+
| ADDR | TYPE | QUANT | VECTOR | CBP | b0 | b1 | ... b5 |
+------+------+-------+--------+-----+----+----+--------+
</pre>
<ul>
<li>ADDR — address of block in image</li>
<li>TYPE — identifies type of macroblock (<a class="mw-redirect" href="/wiki/Intra_frame" title="Intra frame">intra frame</a>, <a href="/wiki/Inter_frame" title="Inter frame">inter frame</a>, <a class="mw-redirect" href="/wiki/B-frame" title="B-frame">bi-directional inter frame</a>)</li>
<li>QUANT — quantization value to vary quantization</li>
<li>VECTOR - <a href="/wiki/Motion_vector" title="Motion vector">motion vector</a></li>
<li>CBP — Coded Block Pattern, this is bit mask indicating for which blocks coefficients are present.</li>
<li>bN — the blocks (4 Y, 1 Cr, 1 Cb)</li>
</ul>
