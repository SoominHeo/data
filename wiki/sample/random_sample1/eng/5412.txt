<p>In <a href="/wiki/Computing" title="Computing">computing</a>, <b>Mantle</b> is a low-overhead <a class="mw-redirect" href="/wiki/Rendering_API" title="Rendering API">rendering API</a> targeted at 3D video games.<sup class="reference" id="cite_ref-whitepaper_3-0"><a href="#cite_note-whitepaper-3">[3]</a></sup> <a href="/wiki/Advanced_Micro_Devices" title="Advanced Micro Devices">AMD</a> originally developed Mantle in cooperation with <a href="/wiki/EA_DICE" title="EA DICE">DICE</a>, starting in 2013.<sup class="reference" id="cite_ref-Dave_Altavilla_1-1"><a href="#cite_note-Dave_Altavilla-1">[1]</a></sup> Mantle was designed as an alternative to <a href="/wiki/Direct3D" title="Direct3D">Direct3D</a> and <a href="/wiki/OpenGL" title="OpenGL">OpenGL</a>, primarily for use on <a href="/wiki/Personal_computer" title="Personal computer">personal computers</a>, although Mantle supports the GPUs present in the <a href="/wiki/PlayStation_4" title="PlayStation 4">PlayStation 4</a> and in the <a href="/wiki/Xbox_One" title="Xbox One">Xbox One</a>.<sup class="reference" id="cite_ref-Dave_Altavilla_1-2"><a href="#cite_note-Dave_Altavilla-1">[1]</a></sup><sup class="reference" id="cite_ref-tomshardware_4-0"><a href="#cite_note-tomshardware-4">[4]</a></sup> According to AMD, Mantle will make a shift in focus after March 2015 to other areas since <a href="/wiki/DirectX#DirectX_12" title="DirectX">DirectX 12</a> and the Mantle-derived <a href="/wiki/Vulkan_(API)" title="Vulkan (API)">Vulkan API</a> are largely replacing it in the <a href="/wiki/Video_game" title="Video game">gaming</a> industry. Therefore, Mantle will be supported by AMD in neither the short term nor long term future, effectively making all future games unable to implement Mantle, and making Mantle unable to be implemented by any older games.<sup class="reference" id="cite_ref-anandtech_mantle_2-1"><a href="#cite_note-anandtech_mantle-2">[2]</a></sup><sup class="reference" id="cite_ref-amd_announcement_5-0"><a href="#cite_note-amd_announcement-5">[5]</a></sup> While supported in drivers for two years, the Mantle API itself was not made public until March 2015.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>

<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Mantle_(API)&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The draw call improvements of Mantle help alleviate cases where the CPU is the bottleneck. The design goals of Mantle are to allow games and applications to utilize the <a href="/wiki/Central_processing_unit" title="Central processing unit">CPUs</a> and <a href="/wiki/Graphics_processing_unit" title="Graphics processing unit">GPUs</a> more efficiently, eliminate CPU bottlenecks by reducing API validation overhead and allowing more effective scaling on multiple CPU cores, provide faster draw routines, and allow greater control over the graphics pipeline by eliminating certain aspects of hardware abstraction inherent to both current prevailing graphics APIs OpenGL and Direct3D.<sup class="reference" id="cite_ref-benchmarks_7-0"><a href="#cite_note-benchmarks-7">[7]</a></sup></p>
<h3><span class="mw-headline" id="CPU-bound_scenarios">CPU-bound scenarios</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Mantle_(API)&amp;action=edit&amp;section=2" title="Edit section: CPU-bound scenarios">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>With a basic implementation, Mantle was designed to improve performance in scenarios where the CPU is the limiting factor:</p>
<ul>
<li>Low-overhead validation and processing of API commands<sup class="reference" id="cite_ref-mantle101_8-0"><a href="#cite_note-mantle101-8">[8]</a></sup><sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></li>
<li>Explicit command buffer control<sup class="reference" id="cite_ref-mantle101_8-1"><a href="#cite_note-mantle101-8">[8]</a></sup></li>
<li>Close to linear performance scaling from reordering command buffers onto multiple CPU cores<sup class="reference" id="cite_ref-mantle101_8-2"><a href="#cite_note-mantle101-8">[8]</a></sup></li>
<li>Reduced runtime shader compilation overhead<sup class="reference" id="cite_ref-mantle101_8-3"><a href="#cite_note-mantle101-8">[8]</a></sup></li>
<li>AMD claims that Mantle can generate up to 9 times more draw calls per second than comparable APIs by reducing CPU overhead.<sup class="reference" id="cite_ref-amd-livestream_10-0"><a href="#cite_note-amd-livestream-10">[10]</a></sup></li>
<li>Multithreaded parallel CPU rendering support for at least 8 cores.<sup class="reference" id="cite_ref-anandtech_11-0"><a href="#cite_note-anandtech-11">[11]</a></sup></li>
</ul>
<h3><span class="mw-headline" id="GPU-bound_scenarios">GPU-bound scenarios</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Mantle_(API)&amp;action=edit&amp;section=3" title="Edit section: GPU-bound scenarios">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Mantle is also designed to improve situations where high resolutions and “maximum detail” settings are used, although to a somewhat lesser degree, as these settings tax GPU resources in a way that is more difficult to improve at the API level. While Mantle provides some built-in features to improve GPU-bound performance, gains in these cases are largely dependent on how well Mantle features and optimizations are being utilized by the game engine. Some of those features include:<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup><sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></p>
<ul>
<li>Reduction of command buffers submissions</li>
<li>Explicit control of resource compression, expands and synchronizations</li>
<li>Asynchronous DMA queue for data uploads independent from the graphics engine</li>
<li>Asynchronous compute queue for overlapping of compute and graphics workloads</li>
<li>Data formats optimizations via flexible buffer/image access</li>
<li>Advanced Anti-Aliasing features for MSAA/EQAA optimizations<sup class="reference" id="cite_ref-whitepaper_3-1"><a href="#cite_note-whitepaper-3">[3]</a></sup><sup class="reference" id="cite_ref-mantle101_8-4"><a href="#cite_note-mantle101-8">[8]</a></sup></li>
<li>Native multi-GPU support<sup class="reference" id="cite_ref-whitepaper_3-2"><a href="#cite_note-whitepaper-3">[3]</a></sup></li>
</ul>
<h3><span class="mw-headline" id="Benchmarks">Benchmarks</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Mantle_(API)&amp;action=edit&amp;section=4" title="Edit section: Benchmarks">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>Performance superior to Direct3D 11<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup></li>
<li>Improved performance in <i><a href="/wiki/Battlefield_4" title="Battlefield 4">Battlefield 4</a></i><sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup><sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup><sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> and up to 319% faster in the <i><a class="new" href="/w/index.php?title=Star_Swarm&amp;action=edit&amp;redlink=1" title="Star Swarm (page does not exist)">Star Swarm</a></i> demo in single GPU configuration<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup><sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup><sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> in extremely CPU-limited situations.</li>
</ul>

<h2><span class="mw-headline" id="Support">Support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Mantle_(API)&amp;action=edit&amp;section=6" title="Edit section: Support">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

<h2><span class="mw-headline" id="Similar_technologies">Similar technologies</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Mantle_(API)&amp;action=edit&amp;section=9" title="Edit section: Similar technologies">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A set of recent OpenGL 4.4 features, coupled with bindless texturing as an extension, can also substantially reduce driver overhead. This approach, termed by the Khronos Group as "<a class="mw-redirect" href="/wiki/AZDO" title="AZDO">AZDO</a>" (Approaching Zero Driver Overhead) has been shown to achieve substantial performance improvements, approaching those stated for Mantle.<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup><sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup> <a href="/wiki/Nvidia" title="Nvidia">Nvidia</a> has extended OpenGL with a number of features that further reduce driver overhead.<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup></p>
<p>After details about DirectX 12 were made public, AMD has stated that they fully intend to support DirectX 12, but at the same time they claimed that Mantle "will [still] do some things faster." They have also claimed that due to similarities in the design philosophy of the two APIs, porting games from Mantle to DirectX 12 will be relatively straightforward,<sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup> and easier than porting from DirectX 11 to 12.<sup class="reference" id="cite_ref-easyport_21-2"><a href="#cite_note-easyport-21">[21]</a></sup></p>
<p>Ultimately, AMD discontinued Mantle as a game API due to the similar aims DirectX 12 and glNext.<sup class="reference" id="cite_ref-anandtech_mantle_2-2"><a href="#cite_note-anandtech_mantle-2">[2]</a></sup><sup class="reference" id="cite_ref-amd_announcement_5-1"><a href="#cite_note-amd_announcement-5">[5]</a></sup> AMD donated the Mantle API to the Khronos group, which developed it into the Vulkan API.<sup class="reference" id="cite_ref-More_on_Vulkan_38-0"><a href="#cite_note-More_on_Vulkan-38">[38]</a></sup><sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[39]</a></sup><sup class="reference" id="cite_ref-40"><a href="#cite_note-40">[40]</a></sup><sup class="reference" id="cite_ref-not-dead-yet_41-0"><a href="#cite_note-not-dead-yet-41">[41]</a></sup><sup class="reference" id="cite_ref-42"><a href="#cite_note-42">[42]</a></sup><sup class="reference" id="cite_ref-43"><a href="#cite_note-43">[43]</a></sup></p>

<h2><span class="mw-headline" id="Recording_and_FPS_Overlay_Software">Recording and FPS Overlay Software</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Mantle_(API)&amp;action=edit&amp;section=11" title="Edit section: Recording and FPS Overlay Software">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>PC gamers and professionals traditionally used programs such as <a href="/wiki/Fraps" title="Fraps">Fraps</a> and <a href="/wiki/Bandicam" title="Bandicam">Bandicam</a> to record gameplay, measure game FPS and display FPS overlay, but because Mantle is new, most traditional recording software does not work with new titles while using the new API.</p>
<p>In partnership with AMD, PC gaming community and game recording software maker <a href="/wiki/Raptr" title="Raptr">Raptr</a> have overhauled their client and have since re-branded it as the <i>AMD Gaming Evolved</i> client in conjunction with AMD's Gaming Evolved initiative in the PC gaming space. Out of the partnership, players who install and use the client while in-game can earn points to spend on digital items like games or computer hardware, chat with friends, keep their game library optimized, check for graphics card driver updates, stream their games to <a href="/wiki/Twitch.tv" title="Twitch.tv">Twitch.tv</a> and record gameplay of their own with a built-in GVR, a feature similar to <a class="mw-redirect" href="/wiki/Nvidia_Shadowplay" title="Nvidia Shadowplay">Nvidia Shadowplay</a> software in its own GeForce Experience software that allows users to define a custom buffer length in their game for retroactive game recording with the push of a button so no moment gets missed and users typically do not need expensive hard drive setups to record to. In late 2014, AMD updated the client to finally support the recording and streaming of titles using Mantle.<sup class="reference" id="cite_ref-44"><a href="#cite_note-44">[44]</a></sup> As of its initial update into the client, the Gaming Evolved software is still the only software to officially support the recording and streaming of Mantle enabled games.</p>
<p>Besides Raptr, D3DGear<sup class="reference" id="cite_ref-45"><a href="#cite_note-45">[45]</a></sup> is the only other commercial game recording software that supports Mantle API based games.</p>
