<h1 class="firstHeading" id="firstHeading" lang="en">Monoid</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><div class="hatnote" role="note">For other uses, see <a class="mw-disambig" href="/wiki/Monoid_(disambiguation)" title="Monoid (disambiguation)">Monoid (disambiguation)</a>.</div>
<div class="hatnote" role="note">Not to be confused with <a class="mw-redirect mw-disambig" href="/wiki/Monad_(disambiguation)" title="Monad (disambiguation)">monad</a>.</div>
<p>In <a href="/wiki/Abstract_algebra" title="Abstract algebra">abstract algebra</a>, a branch of <a href="/wiki/Mathematics" title="Mathematics">mathematics</a>, a <b>monoid</b> is an <a href="/wiki/Algebraic_structure" title="Algebraic structure">algebraic structure</a> with a single <a class="mw-redirect" href="/wiki/Associative" title="Associative">associative</a> <a href="/wiki/Binary_operation" title="Binary operation">binary operation</a> and an <a href="/wiki/Identity_element" title="Identity element">identity element</a>. Monoids are studied in <a href="/wiki/Semigroup" title="Semigroup">semigroup</a> theory as they are semigroups with identity. Monoids occur in several branches of mathematics; for instance, they can be regarded as <a href="/wiki/Category_(mathematics)" title="Category (mathematics)">categories</a> with a single <a class="mw-redirect" href="/wiki/Object_(category_theory)" title="Object (category theory)">object</a>. Thus, they capture the idea of <a href="/wiki/Function_composition" title="Function composition">function composition</a> within a set. In fact, all functions from a set into itself form naturally a monoid with respect to function composition. Monoids are also commonly used in <a href="/wiki/Computer_science" title="Computer science">computer science</a>, both in its foundational aspects and in practical programming. The set of <a href="/wiki/String_(computer_science)" title="String (computer science)">strings</a> built from a given set of <a href="/wiki/Character_(computing)" title="Character (computing)">characters</a> is a <a href="/wiki/Free_monoid" title="Free monoid">free monoid</a>. The <a class="mw-redirect" href="/wiki/Transition_monoid" title="Transition monoid">transition monoid</a> and <a href="/wiki/Syntactic_monoid" title="Syntactic monoid">syntactic monoid</a> are used in describing <a class="mw-redirect" href="/wiki/Finite_state_machines" title="Finite state machines">finite state machines</a>, whereas <a href="/wiki/Trace_monoid" title="Trace monoid">trace monoids</a> and <a href="/wiki/History_monoid" title="History monoid">history monoids</a> provide a foundation for <a class="mw-redirect" href="/wiki/Process_calculi" title="Process calculi">process calculi</a> and <a href="/wiki/Concurrent_computing" title="Concurrent computing">concurrent computing</a>. Some of the more important results in the study of monoids are the <a class="mw-redirect" href="/wiki/Krohn%E2%80%93Rhodes_theorem" title="Krohn–Rhodes theorem">Krohn–Rhodes theorem</a> and the <a href="/wiki/Star_height_problem" title="Star height problem">star height problem</a>. The history of monoids, as well as a discussion of additional general properties, are found in the article on <a href="/wiki/Semigroup" title="Semigroup">semigroups</a>.</p>

<h2><span class="mw-headline" id="Definition">Definition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=1" title="Edit section: Definition">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Suppose that <i>S</i> is a <a href="/wiki/Set_(mathematics)" title="Set (mathematics)">set</a> and • is some <a href="/wiki/Binary_operation" title="Binary operation">binary operation</a> <span class="nowrap"><i>S</i> × <i>S</i> → <i>S</i></span>, then <i>S</i> with • is a <b>monoid</b> if it satisfies the following two axioms:</p>
<dl>
<dt>Associativity</dt>
<dd>For all <i>a</i>, <i>b</i> and <i>c</i> in <i>S</i>, the equation <span class="nowrap">(<i>a</i> • <i>b</i>) • <i>c</i> = <i>a</i> • (<i>b</i> • <i>c</i>)</span> holds.</dd>
<dt>Identity element</dt>
<dd>There exists an element <i>e</i> in <i>S</i> such that for every element <i>a</i> in <i>S</i>, the equations <span class="nowrap"><i>e</i> • <i>a</i> = <i>a</i> • <i>e</i> = <i>a</i></span> hold.</dd>
</dl>
<p>In other words, a monoid is a <a href="/wiki/Semigroup" title="Semigroup">semigroup</a> with an <a href="/wiki/Identity_element" title="Identity element">identity element</a>. It can also be thought of as a <a href="/wiki/Magma_(algebra)" title="Magma (algebra)">magma</a> with associativity and identity. The identity element of a monoid is unique.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> A monoid in which each element has an <a href="/wiki/Inverse_element" title="Inverse element">inverse</a> is a <a href="/wiki/Group_(mathematics)" title="Group (mathematics)">group</a>.</p>

This notation does not imply that it is numbers being multiplied.</p>
<h2><span class="mw-headline" id="Monoid_structures">Monoid structures</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=2" title="Edit section: Monoid structures">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Submonoids">Submonoids</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=3" title="Edit section: Submonoids">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A <b>submonoid</b> of a monoid <span class="nowrap">(<i>M</i>, •)</span> is a <a href="/wiki/Subset" title="Subset">subset</a> <i>N</i> of <i>M</i> that is closed under the monoid operation and contains the identity element <i>e</i> of <i>M</i>.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> Symbolically, <i>N</i> is a submonoid of <i>M</i> if <span class="nowrap"><i>N</i> ⊆ <i>M</i></span>, <span class="nowrap"><i>x</i> • <i>y</i> ∈ <i>N</i></span> whenever <span class="nowrap"><i>x</i>, <i>y</i> ∈ <i>N</i></span>, and <span class="nowrap"><i>e</i> ∈ <i>N</i></span>. <i>N</i> is thus a monoid under the binary operation inherited from <i>M</i>.</p>
<h3><span class="mw-headline" id="Generators">Generators</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=4" title="Edit section: Generators">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A subset <i>S</i> of <i>M</i> is said to be a <b>generator</b> of <i>M</i> if <i>M</i> is the smallest set containing <i>S</i> that is <a href="/wiki/Closure_(mathematics)" title="Closure (mathematics)">closed</a> under the monoid operation, or equivalently <i>M</i> is the result of applying the <a class="mw-redirect" href="/wiki/Finitary_closure_operator" title="Finitary closure operator">finitary closure operator</a> to <i>S</i>. If there is a generator of <i>M</i> that has finite cardinality, then <i>M</i> is said to be <b>finitely generated</b>. Not every set <i>S</i> will generate a monoid, as the generated structure may lack an identity element.</p>
<h3><span class="mw-headline" id="Commutative_monoid">Commutative monoid</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=5" title="Edit section: Commutative monoid">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A monoid whose operation is <a class="mw-redirect" href="/wiki/Commutative" title="Commutative">commutative</a> is called a <b>commutative monoid</b> (or, less commonly, an <b>abelian monoid</b>). Commutative monoids are often written additively. Any commutative monoid is endowed with its <b>algebraic</b> <a href="/wiki/Preorder" title="Preorder">preordering</a> ≤, defined by <span class="nowrap"><i>x</i> ≤ <i>y</i></span> if there exists <i>z</i> such that <span class="nowrap"><i>x</i> + <i>z</i> = <i>y</i></span>.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> An <b>order-unit</b> of a commutative monoid <i>M</i> is an element <i>u</i> of <i>M</i> such that for any element <i>x</i> of <i>M</i>, there exists a positive integer <i>n</i> such that <span class="nowrap"><i>x</i> ≤ <i>nu</i></span>. This is often used in case <i>M</i> is the <a class="mw-redirect" href="/wiki/Ordered_group" title="Ordered group">positive cone</a> of a <a href="/wiki/Partially_ordered_set" title="Partially ordered set">partially ordered</a> <a href="/wiki/Abelian_group" title="Abelian group">abelian group</a> <i>G</i>, in which case we say that <i>u</i> is an order-unit of <i>G</i>.</p>
<h3><span class="mw-headline" id="Partially_commutative_monoid">Partially commutative monoid</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=6" title="Edit section: Partially commutative monoid">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A monoid for which the operation is commutative for some, but not all elements is a <a href="/wiki/Trace_monoid" title="Trace monoid">trace monoid</a>; trace monoids commonly occur in the theory of <a class="mw-redirect" href="/wiki/Concurrent_computation" title="Concurrent computation">concurrent computation</a>.</p>
<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=7" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>Out of the 16 possible <a href="/wiki/Truth_table#Truth_table_for_all_binary_logical_operators" title="Truth table">binary Boolean operators</a>, each of the four that has a two sided identity is also commutative and associative and thus makes the set {False, True} a commutative monoid. Under the standard definitions, <a href="/wiki/Logical_conjunction" title="Logical conjunction">AND</a> and <a href="/wiki/Logical_biconditional" title="Logical biconditional">XNOR</a> have the identity True while <a class="mw-redirect" href="/wiki/Exclusive_disjunction" title="Exclusive disjunction">XOR</a> and <a href="/wiki/Logical_disjunction" title="Logical disjunction">OR</a> have the identity False. The monoids from AND and OR are also <a class="mw-redirect" href="/wiki/Idempotent" title="Idempotent">idempotent</a> while those from XOR and XNOR are not.</li>
<li>The <a href="/wiki/Natural_number" title="Natural number">natural numbers</a>, <b>N</b>, form a commutative monoid under addition (identity element <a class="mw-redirect" href="/wiki/0_(number)" title="0 (number)">zero</a>), or multiplication (identity element <a class="mw-redirect" href="/wiki/1_(number)" title="1 (number)">one</a>). A submonoid of <b>N</b> under addition is called a <a class="mw-redirect" href="/wiki/Numerical_monoid" title="Numerical monoid">numerical monoid</a>.</li>
<li>The <a class="mw-redirect" href="/wiki/Positive_integer" title="Positive integer">positive integers</a>, <span class="nowrap"><b>N</b> ∖ {0}</span>, form a commutative monoid under multiplication (identity element one).</li>
<li>Given a set A, all subsets of A form a commutative monoid under intersection operation (identity element is A itself).</li>
<li>Given a set A, all subsets of A form a commutative monoid under union operation (identity element is the empty set).</li>
<li>Generalizing the previous example, every bounded <a href="/wiki/Semilattice" title="Semilattice">semilattice</a> is an <a class="mw-redirect" href="/wiki/Idempotent" title="Idempotent">idempotent</a> commutative monoid.
<ul>
<li>In particular, any bounded <a href="/wiki/Lattice_(order)" title="Lattice (order)">lattice</a> can be endowed with both a <a href="/wiki/Join_and_meet" title="Join and meet">meet</a>- and a <a href="/wiki/Join_and_meet" title="Join and meet">join</a>- monoid structure. The identity elements are the lattice's top and its bottom, respectively. Being lattices, <a href="/wiki/Heyting_algebra" title="Heyting algebra">Heyting algebras</a> and <a href="/wiki/Boolean_algebra_(structure)" title="Boolean algebra (structure)">Boolean algebras</a> are endowed with these monoid structures.</li>
</ul>
</li>
<li>Every <a class="mw-redirect" href="/wiki/Singleton_set" title="Singleton set">singleton set</a> {<i>x</i>} closed under a binary operation • forms the trivial (one-element) monoid, which is also the <a href="/wiki/Trivial_group" title="Trivial group">trivial group</a>.</li>
<li>Every <a href="/wiki/Group_(mathematics)" title="Group (mathematics)">group</a> is a monoid and every <a href="/wiki/Abelian_group" title="Abelian group">abelian group</a> a commutative monoid.</li>
<li>Any <a href="/wiki/Semigroup" title="Semigroup">semigroup</a> <i>S</i> may be turned into a monoid simply by adjoining an element <i>e</i> not in <i>S</i> and defining <span class="nowrap"><i>e</i> • <i>s</i> = <i>s</i> = <i>s</i> • <i>e</i></span> for all <span class="nowrap"><i>s</i> ∈ <i>S</i></span>. This conversion of any semigroup to the monoid is done by the <a class="mw-redirect" href="/wiki/Free_functor" title="Free functor">free functor</a> between the <a class="new" href="/w/index.php?title=Category_of_semigroups&amp;action=edit&amp;redlink=1" title="Category of semigroups (page does not exist)">category of semigroups</a> and the <a class="mw-redirect" href="/wiki/Category_of_monoids" title="Category of monoids">category of monoids</a>.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup>
<ul>
<li>Thus, an idempotent monoid (sometimes known as <i>find-first</i>) may be formed by adjoining an identity element <i>e</i> to the <a class="mw-redirect" href="/wiki/Left_zero_semigroup" title="Left zero semigroup">left zero semigroup</a> over a set <i>S</i>. The opposite monoid (sometimes called <i>find-last</i>) is formed from the <a class="mw-redirect" href="/wiki/Right_zero_semigroup" title="Right zero semigroup">right zero semigroup</a> over <i>S</i>.
<ul>
<li>Adjoin an identity <i>e</i> to the left-zero semigroup with two elements {<i>lt</i>; <i>gt</i>}. Then the resulting idempotent monoid <span class="nowrap">{<i>lt</i>; <i>e</i>; <i>gt</i>}</span> models the <a href="/wiki/Lexicographical_order" title="Lexicographical order">lexicographical order</a> of a sequence given the orders of its elements, with <i>e</i> representing equality.</li>
</ul>
</li>
</ul>
</li>
<li>The elements of any <a class="mw-redirect" href="/wiki/Unital_algebra" title="Unital algebra">unital</a> <a class="mw-redirect" href="/wiki/Ring_(algebra)" title="Ring (algebra)">ring</a>, with addition or multiplication as the operation.
<ul>
<li>The <a href="/wiki/Integer" title="Integer">integers</a>, <a href="/wiki/Rational_number" title="Rational number">rational numbers</a>, <a href="/wiki/Real_number" title="Real number">real numbers</a> or <a href="/wiki/Complex_number" title="Complex number">complex numbers</a>, with addition or multiplication as operation.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></li>
<li>The set of all <i>n</i> by <i>n</i> <a href="/wiki/Matrix_(mathematics)" title="Matrix (mathematics)">matrices</a> over a given ring, with <a href="/wiki/Matrix_addition" title="Matrix addition">matrix addition</a> or <a href="/wiki/Matrix_multiplication" title="Matrix multiplication">matrix multiplication</a> as the operation.</li>
</ul>
</li>
<li>The set of all finite <a href="/wiki/String_(computer_science)" title="String (computer science)">strings</a> over some fixed alphabet Σ forms a monoid with <a class="mw-redirect" href="/wiki/String_concatenation" title="String concatenation">string concatenation</a> as the operation. The <a href="/wiki/Empty_string" title="Empty string">empty string</a> serves as the identity element. This monoid is denoted Σ<sup>∗</sup> and is called the <b><a href="/wiki/Free_monoid" title="Free monoid">free monoid</a></b> over Σ.</li>
<li>Given any monoid <i>M</i>, the <i>opposite monoid</i> M<sup>op</sup> has the same carrier set and identity element as <i>M</i>, and its operation is defined by <i>x</i> •<sup>op</sup> <span class="nowrap"><i>y</i> = <i>y</i> • <i>x</i></span>. Any <a class="mw-redirect" href="/wiki/Commutative_monoid" title="Commutative monoid">commutative monoid</a> is the opposite monoid of itself.</li>
<li>Given two sets <i>M</i> and <i>N</i> endowed with monoid structure (or, in general, any finite number of monoids, <i>M</i><sub>1</sub>, ..., <i>M<sub>k</sub></i>), their <a href="/wiki/Cartesian_product" title="Cartesian product">cartesian product</a> <span class="nowrap"><i>M</i> × <i>N</i></span> is also a monoid (respectively, <span class="nowrap"><i>M<sub>1</sub></i> × ... × <i>M<sub>k</sub></i></span>). The associative operation and the identity element are defined pairwise.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></li>
<li>Fix a monoid <i>M</i>. The set of all functions from a given set to <i>M</i> is also a monoid. The identity element is a <a href="/wiki/Constant_function" title="Constant function">constant function</a> mapping any value to the identity of <i>M</i>; the associative operation is defined <a href="/wiki/Pointwise" title="Pointwise">pointwise</a>.</li>
<li>Fix a monoid <i>M</i> with the operation • and identity element <i>e</i>, and consider its <a href="/wiki/Power_set" title="Power set">power set</a> <i>P</i>(<i>M</i>) consisting of all <a href="/wiki/Subset" title="Subset">subsets</a> of <i>M</i>. A binary operation for such subsets can be defined by <span class="nowrap"><i>S</i> • <i>T</i> = { <i>s</i> • <i>t</i> : <i>s</i> ∈ <i>S</i>, <i>t</i> ∈ <i>T</i> }</span>. This turns <i>P</i>(<i>M</i>) into a monoid with identity element {<i>e</i>}. In the same way the power set of a group <i>G</i> is a monoid under the <a href="/wiki/Product_of_group_subsets" title="Product of group subsets">product of group subsets</a>.</li>
<li>Let <i>S</i> be a set. The set of all functions <span class="nowrap"><i>S</i> → <i>S</i></span> forms a monoid under <a href="/wiki/Function_composition" title="Function composition">function composition</a>. The identity is just the <a href="/wiki/Identity_function" title="Identity function">identity function</a>. It is also called the <b><a class="mw-redirect" href="/wiki/Full_transformation_monoid" title="Full transformation monoid">full transformation monoid</a></b> of <i>S</i>. If <i>S</i> is finite with <i>n</i> elements, the monoid of functions on <i>S</i> is finite with <i>n</i><sup><i>n</i></sup> elements.</li>
<li>Generalizing the previous example, let <i>C</i> be a <a href="/wiki/Category_(mathematics)" title="Category (mathematics)">category</a> and <i>X</i> an object in <i>C</i>. The set of all <a href="/wiki/Endomorphism" title="Endomorphism">endomorphisms</a> of <i>X</i>, denoted End<sub><i>C</i></sub>(<i>X</i>), forms a monoid under composition of <a href="/wiki/Morphism" title="Morphism">morphisms</a>. For more on the relationship between category theory and monoids see below.</li>
<li>The set of <a href="/wiki/Homeomorphism" title="Homeomorphism">homeomorphism</a> <a href="/wiki/Class_(set_theory)" title="Class (set theory)">classes</a> of <a class="mw-redirect" href="/wiki/Compact_surface" title="Compact surface">compact surfaces</a> with the <a href="/wiki/Connected_sum" title="Connected sum">connected sum</a>. Its unit element is the class of the ordinary 2-sphere. Furthermore, if <i>a</i> denotes the class of the <a href="/wiki/Torus" title="Torus">torus</a>, and <i>b</i> denotes the class of the projective plane, then every element <i>c</i> of the monoid has a unique expression the form <span class="nowrap"><i>c</i> = <i>na</i> + <i>mb</i></span> where <i>n</i> is the integer ≥ 0 and <i>m</i> = 0, 1, or 2. We have <span class="nowrap">3<i>b</i> = <i>a</i> + <i>b</i></span>.</li>

<h2><span class="mw-headline" id="Properties">Properties</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=8" title="Edit section: Properties">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In a monoid, one can define positive integer powers of an element <i>x</i> : <i>x</i><sup>1</sup> = <i>x</i>, and <i>x</i><sup>n</sup> = <i>x</i> • ... • <i>x</i> (<i>n</i> times) for <i>n</i> &gt; 1 . The rule of powers <i>x</i><sup><i>n</i> + <i>p</i></sup> = <i>x</i><sup><i>n</i></sup> • <i>x</i><sup><i>p</i></sup> is obvious.</p>
<p>From the definition of a monoid, one can show that the identity element <i>e</i> is unique. Then, for any <i>x</i>, one can set <i>x</i><sup>0</sup> = <i>e</i> and the rule of powers is still true with nonnegative exponents.</p>
<p>It is possible to define <a href="/wiki/Inverse_element" title="Inverse element">invertible elements</a>: an element <i>x</i> is called invertible if there exists an element <i>y</i> such that <span class="nowrap"><i>x</i> • <i>y</i> = <i>e</i></span> and <span class="nowrap"><i>y</i> • <i>x</i> = <i>e</i></span>. The element <i>y</i> is called the inverse of <i>x</i>. If <i>y</i> and <i>z</i> are inverses of <i>x</i>, then by associativity <span class="nowrap"><i>y</i> = (<i>zx</i>)<i>y</i> = <i>z</i>(<i>xy</i>) = <i>z</i></span>. Thus inverses, if they exist, are unique.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<p>If <i>y</i> is the inverse of <i>x</i>, one can define negative powers of <i>x</i> by setting <span class="nowrap"><i>x</i><sup>−1</sup> = <i>y</i></span> and <span class="nowrap"><i>x</i><sup>−<i>n</i></sup> = <i>y</i> • ... • <i>y</i></span> (<i>n</i> times) for <span class="nowrap"><i>n</i> &gt; 1</span>. And the rule of exponents is still verified for all <i>n</i>, <i>p</i> rational integers. This is why the inverse of <i>x</i> is usually written <i>x</i><sup>−1</sup>. The set of all invertible elements in a monoid <i>M</i>, together with the operation •, forms a <a href="/wiki/Group_(mathematics)" title="Group (mathematics)">group</a>. In that sense, every monoid contains a group (possibly only the <a href="/wiki/Trivial_group" title="Trivial group">trivial group</a> consisting of only the identity).</p>
<p>However, not every monoid sits inside a group. For instance, it is perfectly possible to have a monoid in which two elements <i>a</i> and <i>b</i> exist such that <span class="nowrap"><i>a</i> • <i>b</i> = <i>a</i></span> holds even though <i>b</i> is not the identity element. Such a monoid cannot be embedded in a group, because in the group we could multiply both sides with the inverse of <i>a</i> and would get that <span class="nowrap"><i>b</i> = <i>e</i></span>, which isn't true. A monoid <span class="nowrap">(<i>M</i>, •)</span> has the <a href="/wiki/Cancellation_property" title="Cancellation property">cancellation property</a> (or is <a class="mw-redirect" href="/wiki/Cancellative" title="Cancellative">cancellative</a>) if for all <i>a</i>, <i>b</i> and <i>c</i> in <i>M</i>, <span class="nowrap"><i>a</i> • <i>b</i> = <i>a</i> • <i>c</i></span> always implies <span class="nowrap"><i>b</i> = <i>c</i></span> and <span class="nowrap"><i>b</i> • <i>a</i> = <i>c</i> • <i>a</i></span> always implies <span class="nowrap"><i>b</i> = <i>c</i></span>. A commutative monoid with the cancellation property can always be embedded in a group via the <a href="/wiki/Grothendieck_group" title="Grothendieck group">Grothendieck construction</a>. That is how the additive group of the integers (a group with operation +) is constructed from the additive monoid of natural numbers (a commutative monoid with operation + and cancellation property). However, a non-commutative cancellative monoid need not be embeddable in a group.</p>
<p>If a monoid has the cancellation property and is <i>finite</i>, then it is in fact a group. Proof: Fix an element <i>x</i> in the monoid. Since the monoid is finite, <span class="nowrap"><i>x</i><sup><i>n</i></sup> = <i>x</i><sup><i>m</i></sup></span> for some <span class="nowrap"><i>m</i> &gt; <i>n</i> &gt; 0</span>. But then, by cancellation we have that <span class="nowrap"><i>x</i><sup><i>m</i> − <i>n</i></sup> = <i>e</i></span> where <i>e</i> is the identity. Therefore, <span class="nowrap"><i>x</i> • <i>x</i><sup><i>m</i> − <i>n</i> − 1</sup> = <i>e</i></span>, so <i>x</i> has an inverse.</p>
<p>The right- and left-cancellative elements of a monoid each in turn form a submonoid (i.e. obviously include the identity and not so obviously are closed under the operation). This means that the cancellative elements of any commutative monoid can be extended to a group.</p>
<p>It turns out that requiring the cancellative property in a monoid is not required to perform the Grothendieck construction – commutativity is sufficient. However, if the original monoid has an <a href="/wiki/Absorbing_element" title="Absorbing element">absorbing element</a> then its Grothendieck group is the trivial group. Hence the homomorphism is, in general, not injective.</p>
<p>An <b>inverse monoid</b> is a monoid where for every <i>a</i> in <i>M</i>, there exists a unique <i>a</i><sup>−1</sup> in <i>M</i> such that <span class="nowrap"><i>a</i> = <i>a</i> • <i>a</i><sup>−1</sup> • <i>a</i></span> and <span class="nowrap"><i>a</i><sup>−1</sup> = <i>a</i><sup>−1</sup> • <i>a</i> • <i>a</i><sup>−1</sup></span>. If an inverse monoid is cancellative, then it is a group.</p>
<p>In the opposite direction, a <b><a href="/wiki/Zerosumfree_monoid" title="Zerosumfree monoid">zerosumfree monoid</a></b> is an additively written monoid in which <span class="nowrap"><i>a</i> + <i>b</i> = 0</span> implies that <span class="nowrap"><i>a</i> = 0</span> and <span class="nowrap"><i>b</i> = 0</span>:<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> equivalently, that no element other than zero has an additive inverse.</p>
<h2><span class="mw-headline" id="Acts_and_operator_monoids">Acts and operator monoids</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=9" title="Edit section: Acts and operator monoids">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a class="mw-redirect" href="/wiki/Monoid_act" title="Monoid act">monoid act</a></div>
<p>Let <i>M</i> be a monoid, with the binary operation denoted by • and the identity element denoted by <i>e</i>. Then a (left) <b><i>M</i>-act</b> (or left act over <i>M</i>) is a set <i>X</i> together with an operation <span class="nowrap">⋅ : <i>M</i> × <i>X</i> → <i>X</i></span> which is compatible with the monoid structure as follows:</p>
<ul>
<li>for all <i>x</i> in <i>X</i>: <span class="nowrap"><i>e</i> ⋅ <i>x</i> = <i>x</i></span>;</li>
<li>for all <i>a</i>, <i>b</i> in <i>M</i> and <i>x</i> in <i>X</i>: <span class="nowrap"><i>a</i> ⋅ (<i>b</i> ⋅ <i>x</i>) = (<i>a</i> • <i>b</i>) ⋅ <i>x</i></span>.</li>
</ul>
<p>This is the analogue in monoid theory of a (left) <a href="/wiki/Group_action" title="Group action">group action</a>. Right <i>M</i>-acts are defined in a similar way. A monoid with an act is also known as an <b><a class="mw-redirect" href="/wiki/Operator_monoid" title="Operator monoid">operator monoid</a></b>. Important examples include <a href="/wiki/Transition_system" title="Transition system">transition systems</a> of <a class="mw-redirect" href="/wiki/Semiautomata" title="Semiautomata">semiautomata</a>. A <a href="/wiki/Transformation_semigroup" title="Transformation semigroup">transformation semigroup</a> can be made into an operator monoid by adjoining the identity transformation.</p>
<h2><span class="mw-headline" id="Monoid_homomorphisms">Monoid homomorphisms</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=10" title="Edit section: Monoid homomorphisms">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">

<p>A <a href="/wiki/Homomorphism" title="Homomorphism">homomorphism</a> between two monoids <span class="nowrap">(<i>M</i>, ∗)</span> and <span class="nowrap">(<i>N</i>, •)</span> is a function <span class="nowrap"><i>f</i> : <i>M</i> → <i>N</i></span> such that</p>
<ul>
<li><i>f</i>(<i>x</i> ∗ <i>y</i>) = <i>f</i>(<i>x</i>) • <i>f</i>(<i>y</i>) for all <i>x</i>, <i>y</i> in <i>M</i></li>
<li><i>f</i>(<i>e</i><sub><i>M</i></sub>) = <i>e</i><sub><i>N</i></sub>,</li>
</ul>
<p>where <i>e</i><sub><i>M</i></sub> and <i>e</i><sub><i>N</i></sub> are the identities on <i>M</i> and <i>N</i> respectively. Monoid homomorphisms are sometimes simply called <b>monoid morphisms</b>.</p>
<p>Not every <a class="mw-redirect" href="/wiki/Semigroup_homomorphism" title="Semigroup homomorphism">semigroup homomorphism</a> is a monoid homomorphism, since it may not map the identity to the identity of the target monoid, even though the element it maps the identity to will be an identity of the image of the mapping. In contrast, a semigroup homomorphism between groups is always a <a href="/wiki/Group_homomorphism" title="Group homomorphism">group homomorphism</a>, as it necessarily preserves the identity. Since for monoids this isn't always true, it is necessary to state this as a separate requirement.</p>
<p>A <a class="mw-redirect" href="/wiki/Bijective" title="Bijective">bijective</a> monoid homomorphism is called a monoid <a href="/wiki/Isomorphism" title="Isomorphism">isomorphism</a>. Two monoids are said to be isomorphic if there is a monoid isomorphism between them.</p>
<h2><span class="mw-headline" id="Equational_presentation">Equational presentation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=11" title="Edit section: Equational presentation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/Presentation_of_a_monoid" title="Presentation of a monoid">Presentation of a monoid</a></div>
<p>Monoids may be given a <b>presentation</b>, much in the same way that groups can be specified by means of a <a class="mw-redirect" href="/wiki/Group_presentation" title="Group presentation">group presentation</a>. One does this by specifying a set of generators Σ, and a set of relations on the <a href="/wiki/Free_monoid" title="Free monoid">free monoid</a> Σ<sup>∗</sup>. One does this by extending (finite) <a href="/wiki/Binary_relation" title="Binary relation">binary relations</a> on Σ<sup>∗</sup> to monoid congruences, and then constructing the quotient monoid, as above.</p>
<p>Given a binary relation <span class="nowrap"><i>R</i> ⊂ Σ<sup>∗</sup> × Σ<sup>∗</sup></span>, one defines its symmetric closure as <span class="nowrap"><i>R</i> ∪ <i>R</i><sup>−1</sup></span>. This can be extended to a symmetric relation <span class="nowrap"><i>E</i> ⊂ Σ<sup>∗</sup> × Σ<sup>∗</sup></span> by defining <span class="nowrap"><i>x</i> ~<sub><i>E</i></sub> <i>y</i></span> if and only if <span class="nowrap"><i>x</i> = <i>sut</i></span> and <span class="nowrap"><i>y</i> = <i>svt</i></span> for some strings <span class="nowrap"><i>u</i>, <i>v</i>, <i>s</i>, <i>t</i> ∈ Σ<sup>∗</sup></span> with <span class="nowrap">(<i>u</i>,<i>v</i>) ∈ <i>R</i> ∪ <i>R</i><sup>−1</sup></span>. Finally, one takes the reflexive and transitive closure of <i>E</i>, which is then a monoid congruence.</p>

<p>Monoids can be viewed as a special class of <a href="/wiki/Category_theory" title="Category theory">categories</a>. Indeed, the axioms required of a monoid operation are exactly those required of <a href="/wiki/Morphism" title="Morphism">morphism</a> composition when restricted to the set of all morphisms whose source and target is a given object.<sup class="reference" id="cite_ref-Awo10_10-0"><a href="#cite_note-Awo10-10">[10]</a></sup> That is,</p>
<dl>
<dd><i>A monoid is, essentially, the same thing as a category with a single object.</i></dd>
</dl>
<p>More precisely, given a monoid <span class="nowrap">(<i>M</i>, •)</span>, one can construct a small category with only one object and whose morphisms are the elements of <i>M</i>. The composition of morphisms is given by the monoid operation •.</p>
<p>Likewise, monoid homomorphisms are just <a href="/wiki/Functor" title="Functor">functors</a> between single object categories.<sup class="reference" id="cite_ref-Awo10_10-1"><a href="#cite_note-Awo10-10">[10]</a></sup> So this construction gives an <a href="/wiki/Equivalence_of_categories" title="Equivalence of categories">equivalence</a> between the <a class="mw-redirect" href="/wiki/Category_of_monoids" title="Category of monoids">category of (small) monoids</a> <b>Mon</b> and a full subcategory of the category of (small) categories <b>Cat</b>. Similarly, the <a href="/wiki/Category_of_groups" title="Category of groups">category of groups</a> is equivalent to another full subcategory of <b>Cat</b>.</p>
<p>In this sense, category theory can be thought of as an extension of the concept of a monoid. Many definitions and theorems about monoids can be generalised to small categories with more than one object. For example, a quotient of a category with one object is just a quotient monoid.</p>
<p>Monoids, just like other algebraic structures, also form their own category, <b>Mon</b>, whose objects are monoids and whose morphisms are monoid homomorphisms.<sup class="reference" id="cite_ref-Awo10_10-2"><a href="#cite_note-Awo10-10">[10]</a></sup></p>
<p>There is also a notion of <a href="/wiki/Monoid_(category_theory)" title="Monoid (category theory)">monoid object</a> which is an abstract definition of what is a monoid in a category. A monoid object in <b><a href="/wiki/Category_of_sets" title="Category of sets">Set</a></b> is just a monoid.</p>
<h2><span class="mw-headline" id="Monoids_in_computer_science">Monoids in computer science</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=13" title="Edit section: Monoids in computer science">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In computer science, many <a class="mw-redirect" href="/wiki/Abstract_data_types" title="Abstract data types">abstract data types</a> can be endowed with a monoid structure. For instance, many iterative algorithms need to update some kind of "running total" at each iteration; this pattern may be elegantly expressed by a monoid operation. Alternatively, the associativity of monoid operations ensures that the operation can be <a class="mw-redirect" href="/wiki/Parallelization" title="Parallelization">parallelized</a> by employing a <a href="/wiki/Prefix_sum" title="Prefix sum">prefix sum</a> or similar algorithm, in order to utilize multiple cores or processors efficiently.</p>

<h2><span class="mw-headline" id="MapReduce">MapReduce</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Monoid&amp;action=edit&amp;section=14" title="Edit section: MapReduce">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Probably the most important application of monoids in computer science is so-called <a href="/wiki/MapReduce" title="MapReduce">MapReduce</a> programming model (see <a class="external text" href="http://erikerlandson.github.io/blog/2016/09/05/expressing-map-reduce-as-a-left-folding-monoid/" rel="nofollow">Encoding Map-Reduce As A Monoid With Left Folding</a>). <a href="/wiki/MapReduce" title="MapReduce">MapReduce</a>, in computing, consists of two or three operations. Given a dataset, "Map" consists of mapping arbitrary data to elements of a specific monoid. "Reduce" consists of folding those elements, so that in the end we produce just one element. As described above, this operation can be highly parallelized. The only problem is that the monoid may be pretty large, so it is distributed through the cloud, and the results of fold cannot be stored in one node. If this is the case, an operation of "Shuffling" is added, to group results of mapping.</p>
<p>For example, if we have a <a href="/wiki/Multiset" title="Multiset">multiset</a>, in a program it is represented as a map from elements to their numbers. Elements are called keys in this case. The number of distinct keys may be too big, and in this case the <a href="/wiki/Multiset" title="Multiset">multiset</a> is being sharded. To finalize reduction properly, the "Shuffling" stage regroups the data among the nodes. If we do not need this step, the whole Map/Reduce consists of mapping and reducing; both operation are parallelizable, the former due to its element-wise nature, the latter due to associativity of the monoid.</p>
