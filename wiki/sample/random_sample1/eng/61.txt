<h1 class="firstHeading" id="firstHeading" lang="en">Shortcut (computing)</h1>
<div class="mw-body-content" id="bodyContent">

<p>In <a href="/wiki/Computing" title="Computing">computing</a>, a <b>file shortcut</b> is a <a href="/wiki/Handle_(computing)" title="Handle (computing)">handle</a> in a <a href="/wiki/User_interface" title="User interface">user interface</a> that allows the user to find a file or <a href="/wiki/System_resource" title="System resource">resource</a> located in a different <a href="/wiki/Directory_(computing)" title="Directory (computing)">directory</a> or <a class="mw-redirect" href="/wiki/Folder_(computing)" title="Folder (computing)">folder</a> from the place where the shortcut is located.</p>
<p>Shortcuts are typically implemented as a small file containing a target <a class="mw-redirect" href="/wiki/URI" title="URI">URI</a> or <a href="/wiki/Globally_unique_identifier" title="Globally unique identifier">GUID</a> to an <i>object</i>, or the name of a target program file that the shortcut represents. The shortcut might additionally specify <a class="mw-redirect" href="/wiki/Command-line_argument" title="Command-line argument">parameters</a> to be passed to the target program when it is run. Each shortcut can have its own icon. Shortcuts are very commonly placed on a <a href="/wiki/Desktop_environment" title="Desktop environment">desktop</a>, in an <a class="mw-redirect" href="/wiki/Application_launcher_panel" title="Application launcher panel">application launcher panel</a> such as the <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> <a href="/wiki/Start_menu" title="Start menu">Start menu</a>, or in the main menu of a <a href="/wiki/Desktop_environment" title="Desktop environment">desktop environment</a>. The functional equivalent in the Macintosh operating system is called an <a href="/wiki/Alias_(Mac_OS)" title="Alias (Mac OS)">alias</a>, and a <a href="/wiki/Symbolic_link" title="Symbolic link">symbolic link</a> (or <b>symlink</b>) in <a class="mw-redirect" href="/wiki/UNIX" title="UNIX">UNIX</a>-like systems.</p>

<h2><span class="mw-headline" id="Implementations">Implementations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Shortcut_(computing)&amp;action=edit&amp;section=1" title="Edit section: Implementations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Microsoft_Windows">Microsoft Windows</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Shortcut_(computing)&amp;action=edit&amp;section=2" title="Edit section: Microsoft Windows">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

<p>File shortcuts (also known as <i>shell links</i>) were introduced in <a href="/wiki/Windows_95" title="Windows 95">Windows 95</a>.<sup class="reference" id="cite_ref-WinLNK_1-0"><a href="#cite_note-WinLNK-1">[1]</a></sup> <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> uses <tt>.lnk</tt> as the <a href="/wiki/Filename" title="Filename">filename</a> extension for shortcuts to local files, and <tt>.URL</tt> for shortcuts to remote files, like web pages. Commonly referred to as "shortcuts" or "link files", both are displayed with a curled arrow overlay icon by default, and no <a href="/wiki/Filename_extension" title="Filename extension">filename extension</a>. (The extension remains hidden in Windows Explorer even when "Hide extensions for known file types" is unchecked in <i>File Type</i> options, because it is controlled by the <tt>NeverShowExt</tt> option in <tt>HKEY_CLASSES_ROOT\lnkfile</tt> in the Registry. The <tt>IsShortcut</tt> option causes the arrow to be displayed.) Shortcut files can be used to launch programs in minimized or maximized window states if the program supports it.</p>
<p>Microsoft Windows <tt>.lnk</tt> files operate as <a class="mw-redirect" href="/wiki/Windows_Explorer" title="Windows Explorer">Windows Explorer</a> extensions, rather than file system extensions. As a shell extension, <tt>.lnk</tt> files cannot be used in place of the file except in Windows Explorer, and have other uses in Windows Explorer in addition to use as a shortcut to a local file (or GUID). These files also begin with "L".</p>
<p>Although shortcuts, when created, point to specific files or folders, they may break if the target is moved to another location. When a shortcut file that points to a nonexistent target is opened, Explorer will attempt to repair the shortcut. <a href="/wiki/Windows_9x" title="Windows 9x">Windows 9x</a>-based versions of Windows use a simple search algorithm to fix broken shortcuts.<sup class="reference" id="cite_ref-WinLNK_1-1"><a href="#cite_note-WinLNK-1">[1]</a></sup> On <a href="/wiki/Windows_NT" title="Windows NT">Windows NT</a>-based operating systems and the <a href="/wiki/NTFS" title="NTFS">NTFS</a> file system, the target object's unique identifier is stored in the shortcut file and Windows can use the <i>Distributed Link Tracking</i> <a href="/wiki/Windows_service" title="Windows service">service</a> for tracking the targets of shortcuts, so that the shortcut may be silently updated if the target moves to another hard drive.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> <a href="/wiki/Windows_Installer" title="Windows Installer">Windows Installer</a>, introduced in <a href="/wiki/Windows_2000" title="Windows 2000">Windows 2000</a>, added another special type of shortcuts called "Advertised Shortcuts."</p>
<p>File shortcuts in Windows can store a <a href="/wiki/Working_directory" title="Working directory">working directory</a> path besides the target path. Environment variables can be used. A <a class="mw-redirect" href="/wiki/Hotkey" title="Hotkey">hotkey</a> can be defined in the shortcut's properties for shortcuts that are located in the <a class="mw-redirect" href="/wiki/Start_Menu" title="Start Menu">Start Menu</a> folders, pinned to the Taskbar or the Desktop. In Windows 2000 onwards, file shortcuts can store comments which are displayed as a tooltip when the mouse hovers over the shortcut.</p>
<p>Generally, the effect of double-clicking a shortcut is intended to be the same as double-clicking the application or document to which it refers, but Windows shortcuts contain separate properties for the target file and the "Start In" directory. If the latter parameter is not entered, attempting to use the shortcut for some programs may generate "missing DLL" errors not present when the application is accessed directly.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<p><a href="/wiki/Symbolic_link" title="Symbolic link">File system links</a> can also be created on Windows systems (Vista and up). They serve a similar function, although they are a feature of the <a href="/wiki/File_system" title="File system">file system</a>. Windows shortcuts are files and work independently of the file system, through Explorer.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p>Beginning with <a href="/wiki/Windows_7" title="Windows 7">Windows 7</a>, some shortcuts also store Application User Model IDs (AppUserModelIDs).<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> Instead of the target command line, AppUserModelIDs may directly be used to launch applications. Shortcuts with AppUserModelIDs are used by some desktop programs and all <a class="mw-redirect" href="/wiki/WinRT" title="WinRT">WinRT</a> Modern apps for launching.</p>
<p>Although Windows does not provide convenient tools to create it, Explorer supports a "folder link" or "shell link folder": a folder with the system attribute set, containing a hidden "desktop.ini" (folder customization) file which tells Explorer to look in that same folder for a "target.lnk" shortcut file pointing to another folder. When viewed in Explorer, the shell link folder then appears to have the contents of the target folder in it—that is, the customized folder becomes the effective shortcut.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> This technique is used by Microsoft Windows for items like WebDAV folders. The advent of file system links in Windows Vista and up has made shell link folders less useful.</p>
<h3><span class="mw-headline" id="Unix">Unix</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Shortcut_(computing)&amp;action=edit&amp;section=3" title="Edit section: Unix">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Some desktop environments for Unix-like operating systems, such as <a href="/wiki/GNOME" title="GNOME">GNOME</a> or <a href="/wiki/KDE" title="KDE">KDE</a> provide <a href="/wiki/Freedesktop.org" title="Freedesktop.org">freedesktop.org</a> <tt>.desktop</tt><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> files. These can be used to point to local or remote files, folders, and applications. <a href="/wiki/Symbolic_link" title="Symbolic link">Symbolic links</a> can also be created on Unix systems, which serve a similar function.</p>

<h3><span class="mw-headline" id="Mac">Mac</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Shortcut_(computing)&amp;action=edit&amp;section=5" title="Edit section: Mac">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Macintosh does not have extensions for shortcuts, although symbolic links can be created within the Unix subsystem. The extension .webloc is used for Internet URLs within the Safari browser. The correct term for file "shortcuts" is <a href="/wiki/Alias_(Mac_OS)" title="Alias (Mac OS)">"alias"</a>. Aliases in Macintosh <a class="mw-redirect" href="/wiki/System_7_(Macintosh)" title="System 7 (Macintosh)">System 7</a> through <a href="/wiki/Mac_OS_8" title="Mac OS 8">Mac OS 9</a> distinguished themselves visually to the user by the fact that their file names were in italics. In Mac OS 8 and later, another distinguishing mark was added, badging with an "alias arrow" – a black arrow on a small white square – similar to that used for shortcuts in Microsoft Windows. In <a class="mw-redirect" href="/wiki/Mac_OS_X" title="Mac OS X">Mac OS X</a>, the filenames of aliases are no longer italicized, but the arrow badge remains. Additionally, an alias retains its dynamic reference to an object and does not have to be specified even when calling files on remote servers.</p>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Shortcut_(computing)&amp;action=edit&amp;section=6" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>To execute an <a href="/wiki/Application_software" title="Application software">application</a> or <a href="/wiki/Rendering_(computer_graphics)" title="Rendering (computer graphics)">render</a> a <a href="/wiki/Computer_file" title="Computer file">file</a> in early <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interfaces</a>, the user had to click on the representation of the actual file or executable in the location where the application or file was. The concept of disassociating the executable from the <a href="/wiki/Icon_(computing)" title="Icon (computing)">icon</a> representing an instruction to perform a task associated with that file or executable so that they may be grouped by function or task rather than physical organisation in the file structure was first described in the research paper "A Task Oriented Front End For The Windows Graphical User Interface", published in 1991 by <a href="/wiki/Kingston_University" title="Kingston University">Kingston University</a> and presented to both <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> and <a href="/wiki/PARC_(company)" title="PARC (company)">Xerox EuroPARC</a> that same year under an academia/business technology sharing agreement. A simplified form of this research was incorporated into <a class="mw-redirect" href="/wiki/System_7_(Macintosh)" title="System 7 (Macintosh)">System 7</a> in 1991, and four years later into <a href="/wiki/Windows_95" title="Windows 95">Windows 95</a>.</p>
