<p>This is a <b>list of <a href="/wiki/Complexity_class" title="Complexity class">complexity classes</a></b> in <a href="/wiki/Computational_complexity_theory" title="Computational complexity theory">computational complexity theory</a>. For other computational and complexity subjects, see <a href="/wiki/List_of_computability_and_complexity_topics" title="List of computability and complexity topics">list of computability and complexity topics</a>.</p>
<p>Many of these classes have a 'co' partner which consists of the <a href="/wiki/Complement_(complexity)" title="Complement (complexity)">complements</a> of all languages in the original class. For example if a language L is in NP then the complement of L is in co-NP. (This does not mean that the complement of NP is co-NP - there are languages which are known to be in both, and other languages which are known to be in neither.)</p>
<p>"The hardest problems" of a class refer to problems which belong to the class such that every other problem of that class can be reduced to it. Furthermore, the reduction is also a problem of the given class, or its subset.</p>
