<p>A <b>complex programmable logic device</b> (<b>CPLD</b>) is a <a href="/wiki/Programmable_logic_device" title="Programmable logic device">programmable logic device</a> with complexity between that of <a href="/wiki/Programmable_Array_Logic" title="Programmable Array Logic">PALs</a> and <a href="/wiki/Field-programmable_gate_array" title="Field-programmable gate array">FPGAs</a>, and architectural features of both. The main building block of the CPLD is a <a href="/wiki/Macrocell_array" title="Macrocell array">macrocell</a>, which contains logic implementing <a href="/wiki/Disjunctive_normal_form" title="Disjunctive normal form">disjunctive normal form</a> expressions and more specialized logic operations.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>

<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Complex_programmable_logic_device&amp;action=edit&amp;section=1" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Some of the CPLD features are in common with <a class="mw-redirect" href="/wiki/Programmable_array_logic" title="Programmable array logic">PALs</a>:</p>
<ul>
<li>Non-volatile configuration memory. Unlike many FPGAs, an external configuration <a href="/wiki/Read-only_memory" title="Read-only memory">ROM</a> isn't required, and the CPLD can function immediately on system start-up.</li>
<li>For many legacy CPLD devices, routing constrains most logic blocks to have input and output signals connected to external pins, reducing opportunities for internal state storage and deeply layered logic. This is usually not a factor for larger CPLDs and newer CPLD product families.</li>
</ul>
<p>Other features are in common with <a class="mw-redirect" href="/wiki/Field_programmable_gate_array" title="Field programmable gate array">FPGAs</a>:</p>
<ul>
<li>Large number of gates available. CPLDs typically have the equivalent of thousands to tens of thousands of <a href="/wiki/Logic_gate" title="Logic gate">logic gates</a>, allowing implementation of moderately complicated data processing devices. PALs typically have a few hundred gate equivalents at most, while FPGAs typically range from tens of thousands to several million.</li>
<li>Some provisions for logic more flexible than <a href="/wiki/Disjunctive_normal_form" title="Disjunctive normal form">sum-of-product</a> expressions, including complicated feedback paths between macro cells, and specialized logic for implementing various commonly used functions, such as <a href="/wiki/Integer" title="Integer">integer</a> <a href="/wiki/Arithmetic" title="Arithmetic">arithmetic</a>.</li>
</ul>
<p>The most noticeable difference between a large CPLD and a small FPGA is the presence of on-chip non-volatile memory in the CPLD, which allows CPLDs to be used for "<a class="mw-redirect" href="/wiki/Boot_loader" title="Boot loader">boot loader</a>" functions, before handing over control to other devices not having their own permanent program storage. A good example is where a CPLD is used to load configuration data for an FPGA from non-volatile memory.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<h2><span class="mw-headline" id="Distinctions">Distinctions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Complex_programmable_logic_device&amp;action=edit&amp;section=2" title="Edit section: Distinctions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>CPLDs were an evolutionary step from even smaller devices that preceded them, <a href="/wiki/Programmable_logic_array" title="Programmable logic array">PLAs</a> (first shipped by <a href="/wiki/Signetics" title="Signetics">Signetics</a>), and <a class="mw-redirect" href="/wiki/Programmable_array_logic" title="Programmable array logic">PALs</a>. These in turn were preceded by <a href="/wiki/Logic_family" title="Logic family">standard logic</a> products, that offered no programmability and were used to build logic functions by physically wiring several standard logic chips together (usually with wiring on a printed circuit board, but sometimes, especially for prototyping, using <a href="/wiki/Wire_wrap" title="Wire wrap">wire wrap</a> wiring).</p>
<p>The main distinction between FPGA and CPLD device architectures is that FPGAs are internally based on <a href="/wiki/Lookup_table" title="Lookup table">look-up tables</a> (LUTs) while CPLDs form the logic functions with <a class="new" href="/w/index.php?title=Sea-of-gates&amp;action=edit&amp;redlink=1" title="Sea-of-gates (page does not exist)">sea-of-gates</a> (for example, sum of products).</p>
