<p>A <b>boot disk</b> is a removable digital data storage medium from which a <a href="/wiki/Computer" title="Computer">computer</a> can load and run (<a href="/wiki/Booting" title="Booting">boot</a>) an <a href="/wiki/Operating_system" title="Operating system">operating system</a> or utility program. The computer must have a built-in program which will load and execute a program from a boot disk meeting certain standards.</p>
<p>While almost all modern computers can boot from a hard drive containing the operating system and other software, they would not normally be called boot disks (because they are not removable media). <a href="/wiki/CD-ROM" title="CD-ROM">CD-ROMs</a> are the most common forms of media used, but other media, such as magnetic or paper tape drives, <a href="/wiki/Zip_drive" title="Zip drive">zip drives</a>, and more recently <a href="/wiki/USB_flash_drive" title="USB flash drive">USB flash drives</a> can be used. The computer's <a href="/wiki/BIOS" title="BIOS">BIOS</a> must support booting from the device in question.</p>

<h2><span class="mw-headline" id="Uses">Uses</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Boot_disk&amp;action=edit&amp;section=1" title="Edit section: Uses">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Boot disks are used for:</p>
<ul>
<li>Operating system installation</li>
<li><a href="/wiki/Data_recovery" title="Data recovery">Data recovery</a></li>
<li><a href="/wiki/Data_remanence#Purging" title="Data remanence">Data purging</a></li>
<li>Hardware or software troubleshooting</li>
<li><a class="mw-redirect" href="/wiki/BIOS_flashing" title="BIOS flashing">BIOS flashing</a></li>
<li>Customizing an operating environment</li>
<li>Software demonstration</li>
<li>Running a temporary operating environment, such as when using a <a href="/wiki/Live_USB" title="Live USB">Live USB</a> drive.</li>
<li>Administrative access in case of lost password is possible with an appropriate boot disk with some operating systems</li>
<li>Games (e.g. for <a class="mw-redirect" href="/wiki/Amiga_games" title="Amiga games">Amiga</a> home computers)</li>
</ul>
<h2><span class="mw-headline" id="Process">Process</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Boot_disk&amp;action=edit&amp;section=2" title="Edit section: Process">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The term <i>boot</i> comes from the idea of lifting oneself by one's own <a class="mw-redirect" href="/wiki/Bootstrap" title="Bootstrap">bootstraps</a>: the computer contains a tiny program (bootstrap loader) which will load and run a program found on a boot device. This program may itself be a small program designed to load a larger and more capable program, i.e., the full operating system. To enable booting without the requirement either for a <a href="/wiki/Mass_storage" title="Mass storage">mass storage</a> device or to write to the boot medium, it is usual for the boot program to use some system <a class="mw-redirect" href="/wiki/RAM" title="RAM">RAM</a> as a <a class="mw-redirect" href="/wiki/RAM_disk" title="RAM disk">RAM disk</a> for temporary <a href="/wiki/Computer_file" title="Computer file">file</a> storage.</p>
<p>As an example, any computer compatible with the <a class="mw-redirect" href="/wiki/IBM_PC" title="IBM PC">IBM PC</a> is able with built-in software to load the contents of the first 512 bytes of a floppy and to execute it if it is a viable program; boot floppies have a very simple loader program in these bytes. The process is vulnerable to abuse; data floppies could have a virus written to their first sector which silently infects the host computer if switched on with the disk in the drive.</p>
<h2><span class="mw-headline" id="Media">Media</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Boot_disk&amp;action=edit&amp;section=3" title="Edit section: Media">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Bootable <a href="/wiki/Floppy_disk" title="Floppy disk">floppy disks</a> (("boot floppies") for PCs usually contain <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>, a workalike such as <a href="/wiki/FreeDOS" title="FreeDOS">FreeDOS</a>, or miniature versions of <a href="/wiki/Linux" title="Linux">Linux</a>. The most commonly available floppy disk can hold only 1.4 <a href="/wiki/Megabyte" title="Megabyte">MB</a> of data in its standard format, making it impractical for loading large <a href="/wiki/Operating_system" title="Operating system">operating systems</a>. The use of boot floppies is in decline, due to the availability of other higher-capacity options, such as <a href="/wiki/CD-ROM" title="CD-ROM">CD-ROMs</a> or <a class="mw-redirect" href="/wiki/USB_flash_drives" title="USB flash drives">USB flash drives</a>.</p>
<h2><span class="mw-headline" id="Device_selection">Device selection</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Boot_disk&amp;action=edit&amp;section=4" title="Edit section: Device selection">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A modern PC is configured to attempt to boot from various devices in a certain order. If a computer is not booting from the device desired, such as the floppy drive, the user may have to enter the BIOS setup function by pressing a special key when the computer is first turned on (such as Delete, F1, F2, F10 or F12), and then changing the boot order. More recent BIOSes permit the interruption of the final stage of the boot process by pressing a function key (usually F11). This results in a list of bootable devices being presented, from which a selection may be made.</p>
<p>Modern Apple computers will boot from an appropriate disk if the user presses the C key while the machine is starting.</p>
<h2><span class="mw-headline" id="Requirements">Requirements</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Boot_disk&amp;action=edit&amp;section=5" title="Edit section: Requirements">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Different operating systems use different boot disk contents. All boot discs must be compatible with the computer they are designed for.</p>
<dl>
<dt>MS-DOS/PC DOS</dt>
</dl>
<ul>
<li>A valid <a href="/wiki/Boot_sector" title="Boot sector">boot sector</a> on the floppy disk, not in the format of a normal disk <a href="/wiki/Computer_file" title="Computer file">file</a>.</li>
<li><a href="/wiki/IO.SYS" title="IO.SYS">IO.SYS</a> or <a href="/wiki/IBMBIO.COM" title="IBMBIO.COM">IBMBIO.COM</a></li>
<li><a href="/wiki/MSDOS.SYS" title="MSDOS.SYS">MSDOS.SYS</a> or <a href="/wiki/IBMDOS.COM" title="IBMDOS.COM">IBMDOS.COM</a></li>
<li><a href="/wiki/COMMAND.COM" title="COMMAND.COM">COMMAND.COM</a></li>
</ul>
<p>All files must be for the same version; although MS-DOS 5 and 6 use a file called COMMAND.COM, they are not interchangeable. Complete boot disks can be prepared in one operation by an installed operating system; details vary.</p>
<dl>
<dt>FreeDOS</dt>
</dl>
<ul>
<li>A valid boot sector on the disk</li>
<li>COMMAND.COM</li>
<li>KERNEL.SYS</li>
</ul>
<dl>
<dt>Linux</dt>
</dl>
<ul>
<li>A bootloader such as <a href="/wiki/SYSLINUX" title="SYSLINUX">SYSLINUX</a> or <a class="mw-redirect" href="/wiki/GRUB" title="GRUB">GRUB</a></li>
<li><a href="/wiki/Linux_kernel" title="Linux kernel">Linux kernel</a></li>
<li>Initial ram disk (initrd)</li>
</ul>
