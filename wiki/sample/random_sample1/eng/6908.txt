<p>A <b>stop error</b>, better known as a <b>Blue Screen of Death</b> (also known as a <b>blue screen</b> or <b>BSoD</b>) is an <a class="mw-redirect" href="/wiki/Error_screen" title="Error screen">error screen</a> displayed on a <a class="mw-redirect" href="/wiki/Windows" title="Windows">Windows</a> computer system after a <a href="/wiki/Fatal_system_error" title="Fatal system error">fatal system error</a>, also known as a system <a href="/wiki/Crash_(computing)" title="Crash (computing)">crash</a>: when the <a href="/wiki/Operating_system" title="Operating system">operating system</a> reaches a condition where it can no longer operate safely. </p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Blue_Screen_of_Death&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

<p>BSoDs have been present in <a href="/wiki/Windows_NT_3.1" title="Windows NT 3.1">Windows NT 3.1</a> (the first version of the <a class="mw-redirect" href="/wiki/Windows_NT_family" title="Windows NT family">Windows NT family</a>, released in 1993) and all Windows operating systems released afterwards. (See <a href="/wiki/History_of_Microsoft_Windows" title="History of Microsoft Windows">History of Microsoft Windows</a>.) BSoDs can be caused by poorly written <a href="/wiki/Device_driver" title="Device driver">device drivers</a> or malfunctioning hardware, such as faulty <a href="/wiki/Computer_memory" title="Computer memory">memory</a>, power supply issues, overheating of components, or hardware running beyond its specification limits. In the <a href="/wiki/Windows_9x" title="Windows 9x">Windows 9x</a> era, incompatible <a href="/wiki/Dynamic-link_library" title="Dynamic-link library">DLLs</a> or bugs in the operating system <a href="/wiki/Kernel_(operating_system)" title="Kernel (operating system)">kernel</a> could also cause BSoDs. Because of the instability and lack of memory protection in Windows 9x, BSoDs were much more common.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>]</sup></p>

<h2><span class="mw-headline" id="Format">Format</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Blue_Screen_of_Death&amp;action=edit&amp;section=3" title="Edit section: Format">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Until <a href="/wiki/Windows_Server_2012" title="Windows Server 2012">Windows Server 2012</a>, BSoDs showed <a href="/wiki/Silver_(color)" title="Silver (color)">silver</a> text on a <a href="/wiki/Navy_blue" title="Navy blue">navy blue</a> background with information about current memory values and register values. Windows Server 2012, <a href="/wiki/Windows_8" title="Windows 8">Windows 8</a> and <a href="/wiki/Windows_10" title="Windows 10">Windows 10</a> use a <a href="/wiki/Cerulean" title="Cerulean">cerulean</a> background instead.</p>
<p>Windows 95, <a href="/wiki/Windows_98" title="Windows 98">98</a> and <a href="/wiki/Windows_ME" title="Windows ME">ME</a> BSoDs use 80×25 <a href="/wiki/Text_mode" title="Text mode">text mode</a>. BSoDs in the Windows NT family use 80×50 text mode on a 720×400 screen. Windows XP BSoDs use the <a class="mw-redirect" href="/wiki/Lucida_Console" title="Lucida Console">Lucida Console</a> font while the Windows Vista and 7 BSoD uses the <a href="/wiki/Consolas" title="Consolas">Consolas</a> font. Windows 8, Windows Server 2012 use <a class="mw-redirect" href="/wiki/Segoe_UI" title="Segoe UI">Segoe UI</a> and attempt to render the BSoD at native resolution, otherwise defaulting to 640x480.</p>
<h2><span class="mw-headline" id="Windows_NT">Windows NT</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Blue_Screen_of_Death&amp;action=edit&amp;section=4" title="Edit section: Windows NT">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tleft">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Windows_XP_BSOD.png"><img alt="" class="thumbimage" data-file-height="480" data-file-width="640" height="165" src="//upload.wikimedia.org/wikipedia/commons/thumb/a/a8/Windows_XP_BSOD.png/220px-Windows_XP_BSOD.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/a/a8/Windows_XP_BSOD.png/330px-Windows_XP_BSOD.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/a/a8/Windows_XP_BSOD.png/440px-Windows_XP_BSOD.png 2x" width="220"/></a>

<div class="magnify"><a class="internal" href="/wiki/File:Windows_XP_BSOD.png" title="Enlarge"></a></div>
A stop error as seen in <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a>, <a href="/wiki/Windows_Vista" title="Windows Vista">Vista</a>, and <a href="/wiki/Windows_7" title="Windows 7">7</a></div>
</div>
</div>
<p>In <a href="/wiki/Windows_NT" title="Windows NT">Windows NT</a> family of operating systems, the blue screen of death (officially known as a <b>stop error</b> and referred to as "<a class="mw-redirect" href="/wiki/Bug_check" title="Bug check">bug check</a>" in the Windows <a href="/wiki/Software_development_kit" title="Software development kit">software development kit</a> and <a class="mw-redirect" href="/wiki/Driver_development_kit" title="Driver development kit">driver development kit</a> documentation) occurs when the kernel or a driver running in <a class="mw-redirect" href="/wiki/Kernel_mode" title="Kernel mode">kernel mode</a> encounters an error from which it cannot recover. This is usually caused by an <a class="mw-redirect" href="/wiki/Illegal_operation" title="Illegal operation">illegal operation</a> being performed. The only safe action the operating system can take in this situation is to <a href="/wiki/Triple_fault" title="Triple fault">restart the computer</a>. As a result, data may be lost, as users are not given an opportunity to save data that has not yet been saved to the hard drive.</p>
<p>The text on the error screen contains the code of the error and its symbolic name (e.g. "0x0000001E, KMODE_EXCEPTION_NOT_HANDLED") along with four error-dependent values in parentheses that are there to help software engineers fix the problem that occurred. Depending on the error code, it may display the address where the problem occurred, along with the driver which is loaded at that address. Under Windows NT, the second and third sections of the screen may contain information on all loaded drivers and a stack dump, respectively. The driver information is in three columns; the first lists the base address of the driver, the second lists the driver's creation date (as a <a class="mw-redirect" href="/wiki/Unix_timestamp" title="Unix timestamp">Unix timestamp</a>), and the third lists the name of the driver.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></p>

</div>
</div>
<p>By default, Windows will create a <a class="mw-redirect" href="/wiki/Memory_dump" title="Memory dump">memory dump</a> file when a stop error occurs. Depending on the OS version, there may be several formats this can be saved in, ranging from a 64kB "minidump" (introduced in Windows 2000) to a "complete dump" which is effectively a copy of the entire contents of physical memory (<a class="mw-redirect" href="/wiki/RAM" title="RAM">RAM</a>). The resulting memory dump file may be debugged later, using a kernel <a href="/wiki/Debugger" title="Debugger">debugger</a>. For Windows WinDBG or KD debuggers from <a href="/wiki/WinDbg" title="WinDbg">Debugging Tools for Windows</a> are used.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> A debugger is necessary to obtain a <a href="/wiki/Call_stack" title="Call stack">stack</a> trace, and may be required to ascertain the true cause of the problem; as the information on-screen is limited and thus possibly misleading, it may hide the true source of the error. By default, Windows XP is configured to save only a 64kB minidump when it encounters a stop error, and to then automatically reboot the computer. Because this process happens very quickly, the blue screen may be seen only for an instant or not at all. Users have sometimes noted this as a <a href="/wiki/Booting#Random_reboot" title="Booting">random reboot</a> rather than a traditional stop error, and are only aware of an issue after Windows reboots and displays a notification that it has recovered from a serious error. This happens only when the computer has a function called "Auto Restart" enabled, which can be disabled in the Control Panel which in turn shows the stop error.</p>
<p>Microsoft Windows can also be configured to send live debugging information to a kernel debugger running on a separate <a href="/wiki/Computer" title="Computer">computer</a>. If a stop error is encountered while a live kernel debugger is attached to the system, Windows will halt execution and cause the debugger to break in, rather than displaying the BSoD. The debugger can then be used to examine the contents of memory and determine the source of the problem.</p>
<p>A BSoD can also be caused by a critical boot loader error, where the operating system is unable to access the boot partition due to incorrect storage drivers, a damaged file system or similar problems. The error code in this situation is STOP 0x0000007B (INACCESSIBLE_BOOT_DEVICE). In such cases, there is no memory dump saved. Since the system is unable to boot from the hard drive in this situation, correction of the problem often requires using the repair tools found on the Windows installation disc.</p>
<h3><span class="mw-headline" id="Details">Details</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Blue_Screen_of_Death&amp;action=edit&amp;section=5" title="Edit section: Details">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Before <a href="/wiki/Windows_Server_2012" title="Windows Server 2012">Windows Server 2012</a>, each BSoD displayed an error name in uppercase (e.g. APC_INDEX_MISMATCH), a hexadecimal error number (e.g. 0x00000001) and four parameters. The last two are shown together in the following format:<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup></p>
<blockquote>
<p>error code (parameter 1, parameter 2, parameter 3, parameter 4) error name</p>
</blockquote>
<p>Depending on the error number and its nature, all, some, or even none of the parameters contain data pertaining to what went wrong, and/or where it happened. In addition, the error screens showed four paragraphs of general explanation and advice and may have included other technical data such the file name of the culprit and memory addresses.</p>
<p>With the release of Windows Server 2012, the BSoD was changed, removing all of the above in favor of the error name, and a concise description. Windows 8 added a sad emoticon as well. The hexadecimal error code and parameters can still be found in the <a class="mw-redirect" href="/wiki/Windows_Event_Log" title="Windows Event Log">Windows Event Log</a> or in <a class="mw-redirect" href="/wiki/Memory_dump" title="Memory dump">memory dumps</a>. Windows 10 Build 14393 added a QR code for quick troubleshooting.</p>
<h2><span class="mw-headline" id="Windows_9x">Windows 9x</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Blue_Screen_of_Death&amp;action=edit&amp;section=6" title="Edit section: Windows 9x">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Windows_9X_BSOD.png"><img alt="" class="thumbimage" data-file-height="400" data-file-width="640" height="138" src="//upload.wikimedia.org/wikipedia/commons/thumb/3/3b/Windows_9X_BSOD.png/220px-Windows_9X_BSOD.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/3/3b/Windows_9X_BSOD.png/330px-Windows_9X_BSOD.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/3/3b/Windows_9X_BSOD.png/440px-Windows_9X_BSOD.png 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Windows_9X_BSOD.png" title="Enlarge"></a></div>
A blue screen of death, as appears on <a href="/wiki/Windows_9x" title="Windows 9x">Windows 9x</a></div>
</div>
</div>
<p>The blue screen of death frequently occurs in Microsoft's home desktop operating systems <a href="/wiki/Windows_95" title="Windows 95">Windows 95</a>, <a href="/wiki/Windows_98" title="Windows 98">98</a>, and <a href="/wiki/Windows_ME" title="Windows ME">ME</a>. In these operating systems, the BSoD is the main way for <a href="/wiki/VxD" title="VxD">virtual device drivers</a> to report errors to the user. It is internally referred to by the name of "<code>_VWIN32_FaultPopup</code>". A Windows 9x BSoD gives the user the option either to restart or continue. However, VxDs do not display BSoDs frivolously — they usually indicate a problem that cannot be fixed without restarting the computer, and hence after a BSoD is displayed the system is usually unstable or unresponsive.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (November 2015)">citation needed</span></a></i>]</sup></p>
<p>The most common BSoD is on a 25×80 screen which is the operating system's way of reporting an interrupt caused by a processor exception; it is a more serious form of the <a href="/wiki/General_protection_fault" title="General protection fault">general protection fault</a> dialog boxes. The memory address of the error is given and the error type is a hexadecimal number from 00 to 11 (0 to 17 decimal). The error codes are as follows:<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></p>
<ul>
<li>00: Division <a href="/wiki/Trap_(computing)" title="Trap (computing)">fault</a></li>
<li>02: <a href="/wiki/Non-maskable_interrupt" title="Non-maskable interrupt">Non-Maskable Interrupt</a></li>
<li>04: Overflow Trap</li>
<li>05: Bounds Check Fault</li>
<li>06: Invalid Opcode Fault</li>
<li>07: "<a href="/wiki/Coprocessor" title="Coprocessor">Coprocessor</a> Not Available" Fault</li>
<li>08: Double Fault</li>
<li>09: Coprocessor Segment Overrun</li>
<li>0A: Invalid Task State Segment Fault</li>
<li>0B: Not Present Fault</li>
<li>0C: Stack Fault</li>
<li>0D: General Protection Fault</li>
<li>0E: Page Fault</li>
<li>10: Coprocessor Error Fault</li>
<li>11: Alignment Check Fault</li>
</ul>
<p>Reasons for BSoDs include:</p>
<ul>
<li>Problems that occur with incompatible versions of <a href="/wiki/Dynamic-link_library" title="Dynamic-link library">DLLs</a>: Windows loads these DLLs into memory when they are needed by application programs; if versions are changed, the next time an application loads the DLL it may be different from what the application expects. These incompatibilities increase over time as more new software is installed, and is one of the main reasons why a freshly-installed copy of Windows is more stable than an "old" one.</li>
<li>Faulty or poorly written <a href="/wiki/Device_driver" title="Device driver">device drivers</a></li>
<li>Hardware incompatibilities</li>
</ul>
<p>Damaged hardware may also cause a BSoD.</p>
<p>In Windows 95 and 98, a BSoD occurs when the system attempts to access the file "<code>c:\con\con</code>" or "<code>c:\aux\aux</code>" on the hard drive. This could be inserted on a website to crash visitors' machines. On 16 March 2000, Microsoft released a <a href="/wiki/Patch_(computing)" title="Patch (computing)">security update</a> to resolve this issue.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<p>A famous instance of a Windows 9x BSoD occurred during a presentation of a Windows 98 <a class="mw-redirect" href="/wiki/Beta_software" title="Beta software">Beta</a> by <a href="/wiki/Bill_Gates" title="Bill Gates">Bill Gates</a> at <a href="/wiki/COMDEX" title="COMDEX">COMDEX</a> on April 20, 1998: The demo PC crashed with a BSoD when his assistant, Chris Capossela, connected a scanner to the PC to demonstrate Windows 98's support for <a href="/wiki/Plug_and_play" title="Plug and play">Plug and Play</a> devices. This event brought thunderous applause from the crowd and Gates replied after a nervous pause: "That must be why we're not shipping Windows 98 yet."<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup></p>
