<h1 class="firstHeading" id="firstHeading" lang="en">Nonlinear control</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">

<div class="thumbinner" style="width:442px;"><a class="image" href="/wiki/File:Feedback_loop_with_descriptions.svg"><img alt="" class="thumbimage" data-file-height="126" data-file-width="500" height="111" src="//upload.wikimedia.org/wikipedia/commons/thumb/2/24/Feedback_loop_with_descriptions.svg/440px-Feedback_loop_with_descriptions.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/2/24/Feedback_loop_with_descriptions.svg/660px-Feedback_loop_with_descriptions.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/2/24/Feedback_loop_with_descriptions.svg/880px-Feedback_loop_with_descriptions.svg.png 2x" width="440"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Feedback_loop_with_descriptions.svg" title="Enlarge"></a></div>
A feedback <a href="/wiki/Control_system" title="Control system">control system</a>. It is desired to control a system (often called the <i>plant</i>) so its output follows a desired <i>reference</i> signal. A <i>sensor</i> monitors the output and a <i>controller</i> subtracts the actual output from the desired reference output, and applies this error signal to the system to bring the output closer to the reference. In a nonlinear control system at least one of the blocks, system, sensor, or controller, is nonlinear.</div>
</div>
</div>
<p><b>Nonlinear control</b> theory is the area of <a href="/wiki/Control_theory" title="Control theory">control theory</a> which deals with systems that are <a href="/wiki/Nonlinear_system" title="Nonlinear system">nonlinear</a>, <a href="/wiki/Time-variant_system" title="Time-variant system">time-variant</a>, or both. Control theory is an interdisciplinary branch of engineering and <a href="/wiki/Mathematics" title="Mathematics">mathematics</a> that is concerned with the behavior of <a href="/wiki/Dynamical_system" title="Dynamical system">dynamical systems</a> with inputs, and how to modify the output by changes in the input using <a href="/wiki/Feedback" title="Feedback">feedback</a>. The system to be controlled is called the "<a href="/wiki/Plant_(control_theory)" title="Plant (control theory)">plant</a>". In order to make the output of a system follow a desired reference signal a <a href="/wiki/Controller_(control_theory)" title="Controller (control theory)">controller</a> is designed which compares the output of the plant to the desired output, and provides <a href="/wiki/Feedback" title="Feedback">feedback</a> to the plant to modify the output to bring it closer to the desired output. Control theory is divided into two branches:</p>
<p>Linear control theory applies to systems made of <a href="/wiki/Linear_function" title="Linear function">linear</a> devices; which means they obey the <a href="/wiki/Superposition_principle" title="Superposition principle">superposition principle</a>; the output of the device is proportional to its input. Systems with this property are governed by <a href="/wiki/Linear_equation" title="Linear equation">linear</a> <a href="/wiki/Differential_equation" title="Differential equation">differential equations</a>. A major subclass is systems which in addition have parameters which do not change with time, called <i><a class="mw-redirect" href="/wiki/Linear_time_invariant" title="Linear time invariant">linear time invariant</a></i> (LTI) systems. These systems are amenable to powerful <a href="/wiki/Frequency_domain" title="Frequency domain">frequency domain</a> mathematical techniques of great generality, such as the <a href="/wiki/Laplace_transform" title="Laplace transform">Laplace transform</a>, <a href="/wiki/Fourier_transform" title="Fourier transform">Fourier transform</a>, and <a class="mw-redirect" href="/wiki/Z_transform" title="Z transform">Z transform</a>, <a class="mw-redirect" href="/wiki/Root-locus" title="Root-locus">root-locus</a>, <a href="/wiki/Bode_plot" title="Bode plot">Bode plot</a>, and <a href="/wiki/Nyquist_stability_criterion" title="Nyquist stability criterion">Nyquist stability criterion</a>. These lead to a description of the system using terms like <a href="/wiki/Bandwidth_(signal_processing)" title="Bandwidth (signal processing)">bandwidth</a>, <a href="/wiki/Frequency_response" title="Frequency response">frequency response</a>, <a class="mw-redirect" href="/wiki/Eigenvalue" title="Eigenvalue">eigenvalues</a>, <a href="/wiki/Gain_(electronics)" title="Gain (electronics)">gain</a>, <a class="mw-redirect" href="/wiki/Resonant_frequency" title="Resonant frequency">resonant frequencies</a>, <a href="/wiki/Pole_(complex_analysis)" title="Pole (complex analysis)">poles</a>, and <a href="/wiki/Zero_(complex_analysis)" title="Zero (complex analysis)">zeros</a>, which give solutions for system response and design techniques to most problems of interest.</p>
<p>Nonlinear control theory covers a wider class of systems that do not obey the superposition principle. It applies to more real-world systems, because all real control systems are nonlinear. These systems are often governed by <a class="mw-redirect" href="/wiki/Nonlinear_differential_equation" title="Nonlinear differential equation">nonlinear differential equations</a>. The mathematical techniques which have been developed to handle them are more rigorous and much less general, often applying only to narrow categories of systems. These include <a href="/wiki/Limit_cycle" title="Limit cycle">limit cycle</a> theory, <a href="/wiki/Poincar%C3%A9_map" title="Poincaré map">Poincaré maps</a>, <a class="mw-redirect" href="/wiki/Liapunov_function" title="Liapunov function">Liapunov stability theory</a>, and <a href="/wiki/Describing_function" title="Describing function">describing functions</a>. If only solutions near a stable point are of interest, nonlinear systems can often be <a href="/wiki/Linearization" title="Linearization">linearized</a> by approximating them by a linear system obtained by expanding the nonlinear solution in a <a class="mw-redirect" href="/wiki/Infinite_series" title="Infinite series">series</a>, and then linear techniques can be used.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> Nonlinear systems are often analyzed using <a href="/wiki/Numerical_method" title="Numerical method">numerical methods</a> on <a href="/wiki/Computer" title="Computer">computers</a>, for example by <a href="/wiki/Simulation" title="Simulation">simulating</a> their operation using a <a href="/wiki/Simulation_language" title="Simulation language">simulation language</a>. Even if the plant is linear, a nonlinear controller can often have attractive features such as simpler implementation, faster speed, more accuracy, or reduced control energy, which justify the more difficult design procedure.</p>
<p>An example of a nonlinear control system is a <a href="/wiki/Thermostat" title="Thermostat">thermostat</a>-controlled heating system. A building heating system such as a furnace has a nonlinear response to changes in temperature; it is either "on" or "off", it does not have the fine control in response to temperature differences that a proportional (linear) device would have. Therefore, the furnace is off until the temperature falls below the "turn on" setpoint of the thermostat, when it turns on. Due to the heat added by the furnace, the temperature increases until it reaches the "turn off" setpoint of the thermostat, which turns the furnace off, and the cycle repeats. This cycling of the temperature about the desired temperature is called a <i><a href="/wiki/Limit_cycle" title="Limit cycle">limit cycle</a></i>, and is characteristic of nonlinear control systems.</p>
<p></p>

<h2><span class="mw-headline" id="Properties_of_nonlinear_systems">Properties of nonlinear systems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Nonlinear_control&amp;action=edit&amp;section=1" title="Edit section: Properties of nonlinear systems">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Some properties of nonlinear dynamic systems are</p>
<ul>
<li>They do not follow the principle of <a href="/wiki/Superposition_principle" title="Superposition principle">superposition</a> (linearity and homogeneity).</li>
<li>They may have multiple isolated equilibrium points.</li>
<li>They may exhibit properties such as <a href="/wiki/Limit_cycle" title="Limit cycle">limit cycle</a>, <a href="/wiki/Bifurcation_theory" title="Bifurcation theory">bifurcation</a>, <a href="/wiki/Chaos_theory" title="Chaos theory">chaos</a>.</li>
<li>Finite escape time: Solutions of nonlinear systems may not exist for all times.</li>
</ul>

<p>There are several well-developed techniques for analyzing nonlinear feedback systems:</p>
<ul>
<li><a href="/wiki/Describing_function" title="Describing function">Describing function</a> method</li>

<li><a href="/wiki/Singular_perturbation" title="Singular perturbation">Singular perturbation</a> method</li>

<li><a class="new" href="/w/index.php?title=Passivity_analysis&amp;action=edit&amp;redlink=1" title="Passivity analysis (page does not exist)">Passivity analysis</a></li>

<h2><span class="mw-headline" id="Nonlinear_feedback_analysis_.E2.80.93_The_Lur.27e_problem">Nonlinear feedback analysis – The Lur'e problem</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Nonlinear_control&amp;action=edit&amp;section=3" title="Edit section: Nonlinear feedback analysis – The Lur'e problem">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:402px;"><a class="image" href="/wiki/File:Lur%27e_Problem_Block.jpg"><img alt="" class="thumbimage" data-file-height="312" data-file-width="774" height="161" src="//upload.wikimedia.org/wikipedia/commons/thumb/3/3c/Lur%27e_Problem_Block.jpg/400px-Lur%27e_Problem_Block.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/3/3c/Lur%27e_Problem_Block.jpg/600px-Lur%27e_Problem_Block.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/3/3c/Lur%27e_Problem_Block.jpg 2x" width="400"/></a>

<h3><span class="mw-headline" id="Absolute_stability_problem">Absolute stability problem</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Nonlinear_control&amp;action=edit&amp;section=4" title="Edit section: Absolute stability problem">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Consider:</p>
<ol>
<li>(A,B) is controllable and (C,A) is observable</li>
<li>two real numbers a, b with a&lt;b, defining a sector for function Φ</li>
</ol>
<p>The problem is to derive conditions involving only the transfer matrix H(s) and {a,b} such that x=0 is a globally uniformly asymptotically stable equilibrium of the system. This is known as the Lur'e problem. There are two well-known wrong conjections on absolute stability:</p>
<ul>
<li>The <a href="/wiki/Aizerman%27s_conjecture" title="Aizerman's conjecture">Aizerman's conjecture</a></li>
<li>The <a href="/wiki/Kalman%27s_conjecture" title="Kalman's conjecture">Kalman's conjecture</a>.</li>
</ul>
<p>There are counterexamples to Aizerman's and Kalman's conjectures such that nonlinearity belongs to the sector of linear stability and unique stable equilibrium coexists with a stable periodic solution -- <a href="/wiki/Hidden_oscillation" title="Hidden oscillation">hidden oscillation</a>.</p>
<p>There are two main theorems concerning the problem:</p>
<ul>
<li>The <a href="/wiki/Circle_criterion" title="Circle criterion">Circle criterion</a></li>
<li>The <a class="mw-redirect" href="/wiki/Vasile_M_Popov" title="Vasile M Popov">Popov</a> criterion.</li>
</ul>
<p>which give sufficient conditions of absolute stability.</p>
<h4><span class="mw-headline" id="Popov_criterion">Popov criterion</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Nonlinear_control&amp;action=edit&amp;section=5" title="Edit section: Popov criterion">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>The sub-class of Lur'e systems studied by <a class="mw-redirect" href="/wiki/Vasile_M_Popov" title="Vasile M Popov">Popov</a> is described by:</p>
<p><span><span class="mwe-math-mathml-inline mwe-math-mathml-a11y mw-math-element" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML">
<semantics>

<p><b>Theorem:</b> Consider the system (1)-(2) and suppose</p>
<ol>
<li>A is <a class="mw-redirect" href="/wiki/Routh_Hurwitz_stability_criterion" title="Routh Hurwitz stability criterion">Hurwitz</a></li>
<li>(A,b) is controllable</li>
<li>(A,c) is observable</li>
<li>d&gt;0 and</li>
<li>Φ ∈ (0,∞)</li>
</ol>
<p>then the system is <a class="mw-redirect" href="/wiki/Globally_asymptotically_stable" title="Globally asymptotically stable">globally asymptotically stable</a> if there exists a number r&gt;0 such that<br/>
inf<sub>ω ∈ R</sub> Re[(1+jωr)h(jω)] &gt; 0 .</p>
<p>Things to be noted:</p>
<ul>
<li>The Popov criterion is applicable only to autonomous systems</li>
<li>The system studied by Popov has a pole at the origin and there is no direct pass-through from input to output</li>
<li>The nonlinearity Φ must satisfy an open sector condition</li>
</ul>
