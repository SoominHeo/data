<h1 class="firstHeading" id="firstHeading" lang="en">Color management</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p>In digital imaging systems, <b>color management</b> is the controlled <a href="/wiki/Data_conversion" title="Data conversion">conversion</a> between the <a href="/wiki/Color" title="Color">color</a> representations of various devices, such as <a href="/wiki/Image_scanner" title="Image scanner">image scanners</a>, <a href="/wiki/Digital_camera" title="Digital camera">digital cameras</a>, monitors, TV screens, film printers, <a class="mw-redirect" href="/wiki/Computer_printer" title="Computer printer">computer printers</a>, offset presses, and corresponding media.</p>
<p>The primary goal of color management is to obtain a good match across color devices; for example, the colors of one frame of a video should appear the same on a computer <a class="mw-redirect" href="/wiki/LCD" title="LCD">LCD monitor</a>, on a plasma TV screen, and as a printed poster. Color management helps to achieve the same appearance on all of these devices, provided the devices are capable of delivering the needed color intensities. With photography it is often critical that your prints or online gallery appear how they were intended. Color management cannot guarantee identical color reproduction, as this is rarely possible, but it can at least give you more control over any changes which may occur.<sup class="reference" id="cite_ref-cambridgeincolour.com_1-0"><a href="#cite_note-cambridgeincolour.com-1">[1]</a></sup></p>
<p>Parts of this technology are implemented in the <a href="/wiki/Operating_system" title="Operating system">operating system</a> (OS), helper libraries, the application, and devices. A cross-platform view of color management is the use of an ICC-compatible color management system. The <a href="/wiki/International_Color_Consortium" title="International Color Consortium">International Color Consortium</a> (ICC) is an industry consortium that has defined:</p>
<ul>
<li>An open standard for a <i>Color Matching Module</i> (CMM) at the OS level</li>
<li><a href="/wiki/ICC_profile" title="ICC profile">color profiles</a> for:
<ul>
<li>Devices, including devicelink-profiles that represent a complete color transformation from source device to target device</li>
<li><i>Working spaces</i>, the color spaces in which color data is meant to be manipulated</li>
</ul>
</li>
</ul>
<p>There are other approaches to color management besides using <a href="/wiki/ICC_profile" title="ICC profile">ICC profiles</a>. This is partly due to history and partly because of other needs than the ICC standard covers. The film and broadcasting industries make use of some of the same concepts, but they frequently rely on more limited boutique solutions. The film industry, for instance, often uses <a class="mw-redirect" href="/wiki/3D_LUT" title="3D LUT">3D LUTs</a> (<a href="/wiki/Lookup_table" title="Lookup table">lookup table</a>) to represent a complete color transformation for a specific RGB encoding. At the consumer level, color management currently applies more to still images than video, in which color management is still in its infancy.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p></p>

<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ol>
<li>Characterize. Every color-managed device requires a personalized table, or "color profile," which characterizes the color response of that particular device.</li>
<li>Standardize. Each color profile describes these colors relative to a standardized set of reference colors (the "Profile Connection Space").</li>
<li>Translate. Color-managed software then uses these standardized profiles to translate color from one device to another. This is usually performed by a color management module (CMM).<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></li>
</ol>
<h2><span class="mw-headline" id="Hardware">Hardware</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=2" title="Edit section: Hardware">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Characterization">Characterization</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=3" title="Edit section: Characterization">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">See also: <a href="/wiki/ICC_profile" title="ICC profile">ICC profile</a></div>
<p>To describe the behavior of various output devices, they must be compared (measured) in relation to a standard <a href="/wiki/Color_space" title="Color space">color space</a>. Often a step called linearization is performed first, to undo the effect of <a href="/wiki/Gamma_correction" title="Gamma correction">gamma correction</a> that was done to get the most out of limited <a href="/wiki/8-bit_color" title="8-bit color">8-bit color</a> paths. Instruments used for measuring device colors include <a href="/wiki/Tristimulus_colorimeter" title="Tristimulus colorimeter">colorimeters</a> and <a class="mw-redirect" href="/wiki/Spectrophotometer" title="Spectrophotometer">spectrophotometers</a>. As an intermediate result, the device <a href="/wiki/Gamut" title="Gamut">gamut</a> is described in the form of scattered measurement data. The transformation of the scattered measurement data into a more regular form, usable by the application, is called <i>profiling</i>. Profiling is a complex process involving mathematics, intense computation, judgment, testing, and iteration. After the profiling is finished, an idealized color description of the device is created. This description is called a <i>profile</i>.</p>
<h3><span class="mw-headline" id="Calibration">Calibration</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=4" title="Edit section: Calibration">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Color_calibration" title="Color calibration">Color calibration</a></div>
<p>Calibration is like characterization, except that it can include the adjustment of the device, as opposed to just the measurement of the device. Color management is sometimes sidestepped by calibrating devices to a common standard color space such as <a href="/wiki/SRGB" title="SRGB">sRGB</a>; when such calibration is done well enough, no color translations are needed to get all devices to handle colors consistently. This avoidance of the complexity of color management was one of the goals in the development of sRGB.</p>

<h3><span class="mw-headline" id="Embedding">Embedding</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=6" title="Edit section: Embedding">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Image formats themselves (such as <a href="/wiki/TIFF" title="TIFF">TIFF</a>, <a href="/wiki/JPEG" title="JPEG">JPEG</a>, <a href="/wiki/Portable_Network_Graphics" title="Portable Network Graphics">PNG</a>, <a href="/wiki/Encapsulated_PostScript" title="Encapsulated PostScript">EPS</a>, <a href="/wiki/Portable_Document_Format" title="Portable Document Format">PDF</a>, and <a href="/wiki/Scalable_Vector_Graphics" title="Scalable Vector Graphics">SVG</a>) may contain embedded <a class="mw-redirect" href="/wiki/ICC_Profile" title="ICC Profile">color profiles</a> but are not required to do so by the image format. The <a href="/wiki/International_Color_Consortium" title="International Color Consortium">International Color Consortium</a> standard was created to bring various developers and manufacturers together. The ICC standard permits the exchange of output device characteristics and color spaces in the form of <a href="/wiki/Metadata" title="Metadata">metadata</a>. This allows the embedding of color profiles into images as well as storing them in a database or a profile directory.</p>
<h3><span class="mw-headline" id="Working_spaces">Working spaces</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=7" title="Edit section: Working spaces">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Working spaces, such as <a href="/wiki/SRGB" title="SRGB">sRGB</a>, <a href="/wiki/Adobe_RGB_color_space" title="Adobe RGB color space">Adobe RGB</a> or <a class="mw-redirect" href="/wiki/ProPhoto" title="ProPhoto">ProPhoto</a> are color spaces that facilitate good results while editing. For instance, pixels with equal values of R,G,B should appear neutral. Using a large (gamut) working space will lead to <a href="/wiki/Posterization" title="Posterization">posterization</a>, while using a small working space will lead to <a href="/wiki/Clipping_(photography)" title="Clipping (photography)">clipping</a>.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> This trade-off is a consideration for the critical image editor.</p>
<h2><span class="mw-headline" id="Color_transformation">Color transformation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=8" title="Edit section: Color transformation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Color transformation, or color space conversion, is the transformation of the representation of a color from one <a href="/wiki/Color_space" title="Color space">color space</a> to another. This calculation is required whenever data is exchanged inside a color-managed chain and carried out by a <a href="/wiki/Color_management#Color_management_module" title="Color management">Color Matching Module</a>. Transforming profiled color information to different output devices is achieved by referencing the profile data into a standard color space. It makes it easier to convert colors from one device to a selected standard color space and from that to the colors of another device. By ensuring that the reference color space covers the many possible colors that humans can see, this concept allows one to exchange colors between many different color output devices. Color transformations can be represented by two profiles (source profile and target profile) or by a devicelink profile. In this process there are approximations involved which make sure that the image keeps its important color qualities and also gives an opportunity to control on how the colors are being changed.<sup class="reference" id="cite_ref-cambridgeincolour.com_1-1"><a href="#cite_note-cambridgeincolour.com-1">[1]</a></sup></p>
<h3><span class="mw-headline" id="Profile_connection_space">Profile connection space</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=9" title="Edit section: Profile connection space">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In the terminology of the <a href="/wiki/International_Color_Consortium" title="International Color Consortium">International Color Consortium</a>, a translation between two color spaces can go through a <i>profile connection space</i> (PCS): Color Space 1 → PCS (<a class="mw-redirect" href="/wiki/CIELAB" title="CIELAB">CIELAB</a> or <a class="mw-redirect" href="/wiki/CIEXYZ" title="CIEXYZ">CIEXYZ</a>) → Color space 2; conversions into and out of the PCS are each specified by a profile.<sup class="reference" id="cite_ref-rodney_5-0"><a href="#cite_note-rodney-5">[5]</a></sup></p>
<h3><span class="mw-headline" id="Gamut_mapping">Gamut mapping</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=10" title="Edit section: Gamut mapping">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In nearly every translation process, we have to deal with the fact that the color <a href="/wiki/Gamut" title="Gamut">gamut</a> of different devices vary in range which makes an accurate reproduction impossible.<sup class="reference" id="cite_ref-cambridgeincolour.com_1-2"><a href="#cite_note-cambridgeincolour.com-1">[1]</a></sup> They therefore need some rearrangement near the borders of the gamut. Some colors must be shifted to the inside of the gamut, as they otherwise cannot be represented on the output device and would simply be clipped. This so-called gamut mismatch occurs for example, when we translate from the RGB color space with a wider gamut into the CMYK color space with a narrower gamut range. In this example, the dark highly saturated purplish-blue color of a typical computer monitor’s "blue" primary is impossible to print on paper with a typical <a class="mw-redirect" href="/wiki/CMYK" title="CMYK">CMYK</a> printer. The nearest approximation within the printer’s gamut will be much less saturated. Conversely, an inkjet printer’s "cyan" primary, a saturated mid-brightness greenish-blue, is outside the gamut of a typical computer monitor. The color management system can utilize various methods to achieve desired results and give experienced users control of the gamut mapping behavior.</p>
<h3><span class="mw-headline" id="Rendering_intent">Rendering intent</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=11" title="Edit section: Rendering intent">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>When the gamut of source color space exceeds that of the destination, saturated colors are liable to become <a href="/wiki/Clipping_(photography)" title="Clipping (photography)">clipped</a> (inaccurately represented), or more formally <a href="/wiki/Burned_(image)" title="Burned (image)">burned</a>. The color management module can deal with this problem in several ways. The ICC specification includes four different <b>rendering intents</b>: <i>absolute colorimetric</i>, <i>relative colorimetric</i>, <i>perceptual</i>, and <i>saturation</i>.<sup class="reference" id="cite_ref-rodney_5-1"><a href="#cite_note-rodney-5">[5]</a></sup><sup class="reference" id="cite_ref-cic_6-0"><a href="#cite_note-cic-6">[6]</a></sup><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></p>
<dl>
<dt>Absolute colorimetric </dt>
</dl>
<p>Absolute colorimetry and relative colorimetry actually use the same table but differ in the adjustment for the white point media. If the output device has a much larger gamut than the source profile, i.e., all the colors in the source can be represented in the output, using the absolute colorimetry rendering intent would "ideally" (ignoring noise, precision, etc.) give an exact output of the specified CIELAB values. Perceptually, the colors may appear incorrect, but instrument measurements of the resulting output would match the source. Colors outside of the proof print system's possible color are mapped to the boundary of the color gamut. Absolute colorimetry is useful to get an exact specified color (e.g., IBM blue), or to quantify the accuracy of mapping methods.</p>
<dl>
<dt>Relative colorimetric </dt>
</dl>
<p>The goal in relative colorimetry is to be truthful to the specified color, with only a correction for the media. Relative colorimetry is useful in proofing applications, since you are using it to get an idea of how a print on one device will appear on a different device. Media differences are the only thing you really would like to adjust for. Obviously there has to be some gamut mapping going on also. Usually this is done in a way where hue and lightness are maintained at the cost of reduced saturation. Relative colorimetric is the default rendering intent on most systems.</p>
<dl>
<dt>Perceptual and Saturation </dt>
</dl>
<p>The perceptual and saturation intents are where the results really depend upon the profile maker. This is even how some of the competitors in this market differentiate themselves. These intents should be created by the profile maker so that pleasing images occur with the perceptual intent while eye-catching business graphics occur with the saturation intent. This is achieved through the use of different perceptual remaps of the data as well as different gamut mapping methods. Perceptual rendering is recommended for color separation.</p>
<p>In practice, photographers almost always use relative or perceptual intent, as for natural images, absolute causes <a class="mw-redirect" href="/wiki/Color_cast" title="Color cast">color cast</a>, while saturation produces unnatural colors.<sup class="reference" id="cite_ref-cic_6-1"><a href="#cite_note-cic-6">[6]</a></sup> Relative intent handles out-of-gamut by clipping (burning) these colors to the edge of the gamut, leaving in-gamut colors unchanged, while perceptual intent smoothly moves out-of-gamut colors into gamut, preserving gradations, but distorts in-gamut colors in the process. If an entire image is in-gamut, relative is perfect, but when there are out of gamut colors, which is more preferable depends on a case-by-case basis.</p>
<p>Saturation intent is most useful in charts and diagrams, where there is a discrete palette of colors that the designer wants saturated to make them intense, but where specific hue is less important.</p>
<p>Before the actual rendering intent is carried out, you can temporarily simulate the rendering by <a class="mw-redirect" href="/wiki/Soft_proofing" title="Soft proofing">soft proofing</a>.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> It is a useful tool as it predicts the outcome of the colors and is available as an application in many color management systems.</p>
<h2><span class="mw-headline" id="Implementation">Implementation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=12" title="Edit section: Implementation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Color_management_module">Color management module</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=13" title="Edit section: Color management module">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><i>Color matching module</i> (also -<i>method</i> or -<i>system</i>) is a software algorithm that adjusts the numerical values that get sent to or received from different devices so that the perceived color they produce remains consistent. The key issue here is how to deal with a color that cannot be reproduced on a certain device in order to show it through a different device as if it were visually the same color, just as when the reproducible color range between color transparencies and printed matters are different. There is no common method for this process, and the performance depends on the capability of each color matching method.</p>
<p>Some well known CMMs are <a class="mw-redirect" href="/wiki/ColorSync" title="ColorSync">ColorSync</a>, Adobe CMM, <a class="mw-redirect" href="/wiki/LittleCMS" title="LittleCMS">LittleCMS</a>, and ArgyllCMS.</p>
<h3><span class="mw-headline" id="Operating_system_level">Operating system level</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=14" title="Edit section: Operating system level">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Apple_Inc." title="Apple Inc.">Apple's</a> <a href="/wiki/Classic_Mac_OS" title="Classic Mac OS">classic Mac OS</a> and <a href="/wiki/MacOS" title="MacOS">macOS</a> operating systems have provided OS-level color management APIs since 1993, through <a class="mw-redirect" href="/wiki/ColorSync" title="ColorSync">ColorSync</a>. macOS has added automatic color management (assuming sRGB for most things) automatically in the OS, and applications have to work around this to provide more accurate color management.</p>
<p>Since 1997 color management in Windows is available through an ICC color management system (ICM). Beginning with <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a>, <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> introduced a new color architecture known as <a href="/wiki/Windows_Color_System" title="Windows Color System">Windows Color System</a>.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> WCS supplements the <i>Image Color Management</i> (ICM) system in Windows 2000 and Windows XP, originally written by <a href="/wiki/Heidelberger_Druckmaschinen" title="Heidelberger Druckmaschinen">Heidelberg</a>.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup><sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></p>
<p>Operating systems that use the <a href="/wiki/X_Window_System" title="X Window System">X Window System</a> for graphics can use <a href="/wiki/ICC_profile" title="ICC profile">ICC profiles</a>, and support for <a href="/wiki/Linux_color_management" title="Linux color management">color management on Linux</a>, still less mature than on other platforms, is coordinated through OpenICC at <a href="/wiki/Freedesktop.org" title="Freedesktop.org">freedesktop.org</a> and makes use of <a class="mw-redirect" href="/wiki/LittleCMS" title="LittleCMS">LittleCMS</a>.</p>
<h3><span class="mw-headline" id="File_level">File level</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Color_management&amp;action=edit&amp;section=15" title="Edit section: File level">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Certain image filetypes (<a href="/wiki/TIFF" title="TIFF">TIFF</a> and <a class="mw-redirect" href="/wiki/Photoshop" title="Photoshop">Photoshop</a>) include the notion of <a class="mw-redirect" href="/wiki/Color_channels" title="Color channels">color channels</a> for specifying the <i>color mode</i> of the file. The most commonly used channels are <a class="mw-redirect" href="/wiki/RGB" title="RGB">RGB</a> (for display and printing) and <a class="mw-redirect" href="/wiki/CMYK" title="CMYK">CMYK</a> (for commercial printing). An additional <a href="/wiki/Channel_(digital_image)#Alpha_channel" title="Channel (digital image)"><i>alpha</i> channel</a> may specify a transparency mask value. Some image software (such as <a class="mw-redirect" href="/wiki/Photoshop" title="Photoshop">Photoshop</a>) perform automatic <a class="mw-redirect" href="/wiki/Color_separation" title="Color separation">color separation</a> to maintain color information in CMYK mode using a specified <a href="/wiki/ICC_profile" title="ICC profile">ICC profile</a> such as <b>US Web Coated (SWOP) v2</b>.</p>

<p>As of 2012<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Color_management&amp;action=edit">[update]</a></sup><sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Manual_of_Style/Dates_and_numbers#Chronological_items" title="Wikipedia:Manual of Style/Dates and numbers"><span title="The time period mentioned near this tag is ambiguous. (August 2012)">when?</span></a></i>]</sup>, notable browser support for color management is:</p>
<ul>
<li>Firefox: from version 3.5 enabled by default for ICC v2 tagged images,<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> version 8.0 has ICC v4 profiles support,<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> but it needs to be activated manually.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></li>
<li>Internet Explorer: version 9 is the first Microsoft browser to partly support ICC profiles, but it does not render images correctly according to the Windows ICC settings (it only converts non-sRGB images to the sRGB profile) and therefore provides no real color management at all<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (August 2012)">citation needed</span></a></i>]</sup></li>
<li><a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a>: uses the system provided ICC v2 and v4 support on <a href="/wiki/MacOS" title="MacOS">macOS</a>, and from version 22 supports ICC v2 profiles by default on other platforms.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup></li>
<li>Safari: has support starting with version 2.0<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (August 2012)">citation needed</span></a></i>]</sup></li>
<li>Opera: has support since 12.10<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup> for ICC v4.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup></li>
<li><a href="/wiki/Pale_Moon_(web_browser)" title="Pale Moon (web browser)">Pale Moon</a> supported ICC v2 from its first release, and v4 since Pale Moon 20.2 (2013).<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup></li>
</ul>
