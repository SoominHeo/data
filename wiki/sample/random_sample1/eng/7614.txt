<p>In <a href="/wiki/Computer_science" title="Computer science">computer science</a>, <b>selection sort</b> is a <a href="/wiki/Sorting_algorithm" title="Sorting algorithm">sorting algorithm</a>, specifically an <a href="/wiki/In-place_algorithm" title="In-place algorithm">in-place</a> <a href="/wiki/Comparison_sort" title="Comparison sort">comparison sort</a>. It has <a href="/wiki/Big_O_notation" title="Big O notation">O</a>(<i>n</i><sup>2</sup>) <a href="/wiki/Time_complexity" title="Time complexity">time complexity</a>, making it inefficient on large lists, and generally performs worse than the similar <a href="/wiki/Insertion_sort" title="Insertion sort">insertion sort</a>. Selection sort is noted for its simplicity, and it has performance advantages over more complicated algorithms in certain situations, particularly where auxiliary memory is limited.</p>
<p>The algorithm divides the input list into two parts: the sublist of items already sorted, which is built up from left to right at the front (left) of the list, and the sublist of items remaining to be sorted that occupy the rest of the list. Initially, the sorted sublist is empty and the unsorted sublist is the entire input list. The algorithm proceeds by finding the smallest (or largest, depending on sorting order) element in the unsorted sublist, exchanging (swapping) it with the leftmost unsorted element (putting it in sorted order), and moving the sublist boundaries one element to the right.</p>

<p>Selection sort can also be used on list structures that make add and remove efficient, such as a <a href="/wiki/Linked_list" title="Linked list">linked list</a>. In this case it is more common to <i>remove</i> the minimum element from the remainder of the list, and then <i>insert</i> it at the end of the values sorted so far. For example:</p>
<pre style="overflow:auto; width:auto;">

11 64 25 12 22

11 12 64 25 22

11 12 22 64 25

11 12 22 25 64
</pre>
<h2><span class="mw-headline" id="Implementation">Implementation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Selection_sort&amp;action=edit&amp;section=2" title="Edit section: Implementation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

<pre>
<span class="cm">/* a[0] to a[n-1] is the array to sort */</span>
<span class="kt">int</span> <span class="n">i</span><span class="p">,</span><span class="n">j</span><span class="p">;</span>
<span class="kt">int</span> <span class="n">n</span><span class="p">;</span>

  <span class="p">{</span>

<h2><span class="mw-headline" id="Analysis">Analysis</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Selection_sort&amp;action=edit&amp;section=3" title="Edit section: Analysis">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Selection sort is not difficult to analyze compared to other sorting algorithms since none of the loops depend on the data in the array. Selecting the lowest element requires scanning all <i>n</i> elements (this takes <i>n</i> − 1 comparisons) and then swapping it into the first position. Finding the next lowest element requires scanning the remaining <i>n</i> − 1 elements and so on, for (<i>n</i> − 1) + (<i>n</i> − 2) + ... + 2 + 1 = <i>n</i>(<i>n</i> - 1) / 2 ∈ Θ(<i>n</i><sup>2</sup>) comparisons (see <a href="/wiki/Arithmetic_progression" title="Arithmetic progression">arithmetic progression</a>).<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> Each of these scans requires one swap for <i>n</i> − 1 elements (the final element is already in place).</p>
<h2><span class="mw-headline" id="Comparison_to_other_sorting_algorithms">Comparison to other sorting algorithms</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Selection_sort&amp;action=edit&amp;section=4" title="Edit section: Comparison to other sorting algorithms">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Among simple average-case Θ(<i>n</i><sup>2</sup>) algorithms, selection sort almost always outperforms <a href="/wiki/Bubble_sort" title="Bubble sort">bubble sort</a> and <a href="/wiki/Gnome_sort" title="Gnome sort">gnome sort</a>. <a href="/wiki/Insertion_sort" title="Insertion sort">Insertion sort</a> is very similar in that after the <i>k</i>th iteration, the first <i>k</i> elements in the array are in sorted order. Insertion sort's advantage is that it only scans as many elements as it needs in order to place the <i>k</i> + 1st element, while selection sort must scan all remaining elements to find the <i>k</i> + 1st element.</p>
<p>Simple calculation shows that insertion sort will therefore usually perform about half as many comparisons as selection sort, although it can perform just as many or far fewer depending on the order the array was in prior to sorting. It can be seen as an advantage for some <a href="/wiki/Real-time_computing" title="Real-time computing">real-time</a> applications that selection sort will perform identically regardless of the order of the array, while insertion sort's running time can vary considerably. However, this is more often an advantage for insertion sort in that it runs much more efficiently if the array is already sorted or "close to sorted."</p>
<p>While selection sort is preferable to insertion sort in terms of number of writes (Θ(<i>n</i>) swaps versus Ο(<i>n</i><sup>2</sup>) swaps), it almost always far exceeds (and never beats) the number of writes that <a href="/wiki/Cycle_sort" title="Cycle sort">cycle sort</a> makes, as cycle sort is theoretically optimal in the number of writes. This can be important if writes are significantly more expensive than reads, such as with <a href="/wiki/EEPROM" title="EEPROM">EEPROM</a> or <a href="/wiki/Flash_memory" title="Flash memory">Flash</a> memory, where every write lessens the lifespan of the memory.</p>
<p>Finally, selection sort is greatly outperformed on larger arrays by Θ(<i>n</i> log <i>n</i>) <a href="/wiki/Divide_and_conquer_algorithm" title="Divide and conquer algorithm">divide-and-conquer algorithms</a> such as <a class="mw-redirect" href="/wiki/Mergesort" title="Mergesort">mergesort</a>. However, insertion sort or selection sort are both typically faster for small arrays (i.e. fewer than 10–20 elements). A useful optimization in practice for the recursive algorithms is to switch to insertion sort or selection sort for "small enough" sublists.</p>
<h2><span class="mw-headline" id="Variants">Variants</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Selection_sort&amp;action=edit&amp;section=5" title="Edit section: Variants">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/Heapsort" title="Heapsort">Heapsort</a> greatly improves the basic algorithm by using an <a href="/wiki/Implicit_data_structure" title="Implicit data structure">implicit</a> <a href="/wiki/Heap_(data_structure)" title="Heap (data structure)">heap</a> <a href="/wiki/Data_structure" title="Data structure">data structure</a> to speed up finding and removing the lowest datum. If implemented correctly, the heap will allow finding the next lowest element in Θ(log <i>n</i>) time instead of Θ(<i>n</i>) for the inner loop in normal selection sort, reducing the total running time to Θ(<i>n</i> log <i>n</i>).</p>
<p>A bidirectional variant of selection sort, called <b>cocktail sort</b>, is an algorithm which finds both the minimum and maximum values in the list in every pass. This reduces the number of scans of the list by a factor of 2, eliminating some loop overhead but not actually decreasing the number of comparisons or swaps. Note, however, that <a class="mw-redirect" href="/wiki/Cocktail_sort" title="Cocktail sort">cocktail sort</a> more often refers to a bidirectional variant of bubble sort.</p>
<p>Selection sort can be implemented as a <a href="/wiki/Sorting_algorithm#Classification" title="Sorting algorithm">stable sort</a>. If, rather than swapping in step 2, the minimum value is inserted into the first position (that is, all intervening items moved down), the algorithm is stable. However, this modification either requires a data structure that supports efficient insertions or deletions, such as a linked list, or it leads to performing Θ(<i>n</i><sup>2</sup>) writes.</p>
<p>In the <b>bingo sort</b> variant, items are ordered by repeatedly looking through the remaining items to find the greatest value and moving all items with that value to their final location.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> Like <a href="/wiki/Counting_sort" title="Counting sort">counting sort</a>, this is an efficient variant if there are many duplicate values. Indeed, selection sort does one pass through the remaining items for each item moved. Bingo sort does one pass for each value (not item): after an initial pass to find the biggest value, the next passes can move every item with that value to its final location while finding the next value as in the following <a href="/wiki/Pseudocode" title="Pseudocode">pseudocode</a> (arrays are <b>zero-based and the for-loop includes both the top and bottom limits</b>, as in <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a>):</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">bingo</span><span class="p">(</span><span class="k">array</span> <span class="n">A</span><span class="p">)</span>

<span class="cm">{ This procedure sorts in ascending order. }</span>
<span class="k">begin</span>
    <span class="nb">max</span> <span class="o">:=</span> <span class="nb">length</span><span class="p">(</span><span class="n">A</span><span class="p">)</span><span class="o">-</span><span class="mi">1</span><span class="o">;</span>

    <span class="cm">{ The first iteration is written to look very similar to the subsequent ones, but</span>
<span class="cm">      without swaps. }</span>
    <span class="n">nextValue</span> <span class="o">:=</span> <span class="n">A</span><span class="p">[</span><span class="nb">max</span><span class="p">]</span><span class="o">;</span>
    <span class="k">for</span> <span class="n">i</span> <span class="o">:=</span> <span class="nb">max</span> <span class="o">-</span> <span class="mi">1</span> <span class="k">downto</span> <span class="mi">0</span> <span class="k">do</span>
        <span class="k">if</span> <span class="n">A</span><span class="p">[</span><span class="n">i</span><span class="p">]</span> <span class="o">&gt;</span> <span class="n">nextValue</span> <span class="k">then</span>
            <span class="n">nextValue</span> <span class="o">:=</span> <span class="n">A</span><span class="p">[</span><span class="n">i</span><span class="p">]</span><span class="o">;</span>
    <span class="k">while</span> <span class="p">(</span><span class="nb">max</span> <span class="o">&gt;</span> <span class="mi">0</span><span class="p">)</span> <span class="k">and</span> <span class="p">(</span><span class="n">A</span><span class="p">[</span><span class="nb">max</span><span class="p">]</span> <span class="o">=</span> <span class="n">nextValue</span><span class="p">)</span> <span class="k">do</span>
        <span class="nb">max</span> <span class="o">:=</span> <span class="nb">max</span> <span class="o">-</span> <span class="mi">1</span><span class="o">;</span>

    <span class="k">while</span> <span class="nb">max</span> <span class="o">&gt;</span> <span class="mi">0</span> <span class="k">do</span> <span class="k">begin</span>
        <span class="n">value</span> <span class="o">:=</span> <span class="n">nextValue</span><span class="o">;</span>
        <span class="n">nextValue</span> <span class="o">:=</span> <span class="n">A</span><span class="p">[</span><span class="nb">max</span><span class="p">]</span><span class="o">;</span>
        <span class="k">for</span> <span class="n">i</span> <span class="o">:=</span> <span class="nb">max</span> <span class="o">-</span> <span class="mi">1</span> <span class="k">downto</span> <span class="mi">0</span> <span class="k">do</span>
             <span class="k">if</span> <span class="n">A</span><span class="p">[</span><span class="n">i</span><span class="p">]</span> <span class="o">=</span> <span class="n">value</span> <span class="k">then</span> <span class="k">begin</span>
                 <span class="nb">swap</span><span class="p">(</span><span class="n">A</span><span class="p">[</span><span class="n">i</span><span class="p">]</span><span class="o">,</span> <span class="n">A</span><span class="p">[</span><span class="nb">max</span><span class="p">])</span><span class="o">;</span>
                 <span class="nb">max</span> <span class="o">:=</span> <span class="nb">max</span> <span class="o">-</span> <span class="mi">1</span><span class="o">;</span>
             <span class="k">end</span> <span class="k">else</span> <span class="k">if</span> <span class="n">A</span><span class="p">[</span><span class="n">i</span><span class="p">]</span> <span class="o">&gt;</span> <span class="n">nextValue</span> <span class="k">then</span>
                 <span class="n">nextValue</span> <span class="o">:=</span> <span class="n">A</span><span class="p">[</span><span class="n">i</span><span class="p">]</span><span class="o">;</span>
        <span class="k">while</span> <span class="p">(</span><span class="nb">max</span> <span class="o">&gt;</span> <span class="mi">0</span><span class="p">)</span> <span class="k">and</span> <span class="p">(</span><span class="n">A</span><span class="p">[</span><span class="nb">max</span><span class="p">]</span> <span class="o">=</span> <span class="n">nextValue</span><span class="p">)</span> <span class="k">do</span>
            <span class="nb">max</span> <span class="o">:=</span> <span class="nb">max</span> <span class="o">-</span> <span class="mi">1</span><span class="o">;</span>
    <span class="k">end</span><span class="o">;</span>
<span class="k">end</span><span class="o">;</span>
</pre></div>
<p>Thus, if on average there are more than two items with the same value, bingo sort can be expected to be faster because it executes the inner loop fewer times than selection sort.</p>
