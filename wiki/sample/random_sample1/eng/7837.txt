<p>In <a href="/wiki/Computing" title="Computing">computing</a>, <b>source code</b> is any collection of <a href="/wiki/Computer_program" title="Computer program">computer instructions</a>, possibly with <a href="/wiki/Comment_(computer_programming)" title="Comment (computer programming)">comments</a>, written using a <a class="mw-redirect" href="/wiki/Human-readable" title="Human-readable">human-readable</a> <a href="/wiki/Programming_language" title="Programming language">programming language</a>, usually as <a href="/wiki/Plain_text" title="Plain text">ordinary text</a>. The source code of a program is specially designed to facilitate the work of computer <a href="/wiki/Programmer" title="Programmer">programmers</a>, who specify the actions to be performed by a computer mostly by writing source code. The source code is often transformed by an <a class="mw-redirect" href="/wiki/Assembler_(computing)" title="Assembler (computing)">assembler</a> or <a href="/wiki/Compiler" title="Compiler">compiler</a> into <a href="/wiki/Binary_number" title="Binary number">binary</a> <a href="/wiki/Machine_code" title="Machine code">machine code</a> understood by the computer. The machine code might then be stored for <a href="/wiki/Execution_(computing)" title="Execution (computing)">execution</a> at a later time. Alternatively, source code may be <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreted</a> and thus immediately executed.</p>
<p>Most <a href="/wiki/Application_software" title="Application software">application software</a> is distributed in a form that includes only <a href="/wiki/Executable" title="Executable">executable</a> files. If the source code were included it would be useful to a <a href="/wiki/User_(computing)" title="User (computing)">user</a>, programmer or a <a href="/wiki/System_administrator" title="System administrator">system administrator</a>, any of whom might wish to study and/or modify the program.</p>

<h2><span class="mw-headline" id="Definitions">Definitions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Source_code&amp;action=edit&amp;section=1" title="Edit section: Definitions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The Linux Information Project defines source code as:<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<blockquote>
<p>Source code (also referred to as source or code) is the version of software as it is originally written (i.e., typed into a computer) by a human in <a href="/wiki/Plain_text" title="Plain text">plain text</a> (i.e., human readable alphanumeric characters).</p>
</blockquote>
<p>The notion of source code may also be taken more broadly, to include machine code and notations in graphical languages, neither of which are textual in nature. An example from an article presented on the annual IEEE conference and on Source Code Analysis and Manipulation:<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<blockquote>
<p>For the purpose of clarity ‘<b>source code</b>’ is taken to mean any fully executable description of a software system. It is therefore so construed as to include machine code, very high level languages and executable graphical representations of systems.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
</blockquote>
<p>Often there are several steps of <a class="mw-redirect" href="/wiki/Program_translation" title="Program translation">program translation</a> or <a href="/wiki/Minification_(programming)" title="Minification (programming)">minification</a> between the original source code typed by a human and an executable program. While some, like the <a class="mw-redirect" href="/wiki/Free_software_foundation" title="Free software foundation">FSF</a>, argue that an intermediate file "is not real source code and does not count as source code",<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> others find it convenient to refer to each intermediate file as the source code for the next steps.</p>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Source_code&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The earliest programs for <a href="/wiki/Stored-program_computer" title="Stored-program computer">stored-program computers</a> were entered in binary through the <a href="/wiki/Front_panel" title="Front panel">front panel</a> switches of the computer. This <a href="/wiki/First-generation_programming_language" title="First-generation programming language">first-generation programming language</a> had no distinction between source code and <a href="/wiki/Machine_code" title="Machine code">machine code</a>.</p>
<p>When IBM first offered software to work with its machine, the source code was provided at no additional charge. At that time, the cost of developing and supporting software was included in the price of the hardware. For decades, IBM distributed source code with its software product licenses, until 1983.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>Most early computer magazines published source code as <a href="/wiki/Type-in_program" title="Type-in program">type-in programs</a>.</p>
<p>Occasionally the entire source code to a large program is published as a hardback book, such as <i>"Computers &amp; Typesetting, Volume B: <a href="/wiki/TeX" title="TeX">TeX</a>: The Program"</i>, <i>"<a href="/wiki/Pretty_Good_Privacy" title="Pretty Good Privacy">PGP Source Code and Internals</a>"</i>, <i>"PC <a href="/wiki/SpeedScript" title="SpeedScript">SpeedScript</a>"</i>, and <i>"<a class="mw-redirect" href="/wiki/UC/OS" title="UC/OS">uC/OS</a>: The Real-Time Kernel"</i>.</p>
<h2><span class="mw-headline" id="Organization">Organization</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Source_code&amp;action=edit&amp;section=3" title="Edit section: Organization">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The source code which constitutes a <a href="/wiki/Computer_program" title="Computer program">program</a> is usually held in one or more <a href="/wiki/Text_file" title="Text file">text files</a> stored on a computer's <a class="mw-redirect" href="/wiki/Hard_disk" title="Hard disk">hard disk</a>; usually these files are carefully arranged into a <a class="mw-redirect" href="/wiki/Directory_(file_systems)" title="Directory (file systems)">directory tree</a>, known as a <b>source tree</b>. Source code can also be stored in a database (as is common for <a href="/wiki/Stored_procedure" title="Stored procedure">stored procedures</a>) or elsewhere.</p>
<p>The source code for a particular piece of software may be contained in a single file or many files. Though the practice is uncommon, a program's source code can be written in different programming languages.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> For example, a program written primarily in the <a href="/wiki/C_(programming_language)" title="C (programming language)">C programming language</a>, might have portions written in <a href="/wiki/Assembly_language" title="Assembly language">assembly language</a> for optimization purposes. It is also possible for some components of a piece of software to be written and compiled separately, in an arbitrary programming language, and later integrated into the software using a technique called <a class="mw-redirect" href="/wiki/Library_linking" title="Library linking">library linking</a>. In some languages, such as <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, this can be done at <a href="/wiki/Run_time_(program_lifecycle_phase)" title="Run time (program lifecycle phase)">run time</a> (each class is compiled into a separate file that is linked by the interpreter at runtime).</p>
<p>Yet another method is to make the main program an interpreter for a programming language,<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2008)">citation needed</span></a></i>]</sup> either designed specifically for the application in question or general-purpose, and then write the bulk of the actual user functionality as <a href="/wiki/Macro_(computer_science)" title="Macro (computer science)">macros</a> or other forms of add-ins in this language, an approach taken for example by the <a href="/wiki/GNU_Emacs" title="GNU Emacs">GNU Emacs</a> text editor.</p>
<p>The <b>code base</b> of a <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a> project is the larger collection of all the source code of all the <a href="/wiki/Computer_program" title="Computer program">computer programs</a> which make up the project. It has become common practice to maintain code bases in <a class="mw-redirect" href="/wiki/Version_control_system" title="Version control system">version control systems</a>. Moderately complex software customarily requires the compilation or assembly of several, sometimes dozens or even hundreds, of different source code files. In these cases, instructions for compilations, such as a <a href="/wiki/Makefile" title="Makefile">Makefile</a>, are included with the source code. These describe the relationships among the source code files, and contain information about how they are to be compiled.</p>
<p>The <a class="mw-redirect" href="/wiki/Revision_control" title="Revision control">revision control</a> system is another tool frequently used by developers for source code maintenance.</p>
<h2><span class="mw-headline" id="Purposes">Purposes</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Source_code&amp;action=edit&amp;section=4" title="Edit section: Purposes">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Source code is primarily used as input to the process that produces an executable program (i.e., it is <a href="/wiki/Compiler" title="Compiler">compiled</a> or <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreted</a>). It is also used as a method of communicating <a href="/wiki/Algorithm" title="Algorithm">algorithms</a> between people (e.g., <a class="mw-redirect" href="/wiki/Code_snippets" title="Code snippets">code snippets</a> in books).<sup class="reference" id="cite_ref-Spinellis_8-0"><a href="#cite_note-Spinellis-8">[8]</a></sup></p>
<p><a href="/wiki/Programmer" title="Programmer">Computer programmers</a> often find it helpful to review existing source code to learn about programming techniques.<sup class="reference" id="cite_ref-Spinellis_8-1"><a href="#cite_note-Spinellis-8">[8]</a></sup> The sharing of source code between developers is frequently cited as a contributing factor to the maturation of their programming skills.<sup class="reference" id="cite_ref-Spinellis_8-2"><a href="#cite_note-Spinellis-8">[8]</a></sup> Some people consider source code an expressive <a class="mw-redirect" href="/wiki/Media_(arts)" title="Media (arts)">artistic medium</a>.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
<p><a href="/wiki/Porting" title="Porting">Porting</a> software to other <a class="mw-redirect" href="/wiki/Computer_platform" title="Computer platform">computer platforms</a> is usually prohibitively difficult without source code. Without the source code for a particular piece of software, portability is generally computationally expensive.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (October 2008)">citation needed</span></a></i>]</sup> Possible porting options include <a href="/wiki/Binary_translation" title="Binary translation">binary translation</a> and emulation of the original platform.</p>
<p><a class="mw-redirect" href="/wiki/Decompilation" title="Decompilation">Decompilation</a> of an executable program can be used to generate source code, either in <a class="mw-redirect" href="/wiki/Assembly_code" title="Assembly code">assembly code</a> or in a <a href="/wiki/High-level_programming_language" title="High-level programming language">high-level language</a>.</p>
<p>Programmers frequently adapt source code from one piece of software to use in other projects, a concept known as <a class="mw-redirect" href="/wiki/Software_reusability" title="Software reusability">software reusability</a>.</p>
<h2><span class="mw-headline" id="Legal_aspects">Legal aspects</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Source_code&amp;action=edit&amp;section=5" title="Edit section: Legal aspects">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">See also: <a href="/wiki/History_of_free_and_open-source_software" title="History of free and open-source software">History of free and open-source software</a></div>
<p>Before 1974, when the US Commission on New Technological Uses of Copyrighted Works (CONTU) decided that <i>"computer programs, to the extent that they embody an author's original creation, are proper subject matter of copyright"</i>,<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup><sup class="reference" id="cite_ref-sail_book_11-0"><a href="#cite_note-sail_book-11">[11]</a></sup> software and its source code was not <a href="/wiki/Copyright" title="Copyright">copyrightable</a> and therefore always <a class="mw-redirect" href="/wiki/Public_domain_software" title="Public domain software">public domain software</a>. The 1983 <i><a class="mw-redirect" href="/wiki/Apple_v._Franklin" title="Apple v. Franklin">Apple v. Franklin</a></i> legislation clarified also for <a href="/wiki/Object_code" title="Object code">object code</a> that the Copyright Act gave computer programs the copyright status of literary works.</p>
<p>In 1999, in the United States court case <i><a href="/wiki/Bernstein_v._United_States" title="Bernstein v. United States">Bernstein v. United States</a></i> it was decided if source code should be considered a constitutionally protected form of <a class="mw-redirect" href="/wiki/Free_speech" title="Free speech">free speech</a>. Proponents of free speech argued that because source code conveys information to programmers, is written in a language, and can be used to share humor and other artistic pursuits, it is a protected form of communication. <a href="/wiki/University_of_California" title="University of California">University of California</a> <a href="/wiki/Mathematics" title="Mathematics">mathematics</a> professor and cryptology specialist <a class="mw-redirect" href="/wiki/Dan_Bernstein" title="Dan Bernstein">Dan Bernstein</a> brought the case to court as at the time, <a class="mw-redirect" href="/wiki/Encryption_algorithm" title="Encryption algorithm">encryption algorithms</a> were classified as <a class="mw-redirect" href="/wiki/Munition" title="Munition">munitions</a> by the United States government. Exporting encryption to other countries was considered an issue of <a href="/wiki/National_security" title="National security">national security</a>, and had to be approved by the <a href="/wiki/United_States_Department_of_State" title="United States Department of State">State Department</a>.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> On Bernstein's behalf the <a href="/wiki/Electronic_Frontier_Foundation" title="Electronic Frontier Foundation">Electronic Frontier Foundation</a> sued the <a href="/wiki/Federal_government_of_the_United_States" title="Federal government of the United States">U.S. government</a>.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup><sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> The court ruling in the case declared software as protected speech under the <a class="mw-redirect" href="/wiki/First_Amendment" title="First Amendment">First Amendment</a>, and national restrictions on encryption software were overturned.</p>
<h3><span class="mw-headline" id="Licensing">Licensing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Source_code&amp;action=edit&amp;section=6" title="Edit section: Licensing">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Software_license" title="Software license">Software license</a></div>
<p>An author of a <a href="/wiki/Threshold_of_originality" title="Threshold of originality">non-trivial work</a> like software,<sup class="reference" id="cite_ref-sail_book_11-1"><a href="#cite_note-sail_book-11">[11]</a></sup> has several <a href="/wiki/Exclusive_right" title="Exclusive right">exclusive rights</a>, among them the copyright for the source code and <a href="/wiki/Object_code" title="Object code">object code</a>.<sup class="reference" id="cite_ref-fsm_15-0"><a href="#cite_note-fsm-15">[15]</a></sup> The author has the right and possibility to grant customers and users of his software some of his exclusive rights in form of <a class="mw-redirect" href="/wiki/Software_licensing" title="Software licensing">software licensing</a>. Software, and its accompanying source code, can be associated with several licensing paradigms; the most important distinction is <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> vs <a href="/wiki/Proprietary_software" title="Proprietary software">proprietary software</a>.</p>
<p>Generally speaking, software is <i>open source</i> if the source code is free to use, distribute, modify and study, and <i>proprietary</i> if the source code is kept secret, or is privately owned and restricted. One of the first software license to be published and to explicitly grant these freedoms was the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a> in 1989, the <a class="mw-redirect" href="/wiki/BSD_license" title="BSD license">BSD license</a> is another early example from 1990.</p>
<p>For proprietary software, the provisions of the various copyright laws, <a href="/wiki/Trade_secret" title="Trade secret">trade secrecy</a> and <a href="/wiki/Patent" title="Patent">patents</a> are used to keep the source code closed. Additionally, many pieces of <a href="/wiki/Retail_software" title="Retail software">retail software</a> come with an <a href="/wiki/End-user_license_agreement" title="End-user license agreement">end-user license agreement</a> (EULA) which typically prohibits <a class="mw-redirect" href="/wiki/Decompilation" title="Decompilation">decompilation</a>, <a href="/wiki/Reverse_engineering" title="Reverse engineering">reverse engineering</a>, analysis, modification, or circumventing of <a href="/wiki/Copy_protection" title="Copy protection">copy protection</a>. Types of source code protection –  beyond traditional <a href="/wiki/Compiler" title="Compiler">compilation</a> to <a href="/wiki/Object_code" title="Object code">object code</a> –  include code encryption, <a class="mw-redirect" href="/wiki/Code_obfuscation" title="Code obfuscation">code obfuscation</a> or <a href="/wiki/Code_morphing" title="Code morphing">code morphing</a>.</p>
<h2><span class="mw-headline" id="Quality">Quality</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Source_code&amp;action=edit&amp;section=7" title="Edit section: Quality">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/Software_quality" title="Software quality">Software quality</a></div>
<p>The way a program is written can have important consequences for its maintainers. <a href="/wiki/Coding_conventions" title="Coding conventions">Coding conventions</a>, which stress <a href="/wiki/Readability" title="Readability">readability</a> and some language-specific conventions, are aimed at the maintenance of the software source code, which involves debugging and updating. Other priorities, such as the speed of the program's execution, or the ability to compile the program for multiple architectures, often make code readability a less important consideration, since code <i>quality</i> generally depends on its <i>purpose</i>.</p>
