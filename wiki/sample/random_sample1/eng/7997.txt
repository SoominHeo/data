<h1 class="firstHeading" id="firstHeading" lang="en">Ordered field</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p>In <a href="/wiki/Mathematics" title="Mathematics">mathematics</a>, an <b>ordered field</b> is a <a href="/wiki/Field_(mathematics)" title="Field (mathematics)">field</a> together with a <a href="/wiki/Total_order" title="Total order">total ordering</a> of its elements that is compatible with the field operations. Historically, the <a class="mw-redirect" href="/wiki/Axiomatization" title="Axiomatization">axiomatization</a> of an ordered field was abstracted gradually from the <a href="/wiki/Real_number" title="Real number">real numbers</a>, by mathematicians including <a href="/wiki/David_Hilbert" title="David Hilbert">David Hilbert</a>, <a href="/wiki/Otto_H%C3%B6lder" title="Otto Hölder">Otto Hölder</a> and <a href="/wiki/Hans_Hahn_(mathematician)" title="Hans Hahn (mathematician)">Hans Hahn</a>. This grew eventually into the <a href="/wiki/Artin%E2%80%93Schreier_theory" title="Artin–Schreier theory">Artin–Schreier theory</a> of ordered fields and <a href="/wiki/Formally_real_field" title="Formally real field">formally real fields</a>.</p>
<p>An ordered field necessarily has <a href="/wiki/Characteristic_(algebra)" title="Characteristic (algebra)">characteristic</a> 0 since the elements <span class="nowrap">0 &lt; 1 &lt; 1 + 1 &lt; 1 + 1 + 1 &lt; ...</span> necessarily are all distinct. Thus, an ordered field necessarily contains an infinite number of elements: a <a href="/wiki/Finite_field" title="Finite field">finite field</a> cannot be ordered.</p>
<p>Every <a href="/wiki/Field_extension" title="Field extension">subfield</a> of an ordered field is also an ordered field in the inherited order. Every ordered field contains an ordered subfield that is <a href="/wiki/Isomorphism" title="Isomorphism">isomorphic</a> to the <a href="/wiki/Rational_number" title="Rational number">rational numbers</a>. Any <a class="mw-redirect" href="/wiki/Dedekind-complete" title="Dedekind-complete">Dedekind-complete</a> ordered field is isomorphic to the real numbers. <a href="/wiki/Square_(algebra)" title="Square (algebra)">Squares</a> are necessarily non-negative in an ordered field. This implies that the <a href="/wiki/Complex_number" title="Complex number">complex numbers</a> cannot be ordered since the square of the <a href="/wiki/Imaginary_unit" title="Imaginary unit">imaginary unit</a> <i>i</i> is <a class="mw-redirect" href="/wiki/%E2%88%921_(number)" title="−1 (number)">−1</a>. Every ordered field is a <a href="/wiki/Formally_real_field" title="Formally real field">formally real field</a>.</p>

<h2><span class="mw-headline" id="Definitions">Definitions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordered_field&amp;action=edit&amp;section=1" title="Edit section: Definitions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>There are two equivalent common definitions of an ordered field. The definition of <b>total order</b> appeared first historically and is a first-order axiomatization of the ordering ≤ as a <a class="mw-redirect" href="/wiki/Binary_predicate" title="Binary predicate">binary predicate</a>. Artin and Schreier gave the definition in terms of <b>positive cone</b> in 1926, which axiomatizes the subcollection of nonnegative elements. Although the latter is higher-order, viewing positive cones as <i>maximal</i> prepositive cones provides a larger context in which field orderings are <i>extremal</i> partial orderings.</p>
<h3><span class="mw-headline" id="Total_order">Total order</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordered_field&amp;action=edit&amp;section=2" title="Edit section: Total order">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A <a href="/wiki/Field_(mathematics)" title="Field (mathematics)">field</a> (<i>F</i>, +, ×) together with a <a href="/wiki/Total_order" title="Total order">total order</a> ≤ on <i>F</i> is an <b>ordered field</b> if the order satisfies the following properties for all <i>a</i>, <i>b</i> and <i>c</i> in <i>F</i>:</p>
<ul>
<li>if <i>a</i> ≤ <i>b</i> then <i>a</i> + <i>c</i> ≤ <i>b</i> + <i>c</i>, and</li>
<li>if 0 ≤ <i>a</i> and 0 ≤ <i>b</i> then 0 ≤ <i>a</i> × <i>b</i>.</li>
</ul>
<p>The symbol for <a href="/wiki/Multiplication" title="Multiplication">multiplication</a> will be henceforth omitted.</p>
<h3><span class="mw-headline" id="Positive_cone">Positive cone</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordered_field&amp;action=edit&amp;section=3" title="Edit section: Positive cone">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A <b>prepositive cone</b> or <b>preordering</b> of a field <i>F</i> is a <a href="/wiki/Subset" title="Subset">subset</a> <i>P</i> ⊂ <i>F</i> that has the following properties:<sup class="reference" id="cite_ref-Lam289_1-0"><a href="#cite_note-Lam289-1">[1]</a></sup></p>
<ul>
<li>For <i>x</i> and <i>y</i> in <i>P</i>, both <i>x</i> + <i>y</i> and <i>xy</i> are in <i>P</i>.</li>
<li>If <i>x</i> is in <i>F</i>, then <i>x</i><sup>2</sup> is in <i>P</i>.</li>
<li>The element −1 is not in <i>P</i>.</li>
</ul>
<p>A <b>preordered field</b> is a field equipped with a preordering <i>P</i>. Its non-zero elements <i>P</i><sup>∗</sup> form a <a href="/wiki/Subgroup" title="Subgroup">subgroup</a> of the multiplicative group of <i>F</i>.</p>
<p>If in addition, the set <i>F</i> is the union of <i>P</i> and −<i>P</i>, we call <i>P</i> a <b>positive cone</b> of <i>F</i>. The non-zero elements of <i>P</i> are called the <b>positive</b> elements of <i>F</i>.</p>
<p>An <b>ordered field</b> is a field <i>F</i> together with a positive cone <i>P</i>.</p>
<p>The preorderings on <i>F</i> are precisely the intersections of families of positive cones on <i>F</i>. The positive cones are the maximal preorderings.<sup class="reference" id="cite_ref-Lam289_1-1"><a href="#cite_note-Lam289-1">[1]</a></sup></p>
<h3><span class="mw-headline" id="Equivalence_of_the_two_definitions">Equivalence of the two definitions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordered_field&amp;action=edit&amp;section=4" title="Edit section: Equivalence of the two definitions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Let <i>F</i> be a field. There is a bijection between the field orderings of <i>F</i> and the positive cones of <i>F</i>.</p>
<p>Given a field ordering ≤ as in the first definition, the set of elements such that <i>x</i> ≥ 0 forms a positive cone of <i>F</i>. Conversely, given a positive cone <i>P</i> of <i>F</i> as in the second definition, one can associate a total ordering ≤<sub><i>P</i></sub> on <i>F</i> by setting <i>x</i> ≤<sub><i>P</i></sub> <i>y</i> to mean <i>y</i> − <i>x</i> ∈ <i>P</i>. This total ordering ≤<sub><i>P</i></sub> satisfies the properties of the first definition.</p>

<p>For every <i>a</i>, <i>b</i>, <i>c</i>, <i>d</i> in <i>F</i>:</p>
<ul>
<li>Either −<i>a</i> ≤ 0 ≤ <i>a</i> or <i>a</i> ≤ 0 ≤ −<i>a</i></li>
<li>One can "add inequalities": if <i>a</i> ≤ <i>b</i> and <i>c</i> ≤ <i>d</i>, then <i>a</i> + <i>c</i> ≤ <i>b</i> + <i>d</i></li>
<li>One can "multiply inequalities with positive elements": if <i>a</i> ≤ <i>b</i> and 0 ≤ <i>c</i>, then <i>ac</i> ≤ <i>bc</i></li>
<li><a class="mw-redirect" href="/wiki/Transitive_property" title="Transitive property">Transitivity</a> of inequality: if <i>a</i> &lt; <i>b</i> and <i>b</i> &lt; <i>c</i>, then <i>a</i> &lt; <i>c</i></li>
<li>If <i>x</i> &lt; <i>y</i> and <i>x</i>, <i>y</i> &gt; 0, then 1/<i>y</i> &lt; 1/<i>x</i></li>
<li>1 is positive</li>
<li>An ordered field has <a href="/wiki/Characteristic_(algebra)" title="Characteristic (algebra)">characteristic</a> 0. (Since 1 &gt; 0, then 1 + 1 &gt; 0, and 1 + 1 + 1 &gt; 0, etc. If the field had characteristic <i>p</i> &gt; 0, then −1 would be the sum of <i>p</i> − 1 ones, but −1 is not positive.) In particular, finite fields cannot be ordered.</li>
<li>Squares are non-negative: 0 ≤ <i>a</i><sup>2</sup> for all <i>a</i> in <i>F</i></li>
</ul>
<p>Every subfield of an ordered field is also an ordered field (inheriting the induced ordering). The smallest subfield is <a href="/wiki/Isomorphism" title="Isomorphism">isomorphic</a> to the <a href="/wiki/Rational_number" title="Rational number">rationals</a> (as for any other field of characteristic 0), and the order on this rational subfield is the same as the order of the rationals themselves. If every element of an ordered field lies between two elements of its rational subfield, then the field is said to be <i><a href="/wiki/Archimedean_property" title="Archimedean property">Archimedean</a></i>. Otherwise, such field is a <a href="/wiki/Non-Archimedean_ordered_field" title="Non-Archimedean ordered field">non-Archimedean ordered field</a> and contains <a href="/wiki/Infinitesimal" title="Infinitesimal">infinitesimals</a>. For example, the <a href="/wiki/Real_number" title="Real number">real numbers</a> form an Archimedean field, but <a class="mw-redirect" href="/wiki/Hyperreal_numbers" title="Hyperreal numbers">hyperreal numbers</a> form a non-Archimedean field, because it <a href="/wiki/Field_extension" title="Field extension">extends</a> real numbers with elements greater than any standard <a href="/wiki/Natural_number" title="Natural number">natural number</a>.<sup class="reference" id="cite_ref-BairHenry_2-0"><a href="#cite_note-BairHenry-2">[2]</a></sup></p>
<p>An ordered field <i>K</i> is isomorphic to the real number field if every non-empty subset of <i>K</i> with an upper bound in <i>K</i> has a <a class="mw-redirect" href="/wiki/Least_upper_bound" title="Least upper bound">least upper bound</a> in <i>K</i>. This property implies that the field is Archimedean.</p>
<h3><span class="mw-headline" id="Vector_spaces_over_an_ordered_field">Vector spaces over an ordered field</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordered_field&amp;action=edit&amp;section=7" title="Edit section: Vector spaces over an ordered field">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Vector_space" title="Vector space">Vector spaces</a> (particularly, <a href="/wiki/Examples_of_vector_spaces#Coordinate_space" title="Examples of vector spaces"><i>n</i>-spaces</a>) over an ordered field exhibit some special properties and have some specific structures, namely: <a href="/wiki/Orientation_(vector_space)" title="Orientation (vector space)">orientation</a>, <a href="/wiki/Convex_analysis" title="Convex analysis">convexity</a>, and <a href="/wiki/Inner_product_space" title="Inner product space">positively-definite inner product</a>. See <a href="/wiki/Real_coordinate_space#Geometric_properties_and_uses" title="Real coordinate space">Real coordinate space#Geometric properties and uses</a> for discussion of those properties of <b>R</b><sup><i>n</i></sup>, which can be generalized to vector spaces over other ordered fields.</p>
<h2><span class="mw-headline" id="Which_fields_can_be_ordered.3F">Which fields can be ordered?</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordered_field&amp;action=edit&amp;section=8" title="Edit section: Which fields can be ordered?">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Every ordered field is a <a href="/wiki/Formally_real_field" title="Formally real field">formally real field</a>, i.e., 0 cannot be written as a sum of nonzero squares.<sup class="reference" id="cite_ref-Lam41_3-0"><a href="#cite_note-Lam41-3">[3]</a></sup><sup class="reference" id="cite_ref-Lam232_4-0"><a href="#cite_note-Lam232-4">[4]</a></sup></p>
<p>Conversely, every formally real field can be equipped with a compatible total order, that will turn it into an ordered field. (This order need not be uniquely determined.) The proof uses <a href="/wiki/Zorn%27s_lemma" title="Zorn's lemma">Zorn's lemma</a>.<sup class="reference" id="cite_ref-Lam236_5-0"><a href="#cite_note-Lam236-5">[5]</a></sup></p>
<p><a href="/wiki/Finite_field" title="Finite field">Finite fields</a> and more generally fields of finite <a href="/wiki/Characteristic_(algebra)" title="Characteristic (algebra)">characteristic</a> cannot be turned into ordered fields, because in characteristic <i>p</i>, the element −1 can be written as a sum of (<i>p</i> − 1) squares 1<sup>2</sup>. The <a href="/wiki/Complex_number" title="Complex number">complex numbers</a> also cannot be turned into an ordered field, as −1 is a square (of the imaginary number <i>i</i>) and would thus be positive. Also, the <a class="mw-redirect" href="/wiki/P-adic_numbers" title="P-adic numbers">p-adic numbers</a> cannot be ordered, since <b>Q</b><sub>2</sub> contains a square root of −7 and <b>Q</b><sub><i>p</i></sub> (<i>p</i> &gt; 2) contains a square root of 1 − <i>p</i>.</p>
<h2><span class="mw-headline" id="Topology_induced_by_the_order">Topology induced by the order</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordered_field&amp;action=edit&amp;section=9" title="Edit section: Topology induced by the order">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>If <i>F</i> is equipped with the <a href="/wiki/Order_topology" title="Order topology">order topology</a> arising from the total order ≤, then the axioms guarantee that the operations + and × are <a class="mw-redirect" href="/wiki/Continuous_function_(topology)" title="Continuous function (topology)">continuous</a>, so that <i>F</i> is a <a class="mw-redirect" href="/wiki/Topological_field" title="Topological field">topological field</a>.</p>
<h2><span class="mw-headline" id="Harrison_topology">Harrison topology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordered_field&amp;action=edit&amp;section=10" title="Edit section: Harrison topology">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

<h2><span class="mw-headline" id="Fans_and_superordered_fields">Fans and superordered fields</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordered_field&amp;action=edit&amp;section=11" title="Edit section: Fans and superordered fields">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A <b>fan</b> on <i>F</i> is a preordering <i>T</i> with the property that if <i>S</i> is a subgroup of index 2 in <i>F</i><sup>∗</sup> containing <i>T</i> - {0} and not containing −1 then <i>S</i> is an ordering (that is, <i>S</i> is closed under addition).<sup class="reference" id="cite_ref-L8339_8-0"><a href="#cite_note-L8339-8">[8]</a></sup> A <b>superordered field</b> is a totally real field in which the set of sums of squares forms a fan.<sup class="reference" id="cite_ref-L8345_9-0"><a href="#cite_note-L8345-9">[9]</a></sup></p>
