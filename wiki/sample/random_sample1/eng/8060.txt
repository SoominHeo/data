<h1 class="firstHeading" id="firstHeading" lang="en">Superscalar processor</h1>
<div class="mw-body-content" id="bodyContent">

<p>A <b>superscalar processor</b> is a <a href="/wiki/Central_processing_unit" title="Central processing unit">CPU</a> that implements a form of <a class="mw-redirect" href="/wiki/Parallel_computer" title="Parallel computer">parallelism</a> called <a href="/wiki/Instruction-level_parallelism" title="Instruction-level parallelism">instruction-level parallelism</a> within a single processor. It therefore allows for more <a href="/wiki/Throughput" title="Throughput">throughput</a> (the number of instructions that can be executed in a unit of time) than would otherwise be possible at a given <a href="/wiki/Clock_rate" title="Clock rate">clock rate</a>. A superscalar processor can execute more than one instruction during a clock cycle by simultaneously dispatching multiple instructions to different <a href="/wiki/Execution_unit" title="Execution unit">execution units</a> on the processor. Each execution unit is not a separate processor (or a core if the processor is a <a href="/wiki/Multi-core_processor" title="Multi-core processor">multi-core processor</a>), but an execution resource within a single CPU such as an <a href="/wiki/Arithmetic_logic_unit" title="Arithmetic logic unit">arithmetic logic unit</a>.</p>
<p>In <a href="/wiki/Flynn%27s_taxonomy" title="Flynn's taxonomy">Flynn's taxonomy</a>, a single-core superscalar processor is classified as an <a href="/wiki/SISD" title="SISD">SISD</a> processor (Single Instruction stream, Single Data stream), though many superscalar processors support short vector operations and so could be classified as <a href="/wiki/SIMD" title="SIMD">SIMD</a> (Single Instruction stream, Multiple Data streams). A <a class="mw-redirect" href="/wiki/Multi-core" title="Multi-core">multi-core</a> superscalar processor is classified as an <a href="/wiki/MIMD" title="MIMD">MIMD</a> processor (Multiple Instruction streams, Multiple Data streams).</p>
<p>While a superscalar CPU is typically also <a class="mw-redirect" href="/wiki/Instruction_pipeline" title="Instruction pipeline">pipelined</a>, pipelining and superscalar execution are considered different performance enhancement techniques. The former executes multiple instructions in the same execution unit in parallel by dividing the execution unit into different phases, whereas the latter executes multiple instructions in parallel by using multiple execution units.</p>
<p>The superscalar technique is traditionally associated with several identifying characteristics (within a given CPU):</p>
<ul>
<li>Instructions are issued from a sequential instruction stream</li>
<li>The CPU dynamically checks for <a class="mw-redirect" href="/wiki/Data_dependencies" title="Data dependencies">data dependencies</a> between instructions at run time (versus software checking at <a href="/wiki/Compile_time" title="Compile time">compile time</a>)</li>
<li>The CPU can execute multiple instructions per clock cycle</li>
</ul>
<p></p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Superscalar_processor&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/Seymour_Cray" title="Seymour Cray">Seymour Cray</a>'s <a href="/wiki/CDC_6600" title="CDC 6600">CDC 6600</a> from 1966 is often mentioned as the first superscalar design. The Motorola <a href="/wiki/MC88100" title="MC88100">MC88100</a> (1988), the <a href="/wiki/Intel_i960" title="Intel i960">Intel i960</a>CA (1989) and the <a class="mw-redirect" href="/wiki/AMD_29000" title="AMD 29000">AMD 29000</a>-series 29050 (1990) microprocessors were the first commercial single-chip superscalar microprocessors. <a class="mw-redirect" href="/wiki/RISC" title="RISC">RISC</a> microprocessors like these were the first to have superscalar execution, because RISC architectures frees transistors and die area which could be used to include multiple execution units (this was why RISC designs were faster than <a class="mw-redirect" href="/wiki/Complex_instruction_set_computer" title="Complex instruction set computer">CISC</a> designs through the 1980s and into the 1990s).</p>
<p>Except for CPUs used in <a href="/wiki/Low-power_electronics" title="Low-power electronics">low-power</a> applications, <a href="/wiki/Embedded_system" title="Embedded system">embedded systems</a>, and <a href="/wiki/Battery_(electricity)" title="Battery (electricity)">battery</a>-powered devices, essentially all general-purpose CPUs developed since about 1998 are superscalar.</p>
<p>The <a href="/wiki/P5_(microarchitecture)" title="P5 (microarchitecture)">P5</a> <a class="mw-redirect" href="/wiki/Pentium_(brand)" title="Pentium (brand)">Pentium</a> was the first superscalar x86 processor; the <a class="mw-redirect" href="/wiki/Nx586" title="Nx586">Nx586</a>, <a href="/wiki/P6_(microarchitecture)" title="P6 (microarchitecture)">P6</a> <a href="/wiki/Pentium_Pro" title="Pentium Pro">Pentium Pro</a> and <a href="/wiki/AMD_K5" title="AMD K5">AMD K5</a> were among the first designs which decode <a href="/wiki/X86" title="X86">x86</a>-instructions asynchronously into dynamic <a href="/wiki/Microcode" title="Microcode">microcode</a>-like <i><a class="mw-redirect" href="/wiki/Micro-op" title="Micro-op">micro-op</a></i> sequences prior to actual execution on a superscalar <a href="/wiki/Microarchitecture" title="Microarchitecture">microarchitecture</a>; this opened up for dynamic scheduling of buffered <i>partial</i> instructions and enabled more parallelism to be extracted compared to the more rigid methods used in the simpler <a href="/wiki/P5_(microarchitecture)" title="P5 (microarchitecture)">P5</a> <a class="mw-redirect" href="/wiki/Pentium_(brand)" title="Pentium (brand)">Pentium</a>; it also simplified <a href="/wiki/Speculative_execution" title="Speculative execution">speculative execution</a> and allowed higher clock frequencies compared to designs such as the advanced <a href="/wiki/Cyrix_6x86" title="Cyrix 6x86">Cyrix 6x86</a>.</p>
<h2><span class="mw-headline" id="Scalar_to_superscalar">Scalar to superscalar</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Superscalar_processor&amp;action=edit&amp;section=2" title="Edit section: Scalar to superscalar">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The simplest processors are <a href="/wiki/Scalar_processor" title="Scalar processor">scalar processors</a>. Each instruction executed by a scalar processor typically manipulates one or two data items at a time. By contrast, each instruction executed by a <a href="/wiki/Vector_processor" title="Vector processor">vector processor</a> operates simultaneously on many data items. An analogy is the difference between <a href="/wiki/Scalar_(mathematics)" title="Scalar (mathematics)">scalar</a> and vector arithmetic. A superscalar processor is a mixture of the two. Each instruction processes one data item, but there are multiple execution units within each CPU thus multiple instructions can be processing separate data items concurrently.</p>
<p>Superscalar CPU design emphasizes improving the instruction dispatcher accuracy, and allowing it to keep the multiple execution units in use at all times. This has become increasingly important as the number of units has increased. While early superscalar CPUs would have two <a href="/wiki/Arithmetic_logic_unit" title="Arithmetic logic unit">ALUs</a> and a single <a class="mw-redirect" href="/wiki/Floating_point_unit" title="Floating point unit">FPU</a>, a modern design such as the <a href="/wiki/PowerPC_970" title="PowerPC 970">PowerPC 970</a> includes four ALUs, two FPUs, and two <a href="/wiki/SIMD" title="SIMD">SIMD</a> units. If the dispatcher is ineffective at keeping all of these units fed with instructions, the performance of the system will be no better than that of a simpler, cheaper design.</p>
<p>A superscalar processor usually sustains an execution rate in excess of one <a href="/wiki/Cycles_per_instruction" title="Cycles per instruction">instruction per machine cycle</a>. But merely processing multiple instructions concurrently does not make an architecture superscalar, since <a class="mw-redirect" href="/wiki/Instruction_pipeline" title="Instruction pipeline">pipelined</a>, <a class="mw-redirect" href="/wiki/Multiprocessor" title="Multiprocessor">multiprocessor</a> or <a class="mw-redirect" href="/wiki/Multi-core_(computing)" title="Multi-core (computing)">multi-core</a> architectures also achieve that, but with different methods.</p>
<p>In a superscalar CPU the dispatcher reads instructions from memory and decides which ones can be run in parallel, dispatching each to one of the several execution units contained inside a single CPU. Therefore, a superscalar processor can be envisioned having multiple parallel pipelines, each of which is processing instructions simultaneously from a single instruction thread.</p>
<h2><span class="mw-headline" id="Limitations">Limitations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Superscalar_processor&amp;action=edit&amp;section=3" title="Edit section: Limitations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Available performance improvement from superscalar techniques is limited by three key areas:</p>
<ol>
<li>The degree of intrinsic parallelism in the instruction stream (instructions requiring the same computational resources from the CPU).</li>
<li>The complexity and time cost of dependency checking logic and <a href="/wiki/Register_renaming" title="Register renaming">register renaming</a> circuitry</li>
<li>The branch instruction processing.</li>
</ol>
<p>Existing binary executable programs have varying degrees of intrinsic parallelism. In some cases instructions are not dependent on each other and can be executed simultaneously. In other cases they are inter-dependent: one instruction impacts either resources or results of the other. The instructions <code>a = b + c; d = e + f</code> can be run in parallel because none of the results depend on other calculations. However, the instructions <code>a = b + c; b = e + f</code> might not be runnable in parallel, depending on the order in which the instructions complete while they move through the units.</p>
<p>When the number of simultaneously issued instructions increases, the cost of dependency checking increases extremely rapidly. This is exacerbated by the need to check dependencies at run time and at the CPU's clock rate. This cost includes additional logic gates required to implement the checks, and time delays through those gates. <p>Even though the instruction stream may contain no inter-instruction dependencies, a superscalar CPU must nonetheless check for that possibility, since there is no assurance otherwise and failure to detect a dependency would produce incorrect results.</p>
<p>No matter how advanced the <a href="/wiki/Semiconductor_device_fabrication" title="Semiconductor device fabrication">semiconductor process</a> or how fast the switching speed, this places a practical limit on how many instructions can be simultaneously dispatched. While process advances will allow ever greater numbers of execution units (e.g., ALUs), the burden of checking instruction dependencies grows rapidly, as does the complexity of register renaming circuitry to mitigate some dependencies. Collectively the <a href="/wiki/CPU_power_dissipation" title="CPU power dissipation">power consumption</a>, complexity and gate delay costs limit the achievable superscalar speedup to roughly eight simultaneously dispatched instructions.</p>
<p>However even given infinitely fast dependency checking logic on an otherwise conventional superscalar CPU, if the instruction stream itself has many dependencies, this would also limit the possible speedup. Thus the degree of intrinsic parallelism in the code stream forms a second limitation.</p>
<h2><span class="mw-headline" id="Alternatives">Alternatives</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Superscalar_processor&amp;action=edit&amp;section=4" title="Edit section: Alternatives">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Collectively, these limits drive investigation into alternative architectural changes such as <a href="/wiki/Very_long_instruction_word" title="Very long instruction word">very long instruction word</a> (VLIW), <a href="/wiki/Explicitly_parallel_instruction_computing" title="Explicitly parallel instruction computing">explicitly parallel instruction computing</a> (EPIC), <a href="/wiki/Simultaneous_multithreading" title="Simultaneous multithreading">simultaneous multithreading</a> (SMT), and <a class="mw-redirect" href="/wiki/Multi-core_(computing)" title="Multi-core (computing)">multi-core computing</a>.</p>
<p>With VLIW, the burdensome task of dependency checking by <a class="mw-redirect" href="/wiki/Hardware_logic" title="Hardware logic">hardware logic</a> at run time is removed and delegated to the <a href="/wiki/Compiler" title="Compiler">compiler</a>. <a href="/wiki/Explicitly_parallel_instruction_computing" title="Explicitly parallel instruction computing">Explicitly parallel instruction computing</a> (EPIC) is like VLIW, with extra cache prefetching instructions.</p>
<p>Simultaneous multithreading, often abbreviated as SMT, is a technique for improving the overall efficiency of superscalar processors. SMT permits multiple independent threads of execution to better utilize the resources provided by modern processor architectures.</p>
<p>Superscalar processors differ from <a href="/wiki/Multi-core_processor" title="Multi-core processor">multi-core processors</a> in that the several execution units are not entire processors. A single processor is composed of finer-grained execution units such as the <a href="/wiki/Arithmetic_logic_unit" title="Arithmetic logic unit">ALU</a>, <a href="/wiki/Integer_(computer_science)" title="Integer (computer science)">integer</a> <a class="mw-redirect" href="/wiki/Multiplication_ALU" title="Multiplication ALU">multiplier</a>, integer shifter, <a href="/wiki/Floating-point_unit" title="Floating-point unit">FPU</a>, etc. There may be multiple versions of each execution unit to enable execution of many instructions in parallel. This differs from a multi-core processor that concurrently processes instructions from <i>multiple</i> threads, one thread per <a href="/wiki/Central_processing_unit" title="Central processing unit">processing unit</a> (called "core"). It also differs from a <a href="/wiki/Instruction_pipelining" title="Instruction pipelining">pipelined processor</a>, where the multiple instructions can concurrently be in various stages of execution, <a href="/wiki/Assembly_line" title="Assembly line">assembly-line</a> fashion.</p>
<p>The various alternative techniques are not mutually exclusive—they can be (and frequently are) combined in a single processor. Thus a multicore CPU is possible where each core is an independent processor containing multiple parallel pipelines, each pipeline being superscalar. Some processors also include <a href="/wiki/Vector_processor" title="Vector processor">vector</a> capability.</p>
