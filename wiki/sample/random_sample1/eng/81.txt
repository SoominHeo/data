<p><b>Embedded OpenType</b> (<b>EOT</b>) <a class="mw-redirect" href="/wiki/Fonts" title="Fonts">fonts</a> are a compact form of <a href="/wiki/OpenType" title="OpenType">OpenType</a> fonts designed by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> for use as embedded fonts on <a href="/wiki/Web_page" title="Web page">web pages</a>. These files use the <a class="mw-redirect" href="/wiki/File_extension" title="File extension">extension</a> <code>.eot</code>. They are supported only by <a href="/wiki/Internet_Explorer" title="Internet Explorer">Microsoft Internet Explorer</a>, as opposed to competing <a href="/wiki/Web_Open_Font_Format" title="Web Open Font Format">WOFF</a> files.</p>
<p>These font files can be created from existing <a href="/wiki/TrueType" title="TrueType">TrueType</a> font files using Microsoft's <a class="mw-redirect" href="/wiki/Web_Embedding_Fonts_Tool" title="Web Embedding Fonts Tool">Web Embedding Fonts Tool</a> (WEFT), and other proprietary and open source software (see ‚ÄúExternal links‚Äù below).</p>
<p>The font files are made small in size by use of <a href="/wiki/Subsetting" title="Subsetting">subsetting</a> (only including the needed characters), and by <a href="/wiki/Data_compression" title="Data compression">data compression</a> (LZ compression, part of <a class="mw-redirect" href="/wiki/Agfa" title="Agfa">Agfa</a>'s <a class="external text" href="http://www.w3.org/Printing/porell.html" rel="nofollow">MicroType Express</a>). Like OTF fonts, EOT supports both Postscript and TrueType outlines for the glyphs.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p>Simply including fonts in webpages might lead to unrestricted copying of copyrighted font files. Embedded OpenType includes features to discourage copying. Subsetting reduces the value of copying, as subsetted fonts will typically omit more than half of the characters. Other copy protection measures used are <a href="/wiki/Encryption" title="Encryption">encryption</a> and a list of "trusted roots" at the source end, and a proprietary decrypting library at the receiving end.</p>
<p>If the embedded font is not available to the web page for any reason (missing font file, wrong keys in the file, non-support by the web browser), then the second-choice font specification is used, ensuring that the page should be readable even without the intended font.</p>
<p>Embedded OpenType is a proprietary standard supported exclusively by <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> but was <a class="external text" href="http://lists.w3.org/Archives/Public/www-style/2008Apr/0227.html" rel="nofollow">submitted to the W3C in 2007 as part of CSS3</a>, which was rejected and resubmitted as a <a class="external text" href="http://www.w3.org/Submission/2008/01/" rel="nofollow">standalone submission March 18, 2008</a>. The <a class="external text" href="http://www.w3.org/Submission/2008/01/Comment" rel="nofollow">W3C team comment</a> on the submission states that the "W3C plans to submit a proposal to the W3C members for a working group whose goal is to try and develop EOT into a W3C Recommendation." However, the W3C ultimately chose a different web font format (<a class="mw-redirect" href="/wiki/WOFF" title="WOFF">WOFF</a>) as a W3C Recommendation.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>

<h2><span class="mw-headline" id="Web_Embedding_Fonts_Tool">Web Embedding Fonts Tool</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Embedded_OpenType&amp;action=edit&amp;section=1" title="Edit section: Web Embedding Fonts Tool">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The Web Embedding Fonts Tool, or WEFT, is <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>'s utility for generating embeddable <a class="mw-redirect" href="/wiki/Web_fonts" title="Web fonts">web fonts</a>.</p>
<p>WEFT is used by <a href="/wiki/Webmaster" title="Webmaster">webmasters</a> to create 'font objects' that are linked to their <a href="/wiki/Web_page" title="Web page">web pages</a> so that users using Microsoft's <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> <a href="/wiki/Web_browser" title="Web browser">web browser</a> will see the pages displayed in the font style contained within the font object.</p>
<p>WEFT scans the <a href="/wiki/HTML" title="HTML">HTML</a> document file(s), the TrueType font file(s), and some additional parameters. It adjusts the HTML files and creates Embedded OpenType files for inclusion on the web site. These files usually use the <a class="mw-redirect" href="/wiki/File_extension" title="File extension">extension</a> "<code>.eot</code>".</p>
<p>WEFT can embed most fonts, but it will not embed fonts that have been designated as 'no embedding' fonts by their designers. WEFT may reject other fonts because problems have been identified.</p>
<p>Embedded fonts are widely used to generate non-English-language websites.</p>
<p>As of January 2015, the most recent version of the tool (<b>WEFT 5.3.2</b>) was released on 25 February 2003.</p>
<p>An open source alternative is "ttf2eot" <a class="external autonumber" href="http://code.google.com/p/ttf2eot/" rel="nofollow">[1]</a>.</p>
<p>Microsoft PowerPoint 2007 and 2010 also generate .eot files with the ".fntdata" <a class="external autonumber" href="http://msdn.microsoft.com/en-us/library/ff529562(office.12).aspx" rel="nofollow">[2]</a> extension when fonts are selected to be embedded in a presentation by the PowerPoint client application. These .eot files can be extracted from the ".pptx" file and used directly on web pages.</p>
<h2><span class="mw-headline" id="Security_issues">Security issues</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Embedded_OpenType&amp;action=edit&amp;section=2" title="Edit section: Security issues">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Critical security update for <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a> KB969947 resolves several security issues that "could allow remote code execution if a user viewed content rendered in a specially crafted Embedded OpenType (EOT) font". Such fonts could be embedded in Web sites, including those that host user-provided content.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
