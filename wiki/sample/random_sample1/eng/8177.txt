<p><b>Squid</b> is a <a href="/wiki/Web_cache" title="Web cache">caching</a> and forwarding HTTP <a class="mw-redirect" href="/wiki/Web_proxy" title="Web proxy">web proxy</a>. It has a wide variety of uses, from speeding up a <a href="/wiki/Web_server" title="Web server">web server</a> by caching repeated requests; to caching <a href="/wiki/World_Wide_Web" title="World Wide Web">web</a>, <a href="/wiki/Domain_Name_System" title="Domain Name System">DNS</a> and other <a href="/wiki/Computer_network" title="Computer network">computer network</a> lookups for a group of people sharing network resources, to aiding security by filtering traffic. Although primarily used for <a class="mw-redirect" href="/wiki/HTTP" title="HTTP">HTTP</a> and <a href="/wiki/File_Transfer_Protocol" title="File Transfer Protocol">FTP</a>, Squid includes limited support for several other <a class="mw-redirect" href="/wiki/Protocol_(computing)" title="Protocol (computing)">protocols</a> including <a href="/wiki/Transport_Layer_Security" title="Transport Layer Security">TLS</a>, <a class="mw-redirect" href="/wiki/Secure_Sockets_Layer" title="Secure Sockets Layer">SSL</a>, <a class="mw-redirect" href="/wiki/Internet_Gopher" title="Internet Gopher">Internet Gopher</a> and <a href="/wiki/HTTPS" title="HTTPS">HTTPS</a>.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> Squid does not support the <a href="/wiki/SOCKS" title="SOCKS">SOCKS</a> protocol.</p>
<p>Squid was originally designed to run as a <a class="mw-redirect" href="/wiki/Daemon_(computer_software)" title="Daemon (computer software)">daemon</a> on <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> systems. A Windows port was maintained up to version 2.7. New versions available on Windows use the <a href="/wiki/Cygwin" title="Cygwin">Cygwin</a> environment.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> Squid is <a href="/wiki/Free_software" title="Free software">free software</a> released under the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a>.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Squid_(software)&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Squid was originally developed as the <i>Harvest object cache</i>,<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> part of the <a href="/wiki/Harvest_project" title="Harvest project">Harvest project</a> at the <a href="/wiki/University_of_Colorado_Boulder" title="University of Colorado Boulder">University of Colorado Boulder</a>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> Further work on the program was completed at the <a href="/wiki/University_of_California,_San_Diego" title="University of California, San Diego">University of California, San Diego</a> and funded via two grants from the <a href="/wiki/National_Science_Foundation" title="National Science Foundation">National Science Foundation</a>.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> Duane Wessels forked the "last pre-commercial version of Harvest" and renamed it to Squid to avoid confusion with the commercial fork called Cached 2.0, which became <a href="/wiki/NetCache" title="NetCache">NetCache</a>.<sup class="reference" id="cite_ref-auug-keynote_9-0"><a href="#cite_note-auug-keynote-9">[9]</a></sup><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> Squid version 1.0.0 was released in July 1996.<sup class="reference" id="cite_ref-auug-keynote_9-1"><a href="#cite_note-auug-keynote-9">[9]</a></sup></p>
<p>Squid is now developed almost exclusively through volunteer efforts.</p>
<h2><span class="mw-headline" id="Basic_functionality">Basic functionality</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Squid_(software)&amp;action=edit&amp;section=2" title="Edit section: Basic functionality">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>After a Squid proxy server is installed, <a href="/wiki/Web_browser" title="Web browser">web browsers</a> can be configured to use it as a <a href="/wiki/Proxy_server" title="Proxy server">proxy</a> HTTP server, allowing Squid to retain copies of the documents returned, which, on repeated requests for the same documents, can reduce access time as well as <a href="/wiki/Bandwidth_(computing)" title="Bandwidth (computing)">bandwidth</a> consumption. This is often useful for <a href="/wiki/Internet_service_provider" title="Internet service provider">Internet service providers</a> to increase speed to their customers, and <a href="/wiki/Local_area_network" title="Local area network">LANs</a> that share an <a href="/wiki/Internet" title="Internet">Internet</a> connection. Because the caching servers are controlled by the web service operator, caching proxies do not anonymize the user and should not be confused with anonymizing proxies.</p>
<p>A client program (e.g. browser) either has to specify explicitly the proxy server it wants to use (typical for ISP customers), or it could be using a proxy without any extra configuration: "transparent caching", in which case all outgoing HTTP requests are intercepted by Squid and all responses are cached. The latter is typically a corporate set-up (all clients are on the same LAN) and often introduces the privacy concerns mentioned above.</p>
<p>Squid has some features that can help <a href="/wiki/Anonymity" title="Anonymity">anonymize</a> connections, such as disabling or changing specific header fields in a <a href="/wiki/Client_(computing)" title="Client (computing)">client's</a> HTTP requests. Whether these are set, and what they are set to do, is up to the person who controls the computer running Squid. People requesting pages through a network which transparently uses Squid may not know whether this information is being logged.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> Within UK organisations at least, users should be informed if computers or internet connections are being monitored.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<h2><span class="mw-headline" id="Reverse_proxy">Reverse proxy</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Squid_(software)&amp;action=edit&amp;section=3" title="Edit section: Reverse proxy">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The above setup—caching the contents of an unlimited number of <a class="mw-redirect" href="/wiki/Webserver" title="Webserver">webservers</a> for a limited number of clients—is the classical one. Another setup is "<a href="/wiki/Reverse_proxy" title="Reverse proxy">reverse proxy</a>" or "webserver acceleration" (using <tt>http_port 80 accel vhost</tt>). In this mode, the cache serves an unlimited number of clients for a limited number of—or just one—web servers.</p>
<p>As an example, if <i>slow.example.com</i> is a "real" web server, and <i>www.example.com</i> is the Squid cache server that "accelerates" it, the first time any page is requested from <i>www.example.com</i>, the cache server would get the actual page from <i>slow.example.com</i>, but later requests would get the stored copy directly from the accelerator (for a configurable period, after which the stored copy would be discarded). The end result, without any action by the clients, is less traffic to the source server, meaning less CPU and memory usage, and less need for bandwidth. This does, however, mean that the source server cannot accurately report on its traffic numbers without additional configuration, as all requests would seem to have come from the reverse proxy. A way to adapt the reporting on the source server is to use the <a href="/wiki/X-Forwarded-For" title="X-Forwarded-For">X-Forwarded-For</a> HTTP header reported by the reverse proxy, to get the real client's IP address.</p>
<p>It is possible for a single Squid server to serve both as a normal and a reverse proxy simultaneously. For example, a business might host its own website on a web server, with a Squid server acting as a reverse proxy between clients (customers accessing the website from outside the business) and the web server. The same Squid server could act as a classical web cache, caching HTTP requests from clients within the business (i.e., employees accessing the internet from their workstations), so accelerating web access and reducing bandwidth demands.</p>
<h2><span class="mw-headline" id="Media-range_limitations">Media-range limitations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Squid_(software)&amp;action=edit&amp;section=4" title="Edit section: Media-range limitations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>This feature is used extensively by video streaming websites such as <a href="/wiki/YouTube" title="YouTube">YouTube</a>, so that if a user clicks to the middle of the video progress bar, the server can begin to send data from the middle of the file, rather than sending the entire file from the beginning and the user waiting for the preceding data to finish loading.</p>
<p>Partial downloads are also extensively used by Microsoft <a href="/wiki/Windows_Update" title="Windows Update">Windows Update</a> so that extremely large update packages can download in the background and pause halfway through the download, if the user turns off their computer or disconnects from the Internet.</p>
<p>The <a href="/wiki/Metalink" title="Metalink">Metalink</a> download format enables clients to do <a class="mw-redirect" href="/wiki/Segmented_downloading" title="Segmented downloading">segmented downloads</a> by issuing partial requests and spreading these over a number of mirrors.</p>
<p>Squid can relay partial requests to the origin web server. In order for a partial request to be satisfied at a fast speed from cache, Squid requires a full copy of the same object to already exist in its storage.</p>
<p>If a proxy video user is watching a video stream and browses to a different page before the video completely downloads, Squid cannot keep the partial download for reuse and simply discards the data. Special configuration is required to force such downloads to continue and be cached.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></p>
