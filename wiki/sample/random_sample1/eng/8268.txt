<p><b>SteamOS</b> is a <a href="/wiki/Debian" title="Debian">Debian</a>-based <a href="/wiki/Linux" title="Linux">Linux</a> <a href="/wiki/Operating_system" title="Operating system">operating system</a> by <a href="/wiki/Valve_Corporation" title="Valve Corporation">Valve Corporation</a> and is the primary operating system for Valve's <a href="/wiki/Steam_Machine_(hardware_platform)" title="Steam Machine (hardware platform)">Steam Machine</a> <a href="/wiki/Video_game_console" title="Video game console">video game console</a>.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> It was released alongside the start of end-user beta testing of Steam Machines in December 2013.</p>

<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=SteamOS&amp;action=edit&amp;section=1" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>SteamOS is designed primarily for playing video games away from a <a href="/wiki/Personal_computer" title="Personal computer">PC</a> (such as from the couch in one's living room) by providing a <a href="/wiki/Video_game_console" title="Video game console">console</a>-like experience using generic PC hardware that can connect directly to a television. It can run games natively that have been developed for Linux and purchased from the <a href="/wiki/Steam_(software)" title="Steam (software)">Steam</a> store. Users are also able to stream games from their Windows, Mac or Linux computers to one running SteamOS, and it incorporates the same family sharing and restrictions as Steam on the desktop.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> Valve claims that it has "achieved significant performance increases in graphics processing" through SteamOS.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> The operating system is <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a>, allowing users to build on or adapt the <a href="/wiki/Source_code" title="Source code">source code</a>, though the actual Steam client is closed.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup><sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>Since SteamOS is solely for playing games without use of mouse or keyboard, it does not have many built-in functions beyond web browsing and playing games; for example, there is no <a href="/wiki/File_manager" title="File manager">file manager</a> or <a href="/wiki/Image_viewer" title="Image viewer">image viewer</a> installed by default. Users can, however, access the available <a href="/wiki/GNOME" title="GNOME">GNOME</a> desktop environment and perform tasks like installing other software.<sup class="reference" id="cite_ref-ars_7-0"><a href="#cite_note-ars-7">[7]</a></sup> Though the OS does not, in its current form, support streaming services, Valve is in talks with streaming companies such as <a href="/wiki/Spotify" title="Spotify">Spotify</a> and <a href="/wiki/Netflix" title="Netflix">Netflix</a> to bring their features to SteamOS.,<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup><sup class="reference" id="cite_ref-TechRadar_9-0"><a href="#cite_note-TechRadar-9">[9]</a></sup> However Steam does have full-length films from indie movie makers available from their store. The OS natively supports <a href="/wiki/Nvidia" title="Nvidia">Nvidia</a>, <a href="/wiki/Intel" title="Intel">Intel</a>, and <a href="/wiki/Advanced_Micro_Devices" title="Advanced Micro Devices">AMD</a> graphics processors.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup><sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></p>
<p>Valve stated that it has added support for movies, television, and music functionality to SteamOS, however the video content is only from Steam's store which has a small amount of films, while music playback only supports local music collections. In October 2015, an update allowed Netflix and other DRM protected content to function in the native built-in browser.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<p>The current system hardware requirements for default SteamOS installations include:<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></p>
<ul>
<li><a href="/wiki/List_of_Intel_microprocessors" title="List of Intel microprocessors">Intel</a> or <a href="/wiki/List_of_AMD_microprocessors" title="List of AMD microprocessors">AMD</a> 64-bit capable processor</li>
<li>4 GB (or more) <a class="mw-redirect" href="/wiki/RAM" title="RAM">RAM</a></li>
<li>200 GB or larger <a class="mw-redirect" href="/wiki/Hard_disk" title="Hard disk">hard disk</a></li>
<li><a href="/wiki/List_of_Nvidia_graphics_processing_units" title="List of Nvidia graphics processing units">NVIDIA</a> (<a href="/wiki/Fermi_(microarchitecture)" title="Fermi (microarchitecture)">Fermi</a> graphics cards or newer),<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> <a href="/wiki/List_of_Intel_graphics_processing_units" title="List of Intel graphics processing units">Intel</a>, or <a href="/wiki/List_of_AMD_graphics_processing_units" title="List of AMD graphics processing units">AMD</a> graphics card (<a href="/wiki/Radeon_HD_5000_Series" title="Radeon HD 5000 Series">Radeon HD 5XXX</a> or newer)</li>
<li><a href="/wiki/USB" title="USB">USB</a> port for installation</li>
<li><a href="/wiki/Unified_Extensible_Firmware_Interface" title="Unified Extensible Firmware Interface">UEFI</a> boot support</li>
</ul>
<p>A custom installer method is also available, which can require additional configuration steps. This method allows for smaller hard disk sizes. There is also an ISO image installer that supports legacy BIOS motherboard.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> The installers can be sourced through Valve's repository.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></p>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=SteamOS&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>During a panel at <a href="/wiki/LinuxCon" title="LinuxCon">LinuxCon</a> in 2013, Valve co-founder and executive director <a href="/wiki/Gabe_Newell" title="Gabe Newell">Gabe Newell</a> stated that he believed "Linux and open source are the future of gaming" (though his beliefs about games themselves being open source is unclear), going on to say that the company is aiding game developers who want to make games compatible with Linux, and that they would be making an announcement the following week related to introducing Linux into the living room.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> On September 20, 2013, Valve posted a statement on its website titled <i>The Steam Universe is Expanding in 2014</i> which teased three new announcements from them related to "even more ways to connect the dots for customers who want Steam in the living-room."<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup><sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup> The first announcement was revealed on September 23 as SteamOS, with Valve saying they had "come to the conclusion that the environment best suited to delivering value to customers is an operating system built around Steam itself."<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> A large focus of the reveal was the openness of the operating system, with it being announced that users would be able to alter or replace any part of the software, and that it would be free.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup></p>
<p>In October 2013, Valve announced <i>Steam Dev Days</i>; a two-day developer conference where video game developers will be able to test and provide feedback on SteamOS and Steam Machines.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup> In October 2013, Nvidia also announced their collaboration with Valve to support SteamOS with the help of a development suite called <a href="/wiki/Nvidia_GameWorks" title="Nvidia GameWorks">Nvidia GameWorks</a> which incorporates <a href="/wiki/PhysX" title="PhysX">PhysX</a>, <a href="/wiki/OptiX" title="OptiX">OptiX</a>, <a class="new" href="/w/index.php?title=VisualFX&amp;action=edit&amp;redlink=1" title="VisualFX (page does not exist)">VisualFX</a> and other Nvidia-proprietary <a href="/wiki/Application_programming_interface" title="Application programming interface">APIs</a> and implementations thereof.<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup></p>
<p>In November 2013, Valve confirmed that they would not be making any exclusive games for SteamOS, and were also encouraging other developers not to as it goes against their philosophy of selling games wherever customers are.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup> In December, Valve announced that a beta version of SteamOS would be released for download on December 13, 2013.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> When this beta version released, Valve suggested waiting until 2014 to use it unless the user was confident using Linux operating systems.<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup></p>
<p>In mid-October 2015, preorders of the Steam Controller,<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup> Steam Link,<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup> and Alienware branded Steam Machines became available.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> The official release date for Steam Machines was on November 10, 2015.<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup></p>

<h2><span class="mw-headline" id="Performance">Performance</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=SteamOS&amp;action=edit&amp;section=4" title="Edit section: Performance">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In December 2013, <a href="/wiki/Phoronix" title="Phoronix">Phoronix</a> compared three Nvidia graphics cards on SteamOS and Windows 8.1.<sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup> Overall, the Nvidia's proprietary Linux graphics driver can deliver comparable performance to that of the Windows drivers due to the largely shared code-base between the platforms.</p>
<p>In January 2014, <a href="/wiki/GameSpot" title="GameSpot">GameSpot</a> compared the performance to games running on Windows using identical hardware and settings with an AMD graphics card and a Nvidia graphics card. On the AMD graphics card, they found that <i><a href="/wiki/Dota_2" title="Dota 2">Dota 2</a></i>, <i><a href="/wiki/Left_4_Dead_2" title="Left 4 Dead 2">Left 4 Dead 2</a></i> and <i><a href="/wiki/Metro:_Last_Light" title="Metro: Last Light">Metro: Last Light</a></i> all ran at considerably fewer frames-per-second under SteamOS. <i>Left 4 Dead 2</i> also suffered from stuttering, which they attributed to a <a href="/wiki/Device_driver" title="Device driver">device driver</a> problem. On the Nvidia graphics card <i>Metro: Last Light</i> ran at slightly higher frames per second, and <i>Dota 2</i> ran at the same rate. They state that for <i>Left 4 Dead 2</i> the Nvidia card actually performed better under SteamOS, but did not specify how as their chart indicated it performed at a lower frame rate. On both cards, <i>Left 4 Dead 2</i> and <i>Dota 2</i> both had longer load times compared to Windows.<sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup></p>
<p>On the official release of Steam Machines in November 2015, Ars Technica compared the rendering performance of cross-platform games on SteamOS and <a href="/wiki/Windows_10" title="Windows 10">Windows 10</a> running on the same machine, using average frame-per-second measurements, and found that games were rendered between 21% and 58% slower on SteamOS compared to Windows 10. Ars Technica considered this might be due to the inexperience of developers optimizing on <a href="/wiki/OpenGL" title="OpenGL">OpenGL</a> in contrast to <a href="/wiki/DirectX" title="DirectX">DirectX</a>, and believed that the performance might improve with future titles.<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup> Ars Technica noted that its benchmark, comprising only six games on a single computer, was far from comprehensive. Games designed with OpenGL and Linux in mind might run faster than their Windows counterparts. In fact, benchmarking by <a href="/wiki/Phoronix" title="Phoronix">Phoronix</a> showed Ubuntu 15.04 outclassing Windows 10 when running <a href="/wiki/Free_software" title="Free software">free (libre)</a> Quake clone <a href="/wiki/OpenArena" title="OpenArena">OpenArena</a>.</p>
<p>The primary tests of the Phoronix benchmarks were OpenArena, <a href="/wiki/Xonotic" title="Xonotic">Xonotic</a>, and GpuTest. In OpenArena and in the synthetic GpuTest, Ubuntu 15.04 was marginally faster than Windows 10 for each of the several graphics cards tested. However, Windows 10 did much better than Ubuntu 15.04 for Xonotic v0.8.<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup></p>
<h2><span class="mw-headline" id="Reception">Reception</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=SteamOS&amp;action=edit&amp;section=5" title="Edit section: Reception">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>On the gaming front, following the initial announcement many video game developers have shared their thoughts on SteamOS. <i><a href="/wiki/Minecraft" title="Minecraft">Minecraft</a></i> creator <a href="/wiki/Markus_Persson" title="Markus Persson">Markus Persson</a> described it as "amazing news", and <i><a href="/wiki/Thomas_Was_Alone" title="Thomas Was Alone">Thomas Was Alone</a></i> developer <a href="/wiki/Mike_Bithell" title="Mike Bithell">Mike Bithell</a> called it "encouraging" for <a href="/wiki/Indie_game" title="Indie game">indie games</a>.<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup><sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup> Other developers such as <a class="mw-redirect" href="/wiki/EA_Digital_Illusions_CE" title="EA Digital Illusions CE">DICE</a>, creators of the <i><a href="/wiki/Battlefield_(series)" title="Battlefield (series)">Battlefield</a></i> series, and <a class="mw-redirect" href="/wiki/The_Creative_Assembly" title="The Creative Assembly">The Creative Assembly</a>, developers of the <i><a href="/wiki/Total_War_(series)" title="Total War (series)">Total War</a></i> series, have stated that they may support their games on Linux and SteamOS.<sup class="reference" id="cite_ref-38"><a href="#cite_note-38">[38]</a></sup><sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[39]</a></sup></p>
<p>On the operating system front, <a href="/wiki/Gearbox_Software" title="Gearbox Software">Gearbox Software</a> head <a href="/wiki/Randy_Pitchford" title="Randy Pitchford">Randy Pitchford</a> commented that he believed the operating system needed a unique application to attract developers, saying "without that must-buy product driving us all towards this stuff, I expect that the industry at large will watch curiously, but remain largely unaffected."<sup class="reference" id="cite_ref-40"><a href="#cite_note-40">[40]</a></sup> <a href="/wiki/Richard_Stallman" title="Richard Stallman">Richard Stallman</a>, head of the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a>, is cautiously supportive, although he does not condone the use of non-free games or DRM.<sup class="reference" id="cite_ref-41"><a href="#cite_note-41">[41]</a></sup></p>
<p>The SteamOS beta release received mixed reviews. In <a href="/wiki/TechRadar" title="TechRadar">TechRadar</a>'s review Henry Winchester praised the easy to navigate interface and future potential, but criticized the hard installation and lack of extra features compared to the Steam software.<sup class="reference" id="cite_ref-TechRadar_9-1"><a href="#cite_note-TechRadar-9">[9]</a></sup> <a href="/wiki/Eurogamer" title="Eurogamer">Eurogamer</a>'s Thomas Morgan did not incur installation problems however commented negatively on the lack of options available for detecting monitor resolutions and audio output and the lack of games available natively on the operating system. He did, however, also respond positively to the user interface and called it "a positive start."<sup class="reference" id="cite_ref-42"><a href="#cite_note-42">[42]</a></sup></p>
<p>Since then, outlets such as <a href="/wiki/Ars_Technica" title="Ars Technica">Ars Technica</a> have revisited the SteamOS since its initial debut, offering observations on the platform's growth, pros, and cons.<sup class="reference" id="cite_ref-43"><a href="#cite_note-43">[43]</a></sup> Both <a href="/wiki/Falcon_Northwest" title="Falcon Northwest">Falcon Northwest</a> and <a href="/wiki/Origin_PC" title="Origin PC">Origin PC</a>, computer manufacturers that were planning on offering Steam Machine hardware, opted to not ship a SteamOS-enabled machine in 2015 due to limitations of SteamOS over Windows; Falcon Northwest have said they will still consider shipping machines with SteamOS in the future if performance improves.<sup class="reference" id="cite_ref-44"><a href="#cite_note-44">[44]</a></sup><sup class="reference" id="cite_ref-45"><a href="#cite_note-45">[45]</a></sup></p>
