<p><b>AMD Steamroller Family 15h</b> is a <a href="/wiki/Microarchitecture" title="Microarchitecture">microarchitecture</a> developed by <a href="/wiki/Advanced_Micro_Devices" title="Advanced Micro Devices">AMD</a> for <a class="mw-redirect" href="/wiki/AMD_APU" title="AMD APU">AMD APUs</a>, which succeeded <a href="/wiki/Piledriver_(microarchitecture)" title="Piledriver (microarchitecture)">Piledriver</a> in the beginning of 2014 as the third-generation <a href="/wiki/Bulldozer_(microarchitecture)" title="Bulldozer (microarchitecture)">Bulldozer</a>-based microarchitecture.<sup class="reference" id="cite_ref-anandtech-kaveri-review_3-0"><a href="#cite_note-anandtech-kaveri-review-3">[3]</a></sup> Steamroller APUs continue to use two-core modules as their predecessors, while aiming at achieving greater levels of parallelism.</p>

<h2><span class="mw-headline" id="Microarchitecture">Microarchitecture</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Steamroller_(microarchitecture)&amp;action=edit&amp;section=1" title="Edit section: Microarchitecture">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><i>Steamroller</i> still features two-core modules found in <i>Bulldozer</i> and <i>Piledriver</i> designs called <a class="mw-redirect" href="/wiki/Clustered_multi-thread" title="Clustered multi-thread">clustered multi-thread</a> (CMT), meaning that one module is equal to a dual-core processor.<sup class="reference" id="cite_ref-xbit-AMD-2013_4-0"><a href="#cite_note-xbit-AMD-2013-4">[4]</a></sup> The focus of <i>Steamroller</i> is for greater parallelism.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> Improvements center on independent instruction decoders for each core within a module, 25% more of the maximum width dispatches per thread, better instruction schedulers, improved perceptron branch predictor, larger and smarter caches, up to 30% fewer instruction cache misses, branch misprediction rate reduced by 20%, dynamically resizable L2 cache, micro-operations queue,<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> more internal register resources and improved memory controller. Another improvement over the <i>Piledriver</i> cores were the addition of new CPU instructions, such as <a href="/wiki/High_Efficiency_Video_Coding" title="High Efficiency Video Coding">HEVC</a>.</p>
<p>AMD estimated that these improvements will increase <a href="/wiki/Instructions_per_cycle" title="Instructions per cycle">instructions per cycle</a> (IPC) up to 30% compared to the first-generation Bulldozer core while maintaining Piledriver's high clock rates with decreased power consumption.<sup class="reference" id="cite_ref-xbit-AMD-2013_4-1"><a href="#cite_note-xbit-AMD-2013-4">[4]</a></sup> The final result was a 9% single-threaded IPC improvement, and 18% multi-threaded IPC improvement over Piledriver.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></p>
<p>Steamroller, the microarchitecture for CPUs, as well as <a href="/wiki/Graphics_Core_Next" title="Graphics Core Next">Graphics Core Next</a>, the microarchitecture for GPUs, are paired together in certain product lines, and both support certain features specified in <a href="/wiki/Heterogeneous_System_Architecture" title="Heterogeneous System Architecture">Heterogeneous System Architecture</a> in hardware.</p>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Steamroller_(microarchitecture)&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In 2011, AMD announced a third-generation Bulldozer-based line of processors for 2013,<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> with <i>Next Generation Bulldozer</i> as the working title, using the 28 nm manufacturing process.<sup class="reference" id="cite_ref-ir.amd.com_9-0"><a href="#cite_note-ir.amd.com-9">[9]</a></sup></p>
<p>On 21 September 2011, leaked AMD slides indicated that this third generation of Bulldozer core was codenamed <i>Steamroller</i>.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup><sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></p>
<p>In January 2014, <i>Kaveri</i> APUs became available.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<h2><span class="mw-headline" id="Processors">Processors</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Steamroller_(microarchitecture)&amp;action=edit&amp;section=3" title="Edit section: Processors">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="APU_lines"><span id="KAVERI"></span><span id="BERLIN"></span>APU lines</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Steamroller_(microarchitecture)&amp;action=edit&amp;section=4" title="Edit section: APU lines">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main articles: <a class="mw-redirect" href="/wiki/List_of_AMD_Accelerated_Processing_Unit_microprocessors#.22Kaveri.22_.282014.2C_28_nm.29" title="List of AMD Accelerated Processing Unit microprocessors">"Kaveri" desktop processors</a> and <a class="mw-redirect" href="/wiki/List_of_AMD_Accelerated_Processing_Unit_microprocessors#.22Kaveri.22_.282014.2C_28_nm.29_2" title="List of AMD Accelerated Processing Unit microprocessors">"Kaveri" mobile processors</a></div>
<p>There are two main <a href="/wiki/AMD_Accelerated_Processing_Unit" title="AMD Accelerated Processing Unit">APU</a> lines announced:</p>
<ol>
<li>Kaveri A-series APU
<ul>
<li>Desktop budget and mainstream markets (FM2+): The <i>Trinity</i> / <i>Richland</i> APU line was replaced in January 2014 by the <i>Kaveri</i> APU line, as the third generation of A10, A8, A6 and A4 series for the desktop market. Currently known new model is a quad-core A10-7850K APU, with a 3.7 GHz core frequency and 4 MB L2 cache, incorporating a 720 MHz GPU with 512 stream processors and over 856 <a class="mw-redirect" href="/wiki/GFLOPS" title="GFLOPS">GFLOPS</a> of total <a class="mw-redirect" href="/wiki/Processing_power" title="Processing power">processing power</a>.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup><br/>
An upcoming product is the A10-7890K. These APUs have two to four enhanced <i>Steamroller B</i> cores, a GCN 1.1 <a class="mw-redirect" href="/wiki/AMD_Radeon_Rx_200_Series" title="AMD Radeon Rx 200 Series">Volcanic Islands</a> (Sea Islands for Godavari / Kaveri-refresh A10-7870K integrated GPU,<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup><sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> and two integrated DDR3 memory controllers. <i>Kaveri</i> APUs utilize the new FM2+ socket, thus they are not backwards compatible with the previous generation of FM2 motherboards.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></li>
<li><a href="/wiki/Semiconductor_intellectual_property_core" title="Semiconductor intellectual property core">SIP blocks</a>: <a href="/wiki/Unified_Video_Decoder" title="Unified Video Decoder">Unified Video Decoder</a>, <a href="/wiki/Video_Coding_Engine" title="Video Coding Engine">Video Coding Engine</a>, <a href="/wiki/AMD_TrueAudio" title="AMD TrueAudio">TrueAudio</a><sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></li>
<li><a href="/wiki/AMD_Eyefinity" title="AMD Eyefinity">AMD Eyefinity</a> up to 4 monitors,<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> 4K Ultra HD support, DisplayPort 1.2 Support<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup></li>
<li><a href="/wiki/Socket_FM2%2B" title="Socket FM2+">Socket FM2+</a>-only, Socket FM2 is <i>not</i> supported,<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> support for <a href="/wiki/PCI_Express#3.0" title="PCI Express">PCIe 3.0</a></li>
<li>Two or four CPU cores based on the Steamroller microarchitecture</li>
<li>Three to eight Compute Units (CUs) based on the revised <a href="/wiki/Graphics_Core_Next" title="Graphics Core Next">Graphics Core Next</a> (GCN)<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> microarchitecture; 1 Compute Unit (CU) consists of 64 <a href="/wiki/Unified_shader_model" title="Unified shader model">Unified Shader Processors</a> : 4 <a href="/wiki/Texture_mapping_unit" title="Texture mapping unit">Texture Mapping Units</a> (TMUs) : 1 <a href="/wiki/Render_output_unit" title="Render output unit">Render Output Unit</a> (ROPs)</li>
<li><a href="/wiki/AMD_Accelerated_Processing_Unit#HSA" title="AMD Accelerated Processing Unit">AMD Heterogeneous System Architecture</a> (HSA) 2.0</li>
<li>Dual-channel (2x64-bit) <a href="/wiki/DDR3_SDRAM" title="DDR3 SDRAM">DDR3</a> memory controller</li>
<li>Integrated custom <a href="/wiki/ARM_Cortex-A5" title="ARM Cortex-A5">ARM Cortex-A5</a> co-processor<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup> with <a class="mw-redirect" href="/wiki/TrustZone" title="TrustZone">TrustZone</a> Security Extensions<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup></li>
<li>Select models support <a href="/wiki/AMD_Hybrid_Graphics" title="AMD Hybrid Graphics">AMD Hybrid Graphics</a> by using a Radeon R7 240 or R7 250 discrete graphics card.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup></li>
</ul>
</li>
<li>Berlin APU
<ul>
<li>Enterprise and server markets: The <i>Berlin</i> APU will be similar to <i>Kaveri</i>, featuring four <i>Steamroller</i> cores, up to 512 stream processors, and support for <a href="/wiki/ECC_memory" title="ECC memory">ECC memory</a>.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup></li>
</ul>
</li>
</ol>
<h3><span class="mw-headline" id="FX_lines">FX lines</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Steamroller_(microarchitecture)&amp;action=edit&amp;section=5" title="Edit section: FX lines">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/List_of_AMD_FX_microprocessors" title="List of AMD FX microprocessors">List of AMD FX microprocessors</a></div>
<p>In November 2013, AMD confirmed it will not update the <a href="/wiki/List_of_AMD_FX_microprocessors" title="List of AMD FX microprocessors">FX series</a> in 2014, neither its current <a href="/wiki/Socket_AM3%2B" title="Socket AM3+">Socket AM3+</a> version, nor will it receive a <i>Steamroller</i> version with a new socket.<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup><sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup> An unconfirmed article, citing anonymous sources with knowledge of AMD's internal roadmap extending into 2015, also did not mention plans for a new FX processor.<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup></p>
<h3><span class="mw-headline" id="Server_lines">Server lines</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Steamroller_(microarchitecture)&amp;action=edit&amp;section=6" title="Edit section: Server lines">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/List_of_AMD_Opteron_microprocessors" title="List of AMD Opteron microprocessors">List of AMD Opteron microprocessors</a></div>
<p>AMD's server roadmaps for 2014 show:<sup class="reference" id="cite_ref-tech_report_berlin_warsaw_29-0"><a href="#cite_note-tech_report_berlin_warsaw-29">[29]</a></sup><sup class="reference" id="cite_ref-WCCF_Opteron_Roadmap_30-0"><a href="#cite_note-WCCF_Opteron_Roadmap-30">[30]</a></sup></p>
<ul>
<li><i>Berlin</i> APU - quad-core x86 Steamroller architecture (as described above) for 1 Processor (1P) compute and media clusters
<ul>
<li>focused on cloud and media streaming</li>
</ul>
</li>
<li><i>Berlin</i> CPU - quad-core x86 Steamroller architecture for 1P web and enterprise services clusters
<ul>
<li>focused on big data processing and <a class="mw-redirect" href="/wiki/High-performance_computing" title="High-performance computing">HPC</a></li>
</ul>
</li>
<li><i>Seattle</i> CPU - 4/8 core <a href="/wiki/ARM_architecture#ARMv8-A" title="ARM architecture">AArch64</a> <a href="/wiki/ARM_Cortex-A57" title="ARM Cortex-A57">Cortex-A57</a> architecture (Opteron A1100) for 1P web and enterprise services clusters <sup class="reference" id="cite_ref-Tech_Times_Seattle_31-0"><a href="#cite_note-Tech_Times_Seattle-31">[31]</a></sup>
<ul>
<li>low-end applications with power-saving options</li>
</ul>
</li>
<li><i>Warsaw</i> CPU - up to 16 core <a href="/wiki/X86-64" title="X86-64">x86</a> <a href="/wiki/Piledriver_(microarchitecture)" title="Piledriver (microarchitecture)">Piledriver</a> (2nd gen Bulldozer) architecture (<a href="/wiki/List_of_AMD_Opteron_microprocessors#Opteron_6300P-series_.22Warsaw.22_.2832_nm.29" title="List of AMD Opteron microprocessors">Opteron 6338P and 6370P</a>) for 2P/4P servers <sup class="reference" id="cite_ref-tech_report_warsaw_32-0"><a href="#cite_note-tech_report_warsaw-32">[32]</a></sup>
<ul>
<li>no plans for Steamroller (3rd gen <a href="/wiki/Bulldozer_(microarchitecture)" title="Bulldozer (microarchitecture)">Bulldozer</a>) or <a href="/wiki/Excavator_(microarchitecture)" title="Excavator (microarchitecture)">Excavator</a> (4th gen Bulldozer) architectures on high-end multi-processor (2P/4P) platforms</li>
</ul>
</li>
</ul>
<p>However, plans for <i>Steamroller</i> Opteron products were cancelled, likely due to the poor energy efficiency achieved in this generation of the <i>Bulldozer</i> architecture. Energy efficiency was greatly increased in the following generation, (<i>Excavator</i>), which exceeded <i>Jaguar</i> in performance per watt, and approximately doubled performance/watt over <i>Steamroller</i> (for example 20.74 pt/W vs 10.85 pt/W when comparing similar mobile APUs using rough arbitrary metrics). <sup class="reference" id="cite_ref-jaguar-excavator_comparison_33-0"><a href="#cite_note-jaguar-excavator_comparison-33">[33]</a></sup> <sup class="reference" id="cite_ref-jaguar-steamroller_comparison_34-0"><a href="#cite_note-jaguar-steamroller_comparison-34">[34]</a></sup></p>
