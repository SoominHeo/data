<p><b>Office Open XML</b> (also informally known as <b>OOXML</b> or <b>OpenXML</b> or earlier as <b>MOX</b><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>) is a <a class="mw-redirect" href="/wiki/ZIP_(file_format)" title="ZIP (file format)">zipped</a>, <a href="/wiki/XML" title="XML">XML</a>-based <a href="/wiki/File_format" title="File format">file format</a> developed by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> for representing <a href="/wiki/Spreadsheet" title="Spreadsheet">spreadsheets</a>, <a href="/wiki/Chart" title="Chart">charts</a>, <a href="/wiki/Presentation" title="Presentation">presentations</a> and <a class="mw-redirect" href="/wiki/Word_processing" title="Word processing">word processing</a> documents. The format was initially standardised by <a href="/wiki/Ecma_International" title="Ecma International">Ecma</a> (as ECMA-376), and by the <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a> and <a href="/wiki/International_Electrotechnical_Commission" title="International Electrotechnical Commission">IEC</a> (as ISO/IEC 29500) in later versions.</p>
<p>Starting with <a href="/wiki/Microsoft_Office_2007" title="Microsoft Office 2007">Microsoft Office 2007</a>, the Office Open XML file formats have become the default<sup class="reference" id="cite_ref-microsoft.com_4-0"><a href="#cite_note-microsoft.com-4">[4]</a></sup> target file format of <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a>.<sup class="reference" id="cite_ref-courierpress.com_5-0"><a href="#cite_note-courierpress.com-5">[5]</a></sup><sup class="reference" id="cite_ref-knowledge.wharton.upenn.edu_6-0"><a href="#cite_note-knowledge.wharton.upenn.edu-6">[6]</a></sup> <a href="/wiki/Microsoft_Office_2010" title="Microsoft Office 2010">Microsoft Office 2010</a> provides read support for ECMA-376, read/write support for ISO/IEC 29500 Transitional, and read support for ISO/IEC 29500 Strict.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> <a href="/wiki/Microsoft_Office_2013" title="Microsoft Office 2013">Microsoft Office 2013</a> and <a href="/wiki/Microsoft_Office_2016" title="Microsoft Office 2016">Microsoft Office 2016</a> additionally support both reading and writing of ISO/IEC 29500 Strict.<sup class="reference" id="cite_ref-Office_2013_File_Formats_8-0"><a href="#cite_note-Office_2013_File_Formats-8">[8]</a></sup></p>

<h2><span class="mw-headline" id="Background">Background</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Office_Open_XML&amp;action=edit&amp;section=1" title="Edit section: Background">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In 2000, Microsoft released an initial version of an <a href="/wiki/XML" title="XML">XML</a>-based format for Microsoft Excel, which was incorporated in Office XP. In 2002, a new file format for Microsoft Word followed.<sup class="reference" id="cite_ref-ooxmlhistory_9-0"><a href="#cite_note-ooxmlhistory-9">[9]</a></sup> The Excel and Word formats—known as the <a href="/wiki/Microsoft_Office_XML_formats" title="Microsoft Office XML formats">Microsoft Office XML formats</a>—were later incorporated into the 2003 release of Microsoft Office.</p>
<p>Microsoft announced in November 2005 that it would co-sponsor standardization of the new version of their XML-based formats through <a href="/wiki/Ecma_International" title="Ecma International">Ecma International</a> as "Office Open XML".<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup><sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> The presentation was made to Ecma by Microsoft's <a href="/wiki/Jean_Paoli" title="Jean Paoli">Jean Paoli</a> and Isabelle Valet-Harper.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup><sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></p>
<h2><span class="mw-headline" id="Standardization_process">Standardization process</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Office_Open_XML&amp;action=edit&amp;section=2" title="Edit section: Standardization process">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/Standardization_of_Office_Open_XML" title="Standardization of Office Open XML">Standardization of Office Open XML</a></div>
<p>Microsoft submitted initial material to <a href="/wiki/Ecma_International" title="Ecma International">Ecma International</a> Technical Committee TC45, where it was standardized to become ECMA-376, approved in December 2006.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup></p>
<p>This standard was then fast-tracked in the <a class="mw-redirect" href="/wiki/JTC_1" title="JTC 1">Joint Technical Committee 1</a> of ISO and IEC. After initially <a class="mw-redirect" href="/wiki/Office_Open_XML_Intermediate_5_Month_Ballot_Results" title="Office Open XML Intermediate 5 Month Ballot Results">failing to pass</a>, an amended version of the format received the necessary votes for approval as an ISO/IEC Standard as the result of a <a class="mw-redirect" href="/wiki/ISO/IEC_JTC1" title="ISO/IEC JTC1">JTC 1</a> fast-tracking standardization process that concluded in April 2008.<sup class="reference" id="cite_ref-ISOIECapproval_15-0"><a href="#cite_note-ISOIECapproval-15">[15]</a></sup> The resulting four-part International Standard (designated ISO/IEC 29500:2008) was published in November 2008<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> and can be downloaded from the <a href="/wiki/Information_Technology_Task_Force" title="Information Technology Task Force">ITTF</a>.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> A technically equivalent set of texts is published by Ecma as <i>ECMA-376 Office Open XML File Formats — 2nd edition</i> (December 2008); they can be downloaded from their web site.<sup class="reference" id="cite_ref-StandardECMA376_18-0"><a href="#cite_note-StandardECMA376-18">[18]</a></sup></p>
<p>The ISO standardization of Office Open XML was controversial and embittered,<sup class="reference" id="cite_ref-infoworld-embittered_19-0"><a href="#cite_note-infoworld-embittered-19">[19]</a></sup> with much discussion both about the specification and about the standardization process.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> According to <i><a href="/wiki/InfoWorld" title="InfoWorld">InfoWorld</a></i>:</p>
<blockquote class="templatequote">
<p>OOXML was opposed by many on grounds it was unneeded, as software makers could use <a href="/wiki/OpenDocument" title="OpenDocument">OpenDocument Format</a> (ODF), a less complicated office software format that was already an international standard.<sup class="reference" id="cite_ref-infoworld-embittered_19-1"><a href="#cite_note-infoworld-embittered-19">[19]</a></sup></p>
<div class="templatequotecite"><cite>— InfoWorld</cite></div>
</blockquote>
<p>The same <i>InfoWorld</i> article reported that <a href="/wiki/IBM" title="IBM">IBM</a> (which supports the <a class="mw-redirect" href="/wiki/ODF" title="ODF">ODF</a> format) threatened to leave standards bodies that it said allow dominant corporations like Microsoft to wield undue influence. The article further says that Microsoft was accused of co-opting the standardization process by leaning on countries to ensure that it got enough votes at the ISO for Office Open XML to pass, although it does not specify exactly who accused Microsoft.<sup class="reference" id="cite_ref-infoworld-embittered_19-2"><a href="#cite_note-infoworld-embittered-19">[19]</a></sup></p>
<p><a href="/wiki/Richard_Stallman" title="Richard Stallman">Richard Stallman</a> of the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a> has stated that "Microsoft offers a gratis patent license for OOXML on terms which do not allow <a href="/wiki/Free_software" title="Free software">free</a> implementations."<sup class="reference" id="cite_ref-stallman_21-0"><a href="#cite_note-stallman-21">[21]</a></sup></p>
<h2><span class="mw-headline" id="Licensing">Licensing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Office_Open_XML&amp;action=edit&amp;section=3" title="Edit section: Licensing">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Under the Ecma International code of conduct in patent matters,<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup> participating and approving member organisations of ECMA are required to make available their patent rights on a <a href="/wiki/Reasonable_and_non-discriminatory_licensing" title="Reasonable and non-discriminatory licensing">reasonable and non-discriminatory</a> (RAND) basis.</p>
<p>Holders of patents which concern ISO/IEC International Standards may agree to a standardized license governing the terms under which such patents may be licensed, in accord with the <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a>/<a href="/wiki/International_Electrotechnical_Commission" title="International Electrotechnical Commission">IEC</a>/<a class="mw-redirect" href="/wiki/ITU" title="ITU">ITU</a> common patent policy.<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup></p>
<p>Microsoft, the main contributor to the standard, provided a covenant not to sue<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup> for its patent licensing. The covenant received a mixed reception, with some like the <a href="/wiki/Groklaw" title="Groklaw">Groklaw</a> <a href="/wiki/Blog" title="Blog">blog</a> criticizing it,<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> and others such as <a href="/wiki/Lawrence_Rosen_(attorney)" title="Lawrence Rosen (attorney)">Lawrence Rosen</a>, (an attorney and lecturer at <a href="/wiki/Stanford_Law_School" title="Stanford Law School">Stanford Law School</a>), endorsing it.<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup></p>
<p>Microsoft has added the format to their <i><a href="/wiki/Microsoft_Open_Specification_Promise" title="Microsoft Open Specification Promise">Open Specification Promise</a></i><sup class="reference" id="cite_ref-OSP_27-0"><a href="#cite_note-OSP-27">[27]</a></sup> in which</p>
<blockquote>
<p>Microsoft irrevocably promises not to assert any Microsoft Necessary Claims against you for making, using, selling, offering for sale, importing or distributing any implementation to the extent it conforms to a Covered Specification […]</p>
</blockquote>
<p>This is limited to applications which do not deviate from the ISO/IEC 29500:2008 or Ecma-376 standard and to parties that do not "file, maintain or voluntarily participate in a patent infringement lawsuit against a Microsoft implementation of such Covered Specification".<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup><sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> The Open Specification Promise was included in documents submitted to ISO/IEC in support of the ECMA-376 fast-track submission.<sup class="reference" id="cite_ref-JTC_licensing_30-0"><a href="#cite_note-JTC_licensing-30">[30]</a></sup> Ecma International asserted that, "The OSP enables both open source and commercial software to implement [the specification]".<sup class="reference" id="cite_ref-ecma-responses-pdf_31-0"><a href="#cite_note-ecma-responses-pdf-31">[31]</a></sup></p>
<h2><span class="mw-headline" id="Versions">Versions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Office_Open_XML&amp;action=edit&amp;section=4" title="Edit section: Versions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The Office Open XML specification exists in a number of versions.</p>
<h3><span class="mw-headline" id="ECMA-376_1st_edition_.282006.29">ECMA-376 1st edition (2006)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Office_Open_XML&amp;action=edit&amp;section=5" title="Edit section: ECMA-376 1st edition (2006)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The ECMA standard is structured in five parts to meet the needs of different audiences.<sup class="reference" id="cite_ref-StandardECMA376_18-1"><a href="#cite_note-StandardECMA376-18">[18]</a></sup></p>
<dl>
<dt>Part 1. Fundamentals</dt>
<dd>
<ul>
<li>Vocabulary, notational conventions and abbreviations</li>
<li>Summary of primary and supporting markup languages</li>
<li>Conformance conditions and interoperability guidelines</li>
<li>Constraints within the Open Packaging Conventions that apply to each document type</li>
</ul>
</dd>
</dl>
<dl>
<dt>Part 2. Open Packaging Conventions</dt>
<dd>
<ul>
<li>The <a href="/wiki/Open_Packaging_Conventions" title="Open Packaging Conventions">Open Packaging Conventions</a> (OPC), for the package model and physical package, is defined and used by various document types in various applications from multiple vendors.</li>
<li>It defines core properties, thumbnails, digital signatures, and authorizations &amp; encryption capabilities for parts or all of the contents in the package.</li>
<li>XML schemas for the OPC are declared as XML Schema Definitions (XSD) and (non-normatively) using <a href="/wiki/RELAX_NG" title="RELAX NG">RELAX NG</a> (ISO/IEC 19757-2)</li>
</ul>
</dd>
</dl>
<dl>
<dt>Part 3. Primer</dt>
<dd>
<ul>
<li>Informative (non-normative) introduction to WordprocessingML, SpreadsheetML, PresentationML, DrawingML, VML and Shared MLs, providing context and illustrating elements through examples and diagrams</li>
<li>Describes the custom XML data-storing facility within a package to support integration with business data</li>
</ul>
</dd>
</dl>
<dl>
<dt>Part 4. Markup Language Reference</dt>
<dd>
<ul>
<li>Contains the reference material for WordprocessingML, SpreadsheetML, PresentationML, DrawingML, Shared MLs and Custom XML Schema, defining every element and attribute including the element hierarchy (parent/child relationships)</li>
<li>XML schemas for the markup languages are declared as XSD and (non-normatively) using RELAX NG</li>
<li>Defines the custom XML data-storing facility</li>
</ul>
</dd>
</dl>
<dl>
<dt>Part 5. Markup Compatibility and Extensibility</dt>
<dd>
<ul>
<li>Describes extension facilities of OpenXML documents and specifies elements &amp; attributes through which applications can operate across different extensions.</li>
</ul>
</dd>
</dl>
<p>Later versions of the ECMA-376 standard are aligned and technically equivalent to the corresponding ISO standard.</p>
<h3><span class="mw-headline" id="ISO.2FIEC_29500:2008">ISO/IEC 29500:2008</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Office_Open_XML&amp;action=edit&amp;section=6" title="Edit section: ISO/IEC 29500:2008">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The ISO/IEC standard is structured into four parts:<sup class="reference" id="cite_ref-StandardIS29500_32-0"><a href="#cite_note-StandardIS29500-32">[32]</a></sup> Parts 1, 2 and 3 are independent standards; for example, Part 2, specifying <a href="/wiki/Open_Packaging_Conventions" title="Open Packaging Conventions">Open Packaging Conventions</a>, is used by other file formats including <a href="/wiki/Open_XML_Paper_Specification" title="Open XML Paper Specification">XPS</a> and <a href="/wiki/Design_Web_Format" title="Design Web Format">Design Web Format</a>. Part 4 is to be read as a modification to Part 1, which it requires.</p>
<p>A technically equivalent set of texts is also published by Ecma as ECMA-376 2nd edition (2008).</p>
<dl>
<dt>Part 1. Fundamentals &amp; Markup Language Reference</dt>
<dd>Consisting of 5560 pages, this part contains:
<ul>
<li>Conformance definitions</li>
<li>Reference material for the XML document markup languages defined by the Standard</li>
<li>XML schemas for the document markup languages declared using <a href="/wiki/XML_Schema_(W3C)" title="XML Schema (W3C)">XSD</a> and (non-normatively) <a href="/wiki/RELAX_NG" title="RELAX NG">RELAX NG</a></li>
<li>Defines the foreign markup facilities</li>
</ul>
</dd>
</dl>
<dl>
<dt>Part 2. Open Packaging Conventions</dt>
<dd>Consisting of 129 pages, this part contains:
<ul>
<li>A description of the Open Packaging Conventions (package model, physical package)</li>
<li>Core properties, <a href="/wiki/Thumbnail" title="Thumbnail">thumbnails</a> and <a href="/wiki/Digital_signature" title="Digital signature">digital signatures</a></li>
<li>XML schemas for the OPC are declared using <a href="/wiki/XML_Schema_(W3C)" title="XML Schema (W3C)">XSD</a> and (non-normatively) <a href="/wiki/RELAX_NG" title="RELAX NG">RELAX NG</a></li>
</ul>
</dd>
</dl>
<dl>
<dt>Part 3. Markup Compatibility and Extensibility</dt>
<dd>Consisting of 40 pages, this part contains:
<ul>
<li>A description of <i>extensions</i>: elements &amp; attributes which define mechanisms allowing applications to specify alternative means of negotiating content</li>
<li>Extensibility rules are expressed using <a href="/wiki/Namespace-based_Validation_Dispatching_Language" title="Namespace-based Validation Dispatching Language">NVDL</a></li>
</ul>
</dd>
</dl>
<dl>
<dt>Part 4. Transitional Migration Features</dt>
<dd>Consisting of 1464 pages, this part contains:
<ul>
<li>Legacy material such as compatibility settings and the graphics markup language VML</li>
<li>A list of syntactic differences between this text and ECMA-376 1st Edition</li>
</ul>
</dd>
</dl>
<p>The standard specifies two levels of document &amp; application conformance, <i>strict</i> and <i>transitional,</i> for each of WordprocessingML, PresentationML and SpreadsheetML, and also specifies applications' descriptions of <i>base</i> and <i>full</i>.</p>
<h3><span class="mw-headline" id="Compatibility_between_versions">Compatibility between versions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Office_Open_XML&amp;action=edit&amp;section=7" title="Edit section: Compatibility between versions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The intent of the changes from ECMA-376 1st Edition to ISO/IEC 29500:2008 was that a valid ECMA-376 document would also be a valid ISO 29500 Transitional document;<sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup> however, at least one change introduced at the BRM—refusing to allow further values for xsd:boolean—had the effect of breaking backwards-compatibility for most documents.<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup> A fix for this had been suggested to <a class="mw-redirect" href="/wiki/ISO/IEC_JTC1" title="ISO/IEC JTC1">ISO/IEC JTC1</a>/SC34/WG4, and was approved in June 2009 as a recommendation for the first revision to Office Open XML.<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup></p>
<p>Applications capable of reading documents compliant to ECMA-376 Edition 1 would regard ISO/IEC 29500-4 Transitional documents containing ISO 8601 dates as corrupt.<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup></p>
<h2><span class="mw-headline" id="Application_support">Application support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Office_Open_XML&amp;action=edit&amp;section=8" title="Edit section: Application support">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/List_of_software_that_supports_Office_Open_XML" title="List of software that supports Office Open XML">List of software that supports Office Open XML</a></div>
<p>Some older versions of Microsoft Word and Microsoft Office are able to read and write <code>.docx</code> files after installation of the free compatibility pack provided by Microsoft,<sup class="reference" id="cite_ref-omso_37-0"><a href="#cite_note-omso-37">[37]</a></sup> although some items, such as equations, are converted into images that cannot be edited.<sup class="reference" id="cite_ref-38"><a href="#cite_note-38">[38]</a></sup></p>
<p>Starting with <a href="/wiki/Microsoft_Office_2007" title="Microsoft Office 2007">Microsoft Office 2007</a>, the Office Open XML file formats have become the default file format<sup class="reference" id="cite_ref-microsoft.com_4-1"><a href="#cite_note-microsoft.com-4">[4]</a></sup> of <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a>.<sup class="reference" id="cite_ref-courierpress.com_5-1"><a href="#cite_note-courierpress.com-5">[5]</a></sup><sup class="reference" id="cite_ref-knowledge.wharton.upenn.edu_6-1"><a href="#cite_note-knowledge.wharton.upenn.edu-6">[6]</a></sup> However, due to the changes introduced in the Office Open XML standard, Office 2007 is not wholly in compliance with ISO/IEC 29500:2008.<sup class="reference" id="cite_ref-notOOXML_39-0"><a href="#cite_note-notOOXML-39">[39]</a></sup> <a href="/wiki/Microsoft_Office_2010" title="Microsoft Office 2010">Office 2010</a> includes support for the ISO/IEC 29500:2008-compliant version of Office Open XML, but it can only save documents conforming to the <i>transitional</i>, not the <i>strict</i>, schemas of the specification.<sup class="reference" id="cite_ref-docx23_40-0"><a href="#cite_note-docx23-40">[40]</a></sup><sup class="reference" id="cite_ref-docx22_41-0"><a href="#cite_note-docx22-41">[41]</a></sup> Note that the intent of the ISO/IEC is to allow the removal of the transitional variant from the ISO/IEC 29500 standard.<sup class="reference" id="cite_ref-docx22_41-1"><a href="#cite_note-docx22-41">[41]</a></sup></p>
<p>The ability to read and write Office Open XML format is, however, not limited to Microsoft Office; other office products are also able to read &amp; write this format:</p>
<ul>
<li><a href="/wiki/SoftMaker_Office" title="SoftMaker Office">SoftMaker Office 2010</a> is able to read and write <code>DOCX</code> and <code>XLSX</code> files in its word processor &amp; spreadsheet applications.</li>
<li><a href="/wiki/LibreOffice" title="LibreOffice">LibreOffice</a> is able to open and save Office Open XML files.<sup class="reference" id="cite_ref-42"><a href="#cite_note-42">[42]</a></sup></li>
<li><a href="/wiki/OpenOffice.org" title="OpenOffice.org">OpenOffice.org</a> and <a href="/wiki/Apache_OpenOffice" title="Apache OpenOffice">Apache OpenOffice</a> from version 3.0 has been able to import Office Open XML files, but not save them.<sup class="reference" id="cite_ref-43"><a href="#cite_note-43">[43]</a></sup> Version 3.2 improved this feature with read support even for password-protected Office Open XML files.<sup class="reference" id="cite_ref-44"><a href="#cite_note-44">[44]</a></sup><sup class="reference" id="cite_ref-45"><a href="#cite_note-45">[45]</a></sup><sup class="reference" id="cite_ref-46"><a href="#cite_note-46">[46]</a></sup></li>
<li>The <a href="/wiki/Go-oo" title="Go-oo">Go-oo</a> fork of OpenOffice could also write OOXML files.</li>
<li><a href="/wiki/KOffice" title="KOffice">KOffice</a> from version 2.2 and later was able to import OOXML files.</li>
<li><a href="/wiki/Calligra_Suite" title="Calligra Suite">Calligra Suite</a> is able to import Office Open XML files.</li>
<li><a href="/wiki/NeoOffice" title="NeoOffice">NeoOffice</a>, a OpenOffice.org fork for OS X is able to import but not save OOXML.</li>
</ul>
<p>Other office products that offer import support for the Office Open XML formats include:</p>

<li><a href="/wiki/TextEdit" title="TextEdit">TextEdit</a> (included with <a class="mw-redirect" href="/wiki/OS_X" title="OS X">OS X</a>)</li>
<li><a href="/wiki/IWork" title="IWork">iWork</a></li>
<li><a class="mw-redirect" href="/wiki/IBM_Lotus_Notes" title="IBM Lotus Notes">IBM Lotus Notes</a></li>
