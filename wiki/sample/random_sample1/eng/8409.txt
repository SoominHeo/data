<h1 class="firstHeading" id="firstHeading" lang="en">Simula</h1>

<p><b>Simula</b> is the name of two <a href="/wiki/Simulation_language" title="Simulation language">simulation</a> <a href="/wiki/Programming_language" title="Programming language">programming languages</a>, Simula I and Simula 67, developed in the 1960s at the <a href="/wiki/Norwegian_Computing_Center" title="Norwegian Computing Center">Norwegian Computing Center</a> in <a href="/wiki/Oslo" title="Oslo">Oslo</a>, by <a href="/wiki/Ole-Johan_Dahl" title="Ole-Johan Dahl">Ole-Johan Dahl</a> and <a href="/wiki/Kristen_Nygaard" title="Kristen Nygaard">Kristen Nygaard</a>. <a href="/wiki/Syntax_(programming_languages)" title="Syntax (programming languages)">Syntactically</a>, it is a fairly faithful superset of <a href="/wiki/ALGOL_60" title="ALGOL 60">ALGOL 60</a>.<sup class="reference" id="cite_ref-CommonBase_1-0"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:1.3.1</sup></p>
<p>Simula 67 introduced <a class="mw-redirect" href="/wiki/Object_(computing)" title="Object (computing)">objects</a>,<sup class="reference" id="cite_ref-CommonBase_1-1"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:2, 5.3</sup> <a class="mw-redirect" href="/wiki/Class_(computer_science)" title="Class (computer science)">classes</a>,<sup class="reference" id="cite_ref-CommonBase_1-2"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:1.3.3, 2</sup> <a href="/wiki/Inheritance_(object-oriented_programming)" title="Inheritance (object-oriented programming)">inheritance</a> and <a class="mw-redirect" href="/wiki/Subclass_(computer_science)" title="Subclass (computer science)">subclasses</a>,<sup class="reference" id="cite_ref-CommonBase_1-3"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:2.2.1</sup> <a href="/wiki/Virtual_function" title="Virtual function">virtual procedures</a>,<sup class="reference" id="cite_ref-CommonBase_1-4"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:2.2.3</sup> <a href="/wiki/Coroutine" title="Coroutine">coroutines</a>,<sup class="reference" id="cite_ref-CommonBase_1-5"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:9.2</sup> and <a href="/wiki/Discrete_event_simulation" title="Discrete event simulation">discrete event simulation</a>,<sup class="reference" id="cite_ref-CommonBase_1-6"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:14.2</sup> and features <a href="/wiki/Garbage_collection_(computer_science)" title="Garbage collection (computer science)">garbage collection</a>.<sup class="reference" id="cite_ref-CommonBase_1-7"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:9.1</sup> Also other forms of <a href="/wiki/Subtyping" title="Subtyping">subtyping</a> (besides inheriting subclasses) were introduced in Simula derivatives.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2013)">citation needed</span></a></i>]</sup></p>
<p>Simula is considered the first <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented</a> <a href="/wiki/Programming_paradigm" title="Programming paradigm">programming</a> language. As its name suggests, Simula was designed for doing <a class="mw-redirect" href="/wiki/Simulations" title="Simulations">simulations</a>, and the needs of that <a href="/wiki/Problem_domain" title="Problem domain">domain</a> provided the framework for many of the features of object-oriented languages today.</p>
<p>Simula has been used in a wide range of applications such as simulating <a href="/wiki/Very-large-scale_integration" title="Very-large-scale integration">VLSI</a> designs, <a href="/wiki/Process_modeling" title="Process modeling">process modeling</a>, <a class="mw-redirect" href="/wiki/Protocol_(computing)" title="Protocol (computing)">protocols</a>, <a href="/wiki/Algorithm" title="Algorithm">algorithms</a>, and other applications such as <a href="/wiki/Typesetting" title="Typesetting">typesetting</a>, <a href="/wiki/Computer_graphics" title="Computer graphics">computer graphics</a>, and <a href="/wiki/Education" title="Education">education</a>. The influence of Simula is often understated, and Simula-type objects are reimplemented in <a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/Object_Pascal" title="Object Pascal">Object Pascal</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a> and several other languages. The creator of C++, <a href="/wiki/Bjarne_Stroustrup" title="Bjarne Stroustrup">Bjarne Stroustrup</a>, has acknowledged that Simula 67 was the greatest influence on him to develop C++, to bring the kind of productivity enhancements offered by Simula to the raw computational speed offered by lower level languages like <a href="/wiki/BCPL" title="BCPL">BCPL</a>.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The following account is based on Jan Rune Holmevik's historical essay.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<p><a href="/wiki/Kristen_Nygaard" title="Kristen Nygaard">Kristen Nygaard</a> started writing computer simulation programs in 1957. Nygaard saw a need for a better way to describe the heterogeneity and the <a class="mw-redirect" href="/wiki/Instruction_(computer_science)" title="Instruction (computer science)">operation</a> of a <a href="/wiki/System" title="System">system</a>. To go further with his ideas on a formal <a href="/wiki/Computer_language" title="Computer language">computer language</a> for describing a system, Nygaard realized that he needed someone with more <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a> skills than he had. <a href="/wiki/Ole-Johan_Dahl" title="Ole-Johan Dahl">Ole-Johan Dahl</a> joined him on his work January 1962. The decision of linking the language up to <a href="/wiki/ALGOL_60" title="ALGOL 60">ALGOL 60</a> was made shortly after. By May 1962 the main concepts for a <a href="/wiki/Simulation" title="Simulation">simulation</a> <a href="/wiki/Programming_language" title="Programming language">language</a> were set. "SIMULA I" was born, a special purpose <a href="/wiki/Programming_language" title="Programming language">programming language</a> for <a href="/wiki/Simulation" title="Simulation">simulating</a> discrete event systems.</p>
<p>Kristen Nygaard was invited to <a href="/wiki/UNIVAC" title="UNIVAC">UNIVAC</a> late May 1962 in connection with the marketing of their new <a class="mw-redirect" href="/wiki/UNIVAC_1107" title="UNIVAC 1107">UNIVAC 1107</a> computer. At that visit Nygaard presented the ideas of Simula to <a href="/wiki/Bob_Bemer" title="Bob Bemer">Robert Bemer</a>, the director of systems programming at <a class="mw-redirect" href="/wiki/Univac" title="Univac">Univac</a>. Bemer was a sworn <a href="/wiki/ALGOL" title="ALGOL">ALGOL</a> fan and found the Simula project compelling. Bemer was also chairing a session at the second international conference on information processing hosted by <a href="/wiki/International_Federation_for_Information_Processing" title="International Federation for Information Processing">IFIP</a>. He invited Nygaard, who presented the paper "SIMULA -- An Extension of ALGOL to the Description of Discrete-Event Networks".</p>
<p><a href="/wiki/Norwegian_Computing_Center" title="Norwegian Computing Center">Norwegian Computing Center</a> got a <a class="mw-redirect" href="/wiki/UNIVAC_1107" title="UNIVAC 1107">UNIVAC 1107</a> August 1963 at a considerable discount, on which Dahl implemented the SIMULA I under contract with UNIVAC. The implementation was based on the UNIVAC <a href="/wiki/ALGOL_60" title="ALGOL 60">ALGOL 60</a> compiler. SIMULA I was fully operational on the UNIVAC 1107 by January 1965. In the following couple of years Dahl and Nygaard spent a lot of time teaching Simula. Simula spread to several countries around the world and SIMULA I was later implemented on <a href="/wiki/Burroughs_large_systems" title="Burroughs large systems">Burroughs B5500 computers</a> and the Russian <a href="/wiki/Ural_(computer)" title="Ural (computer)">URAL-16 computer</a>.</p>
<p>In 1966 <a class="mw-redirect" href="/wiki/C._A._R._Hoare" title="C. A. R. Hoare">C. A. R. Hoare</a> introduced the concept of record class construct, which Dahl and Nygaard extended with the concept of prefixing and other features to meet their requirements for a generalized process concept. Dahl and Nygaard presented their paper on <a class="mw-redirect" href="/wiki/Class_(computer_science)" title="Class (computer science)">Class</a> and <a class="mw-redirect" href="/wiki/Subclass_(computer_science)" title="Subclass (computer science)">Subclass</a> Declarations at the <a href="/wiki/International_Federation_for_Information_Processing" title="International Federation for Information Processing">IFIP</a> Working Conference on <a href="/wiki/Simulation_language" title="Simulation language">simulation languages</a> in <a href="/wiki/Oslo" title="Oslo">Oslo</a>, May 1967. This paper became the first formal definition of Simula 67. In June 1967 a conference was held to standardize the language and initiate a number of implementations. Dahl proposed to unify the <a href="/wiki/Data_type" title="Data type">type</a> and the class concept. This led to serious discussions, and the proposal was rejected by the board. SIMULA 67 was formally standardized on the first meeting of the SIMULA Standards Group (SSG) in February 1968.</p>
<p>Simula was influential in the development of <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a> and later <a class="mw-redirect" href="/wiki/Object-oriented_programming_language" title="Object-oriented programming language">object-oriented programming languages</a>. It also helped inspire the <a href="/wiki/Actor_model" title="Actor model">actor model</a> of concurrent computation although Simula only supports <a class="mw-redirect" href="/wiki/Co-routines" title="Co-routines">co-routines</a> and not true <a href="/wiki/Concurrency_(computer_science)" title="Concurrency (computer science)">concurrency</a>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2015)">citation needed</span></a></i>]</sup></p>
<p>In the late sixties and the early seventies there were four main implementations of Simula:</p>
<ul>
<li><a href="/wiki/UNIVAC" title="UNIVAC">UNIVAC 1100</a> by <a href="/wiki/Norwegian_Computing_Center" title="Norwegian Computing Center">NCC</a></li>
<li><a class="mw-redirect" href="/wiki/System/360" title="System/360">System/360</a> and <a class="mw-redirect" href="/wiki/System/370" title="System/370">System/370</a> by <a href="/wiki/Norwegian_Computing_Center" title="Norwegian Computing Center">NCC</a></li>
<li><a href="/wiki/CDC_3000" title="CDC 3000">CDC 3000</a> by <a href="/wiki/University_of_Oslo" title="University of Oslo">University of Oslo</a>'s Joint Computer Installation at Kjeller</li>
<li><a href="/wiki/TOPS-10" title="TOPS-10">TOPS-10</a> by <a href="/wiki/Swedish_National_Defence_Research_Institute" title="Swedish National Defence Research Institute">Swedish National Defence Research Institute</a> (FOA)</li>
</ul>
<p>These implementations were ported to a wide range of platforms. The <a href="/wiki/TOPS-10" title="TOPS-10">TOPS-10</a> implemented the concept of public, protected, and private member variables and procedures, that later was integrated into Simula 87. Simula 87 is the latest standard and is ported to a wide range of platforms. There are mainly three implementations:</p>
<ul>
<li>Simula AS</li>
<li>Lund Simula</li>
<li>GNU Cim<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></li>
</ul>
<p>In November 2001 Dahl and Nygaard were awarded the <a href="/wiki/IEEE_John_von_Neumann_Medal" title="IEEE John von Neumann Medal">IEEE John von Neumann Medal</a> by the <a class="mw-redirect" href="/wiki/Institute_of_Electrical_and_Electronic_Engineers" title="Institute of Electrical and Electronic Engineers">Institute of Electrical and Electronic Engineers</a> "For the introduction of the concepts underlying <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a> through the design and implementation of SIMULA 67". In February 2002 they received the 2001 A. M. <a href="/wiki/Turing_Award" title="Turing Award">Turing Award</a> by the <a href="/wiki/Association_for_Computing_Machinery" title="Association for Computing Machinery">Association for Computing Machinery</a> (ACM), with the citation: "For ideas fundamental to the emergence of object oriented programming, through their design of the programming languages Simula I and Simula 67." Unfortunately neither Dahl, nor Nygaard could make it to the ACM Turing Award Lecture,<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> scheduled to be delivered at the <a href="/wiki/OOPSLA" title="OOPSLA">OOPSLA</a> 2002 conference in Seattle, as they both died within two months of each other in June and August, respectively.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p><a href="/wiki/Simula_Research_Laboratory" title="Simula Research Laboratory">Simula Research Laboratory</a> is a research institute named after the Simula language, and Nygaard held a part-time position there from the opening in 2001.</p>
<p>The new Computer Science building at the <a href="/wiki/University_of_Oslo" title="University of Oslo">University of Oslo</a> is named Ole Johan Dahl's House, after one of the two inventors of Simula. The main auditorium in Ole Johan Dahl's House is named Simula.</p>
<p>Simula is still used for various types of university courses, for instance, Jarek Sklenar teaches Simula to students at University of Malta.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></p>
<h2><span class="mw-headline" id="Sample_code">Sample code</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=2" title="Edit section: Sample code">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Minimal_program">Minimal program</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=3" title="Edit section: Minimal program">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The empty <a href="/wiki/Computer_file" title="Computer file">computer file</a> is the minimal <a href="/wiki/Computer_program" title="Computer program">program</a> in Simula, measured by the size of the <a href="/wiki/Source_code" title="Source code">source code</a>. It consists of one thing only; a dummy <a class="mw-redirect" href="/wiki/Statement_(programming)" title="Statement (programming)">statement</a>.</p>
<p>However, the minimal program is more conveniently represented as an empty block:</p>
<pre>
<b>Begin</b>
<b>End</b>;
</pre>
<p>It begins executing and immediately terminates. The language does not have any <a class="mw-redirect" href="/wiki/Return_value" title="Return value">return value</a> from the program itself.</p>
<h3><span class="mw-headline" id="Classic_Hello_world">Classic Hello world</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=4" title="Edit section: Classic Hello world">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>An example of a <a class="mw-redirect" href="/wiki/Hello_world_program" title="Hello world program">Hello world program</a> in Simula:</p>
<pre>
<b>Begin</b>
   <b>OutText</b> ("Hello World!");
   <b>Outimage</b>;
<b>End</b>;
</pre>
<p>Simula is <a class="mw-redirect" href="/wiki/Case-insensitive" title="Case-insensitive">case-insensitive</a>.</p>
<h3><span class="mw-headline" id="Classes.2C_subclasses_and_virtual_procedures">Classes, subclasses and virtual procedures</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=5" title="Edit section: Classes, subclasses and virtual procedures">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A more realistic example with use of classes,<sup class="reference" id="cite_ref-CommonBase_1-8"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:1.3.3, 2</sup> subclasses<sup class="reference" id="cite_ref-CommonBase_1-9"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:2.2.1</sup> and virtual procedures:<sup class="reference" id="cite_ref-CommonBase_1-10"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:2.2.3</sup></p>
<pre>
<b>Begin</b>
   <b>Class</b> Glyph;
      <b>Virtual</b>: <b>Procedure</b> print <b>Is</b> <b>Procedure</b> print;
   <b>Begin</b>
   <b>End</b>;

   Glyph <b>Class</b> Char (c);
      <b>Character</b> c;
   <b>Begin</b>
      <b>Procedure</b> print;
        OutChar(c);
   <b>End</b>;

   Glyph <b>Class</b> Line (elements);
      <b>Ref</b> (Glyph) <b>Array</b> elements;
   <b>Begin</b>
      <b>Procedure</b> print;
      <b>Begin</b>
         <b>Integer</b> i;
         <b>For</b> i:= 1 <b>Step</b> 1 <b>Until</b> UpperBound (elements, 1) <b>Do</b>
            elements (i).print;
         OutImage;
      <b>End</b>;
   <b>End</b>;

   <b>Ref</b> (Glyph) rg;
   <b>Ref</b> (Glyph) <b>Array</b> rgs (1 : 4);

   <i>! Main program;</i>
   rgs (1):- <b>New</b> Char ('A');
   rgs (2):- <b>New</b> Char ('b');
   rgs (3):- <b>New</b> Char ('b');
   rgs (4):- <b>New</b> Char ('a');
   rg:- <b>New</b> Line (rgs);
   rg.print;
<b>End</b>;
</pre>
<p>The above example has one <a class="mw-redirect" href="/wiki/Superclass_(computer_science)" title="Superclass (computer science)">super class</a> (Glyph) with two <a class="mw-redirect" href="/wiki/Subclass_(computer_science)" title="Subclass (computer science)">subclasses</a> (Char and Line). There is one <a href="/wiki/Virtual_function" title="Virtual function">virtual procedure</a> with two <a href="/wiki/Implementation" title="Implementation">implementations</a>. The execution starts by executing the main program. Simula does not have the concept of <a class="mw-redirect" href="/wiki/Abstract_class" title="Abstract class">abstract classes</a> since classes with <a class="mw-redirect" href="/wiki/Pure_virtual_function" title="Pure virtual function">pure virtual procedures</a> can be <a class="mw-redirect" href="/wiki/Instantiation_(computer_science)" title="Instantiation (computer science)">instantiated</a>. This means that in the above example all classes can be instantiated. Calling a <a class="mw-redirect" href="/wiki/Pure_virtual_function" title="Pure virtual function">pure virtual procedure</a> will however produce a <a class="mw-redirect" href="/wiki/Run-time_error" title="Run-time error">run-time error</a>.</p>
<h3><span class="mw-headline" id="Call_by_name">Call by name</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=6" title="Edit section: Call by name">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Simula supports <a class="mw-redirect" href="/wiki/Call_by_name" title="Call by name">call by name</a><sup class="reference" id="cite_ref-CommonBase_1-11"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:8.2.3</sup> so the <a href="/wiki/Jensen%27s_Device" title="Jensen's Device">Jensen's Device</a> can easily be implemented. However, the default transmission mode for simple parameter is <a class="mw-redirect" href="/wiki/Call_by_value" title="Call by value">call by value</a>, contrary to <a href="/wiki/ALGOL" title="ALGOL">ALGOL</a> which used <a class="mw-redirect" href="/wiki/Call_by_name" title="Call by name">call by name</a>. The source code for the <a href="/wiki/Jensen%27s_Device" title="Jensen's Device">Jensen's Device</a> must therefore specify <a class="mw-redirect" href="/wiki/Call_by_name" title="Call by name">call by name</a> for the parameters when compiled by a Simula compiler.</p>
<p>Another much simpler example is the <a href="/wiki/Arithmetic_function#Summation_functions" title="Arithmetic function">summation function <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML">

</mrow>

</semantics>
</math></span><img alt="\sum " aria-hidden="true" class="mwe-math-fallback-image-inline" src="https://wikimedia.org/api/rest_v1/media/math/render/svg/f1d4e06539576633987e902f402ed46728d573b6" style="vertical-align: -1.338ex; width:3.365ex; height:3.843ex;"/></span></a> which can be implemented as follows:</p>
<pre>
<b>Real</b> <b>Procedure</b> Sigma (k, m, n, u);

<h3><span class="mw-headline" id="Simulation">Simulation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=7" title="Edit section: Simulation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Simula includes a <a href="/wiki/Simulation" title="Simulation">simulation</a><sup class="reference" id="cite_ref-CommonBase_1-12"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:14.2</sup> package for doing <a href="/wiki/Discrete_event_simulation" title="Discrete event simulation">discrete event simulations</a>. This simulation package is based on Simula's <a class="mw-redirect" href="/wiki/Object_oriented" title="Object oriented">object oriented</a> features and its <a href="/wiki/Coroutine" title="Coroutine">coroutine</a><sup class="reference" id="cite_ref-CommonBase_1-13"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:9.2</sup> concept.</p>
<p>Sam, Sally, and Andy are shopping for clothes. They have to share one fitting room. Each one of them is browsing the store for about 12 minutes and then uses the fitting room exclusively for about three minutes, each following a normal distribution. A simulation of their fitting room experience is as follows:</p>
<pre>
Simulation <b>Begin</b>
   <b>Class</b> FittingRoom; <b>Begin</b>
      <b>Ref</b> (Head) door;
      <b>Boolean</b> inUse;
      <b>Procedure</b> request; <b>Begin</b>
         <b>If</b> inUse <b>Then</b> <b>Begin</b>
             Wait (door);
             door.First.Out;
         <b>End</b>;
         inUse:= <b>True</b>;
      <b>End</b>;
      <b>Procedure</b> leave; <b>Begin</b>
         inUse:= <b>False</b>;
         <b>Activate</b> door.First;
      <b>End</b>;
      door:- <b>New</b> Head;
   <b>End</b>;
  
   <b>Procedure</b> report (message); <b>Text</b> message; <b>Begin</b>
      OutFix (Time, 2, 0); OutText (": " &amp; message); OutImage;
   <b>End</b>;

   Process <b>Class</b> Person (pname); <b>Text</b> pname; <b>Begin</b>
      <b>While</b> <b>True</b> <b>Do</b> <b>Begin</b>
         Hold (Normal (12, 4, u));
         report  (pname &amp; " is requesting the fitting room");
         fittingroom1.request;
         report (pname &amp; " has entered the fitting room");
         Hold (Normal (3, 1, u));
         fittingroom1.leave;
         report (pname &amp; " has left the fitting room");
      <b>End</b>;
   <b>End</b>;

   <b>Integer</b> u;
   <b>Ref</b> (FittingRoom) fittingRoom1;

   fittingRoom1:- <b>New</b> FittingRoom;
   <b>Activate</b> <b>New</b> Person ("Sam");
   <b>Activate</b> <b>New</b> Person ("Sally");
   <b>Activate</b> <b>New</b> Person ("Andy");
   Hold (100);
<b>End</b>;
</pre>
<p>The main block is prefixed with <code>Simulation</code> for enabling simulation. The simulation package can be used on any block and simulations can even be nested when simulating someone doing simulations.</p>
<p>The fitting room object uses a queue (<code>door</code>) for getting access to the fitting room. When someone requests the fitting room and it's in use they must wait in this queue (<code>Wait (door)</code>). When someone leaves the fitting room the first one (if any) is released from the queue (<code><b>Activate</b> door.first</code>) and accordingly removed from the door queue (<code>door.First.Out</code>).</p>
<p>Person is a subclass of Process and its activity is described using hold (time for browsing the store and time spent in the fitting room) and calls procedures in the fitting room object for requesting and leaving the fitting room.</p>
<p>The main program creates all the objects and activates all the person objects to put them into the event queue. The main program holds for 100 minutes of simulated time before the program terminates.</p>
