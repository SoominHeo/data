<h1 class="firstHeading" id="firstHeading" lang="en">System V printing system</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p>The <a href="/wiki/Printer_(computing)" title="Printer (computing)">printing</a> subsystem of <a href="/wiki/UNIX_System_V" title="UNIX System V">UNIX System V</a> is one of several standardized systems for printing on Unix, and is typical of commercial System V-based Unix versions such as <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">Solaris</a> and <a class="mw-redirect" href="/wiki/SCO_OpenServer" title="SCO OpenServer">SCO OpenServer</a>. A system running this print architecture could traditionally be identified by the use of the user command <span style="font-family: monospace, monospace;">lp</span> as the primary interface to the print system, as opposed to the <a href="/wiki/Berkeley_Software_Distribution" title="Berkeley Software Distribution">BSD</a> <a href="/wiki/Line_Printer_Daemon_protocol" title="Line Printer Daemon protocol"><span style="font-family: monospace, monospace;">lpr</span></a> command (though some systems provide <span style="font-family: monospace, monospace;">lpr</span> as an alias to <span style="font-family: monospace, monospace;">lp</span>).</p>
<p>Typical user commands available to the System V printing system are:</p>
<ul>
<li><span style="font-family: monospace, monospace;">lp</span>: the user command to print a document</li>
<li><span style="font-family: monospace, monospace;">lpstat</span>: shows the current print queue</li>
<li><span style="font-family: monospace, monospace;">cancel</span>: deletes a job from the print queue</li>
<li><span style="font-family: monospace, monospace;">lpadmin</span>: a system administration command that configures the print system</li>
<li><span style="font-family: monospace, monospace;">lpmove</span>: a system administration command that moves jobs between print queues</li>
</ul>
<p></p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=System_V_printing_system&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In the Unix programming model, <a href="/wiki/Device_file" title="Device file">device files</a> are special files that act as access points to peripheral devices such as printers. For example, the first <a href="/wiki/Line_printer" title="Line printer">line printer</a> on a Unix system might be represented by a file <span style="font-family: monospace, monospace;">lp1</span> in the device (<span style="font-family: monospace, monospace;">/dev</span>) directory, <i>i.e.</i>, <span style="font-family: monospace, monospace;">/dev/lp1</span>. Using the file metaphor, a document could by printed by "copying" the file onto the device: <span style="font-family: monospace, monospace;">cp document /dev/lp1</span>. While this worked well enough for the case where there was one printer per user, this model did not scale out well to <a href="/wiki/Time-sharing" title="Time-sharing">multiple users</a> having to share one printer.<sup class="reference" id="cite_ref-Kofler1997_1-0"><a href="#cite_note-Kofler1997-1">[1]</a></sup> The solution was to create a queue (or "spool") of documents to be printed and use a <a href="/wiki/Daemon_(computing)" title="Daemon (computing)">daemon</a> (system process) to manage this queue and send the documents to the printer in the order in which they arrived.</p>
<p>Such a system, with an <span style="font-family: monospace, monospace;">lp</span> command to send documents to the queue, was first introduced in 1973 in <a href="/wiki/Research_Unix" title="Research Unix">Version 4</a> of Unix.<sup class="reference" id="cite_ref-Arthur1986_2-0"><a href="#cite_note-Arthur1986-2">[2]</a></sup> By the release of <a href="/wiki/UNIX_System_V#SVR4" title="UNIX System V">System V Release 4</a>, the suite of utilities had grown to include commands for canceling print jobs, moving jobs among queues, enabling and disabling queues, enabling and disabling a job scheduler daemon, and status reports of the print system.<sup class="reference" id="cite_ref-Nemeth1985_3-0"><a href="#cite_note-Nemeth1985-3">[3]</a></sup> The <span style="font-family: monospace, monospace;">lp</span> command handled queue documents to be printed<sup class="reference" id="cite_ref-Nemeth1985_3-1"><a href="#cite_note-Nemeth1985-3">[3]</a></sup> and had over 20 different options that controlled the appearance of the document and its place in the queue,<sup class="reference" id="cite_ref-Rosen1990_4-0"><a href="#cite_note-Rosen1990-4">[4]</a></sup> and even handled email notification of the user once the document had finished printing.<sup class="reference" id="cite_ref-Arthur1986_2-1"><a href="#cite_note-Arthur1986-2">[2]</a></sup> The command returned a "job id" which could be used by the <i>cancel</i> or <i>lpstat</i> commands to remove the job from the queue or check on its progress, respectively.<sup class="reference" id="cite_ref-Sobell1985_5-0"><a href="#cite_note-Sobell1985-5">[5]</a></sup> While the system was considered to be quite complex to set up and administer, most uses were expected to only use these three commands.<sup class="reference" id="cite_ref-Rosen1990_4-1"><a href="#cite_note-Rosen1990-4">[4]</a></sup></p>
<p>With its distribution in the influential AT&amp;T Unix System V, the interface if not the implementation became the standard for users' control over printers. The <span style="font-family: monospace, monospace;">lp</span> command was included as a requirement in the <a href="/wiki/POSIX#POSIX.2" title="POSIX">POSIX</a>.2 standard,<sup class="reference" id="cite_ref-Gardner1994_6-0"><a href="#cite_note-Gardner1994-6">[6]</a></sup> and a command by that name appeared in the subsequent lpr, <a href="/wiki/LPRng" title="LPRng">LPRng</a> and <a href="/wiki/CUPS" title="CUPS">CUPS</a> printing systems. (In SVR4 derivates like <a class="mw-redirect" href="/wiki/SCO_OpenServer" title="SCO OpenServer">SCO UNIX</a>, the <span style="font-family: monospace, monospace;">lp</span> command was simply an alias for the <span style="font-family: monospace, monospace;">lpr</span> command used by the BSD-based <span style="font-family: monospace, monospace;">lpr</span> system.<sup class="reference" id="cite_ref-Cutler1994_7-0"><a href="#cite_note-Cutler1994-7">[7]</a></sup>)</p>
<p>As late as 1996, <i>Running <a href="/wiki/Linux" title="Linux">Linux</a></i> stated "The Linux printing software consists of the UNIX standard <i>lp</i> and <i>lpr</i> software,"<sup class="reference" id="cite_ref-Welsh1995_8-0"><a href="#cite_note-Welsh1995-8">[8]</a></sup> but by 1999 support for <i>lp</i> was waning and the third edition simply stated "The <i>lpr</i> command prints a document on Linux."<sup class="reference" id="cite_ref-Welsh1999_9-0"><a href="#cite_note-Welsh1999-9">[9]</a></sup> By 2003, a survey of the Debian, <a href="/wiki/Mandriva_Linux" title="Mandriva Linux">Mandrake</a>, <a href="/wiki/Red_Hat" title="Red Hat">Red Hat</a>, <a href="/wiki/Slackware" title="Slackware">Slackware</a> and <a href="/wiki/SUSE_Linux_distributions" title="SUSE Linux distributions">SuSE</a> distributions showed that all of them were running some combination of lpr, LPRng and CUPS.<sup class="reference" id="cite_ref-Smith2003_10-0"><a href="#cite_note-Smith2003-10">[10]</a></sup></p>
<p>The original System V printing system remains proprietary; however, the Solaris print system, heavily modified from the original, has been released as <a class="mw-redirect" href="/wiki/Open_source_software" title="Open source software">open source software</a> as part of the <a href="/wiki/OpenSolaris" title="OpenSolaris">OpenSolaris</a> project. The <a class="mw-redirect" href="/wiki/Common_Unix_Printing_System" title="Common Unix Printing System">Common Unix Printing System</a> emulates both System V and Berkeley print architectures on the interface level, though its internal architecture is different from both.</p>
<h2><span class="mw-headline" id="Criticism">Criticism</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=System_V_printing_system&amp;action=edit&amp;section=2" title="Edit section: Criticism">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In his introduction to a simplified configuration system for lp, author Peter Gray of the <a href="/wiki/University_of_Wollongong" title="University of Wollongong">University of Wollongong</a> described several weaknesses of the version shipping with the then-current <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">Solaris (operating system)</a> version 2.</p>
<ul>
<li>As opposed to the single daemon used by the simpler BSD lpr system, the lp system used separate daemons, one for scheduling and one for remote communication.</li>
<li>The lpr system could be controlled with a single configuration file while lp requires a separate program for administration.</li>
<li>The lp system did support permissions, but the model did not scale to hundreds of users.</li>
</ul>
<p>As a result, Gray observed that "many administrators choose to simply run the old lpr/lpd system on the SVR4 boxes."<sup class="reference" id="cite_ref-Gray1997_11-0"><a href="#cite_note-Gray1997-11">[11]</a></sup></p>
