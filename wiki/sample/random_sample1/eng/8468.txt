<p><b>Secure copy</b> or <b>SCP</b> is a means of securely transferring <a href="/wiki/Computer_file" title="Computer file">computer files</a> between a local host and a remote <a href="/wiki/Server_(computing)" title="Server (computing)">host</a> or between two remote hosts. It is based on the <a href="/wiki/Secure_Shell" title="Secure Shell">Secure Shell</a> (SSH) protocol.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> "SCP" commonly refers to the Secure Copy Protocol or the program itself.<sup class="reference" id="cite_ref-Pechanec_2-0"><a href="#cite_note-Pechanec-2">[2]</a></sup></p>
<p></p>

<h2><span class="mw-headline" id="Secure_Copy_Protocol">Secure Copy Protocol</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Secure_copy&amp;action=edit&amp;section=1" title="Edit section: Secure Copy Protocol">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The <b>SCP</b> is a <a class="mw-redirect" href="/wiki/Network_protocol" title="Network protocol">network protocol</a>, based on the <a class="mw-redirect" href="/wiki/BSD" title="BSD">BSD</a> <a class="mw-redirect" href="/wiki/Rcp_(Unix)" title="Rcp (Unix)">RCP</a> protocol,<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> which supports <a href="/wiki/File_transfer" title="File transfer">file transfers</a> between hosts on a network. SCP uses <a href="/wiki/Secure_Shell" title="Secure Shell">Secure Shell</a> (SSH) for data transfer and uses the same mechanisms for authentication, thereby ensuring the <a href="/wiki/Information_security#Authenticity" title="Information security">authenticity</a> and <a href="/wiki/Confidentiality" title="Confidentiality">confidentiality</a> of the <a href="/wiki/Data_in_transit" title="Data in transit">data in transit</a>. A client can send (upload) files to a server, optionally including their basic attributes (permissions, timestamps). Clients can also request files or directories from a server (download). SCP runs over <a class="mw-redirect" href="/wiki/TCP_port" title="TCP port">TCP port</a> 22 by default. Like RCP, there is no <a href="/wiki/Request_for_Comments" title="Request for Comments">RFC</a> that defines the specifics of the protocol.</p>
<h3><span class="mw-headline" id="Function">Function</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Secure_copy&amp;action=edit&amp;section=2" title="Edit section: Function">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Normally, a client initiates an SSH connection to the remote host, and requests an SCP process to be started on the remote server. The remote SCP process can operate in one of two modes:</p>
<ul>
<li>source mode, which reads files (usually from disk) and sends them back to the client, or</li>
<li>sink mode, which accepts the files sent by the client and writes them (usually to disk) on the remote host.</li>
</ul>
<p>For most SCP clients, source mode is generally triggered with the -f flag (from), while sink mode is triggered with -t (to).<sup class="reference" id="cite_ref-Pechanec_2-1"><a href="#cite_note-Pechanec-2">[2]</a></sup> These flags are used internally and are not documented outside the SCP source code.</p>
<h3><span class="mw-headline" id="Remote_to_remote_mode">Remote to remote mode</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Secure_copy&amp;action=edit&amp;section=3" title="Edit section: Remote to remote mode">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In the past, in remote-to-remote secure copy, the SCP client opens an SSH connection to the source host and requests that it, in turn, open an SCP connection to the destination. (Remote-to-remote mode did not support opening two SCP connections and using the originating client as an intermediary). It is important to note that SCP cannot be used to remotely copy from the source to the destination when operating in password or keyboard-interactive authentication mode, as this would reveal the destination server's authentication credentials to the source. It is, however, possible with key-based or <a class="mw-redirect" href="/wiki/GSSAPI" title="GSSAPI">GSSAPI</a> methods that do not require user input.<sup class="reference" id="cite_ref-Pechanec_2-2"><a href="#cite_note-Pechanec-2">[2]</a></sup></p>
<p>Recently, remote-to-remote mode supports routing traffic through the client which originated the transfer, even though it is a 3rd party to the transfer. This way, authorization credentials must reside only on the originating client, the 3rd party.</p>
<h3><span class="mw-headline" id="Issues_using_talkative_shell_profiles">Issues using talkative shell profiles</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Secure_copy&amp;action=edit&amp;section=4" title="Edit section: Issues using talkative shell profiles">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>SCP does not expect text communicating with the ssh login shell. Text transmitted due to the ssh profile (e.g. echo "Welcome" in the .bashrc file) is interpreted as an error message, and a null line (echo "") causes scp to deadlock waiting for the error message to complete.<sup class="reference" id="cite_ref-Pechanec_2-3"><a href="#cite_note-Pechanec-2">[2]</a></sup></p>
<h2><span class="mw-headline" id="Secure_Copy_.28remote_file_copy_program.29">Secure Copy (remote file copy program)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Secure_copy&amp;action=edit&amp;section=5" title="Edit section: Secure Copy (remote file copy program)">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The <b>SCP program</b> is a software tool implementing the SCP protocol as a service daemon or client. It is a program to perform secure copying. The SCP server program is typically the same program as the SCP client. The SCP Server software can be installed on a Regular Machine and be configured to only accept SCP Traffic on port 22 through the firewall for greater security.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p>Perhaps the most widely used SCP program is the <a class="mw-redirect" href="/wiki/Command_line" title="Command line">command line</a> <tt>scp</tt> program, which is provided in most SSH implementations. The <tt>scp</tt> program is the secure analog of the <tt>rcp</tt> command. The <tt>scp</tt> program must be part of all SSH servers that want to provide SCP service, as <tt>scp</tt> functions as SCP server too.</p>
<p>Some SSH implementations provide the <tt>scp2</tt> program, which uses the <a class="mw-redirect" href="/wiki/SSH_file_transfer_protocol" title="SSH file transfer protocol">SFTP</a> protocol instead of SCP, but provides the very same <a class="mw-redirect" href="/wiki/Command_line" title="Command line">command line</a> interface as <tt>scp</tt>. <tt>scp</tt> is then typically a <a href="/wiki/Symbolic_link" title="Symbolic link">symbolic link</a> to <tt>scp2</tt>.</p>
<p>Typically, a syntax of <tt>scp</tt> program<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> is like the syntax of <a href="/wiki/Cp_(Unix)" title="Cp (Unix)"><tt>cp</tt> (copy)</a>:</p>
<p>Copying file to host:</p>
<pre>
<tt>scp <i>SourceFile</i> <i>user</i>@<i>host</i>:<i>directory</i>/<i>TargetFile</i></tt>
</pre>
<p>Copying file from host:</p>
<pre>
<tt>scp <i>user</i>@<i>host</i>:<i>directory</i>/<i>SourceFile</i> <i>TargetFile</i></tt>
<tt>scp -r <i>user</i>@<i>host</i>:<i>directory</i>/<i>SourceFolder</i> <i>TargetFolder</i></tt>
</pre>
<p>Note that if the remote host uses a port other than the default of 22, it can be specified in the command. For example, copying a file from host:</p>
<pre>
<tt>scp -P 2222 <i>user</i>@<i>host</i>:<i>directory</i>/<i>SourceFile</i> <i>TargetFile</i></tt>
</pre>
<p>As the <b>Secure Copy Protocol</b> implements file transfers only, <a class="mw-redirect" href="/wiki/GUI" title="GUI">GUI</a> SCP clients are rare, as implementing it requires additional functionality (<a class="mw-redirect" href="/wiki/Directory_(file_systems)" title="Directory (file systems)">directory</a> listing at least). For example, <a href="/wiki/WinSCP" title="WinSCP">WinSCP</a> defaults to the SFTP protocol. Even when operating in SCP mode, clients like WinSCP are typically not pure SCP clients, as they must use other means to implement the additional functionality (like the <tt>ls</tt> command). This in turn brings platform-dependency problems.</p>
<p>More comprehensive tools for managing files over SSH are <a href="/wiki/SSH_File_Transfer_Protocol" title="SSH File Transfer Protocol">SFTP</a> clients.</p>
