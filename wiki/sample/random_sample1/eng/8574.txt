<h1 class="firstHeading" id="firstHeading" lang="en">Cylinder-head-sector</h1>
<div class="mw-body-content" id="bodyContent">

<p><b>Cylinder-head-sector</b>, also known as <b>CHS</b>, is an early method for giving addresses to each physical block of data on a <a href="/wiki/Hard_disk_drive" title="Hard disk drive">hard disk drive</a>. In the case of <i>floppy</i> drives, for which the same exact <i>diskette</i> medium can be truly <i><a href="/wiki/Disk_formatting" title="Disk formatting">low-level formatted</a></i> to different capacities, this is still true.</p>
<p>Though CHS values no longer have a direct physical relationship to the data stored on modern storage media except for floppy disks, virtual CHS values (which can be <i>translated</i> by disk electronics or software) are still being used by many utility programs and file systems.</p>

<h2><span class="mw-headline" id="Definitions">Definitions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Cylinder-head-sector&amp;action=edit&amp;section=1" title="Edit section: Definitions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>CHS addressing is the process of identifying individual <a href="#Sectors">sectors</a> on a disk by their position in a <a href="#Tracks">track</a>, where the track is determined by the <a href="#Heads">head</a> and <a href="#Cylinders">cylinder</a> numbers. The terms are explained bottom up, for disk addressing the <i>sector</i> is the smallest unit. Disk controllers can introduce address translations to map logical to physical positions, e.g., <a href="/wiki/Zone_bit_recording" title="Zone bit recording">zone bit recording</a> stores fewer sectors in shorter (inner) tracks, physical disk formats are not necessarily cylindrical, and sector numbers in a track can be skewed.</p>
<h3><span class="mw-headline" id="Sectors">Sectors</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Cylinder-head-sector&amp;action=edit&amp;section=2" title="Edit section: Sectors">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Floppy disks and controllers use physical sector sizes of 128, 256, 512 and 1024 bytes (e.g., PC/AX), whereby formats with 512 bytes per physical sector became dominant in the 1980s.<sup class="reference" id="cite_ref-Ecma-107_1-0"><a href="#cite_note-Ecma-107-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>The most common physical sector size for harddisks today is 512 bytes, but there have been hard disks with 520 bytes per sector as well for non-IBM compatible machines. In 2005 some <a href="/wiki/Seagate_Technology" title="Seagate Technology">Seagate</a> custom hard disks used sector sizes of 1024 bytes per sector. <a href="/wiki/Advanced_Format" title="Advanced Format">Advanced Format</a> hard disks use 4096 bytes per physical sector (<a class="mw-redirect" href="/wiki/4Kn" title="4Kn">4Kn</a>)<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> since 2010, but will also be able to emulate 512 byte sectors (<a class="mw-redirect" href="/wiki/512e" title="512e">512e</a>) for a transitional period.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p><a href="/wiki/Magneto-optical_drive" title="Magneto-optical drive">Magneto-optical drives</a> use sector sizes of 512 and 1024 bytes on 5.25-inch drives and 512 and 2048 bytes on 3.5-inch drives.</p>
<p>In <code><abbr title="Cylinder">C</abbr><abbr title="Head">H</abbr><abbr title="Sector">S</abbr></code> addressing the <i>sector</i> numbers always start at <b>1</b>, there is no <i>sector 0</i><sup class="reference plainlinks nourlexpansion" id="ref_a"><a class="external autonumber" href="//en.wikipedia.org/wiki/Cylinder-head-sector#endnote_a">[1]</a></sup>, which can lead to confusion since logical sector addressing schemes (e.g., with <a href="/wiki/Logical_block_addressing" title="Logical block addressing">Logical block addressing</a> (LBA), or with "relative sector addressing" in DOS) typically start counting with 0.</p>
<p>For physical disk geometries the maximal sector number is determined by the <i>low level format</i> of the disk. However, for disk access with the <a href="/wiki/BIOS" title="BIOS">BIOS</a> of IBM-PC compatible machines, the sector number was encoded in six bits, resulting in a maximal number of 111111 (63) sectors per track. This maximum is still in use for virtual CHS geometries.</p>
<h3><span class="mw-headline" id="Tracks">Tracks</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Cylinder-head-sector&amp;action=edit&amp;section=3" title="Edit section: Tracks">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/Track_(disk_drive)" title="Track (disk drive)">tracks</a> are the thin <a href="/wiki/Concentric" title="Concentric">concentric</a> circular strips of sectors. At least one head is required to read a single track. With respect to disk geometries the terms <i>track</i> and <i>cylinder</i> are closely related. For a single or double sided <a href="/wiki/Floppy_disk" title="Floppy disk">floppy disk</a> <i>track</i> is the common term; and for more than two heads <i>cylinder</i> is the common term. Strictly speaking a <i>track</i> is a given <code><abbr title="Cylinder">C</abbr><abbr title="Head">H</abbr></code> combination consisting of <code><abbr title="Sectors Per Track">SPT</abbr></code> sectors, while a <i>cylinder</i> consists of <code>SPT×<abbr title="Heads">H</abbr></code> sectors.</p>
<h3><span class="mw-headline" id="Cylinders">Cylinders</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Cylinder-head-sector&amp;action=edit&amp;section=4" title="Edit section: Cylinders">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A cylinder is a division of data in a <a href="/wiki/Disk_storage" title="Disk storage">disk drive</a>, as used in the CHS addressing mode of a <a class="mw-redirect" href="/wiki/Fixed_Block_Architecture" title="Fixed Block Architecture">Fixed Block Architecture</a> disk or the cylinder–head–record (CCHHR) addressing mode of a <a class="mw-redirect" href="/wiki/CKD_disk" title="CKD disk">CKD disk</a>.</p>
<p>The concept is concentric, hollow, <a class="mw-redirect" href="/wiki/Cylinder_(geometry)" title="Cylinder (geometry)">cylindrical</a> slices through the physical disks (<a class="mw-redirect" href="/wiki/Hard_disk_platter" title="Hard disk platter">platters</a>), collecting the respective circular tracks aligned through the stack of platters. The number of cylinders of a disk drive exactly equals the number of tracks on a single surface in the drive. It comprises the same track number on each platter, spanning all such tracks across each platter surface that is able to store data (without regard to whether or not the track is "bad"). Cylinders are vertically formed by <a href="/wiki/Track_(disk_drive)" title="Track (disk drive)">tracks</a>. In other words, track 12 on platter 0 plus track 12 on platter 1 etc. is cylinder 12.</p>
<p>Other forms of <a class="mw-redirect" href="/wiki/Direct_access_storage_device" title="Direct access storage device">Direct Access Storage Device</a> (DASD), such as <a href="/wiki/Drum_memory" title="Drum memory">drum memory</a> devices or the <a href="/wiki/IBM_2321_Data_Cell" title="IBM 2321 Data Cell">IBM 2321 Data Cell</a>, might give blocks addresses that include a cylinder address, although the cylinder address doesn't select a (geometric) cylindrical slice of the device.</p>
<h3><span class="mw-headline" id="Heads">Heads</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Cylinder-head-sector&amp;action=edit&amp;section=5" title="Edit section: Heads">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A device called a <a href="/wiki/Disk_read-and-write_head" title="Disk read-and-write head">head</a> reads and writes data in a hard drive by manipulating the magnetic medium that composes the surface of an associated disk platter. Naturally, a platter has 2 sides and thus 2 surfaces on which data can be manipulated; usually there are 2 heads per platter, one per side. (Sometimes the term <i>side</i> is substituted for <i>head,</i> since platters might be separated from their head assemblies, as with the removable media of a <i>floppy</i> drive.)</p>
<p>The <code><abbr title="Cylinder">C</abbr><abbr title="Head">H</abbr><abbr title="Sector">S</abbr></code> addressing supported in IBM-PC compatible <a href="/wiki/BIOS" title="BIOS">BIOSes</a> code used eight bits for - theoretically up to 256 heads counted as head <b>0</b> up to <b>255</b> (<code>FFh</code>). However, a bug in all versions of <a href="/wiki/MS-DOS" title="MS-DOS">Microsoft DOS</a>/<a href="/wiki/IBM_PC_DOS" title="IBM PC DOS">IBM PC DOS</a> up to and including 7.10 will cause these operating systems to crash on boot when encountering volumes with 256 heads<sup class="reference plainlinks nourlexpansion" id="ref_b"><a class="external autonumber" href="//en.wikipedia.org/wiki/Cylinder-head-sector#endnote_b">[2]</a></sup>. Therefore, all compatible BIOSes will use mappings with up to 255 heads (<code>00h..FEh</code>) only, including in virtual <tt>255×63</tt> geometries.</p>
<p>This historical oddity can affect the maximum disk size in old BIOS <a href="/wiki/INT_13H" title="INT 13H">INT 13h</a> code as well as old <a class="mw-redirect" href="/wiki/PC_DOS" title="PC DOS">PC DOS</a> or similar operating systems:</p>
<p><tt>(512 bytes/sector)×(63 sectors/track)×(255 heads (tracks/cylinder))×(1024 cylinders)=8032.5</tt> <a href="/wiki/Mebibyte" title="Mebibyte">MiB</a>, but actually <tt>512×63×256×1024=8064</tt> <a href="/wiki/Mebibyte" title="Mebibyte">MiB</a> yields what is known as <b>8 <a href="/wiki/Gibibyte" title="Gibibyte">GiB</a></b> limit.<sup class="reference" id="cite_ref-eide_5-0"><a href="#cite_note-eide-5">[5]</a></sup> In this context relevant definition of 8 <a href="/wiki/Gibibyte" title="Gibibyte">GiB</a> = 8192 <a href="/wiki/Mebibyte" title="Mebibyte">MiB</a> is another incorrect limit, because it would require CHS <tt>512×64×256</tt> with 64 sectors per track.</p>
<p><i>Tracks</i> and <i>cylinders</i> are counted from 0, i.e., track 0 is the first (outer-most) track on <a href="/wiki/Floppy_disk" title="Floppy disk">floppy</a> or other cylindrical disks. Old <a href="/wiki/BIOS" title="BIOS">BIOS</a> code supported ten bits in CHS addressing with up to 1024 cylinders (<tt>1024=2<sup>10</sup></tt>). Adding six bits for <a href="#Sectors">sectors</a> and eight bits for <a href="#Heads">heads</a> results in the 24 bits supported by <a href="/wiki/INT_13H" title="INT 13H">BIOS interrupt 13h</a>. Subtracting the disallowed sector number 0 in <tt>1024×256</tt> tracks corresponds to 128 <a href="/wiki/Mebibyte" title="Mebibyte">MiB</a> for a sector size of 512 bytes (<tt>128 <a href="/wiki/Mebibyte" title="Mebibyte">MiB</a>=1024×256×(512 byte/sector)</tt>); and <tt>8192-128=8064</tt> confirms the (roughly) <b>8 <a href="/wiki/Gibibyte" title="Gibibyte">GiB</a></b> limit.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>CHS addressing starts at <code>0/0/1</code> with a maximal value <code>1023/255/63</code> for <tt>24=10+8+6</tt> bits, or <code>1023/254/63</code> for 24 bits limited to 255 <a href="#Heads">heads</a>. CHS values used to specify the geometry of a disk have to count cylinder 0 and head 0 resulting in a maximum (<code>1024/256/63</code> or) <code>1024/255/63</code> for 24 bits with (256 or) 255 heads. In CHS tuples specifying a geometry S actually means sectors per track, and where the (virtual) geometry still matches the capacity the disk contains <code>C×H×S</code> sectors. As larger hard disks have come into use, a cylinder has become also a logical disk structure, standardised<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (July 2011)">citation needed</span></a></i>]</sup> at 16 065 sectors (<tt>16065=255×63</tt>).</p>
<p>CHS addressing with 28 bits (<a href="/wiki/Parallel_ATA#EIDE_and_ATA-2" title="Parallel ATA">EIDE and ATA-2</a>) permits eight bits for sectors still starting at 1, i.e., sectors 1…255, four bits for heads 0…15, and sixteen bits for cylinders 0…65535.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> This results in a roughly <b>128 <a href="/wiki/Gibibyte" title="Gibibyte">GiB</a></b> limit; actually <tt>65536×16×255=267386880</tt> sectors corresponding to 130560 <a href="/wiki/Mebibyte" title="Mebibyte">MiB</a> for a sector size of 512 bytes.<sup class="reference" id="cite_ref-eide_5-1"><a href="#cite_note-eide-5">[5]</a></sup> The <tt>28=16+4+8</tt> bits in the <a href="/wiki/Parallel_ATA#EIDE_and_ATA-2" title="Parallel ATA">ATA-2</a> specification are also covered by <a href="/wiki/Ralf_Brown%27s_Interrupt_List" title="Ralf Brown's Interrupt List">Ralf Brown's Interrupt List</a>, and an old working draft of this now expired standard was published.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<p>With an old <a href="/wiki/BIOS" title="BIOS">BIOS</a> limit of 1024 cylinders and the <a href="/wiki/Parallel_ATA" title="Parallel ATA">ATA</a> limit of 16 heads<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> the combined effect was <tt>1024×16×63=1032192</tt> sectors, i.e., a <b>504 <a href="/wiki/Mebibyte" title="Mebibyte">MiB</a></b> limit for sector size 512. <a href="/wiki/BIOS" title="BIOS">BIOS</a> translation schemes known as <abbr title="Extended CHS">ECHS</abbr> and <i>revised ECHS</i> mitigated this limitation by using 128 or 240 instead of 16 heads, simultaneously reducing the numbers of cylinders and sectors to fit into <code>1024/128/63</code> (ECHS limit: 4032 <a href="/wiki/Mebibyte" title="Mebibyte">MiB</a>) or <code>1024/240/63</code> (revised ECHS limit: 7560 <a href="/wiki/Mebibyte" title="Mebibyte">MiB</a>) for the given total number of sectors on a disk.<sup class="reference" id="cite_ref-eide_5-2"><a href="#cite_note-eide-5">[5]</a></sup></p>
<h3><span class="mw-headline" id="Blocks_and_clusters">Blocks and clusters</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Cylinder-head-sector&amp;action=edit&amp;section=6" title="Edit section: Blocks and clusters">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/Unix" title="Unix">Unix</a> communities employ the term <i>block</i> to refer to a sector or group of sectors. For example, the Linux <a href="/wiki/Fdisk" title="Fdisk">fdisk</a> utility, before version 2.25,<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> displayed partition sizes using 1024-byte <i>blocks</i>.</p>
<p><i>Clusters</i> are allocation units for data on various file systems (<a href="/wiki/File_Allocation_Table" title="File Allocation Table">FAT</a>, <a href="/wiki/NTFS" title="NTFS">NTFS</a>, etc.), where <i>data</i> mainly consists of files. <i>Clusters</i> are not directly affected by the physical or virtual geometry of the disk, i.e., a cluster can begin at a sector near the end of a given <code><abbr title="Cylinder">C</abbr><abbr title="Head">H</abbr></code> track, and end in a sector on the physically or logically next <code><abbr title="Cylinder">C</abbr><abbr title="Head">H</abbr></code> <a href="#Tracks">track</a>.</p>
<h2><span class="mw-headline" id="CHS_to_LBA_mapping">CHS to LBA mapping</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Cylinder-head-sector&amp;action=edit&amp;section=7" title="Edit section: CHS to LBA mapping">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a class="mw-redirect" href="/wiki/Logical_Block_Addressing#CHS_conversion" title="Logical Block Addressing">Logical Block Addressing § CHS conversion</a></div>
<p>In 2002 the <a href="/wiki/Parallel_ATA" title="Parallel ATA">ATA-6</a> specification introduced an optional 48 bits <a class="mw-redirect" href="/wiki/Logical_Block_Addressing" title="Logical Block Addressing">Logical Block Addressing</a> and declared CHS addressing as obsolete, but still allowed to implement the ATA-5 translations.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> Unsurprisingly the CHS to LBA translation formula given below also matches the last ATA-5 CHS translation. In the ATA-5 specification CHS support was mandatory for up to 16 514 064 sectors and optional for larger disks. The ATA-5 limit corresponds to CHS <code>16383 16 63</code> or equivalent disk capacities (16514064 = 16383×16×63 = 1032×254×63), and requires 24 = 14+4+6 bits (16383 + 1 = 2<sup>14</sup>).<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<p>CHS <a href="/wiki/Tuple" title="Tuple">tuples</a> can be mapped onto LBA addresses using the following formula:</p>
<dl>
<dd><i>A</i> = (<i>c</i> ⋅ <i>N</i><sub>heads</sub> + <i>h</i>) ⋅ <i>N</i><sub>sectors</sub> + (<i>s</i> − 1),</dd>
</dl>
<p>where <i>A</i> is the LBA address, <i>N</i><sub>heads</sub> is the number of heads on the disk, <i>N</i><sub>sectors</sub> is the maximum number of sectors per track, and (<i>c</i>, <i>h</i>, <i>s</i>) is the CHS address.</p>
<p>A <i>Logical Sector Number<small> </small></i> formula in the <a href="/wiki/Ecma_International" title="Ecma International">ECMA</a>-107<sup class="reference" id="cite_ref-Ecma-107_1-1"><a href="#cite_note-Ecma-107-1">[1]</a></sup> and <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a>/<a href="/wiki/International_Electrotechnical_Commission" title="International Electrotechnical Commission">IEC</a> 9293:1994<sup class="reference" id="cite_ref-ISO_9293_1994_13-0"><a href="#cite_note-ISO_9293_1994-13">[13]</a></sup> (superseding ISO 9293:1987<sup class="reference" id="cite_ref-ISO_9293_1987_14-0"><a href="#cite_note-ISO_9293_1987-14">[14]</a></sup>) standards for <a href="/wiki/File_Allocation_Table" title="File Allocation Table">FAT</a> file systems matches exactly the LBA formula given above: <i>Logical Block Address</i> and <i>Logical Sector Number</i> (LSN) are synonyms.<sup class="reference" id="cite_ref-Ecma-107_1-2"><a href="#cite_note-Ecma-107-1">[1]</a></sup><sup class="reference" id="cite_ref-ISO_9293_1994_13-1"><a href="#cite_note-ISO_9293_1994-13">[13]</a></sup><sup class="reference" id="cite_ref-ISO_9293_1987_14-1"><a href="#cite_note-ISO_9293_1987-14">[14]</a></sup> The formula does not use the number of cylinders, but requires the number of heads and the number of sectors per track in the disk geometry, because the same CHS tuple addresses different logical sector numbers depending on the geometry. <b>Examples</b>:</p>
<dl>
<dd>For geometry <code>1020 16 63</code> of a disk with 1028160 sectors, CHS <code>3 2 1</code> is LBA <tt> 3150=(3× 16+2)× 63 + (1-1)</tt></dd>
<dd>For geometry <code>1008 4 255</code> of a disk with 1028160 sectors, CHS <code>3 2 1</code> is LBA <tt> 3570=(3×  4+2)×255 + (1-1)</tt></dd>
<dd>For geometry <code> 64 255 63</code> of a disk with 1028160 sectors, CHS <code>3 2 1</code> is LBA <tt>48321=(3×255+2)× 63 + (1-1)</tt></dd>
<dd>For geometry <code>2142 15 32</code> of a disk with 1028160 sectors, CHS <code>3 2 1</code> is LBA <tt> 1504=(3× 15+2)× 32 + (1-1)</tt></dd>
</dl>
<p>To help visualize the sequencing of sectors into a linear LBA model, note that:</p>
<dl>
<dd>The first LBA sector is sector # zero, the same sector in a CHS model is called sector # one.</dd>
<dd>All the sectors of each head/track get counted before incrementing to the next head/track.</dd>
<dd>All the heads/tracks of the same cylinder get counted before incrementing to the next cylinder.</dd>
<dd>The outside half of a whole Hard Drive would be the first half of the drive.</dd>
</dl>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Cylinder-head-sector&amp;action=edit&amp;section=8" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Cylinder Head Record format has been used by <a class="mw-redirect" href="/wiki/Count_Key_Data" title="Count Key Data">Count Key Data</a> (CKD) hard disks on <a class="mw-redirect" href="/wiki/IBM_mainframes" title="IBM mainframes">IBM mainframes</a> since at least the 1960s. This is largely comparable to the Cylinder Head Sector format used by PCs, save that the sector size was not fixed but could vary from track to track based on the needs of each application. In contemporary use, the disk geometry presented to the mainframe is emulated by the storage firmware, and no longer has any relation to physical disk geometry.</p>
<p>Earlier hard drives used in the PC, such as <a href="/wiki/Modified_Frequency_Modulation" title="Modified Frequency Modulation">MFM</a> and <a class="mw-redirect" href="/wiki/Run_Length_Limited" title="Run Length Limited">RLL</a> drives, divided each cylinder into an equal number of sectors, so the CHS values matched the physical properties of the drive. A drive with a CHS tuple of <code>500 4 32</code> would have 500 tracks per side on each platter, two platters (4 heads), and 32 sectors per track, with a total of 32 768 000 <a class="mw-redirect" href="/wiki/Bytes" title="Bytes">bytes</a> (31.25 <a href="/wiki/Mebibyte" title="Mebibyte">MiB</a>).</p>
<p><a class="mw-redirect" href="/wiki/Advanced_Technology_Attachment" title="Advanced Technology Attachment">ATA/IDE</a> drives were much more efficient at storing data and have replaced the now <i>archaic</i> MFM and RLL drives. They use <a href="/wiki/Zone_bit_recording" title="Zone bit recording">zone bit recording</a> (ZBR), where the number of sectors dividing each track varies with the location of groups of tracks on the surface of the platter. Tracks nearer to the edge of the platter contain more blocks of data than tracks close to the spindle, because there is more physical space within a given track near the edge of the platter. Thus, the CHS addressing scheme cannot correspond directly with the physical geometry of such drives, due to the varying number of sectors per track for different regions on a platter. Because of this, many drives still have a surplus of sectors (less than 1 cylinder in size) at the end of the drive, since the total number of sectors rarely, if ever, ends on a cylinder boundary.</p>
<p>An ATA/IDE drive can be set in the system <a href="/wiki/BIOS" title="BIOS">BIOS</a> with any configuration of cylinders, heads and sectors that do not exceed the capacity of the drive (or the BIOS), since the drive will convert any given CHS value into an actual address for its specific hardware configuration. This however can cause compatibility problems.</p>
<p>For operating systems such as <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> <a href="/wiki/DOS" title="DOS">DOS</a> or older version of <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a>, each partition must start and end at a cylinder boundary. Only some of the most modern <a href="/wiki/Operating_system" title="Operating system">operating systems</a> (Windows XP included) may disregard this rule, but doing so can still cause some compatibility issues, especially if the user wants to perform <a class="mw-redirect" href="/wiki/Dual_boot" title="Dual boot">dual booting</a> on the same drive. Microsoft does not follow this rule with internal disk partition tools since Windows Vista.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup></p>

<h2><span class="mw-headline" id="Notes">Notes</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Cylinder-head-sector&amp;action=edit&amp;section=11" title="Edit section: Notes">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<dl>
<dd>1.<span class="citation wikicite" id="endnote_a"><b><a href="#ref_a">^</a></b></span> This rule is true at least for all formats where the physical sectors are named 1 upwards. However, there are a few odd floppy formats (e.g., the 640 <a href="/wiki/Kibibyte" title="Kibibyte">KiB</a> format used by BBC Master 512 with DOS Plus 2.1), where the first sector in a track is named "0" not "1".</dd>
<dd>2.<span class="citation wikicite" id="endnote_b"><b><a href="#ref_b">^</a></b></span> While computers begin counting at 0, DOS would begin counting at 1. In order to do this, DOS would add a 1 to the head count before displaying it on the screen. However, instead of converting the 8-bit unsigned integer to a larger size (such as a 16-bit integer) first, DOS just added the 1. This would <a href="/wiki/Integer_overflow" title="Integer overflow">overflow</a> a head count of 255 (<code>0xFF</code>) into 0 (<code>0x100 &amp; 0xFF = 0x00</code>) instead of the 256 that would be expected. This was fixed with DOS 8, but by then, it had become a <a href="/wiki/De_facto" title="De facto">de facto</a> standard to not use a head value of 255.</dd>
</dl>
