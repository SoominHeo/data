<p>The <b>real-time web</b> is a network web using technologies and practices that enable users to receive information as soon as it is published by its authors, rather than requiring that they or their software check a source periodically for updates.</p>

<h2><span class="mw-headline" id="Difference_from_real-time_computing">Difference from real-time computing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Real-time_web&amp;action=edit&amp;section=1" title="Edit section: Difference from real-time computing">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The real-time web is fundamentally different from <a href="/wiki/Real-time_computing" title="Real-time computing">real-time computing</a> since there is no knowing when, or if, a response will be received. The information types transmitted this way are often short messages, status updates, news alerts, or links to longer documents. The content is often "soft" in that it is based on the <a href="/wiki/Social_web" title="Social web">social web</a>—people's opinions, attitudes, thoughts, and interests—as opposed to hard news or facts.</p>
<h2><span class="mw-headline" id=".28Old.29_True-realtime_web_.28an_.22alternate.22_model.29">(Old) True-realtime web (an "alternate" model)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Real-time_web&amp;action=edit&amp;section=2" title='Edit section: (Old) True-realtime web (an "alternate" model)'>edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>From another point of view, the real-time web consists in making the client interface (or the web side; or the web layer) of a web application, to communicate continuously with the corresponding real-time server, during every user connection. As a fast pic of the client/server model, imagine each client object (each web module of the web <a class="mw-redirect" href="/wiki/GUI" title="GUI">GUI</a> of an application) having its object class alive as a sub process (of its user session) in the server environment. In this scenario, the web is considered as the human entrance (interface) to the real-time environment: at each connected web URL, or Internet real-time zone, corresponds a different "front-end" web application. The real-time server acts as a <a class="new" href="/w/index.php?title=Logic_network_operating_system&amp;action=edit&amp;redlink=1" title="Logic network operating system (page does not exist)">logic network operating system</a> for the programmable array of applications; handles the array of connected users for each application; attends for connections from real-world appliances and second level real-time servers. Applications behaviours and the intercommunication procedures between online services or applications, online users, and connected devices or appliances, are settled in the corresponding source code of each real-time service written in the real-time-interpreted programming language of the centric server.</p>
<p>As opposite to previous scenario, real-time web is exactly soft <a href="/wiki/Real-time_computing" title="Real-time computing">real-time computing</a>: the round trip of a data ping-pong signal from the real-time server to the client must take about 1s (max) to be considered real-time and not to be annoying for humans (or users) during their connections.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2016)">citation needed</span></a></i>]</sup> About the dispute between social web and real-time web, we can say real-time web is social by default and it is not true the contrary (WEB-r comes before Web 2.0). The WEB-r model is called <a class="new" href="/w/index.php?title=True-realtime_web&amp;action=edit&amp;redlink=1" title="True-realtime web (page does not exist)">true-realtime web</a> to highlight the differences with the defective (de facto) model of real-time web generally perceived. From the industry point of view, this model of (general) real-time Internet can also be defined as <a class="new" href="/w/index.php?title=Electronic_web&amp;action=edit&amp;redlink=1" title="Electronic web (page does not exist)">electronic web</a>, that comes with the intrinsic meaning of not being limited to the web side of the Net, and with the direct reference to its server/rest-of-the-world perspective as a mechanism of a single clock.</p>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Real-time_web&amp;action=edit&amp;section=3" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Examples of real-time web are Facebook's newsfeed, and Twitter, implemented in social networking, search, and news sites. Benefits are said to include increased user engagement ("flow") and decreased server loads. In December 2009 real-time search facilities were added to <a href="/wiki/Google_Search" title="Google Search">Google Search</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p>The absolutely first realtime web implementation worldwide have been the WIMS true-realtime server and its web apps in 2001-2011 (WIMS = Web Interactive Management System); based on the WEB-r model of above; built in Java (serverside) and Adobe Flash (clientside). The true-realtime web model was born in 2000 at mc2labs.net by an Italian independent researcher.</p>
<h2><span class="mw-headline" id="Real-time_search">Real-time search</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Real-time_web&amp;action=edit&amp;section=4" title="Edit section: Real-time search">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A problem created by the rapid pace and huge volume of information created by real-time web technologies and practices is finding relevant information. One approach, known as <b>real-time search</b>, is the concept of searching for and finding information online as it is produced. Advancements in web search technology coupled with growing use of <a href="/wiki/Social_media" title="Social media">social media</a> enable online activities to be queried as they occur. A traditional <a class="mw-redirect" href="/wiki/Web_search" title="Web search">web search</a> <a href="/wiki/Web_crawler" title="Web crawler">crawls</a> and <a class="mw-redirect" href="/wiki/Index_(search_engine)" title="Index (search engine)">indexes</a> web pages periodically, returning results based on relevance to the search query. <a href="/wiki/Google_Real-Time_Search" title="Google Real-Time Search">Google Real-Time Search</a> was available in <a href="/wiki/Google_Search" title="Google Search">Google Search</a> until July 2011.</p>
