<p><b>Ogg</b> is a <a class="mw-redirect" href="/wiki/Free_file_format" title="Free file format">free</a>, <a href="/wiki/Open_format" title="Open format">open</a> <a class="mw-redirect" href="/wiki/Container_format_(digital)" title="Container format (digital)">container format</a> maintained by the <a href="/wiki/Xiph.Org_Foundation" title="Xiph.Org Foundation">Xiph.Org Foundation</a>. The creators of the Ogg format state that it is unrestricted by <a href="/wiki/Software_patent" title="Software patent">software patents</a><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> and is designed to provide for efficient <a href="/wiki/Streaming_media" title="Streaming media">streaming</a> and manipulation of high quality <a class="mw-redirect" href="/wiki/Digital_multimedia" title="Digital multimedia">digital multimedia</a>. Its name is derived from "ogging", <a href="/wiki/Jargon" title="Jargon">jargon</a> from the computer game <i><a href="/wiki/Netrek" title="Netrek">Netrek</a></i>.<sup class="reference" id="cite_ref-ogging_3-0"><a href="#cite_note-ogging-3">[3]</a></sup></p>
<p>The Ogg container format can <a href="/wiki/Time-division_multiplexing" title="Time-division multiplexing">multiplex</a> a number of independent <a href="/wiki/Stream_(computing)" title="Stream (computing)">streams</a> for <a href="/wiki/Sound" title="Sound">audio</a>, <a href="/wiki/Video" title="Video">video</a>, text (such as <a class="mw-redirect" href="/wiki/Subtitles" title="Subtitles">subtitles</a>), and <a href="/wiki/Metadata" title="Metadata">metadata</a>.</p>
<p>In the Ogg multimedia framework, <a href="/wiki/Theora" title="Theora">Theora</a> provides a <a class="mw-redirect" href="/wiki/Lossy" title="Lossy">lossy</a> video layer. The <a href="/wiki/Audio_coding_format" title="Audio coding format">audio layer</a> is most commonly provided by the music-oriented <a href="/wiki/Vorbis" title="Vorbis">Vorbis</a> <a href="/wiki/Bitstream_format" title="Bitstream format">format</a> or its successor <a href="/wiki/Opus_(audio_format)" title="Opus (audio format)">Opus</a>. <a class="mw-redirect" href="/wiki/Lossless" title="Lossless">Lossless</a> audio compression formats include <a href="/wiki/FLAC" title="FLAC">FLAC</a>, and <a class="mw-redirect" href="/wiki/OggPCM" title="OggPCM">OggPCM</a>.</p>
<p>Before 2007, the .ogg <a href="/wiki/Filename_extension" title="Filename extension">filename extension</a> was used for all files whose content used the Ogg container format. Since 2007, the Xiph.Org Foundation recommends that .ogg only be used for Ogg Vorbis audio files. The Xiph.Org Foundation decided to create a new set of file extensions and media types to describe different types of content such as .oga for audio only files, .ogv for video with or without sound (including Theora), and .ogx for multiplexed Ogg.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p>As of August 4, 2011, the current version of the Xiph.Org Foundation's reference implementation, is libogg 1.3.0.<sup class="reference" id="cite_ref-release-1.3.0_5-0"><a href="#cite_note-release-1.3.0-5">[5]</a></sup> Another version, libogg2, has been in development, but is awaiting a rewrite as of 2008.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> Both <a href="/wiki/Library_(computing)" title="Library (computing)">software libraries</a> are <a href="/wiki/Free_software" title="Free software">free software</a>, released under the <a class="mw-redirect" href="/wiki/New_BSD_License" title="New BSD License">New BSD License</a>. Ogg reference implementation was separated from Vorbis on September 2, 2000.<sup class="reference" id="cite_ref-ogg-separated_7-0"><a href="#cite_note-ogg-separated-7">[7]</a></sup></p>
<p>Because the format is free, and its reference implementation is not subject to restrictions associated with <a href="/wiki/Copyright" title="Copyright">copyright</a>, Ogg's various codecs have been incorporated into a number of different free and <a href="/wiki/Proprietary_software" title="Proprietary software">proprietary</a> <a class="mw-redirect" href="/wiki/Media_player_(application_software)" title="Media player (application software)">media players</a>, both commercial and non-commercial, as well as <a href="/wiki/Portable_media_player" title="Portable media player">portable media players</a> and <a class="mw-redirect" href="/wiki/GPS" title="GPS">GPS</a> receivers from different manufacturers.</p>

<h2><span class="mw-headline" id="Naming">Naming</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ogg&amp;action=edit&amp;section=1" title="Edit section: Naming">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>It is sometimes assumed that the name "Ogg" comes from the character of <a href="/wiki/Nanny_Ogg" title="Nanny Ogg">Nanny Ogg</a> in <a href="/wiki/Terry_Pratchett" title="Terry Pratchett">Terry Pratchett</a>'s <i><a href="/wiki/Discworld" title="Discworld">Discworld</a></i> novels, but the Ogg developers say that is not true.<sup class="reference" id="cite_ref-xiphname_8-0"><a href="#cite_note-xiphname-8">[8]</a></sup> Ogg is derived from "ogging", <a href="/wiki/Jargon" title="Jargon">jargon</a> from the computer game <i><a href="/wiki/Netrek" title="Netrek">Netrek</a></i>, which came to mean doing something forcefully, possibly without consideration of the drain on future resources.<sup class="reference" id="cite_ref-ogging_3-1"><a href="#cite_note-ogging-3">[3]</a></sup> At its inception, the Ogg project was thought to be somewhat ambitious given the power of the <a href="/wiki/IBM_PC_compatible" title="IBM PC compatible">PC</a> hardware of the time.<sup class="reference" id="cite_ref-xiphname_8-1"><a href="#cite_note-xiphname-8">[8]</a></sup> Still, to quote the same reference: "Vorbis, on the other hand is named after the Terry Pratchett <a href="/wiki/Religions_of_the_Discworld#Vorbis" title="Religions of the Discworld">character</a> from the book <i><a href="/wiki/Small_Gods" title="Small Gods">Small Gods</a></i>".</p>
<p>The <a href="/wiki/Vorbis" title="Vorbis">Ogg Vorbis</a> project started in 1993. It was originally named "Squish" but that name was already trademarked, so the project underwent a name change. The new name, "OggSquish", was used until 2001 when it was changed again to "Ogg". Ogg has since come to refer to the <a class="mw-redirect" href="/wiki/Container_format_(digital)" title="Container format (digital)">container format</a>, which is now part of the larger Xiph.org multimedia project. Today, "Squish" (now known as "Vorbis") refers to a particular <a href="/wiki/Audio_coding_format" title="Audio coding format">audio coding format</a> typically used with the Ogg container format.<sup class="reference" id="cite_ref-xiphname-1_9-0"><a href="#cite_note-xiphname-1-9">[9]</a></sup></p>
<h2><span class="mw-headline" id="File_format">File format</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ogg&amp;action=edit&amp;section=2" title="Edit section: File format">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The "Ogg" <a href="/wiki/Bitstream_format" title="Bitstream format">bitstream format</a>, spearheaded by the Xiph.Org Foundation, has been created as the framework of a larger initiative aimed at developing a set of components for the coding and decoding of multimedia content, which are available free of charge and freely re-implementable in software.</p>
<p>The format consists of chunks of data each called an "<a href="/wiki/Ogg_page" title="Ogg page">Ogg page</a>". Each page begins with the characters, "OggS", to identify the file as Ogg format.</p>
<p>A "serial number" and "page number" in the page header identifies each page as part of a series of pages making up a <a href="/wiki/Bitstream" title="Bitstream">bitstream</a>. Multiple bitstreams may be <a href="/wiki/Multiplexing" title="Multiplexing">multiplexed</a> in the file where pages from each bitstream are ordered by the seek time of the contained data. Bitstreams may also be appended to existing files, a process known as "chaining", to cause the bitstreams to be decoded in sequence.</p>
<p>A <a href="/wiki/BSD_licenses" title="BSD licenses">BSD-licensed</a> library, called "libvorbis", is available to encode and decode data from "Vorbis" streams. Independent Ogg implementations are used in several projects such as <a href="/wiki/RealPlayer" title="RealPlayer">RealPlayer</a> and a set of <a href="/wiki/DirectShow" title="DirectShow">DirectShow</a> filters.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></p>
<p>Mogg, the "Multi-Track-Single-Logical-Stream Ogg-Vorbis", is the multi-channel or multi-track Ogg file format.</p>
<h2><span class="mw-headline" id="Page_structure">Page structure</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ogg&amp;action=edit&amp;section=3" title="Edit section: Page structure">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/Ogg_page" title="Ogg page">Ogg page</a></div>
<p>The following is the field layout of an Ogg page header:</p>
<div class="center">
<div class="floatnone"><a class="image" href="/wiki/File:Ogg_page_header_structure_(en).svg" title="The field layout of an Ogg page header"><img alt="The field layout of an Ogg page header" data-file-height="315" data-file-width="655" height="246" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ee/Ogg_page_header_structure_%28en%29.svg/512px-Ogg_page_header_structure_%28en%29.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ee/Ogg_page_header_structure_%28en%29.svg/768px-Ogg_page_header_structure_%28en%29.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ee/Ogg_page_header_structure_%28en%29.svg/1024px-Ogg_page_header_structure_%28en%29.svg.png 2x" width="512"/></a></div>
</div>
<dl>
<dt>Capture pattern – 32 bits</dt>
<dd>The capture pattern or sync code is a <a href="/wiki/Magic_number_(programming)" title="Magic number (programming)">magic number</a> used to ensure synchronization when parsing Ogg files. Every page starts with the four ASCII character sequence, "OggS". This assists in resynchronizing a parser in cases where data has been lost or is corrupted, and is a sanity check before commencing parsing the page structure.</dd>
</dl>
<dl>
<dt>Version – 8 bits</dt>
<dd>This field indicates the version of the Ogg bitstream format, to allow for future expansion. It is currently mandated to be 0.</dd>
</dl>
<dl>
<dt>Header type – 8 bits</dt>
<dd>This is an 8 bit field of flags, which indicates the type of page that follows.</dd>
</dl>
<dl>
<dd>
<table class="wikitable">
<tr>
<th style="width:5%;">Bit</th>
<th style="width:10%;">Value</th>
<th style="width:10%;">Flag</th>
<th style="width:75%;">Page type</th>
</tr>
<tr>
<td>0</td>
<td>0x01</td>
<td>Continuation</td>
<td>The first packet on this page is a continuation of the previous packet in the logical bitstream.</td>
</tr>
<tr>
<td>1</td>
<td>0x02</td>
<td>BOS</td>
<td>Beginning Of Stream. This page is the first page in the logical bitstream. The BOS flag must be set on the first page of every logical bitstream, and must not be set on any other page.</td>
</tr>
<tr>
<td>2</td>
<td>0x04</td>
<td>EOS</td>
<td>End Of Stream. This page is the last page in the logical bitstream. The EOS flag must be set on the final page of every logical bitstream, and must not be set on any other page.</td>
</tr>
</table>
</dd>
</dl>
<dl>
<dt>Granule position – 64 bits</dt>
<dd>A granule position is the time marker in Ogg files. It is an abstract value, whose meaning is determined by the codec. It may, for example, be a count of the number of samples, the number of frames or a more complex scheme.</dd>
</dl>
<dl>
<dt>Bitstream serial number – 32 bits</dt>
<dd>This field is a serial number that identifies a page as belonging to a particular logical bitstream. Each logical bitstream in a file has a unique value, and this field allows implementations to deliver the pages to the appropriate decoder. In a typical Vorbis and Theora file, one stream is the audio (Vorbis), and the other is the video (Theora)</dd>
</dl>
<dl>
<dt>Page sequence number – 32 bits</dt>
<dd>This field is a monotonically increasing field for each logical bitstream. The first page is 0, the second 1, etc. This allows implementations to detect when data has been lost.</dd>
</dl>
<dl>
<dt><a href="/wiki/Checksum" title="Checksum">Checksum</a> – 32 bits</dt>
<dd>This field provides a <a class="mw-redirect" href="/wiki/CRC32" title="CRC32">CRC32</a> checksum of the data in the entire page (including the page header, calculated with the checksum field set to 0). This allows verification that the data has not been corrupted since it was created. Pages that fail the checksum should be discarded. The checksum is generated using a polynomial value of 0x04C11DB7.</dd>
</dl>
<dl>
<dt>Page segments – 8 bits</dt>
<dd>This field indicates the number of segments that exist in this page. It also indicates how many bytes are in the segment table that follows this field. There can be a maximum of 255 segments in any one page.</dd>
</dl>
<dl>
<dt>Segment table</dt>
<dd>The segment table is a vector of 8-bit values, each indicating the length of the corresponding segment within the page body. The number of segments is determined from the preceding Page Segments field. Each segment is between 0 and 255 bytes in length.</dd>
</dl>
<p>The segments provide a way to group segments into packets, which are meaningful units of data for the decoder. When the segment's length is indicated to be 255, this indicates that the following segment is to be concatenated to this one and is part of the same packet. When the segment's length is 0–254, this indicates that this segment is the final segment in this packet. Where a packet's length is a multiple of 255, the final segment is length 0.</p>
<p>Where the final packet continues on the next page, the final segment value is 255, and the continuation flag is set on the following page to indicate that the start of the new page is a continuation of last page.</p>
<h2><span class="mw-headline" id="Metadata">Metadata</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ogg&amp;action=edit&amp;section=4" title="Edit section: Metadata">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<table class="plainlinks metadata ambox ambox-content ambox-Update" role="presentation">
<tr>
<td class="mbox-image">
<div style="width:52px"><a class="image" href="/wiki/File:Ambox_current_red.svg"><img alt="Ambox current red.svg" data-file-height="290" data-file-width="360" height="34" src="//upload.wikimedia.org/wikipedia/commons/thumb/9/98/Ambox_current_red.svg/42px-Ambox_current_red.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/9/98/Ambox_current_red.svg/63px-Ambox_current_red.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/9/98/Ambox_current_red.svg/84px-Ambox_current_red.svg.png 2x" width="42"/></a></div>

<td class="mbox-text"><span class="mbox-text-span">This article needs to be <b>updated</b>. <span class="hide-when-compact">Please update this section to reflect recent events or newly available information.</span> <small><i>(November 2009)</i></small></span></td>
</tr>
</table>
<p>Currently, there is no official standard for including metadata in Ogg containers. Implementation is still a long way off as of Quarter 3, 2007. The Xiph.Org Foundation are currently welcoming suggestions and feedback. Suggested metadata implementation methods include the following:<sup class="reference" id="cite_ref-oggmeta_11-0"><a href="#cite_note-oggmeta-11">[11]</a></sup></p>
<ul>
<li><a class="mw-redirect" href="/wiki/Dublin_core" title="Dublin core">Dublin core</a> in <a href="/wiki/Resource_Description_Framework" title="Resource Description Framework">RDF</a></li>
<li>XML-encoding (options include generic RDF, <a href="/wiki/Continuous_Media_Markup_Language" title="Continuous Media Markup Language">CMML</a>, and <a href="/wiki/Extensible_Metadata_Platform" title="Extensible Metadata Platform">XMP</a>)</li>
<li><a href="/wiki/MusicBrainz" title="MusicBrainz">MusicBrainz</a> XML MetaData</li>
<li>Ogg Skeleton is emerging as the way to approach machineable metadata.</li>
</ul>
<p>Metadata must currently be included in the <a href="/wiki/Codec" title="Codec">codec</a>. There is fairly good software support for Vorbis metadata—often referred to as comments. But software support for Theora and FLAC comments in Ogg containers is very limited.</p>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ogg&amp;action=edit&amp;section=5" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:154px;"><a class="image" href="/wiki/File:Play_ogg.png"><img alt="" class="thumbimage" data-file-height="51" data-file-width="152" height="51" src="//upload.wikimedia.org/wikipedia/commons/1/11/Play_ogg.png" width="152"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Play_ogg.png" title="Enlarge"></a></div>
The <i>Play Ogg</i> web button</div>
</div>
</div>
<p>The Ogg project began with a simple audio compression package as part of a larger project in 1993.<sup class="reference" id="cite_ref-xiphname_8-2"><a href="#cite_note-xiphname-8">[8]</a></sup> The software was originally named <i>Squish</i> but due to an existing trade mark it was renamed to <i><a href="/wiki/Ogg_Squish" title="Ogg Squish">OggSquish</a></i>. This name was later used for the whole Ogg project. In 1997, the Xiphophorus OggSquish was described as "an attempt both to create a flexible compressed audio format for modern audio applications as well as to provide the first audio format that is common on any and every modern computer platform".<sup class="reference" id="cite_ref-oggsquish-1997_12-0"><a href="#cite_note-oggsquish-1997-12">[12]</a></sup> The OggSquish was in 2000 referred to as "a group of several related multimedia and signal processing projects". In 2000, two projects were in active development for planned release: Ogg Vorbis format and libvorbis - the reference implementation of Vorbis. Research also included work on future video and lossless audio coding.<sup class="reference" id="cite_ref-xiphname_8-3"><a href="#cite_note-xiphname-8">[8]</a></sup><sup class="reference" id="cite_ref-squish-ogg_13-0"><a href="#cite_note-squish-ogg-13">[13]</a></sup><sup class="reference" id="cite_ref-oggsquish-bitstream_14-0"><a href="#cite_note-oggsquish-bitstream-14">[14]</a></sup><sup class="reference" id="cite_ref-oggsquish-2001_15-0"><a href="#cite_note-oggsquish-2001-15">[15]</a></sup><sup class="reference" id="cite_ref-oggsquish-ogg_16-0"><a href="#cite_note-oggsquish-ogg-16">[16]</a></sup> In 2001, OggSquish was renamed to <i>Ogg</i> and it was described as "the umbrella for a group of several related multimedia and signal processing projects".<sup class="reference" id="cite_ref-xiphname-2_17-0"><a href="#cite_note-xiphname-2-17">[17]</a></sup> Ogg has come to stand for the <a href="/wiki/File_format" title="File format">file format</a>, as part of the larger Xiph.org multimedia project. Squish became just the name of one of the Ogg codecs.<sup class="reference" id="cite_ref-xiphname-1_9-1"><a href="#cite_note-xiphname-1-9">[9]</a></sup> In 2009, Ogg is described as "a multimedia container format, and the native file and stream format for the Xiph.org multimedia codecs".<sup class="reference" id="cite_ref-ogg-2009_18-0"><a href="#cite_note-ogg-2009-18">[18]</a></sup></p>
<p>Ogg reference implementation was separated from Vorbis on September 2, 2000.<sup class="reference" id="cite_ref-ogg-separated_7-1"><a href="#cite_note-ogg-separated-7">[7]</a></sup></p>
<p>In May 2003, two Internet <a class="mw-redirect" href="/wiki/Requests_for_Comments" title="Requests for Comments">RFCs</a> were published relating to the format. The Ogg bitstream was defined in <a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc3533" rel="nofollow">RFC 3533</a> (which is classified as 'informative') and its <a class="mw-redirect" href="/wiki/MIME_content_type" title="MIME content type">Internet content type</a> (<code>application/ogg</code>) in <a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc3534" rel="nofollow">RFC 3534</a> (which is, as of 2006<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Ogg&amp;action=edit">[update]</a></sup>, a proposed standard protocol). In September 2008, <a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc3534" rel="nofollow">RFC 3534</a> was obsoleted by <a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc5334" rel="nofollow">RFC 5334</a>, which added content types <code>video/ogg</code>, <code>audio/ogg</code> and filename extensions .ogx, .ogv, .oga, .spx.</p>
<h3><span class="mw-headline" id="OGM">OGM</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ogg&amp;action=edit&amp;section=6" title="Edit section: OGM">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In 2002, the lack of formal video support in Ogg resulted in the creation of the <i>OGM</i> file format, a hack on Ogg that allowed embedding of video from the Microsoft <a href="/wiki/DirectShow" title="DirectShow">DirectShow</a> framework into an Ogg-based wrapper. OGM was initially supported only by closed source Windows-only tools, but the code base was subsequently opened. Later, video (and subtitle) support were formally specified for Ogg but in a manner incompatible with OGM. Independently, the <a href="/wiki/Matroska" title="Matroska">Matroska</a> container format reached maturity and provided another alternative for people interested in combining Vorbis audio and arbitrary video codecs. As a result, OGM is no longer supported or developed and is formally discouraged by Xiph.org.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup> Today, video in Ogg is found with the .ogv file extension, which is formally specified and officially supported.</p>
<h3><span class="mw-headline" id="2006">2006</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ogg&amp;action=edit&amp;section=7" title="Edit section: 2006">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Although Ogg hadn't reached anywhere near the ubiquity of the <a class="mw-redirect" href="/wiki/MPEG" title="MPEG">MPEG</a> standards<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup><sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> (e.g., <a href="/wiki/MP3" title="MP3">MP3</a>/<a class="mw-redirect" href="/wiki/MP4" title="MP4">MP4</a>), as of 2006<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Ogg&amp;action=edit">[update]</a></sup>, it was commonly used to encode <a href="/wiki/Free_content" title="Free content">free content</a> (such as free music, multimedia on <a class="mw-redirect" href="/wiki/Wikimedia" title="Wikimedia">Wikimedia</a> projects and <a href="/wiki/Creative_Commons" title="Creative Commons">Creative Commons</a> files) and had started to be supported by a significant minority of <a class="mw-redirect" href="/wiki/Digital_audio_player" title="Digital audio player">digital audio players</a>. Also supporting the Ogg format were many popular video game engines, including <a href="/wiki/Doom_3" title="Doom 3">Doom 3</a>, <a href="/wiki/Unreal_Tournament_2004" title="Unreal Tournament 2004">Unreal Tournament 2004</a>, <a href="/wiki/Halo:_Combat_Evolved" title="Halo: Combat Evolved">Halo: Combat Evolved</a>, <a class="mw-redirect" href="/wiki/Jets_n_guns" title="Jets n guns">Jets n Guns</a>, <a class="mw-redirect" href="/wiki/Mafia:_The_City_of_Lost_Heaven" title="Mafia: The City of Lost Heaven">Mafia: The City of Lost Heaven</a>, <a href="/wiki/Myst_IV:_Revelation" title="Myst IV: Revelation">Myst IV: Revelation</a>, <a class="mw-redirect" href="/wiki/Stepmania" title="Stepmania">Stepmania</a>, <a class="mw-redirect" href="/wiki/Serious_Sam:_The_Second_Encounter" title="Serious Sam: The Second Encounter">Serious Sam: The Second Encounter</a>, <a class="mw-redirect" href="/wiki/Lineage_2" title="Lineage 2">Lineage 2</a>, <a href="/wiki/Vendetta_Online" title="Vendetta Online">Vendetta Online</a>, <a href="/wiki/Battlefield_2" title="Battlefield 2">Battlefield 2</a>, and the <a class="mw-redirect" href="/wiki/Grand_Theft_Auto_(series)" title="Grand Theft Auto (series)">Grand Theft Auto</a> engines, as well as the audio files of the <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>-based game, <a href="/wiki/Minecraft" title="Minecraft">Minecraft</a>. The more popular Vorbis codec had built-in support on many software players, and extensions were available for nearly all the rest.</p>
<h3><span class="mw-headline" id="2007">2007</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ogg&amp;action=edit&amp;section=8" title="Edit section: 2007">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>On May 16, 2007, the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a> started a campaign to increase the use of Vorbis "as an ethically, legally and technically superior audio alternative to the proprietary MP3 format."<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup> People were also encouraged to support the campaign by adding a web button to their <a href="/wiki/Website" title="Website">website</a> or <a href="/wiki/Blog" title="Blog">blog</a>. For those who don't want to download and use FSF's suggested Ogg player (<a href="/wiki/VLC_media_player" title="VLC media player">VLC</a>), the <a class="mw-redirect" href="/wiki/Xiph" title="Xiph">Xiph.Org Foundation</a> had an official codec<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup> for <a href="/wiki/QuickTime" title="QuickTime">QuickTime</a>-based applications in <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a> and <a class="mw-redirect" href="/wiki/Mac_OS_X" title="Mac OS X">Mac OS X</a>, such as <a href="/wiki/ITunes" title="ITunes">iTunes</a> players and <a href="/wiki/IMovie" title="IMovie">iMovie</a> applications; and Windows users could install a <a href="/wiki/Windows_Media_Player" title="Windows Media Player">Windows Media Player</a> Ogg codec.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup></p>
<h3><span class="mw-headline" id="2009">2009</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ogg&amp;action=edit&amp;section=9" title="Edit section: 2009">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>By June 30, 2009, the Ogg container, through the use of the Theora and Vorbis, was the only container format included in <a class="mw-redirect" href="/wiki/Mozilla_Firefox_3.5" title="Mozilla Firefox 3.5">Firefox 3.5</a> web browser's implementation of the <a class="mw-redirect" href="/wiki/HTML_5" title="HTML 5">HTML 5</a> <code>&lt;video&gt;</code> and <code>&lt;audio&gt;</code> elements.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup><sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup> This was in accordance with the original recommendation outlined in, but later removed from, the HTML 5 draft specification (see <a class="mw-redirect" href="/wiki/Ogg_controversy" title="Ogg controversy">Ogg controversy</a>).</p>
<h3><span class="mw-headline" id="2010">2010</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ogg&amp;action=edit&amp;section=10" title="Edit section: 2010">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>On March 3, 2010, a technical analysis by an <a href="/wiki/FFmpeg" title="FFmpeg">FFmpeg</a> developer was critical about the general purpose abilities of Ogg as a multimedia container format.<sup class="reference" id="cite_ref-ogg-analysis_27-0"><a href="#cite_note-ogg-analysis-27">[27]</a></sup> In a lengthy response, the creator of Ogg refuted these claims.<sup class="reference" id="cite_ref-ogg-analysis-answer_28-0"><a href="#cite_note-ogg-analysis-answer-28">[28]</a></sup></p>
<h2><span class="mw-headline" id="Ogg_codecs">Ogg codecs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ogg&amp;action=edit&amp;section=11" title="Edit section: Ogg codecs">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Ogg is only a container format. The actual audio or video encoded by a codec is stored inside an Ogg container. Ogg containers may contain streams encoded with multiple codecs, for example, a video file with sound contains data encoded by both an audio codec and a video codec.</p>
<p>Being a container format, Ogg can embed audio and video in various formats<sup class="reference" id="cite_ref-ogg-mime_29-0"><a href="#cite_note-ogg-mime-29">[29]</a></sup><sup class="reference" id="cite_ref-rfc5334_30-0"><a href="#cite_note-rfc5334-30">[30]</a></sup> (such as <a class="mw-redirect" href="/wiki/Dirac_(codec)" title="Dirac (codec)">Dirac</a>, <a href="/wiki/Multiple-image_Network_Graphics" title="Multiple-image Network Graphics">MNG</a>, <a href="/wiki/CELT" title="CELT">CELT</a>, <a href="/wiki/MPEG-4" title="MPEG-4">MPEG-4</a>, <a href="/wiki/MP3" title="MP3">MP3</a> and others) but Ogg was intended to be, and usually is, used with the following <a class="mw-redirect" href="/wiki/Xiph.org" title="Xiph.org">Xiph.org</a> free codecs:</p>
<ul>
<li>Audio
<ul>
<li><a class="mw-redirect" href="/wiki/Lossy_data_compression" title="Lossy data compression">Lossy</a>
<ul>
<li><a href="/wiki/Speex" title="Speex">Speex</a>: handles voice data at low bitrates (~8–32 kbit/s/channel)</li>
<li><a href="/wiki/Vorbis" title="Vorbis">Vorbis</a>: handles general audio data at mid to high-level <a href="/wiki/Variable_bitrate" title="Variable bitrate">variable bitrates</a> (≈16–500 kbit/s per channel)</li>
<li><a class="mw-redirect" href="/wiki/Opus_(codec)" title="Opus (codec)">Opus</a>: handles voice, music and generic audio at low and high variable bitrates (≈6–510 kbit/s per channel)</li>
</ul>
</li>
<li><a class="mw-redirect" href="/wiki/Lossless_data_compression" title="Lossless data compression">Lossless</a>
<ul>
<li><a href="/wiki/FLAC" title="FLAC">FLAC</a> handles archival and high fidelity audio data.</li>
</ul>
</li>
<li>Uncompressed
<ul>
<li>OggPCM: handles uncompressed <a href="/wiki/Pulse-code_modulation" title="Pulse-code modulation">PCM</a> audio. It is broadly comparable to <a href="/wiki/WAV" title="WAV">WAV</a>.<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup></li>
</ul>
</li>
</ul>
</li>
<li>Video
<ul>
<li><a class="mw-redirect" href="/wiki/Lossy_data_compression" title="Lossy data compression">Lossy</a>
<ul>
<li><a href="/wiki/Theora" title="Theora">Theora</a>: based upon <a class="mw-redirect" href="/wiki/On2" title="On2">On2</a>'s <a href="/wiki/VP3" title="VP3">VP3</a>, it is targeted at competing with <a href="/wiki/MPEG-4" title="MPEG-4">MPEG-4</a> video (for example, encoded with <a href="/wiki/DivX" title="DivX">DivX</a> or <a href="/wiki/Xvid" title="Xvid">Xvid</a>), <a href="/wiki/RealVideo" title="RealVideo">RealVideo</a>, or <a href="/wiki/Windows_Media_Video" title="Windows Media Video">Windows Media Video</a>.</li>
<li><a href="/wiki/Daala" title="Daala">Daala</a>: a video coding format under development.</li>
<li>Tarkin: an experimental and now obsolete video codec developed in 2000, 2001 and 2002 utilizing <a href="/wiki/Discrete_wavelet_transform" title="Discrete wavelet transform">discrete wavelet transforms</a> in the three <a href="/wiki/Dimension" title="Dimension">dimensions</a> of width, height, and time.<sup class="reference" id="cite_ref-oggsquish-2001_15-1"><a href="#cite_note-oggsquish-2001-15">[15]</a></sup><sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup><sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup><sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup> It has been put on hold after Theora became the main focus for video encoding (in August 2002).<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup></li>
<li><a class="mw-redirect" href="/wiki/Dirac_(codec)" title="Dirac (codec)">Dirac</a>: a free and open video format developed by the <a href="/wiki/BBC" title="BBC">BBC</a>. Uses wavelet encoding.<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup></li>
</ul>
</li>
<li><a class="mw-redirect" href="/wiki/Lossless_data_compression" title="Lossless data compression">Lossless</a>
<ul>
<li><a class="mw-redirect" href="/wiki/Dirac_(codec)" title="Dirac (codec)">Dirac</a>: a part of the specification of dirac covers lossless compression.</li>
<li><a href="/wiki/Daala" title="Daala">Daala</a>: a video coding format under development.</li>
</ul>
</li>
</ul>
</li>
<li><span id="ogg-codecs-text">Text</span>
<ul>
<li>Writ: incomplete draft for a text codec designed to embed subtitles or captions, discontinued in 2007.<sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup></li>
<li><a href="/wiki/Continuous_Media_Markup_Language" title="Continuous Media Markup Language">Continuous Media Markup Language</a>: a text/application codec for timed metadata, captioning, and formatting.</li>
<li><a href="/wiki/Annodex" title="Annodex">Annodex</a>: A free and open source set of standards developed by <a class="mw-redirect" href="/wiki/CSIRO" title="CSIRO">CSIRO</a> to annotate and index networked media.</li>
<li><a class="external text" href="//wiki.xiph.org/OggKate" rel="nofollow">OggKate</a>: An overlay codec, originally designed for karaoke and text, that can be multiplexed in Ogg.</li>
</ul>
