<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Amarok_(software)&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The project was originally started by Mark Kretschmann as a means of improving <a href="/wiki/XMMS" title="XMMS">XMMS</a> due to several usability problems, which interfered with the addition of new files to the playlist due to several user interface elements existing for one task. The original amaroK was created based upon the idea of a two-pane interface seen in <a href="/wiki/Midnight_Commander" title="Midnight Commander">Midnight Commander</a>, and the first version of the software released solely by Kretschmann, was based upon the ideal of allowing users to drag-and-drop music into an interface in which the playlist was displayed on the right and information on the left.</p>
<p>After the initial release of AmaroK, several developers joined the project to form the “Three M’s” the first of whom was Max Howell, who acted as an interface designer and programmer for the project, alongside Muesli (Christian Muehlhaeuser), who also provided user interface insight and programming till the late 1.4 versions.</p>
<p>The program was originally named <b>amaroK</b> after a <a href="/wiki/Mike_Oldfield" title="Mike Oldfield">Mike Oldfield</a> album <a href="/wiki/Amarok_(Mike_Oldfield_album)" title="Amarok (Mike Oldfield album)">of the same name</a>.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> Later the artwork changed to reference <a href="/wiki/Amarok_(wolf)" title="Amarok (wolf)">Amarok, a wolf</a> in <a href="/wiki/Inuit" title="Inuit">Inuit</a> mythology. The app's capitalisation was changed to Amarok in June 2006.</p>
<p>A new major version of Amarok, version 2.0, was released on December 12, 2008. On June 3, 2009, version 2.1 was released which reintroduced a few of the 1.4 features which had been missing from the initial 2.0 release, and introduced some features such as native <a href="/wiki/ReplayGain" title="ReplayGain">ReplayGain</a> support for the first time.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Amarok_(software)&amp;action=edit&amp;section=2" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Basic_uses_and_functions">Basic uses and functions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Amarok_(software)&amp;action=edit&amp;section=3" title="Edit section: Basic uses and functions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Amarok serves many functions in addition to the basic function of playing music files. For example, Amarok can be used to organize a library of music into folders according to genre, artist, and album, can edit tags attached to most music formats, associate album art, attach lyrics, and automatically "score" music by keeping play count statistics.</p>
<p>Although a more technical list of features is listed below, here are the primary functions or uses for Amarok:</p>
<ul>
<li>Playing media files in various formats including but not limited to (depending on the setup) <a href="/wiki/FLAC" title="FLAC">FLAC</a>, <a href="/wiki/Ogg" title="Ogg">Ogg</a>, <a href="/wiki/Opus_(audio_format)" title="Opus (audio format)">Opus</a>, <a href="/wiki/MP3" title="MP3">MP3</a>, <a href="/wiki/Advanced_Audio_Coding" title="Advanced Audio Coding">AAC</a>, <a href="/wiki/WAV" title="WAV">WAV</a>, <a href="/wiki/Windows_Media_Audio" title="Windows Media Audio">Windows Media Audio</a>, <a href="/wiki/Apple_Lossless" title="Apple Lossless">Apple Lossless</a>, <a href="/wiki/WavPack" title="WavPack">WavPack</a>, <a href="/wiki/TTA_(codec)" title="TTA (codec)">TTA</a> and <a href="/wiki/Musepack" title="Musepack">Musepack</a>. Amarok does not play digital music files embedded with <a href="/wiki/Digital_rights_management" title="Digital rights management">DRM</a>.</li>
<li>Tagging digital music files (currently <a href="/wiki/FLAC" title="FLAC">FLAC</a>, <a class="mw-redirect" href="/wiki/Ogg_Vorbis" title="Ogg Vorbis">Ogg Vorbis</a>, <a href="/wiki/Windows_Media_Audio" title="Windows Media Audio">WMA</a>, <a href="/wiki/Advanced_Audio_Coding" title="Advanced Audio Coding">AAC</a>, <a href="/wiki/MP3" title="MP3">MP3</a>, and <a href="/wiki/RealMedia" title="RealMedia">RealMedia</a>).</li>
<li>Associating cover art with a particular album, and retrieving the cover art from <a href="/wiki/Amazon.com" title="Amazon.com">Amazon</a>.</li>
<li>Creating and editing playlists, including smart and dynamic playlists. The dynamic playlists can use such information as the "score" given to a song by an Amarok script, and the playcount which is stored in the database.</li>
<li>Synchronizing, retrieving, playing, or uploading music to the following digital music players: <a href="/wiki/IPod" title="IPod">iPod</a>, <a href="/wiki/Iriver" title="Iriver">iriver</a> iFP, <a href="/wiki/Creative_NOMAD" title="Creative NOMAD">Creative NOMAD</a>, <a class="mw-redirect" href="/wiki/Creative_ZEN" title="Creative ZEN">Creative ZEN</a>, <a href="/wiki/Media_Transfer_Protocol" title="Media Transfer Protocol">MTP</a>, <a href="/wiki/Rio_Karma" title="Rio Karma">Rio Karma</a> and <a href="/wiki/USB" title="USB">USB</a> devices with <a href="/wiki/File_Allocation_Table" title="File Allocation Table">VFAT</a> (generic MP3 players) support.</li>
<li>Displaying artist information from Wikipedia and retrieving song lyrics.</li>
<li><a href="/wiki/Last.fm" title="Last.fm">Last.fm</a> support, including submitting played tracks (including those played on some digital music players) to Last.fm, retrieving similar artists and playing Last.fm streams.</li>
<li><a href="/wiki/Podcast" title="Podcast">Podcast</a>.</li>
</ul>
<p>From version 1.4.4, Amarok introduced the integration of <a href="/wiki/Magnatune" title="Magnatune">Magnatune</a>, a non-<a href="/wiki/Digital_rights_management" title="Digital rights management">DRM</a> digital music store, enabling users to purchase music in <a href="/wiki/Ogg" title="Ogg">Ogg</a>, <a href="/wiki/FLAC" title="FLAC">FLAC</a>, <a href="/wiki/WAV" title="WAV">WAV</a>, and <a href="/wiki/MP3" title="MP3">MP3</a> formats.</p>
<p>Some of these features depend on other programs or libraries that must be installed on the computer to operate.</p>
<h3><span class="mw-headline" id="Technical_features">Technical features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Amarok_(software)&amp;action=edit&amp;section=4" title="Edit section: Technical features">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>Three main window panes: playlist browser, collection and player window.</li>
<li><a class="mw-redirect" href="/wiki/Systray" title="Systray">Systray</a> (panel notification area) icon support.</li>
<li>Song collection, which includes specific folders on the filesystem.
<ul>
<li>Searching Files/Artists/Album/Genre in Collection can be performed using Simple and Advanced options</li>
</ul>
</li>
<li>Intelligent Playlists support (Dynamic Playlists)</li>
<li>Integration with online services such as <a href="/wiki/Magnatune" title="Magnatune">Magnatune</a>, <a href="/wiki/Jamendo" title="Jamendo">Jamendo</a>, <a class="mw-redirect" href="/wiki/MP3tunes" title="MP3tunes">MP3tunes</a>, <a href="/wiki/Last.fm" title="Last.fm">Last.fm</a> and <a class="mw-redirect" href="/wiki/Shoutcast" title="Shoutcast">Shoutcast</a>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></li>
<li>Songs can be rated both dynamically (based on how much the song is played) and by hand (giving rating of 1-5 stars to the song).</li>
<li>Amarok File Tracking (since 1.4.3): Stores file checksum in the collection. This allows the file be moved around in the filesystem without Amarok losing track of the song statistics.</li>
<li>Collection filter (newest songs, highest rated, most played, etc.).</li>
<li>Support for Phonon engine. The backend of Phonon dictates which media types Amarok can play and how they play. When using <a href="/wiki/Xine" title="Xine">Xine</a>, Amarok supports <a class="mw-redirect" href="/wiki/Crossfade_(audio_engineering)" title="Crossfade (audio engineering)">crossfading</a>, but not seeking in seekpointless FLAC files. When using <a class="mw-redirect" href="/wiki/Gstreamer" title="Gstreamer">Gstreamer</a>, Amarok supports seeking and gapless playback, but not crossfading.</li>
<li>Support for <a href="/wiki/Moodbar" title="Moodbar">moodbars</a>.</li>
<li>Uses TagLib for tags.</li>
<li>Amarok can be controlled via <a href="/wiki/D-Bus" title="D-Bus">D-Bus</a>.</li>
<li>Amarok Scripts, for example, when writing lyrics fetching plugins, can be written in <a href="/wiki/QtScript" title="QtScript">QtScript</a>.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></li>
</ul>
<h2><span class="mw-headline" id="Amarok_2.0">Amarok 2.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Amarok_(software)&amp;action=edit&amp;section=5" title="Edit section: Amarok 2.0">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Amarok 2 was released on 10 December 2008.. New features include:</p>
<ul>
<li>Tight integration with online services such as <a href="/wiki/Magnatune" title="Magnatune">Magnatune</a>, <a href="/wiki/Jamendo" title="Jamendo">Jamendo</a>, MP3tunes, <a href="/wiki/Last.fm" title="Last.fm">Last.fm</a> and <a class="mw-redirect" href="/wiki/Shoutcast" title="Shoutcast">Shoutcast</a>.</li>
<li>Completely overhauled scripting API and plugin support to allow better integration into Amarok.</li>
<li>Migration from the <a class="mw-redirect" href="/wiki/KDE_3" title="KDE 3">KDE 3</a> to <a href="/wiki/KDE_Software_Compilation_4" title="KDE Software Compilation 4">KDE Software Compilation 4</a>, and utilization of core technologies such as <a class="mw-redirect" href="/wiki/Solid_(KDE)" title="Solid (KDE)">Solid</a>, <a class="mw-redirect" href="/wiki/Phonon_(KDE)" title="Phonon (KDE)">Phonon</a>, and <a class="mw-redirect" href="/wiki/Plasma_(KDE)" title="Plasma (KDE)">Plasma</a>.</li>
</ul>
<p>The user interface had been redesigned to make context information like lyrics and albums from the same artist more accessible and allow the user to decide which information is available by adding applets to the Context View in the middle.</p>
<h3><span class="mw-headline" id="Changes_since_1.4">Changes since 1.4</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Amarok_(software)&amp;action=edit&amp;section=6" title="Edit section: Changes since 1.4">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>Service Framework: This integrates networked music sources directly into Amarok. This includes online music stores, media servers, Web music lockers, and more. Due to this the users will be able to get easy access to music provided by <a href="/wiki/Magnatune" title="Magnatune">Magnatune</a>, <a href="/wiki/Jamendo" title="Jamendo">Jamendo</a>, <a href="/wiki/Last.fm" title="Last.fm">Last.fm</a>, MP3Tunes Locker and <a href="/wiki/Ampache" title="Ampache">Ampache</a>.</li>
<li>Biased Playlists: Biased Playlists extend the old Dynamic Playlists. They allow users to define an automatically populated playlist, based on specific probability-driven criteria called "biases".</li>
<li>Context View: This view occupies the central place of the Amarok’s window, replacing the old Context Browser from the 1.x series. It displays contextual information about the music one plays, like the album cover, track rating, labels, lyrics, artist information, related songs and artists and track mood.</li>
<li>Other Features: Additional changes in Amarok 2.0 include new SVG-based scalable theme, advanced scripting, dynamic collections, usability changes, updated media device handling, new podcast manager and support for more software platforms.</li>
</ul>

<p>The transition from version 1.4 to version 2 was criticized by many users. As a consequence some new projects have been established, based on Amarok version 1.4.x.</p>
<ul>
<li>Amarok Live was a <a href="/wiki/Live_CD" title="Live CD">live CD</a> <a href="/wiki/Linux_distribution" title="Linux distribution">Linux distribution</a> with the sole purpose to demonstrate Amarok.</li>
<li><a href="/wiki/Clementine_(software)" title="Clementine (software)">Clementine</a> is a fork of Amarok 1.4<sup class="reference" id="cite_ref-amarok14port_8-0"><a href="#cite_note-amarok14port-8">[8]</a></sup> to port it to the Qt 4 framework.</li>

<li>Gereqi is a clone of Amarok 1.4, written in <a href="/wiki/PySide" title="PySide">PySide</a>.</li>
<li>Pana was a fork of Amarok 1.4. The intention was to keep the original program alive. The author has now discontinued the project.</li>
</ul>
