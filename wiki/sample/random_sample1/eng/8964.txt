<h1 class="firstHeading" id="firstHeading" lang="en">Einstein notation</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p>In <a href="/wiki/Mathematics" title="Mathematics">mathematics</a>, especially in applications of <a href="/wiki/Linear_algebra" title="Linear algebra">linear algebra</a> to <a href="/wiki/Physics" title="Physics">physics</a>, the <b>Einstein notation</b> or <b>Einstein summation convention</b> is a notational convention that implies summation over a set of indexed terms in a formula, thus achieving notational brevity. As part of mathematics it is a notational subset of <a href="/wiki/Ricci_calculus" title="Ricci calculus">Ricci calculus</a>; however, it is often used in applications in physics that do not distinguish between <a href="/wiki/Tangent_space" title="Tangent space">tangent</a> and <a href="/wiki/Cotangent_space" title="Cotangent space">cotangent</a> spaces. It was introduced to physics by <a href="/wiki/Albert_Einstein" title="Albert Einstein">Albert Einstein</a> in 1916.<sup class="reference" id="cite_ref-Ein1916_1-0"><a href="#cite_note-Ein1916-1">[1]</a></sup></p>
<p></p>

<h2><span class="mw-headline" id="Introduction">Introduction</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Einstein_notation&amp;action=edit&amp;section=1" title="Edit section: Introduction">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Statement_of_convention">Statement of convention</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Einstein_notation&amp;action=edit&amp;section=2" title="Edit section: Statement of convention">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>According to this convention, when an index variable appears twice in a single term and is not otherwise defined (see <a class="mw-redirect" href="/wiki/Free_and_bound_variables" title="Free and bound variables">free and bound variables</a>), it implies summation of that term over all the values of the index. <p>The upper indices are not <a href="/wiki/Exponentiation" title="Exponentiation">exponents</a> but are indices of coordinates, <a href="/wiki/Coefficient" title="Coefficient">coefficients</a> or <a class="mw-redirect" href="/wiki/Basis_vector" title="Basis vector">basis vectors</a>. That is, in this context <span class="texhtml"><i>x</i><sup>2</sup></span> should be understood as the second component of <span class="texhtml"><b>x</b></span> rather than the square of <span class="texhtml"><b>x</b></span> (this can occasionally lead to ambiguity). Typically <span class="texhtml">(<i>x</i><sup>1</sup>, <i>x</i><sup>2</sup>, <i>x</i><sup>3</sup>)</span> would be equivalent to the traditional <span class="texhtml">(<i>x</i>, <i>y</i>, <i>z</i>)</span>.</p>
<p>In <a href="/wiki/General_relativity" title="General relativity">general relativity</a>, a common convention is that</p>
<ul>
<li>the <a href="/wiki/Greek_alphabet" title="Greek alphabet">Greek alphabet</a> is used for space and time components, where indices take on values 0, 1, 2, or 3 (frequently used letters are <span class="texhtml"><i>μ</i>, <i>ν</i>, ...</span>),</li>
<li>the <a href="/wiki/Latin_alphabet" title="Latin alphabet">Latin alphabet</a> is used for spatial components only, where indices take on values 1, 2, or 3 (frequently used letters are <span class="texhtml"><i>i</i>, <i>j</i>, ...</span>),</li>
</ul>
<p>In general, indices can range over any <a href="/wiki/Indexed_family" title="Indexed family">indexing set</a>, including an <a href="/wiki/Infinite_set" title="Infinite set">infinite set</a>. This should not be confused with a typographically similar convention used to distinguish between <a class="mw-redirect" href="/wiki/Tensor_index_notation" title="Tensor index notation">tensor index notation</a> and the closely related but distinct basis-independent <a href="/wiki/Abstract_index_notation" title="Abstract index notation">abstract index notation</a>.</p>
<p>An index that is summed over is a <i>summation index</i>, in this case "<span class="texhtml"><i>i</i></span>". It is also called a <a class="mw-redirect" href="/wiki/Bound_variable" title="Bound variable">dummy index</a> since any symbol can replace "<span class="texhtml"><i>i</i></span>" without changing the meaning of the expression provided that it does not collide with index symbols in the same term.</p>
<p>An index that is not summed over is a <a class="mw-redirect" href="/wiki/Free_variable" title="Free variable"><i>free index</i></a> and should appear only once per term. If such an index does appear, it usually also appears in terms belonging to the same sum, with the exception of special values such as zero.</p>
<h3><span class="mw-headline" id="Application">Application</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Einstein_notation&amp;action=edit&amp;section=3" title="Edit section: Application">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Einstein notation can be applied in slightly different ways. Typically, each index occurs once in an upper (superscript) and once in a lower (subscript) position in a term; however, the convention can be applied more generally to any repeated indices within a term.<sup class="reference" id="cite_ref-wolfram_2-0"><a href="#cite_note-wolfram-2">[2]</a></sup> When dealing with <a href="/wiki/Covariance_and_contravariance_of_vectors" title="Covariance and contravariance of vectors">covariant and contravariant</a> vectors, where the position of an index also indicates the type of vector, the first case usually applies; a covariant vector can only be contracted with a contravariant vector, corresponding to summation of the products of coefficients. On the other hand, when there is a fixed coordinate basis (or when not considering coordinate vectors), one may choose to use only subscripts; <a href="#Superscripts_and_subscripts_vs._only_subscripts">see below</a>.</p>
<h2><span class="mw-headline" id="Vector_representations">Vector representations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Einstein_notation&amp;action=edit&amp;section=4" title="Edit section: Vector representations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Superscripts_and_subscripts_versus_only_subscripts">Superscripts and subscripts versus only subscripts</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Einstein_notation&amp;action=edit&amp;section=5" title="Edit section: Superscripts and subscripts versus only subscripts">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In terms of <a href="/wiki/Covariance_and_contravariance_of_vectors" title="Covariance and contravariance of vectors">covariance and contravariance of vectors</a>,</p>
<ul>
<li>upper indices represent components of <a href="/wiki/Covariance_and_contravariance_of_vectors" title="Covariance and contravariance of vectors">contravariant vectors</a> (<a href="/wiki/Coordinate_vector" title="Coordinate vector">vectors</a>),</li>
<li>lower indices represent components of <a class="mw-redirect" href="/wiki/Covariant_vector" title="Covariant vector">covariant</a> vectors (<a class="mw-redirect" href="/wiki/Covector" title="Covector">covectors</a>).</li>
</ul>
<p>They transform contravariantly or covariantly, respectively, with respect to change of basis.</p>

<h3><span class="mw-headline" id="Abstract_description">Abstract description</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Einstein_notation&amp;action=edit&amp;section=7" title="Edit section: Abstract description">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The virtue of Einstein notation is that it represents the invariant quantities with a simple notation.</p>
<p>In physics, a <a href="/wiki/Scalar_(physics)" title="Scalar (physics)">scalar</a> is invariant under transformations of <a class="mw-redirect" href="/wiki/Basis_(mathematics)" title="Basis (mathematics)">basis</a>. In particular, a <a href="/wiki/Lorentz_scalar" title="Lorentz scalar">Lorentz scalar</a> is invariant under a Lorentz transformation. The individual terms in the sum are not. When the basis is changed, the <i>components</i> of a vector change by a linear transformation described by a matrix. This led Einstein to propose the convention that repeated indices imply the summation is to be done.</p>
<p>As for covectors, they change by the inverse matrix. This is designed to guarantee that the linear function associated with the covector, the sum above, is the same no matter what the basis is.</p>
<p>The value of the Einstein convention is that it applies to other vector spaces built from <span class="texhtml"><i>V</i></span> using the <a href="/wiki/Tensor_product" title="Tensor product">tensor product</a> and <a href="/wiki/Dual_space" title="Dual space">duality</a>.
