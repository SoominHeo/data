<p><b>DD-WRT</b> is a <a href="/wiki/GNU" title="GNU">GNU</a>/<a href="/wiki/Linux_kernel" title="Linux kernel">Linux</a>-based <a href="/wiki/Firmware" title="Firmware">firmware</a> for <a href="/wiki/Wireless_router" title="Wireless router">wireless routers</a> and <a href="/wiki/Wireless_access_point" title="Wireless access point">access points</a>. Originally designed for the <a href="/wiki/Linksys_WRT54G_series" title="Linksys WRT54G series">Linksys WRT54G series</a>, it now runs on a wide variety of models. DD-WRT is one of a handful of <a href="/wiki/List_of_router_firmware_projects" title="List of router firmware projects">third-party firmware projects</a> designed to replace manufacturer's original firmware with custom firmware offering additional features or functionality.</p>
<p>The firmware project's name was taken in part from the Linksys WRT54G model router, a home router popular in 2002–2004. "DD" are the <a href="/wiki/Vehicle_registration_plates_of_Germany" title="Vehicle registration plates of Germany">German license-plate</a> letters for vehicles from Dresden, where the BrainSlayers development team lived.<sup class="reference" id="cite_ref-GPL_wiki_2-0"><a href="#cite_note-GPL_wiki-2">[2]</a></sup> "WRT", also used by the <a href="/wiki/OpenWrt" title="OpenWrt">OpenWrt</a> router firmware project, comes from the generic abbreviation for "Wireless RouTer", which may have been the original Linksys meaning.</p>
<p><a href="/wiki/Melco" title="Melco">Buffalo Technology</a> and other companies have shipped routers with factory-installed, customized versions of DD-WRT.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> In January 2016, Linksys itself started to offer DD-WRT firmware for their routers.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>

<p>Over the past years, DD-WRT became popular among VPN users due to its support of OpenVPN protocol. This provides an ability to use various VPNs on a DD-WRT in order to secure the whole network. As a result all traffic which passes through DD-WRT router gets a VPN protection.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[7]</a></sup></p>
<p>It is possible to build a custom firmware package with the desired feature-set, working within the limitations of available ROM.<sup class="reference" id="cite_ref-Firmware_Modification_Kit_10-0"><a href="#cite_note-Firmware_Modification_Kit-10">[8]</a></sup></p>
<h3><span class="mw-headline" id="ROM_requirements">ROM requirements</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DD-WRT&amp;action=edit&amp;section=2" title="Edit section: ROM requirements">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>All versions require at least 4 MB of flash memory, unless noted otherwise.</li>
<li>Micro+ requires at least 2 MB of flash memory plus 128 KB <a href="/wiki/Common_Firmware_Environment" title="Common Firmware Environment">Common Firmware Environment</a> (CFE) storage space.</li>
</ul>
<h2><span class="mw-headline" id="Version_history">Version history</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DD-WRT&amp;action=edit&amp;section=3" title="Edit section: Version history">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>Version 16 (the first DD-WRT version) – 22 January 2005<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[9]</a></sup> – Created as a branch of Sveasoft <a class="new" href="/w/index.php?title=Alchemy_firmware&amp;action=edit&amp;redlink=1" title="Alchemy firmware (page does not exist)">Alchemy</a>, which in turn is based on the Linksys WRT54G firmware</li>
<li>Version 22 – 25 July 2005<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[10]</a></sup> – The last version to use the Alchemy kernel</li>
<li>Version 23 – 25 December 2005<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[11]</a></sup> – The first version to use the <a href="/wiki/OpenWrt" title="OpenWrt">OpenWrt</a> kernel instead of the Alchemy kernel</li>
<li>Version 23 Service Pack 1 – 16 May 2006 – Much of the code was overhauled and rewritten during the development of this release, and many new features were added.</li>
<li>Version 23 Service Pack 2 – 14 September 2006 – The interface was overhauled, and some new features were added. Some additional router models are supported.</li>
<li>Version 24 – 18 May 2008 – Allows up to 16 virtual interfaces with different <a href="/wiki/Service_set_(802.11_network)#Service_set_identifier_.28SSID.29" title="Service set (802.11 network)">SSIDs</a> and <a href="/wiki/Wireless_security" title="Wireless security">encryption protocols</a>. It can run on some <a href="/wiki/PowerPC" title="PowerPC">PowerPC</a>, IXP425-based router boards, Atheros WiSOC, and X86-based systems. It can also run to some extent on routers with low flash memory (ex. <a href="/wiki/Linksys_WRT54G_series#Hardware_and_revisions" title="Linksys WRT54G series">WRT54Gv</a>8 or <a href="/wiki/Linksys_WRT54G_series#WRT54GS" title="Linksys WRT54G series">WRT54GSv</a>7).</li>
<li>Version 24 Service Pack 1 – 26 July 2008 – Contains an urgent DNS security fix for an issue in <a href="/wiki/Dnsmasq" title="Dnsmasq">dnsmasq</a>, site survey security fixes, longer passwords, and flexible OpenVPN configurations. It can also run on additional hardware, including WRT300 v1.1, WRT310N, WRT600N, Tonze AP42X Pronghorn SBC, Ubiquiti LSX and Netgear, Belkin, and USR devices.</li>
</ul>
<p>From the last version forward, builds have been constantly released on DD-WRT forums (with a beta tag) for different routers. Some of these are considered highly stable while others are not working at all.</p>
<p>Classical features of routers can be fully enabled with version 24 Service Pack 2 build 14929 (08/12/10) standard,<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[12]</a></sup> which can be installed with the micro version first (rather than the standard version).</p>
