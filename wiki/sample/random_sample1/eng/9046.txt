<p><b>Apache Hadoop</b> (<span class="nowrap"><small>pronunciation:</small> <span class="IPA nopopups"><a href="/wiki/Help:IPA_for_English" title="Help:IPA for English">/<span style="border-bottom:1px dotted"><span title="'h' in 'hi'">h</span><span title="/ə/ 'a' in 'about'">ə</span><span title="/ˈ/ primary stress follows">ˈ</span><span title="'d' in 'dye'">d</span><span title="/uː/ long 'oo' in 'food'">uː</span><span title="'p' in 'pie'">p</span></span>/</a></span></span>) is an <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open-source</a> <a href="/wiki/Software_framework" title="Software framework">software framework</a> used for <a href="/wiki/Clustered_file_system" title="Clustered file system">distributed storage</a> and processing of very large <a href="/wiki/Data_set" title="Data set">data sets</a>. It consists of <a href="/wiki/Computer_cluster" title="Computer cluster">computer clusters</a> built from <a class="mw-redirect" href="/wiki/Commodity_hardware" title="Commodity hardware">commodity hardware</a>. All the modules in Hadoop are designed with a fundamental assumption that hardware failures are common occurrences and should be automatically handled by the framework.<sup class="reference" id="cite_ref-homepage_2-1"><a href="#cite_note-homepage-2">[2]</a></sup></p>
<p>The core of Apache Hadoop consists of a storage part, known as Hadoop Distributed File System (HDFS), and a processing part which is a <a href="/wiki/MapReduce" title="MapReduce">MapReduce</a> <a href="/wiki/Programming_model" title="Programming model">programming model</a>. Hadoop splits files into large blocks and distributes them across nodes in a cluster. It then transfers <a href="/wiki/JAR_(file_format)" title="JAR (file format)">packaged code</a> into nodes to process the data in parallel. This approach takes advantage of <a class="mw-redirect" href="/wiki/Data_locality" title="Data locality">data locality</a><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> –  nodes manipulating the data they have access to –  to allow the dataset to be <a class="mw-redirect" href="/wiki/Distributed_processing" title="Distributed processing">processed</a> faster and more efficiently than it would be in a more conventional <a href="/wiki/Supercomputer_architecture" title="Supercomputer architecture">supercomputer architecture</a> that relies on a <a class="mw-redirect" href="/wiki/Parallel_file_system" title="Parallel file system">parallel file system</a> where computation and data are distributed via high-speed networking.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p>The base Apache Hadoop framework is composed of the following modules:</p>
<ul>
<li><i>Hadoop Common</i> – contains libraries and utilities needed by other Hadoop modules;</li>
<li><i>Hadoop Distributed File System (HDFS)</i> – a distributed file-system that stores data on commodity machines, providing very high aggregate bandwidth across the cluster;</li>
<li><i>Hadoop YARN</i> – a resource-management platform responsible for managing computing resources in clusters and using them for scheduling of users' applications;<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup><sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> and</li>
<li><i>Hadoop MapReduce</i> – an implementation of the <a href="/wiki/MapReduce" title="MapReduce">MapReduce</a> programming model for large scale data processing.</li>
</ul>
<p>The term <i>Hadoop</i> has come to refer not just to the base modules above, but also to the <i>ecosystem</i>,<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> or collection of additional software packages that can be installed on top of or alongside Hadoop, such as <a href="/wiki/Pig_(programming_tool)" title="Pig (programming tool)">Apache Pig</a>, <a href="/wiki/Apache_Hive" title="Apache Hive">Apache Hive</a>, <a href="/wiki/Apache_HBase" title="Apache HBase">Apache HBase</a>, <a href="/wiki/Apache_Phoenix" title="Apache Phoenix">Apache Phoenix</a>, <a href="/wiki/Apache_Spark" title="Apache Spark">Apache Spark</a>, <a href="/wiki/Apache_ZooKeeper" title="Apache ZooKeeper">Apache ZooKeeper</a>, <a href="/wiki/Cloudera_Impala" title="Cloudera Impala">Cloudera Impala</a>, <a href="/wiki/Apache_Flume" title="Apache Flume">Apache Flume</a>, <a class="mw-redirect" href="/wiki/Apache_Sqoop" title="Apache Sqoop">Apache Sqoop</a>, <a href="/wiki/Apache_Oozie" title="Apache Oozie">Apache Oozie</a>, <a class="mw-redirect" href="/wiki/Apache_Storm" title="Apache Storm">Apache Storm</a>.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<p>Apache Hadoop's MapReduce and HDFS components were inspired by <a href="/wiki/Google" title="Google">Google</a> papers on their <a href="/wiki/MapReduce" title="MapReduce">MapReduce</a> and <a href="/wiki/Google_File_System" title="Google File System">Google File System</a>.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
<p>The Hadoop framework itself is mostly written in the <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java programming language</a>, with some native code in <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> and <a href="/wiki/Command-line_interface" title="Command-line interface">command line</a> utilities written as <a class="mw-redirect" href="/wiki/Shell_scripts" title="Shell scripts">shell scripts</a>. Though MapReduce Java code is common, any programming language can be used with "Hadoop Streaming" to implement the "map" and "reduce" parts of the user's program.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> Other projects in the Hadoop ecosystem expose richer user interfaces.</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The genesis of Hadoop came from the Google File System paper<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> that was published in October 2003. This paper spawned another research paper from Google –  MapReduce: Simplified Data Processing on Large Clusters.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> Development started on the <a href="/wiki/Apache_Nutch" title="Apache Nutch">Apache Nutch</a> project, but was moved to the new Hadoop subproject in January 2006.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> <a href="/wiki/Doug_Cutting" title="Doug Cutting">Doug Cutting</a>, who was working at <a href="/wiki/Yahoo!" title="Yahoo!">Yahoo!</a> at the time,<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> named it after his son's toy elephant.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> The initial code that was factored out of Nutch consisted of 5k lines of code for NDFS and 6k lines of code for MapReduce.</p>
<p>The first committer added to the Hadoop project was Owen O’Malley in March 2006.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> Hadoop 0.1.0 was released in April 2006<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> and continues to evolve by the many contributors<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> to the Apache Hadoop project.</p>

<h2><span class="mw-headline" id="Architecture">Architecture</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=3" title="Edit section: Architecture">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">See also: <a href="#Hadoop_distributed_file_system">Hadoop Distributed File System</a>, <a href="/wiki/Apache_HBase" title="Apache HBase">Apache HBase</a>, and <a href="/wiki/MapReduce" title="MapReduce">MapReduce</a></div>
<p>Hadoop consists of the <i>Hadoop Common</i> package, which provides file system and OS level abstractions, a MapReduce engine (either MapReduce/MR1 or YARN/MR2)<sup class="reference" id="cite_ref-58"><a href="#cite_note-58">[58]</a></sup> and the <a href="#Hadoop_distributed_file_system">Hadoop Distributed File System</a> (HDFS). The Hadoop Common package contains the necessary <a href="/wiki/JAR_(file_format)" title="JAR (file format)">Java ARchive (JAR)</a> files and scripts needed to start Hadoop.</p>
<p>For effective scheduling of work, every Hadoop-compatible file system should provide location awareness: the name of the rack (more precisely, of the network switch) where a worker node is. Hadoop applications can use this information to execute code on the node where the data is, and, failing that, on the same rack/switch to reduce backbone traffic. HDFS uses this method when replicating data for data redundancy across multiple racks. This approach reduces the impact of a rack power outage or switch failure; if one of these hardware failures occurs, the data will remain available.<sup class="reference" id="cite_ref-59"><a href="#cite_note-59">[59]</a></sup></p>
<div class="thumb tright">
<div class="thumbinner" style="width:262px;"><a class="image" href="/wiki/File:Hadoop_1.png"><img alt="Hadoop cluster" class="thumbimage" data-file-height="404" data-file-width="520" height="202" src="//upload.wikimedia.org/wikipedia/en/thumb/2/2b/Hadoop_1.png/260px-Hadoop_1.png" srcset="//upload.wikimedia.org/wikipedia/en/thumb/2/2b/Hadoop_1.png/390px-Hadoop_1.png 1.5x, //upload.wikimedia.org/wikipedia/en/2/2b/Hadoop_1.png 2x" width="260"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Hadoop_1.png" title="Enlarge"></a></div>
A multi-node Hadoop cluster</div>
</div>
</div>
<p>A small Hadoop cluster includes a single master and multiple worker nodes. The master node consists of a Job Tracker, Task Tracker, NameNode, and DataNode. A slave or <i>worker node</i> acts as both a DataNode and TaskTracker, though it is possible to have data-only worker nodes and compute-only worker nodes. These are normally used only in nonstandard applications.<sup class="reference" id="cite_ref-michael-noll.com_2_60-0"><a href="#cite_note-michael-noll.com_2-60">[60]</a></sup></p>
<p>Hadoop requires <a class="mw-redirect" href="/wiki/JRE" title="JRE">Java Runtime Environment</a> (JRE) 1.6 or higher. The standard startup and shutdown scripts require that <a href="/wiki/Secure_Shell" title="Secure Shell">Secure Shell</a> (ssh) be set up between nodes in the cluster.<sup class="reference" id="cite_ref-michael-noll.com_1_61-0"><a href="#cite_note-michael-noll.com_1-61">[61]</a></sup></p>
<p>In a larger cluster, HDFS nodes are managed through a dedicated NameNode server to host the file system index, and a secondary NameNode that can generate snapshots of the namenode's memory structures, thereby preventing file-system corruption and loss of data. Similarly, a standalone JobTracker server can manage job scheduling across nodes. When Hadoop MapReduce is used with an alternate file system, the NameNode, secondary NameNode, and DataNode architecture of HDFS are replaced by the file-system-specific equivalents.</p>
<h3><span class="mw-headline" id="File_systems">File systems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=4" title="Edit section: File systems">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<h4><span class="mw-headline" id="Hadoop_distributed_file_system"><span id="HDFS"></span>Hadoop distributed file system</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=5" title="Edit section: Hadoop distributed file system">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>The Hadoop distributed file system (HDFS) is a distributed, scalable, and portable <a class="mw-redirect" href="/wiki/Distributed_file_system" title="Distributed file system">file system</a> written in Java for the Hadoop framework. Some consider HDFS to instead be a <a href="/wiki/Distributed_data_store" title="Distributed data store">data store</a> due to its lack of <a href="/wiki/POSIX" title="POSIX">POSIX</a> compliance and inability to be mounted,<sup class="reference" id="cite_ref-62"><a href="#cite_note-62">[62]</a></sup> but it does provide shell commands and Java API methods that are similar to other file systems.<sup class="reference" id="cite_ref-63"><a href="#cite_note-63">[63]</a></sup> A Hadoop cluster has nominally a single namenode plus a cluster of datanodes, although <a href="/wiki/Redundancy_(engineering)" title="Redundancy (engineering)">redundancy</a> options are available for the namenode due to its criticality. Each datanode serves up blocks of data over the network using a block protocol specific to HDFS. The file system uses <a class="mw-redirect" href="/wiki/TCP/IP" title="TCP/IP">TCP/IP</a> <a class="mw-redirect" href="/wiki/Internet_socket" title="Internet socket">sockets</a> for communication. Clients use <a href="/wiki/Remote_procedure_call" title="Remote procedure call">remote procedure call</a> (RPC) to communicate between each other.</p>
<p>HDFS stores large files (typically in the range of gigabytes to terabytes<sup class="reference" id="cite_ref-64"><a href="#cite_note-64">[64]</a></sup>) across multiple machines. It achieves reliability by <a class="mw-redirect" href="/wiki/Replication_(computer_science)" title="Replication (computer science)">replicating</a> the data across multiple hosts, and hence theoretically does not require <a href="/wiki/RAID" title="RAID">RAID</a> storage on hosts (but to increase I/O performance some RAID configurations are still useful). With the default replication value, 3, data is stored on three nodes: two on the same rack, and one on a different rack. Data nodes can talk to each other to rebalance data, to move copies around, and to keep the replication of data high. HDFS is not fully <a href="/wiki/POSIX" title="POSIX">POSIX</a>-compliant, because the requirements for a POSIX file-system differ from the target goals for a Hadoop application. The trade-off of not having a fully POSIX-compliant file-system is increased performance for data <a href="/wiki/Throughput" title="Throughput">throughput</a> and support for non-POSIX operations such as Append.<sup class="reference" id="cite_ref-openlibrary1_65-0"><a href="#cite_note-openlibrary1-65">[65]</a></sup></p>
<p>HDFS added the high-availability capabilities, as announced for release 2.0 in May 2012,<sup class="reference" id="cite_ref-failover_66-0"><a href="#cite_note-failover-66">[66]</a></sup> letting the main metadata server (the NameNode) fail over manually to a backup. The project has also started developing automatic <a class="mw-redirect" href="/wiki/Fail-over" title="Fail-over">fail-over</a>.</p>
<p>The HDFS file system includes a so-called <i>secondary namenode</i>, a misleading name that some might incorrectly interpret as a backup namenode for when the primary namenode goes offline. In fact, the secondary namenode regularly connects with the primary namenode and builds snapshots of the primary namenode's directory information, which the system then saves to local or remote directories. These checkpointed images can be used to restart a failed primary namenode without having to replay the entire journal of file-system actions, then to edit the log to create an up-to-date directory structure. Because the namenode is the single point for storage and management of metadata, it can become a bottleneck for supporting a huge number of files, especially a large number of small files. HDFS Federation, a new addition, aims to tackle this problem to a certain extent by allowing multiple namespaces served by separate namenodes. Moreover, there are some issues in HDFS, namely, small file issue, scalability problem, Single Point of Failure (SPoF), and bottleneck in huge metadata request. An advantage of using HDFS is data awareness between the job tracker and task tracker. The job tracker schedules map or reduce jobs to task trackers with an awareness of the data location. For example: if node A contains data (x,y,z) and node B contains data (a,b,c), the job tracker schedules node B to perform map or reduce tasks on (a,b,c) and node A would be scheduled to perform map or reduce tasks on (x,y,z). This reduces the amount of traffic that goes over the network and prevents unnecessary data transfer. When Hadoop is used with other file systems, this advantage is not always available. This can have a significant impact on job-completion times, which has been demonstrated when running data-intensive jobs.<sup class="reference" id="cite_ref-67"><a href="#cite_note-67">[67]</a></sup></p>
<p>HDFS was designed for mostly immutable files<sup class="reference" id="cite_ref-openlibrary1_65-1"><a href="#cite_note-openlibrary1-65">[65]</a></sup> and may not be suitable for systems requiring concurrent write-operations.</p>
<p>HDFS can be <a href="/wiki/Mount_(computing)" title="Mount (computing)">mounted</a> directly with a <a href="/wiki/Filesystem_in_Userspace" title="Filesystem in Userspace">Filesystem in Userspace</a> (FUSE) <a href="/wiki/Virtual_file_system" title="Virtual file system">virtual file system</a> on <a href="/wiki/Linux" title="Linux">Linux</a> and some other <a href="/wiki/Unix" title="Unix">Unix</a> systems.</p>
<p>File access can be achieved through the native Java <a href="/wiki/Application_programming_interface" title="Application programming interface">application programming interface</a> (API), the <a class="mw-redirect" href="/wiki/Thrift_(protocol)" title="Thrift (protocol)">Thrift</a> API to generate a client in the language of the users' choosing (C++, Java, Python, PHP, Ruby, Erlang, Perl, Haskell, C#, <a href="/wiki/Cocoa_(API)" title="Cocoa (API)">Cocoa</a>, Smalltalk, and <a href="/wiki/OCaml" title="OCaml">OCaml</a>), the <a href="/wiki/Command-line_interface" title="Command-line interface">command-line interface</a>, browsed through the HDFS-UI <a href="/wiki/Web_application" title="Web application">Web application</a> (webapp) over <a class="mw-redirect" href="/wiki/HTTP" title="HTTP">HTTP</a>, or via 3rd-party network client libraries.<sup class="reference" id="cite_ref-68"><a href="#cite_note-68">[68]</a></sup></p>
<p>HDFS is designed for portability across various hardware platforms and compatibility with a variety of underlying operating systems. The HDFS design introduces portability limitations that result in some performance bottlenecks, since the Java implementation can't use features that are exclusive to the platform on which HDFS is running.<sup class="reference" id="cite_ref-69"><a href="#cite_note-69">[69]</a></sup> Due to its widespread integration into enterprise-level infrastructures, monitoring HDFS performance at scale has become an increasingly important issue. Monitoring end-to-end performance requires tracking metrics from datanodes, namenodes, and the underlying operating system.<sup class="reference" id="cite_ref-70"><a href="#cite_note-70">[70]</a></sup> There are currently several monitoring platforms to track HDFS performance, including HortonWorks, Cloudera, and Datadog.</p>
<h4><span class="mw-headline" id="Other_file_systems">Other file systems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=6" title="Edit section: Other file systems">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Hadoop works directly with any distributed file system that can be mounted by the underlying operating system simply by using a <code>file://</code> URL; however, this comes at a price, the loss of locality. To reduce network traffic, Hadoop needs to know which servers are closest to the data; this is information that Hadoop-specific file system bridges can provide.</p>
<p>In May 2011, the list of supported file systems bundled with Apache Hadoop were:</p>
<ul>
<li>HDFS: Hadoop's own rack-aware file system.<sup class="reference" id="cite_ref-71"><a href="#cite_note-71">[71]</a></sup> This is designed to scale to tens of petabytes of storage and runs on top of the file systems of the underlying <a href="/wiki/Operating_system" title="Operating system">operating systems</a>.</li>
<li><a class="mw-redirect" href="/wiki/FTP" title="FTP">FTP</a> File system: this stores all its data on remotely accessible FTP servers.</li>
<li><a class="mw-redirect" href="/wiki/Amazon_Simple_Storage_Service" title="Amazon Simple Storage Service">Amazon S3 (Simple Storage Service)</a> file system. This is targeted at clusters hosted on the <a href="/wiki/Amazon_Elastic_Compute_Cloud" title="Amazon Elastic Compute Cloud">Amazon Elastic Compute Cloud</a> server-on-demand infrastructure. There is no rack-awareness in this file system, as it is all remote.</li>
<li><a class="external text" href="http://azure.microsoft.com/en-us/documentation/articles/hdinsight-use-blob-storage" rel="nofollow">Windows Azure Storage Blobs (WASB)</a> file system. WASB, an extension on top of HDFS, allows distributions of Hadoop to access data in Azure blob stores without moving the data permanently into the cluster.</li>
</ul>
<p>A number of third-party file system bridges have also been written, none of which are currently in Hadoop distributions. However, some commercial distributions of Hadoop ship with an alternative filesystem as the default –  specifically IBM and MapR.</p>
<ul>
<li>In 2009, <a href="/wiki/IBM" title="IBM">IBM</a> discussed running Hadoop over the <a href="/wiki/IBM_General_Parallel_File_System" title="IBM General Parallel File System">IBM General Parallel File System</a>.<sup class="reference" id="cite_ref-72"><a href="#cite_note-72">[72]</a></sup> The source code was published in October 2009.<sup class="reference" id="cite_ref-73"><a href="#cite_note-73">[73]</a></sup></li>
<li>In April 2010, <a class="new" href="/w/index.php?title=Parascale&amp;action=edit&amp;redlink=1" title="Parascale (page does not exist)">Parascale</a> published the source code to run Hadoop against the Parascale file system.<sup class="reference" id="cite_ref-74"><a href="#cite_note-74">[74]</a></sup></li>
<li>In April 2010, <a class="new" href="/w/index.php?title=Appistry&amp;action=edit&amp;redlink=1" title="Appistry (page does not exist)">Appistry</a> released a Hadoop file system driver for use with its own CloudIQ Storage product.<sup class="reference" id="cite_ref-75"><a href="#cite_note-75">[75]</a></sup></li>
<li>In June 2010, <a href="/wiki/Hewlett-Packard" title="Hewlett-Packard">HP</a> discussed a location-aware <a href="/wiki/IBRIX_Fusion" title="IBRIX Fusion">IBRIX Fusion</a> file system driver.<sup class="reference" id="cite_ref-76"><a href="#cite_note-76">[76]</a></sup></li>
<li>In May 2011, <a href="/wiki/MapR" title="MapR">MapR</a> Technologies, Inc. announced the availability of an alternative file system for Hadoop, <a href="/wiki/MapR_FS" title="MapR FS">MapR FS</a>, which replaced the HDFS file system with a full random-access read/write file system.</li>
</ul>
<h3><span class="mw-headline" id="JobTracker_and_TaskTracker:_the_MapReduce_engine">JobTracker and TaskTracker: the MapReduce engine</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=7" title="Edit section: JobTracker and TaskTracker: the MapReduce engine">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/MapReduce" title="MapReduce">MapReduce</a></div>
<p>Above the file systems comes the <a href="/wiki/MapReduce" title="MapReduce">MapReduce</a> Engine, which consists of one <i>JobTracker</i>, to which client applications submit MapReduce jobs. The JobTracker pushes work out to available <i>TaskTracker</i> nodes in the cluster, striving to keep the work as close to the data as possible. With a rack-aware file system, the JobTracker knows which node contains the data, and which other machines are nearby. If the work cannot be hosted on the actual node where the data resides, priority is given to nodes in the same rack. This reduces network traffic on the main backbone network. If a TaskTracker fails or times out, that part of the job is rescheduled. The TaskTracker on each node spawns a separate <a class="mw-redirect" href="/wiki/Java_Virtual_Machine" title="Java Virtual Machine">Java Virtual Machine</a> process to prevent the TaskTracker itself from failing if the running job crashes its JVM. A heartbeat is sent from the TaskTracker to the JobTracker every few minutes to check its status. The Job Tracker and TaskTracker status and information is exposed by <a href="/wiki/Jetty_(web_server)" title="Jetty (web server)">Jetty</a> and can be viewed from a web browser.</p>
<p>Known limitations of this approach are:</p>
<ul>
<li>The allocation of work to TaskTrackers is very simple. Every TaskTracker has a number of available <i>slots</i> (such as "4 slots"). Every active map or reduce task takes up one slot. The Job Tracker allocates work to the tracker nearest to the data with an available slot. There is no consideration of the current <a href="/wiki/Load_(computing)" title="Load (computing)">system load</a> of the allocated machine, and hence its actual availability.</li>
<li>If one TaskTracker is very slow, it can delay the entire MapReduce job –  especially towards the end of a job, where everything can end up waiting for the slowest task. With speculative execution enabled, however, a single task can be executed on multiple slave nodes.</li>
</ul>
<h4><span class="mw-headline" id="Scheduling">Scheduling</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=8" title="Edit section: Scheduling">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>By default Hadoop uses <a href="/wiki/FIFO_(computing_and_electronics)" title="FIFO (computing and electronics)">FIFO</a> scheduling, and optionally 5 scheduling priorities to schedule jobs from a work queue.<sup class="reference" id="cite_ref-77"><a href="#cite_note-77">[77]</a></sup> In version 0.19 the job scheduler was refactored out of the JobTracker, while adding the ability to use an alternate scheduler (such as the <i>Fair scheduler</i> or the <i>Capacity scheduler</i>, described next).<sup class="reference" id="cite_ref-78"><a href="#cite_note-78">[78]</a></sup></p>
<h5><span class="mw-headline" id="Fair_scheduler">Fair scheduler</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=9" title="Edit section: Fair scheduler">edit</a><span class="mw-editsection-bracket">]</span></span></h5>
<p>The fair scheduler was developed by <a href="/wiki/Facebook" title="Facebook">Facebook</a>.<sup class="reference" id="cite_ref-79"><a href="#cite_note-79">[79]</a></sup> The goal of the fair scheduler is to provide fast response times for small jobs and <a href="/wiki/Quality_of_service" title="Quality of service">QoS</a> for production jobs. The fair scheduler has three basic concepts.<sup class="reference" id="cite_ref-80"><a href="#cite_note-80">[80]</a></sup></p>
<ol>
<li>Jobs are grouped into <a href="/wiki/Pool_(computer_science)" title="Pool (computer science)">pools</a>.</li>
<li>Each pool is assigned a guaranteed minimum share.</li>
<li>Excess capacity is split between jobs.</li>
</ol>
<p>By default, jobs that are uncategorized go into a default pool. Pools have to specify the minimum number of map slots, reduce slots, and a limit on the number of running jobs.</p>
<h5><span class="mw-headline" id="Capacity_scheduler">Capacity scheduler</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=10" title="Edit section: Capacity scheduler">edit</a><span class="mw-editsection-bracket">]</span></span></h5>
<p>The capacity scheduler was developed by <a class="mw-redirect" href="/wiki/Yahoo" title="Yahoo">Yahoo</a>. The capacity scheduler supports several features that are similar to the fair scheduler.<sup class="reference" id="cite_ref-81"><a href="#cite_note-81">[81]</a></sup></p>
<ul>
<li>Queues are allocated a fraction of the total resource capacity.</li>
<li>Free resources are allocated to queues beyond their total capacity.</li>
<li>Within a queue a job with a high level of priority has access to the queue's resources.</li>
</ul>
<p>There is no <a href="/wiki/Preemption_(computing)" title="Preemption (computing)">preemption</a> once a job is running.</p>
<h3><span class="mw-headline" id="Other_applications">Other applications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=11" title="Edit section: Other applications">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The HDFS file system is not restricted to MapReduce jobs. It can be used for other applications, many of which are under development at Apache. The list includes the <a class="mw-redirect" href="/wiki/HBase" title="HBase">HBase</a> database, the <a href="/wiki/Apache_Mahout" title="Apache Mahout">Apache Mahout</a> <a href="/wiki/Machine_learning" title="Machine learning">machine learning</a> system, and the <a href="/wiki/Apache_Hive" title="Apache Hive">Apache Hive</a> <a class="mw-redirect" href="/wiki/Data_Warehouse" title="Data Warehouse">Data Warehouse</a> system. Hadoop can in theory be used for any sort of work that is batch-oriented rather than real-time, is very data-intensive, and benefits from parallel processing of data. It can also be used to complement a real-time system, such as <a href="/wiki/Lambda_architecture" title="Lambda architecture">lambda architecture</a>, Apache Storm, Flink and Spark Streaming.<sup class="reference" id="cite_ref-82"><a href="#cite_note-82">[82]</a></sup></p>
<p>As of October 2009<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Apache_Hadoop&amp;action=edit">[update]</a></sup>, commercial applications of Hadoop<sup class="reference" id="cite_ref-83"><a href="#cite_note-83">[83]</a></sup> included:-</p>
<ul>
<li>log and/or clickstream analysis of various kinds</li>
<li>marketing analytics</li>
<li>machine learning and/or sophisticated data mining</li>
<li>image processing</li>
<li>processing of XML messages</li>
<li>web crawling and/or text processing</li>
<li>general archiving, including of relational/tabular data, e.g. for compliance</li>
</ul>
<h2><span class="mw-headline" id="Prominent_users">Prominent users</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=12" title="Edit section: Prominent users">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>On February 19, 2008, <a class="mw-redirect" href="/wiki/Yahoo!_Inc." title="Yahoo! Inc.">Yahoo! Inc.</a> launched what it claimed was the world's largest Hadoop production application. The Yahoo! Search Webmap is a Hadoop application that runs on a Linux cluster with more than 10,000 <a class="mw-redirect" href="/wiki/Multi-core" title="Multi-core">cores</a> and produced data that was used in every Yahoo! web search query.<sup class="reference" id="cite_ref-84"><a href="#cite_note-84">[84]</a></sup> There are multiple Hadoop clusters at Yahoo! and no HDFS file systems or MapReduce jobs are split across multiple data centers. Every Hadoop cluster node bootstraps the Linux image, including the Hadoop distribution. Work that the clusters perform is known to include the index calculations for the Yahoo! search engine. In June 2009, Yahoo! made the source code of the Hadoop version it runs available to the public via the open-source community.<sup class="reference" id="cite_ref-85"><a href="#cite_note-85">[85]</a></sup></p>
<p>In 2010, <a href="/wiki/Facebook" title="Facebook">Facebook</a> claimed that they had the largest Hadoop cluster in the world with 21 <a href="/wiki/Petabyte" title="Petabyte">PB</a> of storage.<sup class="reference" id="cite_ref-86"><a href="#cite_note-86">[86]</a></sup> In June 2012, they announced the data had grown to 100 PB<sup class="reference" id="cite_ref-87"><a href="#cite_note-87">[87]</a></sup> and later that year they announced that the data was growing by roughly half a PB per day.<sup class="reference" id="cite_ref-88"><a href="#cite_note-88">[88]</a></sup></p>
<p>As of 2013<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Apache_Hadoop&amp;action=edit">[update]</a></sup>, Hadoop adoption had become widespread: more than half of the Fortune 50 used Hadoop.<sup class="reference" id="cite_ref-89"><a href="#cite_note-89">[89]</a></sup></p>
<h2><span class="mw-headline" id="Hadoop_hosting_in_the_cloud">Hadoop hosting in the cloud</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=13" title="Edit section: Hadoop hosting in the cloud">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Hadoop can be deployed in a traditional onsite datacenter as well as in <a href="/wiki/Cloud_computing" title="Cloud computing">the cloud</a>.<sup class="reference" id="cite_ref-90"><a href="#cite_note-90">[90]</a></sup> The cloud allows organizations to deploy Hadoop without hardware to acquire or specific setup expertise.<sup class="reference" id="cite_ref-91"><a href="#cite_note-91">[91]</a></sup> Vendors who currently have an offer for the cloud include <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>, <a href="/wiki/Amazon.com" title="Amazon.com">Amazon</a>, <a href="/wiki/IBM" title="IBM">IBM</a>,<sup class="reference" id="cite_ref-92"><a href="#cite_note-92">[92]</a></sup> <a href="/wiki/Google" title="Google">Google</a> and <a href="/wiki/Oracle_Corporation" title="Oracle Corporation">Oracle</a>.<sup class="reference" id="cite_ref-93"><a href="#cite_note-93">[93]</a></sup></p>
<h3><span class="mw-headline" id="On_Microsoft_Azure">On Microsoft Azure</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=14" title="Edit section: On Microsoft Azure">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Azure HDInsight<sup class="reference" id="cite_ref-azure.microsoft.com_94-0"><a href="#cite_note-azure.microsoft.com-94">[94]</a></sup> is a service that deploys Hadoop on <a href="/wiki/Microsoft_Azure" title="Microsoft Azure">Microsoft Azure</a>. HDInsight uses <a href="/wiki/Hortonworks" title="Hortonworks">Hortonworks</a> HDP and was jointly developed for HDI with <a href="/wiki/Hortonworks" title="Hortonworks">Hortonworks</a>. HDI allows programming extensions with .NET (in addition to Java). HDInsight also supports creation of Hadoop clusters using Linux with Ubuntu.<sup class="reference" id="cite_ref-azure.microsoft.com_94-1"><a href="#cite_note-azure.microsoft.com-94">[94]</a></sup> By deploying HDInsight in the cloud, organizations can spin up the number of nodes they want and only get charged for the compute and storage that is used.<sup class="reference" id="cite_ref-azure.microsoft.com_94-2"><a href="#cite_note-azure.microsoft.com-94">[94]</a></sup> <a href="/wiki/Hortonworks" title="Hortonworks">Hortonworks</a> implementations can also move data from the on-premises datacenter to the cloud for backup, development/test, and bursting scenarios.<sup class="reference" id="cite_ref-azure.microsoft.com_94-3"><a href="#cite_note-azure.microsoft.com-94">[94]</a></sup> It is also possible to run Cloudera or Hortonworks Hadoop clusters on Azure Virtual Machines.</p>
<h3><span class="mw-headline" id="On_Amazon_EC2.2FS3_services">On Amazon EC2/S3 services</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=15" title="Edit section: On Amazon EC2/S3 services">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>It is possible to run Hadoop on <a href="/wiki/Amazon_Elastic_Compute_Cloud" title="Amazon Elastic Compute Cloud">Amazon Elastic Compute Cloud</a> (EC2) and <a class="mw-redirect" href="/wiki/Amazon_Simple_Storage_Service" title="Amazon Simple Storage Service">Amazon Simple Storage Service</a> (S3).<sup class="reference" id="cite_ref-95"><a href="#cite_note-95">[95]</a></sup> As an example, <a href="/wiki/The_New_York_Times" title="The New York Times">The New York Times</a> used 100 Amazon EC2 instances and a Hadoop application to process 4 TB of raw image <a href="/wiki/TIFF" title="TIFF">TIFF</a> data (stored in S3) into 11 million finished <a class="mw-redirect" href="/wiki/PDF" title="PDF">PDFs</a> in the space of 24 hours at a computation cost of about $240 (not including bandwidth).<sup class="reference" id="cite_ref-96"><a href="#cite_note-96">[96]</a></sup></p>
<p>There is support for the S3 object store in the Apache Hadoop releases, though this is below what one expects from a traditional POSIX filesystem. Specifically, operations such as rename() and delete() on directories are not atomic, and can take time proportional to the number of entries and the amount of data in them.</p>
<h3><span class="mw-headline" id="Amazon_Elastic_MapReduce">Amazon Elastic MapReduce</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=16" title="Edit section: Amazon Elastic MapReduce">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Elastic MapReduce (EMR)<sup class="reference" id="cite_ref-97"><a href="#cite_note-97">[97]</a></sup> was introduced by <a href="/wiki/Amazon.com" title="Amazon.com">Amazon.com</a> in April 2009. Provisioning of the Hadoop cluster, running and terminating jobs, and handling data transfer between EC2(VM) and S3(Object Storage) are automated by Elastic MapReduce. <a href="/wiki/Apache_Hive" title="Apache Hive">Apache Hive</a>, which is built on top of Hadoop for providing data warehouse services, is also offered in Elastic MapReduce.<sup class="reference" id="cite_ref-98"><a href="#cite_note-98">[98]</a></sup></p>
<p>Support for using Spot Instances<sup class="reference" id="cite_ref-99"><a href="#cite_note-99">[99]</a></sup> was later added in August 2011.<sup class="reference" id="cite_ref-100"><a href="#cite_note-100">[100]</a></sup> Elastic MapReduce is fault-tolerant for slave failures,<sup class="reference" id="cite_ref-101"><a href="#cite_note-101">[101]</a></sup> and it is recommended to only run the Task Instance Group on spot instances to take advantage of the lower cost while maintaining availability.<sup class="reference" id="cite_ref-102"><a href="#cite_note-102">[102]</a></sup></p>
<h3><span class="mw-headline" id="On_CenturyLink_Cloud_.28CLC.29">On CenturyLink Cloud (CLC)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=17" title="Edit section: On CenturyLink Cloud (CLC)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>CenturyLink Cloud<sup class="reference" id="cite_ref-103"><a href="#cite_note-103">[103]</a></sup> offers Hadoop via both a managed and un-managed model via their Hadoop<sup class="reference" id="cite_ref-104"><a href="#cite_note-104">[104]</a></sup> offering. CLC also offers customers several managed Cloudera Blueprints, the newest managed service in the CenturyLink Cloud big data Blueprints portfolio, which also includes Cassandra and MongoDB solutions.<sup class="reference" id="cite_ref-105"><a href="#cite_note-105">[105]</a></sup></p>
<h3><span class="mw-headline" id="Google_Cloud_Platform">Google Cloud Platform</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=18" title="Edit section: Google Cloud Platform">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>There are multiple ways to run the Hadoop ecosystem on <a href="/wiki/Google_Cloud_Platform" title="Google Cloud Platform">Google Cloud Platform</a> ranging from self-managed to Google-managed.<sup class="reference" id="cite_ref-106"><a href="#cite_note-106">[106]</a></sup></p>
<ul>
<li><a href="/wiki/Google_Cloud_Dataproc" title="Google Cloud Dataproc">Google Cloud Dataproc</a> – a managed Spark and Hadoop service<sup class="reference" id="cite_ref-107"><a href="#cite_note-107">[107]</a></sup></li>
<li><a class="external text" href="https://cloud.google.com/hadoop/bdutil" rel="nofollow">command line tools (bdutil)</a> – a collection of shell scripts to manually create and manage Spark and Hadoop clusters<sup class="reference" id="cite_ref-108"><a href="#cite_note-108">[108]</a></sup></li>
<li>third party Hadoop distributions:-
<ul>
<li>Cloudera – using the Cloudera Director Plugin for Google Cloud Platform<sup class="reference" id="cite_ref-109"><a href="#cite_note-109">[109]</a></sup></li>
<li>Hortonworks – using bdutil support for Hortonworks HDP<sup class="reference" id="cite_ref-110"><a href="#cite_note-110">[110]</a></sup></li>
<li>MapR – using bdutil support for MapR<sup class="reference" id="cite_ref-111"><a href="#cite_note-111">[111]</a></sup></li>
</ul>
</li>
</ul>
<p>Google also offers connectors for using other Google Cloud Platform products with Hadoop, such as a <a class="external text" href="https://www.mapr.com/resources/mapr-google-cloud-platform" rel="nofollow">Google Cloud Storage connector</a> for using <a href="/wiki/Google_Storage" title="Google Storage">Google Cloud Storage</a> and a <a class="external text" href="https://cloud.google.com/hadoop/bigquery-connector" rel="nofollow">Google BigQuery connector</a> for using <a href="/wiki/BigQuery" title="BigQuery">Google BigQuery</a>.</p>

<p>A number of companies offer commercial implementations or support for Hadoop.<sup class="reference" id="cite_ref-112"><a href="#cite_note-112">[112]</a></sup></p>
<h3><span class="mw-headline" id="Branding">Branding</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=20" title="Edit section: Branding">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The Apache Software Foundation has stated that only software officially released by the Apache Hadoop Project can be called <i>Apache Hadoop</i> or <i>Distributions of Apache Hadoop</i>.<sup class="reference" id="cite_ref-113"><a href="#cite_note-113">[113]</a></sup> The naming of products and derivative works from other vendors and the term "compatible" are somewhat controversial within the Hadoop developer community.<sup class="reference" id="cite_ref-114"><a href="#cite_note-114">[114]</a></sup></p>
<h2><span class="mw-headline" id="Papers">Papers</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Apache_Hadoop&amp;action=edit&amp;section=21" title="Edit section: Papers">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Some papers influenced the birth and growth of Hadoop and big data processing. Here is a partial list:</p>
<ul>
<li>Jeffrey Dean, Sanjay Ghemawat (2004) <a class="external text" href="https://www.usenix.org/legacy/publications/library/proceedings/osdi04/tech/full_papers/dean/dean_html/index.html" rel="nofollow">MapReduce: Simplified Data Processing on Large Clusters</a>, Google. This paper inspired Doug Cutting to develop an open-source implementation of the Map-Reduce framework. He named it Hadoop, after his son's toy elephant.</li>
<li>Michael Franklin, Alon Halevy, David Maier (2005) <a class="external text" href="http://www.eecs.berkeley.edu/~franklin/Papers/dataspaceSR.pdf" rel="nofollow">From Databases to Dataspaces: A New Abstraction for Information Management</a>. The authors highlight the need for storage systems to accept all data formats and to provide APIs for data access that evolve based on the storage system’s understanding of the data.</li>
<li>Fay Chang et al. (2006) <a class="external text" href="http://static.googleusercontent.com/external_content/untrusted_dlcp/research.google.com/en/us/archive/bigtable-osdi06.pdf" rel="nofollow">Bigtable: A Distributed Storage System for Structured Data</a>, Google.</li>
<li>Robert Kallman et al. (2008) <a class="external text" href="http://www.vldb.org/pvldb/1/1454211.pdf" rel="nofollow">H-store: a high-performance, distributed main memory transaction processing system</a></li>
</ul>
