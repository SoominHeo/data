<p>The <b>Data Plane Development Kit</b> (<b>DPDK</b>) is a set of <a class="mw-redirect" href="/wiki/Data_plane" title="Data plane">data plane</a> libraries and <a href="/wiki/Network_interface_controller" title="Network interface controller">network interface controller</a> drivers for fast <a href="/wiki/Packet_processing" title="Packet processing">packet processing</a>. The DPDK provides a programming framework for <a class="mw-redirect" href="/wiki/Intel_x86" title="Intel x86">Intel x86</a> processors and enables faster development of high speed data packet networking applications.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> It scales from <a href="/wiki/Intel_Atom" title="Intel Atom">Intel Atom</a> processors to <a class="mw-redirect" href="/wiki/Intel_Xeon" title="Intel Xeon">Intel Xeon</a> processors. Support for other <a class="mw-redirect" href="/wiki/Processor_architecture" title="Processor architecture">processor architectures</a> such as IBM <a href="/wiki/POWER8" title="POWER8">POWER8</a> <a class="mw-redirect" href="/wiki/EZchip" title="EZchip">EZchip</a> <a href="/wiki/TILE-Gx" title="TILE-Gx">TILE-Gx</a> and <a class="mw-redirect mw-disambig" href="/wiki/ARM" title="ARM">ARM</a> have been added.<sup class="reference" id="cite_ref-DPDK-POWER8_3-0"><a href="#cite_note-DPDK-POWER8-3">[3]</a></sup> It is provided and supported under the open source<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> <a class="mw-redirect" href="/wiki/BSD_license" title="BSD license">BSD license</a>.</p>

<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Data_Plane_Development_Kit&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The DPDK framework creates a set of libraries for specific hardware/software environments through the creation of an Environment Abstraction Layer (EAL).<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> The EAL hides the environmental specific and provides a standard programming interface to libraries, available hardware accelerators and other hardware and operating system (Linux, FreeBSD) elements. Once the EAL is created for a specific environment, developers link to the library to create their applications. For instance, EAL provides the frameworks to support Linux, FreeBSD, Intel IA 32- or 64-bit, IBM Power8, EZchip TILE-Gx and ARM.</p>
<p>The EAL also provides additional services including time references, <a class="mw-redirect" href="/wiki/PCIe" title="PCIe">PCIe</a> bus access, trace and debug functions and alarm operations.</p>
<p>The DPDK implements a low overhead <a class="mw-redirect" href="/wiki/Run-to-completion" title="Run-to-completion">run-to-completion</a> model for fast data plane performance and accesses devices via <a href="/wiki/Polling_(computer_science)" title="Polling (computer science)">polling</a> to eliminate the performance overhead of <a href="/wiki/Interrupt" title="Interrupt">interrupt</a> processing.</p>
<p>The DPDK also includes software examples that highlight best practices for software architecture, tips for data structure design and storage, application profiling and performance tuning utilities and tips that address common network performance deficits.</p>
<h2><span class="mw-headline" id="Libraries">Libraries</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Data_Plane_Development_Kit&amp;action=edit&amp;section=2" title="Edit section: Libraries">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The DPDK includes data plane libraries and optimized NIC drivers for the following:<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<ul>
<li>A queue manager implements lockless queues</li>
<li>A buffer manager pre-allocates fixed size buffers</li>
<li>A memory manager allocates pools of objects in memory and uses a ring to store free objects; ensures that objects are spread equally on all DRAM channels</li>
<li>Poll mode drivers (PMD) are designed to work without asynchronous notifications, reducing overhead</li>
<li>A packet framework – set of libraries that are helpers to develop packet processing</li>
</ul>
<p>All libraries are stored in the dpdk/lib/librte_* directories</p>
<h3><span class="mw-headline" id="Plugins">Plugins</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Data_Plane_Development_Kit&amp;action=edit&amp;section=3" title="Edit section: Plugins">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The EAL allows loading some plugins using the <tt>-d file.so</tt> option without recompiling any applications that use the DPDK libraries. The following plugins are available:</p>
<ul>
<li><tt><a class="external text" href="http://dpdk.org/browse/virtio-net-pmd/refs/" rel="nofollow">librte_pmd_virtio.so</a></tt> –  provides PMD Ethernet layer supporting Virtio paravirtualized NIC</li>
<li><tt><a class="external text" href="http://www.dpdk.org/browse/vmxnet3-usermap/refs/" rel="nofollow">librte_pmd_vmxnet3.so</a></tt> –  provides PMD Ethernet layer supporting Vmxnet3 paravitualized NIC</li>
<li><tt><a class="external text" href="http://www.dpdk.org/browse/memnic/refs/" rel="nofollow">librte_pmd_memnic_copy.so</a></tt> –  provides a Virtual PMD Ethernet layer through shared memory based on 2 memory copies of packets</li>
<li><tt>librte_pmd_mlx4.so</tt> –  provides PMD Ethernet layer for Mellanox ConnectX-3 40G NIC</li>
<li><tt>librte_crypto_nitrox.so</tt> –  provides cryptography layer for Cavium Nitrox PCI boards</li>
<li><tt>librte_crypto_quickassist.so</tt> –  provides cryptography layer for Intel Colleto Creek PCI boards.</li>
</ul>
<h2><span class="mw-headline" id="Environment">Environment</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Data_Plane_Development_Kit&amp;action=edit&amp;section=4" title="Edit section: Environment">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The DPDK was originally designed to run using a bare-metal mode which is currently deprecated. Actually, DPDK's EAL provides support for <a href="/wiki/Linux" title="Linux">Linux</a> or <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a> <a href="/wiki/User_space" title="User space">userland application</a>.</p>
<p>EAL can be extended in order to support any processors.</p>

<h2><span class="mw-headline" id="Projects">Projects</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Data_Plane_Development_Kit&amp;action=edit&amp;section=6" title="Edit section: Projects">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Opensource">Opensource</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Data_Plane_Development_Kit&amp;action=edit&amp;section=7" title="Edit section: Opensource">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/PfSense" title="PfSense">pfSense</a> project published a road map on 25 February 2015, in which developer Jim Thompson announced the rewriting of the pfSense core—including <a href="/wiki/PF_(firewall)" title="PF (firewall)">pf</a>, <a href="/wiki/Network_packet" title="Network packet">network packet</a> forwarding and shaping, <a class="mw-redirect" href="/wiki/Link_bonding" title="Link bonding">link bonding</a>, <a href="/wiki/IPsec" title="IPsec">IPsec</a>—using Intel's DPDK: "We have a goal of being able to forward, with packet filtering at rates of at least 14.88Mpps. This is 'line rate' on a 10Gbps interface. There is simply no way to use today's FreeBSD (or linux) in-kernel stacks for this type of load."<sup class="reference" id="cite_ref-pFSenseRoadmap_15-0"><a href="#cite_note-pFSenseRoadmap-15">[15]</a></sup></p>
<p><a href="/wiki/Open_vSwitch" title="Open vSwitch">OVS</a> has a limited set of feature running userland that can be leveraged to bypass the Linux kernel OVS processing. This use case of OVS with DPDK userland is usually named OVS-DPDK. It is mostly deployed with Openstack Neutron but it assumes that many features and SDN capabilities of Openstack are disabled. For instance, when OVS-DPDK is used, Neutron provides a lower lever of security than when OVS kernel is used (no stateful firewalling, less security group).</p>
<p>The FD.IO VPP platform is an extensible framework that provides out-of-the-box production quality switch/router functionality. It is the open source version of Cisco's Vector Packet Processing (VPP) technology: a high performance, packet-processing stack that can run on commodity CPUs, and can leverage the Poll Mode Drivers for both NICs and cryptographic acceleration hardware and libraries. <sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></p>
<p>TRex is an open source, low cost, stateful and stateless traffic generator fueled by DPDK. It generates L4-7 traffic based on pre-processing and smart replay of real traffic templates. TRex amplifies both client and server side traffic and can scale to 200Gb/sec with one UCS using Intel XL710. TRex also supports multiple streams, ability to change any packet field and provides per stream statistics, latency and jitter. <sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<p>It has been estimated that more than 20 opensource projects are now using DPDK in various fashions.</p>
<h3><span class="mw-headline" id="Platforms_and_solutions">Platforms and solutions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Data_Plane_Development_Kit&amp;action=edit&amp;section=8" title="Edit section: Platforms and solutions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Since DPDK was launched, very quickly many platforms have integrated this userland library for some IOs. The platforms are:</p>
<ul>
<li><a href="/wiki/6WIND" title="6WIND">6WIND</a> 6WINDGate, Virtual Accelerator and their Turbo Series which provide the highest performance and the widest set of features leveraging the DPDK.</li>
<li>OVS-DPDK from <a href="/wiki/Open_vSwitch" title="Open vSwitch">OVS</a> which provides a subset of OVS support.</li>
</ul>
