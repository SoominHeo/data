<h1 class="firstHeading" id="firstHeading" lang="en">Delete (SQL)</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p>In the database structured query language (<a href="/wiki/SQL" title="SQL">SQL</a>), the <b>DELETE</b> statement removes one or more records from a <a href="/wiki/Table_(database)" title="Table (database)">table</a>. A subset may be defined for deletion using a condition, otherwise all records are removed.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> Some DBMSs, like MySQL, allow deletion of rows from multiple tables with one DELETE statement (this is sometimes called multi-table DELETE).</p>
<p></p>

<h2><span class="mw-headline" id="Usage">Usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Delete_(SQL)&amp;action=edit&amp;section=1" title="Edit section: Usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The <code>DELETE</code> statement follows the syntax:</p>
<dl>
<dd><b><code>DELETE</code></b> <code>FROM</code> <i>table_name</i> [<b><code>WHERE</code></b> condition];</dd>
</dl>
<p>Any rows that match the <a href="/wiki/Where_(SQL)" title="Where (SQL)"><code>WHERE</code> condition</a> will be removed from the table. If the <code>WHERE</code> clause is omitted, all rows in the table are removed. The <code>DELETE</code> statement should thus be used with caution.</p>
<p>The <code>DELETE</code> statement does not return any rows; that is, it will not generate a <a href="/wiki/Result_set" title="Result set">result set</a>.</p>
<p>Executing a <code>DELETE</code> statement can cause <a href="/wiki/Database_trigger" title="Database trigger">triggers</a> to run that can cause deletes in other tables. For example, if two tables are linked by a <a href="/wiki/Foreign_key" title="Foreign key">foreign key</a> and rows in the referenced table are deleted, then it is common that rows in the referencing table would also have to be deleted to maintain <a href="/wiki/Referential_integrity" title="Referential integrity">referential integrity</a>.</p>
<div class="hatnote" role="note">See also: <a href="/wiki/Propagation_constraint" title="Propagation constraint">propagation constraint</a></div>
<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Delete_(SQL)&amp;action=edit&amp;section=2" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Delete rows from table <i>pies</i> where the column <i>flavour</i> equals <i>Lemon Meringue</i>:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">DELETE</span> <span class="k">FROM</span> <span class="n">pies</span>
 <span class="k">WHERE</span> <span class="n">flavour</span><span class="o">=</span><span class="s1">'Lemon Meringue'</span><span class="p">;</span>
</pre></div>
<p>Delete rows in <i>trees</i>, if the value of <i>height</i> is smaller than 80.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">DELETE</span> <span class="k">FROM</span> <span class="n">trees</span>
 <span class="k">WHERE</span> <span class="n">height</span> <span class="o">&lt;</span> <span class="mi">80</span><span class="p">;</span>
</pre></div>
<p>Delete all rows from <i>mytable</i>:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">DELETE</span> <span class="k">FROM</span> <span class="n">mytable</span><span class="p">;</span>
</pre></div>
<p>Delete rows from <i>mytable</i> using a subquery in the where condition:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">DELETE</span> <span class="k">FROM</span> <span class="n">mytable</span>
 <span class="k">WHERE</span> <span class="n">id</span> <span class="k">IN</span> <span class="p">(</span>
       <span class="k">SELECT</span> <span class="n">id</span>
         <span class="k">FROM</span> <span class="n">mytable2</span>
      <span class="p">);</span>
</pre></div>
<p>Delete rows from <i>mytable</i> using a list of values:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">DELETE</span> <span class="k">FROM</span> <span class="n">mytable</span>
 <span class="k">WHERE</span> <span class="n">id</span> <span class="k">IN</span> <span class="p">(</span>
       <span class="n">value1</span><span class="p">,</span>
       <span class="n">value2</span><span class="p">,</span>
       <span class="n">value3</span><span class="p">,</span>
       <span class="n">value4</span><span class="p">,</span>
       <span class="n">value5</span>
      <span class="p">);</span>
</pre></div>
<h2><span class="mw-headline" id="Example_with_related_tables">Example with related tables</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Delete_(SQL)&amp;action=edit&amp;section=3" title="Edit section: Example with related tables">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Suppose there is a simple database that lists people and addresses. More than one person can live at a particular address and a person can live at more than one address (this is an example of a <a href="/wiki/Many-to-many_(data_model)" title="Many-to-many (data model)">many-to-many relationship</a>). The database only has three tables, <i>person</i>, <i>address</i>, and <i>pa</i>, with the following data:</p>
<table>
<tr valign="top">
<td>
<table border="1" class="wikitable">
<caption>person</caption>
<tr>
<th>pid</th>
<th>name</th>
</tr>
<tr>
<td>1</td>
<td>Joe</td>
</tr>
<tr>
<td>2</td>
<td>Bob</td>
</tr>
<tr>
<td>3</td>
<td>Ann</td>
</tr>
</table>
</td>
<td>
<table border="1" class="wikitable">
<caption>address</caption>
<tr>
<th>aid</th>
<th>description</th>
</tr>
<tr>
<td>100</td>
<td>2001 Main St.</td>
</tr>
<tr>
<td>200</td>
<td>35 Pico Blvd.</td>
</tr>
</table>
</td>
<td>
<table border="1" class="wikitable">
<caption>pa</caption>
<tr>
<th>pid</th>
<th>aid</th>
</tr>
<tr>
<td>1</td>
<td>100</td>
</tr>
<tr>
<td>2</td>
<td>100</td>
</tr>
<tr>
<td>3</td>
<td>100</td>
</tr>
<tr>
<td>1</td>
<td>200</td>
</tr>
</table>
</td>
</tr>
</table>
<p>The <i>pa</i> table relates the person and address tables, showing that Joe, Bob and Ann all live at 2001 Main Street, but Joe also takes up residence on Pico Boulevard.</p>
<p>In order to remove joe from the database, two deletes must be executed:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
 <span class="k">DELETE</span> <span class="k">FROM</span> <span class="n">person</span> <span class="k">WHERE</span> <span class="n">pid</span><span class="o">=</span><span class="mi">1</span><span class="p">;</span>
 <span class="k">DELETE</span> <span class="k">FROM</span> <span class="n">pa</span> <span class="k">WHERE</span> <span class="n">pid</span><span class="o">=</span><span class="mi">1</span><span class="p">;</span>
</pre></div>
<p>To maintain referential integrity, Joe's records must be removed from both <i>person</i> and <i>pa</i>. The means by which integrity is sustained can happen differently in varying relational database management systems<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (February 2007)">citation needed</span></a></i>]</sup>. It could be that beyond just having three tables, the database also has been set up with a trigger so that whenever a row is deleted from <i>person</i> any linked rows would be deleted from <i>pa</i>. Then the first statement:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
 <span class="k">DELETE</span> <span class="k">FROM</span> <span class="n">person</span> <span class="k">WHERE</span> <span class="n">pid</span><span class="o">=</span><span class="mi">1</span><span class="p">;</span>
</pre></div>
<p>would <i>automatically</i> trigger the second:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
 <span class="k">DELETE</span> <span class="k">FROM</span> <span class="n">pa</span> <span class="k">WHERE</span> <span class="n">pid</span><span class="o">=</span><span class="mi">1</span><span class="p">;</span>
</pre></div>
<h2><span class="mw-headline" id="Related_commands">Related commands</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Delete_(SQL)&amp;action=edit&amp;section=4" title="Edit section: Related commands">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Deleting all rows from a table can be very time consuming. Some <a class="mw-redirect" href="/wiki/DBMS" title="DBMS">DBMS</a><sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (September 2009)">clarification needed</span></a></i>]</sup> offer a <a href="/wiki/Truncate_(SQL)" title="Truncate (SQL)">TRUNCATE TABLE</a> command that works a lot quicker, as it only alters metadata and typically does not spend time enforcing constraints or firing triggers.</p>
<p>DELETE only deletes the rows. For deleting a table entirely the <a class="mw-redirect" href="/wiki/Drop_(SQL)" title="Drop (SQL)">DROP</a> command can be used.</p>
