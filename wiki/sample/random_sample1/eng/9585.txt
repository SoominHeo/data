<p>Adobe <b>ColdFusion</b> is a commercial <a href="/wiki/Rapid_application_development" title="Rapid application development">rapid web application development</a> <a href="/wiki/Computing_platform" title="Computing platform">platform</a> created by <a href="/wiki/Joseph_J._Allaire" title="Joseph J. Allaire">JJ Allaire</a> in 1995.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> (The <a href="/wiki/Programming_language" title="Programming language">programming language</a> used with that platform is also commonly called ColdFusion, though is more accurately known as <a class="mw-redirect" href="/wiki/CFML" title="CFML">CFML</a>.) ColdFusion was originally designed to make it easier to connect simple <a href="/wiki/HTML" title="HTML">HTML</a> pages to a <a href="/wiki/Database" title="Database">database</a>. By Version 2 (1996), it became a full platform that included an <a href="/wiki/Integrated_development_environment" title="Integrated development environment">IDE</a> in addition to a full <a href="/wiki/Scripting_language" title="Scripting language">scripting language</a>.</p>

<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>One of the distinguishing features of ColdFusion is its associated scripting language, <a href="/wiki/ColdFusion_Markup_Language" title="ColdFusion Markup Language">ColdFusion Markup Language</a> (CFML). CFML compares to the scripting components of <a href="/wiki/Active_Server_Pages" title="Active Server Pages">ASP</a>, <a href="/wiki/JavaServer_Pages" title="JavaServer Pages">JSP</a>, and <a href="/wiki/PHP" title="PHP">PHP</a> in purpose and features, but its tag syntax more closely resembles <a href="/wiki/HTML" title="HTML">HTML</a>, while its script syntax resembles <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>. <i>ColdFusion</i> is often used synonymously with <i><a href="/wiki/ColdFusion_Markup_Language" title="ColdFusion Markup Language">CFML</a></i>, but there are <a href="#Alternative_server_environments">additional CFML application servers</a> besides ColdFusion, and ColdFusion supports programming languages other than CFML, such as server-side <a class="mw-redirect" href="/wiki/Actionscript" title="Actionscript">Actionscript</a> and embedded scripts that can be written in a <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>-like language known as <a href="/wiki/CFScript" title="CFScript">CFScript</a>.</p>
<p>Originally a product of <a href="/wiki/Allaire_Corporation" title="Allaire Corporation">Allaire</a> and released on July 2, 1995, ColdFusion was developed by brothers <a href="/wiki/Joseph_J._Allaire" title="Joseph J. Allaire">Joseph J. "JJ"</a> and <a href="/wiki/Jeremy_Allaire" title="Jeremy Allaire">Jeremy Allaire</a>. In 2001 <a href="/wiki/Allaire_Corporation" title="Allaire Corporation">Allaire</a> was acquired by <a href="/wiki/Macromedia" title="Macromedia">Macromedia</a>, which in turn was acquired by <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe Systems</a> Inc in 2005.</p>
<p>ColdFusion is most often used for data-driven <a href="/wiki/Website" title="Website">websites</a> or <a href="/wiki/Intranet" title="Intranet">intranets</a>, but can also be used to generate remote services such as <a class="mw-redirect" href="/wiki/REST" title="REST">REST</a> services, <a class="mw-redirect" href="/wiki/Websockets" title="Websockets">websockets</a>, <a href="/wiki/SOAP" title="SOAP">SOAP</a> <a href="/wiki/Web_service" title="Web service">web services</a> or <a href="/wiki/Adobe_Flash" title="Adobe Flash">Flash</a> remoting. It is especially well-suited as the server-side technology to the client-side <a class="mw-disambig" href="/wiki/Ajax" title="Ajax">ajax</a>.</p>
<p>ColdFusion can also handle asynchronous events such as <a class="mw-redirect" href="/wiki/Short_message_service" title="Short message service">SMS</a> and <a href="/wiki/Instant_messaging" title="Instant messaging">instant messaging</a> via its gateway interface, available in ColdFusion MX 7 Enterprise Edition.</p>
<h3><span class="mw-headline" id="Main_features">Main features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=2" title="Edit section: Main features">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>ColdFusion provides a number of additional features out of the box. The main things are:</p>
<ul>
<li>Simplified database access</li>
<li>Client and server cache management</li>
<li>Client-side code generation, especially for form <a class="mw-redirect" href="/wiki/Widget_(computing)" title="Widget (computing)">widgets</a> and validation</li>
<li>Conversion from <a class="mw-redirect" href="/wiki/HyperText_Markup_Language" title="HyperText Markup Language">HTML</a> to <a href="/wiki/Portable_Document_Format" title="Portable Document Format">PDF</a></li>
<li>Data retrieval from common enterprise systems such as <a href="/wiki/Active_Directory" title="Active Directory">Active Directory</a>, <a href="/wiki/Lightweight_Directory_Access_Protocol" title="Lightweight Directory Access Protocol">LDAP</a>, <a href="/wiki/Simple_Mail_Transfer_Protocol" title="Simple Mail Transfer Protocol">SMTP</a>, <a href="/wiki/Post_Office_Protocol" title="Post Office Protocol">POP</a>, <a class="mw-redirect" href="/wiki/HyperText_Transfer_Protocol" title="HyperText Transfer Protocol">HTTP</a>, <a href="/wiki/File_Transfer_Protocol" title="File Transfer Protocol">FTP</a>, <a href="/wiki/Microsoft_Exchange_Server" title="Microsoft Exchange Server">Microsoft Exchange Server</a> and common data formats such as <a href="/wiki/RSS" title="RSS">RSS</a> and <a href="/wiki/Atom_(standard)" title="Atom (standard)">Atom</a></li>
<li>File <a class="mw-redirect" href="/wiki/Index_(database)" title="Index (database)">indexing</a> and searching service based on <a href="/wiki/Apache_Solr" title="Apache Solr">Apache Solr</a></li>
<li><a href="/wiki/Graphical_user_interface" title="Graphical user interface">GUI</a> administration</li>
<li>Server, application, client, session, and request <a class="mw-redirect" href="/wiki/Scope_(programming)" title="Scope (programming)">scopes</a></li>
<li><a href="/wiki/XML" title="XML">XML</a> parsing, <a href="/wiki/XPath" title="XPath">querying (XPath)</a>, validation and <a href="/wiki/XSLT" title="XSLT">transformation (XSLT)</a></li>
<li>Server clustering</li>
<li>Task scheduling</li>
<li><a href="/wiki/Chart" title="Chart">Graphing</a> and <a class="mw-redirect" href="/wiki/Enterprise_reporting" title="Enterprise reporting">reporting</a></li>
<li>Simplified file manipulation including <a href="/wiki/Raster_graphics" title="Raster graphics">raster graphics</a> (and <a href="/wiki/CAPTCHA" title="CAPTCHA">CAPTCHA</a>) and <a class="mw-redirect" href="/wiki/ZIP_(file_format)" title="ZIP (file format)">zip archives</a> (introduction of video manipulation is planned in a future release)</li>
<li>Simplified <a href="/wiki/Web_service" title="Web service">web service</a> implementation (with automated <a class="mw-redirect" href="/wiki/Web_services_description_language" title="Web services description language">WSDL</a> generation / transparent <a class="mw-redirect" href="/wiki/SOAP_(protocol)" title="SOAP (protocol)">SOAP</a> handling for both creating and consuming services - as an example, <a href="/wiki/ASP.NET" title="ASP.NET">ASP.NET</a><sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> has no native equivalent for <code class="mw-highlight" dir="ltr"><span class="p">&lt;</span><span class="nt">CFINVOKE</span> <span class="na">WEBSERVICE</span><span class="o">=</span><span class="s">"http://host/tempconf.cfc?wsdl"</span> <span class="na">METHOD</span><span class="o">=</span><span class="s">"Celsius2Fahrenheit"</span> <span class="na">TEMP</span><span class="o">=</span><span class="s">"#tempc#"</span> <span class="na">RETURNVARIABLE</span><span class="o">=</span><span class="s">"tempf"</span><span class="p">&gt;</span></code><sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup>)</li>
</ul>
<p>Other implementations of CFML offer similar or enhanced functionality, such as running in a <a href="/wiki/.NET_Framework" title=".NET Framework">.NET</a> environment or image manipulation.</p>
<p>The engine was written in C and featured, among other things, a built-in scripting language (CFScript), plugin modules written in Java, and a syntax very similar to HTML. The equivalent to an HTML element, a ColdFusion tag begins with the letters "CF" followed by a name that is indicative of what the tag is interpreted to, in HTML. E.g. &lt;cfoutput&gt; to begin the output of variables or other content.</p>
<p>In addition to CFScript and plugins (as described), CFStudio provided a design platform with a <a href="/wiki/WYSIWYG" title="WYSIWYG">WYSIWYG</a> display. In addition to ColdFusion, CFStudio also supports syntax in other languages popular for backend programming, such as Perl. In addition to making backend functionality easily available to the non-programmer, (version 4.0 and forward in particular) integrated easily with the <a href="/wiki/Apache_HTTP_Server" title="Apache HTTP Server">Apache Web Server</a> and with <a href="/wiki/Internet_Information_Services" title="Internet Information Services">Internet Information Services</a>.</p>
<h3><span class="mw-headline" id="Other_features">Other features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=3" title="Edit section: Other features">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>All versions of ColdFusion prior to 6.0 were written using Microsoft <a href="/wiki/Visual_C%2B%2B" title="Visual C++">Visual C++</a>. This meant that ColdFusion was largely limited to running on <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>, although Allaire did successfully port ColdFusion to <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun</a> <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">Solaris</a> starting with version 3.1.</p>
<p>The <a href="/wiki/Allaire_Corporation" title="Allaire Corporation">Allaire</a> company was sold to <a href="/wiki/Macromedia" title="Macromedia">Macromedia</a>, then Macromedia was sold to <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe</a>. Earlier versions were not as robust as the versions available from version 4.0 forward.</p>
<p>With the release of ColdFusion MX 6.0, the engine had been re-written in <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> and supported its own runtime environment, which was easily replaced through its configuration options with the runtime environment from Sun. Version 6.1 included the ability to code and debug Shockwave Flash.</p>

<h2><span class="mw-headline" id="Versions">Versions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=5" title="Edit section: Versions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Cold_Fusion_3.1">Cold Fusion 3.1</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=6" title="Edit section: Cold Fusion 3.1">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Version 3.1 brought about a port to the <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">Sun Solaris</a> operating system. Cold Fusion studio gained a live page preview and HTML syntax checker.</p>
<h3><span class="mw-headline" id="ColdFusion_4">ColdFusion 4</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=7" title="Edit section: ColdFusion 4">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>"Cold Fusion" moniker renamed simply as "ColdFusion" - possibly to distinguish it from <a href="/wiki/Cold_fusion" title="Cold fusion">Cold fusion</a> theory.</p>
<h3><span class="mw-headline" id="ColdFusion_4.5">ColdFusion 4.5</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=8" title="Edit section: ColdFusion 4.5">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Version 4.5 brought the ability to natively invoke Java objects, execute system commands, and talk directly to a Java EE server.</p>
<h3><span class="mw-headline" id="ColdFusion_5">ColdFusion 5</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=9" title="Edit section: ColdFusion 5">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>First release from <a href="/wiki/Macromedia" title="Macromedia">Macromedia</a> after the <a href="/wiki/Allaire_Corporation" title="Allaire Corporation">Allaire Corporation</a> acquisition. The last to be legacy coded for a specific platform.</p>
<p>On January 16, 2001, Allaire announced a pending merger with Macromedia. Macromedia continued its development and released the product under the name ColdFusion 5.0. It retained the name "ColdFusion" through the remainder of version 5 releases.</p>
<h3><span class="mw-headline" id="ColdFusion_MX_6">ColdFusion MX 6</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=10" title="Edit section: ColdFusion MX 6">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Prior to 2000, Edwin Smith, an Allaire architect on JRun and later the Flash Player, initiated a project codenamed "Neo".<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> This project was later revealed as a ColdFusion Server re-written completely using <a class="mw-redirect" href="/wiki/Java_platform" title="Java platform">Java</a>. This made portability easier and provided a layer of security on the server, because it ran inside a Java Runtime Environment.</p>
<p>In June 2002 Macromedia released the version 6.0 product under a slightly different name, ColdFusion MX, allowing the product to be associated with both the Macromedia brand and its original branding. ColdFusion MX was completely rebuilt from the ground up and was based on the <a class="mw-redirect" href="/wiki/Java_EE" title="Java EE">Java EE</a> platform. ColdFusion MX was also designed to integrate well with <a class="mw-redirect" href="/wiki/Macromedia_Flash" title="Macromedia Flash">Macromedia Flash</a> using Flash Remoting.</p>
<p>With the release of ColdFusion MX, the CFML language API was released with an <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">OOP</a> interface.</p>
<h3><span class="mw-headline" id="ColdFusion_MX_7">ColdFusion MX 7</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=11" title="Edit section: ColdFusion MX 7">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>With the release of ColdFusion 7.0 on February 7, 2005, the naming convention was amended, rendering the product name "Macromedia ColdFusion MX 7" (the codename for CFMX7 was "Blackstone"). CFMX 7 added Flash-based and <a href="/wiki/XForms" title="XForms">XForms</a>-based web forms, and a report builder that output in <a class="mw-redirect" href="/wiki/Adobe_PDF" title="Adobe PDF">Adobe PDF</a> as well as <a href="/wiki/FlashPaper" title="FlashPaper">FlashPaper</a>, <a href="/wiki/Rich_Text_Format" title="Rich Text Format">RTF</a> and <a href="/wiki/Microsoft_Excel" title="Microsoft Excel">Excel</a>. The Adobe PDF output is also available as a wrapper to any HTML page, converting that page to a quality printable document. The enterprise edition also added <a class="mw-redirect" href="/wiki/Gateway_(computer_networking)" title="Gateway (computer networking)">Gateways</a>. These provide interaction with non-HTTP request services such as IM Services, SMS, Directory Watchers, and an asynchronous execution. XML support was boosted in this version to include native schema checking.</p>
<p>ColdFusion MX 7.0.1 (codename "Merrimack") added support for <a href="/wiki/MacOS" title="MacOS">Mac OS X</a>, improvements to Flash forms, RTF support for CFReport, the new CFCPRoxy feature for Java/CFC integration, and more. ColdFusion MX 7.0.2 (codenamed "Mystic") included advanced features for working with Adobe Flex 2 as well as more improvements for the CF Report Builder.</p>
<h3><span class="mw-headline" id="Adobe_ColdFusion_8">Adobe ColdFusion 8</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=12" title="Edit section: Adobe ColdFusion 8">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>On July 30, 2007, <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe Systems</a> released <b>ColdFusion 8</b>, dropping "MX" from its name.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> During beta testing the codename used was "Scorpio" (the eighth sign of the zodiac and the eighth iteration of ColdFusion as a commercial product). More than 14,000 developers worldwide were active in the beta process - many more testers than the 5,000 <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe Systems</a> originally expected. The ColdFusion development team consisted of developers based in Newton/Boston, Massachusetts and offshore in Bangalore, India.</p>
<p>Some of the new features are the CFPDFFORM tag, which enables integration with <a href="/wiki/Adobe_Acrobat" title="Adobe Acrobat">Adobe Acrobat</a> forms, some image manipulation functions, <a class="mw-redirect" href="/wiki/Microsoft_.NET" title="Microsoft .NET">Microsoft .NET</a> integration, and the CFPRESENTATION tag, which allows the creation of dynamic presentations using <a class="mw-redirect" href="/wiki/Adobe_Acrobat_Connect" title="Adobe Acrobat Connect">Adobe Acrobat Connect</a>, the Web-based collaboration solution formerly known as <a class="mw-redirect" href="/wiki/Macromedia_Breeze" title="Macromedia Breeze">Macromedia Breeze</a>. In addition, the ColdFusion Administrator for the Enterprise version ships with built-in server monitoring. ColdFusion 8 is available on several <a class="mw-redirect" href="/wiki/Operating_systems" title="Operating systems">operating systems</a> including <a href="/wiki/Linux" title="Linux">Linux</a>, Mac OS X and <a href="/wiki/Windows_Server_2003" title="Windows Server 2003">Windows Server 2003</a>.</p>
<p>Other additions to ColdFusion 8 are built-in <a href="/wiki/Ajax_(programming)" title="Ajax (programming)">Ajax</a> widgets, file archive manipulation (CFZIP), <a href="/wiki/Microsoft_Exchange_Server" title="Microsoft Exchange Server">Microsoft Exchange</a> server integration (CFEXCHANGE), image manipulation including automatic <a href="/wiki/CAPTCHA" title="CAPTCHA">CAPTCHA</a> generation (CFIMAGE), multi-threading, per-application settings, <a href="/wiki/Atom_(standard)" title="Atom (standard)">Atom</a> and <a href="/wiki/RSS" title="RSS">RSS</a> feeds, reporting enhancements, stronger encryption libraries, array and structure improvements, improved database interaction, extensive performance improvements, PDF manipulation and merging capabilities (CFPDF), interactive debugging, embedded database support with <a href="/wiki/Apache_Derby" title="Apache Derby">Apache Derby</a>, and a more <a href="/wiki/ECMAScript" title="ECMAScript">ECMAScript</a> compliant CFSCRIPT.</p>
<p>For development of ColdFusion applications, several tools are available: primarily <a href="/wiki/Adobe_Dreamweaver" title="Adobe Dreamweaver">Adobe Dreamweaver</a> CS4, <a href="/wiki/Macromedia_HomeSite" title="Macromedia HomeSite">Macromedia HomeSite</a> 5.x, <a class="new" href="/w/index.php?title=CFEclipse&amp;action=edit&amp;redlink=1" title="CFEclipse (page does not exist)">CFEclipse</a>, <a href="/wiki/Eclipse_(software)" title="Eclipse (software)">Eclipse</a> and others. "Tag updaters" are available for these applications to update their support for the new ColdFusion 8 features.</p>
<h3><span class="mw-headline" id="Adobe_ColdFusion_9">Adobe ColdFusion 9</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=13" title="Edit section: Adobe ColdFusion 9">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>ColdFusion 9 (Codenamed: <a href="/wiki/Centaur" title="Centaur">Centaur</a>) was released on October 5, 2009. New features for CF9 include:</p>
<ul>
<li>Ability to code ColdFusion Components (CFCs) entirely in CFScript.</li>
<li>An explicit "local" scope that does not require local variables to be declared at the top of the function.</li>
<li>Implicit getters/setters for CFC.</li>
<li>Implicit constructors via method called "init" or method with same name as CFC.</li>
<li>New CFFinally tag for <a href="/wiki/Exception_handling_syntax" title="Exception handling syntax">Exception handling syntax</a> and CFContinue tag for <a href="/wiki/Control_flow" title="Control flow">Control flow</a>.</li>
<li><a href="/wiki/Object-relational_mapping" title="Object-relational mapping">Object-relational mapping</a> (ORM) Database integration through <a class="mw-redirect" href="/wiki/Hibernate_(Java)" title="Hibernate (Java)">Hibernate (Java)</a>.</li>
<li>Server.cfc file with onServerStart and onServerEnd methods.</li>
<li>Tighter integration with <a class="mw-redirect" href="/wiki/Adobe_Flex" title="Adobe Flex">Adobe Flex</a> and <a href="/wiki/Adobe_AIR" title="Adobe AIR">Adobe AIR</a>.</li>
<li>Integration with key Microsoft products including Word, Excel, SharePoint, Exchange, and PowerPoint.</li>
<li>In Memory Management - or Virtual File System: an ability to treat content in memory as opposed to using the HDD.</li>
<li>Exposed as Services - an ability to access, securely, functions of the server externally.</li>
</ul>
<h3><span class="mw-headline" id="Adobe_ColdFusion_10">Adobe ColdFusion 10</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=14" title="Edit section: Adobe ColdFusion 10">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>ColdFusion 10 (Codenamed: Zeus) was released on May 15, 2012. New or improved features available in all editions (Standard, Enterprise, and Developer) include (but are not limited to):</p>
<ul>
<li>Security enhancements</li>
<li>Hotfix installer and notification</li>
<li>Improved scheduler (based on a version of quartz)</li>
<li>Improved web services support (WSDL 2.0, SOAP 1.2)</li>
<li>Support for <a href="/wiki/HTML5" title="HTML5">HTML5</a> web sockets</li>
<li><a href="/wiki/Apache_Tomcat" title="Apache Tomcat">Tomcat</a> integration</li>
<li>Support for RESTful web services</li>
<li>Language enhancements (closures, and more)</li>
<li>Search integration with Apache Solr</li>
<li>HTML5 video player and Adobe Flash Player</li>
<li>Flex and Adobe AIR lazy loading</li>
<li>XPath integration</li>
<li>HTML5 enhancements</li>
</ul>
<p>Additional new or improved features in ColdFusion Enterprise or Developer editions include (but are not limited to):</p>
<ul>
<li>Dynamic and interactive HTML5 charting</li>
<li>Improved and revamped scheduler (additional features over what is added in CF10 Standard)</li>
<li>Object relational mapping enhancements</li>
</ul>
<p>The lists above were obtained from the Adobe web site pages describing "new features", as listed first in the links in the following list.</p>
<p>For additional information on new features, see:</p>

<p>CF10 was originally referred to by the codename Zeus, after first being confirmed as coming by Adobe at <a href="/wiki/Adobe_MAX" title="Adobe MAX">Adobe MAX</a> 2010, and during much of its prerelease period. It was also commonly referred to as "ColdFusion next" and "ColdFusion X" in blogs, on Twitter, etc., before Adobe finally confirmed it would be "ColdFusion 10". For much of 2010, ColdFusion Product Manager Adam Lehman toured the US setting up countless meetings with customers, developers, and user groups to formulate a master blueprint for the next feature set. In September 2010, he presented the plans to Adobe where they were given full support and approval by upper management.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup></p>
<p>The first public beta of ColdFusion 10 was released via Adobe Labs on 17 February 2012.</p>
<h3><span class="mw-headline" id="Adobe_ColdFusion_11">Adobe ColdFusion 11</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=15" title="Edit section: Adobe ColdFusion 11">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>ColdFusion 11 (Codenamed: Splendor) was released on April 29, 2014.</p>
<p>New or improved features available in all editions (Standard, Enterprise, and Developer) include:</p>
<ul>
<li>End-to-end mobile development</li>
<li>A new lightweight edition (ColdFusion Express)</li>
<li>Language enhancements</li>
<li>WebSocket enhancements</li>
<li>PDF generation enhancements</li>
<li>Security enhancements</li>
<li>Social enhancements</li>
<li>REST enhancements</li>
<li>Charting enhancements</li>
<li>Compression enhancements</li>
<li>and more</li>
</ul>
<p>For more on each of the above, see <a class="external text" href="https://helpx.adobe.com/coldfusion/developing-applications/changes-in-coldfusion.html" rel="nofollow">New in ColdFusion 11</a>.</p>
<p>ColdFusion 11 also removed many features previously identified simply as "deprecated" or no longer supported in earlier releases. For example, the CFLOG tag long offered date and time attributes which were deprecated (and redundant, as the date and time is always logged). As of CF11, their use would not cause the CFLOG tag to fail. For a complete list, see <a class="external text" href="https://helpx.adobe.com/coldfusion/deprecated-features.html" rel="nofollow">Deprecated Features</a>.</p>

<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=19" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="PDF_generation">PDF generation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=20" title="Edit section: PDF generation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>ColdFusion can generate <a class="mw-redirect" href="/wiki/PDF" title="PDF">PDF</a> documents using standard HTML (i.e. no additional coding is needed to generate documents for print). CFML authors place HTML and CSS within a pair of cfdocument tags (or new in ColdFusion 11, cfhtmltopdf tags). The generated document can then either be saved to disk or sent to the client's browser. ColdFusion 8 introduced also the cfpdf tag to allow for control over PDF documents including PDF forms, and merging of PDFs. These tags however do not use Adobe's PDF engine but cfdocument uses a combination of the commercial <a href="/wiki/JPedal" title="JPedal">JPedal</a> Java PDF library and the free and open source Java library <a href="/wiki/IText" title="IText">iText</a>, and cfhtmltopdf uses an embedded <a href="/wiki/WebKit" title="WebKit">WebKit</a> implementation.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup></p>
<h3><span class="mw-headline" id="ColdFusion_Components_.28Objects.29">ColdFusion Components (Objects)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=21" title="Edit section: ColdFusion Components (Objects)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>ColdFusion was originally not an <a class="mw-redirect" href="/wiki/Object-oriented_programming_language" title="Object-oriented programming language">object-oriented programming language</a> like <a href="/wiki/PHP" title="PHP">PHP</a> versions 3 and below. ColdFusion falls into the category of OO languages that do not support multiple inheritance (along with Java, Smalltalk, etc.).<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> With the MX release (6+), ColdFusion introduced basic OO functionality with the <i>component</i> language construct which resembles classes in OO languages. Each <i>component</i> may contain any number of properties and methods. One component may also extend another (<a class="mw-redirect" href="/wiki/Inheritance_(computer_science)" title="Inheritance (computer science)">Inheritance</a>). Components only support single inheritance. Object handling feature set and performance enhancing has occurred with subsequent releases. With the release of ColdFusion 8, Java-style <a class="mw-redirect" href="/wiki/Interface_(programming)" title="Interface (programming)">interfaces</a> are supported. ColdFusion components use the file extension <i>cfc</i> to differentiate them from ColdFusion templates (.cfm).</p>
<h4><span class="mw-headline" id="Remoting">Remoting</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=22" title="Edit section: Remoting">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Component methods may be made available as web services with no additional coding and configuration. All that is required is for a method's access to be declared 'remote'. ColdFusion automatically generates a <a href="/wiki/Web_Services_Description_Language" title="Web Services Description Language">WSDL</a> at the URL for the component in this manner: http://path/to/components/Component.cfc?wsdl. Aside from SOAP, the services are offered in Flash Remoting binary format.</p>
<p>Methods which are declared remote may also be invoked via an HTTP GET or POST request. Consider the GET request as shown.<br/></p>
<pre>
http://path/to/components/Component.cfc?method=search&amp;query=your+query&amp;mode=strict
</pre>
<p>This will invoke the component's search function, passing "your query" and "strict" as arguments.</p>
<p>This type of invocation is well-suited for <a href="/wiki/Ajax_(programming)" title="Ajax (programming)">Ajax</a>-enabled applications. ColdFusion 8 introduced the ability to serialize ColdFusion data structures to <a href="/wiki/JSON" title="JSON">JSON</a> for consumption on the client.</p>
<p>The ColdFusion server will automatically generate documentation for a component if you navigate to its URL and insert the appropriate code within the component's declarations. This is an application of component introspection, available to developers of ColdFusion components. Access to a component's documentation requires a password. A developer can view the documentation for all components known to the ColdFusion server by navigating to the ColdFusion URL. This interface resembles the <a href="/wiki/Javadoc" title="Javadoc">Javadoc</a> HTML documentation for Java classes.</p>
<h3><span class="mw-headline" id="Custom_Tags">Custom Tags</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=23" title="Edit section: Custom Tags">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>ColdFusion provides several ways to implement custom <a href="/wiki/Markup_language" title="Markup language">markup language</a> tags, i.e. those not included in the core ColdFusion language. These are especially useful for providing a familiar interface for web designers and content authors familiar with HTML but not <a href="/wiki/Imperative_programming" title="Imperative programming">imperative programming</a>.</p>
<p>The traditional and most common way is using CFML. A standard CFML page can be interpreted as a tag, with the tag name corresponding to the file name prefixed with "cf_". For example, the file IMAP.cfm can be used as the tag "cf_imap". Attributes used within the tag are available in the ATTRIBUTES scope of the tag implementation page. CFML pages are accessible in the same directory as the calling page, via a special directory in the ColdFusion web application, or via a CFIMPORT tag in the calling page. The latter method does not necessarily require the "cf_" prefix for the tag name.</p>
<p>A second way is the development of CFX tags using Java or C++. CFX tags are prefixed with "cfx_", for example "cfx_imap". Tags are added to the ColdFusion runtime environment using the ColdFusion administrator, where JAR or DLL files are registered as custom tags.</p>
<p>Finally, ColdFusion supports JSP tag libraries from the JSP 2.0 language specification. JSP tags are included in CFML pages using the CFIMPORT tag.</p>
<h2><span class="mw-headline" id="Interactions_with_other_programming_languages">Interactions with other programming languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=24" title="Edit section: Interactions with other programming languages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="ColdFusion_and_Java">ColdFusion and Java</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=25" title="Edit section: ColdFusion and Java">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The standard ColdFusion installation allows the deployment of ColdFusion as a <a class="mw-redirect" href="/wiki/WAR_(Sun_file_format)" title="WAR (Sun file format)">WAR file</a> or <a href="/wiki/EAR_(file_format)" title="EAR (file format)">EAR file</a> for deployment to standalone application servers, such as <a class="mw-redirect" href="/wiki/Macromedia_JRun" title="Macromedia JRun">Macromedia JRun</a>, and IBM <a class="mw-redirect" href="/wiki/WebSphere" title="WebSphere">WebSphere</a>. ColdFusion can also be deployed to servlet containers such as <a href="/wiki/Apache_Tomcat" title="Apache Tomcat">Apache Tomcat</a> and <a href="/wiki/Jetty_(web_server)" title="Jetty (web server)">Mortbay Jetty</a>, but because these platforms do not officially support ColdFusion, they leave many of its features inaccessible. As of ColdFusion 10 <a class="mw-redirect" href="/wiki/Macromedia_JRun" title="Macromedia JRun">Macromedia JRun</a> was replaced by <a href="/wiki/Apache_Tomcat" title="Apache Tomcat">Apache Tomcat</a>.</p>
<p>Because ColdFusion is a Java EE application, ColdFusion code can be mixed with Java classes to create a variety of applications and use existing Java libraries. ColdFusion has access to all underlying Java classes, supports JSP custom tag libraries, and can access JSP functions after retrieving the JSP page context (<i>GetPageContext()</i>).</p>
<p>Prior to ColdFusion 7.0.1, ColdFusion components could only be used by Java or .NET by declaring them as web services. However, beginning in ColdFusion MX 7.0.1, ColdFusion components can now be used directly within Java classes using the CFCProxy class.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup></p>
<p>Recently, there has been much interest in Java development using alternate languages such as <a href="/wiki/Jython" title="Jython">Jython</a>, <a href="/wiki/Groovy_(programming_language)" title="Groovy (programming language)">Groovy</a> and <a href="/wiki/JRuby" title="JRuby">JRuby</a>. ColdFusion was one of the first scripting platforms to allow this style of Java development.</p>
<h3><span class="mw-headline" id="ColdFusion_and_.NET">ColdFusion and .NET</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=26" title="Edit section: ColdFusion and .NET">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>ColdFusion 8 natively supports <a href="/wiki/.NET_Framework" title=".NET Framework">.NET</a> within the CFML syntax. ColdFusion developers can simply call any .NET assembly without needing to recompile or alter the assemblies in any way. Data types are automatically translated between ColdFusion and .NET (example: .NET DataTable → ColdFusion Query).</p>
<p>A unique feature for a Java EE vendor, ColdFusion 8 offers the ability to access .NET Assemblies remotely through proxy (without the use of <a href="/wiki/.NET_Remoting" title=".NET Remoting">.NET Remoting</a>). This allows ColdFusion users to leverage .NET without having to be installed on a Windows operating system.</p>
<h2><span class="mw-headline" id="Acronyms">Acronyms</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=27" title="Edit section: Acronyms">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The <a href="/wiki/Acronym" title="Acronym">acronym</a> for the ColdFusion Markup Language is <i><b>CFML</b></i>. When ColdFusion templates are saved to disk, they are traditionally given the extension .cfm or .cfml. The .cfc extension is used for ColdFusion Components. The original extension was DBM or DBML, which stood for Database Markup Language. When talking about ColdFusion, most users use the acronym CF and this is used for numerous ColdFusion resources such as user groups (CFUGs) and sites.</p>
<p>CFMX is the common abbreviation for ColdFusion versions 6 and 7 (a.k.a. ColdFusion MX).</p>
<h2><span class="mw-headline" id="Alternative_server_environments">Alternative server environments</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=28" title="Edit section: Alternative server environments">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>ColdFusion originated as proprietary technology based on Web technology industry standards. However, it is becoming a less closed technology through the availability of competing products. Such alternative products include (in alphabetical order):</p>
<ul>
<li><a href="/wiki/BlueDragon" title="BlueDragon">BlueDragon</a> - Proprietary .NET-based CFML Engine and Free Open Source Java-based CFML Engine (Open BlueDragon).</li>
<li>Coral Web Builder</li>
<li><a href="/wiki/IgniteFusion" title="IgniteFusion">IgniteFusion</a></li>
<li><a href="/wiki/Lucee" title="Lucee">Lucee</a> - Free, open source CFML Engine forked from Railo. Lucee's aim is to provide the functionality of CFML using less resources and giving better performance and to move CFML past its roots and into a modern and dynamic web programming platform. Lucee is backed by community supporters and members of the Lucee Association.</li>
<li><a href="/wiki/Railo" title="Railo">Railo</a> - Free, Open Source CFML Engine. It comes in three main product editions, and other versions.</li>
<li><a href="/wiki/SmithProject" title="SmithProject">SmithProject</a></li>
</ul>
<p>The argument can be made that ColdFusion is even less platform-bound than raw Java EE or .NET, simply because ColdFusion will run on top of a .NET app server (New Atlanta), or on top of any servlet container or Java EE application server (<a class="mw-redirect" href="/wiki/JRun" title="JRun">JRun</a>, <a href="/wiki/IBM_WebSphere_Application_Server" title="IBM WebSphere Application Server">WebSphere</a>, <a class="mw-redirect" href="/wiki/JBoss" title="JBoss">JBoss</a>, <a href="/wiki/Apache_Geronimo" title="Apache Geronimo">Geronimo</a>, <a href="/wiki/Apache_Tomcat" title="Apache Tomcat">Tomcat</a>, <a class="mw-redirect" href="/wiki/Resin_Server" title="Resin Server">Resin Server</a>, <a href="/wiki/Jetty_(web_server)" title="Jetty (web server)">Jetty (web server)</a>, etc.). In theory, a ColdFusion application could be moved unchanged from a Java EE application server to a .NET application server.</p>
<h2><span class="mw-headline" id="Vulnerabilities">Vulnerabilities</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adobe_ColdFusion&amp;action=edit&amp;section=29" title="Edit section: Vulnerabilities">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In March 2013, a known issue affecting ColdFusion 8, 9 and 10 left the <a href="/wiki/National_Vulnerability_Database" title="National Vulnerability Database">National Vulnerability Database</a> open to attack.<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup> The vulnerability had been identified and a patch released by Adobe for CF9 and CF10 in January.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup></p>
<p>In April 2013, a ColdFusion vulnerability was blamed by <a href="/wiki/Linode" title="Linode">Linode</a> for an intrusion into the Linode Manager control panel website.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> A security bulletin and <a href="/wiki/Hotfix" title="Hotfix">hotfix</a> for this had been issued by Adobe a week earlier.<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup></p>
<p>In May 2013, Adobe identified another critical vulnerability, reportedly already being exploited in the wild, which targets all recent versions of ColdFusion on any servers where the web-based administrator and API have not been locked down. The vulnerability allows unauthorized users to upload malicious scripts and potentially gain full control over the server.<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup><sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup> A security bulletin and <a href="/wiki/Hotfix" title="Hotfix">hotfix</a> for this was issued by Adobe 6 days later.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> This is currently the only known vulnerability for ColdFusion 9 and ColdFusion 10 (currently supported versions).<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup><sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup></p>
<p>In April 2015 there was reported a Cross-site scripting (XSS) vulnerability in Adobe ColdFusion 10 before Update 16, and in ColdFusion 11 before Update 5, that allows remote attackers to inject arbitrary web script or HTML;<sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup> however, it's exploitable only by users who have authenticated through the administration panel. <sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup></p>
