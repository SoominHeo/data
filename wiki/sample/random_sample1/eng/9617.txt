<p>In <a href="/wiki/Computing" title="Computing">computing</a>, an <b>unparser</b> is a system that constructs a set of characters or image components from a given <a href="/wiki/Parse_tree" title="Parse tree">parse tree</a>.<sup class="reference" id="cite_ref-Ikuo_1-0"><a href="#cite_note-Ikuo-1">[1]</a></sup><sup class="reference" id="cite_ref-Ehrig_2-0"><a href="#cite_note-Ehrig-2">[2]</a></sup></p>
<p>An unparser is in effect the reverse of a traditional <a href="/wiki/Parsing" title="Parsing">parser</a> which takes a string of characters and produces a parse tree. Unparsing generally involves the application of a specific set of rules to the parse tree as a "<a href="/wiki/Tree_traversal" title="Tree traversal">tree walk</a>" takes place.<sup class="reference" id="cite_ref-Ikuo_1-1"><a href="#cite_note-Ikuo-1">[1]</a></sup></p>
<p>Given that the tree may involve both textual and graphic elements, the unparser may have two separate modules, each of which handles the relevant components.<sup class="reference" id="cite_ref-Ehrig_2-1"><a href="#cite_note-Ehrig-2">[2]</a></sup> In such cases the "master unparser" looks up the "master unparse table" to determine if a given nested structure should be handled by one module, or the other.<sup class="reference" id="cite_ref-Ehrig_2-2"><a href="#cite_note-Ehrig-2">[2]</a></sup></p>
