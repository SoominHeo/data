<h1 class="firstHeading" id="firstHeading" lang="en">Arial Unicode MS</h1>

<p>In digital <a href="/wiki/Typography" title="Typography">typography</a>, the <a href="/wiki/TrueType" title="TrueType">TrueType</a> font <b>Arial Unicode MS</b> is an extended version of the <a href="/wiki/Typeface" title="Typeface">font</a> <a href="/wiki/Arial" title="Arial">Arial</a>. Compared to Arial, it includes higher line height, omits <a href="/wiki/Kerning#Kerning_pairs" title="Kerning">kerning pairs</a> and adds enough glyphs to cover a large subset of <a href="/wiki/Unicode" title="Unicode">Unicode</a> 2.1—thus supporting most <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> <a href="/wiki/Code_page" title="Code page">code pages</a>, but also requiring much more <a class="mw-redirect" href="/wiki/Computer_storage" title="Computer storage">storage</a> space (22 <a href="/wiki/Megabyte" title="Megabyte">megabytes</a>).<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> It also adds <a href="/wiki/Ideogram" title="Ideogram">Ideographic</a> layout tables, but unlike Arial, it mandates no <a href="/wiki/Spatial_anti-aliasing" title="Spatial anti-aliasing">smoothing</a> in the 14–18 <a href="/wiki/Point_(typography)" title="Point (typography)">point</a> range, and contains Roman (upright) glyphs only; there is no oblique (<a href="/wiki/Italic_type" title="Italic type">italic</a>) version. Arial Unicode MS is normally distributed with <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a>, but it is also bundled with <a class="mw-redirect" href="/wiki/Mac_OS_X_v10.5" title="Mac OS X v10.5">Mac OS X v10.5</a> and later. It may also be purchased separately (as <i>Arial Unicode</i>) from <a href="/wiki/Ascender_Corporation" title="Ascender Corporation">Ascender Corporation</a>, who licenses the font from Microsoft.</p>
<p>When rendered with the same engine and without making adjustments for the different font metrics, the glyphs that appear in both Arial and Arial Unicode MS appear to be slightly wider, and thus rounder, in Arial Unicode MS. Horizontal text may also appear to have more inter-line spacing in Arial Unicode MS. This is due to larger bounding boxes (Arial Unicode MS needs more room for some of its extended glyphs) and the limitations of renderers, not changes in the glyph shapes. The lack of kerning pairs in Arial Unicode MS may also affect inter-glyph spacing in some renderers (for example the Adobe Flash Player).</p>
<p>Arial Unicode MS also includes <a class="mw-redirect" href="/wiki/Hebrew" title="Hebrew">Hebrew</a> glyphs different from the Hebrew glyphs found in Arial. They are based on the shapes of the Hebrew glyphs in <a href="/wiki/Tahoma_(typeface)" title="Tahoma (typeface)">Tahoma</a>, but are adjusted to the weight, proportions and style of Arial.</p>

<h2><span class="mw-headline" id="History_and_availability">History and availability</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Arial_Unicode_MS&amp;action=edit&amp;section=1" title="Edit section: History and availability">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Arial was designed by Robin Nicholas and Patricia Saunders in 1982 and was released as TrueType font in 1990. From 1993 to 1999, it was extended as Arial Unicode MS (with its first release as a TrueType font in 1998) by the following members of <a class="mw-redirect" href="/wiki/Monotype_Corporation" title="Monotype Corporation">Monotype Typography</a>'s Monotype Type Drawing Office, under contract to Microsoft: Brian Allen, Evert Bloemsma, Jelle Bosma, Joshua Hadley, Wallace Ho, Kamal Mansour, Steve Matteson, and Thomas Rickner.</p>
<p>From mid-2001 through mid-2002, Arial Unicode MS was also available as a separate download for licensed users of the standalone version of <a href="/wiki/Microsoft_Publisher" title="Microsoft Publisher">Microsoft Publisher</a> 2000 SR-1, which did not ship with the font. The freely downloadable version was withdrawn after Microsoft Publisher 2002, which included the font, began shipping. The withdrawal coincided with the withdrawal of the free downloads of Microsoft's "<a href="/wiki/Core_fonts_for_the_Web" title="Core fonts for the Web">Core fonts for the Web</a>". Numerous companies, organizations, educational establishments and even governments were directing users to the download without referencing the need for a valid Publisher or Office license or any Microsoft operating system.</p>
<p>Monotype Imaging still owns the Arial and Arial Unicode MS trademarks, but Microsoft once retained exclusive licensing rights to the fonts.</p>
<p>On 11 April 2005, <a href="/wiki/Ascender_Corporation" title="Ascender Corporation">Ascender Corporation</a> announced it had entered an agreement with Microsoft which enables Ascender to distribute Microsoft fonts, including the Windows Core Fonts, the Microsoft Web Fonts and the many multilingual fonts currently supplied by Microsoft.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> Called <i>Arial Unicode</i>, it is sold for approximately <a href="/wiki/United_States_dollar" title="United States dollar">$</a>99 per 5 users.</p>
<p>The font is also apparently licensed to <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple</a>, who announced on October 16, 2007 that their flagship operating system, <a class="mw-redirect" href="/wiki/Mac_OS_X_v10.5" title="Mac OS X v10.5">Mac OS X v10.5</a> ("Leopard"), would be bundled with Arial Unicode.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> Leopard also ships with several other previously Microsoft-only fonts, including <a class="mw-redirect" href="/wiki/Microsoft_Sans_Serif" title="Microsoft Sans Serif">Microsoft Sans Serif</a>, <a href="/wiki/Tahoma_(typeface)" title="Tahoma (typeface)">Tahoma</a> and <a href="/wiki/Wingdings" title="Wingdings">Wingdings</a>.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p>Monotype Imaging currently also licenses Arial Unicode on its own. It was also bundled by Monotype as part of iPhone Compatibility Font Set.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<p>Arial Unicode MS is no longer available in Microsoft Office 2016.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<h3><span class="mw-headline" id="Versions">Versions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Arial_Unicode_MS&amp;action=edit&amp;section=2" title="Edit section: Versions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Version 0.84 was supplied with Microsoft Office 2000 and the standalone versions of that suite's applications—except Publisher 2000 SR-1. It includes 51,180 glyphs (38,911 characters), supports 32 code pages, and contains Latin and Han Ideographic <a href="/wiki/OpenType" title="OpenType">OpenType</a> layout tables. The code pages supported are <a class="mw-redirect" href="/wiki/Code_page_1250" title="Code page 1250">1250</a> (Latin 2: East Europe), <a class="mw-redirect" href="/wiki/Code_page_1251" title="Code page 1251">1251</a> (Cyrillic), <a class="mw-redirect" href="/wiki/Code_page_1252" title="Code page 1252">1252</a> (Latin 1), <a class="mw-redirect" href="/wiki/Code_page_1253" title="Code page 1253">1253</a> (Greek), <a class="mw-redirect" href="/wiki/Code_page_1254" title="Code page 1254">1254</a> (Turkish), <a class="mw-redirect" href="/wiki/Code_page_1255" title="Code page 1255">1255</a> (Hebrew), <a class="mw-redirect" href="/wiki/Code_page_1256" title="Code page 1256">1256</a> (Arabic), <a class="mw-redirect" href="/wiki/Code_page_1257" title="Code page 1257">1257</a> (Windows Baltic), <a class="mw-redirect" href="/wiki/Code_page_1258" title="Code page 1258">Code page 1258</a> (Vietnamese), <a href="/wiki/Code_page_437" title="Code page 437">437</a> (US), <a class="new" href="/w/index.php?title=Code_page_708&amp;action=edit&amp;redlink=1" title="Code page 708 (page does not exist)">708</a> (Arabic; ASMO 708), <a href="/wiki/Code_page_737" title="Code page 737">737</a> (Greek), <a href="/wiki/Code_page_775" title="Code page 775">775</a> (MS-DOS Baltic), <a href="/wiki/Code_page_850" title="Code page 850">850</a> (WE/Latin 1), <a href="/wiki/Code_page_852" title="Code page 852">852</a> (Latin 2), <a href="/wiki/Code_page_855" title="Code page 855">855</a> (IBM Cyrillic; primarily Russian), <a href="/wiki/Code_page_857" title="Code page 857">857</a> (MS-DOS IBM Turkish), <a href="/wiki/Code_page_860" title="Code page 860">860</a> (MS-DOS Portuguese), <a href="/wiki/Code_page_861" title="Code page 861">861</a> (MS-DOS Icelandic), <a href="/wiki/Code_page_862" title="Code page 862">862</a> (Hebrew), <a href="/wiki/Code_page_863" title="Code page 863">863</a> (MS-DOS Canadian French), <a href="/wiki/Code_page_864" title="Code page 864">864</a> (Arabic), <a href="/wiki/Code_page_865" title="Code page 865">865</a> (MS-DOS Nordic), <a href="/wiki/Code_page_866" title="Code page 866">866</a> (MS-DOS Russian), <a href="/wiki/Code_page_869" title="Code page 869">869</a> (IBM Greek), <a class="mw-redirect" href="/wiki/Code_page_874" title="Code page 874">874</a> (Thai), <a href="/wiki/Code_page_932" title="Code page 932">932</a> (JIS/Japan), <a href="/wiki/Code_page_936" title="Code page 936">936</a> (Chinese: Simplified), <a href="/wiki/Code_page_949" title="Code page 949">949</a> (Korean Wansung), <a href="/wiki/Code_page_950" title="Code page 950">950</a> (Chinese: Traditional), "<a class="mw-redirect" href="/wiki/MacRoman" title="MacRoman">Macintosh Character Set</a>" (US Roman), and "<a href="/wiki/Code_page_437" title="Code page 437">Windows OEM Character Set</a>". It covers all code points containing non-control characters in Unicode 2.0 and allows only preview and print embedding.</p>
<p>Version 0.86 has the same coverage and support as 0.84.</p>
<p>Versions 1.00 and 1.01 were supplied with Microsoft Office 2002 (Microsoft Office XP), Microsoft Office 2003 and the standalone versions of those suites' applications. It includes 50,377 glyphs (38,917 characters), which reduces Combining Diacritical Marks to 72, increases Miscellaneous Technical characters to 123, increases Private Use Area characters to 43, reduces Spacing Modifier Letters to 57. <a class="new" href="/w/index.php?title=Code_page_1361&amp;action=edit&amp;redlink=1" title="Code page 1361 (page does not exist)">Code page 1361</a> (Korean Johab) was added. It adds layout tables for Devanagari, Gujarati, Gurmukhi, Kana (Hiragana &amp; Katakana), Kannada, and Tamil. Its Han Ideographic tables were updated to support vertical writing.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> It covers all code points containing non-control characters in Unicode 2.1 and allows editable embedding.</p>
<h2><span class="mw-headline" id="Bugs">Bugs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Arial_Unicode_MS&amp;action=edit&amp;section=3" title="Edit section: Bugs">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:178px;"><a class="image" href="/wiki/File:Arialunicodebug.png"><img alt="" class="thumbimage" data-file-height="192" data-file-width="176" height="192" src="//upload.wikimedia.org/wikipedia/en/7/70/Arialunicodebug.png" width="176"/></a>
<div class="thumbcaption">Demonstration of the double-width <a href="/wiki/Diacritic" title="Diacritic">diacritic</a> bug in Arial Unicode MS. The first row shows how Arial Unicode MS incorrectly renders a diacritic that is correctly placed between the letters "k" and "p". The second row shows how the diacritic is rendered in the correct position only if placed after the "p". The third row shows how the correct placement is rendered in TITUS Cyberbit Basic, which does not have the bug.</div>
</div>
</div>
<p>All versions of Arial Unicode MS deal with double-width <a href="/wiki/Diacritic" title="Diacritic">diacritic characters</a> incorrectly, drawing them too far to the left by one character width. According to the <a class="external text" href="http://www.unicode.org/versions/Unicode4.0.0/ch07.pdf" rel="nofollow">Unicode Standard 4.0.0, section 7.7</a> combining double diacritics go between the two characters to be marked. However, to make text look correct in Arial Unicode MS, the double-width diacritic must be placed after both characters to be marked. This means that it is not possible to make text that renders these characters correctly in both Arial Unicode MS and in other (correctly designed) Unicode fonts. This bug affects the rendering of text written in the <a href="/wiki/International_Phonetic_Alphabet" title="International Phonetic Alphabet">International Phonetic Alphabet</a> and in <a class="mw-redirect" href="/wiki/American_Library_Association-Library_of_Congress_Romanization_Tables" title="American Library Association-Library of Congress Romanization Tables">ALA-LC Romanization</a> for non-Latin-script languages. If the displayed font in your browser draws the diacritics correctly, they should appear over the characters: k͠p, k͡p.</p>
<p>The letters in Latin small <a class="mw-redirect" href="/wiki/Typographical_ligature" title="Typographical ligature">ligatures</a> fi, fl, ffi, ffl, long st, st aren't connected, except for the two f's in the ffi and ffl ligatures. However, there is nothing mandating that these must be connected - while they are indistinguishable from the individual letters placed next to each other, there is no semantic difference between the ligature and the individual characters.</p>
