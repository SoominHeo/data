<p><b>FFmpeg</b> is a <a href="/wiki/Free_software" title="Free software">free software</a> project that produces <a href="/wiki/Library_(computing)" title="Library (computing)">libraries</a> and <a href="/wiki/Computer_program" title="Computer program">programs</a> for handling <a href="/wiki/Multimedia" title="Multimedia">multimedia</a> data. FFmpeg includes <b><a href="/wiki/Libavcodec" title="Libavcodec">libavcodec</a></b>, an audio/video <a href="/wiki/Codec" title="Codec">codec</a> library used by several other projects, <b>libavformat</b> (Lavf),<sup class="reference" id="cite_ref-Lavf_6-0"><a href="#cite_note-Lavf-6">[6]</a></sup> an audio/video <a class="mw-redirect" href="/wiki/Container_format_(digital)" title="Container format (digital)">container</a> <a href="/wiki/Multiplexing" title="Multiplexing">mux and demux</a> library, and the <b>ffmpeg</b> <a class="mw-redirect" href="/wiki/Command_line" title="Command line">command line</a> program for <a href="/wiki/Transcoding" title="Transcoding">transcoding</a> multimedia files. FFmpeg is published under the <a href="/wiki/GNU_Lesser_General_Public_License" title="GNU Lesser General Public License">GNU Lesser General Public License</a> 2.1+ or <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a> 2+ (depending on which options are enabled).<sup class="reference" id="cite_ref-legal_7-0"><a href="#cite_note-legal-7">[7]</a></sup></p>
<p>The name of the project is inspired by the <a href="/wiki/Moving_Picture_Experts_Group" title="Moving Picture Experts Group">MPEG</a> video standards group, together with "FF" for "fast forward".<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> The logo uses a <a href="/wiki/Zigzag" title="Zigzag">zigzag</a> pattern that shows how MPEG video codecs handle <a href="/wiki/Entropy_encoding" title="Entropy encoding">entropy encoding</a>.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>

<p>The project was started by <a href="/wiki/Fabrice_Bellard" title="Fabrice Bellard">Fabrice Bellard</a><sup class="reference" id="cite_ref-legal_7-1"><a href="#cite_note-legal-7">[7]</a></sup> (using the <a href="/wiki/Pseudonym" title="Pseudonym">pseudonym</a> "Gérard Lantau") in 2000, and was led by Michael Niedermayer from 2004 until 2015.<sup class="reference" id="cite_ref-:0_10-0"><a href="#cite_note-:0-10">[10]</a></sup> Some FFmpeg developers were also part of the <a href="/wiki/MPlayer" title="MPlayer">MPlayer</a> project.</p>
<p>The project publishes a new release every three months on average. While release versions are available from the website for download, FFmpeg developers recommend that users compile the software from source using the latest build from their <a href="/wiki/Source_code" title="Source code">source code</a> <a class="mw-redirect" href="/wiki/Git_(software)" title="Git (software)">Git</a> <a class="mw-redirect" href="/wiki/Version_control_system" title="Version control system">version control system</a>.<sup class="reference" id="cite_ref-release_policy_11-0"><a href="#cite_note-release_policy-11">[11]</a></sup></p>
<p>On January 10, 2014, two Google employees announced that over 1000 bugs had been fixed in FFmpeg during the previous two years by means of <a class="mw-redirect" href="/wiki/Fuzz_testing" title="Fuzz testing">fuzz testing</a>.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<h3><span class="mw-headline" id="Codec_history">Codec history</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=2" title="Edit section: Codec history">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Two <a href="/wiki/Video_coding_format" title="Video coding format">video coding formats</a> with corresponding codecs and one <a href="/wiki/Digital_container_format" title="Digital container format">container format</a> have been created within the FFmpeg project so far. The two video codecs are the lossless <a href="/wiki/FFV1" title="FFV1">FFV1</a>, and the lossless and lossy Snow codec. Development of Snow has stalled, while its bit-stream format has not been finalized yet, making it experimental since 2011. The multimedia container format called NUT is no longer being actively developed, but still maintained.<sup class="reference" id="cite_ref-nut_13-0"><a href="#cite_note-nut-13">[13]</a></sup></p>
<p>In summer 2010, Fiona Glaser, Ronald Bultje, and David Conrad of the FFmpeg Team announced the ffvp8 decoder. Through testing, they determined that ffvp8 was faster than Google's own <a href="/wiki/Libvpx" title="Libvpx">libvpx</a> decoder.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup><sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> Starting with version 0.6, FFmpeg also supported <a href="/wiki/WebM" title="WebM">WebM</a> and <a href="/wiki/VP8" title="VP8">VP8</a>.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></p>
<p>In October 2013, a native <a href="/wiki/VP9" title="VP9">VP9</a><sup class="reference" id="cite_ref-LaunchpadVP9Decoder_17-0"><a href="#cite_note-LaunchpadVP9Decoder-17">[17]</a></sup> and the OpenHEVC decoder, an open source <a href="/wiki/High_Efficiency_Video_Coding" title="High Efficiency Video Coding">High Efficiency Video Coding</a> (HEVC) decoder, were added to FFmpeg.<sup class="reference" id="cite_ref-FFmpegHEVCOctober2013Softpedia_18-0"><a href="#cite_note-FFmpegHEVCOctober2013Softpedia-18">[18]</a></sup> In 2016 the native <a href="/wiki/Advanced_Audio_Coding" title="Advanced Audio Coding">AAC</a> encoder was considered stable, removing support for the two external AAC encoders from <a href="/wiki/VisualOn" title="VisualOn">VisualOn</a> and <a href="/wiki/FAAC" title="FAAC">FAAC</a>. FFmpeg 3.0 (nicknamed <i>"Einstein" </i>) retained build support for the <a href="/wiki/Fraunhofer_FDK_AAC" title="Fraunhofer FDK AAC">Fraunhofer FDK AAC</a> encoder.<sup class="reference" id="cite_ref-Einstein_19-0"><a href="#cite_note-Einstein-19">[19]</a></sup></p>
<h2><span class="mw-headline" id="Components">Components</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=3" title="Edit section: Components">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Command_line_tools">Command line tools</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=4" title="Edit section: Command line tools">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li><i>ffmpeg</i> is a command-line tool that converts audio or video formats. It can also capture and encode in real-time from various hardware and software sources such as a TV capture card.</li>
<li><i>ffserver</i> is an <a href="/wiki/Hypertext_Transfer_Protocol" title="Hypertext Transfer Protocol">HTTP</a> and <a href="/wiki/Real_Time_Streaming_Protocol" title="Real Time Streaming Protocol">RTSP</a> multimedia streaming server for live and recorded broadcasts. It can also be used to time shift live broadcasts.</li>
<li><i>ffplay</i> is a simple media player utilizing <a href="/wiki/Simple_DirectMedia_Layer" title="Simple DirectMedia Layer">SDL</a> and the FFmpeg libraries.</li>
<li><i>ffprobe</i> is a command-line tool to display media information (text, <a href="/wiki/Comma-separated_values" title="Comma-separated values">CSV</a>, <a href="/wiki/XML" title="XML">XML</a>, <a href="/wiki/JSON" title="JSON">JSON</a>), see also <a class="mw-redirect" href="/wiki/Mediainfo" title="Mediainfo">Mediainfo</a>.</li>
</ul>
<h3><span class="mw-headline" id="Libraries">Libraries</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=5" title="Edit section: Libraries">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li><i>libswresample</i> is a library containing audio <a class="mw-redirect" href="/wiki/Resampling_(audio)" title="Resampling (audio)">resampling</a> routines.</li>
<li><i>libavresample</i> is a library containing audio resampling routines from the <a href="/wiki/Libav" title="Libav">Libav</a> project, similar to <i>libswresample</i> from <i>ffmpeg</i>.</li>
<li><i><a href="/wiki/Libavcodec" title="Libavcodec">libavcodec</a></i> is a library containing all of the native FFmpeg audio/video encoders and decoders. Most codecs were developed from scratch to ensure best performance and high code reusability.</li>
<li><i>libavformat</i> (Lavf)<sup class="reference" id="cite_ref-Lavf_6-1"><a href="#cite_note-Lavf-6">[6]</a></sup> is a library containing demuxers and muxers for audio/video container formats.</li>
<li><i>libavutil</i> is a helper library containing routines common to different parts of FFmpeg. This library includes hash functions (<a href="/wiki/Adler-32" title="Adler-32">Adler-32</a>, <a href="/wiki/Cyclic_redundancy_check" title="Cyclic redundancy check">CRC</a>, <a href="/wiki/MD5" title="MD5">MD5</a>, <a href="/wiki/RIPEMD" title="RIPEMD">RIPEMD</a>, <a href="/wiki/SHA-1" title="SHA-1">SHA-1</a>. <a href="/wiki/SHA-2" title="SHA-2">SHA-2</a>, <a href="/wiki/MurmurHash" title="MurmurHash">MurmurHash</a>3, <a href="/wiki/Hash-based_message_authentication_code" title="Hash-based message authentication code">HMAC</a> MD-5, HMAC SHA-1 and HMAC SHA-2), ciphers (<a href="/wiki/Data_Encryption_Standard" title="Data Encryption Standard">DES</a>, <a href="/wiki/RC4" title="RC4">RC4</a>, <a href="/wiki/Advanced_Encryption_Standard" title="Advanced Encryption Standard">AES</a>, AES-CTR, <a href="/wiki/Tiny_Encryption_Algorithm" title="Tiny Encryption Algorithm">TEA</a>, <a href="/wiki/XTEA" title="XTEA">XTEA</a>, <a href="/wiki/Blowfish_(cipher)" title="Blowfish (cipher)">Blowfish</a>, <a href="/wiki/CAST-128" title="CAST-128">CAST-128</a>, <a href="/wiki/Twofish" title="Twofish">Twofish</a> and <a href="/wiki/Camellia_(cipher)" title="Camellia (cipher)">Camellia</a>), <a href="/wiki/Lempel%E2%80%93Ziv%E2%80%93Oberhumer" title="Lempel–Ziv–Oberhumer">LZO</a> decompressor and <a href="/wiki/Base64" title="Base64">Base64</a> encoder/decoder.</li>
<li><i>libpostproc</i> is a library containing older h263 based <a class="mw-redirect" href="/wiki/Video_postprocessing" title="Video postprocessing">video postprocessing</a> routines.</li>
<li><i>libswscale</i> is a library containing video <a href="/wiki/Image_scaling" title="Image scaling">image scaling</a> and <a href="/wiki/Color_space" title="Color space">colorspace</a>/pixelformat conversion routines.</li>
<li><i>libavfilter</i> is the substitute for vhook which allows the video/audio to be modified or examined between the decoder and the encoder. Filters have been ported from many projects including <a href="/wiki/MPlayer" title="MPlayer">MPlayer</a> and <a class="mw-redirect" href="/wiki/Avisynth" title="Avisynth">avisynth</a></li>
</ul>
<h2><span class="mw-headline" id="Supported_hardware">Supported hardware</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=6" title="Edit section: Supported hardware">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="CPUs">CPUs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=7" title="Edit section: CPUs">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>FFmpeg encompasses software implementations of video and audio compressing and decompressing algorithms. These can be compiled and run on diverse instruction sets.</p>
<p>Many widespread <a href="/wiki/Instruction_set" title="Instruction set">instruction sets</a> are supported by FFmpeg, like <a href="/wiki/X86" title="X86">x86</a> (<a href="/wiki/IA-32" title="IA-32">IA-32</a> and <a href="/wiki/X86-64" title="X86-64">x86-64</a>), PPC (<a href="/wiki/PowerPC" title="PowerPC">PowerPC</a>), <a href="/wiki/ARM_architecture" title="ARM architecture">ARM</a>, <a href="/wiki/DEC_Alpha" title="DEC Alpha">DEC Alpha</a>, <a href="/wiki/SPARC" title="SPARC">SPARC</a>, and <a class="mw-redirect" href="/wiki/MIPS_architecture" title="MIPS architecture">MIPS</a>.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup></p>
<h3><span class="mw-headline" id="Special_purpose_hardware">Special purpose hardware</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=8" title="Edit section: Special purpose hardware">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Various <a href="/wiki/Application-specific_integrated_circuit" title="Application-specific integrated circuit">application-specific integrated circuit</a> related to video and audio compression and decompression do exist. Such ASIC can perform the computation for audio/video decompression or compression partly or fully to offload these from the host CPU. To make use of such ASIC, instead of an complete implementation of some algorithm, only the API is required. There are numerous ASICs and APIs available, of which several are supported by FFmpeg.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup></p>

<h2><span class="mw-headline" id="Supported_codecs_and_formats">Supported codecs and formats</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=9" title="Edit section: Supported codecs and formats">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Image_formats">Image formats</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=10" title="Edit section: Image formats">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>FFmpeg supports many common and some uncommon image formats.</p>
<p>The <span id="PGMYUV"></span> <b>PGMYUV</b> image format is a homebrewn variant of the binary (P5) PGM <a href="/wiki/Netpbm_format" title="Netpbm format">Netpbm format</a>. FFmpeg also supports 16-bit depths of the PGM and PPM formats, and the binary (P7) <a href="/wiki/Netpbm#PAM_graphics_format" title="Netpbm">PAM</a> format with or without alpha channel, depth 8 bit or 16 bit for <code>pix_fmts</code> <i>monob, gray, gray16be, rgb24, rgb48be, ya8, rgba, rgb64be</i>.</p>

<h3><span class="mw-headline" id="Muxers">Muxers</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=12" title="Edit section: Muxers">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

<h2><span class="mw-headline" id="Legal_aspects">Legal aspects</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=22" title="Edit section: Legal aspects">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>FFmpeg contains more than 100 codecs,<sup class="reference" id="cite_ref-46"><a href="#cite_note-46">[38]</a></sup> most of which use compression techniques of one kind or another. Many such compression techniques may be subject to legal claims relating to <a href="/wiki/Software_patent" title="Software patent">software patents</a>.<sup class="reference" id="cite_ref-47"><a href="#cite_note-47">[39]</a></sup> Such claims may be enforceable in countries like the United States which have implemented software patents, but are considered unenforceable or void in member countries of the <a href="/wiki/European_Union" title="European Union">European Union</a>, for example.</p>
<h2><span class="mw-headline" id="FFmtech_Foundation">FFmtech Foundation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=23" title="Edit section: FFmtech Foundation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In June 2011 an election was organized to establish the board of FFmtech foundation,<sup class="reference" id="cite_ref-48"><a href="#cite_note-48">[40]</a></sup> a non-profit organization dedicated for managing donation funds. It was designed to offer reimbursement for expenses and work done to FFmpeg and <a href="/wiki/Libav" title="Libav">Libav</a>. However, according to FFmpeg developer Ronald Bultje the results have been doubtful.<sup class="reference" id="cite_ref-49"><a href="#cite_note-49">[41]</a></sup></p>
<h2><span class="mw-headline" id="Projects_using_FFmpeg">Projects using FFmpeg</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=24" title="Edit section: Projects using FFmpeg">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main category: <a href="/wiki/Category:Software_that_uses_FFmpeg" title="Category:Software that uses FFmpeg">Software that uses FFmpeg</a></div>
<p>FFmpeg is used by software such as <a href="/wiki/VLC_media_player" title="VLC media player">VLC media player</a>, <a href="/wiki/Xine" title="Xine">xine</a>, <a href="/wiki/Plex_(software)" title="Plex (software)">Plex</a>, <a href="/wiki/Blender_(software)" title="Blender (software)">Blender</a>, <a href="/wiki/YouTube" title="YouTube">YouTube</a>,<sup class="reference" id="cite_ref-50"><a href="#cite_note-50">[42]</a></sup> and <a class="mw-redirect" href="/wiki/MPC-HC" title="MPC-HC">MPC-HC</a>;<sup class="reference" id="cite_ref-projects_51-0"><a href="#cite_note-projects-51">[43]</a></sup> it handles video and audio playback in <a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a>,<sup class="reference" id="cite_ref-projects_51-1"><a href="#cite_note-projects-51">[43]</a></sup> and Linux version of Firefox.<sup class="reference" id="cite_ref-52"><a href="#cite_note-52">[44]</a></sup> <a href="/wiki/Graphical_user_interface" title="Graphical user interface">Graphical user interface</a> <a class="mw-redirect" href="/wiki/Front-end_(computing)" title="Front-end (computing)">front-ends</a> for FFmpeg have been developed, including Avanti,<sup class="reference" id="cite_ref-53"><a href="#cite_note-53">[45]</a></sup> and <a href="/wiki/XMedia_Recode" title="XMedia Recode">XMedia Recode</a>. JavaCV, a <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> wrapper for <a href="/wiki/OpenCV" title="OpenCV">OpenCV</a>, includes a supplementary Java wrapper for FFmpeg.<sup class="reference" id="cite_ref-54"><a href="#cite_note-54">[46]</a></sup></p>
<p>FFmpeg is used by <a href="/wiki/Ffdshow" title="Ffdshow">ffdshow</a>, <a class="mw-redirect" href="/wiki/LAV_Filters" title="LAV Filters">LAV Filters</a>, <a class="mw-redirect" href="/wiki/GStreamer_FFmpeg_plug-in" title="GStreamer FFmpeg plug-in">GStreamer FFmpeg plug-in</a>, <a href="/wiki/Perian" title="Perian">Perian</a> and <a href="/wiki/OpenMAX#Layers" title="OpenMAX">OpenMAX IL</a> to expand the encoding and decoding capabilities of their respective multimedia platform.</p>

<h3><span class="mw-headline" id="Libav">Libav</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FFmpeg&amp;action=edit&amp;section=26" title="Edit section: Libav">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Libav" title="Libav">Libav</a></div>
<p>On March 13, 2011, a group of FFmpeg developers decided to fork the project under the name "<a href="/wiki/Libav" title="Libav">Libav</a>".<sup class="reference" id="cite_ref-55"><a href="#cite_note-55">[47]</a></sup><sup class="reference" id="cite_ref-56"><a href="#cite_note-56">[48]</a></sup><sup class="reference" id="cite_ref-57"><a href="#cite_note-57">[49]</a></sup> The event was related to an issue in project management, in which developers disagreed with the leadership of FFmpeg.<sup class="reference" id="cite_ref-58"><a href="#cite_note-58">[50]</a></sup><sup class="reference" id="cite_ref-59"><a href="#cite_note-59">[51]</a></sup><sup class="reference" id="cite_ref-60"><a href="#cite_note-60">[52]</a></sup></p>
