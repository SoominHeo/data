<p><b>어도비 플래시</b>(<span lang="en" xml:lang="en">Adobe Flash</span>, 이전 이름: 매크로미디어 플래시)는 <a href="/wiki/%EC%96%B4%EB%8F%84%EB%B9%84_%EC%8B%9C%EC%8A%A4%ED%85%9C%EC%A6%88" title="어도비 시스템즈">어도비 시스템즈</a>사의 상호 작용적인 벡터 기반의 웹사이트를 제작할 수 있는 웹 저작 도구이자 소프트웨어 플랫폼이다.</p>
<p>대한민국에서는 <a href="/wiki/%EB%A7%88%EC%8B%9C%EB%A7%88%EB%A1%9C" title="마시마로">마시마로</a>, <a href="/wiki/%EC%A1%B8%EB%9D%BC%EB%A7%A8" title="졸라맨">졸라맨</a> 같은 <a href="/wiki/%ED%94%8C%EB%9E%98%EC%8B%9C_%EC%95%A0%EB%8B%88%EB%A9%94%EC%9D%B4%EC%85%98" title="플래시 애니메이션">플래시 애니메이션</a>의 대중화를 이끌었으며, 액션 스크립트의 진화 (1.0 &gt; 2.0 &gt; 3.0)와 더불어 RIA (플래시 애니메이션 기술과 서버 응용 기술의 통합을 통한 사용자 중심의 웹페이지 제작), AIR (RIA의 데스크톱 확장) 등의 다양한 응용 프로그램과의 상호 연동을 통하여 그 사용 범위가 확장되고 있다.</p>

<h2><span class="mw-headline" id=".EC.A7.80.EC.9B.90_.ED.94.8C.EB.9E.AB.ED.8F.BC">지원 플랫폼</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%96%B4%EB%8F%84%EB%B9%84_%ED%94%8C%EB%9E%98%EC%8B%9C&amp;action=edit&amp;section=4" title="부분 편집: 지원 플랫폼">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>어도비 플래시 플레이어는 인터넷 익스플로러 5 이상, 파이어폭스 등 <a class="mw-redirect" href="/wiki/ActiveX" title="ActiveX">ActiveX</a>와 플러그인을 지원하는 브라우저에서 제공된다. 운영 체제는 <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_98" title="윈도 98">윈도 98</a> 이상에서 작동한다. 또한 리눅스와 <a href="/wiki/%EC%86%94%EB%9D%BC%EB%A6%AC%EC%8A%A4_(%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C)" title="솔라리스 (운영 체제)">솔라리스</a> 운영 체제에서도 작동한다.</p>

<td>.as 파일은 단순한 소스 파일들에 있는 액션 스크립트 <a href="/wiki/%EC%86%8C%EC%8A%A4_%EC%BD%94%EB%93%9C" title="소스 코드">소스 코드</a>를 포함한다. FLA 파일들은 또한 액션스크립트 코드를 직접 포함하기도 하지만 각 외부 .as 파일들은 자주 구조적인 이유로 또는 코드를 버전화(versioning) 응용 프로그램에 노출시키기 위해 병합한다. 이들은 <b>.actionscript</b> 확장자를 사용하기도 한다.</td>
</tr>
<tr>
<td><b>.swd</b></td>
<td>.swd 파일은 플래시 개발 도중에 쓰이는 임시 오류 수정 파일이다. 플래시 프로젝트가 개발이 일단 끝나면 이러한 파일들은 필요 없으므로 지워도 된다.</td>
</tr>
<tr>
<td><b>.asc</b></td>
<td>.asc 파일은 서버사이드 액션스크립트를 포함하며, 효과적이고 유동적인 클라이언트 서버 매크로미디어 플래시 커뮤니케이션 서버 MX 응용 프로그램들을 개발하는 데에 쓰인다.</td>
</tr>
<tr>
<td><b>.<a class="mw-redirect" href="/wiki/FLV" title="FLV">flv</a></b></td>
<td>.flv 파일은 어도비 플래시, <a class="mw-redirect" href="/wiki/Ffmpeg" title="Ffmpeg">ffmpeg</a>, <a class="new" href="/w/index.php?title=%EC%86%8C%EB%A0%8C%EC%8A%A8_%EC%8A%A4%ED%80%B4%EC%A6%88&amp;action=edit&amp;redlink=1" title="소렌슨 스퀴즈 (없는 문서)">소렌슨 스퀴즈</a>, <a class="mw-redirect" href="/wiki/%EB%A6%AC%EC%96%BC_%ED%94%8C%EB%A0%88%EC%9D%B4%EC%96%B4" title="리얼 플레이어">리얼 플레이어</a>나 <a class="new" href="/w/index.php?title=On_Flix&amp;action=edit&amp;redlink=1" title="On Flix (없는 문서)">On Flix</a>로 만든 플래시 영상 파일이다.</td>
</tr>
<tr>
<td><b>.swc</b></td>
<td>.swc 파일은 구성 요소들을 배포하는 데 쓰인다. 이 파일은 컴파일된 클립, 구성 요소의 액션 스크립트 클래스 파일, 그리고 구성 요소에 대한 설명이 들어있는 다른 파일들을 가지고 있다.</td>
</tr>
<tr>
<td><b>.jsfl</b></td>
<td>.jsfl 파일은 플래시 저작 환경에서 기능을 추가하는 데 쓰인다. 이 파일은 자바스크립트 코드를 포함하며 플래시 자바스크립트 API에 접근할 수 있다.</td>
</tr>
<tr>
<td><b>.swt</b></td>
<td>.swt 파일은 .swf 파일의 템플릿 형식이며 <a class="new" href="/w/index.php?title=%EB%A7%A4%ED%81%AC%EB%A1%9C%EB%AF%B8%EB%94%94%EC%96%B4_%EC%A0%9C%EB%84%A4%EB%A0%88%EC%9D%B4%ED%84%B0&amp;action=edit&amp;redlink=1" title="매크로미디어 제네레이터 (없는 문서)">매크로미디어 제네레이터</a>에서 쓰인다.</td>
</tr>
<tr>
<td><b>.flp</b></td>
<td>.flp 파일은 플래시 프로젝트 안에 포함된 모든 문서 파일들을 참조하는 데 쓰이는 XML 파일이다. 플래시 프로젝트는 사용자가 여러 개의 관련된 파일들을 한데 묶을 수 있게 해 주며, 이로써 플래시 프로젝트를 조직하고 컴파일하고 만드는 데 도움을 준다.</td>
</tr>
<tr>
<td><b>.spl</b></td>
<td>.spl 파일은 <a class="new" href="/w/index.php?title=%ED%93%A8%EC%B2%98%EC%8A%A4%ED%94%8C%EB%9E%98%EC%8B%9C&amp;action=edit&amp;redlink=1" title="퓨처스플래시 (없는 문서)">퓨처스플래시</a> 문서이다.</td>
</tr>
<tr>
<td><b>.aso</b></td>
<td>.aso 파일은 플래시 개발 도중에 쓰이는 캐시 파일이며, 컴파일된 액션스크립트 바이트 코드를 포함한다. ASO 파일은 이와 일치하는 클래스 파일들의 내용이 바뀌었음을 확인할 때 다시 만들어진다. 가끔 플래시 IDE는 다시 컴파일이 필요하다는 것을 인지하지 못하기 때문에 이러한 캐시 파일들은 사용자가 하나씩 지워 주어야 한다. Win32 / 플래시8에서는 %USERPROFILE%\Local Settings\Application Data\Macromedia\Flash8\en\Configuration\Classes\aso 에 위치해 있다.</td>
</tr>
<tr>
<td><b>.lmv</b></td>
<td>.lmv 파일은 .fla 파일에 같다. 이 파일은 또한 .swf 파일로도 컴파일할 수 있는 오픈 소스 프로그램 Liveswif를 위한 소스 자료이다.</td>
</tr>
</table>
