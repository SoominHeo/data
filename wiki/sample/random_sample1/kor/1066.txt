<p><b>HFS 플러스</b>(HFS Plus, HFS+)는 <a href="/wiki/%EC%95%A0%ED%94%8C_(%EA%B8%B0%EC%97%85)" title="애플 (기업)">애플</a>이 개발한 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="파일 시스템">파일 시스템</a>이다. <a class="mw-redirect" href="/wiki/OS_X" title="OS X">OS X</a>의 주 파일 시스템으로 사용된다. <a href="/wiki/%EB%A7%A4%ED%82%A8%ED%86%A0%EC%8B%9C" title="매킨토시">매킨토시</a> 컴퓨터(또는 <a href="/wiki/%EB%A7%A5_OS" title="맥 OS">맥 OS</a>를 구동하는 다른 시스템)의 주 파일 시스템으로 사용되었던 <a href="/wiki/%EA%B3%84%EC%B8%B5%EC%A0%81_%ED%8C%8C%EC%9D%BC_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="계층적 파일 시스템">계층적 파일 시스템</a>(HFS)을 대체하기 위해 개발되었다. 디지털 음악 플레이어인 <a href="/wiki/%EC%95%84%EC%9D%B4%ED%8C%9F" title="아이팟">아이팟</a>에서 사용되는 포맷 중 하나이기도 하다. 이전 버전에 해당하는 HFS가 <i>맥 OS 스탠다드</i> 또는 <i>HFS 스탠다드</i>로 불렸기 때문에 HFS 플러스는 <b>맥 OS 익스텐디드</b>(Mac OS Extended) 또는 "HFS 익스텐디드"(HFS Extended)로 불리기도 한다. 애플은 이 파일 시스템을 개발할 때 이를 코드네임 세쿼이아(Sequoia)로 불렀다.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<p>HFS 플러스는 HFS의 개량 버전으로서, 훨씬 더 큰 크기의 파일을 지원(블록 주소가 16 비트에서 32 비트 길이로 확장됨)하며, 파일 및 폴더 이름에 <a href="/wiki/%EC%9C%A0%EB%8B%88%EC%BD%94%EB%93%9C" title="유니코드">유니코드</a>를 사용(기존의 맥 OS 로만 또는 다른 문자셋을 대체)하고, <a href="/wiki/%EC%9C%A0%EB%8B%88%EC%BD%94%EB%93%9C_%EC%A0%95%EA%B7%9C%ED%99%94" title="유니코드 정규화">유니코드 정규화 방식 D</a>와 거의 동일한 방식의 정규화(‘å’와 같은 조합문자가 파일명에서 두 개의 코드로 저장되며 <a href="/wiki/%EC%9C%A0%EB%8B%88%EC%BD%94%EB%93%9C_%EB%B8%94%EB%A1%9D" title="유니코드 블록">유니코드 기본 다국어 평면</a> 밖의 문자들 또한 두 개의 코드로 저장됨)를 적용한다.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> HFS 플러스에서는 파일 이름은 최대 255 UTF-16 코드 길이가 될 수 있으며, <a href="/wiki/NTFS" title="NTFS">NTFS</a>와 유사한 n-포크(n-forked) 파일이 허용되었다. 2005년까지만 해도 데이터 포크와 리소스 포크 외에 포크를 사용하는 시스템은 거의 없었다. HFS가 16비트 할당 맵핑 테이블을 사용했던데 반해 HFS 플러스는 32비트 할당 맵핑 테이블을 사용한다. HFS의 경우 할당 맵핑 테이블 크기의 제약으로 인해 디스크당 최대 65,536개의 할당 블록만 관리할 수 있다는 점이 심각한 제약사항이었다. 이는 과거 디스크 용량이 작던 시절에는 문제가 되지 않았으나, 디스크 용량이 점점 커짐에 따라 파일이 사용할 수 있는 가장 작은 영역(단일 할당 블록)의 크기도 매우 커져서 전체 디스크 용량을 낭비하게 된다는 문제를 발생시켰다. 예를 들어, 1GB 용량의 디스크에 HFS를 적용하면 할당 블록의 사이즈가 16KB가 되어 1바이트짜리 파일도 항상 16KB의 디스크 용량을 차지하게 되는 것이다. 다른 대부분의 파일 시스템들과 달리 HFS 플러스는 디렉토리에 대한 하드 링크를 지원한다.</p>
<p>대부분의 볼륨 <a href="/wiki/%EB%A9%94%ED%83%80%EB%8D%B0%EC%9D%B4%ED%84%B0" title="메타데이터">메타데이터</a>를 저장하는 데에 HFS 플러스는 HFS와 마찬가지로 <a href="/wiki/B_%ED%8A%B8%EB%A6%AC" title="B 트리">B 트리</a>를 사용한다.</p>
<h2><span class="mw-headline" id=".EC.97.AD.EC.82.AC">역사</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HFS_%ED%94%8C%EB%9F%AC%EC%8A%A4&amp;action=edit&amp;section=1" title="부분 편집: 역사">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>HFS 플러스는 <a href="/wiki/1998%EB%85%84" title="1998년">1998년</a> <a href="/wiki/1%EC%9B%94_19%EC%9D%BC" title="1월 19일">1월 19일</a> <a href="/wiki/%EB%A7%A5_OS_8" title="맥 OS 8">맥 OS 8.1</a>의 릴리즈와 함께 도입되었다.<sup class="reference" id="cite_ref-tn1150_1-1"><a href="#cite_note-tn1150-1">[1]</a></sup></p>
<p><a href="/wiki/2002%EB%85%84" title="2002년">2002년</a> <a href="/wiki/11%EC%9B%94_11%EC%9D%BC" title="11월 11일">11월 11일</a> <a href="/wiki/%EB%A7%A5_OS_X_10.2" title="맥 OS X 10.2">맥 OS X v10.2.2</a> 업데이트의 릴리즈와 함께 애플은 HFS 플러스에 데이터 신뢰성을 향상시키는 <a href="/wiki/%EC%A0%80%EB%84%90%EB%A7%81_%ED%8C%8C%EC%9D%BC_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="저널링 파일 시스템">저널링</a> 옵션을 추가하였다. 맥 OS X 서버에서는 이 기능에 쉽게 접근할 수 있었으나, 데스크탑 클라이언트에서는 커맨드 라인을 통해서만 접근이 가능했다.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> <a class="mw-redirect" href="/wiki/%EB%A7%A5_OS_X_10.3" title="맥 OS X 10.3">맥 OS X 10.3</a>에 와서는, 맥의 모든 HFS 플러스 볼륨에 저널링이 기본으로 적용되었다. 시스템 내에서 저널링이 적용된 HFS 플러스 볼륨은 <b>HFSJ</b>로 표시된다.</p>
