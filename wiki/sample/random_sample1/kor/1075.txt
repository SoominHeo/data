<p><b>HTTP 404</b> 또는 <b>찾을 수 없습니다</b> <a href="/wiki/%EC%98%A4%EB%A5%98_%EB%A9%94%EC%8B%9C%EC%A7%80" title="오류 메시지">오류 메시지</a>는 <a class="mw-disambig" href="/wiki/%ED%81%B4%EB%9D%BC%EC%9D%B4%EC%96%B8%ED%8A%B8" title="클라이언트">클라이언트</a>가 서버와 통신할 수는 있지만 서버가 요청한 바를 찾을 수 없다는 것을 가리키는 <a href="/wiki/HTTP" title="HTTP">HTTP</a> <a class="mw-redirect" href="/wiki/HTTP_%EC%83%81%ED%83%9C_%EC%BD%94%EB%93%9C_%EB%AA%A9%EB%A1%9D" title="HTTP 상태 코드 목록">표준 응답 코드</a>이다. 404 오류들은 대상 서버로 연결할 수 없다는 뜻의 "<a href="/wiki/%EC%84%9C%EB%B2%84" title="서버">서버</a>를 찾을 수 없습니다" 오류와 비슷한 메시지와 혼동해서는 안 된다.</p>

<h2><span class="mw-headline" id=".EA.B0.9C.EC.9A.94">개요</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTTP_404&amp;action=edit&amp;section=1" title="부분 편집: 개요">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>HTTP를 통해 통신할 때 서버는 <a href="/wiki/HTTP" title="HTTP">HTTP</a> 문서(<a href="/wiki/%EC%9B%B9_%ED%8E%98%EC%9D%B4%EC%A7%80" title="웹 페이지">웹 페이지</a>)에 대한 <a href="/wiki/%EC%9B%B9_%EB%B8%8C%EB%9D%BC%EC%9A%B0%EC%A0%80" title="웹 브라우저">웹 브라우저</a>의 요청과 같은 요청에 수많은 응답 코드와 선택적, 의무적, 또는 허용하지 않는 (상태 코드 기반으로) 메시지를 보여줌으로써 응답할 것을 요구받는다. 404 코드에서 처음 4는 <a href="/wiki/URL" title="URL">URL</a>을 잘못 입력한 것과 같은 클라이언트 오류를 가리킨다. 그 뒤에 따르는 2개의 숫자들은 특정한 오류와 마주쳤음을 가리킨다. HTTP의 세 개의 숫자 코드 이용은 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%EC%A0%84%EC%86%A1_%ED%94%84%EB%A1%9C%ED%86%A0%EC%BD%9C" title="파일 전송 프로토콜">FTP</a>와 <a class="new" href="/w/index.php?title=%EB%84%A4%ED%8A%B8%EC%9B%8C%ED%81%AC_%EB%89%B4%EC%8A%A4_%EC%A0%84%EC%86%A1_%ED%94%84%EB%A1%9C%ED%86%A0%EC%BD%9C&amp;action=edit&amp;redlink=1" title="네트워크 뉴스 전송 프로토콜 (없는 문서)">NNTP</a>와 같은 초기의 프로토콜에서의 해당 코드와 비슷하다.</p>
<p>HTTP 수준에서 404 응답 코드는 사람이 읽을 수 있는 "이유 설명"과 함께 나온다. HTTP 규격은 "찾을 수 없습니다"라는 설명을 제안하며<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> 수많은 웹 서버들은 기본적으로 404 코드와 "찾을 수 없습니다" 설명이 들어간 HTML 메시지를 제공한다. 웹 서버들은 일반적으로 더 자연스러운 설명, 상표화된 페이지, 또 이따금씩 검색 폼을 보여 주도록 구성되어 있을 수 있지만 사용자로부터 보이지 않는 프로토콜 수준의 설명을 따로 수정하는 것은 드문 일이다.</p>
<p>그러나 <a href="/wiki/%EC%9D%B8%ED%84%B0%EB%84%B7_%EC%9D%B5%EC%8A%A4%ED%94%8C%EB%A1%9C%EB%9F%AC" title="인터넷 익스플로러">인터넷 익스플로러</a> (<a href="/wiki/%EC%9D%B8%ED%84%B0%EB%84%B7_%EC%9D%B5%EC%8A%A4%ED%94%8C%EB%A1%9C%EB%9F%AC_7" title="인터넷 익스플로러 7">인터넷 익스플로러 7</a> 이전)는 512 바이트가 넘지 않을 경우 사용자가 지정한 페이지를 보여 주지 않고 친근한 오류 메시지를 보여 준다. 이러한 기본적인 동작은 인터넷 옵션의 도구에 들어가 고급 탭에서 "친근한 HTTP 오류 메시지 표시" 체크 상자에 체크를 없앰으로써 변경할 수 있다.</p>
<p>404 오류는 페이지가 이동되거나 삭제되었을 때 자주 반환된다. 전자의 경우 URL을 다시 기록하거나 대부분의 서버 구성 파일에서 구성할 수 있는 301 "영구적으로 이동하였음" 오류 응답을 반환한다. 후자의 경우 410 "사라짐" 메시지가 반환된다. 이 두 가지 옵션들이 특별한 서버 구성을 요구하기 때문에 대부분의 웹사이트들은 이들을 이용하지 않는다.</p>
<p>404 오류들은 주어진 URL이 존재하지 않는 서버 이름을 가리킬 때 나타나는 <a class="mw-redirect" href="/wiki/%EB%8F%84%EB%A9%94%EC%9D%B8_%EC%9D%B4%EB%A6%84_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="도메인 이름 시스템">DNS</a> 오류와 혼동해서는 안 된다. 404 오류는 서버 자체가 존재하지만 서버가 응답한 페이지를 찾지 못 했음을 나타낸다.</p>
<p>404 오류가 발생할 경우 사이트 관리자는 IIS에서 바인딩 편집에서 DNS에 등록된 도메인을 추가해 주면 해결이 가능하다.</p>
