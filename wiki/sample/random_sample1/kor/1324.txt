<h1 class="firstHeading" id="firstHeading" lang="ko">NP-완전</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">위키백과, 우리 모두의 백과사전.</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					이동:					<a href="#mw-head">둘러보기</a>, 					<a href="#p-search">검색</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><b>NP-완전</b>(NP-complete, <b>NP-C</b>, <b>NPC</b>)은 <a href="/wiki/NP_(%EB%B3%B5%EC%9E%A1%EB%8F%84)" title="NP (복잡도)">NP</a> 집합에 속하는 <a href="/wiki/%EA%B2%B0%EC%A0%95_%EB%AC%B8%EC%A0%9C" title="결정 문제">결정 문제</a> 중에서 가장 어려운 문제의 부분집합으로, 모든 NP 문제를 다항 시간 내에 NP-완전 문제로 <a href="/wiki/%ED%99%98%EC%82%B0_(%EB%B3%B5%EC%9E%A1%EB%8F%84)" title="환산 (복잡도)">환산</a>할 수 있다. NP-완전 문제 중 하나라도 <a href="/wiki/P_(%EB%B3%B5%EC%9E%A1%EB%8F%84)" title="P (복잡도)">P</a>에 속한다는 것을 증명한다면 모든 NP 문제가 P에 속하기 때문에, <a href="/wiki/P-NP_%EB%AC%B8%EC%A0%9C" title="P-NP 문제">P-NP 문제</a>가 P=NP의 형태로 풀리게 된다. 반대로 NP-완전 문제 중의 하나가 P에 속하지 않는다는 것이 증명된다면 P=NP에 대한 반례가 되어 P-NP 문제는 P≠NP의 형태로 풀리게 된다.</p>

<h2><span class="mw-headline" id=".EC.A0.95.EC.9D.98">정의</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NP-%EC%99%84%EC%A0%84&amp;action=edit&amp;section=1" title="부분 편집: 정의">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>NP-완전은 다음의 조건을 만족하는 <a href="/wiki/%EA%B2%B0%EC%A0%95_%EB%AC%B8%EC%A0%9C" title="결정 문제">결정 문제</a> <i>C</i>의 집합이다:</p>
<ol>
<li><i>C</i>가 NP에 속한다.</li>
<li>NP에 속하는 모든 문제를 <a href="/wiki/%EB%8B%A4%ED%95%AD_%EC%8B%9C%EA%B0%84" title="다항 시간">다항 시간</a> 안에 <i>C</i>로 변환할 수 있다. 즉, 다항 시간 <a href="/wiki/%ED%99%98%EC%82%B0_(%EB%B3%B5%EC%9E%A1%EB%8F%84)" title="환산 (복잡도)">환산</a>을 할 수 있다.</li>
</ol>
<p>여기에서 두 번째 조건은 <a href="/wiki/NP-%EB%82%9C%ED%95%B4" title="NP-난해">NP-난해</a>의 정의이고, 즉 NP-완전은 NP-난해 중 NP인 문제들의 집합이다. 또한 위 정의에서 알 수 있듯이, NP-완전인 <i>C</i>를 다항시간 안에 풀 수 있다면 모든 NP-완전 문제를 다항시간 안에 풀 수 있다.</p>
<h2><span class="mw-headline" id=".EC.97.AD.EC.82.AC">역사</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NP-%EC%99%84%EC%A0%84&amp;action=edit&amp;section=2" title="부분 편집: 역사">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>NP-완전이라는 개념은 <a href="/wiki/1971%EB%85%84" title="1971년">1971년</a>에 <a href="/wiki/%EC%8A%A4%ED%8B%B0%EB%B8%90_%EC%BF%A1" title="스티븐 쿡">스티븐 쿡</a>이 제시했지만, 그가 용어를 처음 만든 것은 아니다. 쿡은 <a href="/wiki/%EC%B6%A9%EC%A1%B1_%EA%B0%80%EB%8A%A5%EC%84%B1_%EB%AC%B8%EC%A0%9C" title="충족 가능성 문제">충족 가능성 문제</a>(SAT)가 위 정의의 두 가지 조건을 모두 만족한다는 것을 증명했다(<a href="/wiki/%EC%BF%A1-%EB%A0%88%EB%B9%88_%EC%A0%95%EB%A6%AC" title="쿡-레빈 정리">쿡-레빈 정리</a>). <a href="/wiki/1972%EB%85%84" title="1972년">1972년</a>에 <a class="new" href="/w/index.php?title=%EB%A6%AC%EC%B2%98%EB%93%9C_%EC%B9%B4%ED%94%84&amp;action=edit&amp;redlink=1" title="리처드 카프 (없는 문서)">리처드 카프</a>는 21가지의 다른 문제들도 마찬가지로 위 두 가지 조건을 만족한다는 것을 증명했다. 쿡이 최초의 NP-완전 문제를 발견한 이후로, 많은 사람들이 NP-완전 문제라고 이미 알려진 문제로부터 환산하는 방법으로 수많은 문제들 역시 NP-완전임을 증명했다. <a class="new" href="/w/index.php?title=%EB%A7%88%EC%9D%B4%ED%81%B4_%EA%B2%8C%EB%A6%AC&amp;action=edit&amp;redlink=1" title="마이클 게리 (없는 문서)">게리</a>와 <a href="/wiki/%EB%8D%B0%EC%9D%B4%EB%B9%84%EB%93%9C_S._%EC%A1%B4%EC%8A%A8" title="데이비드 S. 존슨">존슨</a>이 <a href="/wiki/1979%EB%85%84" title="1979년">1979년</a>에 <i>Computers and Intractability: A Guide to NP-Completeness</i>를 통해 여러 NP-완전 문제들을 소개하였다.</p>
<h2><span class="mw-headline" id=".EA.B7.BC.EC.82.AC_.EC.95.8C.EA.B3.A0.EB.A6.AC.EC.A6.98">근사 알고리즘</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NP-%EC%99%84%EC%A0%84&amp;action=edit&amp;section=3" title="부분 편집: 근사 알고리즘">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>현재 NP-완전 문제를 다항 시간 안에 푸는 알고리즘은 발견되지 않았고, 또한 그러한 알고리즘이 존재할 것인지도 알려지지 않았다. 대신, 특정한 NP-완전 문제에 대한 <a href="/wiki/%EA%B7%BC%EC%82%AC_%EC%95%8C%EA%B3%A0%EB%A6%AC%EC%A6%98" title="근사 알고리즘">근사 알고리즘</a>이 알려져 있다.</p>
