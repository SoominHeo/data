<div id="siteSub">위키백과, 우리 모두의 백과사전.</div>

<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><b>OSx86</b>은 <a href="/wiki/%EC%95%A0%ED%94%8C_(%EA%B8%B0%EC%97%85)" title="애플 (기업)">애플</a>의 <a class="mw-redirect" href="/wiki/OS_X" title="OS X">OS X</a> <a href="/wiki/%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C" title="운영 체제">운영 체제</a>를 비 매킨토시 환경에서 깔리게 하는 해킹 프로젝트이다. OSx86은 <a href="/wiki/2005%EB%85%84" title="2005년">2005년</a> <a href="/wiki/6%EC%9B%94" title="6월">6월</a> 애플의 세계 개발자 회의(Worldwide Developers Conference, WWDC)에서, 예전에 쓰던 파워 PC를 인텔 마이크로프로세서로 전환할 것을 발표한 것에 기인한다.</p>
<p>최초의 시도는 애플이 999달러에 판매한 개발자 전환 키트의 DVD가 누출된 것이었다. OSx86의 첫 번째 패치는 개발자 전환 키트에 포함된 <a class="mw-redirect" href="/wiki/%EB%A1%9C%EC%A7%81%EB%B3%B4%EB%93%9C" title="로직보드">로직보드</a>에 장착된 <a href="/wiki/%EC%8B%A0%EB%A2%B0_%ED%94%8C%EB%9E%AB%ED%8F%BC_%EB%AA%A8%EB%93%88" title="신뢰 플랫폼 모듈">신뢰 플랫폼 모듈</a>(Trusted Platform Module)을 우회하는 데 초점을 맞췄다. TPM은 파워피씨 프로그램을 인텔 기반의 매킨토시에서 구동하게 만들어 주는 <a href="/wiki/%EB%A1%9C%EC%A0%9C%ED%83%80_(%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4)" title="로제타 (소프트웨어)">로제타</a>를 기반으로 하는데, 로제타를 제거하면, OS X은 비 매킨토시에서 설치가 되었다. 로제타는 SSE3 명령을 필요로 하는데, 이 패치는 SSE2 상태를 에뮬레이트하고, SSE3를 지원하지 않는 <a href="/wiki/%EC%A4%91%EC%95%99_%EC%B2%98%EB%A6%AC_%EC%9E%A5%EC%B9%98" title="중앙 처리 장치">CPU</a>에서 설치를 지원하고 있다.</p>
<p>2005년 10월, 애플에서 개발자 키트를 10.4.3로 업데이트하면서, <a href="/wiki/NX_%EB%B9%84%ED%8A%B8" title="NX 비트">NX 비트</a>를 지원하는 마이크로프로세서가 필요하게 하였다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> 패치는 이 업데이트를 회피해서 다시 나오게 되었다. <sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>이 같은 방법으로 OS X을 이용하는 컴퓨터를 때로 <b>해킨토시</b>(Hackintosh)라 부르는데, 이는 본래 맥 시스템을 운용하도록 개조된 <a class="new" href="/w/index.php?title=%EB%A6%AC%EC%82%AC_2/10&amp;action=edit&amp;redlink=1" title="리사 2/10 (없는 문서)">리사 2/10</a>을 가리키는 용어기도 하였다.</p>

<h2><span class="mw-headline" id=".EB.A7.A5_OS_X_10.4_.ED.83.80.EC.9D.B4.EA.B1.B0">맥 OS X 10.4 타이거</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OSx86&amp;action=edit&amp;section=1" title="부분 편집: 맥 OS X 10.4 타이거">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/2006%EB%85%84" title="2006년">2006년</a> <a href="/wiki/1%EC%9B%94_10%EC%9D%BC" title="1월 10일">1월 10일</a>, 애플은 최초의 인텔 기반의 맥킨토시에 돌아가는 맥 OS X 10.4.4 버전을 발매하였다. 이 컴퓨터는 x86에서 쓰이던 BIOS와 다른 <a class="mw-redirect" href="/wiki/%ED%99%95%EC%9E%A5_%ED%8E%8C%EC%9B%A8%EC%96%B4_%EC%9D%B8%ED%84%B0%ED%8E%98%EC%9D%B4%EC%8A%A4" title="확장 펌웨어 인터페이스">EFI</a>(확장 펌웨어 인터페이스) 칩을 장착하였다. <a href="/wiki/2006%EB%85%84" title="2006년">2006년</a> <a href="/wiki/1%EC%9B%94_14%EC%9D%BC" title="1월 14일">1월 14일</a> 인터넷에서 Maxxuss라는 이름의 사용자가 최초로 OS X 10.4.4 버전을 크랙하였다. <sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> 그러나, 몇시간 안에 애플에서는 10.4.5 업데이트를 하였다.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup>, 그러나, Maxxuss가 2주내에 그에 대한 패치를 내놓았다. <sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> <a href="/wiki/2006%EB%85%84" title="2006년">2006년</a> <a href="/wiki/4%EC%9B%94_3%EC%9D%BC" title="4월 3일">4월 3일</a> 애플에서는 맥 OS X의 10.4.6 업데이트를 내놓았다.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> 그리고 2주안에 10.4.6. 커널은 포함되지 않았지만 말이다. 10.4.6. 패치를 처음 내놓은 SemjaZa와 컴파일한 JaS는 6월에 매킨토시가 아닌 컴퓨터에서 돌아가는 10.4.7. 패치를 내놓았지만, 여전히 10.4.4. 커널을 사용하고 있다.</p>
<p>10.4.8 업데이트가 나와도, 패치의 커널은 아직도 10.4.4를 쓰고 있었다. 그러나 많은 업데이트들이 프레임워크에 대해서 10.4.8 커널의 업데이트가 필요하였다. 애플은 또 예전보다 SSE3 구조를 더 많이 쓰도록 만들어, SSE2 사용자들의 시스템은 제 성능을 발휘하지 못하게 되었다.</p>
<p>이러한 까닭에, Mifki/Vitaliy 와 Semthex라는 두 명의 숙련된 프로그래머에 의해서, 10.4.8. 커널이 극적으로 등장하게 되었다. 그들은 <a href="/wiki/%EC%98%A4%ED%94%88_%EC%86%8C%EC%8A%A4" title="오픈 소스">오픈 소스</a>인 XNU 트리에서 출발하여, 비 맥킨토시 하드웨어에서 돌릴 수 있는 패치를 적용할 수 있었다. Mifki이 커널을 공개함에 따라, 여러 패치들이 뒤이어 나오게 되었고, 닫힌 애플의 하드웨어에서 잘 돌아가게 되었다. 자신의 일을 커뮤니티에 돌린 Semthex의 커널은 여러 하드웨어를 사용할 수 있게 되었고, 여러 중대한 결점들을 제거하였다.</p>
<p>이 커널들은 여러 kexts/framework에서 벌어지는 일들을 허용해서, 일반 PC를 정품 매킨토시처럼 만들었다. Mifki가 그의 커널을 업데이트만 했지만, Semthex는 자신의 커널의 새 버전에 <a href="/wiki/%EC%96%B4%EB%93%9C%EB%B0%B4%EC%8A%A4%ED%8A%B8_%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C_%EB%94%94%EB%B0%94%EC%9D%B4%EC%8B%9C%EC%8A%A4" title="어드밴스트 마이크로 디바이시스">AMD</a>, VMware와 SSE2를 지원하게 되었다. Semthex는 자신의 해킹 커널의 소스 코드를 자신의 웹사이트에 공개하였다. 12월 24일 Semthex는 그의 최초의 소스 트리와 파일이 다른 SSE3 커널을 크리스마스 선물로 공개하였다. Semthex와 Rufus가 개발한 SSE2 에뮬레이션에 대한 특별한 배려를 하여야 하는데, 이 에뮬레이션은 OSx86의 역사에서 최초의 SSE3 구조를 완벽하게 가상으로 구현한 것이다. 예전 에뮬레이터는 불완전하였고, 새로 나온 에뮬레이터보다 느렸다. 이 효과는 높은 3차원 프로그램과, 아이튠스 프로그램을 돌릴 때 뚜렷하게 나타났다.</p>
<p>타이거 시리즈가 10.4.11을 마지막으로 개발이 중단되자 타이거 버전 해킨토시 개발도 XxX의 XxX 10.4.11을 마지막으로 중단되었다.</p>
<h2><span class="mw-headline" id=".EB.A7.A5_OS_X_10.5_.EB.A0.88.ED.8D.BC.EB.93.9C">맥 OS X 10.5 레퍼드</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OSx86&amp;action=edit&amp;section=2" title="부분 편집: 맥 OS X 10.5 레퍼드">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>해킨토시는 애플의 <a href="/wiki/%EB%A7%A5_OS_X_10.5" title="맥 OS X 10.5">레퍼드</a> 출시와 함께 새로운 전환기를 맞게 된다. Kalyway라는 이름의 해커가 이전 타이거 시리즈보다 훨씬 안정적이고 부드러운 《Kalyway 10.5》 와 《Kalyway 10.5.1》을 공개하였고, 얼마 지나지 않아 보다 손쉬운 설치를 위해 DVD 인스톨 버전도 내놓았다. DVD 인스톨 버전으로 인해 해킨토시의 설치가 훨씬 쉬워졌고 덕분에 해킨토시는 빠른 속도로 확산되었다. 애플이 10.5.2 업데이트를 내놓자 eddie라는 해커는 JaS라는 해커와 함께 《Leo4All》를 개발하였다. 현재 해킨토시는 10.5.8까지 나와있으며 가장 흔하게 구할 수 있는 버전은 Kalyway 버전과 Leo4All 버전과 iAtkos 버전과 iDeneb, iPc 버전이다.</p>
<h2><span class="mw-headline" id=".EB.A7.A5_OS_X_10.6_.EC.8A.A4.EB.85.B8_.EB.A0.88.ED.8D.BC.EB.93.9C"><a href="/wiki/%EB%A7%A5_OS_X_10.6" title="맥 OS X 10.6">맥 OS X 10.6 스노 레퍼드</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OSx86&amp;action=edit&amp;section=3" title="부분 편집: 맥 OS X 10.6 스노 레퍼드">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>매킨토시의 10.6 스노 레퍼드이다.</p>
<h2><span class="mw-headline" id=".ED.95.B4.ED.82.A8.ED.86.A0.EC.8B.9C_PC.EC.9D.98_.EC.B6.9C.EC.8B.9C">해킨토시 PC의 출시</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OSx86&amp;action=edit&amp;section=4" title="부분 편집: 해킨토시 PC의 출시">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Psystar_PC">Psystar PC</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OSx86&amp;action=edit&amp;section=5" title="부분 편집: Psystar PC">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/2008%EB%85%84" title="2008년">2008년</a> <a href="/wiki/2%EC%9B%94" title="2월">2월</a>, 미국의 중소업체 Psystar는 "세계 최초의 해킨토시 탑재 PC"를 출시하였다. 이 PC의 특징은 메인보드 자체에 EFI 에뮬레이터를 탑재하였다는 것이다. 따라서 <a href="/wiki/%EC%95%A0%ED%94%8C_(%EA%B8%B0%EC%97%85)" title="애플 (기업)">애플</a>에서 파는 정품DVD를 Psystar PC에 삽입하면 그 DVD는 PC를 정품 매킨토시로 인식하여 설치를 진행한다.</p>
<p>이에 관련해 애플은 <a class="mw-redirect" href="/wiki/%EC%B5%9C%EC%A2%85_%EC%82%AC%EC%9A%A9%EC%9E%90_%EB%9D%BC%EC%9D%B4%EC%84%A0%EC%8A%A4_%EB%8F%99%EC%9D%98" title="최종 사용자 라이선스 동의">EULA</a>을 위반했다며 소송을 걸 것이라고 맞섰다.</p>
