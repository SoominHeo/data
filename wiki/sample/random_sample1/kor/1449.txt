<p><b>SCons</b> 는 <a href="/wiki/%EC%98%A4%ED%94%88_%EC%86%8C%EC%8A%A4" title="오픈 소스">오픈 소스</a> 소프트웨어 빌드 도구이다. SCons는 <a href="/wiki/Autoconf" title="Autoconf">autoconf</a>/<a href="/wiki/Automake" title="Automake">automake</a>의 기능과 <a class="new" href="/w/index.php?title=Ccache&amp;action=edit&amp;redlink=1" title="Ccache (없는 문서)">ccache</a>와 같은 컴파일러 캐시를 통합한, 고전적인 <a href="/wiki/Make_(%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4)" title="Make (소프트웨어)">Make</a> 유틸리티의 대체품이다. 이전의 도구들과 비교하여, SCons는 더 쓰기 쉽고, 더 신뢰할 수 있고, 더 빠른 것을 목표로 한다.</p>

<p></p>
<h2><span class="mw-headline" id=".EC.A3.BC.EC.9A.94_.EA.B8.B0.EB.8A.A5">주요 기능</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=SCons&amp;action=edit&amp;section=1" title="부분 편집: 주요 기능">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>설정 파일이 <a href="/wiki/%ED%8C%8C%EC%9D%B4%EC%8D%AC" title="파이썬">파이썬</a> 스크립트이다: 사용자가 작성한 빌드 스크립트에 완전한 범용 프로그래밍 언어의 기능을 사용할 수 있다.</li>
<li><a class="mw-redirect" href="/wiki/C_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4" title="C 프로그래밍 언어">C</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a>과 <a href="/wiki/%ED%8F%AC%ED%8A%B8%EB%9E%80" title="포트란">포트란</a>에 대해, 자동 <a class="new" href="/w/index.php?title=%EC%A2%85%EC%86%8D%EC%84%B1_(%EC%BB%B4%ED%93%A8%ED%84%B0_%EA%B3%BC%ED%95%99)&amp;action=edit&amp;redlink=1" title="종속성 (컴퓨터 과학) (없는 문서)">종속성</a> 분석(Automatic dependency analysis)이 내장되어 있다.
<ul>
<li>make와 달리, 종속성을 획득하기 위한 "<tt>make depend</tt>"나 "<tt>make clean</tt>"<sup class="reference" id="cite_ref-autotools_note_1-0"><a href="#cite_note-autotools_note-1">[1]</a></sup>과 같은 별도의 명령어는 필요하지 않다. 종속성 분석(Dependency analysis)은 다른 언어나 파일 종류를 위한 사용자 정의 종속성 스캐너를 통해 쉽게 확장할 수 있다.</li>
<li><a href="/wiki/GNU_%EB%B9%8C%EB%93%9C_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="GNU 빌드 시스템">autotools</a>와 달리, <a href="/wiki/GNU_%EC%BB%B4%ED%8C%8C%EC%9D%BC%EB%9F%AC_%EB%AA%A8%EC%9D%8C" title="GNU 컴파일러 모음">gcc</a>의 내장 종속성 분석을 사용하지 않는다. 그 대신, "<tt>#include</tt>"에 대한 <a href="/wiki/%EC%A0%95%EA%B7%9C_%ED%91%9C%ED%98%84%EC%8B%9D" title="정규 표현식">regexp</a> 스캔을 모든 C/C++ 소스 파일에 행한다. gcc 기반 빌드에서는, 종속성 생성이 더 느려질 수 있고(추가적인 스캔이 언제나 필요하므로). 신뢰성이 더 낮아질 수 있다(<tt>-DSOMETHING</tt>와 같은 전처리기 플래그 같은 것이 무시된다는 의미에서). 그러나 이것은 비-gcc 컴파일러에 호환성이 더 좋은 방식이다.</li>
</ul>
</li>
<li><a class="mw-redirect" href="/wiki/C_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4" title="C 프로그래밍 언어">C</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a>, <a class="mw-redirect" href="/wiki/D_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4" title="D 프로그래밍 언어">D</a>, <a href="/wiki/%EC%9E%90%EB%B0%94_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="자바 (프로그래밍 언어)">Java</a>, <a href="/wiki/%ED%8F%AC%ED%8A%B8%EB%9E%80" title="포트란">Fortran</a>, <a class="mw-redirect" href="/wiki/Objective-C" title="Objective-C">Objective-C</a>, <a href="/wiki/Yacc" title="Yacc">Yacc</a>, <a class="new" href="/w/index.php?title=Lex_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4&amp;action=edit&amp;redlink=1" title="Lex 프로그래밍 언어 (없는 문서)">Lex</a>, <a class="mw-redirect" href="/wiki/Qt_(%ED%88%B4%ED%82%B7)" title="Qt (툴킷)">Qt</a>과 <a href="/wiki/SWIG" title="SWIG">SWIG</a>와 <a href="/wiki/TeX" title="TeX">TeX</a> 및 <a href="/wiki/LaTeX" title="LaTeX">LaTeX</a> 문서 빌드를 기본 지원한다. 다른 언어나 파일 종류는 사용자 정의 빌더(Builder)를 통해 지원할 수 있다.</li>
<li>소스 코드 및 빌드 전 대상(pre-built targets)에 대한 중심 저장소(central repositories)에서의 빌드를 지원한다.</li>
<li><a class="new" href="/w/index.php?title=SCCS&amp;action=edit&amp;redlink=1" title="SCCS (없는 문서)">SCCS</a>, <a class="mw-disambig" href="/wiki/RCS" title="RCS">RCS</a>(Revision Control System), <a href="/wiki/CVS" title="CVS">CVS</a>(Concurrent Versions System), <a href="/wiki/%EC%84%9C%EB%B8%8C%EB%B2%84%EC%A0%84" title="서브버전">서브버전</a>, <a class="new" href="/w/index.php?title=%EB%B9%84%ED%8A%B8%ED%82%A4%ED%8D%BC&amp;action=edit&amp;redlink=1" title="비트키퍼 (없는 문서)">비트키퍼</a>와 <a class="mw-redirect" href="/wiki/%ED%8D%BC%ED%8F%AC%EC%8A%A4" title="퍼포스">퍼포스</a>(Perforce)에서 소스를 가져오는 것을 기본 지원한다.</li>
<li>.dsp, .dsw, .sln와 .vcproj 파일의 생성을 포함한 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EB%B9%84%EC%A3%BC%EC%96%BC_%EC%8A%A4%ED%8A%9C%EB%94%94%EC%98%A4" title="마이크로소프트 비주얼 스튜디오">마이크로소프트 비주얼 스튜디오 .NET</a>와 이전의 비주얼 스튜디오 버전을 기본 지원한다.</li>
<li>파일 내용 변화 감지에 <a href="/wiki/MD5" title="MD5">MD5</a> 시그너처를 사용한다: 옵션 설정으로, 전통적인 타임스탬프(timestamps) 방식도 지원한다.</li>
<li>디렉터리 체계와 관계없이 지정한 작업들(jobs)의 수를 유지하는 병렬 빌드를 지원한다.</li>
<li><tt>#include</tt> 파일과 라이브러리, 함수와 <tt>typedef</tt>들을 탐색하는 통합된 <a href="/wiki/Autoconf" title="Autoconf">Autoconf</a>와 유사한 기능을 지원한다.</li>
<li>모든 종속성을 전역적으로 조망한다: 복수의 빌드 패스나 타겟 순서 재지정이 필요하지 않다.</li>
<li>다중 빌드를 가속하기 위한 캐시 파일을 공유하는 능력이 있다: <a class="new" href="/w/index.php?title=Ccache&amp;action=edit&amp;redlink=1" title="Ccache (없는 문서)">ccache</a>와 유사하지만 C/C++ 컴파일 뿐만 아니라, 다른 모든 파일 종류를 지원한다.</li>
<li>크로스 플랫폼 빌드(cross-platform builds)를 위해 처음부터 차례로 끝까지 다시 설계했다: <a class="mw-redirect" href="/wiki/Linux" title="Linux">Linux</a>, 다른 <a href="/wiki/POSIX" title="POSIX">POSIX</a> 호환 운영 체제(<a class="mw-redirect" href="/wiki/AIX_(%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C)" title="AIX (운영 체제)">AIX</a>, <a class="new" href="/w/index.php?title=BSD/OS&amp;action=edit&amp;redlink=1" title="BSD/OS (없는 문서)">*BSD systems</a>, <a href="/wiki/HP-UX" title="HP-UX">HP-UX</a>, <a href="/wiki/IRIX" title="IRIX">IRIX</a> 와 <a href="/wiki/%EC%86%94%EB%9D%BC%EB%A6%AC%EC%8A%A4_(%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C)" title="솔라리스 (운영 체제)">솔라리스</a>, <a class="mw-redirect" href="/wiki/Mac_OS_X" title="Mac OS X">Mac OS X</a>), <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">윈도</a> 와 <a href="/wiki/OS/2" title="OS/2">OS/2</a>를 지원한다.</li>
</ul>
<h2><span class="mw-headline" id=".EA.B0.80.EC.9E.A5_.EB.8B.A8.EC.88.9C.ED.95.9C_SConstruct_.ED.8C.8C.EC.9D.BC">가장 단순한 SConstruct 파일</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=SCons&amp;action=edit&amp;section=2" title="부분 편집: 가장 단순한 SConstruct 파일">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
 <span class="n">Program</span><span class="p">(</span><span class="s1">'main.c'</span><span class="p">)</span>
</pre></div>
<p>사용자가 '<tt>scons</tt>' 명령을 실행하면, scons는 'main' 실행 파일 (<a class="mw-redirect" href="/wiki/Unix" title="Unix">Unix</a> 호환 OS에서) 또는 'main.exe' 실행 파일 (<a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">윈도</a> 에서)을 빌드한다.</p>
<h2><span class="mw-headline" id=".EC.97.AD.EC.82.AC">역사</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=SCons&amp;action=edit&amp;section=3" title="부분 편집: 역사">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>SCons는 2000년 8월의 SC<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> Build competition에서의 최우수상을 받은 'ScCons'라는 빌드 도구 설계에서 시작했다. 이 설계는 <a class="external text" href="http://www.dsmit.com/cons/" rel="nofollow">Cons</a> 라는 빌드 도구에 기반한 것이었다.</p>
