<p><b>제이펙</b>(<span lang="en" xml:lang="en">Joint Photographic Experts Group; <b>JPEG</b></span> <small>[ˈdʒeɪpɛɡ]<sup>[<a href="/wiki/%EC%9C%84%ED%82%A4%EB%B0%B1%EA%B3%BC:%EC%98%81%EC%96%B4%EC%9D%98_%ED%95%9C%EA%B8%80_%ED%91%9C%EA%B8%B0" title="위키백과:영어의 한글 표기">*</a>]</sup></small>, <span style="font-size: smaller;"><a href="/wiki/%EB%AC%B8%ED%99%94%EC%96%B4" title="문화어">문화어</a>:</span> 합동사진전문가단체<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup>)은 정지 화상을 위해서 만들어진 <a href="/wiki/%EC%86%90%EC%8B%A4_%EC%95%95%EC%B6%95" title="손실 압축">손실 압축</a> 방법 표준이다. 이 표준은 <a href="/wiki/%EA%B5%AD%EC%A0%9C_%ED%91%9C%EC%A4%80%ED%99%94_%EA%B8%B0%EA%B5%AC" title="국제 표준화 기구">ISO</a>와 <a class="new" href="/w/index.php?title=ITU_%EC%A0%84%EA%B8%B0%ED%86%B5%EC%8B%A0_%ED%91%9C%EC%A4%80%ED%99%94_%EB%B6%80%EB%AC%B8&amp;action=edit&amp;redlink=1" title="ITU 전기통신 표준화 부문 (없는 문서)">ITU-T</a>에서 제정하였다. JPEG를 사용하는 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%98%95%EC%8B%9D" title="파일 형식">파일 형식</a>들도 보통 JPEG 이미지라 불리며, .jpg, .jpeg, .jpe 등의 <a class="mw-redirect" href="/wiki/%ED%99%95%EC%9E%A5%EC%9E%90" title="확장자">확장자</a>를 사용한다. 16777216색과 256색 그레이로 저장 할 수 있다. JPEG 표준은 이미지가 어떻게 연속된 <a href="/wiki/%EB%B0%94%EC%9D%B4%ED%8A%B8" title="바이트">바이트</a>로 바뀌는지만을 규정한다. 독립 JPEG 그룹(Independent JPEG Group; IJG)에서 만든 JPEG의 확장인 <b>JFIF</b>(JPEG File Interchange Format)는 JPEG 스트림을 저장과 전송에 적합한 형태로 담는 이미지 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%98%95%EC%8B%9D" title="파일 형식">파일 형식</a>이다. <a href="/wiki/%EB%94%94%EC%A7%80%ED%84%B8_%EC%B9%B4%EB%A9%94%EB%9D%BC" title="디지털 카메라">디지털 카메라</a>의 사진 저장 방식으로는 다른 확장인 <a class="new" href="/w/index.php?title=EXIF&amp;action=edit&amp;redlink=1" title="EXIF (없는 문서)">EXIF</a> JPEG 형식이 더 자주 사용된다. 일반적으로 JPEG 파일이라고 할 때는 JFIF 형식이거나 EXIF JPEG 형식을 가리키지만, <a class="new" href="/w/index.php?title=JNG&amp;action=edit&amp;redlink=1" title="JNG (없는 문서)">JNG</a>와 같은 JPEG 기반의 다른 파일 형식도 존재한다.</p>
<p>손실 압축 형식이지만 파일 크기가 작기 때문에 웹에서 널리 쓰인다. 압축률을 높이면 파일 크기는 작아지지만 이미지 품질은 더욱 떨어진다.</p>
<p>JPEG/JFIF는 <a href="/wiki/%EC%9B%94%EB%93%9C_%EC%99%80%EC%9D%B4%EB%93%9C_%EC%9B%B9" title="월드 와이드 웹">웹</a> 상에서 사진 등의 화상을 보관하고 전송하는 데 가장 널리 사용되는 파일 형식이다. 하지만 이 압축 방법은 문자, 선, 세밀한 격자 등 고주파 성분이 많은 이미지의 변환에서는 <a href="/wiki/GIF" title="GIF">GIF</a>나 <a href="/wiki/PNG" title="PNG">PNG</a>에 비해 불리하며, 나쁜 품질을 보이는 경우가 많다. JPEG 표준에도 <a href="/wiki/%EB%B9%84%EC%86%90%EC%8B%A4_%EC%95%95%EC%B6%95" title="비손실 압축">비손실 압축</a> 방법이 정의되어 있지만 특허 문제와 압축률 등의 이유로 잘 사용되지는 않으며, <a href="/wiki/PNG" title="PNG">PNG</a> 등의 비손실 압축을 지원하는 포맷을 많이 사용한다.</p>

<h2><span class="mw-headline" id=".EC.95.95.EC.B6.95_.EA.B3.BC.EC.A0.95">압축 과정</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JPEG&amp;action=edit&amp;section=1" title="부분 편집: 압축 과정">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>JPEG 표준 안에는 많은 선택사항들이 있지만, 그중 대부분은 거의 사용되지 않는다. 아래 설명은 <a class="mw-redirect" href="/wiki/%ED%94%BD%EC%85%80" title="픽셀">픽셀</a>당 24비트 (빨강, 초록, 파랑 각 8비트씩)의 색상정보를 가진 데이터를 변환하는 일반적인 방법에 대한 간단한 설명이다. 아래 설명한 방식에선 <a href="/wiki/%EC%86%90%EC%8B%A4_%EC%95%95%EC%B6%95" title="손실 압축">손실 압축</a> 방식을 사용한다.</p>
<h3><span class="mw-headline" id=".EC.83.89.EA.B3.B5.EA.B0.84_.EB.B3.80.ED.99.98">색공간 변환</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JPEG&amp;action=edit&amp;section=2" title="부분 편집: 색공간 변환">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>우선 각 <a class="mw-redirect" href="/wiki/%ED%94%BD%EC%85%80" title="픽셀">픽셀</a>의 RGB 데이터를 <a href="/wiki/YCbCr" title="YCbCr">YCbCr</a>이라는 다른 <a class="mw-redirect" href="/wiki/%EC%83%89%EA%B3%B5%EA%B0%84" title="색공간">색공간</a> 데이터로 변환한다. 이 색공간은 <a href="/wiki/NTSC" title="NTSC">NTSC</a>나 <a href="/wiki/PAL" title="PAL">PAL</a> 방식의 컬러 <a href="/wiki/%ED%85%94%EB%A0%88%EB%B9%84%EC%A0%84" title="텔레비전">텔레비전</a> 전송에 쓰이는 색공간과 유사하다. Y 성분은 픽셀의 <a href="/wiki/%EB%A3%A8%EB%A7%88_(%EB%B9%84%EB%94%94%EC%98%A4)" title="루마 (비디오)">루마</a> 정보를 가지고 있으며, Cb와 Cr 성분은 색차 정보(chroma components)를 가지고 있다. 사람의 눈이 색상 성분보다 휘도 성분에 더 민감하기 때문에 색상 정보를 더 많이 압축하기 위해서 YCbCr 색공간으로 변환한다. 색공간 변환 시 제한된 정밀도 (채널당 8 비트) 때문에 정보가 완전히 보전되지 않는다. 따라서 뒤에서 <a href="/wiki/%EC%96%91%EC%9E%90%ED%99%94_(%EC%A0%95%EB%B3%B4_%EC%9D%B4%EB%A1%A0)" title="양자화 (정보 이론)">양자화</a>를 전혀하지 않더라도 원본과 완전히 일치하지 않는다. 즉, JPEG 저장시 화질을 100%으로 하더라도 원본과 차이가 있게 된다.</p>
<h3><span class="mw-headline" id=".ED.81.AC.EB.A1.9C.EB.A7.88_.EC.84.9C.EB.B8.8C.EC.83.98.ED.94.8C.EB.A7.81">크로마 서브샘플링</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JPEG&amp;action=edit&amp;section=3" title="부분 편집: 크로마 서브샘플링">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>때로는 Cb와 Cr 성분의 수를 줄인다. (이를 "다운 샘플링", 혹은 "크로마 서브샘플링"이라 한다.) 이어지는 설명은 각 성분에 대한 것으로, 성분별로 같은 절차가 진행된다.</p>
<h3><span class="mw-headline" id=".EC.96.91.EC.9E.90.ED.99.94">양자화</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JPEG&amp;action=edit&amp;section=4" title="부분 편집: 양자화">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>사람의 눈은 명도에 대해 비교적 넓은 영역에서 작은 차이를 구분할 수 있다. 그러나 고주파의 명도 변화에서는 그 구별 능력이 떨어진다. 이 사실에 근거하여 고주파 성분의 정보의 많은 부분은 버릴 수 있다. 이런 작업은 주파수 영역의 각 성분에 대해 특정 상수로 나누고 정수 몫만을 취함으로써 수행된다. 이 작업이 전체 압축 과정에서 가장 손실이 큰 부분이다. 이 방법으로 고주파 성분들은 거의 0이나 0에 근접한 양수, 혹은 음수가 된다. (여기서 말하는 주파수는 전자기파의 주파수를 말하는 것이 아니라, 데이터를 격자 상에 늘어 놓았을 때의 공간적인 변화를 말하는 것에 주의한다.) 그런데 정지 영상의 특성상 평균적으로 픽셀값이 확 변하는 곳이 적다(에너지 집중 현상으로 인해 대부분 저주파수에 모여있다. 그래서 양자화를 할 때 가중치 양자화를 적용함으로써 에너지가 큰 저주파는 많이 살려내고, 에너지가 거의 없어서 잘라내도 총합 에너지엔 거의 차이가 없는 고주파를 많이 제거한다.).</p>
<h3><span class="mw-headline" id=".EC.97.94.ED.8A.B8.EB.A1.9C.ED.94.BC_.EB.B6.80.ED.98.B8.ED.99.94">엔트로피 부호화</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JPEG&amp;action=edit&amp;section=5" title="부분 편집: 엔트로피 부호화">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>이 과정은 <a href="/wiki/%EB%B9%84%EC%86%90%EC%8B%A4_%EC%95%95%EC%B6%95" title="비손실 압축">비손실 압축</a> 방식으로서, 8×8 격자의 성분들을 저주파부터 지그재그로 일렬로 세운 뒤. 반복된 0에만 <a class="mw-redirect" href="/wiki/%EB%B0%98%EB%B3%B5_%EA%B8%B8%EC%9D%B4_%EB%B6%80%ED%98%B8%ED%99%94" title="반복 길이 부호화">RLE</a>를 적용하고, 그 결과들에 대해 <a href="/wiki/%ED%97%88%ED%94%84%EB%A7%8C_%EB%B6%80%ED%98%B8%ED%99%94" title="허프만 부호화">허프만 부호화</a>를 한다. 표준은 허프만 부호화 대신에 더 압축률이 좋은 <a href="/wiki/%EC%82%B0%EC%88%A0_%EB%B6%80%ED%98%B8%ED%99%94" title="산술 부호화">산술 부호화</a>도 가능하다고 기술하지만 산술 코딩은 인코딩, 디코딩시의 지연, 경제성 등의 이유를 들어 잘 쓰이지 않는다.</p>
<h2><span class="mw-headline" id=".EC.95.95.EC.B6.95_.ED.95.B4.EC.A0.9C_.EA.B3.BC.EC.A0.95">압축 해제 과정</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JPEG&amp;action=edit&amp;section=6" title="부분 편집: 압축 해제 과정">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>압축 해제된 JPEG 데이터를 얻는 과정은 압축 과정을 역으로 취함으로써 이루어진다.</p>
