<h1 class="firstHeading" id="firstHeading" lang="ko">오픈도큐먼트</h1>

<p><b>오픈도큐먼트</b>(OpenDocument, <b>OD</b>) 또는 <b>개방형문서</b>(Open Document Format for Office Applications, <b>ODF</b>)는 <a href="/wiki/%EC%8A%A4%ED%94%84%EB%A0%88%EB%93%9C%EC%8B%9C%ED%8A%B8" title="스프레드시트">스프레드시트</a>, <a href="/wiki/%EC%B0%A8%ED%8A%B8" title="차트">차트</a>, <a href="/wiki/%ED%94%84%EB%A0%88%EC%A0%A0%ED%85%8C%EC%9D%B4%EC%85%98" title="프레젠테이션">프레젠테이션</a>, <a href="/wiki/%EB%8D%B0%EC%9D%B4%ED%84%B0%EB%B2%A0%EC%9D%B4%EC%8A%A4" title="데이터베이스">데이터베이스</a>, <a href="/wiki/%EC%9B%8C%EB%93%9C_%ED%94%84%EB%A1%9C%EC%84%B8%EC%84%9C" title="워드 프로세서">워드 프로세서</a>를 비롯한 사무용 전자 문서를 위한 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%98%95%EC%8B%9D" title="파일 형식">파일 형식</a>이다.</p>
<p>이 형식은 원래 <a href="/wiki/%EC%98%A4%ED%94%88%EC%98%A4%ED%94%BC%EC%8A%A4" title="오픈오피스">오픈오피스</a>에서 만들고 구현한 <a href="/wiki/XML" title="XML">XML</a> 파일 형식을 바탕으로, <a class="new" href="/w/index.php?title=OASIS&amp;action=edit&amp;redlink=1" title="OASIS (없는 문서)">OASIS</a>(Organization for the Advancement of Structured Information Standards) 컨소시엄이 표준화하였다. <a href="/wiki/2006%EB%85%84" title="2006년">2006년</a>에는 <a href="/wiki/%EA%B5%AD%EC%A0%9C_%ED%91%9C%EC%A4%80%ED%99%94_%EA%B8%B0%EA%B5%AC" title="국제 표준화 기구">국제 표준화 기구</a> 및 <a href="/wiki/%EA%B5%AD%EC%A0%9C_%EC%A0%84%EA%B8%B0_%ED%91%9C%EC%A4%80_%ED%9A%8C%EC%9D%98" title="국제 전기 표준 회의">국제 전기 표준 회의</a>의 인증을 받아 <i>ISO/IEC 26300:2006</i><sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> 으로 발표되었다. 오픈도큐먼트 표준은 <a href="/wiki/%EA%B0%9C%EB%B0%A9%ED%98%95_%ED%91%9C%EC%A4%80" title="개방형 표준">개방형 표준</a>의 정의에 부합하며, 누구나 자유롭게 표준을 열람하고 구현할 수 있다.</p>

<h2><span class="mw-headline" id=".EA.B0.9C.EC.9A.94">개요</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%98%A4%ED%94%88%EB%8F%84%ED%81%90%EB%A8%BC%ED%8A%B8&amp;action=edit&amp;section=1" title="부분 편집: 개요">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>오픈도큐먼트 문서들이 보편적으로 사용하는 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%99%95%EC%9E%A5%EC%9E%90" title="파일 확장자">파일 확장자</a>는 다음과 같다.</p>
<ul>
<li><code>.odt</code> : <a href="/wiki/%EC%9B%8C%EB%93%9C_%ED%94%84%EB%A1%9C%EC%84%B8%EC%84%9C" title="워드 프로세서">워드 프로세서</a> (텍스트) 문서</li>
<li><code>.ods</code> : <a href="/wiki/%EC%8A%A4%ED%94%84%EB%A0%88%EB%93%9C%EC%8B%9C%ED%8A%B8" title="스프레드시트">스프레드시트</a></li>
<li><code>.odp</code> : <a href="/wiki/%ED%94%84%EB%A0%88%EC%A0%A0%ED%85%8C%EC%9D%B4%EC%85%98" title="프레젠테이션">프레젠테이션</a></li>
<li><code>.odb</code> : <a href="/wiki/%EB%8D%B0%EC%9D%B4%ED%84%B0%EB%B2%A0%EC%9D%B4%EC%8A%A4" title="데이터베이스">데이터베이스</a></li>
<li><code>.odg</code> : <a href="/wiki/%EA%B7%B8%EB%9E%98%ED%94%BD" title="그래픽">그래픽</a> (도형)</li>
<li><code>.odf</code> : <a href="/wiki/%EC%88%98%EC%8B%9D" title="수식">수식</a></li>
</ul>
<p>기본적인 오픈도큐먼트 문서는 최상위 요소가 <code>&lt;office:document&gt;</code>인 <a href="/wiki/XML" title="XML">XML</a> 문서이며, 그림과 같은 다른 파일들이 포함된 경우에는 <a class="mw-disambig" href="/wiki/ZIP" title="ZIP">ZIP</a> 형식으로 하나의 파일로 압축할 수도 있다. 오픈도큐먼트는 내용과 스타일, 메타데이터, 프로그램 설정 등을 네 개의 XML 파일에 분리하여 저장함으로써 작업의 분리(SoC)를 구현한다.</p>
<h2><span class="mw-headline" id=".EA.B5.AC.EC.84.B1">구성</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%98%A4%ED%94%88%EB%8F%84%ED%81%90%EB%A8%BC%ED%8A%B8&amp;action=edit&amp;section=2" title="부분 편집: 구성">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>ODF는 하나의 XML 문서로도 표현이 가능하지만, 일반적으로 여러개의 XML 문서와 그림과 같은 이진 파일을 하나의 zip 컨테이너 안에 묶는 형태이다. ODF 파일의 구성은 다음과 같다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<ul>
<li>mimetype</li>
</ul>
<dl>
<dd>문서의 MIME 타입을 기술한다.</dd>
</dl>
<ul>
<li>meta.xml</li>
</ul>
<dl>
<dd>문서의 메타데이터를 포함한다.</dd>
</dl>
<ul>
<li>content.xml</li>
</ul>
<dl>
<dd>문서의 실제 내용을 담고 있는 파일이다.</dd>
</dl>
<ul>
<li>styles.xml</li>
</ul>
<dl>
<dd>문서에 사용되는 스타일을 담고 있는 파일이다.</dd>
</dl>
<ul>
<li>Pictures (폴더)</li>
</ul>
<dl>
<dd>문서에서 사용하는 그림 파일을 담고 있는 폴더이다. 문서에서 그림을 참조할 때는 XLink 표준을 사용한다.</dd>

<h2><span class="mw-headline" id=".ED.91.9C.EC.A4.80.ED.99.94">표준화</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%98%A4%ED%94%88%EB%8F%84%ED%81%90%EB%A8%BC%ED%8A%B8&amp;action=edit&amp;section=3" title="부분 편집: 표준화">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>오픈도큐먼트의 표준은 <a class="new" href="/w/index.php?title=OASIS&amp;action=edit&amp;redlink=1" title="OASIS (없는 문서)">OASIS</a> 컨소시엄 산하 기술위원회(Technical Committee; TC)가 제정하였다. ODF-TC 에는 많은 업체들과 개인이 동등한 자격으로 참가하고 있으며, 많은 오피스 프로그램과 관련 문서 시스템의 개발자들이 표준화 작업을 진행하고 있다.</p>

<li><a href="/wiki/2005%EB%85%84" title="2005년">2005년</a> <a href="/wiki/5%EC%9B%94_11%EC%9D%BC" title="5월 11일">5월 11일</a> : 오픈도큐먼트를 OASIS 표준으로 승인</li>

<p>ODF 표준의 구현 방식이 굉장히 다양하기 때문에, 실제 구현 수준과 완성도도 프로그램에 따라 다르다. 따라서 <a class="new" href="/w/index.php?title=%EC%A0%95%EB%B3%B4_%EC%B2%98%EB%A6%AC_%EC%83%81%ED%98%B8_%EC%9A%B4%EC%9A%A9&amp;action=edit&amp;redlink=1" title="정보 처리 상호 운용 (없는 문서)">정보 처리 상호 운용</a> 테스트가 필요하다. 오픈도큐먼트 협회에서는 다양한 구현 방식에 따른 테스트와 점수를 공개하고 있다.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>오픈도큐먼트 재단과 협력 단체들은 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">마이크로소프트</a> 社의 제품에서도 오픈도큐먼트를 사용할 수 있는 플러그인과 필터를 개발한다고 밝혔다.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup><sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<p><a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%98%A4%ED%94%BC%EC%8A%A4" title="마이크로소프트 오피스">마이크로소프트 오피스</a>는 오픈도큐먼트를 지원하지 않지만, 오픈도큐먼트와 <a href="/wiki/%EC%98%A4%ED%94%BC%EC%8A%A4_%EC%98%A4%ED%94%88_XML" title="오피스 오픈 XML">오피스 오픈 XML</a> 변환기 <sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> 를 통해 형식을 바꿀 수 있다. 이 프로젝트의 결과로 <a href="/wiki/%EC%86%8C%EC%8A%A4%ED%8F%AC%EC%A7%80" title="소스포지">소스포지</a>의 <a class="external text" href="http://sourceforge.net/projects/odf-converter" rel="nofollow">ODF add-in for Word</a> 프로젝트는 마이크로소프트로부터 자금을 지원 받았으며, 이 프로젝트의 목적은 <a class="mw-redirect" href="/wiki/BSD_%EB%9D%BC%EC%9D%B4%EC%84%A0%EC%8A%A4" title="BSD 라이선스">BSD 라이선스</a> 하에서 마이크로소프트 오피스를 자유롭게 사용할 수 있도록 하는 것이다. <a href="/wiki/2007%EB%85%84" title="2007년">2007년</a> 초반에 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9B%8C%EB%93%9C" title="마이크로소프트 워드">마이크로소프트 워드</a>를 위한 <a class="external text" href="http://odf-converter.sourceforge.net/download.html" rel="nofollow">version 1.0</a> 을 내놓았으며, 2007년 후반에 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%97%91%EC%85%80" title="마이크로소프트 엑셀">엑셀</a>과 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%ED%8C%8C%EC%9B%8C%ED%8F%AC%EC%9D%B8%ED%8A%B8" title="마이크로소프트 파워포인트">파워포인트</a>를 위한 버전을 내놓을 계획이다. 또한 선(Sun) 社에서도 마이크로소프트 오피스 2000, XP, 2003 을 위한 오픈도큐먼트 플러그인을 내놓았다.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></p>
<h2><span class="mw-headline" id=".EC.82.AC.EC.9A.A9_.ED.97.88.EA.B0.80">사용 허가</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%98%A4%ED%94%88%EB%8F%84%ED%81%90%EB%A8%BC%ED%8A%B8&amp;action=edit&amp;section=6" title="부분 편집: 사용 허가">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>오픈도큐먼트 규격은 <a class="external text" href="http://www.oasis-open.org/committees/tc_home.php?wg_abbrev=office" rel="nofollow">자유롭게 다운로드하여 사용</a> 할 수 있다.</p>
<p>주요 기여자인 <a href="/wiki/%EC%8D%AC_%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%8B%9C%EC%8A%A4%ED%85%9C%EC%A6%88" title="썬 마이크로시스템즈">썬 마이크로시스템즈</a> 사는 <a class="external text" href="http://www.oasis-open.org/committees/office/ipr.php" rel="nofollow">취소불가한 지적 재산 조항</a> 을 만들어 오픈도큐먼트 규격을 이용한 모든 제품에 대해 미국 및 외국 특허권을 행사하지 않을 것임을 밝혔다. 하지만 이 조항은 오픈도큐먼트를 이용한 제품이 다른 특허나 지적재산권을 침해하는 것에 대해서는 보증하지 않는다.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> 이 조항은 썬 사가 참여한 오픈도큐먼트 규격 버전에 한정되어 적용된다.</p>
<h2><span class="mw-headline" id=".EC.B1.84.ED.83.9D_.EA.B5.AD.EA.B0.80">채택 국가</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%98%A4%ED%94%88%EB%8F%84%ED%81%90%EB%A8%BC%ED%8A%B8&amp;action=edit&amp;section=7" title="부분 편집: 채택 국가">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>대한민국 정부는 OpenDocument를 2007년 한국 산업 표준인 KS X ISO/IEC 26300으로 채택하였다. 그러나 여전히 공공 문서는 .hwp 포맷과 형식으로 만들어지고 배포된다. 이러한 것에 대한 법률적인 규제가 2007년 이후로 만들어지지 않았으며, 한컴오피스 2010부터 개방형 문서를 지원하기 시작했다. 나머지 국가들의 지원 현황은 다음과 같다.</p>
