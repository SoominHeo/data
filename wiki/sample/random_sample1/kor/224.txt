<p><b>닷넷 프레임워크</b>(<span lang="en" xml:lang="en">.NET Framework</span>, 이전 이름: 닷넷)는 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">마이크로소프트</a>에서 개발한 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84%EC%9A%B0" title="마이크로소프트 윈도우">윈도우</a> 프로그램 개발 및 실행 환경이다. 네트워크 작업, 인터페이스 등의 많은 작업을 캡슐화하였고, 공통 언어 런타임(Common Language Runtime)(CLR)이라는 이름의 <a href="/wiki/%EA%B0%80%EC%83%81_%EB%A8%B8%EC%8B%A0" title="가상 머신">가상 머신</a> 위에서 작동한다.</p>

<h3><span class="mw-headline" id=".EA.B3.B5.ED.86.B5_.EC.96.B8.EC.96.B4_.EA.B8.B0.EB.B0.98">공통 언어 기반</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8B%B7%EB%84%B7_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=3" title="부분 편집: 공통 언어 기반">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a href="/wiki/%EA%B3%B5%ED%86%B5_%EC%96%B8%EC%96%B4_%EA%B8%B0%EB%B0%98" title="공통 언어 기반">공통 언어 기반</a>입니다.</div>
<p>.NET 프레임워크의 가장 중요한 콤포넌트는 <a href="/wiki/%EA%B3%B5%ED%86%B5_%EC%96%B8%EC%96%B4_%EA%B8%B0%EB%B0%98" title="공통 언어 기반">공통 언어 기반</a>(<span lang="en" xml:lang="en">Common Language Infrastructure</span>)이다. 다른 말로 CLI라고 한다. CLI를 둔 목적은 애플리케이션의 개발과 실행 시 언어에 종속적이지 않은 플랫폼을 제공하기 위해서이다. 예외 처리, 가비지 콜렉션, 보안, 호환 등을 위한 <a class="new" href="/w/index.php?title=%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4_%EC%BD%A4%ED%8F%AC%EB%84%8C%ED%8A%B8&amp;action=edit&amp;redlink=1" title="소프트웨어 콤포넌트 (없는 문서)">소프트웨어 콤포넌트</a>를 포함한다. <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">마이크로소프트</a>가 구현한 CLI를 일컬어, 공통 언어 런타임(<span lang="en" xml:lang="en">Common Language Runtime, CLR</span>)이라고 한다.</p>
<p>CLR은 다음과 같은 4개의 주요 부분으로 구성된다.</p>
<ul>
<li><a class="new" href="/w/index.php?title=%EA%B3%B5%ED%86%B5_%ED%83%80%EC%9E%85_%EC%8B%9C%EC%8A%A4%ED%85%9C&amp;action=edit&amp;redlink=1" title="공통 타입 시스템 (없는 문서)">공통 타입 시스템</a> (CTS)</li>
<li><a class="mw-redirect" href="/wiki/%EA%B3%B5%ED%86%B5_%EC%96%B8%EC%96%B4_%EC%8A%A4%ED%8E%99" title="공통 언어 스펙">공통 언어 스펙</a> (CLS)</li>
</ul>
<ul>
<li><a class="mw-redirect" href="/wiki/%EC%A0%80%EC%8A%A4%ED%8A%B8-%EC%9D%B8-%ED%83%80%EC%9E%84_%EC%BB%B4%ED%8C%8C%EC%9D%BC%EB%9F%AC" title="저스트-인-타임 컴파일러">저스트-인-타임 컴파일러</a> (JIT)</li>
<li><a class="new" href="/w/index.php?title=%EA%B0%80%EC%83%81_%EC%8B%A4%ED%96%89_%EC%8B%9C%EC%8A%A4%ED%85%9C&amp;action=edit&amp;redlink=1" title="가상 실행 시스템 (없는 문서)">가상 실행 시스템</a> (VES)</li>
</ul>
<h3><span class="mw-headline" id=".EC.96.B4.EC.85.88.EB.B8.94.EB.A6.AC">어셈블리</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8B%B7%EB%84%B7_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=4" title="부분 편집: 어셈블리">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a class="new" href="/w/index.php?title=%EB%8B%B7%EB%84%B7_%EC%96%B4%EC%85%88%EB%B8%94%EB%A6%AC&amp;action=edit&amp;redlink=1" title="닷넷 어셈블리 (없는 문서)">닷넷 어셈블리</a>입니다.</div>
<p>중간 언어인 MSIL 코드는 <a class="new" href="/w/index.php?title=%EB%8B%B7%EB%84%B7_%EC%96%B4%EC%85%88%EB%B8%94%EB%A6%AC&amp;action=edit&amp;redlink=1" title="닷넷 어셈블리 (없는 문서)">닷넷 어셈블리</a>에 저장된다. 닷넷 어셈블리는 윈도우 구현에서는 <a class="mw-redirect" href="/wiki/%ED%8F%AC%ED%84%B0%EB%B8%94_%EC%9D%B5%EC%8A%A4%ED%81%90%ED%84%B0%EB%B8%94" title="포터블 익스큐터블">포터블 익스큐터블</a>(PE) 파일이 된다. (EXE 혹은 DLL파일이다.) 닷넷 어셈블리는 <a class="new" href="/w/index.php?title=%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4_%EB%B0%B0%ED%8F%AC&amp;action=edit&amp;redlink=1" title="소프트웨어 배포 (없는 문서)">소프트웨어 배포</a> 및 판 높임, 보안 등에 있어서 .NET 유니트 역할을 한다. 어셈블리는 파일 한 개 이상 여러 개의 파일로 구성되는데, 파일들 중 한 개 파일은 매니페스트를 꼭 포함해야 한다. 어셈블리의 완전한 이름은 간단한 텍스트 이름 및 판(버전) 번호, 컬처, <a href="/wiki/%EA%B3%B5%EA%B0%9C_%ED%82%A4_%EC%95%94%ED%98%B8_%EB%B0%A9%EC%8B%9D" title="공개 키 암호 방식">공개 키 암호 방식</a>(<span lang="en" xml:lang="en">public key</span>) 토큰 등을 가지고 있다; 이 중 이름은 꼭 가지고 있어야 하지만, 나머지는 없어도 된다. 공개 열쇠 암호 토큰은 어셈블리가 작성될 때 생성된다. 이 토큰은 어셈블리의 이름과 어셈블리 파일들의 내용을 대표하는 고유의 값이다. <a class="mw-redirect" href="/wiki/%EA%B3%B5%EA%B0%9C_%EC%97%B4%EC%87%A0_%EC%95%94%ED%98%B8" title="공개 열쇠 암호">비밀 열쇠 암호</a>는 어셈블리의 작성자에게만 알려진다. 같은 공개 열쇠 암호 토큰을 갖고 있는 두 개의 어셈블리는 같은 어셈블리라는 사실이 보장된다. 어셈블리가 해커 등에 의해 함부로 변경되면, 공개 열쇠 암호를 이용하여 어셈블리가 변경되었는지를 감지해낼 수 있다.</p>
<h3><span class="mw-headline" id=".EB.A9.94.ED.83.80.EB.8D.B0.EC.9D.B4.ED.84.B0">메타데이터</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8B%B7%EB%84%B7_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=5" title="부분 편집: 메타데이터">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a class="new" href="/w/index.php?title=%EB%8B%B7%EB%84%B7_%EB%A9%94%ED%83%80%EB%8D%B0%EC%9D%B4%ED%84%B0&amp;action=edit&amp;redlink=1" title="닷넷 메타데이터 (없는 문서)">닷넷 메타데이터</a>입니다.</div>
<p>모든 공통 중간 언어(CIL)는 <a class="new" href="/w/index.php?title=%EB%8B%B7%EB%84%B7_%EB%A9%94%ED%83%80%EB%8D%B0%EC%9D%B4%ED%84%B0&amp;action=edit&amp;redlink=1" title="닷넷 메타데이터 (없는 문서)">닷넷 메타데이터</a>를 통해 "스스로 기술""된다. CLR은 적절한 메소드가 불리는지를 보장하기 위해 메타데이터를 검사한다. 메타데이터는 보통 랭귀지 컴파일러에 의해 생성된다. 하지만 개발자 자신이 커스텀 애트리뷰트를 사용하여 자신만의 메타데이터를 작성할 수도 있다. 또한 메터데이터는 어셈블리에 대한 모든 정보를 담고 있다.</p>
<h3><span class="mw-headline" id=".EB.B2.A0.EC.9D.B4.EC.8A.A4_.ED.81.B4.EB.9E.98.EC.8A.A4_.EB.9D.BC.EC.9D.B4.EB.B8.8C.EB.9F.AC.EB.A6.AC_.28BCL.29">베이스 클래스 라이브러리 (BCL)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8B%B7%EB%84%B7_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=6" title="부분 편집: 베이스 클래스 라이브러리 (BCL)">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a class="mw-redirect" href="/wiki/%EB%B2%A0%EC%9D%B4%EC%8A%A4_%ED%81%B4%EB%9E%98%EC%8A%A4_%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="베이스 클래스 라이브러리">베이스 클래스 라이브러리</a>입니다.</div>
<p>베이스 클래스 라이브러리(BCL)는 .NET 프레임워크를 사용하는 모든 언어가 사용할 수 있는 <a href="/wiki/%ED%81%B4%EB%9E%98%EC%8A%A4_(%EC%BB%B4%ED%93%A8%ED%84%B0_%EA%B3%BC%ED%95%99)" title="클래스 (컴퓨터 과학)">클래스</a>들의 라이브러리이다. 흔히 프레임워크 클래스 라이브러리(<span lang="en" xml:lang="en">Framework Class Library</span>) (FCL)이라고 잘못 불린다. (이것은 Microsoft.* 네임스페이스를 포함하는 BCL보다 큰 집합이다.) 파일 읽기나 쓰기, 그래픽 렌더링, 데이터베이스 조작, XML 조작 등과 같은 공통된 기능을 해주는 클래스들을 제공한다.</p>
