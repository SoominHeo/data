<p><b>비주얼 베이직</b>(<span lang="en" xml:lang="en">Visual Basic</span>, <span style="font-size: smaller;"><a href="/wiki/%EB%AC%B8%ED%99%94%EC%96%B4" title="문화어">문화어</a>:</span> 비쥬얼베이씨크)은 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">마이크로소프트</a>에서 만든 <a href="/wiki/%EB%B2%A0%EC%9D%B4%EC%A7%81" title="베이직">베이직</a> <a href="/wiki/%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4" title="프로그래밍 언어">프로그래밍 언어</a>의 일종으로, <a class="mw-redirect" href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EC%8A%A4%ED%8A%9C%EB%94%94%EC%98%A4" title="비주얼 스튜디오">마이크로소프트 비주얼 스튜디오</a> 제품군의 하나이다. 비주얼 베이직은 <a href="/wiki/%EC%82%AC%EA%B1%B4_%EA%B8%B0%EB%B0%98_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="사건 기반 프로그래밍">사건 기반 프로그래밍</a>의 <a href="/wiki/3%EC%84%B8%EB%8C%80_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4" title="3세대 프로그래밍 언어">3세대 프로그래밍 언어</a>이면서, 또한 마이크로소프트의 <a href="/wiki/%EC%BB%B4%ED%8F%AC%EB%84%8C%ED%8A%B8_%EC%98%A4%EB%B8%8C%EC%A0%9D%ED%8A%B8_%EB%AA%A8%EB%8D%B8" title="컴포넌트 오브젝트 모델">컴포넌트 오브젝트 모델</a>에 따른 풍부한 구성 요소를 가진 <a href="/wiki/%ED%86%B5%ED%95%A9_%EA%B0%9C%EB%B0%9C_%ED%99%98%EA%B2%BD" title="통합 개발 환경">통합 개발 환경</a> 및 <a href="/wiki/%EA%B3%A0%EC%86%8D_%EC%9D%91%EC%9A%A9_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%A8_%EA%B0%9C%EB%B0%9C" title="고속 응용 프로그램 개발">RAD</a>을 가리키기도 한다. 비주얼 베이직은 초보자가 접근하기 쉬운 장점을 가지고 있는데, 그 이유는 시각적인 개발 환경과 더불어 <a href="/wiki/%EB%B2%A0%EC%9D%B4%EC%A7%81" title="베이직">베이직</a> 언어의 연장선상에 있기 때문으로 간주된다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p>비주얼 베이직은 <a href="/wiki/%EB%B2%A0%EC%9D%B4%EC%A7%81" title="베이직">베이직</a>의 영향을 받았으며, <a href="/wiki/%EA%B7%B8%EB%9E%98%ED%94%BD_%EC%82%AC%EC%9A%A9%EC%9E%90_%EC%9D%B8%ED%84%B0%ED%8E%98%EC%9D%B4%EC%8A%A4" title="그래픽 사용자 인터페이스">GUI</a> 응용 프로그램의 <a class="mw-redirect" href="/wiki/RAD" title="RAD">RAD</a>개발을 가능하게 해 주며, <a href="/wiki/%EB%8D%B0%EC%9D%B4%ED%84%B0%EB%B2%A0%EC%9D%B4%EC%8A%A4" title="데이터베이스">데이터베이스</a>에 대한 접근을 <a class="new" href="/w/index.php?title=%EB%8D%B0%EC%9D%B4%ED%84%B0_%EC%95%A1%EC%84%B8%EC%8A%A4_%EC%98%A4%EB%B8%8C%EC%A0%9D%ED%8A%B8&amp;action=edit&amp;redlink=1" title="데이터 액세스 오브젝트 (없는 문서)">데이터 액세스 오브젝트</a>, <a href="/wiki/%EC%9B%90%EA%B2%A9_%EB%8D%B0%EC%9D%B4%ED%84%B0_%EC%98%A4%EB%B8%8C%EC%A0%9D%ED%8A%B8" title="원격 데이터 오브젝트">원격 데이터 오브젝트</a>, 또는 <a class="mw-redirect" href="/wiki/ActiveX_%EB%8D%B0%EC%9D%B4%ED%84%B0_%EC%98%A4%EB%B8%8C%EC%A0%9D%ED%8A%B8" title="ActiveX 데이터 오브젝트">ActiveX 데이터 오브젝트</a>들, 그리고 <a class="mw-redirect" href="/wiki/Active_X" title="Active X">Active X</a>컨트롤과 개체의 생성을 통해 가능하게 한다. <a href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%ED%8F%AC_%EC%95%A0%ED%94%8C%EB%A6%AC%EC%BC%80%EC%9D%B4%EC%85%98" title="비주얼 베이직 포 애플리케이션">비주얼 베이직 포 애플리케이션</a>이나 <a class="mw-redirect" href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8" title="비주얼 베이직 스크립트">비주얼 베이직 스크립트</a>와 같은 <a href="/wiki/%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8_%EC%96%B8%EC%96%B4" title="스크립트 언어">스크립트 언어</a>들은 문법적으로 비주얼 베이직과 유사하지만, 그 기능은 다르다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>프로그래머는 비주얼 베이직에서 제공하는 구성 요소를 응용 프로그램에 삽입할 수 있다. 또한 비주얼 베이직으로 만든 프로그램은 <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_API" title="윈도 API">윈도 API</a>를 사용할 수 있으나, 추가적인 외부 함수 선언이 필요하다.</p>
<p>비주얼 베이직의 마지막 버전은 1998년에 출시된 비주얼 베이직 6.0이다. 마이크로소프트의 연장 지원 기간은 2008년 3월에 끝났으며, 공식적인 후계 제품은 <a href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%EB%8B%B7%EB%84%B7" title="비주얼 베이직 닷넷">비주얼 베이직 닷넷</a>이다.</p>

<h2><span class="mw-headline" id=".ED.8A.B9.EC.A7.95">특징</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;section=1" title="부분 편집: 특징">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">

<p>프로그래밍 언어 <a href="/wiki/%EB%B2%A0%EC%9D%B4%EC%A7%81" title="베이직">베이직</a>과 마찬가지로, 비주얼 베이직은 쉽게 배우고 쉽게 사용할 수 있도록 설계되었다. 비주얼 베이직은 간단한 <a class="mw-redirect" href="/wiki/GUI" title="GUI">GUI</a>기반의 응용 프로그램의 개발 뿐만 아니라, 복잡한 프로그램의 개발까지도 가능하게 한다. 비주얼 베이직에서의 프로그래밍은 <a class="mw-disambig" href="/wiki/%ED%8F%BC" title="폼">폼</a>(Form)위에 놓인, 시적으로 정렬된 <a href="/wiki/GUI_%EC%9C%84%EC%A0%AF" title="GUI 위젯">GUI 위젯</a>(비주얼 베이직에서는 컨트롤이라고 한다)들의 조합이다. 이 컨트롤들은 특별한 속성과 역할을 담당하고 있으며, 기능의 확장을 위해 추가적인 코드의 입력이 가능하다.</p>
<p>기본적인 속성과 역할이 이들 컨트롤에 기본값으로 설정되어 있기 때문에, 간단한 프로그램의 경우 프로그래머는 그다지 많은 코딩 없이 만들 수 있다.</p>
<p>성능상의 문제가 이전 버전에서 많이 보고되었지만, 기술의 발전에 따른 컴퓨터 성능 향상과 원시 코드(Native Code) 컴파일을 통해 이 문제는 많이 사라졌다.</p>
<p>비주얼 베이직 5.0부터 원시 코드로 컴파일된 실행 파일 작성이 가능해 졌지만, 비주얼 베이직으로 작성된 실행 파일은 여전히 1메가바이트 정도 크기의 라이브러리 파일을 필요로 한다. 이 파일은 <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_2000" title="윈도 2000">윈도 2000</a>과 이후 윈도 버전에서 기본적으로 설치되었지만, 그 이전의 <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_95" title="윈도 95">윈도 95</a>, <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_98" title="윈도 98">윈도 98</a>, <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_NT" title="윈도 NT">윈도 NT</a>에는 포함되어 있지 않았다. 때문에 비주얼 베이직으로 만든 실행 파일을 실행하기 위해서는 이 라이브러리 파일 또한 함께 배포하여야 했다.</p>
<p>폼은 <a class="mw-redirect" href="/wiki/%EB%81%8C%EC%96%B4%EB%8B%A4_%EB%86%93%EA%B8%B0" title="끌어다 놓기">끌어다 놓기</a> 방식으로 만들어진다. 커서를 통해 텍스트 상자, 단추 등과 같은 컨트롤들을 폼 위에 놓는다. 컨트롤들은 속성을 가지며, 이벤트 처리자를 통해 서로 연결된다. 기본값들은 컨트롤이 만들어질 때 함께 제공된 것이지만, 프로그래머에 의해 변경될 수 있다. 많은 속성값들은 프로그램이 실행 중일 때, 응용 프로그램의 사용자의 조작이나 외부 환경의 변화에 의해 변경될 수 있으며, 이것은 동적인 응용 프로그램을 가능하게 해 준다. 예를 들면, 창의 'Resize(크기 변경)' 이벤트 처리자에 컨트롤들의 위치를 재조정하는 코드를 삽입할 수 있다. 이러한 방법을 통해서, 특정 컨트롤이 계속해서 창의 중앙에 위치하게 하거나, 창 전체를 다 채우는 등의 제어를 할 수 있게 된다. 텍스트 상자의 'KeyPress(키 눌림)' 이벤트 처리자에 코드를 삽입함으로써, 해당 응용 프로그램은 자동적으로 입력된 문자의 대소문자를 변환하거나, 특정 문자의 삽입을 막을 수도 있다.</p>
<p>비주얼 베이직은 실행 파일, ActiveX 컨트롤, DLL 파일을 만들 수 있다. 하지만 비주얼 베이직의 가장 기본적인 사용 용도는 <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">마이크로소프트 윈도</a>용 응용 프로그램과 데이터베이스 조작 프로그램의 개발에 있다. 덜 기능적인 대화 상자들은 팝업 능력에 사용된다. 프로그래머가 추가적인 코드를 적당한 이벤트 처리자에 삽입해야 하긴 하지만, 컨트롤들은 응용 프로그램에 기본적인 기능성을 제공한다. 예를 들면, 문자열의 입력이 가능한 드롭 다운 리스트 컨트롤은 자동적으로 자신이 가진 목록을 보여주며, 사용자가 항목을 선택할 수 있게 한다. 사용자에 의해 항목이 선택될 때, 이벤트 처리자가 호출되며, 이 이벤트 처리자는 프로그래머가 추가적으로 코딩한 코드를 처리하는데, 이때 사용자가 선택한 항목의 관련 항목의 출력과 같은, 사용자의 컨트롤에 대한 구체적인 동작에 따른 처리를 할 수 있다.</p>
<p>한편, 비주얼 베이직으로 사용자 인터페이스가 존재하지 않는 프로그램이나 추가적으로 제공되는 <a href="/wiki/%EC%BB%B4%ED%8F%AC%EB%84%8C%ED%8A%B8_%EC%98%A4%EB%B8%8C%EC%A0%9D%ED%8A%B8_%EB%AA%A8%EB%8D%B8" title="컴포넌트 오브젝트 모델">컴포넌트 오브젝트 모델</a>을 통한 Active X 개체를 구성할 수도 있다. 이것은 서버 측면에서의 처리나 추가 모듈을 가능하게 해 준다.</p>
<p>비주얼 베이직은 <a class="mw-redirect" href="/wiki/%EB%A0%88%ED%8D%BC%EB%9F%B0%EC%8A%A4_%EC%B9%B4%EC%9A%B4%ED%8C%85" title="레퍼런스 카운팅">레퍼런스 카운팅</a>을 통한 <a href="/wiki/%EC%93%B0%EB%A0%88%EA%B8%B0_%EC%88%98%EC%A7%91_(%EC%BB%B4%ED%93%A8%ED%84%B0_%EA%B3%BC%ED%95%99)" title="쓰레기 수집 (컴퓨터 과학)">쓰레기 수집</a>을 수행하며, 방대한 규모의 추가적인 라이브러리들과, 기본적인 <a href="/wiki/%EA%B0%9D%EC%B2%B4_%EC%A7%80%ED%96%A5_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="객체 지향 프로그래밍">객체 지향 프로그래밍</a>을 지원한다. 대다수의 일반적인 위젯들이 초기 프로젝트 구성에 포함되어 있기 때문에, 비주얼 베이직 프로그래머가 추가적인 라이브러리를 사용하는 경우는 드물다. 다른 대다수의 프로그래밍 언어와는 달리, 비주얼 베이직은 일반적으로 대소문자 구분을 하지 않으며, 대신에 키워드를 표준 대소문자 설정으로 변환한 뒤, 강제로 변수명의 대소문자를 기호 테이블의 내용으로 변환시킨다. 문자열 비교에 관해서는 대소문자 구분을 하지만, 프로그래머가 원한다면 대소문자 구분을 하지 않을 수도 있다.</p>
<p>비주얼 베이직의 컴파일러는 다른 비주얼 스튜디오 제품군(C, C++)과 호환성이 있지만, IDE상의 제한으로 인해 윈도 모델 DLL이나 스레드의 사용 등은 불가능하다.</p>
<h2><span class="mw-headline" id=".EB.8B.A4.EB.A5.B8_.EC.96.B8.EC.96.B4.EC.99.80.EC.9D.98_.EB.B9.84.EA.B5.90">다른 언어와의 비교</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;section=2" title="부분 편집: 다른 언어와의 비교">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>비주얼 베이직은 <a class="mw-redirect" href="/wiki/C_%EC%96%B8%EC%96%B4" title="C 언어">C 언어</a> 계열의 언어들과 비교했을 때, 다음과 같은 특징이 있다:</p>
<ul>
<li><a class="mw-redirect" href="/wiki/C_%EC%96%B8%EC%96%B4" title="C 언어">C 언어</a>에서 가능한 다중 할당은 불가능하다. 비주얼 베이직에서 코드 <tt>A = B = C</tt> 는 변수 A, B, C의 값을 같게 하지 않는다. "Is B = C?" 불린 결과값이 A에 들어간다.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> 결국 A에 들어가는 결과치는 거짓(0) 또는 참(-1)이 될 것이다.</li>
</ul>
<ul>
<li><a href="/wiki/%EB%B6%88%EB%A6%B0_%EC%9E%90%EB%A3%8C%ED%98%95" title="불린 자료형">불린</a> 상수 <tt>True</tt>는 숫자값 -1을 갖는다.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> 이렇게 처리하는 이유는, 비주얼 베이직에서 불린 자료형은 16비트 부호 있는 정수 형태로 저장되기 때문이다. 이 구조에서 -1은 16진수 1로 계산되고(불린 값은 <tt>True</tt>이다), 0은 16진수 0으로 계산된다.(불린 값은 <tt>False</tt>이다.) 이것은 16비트 부호 있는 정수값 0인 <tt>Not</tt> 연산을 할 때 정수값 -1을 반환하는 것을 통해 명백하게 설명되는데, 다시 말해 <tt>True = Not False</tt>이다. 이러한 비주얼 베이직의 고유한 처리 방식은 <tt>And</tt>, <tt>Or</tt>, <tt>Xor</tt>, <tt>Not</tt>과 같은 정수 비트의 논리 연산에 유용하다.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> 이러한 <tt>True</tt>의 정의는 1970년대 마이크로소프트가 만들었던 <a href="/wiki/%EB%B2%A0%EC%9D%B4%EC%A7%81" title="베이직">베이직</a> 제품에서의 처리와 일치하며, 이것은 당시의 CPU 명령어의 특성과 관련이 있다.</li>
<li>논리 및 비트 연산자가 통합되어 있다. 이것은 논리 연산자와 비트 연산자가 구분되어 있는 <a href="/wiki/%ED%8E%84" title="펄">펄</a>과 같은 C 언어 계열의 언어와는 다르다. 이 요소는 <a href="/wiki/%EB%B2%A0%EC%9D%B4%EC%A7%81" title="베이직">베이직</a>언어의 전통적인 특징이기도 하다.</li>
<li>가변 배열 기반이다. 배열의 정의는 파스칼이나 포트란과 같이, 유사한 최대 인덱스(Upper Bound)와 최소 인덱스(Lower Bound)의 지정을 통해 이루어진다. 또한 <tt>Option Base</tt> 구문을 통해 최소 인덱스의 지정을 할 수 있다. <tt>Option Base</tt> 문의 사용은 비주얼 베이직 코드를 볼 때 혼란을 야기할 수 있는데, 이것을 피하는 최선의 방법은 언제나 명시적으로 배열의 최소 인덱스를 지정하는 것이다. 이 최소 인덱스는 0 또는 1뿐만 아니라, 선언에 따라서는 다른 값이 될 수도 있기 때문이다. 이러한 방법으로, 최소 인덱스와 최대 인덱스 모두 프로그래머의 제어가 가능하다. 대조적으로, 서브스크립트 제한적 언어(Subscript-Limited Language)들에서, 배열의 최소 인덱스는 가변적이지 않다. 이러한 독특한 특성은 <a href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%EB%8B%B7%EB%84%B7" title="비주얼 베이직 닷넷">비주얼 베이직 닷넷</a>에서도 나타나지만, <a class="mw-redirect" href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8" title="비주얼 베이직 스크립트">비주얼 베이직 스크립트</a>에는 없다.</li>
</ul>
<dl>
<dd><code>OPTION BASE</code> 는 1970년대 후반 ANSI Minimal BASIC을 통해 표준으로 지정되었다.</dd>
</dl>
<ul>
<li>상대적으로 <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">마이크로소프트 윈도</a> 및 <a href="/wiki/%EC%BB%B4%ED%8F%AC%EB%84%8C%ED%8A%B8_%EC%98%A4%EB%B8%8C%EC%A0%9D%ED%8A%B8_%EB%AA%A8%EB%8D%B8" title="컴포넌트 오브젝트 모델">컴포넌트 오브젝트 모델</a>과 강력하게 통합되어 있다.</li>
<li>실수를 정수로 바꿀 때, 기본적으로 <code>Round</code>함수를 통해 <a href="/wiki/%EB%B0%98%EC%98%AC%EB%A6%BC" title="반올림">반올림</a>을 한다. 반올림 방법으로는 Round-to-even 을 사용한다.</li>
<li>일반적인 나누기 연산자인 <tt>/</tt>를 통해 자동적으로 정수가 실수로 변환되므로, 짝수 정수부에 의한 홀수 정수부는 직관적으로 올바른 연산 결과를 보여준다. 한편, 자동으로 반올림을 하는 정수 나누기 연산자인 <tt>\</tt>도 있다.</li>
<li>기본값으로, 만약 변수가 선언되지 않았거나 자료형이 선언되지 않을 경우, 변수는 <tt>Variant</tt>형으로 선언 된다. 그러나 이것은 <tt>DefInt</tt>, <tt>DefBool</tt>, <tt>DefVar</tt>, <tt>DefObj</tt>, <tt>DefStr</tt>와 같은 DefType 구문을 통해 바꿀 수 있다. 비주얼 베이직은 6.0의 경우 12개의 <tt>DefType</tt> 구문이 지원된다. 기본 형은 특별한 접두어인 <tt>#</tt>를 통한 선언을 통해 Double형으로, <tt>!</tt>를 통해 Single형으로, <tt>&amp;</tt>를 통해 Long 형으로, <tt>%</tt>형으로 Integer형으로, <tt>$</tt>를 통해 String형으로, <tt>@</tt>를 통해 Currency 형으로 오버라이드되거나, 예약어 <tt>As (type)</tt>를 통해 오버라이드 될 수 있다. 또한 무조건 명시적인 변수 선언만 허용하도록 할 수도 있는데, 이 때는 <tt>Option Explicit</tt> 명령을 사용한다.</li>
</ul>
<h2><span class="mw-headline" id=".EB.B0.9C.EC.A0.84">발전</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;section=3" title="부분 편집: 발전">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>비주얼 베이직 1.0은 <a href="/wiki/1991%EB%85%84" title="1991년">1991년</a> <a href="/wiki/5%EC%9B%94" title="5월">5월</a> 발표되었다. 사용자 인터페이스 개발에 있어서의 끌어다 놓기 방법은 <a href="/wiki/%EC%95%A8%EB%9F%B0_%EC%BF%A0%ED%8D%BC" title="앨런 쿠퍼">앨런 쿠퍼</a>가 개발한 프로토타입 폼 제작기에서 온 것이며, 쿠퍼가 재직했던 회사에서는 이 프로그램을 <i>트라이포드</i>라고 불렀다. 마이크로소프트는 윈도 3.0용 프로그래밍 가능 폼 제작 시스템의 개발을 위해 쿠퍼의 개발팀과 계약 한 후, 프로젝트의 코드네임을 <i>루비</i>라고 명명하였다. (<a href="/wiki/%EB%A3%A8%EB%B9%84_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="루비 (프로그래밍 언어)">루비 프로그래밍 언어</a>와는 관계 없다.)</p>
<p>트라이포드는 프로그래밍 언어를 포함하고 있지는 않았다. 마이크로소프트는 비주얼 베이직을 만들기 위해 루비와 베이직 언어를 합치기로 결정했다.</p>
<p>루비 인터페이스 개발기는 비주얼 베이직의 "시각적인" 부분이 되었으며, 이것은 다시 마이크로소프트의 취소된 데이터베이스 시스템인 "오메가"용으로 설계된 "EB"(임베디드 베이직 엔진)와 합쳐졌다. 루비는 또한 훗날 <a class="new" href="/w/index.php?title=%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%ED%99%95%EC%9E%A5&amp;action=edit&amp;redlink=1" title="비주얼 베이직 확장 (없는 문서)">비주얼 베이직 확장</a>(VBX)이라고 명명될 추가적인 컨트롤들을 포함하기 위한 <a href="/wiki/%EB%8F%99%EC%A0%81_%EB%A7%81%ED%81%AC_%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="동적 링크 라이브러리">DLL</a> 로드 기능을 지원하였다.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>버전 5.0에서 <a class="mw-redirect" href="/wiki/Win16" title="Win16">Win16</a> 코드와의 호환성을 버리고 전반적인 기능을 강화했다. 버전 6.0에서는 <a href="/wiki/%EC%9D%B8%ED%84%B0%EB%84%B7" title="인터넷">인터넷</a> 관련 기능이 대폭 강화되었다.</p>
<p><a href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%EB%8B%B7%EB%84%B7" title="비주얼 베이직 닷넷">비주얼 베이직 닷넷</a>은 기존 버전과 다르게, 닷넷의 일부분으로 동작하고 객체지향 프로그래밍이 강화되었다. 또한 기존 버전과의 호환성이 없기 때문에 기존 비주얼 베이직 코드를 비주얼 베이직 닷넷 코드로 옮겨 주는 마법사가 있다.</p>
<p>그러나 네이티브 비주얼 베이직은 버전 6.0 이후로 마이크로소프트가 완전히 포기하여 사라진 언어가 되고 말았다. 하지만 2011년 현재, 프로그래밍 입문 언어로서 시스템에 관한 구체적인 지식(변수, 메모리 등)이 없이도 프로그래밍의 시작이 가능하다는 점에서 또, 액셀 등 오피스에서 사용되는 VBA 의 문법이 비주얼 베이직 6.0 버전과 호환됨에 따라 아직도 많은 사람들이 입문용으로도 또 실무용으로도 사용하고 있다.</p>
<h3><span class="mw-headline" id=".EA.B0.9C.EB.B0.9C_.EB.B0.8F_.EB.B0.9C.EC.A0.84_.EA.B3.BC.EC.A0.95">개발 및 발전 과정</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;section=4" title="부분 편집: 개발 및 발전 과정">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>'선더(Thunder)' 프로젝트가 시작되었다.</li>
<li>1991년 5월, 미국 조지아주 애틀랜타에서 열린 컴덱스에서 윈도용 비주얼 베이직 1.0이 발매되었다.</li>
<li>1992년 3월, 도스용 비주얼 베이직 1.0이 발표되었다. 마이크로소프트의 도스 기반 베이직 컴파일러인 <a href="/wiki/%ED%80%B5%EB%B2%A0%EC%9D%B4%EC%A7%81" title="퀵베이직">퀵베이직</a>과 베이직 전문 개발 체계(BASIC Professional Development System)의 차기 버전으로 개발된 것이기 때문에, 언어적인 특성 자체는 윈도용 비주얼 베이직과 전혀 호환되지 않았다. 도스용 비주얼 베이직은 문자 기반 윈도(Character Oriented Windows)를 사용했으며, <a class="new" href="/w/index.php?title=%ED%99%95%EC%9E%A5_%EC%95%84%EC%8A%A4%ED%82%A4&amp;action=edit&amp;redlink=1" title="확장 아스키 (없는 문서)">확장 아스키</a>문자를 통해 GUI를 구현해 내었다.</li>
<li>1992년 11월, 비주얼 베이직 2.0이 발매되었다. 프로그래밍 환경이 더욱 쉽게 개선되었으며, 속도 또한 향상되었다. 특히, 폼이 인스턴스적인 개체가되었으며, 이러한 기본적인 클래스 모듈의 개념은 후에 비주얼 베이직 4.0에서 적용된다.</li>
<li>1993년 7월, 비주얼 베이직 3.0이 발매되었다. 이 때는 스탠다드 에디션과 프로페셔널 에디션으로 구분되어 발매되었다. 이 비주얼 베이직 3.0은 제트(또는 액세스)1.x 데이터베이스를 읽을 수 있는 <a class="new" href="/w/index.php?title=%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%A0%9C%ED%8A%B8_%EB%8D%B0%EC%9D%B4%ED%84%B0%EB%B2%A0%EC%9D%B4%EC%8A%A4_%EC%97%94%EC%A7%84&amp;action=edit&amp;redlink=1" title="마이크로소프트 제트 데이터베이스 엔진 (없는 문서)">마이크로소프트 제트 데이터베이스 엔진</a> 1.1을 포함하고 있었다.</li>
<li>1995년 8월, 비주얼 베이직 4.0이 발매되었다. 비주얼 베이직 4.0은 <a href="/wiki/32%EB%B9%84%ED%8A%B8" title="32비트">32비트</a>와 <a href="/wiki/16%EB%B9%84%ED%8A%B8" title="16비트">16비트</a> 윈도 프로그램 둘 다 개발할 수 있는 첫 번째 버전의 비주얼 베이직이었다. 또한 GUI가 없는 클래스를 비주얼 베이직에서 개발할 수 있는 기능이 추가되었다. 비주얼 베이직 4.0 제품들간의 비호환성은 설치와 운영상의 문제를 낳았다. 이전 버전의 비주얼 베이직은 VBX 컨트롤을 써 왔었지만, 비주얼 베이직 4.0은 OLE 컨트롤(확장자가 .OCX인 파일)을 썼다. 이 OLE 컨트롤은 훗날 ActiveX 컨트롤이라고 이름지어진다.</li>
<li>1997년 2월 발매된 비주얼 베이직 5.0은 32비트 버전으로만 발매되었다. 16비트 프로그램의 개발을 계속 했던 프로그래머는 비주얼 베이직 4.0의 소스 코드를 비주얼 베이직 5.0으로 옮길 수 있었으며, 비주얼 베이직 5.0 프로그램이 비주얼 베이직 4.0으로 옮겨 가기도 쉬웠다. 비주얼 베이직 5.0은 또한 사용자 정의 컨트롤을 만들 수 있는 기능, 또한 원시 코드로 윈도용 실행 파일을 컴파일 할 수 있는 기능, 대량의 계산에서의 속도 향상 등의 기능이 추가되었다. Active X 컨트롤의 개발을 위해 다운로드 받을 수 있는 무료 버전 또한 배포되었는데, 이 버전은 일종의 시험판으로, 일반적인 .exe 프로젝트가 IDE에서는 실행되었지만 컴파일되지는 않았다.</li>
<li>1998년 6월, 많은 영역에서 개선을 이룬 비주얼 베이직 6.0이 발매되었다.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> 특히 비주얼 베이직 6.0은 웹 기반 응용 프로그램의 개발 기능이 추가되었다. 비주얼 베이직 6.0은 2008년 3월 마이크로소프트의 "지원 안함" 단계에 해당 됨으로써 종료되었다. 이처럼 비주얼 베이직 6.0의 개발 환경은 더 이상 지원되지 않기는 하지만, 실행에 필요한 라이브러리는 <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_%EB%B9%84%EC%8A%A4%ED%83%80" title="윈도 비스타">윈도 비스타</a>, <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_%EC%84%9C%EB%B2%84_2008" title="윈도 서버 2008">윈도 서버 2008</a>, 그리고 <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_7" title="윈도 7">윈도 7</a>에 계속 지원되고 있다.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></li>
<li>2005년 3월 31일, 마이크로소프트 비주얼 베이직 6.0의 주요 지원이 종료되었다. 확장 지원은 2008년 3월 종료되었다.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> 이에 대하여, 비주얼 베이직 유저 커뮤니티는 우려를 표현하고 제품의 계속적인 지원을 청원하였다.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> 하지만 마이크로소프트는 기존 입장을 바꾸지 않았다. (하지만 다른 흐름도 있다.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> 아이러니컬하게도, 이 시기(2005년)에 발표된 마이크로소프트의 새로운 <a href="/wiki/%EB%B0%94%EC%9D%B4%EB%9F%AC%EC%8A%A4_%EA%B2%80%EC%82%AC_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="바이러스 검사 소프트웨어">바이러스 검사 소프트웨어</a>인 <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%95%A4%ED%8B%B0%EC%8A%A4%ED%8C%8C%EC%9D%B4%EC%9B%A8%EC%96%B4" title="마이크로소프트 앤티스파이웨어">마이크로소프트 앤티스파이웨어</a>는 비주얼 베이직 6.0으로 개발된 프로그램이었다.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> 이 프로그램의 대체품인 <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_%EB%94%94%ED%8E%9C%EB%8D%94" title="윈도 디펜더">윈도 디펜더</a>는 C++ 코드로 재작성되어 개발되었다.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></li>
</ul>
<h2><span class="mw-headline" id=".EB.8B.A4.EB.A5.B8_.EC.96.B8.EC.96.B4.EB.93.A4.EA.B3.BC.EC.9D.98_.EA.B4.80.EA.B3.84">다른 언어들과의 관계</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;section=5" title="부분 편집: 다른 언어들과의 관계">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>마이크로소프트의 여러 스크립트 언어들은 비주얼 베이직으로부터 파생한 것이다. 비주얼 베이직 자체는 <a href="/wiki/%EB%B2%A0%EC%9D%B4%EC%A7%81" title="베이직">베이직</a>과 강력한 연관성을 가지며, 결과적으로 <a href="/wiki/%EB%8B%B7%EB%84%B7_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC" title="닷넷 프레임워크">닷넷 프레임워크</a> 플랫폼 버전으로 대체되었다.</p>
<p>다음 언어들은 비주얼 베이직에서 파생한 것이다:</p>
<ul>
<li><a href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%ED%8F%AC_%EC%95%A0%ED%94%8C%EB%A6%AC%EC%BC%80%EC%9D%B4%EC%85%98" title="비주얼 베이직 포 애플리케이션">비주얼 베이직 포 애플리케이션</a>(VBA)는 많은 마이크로소프트의 응용 프로그램들(특히 마이크로소프트 오피스) 및 <a class="new" href="/w/index.php?title=%EC%86%94%EB%A6%AC%EB%93%9C%EC%9B%8D%EC%8A%A4&amp;action=edit&amp;redlink=1" title="솔리드웍스 (없는 문서)">솔리드웍스</a>, <a href="/wiki/%EC%98%A4%ED%86%A0%EC%BA%90%EB%93%9C" title="오토캐드">오토캐드</a>,<a class="new" href="/w/index.php?title=%EC%9B%8C%EB%93%9C%ED%8D%BC%ED%8E%99%ED%8A%B8_%EC%98%A4%ED%94%BC%EC%8A%A4&amp;action=edit&amp;redlink=1" title="워드퍼펙트 오피스 (없는 문서)">워드퍼펙트 오피스</a>, <a class="new" href="/w/index.php?title=%EC%95%84%ED%81%ACGIS&amp;action=edit&amp;redlink=1" title="아크GIS (없는 문서)">아크GIS</a>, 그리고 <a class="new" href="/w/index.php?title=ACCPAC&amp;action=edit&amp;redlink=1" title="ACCPAC (없는 문서)">ACCPAC</a>와 같은 서드 파티 제품들에 포함되어 있다. 서로 다른 응용 프로그램들간에 약간의 불일치는 존재하지만, 이것들은 전체적으로 비주얼 베이직 6.0과 같은 언어이며, 같은 런타임 라이브러리를 사용한다.</li>
<li><a class="mw-redirect" href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8" title="비주얼 베이직 스크립트">비주얼 베이직 스크립트</a>는 <a href="/wiki/%EC%95%A1%ED%8B%B0%EB%B8%8C_%EC%84%9C%EB%B2%84_%ED%8E%98%EC%9D%B4%EC%A7%80" title="액티브 서버 페이지">액티브 서버 페이지</a>의 기본 언어이다. 비주얼 베이직 스크립트는 또한 <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">마이크로소프트 윈도</a> 스크립팅과 클라이언트측 웹 페이지 스크립팅에도 사용된다. 비주얼 베이직과 문법적으로 닮아 있지만, 이 언어는 비주얼 베이직 런타임 라이브러리와 대립되는 vbscript.dll을 통해 실행된다는 점에서 두 언어는 구분 된다. ASP와 비주얼 베이직 스크립트는 컴파일된 웹 페이지용 <a href="/wiki/%EB%8B%B7%EB%84%B7_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC" title="닷넷 프레임워크">닷넷 프레임워크</a>를 사용하는 <a class="new" href="/w/index.php?title=%EC%95%A1%ED%8B%B0%EB%B8%8C_%EC%84%9C%EB%B2%84_%ED%8E%98%EC%9D%B4%EC%A7%80_%EB%8B%B7%EB%84%B7&amp;action=edit&amp;redlink=1" title="액티브 서버 페이지 닷넷 (없는 문서)">액티브 서버 페이지 닷넷</a>과 혼동하지 말것.</li>
<li><a href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%EB%8B%B7%EB%84%B7" title="비주얼 베이직 닷넷">비주얼 베이직 닷넷</a>은 마이크로소프트의 비주얼 베이직 6.0의 후속작으로 설계된 제품이며, 마이크로소프트의 <a href="/wiki/%EB%8B%B7%EB%84%B7_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC" title="닷넷 프레임워크">닷넷 프레임워크</a> 플랫폼의 일부분이다. 비주얼 베이직 닷넷은 닷넷 프레임워크를 통해 실행되며, 실행 파일을 컴파일한다. 비주얼 베이직 닷넷은 비주얼 베이직 6.0과 하위 호환성이 없다. 마이크로소프트에서 제공하는 Visual Basic 6.0 to Visual Basic .NET Upgrade Assessment Tool 를 이용하여 변환시 생길 수 있는 문제점들을 확인할 수 있다.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup></li>
<li><a class="new" href="/w/index.php?title=%EC%8A%A4%ED%83%80_%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;redlink=1" title="스타 베이직 (없는 문서)">스타 베이직</a>은 <a class="mw-redirect" href="/wiki/%EC%84%A0_%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%8B%9C%EC%8A%A4%ED%85%9C%EC%A6%88" title="선 마이크로시스템즈">선 마이크로시스템즈</a>가 개발한 <a class="new" href="/w/index.php?title=%EC%8A%A4%ED%83%80_%EC%98%A4%ED%94%BC%EC%8A%A4&amp;action=edit&amp;redlink=1" title="스타 오피스 (없는 문서)">스타 오피스</a> 제품군에 포함되어 있는 인터프리터 언어이며, 비주얼 베이직과 호환성이 있다.</li>
<li><a href="/wiki/%EA%B0%90%EB%B0%94%EC%8A%A4" title="감바스">감바스</a>는 비주얼 베이직의 영향을 받은 <a href="/wiki/GNU" title="GNU">GNU</a>/<a href="/wiki/%EB%A6%AC%EB%88%85%EC%8A%A4" title="리눅스">리눅스</a>용 <a href="/wiki/%EC%9E%90%EC%9C%A0_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="자유 소프트웨어">자유 소프트웨어</a> 프로그래밍 언어이다. 감바스는 비주얼 베이직의 클론은 아니지만, 비주얼 베이직 프로그램을 감바스로 변환하는 기능이 있다.</li>
<li><a class="new" href="/w/index.php?title=K%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;redlink=1" title="K베이직 (없는 문서)">K베이직</a>은 비주얼 베이직의 영향을 받은 리눅스, 매킨토시, 윈도용 <a href="/wiki/%EC%9E%90%EC%9C%A0_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="자유 소프트웨어">자유 소프트웨어</a> 프로그래밍 언어이다. K베이직은는 비주얼 베이직의 클론은 아니지만, 비주얼 베이직 프로그램을 K베이직으로 변환하는 기능이 있다.</li>
</ul>
<h2><span class="mw-headline" id=".EC.A7.80.EC.9B.90_.EC.A4.91.EB.8B.A8">지원 중단</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;section=6" title="부분 편집: 지원 중단">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">MS</a>는 무료 지원과 중요 업데이트 제공을 끝으로 <a href="/wiki/2005%EB%85%84" title="2005년">2005년</a> 3월 말 VB6에 대한 표준 지원을 중단할 것이라고 밝혔다. 하지만 두 서비스 모두 앞으로 3년 동안은 유료 서비스로 남아있게 된다. <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">MS</a>는 <a href="/wiki/2000%EB%85%84" title="2000년">2000년</a> <a href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%EB%8B%B7%EB%84%B7" title="비주얼 베이직 닷넷">VB.NET</a>을 내놓았으며 그 후 VB6 이하 버전을 사용하는 개발자들은 끊임없이 감소해왔다. 비주얼 베이직을 등지고 떠난 많은 개발자들은 <a href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%EB%8B%B7%EB%84%B7" title="비주얼 베이직 닷넷">VB.NET</a>으로 옮겨가지 않고, <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">MS</a>의 언어가 아닌 <a href="/wiki/%EC%9E%90%EB%B0%94_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="자바 (프로그래밍 언어)">자바</a>로 옮겨갔다는 설문 조사 결과가 있다. <a href="/wiki/2005%EB%85%84" title="2005년">2005년</a> 현재, <a class="mw-redirect" href="/wiki/%EB%B6%81%EB%AF%B8" title="북미">북미</a> 전체 개발자 중 <a href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%EB%8B%B7%EB%84%B7" title="비주얼 베이직 닷넷">비주얼 베이직 닷넷</a> 개발자가 34%인데 비해 비주얼 베이직 개발자들은 45%다. 기존 VB6 응용 프로그램을 <a href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81_%EB%8B%B7%EB%84%B7" title="비주얼 베이직 닷넷">VB 닷넷</a>으로 옮기는 게 어려운 점이 문제점으로 지적되고 있다.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> 다만 <a href="/wiki/2008%EB%85%84" title="2008년">2008년</a> VB6 SP6에 대한 일부 취약한 보안 문제와 관련된 패치가 있었다. 그리고 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">마이크로소프트</a>가 추후 공개할 <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_8" title="윈도 8">윈도 8</a>부터 비주얼 베이직 및 기반 프로그램들을 더 이상 지원하지 않는다.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></p>
<h2><span class="mw-headline" id=".EC.98.88.EC.A0.9C_.EC.BD.94.EB.93.9C">예제 코드</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;section=7" title="부분 편집: 예제 코드">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>다음은 이 언어의 예제 코드이다. 창이 호출되면 "Hello, World!" 메시지 상자를 표시한다:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">Private</span> <span class="k">Sub</span> <span class="nf">Form_Load</span><span class="p">()</span>
    <span class="c">' Hello, World라는 내용의 메시지 상자를 띄웁니다.</span>
    <span class="n">MsgBox</span> <span class="p">(</span><span class="s">"Hello, World!"</span><span class="p">)</span>
<span class="k">End</span> <span class="k">Sub</span>
</pre></div>
<p>아래 코드는 Count 변수가 1초에 1씩 올라가도록 한다.(작동을 위해서는 타이머 컨트롤이 필요하다)</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">Option</span> <span class="n">Explicit</span>
<span class="k">Dim</span> <span class="n">Count</span> <span class="ow">As</span> <span class="kt">Integer</span>
<span class="k">Private</span> <span class="k">Sub</span> <span class="nf">Form_Load</span><span class="p">()</span>
    <span class="n">Count</span> <span class="o">=</span> <span class="mi">0</span>
    <span class="n">Timer1</span><span class="p">.</span><span class="n">Interval</span> <span class="o">=</span> <span class="mi">1000</span> <span class="c">' 밀리초 단위</span>
<span class="k">End</span> <span class="k">Sub</span>
<span class="k">Private</span> <span class="k">Sub</span> <span class="nf">Timer1_Timer</span><span class="p">()</span>
    <span class="n">Count</span> <span class="o">=</span> <span class="n">Count</span> <span class="o">+</span> <span class="mi">1</span>
    <span class="n">lblCount</span><span class="p">.</span><span class="n">Caption</span> <span class="o">=</span> <span class="n">Count</span>
<span class="k">End</span> <span class="k">Sub</span>
</pre></div>
