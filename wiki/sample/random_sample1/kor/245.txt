<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><a href="/wiki/%EC%BB%B4%ED%93%A8%ED%8C%85" title="컴퓨팅">컴퓨팅</a>에서 <b>DOC</b> 또는 <b>doc</b> (<a href="/wiki/%EB%AC%B8%EC%84%9C" title="문서">document</a>의 준말)는 <a class="mw-redirect" href="/wiki/%EC%9B%8C%EB%93%9C_%EC%B2%98%EB%A6%AC" title="워드 처리">워드 처리</a> 문서를 위한 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%99%95%EC%9E%A5%EC%9E%90" title="파일 확장자">파일 확장자</a>이다. 대개 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9B%8C%EB%93%9C" title="마이크로소프트 워드">마이크로소프트 워드</a>에 쓰인다. 역사적으로 이 확장자는 다양한 <a href="/wiki/%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C" title="운영 체제">운영 체제</a>에서, 특히 프로그램이나 컴퓨터 하드웨어의 <a href="/wiki/%ED%94%8C%EB%A0%88%EC%9D%B8_%ED%85%8D%EC%8A%A4%ED%8A%B8" title="플레인 텍스트">플레인 텍스트</a> <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%98%95%EC%8B%9D" title="파일 형식">형식</a>으로 사용되었다. 1980년대에 <a href="/wiki/%EC%9B%8C%EB%93%9C%ED%8D%BC%ED%8E%99%ED%8A%B8" title="워드퍼펙트">워드퍼펙트</a>는 DOC를 저들의 <a class="new" href="/w/index.php?title=%EC%82%AC%EC%9C%A0_%ED%8F%AC%EB%A7%B7&amp;action=edit&amp;redlink=1" title="사유 포맷 (없는 문서)">사유 포맷</a> 확장자로 사용하였다. 나중에 1990년대 말에 이르러 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">마이크로소프트</a>는 DOC 확장자를 자신의 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9B%8C%EB%93%9C" title="마이크로소프트 워드">마이크로소프트 워드</a> 워드 처리 포맷으로 사용하였다. 전자의 경우 <a href="/wiki/%EA%B0%9C%EC%9D%B8%EC%9A%A9_%EC%BB%B4%ED%93%A8%ED%84%B0" title="개인용 컴퓨터">PC</a> 세상에서 대부분 사라진 상황이다.</p>

<h2><span class="mw-headline" id=".EB.A7.88.EC.9D.B4.ED.81.AC.EB.A1.9C.EC.86.8C.ED.94.84.ED.8A.B8.EC.9D.98_DOC_.EC.9D.B4.EC.A7.84_.ED.8C.8C.EC.9D.BC_.ED.8F.AC.EB.A7.B7">마이크로소프트의 DOC 이진 파일 포맷</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DOC_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=1" title="부분 편집: 마이크로소프트의 DOC 이진 파일 포맷">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>이진 DOC 파일들은 <a href="/wiki/%EC%84%9C%EC%8B%9D_%EC%9E%88%EB%8A%94_%ED%85%8D%EC%8A%A4%ED%8A%B8_%ED%8F%AC%EB%A7%B7" title="서식 있는 텍스트 포맷">서식 있는 텍스트 포맷</a>(RTF), <a href="/wiki/HTML" title="HTML">하이퍼텍스트 마크업 언어</a>(HTML)과 같은 다른 일부 <a class="new" href="/w/index.php?title=%EB%AC%B8%EC%84%9C_%ED%8C%8C%EC%9D%BC_%ED%98%95%EC%8B%9D&amp;action=edit&amp;redlink=1" title="문서 파일 형식 (없는 문서)">문서 파일 형식</a>에 비해 (<a href="/wiki/%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8_%EC%96%B8%EC%96%B4" title="스크립트 언어">스크립트</a> 및 실행 취소된 정보를 포함하여) 더 많은 <a href="/wiki/%EC%A1%B0%ED%8C%90" title="조판">텍스트 포맷</a> 정보를 포함하기도 하지만 일반적으로 <a class="new" href="/w/index.php?title=%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%98%B8%ED%99%98%EC%84%B1&amp;action=edit&amp;redlink=1" title="컴퓨터 호환성 (없는 문서)">호환성</a>이 광범위하지는 않은 편이다.</p>
<p>DOC 포맷은 마이크로소프트 오피스 워드 포맷 사이에서도 다양하다. 97 이하의 워드 버전의 포맷은 워드 97에서 2003 사이의 마이크로소프트 워드 포맷과는 다르다.</p>
<h3><span class="mw-headline" id=".EC.9D.91.EC.9A.A9_.ED.94.84.EB.A1.9C.EA.B7.B8.EB.9E.A8_.EC.A7.80.EC.9B.90">응용 프로그램 지원</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DOC_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=2" title="부분 편집: 응용 프로그램 지원">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink boilerplate seealso"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> <a class="new" href="/w/index.php?title=%EC%9B%8C%EB%93%9C_%ED%94%84%EB%A1%9C%EC%84%B8%EC%84%9C%EC%9D%98_%EB%B9%84%EA%B5%90&amp;action=edit&amp;redlink=1" title="워드 프로세서의 비교 (없는 문서)">워드 프로세서의 비교</a> 문서를 참고하십시오.</div>
<p>DOC 포맷은 마이크로소프트 오피스 워드에 순수 적용되지만, 일부 기능 제한을 제외하고 .doc 파일을 쓰고 읽을 수 있는 <a href="/wiki/%EC%9B%8C%EB%93%9C_%ED%94%84%EB%A1%9C%EC%84%B8%EC%84%9C" title="워드 프로세서">워드 프로세서</a>로는 <a href="/wiki/%EC%98%A4%ED%94%88%EC%98%A4%ED%94%BC%EC%8A%A4_%EB%9D%BC%EC%9D%B4%ED%84%B0" title="오픈오피스 라이터">오픈오피스 라이터</a>, <a class="new" href="/w/index.php?title=IBM_%EB%A1%9C%ED%84%B0%EC%8A%A4_%EC%8B%AC%ED%8F%AC%EB%8B%88&amp;action=edit&amp;redlink=1" title="IBM 로터스 심포니 (없는 문서)">IBM 로터스 심포니</a>, <a href="/wiki/%EA%B5%AC%EA%B8%80_%EB%AC%B8%EC%84%9C%EB%8F%84%EA%B5%AC" title="구글 문서도구">구글 문서도구</a>, <a href="/wiki/%ED%8E%98%EC%9D%B4%EC%A7%80_(%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4)" title="페이지 (소프트웨어)">애플 페이지</a>, <a href="/wiki/%EC%95%A0%EB%B9%84%EC%9B%8C%EB%93%9C" title="애비워드">애비워드</a> 등이 있다.</p>
<h3><span class="mw-headline" id=".EA.B7.9C.EA.B2.A9">규격</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DOC_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=3" title="부분 편집: 규격">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>.doc 파일 형식이 수년 동안은 공개되지 않은 규격이었기 때문에, 일치되지 않은 포맷 관리가 지속되며 여러 워드 처리 프로그램에서 같은 파일을 다룰 때 포맷 정보가 일부 손실될 수도 있다. MS 오피스 97 이진 파일 포맷의 일부 규격은 1997년에 제한된 라이선스로 출판되었으나 이러한 규격들은 1999년에 온라인 다운로드에서 제거되었다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> 그 이후 버전의 MS 오피스의 이진 파일 규격은 공식 공개되지 않아 이용할 수 없다. DOC 포맷 규격은 요청을 할 경우 마이크로소프트로부터 2006년 이후로<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> 이용을 할 수 있었으나<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> 이는 2008년 2월까지 제한된 <a class="new" href="/w/index.php?title=RAND-Z&amp;action=edit&amp;redlink=1" title="RAND-Z (없는 문서)">RAND-Z</a> 조항에 의해서만 가능했다. <a href="/wiki/%EC%8D%AC_%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%8B%9C%EC%8A%A4%ED%85%9C%EC%A6%88" title="썬 마이크로시스템즈">썬 마이크로시스템즈</a>와 <a href="/wiki/%EC%98%A4%ED%94%88%EC%98%A4%ED%94%BC%EC%8A%A4" title="오픈오피스">오픈오피스</a>가 <a href="/wiki/%EB%A6%AC%EB%B2%84%EC%8A%A4_%EC%97%94%EC%A7%80%EB%8B%88%EC%96%B4%EB%A7%81" title="리버스 엔지니어링">리버스 엔지니어링</a>을 통해 문서를 완성하자,<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> 마이크로소프트는 <a class="new" href="/w/index.php?title=%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%98%A4%ED%94%88_%EA%B7%9C%EA%B2%A9_%EC%95%BD%EC%86%8D&amp;action=edit&amp;redlink=1" title="마이크로소프트 오픈 규격 약속 (없는 문서)">마이크로소프트 오픈 규격 약속</a>을 통해 .DOC 포맷 규격을 공개하였다.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup><sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> 그러나 이 규격은 DOC 포맷에 쓰이는 모든 기능을 기술한 것은 아니므로 리버스 엔지니어링은 여전히 필요하다.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></p>
<h2><span class="mw-headline" id=".EB.8B.A4.EB.A5.B8_.ED.8C.8C.EC.9D.BC_.ED.98.95.EC.8B.9D">다른 파일 형식</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DOC_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=4" title="부분 편집: 다른 파일 형식">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>일부 역사적인 문서들에 따르면 플레인 텍스트 포맷을 위해 .doc 파일 확장자를 사용할 수 있다. .doc 파일 확장자는 사유 포맷으로 워드퍼펙트의 역사적인 버전에서도 사용되었다.</p>
<p>일부 응용 소프트웨어는 다른 파일 포맷의 다른 용어(이를테면 소프트웨어 제조업체의 이름)들의 결합으로 "DOC"라는 이름을 사용한다. 이를테면 <a href="/wiki/%ED%8C%9C_OS" title="팜 OS">팜 OS</a>에서 DOC의 이름은 PalmDoc이며 이는 <a href="/wiki/%EC%A0%84%EC%9E%90%EC%B1%85" title="전자책">전자책</a>과 같은 텍스트 파일 인코딩에 쓰이는 포맷(일반적으로 .pdb 확장자)과는 전혀 관계가 없다.</p>
