<h1 class="firstHeading" id="firstHeading" lang="ko">기계어</h1>

<a href="/wiki/WDC_65816" title="WDC 65816">WDC 65816</a> <a class="mw-redirect" href="/wiki/%EB%8B%A8%EC%9D%BC_%EA%B8%B0%ED%8C%90_%EC%BB%B4%ED%93%A8%ED%84%B0" title="단일 기판 컴퓨터">단일 기판 컴퓨터</a>의 기계어 모니터가 코드의 <a href="/wiki/%EC%97%AD%EC%96%B4%EC%85%88%EB%B8%94%EB%9F%AC" title="역어셈블러">역어셈블리</a>된 모습 및 프로세서 레지스터, 메모리 덤프를 보여주고 있다.</div>

<p><b>기계어</b>(機械語)는 <a href="/wiki/%EC%A4%91%EC%95%99_%EC%B2%98%EB%A6%AC_%EC%9E%A5%EC%B9%98" title="중앙 처리 장치">CPU</a>가 직접 해독하고 실행할 수 있는 <a href="/wiki/%EB%B9%84%ED%8A%B8" title="비트">비트</a> 단위로 쓰인 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%EC%96%B8%EC%96%B4" title="컴퓨터 언어">컴퓨터 언어</a>를 통틀어 일컫는다. 기계어는 프로그램을 나타내는 가장 낮은 단계의 개념이다.</p>
<p>기계어는 <a href="/wiki/%EC%96%B4%EC%85%88%EB%B8%94%EB%A6%AC%EC%96%B4" title="어셈블리어">어셈블리어</a>와 1 : 1로 쓰일 수 있다. 다시 말해 컴퓨터 프로그래밍에서 기계어는 대부분 어셈블리어를 거쳐 짜여지게 된다. 어셈블리어가 아직 만들어지기 전에는 기계어를 직접 입력하여 프로그램을 작성하기도 했으며, 오늘날에도 흔치는 않지만 그렇게 하는 사람이 있다. 기계어는 CPU의 종류에 따라서 서로 다른 코드를 갖게 된다.</p>
<h2><span class="mw-headline" id=".EC.98.88">예</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EA%B8%B0%EA%B3%84%EC%96%B4&amp;action=edit&amp;section=1" title="부분 편집: 예">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<pre>
   6      5     5     5     5      6 bits
[  op  |  rs |  rt |  rd |shamt| funct]  R-type
[  op  |  rs |  rt | address/immediate]  I-type
[  op  |        target address        ]  J-type
</pre>
<p><i>rs</i>, <i>rt</i>, <i>rd</i>는 register 연산자를 지시한다; <i>shamt</i>는 변화량을 제공한다; <i>address</i>나 <i>immediate</i> 필드는 직접 연산을 포함한다.</p>
<p>이를테면, register 1과 2를 추가하여 register 6이라는 결과를 낳게 만들면 다음과 같이 인코딩된다:</p>
<pre>
[  op  |  rs |  rt |  rd |shamt| funct]
    0     1     2     6     0     32     decimal
 000000 00001 00010 00110 00000 100000   binary
</pre>
<p>값을 register 3에 나열된 위치 뒤의 메모리 셀 68 셀에서 가져온 register 8로 불러들인다:</p>
<pre>
[  op  |  rs |  rt | address/immediate]
   35     3     8           68           decimal
 100011 00011 01000 00000 00001 000100   binary
</pre>
<p>address 1024로 Jump:</p>
<pre>
[  op  |        target address        ]
    2                 1024               decimal
 000010 00000 00000 00000 10000 000000   binary
</pre>
