<h1 class="firstHeading" id="firstHeading" lang="ko">INI 파일</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">위키백과, 우리 모두의 백과사전.</div>
<div id="contentSub"><span class="mw-redirectedfrom">(<a class="mw-redirect" href="/w/index.php?title=.ini&amp;redirect=no" title=".ini">.ini</a>에서 넘어옴)</span></div>

<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><b>INI(Initialization) 파일</b> <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%98%95%EC%8B%9D" title="파일 형식">포맷</a>은 <a class="new" href="/w/index.php?title=%EC%84%A4%EC%A0%95_%ED%8C%8C%EC%9D%BC&amp;action=edit&amp;redlink=1" title="설정 파일 (없는 문서)">설정 파일</a>에 대한 <a class="mw-redirect" href="/wiki/De_facto" title="De facto">de facto</a> 표준이다. INI 파일은 단순 구조의 <a href="/wiki/%ED%85%8D%EC%8A%A4%ED%8A%B8_%ED%8C%8C%EC%9D%BC" title="텍스트 파일">텍스트 파일</a>로 이루어져 있다. 보통 <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">마이크로소프트 윈도</a>와 연결되어 있지만 다른 <a href="/wiki/%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C" title="운영 체제">운영 체제</a>에서도 사용할 수 있다. "INI 파일"이라는 이름은 ".INI"라는 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%99%95%EC%9E%A5%EC%9E%90" title="파일 확장자">파일 확장자</a>가 따라오지만, ".CFG", ".conf"<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>, ".TXT"<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> 등의 다른 확장자를 사용하기도 한다.</p>

<h2><span class="mw-headline" id=".ED.98.95.EC.8B.9D">형식</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_%ED%8C%8C%EC%9D%BC&amp;action=edit&amp;section=1" title="부분 편집: 형식">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<dl>
<dt>매개 변수</dt>
</dl>
<p>INI 파일에 포함된 기본 요소는 매개 변수이다. 각 변수는 이름과 값을 가지고 있으며 <a href="/wiki/%EB%93%B1%ED%98%B8" title="등호">등호</a>로 이를 구분한다. "이름"은 등호 왼쪽에 적는다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="na">이름</span> <span class="o">=</span> <span class="s">값</span>
</pre></div>
<dl>
<dt>섹션</dt>
</dl>
<p>매개 변수는 임의의 이름으로 지정된 여러 개의 섹션으로 구분할 수 있다. 이 섹션 이름은 <a href="/wiki/%EA%B4%84%ED%98%B8" title="괄호">괄호</a> ([, ])로 구분한다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">[섹션]</span>
</pre></div>
<dl>
<dt>주석</dt>
</dl>
<p><a class="mw-redirect" href="/wiki/%EC%84%B8%EB%AF%B8%EC%BD%9C%EB%A1%A0" title="세미콜론">세미콜론</a> (;)은 <a href="/wiki/%EC%A3%BC%EC%84%9D_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D)" title="주석 (프로그래밍)">주석</a>의 시작을 가리킨다. 줄이 끝날 때까지 주석을 계속 적을 수 있다. 세미콜론 사이의 모든 항목과 줄의 끝 부분까지의 내용은 프로그램의 설정값에서 무시한다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">; 주석</span>
</pre></div>
<h2><span class="mw-headline" id=".EC.98.88">예</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=INI_%ED%8C%8C%EC%9D%BC&amp;action=edit&amp;section=2" title="부분 편집: 예">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>가상의 프로그램(실제로 존재하지는 않음)을 위한 INI 파일의 예는 다음과 같다. 두 개의 섹션을 가지고 있으며, 그 가운데 한 섹션은 소프트웨어의 소유자에 대한 것이며, 나머지 하나는 종업원 명부 데이터베이스 연결에 대한 것이다. 또, 파일을 마지막으로 수정한 날짜, <a class="mw-redirect" href="/wiki/%EB%8F%84%EB%A9%94%EC%9D%B8_%EC%9D%B4%EB%A6%84_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="도메인 이름 시스템">도메인 이름</a> 대신 <a href="/wiki/IP_%EC%A3%BC%EC%86%8C" title="IP 주소">IP 주소</a>가 쓰인 까닭에 대해 주석을 적고 있다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">; 홍길동이 2001년 4월 1일에 마지막으로 수정하였음</span>
<span class="k">[owner]</span>
<span class="na">name</span><span class="o">=</span><span class="s">홍길동</span>
<span class="na">organization</span><span class="o">=</span><span class="s">최고의 제품</span>

<span class="k">[database]</span>
<span class="na">server</span><span class="o">=</span><span class="s">192.0.2.62     ; 네트워크 이름 변환이 동작하지 않는 경우 IP 주소를 사용한다</span>
<span class="na">port</span><span class="o">=</span><span class="s">143</span>
<span class="na">file</span><span class="o">=</span><span class="s">"payroll.dat"</span>
</pre></div>
