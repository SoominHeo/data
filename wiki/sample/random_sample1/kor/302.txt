<h1 class="firstHeading" id="firstHeading" lang="ko">배치 파일</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">위키백과, 우리 모두의 백과사전.</div>
<div id="contentSub"><span class="mw-redirectedfrom">(<a class="mw-redirect" href="/w/index.php?title=.BAT&amp;redirect=no" title=".BAT">.BAT</a>에서 넘어옴)</span></div>

<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><div class="dablink"><a href="/wiki/%EC%9C%84%ED%82%A4%EB%B0%B1%EA%B3%BC:%EB%8F%99%EC%9D%8C%EC%9D%B4%EC%9D%98%EC%96%B4_%EB%AC%B8%EC%84%9C" title="위키백과:동음이의어 문서"><img alt="" data-file-height="200" data-file-width="260" height="18" src="//upload.wikimedia.org/wikipedia/commons/thumb/4/4a/Disambig_grey.svg/23px-Disambig_grey.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/4/4a/Disambig_grey.svg/35px-Disambig_grey.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/4/4a/Disambig_grey.svg/46px-Disambig_grey.svg.png 2x" width="23"/></a> 이 문서는 MS-DOS 스크립트에 관한 것입니다. 다른 종류의 배치 프로그래밍에 대해서는 <a href="/wiki/%EC%85%B8_%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8" title="셸 스크립트">셸 스크립트</a> 문서를 참조하십시오.</div>
<p><a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>, <a href="/wiki/OS/2" title="OS/2">OS/2</a>, <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84%EC%9A%B0" title="마이크로소프트 윈도우">윈도우</a>에서 쓰이는 <b>배치 파일</b>(batch file)은 <a class="mw-redirect" href="/wiki/%EB%AA%85%EB%A0%B9_%EC%A4%84_%EC%9D%B8%ED%84%B0%ED%94%84%EB%A6%AC%ED%84%B0" title="명령 줄 인터프리터">명령 인터프리터</a>에 의해 실행되게끔 고안된 명령어들이 나열되어 있는 <a href="/wiki/%ED%85%8D%EC%8A%A4%ED%8A%B8_%ED%8C%8C%EC%9D%BC" title="텍스트 파일">텍스트 파일</a>이다. 배치 파일이 실행될 때, <a href="/wiki/COMMAND.COM" title="COMMAND.COM">COMMAND.COM</a> 또는 <a href="/wiki/Cmd.exe" title="Cmd.exe">cmd.exe</a>와 같은 <a href="/wiki/%EC%85%B8" title="셸">셸</a> 프로그램이 파일을 읽어 명령어를 줄 단위로 실행한다. 배치 파일은 보통 <a href="/wiki/%EC%8B%A4%ED%96%89_%ED%8C%8C%EC%9D%BC" title="실행 파일">실행 파일</a>을 자동으로, 연속적으로 실행할 때 유용하며 <a href="/wiki/%EC%8B%9C%EC%8A%A4%ED%85%9C_%EA%B4%80%EB%A6%AC%EC%9E%90" title="시스템 관리자">시스템 관리자</a>가 따분한 일들을 자동화하기 위해 자주 사용한다.</p>
<p><a href="/wiki/%EC%9C%A0%EB%8B%89%EC%8A%A4_%EA%B3%84%EC%97%B4" title="유닉스 계열">유닉스 계열</a> 운영체제에서 배치 파일은 <a href="/wiki/%EC%85%B8_%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8" title="셸 스크립트">셸 스크립트</a>와 닮아 있지만, 구문과 명령어가 제한이 되어 있기 때문에 일반 목적의 프로그래밍에 적합하지 않다. 이러한 제한을 해결하려면 DOS 시절의 경우 <a href="/wiki/%EB%85%B8%ED%84%B4_%EC%9C%A0%ED%8B%B8%EB%A6%AC%ED%8B%B0" title="노턴 유틸리티">노턴 유틸리티</a>나 이후에 나온 셸 <a href="/wiki/4DOS" title="4DOS">4DOS</a>와 같은 넓은 용도의 "강화" 명령어를 사용하면 된다.</p>

<h2><span class="mw-headline" id=".ED.8C.8C.EC.9D.BC_.ED.99.95.EC.9E.A5.EC.9E.90">파일 확장자</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B0%B0%EC%B9%98_%ED%8C%8C%EC%9D%BC&amp;action=edit&amp;section=1" title="부분 편집: 파일 확장자">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>도스에서는 배치 파일의 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%99%95%EC%9E%A5%EC%9E%90" title="파일 확장자">확장자</a>로 .BAT를, <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_NT" title="윈도우 NT">윈도우 NT</a>와 <a href="/wiki/OS/2" title="OS/2">OS/2</a>에서는 .CMD를, <a href="/wiki/4DOS" title="4DOS">4DOS</a>에서는 BTM을 도입하였다.</p>
<ul>
<li><b>.bat</b>: <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">마이크로소프트</a>가 배치 파일에 사용한 최초의 파일 확장자. 이 확장자는 MS-DOS와 모든 버전의 윈도우에서 COMMAND.COM 또는 cmd.exe를 통해 실행된다.</li>
<li><b>.cmd</b>: <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_NT" title="윈도우 NT">윈도우 NT</a> 계열에서 배치 파일에 사용되며 해석을 위해 cmd.exe로 내보낸다. COMMAND.COM은 이 파일 확장자를 식별하지 못한다.</li>
<li><b>.btm</b>: <a href="/wiki/4DOS" title="4DOS">4DOS</a>와 <a class="new" href="/w/index.php?title=4NT&amp;action=edit&amp;redlink=1" title="4NT (없는 문서)">4NT</a>에 쓰이는 확장자이다 4DOS와 NT에서 실행되는 스크립트들은 특히 더 긴 이름을 사용할 때 속도가 더 빠른데, 실행을 위해 스크립트를 줄 단위로 실행하지 않고 스크립트를 완전히 불러들인 상태에서 진행하기 때문이다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></li>
</ul>
<h2><span class="mw-headline" id=".EC.98.88.EC.A0.9C">예제</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B0%B0%EC%B9%98_%ED%8C%8C%EC%9D%BC&amp;action=edit&amp;section=2" title="부분 편집: 예제">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>단순한 배치 파일의 예제는 다음과 같다: (rem은 주석 처리를 말함)</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
 <span class="c1">rem 알림:echo off는 [[표준 출력]]에 각 명령을 출력하지 않게 하는 명령어이다.</span>
 <span class="p">@</span><span class="k">echo</span> off
 <span class="c1">rem 알림:echo.는 빈 줄을 하나 출력한다.</span>
 <span class="k">echo</span>.
 <span class="k">echo</span> [[Hello World]], AProgram.exe를 실행하려면 아무 키나 누르십시오!
 <span class="k">pause</span> <span class="p">&gt;</span> nul
 <span class="c1">rem 알림: 배치 파일에 대한 첫 [[매개변수]]는 "%1"로 지정하면 된다.</span>
 AProgram.exe <span class="nv">%1</span>
 <span class="k">if</span> <span class="k">errorlevel</span> <span class="mi">1</span> <span class="k">goto</span> <span class="nl">error</span>
 <span class="k">echo</span>.
 <span class="k">echo</span> AProgram이 끝났습니다.
 <span class="k">goto</span> <span class="nl">end</span>
 <span class="p">:</span><span class="nl">error</span>
 <span class="k">echo</span>.
 <span class="k">echo</span> AProgram에 무언가가 잘못된 것 같습니다.
 <span class="p">:</span><span class="nl">end</span>
</pre></div>
