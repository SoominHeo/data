<h1 class="firstHeading" id="firstHeading" lang="ko">.bss</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">위키백과, 우리 모두의 백과사전.</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					이동:					<a href="#mw-head">둘러보기</a>, 					<a href="#p-search">검색</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><a href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="컴퓨터 프로그래밍">컴퓨터 프로그래밍</a>에서 <b>.bss</b> 또는 <b>bss</b>는 수많은 <a href="/wiki/%EC%BB%B4%ED%8C%8C%EC%9D%BC%EB%9F%AC" title="컴파일러">컴파일러</a>와 <a class="mw-disambig" href="/wiki/%EB%A7%81%EC%BB%A4" title="링커">링커</a>가 처음에 0 값의 비트로 표현되는 <a class="new" href="/w/index.php?title=%EC%A0%95%EC%A0%81_%EB%B3%80%EC%88%98&amp;action=edit&amp;redlink=1" title="정적 변수 (없는 문서)">정적으로 할당된 변수</a>를 포함하는 <a class="new" href="/w/index.php?title=%EB%8D%B0%EC%9D%B4%ED%84%B0_%EC%84%B8%EA%B7%B8%EB%A8%BC%ED%8A%B8&amp;action=edit&amp;redlink=1" title="데이터 세그먼트 (없는 문서)">데이터 세그먼트</a>의 한 부분으로 사용한다. "bss 섹션"(bss section), "bss 세그먼트"(bss segment)라고도 부른다.</p>
<p>일반적으로 데이터가 없는 bss 섹션의 길이만이 <a class="mw-redirect" href="/wiki/%EC%98%A4%EB%B8%8C%EC%A0%9D%ED%8A%B8_%ED%8C%8C%EC%9D%BC" title="오브젝트 파일">오브젝트 파일</a>에 저장된다. <a class="new" href="/w/index.php?title=%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%A8_%EB%A1%9C%EB%8D%94&amp;action=edit&amp;redlink=1" title="프로그램 로더 (없는 문서)">프로그램 로더</a>는 프로그램을 로드할 때 bss 섹션을 위한 메모리를 할당하고 초기화한다. 운영 체제는 zero-fill-on-demand라는 기술을 사용하여 bss 세그먼트를 효율적으로 구현한다. (McKusick &amp; Karels 1986) 임베디드 소프트웨어에서 bss 세그먼트는 <code>main()</code>에 들어가기 전에 C <a class="new" href="/w/index.php?title=%EB%9F%B0%ED%83%80%EC%9E%84_%EC%8B%9C%EC%8A%A4%ED%85%9C&amp;action=edit&amp;redlink=1" title="런타임 시스템 (없는 문서)">런타임 시스템</a>에 의해 0으로 초기화되는 메모리로 매핑된다.</p>
<p>일부 <a class="mw-redirect" href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%EC%95%84%ED%82%A4%ED%85%8D%EC%B2%98" title="컴퓨터 아키텍처">컴퓨터 아키텍처</a>에서 <a class="mw-disambig" href="/wiki/ABI" title="ABI">ABI</a> 또한 조그마한 데이터에 대한 sbss 세그먼트를 지원한다. 일반적으로 이러한 데이터 항목들은 특정한 범위의 주소에만 접근할 수 있는 더 짧은 명령을 이용하여 접근할 수 있다.</p>
<h2><span class="mw-headline" id=".EA.B8.B0.EC.9B.90">기원</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=.bss&amp;action=edit&amp;section=1" title="부분 편집: 기원">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>역사적으로 BSS(Block Started by Symbol)는 <a class="new" href="/w/index.php?title=%EC%9C%A0%EB%82%98%EC%9D%B4%ED%8B%B0%EB%93%9C_%EC%97%90%EC%96%B4%ED%81%AC%EB%9E%98%ED%94%84%ED%8A%B8&amp;action=edit&amp;redlink=1" title="유나이티드 에어크래프트 (없는 문서)">유나이티드 에어크래프트</a>의 로이 넛(Roy Nutt), 램셔(Ramshaw) 등이 <a class="new" href="/w/index.php?title=IBM_704&amp;action=edit&amp;redlink=1" title="IBM 704 (없는 문서)">IBM 704</a>용으로 1950년대 중반에 개발한 어셈블러인 UA-SAP (United Aircraft Symbolic Assembly Program)에서의 <a href="/wiki/%EC%96%B4%EC%85%88%EB%B8%94%EB%A6%AC%EC%96%B4" title="어셈블리어">의사 동작</a>이다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
