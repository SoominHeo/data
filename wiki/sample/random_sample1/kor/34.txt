<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><b>윈도우 드라이버 모델</b> (Windows Driver Model, WDM)은 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%8C%85" title="컴퓨팅">컴퓨팅</a>에서 <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_95" title="윈도우 95">윈도우 95</a>, <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_3.1" title="윈도우 3.1">윈도우 3.1</a>, 윈도우 NT 드라이버 모델과 같은 오래된 버전의 윈도우에서 쓰였던 <a href="/wiki/VxD" title="VxD">VxD</a>를 대체하기 위해 만들어진 것으로, <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_95" title="윈도우 95">윈도우 95</a>, <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_2000" title="윈도우 2000">윈도우 2000</a>에 도입된 <a href="/wiki/%EC%9E%A5%EC%B9%98_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84" title="장치 드라이버">장치 드라이버</a>를 위한 프레임워크이다. <b>Win32 드라이버 모델</b>로도 알려져 있다.</p>

<h2><span class="mw-headline" id=".EA.B0.9C.EC.9A.94">개요</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%9C%88%EB%8F%84%EC%9A%B0_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84_%EB%AA%A8%EB%8D%B8&amp;action=edit&amp;section=1" title="부분 편집: 개요">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>WDM 드라이버는 복잡한 계층으로 되어 있고 <a class="new" href="/w/index.php?title=%EC%9E%85%EC%B6%9C%EB%A0%A5_%EC%9A%94%EC%B2%AD_%ED%8C%A8%ED%82%B7&amp;action=edit&amp;redlink=1" title="입출력 요청 패킷 (없는 문서)">입출력 요청 패킷</a> (IRP)을 통해 서로 통신한다. 마이크로소프트 윈도우 드라이버 모델은 기록되어야 하는 코드의 양을 줄이고 요구 사항을 표준화함으로써 윈도우 98과 윈도우 2000 운영 체제를 위한 통합 드라이버 모델을 정의하였다. WDM 드라이버는 윈도우 98, 윈도우 2000 이전의 운영 체제, 다시 말해 윈도우 95, 윈도우 NT 4.0, 윈도우 3.1 등에서는 동작하지 않는다. WDM을 따르게 되면 <a href="/wiki/X86" title="X86">x86</a> 기반의 컴퓨터에서 윈도우 98, 윈도우 98 SE, <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_Me" title="윈도우 Me">윈도우 Me</a>, 윈도우 2000, <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_XP" title="윈도우 XP">윈도우 XP</a>, <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_%EC%84%9C%EB%B2%84_2003" title="윈도우 서버 2003">윈도우 서버 2003</a>, <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_%EB%B9%84%EC%8A%A4%ED%83%80" title="윈도우 비스타">윈도우 비스타</a> (하위 호환성을 위함)에 걸쳐 <a class="new" href="/w/index.php?title=%EC%9D%B4%EC%A7%84_%EC%BD%94%EB%93%9C&amp;action=edit&amp;redlink=1" title="이진 코드 (없는 문서)">이진 코드 호환성</a>과 소스 호환성을 보장한다. WDM은 <a href="/wiki/%EC%83%81%EC%9C%84_%ED%98%B8%ED%99%98%EC%84%B1" title="상위 호환성">상위 호환성</a>을 전제로 만들어졌으므로 WDM 드라이버는 드라이버가 처음 만들어진 시기보다 더 새로 나온 윈도우의 버전에서도 동작할 수 있지만 새로운 버전의 운영 체제에 도입된 새로운 기능의 이점을 얻을 수 없다는 것을 뜻하기도 한다. WDM은 일반적으로 <a href="/wiki/%ED%95%98%EC%9C%84_%ED%98%B8%ED%99%98%EC%84%B1" title="하위 호환성">하위 호환성</a>은 제공되지 않는다. 다시 말해 WDM 드라이버는 더 오래된 버전의 윈도우에서 동작하는 것을 보장하지는 않는다. 이를테면 <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_XP" title="윈도우 XP">윈도우 XP</a>는 <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_2000" title="윈도우 2000">윈도우 2000</a>용으로 작성된 드라이버를 사용할 수 있으나 윈도우 XP에 도입된 어느 새로운 WDM 기능도 이용할 수는 없다. 그러나 윈도우 XP용으로 쓰여진 드라이버는 윈도우 2000에서 사용할 수 있을 수도 있고 그렇지 않을 수도 있다.</p>
<p>WDM은 윈도우 2000 <a class="new" href="/w/index.php?title=%EB%A7%81_%EB%B3%B4%ED%98%B8&amp;action=edit&amp;redlink=1" title="링 보호 (없는 문서)">커널 모드</a> 드라이버들의 중재 계층에 존재하며 윈도우의 드라이버 기록을 쉽게 하고 기능을 증가시킬 목적으로 도입되었다. WDM이 <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_98" title="윈도우 98">윈도우 98</a> 계열부터 윈도우 2000 계열에 이르기까지 주로 <a class="new" href="/w/index.php?title=%EC%9D%B4%EC%A7%84_%EC%BD%94%EB%93%9C&amp;action=edit&amp;redlink=1" title="이진 코드 (없는 문서)">이진 코드 호환성</a>과 소스 호환성을 보장하기 위해 설계되었지만 이것이 반드시 선호된다는 것을 뜻하는 것은 아니며 특정한 드라이버는 그 둘 중 하나의 운영 체제에 맞추어서 개발될 수 있음을 말해 준다. WDM 드라이버는 다음과 같은 종류로 나뉜다.</p>
<h3><span class="mw-headline" id=".EC.9E.A5.EC.B9.98_.EA.B8.B0.EB.8A.A5_.EB.93.9C.EB.9D.BC.EC.9D.B4.EB.B2.84">장치 기능 드라이버</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%9C%88%EB%8F%84%EC%9A%B0_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84_%EB%AA%A8%EB%8D%B8&amp;action=edit&amp;section=2" title="부분 편집: 장치 기능 드라이버">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><b>기능 드라이버</b>는 장치를 위한 주된 드라이버이다. 기능 드라이버는 일반적으로 장치 제조업체가 만들어내며 (드라이버가 원본 모드로 쓰이는 것이 아니라면) 필수적이다. 기능 드라이버는 하나 이상의 장치를 도와줄 수 있다.</p>
<ul>
<li><a class="new" href="/w/index.php?title=%ED%81%B4%EB%9E%98%EC%8A%A4_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84&amp;action=edit&amp;redlink=1" title="클래스 드라이버 (없는 문서)">클래스 드라이버</a>: 기능 드라이버의 일종으로, <a class="new" href="/w/index.php?title=%EB%AF%B8%EB%8B%88%ED%8F%AC%ED%8A%B8&amp;action=edit&amp;redlink=1" title="미니포트 (없는 문서)">미니포트</a>와 기타 <a class="new" href="/w/index.php?title=%ED%81%B4%EB%9E%98%EC%8A%A4_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84&amp;action=edit&amp;redlink=1" title="클래스 드라이버 (없는 문서)">클래스 드라이버</a>를 맨 위층에서 만들어질 수 있는 내장 프레임워크 드라이버로 생각할 수 있다. 클래스 드라이버는 서로 다른 수준의 WDM 아키텍처 사이에서 상호 작용한다. 다른 드라이버 클래스 사이의 공통 기능은 클래스 드라이버 안에다 기록할 수 있으며 다른 클래스와 미니포트 드라이버가 이를 사용한다. 낮은 수준의 클래스 드라이버는 미니포트 드라이버에 노출된 저만의 인터페이스를 가지고 있지만 높은 수준의 클래스 드라이버는 운영 체제에 따른다. 클래스 드라이버는 원하는 대로 동적으로 로드하거나 언로드할 수 있다. 이들은 하드웨어나 <a href="/wiki/%EB%B2%84%EC%8A%A4_(%EC%BB%B4%ED%93%A8%ED%8C%85)" title="버스 (컴퓨팅)">버스</a>에 따르지 않는 클래스 고유 기능으로, 실제로 열거(Enumeration)와 같이 클래스 고유 기능만 다루는 경우도 있다.</li>
<li><a class="new" href="/w/index.php?title=%EB%AF%B8%EB%8B%88%ED%8F%AC%ED%8A%B8_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84&amp;action=edit&amp;redlink=1" title="미니포트 드라이버 (없는 문서)">미니포트 드라이버</a>: <a class="mw-redirect" href="/wiki/%EB%B2%94%EC%9A%A9_%EC%A7%81%EB%A0%AC_%EB%B2%84%EC%8A%A4" title="범용 직렬 버스">USB</a>, <a class="mw-disambig" href="/wiki/%EC%98%A4%EB%94%94%EC%98%A4" title="오디오">오디오</a>, <a href="/wiki/SCSI" title="SCSI">SCSI</a>, <a class="mw-redirect" href="/wiki/%EB%84%A4%ED%8A%B8%EC%9B%8C%ED%81%AC_%EC%96%B4%EB%8C%91%ED%84%B0" title="네트워크 어댑터">네트워크 어댑터</a>를 위한 기능 드라이버이다. 이들은 일반적으로 윈도우 98에서 윈도우 2000에 이르기까지의 소스 호환성과 이진 호환성을 보장하지만 특정한 버스 <a class="new" href="/w/index.php?title=%ED%81%B4%EB%9E%98%EC%8A%A4_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84&amp;action=edit&amp;redlink=1" title="클래스 드라이버 (없는 문서)">클래스 드라이버</a>를 통해 하드웨어로의 접근을 제어한다.</li>
</ul>
<h3><span class="mw-headline" id=".EB.B2.84.EC.8A.A4_.EB.93.9C.EB.9D.BC.EC.9D.B4.EB.B2.84">버스 드라이버</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%9C%88%EB%8F%84%EC%9A%B0_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84_%EB%AA%A8%EB%8D%B8&amp;action=edit&amp;section=3" title="부분 편집: 버스 드라이버">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><b><a href="/wiki/%EB%B2%84%EC%8A%A4_(%EC%BB%B4%ED%93%A8%ED%8C%85)" title="버스 (컴퓨팅)">버스</a> 드라이버</b>는 버스 컨트롤러, 어댑터, 브리지를 도와 준다. 마이크로소프트는 <a href="/wiki/PCI_%EB%B2%84%EC%8A%A4" title="PCI 버스">PCI</a>, <a href="/wiki/%ED%94%8C%EB%9F%AC%EA%B7%B8_%EC%95%A4_%ED%94%8C%EB%A0%88%EC%9D%B4" title="플러그 앤 플레이">PnPISA</a>, <a href="/wiki/SCSI" title="SCSI">SCSI</a>, <a class="mw-redirect" href="/wiki/%EB%B2%94%EC%9A%A9_%EC%A7%81%EB%A0%AC_%EB%B2%84%EC%8A%A4" title="범용 직렬 버스">USB</a>, <a class="mw-redirect" href="/wiki/IEEE1394" title="IEEE1394">IEEE1394</a>와 같은 가장 일반적인 버스를 위한 버스 드라이버를 제공한다. 각 소프트웨어 제조업체는 필요하다면 저만의 버스 드라이버를 만들 수 있다. 버스 드라이버는 컴퓨터에 같은 종류의, 하나 이상의 버스가 일을 경우 하나 이상의 버스를 도와 줄 수 있다.</p>
<h3><span class="mw-headline" id=".ED.95.84.ED.84.B0_.EB.93.9C.EB.9D.BC.EC.9D.B4.EB.B2.84">필터 드라이버</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%9C%88%EB%8F%84%EC%9A%B0_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84_%EB%AA%A8%EB%8D%B8&amp;action=edit&amp;section=4" title="부분 편집: 필터 드라이버">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a class="new" href="/w/index.php?title=%ED%95%84%ED%84%B0_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84&amp;action=edit&amp;redlink=1" title="필터 드라이버 (없는 문서)">필터 드라이버</a>는 선택적인 드라이버로서, 장치의 동작에 값을 더하거나 수정하는 역할을 하며 비장치 드라이버의 역할을 할 수도 있다. 필터 드라이버는 하나 이상의 장치를 도와 줄 수 있다. 상위 수준의 필터 드라이버는 기능 드라이버를 위한 기본 드라이버 위에 놓일 수 있지만 하위 수준의 필터 드라이버는 기능 드라이버 아래와 버스 드라이버 위에 놓일 수 있다.</p>
<ul>
<li><a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_%EC%84%9C%EB%B9%84%EC%8A%A4" title="윈도우 서비스">장치 서비스</a>: 이것은 <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_%EC%84%9C%EB%B9%84%EC%8A%A4" title="윈도우 서비스">윈도우 서비스</a>로 추가된, 커널 수준의 필터 드라이버의 일종이며 응용 프로그램들이 장치와 함께 동작할 수 있게 도와 준다.</li>
</ul>
<h2><span class="mw-headline" id="VxD.2C_WDM.2C_.EA.B7.B8.EB.A6.AC.EA.B3.A0_.EC.9C.88.EB.8F.84.EC.9A.B0_98">VxD, WDM, 그리고 윈도우 98</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%9C%88%EB%8F%84%EC%9A%B0_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84_%EB%AA%A8%EB%8D%B8&amp;action=edit&amp;section=5" title="부분 편집: VxD, WDM, 그리고 윈도우 98">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_98" title="윈도우 98">윈도우 98</a> 기반 운영 체제(<a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_98" title="윈도우 98">윈도우 98</a>, 윈도우 98 SE, <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_Me" title="윈도우 Me">윈도우 Me</a>)는 WDM와 <a href="/wiki/VxD" title="VxD">VxD</a> 드라이버 표준을 둘 다 사용할 수 있다. 이 드라이버들은 둘 다 같은 하드웨어에 통일되고도 각기 다른 기능을 제공할 수 있다. 그러나 일반적으로 더 새로운 WDM 표준은 더 많은 기능을 제공한다. 이를테면 <a href="/wiki/VxD" title="VxD">VxD</a> 드라이버를 사용하는 <a href="/wiki/TV_%EC%88%98%EC%8B%A0_%EC%B9%B4%EB%93%9C" title="TV 수신 카드">TV 수신 카드</a>는 384 x 288 화소의 해상도로 영상을 캡처할 수 있지만 같은 TV 수신 카드에 WDM 드라이버를 사용하게 되면 768 x 576 화소의 해상도로 캡처할 수 있다. 이는 WDM의 일부인 새로운 <a class="new" href="/w/index.php?title=%EB%B8%8C%EB%A1%9C%EB%93%9C%EC%BA%90%EC%8A%A4%ED%8A%B8_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84_%EC%95%84%ED%82%A4%ED%85%8D%EC%B2%98&amp;action=edit&amp;redlink=1" title="브로드캐스트 드라이버 아키텍처 (없는 문서)">브로드캐스트 드라이버 아키텍처</a> (BDA)의 도움을 받은 것에 기인한다.</p>
<h2><span class="mw-headline" id=".EB.B9.84.ED.8C.90">비판</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%9C%88%EB%8F%84%EC%9A%B0_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84_%EB%AA%A8%EB%8D%B8&amp;action=edit&amp;section=6" title="부분 편집: 비판">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>윈도우 드라이버 모델은 드라이버 소프트웨어 개발자로부터 비평을 받아왔으며 다음과 같은 사항이 주를 이룬다<a class="external autonumber" href="http://www.wd-3.com/archive/FrameworkIntro.htm" rel="nofollow">[1]</a>:</p>
<ul>
<li>WDM은 매우 가파른 학습 곡선이 있다.</li>
<li><a class="new" href="/w/index.php?title=%EC%A0%84%EC%9B%90_%EA%B4%80%EB%A6%AC&amp;action=edit&amp;redlink=1" title="전원 관리 (없는 문서)">전원 관리</a> 이벤트와 <a href="/wiki/%ED%94%8C%EB%9F%AC%EA%B7%B8_%EC%95%A4_%ED%94%8C%EB%A0%88%EC%9D%B4" title="플러그 앤 플레이">플러그 앤 플레이</a>의 상호 작용이 쉽지 않다.</li>
<li>입출력 취소가 거의 불가능하다.</li>
<li>지원 코드의 수천 개의 라인이 드라이버마다 요구된다.</li>
<li>순수 <a href="/wiki/%EB%B3%B4%ED%98%B8_%EB%A7%81" title="보호 링">사용자 모드</a> 드라이버 기록을 지원하지 않는다.</li>
</ul>
<p>마이크로소프트사가 제공한 문서의 품질과 샘플에 대한 수많은 문제도 거론되었다.</p>
<p>이러한 문제로 마이크로소프트는 WDM을 대체할 새로운 프레임워크인 <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84_%ED%8C%8C%EC%9A%B4%EB%8D%B0%EC%9D%B4%EC%85%98" title="윈도우 드라이버 파운데이션">윈도우 드라이버 파운데이션</a>(WDF)을 공개하였다. 여기에는 <a class="new" href="/w/index.php?title=%EC%BB%A4%EB%84%90_%EB%AA%A8%EB%93%9C_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84_%ED%8C%8C%EC%9A%B4%EB%8D%B0%EC%9D%B4%EC%85%98&amp;action=edit&amp;redlink=1" title="커널 모드 드라이버 파운데이션 (없는 문서)">커널 모드 드라이버 파운데이션</a> (KMDF)와 <a class="new" href="/w/index.php?title=%EC%82%AC%EC%9A%A9%EC%9E%90_%EB%AA%A8%EB%93%9C_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84_%ED%8C%8C%EC%9A%B4%EB%8D%B0%EC%9D%B4%EC%85%98&amp;action=edit&amp;redlink=1" title="사용자 모드 드라이버 파운데이션 (없는 문서)">사용자 모드 드라이버 파운데이션</a> (UMDF)를 포함하고 있다. <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_%EB%B9%84%EC%8A%A4%ED%83%80" title="윈도우 비스타">윈도우 비스타</a> 이상의 운영 체제에서는 WDM과 더 새로운 <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84_%ED%8C%8C%EC%9A%B4%EB%8D%B0%EC%9D%B4%EC%85%98" title="윈도우 드라이버 파운데이션">윈도우 드라이버 파운데이션</a>을 둘 다 제공하고 있다. KMDF는 윈도우 2000/XP용으로도 제공되지만 UMDF는 윈도우 XP에서만 사용할 수 있다.</p>
