<p><b>JSON</b>(<b>제이슨</b><sup class="reference" id="cite_ref-Pronunciation_1-0"><a href="#cite_note-Pronunciation-1">[1]</a></sup>, <span lang="en" xml:lang="en">JavaScript Object Notation</span>)은 <a class="new" href="/w/index.php?title=%EC%86%8D%EC%84%B1-%EA%B0%92_%EC%8C%8D&amp;action=edit&amp;redlink=1" title="속성-값 쌍 (없는 문서)">속성-값 쌍</a>으로 이루어진 데이터 오브젝트를 전달하기 위해 인간이 읽을 수 있는 텍스트를 사용하는 <a href="/wiki/%EA%B0%9C%EB%B0%A9%ED%98%95_%ED%91%9C%EC%A4%80" title="개방형 표준">개방형 표준</a> 포맷이다. 비동기 브라우저/서버 통신 (<a href="/wiki/Ajax" title="Ajax">AJAX</a>)을 위해, 넓게는 <a href="/wiki/XML" title="XML">XML</a>(<a href="/wiki/Ajax" title="Ajax">AJAX</a>가 사용)을 대체하는 주요 데이터 포맷이다. 특히, <a href="/wiki/%EC%9D%B8%ED%84%B0%EB%84%B7" title="인터넷">인터넷</a>에서 자료를 주고 받을 때 그 자료를 표현하는 방법으로 알려져 있다. 자료의 종류에 큰 제한은 없으며, 특히 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%A8" title="컴퓨터 프로그램">컴퓨터 프로그램</a>의 <a href="/wiki/%EB%B3%80%EC%88%98" title="변수">변수</a>값을 표현하는 데 적합하다.</p>
<p>본래는 <a href="/wiki/%EC%9E%90%EB%B0%94%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8" title="자바스크립트">자바스크립트</a> 언어로부터 파생되어 <a href="/wiki/%EC%9E%90%EB%B0%94%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8" title="자바스크립트">자바스크립트</a>의 구문 형식을 따르지만 언어 독립형 데이터 포맷이다. 즉, <a href="/wiki/%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4" title="프로그래밍 언어">프로그래밍 언어</a>나 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%8C%85_%ED%94%8C%EB%9E%AB%ED%8F%BC" title="컴퓨팅 플랫폼">플랫폼</a>에 독립적이므로, <a href="/wiki/%EA%B5%AC%EB%AC%B8_%EB%B6%84%EC%84%9D" title="구문 분석">구문 분석</a> 및 JSON 데이터 생성을 위한 코드는 <a href="/wiki/C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="C (프로그래밍 언어)">C</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/C_%EC%83%A4%ED%94%84" title="C 샤프">C#</a>, <a href="/wiki/%EC%9E%90%EB%B0%94_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="자바 (프로그래밍 언어)">자바</a>, <a href="/wiki/%EC%9E%90%EB%B0%94%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8" title="자바스크립트">자바스크립트</a>, <a href="/wiki/%ED%8E%84" title="펄">펄</a>, <a href="/wiki/%ED%8C%8C%EC%9D%B4%EC%8D%AC" title="파이썬">파이썬</a> 등 수많은 <a href="/wiki/%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4" title="프로그래밍 언어">프로그래밍 언어</a>에서 쉽게 이용할 수 있다.</p>
<p>JSON 포맷은 본래 <a class="new" href="/w/index.php?title=%EB%8D%94%EA%B8%80%EB%9D%BC%EC%8A%A4_%ED%81%AC%EB%A1%9D%ED%8F%AC%EB%93%9C&amp;action=edit&amp;redlink=1" title="더글라스 크록포드 (없는 문서)">더글라스 크록포드</a>가 규정하였다. RFC 7159와 ECMA-404라는 두 개의 경쟁 표준에 의해 기술되고 있다. ECMA 표준은 문법만 정의할 정도로 최소한으로만 정의되어 있는 반면 RFC는 시맨틱, 보안적 고려 사항을 일부 제공하기도 한다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> JSON의 공식 인터넷 미디어 타입은 <code>application/json</code>이며, JSON의 파일 확장자는 <code>.json</code>이다.</p>

<h2><span class="mw-headline" id=".EC.9E.90.EB.A3.8C.ED.98.95.EA.B3.BC_.EB.AC.B8.EB.B2.95">자료형과 문법</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JSON&amp;action=edit&amp;section=1" title="부분 편집: 자료형과 문법">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id=".EA.B8.B0.EB.B3.B8_.EC.9E.90.EB.A3.8C.ED.98.95">기본 자료형</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JSON&amp;action=edit&amp;section=2" title="부분 편집: 기본 자료형">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>JSON의 기본 자료형은 다음과 같다:</p>
<ul>
<li>수(Number)</li>
<li><a href="/wiki/%EB%AC%B8%EC%9E%90%EC%97%B4" title="문자열">문자열(String)</a>: 0개 이상의 <a href="/wiki/%EC%9C%A0%EB%8B%88%EC%BD%94%EB%93%9C" title="유니코드">유니코드</a> 문자들의 연속. 문자열은 큰 따옴표(")로 구분하며 역슬래시 <a class="new" href="/w/index.php?title=%EC%9D%B4%EC%8A%A4%EC%BC%80%EC%9D%B4%ED%94%84_%EB%AC%B8%EC%9E%90&amp;action=edit&amp;redlink=1" title="이스케이프 문자 (없는 문서)">이스케이프</a> 문법을 지원한다.</li>
<li><a href="/wiki/%EB%B6%88%EB%A6%B0_%EC%9E%90%EB%A3%8C%ED%98%95" title="불린 자료형">참/거짓(Boolean)</a>: <code>true</code> 또는 <code>false</code> 값</li>
<li><a href="/wiki/%EB%B0%B0%EC%97%B4" title="배열">배열(Array)</a>: 0 이상의 임의의 종류의 값으로 이루어진 <a href="/wiki/%EB%A6%AC%EC%8A%A4%ED%8A%B8_(%EC%BB%B4%ED%93%A8%ED%8C%85)" title="리스트 (컴퓨팅)">순서가 있는 리스트</a>. 대괄호로 나타내며 요소는 쉼표로 구분한다.</li>
<li>객체(Object): 순서가 없는 이름/값 쌍의 집합으로, 이름(키)이 문자열이다.</li>
<li><code><a class="mw-redirect mw-disambig" href="/wiki/Null" title="Null">null</a></code>: 빈 값으로, <code>null</code>을 사용한다.</li>
</ul>

<h3><span class="mw-headline" id=".EB.AC.B8.EC.9E.90.EC.97.B4.28String.29">문자열(String)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JSON&amp;action=edit&amp;section=4" title="부분 편집: 문자열(String)">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>항상 큰 따옴표(")로 묶어야 하며, 그 안에는 유니코드 문자들이 나열된다. 유니코드 중 역슬래시(\)와 큰따옴표(")는 바로 사용할 수 없다. 역슬래시는 제어문자를 표현하기 위해 사용되며 다음과 같은 의미를 지닌다.</p>
<dl>
<dd>\b 백스페이스</dd>
<dd>\f 폼 피드</dd>
<dd>\n 개행</dd>
<dd>\r 캐리지 리턴</dd>
<dd>\t 탭</dd>
<dd>\" 따옴표</dd>
<dd>\/ 슬래시</dd>
<dd>\\ 역슬래시</dd>
<dd>\uHHHH 16진수 네자리로되어 있는 유니코드 문자</dd>
</dl>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="s2">"1234"</span>
<span class="s2">"Love"</span>
<span class="s2">"O-matic"</span>
<span class="s2">"한글"</span>
<span class="s2">"\"JSON\""</span>
</pre></div>
<h3><span class="mw-headline" id=".EB.B0.B0.EC.97.B4.28Array.29">배열(Array)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JSON&amp;action=edit&amp;section=5" title="부분 편집: 배열(Array)">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>배열은 대괄호[]로 나타낸다. 배열의 각 요소는 기본 자료형이거나 배열, 객체이다. 각 요소들은 쉼표(,)로 구별된다. 각 요소가 나타나는 순서에 의미가 있다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
 <span class="p">[</span><span class="mi">10</span><span class="p">,</span> <span class="p">{</span><span class="s2">"v"</span><span class="o">:</span> <span class="mi">20</span><span class="p">},</span> <span class="p">[</span><span class="mi">30</span><span class="p">,</span> <span class="s2">"마흔"</span><span class="p">]]</span>
</pre></div>
<h3><span class="mw-headline" id=".EA.B0.9D.EC.B2.B4.28Object.29">객체(Object)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JSON&amp;action=edit&amp;section=6" title="부분 편집: 객체(Object)">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>객체는 이름/값 쌍의 집합으로, 중괄호{}를 사용한다. 이름은 문자열이기 때문에 반드시 따옴표를 하며, 값은 기본 자료형이거나 배열, 객체이다. 각 쌍들은 쉼표(,)로 구별된다. 각 쌍이 나오는 순서는 의미가 없다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
 <span class="p">{</span><span class="s2">"name2"</span><span class="o">:</span> <span class="mi">50</span><span class="p">,</span> <span class="s2">"name3"</span><span class="o">:</span> <span class="s2">"값3"</span><span class="p">,</span> <span class="s2">"name1"</span><span class="o">:</span> <span class="kc">true</span><span class="p">}</span>
</pre></div>
<p>JSON 메시지 단위는 <a href="/wiki/%EB%B0%B0%EC%97%B4" title="배열">배열</a>이나 <a class="mw-redirect mw-disambig" href="/wiki/%EA%B0%9D%EC%B2%B4" title="객체">객체</a>이다. 위의 두 예는 JSON 메시지가 될 수 있다.</p>
<h2><span class="mw-headline" id=".EC.98.88.EC.A0.9C">예제</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JSON&amp;action=edit&amp;section=7" title="부분 편집: 예제">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>다음은 한 사람에 관한 정보를 갖는 JSON 객체이다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="lineno">1 </span> <span class="p">{</span>
<span class="lineno">2 </span>    <span class="s2">"이름"</span><span class="o">:</span> <span class="s2">"테스트"</span><span class="p">,</span>
<span class="lineno">3 </span>    <span class="s2">"나이"</span><span class="o">:</span> <span class="mi">25</span><span class="p">,</span>
<span class="lineno">4 </span>    <span class="s2">"성별"</span><span class="o">:</span> <span class="s2">"여"</span><span class="p">,</span>
<span class="lineno">5 </span>    <span class="s2">"주소"</span><span class="o">:</span> <span class="s2">"서울특별시 양천구 목동"</span><span class="p">,</span>
<span class="lineno">6 </span>    <span class="s2">"특기"</span><span class="o">:</span> <span class="p">[</span><span class="s2">"농구"</span><span class="p">,</span> <span class="s2">"도술"</span><span class="p">],</span>
<span class="lineno">7 </span>    <span class="s2">"가족관계"</span><span class="o">:</span> <span class="p">{</span><span class="s2">"#"</span><span class="o">:</span> <span class="mi">2</span><span class="p">,</span> <span class="s2">"아버지"</span><span class="o">:</span> <span class="s2">"홍판서"</span><span class="p">,</span> <span class="s2">"어머니"</span><span class="o">:</span> <span class="s2">"춘섬"</span><span class="p">},</span>
<span class="lineno">8 </span>    <span class="s2">"회사"</span><span class="o">:</span> <span class="s2">"경기 안양시 만안구 안양7동"</span><span class="p">;</span>
<span class="lineno">9 </span> <span class="p">}</span>
</pre></div>
<h2><span class="mw-headline" id=".EC.9E.A5.EC.A0.90">장점</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JSON&amp;action=edit&amp;section=8" title="부분 편집: 장점">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>JSON은 텍스트로 이루어져 있으므로, 사람과 기계 모두 읽고 쓰기 쉽다.</li>
<li><a href="/wiki/%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4" title="프로그래밍 언어">프로그래밍 언어</a>와 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%8C%85_%ED%94%8C%EB%9E%AB%ED%8F%BC" title="컴퓨팅 플랫폼">플랫폼</a>에 독립적이므로, 서로 다른 시스템간에 객체를 교환하기에 좋다.</li>
<li><a href="/wiki/%EC%9E%90%EB%B0%94%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8" title="자바스크립트">자바스크립트</a>의 문법을 채용했기 때문에, 자바스크립트에서 <code>eval</code> 명령으로 곧바로 사용할 수 있다. 이런 특성은 자바스크립트를 자주 사용하는 웹 환경에서 유리하다. 그러나 실질적으로 <code>eval</code> 명령을 사용하면 외부에서 악성 코드가 유입될 수 있다. <a href="/wiki/%EB%AA%A8%EC%A7%88%EB%9D%BC_%ED%8C%8C%EC%9D%B4%EC%96%B4%ED%8F%AD%EC%8A%A4" title="모질라 파이어폭스">모질라 파이어폭스</a> 3.5, <a href="/wiki/%EC%9D%B8%ED%84%B0%EB%84%B7_%EC%9D%B5%EC%8A%A4%ED%94%8C%EB%A1%9C%EB%9F%AC" title="인터넷 익스플로러">인터넷 익스플로러</a> 8, <a href="/wiki/%EC%98%A4%ED%8E%98%EB%9D%BC_(%EC%9B%B9_%EB%B8%8C%EB%9D%BC%EC%9A%B0%EC%A0%80)" title="오페라 (웹 브라우저)">오페라</a> 10.5, <a href="/wiki/%EC%82%AC%ED%8C%8C%EB%A6%AC_(%EC%9B%B9_%EB%B8%8C%EB%9D%BC%EC%9A%B0%EC%A0%80)" title="사파리 (웹 브라우저)">사파리</a>, <a href="/wiki/%EA%B5%AC%EA%B8%80_%ED%81%AC%EB%A1%AC" title="구글 크롬">구글 크롬</a> 등 대부분의 최신 <a href="/wiki/%EC%9B%B9_%EB%B8%8C%EB%9D%BC%EC%9A%B0%EC%A0%80" title="웹 브라우저">웹 브라우저</a>는 JSON 전용 파서 기능을 내장하고 있으므로 이런 기능을 사용하는 것이 더 안전할 뿐만 아니라 빠른 방법이다.</li>
</ul>
