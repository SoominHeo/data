<h1 class="firstHeading" id="firstHeading" lang="ko">JAR (파일 포맷)</h1>
<div class="mw-body-content" id="bodyContent">

<p><b>JAR</b>(<span lang="en" xml:lang="en">Java Archive</span>, 자바 아카이브)는 <a href="/wiki/%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="소프트웨어">소프트웨어</a>에서 수많은 <a class="new" href="/w/index.php?title=%EC%9E%90%EB%B0%94_%ED%81%B4%EB%9E%98%EC%8A%A4&amp;action=edit&amp;redlink=1" title="자바 클래스 (없는 문서)">자바 클래스 파일</a>과 연관 <a href="/wiki/%EB%A9%94%ED%83%80%EB%8D%B0%EC%9D%B4%ED%84%B0" title="메타데이터">메타데이터</a>, 리소스(텍스트, 그림 등)을 하나의 파일로 모아서 <a class="mw-redirect" href="/wiki/%EC%9E%90%EB%B0%94_%ED%94%8C%EB%9E%AB%ED%8F%BC" title="자바 플랫폼">자바 플랫폼</a>에 <a href="/wiki/%EC%9D%91%EC%9A%A9_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="응용 소프트웨어">응용 소프트웨어</a>나 <a href="/wiki/%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC_(%EC%BB%B4%ED%93%A8%ED%8C%85)" title="라이브러리 (컴퓨팅)">라이브러리</a>를 배포하기 위한 패키지 파일 포맷이다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p>JAR 파일은 실제로 <a href="/wiki/ZIP_(%ED%8C%8C%EC%9D%BC_%ED%8F%AC%EB%A7%B7)" title="ZIP (파일 포맷)">ZIP 파일 포맷</a>으로 이루어진 <a href="/wiki/%EC%95%95%EC%B6%95_%ED%8C%8C%EC%9D%BC" title="압축 파일">압축 파일</a>로서, 파일 확장자는 <code>.jar</code>이다. 컴퓨터 사용자들은 <a class="mw-redirect" href="/wiki/%EC%9E%90%EB%B0%94_%EA%B0%9C%EB%B0%9C_%ED%82%B7" title="자바 개발 킷">JDK</a>에 포함된 <code>jar</code> 명령어를 이용하여 JAR 파일을 만들거나 압축을 풀 수 있다. 또, <code>zip</code> 도구를 사용할 수도 있으나 압축 시에는 <a class="new" href="/w/index.php?title=%EB%A7%A4%EB%8B%88%ED%8E%98%EC%8A%A4%ED%8A%B8&amp;action=edit&amp;redlink=1" title="매니페스트 (없는 문서)">매니페스트</a> 파일이 처음이어야 하는 경우가 있어서 zip 파일 헤더의 엔트리 순서가 중요하다. JAR 안에서 파일 이름들은 유니코드 텍스트로 되어 있다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>

<h2><span class="mw-headline" id=".EC.84.A4.EA.B3.84">설계</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=JAR_(%ED%8C%8C%EC%9D%BC_%ED%8F%AC%EB%A7%B7)&amp;action=edit&amp;section=1" title="부분 편집: 설계">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>JAR 파일은 자바 런타임이 효율적으로 클래스 집합 및 관련 리소스를 배치(디플로이)할 수 있게 한다. JAR 파일 내의 요소들은 압축할 수 있고 한 차례의 요청으로 애플리케이션 전체를 다운로드할 수 있다. 하나의 자바 애플리케이션을 이루는, 압축되지 않은 수많은 파일들을 하나씩 내려받는 것 보다 JAR 파일 하나를 다운로드하는 것이 더 편리하다. <code>java.util.zip</code> 패키지는 JAR 파일을 읽고 쓰는 클래스들을 포함하고 있다.</p>
<p>JAR 파일은 <code>META-INF/MANIFEST.MF</code>경로에 위치한 <a class="new" href="/w/index.php?title=%EB%A7%A4%EB%8B%88%ED%8E%98%EC%8A%A4%ED%8A%B8_%ED%8C%8C%EC%9D%BC&amp;action=edit&amp;redlink=1" title="매니페스트 파일 (없는 문서)">매니페스트 파일</a>을 선택적으로 포함할 수 있다. 매니페스트 파일 안의 엔트리들은 어떻게 JAR 파일을 이용할지를 결정한다. 이를테면 <a class="new" href="/w/index.php?title=%ED%81%B4%EB%9E%98%EC%8A%A4%ED%8C%A8%EC%8A%A4&amp;action=edit&amp;redlink=1" title="클래스패스 (없는 문서)">클래스패스</a> 엔트리를 사용하면 해당 JAR 파일과 함께 로드할 다른 JAR 파일들을 지정할 수 있다. 이 엔트리는 다른 JAR 파일에 대한 일련의 절대 경로나 상대 경로로 구성된다. JAR 이용을 단순하게 할 의도였으나 실제로는 안정적이지 못한 모습을 보여주었는데, 이는 엔트리 포인트 JAR가 만들어질 때 지정한 정확한 위치의 모든 JAR에 의존하기 때문이다. 라이브러리의 버전이나 위치를 변경하려면 새로운 매니페스트가 필요하다.</p>
<p>JAR 파일의 압축을 풀려면 ZIP 파일을 해제하는 표준 소프트웨어를 사용하거나 <a href="/wiki/%EC%9E%90%EB%B0%94_%EA%B0%80%EC%83%81_%EB%A8%B8%EC%8B%A0" title="자바 가상 머신">자바 가상 머신</a>에 포함된 <code>jar</code> 명령을 이용하면 된다.: <code>jar -xf foo.jar</code></p>
<p>개발자들은 JAR 파일들에 <a href="/wiki/%EB%94%94%EC%A7%80%ED%84%B8_%EC%84%9C%EB%AA%85" title="디지털 서명">디지털 서명</a>을 할 수 있다. 이 경우 서명 정보는 내재된 매니페스트 파일의 일부가 된다. JAR 그 자체는 서명되지 않으나 압축 파일 안의 모든 파일은 저마다 체크섬으로 나열된다. 즉, 체크섬을 통해 서명이 된다는 것을 뜻한다. 여러 개의 엔트리들은 각기 서명함으로써 JAR 파일 자체를 변경하여 JAR 파일을 서명할 수 있지만 서명된 파일들은 그 자체로 유효한 상태로 남아있다. 서명된 JAR 파일들을 자바 런타임이 로드할 때 서명이 유효한지 확인하고 서명이 일치하지 않는 클래스들은 로드를 거부할 수 있다..</p>
