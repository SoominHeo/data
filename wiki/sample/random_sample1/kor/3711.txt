<h1 class="firstHeading" id="firstHeading" lang="ko">덕 타이핑</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">위키백과, 우리 모두의 백과사전.</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					이동:					<a href="#mw-head">둘러보기</a>, 					<a href="#p-search">검색</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><a href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="컴퓨터 프로그래밍">컴퓨터 프로그래밍</a> 분야에서 <b>덕 타이핑</b>(duck typing)은 <a class="mw-redirect" href="/wiki/%EB%8F%99%EC%A0%81_%ED%83%80%EC%9D%B4%ED%95%91" title="동적 타이핑">동적 타이핑</a>의 한 종류로, 객체의 변수 및 메소드의 집합이 객체의 타입을 결정하는 것을 말한다. 클래스 상속이나 인터페이스 구현으로 타입을 구분하는 대신, 덕 타이핑은 객체가 어떤 타입에 걸맞은 변수와 메소드를 지니면 객체를 해당 타입에 속하는 것으로 간주한다. “덕 타이핑”이라는 용어는 다음과 같이 표현될 수 있는 <a class="new" href="/w/index.php?title=%EB%8D%95_%ED%85%8C%EC%8A%A4%ED%8A%B8&amp;action=edit&amp;redlink=1" title="덕 테스트 (없는 문서)">덕 테스트</a>에서 유래했다. (덕은 영어로 오리를 의미한다.)</p>
<blockquote class="toccolours" style="float:none; padding: 10px 15px 10px 15px; display:table;">
<p>만약 어떤 새가 오리처럼 걷고, 헤엄치고, 꽥꽥거리는 소리를 낸다면 나는 그 새를 오리라고 부를 것이다.</p>
</blockquote>
<p>덕 타이핑에서는, 객체의 타입보다 객체가 사용되는 양상이 더 중요하다. 예를 들면, 덕 타이핑이 없는 프로그래밍 언어로는 <tt>오리</tt> 타입의 객체를 인자로 받아 객체의 <tt>걷기</tt> 메소드와 <tt>꽥꽥거리기</tt> 메소드를 차례로 호출하는 함수를 만들 수 있다. 반면에, 같은 함수를 덕 타이핑이 지원되는 언어에서는 인자로 받는 객체의 타입을 검사하지 않도록 만들 수 있다. <tt>걷기</tt> 메소드나 <tt>꽥꽥거리기</tt> 메소드를 호출 할 시점에서 객체에 두 메소드가 없다면 런타임 에러가 발생하고, 두 메소드가 제대로 구현되어 있다면 함수는 정상적으로 작동한다. 여기에는 인자로 받은 객체가 <tt>걷기</tt> 메소드와 <tt>꽥꽥거리기</tt> 메소드를 갖고 있다면 객체를 <tt>오리</tt> 타입으로 간주하겠다는 암시가 깔려있다. 바로 이 점이 앞에서 인용한 덕 테스트의 사상과 일치하기 때문에 덕 타이핑이라는 이름이 붙었다.</p>

<h2><span class="mw-headline" id=".EC.98.88.EC.A0.9C">예제</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8D%95_%ED%83%80%EC%9D%B4%ED%95%91&amp;action=edit&amp;section=1" title="부분 편집: 예제">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>다음 덕 타입트(duck typed) 언어의 <a href="/wiki/%EC%9D%98%EC%82%AC%EC%BD%94%EB%93%9C" title="의사코드">의사코드</a>이다:</p>
<pre>
<b>function</b> calculate(a, b, c) =&gt; <b>return</b> (a+b)*c
</pre>
<pre>
a = calculate (1, 2, 3)
b = calculate ([1, 2, 3], [4, 5, 6], 2)
c = calculate ('apples ', 'and oranges, ', 3)
</pre>
<pre>
<b>print</b> <b>to_string</b> a
<b>print</b> <b>to_string</b> b
<b>print</b> <b>to_string</b> c
</pre>
<p>결과는 다음과 같다:</p>
<pre>
9
[1, 2, 3, 4, 5, 6, 1, 2, 3, 4, 5, 6]
apples and oranges, apples and oranges, apples and oranges,
</pre>

<h2><span class="mw-headline" id=".EA.B0.9C.EB.85.90.EC.A0.81_.EC.98.88.EC.A0.9C">개념적 예제</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8D%95_%ED%83%80%EC%9D%B4%ED%95%91&amp;action=edit&amp;section=4" title="부분 편집: 개념적 예제">편집</a><span class="mw-editsection-bracket">]</span></span></h2>

<h2><span class="mw-headline" id=".EA.B5.AC.ED.98.84">구현</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8D%95_%ED%83%80%EC%9D%B4%ED%95%91&amp;action=edit&amp;section=8" title="부분 편집: 구현">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id=".EC.8A.A4.EB.AA.B0.ED.86.A0.ED.81.AC">스몰토크</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8D%95_%ED%83%80%EC%9D%B4%ED%95%91&amp;action=edit&amp;section=9" title="부분 편집: 스몰토크">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>객체 지향 언어인 <a href="/wiki/%EC%8A%A4%EB%AA%B0%ED%86%A0%ED%81%AC" title="스몰토크">스몰토크</a>는 변수나 메소드 인자에 대한 형 선언이 없으므로, 당연히 덕 타이핑을 지원하고 있다. 스몰토크에서는 동일한 프로토콜(protocol)을 구현하는 클래스들은 보통 동일한 상속 계층에 속하나, 꼭 그럴 필요는 없다.</p>
<h3><span class="mw-headline" id=".ED.8C.8C.EC.9D.B4.EC.8D.AC">파이썬</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8D%95_%ED%83%80%EC%9D%B4%ED%95%91&amp;action=edit&amp;section=10" title="부분 편집: 파이썬">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/%ED%8C%8C%EC%9D%B4%EC%8D%AC" title="파이썬">파이썬</a>에서 덕 타이핑은 매우 많이 쓰이고 있다. <a class="external text" href="http://docs.python.org/tut/node18.html#l2h-46" rel="nofollow">파이썬 튜터리얼의 용어 항목</a>은 덕 타이핑을 다음과 같이 설명하고 있다:</p>
<blockquote class="toccolours" style="float:none; padding: 10px 15px 10px 15px; display:table;">
<p>Pythonic programming style that determines an object's type by inspection of its method or attribute signature rather than by explicit relationship to some type object ("If it looks like a duck and quacks like a duck, it must be a duck.") By emphasizing interfaces rather than specific types, well-designed code improves its flexibility by allowing polymorphic substitution. Duck-typing avoids tests using <code>type()</code> or <code>isinstance()</code>. Instead, it typically employs <code>hasattr()</code> tests or <i>EAFP</i> (Easier to Ask Forgiveness than Permission) programming.</p>
<p><i>객체의 타입을 다른 타입 객체와의 명시적인 관계를 비교하는 것이 아니라, 그 객체의 메서드나 속성들을 비교함으로써 판별하는 파이썬적인 프로그래밍 스타일이다. (“오리처럼 보이고, 오리처럼 운다면 오리임에 틀림없다.”) 특정 타입 대신 인터페이스를 강조함으로써, 잘 디자인된 코드는 다형적 대체를 허용함으로써 유연성을 향상시킬 수 있다. 덕 타이핑을 이용하면 <code>type()</code>이나 <code>isinstance()</code>를 이용한 테스트를 하지 않는다. 대신, 대개 <code>hasattr()</code> 테스트를 이용하거나, 혹은</i> EAFP <i>(Easier to ask forgiveness than permission; 하고 나서 용서를 비는 것이 하기 전에 허락을 구하는 것보다 쉽다) 프로그래밍 기법을 이용한다.</i></p>
<p><br/></p>
</blockquote>
