<h1 class="firstHeading" id="firstHeading" lang="ko">BMP 파일 포맷</h1>
<div class="mw-body-content" id="bodyContent">

<p><b>BMP 파일 포맷</b>은 <a href="/wiki/%EB%B9%84%ED%8A%B8%EB%A7%B5" title="비트맵">비트맵</a> <a class="new" href="/w/index.php?title=%EB%94%94%EC%A7%80%ED%84%B8_%EA%B7%B8%EB%A6%BC&amp;action=edit&amp;redlink=1" title="디지털 그림 (없는 문서)">디지털 그림</a>을 저장하는 데 쓰이는 <a class="new" href="/w/index.php?title=%EA%B7%B8%EB%A6%BC_%ED%8C%8C%EC%9D%BC_%ED%8F%AC%EB%A7%B7&amp;action=edit&amp;redlink=1" title="그림 파일 포맷 (없는 문서)">그림 파일 포맷</a>이다. 특히, <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">마이크로소프트 윈도</a>, <a href="/wiki/OS/2" title="OS/2">OS/2</a> 운영 체제에 널리 쓰인다. 줄여서 <b>비트맵</b>, 또 장치 독립 비트맵의 경우 <b>DIB 파일 포맷</b>이라고 한다.</p>
<p>수많은 <a href="/wiki/%EA%B7%B8%EB%9E%98%ED%94%BD_%EC%82%AC%EC%9A%A9%EC%9E%90_%EC%9D%B8%ED%84%B0%ED%8E%98%EC%9D%B4%EC%8A%A4" title="그래픽 사용자 인터페이스">그래픽 사용자 인터페이스</a>는 자체 내장 그래픽 하부 시스템에서 비트맵을 사용한다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> 이를테면, 마이크로소프트 윈도와 OS/2 플랫폼의 <a href="/wiki/%EA%B7%B8%EB%9E%98%ED%94%BD_%EC%9E%A5%EC%B9%98_%EC%9D%B8%ED%84%B0%ED%8E%98%EC%9D%B4%EC%8A%A4" title="그래픽 장치 인터페이스">GDI</a> 하부 시스템은 <code>.BMP</code>, <code>.DIB</code>의 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%99%95%EC%9E%A5%EC%9E%90" title="파일 확장자">파일 확장자</a> 이름을 가진다.</p>
<p>기본적으로 1~24비트의 색을 표현할 수 있다. 1비트는 2가지 색이며 24비트는 16777216가지 색이다. <a href="/wiki/%EC%95%8C%ED%8C%8C_%EC%B1%84%EB%84%90" title="알파 채널">알파 채널</a>을 포함한 32비트 포맷이 <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_XP" title="윈도 XP">윈도 XP</a>에서 발표되었다. 일반적으로 데이터를 압축하지 않고 사용되지만, <a class="mw-redirect" href="/wiki/%EB%B0%98%EB%B3%B5_%EA%B8%B8%EC%9D%B4_%EB%B6%80%ED%98%B8%ED%99%94" title="반복 길이 부호화">RLE 압축</a> 방식도 지원한다.</p>

<h2><span class="mw-headline" id="BMP_.ED.8F.AC.EB.A7.B7.EC.9D.98_.EC.82.AC.EC.9A.A9">BMP 포맷의 사용</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=BMP_%ED%8C%8C%EC%9D%BC_%ED%8F%AC%EB%A7%B7&amp;action=edit&amp;section=4" title="부분 편집: BMP 포맷의 사용">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>BMP 파일 포맷은 단순하고 윈도와 같은 여러 곳에서 널리 쓰일뿐 아니라 상대적으로 <a href="/wiki/%ED%8A%B9%ED%97%88" title="특허">특허</a>에서 자유로우므로, 수많은 운영 체제가 제공하는 그림 처리 프로그램이 읽고 쓸 수 있는 매우 일반적인 포맷으로 자리잡았다.</p>
<p>대부분의 BMP 파일은 압축을 거의 하지 않아 상대적으로 크기가 큰 반면, 수많은 BMP 파일은 <a href="/wiki/ZIP_(%ED%8C%8C%EC%9D%BC_%ED%8F%AC%EB%A7%B7)" title="ZIP (파일 포맷)">ZIP</a>과 같은 <a class="mw-redirect" href="/wiki/%EB%AC%B4%EC%86%90%EC%8B%A4_%EB%8D%B0%EC%9D%B4%ED%84%B0_%EC%95%95%EC%B6%95" title="무손실 데이터 압축">무손실 데이터 압축</a> 알고리즘으로 상당 용량을 압축할 수 있다.</p>
