<p><b>데이터 분산 서비스</b>(<span lang="en" xml:lang="en">Data Distribution Service</span>, DDS) 는 실시간 시스템의 실시간성(real-time), 규모가변성(scalable), 안전성 (dependable), 고성능 (high performance)를 가능하게 하는 <a class="mw-redirect" href="/wiki/Object_Management_Group" title="Object Management Group">Object Management Group</a>(OMG) 표준 출판/구독 (Publish/Subscribe) 네트워크 커뮤니케이션 미들웨어이다. 분산 환경을 위한 데이터 중심의 출판/구독 프로그래밍 모델에 대한 표준화의 필요성에 의해 만들어졌다. 2개의 메이저 벤더인 미국의 Real-Time Innovations (RTI)와 프랑스의 Thales Group이 OMG에 의해 승인된 DDS 규격을 만들기 위해 협력한 2004년까지 독점적 DDS 솔루션만 사용 가능했다. DDS는 금융거래, 항공 교통관제, 스마트 그리드 관리, 산업자동화 시스템등의 mission critical 애플리케이션을 위해 디자인되었다. 과거에는 국방, 항공 우주산업에 제한적으로 사용되었지만, 현재는 다양한 분야의 지능화 시스템 (Intelligent System)에 사용되는 사례가 늘어나고 있다.</p>

<h2><span class="mw-headline" id=".EB.B2.84.EC.A0.84_.EC.97.AD.EC.82.AC">버전 역사</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8D%B0%EC%9D%B4%ED%84%B0_%EB%B6%84%EC%82%B0_%EC%84%9C%EB%B9%84%EC%8A%A4&amp;action=edit&amp;section=1" title="부분 편집: 버전 역사">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li><b>DDS 1.3</b> - 작업 중인 버전, OMG 회원만 가능.</li>
<li><b>DDS 1.2</b> - 2007.01.01</li>
<li><b>DDS 1.1</b> - 2005.12.04</li>
<li><b>DDS 1.0</b> - 2003.06.01</li>
</ul>
<p>DDS의 표준 스펙은 2단계의 인터페이스로 구성된다:</p>
<ul>
<li>낮은 레벨의 DCPS(Data-Centric Publish-Subscribe) 레벨은 적절한 정보를 적절한 수령인에게 효율적으로 전달하도록 하는데 그 방향을 두고 있다.</li>
<li>높은 레벨의 DLRL(Data Local Reconstruction Layer) 레벨은 애플리케이션에서 DDS를 쉽게 통합 할 수 있도록한다. DLRL레벨의 구현은 선택적이다.</li>
</ul>
<h2><span class="mw-headline" id=".EA.B5.AC.EC.A1.B0">구조</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8D%B0%EC%9D%B4%ED%84%B0_%EB%B6%84%EC%82%B0_%EC%84%9C%EB%B9%84%EC%8A%A4&amp;action=edit&amp;section=2" title="부분 편집: 구조">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="DDS_.EC.97.94.ED.8A.B8.EB.A6.AC">DDS 엔트리</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8D%B0%EC%9D%B4%ED%84%B0_%EB%B6%84%EC%82%B0_%EC%84%9C%EB%B9%84%EC%8A%A4&amp;action=edit&amp;section=3" title="부분 편집: DDS 엔트리">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<dl>
<dt>DomainParticipantFactory</dt>
<dd>DDS의 진입점인 싱글턴(<a href="/wiki/%EC%8B%B1%EA%B8%80%ED%84%B4_%ED%8C%A8%ED%84%B4" title="싱글턴 패턴">Singleton</a>) 팩토리</dd>
</dl>
<dl>
<dt>DomainParticipant</dt>
<dd>특정 도메인 내에서 통신을 위한 진입점. 하나의 도메인에 애플리케이션의 참가자를 표현한다. 또한 DDS 출판(Publish), 구독(Subscribe), 토픽(Topic), 멀티토픽(Multi-Topic), 콘텐트필터드토픽(ContentFilteredTopics)의 생성에 대한 팩토리로서 동작한다.</dd>
</dl>
<dl>
<dt>TopicDescription</dt>
<dd>토픽, 콘텐트필터드토픽, 멀티토픽을 위한 추상화된 기본 클래스</dd>
</dl>
<dl>
<dt>Topic</dt>
<dd>TopicDescription의 분화한 형태로, 대부분의 데이터의 기본 디스크립션이며, 출판되거나 구독된다.</dd>
</dl>
<dl>
<dt>ContentFilterdTopic</dt>
<dd>Topic과 같은 TopicDescription의 분화된 형태로 콘텐츠-기반의 구독을 허용한다.</dd>
</dl>
<dl>
<dt>MultiTopic</dt>
<dd>Topic과 같은 TopicDescription의 분화된 형태로, 여러개의 토픽들로부터 데이터의 결합/필터/재배치를 허용한다.</dd>
</dl>
<dl>
<dt>Publisher</dt>
<dd>출판자(Publisher)는 객체로, 실제로 데이터의 보급을 책임진다.</dd>
</dl>
<dl>
<dt>DataWriter</dt>
<dd>애플리케이션이 주어진 토픽 하에서 출판되기 위해 데이터 값을 설정하는 것을 허용한다.</dd>
</dl>
<dl>
<dt>Subscriber</dt>
<dd>구독자(Subscriber)는 객체로 구독물로부터 데이터 결과를 실제로 받는 것을 책임진다.</dd>
</dl>
<dl>
<dt>DataReader</dt>
<dd>애플리케이션이 받기를 원하는 데이터가 있음을 알리면(토픽, 콘텐트 필터드 토픽 또는 멀티토픽을 이용한 서브스크립션이 만들어진) 허용하고, 덧붙여진 구독자에 의해 받은 데이터를 액세스할 수 있다.</dd>
</dl>
<h3><span class="mw-headline" id="DDS_.EB.AA.A8.EB.8D.B8">DDS 모델</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8D%B0%EC%9D%B4%ED%84%B0_%EB%B6%84%EC%82%B0_%EC%84%9C%EB%B9%84%EC%8A%A4&amp;action=edit&amp;section=4" title="부분 편집: DDS 모델">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>DDS는 복잡한 네트워크 프로그래밍을 단순화 하는 네트워크 미들웨어이다. 노드에 걸쳐 데이터, 이벤트, 그리고 명령어를 주고 받기 위해 출판-구독 모델을 구현했다.</p>
<p>정보를 생성하는 노드(출판자들)는 "토픽"을(예 : 기온, 장소, 압력) 생성하고, 데이터 샘플을 출판한다. DDS는 샘플을 해당 토픽에 관심있어하는 모든 구독자들에게 배달한다</p>
<p>DDS는 모든 전송에 대한 것들을 다룬다. : 메시지 어드레싱, 데이터 마샬링 &amp; 디마샬링(그래서 구독자가 출판자와 다른 플랫폼 하에 있을 수 있다.), 배달, 흐름제어, 재시도, 기타. 어떤 노트도출판, 구독, 양쪽 모두를 동시에 할 수 있다.</p>
<p>DDS 출판-구독 모델은 분산 애플리케이션에서의 복잡한 네트워크 프로그래밍을 가상적으로 제거한다.</p>
<p>DDS는 기본적인 출판-구독 모델 이상의 메커니즘을 지원한다. 통신으로 DDS를 이용하는 애플리케이션이 갖는 주요한 이득은 완전히 분리된다는 점이다. 상호간의 응답을 다루기 위해서 아주 적은 설계시간이 소요된다.</p>
<p>특히, 애플리케이션들은 위치나 존재를 포함한 참가하는 다른 애플리케이션들에 대한 정보를 필요로 하지 않는다. DDS는 사용자 애플리케이션들로부터의 어떤 간섭 요구없이, 다음 내용을 포함한 모든 메시지 전송에 대한 것을 자동적으로 다룬다.</p>
<ul>
<li>메시지를 누가 받을 것인지 결정하는 것</li>
<li>구독자가 어디 위치해 있는지</li>
<li>메시지를 전달할 수 없을 경우 어떻게 되는지</li>
</ul>
<p>DDS는 사용자가 Quality of Service(QoS) 파라미터를 자동-발견 메커니즘을 설정하기위한 방법으로써 명기하는 것을 허용하고 메시지를 보내거나 받을 때 사용되는 행동을 명기한다. 메커니즘은 선행으로 설정되고, 유저 파트에서 더 이상의 노력을 필요로 하진 않는다. 완전한 익명으로 메시지를 교환함으로써 DDS는 분산된 애플리케이션 설계를 단순화하고 모듈러형태의-잘 구조화된 프로그램을 장려한다.DDS는 또한 만약 첫 번째가 실패가 되면 핫-스왑 리던던트 출판을 자동적으로 다룬다. 구독자는 항상 유효한 데이터에 높은 우선순위를 둔 샘플을 취한다. 역시 첫 번째가 복원되었을 때 자동적으로 돌아올 수도 있다. DDS는 Ada, C, C++, C# 그리고 Java API를 포함한 언어로 사용할 수 있다.</p>
