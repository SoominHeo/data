<h1 class="firstHeading" id="firstHeading" lang="ko">EXE</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">위키백과, 우리 모두의 백과사전.</div>
<div id="contentSub"><span class="mw-redirectedfrom">(<a class="mw-redirect" href="/w/index.php?title=.exe&amp;redirect=no" title=".exe">.exe</a>에서 넘어옴)</span></div>
<div class="mw-jump" id="jump-to-nav">
					이동:					<a href="#mw-head">둘러보기</a>, 					<a href="#p-search">검색</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><b>EXE</b>는 일반적인 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%99%95%EC%9E%A5%EC%9E%90" title="파일 확장자">파일 확장자</a>로 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%A8" title="컴퓨터 프로그램">컴퓨터 프로그램</a>의 <a href="/wiki/%EC%8B%A4%ED%96%89_%ED%8C%8C%EC%9D%BC" title="실행 파일">실행 파일</a>을 가리킨다. <a class="mw-redirect" href="/wiki/%EC%98%A4%ED%94%88VMS" title="오픈VMS">오픈VMS</a>, <a href="/wiki/%EB%8F%84%EC%8A%A4" title="도스">도스</a>, <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">마이크로소프트 윈도</a>, <a class="mw-redirect" href="/wiki/%EB%A6%AC%EC%97%91%ED%8A%B8%EC%98%A4%EC%97%90%EC%8A%A4" title="리엑트오에스">리엑트오에스</a>, <a href="/wiki/OS/2" title="OS/2">OS/2</a> 운영 체제에서 사용할 수 있다.</p>
<p>실행 프로그램 자체뿐 아니라, 많은 EXE 파일들은 비트맵, 실행 프로그램이 <a href="/wiki/%EA%B7%B8%EB%9E%98%ED%94%BD_%EC%82%AC%EC%9A%A9%EC%9E%90_%EC%9D%B8%ED%84%B0%ED%8E%98%EC%9D%B4%EC%8A%A4" title="그래픽 사용자 인터페이스">그래픽 사용자 인터페이스</a>를 사용하는, 아이콘과 같은 <a class="new" href="/w/index.php?title=%EB%A6%AC%EC%86%8C%EC%8A%A4_(%EC%BB%B4%ED%93%A8%ED%84%B0_%EA%B3%BC%ED%95%99)&amp;action=edit&amp;redlink=1" title="리소스 (컴퓨터 과학) (없는 문서)">리소스</a>라고 불리는 다른 구성 요소들을 포함할 수 있다.</p>
<p>도스 실행 파일 포맷은 64 <a href="/wiki/%ED%82%AC%EB%A1%9C%EB%B0%94%EC%9D%B4%ED%8A%B8" title="킬로바이트">킬로바이트</a>로 크기가 제한되는 <a href="/wiki/COM_%ED%8C%8C%EC%9D%BC" title="COM 파일">COM</a> 실행 파일과 다르다. 도스 실행 파일 헤더는 여러 개의 세그먼트가 <a class="mw-redirect" href="/wiki/%EA%B8%B0%EC%96%B5_%EC%A7%81%EC%A0%91_%EC%A0%91%EA%B7%BC" title="기억 직접 접근">DMA</a>에서 로드될 수 있으며 64 킬로바이트 이상의 실행 파일을 지원하는 리로케이션 정보를 포함하고 있다.</p>

<h2><span class="mw-headline" id=".ED.8C.8C.EC.9D.BC_.ED.98.95.EC.8B.9D">파일 형식</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EXE&amp;action=edit&amp;section=1" title="부분 편집: 파일 형식">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id=".EB.8F.84.EC.8A.A4">도스</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EXE&amp;action=edit&amp;section=2" title="부분 편집: 도스">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>.exe 확장자와 함께 쓰이는 여러 종류의 다양한 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%98%95%EC%8B%9D" title="파일 형식">파일 형식</a>이 있다.</p>
<dl>
<dt>16-bit DOS MZ 실행 파일 (Executable)</dt>
<dd>원래 도스 실행 파일 포맷. 이것들은 파일 시작 부분의 아스키 코드 "MZ" 문자로 구별된다.</dd>
<dt>16-bit New 실행 파일 (Executable)</dt>
<dd>멀티태스킹 MS-DOS 4.0에서 도입되어, 16-bit OS/2 와 윈도우에서 사용되었으며, NE는 아스키 코드 "NE" 문자로 구별된다.</dd>
</dl>
<h3><span class="mw-headline" id="OS.2F2">OS/2</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EXE&amp;action=edit&amp;section=3" title="부분 편집: OS/2">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<dl>
<dt>32-bit 선형 실행 파일 (Linear Executable)</dt>
<dd>OS/2 2.0에서 도입되었으며, 이것들은 아스키 코드 "LX"로 구별된다. 이것들은 오직 OS/2 2.0와 그 이상 버전에서만 사용 가능하다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> 또한 몇몇 DOS extenders에 의해 사용된다.</dd>
<dt>Mixed 16/32-bit 선형 실행 파일 (Linear Executable)</dt>
<dd>OS/2 2.0에서 도입되었으며, 이것들은 아스키 코드 "LE"로 구별된다. 이 포맷은 <a href="/wiki/VxD" title="VxD">VxD</a> 드라이버로 사용되며, 또한 몇몇 DOS extenders에 의해 사용된다. (<a class="mw-disambig" href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_3.x" title="윈도우 3.x">윈도우 3.x</a>, <a href="/wiki/OS/2" title="OS/2">OS/2</a>, 그리고 윈도우 9x)</dd>
</dl>
<h3><span class="mw-headline" id=".EC.9C.88.EB.8F.84.EC.9A.B0">윈도우</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EXE&amp;action=edit&amp;section=4" title="부분 편집: 윈도우">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<dl>
<dt>16-bit New Executable</dt>
<dd>16비트 또는 32비트 윈도우 실행 파일은 NE 또는 PE에서 실행되며, MZ 코드는 "DOS stub"이라고 불리며 무시된다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> <a href="/wiki/%EB%8F%84%EC%8A%A4" title="도스">도스</a>에서 실행되는 경우, 스텁 코드는 "This program cannot be run in DOS mode" 메시지를 표시하고 종료된다. <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_%EB%A0%88%EC%A7%80%EC%8A%A4%ED%8A%B8%EB%A6%AC" title="윈도우 레지스트리">regedit</a><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> 이나 오래된 WinZIP self extractors 같은 소수의 듀얼 모드 프로그램은 더 기능적인 DOS 섹션을 포함한다.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></dd>
<dt>32-bit Portable Executable</dt>
<dd>윈도우 NT에 도입되었으며, 아스키 코드 "PE"로 구별된다. (비록 시작 부분은 아니지만 이 파일도 "MZ"로 시작한다) <sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></dd>
<dt>64-bit Portable Executable (PE32+)</dt>
<dd>64비트 버전의 윈도우에서 도입되었으며 확장된 필드를 갖는 PE 파일이다. 대부분의 경우 코드는 32비트 또는 64비트 어느 하나에서 돌아갈 수 있게 써질 수 있다. <sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></dd>
</dl>
