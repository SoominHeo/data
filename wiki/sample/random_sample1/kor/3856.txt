<p><b>도메인 네임 등록 대행자</b>(domain name registrar)는 <a href="/wiki/%EC%9D%BC%EB%B0%98_%EC%B5%9C%EC%83%81%EC%9C%84_%EB%8F%84%EB%A9%94%EC%9D%B8" title="일반 최상위 도메인">일반 최상위 도메인</a> (gTLD) 레지스트리나 <a href="/wiki/%EA%B5%AD%EA%B0%80_%EC%BD%94%EB%93%9C_%EC%B5%9C%EC%83%81%EC%9C%84_%EB%8F%84%EB%A9%94%EC%9D%B8" title="국가 코드 최상위 도메인">국가 코드 최상위 도메인</a> 레지스트리 (ccTLD)에 의해 공인된 기구 또는 사업체로서, 지정된 <a href="/wiki/%EB%8F%84%EB%A9%94%EC%9D%B8_%EB%84%A4%EC%9E%84_%EB%A0%88%EC%A7%80%EC%8A%A4%ED%8A%B8%EB%A6%AC" title="도메인 네임 레지스트리">도메인 네임 레지스트리</a>의 지침에 따라 <a href="/wiki/%EC%9D%B8%ED%84%B0%EB%84%B7" title="인터넷">인터넷</a> <a class="mw-redirect" href="/wiki/%EB%8F%84%EB%A9%94%EC%9D%B8_%EC%9D%B4%EB%A6%84" title="도메인 이름">도메인 이름</a>의 등록을 관리하고, 도메인 관련 서비스를 일반에 제공한다.</p>

<h2><span class="mw-headline" id=".EC.97.AD.EC.82.AC">역사</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%EB%84%A4%EC%9E%84_%EB%93%B1%EB%A1%9D_%EB%8C%80%ED%96%89%EC%9E%90&amp;action=edit&amp;section=1" title="부분 편집: 역사">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>1999년 이전에는 <a class="extiw" href="https://en.wikipedia.org/wiki/Network_Solutions" title="en:Network Solutions">Network Solutions</a> (NSI)가 <tt>com</tt>, <tt>net</tt> 그리고 <tt>org</tt> 레지스트리를 운영하였으며, 유일한 등록 대행자이기도 했다. 하지만 몇몇 회사들이 독립적인 등록 대행 서비스를 개발하였고, 그러한 회사 중 NetNames는 일반 도메인 등록과 그에 연관된 서비스들을 판매하는 독자적인 상용 도메인 이름 등록 서비스를 개발하였다. 이것은 도메인 산업에 소매 모델을, 레지스트리에게는 도매 판매라는 역할을 만들어 냈다. NSI는 이 모델을 받아들였고, 마침내 레지스트리와 등록 대행자의 역할을 분리하게 되었다.</p>
<p>1997년, PGMedia가 ROOT 존은 인터넷의 필수 설비이기 때문에 접근을 제한하는 것은 반독점 법에 위반이라고 지적하며 NSI를 고발하였고, 이 때문에 미 <a class="mw-redirect" href="/wiki/%EA%B5%AD%EB%A6%BD%EA%B3%BC%ED%95%99%EC%9E%AC%EB%8B%A8" title="국립과학재단">국립과학재단</a> (NSF)도 여기에 개입하게 된다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> 최종적으로, NSI는 반독점 소송에서는 면제되는 것으로 결론이 났지만, 이 소송으로 인해 도메인 이름 시장은 재편되게 된다.</p>
<p>도메인 이름 등록 사업이 점점 커지고 관련 집단의 압력이 커짐에 따라, 1998년 10월, NSI는 미국 상무성과의 협약을 개정하였다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> 이 개정안은 다수의 등록 대행자를 지원하는 <a class="new" href="/w/index.php?title=%EA%B3%B5%EC%9A%A9_%EB%93%B1%EB%A1%9D_%EC%8B%9C%EC%8A%A4%ED%85%9C&amp;action=edit&amp;redlink=1" title="공용 등록 시스템 (없는 문서)">공용 등록 시스템</a>을 만들 것을 포함하고 있다. 시험 목적의 몇몇 등록 대행자들이 1999년 3월 11일부터 이 시스템을 사용하였고, <a class="mw-redirect" href="/wiki/%EA%B5%AD%EC%A0%9C_%EC%9D%B8%ED%84%B0%EB%84%B7_%EC%A3%BC%EC%86%8C%EC%9E%90%EC%9B%90_%EA%B4%80%EB%A6%AC%EA%B8%B0%EA%B4%80" title="국제 인터넷 주소자원 관리기관">국제 인터넷 주소자원 관리기관</a> (ICANN)의 감독하에 1999년 11월 30일 공식적으로 서비스가 시작되었다. 그 후 지금까지 900여 개 이상의 등록 대행자가 도메인 이름 등록 서비스 시장에 뛰어들었다.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<p>초기 시장 진입자 중 다수는 성장을 거듭하며 경쟁자들을 앞질러 나갔다. <a class="extiw" href="https://en.wikipedia.org/wiki/Go_Daddy" title="en:Go Daddy">Go Daddy</a>는 현재 가장 큰 등록 대행자이다.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> 다른 성공적인 등록 대행자들로는 <a class="extiw" href="https://en.wikipedia.org/wiki/eNom" title="en:eNom">eNom</a>, <a class="extiw" href="https://en.wikipedia.org/wiki/Tucows" title="en:Tucows">Tucows</a>, <a class="extiw" href="https://en.wikipedia.org/wiki/Melbourne_IT" title="en:Melbourne IT">Melbourne IT</a> 등을 들 수 있다. 초기에는 시장을 선도했으나 이후 경쟁자들에게 추월당한 업체로는 <a class="extiw" href="https://en.wikipedia.org/wiki/Network_Solutions" title="en:Network Solutions">Network Solutions</a>, <a class="extiw" href="https://en.wikipedia.org/wiki/Dotster" title="en:Dotster">Dotster</a>등이 있다.</p>
<p>ICANN 공인 등록 대행자들은 미화 4,000불의 고정 수수료와 추가적인 부대 수수료를 내야 한다.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> 등록 대행자 수수료의 합은 총 미화 3백 8십만 불에 이른다. 등록 시스템 자체는 서로 공유하기 때문에 등록 대행자들은 여러 관련 서비스들을 다양한 가격으로 최종 사용자에게 제공함으로써 경쟁하고 있다.</p>
<h2><span class="mw-headline" id=".EC.A7.80.EC.A0.95_.EB.93.B1.EB.A1.9D_.EB.8C.80.ED.96.89.EC.9E.90">지정 등록 대행자</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%EB%84%A4%EC%9E%84_%EB%93%B1%EB%A1%9D_%EB%8C%80%ED%96%89%EC%9E%90&amp;action=edit&amp;section=2" title="부분 편집: 지정 등록 대행자">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>도메인 네임 레지스트리는 도메인 등록 정보를 관리하고, 도메인 등록 대행자는 도메인 네임 레지스트리와 계약을 맺고 도메인 등록 서비스를 제공한다. 최종 사용자는 도메인을 등록할 때 어느 한 등록 대행자를 통해서 하게 되는데, 이때 도메인 등록 서비스를 제공한 등록 대행자가 그 도메인의 지정 등록 대행자가 된다.</p>
<p>지정 등록 대행자만이 중앙 레지스트리 데이터베이스에서 도메인 이름에 관한 정보를 수정 또는 삭제할 수 있다. 최종 사용자가 <a href="#.EB.8F.84.EB.A9.94.EC.9D.B8_.EC.9D.B4.EB.A6.84_.EC.9D.B4.EC.A0.84">도메인 이전</a> 절차를 통해 등록 대행자를 바꾸는 것은 낯선 것이 아니며, 이 절차는 도메인 이름 이전 정책을 따른다.</p>
<p><tt>com</tt> 도메인의 등록 대행자가 도메인을 등록할 때 <tt>com</tt> 레지스트리인 VeriSign에 미화로 최고 7.85불<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup>의 연간 수수료와 ICANN에 연간 관리 수수료로 미화 0.18불을 내야 한다. 대부분의 도메인 등록 대행자는 이를 포함해서 서비스와 상품 가격을 책정하고 있다. 기존 고객 기반이 없는 사업자에게 대규모 등록 대행 산업은 진입 장벽이 높은 편이다.</p>
<p>많은 등록 대행자들이 리셀러 제휴를 통해서도 등록 서비스를 제공하고 있기 때문에, 최종 사용자는 등록 대행자에게 직접 등록하거나, 또는 소매상을 통해 몇 단계를 거쳐 간접적으로 등록을 하기도 한다. 2010년 현재 소매가는 종종 등록 대행자들이 <a href="/wiki/%EC%9B%B9_%ED%98%B8%EC%8A%A4%ED%8C%85_%EC%84%9C%EB%B9%84%EC%8A%A4" title="웹 호스팅 서비스">웹 호스팅 서비스</a>와 같이 다른 상품과 같이 주문할 경우 상당히 낮춰주기도 - 심지어 공짜로도 제공 - 하지만, 단순 도메인 한 개 등록에 연 7.5불에서 연 35불 정도이다.</p>
<p>도메인 이름의 최장 등록 기간은 10년이다. 일부 등록 대행자가 최고 100년 까지도 더 긴 기간을 제공하기도 하지만, 이는 등록 대행자가 고객을 대신에 등록 갱신을 해주는 것이다. 공식적인 등록 데이터베이스에서는 100년 기간 등록은 존재하지 않는다.</p>
<h2><span class="mw-headline" id="DNS_.ED.98.B8.EC.8A.A4.ED.8C.85">DNS 호스팅</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%EB%84%A4%EC%9E%84_%EB%93%B1%EB%A1%9D_%EB%8C%80%ED%96%89%EC%9E%90&amp;action=edit&amp;section=3" title="부분 편집: DNS 호스팅">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a class="new" href="/w/index.php?title=DNS_%ED%98%B8%EC%8A%A4%ED%8C%85&amp;action=edit&amp;redlink=1" title="DNS 호스팅 (없는 문서)">DNS 호스팅</a>, <a href="/wiki/%EB%8F%84%EB%A9%94%EC%9D%B8_%EB%84%A4%EC%9E%84_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="도메인 네임 시스템">도메인 네임 시스템</a>입니다.</div>
<p>도메인을 등록하게 되면, 일련의 <a class="extiw" href="https://en.wikipedia.org/wiki/Start_of_Authority" title="en:Start of Authority">Start of Authority</a> (SOA) record를 상위 도메인에 설정하게 되는데, 이것은 해당 도메인에 대해 공인된 권한을 가지는 네임 서버의 <a href="/wiki/IP_%EC%A3%BC%EC%86%8C" title="IP 주소">IP 주소</a> (또는 도메인 이름)이다. 즉, 실제 도메인 정보가 아니라 어디서 해당 도메인의 정보를 찾을 수 있는 지에 대한 참조만을 제공한다.</p>
<p>도메인을 등록했다고 해서 자동으로 등록된 도메인에 대한 DNS 서비스가 제공되는 것은 아니다. 그래서 대부분의 등록 대행자들은 자신들을 통해 등록한 도메인에 한해 사용자가 원할 경우 <a class="new" href="/w/index.php?title=DNS_%ED%98%B8%EC%8A%A4%ED%8C%85&amp;action=edit&amp;redlink=1" title="DNS 호스팅 (없는 문서)">DNS 호스팅</a>을 무료 서비스로 제공하고 있다. 만약 등록 대행자가 DNS 서비스를 제공하지 않거나, 또는 최종 사용자가 이를 사용하지 않기로 했다면, 최종 사용자는 직접 DNS 서비스를 구하거나 운영해야 할 책임이 있다. DNS 서비스 없이 도메인만 등록하는 것은 인터넷 상에서는 본질적으로 무의미하지만, <a class="new" href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%ED%8C%8C%ED%82%B9&amp;action=edit&amp;redlink=1" title="도메인 파킹 (없는 문서)">도메인 파킹</a> (<b><span lang="en" xml:lang="en">domain parking</span></b>)이나 <a class="new" href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%EC%82%AC%EB%83%A5&amp;action=edit&amp;redlink=1" title="도메인 사냥 (없는 문서)">도메인 사냥</a> (<b><span lang="en" xml:lang="en">cybersquatting</span></b>)과 같은 목적으로 종종 일어난다.</p>
<h2><span class="mw-headline" id="DNSSEC_.EC.A7.80.EC.9B.90">DNSSEC 지원</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%EB%84%A4%EC%9E%84_%EB%93%B1%EB%A1%9D_%EB%8C%80%ED%96%89%EC%9E%90&amp;action=edit&amp;section=4" title="부분 편집: DNSSEC 지원">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a class="new" href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%EB%84%A4%EC%9E%84_%EC%8B%9C%EC%8A%A4%ED%85%9C_%EB%B3%B4%EC%95%88_%ED%99%95%EC%9E%A5&amp;action=edit&amp;redlink=1" title="도메인 네임 시스템 보안 확장 (없는 문서)">도메인 네임 시스템 보안 확장</a> (DNSSEC)은 도메인 네임 시스템 상의 정보 보안을 위한 Internet Engineering Task Force (IETF) 명세 모음이다.</p>
<h2><span class="mw-headline" id=".EB.8F.84.EB.A9.94.EC.9D.B8_.EC.9D.B4.EB.A6.84_.EC.9D.B4.EC.A0.84">도메인 이름 이전</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%EB%84%A4%EC%9E%84_%EB%93%B1%EB%A1%9D_%EB%8C%80%ED%96%89%EC%9E%90&amp;action=edit&amp;section=5" title="부분 편집: 도메인 이름 이전">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>도메인 이름 이전은 도메인 이름의 지정 등록 대행자를 변경하는 절차이다. <a href="/wiki/ICANN" title="ICANN">ICANN</a>은 <i>등록 대행자 간 도메인 이전 정책</i><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup>을 가지고 있는데, 일반적인 이전 절차는 다음과 같다.</p>
<ol>
<li>최종 사용자는 whois에서 관리자 연락처 정보 - 특히 e-mail 주소 - 가 정확한지 확인한다. 그리고 도메인 인도 등록 대행자로부터 인증 코드(<a class="extiw" href="https://en.wikipedia.org/wiki/Transfer_secret" title="en:Transfer secret">EPP 이전 코드</a>)를 받고, 도메인에 잠겨 있으면 잠금을 푼다. 만약 whois 정보가 잘못되었거나 새로 갱신한 직후라면, 최종 사용자는 다음 단계를 진행하기 전에 갱신한 정보가 퍼질 때까지 12~24시간을 기다려야 한다.</li>
<li>최종 사용자는 새 등록 대행자에게 인증 코드와 함께 도메인 이름을 이전하고 싶다는 의사를 전달한다.</li>
<li>도메인 인수 등록 대행자는 네임 보유자 또는 도메인 관리 담당자로부터 빠른 이전 허가서를 얻어야만 한다. 도메인 이전은 인수 등록 대행자가 이들 담당자로부터 이전에 대한 확인을 받았을 때만 진행될 수 있다. 이제 도메인 인수 등록 대행자는 인증 코드를 가지고 온라인에서 도메인 이전을 시작한다.</li>
<li>도메인 인도 등록 대행자는 최종 사용자에게 이 요청이 확실함을 확인하는데, 이 때 최종 사용자가 도메인 이전이 신속하게 진행되도록 온라인 관리 툴에서 한번 더 이전 요청 과정을 진행해야 하는 경우도 있다.</li>
<li>도메인 인도 등록 대행자는 이제 도메인 등록 대행자에게 도메인에 대한 권한을 넘겨준다.</li>
<li>도메인 인수 등록 대행자는 최종 사용자에게 도메인 이전이 완료되었음을 알린다. 도메인 인수 등록 대행자가 자동으로 도메인 서버 정보를 복사해 준다면, 웹 사이트가 이전처럼 잘 작동할 것이지만, 그렇지 않다면 수동으로 이 정보를 새 등록 대행자에 업데이트해야 한다.</li>
</ol>
<p>이 과정이 끝나면 새로운 등록 대행자가 지정 등록 대행자가 된다. 대략 5일 정도 소요되지만, 경우에 따라 인도 등록 대행자가 이를 의도적으로 지연시키기도 한다. 한번 이전되면 60일 동안은 다시 예전 등록 대행자로 돌아가는 것 외에는 재이전을 할 수 없다.</p>
<p>도메인이 만료되기 직전에 도메인을 이전 하는 것은 그리 현명한 일이 아닌데, 이는 최대 14일이 걸릴 수 있는 도메인 이전이 미처 끝나기도 전에, 도메인이 만료될 수도 있기 때문이다. 이 경우 이전은 되지 않고 도메인도 잃어버리게 된다. 따라서 최종 사용자는 만료일까지 충분한 시간 여유를 두고 이전하거나, 이전하기 전에 도메인 등록을 갱신하는 것이 좋다.<a class="external autonumber" href="https://www.dyndns.com/support/kb/domain_expirations.html" rel="nofollow">[1]</a></p>
<p>무슨 이유에서건 일단 도메인이 만료되면, 원 소유자에게로 도메인을 되돌리는 것은 어렵고, 비용이 많이 들며, 심지어는 불가능한 일이기도 하다. 만료된 도메인은 수개월 동안 여러 단계를 거친 후에야 다시 등록할 수 있게 된다.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></p>
<h3><span class="mw-headline" id=".EB.8F.84.EB.A9.94.EC.9D.B8_.EC.9D.B4.EC.A0.84_.EC.82.AC.EA.B8.B0">도메인 이전 사기</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%EB%84%A4%EC%9E%84_%EB%93%B1%EB%A1%9D_%EB%8C%80%ED%96%89%EC%9E%90&amp;action=edit&amp;section=6" title="부분 편집: 도메인 이전 사기">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a class="new" href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%EA%B0%80%EB%A1%9C%EC%B1%84%EA%B8%B0&amp;action=edit&amp;redlink=1" title="도메인 가로채기 (없는 문서)">도메인 가로채기</a>입니다.</div>
<p><a class="new" href="/w/index.php?title=%EA%B3%B5%EC%9A%A9_%EB%93%B1%EB%A1%9D_%EC%8B%9C%EC%8A%A4%ED%85%9C&amp;action=edit&amp;redlink=1" title="공용 등록 시스템 (없는 문서)">공용 등록 시스템</a> 도입으로 도메인 등록의 독점 시장이 개방되었고, 등록 대행자들은 서로 치열하게 경쟁하였다. 이런 와중에 VeriSign, Domain Registry of America와 같은 일부 등록 대행자들은 고객을 눈속임함으로써 자신들의 서비스로 도메인을 이전하게 하는 방법을 쓰기도 했다. 이러한 수법을 <a class="new" href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%EA%B0%80%EB%A1%9C%EC%B1%84%EA%B8%B0&amp;action=edit&amp;redlink=1" title="도메인 가로채기 (없는 문서)">도메인 가로채기</a>라고 한다.</p>
<p>대부분의 도메인 이전 사기는 우편, 팩스 또는 전자 우편을 통한 안내문을 이용한다. 안내문에 사용할 더 많은 정보를 알아내기 위해 최종 사용자에게 전화로 (전화번호는 <a class="mw-redirect" href="/wiki/WHOIS" title="WHOIS">WHOIS</a>에서 얻을 수 있으므로) 접촉하기도 한다. 안내문이 좀 더 신뢰성이 있어 보이도록 <a class="mw-redirect" href="/wiki/WHOIS" title="WHOIS">WHOIS</a> 데이터베이스에서 일반에게 공개되어 있는 정보를 가져다 포함하기도 하고, 복잡한 법률 용어를 사용하여 마치 공식적인 처리를 필요로 하는 안내인 것처럼 최종 사용자를 헷갈리게 하기도 한다. 사기꾼 등록 대행자들은 곧 만료되거나 최근 만료된 도메인들을 찾아 다닌다. 도메인의 만료일은 <a class="mw-redirect" href="/wiki/WHOIS" title="WHOIS">WHOIS</a>를 통해 쉽게 확인할 수 있다.</p>
<h2><span class="mw-headline" id=".EB.82.99.EC.9E.A5_.EB.8F.84.EB.A9.94.EC.9D.B8_.ED.8F.AC.ED.9A.8D.EC.9E.90">낙장 도메인 포획자</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%EB%84%A4%EC%9E%84_%EB%93%B1%EB%A1%9D_%EB%8C%80%ED%96%89%EC%9E%90&amp;action=edit&amp;section=7" title="부분 편집: 낙장 도메인 포획자">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>기존 등록자가 더는 해당 도메인을 원하지 않거나 또는 등록을 제때에 갱신하지 않아서 만료되어 사용 가능해진 도메인 중 고객이 원할 경우 재빨리 등록을 대신 시도해 주는 그런 서비스를 제공하는 등록 대행자를 "떨어진" 도메인을 "잡는"다고 해서 <i>낙장 도메인 포획자</i>라고 한다.</p>
<h2><span class="mw-headline" id=".EB.93.B1.EB.A1.9D_.EB.8C.80.ED.96.89.EC.9E.90_.EC.A0.90.EC.9C.A0.EC.9C.A8_.EC.88.9C.EC.9C.84">등록 대행자 점유율 순위</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EB%A9%94%EC%9D%B8_%EB%84%A4%EC%9E%84_%EB%93%B1%EB%A1%9D_%EB%8C%80%ED%96%89%EC%9E%90&amp;action=edit&amp;section=8" title="부분 편집: 등록 대행자 점유율 순위">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>몇몇 기구에서 도메인 등록 대행자들의 시장 점유율 순위와 각각의 도메인 등록 개수를 게시하기도 한다. 공개된 목록들은 어느 <a href="/wiki/%EC%B5%9C%EC%83%81%EC%9C%84_%EB%8F%84%EB%A9%94%EC%9D%B8" title="최상위 도메인">최상위 도메인</a>을 가지고 했는지, 얼마나 자주 업데이트하는지, 레지스트리가 제공한 절대 숫자를 기본 데이터로 사용했는지 아니면 <a class="new" href="/w/index.php?title=%EC%A1%B4_%ED%8C%8C%EC%9D%BC&amp;action=edit&amp;redlink=1" title="존 파일 (없는 문서)">존 파일</a>에서 가져온 일 변화를 기본 데이터로 했는지에 따라 서로 다르다.</p>
<p>점유율 목록들은 모두 16개의 일반 최상위 도메인과 .us 도메인에 대해서만 다루고 있다. 대부분의 <a href="/wiki/%EA%B5%AD%EA%B0%80_%EC%BD%94%EB%93%9C_%EC%B5%9C%EC%83%81%EC%9C%84_%EB%8F%84%EB%A9%94%EC%9D%B8" title="국가 코드 최상위 도메인">국가 코드 최상위 도메인</a>은 <a class="new" href="/w/index.php?title=%EC%A1%B4_%ED%8C%8C%EC%9D%BC&amp;action=edit&amp;redlink=1" title="존 파일 (없는 문서)">존 파일</a> 남용과 같은 문제로 2003년 이래로 존 파일 제공하지 않고 있다. <sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
<p>공개된 순위와 보고서에는 다음과 같은 것들이 있다.</p>
<ul>
<li>(약 3개월의 시차가 있음) <a href="/wiki/ICANN" title="ICANN">ICANN</a>은 16 일반 최상위 도메인 레지스트리가 작성한 <a class="external text" href="http://www.icann.org/tlds/monthly-reports/index.html" rel="nofollow">보고서</a>를 매달 게시함. 각 <a href="/wiki/ICANN" title="ICANN">ICANN</a> 공인 등록 대행자들에 등록된 도메인의 숫자가 나열되어 있다.</li>
<li>(<a href="/wiki/ICANN" title="ICANN">ICANN</a> 데이터에 기반하기 때문에 3개월의 시차가 있음) <a class="external text" href="http://www.dotandco.net" rel="nofollow">Dotandco.net</a>는 매월 <a class="external text" href="http://www.dotandco.net/ressources/icann_registrars/details/position.en" rel="nofollow">등록 대행자의 규모에 따른 목록</a>]을 발행.</li>
<li>Name Intelligence, Inc.가 운영하는 <a class="external text" href="http://www.domaintools.com/" rel="nofollow">DomainTools.com</a>은 <a class="external text" href="http://www.domaintools.com/internet-statistics/registrar-stats.php" rel="nofollow">등록 대행자 통계</a>를 매년 발행. (2002~2007년 자료만 있음.)</li>
</ul>
