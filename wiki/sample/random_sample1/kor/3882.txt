명령 줄 인터페이스, 디렉터리 구조, 버전 정보를 보여주고 있는 <a href="/wiki/%ED%94%84%EB%A6%AC%EB%8F%84%EC%8A%A4" title="프리도스">프리도스</a>의 스크린샷.</div>

<p><b>도스</b>(disk operating system<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup>, <b>DOS</b>)는 디스크를 기반으로 한 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0" title="컴퓨터">컴퓨터</a>에서 쓰이는 <a href="/wiki/%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C" title="운영 체제">운영 체제</a>로서 디스크에 읽고 쓰기 등의 명령을 수행하는 프로그램이다. 명령어를 직접 치는 <a href="/wiki/%EB%AA%85%EB%A0%B9_%EC%A4%84_%EC%9D%B8%ED%84%B0%ED%8E%98%EC%9D%B4%EC%8A%A4" title="명령 줄 인터페이스">명령 줄 기반</a>이다.</p>
<p>1981년부터 1995년까지, 또 부분적으로 MS-DOS 기반인 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84%EC%9A%B0" title="마이크로소프트 윈도우">마이크로소프트 윈도우</a>(<a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_95" title="윈도우 95">95</a>, <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_98" title="윈도우 98">98</a>, <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_%EB%AF%B8" title="윈도우 미">미</a>)를 포함한 2000년까지는 <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>가 <a href="/wiki/IBM_PC_%ED%98%B8%ED%99%98%EA%B8%B0%EC%A2%85" title="IBM PC 호환기종">IBM PC 호환기종</a> 시장을 장악하였다. 도스는 <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>, <a class="mw-redirect" href="/wiki/PC_DOS" title="PC DOS">PC DOS</a>, <a href="/wiki/DR-DOS" title="DR-DOS">DR-DOS</a>, <a href="/wiki/%ED%94%84%EB%A6%AC%EB%8F%84%EC%8A%A4" title="프리도스">프리도스</a>, <a class="new" href="/w/index.php?title=ROM-DOS&amp;action=edit&amp;redlink=1" title="ROM-DOS (없는 문서)">ROM-DOS</a>, <a class="new" href="/w/index.php?title=PST-DOS&amp;action=edit&amp;redlink=1" title="PST-DOS (없는 문서)">PST-DOS</a>를 포함한 비슷한 명령 줄 시스템의 계열이다.</p>
<p>이 시스템들 중 어느 것도 간단히 "도스"라고 불리진 않았다. (1960년대의 이와 관련이 없는 <a class="new" href="/w/index.php?title=DOS/360&amp;action=edit&amp;redlink=1" title="DOS/360 (없는 문서)">IBM 메인프레임 운영 체제</a>에만 사용되었음) 이와 무관한 수많은 비<a href="/wiki/X86" title="X86">x86</a> 마이크로컴퓨터 <a class="mw-redirect" href="/wiki/%EB%94%94%EC%8A%A4%ED%81%AC_%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C" title="디스크 운영 체제">디스크 운영 체제</a>는 "도스"라는 이름을 그대로 사용하였으며 이들을 사용하는 컴퓨터에 대해 논할 때 단순히 "도스"라고 부르곤 했다. (<a class="new" href="/w/index.php?title=%EC%95%84%EB%AF%B8%EA%B0%80%EB%8F%84%EC%8A%A4&amp;action=edit&amp;redlink=1" title="아미가도스 (없는 문서)">아미가도스</a>, <a class="new" href="/w/index.php?title=AMSDOS&amp;action=edit&amp;redlink=1" title="AMSDOS (없는 문서)">AMSDOS</a>, <a class="new" href="/w/index.php?title=%EC%95%A0%ED%94%8C_%EB%8F%84%EC%8A%A4&amp;action=edit&amp;redlink=1" title="애플 도스 (없는 문서)">애플 도스</a>, <a class="new" href="/w/index.php?title=%EC%95%84%ED%83%80%EB%A6%AC_%EB%8F%84%EC%8A%A4&amp;action=edit&amp;redlink=1" title="아타리 도스 (없는 문서)">아타리 도스</a>, <a class="new" href="/w/index.php?title=%EC%BD%94%EB%AA%A8%EB%8F%84%EC%96%B4_%EB%8F%84%EC%8A%A4&amp;action=edit&amp;redlink=1" title="코모도어 도스 (없는 문서)">코모도어 도스</a>, <a class="new" href="/w/index.php?title=CSI-DOS&amp;action=edit&amp;redlink=1" title="CSI-DOS (없는 문서)">CSI-DOS</a>, <a class="new" href="/w/index.php?title=%ED%94%84%EB%A1%9C%EB%8F%84%EC%8A%A4&amp;action=edit&amp;redlink=1" title="프로도스 (없는 문서)">프로도스</a>, <a class="new" href="/w/index.php?title=TRSDOS&amp;action=edit&amp;redlink=1" title="TRSDOS (없는 문서)">TRSDOS</a>)</p>

<h2><span class="mw-headline" id=".EC.97.AD.EC.82.AC">역사</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=1" title="부분 편집: 역사">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id=".EC.B4.88.EA.B8.B0.EC.9D.98_.EB.8F.84.EC.8A.A4">초기의 도스</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=2" title="부분 편집: 초기의 도스">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/%EB%94%94%EC%A7%80%ED%84%B8_%EB%A6%AC%EC%84%9C%EC%B9%98" title="디지털 리서치">디지털 리서치</a>의 <a href="/wiki/CP/M" title="CP/M">CP/M</a>이 가장 대표적인 도스의 원형이다. 8비트 개인용 컴퓨터에는 CP/M이 널리 사용되었으나 <a href="/wiki/%EC%95%A0%ED%94%8C_II" title="애플 II">애플 II</a>는 독자적인 <a class="new" href="/w/index.php?title=%EC%95%A0%ED%94%8C_%EB%8F%84%EC%8A%A4&amp;action=edit&amp;redlink=1" title="애플 도스 (없는 문서)">애플 도스</a>(정식 명칭은 그냥 DOS)를, <a href="/wiki/MSX" title="MSX">MSX</a>는 <a class="new" href="/w/index.php?title=MSX-DOS&amp;action=edit&amp;redlink=1" title="MSX-DOS (없는 문서)">MSX-DOS</a>를 썼다. 이들 컴퓨터도 나중에 <a href="/wiki/CP/M" title="CP/M">CP/M</a>을 지원하지만 디스크의 포맷이 달라서 서로 바꾸어 쓸 수는 없었다. <a class="mw-redirect" href="/wiki/IBM-PC" title="IBM-PC">IBM-PC</a>에서는 이를 모방한 <a class="new" href="/w/index.php?title=QDOS&amp;action=edit&amp;redlink=1" title="QDOS (없는 문서)">QDOS</a>로부터 <a href="/wiki/PC-DOS" title="PC-DOS">PC-DOS</a>와 <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>가 나왔으며, 나중에 CP/M은 16비트 버전인 CP/M-86 바탕으로 <a href="/wiki/DR-DOS" title="DR-DOS">DR-DOS</a>로 나왔다.</p>
<p>현재는 DR-DOS의 후기작인 <a class="mw-redirect" href="/wiki/%EC%98%A4%ED%94%88%EB%8F%84%EC%8A%A4" title="오픈도스">오픈도스</a>와, <a href="/wiki/NTFS" title="NTFS">NTFS</a> 등을 지원하는 공개 도스 프로젝트인 <a href="/wiki/%ED%94%84%EB%A6%AC%EB%8F%84%EC%8A%A4" title="프리도스">프리도스</a>가 있다. <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>는 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">마이크로소프트</a>가 만든 가장 보편적으로 쓰인 도스이다. <a href="/wiki/PC-DOS" title="PC-DOS">PC-DOS</a>는 <a href="/wiki/IBM" title="IBM">IBM</a>의 도스이다. 버전 5.0까지는 사실상 MS-DOS와 같은 제품이었으나 그 이후 추가된 내용이 달라졌다. <a href="/wiki/DR-DOS" title="DR-DOS">DR-DOS</a>는 <a href="/wiki/%EB%94%94%EC%A7%80%ED%84%B8_%EB%A6%AC%EC%84%9C%EC%B9%98" title="디지털 리서치">디지털 리서치</a>사에서 만든 도스이다. 이것은 나중에 <a class="new" href="/w/index.php?title=%EB%85%B8%EB%B2%A8_%EB%8F%84%EC%8A%A4&amp;action=edit&amp;redlink=1" title="노벨 도스 (없는 문서)">노벨 도스</a>(Novell DOS)로 바뀌었다가 다시 <a class="new" href="/w/index.php?title=%EC%B9%BC%EB%8D%B0%EB%9D%BC_%EC%98%A4%ED%94%88_%EB%8F%84%EC%8A%A4&amp;action=edit&amp;redlink=1" title="칼데라 오픈 도스 (없는 문서)">칼데라 오픈 도스</a>(Caldera OpenDOS)로 바뀐다.</p>
<h3><span class="mw-headline" id=".EC.87.A0.ED.87.B4">쇠퇴</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=3" title="부분 편집: 쇠퇴">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84%EC%9A%B0%EC%9D%98_%EC%97%AD%EC%82%AC" title="마이크로소프트 윈도우의 역사">마이크로소프트 윈도우의 역사</a>입니다.</div>
<p><a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_3.1" title="윈도우 3.1">윈도우 3.1</a> 이하의 버전과 같이 초기 버전의 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84%EC%9A%B0" title="마이크로소프트 윈도우">마이크로소프트 윈도우</a>는 별도 버전의 도스 위에서 실행되었다.<sup class="reference" id="cite_ref-emulate_2-0"><a href="#cite_note-emulate-2">[2]</a></sup> 1990년대 초에 윈도우의 그래픽 셸은 도스 시스템에서 많이 이용되었다. 1995년에 <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_95" title="윈도우 95">윈도우 95</a>가 별도의 도스 라이선스를 요구하지 않은 독립형 운영 체제로 번들되었다. 윈도우 95(및 이를 따랐던 윈도우 95과 ME)는 기본 <a class="mw-redirect" href="/wiki/%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C_%EC%BB%A4%EB%84%90" title="운영 체제 커널">OS 커널</a>을 인계 받았지만 MS-DOS 구성 요소는 호환성을 위해 그대로 남겨두었다. 윈도우 미를 제외한 윈도우 95, 98과 더불어 MS-DOS 구성 요소는 윈도우를 시작하지 않고도 구동이 가능하였다.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-various-77715_4-0"><a href="#cite_note-various-77715-4">[4]</a></sup><sup class="reference" id="cite_ref-various-42623_5-0"><a href="#cite_note-various-42623-5">[5]</a></sup> <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_NT" title="윈도우 NT">윈도우 NT</a> 계열 운영 체제가 널리 사용되면서, 윈도우 사용을 위해 더 이상 도스가 필요없게 되자 PC 사용자들 다수가 도스를 직접 사용하는 일은 중단되었다.</p>
<h3><span class="mw-headline" id=".EA.B3.84.EC.86.8D.EB.90.9C_.EC.9D.B4.EC.9A.A9">계속된 이용</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=4" title="부분 편집: 계속된 이용">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2012년 기준으로, <a href="/wiki/%ED%94%84%EB%A6%AC%EB%8F%84%EC%8A%A4" title="프리도스">프리도스</a>, <a href="/wiki/DR-DOS" title="DR-DOS">DR-DOS</a>, <a class="new" href="/w/index.php?title=ROM-DOS&amp;action=edit&amp;redlink=1" title="ROM-DOS (없는 문서)">ROM-DOS</a>, <a class="new" href="/w/index.php?title=PTS-DOS&amp;action=edit&amp;redlink=1" title="PTS-DOS (없는 문서)">PTS-DOS</a>, <a class="new" href="/w/index.php?title=RxDOS&amp;action=edit&amp;redlink=1" title="RxDOS (없는 문서)">RxDOS</a>, <a class="new" href="/w/index.php?title=REAL/32&amp;action=edit&amp;redlink=1" title="REAL/32 (없는 문서)">REAL/32</a>와 같은 도스 운영 체제를 이용할 수 있다.</p>
<h4><span class="mw-headline" id=".EC.9E.84.EB.B2.A0.EB.94.94.EB.93.9C_.EC.8B.9C.EC.8A.A4.ED.85.9C">임베디드 시스템</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=5" title="부분 편집: 임베디드 시스템">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>하드웨어에 직접 접근하는 도스 구조는 임베디드 장치에서 이용할 때 매우 이상적이다. 최신 판의 DR-DOS가 현재에도 이 시장에 초점을 두고 있다.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> ROM-DOS가 <a class="new" href="/w/index.php?title=%EC%BA%90%EB%85%BC_%ED%8C%8C%EC%9B%8C%EC%83%B7_%ED%94%84%EB%A1%9C&amp;action=edit&amp;redlink=1" title="캐논 파워샷 프로 (없는 문서)">캐논 파워샷 프로</a> 70의 임베디드 시스템에 사용되었다.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></p>
<h4><span class="mw-headline" id=".EC.97.90.EB.AE.AC.EB.A0.88.EC.9D.B4.EC.85.98">에뮬레이션</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=6" title="부분 편집: 에뮬레이션">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p><a href="/wiki/%EB%A6%AC%EB%88%85%EC%8A%A4" title="리눅스">리눅스</a> 하에서 도스 프로그램들을 거의 네이티브 속도로 구동할 수 있는 <a href="/wiki/DOSEMU" title="DOSEMU">DOSEMU</a>라는 리눅스 네이티브 <a href="/wiki/%EA%B0%80%EC%83%81_%EB%A8%B8%EC%8B%A0" title="가상 머신">가상 머신</a>을 이용하여 여러 도스 사본들을 실행할 수 있다. <a href="/wiki/%EB%8F%84%EC%8A%A4%EB%B0%95%EC%8A%A4" title="도스박스">도스박스</a>를 포함하여 여러 버전의 유닉스에서 도스를 구동하기 위한 다른 수많은 <a href="/wiki/%EC%97%90%EB%AE%AC%EB%A0%88%EC%9D%B4%ED%84%B0" title="에뮬레이터">에뮬레이터</a>들이 있다.<sup class="reference" id="cite_ref-DOSBox_8-0"><a href="#cite_note-DOSBox-8">[8]</a></sup><sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></p>
<p>도스 에뮬레이터들은 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84%EC%9A%B0" title="마이크로소프트 윈도우">마이크로소프트 윈도우</a> 하에서 게임이나 다른 도스용 소프트웨어를 구동하기에 적절한 방법인데, 현대 버전은 도스와의 완전한 호환성이 결여되어 있기 때문이다. 현대의 운영 체제에서 오래된 게임을 즐기도록 고안된 <a href="/wiki/%EB%8F%84%EC%8A%A4%EB%B0%95%EC%8A%A4" title="도스박스">도스박스</a>가 대표적인 예이다.<sup class="reference" id="cite_ref-emulate_2-1"><a href="#cite_note-emulate-2">[2]</a></sup><sup class="reference" id="cite_ref-DOSBox_8-1"><a href="#cite_note-DOSBox-8">[8]</a></sup> 다른 대안으로는 <a class="new" href="/w/index.php?title=%EB%B6%84%EB%A5%98:X86_%EC%97%90%EB%AE%AC%EB%A0%88%EC%9D%B4%ED%84%B0&amp;action=edit&amp;redlink=1" title="분류:X86 에뮬레이터 (없는 문서)">PC 에뮬레이터</a> 상에서 도스 운영 체제의 사본으로 도스 응용 프로그램들을 구동하는 것이다. 이러한 접근은 오버헤드는 증가되지만 더 나은 호환성을 제공한다.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></p>
<h2><span class="mw-headline" id=".EC.84.A4.EA.B3.84">설계</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=7" title="부분 편집: 설계">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>모든 MS-DOS형 운영 체제들은 <a class="mw-redirect" href="/wiki/%EC%9D%B8%ED%85%94_x86" title="인텔 x86">인텔 x86</a> 호환 <a href="/wiki/%EC%A4%91%EC%95%99_%EC%B2%98%EB%A6%AC_%EC%9E%A5%EC%B9%98" title="중앙 처리 장치">CPU</a>를 갖춘 컴퓨터, 주로 <a href="/wiki/IBM_PC" title="IBM PC">IBM-PC</a> 및 <a href="/wiki/IBM_PC_%ED%98%B8%ED%99%98%EA%B8%B0%EC%A2%85" title="IBM PC 호환기종">호환기종</a>에서 동작한다. 특정 기계에 의존적인 MS-DOS 버전들은 수많은 비 IBM 호환 <a href="/wiki/X86" title="X86">x86</a> 기반 컴퓨터용으로 개발되었으며, 여기에는 제조업체 이름을 딴 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">마이크로소프트</a>의 배포판에서부터 비 IBM-PC 호환 하드웨어와 동작하도록 설계된 버전에 이르기까지 다양하다. 응용 프로그램들이 직접 하드웨어 접근 대신 도스 API를 사용하는 한 IBM-PC 비호환 머신들에서도 실행이 가능하다. 1985년, 디지털 리서치는 또한 모토로라 68000 CPU용 <a class="new" href="/w/index.php?title=%EC%BB%A8%EC%BB%A4%EB%9F%B0%ED%8A%B8_%EB%8F%84%EC%8A%A4_68K&amp;action=edit&amp;redlink=1" title="컨커런트 도스 68K (없는 문서)">컨커런트 도스 68K</a> 버전 및 1990년대 초에 모토로라 CPU용 <a class="new" href="/w/index.php?title=DOS/NT&amp;action=edit&amp;redlink=1" title="DOS/NT (없는 문서)">DOS/NT</a> 파생 오리지널 <a href="/wiki/%ED%94%84%EB%A6%AC%EB%8F%84%EC%8A%A4" title="프리도스">프리도스</a> 커널 <a class="new" href="/w/index.php?title=DOS-C&amp;action=edit&amp;redlink=1" title="DOS-C (없는 문서)">DOS-C</a>가 있었다. 이 시스템들이 도스 구조와 유사하긴 했지만, 응용 프로그램들은 이러한 x86이 아닌 CPU의 비호환 명령 집합으로 인해 이진 호환성이 없었다. 그러나 고급 언어로 작성된 응용 프로그램들은 쉽게 포팅이 가능했다.</p>
<p>도스는 <a href="/wiki/%EC%9E%AC%EC%A7%84%EC%9E%85%EC%84%B1" title="재진입성">재진입성이 없는</a> 기본 <a class="mw-disambig" href="/wiki/%EC%BB%A4%EB%84%90" title="커널">커널</a> 기능들을 갖춘 단일 사용자, 싱글태스킹 운영 체제이다. 즉, 한 번에 하나의 프로그램 해당 커널 기능들을 사용할 수 있으며 도스 스스로는 하나 이상의 프로그램을 한 번에 실행할 수 있게 하는 기능이 없다. 도스 커널은 문자 입출력, 파일 관리, 메모리 관리, 프로그램 적재 및 종료와 같이 <a href="/wiki/MS-DOS_API" title="MS-DOS API">프로그램을 위한 다양한 함수</a>를 제공한다.</p>
<p>도스는 기본적으로 배치 파일(<a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%99%95%EC%9E%A5%EC%9E%90" title="파일 확장자">파일 확장자</a>는 <code>.BAT</code>)을 통해 <a href="/wiki/%EC%85%B8_%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8" title="셸 스크립트">셸 스크립트</a>를 위한 기능을 제공한다. 이것들은 텍스트 파일로서, 문서 편집기를 이용해서 작성이 가능하다. 컴파일된 프로그램과 동일한 방식으로 실행되며, 배치 파일의 각 줄을 명령 단위로 실행한다. 배치 파일은 또한 여러 내부 명령을 이용할 수 있는데, 이를테면 <a class="mw-redirect" href="/wiki/GOTO" title="GOTO">GOTO</a>와 <a href="/wiki/%EC%A1%B0%EA%B1%B4%EB%AC%B8" title="조건문">조건문</a>을 들 수 있다.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> <a class="new" href="/w/index.php?title=GOSUB&amp;action=edit&amp;redlink=1" title="GOSUB (없는 문서)">GOSUB</a> 및 단순 산술은 DR-DOS COMMAND.COM 및 <a href="/wiki/4DOS" title="4DOS">4DOS</a>와 같은 일부 서드 파티 셸을 통해 지원되지만, 실질적인 형태의 프로그래밍이 지원되는 것은 아니다.</p>
<p>이 운영 체제는 문자 기반 응용 프로그램의 개발을 허용하는 API를 제공하지만 <a href="/wiki/%EA%B7%B8%EB%9E%98%ED%94%BD_%EC%B9%B4%EB%93%9C" title="그래픽 카드">그래픽 카드</a>, <a class="mw-redirect" href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%94%84%EB%A6%B0%ED%84%B0" title="컴퓨터 프린터">프린터</a>, <a href="/wiki/%EB%A7%88%EC%9A%B0%EC%8A%A4" title="마우스">마우스</a>와 같은 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%95%98%EB%93%9C%EC%9B%A8%EC%96%B4" title="컴퓨터 하드웨어">하드웨어</a> 대부분의 접근을 위한 것은 아니다. 프로그래머들이 직접 하드웨어에 접근하게 하려면 각 프로그램이 개별 하드웨어 주변기기에 맞는 자체 <a href="/wiki/%EC%9E%A5%EC%B9%98_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84" title="장치 드라이버">장치 드라이버</a>를 갖추어야 했다. 하드웨어 제조업체들은 사양을 공개하여 대중화된 응용 프로그램들을 위한 장치 드라이버들을 사용할 수 있도록 보증해야 했다.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<h3><span class="mw-headline" id=".EB.B6.80.ED.8A.B8_.EC.8B.9C.ED.80.80.EC.8A.A4">부트 시퀀스</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=8" title="부분 편집: 부트 시퀀스">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>PC 호환 컴퓨터의 <a href="/wiki/%EB%B6%80%ED%8C%85" title="부팅">부트스트랩 로더</a>(<a href="/wiki/%EB%A7%88%EC%8A%A4%ED%84%B0_%EB%B6%80%ED%8A%B8_%EB%A0%88%EC%BD%94%EB%93%9C" title="마스터 부트 레코드">MBR</a> 또는 <a href="/wiki/%EB%B6%80%ED%8A%B8_%EC%84%B9%ED%84%B0" title="부트 섹터">부트 섹터</a>)는 디스크의 첫 섹터인 <a class="new" href="/w/index.php?title=Track0&amp;action=edit&amp;redlink=1" title="Track0 (없는 문서)">Track0</a>에 위치한다. <a class="mw-redirect" href="/wiki/%EB%A1%AC_%EB%B0%94%EC%9D%B4%EC%98%A4%EC%8A%A4" title="롬 바이오스">롬 바이오스</a>는 이 섹터를 주소 0000h:7C00h의 메모리에 불러들이고 오프셋 +1FEh의 55h AAh 서명을 보통 검사한다.</li>
<li>로드된 부트 섹터가 <a href="/wiki/%EB%A7%88%EC%8A%A4%ED%84%B0_%EB%B6%80%ED%8A%B8_%EB%A0%88%EC%BD%94%EB%93%9C" title="마스터 부트 레코드">마스터 부트 레코드</a>(MBR)로 간주되면, 그 자신을 메모리의 0000h:0600h에 재할당하며<sup class="reference" id="cite_ref-dewassoc_com-master_boot_record_13-0"><a href="#cite_note-dewassoc_com-master_boot_record-13">[13]</a></sup>, 그렇지 않다면 이 단계는 무시된다.</li>
<li>0000h:7C00h에 위치하는 섹터 콘텐츠는 현재 VBR을 이룬다. VBR은 운영 체제에 특화되어 있으며 구체적인 동작이 도스 버전마다 다르기 때문에 일반적으로 다른 도스 버전끼리 교환할 수 없다.</li>
<li>VBR이 IO.SYS/IBMBIO.COM 파일의 처음 3개의 섹터만 메모리에 불러들이는 현대의 도스 버전에서는 로드된 부분에는 다른 부트 로더를 포함하며, 이는 0000h:0500h에 저장된 루트 디렉터리 정보를 이용하여 메모리의 나머지 부분을 읽어들인다.</li>
<li>도스 시스템 초기화 코드가 내장 장치 드라이버를 초기화한 다음 MS-DOS 시스템의 <a href="/wiki/MSDOS.SYS" title="MSDOS.SYS">MSDOS.SYS</a>에 위치한 도스 커널을 메모리로 불러들인다. 윈도우 9x의 경우 도스 시스템 초기화 코드와 내장 장치 드라이버와 도스 커널은 하나의 IO.SYS 파일에 병합되어 있으며 MSDOS.SYS는 텍스트 구성 파일로 사용된다.</li>
<li>그 뒤 <a href="/wiki/CONFIG.SYS" title="CONFIG.SYS">CONFIG.SYS</a> 파일을 읽어들여 구성 매개변수들을 <a href="/wiki/%EA%B5%AC%EB%AC%B8_%EB%B6%84%EC%84%9D" title="구문 분석">구문 분석</a>한다. SHELL 변수는 <a href="/wiki/COMMAND.COM" title="COMMAND.COM">COMMAND.COM</a>을 기본으로 하는 셸의 위치를 지정한다.</li>
<li>셸을 불러들여 실행한다.</li>
<li>그 뒤 시작 배치 파일 <a href="/wiki/AUTOEXEC.BAT" title="AUTOEXEC.BAT">AUTOEXEC.BAT</a>이 셸에 의해 실행된다.<sup class="reference" id="cite_ref-evergreen_edu-config-sys_14-0"><a href="#cite_note-evergreen_edu-config-sys-14">[14]</a></sup><sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup></li>
</ul>
<h3><span class="mw-headline" id=".ED.8C.8C.EC.9D.BC_.EC.8B.9C.EC.8A.A4.ED.85.9C">파일 시스템</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=9" title="부분 편집: 파일 시스템">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>도스는 <a href="/wiki/8.3_%ED%8C%8C%EC%9D%BC_%EC%9D%B4%EB%A6%84" title="8.3 파일 이름">8.3 파일 이름</a>을 지원하는 파일 시스템을 사용하며, 여기에서 파일 이름이 8자, 확장자의 경우 3자를 사용한다는 뜻이다. 도스 2를 기점으로 계층적 디렉터리가 지원된다. 또, 각 디렉터리는 8.3 형식이지만 최대 디렉터리 경로 길이는 64자인데, 이는 도스가 관리하는 내부적인 현재 디렉터리 구조(CDs) 테이블 때문이다. 드라이브 이름을 포함하여, 도스가 온전히 지원하는 최대 길이는 마지막의 널 바이트를 포함, "드라이브:\경로\파일명.확장자"를 이용하여 80자이다.</p>
<p>도스는 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%ED%95%A0%EB%8B%B9_%ED%85%8C%EC%9D%B4%EB%B8%94" title="파일 할당 테이블">파일 할당 테이블</a>(FAT) 파일 시스템을 사용한다. 본래 드라이브 당 최대 4078개의 클러스터를 지원하는 <a class="mw-redirect" href="/wiki/FAT12" title="FAT12">FAT12</a>였다. 도스 3.0에는 16비트 할당 엔트리를 사용하여 드라이브 당 최대 65517개의 클러스터를 지원하는 <a class="mw-redirect" href="/wiki/FAT16" title="FAT16">FAT16</a> 지원을 추가하였다. 도스 3.31은 32 MB 드라이브 제한을 제거하고 최대 2 GB까지 지원하는 FAT16B의 지원을 추가하였다. 마지막으로 MS-DOS 7.1(윈도우 9x의 도스 구성 요소)은 32비트 핟랑 엔트리를 사용하는 <a class="mw-redirect" href="/wiki/FAT32" title="FAT32">FAT32</a>의 지원을 추가함으로써 하드 드라이브를 최대 137 GB 이상 지원할 수 있게 되었다.</p>
<p>도스 3.1을 시작으로 파일 리다이렉터 지원이 도스에 추가되었다. 초기에 네트워킹 지원을 위해 사용되었으나 나중에는 <a href="/wiki/MSCDEX" title="MSCDEX">MSCDEX</a>를 이용한 CD-ROM 드라이브 지원에 사용되었다. IBM PC DOS 4.0 또한 IFS(installable file system)를 지원했으나 사용되지 않아 도스 5.0에서 제거되었다.</p>
<h4><span class="mw-headline" id=".EB.93.9C.EB.9D.BC.EC.9D.B4.EB.B8.8C_.EB.AA.85.EB.AA.85_.EA.B3.84.ED.9A.8D">드라이브 명명 계획</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=10" title="부분 편집: 드라이브 명명 계획">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a href="/wiki/%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B8%8C_%EB%AC%B8%EC%9E%90_%ED%95%A0%EB%8B%B9" title="드라이브 문자 할당">드라이브 문자 할당</a>입니다.</div>
<p>도스에서 드라이브들은 식별 가능한 문자들을 가리킨다. 표준화된 관습으로, "A'와 "B"는 <a href="/wiki/%ED%94%8C%EB%A1%9C%ED%94%BC_%EB%94%94%EC%8A%A4%ED%81%AC" title="플로피 디스크">플로피 드라이브</a>를 위해 할당한다. 오직 하나의 플로피 드라이브를 가진 시스템에서 도스는 이 드라이브에 두 개의 문자를 할당하며, 프로그램들이 이들을 교체할 때 사용자에게 디스크를 스왑할 것인지 묻게 된다. 플로피에서 플로피로 복사하거나 다른 곳에서 데이터를 접근하는 동안 특정 플로피로부터 프로그램을 실행하고자 할 때 이용된다. <a class="mw-redirect" href="/wiki/%ED%95%98%EB%93%9C_%EB%94%94%EC%8A%A4%ED%81%AC" title="하드 디스크">하드 드라이브</a>들은 본래 문자 "C"와 "D'로 할당되었다. 도스는 드라이브 당 하나의 활성 파티션만 지원할 수 있었다.</p>

<p>도스에는 내부 문자 장치로 점유하고 있는 까닭에 파일 이름을 사용할 수 없는 예약 장치명이 있다. 또, 이러한 제한은 여러 윈도우 버전에도 영향을 미치면서 어떠한 경우에는 충돌이나 보안 취약성을 야기하기도 한다.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup></p>
<h3><span class="mw-headline" id=".EB.A9.94.EB.AA.A8.EB.A6.AC_.EA.B4.80.EB.A6.AC">메모리 관리</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=12" title="부분 편집: 메모리 관리">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a class="new" href="/w/index.php?title=%EB%8F%84%EC%8A%A4_%EB%A9%94%EB%AA%A8%EB%A6%AC_%EA%B4%80%EB%A6%AC&amp;action=edit&amp;redlink=1" title="도스 메모리 관리 (없는 문서)">도스 메모리 관리</a>입니다.</div>
<p>도스는 원래 인텔 8086/8088 프로세서용으로 설계되었으므로 최대 1 MB의 램에만 직접 접근할 수 있었다. PC 아키텍처로 인해 최대 <a class="mw-redirect" href="/wiki/640_%ED%82%AC%EB%A1%9C%EB%B0%94%EC%9D%B4%ED%8A%B8%EC%9D%98_%EC%9E%A5%EB%B2%BD" title="640 킬로바이트의 장벽">640 KB만 사용 가능</a>하며(<a href="/wiki/%EA%B8%B0%EB%B3%B8_%EB%A9%94%EB%AA%A8%EB%A6%AC" title="기본 메모리">기본 메모리</a>) 그 중 상위 384KB가 예약된다.</p>
<h3><span class="mw-headline" id="OS.2F2.2C_.EC.9C.88.EB.8F.84.EC.9A.B0_.ED.95.98.EC.9D.98_.EB.8F.84.EC.8A.A4">OS/2, 윈도우 하의 도스</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=13" title="부분 편집: OS/2, 윈도우 하의 도스">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a href="/wiki/%EA%B0%80%EC%83%81_%EB%8F%84%EC%8A%A4_%EB%A8%B8%EC%8B%A0" title="가상 도스 머신">가상 도스 머신</a>입니다.</div>
<h2><span class="mw-headline" id=".EC.82.AC.EC.9A.A9.EC.9E.90_.EC.9D.B8.ED.84.B0.ED.8E.98.EC.9D.B4.EC.8A.A4">사용자 인터페이스</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=14" title="부분 편집: 사용자 인터페이스">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="rellink boilerplate seealso"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> <a href="/wiki/%EB%8F%84%EC%8A%A4_%EB%AA%85%EB%A0%B9%EC%96%B4_%EB%AA%A9%EB%A1%9D" title="도스 명령어 목록">도스 명령어 목록</a> 문서를 참고하십시오.</div>
<p>도스 시스템들은 <a href="/wiki/%EB%AA%85%EB%A0%B9_%EC%A4%84_%EC%9D%B8%ED%84%B0%ED%8E%98%EC%9D%B4%EC%8A%A4" title="명령 줄 인터페이스">명령 줄 인터페이스</a>를 이용한다. 프로그램들은 명령 프롬프트에서 파일 이름을 입력하여 시작할 수 있다. 도스 운영 체제들은 시스템 유틸리티와 같은 몇 가지 프로그램들을 포함하고 있으며, 이러한 프로그램들과 관련되지 않는 내부 명령어라 불리는 추가 프로그램들도 제공한다.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<p>사용자에 더욱 친숙한 환경을 제공하려는 시도 속에, 수많은 소프트웨어 제조업체들은 메뉴나 아이콘 기반 인터페이스를 제공하는 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%EA%B4%80%EB%A6%AC%EC%9E%90" title="파일 관리자">파일 관리 프로그램</a>들을 작성하였다. 마이크로소프트 윈도우가 잘 알려진 예인데, <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_9x" title="윈도우 9x">마이크로소프트 윈도우 9x</a>가 프로그램 로더를 기본 내장하면서 가장 흔히 쓰이는 PC 호환 프로그램 로더로서 도스를 대체하였다. <a href="/wiki/%ED%85%8D%EC%8A%A4%ED%8A%B8_%EC%82%AC%EC%9A%A9%EC%9E%90_%EC%9D%B8%ED%84%B0%ED%8E%98%EC%9D%B4%EC%8A%A4" title="텍스트 사용자 인터페이스">텍스트 사용자 인터페이스</a> 프로그램들에는 <a href="/wiki/Mdir" title="Mdir">Mdir</a>, <a class="new" href="/w/index.php?title=%EB%85%B8%ED%84%B4_%EC%BB%A4%EB%A7%A8%EB%8D%94&amp;action=edit&amp;redlink=1" title="노턴 커맨더 (없는 문서)">노턴 커맨더</a>, <a class="new" href="/w/index.php?title=%EB%8F%84%EC%8A%A4_%EB%82%B4%EB%B9%84%EA%B2%8C%EC%9D%B4%ED%84%B0&amp;action=edit&amp;redlink=1" title="도스 내비게이터 (없는 문서)">도스 내비게이터</a>, <a class="new" href="/w/index.php?title=%EB%B3%BC%EC%BD%94%ED%94%84_%EC%BB%A4%EB%A7%A8%EB%8D%94&amp;action=edit&amp;redlink=1" title="볼코프 커맨더 (없는 문서)">볼코프 커맨더</a>, 쿼터데스크 <a class="new" href="/w/index.php?title=DESQview&amp;action=edit&amp;redlink=1" title="DESQview (없는 문서)">DESQview</a>, <a class="new" href="/w/index.php?title=%EB%B3%BC%EB%9E%9C%EB%93%9C_%EC%82%AC%EC%9D%B4%EB%93%9C%ED%82%A5&amp;action=edit&amp;redlink=1" title="볼랜드 사이드킥 (없는 문서)">사이드킥</a>이 있다. <a href="/wiki/%EA%B7%B8%EB%9E%98%ED%94%BD_%EC%82%AC%EC%9A%A9%EC%9E%90_%EC%9D%B8%ED%84%B0%ED%8E%98%EC%9D%B4%EC%8A%A4" title="그래픽 사용자 인터페이스">그래픽 사용자 인터페이스</a> 프로그램에는 디지털 리서치의 <a class="new" href="/w/index.php?title=%EA%B7%B8%EB%9E%98%ED%94%BD_%EC%9D%B8%EB%B0%94%EC%9D%B4%EB%9F%B0%EB%A8%BC%ED%8A%B8_%EB%A7%A4%EB%8B%88%EC%A0%80&amp;action=edit&amp;redlink=1" title="그래픽 인바이런먼트 매니저 (없는 문서)">그래픽 인바이런먼트 매니저</a>(본래 CP/M용으로 작성됨)와 <a class="mw-disambig" href="/wiki/GEOS" title="GEOS">GEOS</a>가 있다.</p>
<p>마침내, 주요 도스 시스템 제조업체들이 자신들만의 환경 관리자를 포함하기 시작하였다. MS-DOS/IBM DOS 4는 <a href="/wiki/%EB%8F%84%EC%8A%A4_%EC%85%B8" title="도스 셸">도스 셸</a>을 포함하였다.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> 이듬해에 출시된 DR DOS 5.0은 GEM 기반의 <a class="new" href="/w/index.php?title=%EB%B7%B0%EB%A7%A5%EC%8A%A4&amp;action=edit&amp;redlink=1" title="뷰맥스 (없는 문서)">뷰맥스</a>를 포함하였다.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup></p>
<h3><span class="mw-headline" id=".EC.A2.85.EB.A3.8C_.ED.9B.84_.EC.83.81.EC.A3.BC_.ED.94.84.EB.A1.9C.EA.B7.B8.EB.9E.A8">종료 후 상주 프로그램</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%8F%84%EC%8A%A4&amp;action=edit&amp;section=15" title="부분 편집: 종료 후 상주 프로그램">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a href="/wiki/%EC%A2%85%EB%A3%8C_%ED%9B%84_%EC%83%81%EC%A3%BC_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%A8" title="종료 후 상주 프로그램">종료 후 상주 프로그램</a>입니다.</div>
<p>도스는 멀티태스킹 운영 체제가 아니다. 그러나 도스는 종료 후 상주 프로그램(TSR)을 제공하여 여러 프로그램들이 메모리에 상주할 수 있게 도와준다.</p>
<p>이러한 프로그램들은 시스템 타이머나 키보드 인터럽트를 후킹하여 이들이 백그라운드에서 작업을 수행하게끔 하거나 현재 실행 중인 프로그램을 적시에 호출할 수 있게 한다. PRINT 명령어는 이를 수행하여 백그라운드 인쇄 스풀링을 구현하였다. <a class="new" href="/w/index.php?title=%EB%B3%BC%EB%9E%9C%EB%93%9C_%EC%82%AC%EC%9D%B4%EB%93%9C%ED%82%A5&amp;action=edit&amp;redlink=1" title="볼랜드 사이드킥 (없는 문서)">볼랜드 사이드킥</a>도 이 기법을 사용한다.</p>
<p>종료 후 상주 프로그램들은 기본적으로 이용할 수 없는 기능을 제공하는 데에도 사용되었다. CED와 <a class="new" href="/w/index.php?title=%EB%8F%84%EC%8A%A4%ED%82%A4&amp;action=edit&amp;redlink=1" title="도스키 (없는 문서)">도스키</a>와 같은 프로그램들은 COMMAND.COM에서 사용할 수 있는 기능 이상으로 명령 줄 편집 기능을 제공하였다. 마이크로소프트 CD-ROM 확장 (MSCDEX)과 같은 프로그램들은 CD-ROM 디스크의 파일을 접근할 수 있게 하였다.</p>
<p>일부 TSR들은 초기 수준의 태스크 전환을 수행할 수 있었다. 이를테면, <a href="/wiki/%EC%85%B0%EC%96%B4%EC%9B%A8%EC%96%B4" title="셰어웨어">셰어웨어</a> 프로그램 백 앤드 포스 (1990년)<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup>은 단축키를 이용하여 현재 실행 중인 프로그램의 상태를 디스크로 저장하고, 다른 프로그램을 불러와 전환할 수 있었다.</p>

<li><b><a href="/wiki/%EC%95%84%EB%9D%BC%ED%81%AC%EB%84%A4_(%EC%9B%B9_%EB%B8%8C%EB%9D%BC%EC%9A%B0%EC%A0%80)" title="아라크네 (웹 브라우저)">아라크네</a></b>: 16비트 그래픽 도스 <a href="/wiki/%EC%9B%B9_%EB%B8%8C%EB%9D%BC%EC%9A%B0%EC%A0%80" title="웹 브라우저">웹 브라우저</a>.</li>
