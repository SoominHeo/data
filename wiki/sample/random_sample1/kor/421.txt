<p><b>EPUB</b>(electronic publication)은 <a class="new" href="/w/index.php?title=%EA%B5%AD%EC%A0%9C_%EB%94%94%EC%A7%80%ED%84%B8_%EC%B6%9C%ED%8C%90_%ED%8F%AC%EB%9F%BC&amp;action=edit&amp;redlink=1" title="국제 디지털 출판 포럼 (없는 문서)">국제 디지털 출판 포럼</a>(<a class="new" href="/w/index.php?title=IDPF&amp;action=edit&amp;redlink=1" title="IDPF (없는 문서)">IDPF</a>, International Digital Publishing Forum)에서 제정한 개방형 자유 전자서적 표준이다. EPUB은 자동공간조정(reflowable)이 가능하게 끔 디자인 되었다. 이는 EPUB으로 만들어진 내용을 볼 때 디스플레이하는 기계의 형식, 크기에 자동으로 최적화되어 보여줄 수 있다는 뜻이다. EPUB는 2007년 9월 이전에 있던 오픈 eBook 표준을 대체하기 위해 국제 디지털 출판 포럼에서 공식 표준으로 채택되었다.</p>

<h2><span class="mw-headline" id=".EA.B8.B0.EB.8A.A5">기능</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=1" title="부분 편집: 기능">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>자유 소프트웨어 개발이 가능한 개방형 표준</li>
<li>자동공간조정과 글자크기 변환</li>
<li>내부에 래스터나 백터 이미지를 담을 수 있음</li>
<li><a href="/wiki/%EB%A9%94%ED%83%80%EB%8D%B0%EC%9D%B4%ED%84%B0" title="메타데이터">메타데이터</a> 포함</li>
<li><a class="mw-disambig" href="/wiki/DRM" title="DRM">DRM</a> 지원</li>
<li><a class="mw-redirect" href="/wiki/CSS" title="CSS">CSS</a> 지원</li>
<li>같은 파일에 대해 대체적인 렌더링 지원</li>
<li>기능 확장을 위한 내외부적 <a href="/wiki/XML" title="XML">XML</a>기능 지원</li>
</ul>
<h2><span class="mw-headline" id=".ED.8C.8C.EC.9D.BC_.ED.98.95.EC.8B.9D">파일 형식</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=2" title="부분 편집: 파일 형식">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>EPUB은 세 가지 정의로 이루어져 있다.</p>
<ul>
<li>Open Publication Structure 2.0, 이는 내용의 형태를 정의한다.</li>
<li>Open Packaging Format (OPF) 2.0, 이는 XML로 구성된 .epub의 파일 구조를 정의한다.</li>
<li>OEBPS Container Format(OCF) 1.0, 모든 파일들을 ZIP으로 압축 저장한다.</li>
</ul>
<p>기본적으로, EPUB은 내부적으로 XHTML이나 DTBook(DAISY 컨소시엄에서 만든 XML 표준.)를 이용하여 내부의 글과 문서 구조를 만들고, CSS의 일부를 이용해 문서의 틀과 형식을 만든다. 목록이나 표 양식, EPUB 메타데이터를 위해 XML을 사용한다. zip 파일을 통해 상기된 다양한 파일들을 하나로 묶어 배포한다.</p>
<h3><span class="mw-headline" id=".EA.B0.9C.EB.B0.A9.ED.98.95_.EC.B6.9C.ED.8C.90_.ED.98.95.EC.8B.9D.28Open_Publication_Structure.29_2.0">개방형 출판 형식(Open Publication Structure) 2.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=3" title="부분 편집: 개방형 출판 형식(Open Publication Structure) 2.0">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>EPUB은 내용을 구성하기위해 이전 버전에서는 XHTML에서 파생된 OEBPS 1.2를 사용했었으나 2.0버전에서는 XHTML 1.1(이나 DTBook)을 사용한다. 그러나 XHTML 구성요소(element)에 대한 제한이 좀 있다. EPUB의 XHTML에서 사용하는 mimetype은 application/xhtml+xml이다. XHTML 모듈과 제한에 대한 자세한 설명은 2.2<sup class="reference" id="cite_ref-ops2.0_spec_1-0"><a href="#cite_note-ops2.0_spec-1">[1]</a></sup>를 참고 하면 된다.</p>
<p>스타일과 레이아웃은 "OPS 스타일 시트"라 불리는 CSS 2.0의 일부를 사용한다. 이 특별한 구문들은 CSS 중 읽기시스템과 몇가지 특수한 부분만을 사용한다. 특수한 부분이란 <code>oeb-page-head, oeb-page-foot,</code>나, <code>oeb-column-number</code>등과 같은 것들이다. 폰트는 <code>@font-face</code>항목이나 OPF 메니페스트 속에 폰트를 포함시켜 설정할 수 있다. <a class="new" href="/w/index.php?title=Internet_media_type&amp;action=edit&amp;redlink=1" title="Internet media type (없는 문서)">mimetype</a>을 위한 EPUB의 CSS 문서형식은 <code>text/css</code> 이다. <sup class="reference" id="cite_ref-ops2.0_spec_1-1"><a href="#cite_note-ops2.0_spec-1">[1]</a></sup> 지원항목과 그에 대한 세부정보는 <a class="external text" href="http://www.idpf.org/2007/ops/OPS_2.0_final_spec.html#Section3.0" rel="nofollow">Section 3.0</a>에서 찾아볼 수 있다.</p>
<p>EPUB에서 이미지는 <a href="/wiki/PNG" title="PNG">PNG</a>, <a href="/wiki/JPEG" title="JPEG">JPEG</a>, <a href="/wiki/GIF" title="GIF">GIF</a>, <a class="mw-redirect" href="/wiki/SVG" title="SVG">SVG</a> 형식을 지원한다. 이들은 각각 <a class="new" href="/w/index.php?title=%EC%9D%B8%ED%84%B0%EB%84%B7_%EB%AF%B8%EB%94%94%EC%96%B4_%ED%83%80%EC%9E%85&amp;action=edit&amp;redlink=1" title="인터넷 미디어 타입 (없는 문서)">mimetype</a>의 <code>image/png, image/jpeg, image/gif, image/svg+xml</code>를 사용한다. 다른 미디어 형식도 허용되나, 제작자가 지원 형식으로 된 대체물을 지정해 두어야 한다.<sup class="reference" id="cite_ref-ops2.0_spec_1-2"><a href="#cite_note-ops2.0_spec-1">[1]</a></sup> mimetypes을 위한 표는 Section 1.3.7의 세부사항<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>을 참고하면 된다.</p>
<p>국제적이며 다중언어 서적을 위해 <a class="mw-redirect" href="/wiki/Unicode" title="Unicode">Unicode</a> 가 사용되며, 제작자는 <a href="/wiki/UTF-8" title="UTF-8">UTF-8</a> or <a href="/wiki/UTF-16" title="UTF-16">UTF-16</a> 인코딩을 사용해야 한다. 하지만, 읽기 시스템에서 유니코드의 모든 유니코드 문자를 위한 폰트를 제공해야 하는 것은 아니지만, 표시할 수 없는 문자를 위한 표시는 해주어야 한다.<sup class="reference" id="cite_ref-ops2.0_spec_1-3"><a href="#cite_note-ops2.0_spec-1">[1]</a></sup></p>
<p>EPUB을 위한 XHTML의 기본 골격은 아래와 같다.</p>

<h2><span class="mw-headline" id=".EC.86.8C.ED.94.84.ED.8A.B8.EC.9B.A8.EC.96.B4">소프트웨어</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=4" title="부분 편집: 소프트웨어">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id=".EC.9D.BD.EA.B8.B0_.EC.A7.80.EC.9B.90">읽기 지원</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=EPUB&amp;action=edit&amp;section=5" title="부분 편집: 읽기 지원">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>EPUB를 읽는 것을 지원하는 장치는 OPS 출판을 지원하는 하드웨어와 소프트웨어의 조합이다.</p>
<table border="1" class="wikitable sortable">
<caption>Reading Systems and Software</caption>
