<p><b>그래픽 인터체인지 포맷</b>(<span lang="en" xml:lang="en">Graphics Interchange Format; GIF</span>)는 <a href="/wiki/%EB%B9%84%ED%8A%B8%EB%A7%B5" title="비트맵">비트맵</a> <a class="mw-redirect" href="/wiki/%EA%B7%B8%EB%9E%98%ED%94%BD_%ED%8C%8C%EC%9D%BC_%ED%8F%AC%EB%A7%B7" title="그래픽 파일 포맷">그래픽 파일 포맷</a>이다. <a href="/wiki/1987%EB%85%84" title="1987년">1987년</a> <a class="new" href="/w/index.php?title=%EC%BB%B4%ED%93%A8%EC%84%9C%EB%B8%8C&amp;action=edit&amp;redlink=1" title="컴퓨서브 (없는 문서)">컴퓨서브</a>가 발표하였으며, <a href="/wiki/%EC%9B%94%EB%93%9C_%EC%99%80%EC%9D%B4%EB%93%9C_%EC%9B%B9" title="월드 와이드 웹">월드 와이드 웹</a>에서 가장 널리 쓰이는 파일 포맷이기도 하다. 특별한 플러그인을 요구하지 않고 여러 환경에서 쉽게 쓸 수 있는 까닭에 다중 프레임 애니메이션을 이용한 배너 광고 등에 널리 쓰였으나, 수많은 웹사이트에서는 GIF 대신 어도비 플래시를 사용하기도 한다.</p>
<p>최대 256 색까지 저장할 수 있는 <a href="/wiki/%EB%B9%84%EC%86%90%EC%8B%A4_%EC%95%95%EC%B6%95" title="비손실 압축">비손실 압축</a> 형식이다. GIF에 쓰인 <a href="/wiki/LZW" title="LZW">LZW</a> 알고리즘에 대한 <a href="/wiki/%ED%8A%B9%ED%97%88" title="특허">특허</a>를 <a href="/wiki/%EC%9C%A0%EB%8B%88%EC%8B%9C%EC%8A%A4" title="유니시스">유니시스</a> 사가 가지고 있다는 것이 알려지고, 또한 256 색의 제한에 한계를 느끼면서 <a href="/wiki/PNG" title="PNG">PNG</a>라는 새로운 표준이 개발되었다.</p>
<p>GIF는 지프(<span class="IPA" title="IPA 발음">/dʒɪf/</span>)나 기프(<span class="IPA" title="IPA 발음">/ɡɪf/</span>)로 읽는다. GIF 포맷의 저자는 <b>지프</b>로 읽는다고 밝혔으나<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup>, 기프라는 발음도 계속 쓰이고 있다.</p>

<h2><span class="mw-headline" id=".EC.9C.A0.EB.8B.88.EC.8B.9C.EC.8A.A4.EC.99.80_LZW_.ED.8A.B9.ED.97.88">유니시스와 LZW 특허</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GIF&amp;action=edit&amp;section=1" title="부분 편집: 유니시스와 LZW 특허">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>GIF에 사용되는 <a href="/wiki/LZW" title="LZW">LZW</a>알고리즘은 유니시스 사에 <a class="external text" href="http://patft.uspto.gov/netacgi/nph-Parser?patentnumber=4,558,302" rel="nofollow">미국 특허 4,558,302</a>로 등록되어 있다. 컴퓨서브가 처음 GIF 포맷을 공개했을 때는 LZW 알고리즘에 특허가 있는 줄 몰랐다. <a href="/wiki/1994%EB%85%84" title="1994년">1994년</a> 12월, 유니시스는 이 특허에 대해 특허료를 받는다고 공고했다. <a href="/wiki/1999%EB%85%84" title="1999년">1999년</a> 8월에는 무료 소프트웨어와 그 사용자에게도 특허료를 받는다고 하여 많은 반발을 불러일으켰다.</p>
<p><a href="/wiki/2003%EB%85%84" title="2003년">2003년</a> <a href="/wiki/6%EC%9B%94_20%EC%9D%BC" title="6월 20일">6월 20일</a>, LZW 알고리즘에 대한 유니시스의 특허가 만료되었으며, 유럽, 일본, 캐나다에서의 특허는 <a href="/wiki/2004%EB%85%84" title="2004년">2004년</a> <a href="/wiki/6%EC%9B%94_18%EC%9D%BC" title="6월 18일">6월 18일</a>, <a href="/wiki/6%EC%9B%94_20%EC%9D%BC" title="6월 20일">6월 20일</a>, <a href="/wiki/7%EC%9B%94_7%EC%9D%BC" title="7월 7일">7월 7일</a> 차례대로 만료되었다<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>.</p>
<p>IBM 역시 LZW 알고리즘에 대한 특허를 가지고 있으나 이 특허에 대한 권리를 주장하지는 않았다. <a href="/wiki/%EC%9E%90%EC%9C%A0_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4_%EC%9E%AC%EB%8B%A8" title="자유 소프트웨어 재단">자유 소프트웨어 재단</a>에 따르면 미국에서의 특허는 <a href="/wiki/2006%EB%85%84" title="2006년">2006년</a> <a href="/wiki/8%EC%9B%94_11%EC%9D%BC" title="8월 11일">8월 11일</a> 만료되었다고 한다.</p>
<h2><span class="mw-headline" id=".EB.8F.99.ED.99.94.EC.83.81_GIF">동화상 GIF</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=GIF&amp;action=edit&amp;section=2" title="부분 편집: 동화상 GIF">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>GIF에서는 사용자가 새로운 블록을 정의할 수 있다. 1990년대에 <a href="/wiki/%EB%84%B7%EC%8A%A4%EC%BC%80%EC%9D%B4%ED%94%84" title="넷스케이프">넷스케이프</a>는 넷스케이프 애플리케이션 블록을 설계하였으며<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> GIF 파일이 정적인 그림 대신 움직이는 그림을 가리키게 하였다. 이러한 애니메이션은 <a href="/wiki/%EB%84%B7%EC%8A%A4%EC%BC%80%EC%9D%B4%ED%94%84_%EB%82%B4%EB%B9%84%EA%B2%8C%EC%9D%B4%ED%84%B0" title="넷스케이프 내비게이터">넷스케이프 내비게이터</a> 버전 2.0에 처음 등장하였으며 그 뒤에 다른 브라우저로 퍼져나갔다.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
<p>동화상 GIF는 여러 장의 그림이나 프레임을 이루어 연속으로 보이게 만들며, 각 그림은 GCE (그래픽 제어 확장) 기능을 통해 그려진다. 그에 이어 기본적으로 모든 프레임에 적용되는 헤더가 뒤따르며 헤더 뒤에는 데이터가 고정 색인들에 위치하지 않은 스트림 지향이 되므로 GCE 시작 위치는 선행하는 GCE의 길이에 따라 달라지게 된다. GCE 안에서 LZE 코드 그림 데이터는 각기 최대 255 바이트 안에 정렬된다. 블록의 크기는 이를 선행하는 바이트에 의해 정의된다. 이를테면 아래는 애니메이션 <i><a href="/wiki/%ED%8C%8C%EC%9D%BC:Rotating_earth_(large).gif" title="파일:Rotating earth (large).gif">Rotating earth (large).gif</a></i>의 구조가 나열되어 있다.</p>

<p><a href="/wiki/%EC%9D%B8%ED%84%B0%EB%84%B7_%EC%9D%B5%EC%8A%A4%ED%94%8C%EB%A1%9C%EB%9F%AC" title="인터넷 익스플로러">인터넷 익스플로러</a>는 프레임레이트가 초당 20 프레임 이상인 경우 GIF 재생 속도를 떨어트리는데, 마이크로소프트는 <a href="/wiki/%EA%B5%AC%EA%B8%80_%ED%81%AC%EB%A1%AC" title="구글 크롬">구글 크롬</a>과 <a href="/wiki/%EC%82%AC%ED%8C%8C%EB%A6%AC_(%EC%9B%B9_%EB%B8%8C%EB%9D%BC%EC%9A%B0%EC%A0%80)" title="사파리 (웹 브라우저)">사파리</a> 또한 일부 GIF 애니메이션의 속도를 떨어트린다고 밝혔다.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
