<h1 class="firstHeading" id="firstHeading" lang="ko">Raw 이미지 포맷</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">위키백과, 우리 모두의 백과사전.</div>
<div id="contentSub"><span class="mw-redirectedfrom">(<a class="mw-redirect" href="/w/index.php?title=.cap&amp;redirect=no" title=".cap">.cap</a>에서 넘어옴)</span></div>
<div class="mw-jump" id="jump-to-nav">
					이동:					<a href="#mw-head">둘러보기</a>, 					<a href="#p-search">검색</a>

<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><b>Raw 이미지 파일</b>은 이미지 파일 포맷 중 하나로 <a href="/wiki/%EB%94%94%EC%A7%80%ED%84%B8_%EC%B9%B4%EB%A9%94%EB%9D%BC" title="디지털 카메라">디지털 카메라</a>나 <a href="/wiki/%EC%9D%B4%EB%AF%B8%EC%A7%80_%EC%8A%A4%EC%BA%90%EB%84%88" title="이미지 스캐너">이미지 스캐너</a>의 이미지 센서로부터 최소한으로 처리한 데이터를 포함하고 있다. Raw 파일들은 영어 낱말의 뜻 그대로 전혀 가공되지 않은 상태이며 <a class="new" href="/w/index.php?title=%EB%9E%98%EC%8A%A4%ED%84%B0_%EA%B7%B8%EB%9E%98%ED%94%BD%EC%8A%A4_%EC%97%90%EB%94%94%ED%84%B0&amp;action=edit&amp;redlink=1" title="래스터 그래픽스 에디터 (없는 문서)">비트맵 그래픽스 에디터</a>를 가지고 사용하거나 <a href="/wiki/%EC%9D%B8%EC%87%84" title="인쇄">인쇄</a>용으로 사용할 준비가 되어 있지 않다는 뜻이 된다. <a class="mw-redirect" href="/wiki/%ED%99%94%EC%9D%B4%ED%8A%B8%EB%B0%B8%EB%9F%B0%EC%8A%A4" title="화이트밸런스">화이트밸런스</a>, <a class="mw-redirect" href="/wiki/%ED%8C%8C%EB%9D%BC%EB%AF%B8%ED%84%B0" title="파라미터">파라미터</a>, <a class="mw-redirect" href="/wiki/%EC%83%89%EA%B3%B5%EA%B0%84" title="색공간">색공간</a> 등이 정해져 있지 않고 촬상매체에 감지된 빛의 세기에 대한 정보만을 가지고 있다.</p>
<p>Raw라는 표현 대신 RAW라는 표현이 쓰이기도 하는데 이는 단일 파일 확장자 .RAW를 대표한다는 잘못된 믿음에서 비롯된다. 실제로 Raw 이미지 포맷의 확장자는 업체나 디지털 카메라 제조사에 따라 여러 가지가 쓰인다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> 자세한 것은 아래의 확장자를 참고.</p>

<h2><span class="mw-headline" id=".EC.9D.B4.EC.A0.90">이점</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_%EC%9D%B4%EB%AF%B8%EC%A7%80_%ED%8F%AC%EB%A7%B7&amp;action=edit&amp;section=2" title="부분 편집: 이점">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>거의 모든 디지털 카메라는 <a class="mw-redirect" href="/wiki/CCD" title="CCD">이미지 센서</a>로부터 이미지를 JPEG 파일로 변환할 때 사진을 찍기 전 사진사가 지정해 두었거나 자동으로 지정된 <a class="mw-redirect" href="/wiki/%ED%99%94%EC%9D%B4%ED%8A%B8%EB%B0%B8%EB%9F%B0%EC%8A%A4" title="화이트밸런스">화이트밸런스</a>, <a class="mw-redirect" href="/wiki/%ED%8C%8C%EB%9D%BC%EB%AF%B8%ED%84%B0" title="파라미터">파라미터</a>, <a class="mw-redirect" href="/wiki/%EC%83%89%EA%B3%B5%EA%B0%84" title="색공간">색공간</a> 등의 설정을 사용하여 처리할 수 있다. raw 파일을 지원하는 카메라들은 파일에 이러한 설정값들을 저장해 두고 있지만 처리 시간을 지연시킨다. 사진사가 별도의 과정을 거쳐야 하는 결과를 낳게 되므로 raw는 별도의 컴퓨터 처리를 한다고 가정할 때에만 쓰이는 것이 보통이다. 그러나 raw는 JPEG보다 수많은 이점을 가지고 있다.:</p>
<ul>
<li>더 높은 화질.</li>
<li>JPEG는 손실 압축 포맷이다. Raw 포맷은 압축되지 않거나 무손실 압축을 사용하므로 이미지는 언제나 원본 그대로를 가지고 있다.</li>
<li>더 섬세한 제어. RAW 변환 소프트웨어를 사용하면 더 많은 변수를 이용할 수 있다.(밝기, 화이트 밸런스, 색조, 채도 등)</li>
<li>카메라의 raw 파일들은 12비트나 14비트의 명암 정보를 가지고 있으며, 가공된 <a href="/wiki/TIFF" title="TIFF">TIFF</a>나 <a href="/wiki/JPEG" title="JPEG">JPEG</a> 파일에 저장되어 있는 <a href="/wiki/%EA%B0%90%EB%A7%88_%EB%B3%B4%EC%A0%95" title="감마 보정">감마 압축</a> 8비트를 사용하지 않는다. 그러므로 데이터는 그림자, 밝은 부분, 채도가 깊은 색에 더 정확성을 제공한다.</li>
<li><a href="/wiki/%EC%83%89_%EA%B3%B5%EA%B0%84" title="색 공간">색 공간</a>은 사용자가 원하는 대로 설정할 수 있다.</li>
</ul>
<h2><span class="mw-headline" id=".EB.8B.A8.EC.A0.90">단점</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Raw_%EC%9D%B4%EB%AF%B8%EC%A7%80_%ED%8F%AC%EB%A7%B7&amp;action=edit&amp;section=3" title="부분 편집: 단점">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>카메라의 raw 파일들은 일반적으로 JPEG 파일보다 2~6배 정보 용량이 더 크다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> 어떠한 raw 포맷은 압축을 전혀 하지 않으며, 어떠한 것들은 <a class="mw-redirect" href="/wiki/%EB%AC%B4%EC%86%90%EC%8B%A4_%EC%95%95%EC%B6%95" title="무손실 압축">무손실 압축</a>을 사용하여 화질에 영향을 미치지 않고 파일 크기를 줄이며 어떠한 것들은 양자화와 필터링을 이미지 데이터에 적용하는 <a href="/wiki/%EC%86%90%EC%8B%A4_%EC%95%95%EC%B6%95" title="손실 압축">손실 압축</a>을 사용한다. raw 포맷을 사용하면 JPEG 포맷의 화질 저하 문제를 피할 수 있지만 <a href="/wiki/%EB%A9%94%EB%AA%A8%EB%A6%AC_%EC%B9%B4%EB%93%9C" title="메모리 카드">메모리 카드</a>에는 보다 적은 양의 사진이 저장된다. 또한 카메라가 raw 이미지를 카드에 기록할 때 JPEG보다 시간이 많이 걸리므로 연속 촬영에서 건져낼 수 있는 사진 수는 비교적 적다.</p>
<p>카메라 제조업체 대부분은 raw 포맷을 사용하고 있지만 표준 raw 포맷이 별도로 지정된 것은 아니다. <a href="/wiki/%EC%96%B4%EB%8F%84%EB%B9%84_%EC%8B%9C%EC%8A%A4%ED%85%9C%EC%A6%88" title="어도비 시스템즈">어도비 시스템즈</a>의 <a href="/wiki/%EB%94%94%EC%A7%80%ED%84%B8_%EB%84%A4%EA%B1%B0%ED%8B%B0%EB%B8%8C" title="디지털 네거티브">디지털 네거티브</a>(DNG) 포맷을 표준으로 인정하는 추세이지만 아직 대다수의 카메라 회사들이 이를 받아들이지 않고 있다.</p>
<p>raw 포맷은 JPEG, TIFF와 같은 표준 포맷과 견주어 활성화되지 않았기 때문에 raw 포맷을 지원하는 소프트웨어가 필요하다. 소프트웨어 개발자들은 최신 카메라의 raw 포맷 지원을 위해 제품을 지속적으로 갱신해야 한다.</p>
