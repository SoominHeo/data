<p><b>PHP</b>(<small>약칭:</small> <span lang="en" xml:lang="en"><b>Hypertext Preprocessor</b></span> <small>하이퍼텍스트 프리프로세서<sup>[<a href="/wiki/%EC%9C%84%ED%82%A4%EB%B0%B1%EA%B3%BC:%EC%98%81%EC%96%B4%EC%9D%98_%ED%95%9C%EA%B8%80_%ED%91%9C%EA%B8%B0" title="위키백과:영어의 한글 표기">*</a>]</sup></small>)는 프로그래밍 언어의 일종이다. 원래는 동적 웹 페이지를 만들기 위해 설계되었으며 이를 구현하기 위해 PHP로 작성된 코드를 HTML 소스 문서 안에 넣으면 PHP 처리 기능이 있는 웹 서버에서 해당 코드를 인식하여 작성자가 원하는 웹 페이지를 생성한다. 근래에는 PHP 코드와 HTML을 별도 파일로 분리하여 작성하는 경우가 일반적이며, PHP 또한 웹서버가 아닌 php-fpm(PHP FastCGI Process Manager)을 통해 실행하는 경우가 늘어나고 있다.</p>
<p>또한 PHP는 명령 줄 인터페이스 방식의 자체 인터프리터를 제공하여 이를 통해 범용 프로그래밍 언어로도 사용할 수 있으며 그래픽 애플리케이션을 제작할 수도 있다.</p>
<p>많은 서버 측 <a href="/wiki/%EC%98%A4%ED%94%88_%EC%86%8C%EC%8A%A4" title="오픈 소스">오픈 소스</a> 소프트웨어는 PHP로 구현되었다. PHP를 바탕으로 하는 프로그램 중 대표적인 예로 블로깅 도구 <a href="/wiki/%EC%9B%8C%EB%93%9C%ED%94%84%EB%A0%88%EC%8A%A4" title="워드프레스">워드프레스</a>와 위키백과를 구동시키는 <a href="/wiki/%EB%AF%B8%EB%94%94%EC%96%B4%EC%9C%84%ED%82%A4" title="미디어위키">미디어위키</a>를 들 수 있다. <a href="/wiki/%EB%8C%80%ED%95%9C%EB%AF%BC%EA%B5%AD" title="대한민국">대한민국</a>의 PHP 기반 <a href="/wiki/%EC%A0%84%EC%9E%90_%EA%B2%8C%EC%8B%9C%ED%8C%90" title="전자 게시판">BBS</a>는 <a href="/wiki/%EC%A0%9C%EB%A1%9C%EB%B3%B4%EB%93%9C" title="제로보드">제로보드</a>, <a href="/wiki/%EA%B7%B8%EB%88%84%EB%B3%B4%EB%93%9C" title="그누보드">그누보드</a>, <a href="/wiki/XpressEngine" title="XpressEngine">XpressEngine</a> [웹솔루션 ,랭크업]등이 있다.</p>
<p>PHP는 <a href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8" title="마이크로소프트">마이크로소프트</a>의 <a href="/wiki/ASP.NET" title="ASP.NET">ASP.NET</a>, <a href="/wiki/%EB%B9%84%EC%A3%BC%EC%96%BC_%EB%B2%A0%EC%9D%B4%EC%A7%81" title="비주얼 베이직">비주얼 베이직</a>, <a href="/wiki/%EB%A7%A4%ED%81%AC%EB%A1%9C%EB%AF%B8%EB%94%94%EC%96%B4" title="매크로미디어">매크로미디어</a>의 <a class="new" href="/w/index.php?title=%EB%A7%A4%ED%81%AC%EB%A1%9C%EB%AF%B8%EB%94%94%EC%96%B4_%EC%BD%9C%EB%93%9C%ED%93%A8%EC%A0%84&amp;action=edit&amp;redlink=1" title="매크로미디어 콜드퓨전 (없는 문서)">콜드퓨전</a>, <a href="/wiki/%EC%8D%AC_%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%8B%9C%EC%8A%A4%ED%85%9C%EC%A6%88" title="썬 마이크로시스템즈">썬 마이크로시스템즈</a>의 <a href="/wiki/%EC%9E%90%EB%B0%94_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="자바 (프로그래밍 언어)">자바</a>나 오픈 소스 커뮤니티의 <a href="/wiki/%ED%8C%8C%EC%9D%B4%EC%8D%AC" title="파이썬">파이썬</a>, <a href="/wiki/%ED%8E%84" title="펄">펄</a>, <a href="/wiki/%EB%A3%A8%EB%B9%84_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="루비 (프로그래밍 언어)">루비</a>에 대한 대안으로 생각될 수 있다.</p>
<p>PHP는 텍스트, 특히 <a href="/wiki/HTML" title="HTML">HTML</a>의 처리에 강점을 가지고 있다. <a href="/wiki/URL" title="URL">URL</a>의 <a class="mw-redirect" href="/wiki/%ED%8C%8C%EC%8B%B1" title="파싱">파싱</a>이나 폼 처리, <a href="/wiki/%EC%A0%95%EA%B7%9C_%ED%91%9C%ED%98%84%EC%8B%9D" title="정규 표현식">정규 표현식</a> 등이 그 한 예이다. 또한 다양한 데이터베이스를 지원하므로 데이터베이스와 사용자간의 다리 역할도 잘 수행한다.</p>
<p>PHP는 PHP 사용 허가서의 규정을 따라 릴리즈된 자유 소프트웨어이다. 그러나 PHP 사용 허가서는 PHP라는 단어의 사용에 제한을 두는 규정을 가지고 있기 때문에 GNU 일반 공중 사용 허가서와 호환되지 않는다.</p>

<h2><span class="mw-headline" id=".EC.97.AD.EC.82.AC">역사</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=1" title="부분 편집: 역사">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>PHP는 1995년 <a href="/wiki/%EB%9D%BC%EC%8A%A4%EB%AC%B4%EC%8A%A4_%EB%9F%AC%EB%8F%84%ED%94%84" title="라스무스 러도프">라스무스 러도프</a>가 처음 만든 것으로, 그 뒤로 개발이 계속되어 오늘날에까지 이르게 되었다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<h3><span class="mw-headline" id="PHP_1.0">PHP 1.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=2" title="부분 편집: PHP 1.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>1995년 6월 8일 발표 라스 매스·라드후가 개발. 초기는 펄로 작성된 스크립트였지만, 그 후 C 언어로 고쳐 작성된 PHP (Personal Home Page Tools)가 된다.</p>
<h3><span class="mw-headline" id="PHP_Version_2_.28PHP.2FFI.29">PHP Version 2 (PHP/FI)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=3" title="부분 편집: PHP Version 2 (PHP/FI)">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>1996년 4월 16일 발표. FI(Form Interpreter, SQL이 만든 DBMS 도구)가 통합되어 1997년에 PHP/FI2.0 이 된다. 정식명칭은 "Personal Home Page Construction Kit/Form Interpreter"이다.</p>
<h3><span class="mw-headline" id="PHP_3.0">PHP 3.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=4" title="부분 편집: PHP 3.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>1998년 6월 6일 발표. PHP/FI를 기반으로, 앤디·가트만즈와 제이브·스라스키에 의해 PHP3(PHP:Hypertext Preprocessor)로 업데이트되었다.</p>
<h3><span class="mw-headline" id="PHP_4.0.0">PHP 4.0.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=5" title="부분 편집: PHP 4.0.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2000년 5월 22일 발표. PHP3 를 큰 폭으로 기능을 확장하고. Zend 엔진을 도입하였다.</p>
<h3><span class="mw-headline" id="PHP_4.1.0">PHP 4.1.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=6" title="부분 편집: PHP 4.1.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2001년 12월 10일 발표. 슈퍼 전역 변수가 추가된다.</p>
<h3><span class="mw-headline" id="PHP_4.2.0">PHP 4.2.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=7" title="부분 편집: PHP 4.2.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2002년 4월 22일 발표. register_globals의 초기값이 Off로 변경된다.</p>
<h3><span class="mw-headline" id="PHP_4.3.0">PHP 4.3.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=8" title="부분 편집: PHP 4.3.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2002년 12월 27일 발표. <a href="/wiki/%EB%AA%85%EB%A0%B9_%EC%A4%84_%EC%9D%B8%ED%84%B0%ED%8E%98%EC%9D%B4%EC%8A%A4" title="명령 줄 인터페이스">명령 줄 인터페이스</a>가 추가된다.</p>
<h3><span class="mw-headline" id="PHP_4.4.0">PHP 4.4.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=9" title="부분 편집: PHP 4.4.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2005년 7월 11일 발표. 같은 날 PHP4의 지원을 2007년 12월 31일에 종료한다고 고지했다. 중대한 보안문제에 관한 수정만, 2008년 8월 8일까지 계속되어 그 후 모든 지원을 종료했다.</p>
<h3><span class="mw-headline" id="PHP_5.0.0">PHP 5.0.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=10" title="부분 편집: PHP 5.0.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2005년 7월 13일 발표. 의사적인 객체 지향이 한층 강화됨과 동시에 클래스 라이브러리로서 SPL가 지원되었다. 또, SQLite 가 표준으로 장착되게 되었다. Zend 엔진 2.0을 도입했다.</p>
<h3><span class="mw-headline" id="PHP_5.1.0">PHP 5.1.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=11" title="부분 편집: PHP 5.1.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2005년 11월 24일 발표. 실행 속도가 개선되고 PDO 확장 기능이 추가된다.</p>
<h3><span class="mw-headline" id="PHP_5.2.0">PHP 5.2.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=12" title="부분 편집: PHP 5.2.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2006년 11월 2일 발표. 확장 모듈에 필터(Filter)가 기본으로 추가된다.</p>
<h3><span class="mw-headline" id="PHP_5.3.0">PHP 5.3.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=13" title="부분 편집: PHP 5.3.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2009년 6월 30일 발표. 아래의 기능이 추가되었다.</p>
<ul>
<li>이름공간 (Namespace)</li>
<li>지연 정적 바인딩(Late static bindings)</li>
<li>goto</li>
<li>클로저(Native closures)</li>
<li> ?: 연산자</li>
<li><a href="/wiki/%EC%93%B0%EB%A0%88%EA%B8%B0_%EC%88%98%EC%A7%91_(%EC%BB%B4%ED%93%A8%ED%84%B0_%EA%B3%BC%ED%95%99)" title="쓰레기 수집 (컴퓨터 과학)">쓰레기 수집</a>(garbage collection)</li>
</ul>
<h3><span class="mw-headline" id="PHP_5.4.0">PHP 5.4.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=14" title="부분 편집: PHP 5.4.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2012년 3월 1일 발표. 단일 상속의 한계를 없앤 트레이트와 짧은 배열 구문이 추가되었다.</p>
<h3><span class="mw-headline" id="PHP_5.5.0">PHP 5.5.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=15" title="부분 편집: PHP 5.5.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2013년 6월 20일 발표. 아래의 기능들이 추가 또는 변경되었다.</p>
<ul>
<li>제너레이터(Generators)와 코루틴(coroutines)</li>
<li>finally 키워드</li>
<li>강화된 패스워드 해슁 함수</li>
<li>배열과 문자열의 디레퍼런싱</li>
<li>상수 클래스명 ::class</li>
<li>empty() 함수가 수식을 지원</li>
<li>foreach 내 이터레이터(Interator)로 논스칼라(non-scalar) 사용</li>
<li>foreach 구문에 list() 사용</li>
<li>op코드 캐시를 위한 Zend OPcache</li>
</ul>
<h3><span class="mw-headline" id="PHP_5.6.0">PHP 5.6.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=16" title="부분 편집: PHP 5.6.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2014년 8월 28일 발표. 아래의 기능들이 추가되었다.</p>
<ul>
<li>선언에 상수 표현식 지원</li>
<li>가변길이 인수에 대한 문법 (...$args)</li>
<li>매개변수 해체(argument unpacking)</li>
<li>거듭제곱 연산자 (**)</li>
<li>추가적인 use 문 (use function, use const)</li>
<li>SAPI 모듈을 위한 새로운 phpdbg 디버거</li>
</ul>
<h3><span class="mw-headline" id="PHP_7.0.0">PHP 7.0.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=17" title="부분 편집: PHP 7.0.0">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>2015년 12월 3일 공개되었다. 특징은 다음과 같다.</p>
<ul>
<li>성능 향상 - PHP 5.6 보다 두배 이상 빠른 속도</li>
<li>메모리 사용률이 현저히 감소</li>
<li>추상 구문 트리(Abstract Syntax Tree)</li>
<li>일관된 64비트 지원(Consistent 64-bit support)</li>
<li>향상된 예외 상속(Improved Exception hierarchy)</li>
<li>많은 치명적 에러들이 예외로 전환 됨</li>
<li>보안 난수 발생기(Secure random number generator)</li>
<li>오래 됐거나 지원하지 않는 SAPI와 확장 지원 중단</li>
<li>Null 병법 연산자(The null coalescing operator (??))</li>
<li>리턴 값, 스칼라 타입 정의</li>
<li>익명 클래스(Anonymous Classes)</li>
<li>Zero cost asserts</li>
</ul>

<span class="p">&lt;/</span><span class="nt">html</span><span class="p">&gt;</span>

<p>또한, 간단한 템플릿 기능도 지원한다. 다만 이는 언어의 기능이라기보다는 하나의 트릭에 가까우며, 로직과 디자인이 완전히 분리되지 않는 문제점이 있다. 따라서 본격적인 템플릿 기능에는 <a class="new" href="/w/index.php?title=Smarty&amp;action=edit&amp;redlink=1" title="Smarty (없는 문서)">Smarty</a> 등의 전문 <a class="new" href="/w/index.php?title=%ED%85%9C%ED%94%8C%EB%A6%BF_%EC%97%94%EC%A7%84&amp;action=edit&amp;redlink=1" title="템플릿 엔진 (없는 문서)">템플릿 엔진</a>이 많이 사용된다. 다음은 템플릿을 써서 비슷한 결과를 출력하는 PHP 코드이다. (<a href="/wiki/HTML" title="HTML">HTML</a> 생성물은 위와 다르지만 화면 출력되는 모습은 같다.)</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">

<h2><span class="mw-headline" id=".EB.AC.B8.EB.B2.95">문법</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=19" title="부분 편집: 문법">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>근본적으로, PHP의 문법은 C스타일을 따른다.</p>
<h3><span class="mw-headline" id="C.EC.96.B8.EC.96.B4.EC.99.80_.EB.8B.A4.EB.A5.B8_.EC.A0.90">C언어와 다른 점</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=20" title="부분 편집: C언어와 다른 점">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>변수는 $기호가 앞에 붙은 단어로 나타내며 함수는 <i>함수 이름(매개변수)</i>의 형식으로 나타낸다.</li>
<li>함수를 선언할 때에는 <code>function</code>이라는 키워드를 앞에 붙여야 한다.</li>
<li>.(Dot) 연산자는 <i>문자열 결합 연산자</i>이며, 구조체 참조 연산자는 <code>-&gt;</code>로 통일된다.</li>
<li>포인터를 제공하지 않으며 기본 매개변수 값은 값에 의한 전달(Call by value)이다.</li>
<li>그 외 class 관련 문법이 <a href="/wiki/C%2B%2B" title="C++">C++</a>과 다르다.</li>
</ul>
<p>PHP4와 PHP5는 객체 문법에 차이를 보이고 있으므로 주의할 필요가 있다.</p>
<h2><span class="mw-headline" id=".EC.9E.A5.EC.A0.90">장점</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=21" title="부분 편집: 장점">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>PHP의 장점은 기존까지 존재해 왔던 언어에 비해 `직관적` 이라는 것이었다. 현재는 이런 인기 후, 유사하거나 더욱 개성이 강한 스크립트 언어들이 '웹 개발'을 위해 태어나게 된다. <sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<h2><span class="mw-headline" id=".ED.8A.B9.EC.A7.95">특징</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=22" title="부분 편집: 특징">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>자바나 C 언어에서 접근하려면 여러 Include 등을 통해야 했던 작업들이 '그냥 존재' 한다. 그래서 '그냥 적어 놓기만' 하면 작동을 한다. 즉 작성 코드의 양이 현저히 적다는 것이다. 그래서 직관적인 코드작성을 가능하게 하고 이것은 소스코드를 '절차적 프로그램 코드'에 가깝도록 만든다. 현재의 PHP는 절차적인 형태에서 '구조적 (클래스- Class)' 프로그램 작성이 가능한 상태로 진화된 상황이고 이런 진화하에서 Drupal, PHPbb, Wordpress 등의 거대한 프로그램들이 연이어 출시할 수 있게 되었다.</p>
<h2><span class="mw-headline" id=".EB.B9.84.ED.8C.90">비판</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=23" title="부분 편집: 비판">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>PHP의 소스가 점점 방대해지고 구조적이 되면서 기존의 PHP가 추구했던 '간결함의 추구'라는 사상은 어떻게 되버리는 것인가라는 비판도 있었다. 그러나 웹의 역사를 보면 알 수 있듯이 웹문서가 초창기 보다 '다채롭고' , '복잡하며', '보다 많은 내용을 담으려하고', 마우스나 키보드를 사용하여 보다 다양한 입력과, 움직임 <sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup>을 첨가했기 때문에 이 PHP의 활용은 사용자의 기호에 달렸다고 볼 수 있다.</p>
<h2><span class="mw-headline" id="PHP.EB.A1.9C_.EC.9E.91.EC.84.B1.EB.90.98.EC.96.B4_.EC.B6.9C.EC.8B.9C.EB.90.9C_.ED.94.84.EB.A1.9C.EA.B7.B8.EB.9E.A8.EB.93.A4">PHP로 작성되어 출시된 프로그램들</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=PHP&amp;action=edit&amp;section=24" title="부분 편집: PHP로 작성되어 출시된 프로그램들">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>PHP는 기존에, 사람들이 생각해오지 않았던 방향으로도 발전해가고 있었다. 웹으로 구현되리라 생각해오지 않았던 ERP, CRM 등의 경영에 관련한 프로그램이 개발하고 있다.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> 소스가 오픈된채 출시된 프로그램이 많으며 그들로는 <a href="/wiki/%EC%9B%8C%EB%93%9C%ED%94%84%EB%A0%88%EC%8A%A4" title="워드프레스">워드프레스</a>, <a href="/wiki/XpressEngine" title="XpressEngine">XpressEngine</a>, <a href="/wiki/%ED%83%9C%ED%84%B0%ED%88%B4%EC%A6%88" title="태터툴즈">태터툴즈</a>, <a class="new" href="/w/index.php?title=PHPbb&amp;action=edit&amp;redlink=1" title="PHPbb (없는 문서)">PHPbb</a>, <a class="mw-redirect" href="/wiki/Drupal" title="Drupal">Drupal</a>, <a href="/wiki/%EC%A0%9C%EB%A1%9C%EB%B3%B4%EB%93%9C" title="제로보드">제로보드</a>, <a href="/wiki/%EA%B7%B8%EB%88%84%EB%B3%B4%EB%93%9C" title="그누보드">그누보드</a>, <a class="new" href="/w/index.php?title=%ED%85%8C%ED%81%AC%EB%85%B8%ED%8A%B8&amp;action=edit&amp;redlink=1" title="테크노트 (없는 문서)">테크노트</a>, <a class="new" href="/w/index.php?title=ExpressionEngine&amp;action=edit&amp;redlink=1" title="ExpressionEngine (없는 문서)">ExpressionEngine</a>, webmaker3.0 등의 프로그램이 있다. 이 프로그램들은 개인 서버에 설치하여도 활용이 가능한 경우가 있으며, 개인/기관의 차원을 넘어, 그 활용의 형태로는 커뮤니티 사이트, 블로그 등을 들 수 있다. 상용 프로그램으론, 따로 이름을 적지 않아도, CRM, ERP 등, 즉, 기업 경영에 활용하기 좋은 형태로, 수많은 프로그램들이 출시되어 실용화된다. 이것은 기존의 C언어와 같은 언어에서 처럼 모듈로 존재하기도 한다.</p>
