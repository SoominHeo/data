<p><b>리눅스 커널</b>(Linux kernel)은 <a href="/wiki/%EC%9C%A0%EB%8B%89%EC%8A%A4_%EA%B3%84%EC%97%B4" title="유닉스 계열">유닉스 계열</a> 운영 체제의 <a href="/wiki/%EC%BB%A4%EB%84%90_(%EC%BB%B4%ED%93%A8%ED%8C%85)" title="커널 (컴퓨팅)">커널</a>이다. <a href="/wiki/GNU_%EC%9D%BC%EB%B0%98_%EA%B3%B5%EC%A4%91_%EC%82%AC%EC%9A%A9_%ED%97%88%EA%B0%80%EC%84%9C" title="GNU 일반 공중 사용 허가서">GNU 일반 공중 사용 허가서</a> 버전 2 (GPLv2) 아래에서 공개되었으며 전 세계적으로 배포자들이 개발한 리눅스는 가장 두드러진 <a href="/wiki/%EC%9E%90%EC%9C%A0_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="자유 소프트웨어">자유 소프트웨어</a> / <a href="/wiki/%EC%98%A4%ED%94%88_%EC%86%8C%EC%8A%A4" title="오픈 소스">오픈 소스</a>의 본보기들 가운데 하나이다.</p>
<p>리눅스 커널은 1991년에 <a href="/wiki/%EB%A6%AC%EB%88%84%EC%8A%A4_%ED%86%A0%EB%A5%B4%EB%B0%9C%EC%8A%A4" title="리누스 토르발스">리누스 토르발스</a>에 의해 생긴 말이다. 일찍이 <a href="/wiki/%EB%AF%B8%EB%8B%89%EC%8A%A4" title="미닉스">미닉스</a> 커뮤니티가 리눅스 커널에 코드와 개념을 제공하였다. 그 당시 <a href="/wiki/GNU_%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B8" title="GNU 프로젝트">GNU 프로젝트</a>는 <a href="/wiki/%EC%9E%90%EC%9C%A0_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="자유 소프트웨어">자유 소프트웨어</a> 운영 체제에 필요한 요소를 많이 만들어 냈지만 자체 커널 <a href="/wiki/GNU_%ED%97%88%EB%93%9C" title="GNU 허드">GNU 허드</a>는 완전하지 않았고 이용성이 없었다. <a href="/wiki/BSD" title="BSD">BSD</a> 운영 체제는 법적 문제로부터 헤어나오지 못했다. 이는 초기 버전의 제한된 기능에도 불구하고 <a href="/wiki/%EB%A6%AC%EB%88%85%EC%8A%A4" title="리눅스">리눅스</a>가 새로운 운영 체제를 사용하기 위한 프로젝트로부터 코드를 채용한 개발자들과 사용자들을 빠른 속도로 모았다는 것을 말해 준다. 오늘날 리눅스 커널은 수많은 프로그래머로부터 기여를 받고 있다.</p>

<p>리눅스 커널은 진정한 <a href="/wiki/%EC%84%A0%EC%A0%90_%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81" title="선점 스케줄링">선점형 멀티태스킹</a> (<a class="mw-redirect" href="/wiki/%EC%82%AC%EC%9A%A9%EC%9E%90_%EB%AA%A8%EB%93%9C" title="사용자 모드">사용자 모드</a> 및, 2.6 시리즈 이후의 경우 <a class="mw-redirect" href="/wiki/%EC%BB%A4%EB%84%90_%EB%AA%A8%EB%93%9C" title="커널 모드">커널 모드</a><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-lwn-22912_4-0"><a href="#cite_note-lwn-22912-4">[4]</a></sup>까지), <a href="/wiki/%EA%B0%80%EC%83%81_%EB%A9%94%EB%AA%A8%EB%A6%AC" title="가상 메모리">가상 메모리</a>, <a class="mw-redirect" href="/wiki/%EA%B3%B5%EC%9C%A0_%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="공유 라이브러리">공유 라이브러리</a>, <a class="new" href="/w/index.php?title=%EB%94%94%EB%A7%A8%EB%93%9C_%ED%8E%98%EC%9D%B4%EC%A7%95&amp;action=edit&amp;redlink=1" title="디맨드 페이징 (없는 문서)">디맨드 로딩</a>, 공유 <a class="new" href="/w/index.php?title=%EC%B9%B4%ED%94%BC_%EC%98%A8_%EB%9D%BC%EC%9D%B4%ED%8A%B8&amp;action=edit&amp;redlink=1" title="카피 온 라이트 (없는 문서)">카피 온 라이트</a> 실행 파일(<a class="new" href="/w/index.php?title=KSM&amp;action=edit&amp;redlink=1" title="KSM (없는 문서)">KSM</a> 공유), <a href="/wiki/%EB%A9%94%EB%AA%A8%EB%A6%AC_%EA%B4%80%EB%A6%AC" title="메모리 관리">메모리 관리</a>, <a href="/wiki/%EC%9D%B8%ED%84%B0%EB%84%B7_%ED%94%84%EB%A1%9C%ED%86%A0%EC%BD%9C_%EC%8A%A4%EC%9C%84%ED%8A%B8" title="인터넷 프로토콜 스위트">인터넷 프로토콜 스위트</a>, <a href="/wiki/%EC%8A%A4%EB%A0%88%EB%93%9C" title="스레드">스레딩</a>을 지원하는 <a class="mw-redirect" href="/wiki/%EB%AA%A8%EB%86%80%EB%A6%AC%EB%94%95_%EC%BB%A4%EB%84%90" title="모놀리딕 커널">모놀리딕 커널</a>이다.</p>
<h3><span class="mw-headline" id=".ED.94.84.EB.A1.9C.EA.B7.B8.EB.9E.98.EB.B0.8D_.EC.96.B8.EC.96.B4">프로그래밍 언어</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A6%AC%EB%88%85%EC%8A%A4_%EC%BB%A4%EB%84%90&amp;action=edit&amp;section=2" title="부분 편집: 프로그래밍 언어">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>리눅스 커널은 <a href="/wiki/%EC%96%B4%EC%85%88%EB%B8%94%EB%A6%AC%EC%96%B4" title="어셈블리어">어셈블리어</a>(GCC의 AT&amp;T 스타일 문법)로 작성된 수많은 짧은 코드와 더불어 <a href="/wiki/GNU_%EC%BB%B4%ED%8C%8C%EC%9D%BC%EB%9F%AC_%EB%AA%A8%EC%9D%8C" title="GNU 컴파일러 모음">GCC</a>(수많은 확장과 변경 사항을 표준 C에 도입)에 의해 지원되는 <a href="/wiki/C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="C (프로그래밍 언어)">C 프로그래밍 언어</a> 버전으로 작성되어 있다.</p>
