<p><b>C++</b>는 <a href="/wiki/AT%26T" title="AT&amp;T">AT&amp;T</a> <a href="/wiki/%EB%B2%A8_%EC%97%B0%EA%B5%AC%EC%86%8C" title="벨 연구소">벨 연구소</a>의 <a href="/wiki/%EB%B9%84%EC%95%BC%EB%84%A4_%EC%8A%A4%ED%8A%B8%EB%A1%AD%EC%8A%A4%ED%8A%B8%EB%A3%B9" title="비야네 스트롭스트룹">비야네 스트롭스트룹</a>이 <a href="/wiki/1983%EB%85%84" title="1983년">1983년</a> 발표하여 발전한 프로그래밍 언어이다. <a href="/wiki/C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="C (프로그래밍 언어)">C 언어</a>의 문법을 대부분 사용할 수 있으며, 객체지향성이 더해진 C 언어의 확장형이라고 생각할 수도 있다. 원래 이름은 <b>C with Classes</b>였으나, <a href="/wiki/%EC%A6%9D%EA%B0%90_%EC%97%B0%EC%82%B0%EC%9E%90" title="증감 연산자">증가 연산자</a>를 도입하면서 1984년에 C++로 이름이 변경되었다.<sup class="reference" id="cite_ref-invention_1-0"><a href="#cite_note-invention-1">[1]</a></sup></p>

<h2><span class="mw-headline" id=".ED.8A.B9.EC.A7.95">특징</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=1" title="부분 편집: 특징">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>C 언어에 객체지향 프로그래밍을 지원하기 위한 내용이 덧붙여진 것이라고 할 수도 있지만, 애초부터 객체지향을 염두에 두고 만들어진 언어와는 다르게, 단지 더 좋은 C 언어로서 수속형 언어로 취급하기도 한다. 초기의 C++은 C 위에 놓인 트랜스레이터로 구현되었다. 즉, C++ 프로그램을 일단 C 프로그램으로 변환하고 나서 C 컴파일러로 컴파일하는 식이었고 따라서 C 언어에 대해 상위 호환성을 갖는 언어였다.</p>
<p>그 후, C 언어의 표준 규격이 바뀔 때 const 수식 등 C++의 기능이 C 언어에 받아들이기도 했다. 현재 C 언어와 C++와의 사이에는 엄격한 호환성은 없다. 특히 <a href="/wiki/C99" title="C99">C99</a>의 출현으로 C 언어와의 호환성은 깨져 버렸다. 따라서 <a href="/wiki/C99" title="C99">C99</a> 이후로 C++은 C의 수퍼셋(superset)이 아니다. 그러나, C++17 표준안에서는 다시 C언어가 C++의 진부분집합이 될 것이 확실해 보인다. 현재 C와 C++가 명확한 구별 없이 혼재되어 사용되는 컴파일러가 대부분이지만 C99 이후의 문법을 C++ 컴파일러에서 컴파일 할 경우 오류가 발생할 수 있다.</p>
<p>다음과 같은 다양한 기능을 가지고 있어 C++ 언어의 표준 규격은 몹시 복잡하다. C++ 표준의 모든 사항을 완전하게 지원하는 컴파일러는 현재 손꼽을 정도이다.</p>
<ul>
<li>다중 상속</li>
<li>템플릿</li>
<li>연산자 오버로드</li>
<li>예외 처리</li>
<li>실행시 형 식별</li>
</ul>
<p>덧붙여 예부터 전해 내려온 절차적인 성격도 그대로 남아 "만능" 설계로, 유연함과 강력함은 다른 언어에 비할 바가 아니지만, 복잡한 언어가 되고 말았다. 이런 점 때문에 보다 객체지향성을 강화하여, 만능설계를 지향하기보다는 단순한 설계를 목표로 한 새로운 언어들인 <a href="/wiki/%EC%9E%90%EB%B0%94_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="자바 (프로그래밍 언어)">Java</a> 언어, <a class="mw-redirect" href="/wiki/D_%EC%96%B8%EC%96%B4" title="D 언어">D 언어</a> 등이 나오기도 했다.</p>
<h3><span class="mw-headline" id=".EC.BA.A1.EC.8A.90.ED.99.94">캡슐화</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=2" title="부분 편집: 캡슐화">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/%EC%BA%A1%EC%8A%90%ED%99%94" title="캡슐화">캡슐화</a>는 2가지 요소를 만족하도록 언어적으로 구현 되어야 한다.</p>
<ul>
<li>데이터와 메서드 결합</li>
</ul>
<p>C++에서는 객체의 저장공간의 위치(this로 표현되는 객체의 데이터 위치 주소값)를 함수에 넘김으로써 데이터와 메서드를 결합한다. this는 객체의 위치 주소값인 포인터 이다.</p>
<ul>
<li>외부에 데이터나 메서드를 은닉</li>
</ul>
<p><a href="/wiki/%EA%B0%9D%EC%B2%B4_%EC%A7%80%ED%96%A5_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="객체 지향 프로그래밍">객체 지향 프로그래밍</a>에서 작성된 프로그램 코드는 재사용이 중요한 요소이다. 재사용하려면 다른 개발자가 사용할 때, 필요한 사용방법만 알면된다. 클래스 내면 깊숙히 알 필요가 없는 경우가 많다. 따라서 객체를 설계할 때, 경우에 따라 <a class="new" href="/w/index.php?title=%EB%A9%A4%EB%B2%84_%EB%B3%80%EC%88%98&amp;action=edit&amp;redlink=1" title="멤버 변수 (없는 문서)">멤버 변수</a>나 <a class="new" href="/w/index.php?title=%EB%A9%A4%EB%B2%84_%ED%95%A8%EC%88%98&amp;action=edit&amp;redlink=1" title="멤버 함수 (없는 문서)">멤버 함수</a>를 다 알릴 필요가 없다. 어떤 요소들은 내부에서 사용하고 비공개로 설정할 필요가 있다. 해당 객체를 사용하거나 상속해서 사용할 때, 필요한 요소만을 공개하고 API을 작성하여 공개하면 된다. 이런 비공개의 방법으로 은닉을 설정하고 캡슐화의 한요소가 된다.</p>
<p>키워드 3개가 있다. private, protected, public을 사용해서 접근을 제한할 수 있다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">class</span> <span class="nc">MyObject</span> <span class="p">{</span>
<span class="k">public</span><span class="o">:</span>
     <span class="n">MyObject</span><span class="p">()</span> <span class="p">{</span> <span class="p">}</span>

     <span class="kt">int</span> <span class="n">getData</span><span class="p">();</span>  <span class="c1">// 은닉된 데이터는 메서드를 사용하여 인터페이스로 쓸 수 있다.</span>

<span class="k">private</span><span class="o">:</span> <span class="c1">// 이 지정자는 자기 클래스에서만 액세스를 한정한다. 은닉하는 방법으로 사용.</span>
     <span class="kt">int</span> <span class="n">m_data</span><span class="p">;</span>
<span class="p">};</span>

<span class="kt">int</span> <span class="n">MyObject</span><span class="o">::</span><span class="n">getData</span><span class="p">()</span>
<span class="p">{</span> 
   <span class="k">return</span> <span class="k">this</span><span class="o">-&gt;</span><span class="n">m_data</span><span class="p">;</span> <span class="c1">// this가 자동으로 넘어온다.</span>
<span class="p">}</span>
</pre></div>
<p>여기서 데이터와 메서드 결합은 멤버함수를 구성할 때 this라는 포인터로 상징되는 것을 개발자 코드내에 코딩없이도 자동으로 넘겨 주어 해당 클래스 내의 멤버변수들을 사용하게 하였다. 이렇게 언어적으로 결합한다.</p>
<p>this는 자동으로 객체의 공간위치값이 메모리의 주소값이다. 이것을 포인터로 사용하였다. 이것으로 결국 어떤 객체인지를 구별하는 수단으로 사용하여, 데이터와 메서드를 결합한다. <b>MyObject::</b>가 붙는 클래스의 멤버함수는 실행할 때 자동으로 모두 this가 넘어간다. static 메서드는 this가 제외된다.</p>
<p>this가 멤버 함수에 넘어가는 방법으로 CPU의 레지스터를 사용할 수도 있다. 멤버 함수에서 멤버 변수를 사용할 때 레지스터에 저장된 객체의 위치 주소값으로 하여 내부 멤버변수의 상대주소값과 합하여 위치값을 다시 설정하고 액세스 하면 된다. 이것은 마치 struct의 시작주소와 내부변수의 상대적 위치값의 합으로 표현하는 주소체계와 유사하다.</p>

<h2><span class="mw-headline" id=".EC.98.88.EC.A0.9C">예제</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=4" title="부분 편집: 예제">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Hello_world_.ED.94.84.EB.A1.9C.EA.B7.B8.EB.9E.A8">Hello world 프로그램</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=5" title="부분 편집: Hello world 프로그램">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/C%2B%2B_%ED%91%9C%EC%A4%80_%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="C++ 표준 라이브러리">C++ 표준 라이브러리</a> 스트림 기능을 사용하여 <a class="mw-redirect" href="/wiki/%ED%91%9C%EC%A4%80_%EC%B6%9C%EB%A0%A5" title="표준 출력">표준 출력</a>을 통해 <tt><a href="/wiki/Hello_world_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%A8" title="Hello world 프로그램">Hello, world!</a></tt>를 출력한다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="cp"># include &lt;iostream&gt; </span><span class="c1">// std::cout을 제공</span>

<span class="k">using</span> <span class="k">namespace</span> <span class="n">std</span><span class="p">;</span> <span class="c1">// std namespace를 사용할 수 있게 함. 즉, cout, endl을 std::cout 같이 길게 안 써도 되게 함.</span>
 
<span class="kt">void</span> <span class="nf">main</span><span class="p">()</span> <span class="p">{</span>
    <span class="n">cout</span> <span class="o">&lt;&lt;</span> <span class="s">"Hello, world!"</span> <span class="o">&lt;&lt;</span> <span class="n">endl</span><span class="p">;</span>
<span class="p">}</span>
</pre></div>
<p>그러나,비쥬얼 C++를 쓰는 사람들은 이렇게 하면 좋은 결과를 얻을 수 있다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="cp"># include &lt;iostream&gt; </span><span class="c1">// std::cout을 제공</span>

<span class="k">using</span> <span class="k">namespace</span> <span class="n">std</span><span class="p">;</span> <span class="c1">// std namespace를 사용할 수 있게 함. 즉, cout, endl을 std::cout 같이 길게 안 써도 되게 함.</span>

<h2><span class="mw-headline" id=".EA.B0.9D.EC.B2.B4.EC.9D.98_.EC.83.9D.EC.84.B1.EA.B3.BC_.EC.86.8C.EB.A9.B8">객체의 생성과 소멸</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=6" title="부분 편집: 객체의 생성과 소멸">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>객체를 구현하기 위해 <a class="mw-disambig" href="/wiki/%ED%81%B4%EB%9E%98%EC%8A%A4" title="클래스">클래스</a>의 선언이 필요하다. 구조적인 설정을 하고 생성하면 된다.</p>
<h3><span class="mw-headline" id=".EA.B0.9D.EC.B2.B4.EC.9D.98_.EC.84.A0.EC.96.B8">객체의 선언</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=7" title="부분 편집: 객체의 선언">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>객체의 모양은 <a class="mw-disambig" href="/wiki/%ED%81%B4%EB%9E%98%EC%8A%A4" title="클래스">클래스</a>를 사용하여 형을 선언하여 프로그램 한다. C언어에서 전역변수, 지역변수가 있듯이 객체도 선언 위치에 따라 생성과 소멸이 다르다.</p>
<p>객체 만들기 :</p>
<ol>
<li><a class="mw-disambig" href="/wiki/%ED%81%B4%EB%9E%98%EC%8A%A4" title="클래스">클래스</a>로 객체의 구조를 만든다. 멤버 변수와 멤버 함수로 구성된다.</li>
<li>정적 또는 동적으로 객체를 생성한다.</li>
</ol>
<dl>
<dd>
<dl>
<dd>
<ul>
<li>전역 정적 객체 : main() 함수 전에 객체의 메모리 공간을 생성하고 생성자가 자동 실행된다. 전역형은 <a href="/wiki/%EB%A9%94%EB%AA%A8%EB%A6%AC_%EB%A7%B5" title="메모리 맵">메모리 맵</a>에서 할당된 특정영역을 사용한다. 따라서 응용 프로그램이 시작하기 위해 <a href="/wiki/%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C" title="운영 체제">운영 체제</a>로 부터 할당 받아 사용한다.</li>
<li>지역 정적 객체 : 객체가 선언된 시점에서 객체의 메모리 공간을 생성하고 생성자가 자동 실행된다. 지역변수처럼 주로 <a href="/wiki/%EC%8A%A4%ED%83%9D" title="스택">스택</a>에 할당된다.</li>
<li>동적 객체 : new에 의해 힙영역에 객체의 메모리 공간을 생성하고 생성자가 자동 실행된다.</li>
</ul>
</dd>
</dl>
</dd>
</dl>
<p>객체 소멸 :</p>
<dl>
<dd>
<ul>
<li>전역 정적 객체 : main() 함수 return후 소멸된다. 응용 프로그램의 종료와 동시에 소멸된다.</li>
<li>지역 정적 객체 : 함수 블럭 또는 객체가 생성된 임의의 자기 블럭이 끝나면 소멸된다.</li>
<li>동적 객체 : delete에 의해 소멸된다.</li>
</ul>
</dd>
</dl>
<p><br/></p>
<h4><span class="mw-headline" id=".EC.A0.91.EA.B7.BC.EC.9E.90">접근자</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=8" title="부분 편집: 접근자">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>클래스는 <a href="/wiki/%EC%BA%A1%EC%8A%90%ED%99%94" title="캡슐화">캡슐화</a>가 가능하다. 따라서 객체를 설계할 때는 특정 멤버변수나 멤버함수를 외부에서 접근을 제한할 필요가 있다.</p>
<p>접근 제어 키워드 3가지:</p>
<ul>
<li>private : 자기 클래스에서만 접근을 허용한다.</li>
<li>public : 모두 접근을 허용한다.</li>
<li>protected : 자기 클래스와 상속을 받은 자식 클래스에서만 접근을 허용한다.</li>
</ul>
<h3><span class="mw-headline" id=".EA.B0.9D.EC.B2.B4.EC.9D.98_.EC.83.9D.EC.84.B1">객체의 생성</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=9" title="부분 편집: 객체의 생성">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>C언어에서는 전역변수와 지역변수 그리고 동적할당에 의한 데이터 저장공간을 만들 수 있다. 마찬가지로 C++에서도 전역변수로 객체를 생성할수도 있고 지역변수로 생성하는것도 가능하다. 또한 new를 통해 동적으로 객체를 생성 시킬 수 있다.</p>
<h4><span class="mw-headline" id=".EC.A0.95.EC.A0.81_.ED.95.A0.EB.8B.B9_.EA.B0.9D.EC.B2.B4">정적 할당 객체</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=10" title="부분 편집: 정적 할당 객체">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>C++에서 변수를 잡듯이 객체도 선언을 통해 이루어 진다. C에서 전역변수와 지역변수로 나누어는 것과 같이, 객체 역시 같은 방식으로 전역 또는 지역 객체로 선언할 수 있다.</p>
<p>전역 정적 객체는 main() 함수가 실행 되기 전 저장공간이 생성되고, 생성자가 호출된다. 객체가 생성되면 메모리에 데이터 저장공간이 생기고 바로 생성자가 호출된다. 따라서 전역 정적 객체 생성자의 호출은 main함수보다 먼저 이루어진다.</p>
<h4><span class="mw-headline" id=".EB.8F.99.EC.A0.81_.ED.95.A0.EB.8B.B9_.EA.B0.9D.EC.B2.B4">동적 할당 객체</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=11" title="부분 편집: 동적 할당 객체">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>new에 의해 생성 되고, delete에 의해 삭제 된다. new로 생성 되면, new 실행 시점에서 객체의 저장공간인 메모리를 확보하고 생성자가 자동실행 된다. 동적객체는 힙영역에 존재하고 포인터를 넘겨 받아서 객체 포인터 변수에 주소값을 저장하고 처리한다.</p>
<p>'new'의 코딩은 함수에 넣을 수 밖에 없으므로 main 함수 시작 이후에야 가능하다.</p>
<h4><span class="mw-headline" id=".EC.9D.B4.EB.AF.B8_.EC.84.A4.EC.A0.95.EB.90.9C_.EB.A9.94.EB.AA.A8.EB.A6.AC_.EA.B3.B5.EA.B0.84.EC.9D.84_.EC.9D.B4.EC.9A.A9.ED.95.98.EC.97.AC_.EA.B0.9D.EC.B2.B4_.EC.A0.95.EC.9D.98.ED.95.98.EA.B8.B0">이미 설정된 메모리 공간을 이용하여 객체 정의하기</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=12" title="부분 편집: 이미 설정된 메모리 공간을 이용하여 객체 정의하기">편집</a><span class="mw-editsection-bracket">]</span></span></h4>

</pre>
<p>모든 함수는 블럭을 포함하므로 함수내에서 생성된 정적객체는 함수의 블럭이 끝나면 <a href="/wiki/%EC%86%8C%EB%A9%B8%EC%9E%90_(%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D)" title="소멸자 (컴퓨터 프로그래밍)">소멸자</a>가 호출되고 객체의 저장공간은 사라진다. 위의 예처럼 임의의 블럭에서 생성된 정적객체 역시 자기의 블럭이 끝나면 소멸자가 호출되고 사라진다.</p>
<h4><span class="mw-headline" id=".ED.81.B4.EB.9E.98.EC.8A.A4_.EB.B3.80.EC.88.98_.EC.84.A0.EC.96.B8_.EC.8B.9C.2C_.ED.81.B4.EB.9E.98.EC.8A.A4_.EA.B0.84_.EA.B5.90.EC.B0.A9.EC.83.81.ED.83.9C">클래스 변수 선언 시, 클래스 간 교착상태</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=14" title="부분 편집: 클래스 변수 선언 시, 클래스 간 교착상태">편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>클래스 내에서 멤버변수로 다른 클래스를 사용할 수 있다. 정적 멤버변수와 포인터가 가능하다. 그러나 2개의 클래스가 서로 정적인 객체를 선언하면 교착상태로 빠져 객체의 크기를 결정할 수 없다. 따라서 불가능해 진다.</p>

<h3><span class="mw-headline" id=".EA.B0.9D.EC.B2.B4.EC.9D.98_.EB.A9.94.EB.AA.A8.EB.A6.AC_.EA.B5.AC.EC.A1.B0">객체의 메모리 구조</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=16" title="부분 편집: 객체의 메모리 구조">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>객체는 클래스에 의해 규정된 구조에 따라 메모리 공간을 확보하고 메모리에 <a href="/wiki/%EC%95%A1%EC%84%B8%EC%8A%A4_(%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%ED%94%84%EB%A1%9C%EC%84%B8%EC%84%9C)" title="액세스 (마이크로프로세서)">액세스</a> 함으로써 데이터 처리가 된다. 이 구조는 기본적으로 struct와 별로 다를 바가 없다. 멤버변수만을 모아 순서데로 나열하여 특정 크기의 메모리를 확보하면 된다. struct에 C++에 필요한 몇가지 추가될 수 있다.</p>
<p>위의 예 프로그램의 객체구조를 알기 위해 다음과 같이 프로그램 하면:</p>

	<span class="n">printf</span><span class="p">(</span><span class="s">"pm=0x%08X</span><span class="se">\n</span><span class="s">"</span><span class="p">,</span> <span class="n">pm</span> <span class="p">);</span>

	<span class="n">printf</span><span class="p">(</span><span class="s">"&amp;counter=0x%08X, value=%d</span><span class="se">\n</span><span class="s">"</span><span class="p">,</span> <span class="o">&amp;</span><span class="n">MyObject</span><span class="o">::</span><span class="n">counter</span><span class="p">,</span> <span class="n">MyObject</span><span class="o">::</span><span class="n">counter</span><span class="p">);</span>

<p><a href="/wiki/%EC%86%8C%EB%A9%B8%EC%9E%90_(%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D)" title="소멸자 (컴퓨터 프로그래밍)">소멸자</a>에서 <tt>virtual</tt>을 붙이는 가장 중요한 이유는 상속과 관련하여 소멸자를 정확히 실행하기 위해서 이다. 소멸자를 호출할 때, 이 테이블을 보고 호출한다.</p>
<p>'<tt>static int MyObject::counter</tt>' 변수는 객체와 분리하여 한개의 전역변수로 처리된다. 따라서 이것은 객체의 메모리 크기에 들어가지 않는다. <tt>sizeof</tt>연산자에 의해 바이트수를 얻은면 16바이트가 된다. 여러개의 같은 클래스의 객체가 존재해도 이 변수는 하나만 존재할 뿐이다. 그리고 정적으로 처리되므로 전역변수만 가능하다.</p>
<p>이제 <tt>virtual</tt>을 삭제하고 객체를 만들면 멤버변수들로만 구성된 구조를 갖는다.</p>

<h2><span class="mw-headline" id=".EB.A9.A4.EB.B2.84_.EB.B3.80.EC.88.98">멤버 변수</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=17" title="부분 편집: 멤버 변수">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>클래스 내부의 변수를 말 한다. 클래스의 속성(attribute) 이다. 기본 자료형과 다른 클래스 들도 정의가 가능하다.</p>
<p>멤버함수는 프로그램 코드이므로 결국 기계어 코드 묶음에 존재한다. 멤버함수가 실행될 때, 객체의 변수들로 구성된 메모리 구조체의 시작 주소값으로 전달되어 조작된다.</p>
<h3><span class="mw-headline" id=".EA.B0.9D.EC.B2.B4_.EA.B3.B5.EA.B0.84.EC.9D.98_.EC.95.A1.EC.84.B8.EC.8A.A4_.EC.8B.9C.2C_.EA.B0.9D.EC.B2.B4.EC.98.81.EC.97.AD_.EC.B4.88.EA.B3.BC_.EC.95.A1.EC.84.B8.EC.8A.A4_.EB.AC.B8.EC.A0.9C">객체 공간의 액세스 시, 객체영역 초과 액세스 문제</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=18" title="부분 편집: 객체 공간의 액세스 시, 객체영역 초과 액세스 문제">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>C에서도 마찬가지 이지만, C++는 단지 메모리의 주소값을 이용하여 액세스하는 단순한 방법을 사용한다. 따라서 배열등에서 정의된 크기보다 큰 인덱스로 쓰기하면, 쓰기 자체에는 아무 문제없이 써진다. 그러나 변수가 차지하고 있는 공간을 넘어서는 문제는 C++에서도 그대로 적용된다. 자바와는 달리 어떤 보호책이 없다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="cp">#include</span> <span class="cpf">&lt;iostream&gt;</span>

<span class="k">class</span> <span class="nc">A</span> <span class="p">{</span>
<span class="k">public</span><span class="o">:</span>
   <span class="n">A</span><span class="p">()</span> <span class="p">{</span> <span class="n">num</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span> <span class="p">}</span>
   <span class="n">A</span><span class="p">(</span><span class="kt">int</span> <span class="n">n</span><span class="p">)</span> <span class="p">{</span> <span class="n">num</span> <span class="o">=</span> <span class="n">n</span><span class="p">;</span> <span class="p">}</span>

   <span class="kt">int</span> <span class="n">num</span><span class="p">;</span>
<span class="p">};</span>

<span class="kt">int</span> <span class="nf">main</span><span class="p">(</span><span class="kt">int</span> <span class="n">argc</span><span class="p">,</span> <span class="kt">char</span><span class="o">*</span> <span class="n">argv</span><span class="p">[])</span>
<span class="p">{</span>
    <span class="n">A</span> <span class="n">a</span><span class="p">(</span><span class="mi">10</span><span class="p">);</span>
    <span class="kt">char</span> <span class="n">buff</span><span class="p">[</span><span class="mi">4</span><span class="p">];</span>
    <span class="kt">char</span> <span class="o">*</span><span class="n">pstr</span> <span class="o">=</span> <span class="n">buff</span><span class="p">;</span>

    <span class="n">std</span><span class="o">::</span><span class="n">cout</span> <span class="o">&lt;&lt;</span> <span class="s">"num="</span> <span class="o">&lt;&lt;</span> <span class="n">a</span><span class="p">.</span><span class="n">num</span> <span class="o">&lt;&lt;</span> <span class="n">std</span><span class="o">::</span><span class="n">endl</span><span class="p">;</span>
    <span class="n">printf</span><span class="p">(</span><span class="s">"&amp;a=0x%08X</span><span class="se">\n</span><span class="s">"</span><span class="p">,</span> <span class="o">&amp;</span><span class="n">a</span><span class="p">);</span>
    <span class="n">printf</span><span class="p">(</span><span class="s">"buff=0x%08X</span><span class="se">\n</span><span class="s">"</span><span class="p">,</span> <span class="n">buff</span><span class="p">);</span>

    <span class="k">for</span> <span class="p">(</span><span class="kt">int</span> <span class="n">cnt</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span><span class="n">cnt</span> <span class="o">&lt;</span> <span class="mi">16</span><span class="p">;</span><span class="n">cnt</span><span class="o">++</span><span class="p">)</span>
	<span class="o">*</span><span class="n">pstr</span><span class="o">++</span> <span class="o">=</span> <span class="n">cnt</span><span class="p">;</span>

    <span class="n">std</span><span class="o">::</span><span class="n">cout</span> <span class="o">&lt;&lt;</span> <span class="s">"num="</span> <span class="o">&lt;&lt;</span>  <span class="n">a</span><span class="p">.</span><span class="n">num</span> <span class="o">&lt;&lt;</span> <span class="n">std</span><span class="o">::</span><span class="n">endl</span><span class="p">;</span>

    <span class="k">return</span> <span class="mi">0</span><span class="p">;</span>
<span class="p">}</span>
</pre></div>
<p>실행 결과 예 (x86 32비트 CPU, 마이크로소프트 비주얼스튜디오):<sup class="reference" id="cite_ref-MemOverf_7-0"><a href="#cite_note-MemOverf-7">[note 1]</a></sup></p>
<pre>
num=10
&amp;a=0x0027F848
buff=0x0027F83C
num=252579084
</pre>
<p>여기에서 10이어야 하는 값이 252579084로 바뀌어 졌다. 이것은 16바이트를 쓰면서 buff영역을 넘어서기 때문이다. buff와 객체 a와는</p>
<pre>
0x0027F848-0x0027F83C=0xC=12
</pre>
<p>의 공간차이가 난다. 따라서 12바이트 이상 쓰기 하면 a 영역의 값이 변한다.</p>

<h2><span class="mw-headline" id=".EB.A9.A4.EB.B2.84_.ED.95.A8.EC.88.98"><a class="new" href="/w/index.php?title=%EB%A9%A4%EB%B2%84_%ED%95%A8%EC%88%98&amp;action=edit&amp;redlink=1" title="멤버 함수 (없는 문서)">멤버 함수</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=19" title="부분 편집: 멤버 함수">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>클래스의 행위(behavior)를 정의한 것으로 기능적으로 설정된 프로그램 묶음이다.</p>
<h3><span class="mw-headline" id="this">this</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=20" title="부분 편집: this">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>모든 멤버함수는 멤버변수들로 구성된 구조적 저장공간이 필요하다. 각각의 다른 객체는 다른 메모리 공간에 존재 한다. 따라서 여러개의 객체를 구별한 포인터 변수가 필요하다. this는 객체의 위치를 정의하는 포인터 변수라고 생각하면 된다.</p>
<p>어느 객체에 조작을 할것인가를 결정하는 포인터 이다. 'this'는 클래스 내에 변수 선언이 필요없고 이미 정의된 키워드 이다.</p>
<p><a href="/wiki/%EC%BA%A1%EC%8A%90%ED%99%94" title="캡슐화">캡슐화</a>의 방법으로 멤버함수는 this을 통해 멤버변수와 묶는 방법으로 사용한다.</p>
<p>다음 예의 구조체와 클래스에서</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">struct</span> <span class="n">MyStruct</span> <span class="p">{</span>
   <span class="kt">int</span> <span class="n">opt</span><span class="p">;</span>
   <span class="kt">int</span> <span class="n">age</span><span class="p">;</span>
   <span class="kt">char</span> <span class="o">*</span><span class="n">name</span><span class="p">;</span>
<span class="p">};</span>

<span class="k">class</span> <span class="nc">MyObject</span> 
<span class="p">{</span>
<span class="k">public</span><span class="o">:</span>
   <span class="n">MyObject</span><span class="p">()</span> <span class="p">{</span> <span class="n">opt</span> <span class="o">=</span> <span class="n">age</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span> <span class="n">name</span> <span class="o">=</span> <span class="nb">NULL</span><span class="p">;}</span>
   <span class="kt">void</span> <span class="n">MyObject</span><span class="o">::</span><span class="n">setMyObject</span><span class="p">(</span><span class="kt">int</span> <span class="n">age</span><span class="p">,</span> <span class="kt">char</span> <span class="o">*</span><span class="n">name</span><span class="p">,</span> <span class="kt">int</span> <span class="n">opt</span> <span class="o">=</span> <span class="mi">0</span><span class="p">);</span>

<span class="k">private</span><span class="o">:</span>
   <span class="kt">int</span> <span class="n">opt</span><span class="p">;</span>
   <span class="kt">int</span> <span class="n">age</span><span class="p">;</span>
   <span class="kt">char</span> <span class="o">*</span><span class="n">name</span><span class="p">;</span>
<span class="p">};</span>

<span class="kt">void</span> <span class="nf">setMyStruct</span><span class="p">(</span><span class="n">MyStruct</span><span class="o">*</span> <span class="n">pthis</span><span class="p">,</span> <span class="kt">int</span> <span class="n">age</span><span class="p">,</span> <span class="kt">char</span> <span class="o">*</span><span class="n">name</span><span class="p">,</span> <span class="kt">int</span> <span class="n">opt</span><span class="cm">/* = 0*/</span><span class="p">)</span>
<span class="p">{</span>
   <span class="n">pthis</span><span class="o">-&gt;</span><span class="n">opt</span> <span class="o">=</span> <span class="n">opt</span><span class="p">;</span>
   <span class="n">pthis</span><span class="o">-&gt;</span><span class="n">age</span> <span class="o">=</span> <span class="n">age</span><span class="p">;</span>
   <span class="n">pthis</span><span class="o">-&gt;</span><span class="n">name</span> <span class="o">=</span> <span class="n">name</span><span class="p">;</span>
<span class="p">}</span>

<span class="kt">void</span> <span class="n">MyObject</span><span class="o">::</span><span class="n">setMyObject</span><span class="p">(</span><span class="kt">int</span> <span class="n">age</span><span class="p">,</span> <span class="kt">char</span> <span class="o">*</span><span class="n">name</span><span class="p">,</span> <span class="kt">int</span> <span class="n">opt</span><span class="cm">/* = 0*/</span><span class="p">)</span>
<span class="p">{</span>
   <span class="k">this</span><span class="o">-&gt;</span><span class="n">opt</span> <span class="o">=</span> <span class="n">opt</span><span class="p">;</span>
   <span class="k">this</span><span class="o">-&gt;</span><span class="n">age</span> <span class="o">=</span> <span class="n">age</span><span class="p">;</span>
   <span class="k">this</span><span class="o">-&gt;</span><span class="n">name</span> <span class="o">=</span> <span class="n">name</span><span class="p">;</span>
<span class="p">}</span>
</pre></div>
<p>위의 예에서 C언어의 struct는 구조체 내의 변수들을 묶어서 메모리에 할당한다.</p>
<ul>
<li>C 언어의 struct : void setMyStruct(MyStruct* <b>pthis</b>, int age, char *name, int opt = 0);</li>
<li>C++ 객체의 멤버함수 : void <b>MyObject::</b>setMyObject(int age, char *name, int opt = 0);</li>
</ul>
<p>위의 2개의 차이는 'MyObject::'을 붙이면 this 포인터가 자동으로 들어간다. 이를 통해 멤버함수를 클래스와 통합한다. 그러나 C언어의 함수는 어떤 변수의 구조체와 연결이 없다. 필요하면 포인터나 변수를 인수를 통해 넘길 뿐이다. 그러나 C++에서는 this는 반드시 포함 시키는 차이가 있다. static 멤버함수는 this가 넘어가는 것은 제외된다.</p>
<h3><span class="mw-headline" id="static_.EB.A9.A4.EB.B2.84_.ED.95.A8.EC.88.98">static 멤버 함수</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=21" title="부분 편집: static 멤버 함수">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>this가 객체의 위치 주소값을 취급하는 자동 포인터로써 멤버함수가 호출되면 자동으로 따라 다닌다. 이말은 이미 객체가 존재 한다는 것이다. 그러나 static을 사용하면 특정 객체의 this을 사용하지 않는다. 객체가 이미 존재하는 것은 상관없이 이미 존재하는 객체의 포인터 값을 모른다. 따라서 호출 할때, 특정 객체가 미리 생성되지 않아도 된다. 이미 생성 되었다면 오히려 인수로 넘겨 주어야 한다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">class</span> <span class="nc">A</span> <span class="p">{</span>
<span class="k">public</span><span class="o">:</span>
   <span class="n">A</span><span class="p">(</span><span class="kt">int</span> <span class="n">n</span><span class="p">)</span> <span class="p">{</span> <span class="n">num</span> <span class="o">=</span> <span class="n">n</span><span class="p">;</span> <span class="p">}</span>
   <span class="kt">int</span> <span class="n">num</span><span class="p">;</span>
   <span class="kt">int</span> <span class="nf">add</span><span class="p">(</span><span class="kt">int</span> <span class="n">n2</span><span class="p">)</span> <span class="p">{</span> <span class="k">return</span> <span class="p">(</span><span class="n">num</span> <span class="o">+=</span> <span class="n">n2</span><span class="p">);</span> <span class="p">}</span>

   <span class="k">static</span> <span class="kt">int</span> <span class="nf">add</span><span class="p">(</span><span class="n">A</span> <span class="o">*</span><span class="n">pa</span><span class="p">,</span> <span class="kt">int</span> <span class="n">n2</span><span class="p">);</span>
<span class="p">};</span>

<span class="kt">int</span> <span class="n">A</span><span class="o">::</span><span class="n">add</span><span class="p">(</span><span class="n">A</span> <span class="o">*</span><span class="n">pa</span><span class="p">,</span> <span class="kt">int</span> <span class="n">n2</span><span class="p">)</span>
<span class="p">{</span>
    <span class="c1">//num += n2;  // 오류 : this을 사용할 수 없다. 객체의 포인터가 넘어오지 않는다.</span>
	 <span class="c1">// error C2597: illegal reference to non-static member 'A::num'</span>
    <span class="n">pa</span><span class="o">-&gt;</span><span class="n">num</span> <span class="o">+=</span> <span class="n">n2</span><span class="p">;</span>  
    <span class="k">return</span> <span class="n">pa</span><span class="o">-&gt;</span><span class="n">num</span><span class="p">;</span>
<span class="p">}</span>

<span class="kt">int</span> <span class="n">main</span><span class="p">()</span>
<span class="p">{</span>
    <span class="n">A</span> <span class="n">a</span><span class="p">(</span><span class="mi">10</span><span class="p">);</span>
    <span class="kt">int</span> <span class="n">r</span> <span class="o">=</span> <span class="n">A</span><span class="o">::</span><span class="n">add</span><span class="p">(</span><span class="o">&amp;</span><span class="n">a</span><span class="p">,</span> <span class="mi">20</span><span class="p">);</span>     <span class="c1">// 결과 30</span>
    <span class="n">std</span><span class="o">::</span><span class="n">cout</span> <span class="o">&lt;&lt;</span> <span class="n">r</span> <span class="o">&lt;&lt;</span> <span class="n">std</span><span class="o">::</span><span class="n">endl</span><span class="p">;</span>
	
    <span class="n">r</span> <span class="o">=</span> <span class="n">a</span><span class="p">.</span><span class="n">add</span><span class="p">(</span><span class="o">&amp;</span><span class="n">a</span><span class="p">,</span> <span class="mi">10</span><span class="p">);</span>           <span class="c1">// 결과 40</span>
    <span class="n">std</span><span class="o">::</span><span class="n">cout</span> <span class="o">&lt;&lt;</span> <span class="n">r</span> <span class="o">&lt;&lt;</span> <span class="n">std</span><span class="o">::</span><span class="n">endl</span><span class="p">;</span>

    <span class="k">return</span> <span class="mi">0</span><span class="p">;</span>
<span class="p">}</span>
</pre></div>
<p>위의 프로그램 예제 코드 처럼, 객체가 없는 상태에서 실행되는 것이 static 멤버함수이기 때문에 객체의 멤버변수를 사용하지 않아도 된다. 클래스 이름으로 호출하면 되고 설령 이미 존재하는 객체의 멤버변수로 하더라고 this 가 넘어가지 않으므로 별 차이가 없다. <a href="/wiki/%EC%BA%A1%EC%8A%90%ED%99%94" title="캡슐화">캡슐화</a> 되지 않는 C 언어의 함수와 별 차이가 없다. C 언어의 함수와의 차이는 단지 '어느 클래스에 속하는냐'만의 의미만 갖는다.</p>

<h3><span class="mw-headline" id="struct.28.EA.B5.AC.EC.A1.B0.EC.B2.B4.29_.EC.99.80_class.28.ED.81.B4.EB.9E.98.EC.8A.A4.29.EB.8A.94_.EB.8F.99.EC.9D.BC">struct(구조체) 와 class(클래스)는 동일</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C%2B%2B&amp;action=edit&amp;section=23" title="부분 편집: struct(구조체) 와 class(클래스)는 동일">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>흔히 구조체는 클래스 이전에 선언되고 주로 여러종류의 기억공간을 선언하기 위한 형태로만 보는 경우가 일반적인데, 이는 틀린 생각이다. 구조체 역시 클래스와 동일하게 생성자와 소멸자를 지정할 수 있고, 코딩이 가능하며 내부호출과 캡슐화가 가능하다. 따라서, 하나의 물체(Object) 를 하나의 구조체(struct) 로 선언하고 사용할 수 있다. 따라서, 구조체 내에서도 모든 함수와 명령을 클래스와 동일하게 사용 가능하다. 다만, 멤버에 대해 접근 지정자를 써 주지 않는 경우 구조체는 public이 되고, 클래스는 private이 된다는 차이가 있다.</p>
