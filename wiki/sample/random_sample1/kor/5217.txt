<h2><span class="mw-headline" id=".EA.B5.AC.EC.A1.B0.ED.99.94.EB.90.9C_.EC.98.88.EC.99.B8_.EC.B2.98.EB.A6.AC_.28SEH_:_Structured_Exception_Handling.29"><span id="SEH"></span>구조화된 예외 처리 (SEH : Structured Exception Handling)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8%EC%9D%98_%EC%98%88%EC%99%B8_%EC%B2%98%EB%A6%AC_%EB%A9%94%EC%BB%A4%EB%8B%88%EC%A6%98&amp;action=edit&amp;section=1" title="부분 편집: 구조화된 예외 처리 (SEH : Structured Exception Handling)">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>마이크로소프트 구조화된 예외 처리 (Microsoft Structured Exception Handling)는 윈도우를 위한 네이티브 예외 처리 메커니즘이며 VEH의 전신이다.<sup class="reference" id="cite_ref-devx_1-0"><a href="#cite_note-devx-1">[1]</a></sup> 이것은 표준 C++ 예외에는 존재하지 않는 <code>finally</code> 메커니즘을 특징으로 갖는다(그러나 대부분의 <a href="/wiki/%EB%AA%85%EB%A0%B9%ED%98%95_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="명령형 프로그래밍">명령형 프로그래밍</a> 언어들에서는 이후 소개되었다). SEH는 실행의 각 <a href="/wiki/%EC%8A%A4%EB%A0%88%EB%93%9C" title="스레드">스레드</a>를 위해 독립적으로 설치되고 처리된다.</p>
<p>SEH의 마이크로소프트 구현은 <a href="/wiki/%EB%B3%BC%EB%9E%9C%EB%93%9C" title="볼랜드">볼랜드</a>, <a class="external text" href="http://patft.uspto.gov/netacgi/nph-Parser?patentnumber=5,628,016" rel="nofollow">미국 특허 5,628,016</a><span class="PDFlink noprint"><a class="external text" href="http://www.pat2pdf.org/pat2pdf/foo.pl?number=5,628,016" rel="nofollow"> </a></span> 라이선스의 특허권에 기반한다. 오픈소스 운영 체제 시스템들은 이 특허권 때문에 SEH 기반 메커니즘을 채택하는 것을 꺼린다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<h3><span class="mw-headline" id=".EC.82.AC.EC.9A.A9">사용</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8%EC%9D%98_%EC%98%88%EC%99%B8_%EC%B2%98%EB%A6%AC_%EB%A9%94%EC%BB%A4%EB%8B%88%EC%A6%98&amp;action=edit&amp;section=2" title="부분 편집: 사용">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>마이크로소프트는 SEH를 단지 컴파일러 수준에서의 프로그래밍 기법으로 지원한다. MS 비주얼 C++ 컴파일러는 이 목적을 위해 세개의 표준화되지 않은 키워드: <code>__try</code>, <code>__except</code> 그리고 <code>__finally</code> 라는 특징을 갖는다. 다른 예외 처리 측면들은 수많은 윈도우 API 함수들의 지원을 받는다.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> 예를 들면 SEH 예외를 직접 일으키기 위한 <code>RaiseException</code>가 있다.</p>
<h3><span class="mw-headline" id=".EA.B5.AC.ED.98.84">구현</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8%EC%9D%98_%EC%98%88%EC%99%B8_%EC%B2%98%EB%A6%AC_%EB%A9%94%EC%BB%A4%EB%8B%88%EC%A6%98&amp;action=edit&amp;section=3" title="부분 편집: 구현">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>윈도우에서 실행의 각 스레드는 자신의 <a href="/wiki/%EC%8A%A4%EB%A0%88%EB%93%9C_%EC%A0%95%EB%B3%B4_%EB%B8%94%EB%A1%9D" title="스레드 정보 블록">스레드 정보 블록</a>의 시작에서 문서화되지 않은 _EXCEPTION_REGISTRATION_RECORD 리스트와의 연결을 갖는다. <code>__try</code> 선언은 근본적으로 컴파일러에서 정의된 <code>EH_prolog</code> 함수를 호출한다. 이 함수는 스택에서 _EXCEPTION_REGISTRATION_RECORD 를 할당하며, 이것은 <code>msvcrt.dll</code>에서 <code>__except_handler3</code> 함수를 가리키고,<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[a]</a></sup><sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[b]</a></sup> 리스트의 헤드에 레코드를 추가한다. <code>__try</code> 블록의 끝에서 컴파일러에서 정의된 역 연산을 수행하는 <code>EH_epilog</code> 함수가 호출된다. 이 컴파일러에서 정의된 루틴들은 인라인화될 수 있다. 모든 프로그래머가 정의한 <code>__except</code> 와 <code>__finally</code> 블록들은 <code>__except_handler3</code> 내부에서 호출된다. 만약 이러한 블록들이 존재한다면 생성된 _EXCEPTION_REGISTRATION_RECORD 는 소수의 추가적인 필드들로 확장되며, 이것은 <code>__except_handler3</code>에 의해 사용된다.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[4]</a></sup></p>
<p>사용자 모드 코드에서의 예외의 경우, 운영 체제는 <sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[c]</a></sup> 핸들러 신호가 예외를 처리하거나 리스트가 끝날 때까지 스레드의 _EXCEPTION_REGISTRATION_RECORD 리스트를 분석하고 순서에 따라 각 예외 처리기를 호출한다. 리스트의 마지막은 항상 <a class="new" href="/w/index.php?title=%EC%9D%BC%EB%B0%98_%EB%B3%B4%ED%98%B8_%EC%9E%A5%EC%95%A0&amp;action=edit&amp;redlink=1" title="일반 보호 장애 (없는 문서)">일반 보호 장애</a>를 표시하는 <code>kernel32!</code><code>UnhandledExceptionFilter</code>이다.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[d]</a></sup> 그 후 리스트는 처리기에게 사용된 자원들을 청소할 기회를 주며 한번 더 순회된다. 마지막으로 실행은 커널 모드로 반환되는데<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[e]</a></sup> 여기는 프로세스가 재개되거나 종료되는 곳이다.</p>
<h2><span class="mw-headline" id=".EB.B2.A1.ED.84.B0.ED.99.94.EB.90.9C_.EC.98.88.EC.99.B8_.EC.B2.98.EB.A6.AC_.28VEH_:Vectored_Exception_Handling.29"><span id="VEH"></span>벡터화된 예외 처리 (VEH :Vectored Exception Handling)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8%EC%9D%98_%EC%98%88%EC%99%B8_%EC%B2%98%EB%A6%AC_%EB%A9%94%EC%BB%A4%EB%8B%88%EC%A6%98&amp;action=edit&amp;section=4" title="부분 편집: 벡터화된 예외 처리 (VEH :Vectored Exception Handling)">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>벡터화된 예외 처리(Vectored Exception Handling)는 <a href="/wiki/%EC%9C%88%EB%8F%84%EC%9A%B0_XP" title="윈도우 XP">윈도우 XP</a>에서 도입되었다.<sup class="reference" id="cite_ref-VEH_10-0"><a href="#cite_note-VEH-10">[5]</a></sup> VEH는 C++과 비주얼 베이직 같은 언어를 사용하는 윈도우 프로그래머에게 사용 가능하게 만들어 졌다. VEH는 SEH를 대체하는 것이 아니라 VEH가 SEH보다 더 높은 우선 순위를 가짐으로써 공존한다.<sup class="reference" id="cite_ref-devx_1-1"><a href="#cite_note-devx-1">[1]</a></sup><sup class="reference" id="cite_ref-VEH_10-1"><a href="#cite_note-VEH-10">[5]</a></sup> SEH와 비교해서, VEH는 더 전통적인 <a class="new" href="/w/index.php?title=%EC%95%8C%EB%A6%BC_%EC%BD%9C%EB%B0%B1_%EC%8A%A4%ED%82%B4&amp;action=edit&amp;redlink=1" title="알림 콜백 스킴 (없는 문서)">알림 콜백 스킴</a>과 닮았다.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[6]</a></sup></p>
<h2><span class="mw-headline" id=".EB.85.B8.ED.8A.B8">노트</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8%EC%9D%98_%EC%98%88%EC%99%B8_%EC%B2%98%EB%A6%AC_%EB%A9%94%EC%BB%A4%EB%8B%88%EC%A6%98&amp;action=edit&amp;section=5" title="부분 편집: 노트">편집</a><span class="mw-editsection-bracket">]</span></span></h2>

<li id="cite_note-5"><span class="mw-cite-backlink"><a href="#cite_ref-5">↑</a></span> <span class="reference-text">VC 런타임과 정적으로 링크된 다른 프로그램들 뿐만 아니라 <code>ntdll.dll</code>와 <code>kernel32.dll</code>는 컴파일된 이 함수를 갖는다.</span></li>

<li id="cite_note-8"><span class="mw-cite-backlink"><a href="#cite_ref-8">↑</a></span> <span class="reference-text">이 메시지는 프로세스의 <a class="external text" href="http://msdn.microsoft.com/en-us/library/ms680548%28VS.85%29.aspx" rel="nofollow">error mode</a>를 변경함으로써 조용해진다; 기본 마지막 처리기는 <a class="external text" href="http://msdn.microsoft.com/en-us/library/ms680634(VS.85).aspx" rel="nofollow">SetUnhandledExceptionFilter</a>로 대체될 수 있다.</span></li>
