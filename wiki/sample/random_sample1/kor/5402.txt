<p><b>맥 OS X 스노 레퍼드</b>(Mac OS X Snow Leopard. 버전 10.6)는 <a href="/wiki/%EC%95%A0%ED%94%8C_(%EA%B8%B0%EC%97%85)" title="애플 (기업)">애플</a>의 <a href="/wiki/%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C" title="운영 체제">운영 체제</a>인 <a class="mw-redirect" href="/wiki/%EB%A7%A5_OS_X" title="맥 OS X">맥 OS X</a>의 일곱번째 주요 버전이다.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> 맥 OS X 스노 레퍼드는 <a href="/wiki/%EC%95%A0%ED%94%8C_(%EA%B8%B0%EC%97%85)" title="애플 (기업)">애플</a>사의 <a class="mw-redirect" href="/wiki/%EC%B5%9C%EA%B3%A0_%EA%B2%BD%EC%98%81%EC%9E%90" title="최고 경영자">최고 경영자</a>인 <a href="/wiki/%EC%8A%A4%ED%8B%B0%EB%B8%8C_%EC%9E%A1%EC%8A%A4" title="스티브 잡스">스티브 잡스</a>가 <a href="/wiki/2008%EB%85%84" title="2008년">2008년</a> <a href="/wiki/6%EC%9B%94_9%EC%9D%BC" title="6월 9일">6월 9일</a>에 <a class="mw-redirect" href="/wiki/WWDC" title="WWDC">WWDC 08</a>에서 발표하였다. 그리고 2009년 8월 28일 정식으로 출시되었다. 가격이 1달러로 상당히 저렴해서 업그레이드 율이 아주 높았다. 맥 OS X 스노 레퍼드는 <a class="mw-redirect" href="/wiki/%EB%A7%A5_OS_X_%EB%A0%88%ED%8D%BC%EB%93%9C" title="맥 OS X 레퍼드">맥 OS X 레퍼드</a>가 나온지 2년 만에 나온 릴리즈이다.</p>
<p>이 운영 체제 판올림은 새로운 기능보다는 성능 향상과 전체적인 효율 증가에 초점이 맞추어져 있다. 또한 <a href="/wiki/%EC%8B%9C%EC%8A%A4%ED%85%9C_7" title="시스템 7">시스템 7</a>.1.2부터 지원되기 시작한 <a href="/wiki/%ED%8C%8C%EC%9B%8CPC" title="파워PC">파워PC</a> 플랫폼을 지원하지 않게 된 첫 버전이다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>스노 레퍼드의 다음 버전은 2011년 7월에 출시된 <a href="/wiki/%EB%A7%A5_OS_X_%EB%9D%BC%EC%9D%B4%EC%96%B8" title="맥 OS X 라이언">맥 OS X 라이언</a>이다.</p>

<h2><span class="mw-headline" id=".EC.8B.9C.EC.8A.A4.ED.85.9C_.EC.9A.94.EA.B5.AC_.EC.82.AC.ED.95.AD">시스템 요구 사항</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%A5_OS_X_10.6&amp;action=edit&amp;section=1" title="부분 편집: 시스템 요구 사항">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>애플은 스노 레퍼드를 설치하기 위한 시스템의 기본 요구사항을 다음과 같이 정했다.</p>
<ul>
<li><a href="/wiki/%EC%9D%B8%ED%85%94" title="인텔">인텔</a> 프로세서 탑재 <a href="/wiki/%EB%A7%A4%ED%82%A8%ED%86%A0%EC%8B%9C" title="매킨토시">매킨토시</a>. 인텔 코어 솔로나 인텔 코어 듀오 등의 <a href="/wiki/IA-32" title="IA-32">IA-32</a> 기반 프로세서는 <a href="/wiki/32%EB%B9%84%ED%8A%B8" title="32비트">32비트</a> 응용 소프트웨어만 구동할 수 있다. 이후 나온 <a class="mw-redirect" href="/wiki/X86_64" title="X86 64">x86 64</a> 아키텍처 프로세서는 <a href="/wiki/64%EB%B9%84%ED%8A%B8" title="64비트">64비트</a> 응용 소프트웨어를 작동할 수 있다.</li>
<li>최소 1GB의 <a class="mw-disambig" href="/wiki/RAM" title="RAM">RAM</a></li>
<li>최소 5GB의 여유 디스크 공간</li>
<li>설치를 위한 <a href="/wiki/DVD" title="DVD">DVD</a> 드라이브 (원격 맥 OS X 설치로도 사용 가능) 혹은 외장 <a href="/wiki/USB" title="USB">USB</a>나 <a class="mw-redirect" href="/wiki/FireWire" title="FireWire">FireWire</a> DVD 드라이브.</li>
</ul>
<p><b>몇 가지 기능을 사용하기 위한 추가 요구사항</b><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<ul>
<li><a href="/wiki/%ED%80%B5%ED%83%80%EC%9E%84" title="퀵타임">퀵타임</a> <a class="mw-redirect" href="/wiki/H.264" title="H.264">H.264</a> 하드웨어 가속 지원. <a class="new" href="/w/index.php?title=%EC%97%94%EB%B9%84%EB%94%94%EC%95%84_%EC%A7%80%ED%8F%AC%EC%8A%A4&amp;action=edit&amp;redlink=1" title="엔비디아 지포스 (없는 문서)">엔비디아 지포스</a> 9400M, 320M, GT 330M 이상의 그래픽 카드 요구.</li>
<li><a href="/wiki/OpenCL" title="OpenCL">OpenCL</a>이 지원되는 <a href="/wiki/%EC%97%94%EB%B9%84%EB%94%94%EC%95%84" title="엔비디아">엔비디아</a>나 <a class="mw-redirect" href="/wiki/ATI" title="ATI">ATI</a> 그래픽카드 요구.</li>
</ul>
<p>스노 레퍼드는 <a href="/wiki/%ED%8C%8C%EC%9B%8CPC" title="파워PC">파워PC</a> 기반 시스템을 더 이상 지원하지 않는다.</p>
<p>기존 레퍼드 및 타이거와는 달리 파워PC 기반 시스템에서만 작동되는 프로그램의 코드를 x86용으로 변환시켜주는 소프트웨어인 <a class="mw-disambig" href="/wiki/%EB%A1%9C%EC%A0%9C%ED%83%80" title="로제타">로제타</a>를 기본 설치가 아닌 추가 설치로 바꾸었다.</p>
<h3><span class="mw-headline" id="IBM_.ED.98.B8.ED.99.98_.EA.B8.B0.EC.A2.85_.EA.B8.B0.EB.B0.98.EC.9D.98_.EB.B9.84_.EB.A7.A4.ED.82.A8.ED.86.A0.EC.8B.9C_.ED.95.98.EB.93.9C.EC.9B.A8.EC.96.B4.EC.97.90.EC.84.9C.EC.9D.98_OS_X">IBM 호환 기종 기반의 비 매킨토시 하드웨어에서의 OS X</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%A5_OS_X_10.6&amp;action=edit&amp;section=2" title="부분 편집: IBM 호환 기종 기반의 비 매킨토시 하드웨어에서의 OS X">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a class="mw-redirect" href="/wiki/%ED%95%B4%ED%82%A8%ED%86%A0%EC%8B%9C" title="해킨토시">해킨토시</a>입니다.</div>
<p>몇몇 사용자들은 지원되지 않는 하드웨어에 맥 OS X 스노 레퍼드를 설치하는 방법을 알아냈다. 한 사용자는 지원되는 하드웨어에 맥 OS X 스노 레퍼드를 설치한 후 하드 드라이브를 미지원 하드웨어로 옮겼다. 또 다른 방법은 스노 레퍼드 설치 DVD를 지원되는 하드웨어에서 부팅한 후 파이워와이어 대상 디스크 모드로 지원되지 않는 하드웨어에 설치하는 것이다.</p>
<p>애플이 맥을 인텔 프로세서 기반으로 바꾼 이후, <a class="mw-redirect" href="/wiki/%ED%95%B4%ED%82%A8%ED%86%A0%EC%8B%9C" title="해킨토시">해킨토시</a> 프로젝트는 맥 OS X 타이거와 맥 OS X 레퍼드를 비 애플 x86 기반 컴퓨터에서 설치할 수 있게 했다. 가장 유명한 방법은 편집된 다윈 부트로더를 편집한 부트 132로 컴퓨터를 리테일로 속이거나 바닐라로 운영 체제를 EFI 기반 매킨토시로 속이는 것이다. 이런 방법들은 수정되지 않은 애플의 순정 DVD를 이용해서 맥 OS X를 설치할 수 있게 하지만, 몇몇 컴퓨터에서만 지원한다.</p>
<h2><span class="mw-headline" id=".EC.83.88.EB.A1.AD.EA.B1.B0.EB.82.98_.EB.B3.80.EA.B2.BD.EB.90.9C_.EB.B6.80.EB.B6.84">새롭거나 변경된 부분</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%A5_OS_X_10.6&amp;action=edit&amp;section=3" title="부분 편집: 새롭거나 변경된 부분">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>맥 OS X 스노 레퍼드는 기존 기능 모음을 더 정교하게 만들어서 운영 체제의 기술적 가능성을 놓이는 것이 목적이었고, 응용 소프트웨어의 효율성을 향상시키는 것이다. 많은 변화는 백그라운드에서 동작하는 시스템에 있고 사용자에게 잘 보이지 않는다. 예를 들어 응용 소프트웨어 "파인더"는 <a class="mw-redirect" href="/wiki/%EC%BD%94%EC%BD%94%EC%95%84_API" title="코코아 API">코코아 API</a>로 완전히 재작성되었다. 소프트웨어의 아주 큰 차이에도 불구하고, 사용자들은 사용자 인터페이스에 따른 변화를 거의 느끼지 못할 것이다. 스노 레퍼드의 변경 사항은 다음과 같다:</p>
<ul>
<li><b>부트 캠프 3.0</b>은 이제 윈도 파티션에서 <a class="mw-redirect" href="/wiki/HFS%2B" title="HFS+">HFS+</a> 파티션을 읽고 복사할 수 있게 한다. 또한 이 버전은 <a class="new" href="/w/index.php?title=%EC%8B%9C%EB%84%A4%EB%A7%88_%EB%94%94%EC%8A%A4%ED%94%8C%EB%A0%88%EC%9D%B4&amp;action=edit&amp;redlink=1" title="시네마 디스플레이 (없는 문서)">시네마 디스플레이</a>의 향상된 기능에 대한 지원과 시동 디스크 제어판의 새로운 커맨드 라인 버전을 추가한다.</li>
<li><b>파인더</b>가 스노 레퍼드의 새로운 기능을 사용하기 위해 <a class="mw-redirect" href="/wiki/%EC%BD%94%EC%BD%94%EC%95%84_API" title="코코아 API">코코아 API</a>로 완전히 재작성됐다.</li>
<li>OS의 하 드디스크 점유율이 기존에 비해 낮아졌다.</li>
<li>풀 멀티터치 트랙패드 제스처를 지원하기 시작했다.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></li>
<li><b>미리 보기</b> PDF 문서를 인식하기 위한 인공 지능 알고리즘 추가.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></li>
<li><b><a href="/wiki/%ED%80%B5%ED%83%80%EC%9E%84" title="퀵타임">퀵타임</a> X</b>, 새로운 버전의 <a href="/wiki/%ED%80%B5%ED%83%80%EC%9E%84" title="퀵타임">퀵타임</a> 플레이어가 완전한 64비트 코코아 프로그램으로 작성되었다. 애플은 퀵타임의 사용자 인터페이스를 풀 스크린 퀵타임 뷰로 바꿔서 전체 창이 비디오를 보여주게 설계했다. 퀵타임은 HTTP 라이브 스트리밍을 지원하고, 고성능 색상 제작을 위한 ColorSync를 활용한다.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> 스노 레퍼드가 엔비디아 지포스 9400M, 320M, GT 330M 그래픽 카드를 가지고 있을 경우에, 퀵타임 X는 CPU 로드를 줄이기 위해 비디오 디코딩 기술을 사용한다.</li>
<li><b><a class="mw-redirect" href="/wiki/%EC%82%AC%ED%8C%8C%EB%A6%AC_%EB%B8%8C%EB%9D%BC%EC%9A%B0%EC%A0%80" title="사파리 브라우저">사파리</a> 4</b> 는 톱 사이트, 커버 플로우, 보이스오버, 표준 지원 추가, 충돌 복구 등의 기능을 추가했다. 사파리 4는 스노 레퍼드에 포함되어 있지만 꼭 필요하지는 않으며, <a href="/wiki/%EB%A7%A5_OS_X_%ED%83%80%EC%9D%B4%EA%B1%B0" title="맥 OS X 타이거">맥 OS X 타이거</a>, <a class="mw-redirect" href="/wiki/%EB%A7%A5_OS_X_%EB%A0%88%ED%8D%BC%EB%93%9C" title="맥 OS X 레퍼드">레퍼드</a> 및 <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">마이크로소프트 윈도</a>에서도 무료로 다운로드할 수 있다.</li>
<li><b>타임 머신</b> 연결 생성과 백업이 훨씬 빨라졌다.</li>
<li><b>VoiceOver</b>가 스노 레퍼드에서 크게 향상되었다. 웹 페이지를 읽는 것은 오토 웹 스팟으로 향상되었고, 최신 애플 노트북에서 트랙패드 제스처는 보이스오버 기능을 조작하는데 사용될 수 있다.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></li>
</ul>
<h3><span class="mw-headline" id=".EC.82.AC.EC.9A.A9.EC.9E.90_.EC.9D.B8.ED.84.B0.ED.8E.98.EC.9D.B4.EC.8A.A4_.ED.96.A5.EC.83.81">사용자 인터페이스 향상</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%A5_OS_X_10.6&amp;action=edit&amp;section=4" title="부분 편집: 사용자 인터페이스 향상">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>파인더가 완전히 <a class="mw-redirect" href="/wiki/%EC%BD%94%EC%BD%94%EC%95%84_API" title="코코아 API">코코아 API</a>로 재작성되었지만, 큰 인터페이스 변경은 없었다. 대신 사용상의 편의를 강조하기 위해 몇몇 인터페이스가 변경되었다. 변경된 사항은 다음과 같다:</p>
<ul>
<li>익스포제가 이제는 왼쪽 클릭이나 독의 아이콘으로도 시작할 수 있다. 창은 새로운 그리드 패턴으로 정렬된다.</li>
<li>기본 감마가 1.8에서 2.2로 변경되었다.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup></li>
<li>빨라진 <a href="/wiki/PDF" title="PDF">PDF</a> 및 <a href="/wiki/JPEG" title="JPEG">JPEG</a> 아이콘 새로 고침.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></li>
<li>네트워크를 검색할 때, 에어포트 <a href="/wiki/%EB%A9%94%EB%89%B4_%EB%AA%A8%EC%9D%8C" title="메뉴 모음">메뉴 모음</a> 아이콘이 네트워크를 찾을 때 애니메이션을 보여주고 사용 가능한 네트워크의 강도를 드롭 다운 메뉴로 보여준다.</li>
<li>디스크의 남은 공간 등을 보여주는 바이트의 단위가 <a class="mw-redirect" href="/wiki/MiB" title="MiB">MiB</a>에서 <a href="/wiki/%EB%A9%94%EA%B0%80%EB%B0%94%EC%9D%B4%ED%8A%B8" title="메가바이트">MB</a>로 바뀌었다. <a href="/wiki/%EB%A9%94%EA%B0%80%EB%B0%94%EC%9D%B4%ED%8A%B8" title="메가바이트">MB</a> 방식은 <a class="mw-redirect" href="/wiki/%ED%95%98%EB%93%9C_%EB%94%94%EC%8A%A4%ED%81%AC" title="하드 디스크">하드 디스크</a> 제조사들이 많이 사용하는 방식이다.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></li>
<li>종료 속도와 잠자기 모드 전환이 빨라졌다.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></li>
</ul>
<h3><span class="mw-headline" id=".EC.A0.9C.EA.B1.B0.EB.90.9C_.EA.B8.B0.EB.8A.A5">제거된 기능</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%A5_OS_X_10.6&amp;action=edit&amp;section=5" title="부분 편집: 제거된 기능">편집</a><span class="mw-editsection-bracket">]</span></span></h3>

<li><a href="/wiki/%EC%9D%91%EC%9A%A9_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="응용 소프트웨어">응용 소프트웨어</a>의 언어를 "정보 수집" 창으로 바꿀 수 없다.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup></li>

<h2><span class="mw-headline" id=".EA.B0.9C.EB.B0.9C.EC.9E.90_.EA.B8.B0.EB.8A.A5">개발자 기능</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%A5_OS_X_10.6&amp;action=edit&amp;section=6" title="부분 편집: 개발자 기능">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="64.EB.B9.84.ED.8A.B8_.EC.95.84.ED.82.A4.ED.85.8D.EC.B2.98">64비트 아키텍처</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%A5_OS_X_10.6&amp;action=edit&amp;section=7" title="부분 편집: 64비트 아키텍처">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/%EB%A7%A5_OS_X_%ED%83%80%EC%9D%B4%EA%B1%B0" title="맥 OS X 타이거">맥 OS X 타이거</a>는 <a href="/wiki/64%EB%B9%84%ED%8A%B8" title="64비트">64비트</a> 응용 소프트웨어에 대한 제한된 지원을 추가했다. <a class="mw-redirect" href="/wiki/%EB%A7%A5_OS_X_%EB%A0%88%ED%8D%BC%EB%93%9C" title="맥 OS X 레퍼드">맥 OS X 레퍼드</a>는 <a href="/wiki/64%EB%B9%84%ED%8A%B8" title="64비트">64비트</a> <a class="mw-disambig" href="/wiki/%EC%95%A0%ED%94%8C%EB%A6%AC%EC%BC%80%EC%9D%B4%EC%85%98" title="애플리케이션">애플리케이션</a>이 대부분의 <a class="mw-redirect" href="/wiki/%EB%A7%A5_OS_X" title="맥 OS X">맥 OS X</a> 라이브러리와 프레임워크를 사용할 수 있도록 지원을 확장했다.</p>
<p>스노 레퍼드에서 대부분의 기본 응용 소프트웨어는 <a href="/wiki/64%EB%B9%84%ED%8A%B8" title="64비트">64비트</a> <a href="/wiki/X86-64" title="X86-64">x86-64</a>에 맞도록 재설계되었다.(<a class="mw-redirect" href="/wiki/%EC%95%84%EC%9D%B4%ED%8A%A0%EC%A6%88" title="아이튠즈">아이튠즈</a>, 프론트 로, 그래퍼, DVD 플레이어 제외) 해당 응용 소프트웨어는 <a href="/wiki/32%EB%B9%84%ED%8A%B8" title="32비트">32비트</a> 프로세서에서 <a href="/wiki/32%EB%B9%84%ED%8A%B8" title="32비트">32비트</a> 모드로 작동할 것이고, <a href="/wiki/64%EB%B9%84%ED%8A%B8" title="64비트">64비트</a> 프로세서에서 <a href="/wiki/64%EB%B9%84%ED%8A%B8" title="64비트">64비트</a> 모드로 작동할 것이다.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup></p>
<p>맥 OS X의 커널은 몇몇 기기에서 <a href="/wiki/64%EB%B9%84%ED%8A%B8" title="64비트">64비트</a> 모드에서 작동하도록 새롭게 만들어졌다. 해당 기기에서 스노 레퍼드는 16 <a href="/wiki/%ED%85%8C%EB%9D%BC%EB%B0%94%EC%9D%B4%ED%8A%B8" title="테라바이트">테라바이트</a>의 메모리를 지원한다. 최신 <a class="new" href="/w/index.php?title=Xserve&amp;action=edit&amp;redlink=1" title="Xserve (없는 문서)">Xserve</a>와 <a href="/wiki/%EB%A7%A5_%ED%94%84%EB%A1%9C" title="맥 프로">맥 프로</a>는 <a href="/wiki/64%EB%B9%84%ED%8A%B8" title="64비트">64비트</a> 모드를 기본으로 작동할 것이고, 최신 <a href="/wiki/%EC%95%84%EC%9D%B4%EB%A7%A5" title="아이맥">아이맥</a>도 <a href="/wiki/64%EB%B9%84%ED%8A%B8" title="64비트">64비트</a> 커널을 지원하지만, 기본으로 설정되어 있지 않다.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> 64비트 커널을 로딩하고 싶은 사용자들은 부팅 직후 키보드에서 6과 4를 누르고 있어야 한다.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup><sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> com.apple.Boot.plist 파일을 변경하면 사용자가 호환되는 컴퓨터에서 계속 64비트 모드로 부팅할 수 있게 한다.</p>
<p>버전 10.6.0에서 다음의 애플 컴퓨터만이 <a href="/wiki/64%EB%B9%84%ED%8A%B8" title="64비트">64비트</a> 커널과 호환된다.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup><sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup></p>

<h3><span class="mw-headline" id=".EA.B7.B8.EB.9E.9C.EB.93.9C_.EC.84.BC.ED.8A.B8.EB.9F.B4_.EB.94.94.EC.8A.A4.ED.8C.A8.EC.B9.98">그랜드 센트럴 디스패치</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%A5_OS_X_10.6&amp;action=edit&amp;section=8" title="부분 편집: 그랜드 센트럴 디스패치">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>그랜드 센트럴 디스패치는 다중 프로세서 코어를 사용해서 더 효율적인 성능을 제공하는 도구이다. 기존 <a href="/wiki/%EC%9D%91%EC%9A%A9_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="응용 소프트웨어">응용 소프트웨어</a>는 <a href="/wiki/%EB%A9%80%ED%8B%B0_%EC%BD%94%EC%96%B4" title="멀티 코어">멀티 코어</a> <a href="/wiki/%EC%A4%91%EC%95%99_%EC%B2%98%EB%A6%AC_%EC%9E%A5%EC%B9%98" title="중앙 처리 장치">CPU</a>용으로 최적화하는 것에 대한 기술적 어려움 때문에, 수많은 컴퓨터 응용 프로그램은 다중 프로세서 코어를 효율적으로 사용하지 않는다.<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> 따라서, 프로세싱 능력이 제대로 사용되지 않는 경우가 많은데, 그랜드 센트럴 디스패치는 프로그래머들이 병렬 프로그래밍 시 효율적으로 다중 코어를 사용할 수 있게 해주는 <a href="/wiki/API" title="API">API</a>를 포함한다.</p>
<p>새로운 C와 오브젝티브 C 언어 기능인 블록스는 그랜드 센트럴 디스패치의 효과를 누리는 응용 소프트웨어를 쉽게 생성할 수 있게 해준다.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup><sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup><sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup></p>
<h3><span class="mw-headline" id="CUPS">CUPS</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%A5_OS_X_10.6&amp;action=edit&amp;section=9" title="부분 편집: CUPS">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>많은 <a href="/wiki/%EC%9C%A0%EB%8B%89%EC%8A%A4_%EA%B3%84%EC%97%B4" title="유닉스 계열">유닉스 계열</a> <a href="/wiki/%EC%9A%B4%EC%98%81_%EC%B2%B4%EC%A0%9C" title="운영 체제">운영 체제</a>에서 쓰이는 프린팅 시스템인 CUPS가 1.4 버전으로 업그레이드되었다. 이 버전은 향상된 <a href="/wiki/%EC%9E%A5%EC%B9%98_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B2%84" title="장치 드라이버">장치 드라이버</a>, 네트워킹, Kerberos 지원, 성능 향상 등이 추가된다.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup></p>
<h2><span class="mw-headline" id=".ED.98.B8.ED.99.98.EC.84.B1">호환성</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%A5_OS_X_10.6&amp;action=edit&amp;section=10" title="부분 편집: 호환성">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>스노 레퍼드는 페러럴즈 데스크톱 3.0, Aperture 2.1.1 이전 버전, <a href="/wiki/%ED%82%A4%EB%85%B8%ED%8A%B8" title="키노트">키노트</a> 2.0.2 이전 버전 등의 일부 소프트웨어와는 호환되지 않는다.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> 또한 애플은 스노 레퍼드랑 호환성 문제가 있는 응용 소프트웨어의 목록을 등록하였다.<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup></p>
<p>기존 <a class="mw-redirect" href="/wiki/%EB%A7%A5_OS_X" title="맥 OS X">맥 OS X</a>에서 사용되던 프린터와 스캐너 드라이버는 스노 레퍼드랑 호환되지 않는다. 스노 레퍼드의 출시와 함께 많은 프린터 제조사들은 맥 OS X 스노 레퍼드용 업데이트된 드라이버를 제공했다.<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup> 네이티브 드라이버가 존재하지 않을 경우 스노 레퍼드는 CUPS와 Gutenprint 오픈 소스 프린터 드라이버를 지원한다. 하지만 이 드라이버는 제한된 기능을 제공한다.</p>
<h2><span class="mw-headline" id=".EB.B3.B4.EC.95.88">보안</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%A5_OS_X_10.6&amp;action=edit&amp;section=11" title="부분 편집: 보안">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>멀웨어가 감지되었을 때 사용자에게 알려주는 악성코드 방지(안티 멀웨어) 기능이 추가되었다.<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup></p>
<p>컴퓨터 보안 전문가 찰리 밀러는 ASLR가 없기 때문에 맥 OS X 스노 레퍼드가 <a class="mw-redirect" href="/wiki/%EB%A7%88%EC%9D%B4%ED%81%AC%EB%A1%9C%EC%86%8C%ED%94%84%ED%8A%B8_%EC%9C%88%EB%8F%84" title="마이크로소프트 윈도">마이크로소프트 윈도</a>보다 보안에 취약하다고 말했다.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> 이 기능은 마이크로소프트가 <a class="mw-redirect" href="/wiki/%EC%9C%88%EB%8F%84_%EB%B9%84%EC%8A%A4%ED%83%80" title="윈도 비스타">윈도 비스타</a>부터 추가한 기능이지만 맥 OS X 스노 레퍼드에서는 존재하지 않았다.<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup> ASLR의 부재에도 불구하고, 애플은 맥 OS X를 스택 보호, 더 많은 <a class="mw-redirect" href="/wiki/%EB%A7%A5_OS_X" title="맥 OS X">맥 OS X</a> 컴포넌트 샌드박싱으로 보안성을 강화하였다.</p>
<h2><span class="mw-headline" id=".ED.8F.89.EA.B0.80">평가</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%A7%A5_OS_X_10.6&amp;action=edit&amp;section=12" title="부분 편집: 평가">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>맥 OS X 스노 레퍼드는 전반적으로 좋은 평가를 받았다.<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[31]</a></sup><sup class="reference" id="cite_ref-32"><a href="#cite_note-32">[32]</a></sup> <a class="mw-redirect" href="/wiki/WWDC" title="WWDC">WWDC</a>2009에서 애플은 스노 레퍼드는 새로운 비주얼 변화는 없을 것이며,<sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup> 대신 릴리즈는 운영 체제가 더 나은 성능을 보이는 것에 초점을 맞출 것이라고 했다.</p>
<p>스노 레퍼드의 가격은 기존 맥 OS X의 가격이었던 129달러가 아닌 29달러로 출시되었다. 이 이유는 눈에 보이는 시스템의 변화가 거의 없기 때문이다.<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup> 하지만 대부분의 리뷰어들은 파인더, <a class="mw-redirect" href="/wiki/ICal" title="ICal">iCal</a> 등의 네이티브 <a class="mw-redirect" href="/wiki/%EB%A7%A5_OS_X" title="맥 OS X">맥 OS X</a> 응용 소프트웨어의 속도가 상당히 향상되었다고 한다. 단일 사용자 업그레이드와 패밀리 팩은 아마존닷컴의 소프트웨어 베스트 셀러 차트에서 출시 1주일 만에 1위와 2위를 왔다갔다했다.<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup></p>
<p>하지만 <a class="mw-redirect" href="/wiki/%EB%A7%A5_OS_X" title="맥 OS X">맥 OS X</a> 10.6과 10.6.1 에서 아주 드문 경우지만 맥 OS X로 업그레이드하면서 이전의 게스트 계정의 사용자 계정 데이터가 삭제되는 경우가 있었다.<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup> 이 버그는 버전 10.6.2에서 해결되었다.<sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup></p>
