<h2><span class="mw-headline" id=".EB.AC.B8.EC.9E.90_.EC.9D.B8.EC.BD.94.EB.94.A9">문자 인코딩</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%AC%B8%EC%9E%90_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=1" title="부분 편집: 문자 인코딩">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="rellink relarticle mainarticle"><img alt="" data-file-height="128" data-file-width="128" height="16" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/16px-Crystal_Clear_app_xmag.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/24px-Crystal_Clear_app_xmag.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ec/Crystal_Clear_app_xmag.svg/32px-Crystal_Clear_app_xmag.svg.png 2x" width="16"/> 이 부분의 본문은 <a href="/wiki/%EB%AC%B8%EC%9E%90_%EC%9D%B8%EC%BD%94%EB%94%A9" title="문자 인코딩">문자 인코딩</a>입니다.</div>
<p>컴퓨터 및 통신 장비는 각 문자를 무언가(일련의 <a href="/wiki/%EB%B9%84%ED%8A%B8" title="비트">비트</a>로 대표되는 <a href="/wiki/%EC%A0%95%EC%88%98%ED%98%95" title="정수형">정수</a>의 양, 일반적으로는 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%EB%84%A4%ED%8A%B8%EC%9B%8C%ED%81%AC" title="컴퓨터 네트워크">네트워크</a>를 통해 전달되거나 저장 가능)에 할당하는 <a href="/wiki/%EB%AC%B8%EC%9E%90_%EC%9D%B8%EC%BD%94%EB%94%A9" title="문자 인코딩">문자 인코딩</a>을 이용하여 문자를 대표한다. 대표적인 예로 <a class="mw-disambig" href="/wiki/%EC%95%84%EC%8A%A4%ED%82%A4" title="아스키">아스키</a>, 그리고 <a href="/wiki/%EC%9C%A0%EB%8B%88%EC%BD%94%EB%93%9C" title="유니코드">유니코드</a>를 위한 <a href="/wiki/UTF-8" title="UTF-8">UTF-8</a> 인</p>
<h2><span class="mw-headline" id="char">char</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%AC%B8%EC%9E%90_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=2" title="부분 편집: char">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a class="mw-redirect" href="/wiki/C_%EC%96%B8%EC%96%B4" title="C 언어">C</a>/<a href="/wiki/C%2B%2B" title="C++">C++</a> 프로그래밍 언어에서의 <b>char</b>는 8비트 정수형 처리 변수이다. 그러나 정수형이라는 것은 CPU 입장에서는 그렇고 인간의 문자 처리를 위해 만들어졌다. character의 약자가 변수를 지정하기 위한 char로 ASCII 코드 값은 기본적으로 7비트 이므로 이것은 8비트 정수형의 부분집합일 뿐이다. 따라서 char 변수는 <a class="mw-disambig" href="/wiki/%EC%95%84%EC%8A%A4%ED%82%A4" title="아스키">아스키</a>(ASCII) 코드 처리를 할 수 있다. 기본적으로 char 변수는 정수형이기 때문에 아스키 코드의 변환 등에 계산을 통해 수행 할 수 있다. 예를 들어 'A'를 'a'로 바꾸려면 정수형 연산으로 가능하다. 따라서 8비트의 문자 코드 처리가 가능하다. 아스키 뿐만 아니라 UTF-8 코드 역시 처리가 가능하다. 문자를 처리할 때 C/C++에서는 문자열(string)의 끝에 0이라는 끝을 알리는 숫자가 들어가기 때문에 정해진 배열 크기 안에서 스트링 처리가 되어야 한다. 초기의 컴퓨터가 미국에서 개발되어 사용되어 질때, 8비트 코드 만으로 문자를 나타내는데 충분 했다. 그러나 최근의 많은 OS에서는 국제화 되면서 UNICODE 등의 16비트 체계를 많이 사용 한다. 따라서 char 변수로는 불가능 하다. char 배열 변수가 처리하고자 하는 충분한 문자열 길이를 갖더라도, UNICODE에서 알파벳의 한글자는 0+ASCII코드의 형태로 변수 영역에 저장 된다. 바로 이 숫자 때문에 한글자만 인식하는 문제가 발생 한다. 이를 해결하기 위해 wchar_t 변수를 사용하면 된다. 초기에 컴퓨터의 문자 처리는 아스키의 7비트 만으로도 충분 했다. 따라서 C언어에서 처리는 char 변수로 충분 했다. 컴퓨터가 국제화 되면서 다양한 언어의 문자를 취급할 필요가 생겼다. 국제화를 하면서 다양한 문자 코드가 있으나 16비트 UNICODE을 많이 사용한다. 따라서 이것을 처리하기 위해 <a class="mw-redirect" href="/wiki/Wchar_t" title="Wchar t">wchar t</a>가 도입되었다.</p>
<h2><span class="mw-headline" id="word_.EB.AC.B8.EC.9E.90">word 문자</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%AC%B8%EC%9E%90_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=3" title="부분 편집: word 문자">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
