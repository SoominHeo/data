<p><b>방어적 프로그래밍</b>(Defensive programming)은 예상치 못한 입력에도 한 <a href="/wiki/%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="소프트웨어">소프트웨어</a>가 계속적 기능 수행을 보장할 수 있도록 고안된 <a class="new" href="/w/index.php?title=%EB%B0%A9%EC%96%B4%EC%A0%81_%EC%84%A4%EA%B3%84&amp;action=edit&amp;redlink=1" title="방어적 설계 (없는 문서)">방어적 설계</a>의 한 형태이다. 이 생각은 <a href="/wiki/%EB%A8%B8%ED%94%BC%EC%9D%98_%EB%B2%95%EC%B9%99" title="머피의 법칙">머피의 법칙</a>이 발현할 가능성을 줄이거나 제거하는 것으로 보일 수 있다. 방어적 프로그래밍은 기법들은 한 소프트웨어가 해롭거나 부주의하게 잘못 사용되어 재앙적 결과를 가져올수도 있을 때 사용된다.</p>
<p>방어적 프로그래밍은 소프트웨어와 소스코드를 아래의 관점에서 향상시키는 한 접근법이다:</p>
<ul>
<li>일반적 품질 - <a href="/wiki/%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4_%EB%B2%84%EA%B7%B8" title="소프트웨어 버그">소프트웨어 버그</a>와 문제의 수를 줄인다.</li>
<li>그 소스 코드를 이해하기 쉽게 만든다 - 그 소스 코드는 읽기 쉽고 이해하기 쉬워서 <a class="mw-redirect" href="/wiki/%EC%BD%94%EB%93%9C_%EB%A6%AC%EB%B7%B0" title="코드 리뷰">코드 리뷰</a>(<a class="extiw" href="https://en.wikipedia.org/wiki/code_audit" title="en:code audit">code audit</a>)에서 승인된다.</li>
<li>그 소프트웨어가 예상치 못한 입력들이나 사용자의 동작들에도 예상 가능한 방식으로 동작하도록 한다.</li>
</ul>

<h2><span class="mw-headline" id=".EC.95.88.EC.A0.84.ED.95.9C_.ED.94.84.EB.A1.9C.EA.B7.B8.EB.9E.98.EB.B0.8D">안전한 프로그래밍</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B0%A9%EC%96%B4%EC%A0%81_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D&amp;action=edit&amp;section=1" title="부분 편집: 안전한 프로그래밍">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>방어적 프로그래밍은 때때로 이 버그를 줄이는 접근을 언급하는 컴퓨터 학자들에 의해 <b>안전한 프로그래밍</b>(secure programming)으로 불린다. <a href="/wiki/%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4_%EB%B2%84%EA%B7%B8" title="소프트웨어 버그">소프트웨어 버그</a>는 잠재적으로 <a class="extiw" href="https://en.wikipedia.org/wiki/code_injection" title="en:code injection">코드 주입</a>, <a href="/wiki/%EC%84%9C%EB%B9%84%EC%8A%A4_%EA%B1%B0%EB%B6%80_%EA%B3%B5%EA%B2%A9" title="서비스 거부 공격">서비스 거부 공격</a> 또는 다른 공격을 위한 <a class="extiw" href="https://en.wikipedia.org/wiki/Hacker_(computer_security)" title="en:Hacker (computer security)">크래커</a>에 의해 이용당할 수 있다.</p>
<p>방어적 프로그래밍과 보통 습관들 사이의 차이는 모든 가능한 에러 상태들을 다루고자 시도하는 프로그래머가 실제로는 그것을 거의 가정하지 않는다는 것이다. 줄여 말하면, 그 프로그래머는 한 특별한 함수 호출 또는 라이브러리가 적대적이게 동작할 수 있다고 가정하지 않고 그것을 그 코드안에서 다룬다. 한 예는 다음과 같다:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kt">int</span> <span class="nf">risky_programming</span><span class="p">(</span><span class="kt">char</span> <span class="o">*</span><span class="n">input</span><span class="p">){</span>
  <span class="kt">char</span> <span class="n">str</span><span class="p">[</span><span class="mi">1000</span><span class="o">+</span><span class="mi">1</span><span class="p">];</span>     <span class="c1">// 널 문자를 위해 하나 추가</span>
  <span class="c1">// ...</span>
  <span class="n">strcpy</span><span class="p">(</span><span class="n">str</span><span class="p">,</span> <span class="n">input</span><span class="p">);</span>   <span class="c1">// copy input</span>
  <span class="c1">// ...</span>
<span class="p">}</span>
</pre></div>
<p>이 함수는 입력이 1000 문자를 넘을 때 충돌을 일으킬 것이다. 몇몇 미숙한 프로그래머들은 사용자들이 그런 긴 입력을 넣지 않을 것이라고 가정하면서 이것이 별 문제가 아니라고 느낄지도 모른다. 한 방어적 프로그래밍을 실천하는 프로그래머는 그 버그를 허용하지 않았을 것이다, 왜냐하면 만약 그 응용 프로그램이 한 알려진 버그를 포함한다면 <a href="/wiki/%EB%A8%B8%ED%94%BC%EC%9D%98_%EB%B2%95%EC%B9%99" title="머피의 법칙">머피의 법칙</a>은 그 버그가 실제 사용 시에 발생할 것이라고 말하기 때문이다. 이 특정 버그는 <a class="mw-redirect" href="/wiki/%EB%B2%84%ED%8D%BC_%EC%98%A4%EB%B2%84%ED%94%8C%EB%A1%9C%EC%9A%B0" title="버퍼 오버플로우">버퍼 오버플로우</a>를 <a href="/wiki/%EC%B7%A8%EC%95%BD%EC%A0%90_%EA%B3%B5%EA%B2%A9" title="취약점 공격">이용</a>하여 일으킬 수 있는 한 취약성을 재연한다. 여기 이 예제에 대한 한 해결 방법이 있다.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kt">int</span> <span class="nf">secure_programming</span><span class="p">(</span><span class="kt">char</span> <span class="o">*</span><span class="n">input</span><span class="p">){</span>
  <span class="kt">char</span> <span class="n">str</span><span class="p">[</span><span class="mi">1000</span><span class="p">];</span>
  <span class="c1">// ...</span>
  <span class="n">strncpy</span><span class="p">(</span><span class="n">str</span><span class="p">,</span> <span class="n">input</span><span class="p">,</span> <span class="k">sizeof</span><span class="p">(</span><span class="n">str</span><span class="p">));</span> <span class="c1">// str의 길이를 초과하지 않도록 입력(input) 복사</span>
  <span class="n">str</span><span class="p">[</span><span class="k">sizeof</span><span class="p">(</span><span class="n">str</span><span class="p">)</span> <span class="o">-</span> <span class="mi">1</span><span class="p">]</span> <span class="o">=</span> <span class="sc">'\0'</span><span class="p">;</span>              <span class="c1">// 만약 strlen(input) == sizeof(str) 여도 strncpy가 NULL 종료 하지 않을 것</span>
  <span class="c1">// ...</span>
<span class="p">}</span>
</pre></div>
