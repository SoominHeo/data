<p><b>버그질라</b>(Bugzilla)는 원래 <a href="/wiki/%EB%AA%A8%EC%A7%88%EB%9D%BC" title="모질라">모질라</a> 프로젝트가 개발하여 사용한 <a class="mw-disambig" href="/wiki/%EC%9B%B9" title="웹">웹</a> 기반의 일반 목적의 <a href="/wiki/%EB%B2%84%EA%B7%B8_%EC%B6%94%EC%A0%81_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="버그 추적 시스템">버그 추적기</a> 및 <a class="new" href="/w/index.php?title=%ED%85%8C%EC%8A%A4%ED%8A%B8_%EC%9E%90%EB%8F%99%ED%99%94&amp;action=edit&amp;redlink=1" title="테스트 자동화 (없는 문서)">테스트 도구</a>이다. 현재 <a href="/wiki/%EB%AA%A8%EC%A7%88%EB%9D%BC_%EA%B3%B5%EC%9A%A9_%ED%97%88%EA%B0%80%EC%84%9C" title="모질라 공용 허가서">모질라 공용 허가서</a>로 <a href="/wiki/%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4_%EC%82%AC%EC%9A%A9%EA%B6%8C" title="소프트웨어 사용권">사용권</a>이 제공된다.</p>
<p>1998년에 <a class="mw-redirect" href="/wiki/%EB%84%B7%EC%8A%A4%EC%BC%80%EC%9D%B4%ED%94%84_%EC%BB%A4%EB%AE%A4%EB%8B%88%EC%BC%80%EC%9D%B4%EC%85%98%EC%8A%A4" title="넷스케이프 커뮤니케이션스">넷스케이프 커뮤니케이션스</a>가 <a href="/wiki/%EC%98%A4%ED%94%88_%EC%86%8C%EC%8A%A4_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="오픈 소스 소프트웨어">오픈 소스 소프트웨어</a>로 공개하였으며 다양한 단체가 <a href="/wiki/%EB%B2%84%EA%B7%B8_%EC%B6%94%EC%A0%81_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="버그 추적 시스템">버그 추적 시스템</a>으로 채택하였다. 버그질라는 <a class="mw-redirect" href="/wiki/%EC%9E%90%EC%9C%A0_%EB%B0%8F_%EC%98%A4%ED%94%88_%EC%86%8C%EC%8A%A4_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="자유 및 오픈 소스 소프트웨어">자유 및 오픈 소스 소프트웨어</a>와 <a href="/wiki/%EC%82%AC%EC%9C%A0_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="사유 소프트웨어">사유</a>의 프로젝트 및 제품 전반에 모두에 쓰인다.</p>

<h2><span class="mw-headline" id=".EC.97.AD.EC.82.AC">역사</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B2%84%EA%B7%B8%EC%A7%88%EB%9D%BC&amp;action=edit&amp;section=1" title="부분 편집: 역사">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>버그질라는 테리(Terry Weissman)가 1998년에 <a href="/wiki/%EB%AA%A8%EC%A7%88%EB%9D%BC_%EC%9E%AC%EB%8B%A8" title="모질라 재단">Mozilla.org</a> 프로젝트로 작성한 것이다. 원래 <a href="/wiki/Tcl" title="Tcl">Tcl</a>로 작성하였다가, 넷스케이프 초기 오픈 소스 코드의 일부가 떨어져나가면서 테리가 버그질라를 <a href="/wiki/%ED%8E%84" title="펄">펄</a>로 포팅하기로 결정하였다. 당시 펄이 더 유명한 언어가 될 것으로 보았고 사람들이 이를 계속 사용하면 좋겠다고 생각했다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>버그질라 2.0은 펄로 포팅한 뒤의 결과물이며 익명 <a href="/wiki/CVS" title="CVS">CVS</a>를 통하여 대중에게 공개된 첫 버전이다.</p>
<p>버그질라 3.0은 2007년 5월 10일에 출시되었다.</p>
<p>버그질라 4.0은 2011년 2월 15일에 출시되었다.</p>

<p>버그질라의 시스템 요구 사항은 다음과 같다:</p>
<ul>
<li>호환 가능한 <a href="/wiki/%EB%8D%B0%EC%9D%B4%ED%84%B0%EB%B2%A0%EC%9D%B4%EC%8A%A4_%EA%B4%80%EB%A6%AC_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="데이터베이스 관리 시스템">데이터베이스 관리 시스템</a></li>
<li><a href="/wiki/%ED%8E%84" title="펄">펄</a> 5</li>
<li>펄 모듈</li>
<li>호환 가능한 <a href="/wiki/%EC%9B%B9_%EC%84%9C%EB%B2%84" title="웹 서버">웹 서버</a></li>
<li>적절한 <a href="/wiki/%EB%A9%94%EC%9D%BC_%EC%A0%84%EC%86%A1_%EC%97%90%EC%9D%B4%EC%A0%84%ED%8A%B8" title="메일 전송 에이전트">메일 전송 에이전트</a>나 <a class="mw-redirect" href="/wiki/SMTP" title="SMTP">SMTP</a> 서버</li>
</ul>
