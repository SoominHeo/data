<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><b>벡터 공간 모델</b>(―空間―, <span style="font-size: smaller;"><a href="/wiki/%EC%98%81%EC%96%B4" title="영어">영어</a>:</span> <span lang="en" xml:lang="en">vector space model</span>) 또는 <b>단어 벡터 모델</b>(單語―, <span style="font-size: smaller;"><a href="/wiki/%EC%98%81%EC%96%B4" title="영어">영어</a>:</span> <span lang="en" xml:lang="en">term vector model</span>)은 텍스트 문서를 단어 색인 등의 식별자로 구성된 <a href="/wiki/%EB%B2%A1%ED%84%B0_%EA%B3%B5%EA%B0%84" title="벡터 공간">벡터</a>로 표현하는 대수적 모델이다. <a href="/wiki/%EC%A0%95%EB%B3%B4_%EA%B2%80%EC%83%89" title="정보 검색">정보 검색</a>, 정보 필터링 및 검색 엔진의 색인이나 연관도 순위에 사용된다. 이 모델이 최초로 적용된 것은 <a href="/wiki/%EC%BD%94%EB%84%AC_%EB%8C%80%ED%95%99%EA%B5%90" title="코넬 대학교">코넬 대학교</a>의 <a class="new" href="/w/index.php?title=SMART_%EC%A0%95%EB%B3%B4_%EA%B2%80%EC%83%89_%EC%8B%9C%EC%8A%A4%ED%85%9C&amp;action=edit&amp;redlink=1" title="SMART 정보 검색 시스템 (없는 문서)">SMART 정보 검색 시스템</a>이었다.</p>

<p>각각의 <a class="new" href="/w/index.php?title=%EC%B0%A8%EC%9B%90_(%EB%B2%A1%ED%84%B0%EA%B3%B5%EA%B0%84)&amp;action=edit&amp;redlink=1" title="차원 (벡터공간) (없는 문서)">차원</a>은 개별 단어에 대응된다. 어떤 단어가 문서에 포함되면, 해당 단어는 0이 아닌 벡터값을 갖는다. 단어 가중치라고도 불리는 이 값을 산출하는 방법에는 여러 가지가 있다. 가장 널리 알려진 방식은 <a href="/wiki/TF-IDF" title="TF-IDF">TF-IDF</a> 방식이다.(아래 사례 항목 참조)</p>
<p>벡터 공간 모델에서 <i>단어</i>(term)의 의미는 그 적용 대상에 따라 달라진다. 일반적으로는 하나의 단어(word)나 키워드, 또는 좀더 긴 구를 의미한다. 벡터의 차원의 크기는 <a href="/wiki/%EB%A7%90%EB%AD%89%EC%B9%98" title="말뭉치">말뭉치</a>에 포함된 단어의 개수와 같다.</p>
<p>벡터 연산은 문서들간의 유사성을 비교하는 데에도 사용될 수 있다.</p>
<h2><span class="mw-headline" id=".EC.9D.91.EC.9A.A9">응용</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B2%A1%ED%84%B0_%EA%B3%B5%EA%B0%84_%EB%AA%A8%EB%8D%B8&amp;action=edit&amp;section=2" title="부분 편집: 응용">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="floatright"><a class="image" href="/wiki/%ED%8C%8C%EC%9D%BC:Vector_space_model.jpg"><img alt="Vector space model.jpg" data-file-height="420" data-file-width="427" height="246" src="//upload.wikimedia.org/wikipedia/commons/thumb/f/ff/Vector_space_model.jpg/250px-Vector_space_model.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/f/ff/Vector_space_model.jpg/375px-Vector_space_model.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/f/ff/Vector_space_model.jpg 2x" width="250"/></a></div>
<p>키워드 검색에서 사용되는 문서의 연관도는 <a class="new" href="/w/index.php?title=%EC%9D%98%EB%AF%B8_%EC%9C%A0%EC%82%AC%EB%8F%84&amp;action=edit&amp;redlink=1" title="의미 유사도 (없는 문서)">문서 유사도</a> 이론으로 계산될 수 있다. 문서 유사도 이론에서 문서 유사도는 문서 벡터간의 각도의 편차를 이용하여 산출된다.</p>
<p>실제 적용시에는, 문서 벡터간의 각도 자체보다는 다음과 같이 각도의 <a class="mw-redirect" href="/wiki/%EC%BD%94%EC%82%AC%EC%9D%B8" title="코사인">코사인</a>값을 이용하는 것이 더 간단하다.</p>

<h2><span class="mw-headline" id=".EC.9E.A5.EC.A0.90">장점</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B2%A1%ED%84%B0_%EA%B3%B5%EA%B0%84_%EB%AA%A8%EB%8D%B8&amp;action=edit&amp;section=4" title="부분 편집: 장점">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>벡터 공간 모델은 <a class="mw-redirect" href="/wiki/%EB%B6%88%EB%A6%B0_%EB%AA%A8%EB%8D%B8" title="불린 모델">불린 모델</a>에 비해 다음과 같은 장점을 지닌다.</p>
<ol>
<li>선형 대수에 기반한 단순한 모델</li>
<li>비(非)이진 단어 가중치</li>
<li>문서와 질의간의 유사도를 연속적인 값으로 계산 가능</li>
<li>연관도에 의한 문서 순위 결정 가능</li>
<li>부분 일치 고려 가능</li>
</ol>
<h2><span class="mw-headline" id=".ED.95.9C.EA.B3.84">한계</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B2%A1%ED%84%B0_%EA%B3%B5%EA%B0%84_%EB%AA%A8%EB%8D%B8&amp;action=edit&amp;section=5" title="부분 편집: 한계">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>벡터 공간 모델은 다음과 같은 한계를 지닌다.</p>
<ol>
<li>길이가 긴 문서는 낮은 유사도값(작은 <a href="/wiki/%EC%8A%A4%EC%B9%BC%EB%9D%BC%EA%B3%B1" title="스칼라곱">스칼라곱</a>과 <a class="new" href="/w/index.php?title=%EC%B0%A8%EC%9B%90%EC%9D%98_%EC%A0%80%EC%A3%BC&amp;action=edit&amp;redlink=1" title="차원의 저주 (없는 문서)">큰 차원</a>)으로 인해 제대로 표현할 수 없다.</li>
<li>검색 키워드는 문서내의 단어와 정확히 일치해야 한다. 부분 일치하는 단어는 <a href="/wiki/%EA%B1%B0%EC%A7%93_%EC%96%91%EC%84%B1" title="거짓 양성">거짓 양성</a> 결과를 가져올 수 있다.</li>
<li>의미 유사성 관련 ― 유사한 내용을 담고 있더라도 사용된 단어들 다르다면 그 문서들의 유사성은 매우 낮게 나오며, 이는 <a href="/wiki/%EA%B1%B0%EC%A7%93_%EC%9D%8C%EC%84%B1" title="거짓 음성">거짓 음성</a> 결과에 해당한다.</li>
<li>단어가 나타나는 순서에 관한 정보가 활용되지 못한다.</li>
<li>단어들이 통계적으로 서로 독립적이라는 이론적 가정에 기반하고 있다.</li>
<li>가중치 계산 방식이 직관적이기는 하지만 포멀한 형식이 아니다.</li>
</ol>
<p>벡터 공간 모델의 이러한 여러 문제점들은 다양한 방법으로 극복될 수 있다. 여기에는 <a class="mw-redirect" href="/wiki/%ED%8A%B9%EC%9E%87%EA%B0%92_%EB%B6%84%ED%95%B4" title="특잇값 분해">특잇값 분해</a>와 같은 수학적 기법과 <a href="/wiki/%EC%9B%8C%EB%93%9C%EB%84%B7" title="워드넷">워드넷</a>과 같은 어휘 데이터베이스 활용이 포함된다.</p>
<h2><span class="mw-headline" id=".EA.B4.80.EB.A0.A8_.EB.AA.A8.EB.8D.B8">관련 모델</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B2%A1%ED%84%B0_%EA%B3%B5%EA%B0%84_%EB%AA%A8%EB%8D%B8&amp;action=edit&amp;section=6" title="부분 편집: 관련 모델">편집</a><span class="mw-editsection-bracket">]</span></span></h2>

<li>Weka: WordVectords 및 단어주머니 모델을 자바로 구현한 널리 알려진 데이터 마이닝 패키지</li>
<li><a class="external text" href="http://codingplayground.blogspot.com/2010/03/compressed-vector-space.html" rel="nofollow">Compressed vector space in C++</a> by Antonio Gulli</li>
