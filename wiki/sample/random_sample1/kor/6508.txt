<p><a href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="컴퓨터 프로그래밍">컴퓨터 프로그래밍</a>에서 <b>변수 영역</b>은 변수가 유효성을 갖는 영역을 가리킨다. 프로그램은 영역을 벗어난 변수를 가리킬 수 없다.</p>
<p>변수 영역을 지정하는 규칙은 크게 <b>정적 영역 규칙</b>과 <b>동적 영역 규칙</b>으로 나뉜다. 정적 영역 규칙을 따르는 언어에서는 변수의 유효 영역이 <a href="/wiki/%EC%BB%B4%ED%8C%8C%EC%9D%BC_%ED%83%80%EC%9E%84" title="컴파일 타임">컴파일 타임</a>에 결정되며, 동적 영역 규칙은 <a href="/wiki/%EB%9F%B0%ED%83%80%EC%9E%84" title="런타임">실행 시간</a>에 결정된다. 대부분의 현대 언어는 정적 영역 규칙을 따르지만, 매크로를 컴파일할때는 동적 영역 규칙을 따르기도 한다.</p>

<h2><span class="mw-headline" id=".EC.A2.85.EB.A5.98">종류</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B3%80%EC%88%98_%EC%98%81%EC%97%AD&amp;action=edit&amp;section=1" title="부분 편집: 종류">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id=".EC.A0.84.EC.97.AD">전역</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B3%80%EC%88%98_%EC%98%81%EC%97%AD&amp;action=edit&amp;section=2" title="부분 편집: 전역">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>가장 간단한 변수 영역은 소스 코드 상의 모든 곳에서 사용할 수 있는 <a href="/wiki/%EC%A0%84%EC%97%AD_%EB%B3%80%EC%88%98" title="전역 변수">전역 변수</a>이다. 전역 변수는 많은 언어에서 나쁜 관습으로 취급되지만, 그런 경우에도 함수명이나 클래스 명은 대부분 전역에서 접근할 수 있다. <a href="/wiki/%EC%9D%B4%EB%A6%84%EA%B3%B5%EA%B0%84" title="이름공간">이름공간</a>을 사용하면 전역 영역에서 변수나 함수명이 충돌하는 것을 막을 수 있다.</p>
<h3><span class="mw-headline" id=".EB.AA.A8.EB.93.88_.EC.98.81.EC.97.AD">모듈 영역</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B3%80%EC%88%98_%EC%98%81%EC%97%AD&amp;action=edit&amp;section=3" title="부분 편집: 모듈 영역">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a class="new" href="/w/index.php?title=%EB%AA%A8%EB%93%88%EB%9D%BC&amp;action=edit&amp;redlink=1" title="모듈라 (없는 문서)">모듈라</a>나 <a href="/wiki/%ED%8C%8C%EC%9D%B4%EC%8D%AC" title="파이썬">파이썬</a>과 같이 <a href="/wiki/%EB%AA%A8%EB%93%88%EC%84%B1_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D)" title="모듈성 (프로그래밍)">모듈</a>을 지원하는 프로그래밍 언어에서는 모듈 단위 변수를 선언할 수 있다. 이 변수들은 같은 모듈 안에서만 접근할 수 있으며, 모듈 밖에서는 변수를 볼 수 없다.</p>
<p><a href="/wiki/C%2B%2B" title="C++">C++</a> 등 모듈 개념을 지원하지 않는 일부 <a href="/wiki/%EA%B0%9D%EC%B2%B4_%EC%A7%80%ED%96%A5_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="객체 지향 프로그래밍">객체 지향</a> 언어에서는 <a class="mw-disambig" href="/wiki/%ED%81%B4%EB%9E%98%EC%8A%A4" title="클래스">클래스</a>가 모듈 영역의 역할을 대신하기도 한다. 모듈과 객체의 개념을 모두 지원하는 파이썬의 경우 모듈 단위 영역과 클래스 단위 영역을 둘 다 지원한다.</p>
<h3><span class="mw-headline" id=".ED.8C.8C.EC.9D.BC_.EC.98.81.EC.97.AD">파일 영역</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B3%80%EC%88%98_%EC%98%81%EC%97%AD&amp;action=edit&amp;section=4" title="부분 편집: 파일 영역">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>파일 영역은 <a href="/wiki/C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="C (프로그래밍 언어)">C</a>와 <a href="/wiki/C%2B%2B" title="C++">C++</a> 및 그 유사 언어에서만 지원하는 개념이다. 이런 언어에서는, 소스 파일의 최상위에서 선언된 변수나 함수는 해당 파일 전체에서 접근할 수 있고, 다른 파일에서는 접근할 수 없다. 파일 영역은 보다 원시적인 형태의 모듈 영역으로 여겨진다.</p>
<h3><span class="mw-headline" id=".ED.95.A8.EC.88.98_.EC.98.81.EC.97.AD">함수 영역</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B3%80%EC%88%98_%EC%98%81%EC%97%AD&amp;action=edit&amp;section=5" title="부분 편집: 함수 영역">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="mw-highlight mw-content-ltr" dir="ltr" style="float:right;margin-left:1em">
<pre>
<span class="k">def</span> <span class="nf">square</span><span class="p">(</span><span class="n">n</span><span class="p">):</span>
  <span class="k">return</span> <span class="n">n</span> <span class="o">*</span> <span class="n">n</span>
 
<span class="k">def</span> <span class="nf">sum_of_squares</span><span class="p">(</span><span class="n">n</span><span class="p">):</span>
  <span class="n">total</span> <span class="o">=</span> <span class="mi">0</span> 
  <span class="n">i</span> <span class="o">=</span> <span class="mi">0</span>
  <span class="k">while</span> <span class="n">i</span> <span class="o">&lt;=</span> <span class="n">n</span><span class="p">:</span>
    <span class="n">total</span> <span class="o">+=</span> <span class="n">square</span><span class="p">(</span><span class="n">i</span><span class="p">)</span>
    <span class="n">i</span> <span class="o">+=</span> <span class="mi">1</span>
  <span class="k">return</span> <span class="n">total</span>
</pre></div>
<p>대부분의 프로그래밍 언어는 함수 내에서만 유효한 <a class="new" href="/w/index.php?title=%EC%A7%80%EC%97%AD_%EB%B3%80%EC%88%98&amp;action=edit&amp;redlink=1" title="지역 변수 (없는 문서)">지역 변수</a>를 제공한다. 함수 영역을 사용하는 지역 변수는 함수가 반환되면 더 이상 사용할 수 없다. 오른쪽의 파이썬 코드에서 변수 <code>n</code>은 함수의 인자로, 함수 밖에서는 사용할 수 없다. 또한 변수 <code>total</code>는 인자는 아니지만 함수 내에서 선언된 지역 변수로, 마찬가지로 함수 밖에서는 사용할 수 없다.</p>
<p>정적 영역 규칙에서는 함수가 다른 함수를 호출할 경우 변수는 영역 밖으로 벗어나게 되며, 해당 변수를 사용할 수 없게 된다. 그러나 동적 영역 규칙에서는 다른 함수가 호출되더라도 변수가 여전히 사용 가능하며, 함수가 최종적으로 종료될 때 비로소 변수가 영역 밖으로 벗어나게 된다. 동적 영역 규칙은 이런 특성으로 인해 소스 코드 단위에서 변수의 사용 가능 여부를 분석할 수 없고, 실행 시의 동작에 따라 변수의 사용 가능 여부가 바뀌게 된다. 이것은 종종 찾아내기 힘든 버그를 만들어낼 수 있기 때문에, 현대 프로그래밍 언어들은 대부분 정적 영역 규칙을 사용한다.</p>
<p>정적 영역 규칙의 경우, 함수 <code>square</code>의 인자 <code>n</code>과 함수 <code>sum_of_squares</code>의 인자 <code>n</code>은 이름은 같지만 서로 별개의 영역에 선언되어 있다. 따라서 두 변수는 완전히 독립되어 서로 어떤 영향도 끼치지 않는다.</p>
<p>만약 오른쪽 코드에서 동적 영역 규칙이 사용될 경우, <code>sun_of_squares</code>의 인자 <code>n</code>은 <code>square</code>의 인자 <code>n</code>과 겹치게 된다. 이 경우 이름 마스킹이 일어나, 기존의 <code>n</code> 변수를 가렸다가, 함수가 반환될 때 다시 원래 변수를 가리키게 될 것이다.</p>
<h3><span class="mw-headline" id=".EB.B8.94.EB.A1.9D_.EC.98.81.EC.97.AD">블록 영역</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B3%80%EC%88%98_%EC%98%81%EC%97%AD&amp;action=edit&amp;section=6" title="부분 편집: 블록 영역">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="mw-highlight mw-content-ltr" dir="ltr" style="float:right;margin-left:1em">
<pre>
<span class="kt">unsigned</span> <span class="kt">int</span> <span class="nf">sum_of_squares</span><span class="p">(</span><span class="kt">unsigned</span> <span class="kt">int</span> <span class="n">n</span><span class="p">)</span> <span class="p">{</span>
  <span class="kt">unsigned</span> <span class="kt">int</span> <span class="n">ret</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span>
  <span class="k">for</span><span class="p">(</span><span class="kt">unsigned</span> <span class="kt">int</span> <span class="n">i</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span> <span class="n">i</span> <span class="o">&lt;=</span> <span class="n">n</span><span class="p">;</span> <span class="o">++</span><span class="n">i</span><span class="p">)</span> <span class="p">{</span>
    <span class="kt">unsigned</span> <span class="kt">int</span> <span class="n">n_squared</span> <span class="o">=</span> <span class="n">i</span> <span class="o">*</span> <span class="n">i</span><span class="p">;</span>
    <span class="n">ret</span> <span class="o">+=</span> <span class="n">n_squared</span><span class="p">;</span>
  <span class="p">}</span>
  <span class="k">return</span> <span class="n">ret</span><span class="p">;</span>
<span class="p">}</span>
</pre></div>
<p><a href="/wiki/%EC%95%8C%EA%B3%A8_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="알골 (프로그래밍 언어)">알골</a>과 그의 자손인 <a href="/wiki/C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="C (프로그래밍 언어)">C</a>, 그리고 그에 영향을 받은 많은 현대 언어들은 블록 단위 지역 변수를 지원한다.</p>
<p>예를 들어 오른쪽의 C 코드에서, 변수 <code>i</code>는 <code>for</code> 문 안에서만 유효한 루프 변수이다. 또한 <code>n_squared</code>는 블록 안에서 선언된 변수로, 블록 안에서만 유효하다. 한편 이 블록은 함수 안에 들어 있으므로, 블록 바깥에서 선언된 변수인 <code>ret</code>는 블록 안에서도 사용 가능하다. 블록 영역은 여러 개의 블록 안에서 선언된 변수들이 서로 간섭하지 않도록 하여, 에러의 가능성을 줄여 준다.</p>
<p>블록 영역 변수는 오른쪽 코드에서처럼 루프나 조건문에서 유용하게 사용할 수 있지만, 변수 영역을 제한하기 위한 용도로 블록을 따로 선언하는 것도 가능하다.</p>
<h3><span class="mw-headline" id=".ED.91.9C.ED.98.84.EC.8B.9D_.EC.98.81.EC.97.AD">표현식 영역</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B3%80%EC%88%98_%EC%98%81%EC%97%AD&amp;action=edit&amp;section=7" title="부분 편집: 표현식 영역">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>대부분의 <a href="/wiki/%ED%95%A8%EC%88%98%ED%98%95_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="함수형 프로그래밍">함수형 프로그래밍</a> 언어는 <code>let</code> 문을 지원한다. 예를 들어 <a href="/wiki/ML_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="ML (프로그래밍 언어)">ML</a>에서 <code>let val x = f() in x * x end</code> 와 같은 표현식에서, 변수 x는 이 표현식 안에서만 유효하다. 이와 같은 표현식은 <code>f()</code>를 두번 호출하는 낭비를 막기 위해 쓰일 수 있다.</p>
<h2><span class="mw-headline" id=".EC.A0.95.EC.A0.81_.EC.98.81.EC.97.AD_.EA.B7.9C.EC.B9.99">정적 영역 규칙</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B3%80%EC%88%98_%EC%98%81%EC%97%AD&amp;action=edit&amp;section=8" title="부분 편집: 정적 영역 규칙">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><i>정적 영역 규칙</i>은 변수의 유효 영역이 소스 코드 상의 구조를 바탕으로 결정된다. 이로 인해 변수 영역은 실행 시간의 <a class="new" href="/w/index.php?title=%ED%98%B8%EC%B6%9C_%EC%8A%A4%ED%83%9D&amp;action=edit&amp;redlink=1" title="호출 스택 (없는 문서)">호출 스택</a>이나 언어의 구현에 독립적이다. 이런 규칙에서는 변수의 유효 영역을 소스 코드만 보고 알아낼 수 있으므로, <i>문법적 영역 규칙</i>(lexical scoping rule)이라고도 한다. 정적 영역 규칙은 <a href="/wiki/%EC%95%8C%EA%B3%A8_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="알골 (프로그래밍 언어)">알골</a>과 그의 자손인 <a href="/wiki/C_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="C (프로그래밍 언어)">C</a>, 그리고 대부분의 현대 프로그래밍 언어에서 사용된다. 이 규칙은 <a href="/wiki/%EA%B5%AC%EC%A1%B0%EC%A0%81_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="구조적 프로그래밍">구조적 프로그래밍</a>에서 변수의 수명 주기를 쉽게 알아볼 수 있게 하여 프로그램의 <a href="/wiki/%EB%AA%A8%EB%93%88%EC%84%B1_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D)" title="모듈성 (프로그래밍)">모듈성</a>을 증가시킨다.</p>
<p>함수를 값으로 반환할 수 있는 <a href="/wiki/%ED%95%A8%EC%88%98%ED%98%95_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="함수형 프로그래밍">함수형 프로그래밍</a> 언어의 경우 정적 영역 규칙은 좀더 복잡하다. 함수를 선언할 당시 자신의 바깥 영역에 있는 변수를 사용할 수 있기 때문이다. 이 경우 내부에 중첩된 함수는 함수 코드와 함수 선언 시의 변수 환경을 같이 갖고 있게 되며, 이런 자료구조를 <a class="new" href="/w/index.php?title=%ED%81%B4%EB%A1%9C%EC%A0%80_(%EC%BB%B4%ED%93%A8%ED%84%B0_%EA%B3%BC%ED%95%99)&amp;action=edit&amp;redlink=1" title="클로저 (컴퓨터 과학) (없는 문서)">클로저</a> 라고 부른다.</p>
<h2><span class="mw-headline" id=".EB.8F.99.EC.A0.81_.EC.98.81.EC.97.AD_.EA.B7.9C.EC.B9.99">동적 영역 규칙</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EB%B3%80%EC%88%98_%EC%98%81%EC%97%AD&amp;action=edit&amp;section=9" title="부분 편집: 동적 영역 규칙">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><i>동적 영역 규칙</i>을 사용하는 언어는 대개 호출 스택에 변수의 바인딩이 포함되어 있다. 변수가 선언될 때 스택의 맨 위에 각 변수가 선언되며, 프로그램의 흐름이 변수 영역을 빠져나갈 때 스택의 맨 위가 팝된다. 호출 스택은 <a href="/wiki/%EB%9F%B0%ED%83%80%EC%9E%84" title="런타임">런타임</a>에만 동작하므로, 컴파일 타임에는 동적 영역 규칙을 실행할 수 없다.</p>
<p>동적 영역 규칙을 사용하는 언어로는 <a href="/wiki/%ED%8E%84" title="펄">펄</a>, <a href="/wiki/%EC%BB%A4%EB%A8%BC_%EB%A6%AC%EC%8A%A4%ED%94%84" title="커먼 리스프">커먼 리스프</a> 등이 있다. 또한 <a class="mw-redirect" href="/wiki/C_%EC%96%B8%EC%96%B4_%EC%A0%84%EC%B2%98%EB%A6%AC%EA%B8%B0" title="C 언어 전처리기">C 언어 전처리기</a>를 비롯, 대부분의 언어에서 <a href="/wiki/%EB%A7%A4%ED%81%AC%EB%A1%9C_(%EC%BB%B4%ED%93%A8%ED%84%B0_%EA%B3%BC%ED%95%99)" title="매크로 (컴퓨터 과학)">매크로</a>의 변수 영역은 동적 영역 규칙과 유사한 규칙을 따른다.</p>
