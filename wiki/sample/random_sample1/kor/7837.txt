<p><b>소스 코드</b>(<span lang="en" xml:lang="en">source code</span>) 또는 <b>원시 코드</b>란 컴퓨터 프로그램을 (사람이 읽을 수 있는) <a href="/wiki/%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4" title="프로그래밍 언어">프로그래밍 언어</a>로 기술한 글을 말한다. 한 개, 또는 여러 개의 텍스트 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%8C%8C%EC%9D%BC" title="컴퓨터 파일">파일</a>로 구성되어 있다. 현대 소프트웨어 개발에서 <a href="/wiki/%EA%B8%B0%EA%B3%84%EC%96%B4" title="기계어">기계어</a>는 극히 일부 영역에서만 쓰이며, 대부분 <a class="mw-redirect" href="/wiki/%EA%B3%A0%EA%B8%89_%EC%96%B8%EC%96%B4" title="고급 언어">고급 언어</a>로 된 소스 코드를 <a href="/wiki/%EC%BB%B4%ED%8C%8C%EC%9D%BC%EB%9F%AC" title="컴파일러">컴파일</a>하여 개발한다. 소프트웨어와 이에 동반하는 소스 코드는 일반적으로 크게 <a href="/wiki/%EC%9E%90%EC%9C%A0_%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="자유 소프트웨어">자유 소프트웨어</a>와 <a class="mw-redirect" href="/wiki/%ED%81%B4%EB%A1%9C%EC%A6%88%EB%93%9C_%EC%86%8C%EC%8A%A4" title="클로즈드 소스">사유 소프트웨어</a> 가운데 하나의 라이선스를 지닌다.</p>

<h2><span class="mw-headline" id=".EB.AA.A9.EC.A0.81">목적</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%86%8C%EC%8A%A4_%EC%BD%94%EB%93%9C&amp;action=edit&amp;section=1" title="부분 편집: 목적">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>소스 코드는 주로 실행 프로그램을 만드는 과정을 입력하는 데 이용된다. (이를테면 <a href="/wiki/%EC%BB%B4%ED%8C%8C%EC%9D%BC%EB%9F%AC" title="컴파일러">컴파일</a>이나 <a href="/wiki/%EC%9D%B8%ED%84%B0%ED%94%84%EB%A6%AC%ED%84%B0" title="인터프리터">해석</a> 과정) 또, 사람들 사이에 <a href="/wiki/%EC%95%8C%EA%B3%A0%EB%A6%AC%EC%A6%98" title="알고리즘">알고리즘</a>을 주고 받는 방식으로 이용되기도 한다. (이를테면 책의 <a class="new" href="/w/index.php?title=%EC%8A%A4%EB%8B%88%ED%8E%AB&amp;action=edit&amp;redlink=1" title="스니펫 (없는 문서)">코드 조각</a>)<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p><a href="/wiki/%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%A8%B8" title="프로그래머">프로그래머</a>는 프로그래밍 기술을 배우려면 기존에 있던 소스 코드를 살펴보는 것이 도움이 된다. 개발자들 사이의 소스 공유는 프로그래밍 기술의 성숙 요소에 기여하는 역할을 하며 자주 인용된다. 일부 사람들은 소스 코드를 풍부한 <a class="new" href="/w/index.php?title=%ED%99%94%EC%9E%AC_(%EC%98%88%EC%88%A0)&amp;action=edit&amp;redlink=1" title="화재 (예술) (없는 문서)">화재</a>로 생각한다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>소프트웨어를 다른 <a class="mw-redirect" href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%94%8C%EB%9E%AB%ED%8F%BC" title="컴퓨터 플랫폼">컴퓨터 플랫폼</a>에 <a class="mw-redirect" href="/wiki/%ED%8F%AC%ED%8C%85" title="포팅">포팅</a>하는 것은 일반적으로 소스 코드 없이는 불가능하다. <a class="new" href="/w/index.php?title=%EC%9D%B4%EC%A7%84_%EB%B2%88%EC%97%AD&amp;action=edit&amp;redlink=1" title="이진 번역 (없는 문서)">이진 번역</a>과 원본 플랫폼의 에뮬레이션과 같이 이용할 수 있는 포팅 옵션들이 있다.</p>
<p>실행 프로그램의 <a class="mw-redirect" href="/wiki/%EB%94%94%EC%BB%B4%ED%8C%8C%EC%9D%BC%EB%9F%AC" title="디컴파일러">디컴파일</a>은 <a href="/wiki/%EA%B3%A0%EA%B8%89_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4" title="고급 프로그래밍 언어">고급 언어</a>에서나 <a href="/wiki/%EC%96%B4%EC%85%88%EB%B8%94%EB%A6%AC%EC%96%B4" title="어셈블리어">어셈블리어</a>로 소스 코드를 만들어내는 데 쓰인다.</p>
