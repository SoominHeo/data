<h2><span class="mw-headline" id=".EC.9C.A0.ED.98.95">유형</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=1" title="부분 편집: 유형">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>장기 스케줄링</li>
<li>중기 스케줄링</li>
<li>단기 스케줄링</li>
</ul>
<h2><span class="mw-headline" id=".EB.8B.A8.EA.B3.84">단계</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=2" title="부분 편집: 단계">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>1단계 스케줄링</li>
</ul>
<dl>
<dd>작업 스케줄링(Job scheduling)이라고도 한다. 어느 작업부터 시스템 내의 자원들을 실제로 사용할 수 있도록 할지를 결정한다. 작업들이 시스템에 들어오는 것을 승인하는 것이기 때문에 승인 스케줄링(Admission scheduling)이라고도 한다.</dd>
</dl>
<ul>
<li>2단계 스케줄링</li>
</ul>
<dl>
<dd>어느 프로세스부터 CPU를 차지할 수 있게 할지를 결정한다. 프로세스들을 보류시키고 다시 활성화시키는 기법을 사용하여 시스템에 대한 단기적인 부하를 조절한다. 그렇게 함으로써 시스템을 적절히 운영할 수 있다. 작업 승인(1단계)와 CPU 배당(3단계) 사이의 완충 작용을 한다.</dd>
</dl>
<ul>
<li>3단계 스케줄링</li>
</ul>
<dl>
<dd><a href="/wiki/%EC%A4%91%EC%95%99_%EC%B2%98%EB%A6%AC_%EC%9E%A5%EC%B9%98" title="중앙 처리 장치">CPU</a>가 사용 가능한 경우 어느 프로세스에게 배당할지를 결정한다.</dd>
</dl>
<h2><span class="mw-headline" id=".EA.B2.B0.EC.A0.95_.EC.8B.9C.EC.A0.90">결정 시점</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=3" title="부분 편집: 결정 시점">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>CPU 스케줄링의 결정 시점은 다음과 같은 프로세스의 상태 변화가 있을 때이다.</p>
<ol>
<li>수행 → 대기</li>
<li>수행 → 준비</li>
<li>대기 → 준비</li>
<li>수행 → 종료</li>
</ol>
<h2><span class="mw-headline" id=".EB.B9.84.EC.84.A0.EC.A0.90.ED.98.95.EA.B3.BC_.EC.84.A0.EC.A0.90.ED.98.95">비선점형과 선점형</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=4" title="부분 편집: 비선점형과 선점형">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>스케줄링 적용 시점에 따라 비선점형과 선점형의 2가지로 구분할 수 있다. 비선점형은 위의 결정 시점 중 1번과 4번의 상황에서만 수행되며, 선점형은 1번에서 4번까지 모든 상황에서 수행된다.</p>
<ul>
<li>비선점형 스케줄링(Non-preemptive Scheduling) : 어떤 프로세스가 CPU를 할당 받으면 그 프로세스가 종료되거나 입출력 요구가 발생하여 자발적으로 중지될 때까지 계속 실행되도록 보장한다. 순서대로 처리되는 공정성이 있고 다음에 처리해야 할 프로세스와 관계없이 응답 시간을 예상할 수 있으며 선점 방식보다 스케줄러 호출 빈도가 낮고 <a href="/wiki/%EB%AC%B8%EB%A7%A5_%EA%B5%90%ED%99%98" title="문맥 교환">문맥 교환</a>에 의한 <a href="/wiki/%EC%98%A4%EB%B2%84%ED%97%A4%EB%93%9C" title="오버헤드">오버헤드</a>가 적다. <a href="/wiki/%EC%9D%BC%EA%B4%84_%EC%B2%98%EB%A6%AC" title="일괄 처리">일괄 처리</a> 시스템에 적합하며, CPU 사용 시간이 긴 하나의 프로세스가 CPU 사용 시간이 짧은 여러 프로세스를 오랫동안 대기시킬 수 있으므로, 처리율이 떨어질 수 있다는 단점이 있다.<sup class="reference" id="cite_ref-jeongik_1-0"><a href="#cite_note-jeongik-1">[1]</a></sup></li>
</ul>
<ul>
<li>선점형 스케줄링(Preemptive Scheduling) : 어떤 프로세스가 CPU를 할당받아 실행 중에 있어도 다른 프로세스가 실행 중인 프로세스를 중지하고 CPU를 강제로 점유할 수 있다. 모든 프로세스에게 CPU 사용 시간을 동일하게 부여할 수 있다. 빠른 응답시간을 요하는 대화식 시분할 시스템에 적합하며 긴급한 프로세서를 제어할 수 있다. '운영 체제가 프로세서 자원을 선점'하고 있다가 각 프로세스의 요청이 있을 때 특정 요건들을 기준으로 자원을 배분하는 방식이다.<sup class="reference" id="cite_ref-jeongik_1-1"><a href="#cite_note-jeongik-1">[1]</a></sup></li>
</ul>
<h2><span class="mw-headline" id=".EC.A0.95.EC.A0.81_.EC.8A.A4.EC.BC.80.EC.A4.84.EB.A7.81.EA.B3.BC_.EB.8F.99.EC.A0.81_.EC.8A.A4.EC.BC.80.EC.A4.84.EB.A7.81">정적 스케줄링과 동적 스케줄링</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=5" title="부분 편집: 정적 스케줄링과 동적 스케줄링">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>프로세스의 우선순위 변동 여부에 따라 정적 스케줄링과 동적 스케줄링으로 구분할 수 있다.</p>
<ul>
<li>정적 스케줄링(Static Scheduling) : 프로세스에 부여된 우선순위가 바뀌지 않는다. 고정우선순위 스케줄링이라고도 한다.</li>
</ul>
<ul>
<li>동적 스케줄링(Dynamic Scheduling) : 스케줄링 과정에서 프로세스의 우선순위를 변동시킨다. 유동우선순위 스케줄링이라고도 한다.</li>
</ul>
<h2><span class="mw-headline" id=".EC.8A.A4.EC.BC.80.EC.A4.84.EB.A7.81_.EC.95.8C.EA.B3.A0.EB.A6.AC.EC.A6.98">스케줄링 알고리즘</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=6" title="부분 편집: 스케줄링 알고리즘">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>스케줄링 알고리즘을 구현할 때 스케줄링의 여러 가지 목적 중 서로 상반되는 것이 있을 수 있다. 알고리즘 작성에는 일괄 처리인지와 대화형인지와, CPU, I/O의 사용 비율, 우선 순위 부여 여부와 부여할 경우 해당 프로세스의 선점 정도, <a href="/wiki/%ED%8E%98%EC%9D%B4%EC%A7%80_%EB%B6%80%EC%9E%AC" title="페이지 부재">페이지 부재</a> 정도가 고려된다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>스케줄링 알고리즘에는 다음과 같은 종류가 있다.</p>
<ul>
<li>비선점 프로세스 스케줄링</li>
</ul>
<ol>
<li><a class="mw-redirect" href="/wiki/FCFS_%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81" title="FCFS 스케줄링">FCFS 스케줄링</a>(First Come First Served Scheduling)</li>
<li><a class="mw-redirect" href="/wiki/SJF_%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81" title="SJF 스케줄링">SJF 스케줄링</a>(Shortest Job First Scheduling)</li>
<li><a href="/wiki/HRRN_%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81" title="HRRN 스케줄링">HRRN 스케줄링</a>(Highest Response Ratio Next Scheduling)</li>
</ol>
<ul>
<li>선점 프로세스 스케줄링</li>
</ul>
<ol>
<li><a class="mw-redirect" href="/wiki/RR_%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81" title="RR 스케줄링">RR 스케줄링</a>(Round Robin Scheduling)</li>
<li><a class="mw-redirect" href="/wiki/SRTF_%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81" title="SRTF 스케줄링">SRTF 스케줄링</a>(Shortest Remaining-Time First Scheduling)</li>
<li><a href="/wiki/%EB%8B%A4%EB%8B%A8%EA%B3%84_%ED%81%90_%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81" title="다단계 큐 스케줄링">다단계 큐 스케줄링</a>(Multilevel Queue Scheduling)</li>
<li><a class="mw-redirect" href="/wiki/%EB%8B%A4%EB%8B%A8%EA%B3%84_%ED%94%BC%EB%93%9C%EB%B0%B1_%ED%81%90_%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81" title="다단계 피드백 큐 스케줄링">다단계 피드백 큐 스케줄링</a>(Multilevel Feedback Queue Scheduling)</li>
<li><a class="mw-redirect" href="/wiki/RM_%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81" title="RM 스케줄링">RM 스케줄링</a>(Rate Monotonic Scheduling)</li>
<li><a class="mw-redirect" href="/wiki/EDF_%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81" title="EDF 스케줄링">EDF 스케줄링</a>(Earliest Deadline First Scheduling)</li>
</ol>
<h2><span class="mw-headline" id=".ED.8F.89.EA.B0.80_.EA.B8.B0.EC.A4.80">평가 기준</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%EC%BC%80%EC%A4%84%EB%A7%81_(%EC%BB%B4%ED%93%A8%ED%8C%85)&amp;action=edit&amp;section=7" title="부분 편집: 평가 기준">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>스케줄링 알고리즘은 다음과 같은 기준을 통해 평가할 수 있다.</p>
<ul>
<li><a class="mw-redirect" href="/wiki/CPU_%EC%82%AC%EC%9A%A9%EB%A5%A0" title="CPU 사용률">CPU 사용률</a>(CPU Utilization) : 전체 시스템 시간 중 CPU가 작업을 처리하는 시간의 비율.</li>
<li><a href="/wiki/%EC%8A%A4%EB%A3%A8%ED%92%8B" title="스루풋">처리량</a>(Throughput) : CPU가 단위 시간당 처리하는 프로세스의 개수.</li>
<li><a href="/wiki/%EC%9D%91%EB%8B%B5_%EC%8B%9C%EA%B0%84" title="응답 시간">응답 시간</a>(Response Time) : <a class="new" href="/w/index.php?title=%EB%8C%80%ED%99%94%EC%8B%9D_%EC%8B%9C%EC%8A%A4%ED%85%9C&amp;action=edit&amp;redlink=1" title="대화식 시스템 (없는 문서)">대화식 시스템</a>에서 요청 후 응답이 오기 시작할 때까지의 시간.</li>
<li>대기 시간(Waiting Time) : 프로세스가 준비 큐 내에서 대기하는 시간의 총합.</li>
<li>반환 시간(Turnaround Time) : 프로세스가 시작해서 끝날 때까지 걸리는 시간.</li>
</ul>
