<p><b>스프링 프레임워크</b>(Spring Framework)는 <a href="/wiki/%EC%9E%90%EB%B0%94_(%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4_%ED%94%8C%EB%9E%AB%ED%8F%BC)" title="자바 (소프트웨어 플랫폼)">자바 플랫폼</a>을 위한 <a class="mw-redirect" href="/wiki/%EC%98%A4%ED%94%88%EC%86%8C%EC%8A%A4" title="오픈소스">오픈소스</a> <a href="/wiki/%EC%95%A0%ED%94%8C%EB%A6%AC%EC%BC%80%EC%9D%B4%EC%85%98_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC" title="애플리케이션 프레임워크">애플리케이션 프레임워크</a>로서 간단히 <b>스프링</b>(Spring)이라고도 불린다. 동적인 웹 사이트를 개발하기 위한 여러 가지 서비스를 제공하고 있다. 대한민국 <a href="/wiki/%EA%B3%B5%EA%B3%B5%EA%B8%B0%EA%B4%80" title="공공기관">공공기관</a>의 웹 서비스 개발 시 사용을 권장하고 있는 <a href="/wiki/%EC%A0%84%EC%9E%90%EC%A0%95%EB%B6%80_%ED%91%9C%EC%A4%80%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC" title="전자정부 표준프레임워크">전자정부 표준프레임워크</a>의 기반 기술로서 쓰이고 있다.</p>

<h2><span class="mw-headline" id=".EC.97.AD.EC.82.AC">역사</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%ED%94%84%EB%A7%81_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=1" title="부분 편집: 역사">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>로드 존슨이 2002년에 출판한 자신의 저서인 <a class="external text" href="http://www.wrox.com/WileyCDA/WroxTitle/productCd-0764543857.html" rel="nofollow">Expert One-on-One J2EE Design and Developement</a> 에 선보인 코드를 기반으로 시작하여 점점 발전하게 되었다. <sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> 이 프레임워크는 2003년 6월에 최초로 <a href="/wiki/%EC%95%84%ED%8C%8C%EC%B9%98_%EB%9D%BC%EC%9D%B4%EC%84%A0%EC%8A%A4" title="아파치 라이선스">아파치 2.0 라이선스</a>로 공개되었으며 주요 버전 이력은 다음과 같다.</p>
<ul>
<li>1.0 : 2004년 3월</li>
<li>2.0 : 2006년 10월</li>
<li>2.5 : 2007년 11월</li>
<li>3.0 : 2009년 12월</li>
<li>3.1 : 2011년 12월</li>
<li>4.0 : 2013년 12월</li>
</ul>
<p>2006년에 1.2.6 버전으로 Jolt Productive Award<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> 와 Jax Innovation Award<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> 를 수상하였다.</p>
<h2><span class="mw-headline" id=".ED.8A.B9.EC.A7.95">특징</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%ED%94%84%EB%A7%81_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=2" title="부분 편집: 특징">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>스프링은 다른 프레임워크에 비해 다음과 같은 특징을 가진다.</p>
<ul>
<li>경량 컨테이너로서 자바 객체를 직접 관리한다. 각각의 객체 생성, 소멸과 같은 라이프 사이클을 관리하며 스프링으로부터 필요한 객체를 얻어올 수 있다.</li>
<li>스프링은 <a class="mw-redirect" href="/wiki/POJO" title="POJO">POJO</a>(Plain Old Java Object) 방식의 프레임워크이다. 일반적인 <a class="mw-redirect" href="/wiki/J2EE" title="J2EE">J2EE</a> 프레임워크에 비해 구현을 위해 특정한 <a class="mw-disambig" href="/wiki/%EC%9D%B8%ED%84%B0%ED%8E%98%EC%9D%B4%EC%8A%A4" title="인터페이스">인터페이스</a>를 구현하거나 상속을 받을 필요가 없어 기존에 존재하는 <a class="mw-disambig" href="/wiki/%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="라이브러리">라이브러리</a> 등을 지원하기에 용이하고 객체가 가볍다.</li>
<li>스프링은 <a href="/wiki/%EC%A0%9C%EC%96%B4_%EB%B0%98%EC%A0%84" title="제어 반전">제어 반전</a>(IoC : Inversion of Control)을 지원한다. 컨트롤의 제어권이 사용자가 아니라 프레임워크에 있어서 필요에 따라 스프링에서 사용자의 코드를 호출한다.</li>
<li>스프링은 <a href="/wiki/%EC%9D%98%EC%A1%B4%EC%84%B1_%EC%A3%BC%EC%9E%85" title="의존성 주입">의존성 주입</a>(DI : Dependency Injection)을 지원한다. 각각의 계층이나 서비스들 간에 의존성이 존재할 경우 프레임워크가 서로 연결시켜준다.</li>
<li>스프링은 <a href="/wiki/%EA%B4%80%EC%A0%90_%EC%A7%80%ED%96%A5_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="관점 지향 프로그래밍">관점 지향 프로그래밍</a>(AOP : Aspect-Oriented Programming)을 지원한다. 따라서 <a href="/wiki/%ED%8A%B8%EB%9E%9C%EC%9E%AD%EC%85%98" title="트랜잭션">트랜잭션</a>이나 로깅, 보안과 같이 여러 모듈에서 공통적으로 사용하는 기능의 경우 해당 기능을 분리하여 관리할 수 있다.</li>
<li>스프링은 영속성과 관련된 다양한 서비스를 지원한다. <a href="/wiki/IBatis" title="IBatis">iBatis</a>나 <a class="new" href="/w/index.php?title=Hibernate&amp;action=edit&amp;redlink=1" title="Hibernate (없는 문서)">Hibernate</a> 등 이미 완성도가 높은 데이터베이스 처리 라이브러리와 연결할 수 있는 인터페이스를 제공한다.</li>
<li>스프링은 확장성이 높다. 스프링 프레임워크에 통합하기 위해 간단하게 기존 라이브러리를 감싸는 정도로 스프링에서 사용이 가능하기 때문에 수많은 라이브러리가 이미 스프링에서 지원되고 있고 스프링에서 사용되는 라이브러리를 별도로 분리하기도 용이하다.</li>
</ul>
<h2><span class="mw-headline" id=".EC.A3.BC.EC.9A.94_.EB.AA.A8.EB.93.88">주요 모듈</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%ED%94%84%EB%A7%81_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=3" title="부분 편집: 주요 모듈">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>스프링에서 사용되는 주요 모듈은 다음과 같다.</p>
<h3><span class="mw-headline" id=".EC.A0.9C.EC.96.B4_.EB.B0.98.EC.A0.84_.EC.BB.A8.ED.85.8C.EC.9D.B4.EB.84.88">제어 반전 컨테이너</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%ED%94%84%EB%A7%81_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=4" title="부분 편집: 제어 반전 컨테이너">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/%EC%A0%9C%EC%96%B4_%EB%B0%98%EC%A0%84" title="제어 반전">제어 반전</a>(IoC: Inversion of Control) 컨테이너는 스프링의 가장 중요하고 핵심적인 기능으로서 자바의 <a href="/wiki/%EB%B0%98%EC%98%81_(%EC%BB%B4%ED%93%A8%ED%84%B0_%EA%B3%BC%ED%95%99)" title="반영 (컴퓨터 과학)">반영</a>(reflection)을 이용해서 객체의 생명주기를 관리하고 <a href="/wiki/%EC%9D%98%EC%A1%B4%EC%84%B1_%EC%A3%BC%EC%9E%85" title="의존성 주입">의존성 주입</a>(Dependency Injection)을 통해 각 계층이나 서비스들간의 의존성을 맞춰준다. 이러한 기능들은 주로 환경설정을 담당하는 XML 파일에 의해 설정되고 수행된다.</p>
<h3><span class="mw-headline" id=".EA.B4.80.EC.A0.90_.EC.A7.80.ED.96.A5_.ED.94.84.EB.A1.9C.EA.B7.B8.EB.9E.98.EB.B0.8D_.ED.94.84.EB.A0.88.EC.9E.84.EC.9B.8C.ED.81.AC">관점 지향 프로그래밍 프레임워크</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%ED%94%84%EB%A7%81_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=5" title="부분 편집: 관점 지향 프로그래밍 프레임워크">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>스프링은 로깅이나 보안, 트랜잭션 등 핵심적인 비즈니스 로직과 관련이 없으나 여러 곳에서 공통적으로 쓰이는 기능들을 분리하여 개발하고 실행 시에 서로 조합할 수 있는 <a href="/wiki/%EA%B4%80%EC%A0%90_%EC%A7%80%ED%96%A5_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D" title="관점 지향 프로그래밍">관점 지향 프로그래밍</a>(AOP)을 지원한다. 기존에 널리 사용되고 있는 강력한 관점 지향 프로그래밍 프레임워크인 <a class="new" href="/w/index.php?title=AspectJ&amp;action=edit&amp;redlink=1" title="AspectJ (없는 문서)">AspectJ</a>도 내부적으로 사용할 수 있으며, 스프링 자체적으로 지원하는 실행시(Runtime)에 조합하는 방식도 지원한다.</p>
<h3><span class="mw-headline" id=".EB.8D.B0.EC.9D.B4.ED.84.B0_.EC.95.A1.EC.84.B8.EC.8A.A4_.ED.94.84.EB.A0.88.EC.9E.84.EC.9B.8C.ED.81.AC">데이터 액세스 프레임워크</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%ED%94%84%EB%A7%81_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=6" title="부분 편집: 데이터 액세스 프레임워크">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>스프링은 <a href="/wiki/%EB%8D%B0%EC%9D%B4%ED%84%B0%EB%B2%A0%EC%9D%B4%EC%8A%A4" title="데이터베이스">데이터베이스</a>에 접속하고 자료를 저장 및 읽어오기 위한 여러 가지 유명한 라이브러리, 즉 <a href="/wiki/JDBC" title="JDBC">JDBC</a>, <a href="/wiki/IBatis" title="IBatis">iBatis</a>(<a class="new" href="/w/index.php?title=MyBatis&amp;action=edit&amp;redlink=1" title="MyBatis (없는 문서)">MyBatis</a>), <a class="new" href="/w/index.php?title=Hibernate&amp;action=edit&amp;redlink=1" title="Hibernate (없는 문서)">Hibernate</a> 등에 대한 지원 기능을 제공하여 데이터베이스 프로그래밍을 쉽게 사용할 수 있다.</p>
<h3><span class="mw-headline" id=".ED.8A.B8.EB.9E.9C.EC.9E.AD.EC.85.98_.EA.B4.80.EB.A6.AC_.ED.94.84.EB.A0.88.EC.9E.84.EC.9B.8C.ED.81.AC">트랜잭션 관리 프레임워크</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%ED%94%84%EB%A7%81_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=7" title="부분 편집: 트랜잭션 관리 프레임워크">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>스프링은 추상화된 <a href="/wiki/%ED%8A%B8%EB%9E%9C%EC%9E%AD%EC%85%98" title="트랜잭션">트랜잭션</a> 관리를 지원하며 XML 설정파일 등을 이용한 선언적인 방식 및 프로그래밍을 통한 방식을 모두 지원한다.</p>
<h3><span class="mw-headline" id=".EB.AA.A8.EB.8D.B8-.EB.B7.B0-.EC.BB.A8.ED.8A.B8.EB.A1.A4.EB.9F.AC_.ED.8C.A8.ED.84.B4">모델-뷰-컨트롤러 패턴</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%ED%94%84%EB%A7%81_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=8" title="부분 편집: 모델-뷰-컨트롤러 패턴">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>스프링은 웹 프로그램밍 개발 시 거의 표준적인 방식인 <b>Spring MVC</b>라 불리는 <a href="/wiki/%EB%AA%A8%EB%8D%B8-%EB%B7%B0-%EC%BB%A8%ED%8A%B8%EB%A1%A4%EB%9F%AC" title="모델-뷰-컨트롤러">모델-뷰-컨트롤러</a>(MVC) 패턴을 사용한다. DispatcherServlet이 Contoller 역할을 담당하여 각종 요청을 적절한 서비스에 분산시켜주며 이를 각 서비스들이 처리를 하여 결과를 생성하고 그 결과는 다양한 형식의 View 서비스들로 화면에 표시될 수 있다.</p>
<h3><span class="mw-headline" id=".EB.B0.B0.EC.B9.98_.ED.94.84.EB.A0.88.EC.9E.84.EC.9B.8C.ED.81.AC">배치 프레임워크</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8A%A4%ED%94%84%EB%A7%81_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC&amp;action=edit&amp;section=9" title="부분 편집: 배치 프레임워크">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>스프링은 특정 시간대에 실행하거나 대용량의 자료를 처리하는데 쓰이는 <a href="/wiki/%EC%9D%BC%EA%B4%84_%EC%B2%98%EB%A6%AC" title="일괄 처리">일괄 처리</a>(Batch Processing)을 지원하는 배치 프레임워크를 제공한다. 기본적으로 스프링 배치는 Quartz 기반으로 동작한다.</p>
