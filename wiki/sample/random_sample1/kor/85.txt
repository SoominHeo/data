<p><b>종속형 시트</b> 또는 <b>캐스케이딩 스타일 시트</b>(<span lang="en" xml:lang="en">Cascading Style Sheets</span>, CSS)는 <a href="/wiki/%EB%A7%88%ED%81%AC%EC%97%85_%EC%96%B8%EC%96%B4" title="마크업 언어">마크업 언어</a>가 실제 표시되는 방법을 기술하는 언어로, <a href="/wiki/HTML" title="HTML">HTML</a>과 <a href="/wiki/XHTML" title="XHTML">XHTML</a>에 주로 쓰이며, <a href="/wiki/XML" title="XML">XML</a>에서도 사용할 수 있다. <a href="/wiki/W3C" title="W3C">W3C</a>의 표준이며, 레이아웃과 스타일을 정의할 때의 자유도가 높다.</p>
<p><a href="/wiki/%EB%A7%88%ED%81%AC%EC%97%85_%EC%96%B8%EC%96%B4" title="마크업 언어">마크업 언어</a>가 웹사이트의 몸체를 담당한다면 <b>CSS</b>는 옷과 액세서리 같은 꾸미는 역할을 담당한다고 할 수 있다. 즉, HTML 구조는 그대로 두고 CSS 파일만 변경해도 전혀 다른 웹사이트처럼 꾸밀 수 있다. <a class="new" href="/w/index.php?title=CSS_Zen_Garden&amp;action=edit&amp;redlink=1" title="CSS Zen Garden (없는 문서)">CSS Zen Garden</a>은 하나의 HTML(필요에 따라 최소한의 수정이 있다.)을 CSS만으로 얼마나 다양하게 표현할 수 있는지 알 수 있는 웹사이트이다.</p>
<p>현재 개발 중인 <a class="mw-redirect" href="/wiki/CSS3" title="CSS3">CSS3</a>의 경우 그림자 효과, 그라데이션, 변형 등 그래픽 편집 프로그램으로 제작한 이미지를 대체할 수 있는 기능이 추가되었다. 또한 다양한 애니메이션 기능이 추가되어 <a href="/wiki/%EC%96%B4%EB%8F%84%EB%B9%84_%ED%94%8C%EB%9E%98%EC%8B%9C" title="어도비 플래시">어도비 플래시</a>를 어느 정도 대체하고 있다.</p>

<h2><span class="mw-headline" id=".EC.97.AD.EC.82.AC">역사</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%A2%85%EC%86%8D%ED%98%95_%EC%8B%9C%ED%8A%B8&amp;action=edit&amp;section=1" title="부분 편집: 역사">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>CSS는 1994년 10월 10일, <a class="new" href="/w/index.php?title=%ED%95%98%EC%BD%A4_%EB%B9%84%EC%9B%80_%EB%A6%AC&amp;action=edit&amp;redlink=1" title="하콤 비움 리 (없는 문서)">하콤 비움 리</a>(Håkon Wium Lie)가 처음 제안하였다.<sup class="reference" id="cite_ref-chss-proposal_1-0"><a href="#cite_note-chss-proposal-1">[1]</a></sup></p>
<h3><span class="mw-headline" id=".EB.B2.84.EC.A0.84.EC.9D.98_.EB.B3.80.ED.99.94">버전의 변화</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%A2%85%EC%86%8D%ED%98%95_%EC%8B%9C%ED%8A%B8&amp;action=edit&amp;section=2" title="부분 편집: 버전의 변화">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>CSS는 지속적으로 새로운 버전이 나오고 있다. 1996년에 도입된 <a class="external text" href="http://www.w3.org/TR/CSS1" rel="nofollow">CSS 1</a>은 CSS의 바탕이 되었다. CSS의 표준으로는 <a class="external text" href="http://www.w3.org/TR/CSS21/" rel="nofollow">CSS 2.1</a>이 있으며 이전 버전에 비하여 새로운 기능과 도구가 추가되었다. 대다수의 <a href="/wiki/%EC%9B%B9_%EB%B8%8C%EB%9D%BC%EC%9A%B0%EC%A0%80" title="웹 브라우저">웹 브라우저</a>는 CSS 2.1를 잘 지원한다.(<a href="/wiki/%EC%9D%B8%ED%84%B0%EB%84%B7_%EC%9D%B5%EC%8A%A4%ED%94%8C%EB%A1%9C%EB%9F%AC" title="인터넷 익스플로러">인터넷 익스플로러</a>의 경우, 버전 7이 되면서 CSS2.1을 지원한다.) 현재 W3C에서는 CSS3을 표준으로 만들고 있다.</p>
<p>CSS는 여러 수준과 프로파일을 가지고 있다. 각 수준의 CSS는 일반적으로 새로운 기능을 담고 있으며 CSS1, CSS2, CSS3, CSS4로 나뉜다. 프로파일들은 일반적으로 특정한 장치나 사용자 인터페이스를 위해 만들어진 하나 이상 수준의 CSS의 하부 집합이다. 현재 휴대용 장치, 프린터, 텔레비전 수상기를 위한 프로파일들이 있다.</p>
<h4><span class="mw-headline" id="CSS1">CSS1</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%A2%85%EC%86%8D%ED%98%95_%EC%8B%9C%ED%8A%B8&amp;action=edit&amp;section=3" title="부분 편집: CSS1">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>첫 CSS 규격은 공식 W3C 권고안이 되었으며 그 이름은 CSS1이다. 1996년 12월에 발표되었다.</p>
<h4><span class="mw-headline" id="CSS2">CSS2</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%A2%85%EC%86%8D%ED%98%95_%EC%8B%9C%ED%8A%B8&amp;action=edit&amp;section=4" title="부분 편집: CSS2">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>CSS2는 W3C가 개발하였으며 <a href="/wiki/1998%EB%85%84" title="1998년">1998년</a> 5월에 권고안으로 발표되었다.</p>
<h4><span class="mw-headline" id="CSS3">CSS3</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%A2%85%EC%86%8D%ED%98%95_%EC%8B%9C%ED%8A%B8&amp;action=edit&amp;section=5" title="부분 편집: CSS3">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>CSS3는 <a href="/wiki/2005%EB%85%84" title="2005년">2005년</a> <a href="/wiki/12%EC%9B%94_5%EC%9D%BC" title="12월 5일">12월 5일</a> 이후 개발 중에 있다.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup> W3C CSS3 로드맵은 요약과 도입부를 제공하고 있다.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> 전체가 모듈화되어 사용자 에이전트가 모듈에 대한 모든 모듈을 지원하지 않거나 자유롭게 선택할 수 있도록 하고 있으며, 다른 세로 글쓰기와 HTML 이외의 규격에까지 관여하는 내용으로 되어 있다. 현재 어떤 모듈도 권고안까지 이른 것은 없다.</p>
<h4><span class="mw-headline" id="CSS4">CSS4</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%A2%85%EC%86%8D%ED%98%95_%EC%8B%9C%ED%8A%B8&amp;action=edit&amp;section=6" title="부분 편집: CSS4">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>하나의 통합된 CSS4 사양은 없는데,<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> 이는 여러 개의 모듈로 나뉘어 있는데 기인하다. 그러나 "레벨 4" 모듈들이 존재한다.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<h2><span class="mw-headline" id=".EB.AC.B8.EB.B2.95">문법</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%A2%85%EC%86%8D%ED%98%95_%EC%8B%9C%ED%8A%B8&amp;action=edit&amp;section=7" title="부분 편집: 문법">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>CSS는 단순한 <a href="/wiki/%EB%AC%B8%EB%B2%95" title="문법">문법</a>을 가지며, 수많은 영어 키워드를 사용하여 다양한 스타일의 프로퍼티의 이름을 규정한다.</p>
<p>스타일 시트는 규칙의 목록으로 구성된다. 각 규칙이나 규칙 집합은 하나 이상의 셀렉터와 하나의 선언 블록을 이룬다.</p>
<h3><span class="mw-headline" id=".EC.85.80.EB.A0.89.ED.84.B0">셀렉터</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%A2%85%EC%86%8D%ED%98%95_%EC%8B%9C%ED%8A%B8&amp;action=edit&amp;section=8" title="부분 편집: 셀렉터">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>CSS에서 셀렉터(selector)는 마크업 자체에 태그와 속성을 일치시킴으로써 어느 부분의 마크업에 스타일을 적용할지 선언한다.</p>
<p>클래스와 ID들은 대소문자를 구분하며 문자로 시작해야 하며 영숫자와 언더바(_)를 포함할 수 있다. 클래스는 어떠한 요소의 어떠한 수의 인스턴스에도 적용할 수 있다. ID는 하나의 요소에만 적용할 수 있다.</p>
<h3><span class="mw-headline" id=".EC.84.A0.EC.96.B8_.EB.B8.94.EB.A1.9D">선언 블록</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%A2%85%EC%86%8D%ED%98%95_%EC%8B%9C%ED%8A%B8&amp;action=edit&amp;section=9" title="부분 편집: 선언 블록">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>선언 블록은 괄호로 이루어진 선언들의 목록으로 이루어진다. 각 선언은 그 자체가 프로퍼티, 콜론( : ), 값으로 구성된다. 한 블록 안에 여러 선언이 있으면, 세미콜론 ( ; )이 개개의 각 선언에 삽입되어야 한다.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<h3><span class="mw-headline" id=".EC.9B.90.EC.B2.9C">원천</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%A2%85%EC%86%8D%ED%98%95_%EC%8B%9C%ED%8A%B8&amp;action=edit&amp;section=10" title="부분 편집: 원천">원본 편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>CSS 정보는 다양한 원천으로부터 제공받을 수 있다. 이러한 원천으로는 웹 브라우저, 사용자, 제작자가 될 수 있다. 제작자로부터의 정보는 더 나아가 인라인, 미디어 타입, 중요성, 셀렉터 특정, 규칙 순서, 상속, 프로퍼티 정의로 분류될 수 있다. CSS 스타일의 정보는 별도의 문서에 있을 수 있고, HTML 문서에 직접 추가할 수도 있다. 사용되는 출력 장치에 따라 다른 스타일을 적용할 수 있다. 이를테면 화면 버전은 인쇄 버전과 매우 다르므로 제작자는 각 매체에 적절한 표현 방식을 사용할 수 있다.</p>
