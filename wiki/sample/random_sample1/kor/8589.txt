<h1 class="firstHeading" id="firstHeading" lang="ko">실행 파일 압축</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">위키백과, 우리 모두의 백과사전.</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					이동:					<a href="#mw-head">둘러보기</a>, 					<a href="#p-search">검색</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ko"><script>function mfTempOpenSection(id){var block=document.getElementById("mf-section-"+id);block.className+=" open-block";block.previousSibling.className+=" open-block";}</script><p><b>실행 파일 압축</b>은 말 그대로 <a href="/wiki/%EC%8B%A4%ED%96%89_%ED%8C%8C%EC%9D%BC" title="실행 파일">실행 파일</a>을 <a href="/wiki/%EB%8D%B0%EC%9D%B4%ED%84%B0_%EC%95%95%EC%B6%95" title="데이터 압축">압축</a>하는 것을 말한다. 수많은 실행 파일 압축 프로그램이 존재하며, 이러한 프로그램들은 <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a> 시절부터 존재해왔다. 줄여서 <b>실행 압축</b>이라고도 부른다.</p>
<h2><span class="mw-headline" id=".EC.9E.A5.EB.8B.A8.EC.A0.90">장단점</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%8B%A4%ED%96%89_%ED%8C%8C%EC%9D%BC_%EC%95%95%EC%B6%95&amp;action=edit&amp;section=1" title="부분 편집: 장단점">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4" title="소프트웨어">소프트웨어</a> 개발자들이 실행 파일 압축을 사용하는 까닭은 많은데, 주로 소프트웨어의 <a href="/wiki/%EA%B8%B0%EC%96%B5_%EC%9E%A5%EC%B9%98" title="기억 장치">기억 장치</a>의 용량을 줄이는 것이 목적이다. 실행 압축 프로그램의 목적은 실행 코드를 압축하는 것이며 <a href="/wiki/Gzip" title="Gzip">gzip</a>, <a href="/wiki/ZIP_(%ED%8C%8C%EC%9D%BC_%ED%8F%AC%EB%A7%B7)" title="ZIP (파일 포맷)">zip</a>, <a href="/wiki/Bzip2" title="Bzip2">Bzip2</a>와 같은 표준 <a href="/wiki/%EB%8D%B0%EC%9D%B4%ED%84%B0_%EC%95%95%EC%B6%95" title="데이터 압축">데이터 압축</a> 기능보다 더 나은 <a class="new" href="/w/index.php?title=%EC%95%95%EC%B6%95%EB%A5%A0&amp;action=edit&amp;redlink=1" title="압축률 (없는 문서)">압축률</a>을 보이고 있다.<sup class="정리_필요 noprint">[<a href="/wiki/%EC%9C%84%ED%82%A4%EB%B0%B1%EA%B3%BC:%EC%B6%9C%EC%B2%98_%EB%B0%9D%ED%9E%88%EA%B8%B0" title="위키백과:출처 밝히기">출처 필요</a>]</sup> <a href="/wiki/CD-ROM" title="CD-ROM">CD-ROM</a>, <a class="mw-redirect" href="/wiki/DVD-ROM" title="DVD-ROM">DVD-ROM</a>, <a href="/wiki/%ED%94%8C%EB%A1%9C%ED%94%BC_%EB%94%94%EC%8A%A4%ED%81%AC" title="플로피 디스크">플로피 디스크</a>와 같은 용량이 제한된 매체에 쓰이며, <a href="/wiki/%EC%9D%B8%ED%84%B0%EB%84%B7" title="인터넷">인터넷</a>의 경우 다운로드 시간을 줄이기 위해서 사용되기도 한다.</p>
<p><a href="/wiki/PE_%ED%8F%AC%EB%A7%B7" title="PE 포맷">PE 포맷</a>으로 되어 있는 실행 파일을 압축할 때의 한 가지 장점으로는 <a href="/wiki/%EC%BB%B4%ED%93%A8%ED%84%B0_%EB%B3%B4%EC%95%88" title="컴퓨터 보안">보안</a>을 들 수 있다. 어떠한 프로그램 안의 데이터를 분석하는 프로그램을 사용하여 프로그래머가 짠 내용을 분석하지 못하게 막을 수 있다. <a class="mw-redirect" href="/wiki/%ED%95%9C%EA%B8%80%ED%99%94_%ED%8C%A8%EC%B9%98" title="한글화 패치">한글 패치</a>를 만들 때에도 이러한 프로그램이 이용되기도 한다.</p>
<p>압축된 실행 파일은 파일 시스템의 공간을 덜 차지하기 때문에 파일 시스템으로부터 데이터가 메모리에 전송되는 시간이 덜 걸린다. 한편, 실행 직후 데이터의 압축을 푸는 시간이 좀 걸릴 수 있다. 다만, 다양한 저장 매체의 속도는 평균적인 프로세서 속도를 따라잡지 못했기 때문에 저장 장치는 자주 병목 현상을 겪는다. 그러므로 압축된 실행 파일은 대부분의 시스템에서 더 빨리 실행될 것이다. 일부 <a href="/wiki/%EB%8C%80%ED%95%9C%EB%AF%BC%EA%B5%AD" title="대한민국">대한민국</a> 웹에서는 압축된 실행 파일을 실행할 때 비록 시스템 메모리를 더 잡아먹지만 속도가 빨라진 것을 바로 느낄 수 있다는 의견이 있던 반면, 그렇지 못하는 경우도 많다는 의견도 있었다.<sup class="정리_필요 noprint">[<a href="/wiki/%EC%9C%84%ED%82%A4%EB%B0%B1%EA%B3%BC:%EC%B6%9C%EC%B2%98_%EB%B0%9D%ED%9E%88%EA%B8%B0" title="위키백과:출처 밝히기">출처 필요</a>]</sup> 현대의 데스크톱 컴퓨터에서 실행 파일이 비정상적으로 크지 않다면 눈치채기 쉽지 않기 때문에 실행 속도를 빠르게 하는 것이 실행 파일을 압축하는 주된 이유라고 볼 수는 없다.</p>
<p>실행 파일마다 압축 프로그램의 특성을 타는 경우가 있다. 이를테면, <a href="/wiki/UPX" title="UPX">UPX</a>로 압축한 실행 파일은 정상 동작하는 반면, <a class="new" href="/w/index.php?title=ASPack&amp;action=edit&amp;redlink=1" title="ASPack (없는 문서)">ASPack</a>로 압축한 실행 파일은 정상 동작하지 않는 경우가 있고 그 반대의 경우도 있다. 또 이러한 실행 파일 압축 프로그램은 <a class="mw-disambig" href="/wiki/%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="라이브러리">라이브러리</a> 파일도 압축할 수 있으며 이러한 라이브러리 파일도 특성을 탈 수 있다. 따라서 실행 압축을 하기 앞서 <a href="/wiki/%EB%B0%B1%EC%97%85" title="백업">백업</a>을 하는 것이 권장되며, 압축을 한 뒤에 적절한 테스트가 요구된다.</p>
<p>디스크의 실행 파일 이미지를 읽는 운영 체제에서(<a href="/wiki/%EA%B0%80%EC%83%81_%EB%A9%94%EB%AA%A8%EB%A6%AC" title="가상 메모리">가상 메모리</a>를 참조하라), 압축된 실행 파일은 이러한 작업의 효율성을 떨어트릴 수 있다. 압축 해제 작업은 메모리 블록을 할당하여 압축되지 않은 데이터를 유지한다. 사용 여부에 관계 없이, 함께 실행되고 있는 다른 응용 프로그램의 메모리 리소스를 차지하려 하면서, 실행 파일이 로드되는 한 계속 할당된다. 운영 체제가 스왑 파일을 사용한다면(보통 메모리 부족을 겪지 않기 위해 사용한다), 압축되지 않은 데이터는 단순히 사용하지 않는 데이터 블록을 버린 뒤 실행 파일 이미지로부터 다시 로드하지 않고 메모리의 남은 공간을 넓힌다. 보통 눈에 잘 띄지 않지만, 실행 파일이 동시에 한 번 이상 실행될 때 문제가 될 수 있다.(운영 체제는 이미 로드된 데이터 블록을 다시 사용할 수 없고, 데이터는 새로운 메모리 블록으로 압축이 풀어져야 하며, 사용하지 않는 경우 독립적으로 스웨핑 처리가 되어야 한다.)</p>
<p>또다른 단점으로는 <a href="/wiki/%EC%A0%95%EC%A0%81_%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="정적 라이브러리">정적 라이브러리</a> 압축 해제 프로그램만 보이는 것처럼, 일부 유틸리티는 <a href="/wiki/%EB%9F%B0%ED%83%80%EC%9E%84_%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC" title="런타임 라이브러리">런타임 라이브러리</a> 독립을 더 이상 확인하지 않는다는 것이다.</p>
<p>또한, <a class="mw-redirect" href="/wiki/%EB%B0%94%EC%9D%B4%EB%9F%AC%EC%8A%A4_%EA%B2%80%EC%82%AC_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%A8" title="바이러스 검사 프로그램">바이러스 검사 프로그램</a>은 압축된 모든 실행 파일을 <a href="/wiki/%EB%B0%94%EC%9D%B4%EB%9F%AC%EC%8A%A4" title="바이러스">바이러스</a>로 판단하는 경우가 있는데 압축 해제 부분이 바이러스에 대한 일부 문자열을 공유하기 때문이다. 요즘 나오는 대부분의 바이러스 검사 프로그램은 몇 가지 실행 압축 계층을 분석하여 내부에 있는 실제의 실행 파일을 검사할 수 있다.</p>
<p>실행 압축은 컴퓨터가 <a href="/wiki/%ED%94%8C%EB%A1%9C%ED%94%BC_%EB%94%94%EC%8A%A4%ED%81%AC" title="플로피 디스크">플로피 디스크</a>, 용량이 적은 <a class="mw-redirect" href="/wiki/%ED%95%98%EB%93%9C_%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B8%8C" title="하드 드라이브">하드 드라이브</a>가 쓰였을 시절에 많이 쓰였다. 컴퓨터가 같은 크기의 공간에 더 많은 소프트웨어를 담을 수 있게 해 주었고 압축 파일의 해제 과정이 필요 없다는 것이 장점이었다. 그러나 실행 압축은 컴퓨터의 기억 공간이 커짐에 따라 잘 쓰이지 않게 되었다.</p>
<p>실행 압축은 <a class="new" href="/w/index.php?title=%EB%8D%B0%EB%AA%A8%EC%94%AC&amp;action=edit&amp;redlink=1" title="데모씬 (없는 문서)">데모씬</a>과 같이 엄청나게 용량이 적은 <a class="new" href="/w/index.php?title=%EB%8D%B0%EB%AA%A8_(%EC%BB%B4%ED%93%A8%ED%84%B0_%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D)&amp;action=edit&amp;redlink=1" title="데모 (컴퓨터 프로그래밍) (없는 문서)">데모</a> 프로그램을 설계하는 데 이용되기도 한다. 크리클럭(<a class="external autonumber" href="http://www.crinkler.net/" rel="nofollow">[1]</a>)가 그 예로, 용량이 4096 바이트밖에 되지 않는다.</p>
