<p><b>아파치 하둡</b>(Apache Hadoop, High-Availability Distributed Object-Oriented Platform)은 대량의 자료를 처리할 수 있는 큰 컴퓨터 클러스터에서 동작하는 분산 응용 프로그램을 지원하는 <a href="/wiki/%ED%94%84%EB%A6%AC%EC%9B%A8%EC%96%B4" title="프리웨어">프리웨어</a> <a href="/wiki/%EC%9E%90%EB%B0%94_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="자바 (프로그래밍 언어)">자바</a> <a href="/wiki/%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4_%ED%94%84%EB%A0%88%EC%9E%84%EC%9B%8C%ED%81%AC" title="소프트웨어 프레임워크">소프트웨어 프레임워크</a>이다. 원래 <a href="/wiki/%EB%84%88%EC%B9%98" title="너치">너치</a>의 <a href="/wiki/%EB%B6%84%EC%82%B0_%ED%8C%8C%EC%9D%BC_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="분산 파일 시스템">분산 처리</a>를 지원하기 위해 개발된 것으로, <a class="mw-redirect" href="/wiki/%EC%95%84%ED%8C%8C%EC%B9%98_%EC%9B%B9_%EC%84%9C%EB%B2%84" title="아파치 웹 서버">아파치</a> <a href="/wiki/%EB%A3%A8%EC%94%AC" title="루씬">루씬</a>의 하부 프로젝트이다<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>. 분산처리 시스템인 <a href="/wiki/%EA%B5%AC%EA%B8%80_%ED%8C%8C%EC%9D%BC_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="구글 파일 시스템">구글 파일 시스템</a>을 대체할 수 있는 하둡 분산 파일 시스템(HDFS: Hadoop Distributed File System)과 <a href="/wiki/%EB%A7%B5%EB%A6%AC%EB%93%80%EC%8A%A4" title="맵리듀스">맵리듀스</a>를 구현한 것이다.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup>.</p>

<h2><span class="mw-headline" id=".EC.97.AD.EC.82.AC">역사</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%95%84%ED%8C%8C%EC%B9%98_%ED%95%98%EB%91%A1&amp;action=edit&amp;section=1" title="부분 편집: 역사">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>하둡은 2006년 <a class="new" href="/w/index.php?title=%EB%8D%94%EA%B7%B8_%EC%BB%A4%ED%8C%85&amp;action=edit&amp;redlink=1" title="더그 커팅 (없는 문서)">더그 커팅</a>과 <a class="new" href="/w/index.php?title=%EB%A7%88%EC%9D%B4%ED%81%AC_%EC%BA%90%ED%8D%BC%EB%A0%90%EB%9D%BC&amp;action=edit&amp;redlink=1" title="마이크 캐퍼렐라 (없는 문서)">마이크 캐퍼렐라</a>(Mike Cafarella)가 개발하였다.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> 당시 커팅은 야후에서 일하고 있었으며<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup>, 이후 하둡은 아파치(Apache) 재단으로 넘어가 공개 소프트웨어로 개발되고 있다. 하둡은 <a href="/wiki/%EA%B5%AC%EA%B8%80" title="구글">구글</a>의 분산 파일 시스템(GFS) 논문이 공개된 후, 그 구조에 대응하는 체계로 개발되었다. 하둡의 로고는 노랑색 아기 코끼리로 표시한다. 이는 하둡의 개발자인 더그 커팅이 자신의 아이가 가지고 놀던 장난감 코끼리의 이름을 따서 하둡이라는 이름을 지었기 때문이다.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> 코끼리는 <a href="/wiki/%EB%B9%85_%EB%8D%B0%EC%9D%B4%ED%84%B0" title="빅 데이터">빅 데이터</a>(Big Data)를 상징하는 동물이다. 2011년 오픈 소스 아파치 하둡(Apache Hadoop) 사업을 위해 야후에서 분사한 기업의 이름을 지을 때도, 코끼리가 주인공인 동화에 나오는 코끼리 이름인 호튼(Horton)을 따와서 <a class="new" href="/w/index.php?title=%ED%98%B8%ED%8A%BC%EC%9B%8D%EC%8A%A4&amp;action=edit&amp;redlink=1" title="호튼웍스 (없는 문서)">호튼웍스</a>(Hortonworks)라고 회사명을 지었다.</p>
<h2><span class="mw-headline" id=".EC.95.84.ED.82.A4.ED.85.8D.EC.B2.98">아키텍처</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%95%84%ED%8C%8C%EC%B9%98_%ED%95%98%EB%91%A1&amp;action=edit&amp;section=2" title="부분 편집: 아키텍처">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>하둡은 <i>하둡 공통 패키지</i>로 구성되어 있다. 이 패키지에는 하둡 파일 시스템(HDFS), OS 수준 앱스트랙션(OS level abstractions) 그리고 맵리듀스(MapReduce) 엔진이 포함되어 있다.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> 또한 필요한 자바 아카이브 파일(Java ARchive, JAR)들과 하둡을 시작할 스크립트, 소스 코드들과 관련 자료들로 구성되어 있다.</p>
<p>작은 하둡 클러스터에는 하나의 마스터와 여러 워커 노드들로 구성 되어 있다. 마스터 노드들은 잡트렉커(JobTracker), 테스크트렉커(TaskTracker), 네임노드(NameNode), 데이터노드(DataNode)로 구성 된다. 슬레이브 또는 워커 노드(Worker Node)는 데이터노드와 테스크트렉커로서 동작을 한다. 하둡은 <a href="/wiki/%EC%9E%90%EB%B0%94_%EA%B0%80%EC%83%81_%EB%A8%B8%EC%8B%A0" title="자바 가상 머신">자바 런타임 환경</a>(Java Runtime Environment, JRE) 1.6 또는 그보다 높은 버전이 필요하다. 일반 시작 및 종료를 위한 스크립트는 클러스터의 노드들간 설정을 하기 위해 <a href="/wiki/%EC%8B%9C%ED%81%90%EC%96%B4_%EC%85%B8" title="시큐어 셸">시큐어 셸</a>(Secure Shell, ssh)이 필요하다.<sup class="reference" id="cite_ref-michael-noll.com_1_8-0"><a href="#cite_note-michael-noll.com_1-8">[8]</a></sup></p>
<p>대용량 하둡 클러스터에는 HDFS가 파일 시스템 인덱스를 관장하기 위한 네임노드 전담 서버를 통해 관리된다. 그리고 2차 네임노드는 네임노드의 메모리 구조 스냅샷을 만들어서 파일시스템에 장애나 데이터의 손실을 줄여준다. 단독 쟙트렉커 서버는 작업 스케줄링을 관리한다.</p>
<h2><span class="mw-headline" id=".ED.95.98.EB.91.A1_.EB.B6.84.EC.82.B0_.ED.8C.8C.EC.9D.BC_.EC.8B.9C.EC.8A.A4.ED.85.9C">하둡 분산 파일 시스템</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%95%84%ED%8C%8C%EC%B9%98_%ED%95%98%EB%91%A1&amp;action=edit&amp;section=3" title="부분 편집: 하둡 분산 파일 시스템">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p><b>하둡 분산 파일 시스템</b>(HDFS, Hadoop distributed file system)은 하둡 프레임워크를 위해 <a href="/wiki/%EC%9E%90%EB%B0%94_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="자바 (프로그래밍 언어)">자바</a> 언어로 작성된 분산 확장 <a href="/wiki/%ED%8C%8C%EC%9D%BC_%EC%8B%9C%EC%8A%A4%ED%85%9C" title="파일 시스템">파일 시스템</a>이다. HDFS은 여러 기계에 대용량 파일들을 나눠서 저장을 한다. 데이터들을 여러 서버에 중복해서 저장을 함으로써 데이터 안정성을 얻는다. 따라서 호스트에 <a href="/wiki/RAID" title="RAID">RAID</a> 저장장치를 사용하지 않아도 된다.</p>
<p>하둡 분산 파일 시스템은 다음과 같은 시스템에서 잘 동작하는 것을 목표로 하고 있다<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup>.</p>
<ul>
<li><b>하드웨어 오동작</b>:하드웨어 수가 많아지면 그중에 일부 하드웨어가 오동작하는 것은 예외 상황이 아니라 항상 발생하는 일이다. 따라서 이런 상황에서 빨리 자동으로 복구하는 것은 HDFS의 중요한 목표다.</li>
<li><b>스트리밍 자료 접근</b>: 범용 파일 시스템과 달리 반응 속도보다는 시간당 처리량에 최적화되어 있다.</li>
<li><b>큰 자료 집합</b>: 한 파일이 기가바이트나 테라바이트 정도의 크기를 갖는 것을 목적으로 설계되었다. 자료 대역폭 총량이 높고, 하나의 클러스터에 수 백개의 노드를 둘 수 있다. 하나의 인스턴스에서 수천만여 파일을 지원한다.</li>
<li><b>간단한 결합 모델</b>: 한번 쓰고 여러번 읽는 모델에 적합한 구조이다. 파일이 한번 작성되고 닫히면 바뀔 필요가 없는 경우를 위한 것이다. 이렇게 함으로써 처리량을 극대화할 수 있다.</li>
<li><b>자료를 옮기는 것보다 계산 작업을 옮기는 것이 비용이 적게 든다</b>: 자료를 많이 옮기면 대역폭이 많이 들기 때문에 네트워크 혼잡으로 인하여 전체 처리량이 감소한다. 가까운 곳에 있는 자료를 처리하게 계산 작업을 옮기면 전체적인 처리량이 더 높아진다.</li>
<li><b>다른 종류의 하드웨어와 소프트웨어 플랫폼과의 호환성:</b> 서로 다른 하드웨어와 소프트웨어 플랫폼들을 묶어 놓아도 잘 동작한다.</li>
</ul>
<h3><span class="mw-headline" id=".EB.84.A4.EC.9E.84.EB.85.B8.EB.93.9C.EC.99.80_.EB.8D.B0.EC.9D.B4.ED.84.B0.EB.85.B8.EB.93.9C">네임노드와 데이터노드</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%95%84%ED%8C%8C%EC%B9%98_%ED%95%98%EB%91%A1&amp;action=edit&amp;section=4" title="부분 편집: 네임노드와 데이터노드">편집</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>HDFS는 마스터/슬레이브(master/slave) 구조를 가진다. HDFS 클러스터는 하나의 네임노드와, 파일 시스템을 관리하고 클라이언트의 접근을 통제하는 마스터 서버로 구성된다. 게다가 클러스터의 각 노드에는 데이터노드가 하나씩 존재하고, 이 데이터 노드는 실행될 때마다 노드에 추가되는 스토리지를 관리한다. HDFS는 네임스페이스를 공개하여서 유저 데이터가 파일에 저장되는 것을 허락한다. 내부적으로 하나의 파일은 하나 이상의 블록으로 나뉘어 있고, 이 블록들은 데이터노드들에 저장되어 있다. 네임노드는 파일과 디렉터리의 읽기(open), 닫기(close), 이름 바꾸기(rename) 등, 파일시스템의 네임스페이스의 여러 기능을 수행한다. 또한, 데이터 노드와 블록들의 맵핑을 결정한다. 데이터 노드는 파일시스템의 클라이언트가 요구하는 읽기(read), 쓰기(write) 기능들을 담당한다. 또한 데이터 노드는 네임노드에서의 생성, 삭제, 복제 등과 같은 기능도 수행한다.</p>
<p>네임노드와 데이터노드는 GNU/Linux OS를 기반으로 하는 상용머신에서 실행하기 위해 디자인된 소프트웨어의 일부이다. HDFS는 자바 언어를 사용하므로 자바가 동작하는 어떠한 컴퓨터에서나 네임노드나 데이터노드 소프트웨어를 실행할 수 있다.</p>
<h2><span class="mw-headline" id=".EC.9D.B4.EC.9A.A9_.EC.82.AC.EB.A1.80">이용 사례</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%95%84%ED%8C%8C%EC%B9%98_%ED%95%98%EB%91%A1&amp;action=edit&amp;section=5" title="부분 편집: 이용 사례">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>하둡은 다양한 기업에서 사용되고 있다. <a class="mw-redirect" href="/wiki/%EC%95%BC%ED%9B%84" title="야후">야후</a>, <a href="/wiki/%ED%8E%98%EC%9D%B4%EC%8A%A4%EB%B6%81" title="페이스북">페이스북</a> 등이 하둡을 서비스의 여러 부분에 많이 사용하고 있는 것으로 알려져있다.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> 또한 데이터 분석 기업들이 하둡을 기반으로 플랫폼을 구축하고 있다.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></p>
