<h2><span class="mw-headline" id=".EC.97.AD.EC.82.AC.EC.99.80_.EA.B8.B0.EB.8A.A5">역사와 기능</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%95%A0%ED%94%8C%EC%86%8C%ED%94%84%ED%8A%B8_%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;section=1" title="부분 편집: 역사와 기능">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>애플은 48KB 램의 애플 2 컴퓨터에 적재할 새로운 베이직을 찾고 있었고 <a class="new" href="/w/index.php?title=%EC%95%8C%ED%85%8C%EC%96%B4_%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;redlink=1" title="알테어 베이직 (없는 문서)">알테어 베이직</a>으로 성공한 마이크로소프트를 선택했다. 애플은 10KB <a href="/wiki/%EC%96%B4%EC%85%88%EB%B8%94%EB%A6%AC%EC%96%B4" title="어셈블리어">어셈블리어</a> 베이직을 "애플소프트"란 이름으로 계약하였다. <a href="/wiki/%EC%96%B4%EC%85%88%EB%B8%94%EB%A6%AC%EC%96%B4" title="어셈블리어">어셈블리어</a> 코드를 공유하였기 때문에 <a class="new" href="/w/index.php?title=%EC%BD%94%EB%AA%A8%EB%8F%84%EC%96%B4_%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;redlink=1" title="코모도어 베이직 (없는 문서)">코모도어 베이직</a>과 같은 다른 <a href="/wiki/MOS_6502" title="MOS 6502">6502</a> 기반 컴퓨터의 베이직 구현과 매우 흡사했다. 행 번호가 있었고 각 행은 <a href="/wiki/%EA%B3%B5%EB%B0%B1_%EA%B8%B0%ED%98%B8" title="공백 기호">공백 기호</a>가 필요하지 않았으며 반복문은 매우 느렸다. 그럼에도 애플소프트 베이직은 이전의 열악한 정수 베이직보다 더 많은 기능을 제공하였다.</p>
<ul>
<li>원자적인 문자열. 문자열은 더 이상 C언어와 같이 문자형의 배열이 아니었다. 대신에 <a href="/wiki/%EC%9E%90%EB%B0%94_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="자바 (프로그래밍 언어)">자바</a>나 <a href="/wiki/%EC%8A%A4%ED%82%B4_(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D_%EC%96%B8%EC%96%B4)" title="스킴 (프로그래밍 언어)">스킴</a>과 같이 <a class="new" href="/w/index.php?title=%EB%A9%94%EB%AA%A8%EB%A6%AC_%EC%88%98%EA%B1%B0&amp;action=edit&amp;redlink=1" title="메모리 수거 (없는 문서)">메모리 수거</a> 객체를 사용했다. 이것은 문자열 배열의 사용이 가능하게 했다. DIM A$(10)은 문자열 배열의 10개짜리 <a href="/wiki/%EB%8F%99%EC%A0%81_%EB%B0%B0%EC%97%B4" title="동적 배열">동적 배열</a> 자료 구조, 즉 벡터가 되었다.</li>
<li>다차원 배열</li>
<li>8비트 지수와 31비트 유효수를 가지는 단정도 <a href="/wiki/%EB%B6%80%EB%8F%99%EC%86%8C%EC%88%98%EC%A0%90" title="부동소수점">부동소수점</a> 변수. 삼각법의 사용이 가능했다.</li>
<li>고해상도 영상</li>
<li>수치형과 문자열의 변환을 위해 CHR$, ASC, STR$, VAL 함수 제공.</li>
<li>LET 문이 부차적으로 가능했다.</li>
<li>사용자 지정 함수: DEF FN문을 이용하고, 파라메터는 한 개만 제공.</li>
<li>PEEK와 POKE문을 이용해서 메모리 직접 제어 가능.</li>
<li>명령어는 대문자만 사용이 가능하며 (문자열에는 소문자도 사용 가능), 행 번호를 다시 정렬해주는 RENUM 기능은 없다.</li>
</ul>
<h2><span class="mw-headline" id=".EA.B2.8C.EC.9E.84">게임</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%EC%95%A0%ED%94%8C%EC%86%8C%ED%94%84%ED%8A%B8_%EB%B2%A0%EC%9D%B4%EC%A7%81&amp;action=edit&amp;section=2" title="부분 편집: 게임">편집</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>그러나 몇 가지 이유로 애플소프트 베이직으로 만들어진 <a href="/wiki/%EC%95%A1%EC%85%98_%EA%B2%8C%EC%9E%84" title="액션 게임">액션 게임</a>은 거의 없었다.</p>
<ul>
<li>당시에는 제한된 메모리와 <a href="/wiki/%EC%A4%91%EC%95%99_%EC%B2%98%EB%A6%AC_%EC%9E%A5%EC%B9%98" title="중앙 처리 장치">CPU</a> 클록을 조심스럽게 사용하던 시대였기 때문에, 게임같은 속도에 민감한 프로그램을 실시간 <a href="/wiki/%EC%9D%B8%ED%84%B0%ED%94%84%EB%A6%AC%ED%84%B0" title="인터프리터">인터프리터</a>로 작성하는 것을 비효율적인 일로 여겼다.</li>
<li>애플소프트 베이직은 산술 연산을 위해 정수 대신 실수를 사용하였다. 이것이 프로그램 수행에 불필요한 오버헤드의 원인이 되어, 전체 프로그램의 효율을 떨어뜨렸다. 모든 정수형 수치는 산술 연산을 위해 실수형으로 변환되었고 계산 후에 정수형 수치로 변환되었다. 마이크로소프트는 이를 결코 <a class="mw-redirect" href="/wiki/%EC%B5%9C%EC%A0%81%ED%99%94_(%EC%A0%95%EB%B3%B4_%EA%B3%B5%ED%95%99)" title="최적화 (정보 공학)">최적화</a>하지 않았다.</li>
<li><a href="/wiki/%EB%B9%84%ED%8A%B8%EB%A7%B5" title="비트맵">비트맵</a> 대신에 <a class="new" href="/w/index.php?title=%EB%AA%A8%EC%96%91_%EA%B5%AC%EC%84%B1%ED%91%9C&amp;action=edit&amp;redlink=1" title="모양 구성표 (없는 문서)">모양 구성표</a>(<span style="font-size: smaller;"><a href="/wiki/%EC%98%81%EC%96%B4" title="영어">영어</a>:</span> <span lang="en" xml:lang="en">Shape Table</span>)만 제공되었다. 영상 하단에 4줄의 텍스트가 붙는 "하드웨어 분할 화면" 이외에 텍스트와 그래픽을 동시에 출력할 방법이 없었다. 128KB의 메모리를 가졌던 <a class="new" href="/w/index.php?title=%EC%95%A0%ED%94%8C_2e&amp;action=edit&amp;redlink=1" title="애플 2e (없는 문서)">애플 2e</a>와 <a class="new" href="/w/index.php?title=%EC%95%A0%ED%94%8C_2c&amp;action=edit&amp;redlink=1" title="애플 2c (없는 문서)">애플 2c</a>의 베이직 인터프리터도 여분의 메모리나 배정도 그래픽을 지원하지 않았고 <a class="new" href="/w/index.php?title=%EC%95%A0%ED%94%8C2GS&amp;action=edit&amp;redlink=1" title="애플2GS (없는 문서)">애플2GS</a>의 16 색상도 지원하지 않았다. (<a href="/wiki/%EB%B9%84%EA%B8%80_%EB%B8%8C%EB%A1%9C%EC%8A%A4" title="비글 브로스">비글 브로스</a>는 이 문제의 해결을 위해 기계어 코드를 제공했다.)</li>
<li>프로그램의 각 행은 <a href="/wiki/%EC%97%B0%EA%B2%B0_%EB%A6%AC%EC%8A%A4%ED%8A%B8" title="연결 리스트">연결 리스트</a>에 저장되어서 <a class="new" href="/w/index.php?title=Goto_(%EB%AA%85%EB%A0%B9%EC%96%B4)&amp;action=edit&amp;redlink=1" title="Goto (명령어) (없는 문서)">Goto</a>의 계산에 <a href="/wiki/%EA%B3%84%EC%82%B0_%EB%B3%B5%EC%9E%A1%EB%8F%84_%EC%9D%B4%EB%A1%A0" title="계산 복잡도 이론">O(n) (선형) 시간</a>이 걸렸다.</li>
<li>음향에 관한 기능을 제공하지 않았다.</li>
